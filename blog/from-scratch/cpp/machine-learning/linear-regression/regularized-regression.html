    Ridge, Lasso, and Elastic Net in C++

    July 2026 · 27 min read

    OLS and plain gradient descent (Part 1, Part 2) minimize squared error alone. When features are highly correlated, or there are more features than can be reliably estimated from the available rows, that objective has a family of near-equally-good solutions, and the one OLS happens to land on can have wildly large, unstable coefficients. Regularization adds a second term to the objective that penalizes large coefficients, trading a small amount of bias for a large reduction in variance. This post covers three flavors of that idea, Ridge (L2), Lasso (L1), and Elastic Net (both), derives the closed-form ridge solution and the coordinate-descent update that solves Lasso and Elastic Net exactly, and works through what each one actually does differently when features are correlated, using a dataset built to have exactly that problem.

    Three Objectives, One Extra Term

    All three variants keep the same squared-error term from Part 1 and add a penalty on the weights (never on the intercept, so the model is still free to match the target's overall level):

    $$J_{\text{ridge}}(\mathbf w) = \frac12\lVert \mathbf y - X\mathbf w\rVert_2^2 + \frac{\lambda}{2}\sum_{j=1}^p w_j^2$$

    $$J_{\text{lasso}}(\mathbf w) = \frac{1}{2n}\lVert \mathbf y - X\mathbf w\rVert_2^2 + \lambda\sum_{j=1}^p |w_j|$$

    $$J_{\text{elastic net}}(\mathbf w) = \frac{1}{2n}\lVert \mathbf y - X\mathbf w\rVert_2^2 + \lambda\left(\alpha\sum_{j=1}^p |w_j| + \frac{1-\alpha}{2}\sum_{j=1}^p w_j^2\right)$$

    with $\alpha$ (called l1_ratio in the code) interpolating between pure Lasso ($\alpha=1$) and pure Ridge ($\alpha=0$). $L2$ shrinks every coefficient smoothly toward zero and has a closed form because the penalty is differentiable everywhere. $L1$ can shrink a coefficient exactly to zero, which is what makes Lasso useful for feature selection, but $|w_j|$ is not differentiable at $w_j=0$, so there is no single matrix formula for the minimizer; the fix is coordinate descent, derived below.

    Two different scaling conventions, on purpose. $J_{\text{ridge}}$ above uses a raw sum of squared errors (no $1/n$), matching fit_ridge's implementation exactly: it builds $X^\top X$ and $X^\top \mathbf y$ as plain sums over rows and adds $\lambda$ directly to the diagonal. $J_{\text{lasso}}$ and $J_{\text{elastic net}}$ use the mean squared error ($1/n$ included), matching fit_elastic_net_centered's implementation and the common glmnet/scikit-learn convention. This means a lambda value tuned for fit_ridge is not on the same numeric scale as one tuned for the coordinate-descent functions, a real gotcha if the two are compared side by side without accounting for the implicit factor of $n$. This post keeps them separate throughout and never compares a raw ridge lambda to a lasso lambda directly.

    Deriving the Ridge Closed Form

    Differentiating $J_{\text{ridge}}$ with respect to $\mathbf w$ (writing $D$ for the diagonal matrix with a 0 in the intercept position and 1s elsewhere, so the intercept is excluded from the penalty) and setting the result to zero:

    $$\nabla J_{\text{ridge}} = -X^\top(\mathbf y - X\mathbf w) + \lambda D\mathbf w = 0 \quad\Longrightarrow\quad (X^\top X + \lambda D)\,\mathbf w = X^\top \mathbf y$$

    Adding $\lambda$ to the diagonal of $X^\top X$ (except at the intercept row) is the entire algorithm: the same partial-pivot solver from Part 1 handles the rest, and a strictly positive $\lambda$ makes the matrix invertible even when $X^\top X$ alone is singular, since adding a positive value to every relevant diagonal entry of a symmetric positive-semidefinite matrix strictly increases its smallest eigenvalue. This is why ridge regression is often reached for directly as a fix for the "singular or near-singular" error from Part 1's solver.

    The rest of this file is a single, independently compilable C++ program; DataError/validate_dataset is the same validation code as Part 1 and Part 2, repeated so this file stands alone.

    Validating the Input

    // Ridge, Lasso, and Elastic Net linear regression from scratch, standard
// library only: closed-form ridge, coordinate-descent lasso/elastic net,
// a warm-started regularization path, and k-fold cross-validation with the
// one-standard-error rule.
#include <bits/stdc++.h>
using namespace std;

using Vector = vector<double>;
using Matrix = vector<Vector>;

struct DataError : runtime_error {
    explicit DataError(const string& msg) : runtime_error(msg) {}
};

static void validate_dataset(const Matrix& X, const Vector& y) {
    if (X.empty()) throw DataError("X has zero rows");
    if (X.size() != y.size()) throw DataError("row count mismatch between X and y");
    size_t d = X[0].size();
    for (auto& row : X) {
        if (row.size() != d) throw DataError("ragged row in X");
        for (double v : row) if (!isfinite(v)) throw DataError("non-finite feature");
    }
    for (double v : y) if (!isfinite(v)) throw DataError("non-finite target");
}

    Standardizing Before Penalizing

    All three variants require standardized features for the penalty to be fair: an unstandardized feature with a large natural scale has a small coefficient even in the unpenalized OLS solution, so a penalty proportional to $w_j^2$ or $|w_j|$ would shrink it far less than a feature on a small scale, for reasons that have nothing to do with which feature actually matters. Standardizing first (fit on training data only, same rule as Part 2) makes every coefficient's starting scale comparable, so the same $\lambda$ penalizes every feature on equal footing:

    // ---------- standardization ----------

struct Scaler {
    Vector mean, stdev;
    void fit(const Matrix& X) {
        size_t n = X.size(), d = X[0].size();
        mean.assign(d, 0.0); stdev.assign(d, 0.0);
        for (auto& row : X) for (size_t j = 0; j < d; ++j) mean[j] += row[j];
        for (double& m : mean) m /= (double)n;
        for (auto& row : X) for (size_t j = 0; j < d; ++j) stdev[j] += (row[j] - mean[j]) * (row[j] - mean[j]);
        for (double& s : stdev) { s = sqrt(s / (double)n); if (s < 1e-12) s = 1.0; }
    }
    Matrix transform(const Matrix& X) const {
        Matrix out = X;
        for (auto& row : out) for (size_t j = 0; j < row.size(); ++j) row[j] = (row[j] - mean[j]) / stdev[j];
        return out;
    }
};

    The Linear Solver

    Reused unchanged from Part 1 for the ridge closed form:

    // ---------- linear solver (for ridge normal equations) ----------

static Vector solve_linear(Matrix A, Vector b) {
    int n = (int)b.size();
    double scale = 0.0;
    for (auto& row : A) for (double v : row) scale = max(scale, fabs(v));
    if (scale == 0.0) scale = 1.0;
    const double tol = 1e-10 * scale;
    for (int col = 0; col < n; ++col) {
        int piv = col;
        for (int r = col + 1; r < n; ++r) if (fabs(A[r][col]) > fabs(A[piv][col])) piv = r;
        if (fabs(A[piv][col]) < tol) throw DataError("solve_linear: singular system at column " + to_string(col));
        swap(A[col], A[piv]); swap(b[col], b[piv]);
        double div = A[col][col];
        for (int c = col; c < n; ++c) A[col][c] /= div;
        b[col] /= div;
        for (int r = 0; r < n; ++r) if (r != col) {
            double m = A[r][col];
            if (m == 0.0) continue;
            for (int c = col; c < n; ++c) A[r][c] -= m * A[col][c];
            b[r] -= m * b[col];
        }
    }
    return b;
}

    Ridge Regression

    // ---------- ridge regression (closed form, intercept unpenalized) ----------

static Vector fit_ridge(const Matrix& X, const Vector& y, double lambda) {
    validate_dataset(X, y);
    if (lambda < 0.0) throw DataError("fit_ridge: lambda must be non-negative");
    int n = (int)X.size(), d = (int)X[0].size() + 1;
    Matrix xtx(d, Vector(d, 0.0));
    Vector xty(d, 0.0);
    for (int i = 0; i < n; ++i) {
        Vector row(d, 1.0);
        for (int j = 1; j < d; ++j) row[j] = X[i][j - 1];
        for (int a = 0; a < d; ++a) {
            xty[a] += row[a] * y[i];
            for (int b = 0; b < d; ++b) xtx[a][b] += row[a] * row[b];
        }
    }
    for (int j = 1; j < d; ++j) xtx[j][j] += lambda; // intercept (index 0) is never penalized
    return solve_linear(xtx, xty);
}

    fit_ridge takes raw (unstandardized) $X$ deliberately in this implementation, since forming $X^\top X$ from raw features and adding $\lambda$ to the diagonal already produces a valid, well-defined penalized fit; standardizing first and converting back afterward (as the Lasso/Elastic Net path below does) is also correct and often preferable for comparability across features, but is not required for ridge to be well-posed the way it is for a fair L1 penalty.

    Coordinate Descent for Lasso and Elastic Net

    Coordinate descent optimizes one coefficient at a time, holding every other coefficient fixed, and cycles through all of them repeatedly until nothing changes. For a convex, separable-in-each-coordinate objective like Elastic Net's, this converges to the global optimum. Isolating $w_j$ in $J_{\text{elastic net}}$, with $\mathbf r^{(j)} = \mathbf y_c - \sum_{k \ne j} X_{s,k} w_k$ the partial residual with feature $j$'s contribution removed:

    $$\min_{w_j}\ \frac{1}{2n}\lVert \mathbf r^{(j)} - X_{s,j} w_j\rVert_2^2 + \lambda\alpha|w_j| + \frac{\lambda(1-\alpha)}{2}w_j^2$$

    Because every column of the standardized design matrix $X_s$ has sample variance 1, $\frac1n X_{s,j}^\top X_{s,j} = 1$, which cancels out of the stationarity condition and leaves a clean closed form for this one-dimensional subproblem. Writing $\rho_j = \frac1n X_{s,j}^\top \mathbf r^{(j)}$ (the correlation of feature $j$ with the residual left over from every other feature), the subgradient stationarity condition for the non-differentiable $|w_j|$ term gives the coordinate-wise update:

    $$w_j \leftarrow \frac{\operatorname{soft}(\rho_j,\ \lambda\alpha)}{1 + \lambda(1-\alpha)}, \qquad \operatorname{soft}(z, t) = \operatorname{sign}(z)\max(|z| - t, 0)$$

    The soft-threshold function shrinks $\rho_j$ toward zero by $\lambda\alpha$ and clips it to exactly zero once $|\rho_j| \le \lambda\alpha$, which is the mechanism that lets Lasso ($\alpha=1$) drop features entirely; the $1 + \lambda(1-\alpha)$ denominator is the extra shrinkage from the L2 part whenever $\alpha < 1$.

    // ---------- coordinate descent for lasso / elastic net ----------

// Objective (on standardized columns, centered y, no intercept term):
//   (1/2n) ||yc - Xs w||^2 + lambda * ( l1_ratio * ||w||_1 + (1-l1_ratio)/2 * ||w||_2^2 )
// Because columns of Xs have unit sample variance, X_j^T X_j / n = 1, which
// simplifies each coordinate update to a soft-threshold of the partial residual.
static double soft_threshold(double z, double thresh) {
    if (z > thresh) return z - thresh;
    if (z < -thresh) return z + thresh;
    return 0.0;
}

struct ENResult {
    Vector w;          // on standardized, centered scale (no intercept term)
    int iters = 0;
    bool converged = false;
};
    static ENResult fit_elastic_net_centered(const Matrix& Xs, const Vector& yc, double lambda,
                                          double l1_ratio, Vector w_init,
                                          int max_iter = 2000, double tol = 1e-7) {
    int n = (int)Xs.size(), d = (int)Xs[0].size();
    Vector w = w_init.empty() ? Vector(d, 0.0) : w_init;
    Vector resid(n);
    for (int i = 0; i < n; ++i) {
        double pred = 0.0;
        for (int j = 0; j < d; ++j) pred += Xs[i][j] * w[j];
        resid[i] = yc[i] - pred;
    }
    ENResult res;
    for (int it = 0; it < max_iter; ++it) {
        double max_change = 0.0;
        for (int j = 0; j < d; ++j) {
            // Add feature j's own contribution back into the residual, so
            // rho_j is the partial residual excluding feature j.
            for (int i = 0; i < n; ++i) resid[i] += Xs[i][j] * w[j];
            double rho = 0.0;
            for (int i = 0; i < n; ++i) rho += Xs[i][j] * resid[i];
            rho /= n;
            double w_new = soft_threshold(rho, lambda * l1_ratio) / (1.0 + lambda * (1.0 - l1_ratio));
            max_change = max(max_change, fabs(w_new - w[j]));
            w[j] = w_new;
            for (int i = 0; i < n; ++i) resid[i] -= Xs[i][j] * w[j];
        }
        res.iters = it + 1;
        if (max_change < tol) { res.converged = true; break; }
    }
    res.w = w;
    return res;
}

    The inner loop is careful to add feature $j$'s contribution back into resid before computing $\rho_j$ and subtract it again after updating w[j], which is what turns the full residual into the correct partial residual $\mathbf r^{(j)}$ for exactly one coordinate at a time without ever recomputing the whole prediction vector from scratch. Convergence is judged by the largest single coordinate change in a full sweep (max_change); once every coordinate moves by less than tol in one full pass, the sweep has settled.

    Choosing the Path: lambda_max and Warm Starts

    Lasso has a useful closed-form fact that Ridge does not: there is an exact, computable $\lambda_{\max}$ above which every coefficient is zero, because the soft-threshold update at $\mathbf w = \mathbf 0$ zeroes out coordinate $j$ whenever $\lambda\alpha \ge |\rho_j|_{\mathbf w=0}$, and $\rho_j$ at $\mathbf w=\mathbf 0$ is just $\frac1n X_{s,j}^\top \mathbf y_c$:

    // Largest lambda at which every lasso coefficient is exactly zero (for
// l1_ratio = 1). Any lambda >= this value gives the all-zero solution, which
// is the natural starting point of a regularization path.
static double lambda_max(const Matrix& Xs, const Vector& yc) {
    int n = (int)Xs.size(), d = (int)Xs[0].size();
    double m = 0.0;
    for (int j = 0; j < d; ++j) {
        double dot = 0.0;
        for (int i = 0; i < n; ++i) dot += Xs[i][j] * yc[i];
        m = max(m, fabs(dot) / n);
    }
    return m;
}

    Sweeping a geometrically spaced sequence of $\lambda$ values from $\lambda_{\max}$ down to some small fraction of it (this file uses $\lambda_{\max}\times 10^{-3}$ as the floor) traces out the full path from the empty model to a nearly unpenalized one. Solving each $\lambda$ from scratch would work, but warm-starting each solve with the previous, nearby $\lambda$'s solution converges in far fewer coordinate sweeps, since consecutive solutions on the path are usually close together, and it also produces a visibly smooth, continuous-looking path instead of independently noisy restarts:

    // Warm-started path: solve at each lambda in decreasing order, reusing the
// previous solution as the starting point for the next (smaller) lambda.
// This is both faster (fewer coordinate sweeps per lambda) and gives a
// smoothly varying coefficient path instead of independent restarts.
static Matrix fit_path(const Matrix& Xs, const Vector& yc, const Vector& lambdas, double l1_ratio) {
    Matrix path;
    Vector w(Xs[0].size(), 0.0);
    for (double lam : lambdas) {
        ENResult r = fit_elastic_net_centered(Xs, yc, lam, l1_ratio, w);
        w = r.w;
        path.push_back(w);
    }
    return path;
}

    Back to Original Units

    Everything above operates on standardized features and a centered target; converting the fitted weights back to the scale the caller actually cares about uses the same chain-rule algebra as Part 2's unstandardize_weights, adjusted for the centered target's mean:

    // Recovers intercept and original-scale coefficients from a fit on
// standardized features and a centered target.
static Vector to_original_scale(const Vector& w_scaled, const Scaler& sc, double y_mean) {
    size_t d = sc.mean.size();
    Vector w(d + 1);
    w[0] = y_mean;
    for (size_t j = 0; j < d; ++j) {
        w[j + 1] = w_scaled[j] / sc.stdev[j];
        w[0] -= w_scaled[j] * sc.mean[j] / sc.stdev[j];
    }
    return w;
}

    Prediction and Evaluation

    static double predict(const Vector& w, const Vector& x) {
    double yhat = w[0];
    for (size_t j = 0; j < x.size(); ++j) yhat += w[j + 1] * x[j];
    return yhat;
}

static double rmse(const Vector& w, const Matrix& X, const Vector& y) {
    double s = 0.0;
    for (size_t i = 0; i < X.size(); ++i) { double e = predict(w, X[i]) - y[i]; s += e * e; }
    return sqrt(s / X.size());
}

    Cross-Validation and the One-Standard-Error Rule

    Picking $\lambda$ by minimizing training error is meaningless (training error only ever improves as $\lambda \to 0$); it has to be picked by estimated generalization error, which $k$-fold cross-validation approximates without needing a separate held-out set. Standard $k$-fold CV fits the whole path on each of $k$ training folds, scores it on the held-out fold, and averages:

    // ---------- k-fold cross-validation with the one-standard-error rule ----------

struct CVResult {
    Vector lambdas;
    Vector mean_rmse;
    Vector se_rmse;
    int idx_min = 0;
    int idx_1se = 0;
};

static CVResult cross_validate_lasso(const Matrix& X, const Vector& y, const Vector& lambdas,
                                      int k = 5, unsigned seed = 0) {
    int n = (int)X.size();
    vector<int> order(n);
    iota(order.begin(), order.end(), 0);
    mt19937 rng(seed);
    shuffle(order.begin(), order.end(), rng);

    CVResult cv;
    cv.lambdas = lambdas;
    cv.mean_rmse.assign(lambdas.size(), 0.0);
    cv.se_rmse.assign(lambdas.size(), 0.0);
    vector<vector<double>> fold_rmse(lambdas.size());

    for (int f = 0; f < k; ++f) {
        vector<int> test_idx, train_idx;
        for (int i = 0; i < n; ++i) {
            if (i % k == f) test_idx.push_back(order[i]);
            else train_idx.push_back(order[i]);
        }
        Matrix X_tr, X_te; Vector y_tr, y_te;
        for (int i : train_idx) { X_tr.push_back(X[i]); y_tr.push_back(y[i]); }
        for (int i : test_idx) { X_te.push_back(X[i]); y_te.push_back(y[i]); }

        Scaler sc; sc.fit(X_tr);
        Matrix Xs_tr = sc.transform(X_tr), Xs_te = sc.transform(X_te);
        double y_mean = accumulate(y_tr.begin(), y_tr.end(), 0.0) / y_tr.size();
        Vector yc_tr(y_tr.size());
        for (size_t i = 0; i < y_tr.size(); ++i) yc_tr[i] = y_tr[i] - y_mean;

        Matrix path = fit_path(Xs_tr, yc_tr, lambdas, 1.0);
        for (size_t li = 0; li < lambdas.size(); ++li) {
            Vector w = to_original_scale(path[li], sc, y_mean);
            fold_rmse[li].push_back(rmse(w, X_te, y_te));
        }
    }
    for (size_t li = 0; li < lambdas.size(); ++li) {
        double m = accumulate(fold_rmse[li].begin(), fold_rmse[li].end(), 0.0) / k;
        double var = 0.0;
        for (double v : fold_rmse[li]) var += (v - m) * (v - m);
        var /= (k - 1);
        cv.mean_rmse[li] = m;
        cv.se_rmse[li] = sqrt(var / k);
    }
    cv.idx_min = (int)(min_element(cv.mean_rmse.begin(), cv.mean_rmse.end()) - cv.mean_rmse.begin());
    double threshold = cv.mean_rmse[cv.idx_min] + cv.se_rmse[cv.idx_min];
    // lambdas are given from large to small; the 1-SE rule picks the *largest*
    // lambda (simplest model) whose mean CV error is still within one SE of
    // the minimum.
    cv.idx_1se = cv.idx_min;
    for (size_t li = 0; li < lambdas.size(); ++li) {
        if (cv.mean_rmse[li] <= threshold) { cv.idx_1se = (int)li; break; }
    }
    return cv;
}

    Taking the $\lambda$ that minimizes mean cross-validated error is the obvious choice, but that minimum is itself a noisy estimate (it comes from averaging only $k$ numbers), and the curve of mean error against $\lambda$ is often quite flat near its minimum: many values of $\lambda$ give statistically indistinguishable performance. The one-standard-error rule uses this directly: compute the standard error of the mean CV error at the minimizing $\lambda$, then walk toward larger $\lambda$ (simpler models, since $\lambda$ shrinks more coefficients toward zero) and stop at the largest $\lambda$ whose mean error is still within one standard error of the minimum. The result is a model that is, within the noise of the cross-validation estimate itself, just as good as the strict minimizer but simpler and typically more stable to refit on a slightly different sample.

    What Each Method Does With Correlated Features

    The worked example's dataset has five standardized features: $x_1$ and $x_2$ are near-duplicates (sample correlation about 0.999, effectively the same measurement recorded twice), $x_3$ has a real, independent effect, and $x_4$, $x_5$ are pure noise. The true generative relationship is $y = 5 + 3x_1 + 2x_3$, with $x_2$, $x_4$, $x_5$ contributing nothing. This is the textbook setup for seeing how the three penalties disagree:

    • Ridge spreads the credit. At $\lambda=5$, ridge assigns $x_1$ and $x_2$ almost identical coefficients (1.3769 and 1.3543 in the worked example below) even though only $x_1$ is in the true model. Because the L2 penalty is strictly convex and smooth, whenever two columns are (near-)identical, the penalty is minimized by splitting the total coefficient evenly between them rather than favoring one arbitrarily; the combined effect ($1.3769+1.3543 \approx 2.73$) is still in the right neighborhood of the true combined coefficient.
    • Lasso picks one, somewhat arbitrarily, and the choice can shift along the path. In the standardized coefficient path below, at $\lambda=1.34$ only $x_2$ is nonzero (1.17); by $\lambda=0.72$, $x_1$ has entered too, and by the smallest $\lambda$, $x_1$ has taken over almost the entire combined coefficient (2.38) while $x_2$ has shrunk to 0.03. With near-duplicate columns, which one Lasso keeps depends on fine numerical differences and the order coordinate descent visits them in; that instability, not just sparsity, is the main practical caveat of using Lasso when predictors are highly correlated.
    • Elastic Net compromises. With $\alpha=0.5$, the worked example splits credit between $x_1$ and $x_2$ (1.4238 and 1.3967) much like ridge, while still shrinking the two noise features ($x_4$, $x_5$) toward small values the way lasso would. This is exactly Elastic Net's motivation: keep ridge's stability under correlated features while retaining some of lasso's sparsity on genuinely irrelevant ones.
    Practical takeaway. If two features are known to be near-duplicates and interpretability of individual coefficients matters, prefer ridge or an elastic net with a non-trivial L2 component; a small $\lambda$ change or a different random seed can otherwise flip which of two correlated lasso coefficients is nonzero, without changing the combined prediction much at all.

    The cross-validation table shows the classic shape: mean RMSE drops sharply as $\lambda$ decreases from $\lambda_{\max}$, flattens out around $\lambda\approx0.03$ to $0.25$, and creeps very slightly back up as $\lambda$ approaches zero (mild overfitting to the 4 noise-adjacent coefficients). The strict minimum sits at $\lambda=0.0252$; the one-standard-error rule instead picks $\lambda=0.2516$, a model with roughly ten times the regularization and a mean RMSE only about 0.06 higher, comfortably inside one standard error of the minimum, and structurally simpler (fewer of the noise features have crept in).
