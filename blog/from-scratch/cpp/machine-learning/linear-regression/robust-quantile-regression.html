    Robust and Quantile Regression in C++

    July 2026 · 26 min read

    Every model in this series so far minimizes squared error, which means a single bad data point, a data-entry typo, a sensor glitch, a mislabeled row, can dominate the fit, since squared error penalizes a residual twice as large four times as much. This post covers three ways to reduce that sensitivity: least absolute deviations (LAD), which uses the absolute residual instead of its square; Huber loss, which blends squared error near zero with a linear tail; and quantile (pinball) loss, which generalizes LAD to any quantile of the conditional distribution, not just the median. All three are optimized by (sub)gradient descent, so the signs of those gradients matter more than anything else in this post, one wrong sign and the fit moves the wrong way and nothing in the output looks obviously broken until it is compared against a known-good answer.

    Why Minimize Something Other Than Squared Error?

    OLS's normal equations (Part 1) are the unique minimizer of squared error, and squared error is the right objective when residuals are genuinely close to Gaussian and every row is trustworthy. Both assumptions break in the same common scenario: a small number of rows have a target value that is wrong, not just noisy, a decimal point typo, a unit conversion error, a sensor reading during a fault. Squared error's derivative grows linearly with the residual, so one row with a residual ten times the typical size contributes ten times the gradient of a typical row, actively pulling the fit toward matching that one bad value at the expense of every other row.

    Three responses to this, in increasing order of how much they change the objective itself:

    • Huber loss keeps squared error's behavior for small, typical residuals (so the fit is still efficient when the data really is roughly Gaussian) and switches to a linear penalty beyond a threshold $\delta$, capping how much any single outlier can influence the gradient.
    • LAD (least absolute deviations) goes further: every residual, large or small, contributes a bounded, constant-magnitude gradient. This is the $\ell_1$ loss, and it estimates the conditional median of the target rather than its mean, which is itself a more outlier-resistant summary statistic.
    • Quantile (pinball) loss generalizes LAD to any quantile $\tau \in (0,1)$, not just the median, by weighting over- and under-prediction asymmetrically. Fitting several quantiles gives a full predictive interval instead of a single point estimate.
    Where RANSAC fits in. RANSAC takes a completely different approach: instead of changing the loss, it repeatedly fits an exact model to a small random subset of points, counts how many of the remaining points agree with that fit within a tolerance (the "inliers"), and keeps the fit with the most inliers. That works well when there is a clear, verifiable notion of an inlier (classic use case: fitting a line or plane to noisy sensor data where a majority of points are known to lie near the true surface), but it does not produce a smooth loss to differentiate, does not naturally extend to more than a handful of parameters, and gives a single best fit rather than a distribution. Huber, LAD, and quantile loss instead reshape the objective itself, which keeps them inside the same gradient-based machinery as every other model in this series and lets them scale to many features.

    Four Loss Shapes, Compared

    LossFormula (in $r = \hat y - y$)Gradient wrt $\hat y$EstimatesUse when
    Squared error$\tfrac12 r^2$$r$ (unbounded)Conditional meanResiduals roughly Gaussian, no bad labels
    LAD ($\ell_1$)$|r|$$\operatorname{sign}(r)$ (bounded)Conditional medianHeavy-tailed noise or occasional bad labels, point estimate only

    $\tau=0.5$ in the pinball loss recovers LAD exactly (both terms become $\tfrac12|r|$), which is why the C++ code below implements LAD as a special case of the general quantile fitting function rather than as a separate one.

    The rest of this file is a single, independently compilable C++ program; validation, the scaler, and the OLS baseline reuse the same code shown in earlier parts, repeated here so the file stands on its own.

    Validating the Input

    // Huber and quantile (pinball) linear regression from scratch, standard
// library only: robust gradient descent with correct signed subgradients,
// multi-quantile fitting with a non-crossing repair, and a comparison
// against ordinary least squares on data with one corrupted label.
#include <bits/stdc++.h>
using namespace std;

using Vector = vector<double>;
using Matrix = vector<Vector>;

struct DataError : runtime_error {
    explicit DataError(const string& msg) : runtime_error(msg) {}
};

static void validate_dataset(const Matrix& X, const Vector& y) {
    if (X.empty()) throw DataError("X has zero rows");
    if (X.size() != y.size()) throw DataError("row count mismatch between X and y");
    size_t d = X[0].size();
    for (auto& row : X) {
        if (row.size() != d) throw DataError("ragged row in X");
        for (double v : row) if (!isfinite(v)) throw DataError("non-finite feature");
    }
    for (double v : y) if (!isfinite(v)) throw DataError("non-finite target");
}

    Scaling and Prediction

    struct Scaler {
    Vector mean, stdev;
    void fit(const Matrix& X) {
        size_t n = X.size(), d = X[0].size();
        mean.assign(d, 0.0); stdev.assign(d, 0.0);
        for (auto& row : X) for (size_t j = 0; j < d; ++j) mean[j] += row[j];
        for (double& m : mean) m /= (double)n;
        for (auto& row : X) for (size_t j = 0; j < d; ++j) stdev[j] += (row[j] - mean[j]) * (row[j] - mean[j]);
        for (double& s : stdev) { s = sqrt(s / (double)n); if (s < 1e-12) s = 1.0; }
    }
    Matrix transform(const Matrix& X) const {
        Matrix out = X;
        for (auto& row : out) for (size_t j = 0; j < row.size(); ++j) row[j] = (row[j] - mean[j]) / stdev[j];
        return out;
    }
};
    static Vector unstandardize_weights(const Vector& w_scaled, const Scaler& sc) {
    size_t d = sc.mean.size();
    Vector w(d + 1);
    w[0] = w_scaled[0];
    for (size_t j = 0; j < d; ++j) {
        w[j + 1] = w_scaled[j + 1] / sc.stdev[j];
        w[0] -= w_scaled[j + 1] * sc.mean[j] / sc.stdev[j];
    }
    return w;
}

static double predict(const Vector& w, const Vector& x) {
    double yhat = w[0];
    for (size_t j = 0; j < x.size(); ++j) yhat += w[j + 1] * x[j];
    return yhat;
}

    The OLS Baseline

    A closed-form OLS fit is computed first, purely as a comparison baseline, using the same partial-pivot solver and normal-equations approach as Part 1:

    // ---------- closed-form OLS baseline, for comparison ----------

static Vector solve_linear(Matrix A, Vector b) {
    int n = (int)b.size();
    double scale = 0.0;
    for (auto& row : A) for (double v : row) scale = max(scale, fabs(v));
    if (scale == 0.0) scale = 1.0;
    const double tol = 1e-10 * scale;
    for (int col = 0; col < n; ++col) {
        int piv = col;
        for (int r = col + 1; r < n; ++r) if (fabs(A[r][col]) > fabs(A[piv][col])) piv = r;
        if (fabs(A[piv][col]) < tol) throw DataError("solve_linear: singular system");
        swap(A[col], A[piv]); swap(b[col], b[piv]);
        double div = A[col][col];
        for (int c = col; c < n; ++c) A[col][c] /= div;
        b[col] /= div;
        for (int r = 0; r < n; ++r) if (r != col) {
            double m = A[r][col];
            if (m == 0.0) continue;
            for (int c = col; c < n; ++c) A[r][c] -= m * A[col][c];
            b[r] -= m * b[col];
        }
    }
    return b;
}

static Vector fit_ols(const Matrix& X, const Vector& y) {
    validate_dataset(X, y);
    int n = (int)X.size(), d = (int)X[0].size() + 1;
    Matrix xtx(d, Vector(d, 0.0));
    Vector xty(d, 0.0);
    for (int i = 0; i < n; ++i) {
        Vector row(d, 1.0);
        for (int j = 1; j < d; ++j) row[j] = X[i][j - 1];
        for (int a = 0; a < d; ++a) {
            xty[a] += row[a] * y[i];
            for (int b = 0; b < d; ++b) xtx[a][b] += row[a] * row[b];
        }
    }
    return solve_linear(xtx, xty);
}

    Deriving the Huber and Pinball (Sub)Gradients

    Both functions below are written in terms of $\text{err} = \hat y - y$ (prediction minus target, not the other way around), and every sign in this section depends on keeping that convention straight throughout.

    Huber. The loss is $\tfrac12\,\text{err}^2$ for $|\text{err}|\le\delta$ and $\delta(|\text{err}|-\tfrac12\delta)$ beyond it. Differentiating each piece with respect to $\text{err}$: the quadratic piece gives exactly $\text{err}$, and the linear piece gives $\pm\delta$ (matching the sign of err), so the gradient is $\text{err}$ clipped to the range $[-\delta,\delta]$. This is precisely what caps a single huge residual's influence: no matter how large $|\text{err}|$ grows, the gradient contribution never exceeds $\delta$ in magnitude.

    Pinball. The loss is $(1-\tau)\,\text{err}$ when $\text{err}>0$ (the model over-predicted) and $-\tau\,\text{err}$ when $\text{err}<0$ (the model under-predicted). Differentiating each linear piece gives a constant: $+(1-\tau)$ when over-predicting, $-\tau$ when under-predicting. At $\tau=0.5$ these are $\pm0.5$, symmetric, and match LAD's $\tfrac12\operatorname{sign}(\text{err})$ up to the constant factor that cancels in gradient descent's learning rate. The asymmetry for $\tau \ne 0.5$ is the entire mechanism behind quantile regression: penalizing over-prediction more than under-prediction (small $\tau$) pushes the fit down until only a $\tau$ fraction of points lie above it, and vice versa for large $\tau$.

    // ---------- loss gradients wrt the prediction, in terms of err = pred - y ----------

// Huber: quadratic near zero, linear beyond delta. The gradient wrt the
// prediction is the residual itself, clipped to +-delta, so a residual
// bigger than delta contributes a bounded push instead of one proportional
// to its (possibly huge) size.
static double huber_grad(double err, double delta) {
    if (err > delta) return delta;
    if (err < -delta) return -delta;
    return err;
}
static double huber_loss(double err, double delta) {
    double a = fabs(err);
    return a <= delta ? 0.5 * err * err : delta * (a - 0.5 * delta);
}

// Pinball (quantile) subgradient wrt the prediction. Over-prediction
// (err > 0) is penalized at rate (1 - tau); under-prediction is penalized
// at rate tau. tau = 0.5 recovers least absolute deviations (LAD).
static double pinball_grad(double err, double tau) {
    return err > 0.0 ? (1.0 - tau) : -tau;
}
static double pinball_loss(double err, double tau) {
    return err > 0.0 ? (1.0 - tau) * err : -tau * err;
}
    How to catch a sign error here. The single most useful sanity check when implementing either of these is to fit $\tau=0.5$ quantile regression on data with a few large positive and negative residuals and confirm it reproduces the same fit as LAD, and to fit Huber with a very large $\delta$ and confirm it converges to the same answer as OLS (a large enough $\delta$ never triggers the linear tail, so Huber degenerates exactly to squared error). Both checks were used to validate the implementation below before trusting its output on the corrupted dataset.

    Choosing Huber's Delta With a Robust Scale Estimate

    Huber's threshold $\delta$ needs to be on the same scale as a "typical" residual, but computing that scale with the ordinary standard deviation is circular: the standard deviation is itself heavily distorted by the same outliers $\delta$ is meant to guard against. The median absolute deviation (MAD), scaled by $1.4826$ to make it consistent with the standard deviation under a Gaussian assumption, is a standard robust alternative: like the median itself, it has a 50% breakdown point, meaning up to half the data can be arbitrarily corrupted before it is affected:

    // Robust scale estimate (1.4826 * MAD of a residual sample).
static double mad_scale(Vector r) {
    size_t n = r.size();
    nth_element(r.begin(), r.begin() + n / 2, r.end());
    double med = r[n / 2];
    if (n % 2 == 0) { double lo = *max_element(r.begin(), r.begin() + n / 2); med = 0.5 * (med + lo); }
    Vector dev(n);
    for (size_t i = 0; i < n; ++i) dev[i] = fabs(r[i] - med);
    nth_element(dev.begin(), dev.begin() + n / 2, dev.end());
    double mad = dev[n / 2];
    if (n % 2 == 0) { double lo = *max_element(dev.begin(), dev.begin() + n / 2); mad = 0.5 * (mad + lo); }
    return 1.4826 * mad;
}

    The usual recipe, used in the worked example below, is to fit a quick preliminary OLS model, compute its residuals, and set $\delta = 1.345 \times \text{mad\_scale}(\text{residuals})$; the constant 1.345 is chosen so that Huber loss with this $\delta$ retains about 95% of OLS's statistical efficiency when the data really is Gaussian, while still bounding the influence of genuine outliers.

    Configuring Robust (Sub)gradient Descent

    Huber's gradient is bounded: it equals the residual inside the quadratic region, so it does shrink toward zero there, but its tail saturates at $\pm\delta$. The pinball subgradient is also bounded and takes the piecewise-constant values $-\tau$ and $1-\tau$ away from its kink. A schedule that decays too aggressively can therefore make progress imperceptibly slow before a finite training run reaches the optimum. Standard convergence theory for convex subgradient descent favors diminishing step sizes whose sum diverges while the sum of their squares converges; a simple $1/t$ schedule is one example, although it can be slow in practice. This compact demonstration instead uses a small constant learning rate so every finite run keeps moving. Constant-step subgradient descent can oscillate near the optimum, so the best-loss iterate seen during training (w_best) is tracked and returned instead of blindly returning the final iterate:

    // ---------- robust gradient descent ----------

// Huber's tail saturates at +-delta; the pinball subgradient is -tau or
// 1-tau. Huber's quadratic core still shrinks with the residual. This short
// demonstration uses a small constant learning rate so a finite run keeps
// moving, then returns the best-loss iterate because constant-step
// subgradient descent can oscillate near the optimum. Production solvers
// commonly use a properly tuned diminishing schedule or a specialized
// convex optimizer.
struct FitConfig {
    int max_epochs = 3000;
    double lr = 0.5;       // constant learning rate on standardized features
    double tol = 1e-10;    // convergence check on the best-loss trace
    int patience = 50;
};

struct FitResult {
    Vector w;              // last iterate (standardized scale)
    Vector w_best;         // best-loss iterate seen (standardized scale)
    vector<double> history;
    int epochs_run = 0;
    bool converged = false;
};

    Fitting Huber Regression

    static FitResult fit_huber(const Matrix& Xs, const Vector& y, double delta, FitConfig cfg) {
    validate_dataset(Xs, y);
    if (delta <= 0.0) throw DataError("fit_huber: delta must be positive");
    int n = (int)Xs.size(), d = (int)Xs[0].size() + 1;
    FitResult res; res.w.assign(d, 0.0);
    double best_loss = numeric_limits<double>::infinity();
    int stall = 0;
    for (int ep = 0; ep < cfg.max_epochs; ++ep) {
        Vector g(d, 0.0);
        for (int i = 0; i < n; ++i) {
            double err = predict(res.w, Xs[i]) - y[i];
            double gr = huber_grad(err, delta);
            g[0] += gr;
            for (int j = 0; j < (int)Xs[i].size(); ++j) g[j + 1] += gr * Xs[i][j];
        }
        for (int j = 0; j < d; ++j) res.w[j] -= cfg.lr * g[j] / n;

        double loss = 0.0;
        for (int i = 0; i < n; ++i) loss += huber_loss(predict(res.w, Xs[i]) - y[i], delta);
        loss /= n;
        if (!isfinite(loss)) throw DataError("fit_huber: loss diverged; reduce the learning rate");
        res.history.push_back(loss);
        res.epochs_run = ep + 1;
        if (loss < best_loss - cfg.tol) { best_loss = loss; res.w_best = res.w; stall = 0; }
        else if (++stall >= cfg.patience) { res.converged = true; break; }
    }
    if (res.w_best.empty()) res.w_best = res.w;
    return res;
}

    Fitting Quantile Regression

    static FitResult fit_quantile(const Matrix& Xs, const Vector& y, double tau, FitConfig cfg) {
    if (tau <= 0.0 || tau >= 1.0) throw DataError("fit_quantile: tau must be in (0, 1)");
    validate_dataset(Xs, y);
    int n = (int)Xs.size(), d = (int)Xs[0].size() + 1;
    FitResult res; res.w.assign(d, 0.0);
    double best_loss = numeric_limits<double>::infinity();
    int stall = 0;
    for (int ep = 0; ep < cfg.max_epochs; ++ep) {
        Vector g(d, 0.0);
        for (int i = 0; i < n; ++i) {
            double err = predict(res.w, Xs[i]) - y[i];
            double gr = pinball_grad(err, tau);
            g[0] += gr;
            for (int j = 0; j < (int)Xs[i].size(); ++j) g[j + 1] += gr * Xs[i][j];
        }
        for (int j = 0; j < d; ++j) res.w[j] -= cfg.lr * g[j] / n;

        double loss = 0.0;
        for (int i = 0; i < n; ++i) loss += pinball_loss(predict(res.w, Xs[i]) - y[i], tau);
        loss /= n;
        if (!isfinite(loss)) throw DataError("fit_quantile: loss diverged; reduce the learning rate");
        res.history.push_back(loss);
        res.epochs_run = ep + 1;
        if (loss < best_loss - cfg.tol) { best_loss = loss; res.w_best = res.w; stall = 0; }
        else if (++stall >= cfg.patience) { res.converged = true; break; }
    }
    if (res.w_best.empty()) res.w_best = res.w;
    return res;
}

    fit_huber and fit_quantile are almost identical, batch (sub)gradient descent over the whole training set each epoch, differing only in which gradient function is called and, for quantile regression, in guarding that $\tau$ is strictly inside $(0,1)$. That symmetry is intentional: both losses reduce to the general pattern "accumulate a bounded per-row gradient contribution, take a constant-size step, track the best loss seen", which is also exactly the shape Huber and quantile regression have in any other language.

    Multi-Quantile Intervals and the Crossing Problem

    Fitting several quantiles independently (say $\tau=0.1$, $0.5$, $0.9$ for an 80% interval) gives a useful predictive band, but each fit is optimized on its own, with no constraint that forces the 90th-percentile prediction to stay above the 50th-percentile prediction at every point. On a small sample, especially near the edges of the training data's feature range, this can and does happen (a "crossing" quantile), which is nonsensical as a predictive interval. The fix used here is the simplest practical one: predict every quantile at a point, then sort the results:

    // Fits several quantiles independently, then repairs any crossing by
// sorting the predicted quantiles at each point. Crossing (e.g. a predicted
// 90th percentile below the predicted 50th) can happen because each tau is
// optimized on its own.
static Matrix predict_quantiles_sorted(const vector<Vector>& weights, const Matrix& X) {
    Matrix preds(X.size(), Vector(weights.size()));
    for (size_t i = 0; i < X.size(); ++i) {
        for (size_t q = 0; q < weights.size(); ++q) preds[i][q] = predict(weights[q], X[i]);
        sort(preds[i].begin(), preds[i].end());
    }
    return preds;
}

    Sorting after the fact repairs an inconsistent ordering wherever it occurs, but it is a patch, not a cure: it does not make the individual quantile estimates more accurate, it only guarantees the reported band is monotone in $\tau$. If crossing happens frequently on a given dataset, that is itself a signal that too few points support the requested tail quantile, a lower-variance approach (fitting all quantiles jointly with a monotonicity constraint, or simply choosing less extreme quantiles) is worth considering before trusting the interval.

    Evaluation

    static double rmse(const Vector& w, const Matrix& X, const Vector& y) {
    double s = 0.0;
    for (size_t i = 0; i < X.size(); ++i) { double e = predict(w, X[i]) - y[i]; s += e * e; }
    return sqrt(s / X.size());
}
static double mae(const Vector& w, const Matrix& X, const Vector& y) {
    double s = 0.0;
    for (size_t i = 0; i < X.size(); ++i) s += fabs(predict(w, X[i]) - y[i]);
    return s / X.size();
}

    Selecting Delta and Tau in Practice

    • Delta. Start with the $1.345\times\text{mad\_scale}$ recipe above using a preliminary OLS fit's residuals. If outliers are especially extreme or common, a smaller $\delta$ trades more bias (a flatter, more median-like fit) for more robustness; a larger $\delta$ moves Huber closer to plain OLS.
    • Central tau. Use $\tau=0.5$ (LAD) when the target's conditional distribution might be skewed and the median is a more representative summary than the mean, or simply as a second, differently-biased check against an OLS or Huber fit: if all three roughly agree, that is evidence the fit is not being driven by a handful of extreme rows.
    • Interval tau pairs. Wider pairs ($\tau=0.1/0.9$ for an 80% interval) need more data to estimate reliably than narrower ones ($\tau=0.2/0.8$ for a 60% interval); on small datasets, extreme quantiles are estimated from only a handful of the most extreme points and can have surprisingly unstable slopes. The animation below shows this directly by comparing a moderate interval against the same dataset.
    • Always compare against OLS on the same data. A large gap between the OLS fit and the Huber/LAD fit is itself diagnostic: it means a meaningful fraction of the squared-error solution's shape is coming from a small number of rows, worth inspecting directly rather than only fixed by switching losses.

    Worked Example

    The dataset is the same 12-row training split used in Parts 1 and 2, with one label corrupted: row 4's price is changed from about 177.6 to 357.6 (a plausible data-entry error, a missing decimal point), simulating exactly the failure mode these losses target.

    int main() {
    // Same housing rows used throughout the series, but row 4's price has a
    // data-entry error (357.6 instead of roughly 177.6): a single corrupted
    // label, which is exactly the failure mode robust losses target.
    Matrix X = {
        {20.5,4},{25.9,5},{23.5,5},{12.5,5},{14.0,4},{25.5,4},
        {8.1,3},{24.4,5},{23.9,3},{17.4,2},{14.1,5},{13.6,2}
    };
    Vector y = {228.0,277.8,241.1,174.4,177.6,265.2,109.5,264.2,232.4,173.0,199.2,144.5};
    y[4] = 357.6;

    Scaler sc; sc.fit(X);
    Matrix Xs = sc.transform(X);
    cout << fixed << setprecision(3);

    Vector w_ols = fit_ols(X, y);
    cout << "OLS (closed form) on corrupted data:\n  w=[" << w_ols[0] << ", " << w_ols[1]
         << ", " << w_ols[2] << "]\n";
    cout << "  (clean-data OLS reference from the earlier post was roughly [16.5, 7.4, 13.5])\n\n";

    Vector resid(X.size());
    for (size_t i = 0; i < X.size(); ++i) resid[i] = predict(w_ols, X[i]) - y[i];
    double delta = 1.345 * mad_scale(resid); // 1.345 * (1.4826 * raw MAD) of preliminary residuals
    cout << "Chosen Huber delta (1.345 * robust MAD scale of OLS residuals) = " << delta << "\n";

    FitConfig cfg;
    FitResult huber_res = fit_huber(Xs, y, delta, cfg);
    Vector w_huber = unstandardize_weights(huber_res.w_best, sc);
    cout << "Huber fit on corrupted data:\n  w=[" << w_huber[0] << ", " << w_huber[1]
         << ", " << w_huber[2] << "]  epochs=" << huber_res.epochs_run
         << "  converged=" << boolalpha << huber_res.converged << "\n\n";

    FitResult lad_res = fit_quantile(Xs, y, 0.5, cfg);
    Vector w_lad = unstandardize_weights(lad_res.w_best, sc);
    cout << "LAD (tau=0.5) fit on corrupted data:\n  w=[" << w_lad[0] << ", " << w_lad[1]
         << ", " << w_lad[2] << "]  epochs=" << lad_res.epochs_run << "\n\n";

    // Multi-quantile interval: 10th, 50th, 90th percentile models.
    vector<double> taus = {0.1, 0.5, 0.9};
    vector<Vector> qweights;
    for (double tau : taus) {
        FitResult r = fit_quantile(Xs, y, tau, cfg);
        qweights.push_back(unstandardize_weights(r.w_best, sc));
    }
    Matrix sorted_preds = predict_quantiles_sorted(qweights, X);
    int covered = 0;
    for (size_t i = 0; i < X.size(); ++i)
        if (y[i] >= sorted_preds[i][0] && y[i] <= sorted_preds[i][2]) ++covered;
    cout << "Quantile interval [tau=0.1, tau=0.9] coverage on corrupted data: "
         << covered << "/" << X.size() << " rows inside the interval (target: about 80%)\n";
    cout << "First 3 rows, sorted quantile predictions (q10, q50, q90):\n";
    for (int i = 0; i < 3; ++i)
        cout << "  row " << i << ": [" << sorted_preds[i][0] << ", " << sorted_preds[i][1]
             << ", " << sorted_preds[i][2] << "]  actual=" << y[i] << "\n";

    cout << "\nMAE comparison on the corrupted training rows (lower means less pulled by the outlier):\n";
    cout << "  OLS   MAE = " << mae(w_ols, X, y) << "  RMSE = " << rmse(w_ols, X, y) << "\n";
    cout << "  Huber MAE = " << mae(w_huber, X, y) << "  RMSE = " << rmse(w_huber, X, y) << "\n";
    cout << "  LAD   MAE = " << mae(w_lad, X, y) << "  RMSE = " << rmse(w_lad, X, y) << "\n";
    return 0;
}
    OLS (closed form) on corrupted data: w=[55.950, 5.040, 18.499] (clean-data OLS reference from the earlier post was roughly [16.5, 7.4, 13.5]) Chosen Huber delta (1.345 * robust MAD scale of OLS residuals) = 22.363 Huber fit on corrupted data: w=[21.391, 7.104, 14.157] epochs=105 converged=true LAD (tau=0.5) fit on corrupted data: w=[12.669, 7.713, 13.070] epochs=1206 Quantile interval [tau=0.1, tau=0.9] coverage on corrupted data: 11/12 rows inside the interval (target: about 80%) First 3 rows, sorted quantile predictions (q10, q50, q90): row 0: [147.107, 223.075, 252.715] actual=228.000 row 1: [166.953, 277.797, 292.683] actual=277.800 row 2: [160.822, 259.285, 286.681] actual=241.100 MAE comparison on the corrupted training rows (lower means less pulled by the outlier): OLS MAE = 27.382 RMSE = 49.524 Huber MAE = 20.016 RMSE = 52.493 LAD MAE = 19.610 RMSE = 53.747

    The corrupted label alone drags the OLS intercept from about 16.5 (the clean-data value from Part 1) to 55.95, more than triple, and both slopes shift meaningfully too. Huber (21.4) and LAD (12.7) both land far closer to the clean-data intercept than OLS does, with LAD closest of all, consistent with LAD's stronger robustness (a bounded, constant-magnitude gradient contribution from every row, including the corrupted one, rather than Huber's contribution which is bounded but still larger for larger residuals up to $\delta$). The MAE comparison at the bottom makes the practical payoff concrete: both robust methods have noticeably lower mean absolute error on this corrupted training set than OLS does, even though OLS is, by construction, the method that minimizes squared error on the very same data; minimizing the wrong thing well is still minimizing the wrong thing. The 80%-target quantile interval covers 11 of 12 rows (about 92%), reasonably close to the target given only 12 rows total.

    Watching an Outlier Pull the Fit

    The animation below uses a single-feature simplification of the worked example above (house size only, so the fit is a 2D line rather than a 3D plane) and recomputes every fit live, in JavaScript, using the same algorithms as the C++ above: closed-form OLS, constant-learning-rate Huber and LAD (sub)gradient descent with best-iterate tracking, and a 60% quantile interval ($\tau=0.2$ and $\tau=0.8$, chosen to be more stable than the worked example's wider 80% interval on this smaller single-feature problem). Step through to watch one corrupted label pull the OLS line away from the rest of the data, while Huber and LAD both stay much closer to where the clean fit was than OLS does.

    Notice that Huber ends up closer to the clean-data intercept than LAD in this single-feature reduction (a diff of about 7 versus about 21), the opposite ranking from the two-feature worked example above where LAD was closest. That is not a contradiction, it is the same lesson as the residual-diagnostics posts: dropping a feature changes the whole residual structure, so which robust method lands closest to a "clean" reference depends on the specific data and outlier configuration, not on a fixed law that one loss always beats the other. What is consistent across both cases is the more important, general result: both Huber and LAD stay far closer to the uncorrupted fit than plain OLS does, because a single extreme residual can only ever contribute a bounded amount to their gradients, while it contributes an unbounded, ever-growing amount to squared error's.
