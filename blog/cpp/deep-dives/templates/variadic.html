    Variadic Templates

    March 2026 · 20 min read

    What Are Variadic Templates?

    Introduced in C++11, variadic templates allow a template to accept an arbitrary number of template arguments. Before C++11, if you wanted a function that accepted 1, 2, 3, or 10 arguments of different types, you needed to write separate overloads for each count. Variadic templates eliminate this entirely.

    They are the foundation behind std::tuple, std::variant, std::make_unique, std::make_shared, std::invoke, and practically every modern C++ utility that works with heterogeneous argument lists.

    // A function that accepts any number of arguments of any types
template <typename... Args>
void log(const Args&... args);

log(42);                          // one argument
log("status", 200, true);        // three arguments, three different types
log(1, 2.0, "three", 'f', 5L);  // five arguments, five types
    Key insight: The ... (ellipsis) has two distinct roles in variadic templates: in the template parameter list, it declares a parameter pack; to the right of a pattern, it expands that pack. These are fundamentally different operations.

    Parameter Packs

    A template parameter pack is a template parameter that accepts zero or more arguments. A function parameter pack is a function parameter that matches the corresponding template parameter pack:

    template <typename... Types>  // Types is a template parameter pack
void f(Types... args);         // args is a function parameter pack

// When called as f(1, 2.0, "hi"):
//   Types = {int, double, const char*}
//   args  = {1, 2.0, "hi"}

    Three Kinds of Parameter Packs

    • Type parameter packs: typename... Ts — accepts zero or more types.
    • Non-type parameter packs: int... Ns — accepts zero or more values.
    • Template template parameter packs: template<typename> class... Containers — accepts zero or more templates.
    // Type parameter pack
template <typename... Ts>
struct TypeList {};

// Non-type parameter pack
template <int... Values>
struct IntList {};

// Template template parameter pack
template <template <typename> class... Containers>
struct ContainerSet {};

    sizeof...

    The sizeof... operator returns the number of elements in a parameter pack at compile time:

    template <typename... Ts>
constexpr std::size_t count() {
    return sizeof...(Ts);
}

count<int, double, char>()  // 3
count<>()                    // 0

template <typename... Args>
void f(Args... args) {
    constexpr auto n = sizeof...(args);  // works on function packs too
}

    Pack Expansion

    Pack expansion is where the real power lies. You write a pattern containing the pack, followed by .... The compiler repeats the pattern once for each element in the pack, separating them with commas:

    template <typename... Ts>
void f(Ts... args) {
    // Pattern: args
    // Expansion: args... becomes arg1, arg2, arg3, ...
    g(args...);
}

// If f(1, 2.0, "hi") is called:
// g(args...) expands to g(1, 2.0, "hi")

    Expansion in Different Contexts

    Pack expansion works in many contexts, not just function call arguments:

    // 1. Function call arguments
g(args...);                    // g(a1, a2, a3)

// 2. Applying a function to each element
g(process(args)...);           // g(process(a1), process(a2), process(a3))

// 3. Initializer lists
auto list = {args...};         // {a1, a2, a3}

// 4. Base class list
template <typename... Bases>
struct Derived : Bases... {};   // Derived inherits from all Bases

// 5. Member initializer list
template <typename... Bases>
struct Derived : Bases... {
    Derived(int x) : Bases(x)... {}  // initialize all bases with x
};

// 6. Template argument list
std::tuple<Ts...> t;           // std::tuple<int, double, string>

// 7. Lambda capture
[args...]() { ... };            // capture all pack elements by value
[&args...]() { ... };           // capture all by reference
    Key insight: The pattern before ... can be any expression involving the pack. process(args)... applies process to each element individually. std::forward<Ts>(args)... perfect-forwards each argument individually.

    Recursive Unpacking Pattern

    Before C++17 fold expressions, the standard way to process each element in a pack was recursion: peel off the first argument, process it, then recurse on the rest.

    // Base case: no arguments
void print() {
    std::cout << "\n";
}

// Recursive case: process first, recurse on rest
template <typename T, typename... Rest>
void print(const T& first, const Rest&... rest) {
    std::cout << first;
    if constexpr (sizeof...(rest) > 0) {
        std::cout << ", ";
    }
    print(rest...);  // recurse with one fewer argument
}

print(1, 2.5, "hello", 'x');
// Output: 1, 2.5, hello, x

    How the recursion unfolds:

    print(1, 2.5, "hello", 'x')
  → cout << 1; print(2.5, "hello", 'x')
    → cout << 2.5; print("hello", 'x')
      → cout << "hello"; print('x')
        → cout << 'x'; print()
          → cout << "\n";

    Each recursive call instantiates a new function with one fewer parameter. The compiler generates four different print functions plus the base case. This is all resolved at compile time—there is no runtime recursion overhead beyond the function call chain.

    Alternative: Comma Operator Trick (C++11/14)

    Before fold expressions, another approach avoided explicit recursion by using the comma operator inside an initializer list expansion:

    template <typename... Args>
void print_all(const Args&... args) {
    // each (cout << args, 0) is evaluated, results form an initializer list
    using expand = int[];
    (void)expand{0, (std::cout << args << " ", 0)...};
    std::cout << "\n";
}

    This works but is cryptic. C++17 fold expressions are the idiomatic replacement.

    Fold Expressions (C++17)

    Fold expressions collapse a parameter pack using a binary operator. They replace the recursive pattern for most use cases and are dramatically simpler:

    Four Forms of Fold

    Form Syntax Expansion
    Unary left fold(... op args)((a1 op a2) op a3) op a4
    Unary right fold(args op ...)a1 op (a2 op (a3 op a4))
    Binary left fold(init op ... op args)((init op a1) op a2) op a3
    Binary right fold(args op ... op init)a1 op (a2 op (a3 op init))
     
    // Sum all arguments
template <typename... Args>
auto sum(Args... args) {
    return (args + ...);   // unary right fold: a1 + (a2 + (a3 + a4))
}

sum(1, 2, 3, 4)  // 10

// Sum with initial value
template <typename... Args>
auto sum_from(int init, Args... args) {
    return (init + ... + args);  // binary left fold: ((init + a1) + a2) + a3
}

sum_from(100, 1, 2, 3)  // 106

    Printing with Fold Expressions

    template <typename... Args>
void print(const Args&... args) {
    ((std::cout << args << " "), ...);  // comma fold
    std::cout << "\n";
}

print(1, 2.5, "hello");
// Output: 1 2.5 hello

    Boolean Folds

    // Are all arguments true?
template <typename... Args>
bool all_true(Args... args) {
    return (args && ...);  // short-circuits
}

// Is any argument equal to the target?
template <typename T, typename... Args>
bool any_of(T target, Args... args) {
    return ((args == target) || ...);
}

all_true(true, true, true)   // true
all_true(true, false, true)  // false
any_of(3, 1, 2, 3, 4)       // true

    Fold over Comma for Side Effects

    template <typename... Args>
void push_all(std::vector<int>& v, Args... args) {
    (v.push_back(args), ...);  // expands to: v.push_back(a1), v.push_back(a2), ...
}

std::vector<int> v;
push_all(v, 1, 2, 3, 4, 5);  // v = {1, 2, 3, 4, 5}
    Key insight: The comma operator fold (expr, ...) is the most versatile fold form. It lets you execute any side-effecting expression for each pack element. Think of it as a compile-time for-each loop.

    Practical Examples

    Implementing make_unique

    template <typename T, typename... Args>
std::unique_ptr<T> make_unique(Args&&... args) {
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

// Usage:
auto p = make_unique<std::string>(5, 'a');  // "aaaaa"

    The std::forward<Args>(args)... expansion creates one std::forward call per argument, preserving value categories (lvalue/rvalue) through perfect forwarding.

    Type-Safe printf

    void safe_printf(const char* fmt) {
    while (*fmt) {
        if (*fmt == '%') throw std::runtime_error("Too few arguments");
        std::cout << *fmt++;
    }
}

template <typename T, typename... Rest>
void safe_printf(const char* fmt, const T& val, const Rest&... rest) {
    while (*fmt) {
        if (*fmt == '%') {
            std::cout << val;
            safe_printf(fmt + 1, rest...);
            return;
        }
        std::cout << *fmt++;
    }
    throw std::runtime_error("Too many arguments");
}

safe_printf("Name: %, Age: %, Score: %\n", "Alice", 30, 95.5);
// Output: Name: Alice, Age: 30, Score: 95.5

    Simplified Tuple

    // Base case: empty tuple
template <typename... Ts>
struct Tuple {};

// Recursive case: store head, inherit tail
template <typename Head, typename... Tail>
struct Tuple<Head, Tail...> : Tuple<Tail...> {
    Head value;
    Tuple(Head h, Tail... t) : Tuple<Tail...>(t...), value(h) {}
};

// Get element by index
template <std::size_t I, typename Head, typename... Tail>
auto get(const Tuple<Head, Tail...>& t) {
    if constexpr (I == 0) {
        return t.value;
    } else {
        return get<I - 1>(static_cast<const Tuple<Tail...>&>(t));
    }
}

Tuple<int, double, std::string> t(42, 3.14, "hello");
get<0>(t)  // 42
get<1>(t)  // 3.14
get<2>(t)  // "hello"

    Index Sequences

    The standard library provides std::index_sequence and std::make_index_sequence to work with compile-time index packs:

    template <typename Tuple, std::size_t... Is>
void print_tuple_impl(const Tuple& t, std::index_sequence<Is...>) {
    ((std::cout << (Is == 0 ? "" : ", ") << std::get<Is>(t)), ...);
}

template <typename... Ts>
void print_tuple(const std::tuple<Ts...>& t) {
    std::cout << "(";
    print_tuple_impl(t, std::index_sequence_for<Ts...>{});
    std::cout << ")\n";
}

print_tuple(std::make_tuple(1, 2.5, "hello"));
// Output: (1, 2.5, hello)

    Advanced Patterns

    Constraining Pack Elements (C++20)

    // All arguments must be integral
template <std::integral... Ts>
auto sum(Ts... args) {
    return (args + ...);
}

sum(1, 2, 3);      // OK
// sum(1, 2.5, 3);  // ERROR: double is not integral

    Pack Expansion in Lambda Captures (C++20)

    template <typename... Args>
auto make_lazy(Args... args) {
    return [...args = std::move(args)]() {
        return (args + ...);
    };
}

auto lazy = make_lazy(1, 2, 3);
lazy()  // 6

    Structured Bindings with Packs (Future)

    C++26 is expected to introduce pack-expanded structured bindings, further simplifying pack operations. For now, index sequences remain the standard approach for positional access.
