    C++17 Parallel Algorithms

    March 2026 · 20 min read

    Parallelism in One Line

    C++17 added execution policies to the standard algorithms. By passing a policy as the first argument, you can parallelize std::sort, std::for_each, std::transform, std::reduce, and over 70 other algorithms with no manual thread management:

    #include <algorithm>
#include <execution>
#include <vector>

std::vector<int> data(10'000'000);

// Sequential (classic)
std::sort(data.begin(), data.end());

// Parallel!
std::sort(std::execution::par, data.begin(), data.end());

// Parallel + vectorized (SIMD)
std::sort(std::execution::par_unseq, data.begin(), data.end());

    Execution Policies

    PolicyParallelismVectorizationNotes
    seqNoNoSame as the non-policy overload.
    parYes (threads)NoMay use thread pool. User function must be thread-safe.
    par_unseqYesYes (SIMD)User function must not acquire mutexes or allocate memory.
    unseq (C++20)NoYes (SIMD)Vectorized on one thread.

    Key Parallel Algorithms

    std::for_each

    std::vector<double> v(1'000'000, 1.0);
std::for_each(std::execution::par, v.begin(), v.end(),
    [](double& x) { x = std::sin(x); });

    std::transform_reduce (Map-Reduce)

    // Dot product: sum of element-wise products
double dot = std::transform_reduce(
    std::execution::par,
    a.begin(), a.end(), b.begin(),
    0.0  // initial value
);

    std::reduce (Parallel Accumulate)

    // std::accumulate is sequential and order-dependent.
// std::reduce may reorder operations (requires associative + commutative op).
auto sum = std::reduce(std::execution::par, v.begin(), v.end(), 0.0);

// WARNING: floating-point addition is not truly associative.
// Results may differ slightly between sequential and parallel.

    Constraints on User Functions

    • par: User-supplied function objects must not introduce data races. They may use mutexes and allocations.
    • par_unseq: User functions must not call blocking functions, acquire mutexes, or perform memory allocations. They must be safe for vectorization (no branches dependent on other elements).

    Exception Handling

    If any element's operation throws an exception during parallel execution, std::terminate() is called. There is no mechanism to catch individual exceptions from parallel algorithm elements. Ensure your operations are noexcept or handle exceptions internally.

    Performance Tips

    • Minimum data size: Parallelism has overhead. For small datasets (<10,000 elements), sequential is often faster.
    • Compiler support: GCC uses Intel TBB as the backend. You need to link with -ltbb. MSVC has native support.
    • Measure, do not assume: Always benchmark to verify parallel actually helps for your specific workload.
    // Compile with TBB on GCC/Clang:
g++ -std=c++20 -O2 -ltbb program.cpp -o program

    Summary

    • C++17 parallel algorithms let you add parallelism by changing one argument.
    • par uses threads; par_unseq uses threads + SIMD.
    • User functions must be thread-safe (no data races).
    • std::reduce and std::transform_reduce replace std::accumulate for parallel reductions.
    • Exceptions in parallel algorithms call std::terminate().
    • Measure before and after to confirm performance benefit.
