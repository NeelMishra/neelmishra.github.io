    Futures and Promises

    March 2026 · 22 min read

    One-Shot Communication Channels

    A future and promise form a one-time communication channel between threads. The promise is the sending end (producer) and the future is the receiving end (consumer). The producer sets a value (or an exception); the consumer waits for and retrieves it.

    Unlike mutexes and condition variables, futures are higher-level: they handle synchronization and value transfer in one abstraction.

    #include <future>
#include <thread>
#include <iostream>

int compute() { return 42; }

int main() {
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();

    std::thread t([&prom]{
        int result = compute();
        prom.set_value(result);  // send
    });

    int value = fut.get();  // blocks until value is available
    std::cout << "Got: " << value << "\n";
    t.join();
}

    ▶ Promise-Future Channel

    Watch a value travel from the promise (producer) to the future (consumer).

    std::promise in Detail

    std::promise<int> prom;

// Set the value (can only be called ONCE)
prom.set_value(42);

// Or set an exception instead
prom.set_exception(std::make_exception_ptr(std::runtime_error("fail")));

// If the promise is destroyed without setting a value,
// the future receives a std::future_error (broken_promise).

    std::future in Detail

    std::future<int> fut = prom.get_future();

// Block until value is ready
int val = fut.get();

// Check without blocking
if (fut.wait_for(std::chrono::milliseconds(0)) == std::future_status::ready) {
    int val = fut.get();
}

// Wait with timeout
auto status = fut.wait_for(std::chrono::seconds(1));
if (status == std::future_status::ready) { /* value available */ }
else if (status == std::future_status::timeout) { /* not yet */ }
else if (status == std::future_status::deferred) { /* lazy */ }
    get() can only be called once. After calling get(), the future is in a moved-from state. Calling get() again is undefined behavior. If you need multiple consumers, use std::shared_future.

    std::packaged_task

    A packaged_task wraps a callable and automatically creates the promise/future pair. Calling the task sets the return value (or exception) on the promise:

    std::packaged_task<int(int, int)> task([](int a, int b) {
    return a + b;
});

std::future<int> fut = task.get_future();

std::thread t(std::move(task), 10, 20);
int result = fut.get();  // 30
t.join();

    Exception Propagation

    If the producer throws, the exception is captured and re-thrown when the consumer calls get():

    std::promise<int> prom;
auto fut = prom.get_future();

std::thread t([&prom]{
    try {
        throw std::runtime_error("oops");
    } catch (...) {
        prom.set_exception(std::current_exception());
    }
});

try {
    int val = fut.get();  // rethrows runtime_error
} catch (const std::exception& e) {
    std::cerr << "Exception from thread: " << e.what() << "\n";
}
t.join();

    std::shared_future

    A shared_future allows multiple threads to wait for and retrieve the same result. It is copyable (unlike future, which is move-only):

    std::promise<int> prom;
std::shared_future<int> sfut = prom.get_future().share();

// Multiple threads can each call sfut.get()
std::thread t1([sfut]{ std::cout << sfut.get(); });
std::thread t2([sfut]{ std::cout << sfut.get(); });
// Both read the same value without data races

prom.set_value(42);
t1.join(); t2.join();

    Summary

    • promise is the producer, future is the consumer of a one-time value.
    • future::get() blocks until the value is ready and can only be called once.
    • packaged_task wraps a callable with an auto-created promise/future pair.
    • Exceptions propagate through the channel: thrown by producer, caught by consumer.
    • shared_future enables multiple consumers.

    Futures and promises are powerful but low-level. The next post covers std::async, which makes task-based parallelism even simpler.
