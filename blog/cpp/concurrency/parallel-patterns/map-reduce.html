    Map-Reduce Pattern

    March 2026

    Overview

    Map-reduce is a two-phase parallel pattern. In the map phase, each element of the input is independently transformed in parallel. In the reduce phase, the transformed results are combined (accumulated) into a single output. Because both phases are embarrassingly parallel (with associative reduction), this pattern scales well across many cores.

    Two Phases. Map applies a function to every element independently. Reduce combines all results using an associative binary operation. The independence in map and the associativity in reduce are what make parallelism safe.

    Map Phase: Parallel Transform

    The map phase applies an independent function to every element. Since there are no dependencies between elements, this is trivially parallelizable. In C++, you can split the range across threads. 

    #include <future>
#include <iostream>
#include <thread>
#include <vector>

template <typename T, typename Func>
std::vector<T> parallel_map(const std::vector<T>& input, Func f,
                             size_t num_threads = 0) {
    if (num_threads == 0)
        num_threads = std::thread::hardware_concurrency();
    if (num_threads == 0) num_threads = 2;

    std::vector<T> output(input.size());
    size_t chunk = (input.size() + num_threads - 1) / num_threads;

    std::vector<std::future<void>> futures;
    for (size_t t = 0; t < num_threads; ++t) {
        size_t lo = t * chunk;
        size_t hi = std::min(lo + chunk, input.size());
        if (lo >= hi) break;

        futures.push_back(std::async(std::launch::async, [&, lo, hi]() {
            for (size_t i = lo; i < hi; ++i)
                output[i] = f(input[i]);
        }));
    }
    for (auto& fut : futures) fut.get();
    return output;
}

int main() {
    std::vector<int> data = {1, 2, 3, 4, 5, 6, 7, 8};
    auto squares = parallel_map(data, [](int x) { return x * x; });
    for (int v : squares) std::cout << v << " ";
    // Output: 1 4 9 16 25 36 49 64
    return 0;
}

    Reduce Phase: Parallel Accumulate

    The reduce phase combines all elements using an associative binary operation. We split the data into chunks, reduce each chunk locally, then combine partial results. 

    #include <future>
#include <numeric>
#include <vector>

template <typename T, typename BinOp>
T parallel_reduce(const std::vector<T>& data, T init, BinOp op,
                  size_t num_threads = 0) {
    if (num_threads == 0)
        num_threads = std::thread::hardware_concurrency();
    if (num_threads == 0) num_threads = 2;

    size_t chunk = (data.size() + num_threads - 1) / num_threads;
    std::vector<std::future<T>> futures;

    for (size_t t = 0; t < num_threads; ++t) {
        size_t lo = t * chunk;
        size_t hi = std::min(lo + chunk, data.size());
        if (lo >= hi) break;

        futures.push_back(std::async(std::launch::async, [&, lo, hi]() {
            return std::accumulate(data.begin() + lo,
                                   data.begin() + hi,
                                   T{}, op);
        }));
    }

    T result = init;
    for (auto& fut : futures) {
        result = op(result, fut.get());
    }
    return result;
}

int main() {
    std::vector<int> data = {1, 2, 3, 4, 5, 6, 7, 8};
    int sum = parallel_reduce(data, 0, std::plus<int>{});
    std::cout << "Sum: " << sum << "\n";  // Sum: 36
    return 0;
}

    Combining Map and Reduce

    The real power of map-reduce emerges when the two phases are composed. Map transforms the data; reduce aggregates the transformed values. We can fuse these into a single pass per chunk for better cache performance. 

    template <typename T, typename MapFn, typename ReduceOp>
T map_reduce(const std::vector<T>& data, T init,
             MapFn map_fn, ReduceOp reduce_op,
             size_t num_threads = 0) {
    if (num_threads == 0)
        num_threads = std::thread::hardware_concurrency();
    if (num_threads == 0) num_threads = 2;

    size_t chunk = (data.size() + num_threads - 1) / num_threads;
    std::vector<std::future<T>> futures;

    for (size_t t = 0; t < num_threads; ++t) {
        size_t lo = t * chunk;
        size_t hi = std::min(lo + chunk, data.size());
        if (lo >= hi) break;

        futures.push_back(std::async(std::launch::async, [&, lo, hi]() {
            T local = T{};
            for (size_t i = lo; i < hi; ++i)
                local = reduce_op(local, map_fn(data[i]));
            return local;
        }));
    }

    T result = init;
    for (auto& fut : futures)
        result = reduce_op(result, fut.get());
    return result;
}

int main() {
    std::vector<int> data = {2, 5, 3, 8, 1, 7};
    // Sum of squares: 4 + 25 + 9 + 64 + 1 + 49 = 152
    int sum_sq = map_reduce(data, 0,
                            [](int x) { return x * x; },
                            std::plus<int>{});
    std::cout << "Sum of squares: " << sum_sq << "\n";
    return 0;
}

    Practical Example: Parallel Word Count

    Word count is the canonical map-reduce problem. Each chunk of text is processed independently to produce a local word count map. The reduce phase merges all maps. 

    #include <future>
#include <iostream>
#include <sstream>
#include <string>
#include <unordered_map>
#include <vector>

using WordMap = std::unordered_map<std::string, int>;

WordMap count_words(const std::string& text) {
    WordMap counts;
    std::istringstream iss(text);
    std::string word;
    while (iss >> word) {
        ++counts[word];
    }
    return counts;
}

WordMap merge_maps(const WordMap& a, const WordMap& b) {
    WordMap result = a;
    for (const auto& [word, count] : b) {
        result[word] += count;
    }
    return result;
}

WordMap parallel_word_count(const std::vector<std::string>& chunks) {
    // Map phase: count words in each chunk in parallel
    std::vector<std::future<WordMap>> futures;
    for (const auto& chunk : chunks) {
        futures.push_back(std::async(std::launch::async, count_words, chunk));
    }

    // Reduce phase: merge all maps
    WordMap result;
    for (auto& fut : futures) {
        result = merge_maps(result, fut.get());
    }
    return result;
}

int main() {
    std::vector<std::string> chunks = {
        "the quick brown fox jumps over the lazy dog",
        "the fox runs fast and the dog sleeps",
        "quick fox quick fox quick fox"
    };
    auto counts = parallel_word_count(chunks);
    for (const auto& [word, count] : counts) {
        std::cout << word << ": " << count << "\n";
    }
    return 0;
}

    C++17: std::transform_reduce

    C++17 introduced std::transform_reduce in <numeric>, which fuses map and reduce into a single algorithm. When used with an execution policy, the standard library handles parallelization automatically. 

    #include <execution>
#include <iostream>
#include <numeric>
#include <vector>

int main() {
    std::vector<int> data = {2, 5, 3, 8, 1, 7};

    // Sequential transform_reduce: sum of squares
    int seq = std::transform_reduce(
        data.begin(), data.end(),
        0,                          // init
        std::plus<>{},              // reduce operation
        [](int x) { return x * x; } // map operation
    );
    std::cout << "Sequential: " << seq << "\n";  // 152

    // Parallel transform_reduce (C++17 execution policy)
    int par = std::transform_reduce(
        std::execution::par,
        data.begin(), data.end(),
        0,
        std::plus<>{},
        [](int x) { return x * x; }
    );
    std::cout << "Parallel:   " << par << "\n";  // 152

    // Dot product using the two-range overload
    std::vector<int> a = {1, 2, 3};
    std::vector<int> b = {4, 5, 6};
    int dot = std::transform_reduce(
        std::execution::par,
        a.begin(), a.end(), b.begin(),
        0  // init
    );
    std::cout << "Dot product: " << dot << "\n";  // 32
    return 0;
}
    Requirements. For std::transform_reduce to be used safely with parallel execution policies, the reduce operation must be both associative and commutative, and the map operation must not modify shared state. std::plus, std::multiplies, and similar stateless functors satisfy these requirements.

    parallel_accumulate Revisited

    Many C++ codebases define a custom parallel_accumulate before C++17 was available. This pattern is essentially a reduce-only map-reduce where the map is the identity function. 

    #include <algorithm>
#include <future>
#include <numeric>
#include <vector>

template <typename Iter, typename T>
T parallel_accumulate(Iter begin, Iter end, T init) {
    auto len = std::distance(begin, end);
    if (len == 0) return init;

    unsigned int hw = std::thread::hardware_concurrency();
    unsigned int num_threads = (hw != 0) ? hw : 2;
    auto block_size = len / num_threads;

    std::vector<std::future<T>> futures;
    Iter block_start = begin;

    for (unsigned int i = 0; i < num_threads - 1; ++i) {
        Iter block_end = block_start;
        std::advance(block_end, block_size);
        futures.push_back(std::async(std::launch::async,
            [](Iter s, Iter e) { return std::accumulate(s, e, T{}); },
            block_start, block_end));
        block_start = block_end;
    }

    // Last chunk handled by current thread
    T last = std::accumulate(block_start, end, T{});

    T result = init + last;
    for (auto& fut : futures) {
        result += fut.get();
    }
    return result;
}

int main() {
    std::vector<double> data(1'000'000, 0.001);
    double sum = parallel_accumulate(data.begin(), data.end(), 0.0);
    std::cout << "Sum: " << sum << "\n";  // ~1000.0
    return 0;
}

    Key Takeaways

    • Map-reduce separates independent transformation (map) from aggregation (reduce).
    • The map function must be side-effect free; the reduce operation must be associative.
    • Fusing map and reduce into a single pass per chunk improves cache performance.
    • C++17 std::transform_reduce with execution policies handles parallelism automatically.
    • Word count is the classic map-reduce application; the pattern generalizes to many aggregation tasks.
