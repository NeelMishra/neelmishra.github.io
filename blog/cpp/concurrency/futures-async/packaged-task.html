    std::packaged_task

    March 2026

    What Is a packaged_task?

    std::packaged_task<R(Args...)> wraps any callable (function, lambda, functor) so that its return value (or thrown exception) is automatically captured in a std::future. Think of it as a promise that knows how to fulfil itself when you invoke the wrapped callable.

    Unlike a raw std::promise, where you manually call set_value, a packaged_task does that for you when it is invoked via operator().

    Header. std::packaged_task lives in <future>, just like std::promise and std::future.

    Basic Usage

    #include <future>
#include <iostream>

int compute(int a, int b) {
    return a * b + 1;
}

int main() {
    // Wrap a plain function
    std::packaged_task<int(int, int)> task(compute);

    // Obtain the future BEFORE invoking the task
    std::future<int> fut = task.get_future();

    // Invoke the task (this stores the return value)
    task(6, 7);

    std::cout << fut.get() << "\n";  // 43
}

    Calling task(6, 7) executes compute(6, 7) internally, stores the return value 43 in the shared state, and marks the future as ready.

    Wrapping Lambdas and Functors

    Any callable that matches the signature works:

    #include <future>
#include <iostream>
#include <string>

int main() {
    // Lambda
    std::packaged_task<std::string(int)> task(
        [](int n) -> std::string {
            return "Value is " + std::to_string(n);
        }
    );

    std::future<std::string> fut = task.get_future();
    task(42);
    std::cout << fut.get() << "\n";  // "Value is 42"
}
    // Functor example
struct Hasher {
    std::size_t operator()(const std::string& s) const {
        return std::hash<std::string>{}(s);
    }
};

std::packaged_task<std::size_t(const std::string&)> task(Hasher{});
auto fut = task.get_future();
task("hello");
std::cout << fut.get() << "\n";

    Moving Tasks into Threads

    packaged_task is move-only (it owns the shared state). Move it into a std::thread to run asynchronously. 

    #include <future>
#include <thread>
#include <iostream>
#include <vector>
#include <numeric>

int main() {
    std::packaged_task<long long(const std::vector<int>&)> task(
        [](const std::vector<int>& v) {
            return std::accumulate(v.begin(), v.end(), 0LL);
        }
    );

    std::future<long long> fut = task.get_future();

    std::vector<int> data(100000, 1);

    // Move task into thread; pass data by const-ref
    std::thread t(std::move(task), std::cref(data));

    std::cout << "Sum: " << fut.get() << "\n";  // 100000
    t.join();
}
    get_future before move. Always extract the future before moving the task into the thread. After the move, the original variable is in a moved-from state and calling get_future() on it is undefined.

    Task Queue Pattern

    A common concurrency pattern is a thread-safe queue of packaged_task objects processed by a worker pool. Step through the diagram to see how tasks flow from producers to a dedicated worker thread.

    Implementation

    #include <future>
#include <thread>
#include <queue>
#include <mutex>
#include <condition_variable>
#include <functional>
#include <iostream>

class TaskQueue {
    std::queue<std::packaged_task<void()>> tasks_;
    std::mutex mtx_;
    std::condition_variable cv_;
    bool stop_ = false;

public:
    // Submit any callable, return a future for its result
    template <typename F>
    std::future<std::invoke_result_t<F>> submit(F&& f) {
        using R = std::invoke_result_t<F>;

        std::packaged_task<R()> task(std::forward<F>(f));
        std::future<R> fut = task.get_future();

        {
            std::lock_guard lk(mtx_);
            // Wrap R()-returning task into void() for uniform storage
            tasks_.push(std::packaged_task<void()>(std::move(task)));
        }
        cv_.notify_one();
        return fut;
    }

    void worker_loop() {
        while (true) {
            std::packaged_task<void()> task;
            {
                std::unique_lock lk(mtx_);
                cv_.wait(lk, [&]{ return stop_ || !tasks_.empty(); });
                if (stop_ && tasks_.empty()) return;
                task = std::move(tasks_.front());
                tasks_.pop();
            }
            task();   // executes and fulfils the promise inside
        }
    }

    void shutdown() {
        {
            std::lock_guard lk(mtx_);
            stop_ = true;
        }
        cv_.notify_all();
    }
};

int main() {
    TaskQueue q;
    std::thread worker(&TaskQueue::worker_loop, &q);

    auto f1 = q.submit([]{ return 10 + 20; });
    auto f2 = q.submit([]{ return std::string("hello"); });

    std::cout << f1.get() << "\n";   // 30
    std::cout << f2.get() << "\n";   // "hello"

    q.shutdown();
    worker.join();
}
    Type-erased storage. The queue stores packaged_task<void()>. When we wrap a packaged_task<R()> inside it, invoking the outer void-returning task still fulfils the inner promise because the wrapped callable is the original packaged_task itself.

    Resetting a packaged_task

    reset() constructs a new shared state, allowing you to invoke the same wrapped callable a second time and get a new future. 

    std::packaged_task<int(int)> task([](int x){ return x * x; });

auto f1 = task.get_future();
task(5);
std::cout << f1.get() << "\n";  // 25

task.reset();                     // new shared state
auto f2 = task.get_future();
task(7);
std::cout << f2.get() << "\n";  // 49

    Automatic Exception Capture

    If the callable throws, packaged_task catches the exception and stores it in the shared state automatically. The consumer receives it on get(). 

    #include <future>
#include <iostream>
#include <stdexcept>

int main() {
    std::packaged_task<int()> task([]{
        throw std::logic_error("bad input");
        return 0;
    });

    auto fut = task.get_future();
    task();

    try {
        fut.get();
    } catch (const std::logic_error& e) {
        std::cout << "Caught: " << e.what() << "\n";
    }
}

    packaged_task vs. promise

    Aspect std::promise std::packaged_task
    Fulfillment Manual set_value/set_exception Automatic on operator()
    Wraps a callable? No Yes
    Reusable? No (one shot) Yes, after reset()
    Move-only? Yes Yes
    Best for Ad-hoc signalling, custom logic Task queues, deferred execution

    When Not to Use packaged_task

    • Simple fire-and-forget. If you want the runtime to manage the thread, prefer std::async.
    • No return value needed. If you only need to run something on another thread with no result, a plain std::thread with a lambda suffices.
    • Multiple consumers. packaged_task yields a std::future (single consumer). If you need many readers, convert to shared_future.

    Complete Example: Parallel Map

    Apply a function to each element of a vector in parallel using packaged tasks and collect results via futures. 

    #include <future>
#include <thread>
#include <vector>
#include <iostream>
#include <cmath>

int main() {
    std::vector<double> input = {1.0, 4.0, 9.0, 16.0, 25.0};
    std::vector<std::future<double>> futures;
    std::vector<std::thread> threads;

    for (double x : input) {
        std::packaged_task<double()> task([x]{
            return std::sqrt(x);
        });
        futures.push_back(task.get_future());
        threads.emplace_back(std::move(task));
    }

    for (auto& f : futures) {
        std::cout << f.get() << " ";
    }
    std::cout << "\n";  // 1 2 3 4 5

    for (auto& t : threads) t.join();
}

    Key Takeaways

    • packaged_task wraps a callable and auto-fulfils a promise on invocation.
    • Always call get_future() before moving the task.
    • Use reset() to reuse a task with a fresh shared state.
    • Exceptions thrown by the callable are captured automatically.
    • Ideal building block for thread pools and task queues.
