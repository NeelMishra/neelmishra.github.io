    std::promise & std::future

    March 2026

    The One-Shot Channel

    When a thread computes a result that another thread needs, the simplest synchronisation primitive is the promise/future pair. A std::promise is the write end and a std::future is the read end of a single-use, thread-safe communication channel provided by <future>.

    The producer calls set_value() (or set_exception()) on the promise exactly once. The consumer calls get() on the future exactly once. The future blocks until the shared state is ready.

    One-shot rule. Both set_value and get may be invoked at most once. Violating this throws std::future_error.

    How the Value Flows

    Step through the diagram below to see a value travel from a promise in the producer thread to a future in the consumer thread.

    Basic Usage

    The canonical pattern: create a promise, extract a future, move the promise into a worker thread, and retrieve the result on the calling thread. 

    #include <future>
#include <thread>
#include <iostream>

int main() {
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();

    std::thread worker([p = std::move(prom)]() mutable {
        // Simulate work
        int result = 6 * 7;
        p.set_value(result);          // fulfil the promise
    });

    int answer = fut.get();            // blocks until value ready
    std::cout << "Answer: " << answer << "\n";

    worker.join();
}

    get() blocks the calling thread until the shared state becomes ready. Once the value is retrieved the future is no longer valid: calling get() again throws std::future_error with error code future_errc::no_state.

    set_value and the Shared State

    promise::set_value() stores a value in the shared state and makes the state ready. For promise<void>, call set_value() with no argument to signal completion. 

    std::promise<void> gate;
std::future<void> signal = gate.get_future();

std::thread t([&gate]{
    // ... do setup work ...
    gate.set_value();   // signal "ready"
});

signal.get();           // blocks until set_value is called
std::cout << "Setup complete\n";
t.join();
    Void promises. std::promise<void> is the idiomatic way to signal a one-time event across threads without transferring data, similar to a one-shot latch.

    set_exception

    If the producer encounters an error, it can propagate an exception to the consumer via the shared state. The consumer sees the exception re-thrown from get(). 

    #include <future>
#include <thread>
#include <iostream>
#include <stdexcept>

int main() {
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();

    std::thread worker([p = std::move(prom)]() mutable {
        try {
            throw std::runtime_error("disk full");
        } catch (...) {
            p.set_exception(std::current_exception());
        }
    });

    try {
        int val = fut.get();
    } catch (const std::exception& e) {
        std::cout << "Caught: " << e.what() << "\n";
    }

    worker.join();
}

    set_exception takes a std::exception_ptr, typically obtained from std::current_exception() inside a catch block.

    Broken Promises

    If a std::promise is destroyed without ever calling set_value() or set_exception(), the runtime stores a std::future_error with error code future_errc::broken_promise in the shared state. The consumer sees this as an exception from get(). 

    #include <future>
#include <iostream>

int main() {
    std::future<int> fut;

    {
        std::promise<int> prom;
        fut = prom.get_future();
        // prom destroyed here without calling set_value
    }

    try {
        int val = fut.get();
    } catch (const std::future_error& e) {
        std::cout << "Error: " << e.what() << "\n";
        std::cout << "Code:  "
                  << (e.code() == std::future_errc::broken_promise
                        ? "broken_promise" : "other")
                  << "\n";
    }
}

    Output:

    Error: broken promise
Code:  broken_promise
    Defensive pattern. Always ensure that every code path through your producer either calls set_value or set_exception. A try/catch wrapper that calls set_exception(std::current_exception()) in the catch block prevents accidental broken promises.

    The get() Exactly Once Rule

    std::future::get() moves the result out of the shared state. After the first call, the future is in a "no shared state" condition. A second call is undefined behaviour caught by a future_error. 

    std::promise<std::string> prom;
std::future<std::string> fut = prom.get_future();
prom.set_value("hello");

std::string s1 = fut.get();    // OK, s1 == "hello"
// std::string s2 = fut.get(); // throws future_error (no_state)

    To allow multiple consumers to read the same result, convert to a std::shared_future (covered in its own article in this series).

    Non-Blocking Checks: wait_for and wait_until

    Instead of blocking unconditionally, you can poll or time-box the wait. 

    #include <future>
#include <thread>
#include <chrono>
#include <iostream>

int main() {
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();

    std::thread worker([p = std::move(prom)]() mutable {
        std::this_thread::sleep_for(std::chrono::milliseconds(300));
        p.set_value(99);
    });

    // Poll with timeout
    while (true) {
        auto status = fut.wait_for(std::chrono::milliseconds(50));
        if (status == std::future_status::ready) {
            std::cout << "Result: " << fut.get() << "\n";
            break;
        }
        if (status == std::future_status::timeout) {
            std::cout << "Still waiting...\n";
        }
    }

    worker.join();
}

    wait_for returns one of three values: future_status::ready, future_status::timeout, or future_status::deferred (the last only applies to deferred tasks from std::async).

    Passing References Through Promises

    std::promise<T&> lets you share a reference to an existing object rather than copying the value. 

    #include <future>
#include <thread>
#include <iostream>
#include <string>

int main() {
    std::string data = "original";

    std::promise<std::string&> prom;
    std::future<std::string&> fut = prom.get_future();

    std::thread t([&prom, &data]{
        data += " + modified";
        prom.set_value(data);
    });

    std::string& ref = fut.get();
    std::cout << ref << "\n";   // "original + modified"

    t.join();
}
    Lifetime caution. When using reference promises, the referenced object must outlive every consumer. This is your responsibility; the shared state does not extend the lifetime of the referent.

    Complete Example: Parallel Pipeline

    Below, two stages of a pipeline communicate via promise/future pairs. Stage 1 downloads data (simulated), Stage 2 processes it. 

    #include <future>
#include <thread>
#include <iostream>
#include <vector>
#include <numeric>

int main() {
    // Stage 1 -> Stage 2 channel
    std::promise<std::vector<int>> prom_data;
    std::future<std::vector<int>>  fut_data = prom_data.get_future();

    // Stage 2 -> main channel
    std::promise<long long> prom_result;
    std::future<long long>  fut_result = prom_result.get_future();

    // Stage 1: produce data
    std::thread stage1([p = std::move(prom_data)]() mutable {
        std::vector<int> data(1000);
        std::iota(data.begin(), data.end(), 1);
        p.set_value(std::move(data));
    });

    // Stage 2: consume data, produce result
    std::thread stage2(
        [fd = std::move(fut_data), pr = std::move(prom_result)]() mutable {
            std::vector<int> data = fd.get();
            long long sum = 0;
            for (int v : data) sum += v;
            pr.set_value(sum);
        }
    );

    long long total = fut_result.get();
    std::cout << "Pipeline result: " << total << "\n"; // 500500

    stage1.join();
    stage2.join();
}

    Common future_error Codes

    Error code When thrown
    broken_promise Promise destroyed without setting a value
    future_already_retrieved get_future() called more than once
    promise_already_satisfied set_value/set_exception called twice
    no_state Operating on a future with no shared state (e.g. after get())

    Key Takeaways

    • std::promise / std::future form a one-shot, thread-safe channel.
    • The producer calls set_value or set_exception exactly once.
    • The consumer calls get() exactly once; subsequent calls throw.
    • If the promise is destroyed unfulfilled, get() throws broken_promise.
    • Use wait_for/wait_until for non-blocking polling.
    • For multiple readers, convert to std::shared_future.
