    Design: Elevator System

    March 2026 · 28 min read

    The elevator system is a classic LLD problem that tests state management, strategy pattern, and the ability to handle concurrent requests. Unlike the parking lot, this problem has a real-time component — the elevator is always doing something (moving, waiting, opening doors).

    1Requirements

    Functional

    • Building with N floors and M elevators
    • Users press Up/Down buttons on each floor (external request)
    • Users press floor number inside the elevator (internal request)
    • Elevators have a maximum capacity (weight or count)
    • Door opens/closes at each stop
    • Display current floor and direction

    Non-Functional

    • Minimize average wait time
    • Multiple elevators should be coordinated (not all rush to the same floor)
    • Extensible scheduling algorithm

    2Elevator State Machine

    Each elevator is a state machine with three states:

    IDLE ↔ MOVING_UP ↔ MOVING_DOWN
    StateBehaviorTransitions
    IDLEDoors closed, waiting at a floorReceives request → MOVING_UP or MOVING_DOWN
    MOVING_UPTraveling upward, stops at requested floorsNo more up requests → MOVING_DOWN or IDLE
    MOVING_DOWNTraveling downward, stops at requested floorsNo more down requests → MOVING_UP or IDLE

    3Scheduling Strategies

    How do you decide which elevator handles which request? This is the heart of the problem — and a perfect place for the Strategy pattern.

    StrategyDescriptionProsCons
    FCFSFirst come, first servedSimpleInefficient вҖ” may zig-zag
    SCAN (Elevator)Go in one direction until end, then reverseFair, no starvationEnd floors wait longer
    LOOKLike SCAN but reverses when no more requests in current directionMore efficient than SCANSlightly more complex
    NearestAssign to the closest idle/same-direction elevatorFast responseCan overload one elevator

    5Implementation

    Enums and Request

    enum class Direction { UP, DOWN, IDLE };

enum class ElevatorState { IDLE, MOVING_UP, MOVING_DOWN };

struct Request {
    int floor;
    Direction direction;  // for external requests; IDLE for internal
};

    Elevator SRP

    class Elevator {
    int id;
    int currentFloor;
    ElevatorState state;
    set<int> upStops;    // floors to stop at going up
    set<int> downStops;  // floors to stop at going down
    int capacity;
    int currentLoad;

public:
    Elevator(int id, int capacity = 10)
        : id(id), currentFloor(0), state(ElevatorState::IDLE),
          capacity(capacity), currentLoad(0) {}

    int getId() const { return id; }
    int getCurrentFloor() const { return currentFloor; }
    ElevatorState getState() const { return state; }
    bool isIdle() const { return state == ElevatorState::IDLE; }

    bool hasCapacity() const { return currentLoad < capacity; }

    void addStop(int floor) {
        if (floor > currentFloor || (floor == currentFloor && state == ElevatorState::MOVING_UP))
            upStops.insert(floor);
        else if (floor < currentFloor || (floor == currentFloor && state == ElevatorState::MOVING_DOWN))
            downStops.insert(floor);
        else if (state == ElevatorState::IDLE) {
            if (floor > currentFloor) upStops.insert(floor);
            else if (floor < currentFloor) downStops.insert(floor);
        }
    }

    // Called each "tick" вҖ” move one floor
    void step() {
        switch (state) {
        case ElevatorState::IDLE:
            if (!upStops.empty()) state = ElevatorState::MOVING_UP;
            else if (!downStops.empty()) state = ElevatorState::MOVING_DOWN;
            break;

        case ElevatorState::MOVING_UP:
            currentFloor++;
            if (upStops.count(currentFloor)) {
                upStops.erase(currentFloor);
                openDoors();
            }
            if (upStops.empty()) {
                state = downStops.empty() ? ElevatorState::IDLE : ElevatorState::MOVING_DOWN;
            }
            break;

        case ElevatorState::MOVING_DOWN:
            currentFloor--;
            if (downStops.count(currentFloor)) {
                downStops.erase(currentFloor);
                openDoors();
            }
            if (downStops.empty()) {
                state = upStops.empty() ? ElevatorState::IDLE : ElevatorState::MOVING_UP;
            }
            break;
        }
    }

    void openDoors() {
        // In real system: unlock doors, wait, close
        cout << "Elevator " << id << " opening doors at floor " << currentFloor << endl;
    }

    int distanceTo(int floor) const { return abs(currentFloor - floor); }

    bool isMovingToward(int floor) const {
        if (state == ElevatorState::MOVING_UP && floor >= currentFloor) return true;
        if (state == ElevatorState::MOVING_DOWN && floor <= currentFloor) return true;
        return false;
    }
};

    Dispatch Strategy OCP DIP

    class DispatchStrategy {
public:
    virtual ~DispatchStrategy() = default;
    virtual Elevator* select(const Request& req,
                             vector<Elevator>& elevators) = 0;
};

// Assign to nearest idle or same-direction elevator
class NearestStrategy : public DispatchStrategy {
public:
    Elevator* select(const Request& req,
                     vector<Elevator>& elevators) override {
        Elevator* best = nullptr;
        int bestDist = INT_MAX;

        for (auto& e : elevators) {
            if (!e.hasCapacity()) continue;

            int dist = e.distanceTo(req.floor);

            // Prefer: idle elevators, then same-direction, then others
            if (e.isIdle()) {
                if (dist < bestDist) { bestDist = dist; best = &e; }
            } else if (e.isMovingToward(req.floor)) {
                if (dist < bestDist) { bestDist = dist; best = &e; }
            }
        }

        // Fallback: any elevator with smallest distance
        if (!best) {
            for (auto& e : elevators) {
                int dist = e.distanceTo(req.floor);
                if (dist < bestDist) { bestDist = dist; best = &e; }
            }
        }
        return best;
    }
};

// LOOK: continue in one direction until no more requests, then reverse
class LOOKStrategy : public DispatchStrategy {
public:
    Elevator* select(const Request& req,
                     vector<Elevator>& elevators) override {
        Elevator* best = nullptr;
        int bestScore = INT_MAX;

        for (auto& e : elevators) {
            if (!e.hasCapacity()) continue;
            int dist = e.distanceTo(req.floor);
            int score = dist;

            // Penalize elevators moving away
            if (!e.isIdle() && !e.isMovingToward(req.floor))
                score += 1000;  // large penalty

            if (score < bestScore) { bestScore = score; best = &e; }
        }
        return best;
    }
};

    Building (Coordinator)

    class Building {
    int numFloors;
    vector<Elevator> elevators;
    unique_ptr<DispatchStrategy> strategy;

public:
    Building(int floors, int numElevators,
             unique_ptr<DispatchStrategy> strat = make_unique<NearestStrategy>())
        : numFloors(floors), strategy(move(strat)) {
        for (int i = 0; i < numElevators; i++)
            elevators.emplace_back(i + 1);
    }

    void setStrategy(unique_ptr<DispatchStrategy> strat) {
        strategy = move(strat);
    }

    // External request: someone presses Up/Down on a floor
    void requestElevator(int floor, Direction dir) {
        Request req{floor, dir};
        Elevator* e = strategy->select(req, elevators);
        if (e) {
            e->addStop(floor);
            cout << "Dispatched elevator " << e->getId()
                 << " to floor " << floor << endl;
        }
    }

    // Internal request: someone inside elevator presses a floor button
    void pressFloor(int elevatorId, int floor) {
        for (auto& e : elevators) {
            if (e.getId() == elevatorId) {
                e.addStop(floor);
                break;
            }
        }
    }

    // Simulate one time step
    void tick() {
        for (auto& e : elevators) e.step();
    }

    void display() const {
        for (const auto& e : elevators) {
            cout << "Elevator " << e.getId()
                 << " @ floor " << e.getCurrentFloor()
                 << " [" << (e.getState() == ElevatorState::IDLE ? "IDLE" :
                            e.getState() == ElevatorState::MOVING_UP ? "UP" : "DOWN")
                 << "]" << endl;
        }
    }
};

    6Putting It Together

    int main() {
    Building building(10, 3);  // 10 floors, 3 elevators

    // Someone on floor 5 presses Up
    building.requestElevator(5, Direction::UP);

    // Someone on floor 2 presses Down
    building.requestElevator(2, Direction::DOWN);

    // Simulate 5 steps
    for (int t = 0; t < 5; t++) {
        cout << "--- Tick " << t + 1 << " ---" << endl;
        building.tick();
        building.display();
    }

    // Person inside elevator 1 presses floor 8
    building.pressFloor(1, 8);

    // Switch to LOOK scheduling at peak hours
    building.setStrategy(make_unique<LOOKStrategy>());
}

    7Design Review

    PrincipleApplication
    SRPElevator manages its own state. DispatchStrategy handles selection. Building coordinates.
    OCPAdd LOOKStrategy, SCANStrategy without changing Building or Elevator.
    DIPBuilding depends on DispatchStrategy interface, not NearestStrategy.
    State patternElevator is a state machine вҖ” could be formalized with State pattern for complex transitions.
    Strategy patternScheduling algorithm is swappable at runtime.

    Common Extensions

    • Emergency stop: Add an EMERGENCY state that halts elevator and opens doors.
    • VIP elevator: Certain elevators serve only specific floors (subclass or config).
    • Weight-based capacity: Track cumulative weight, not just passenger count.
    • Maintenance mode: Take an elevator offline, redistribute its requests.
    • Peak-hour scheduling: Switch strategy based on time of day (Building already supports this via setStrategy).
