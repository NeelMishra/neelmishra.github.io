    Systems of Linear Equations

    July 2026 · 25 min read

    Problem Setup

    Every system of linear equations can be written as Ax = b, where A is an m × n coefficient matrix, x is the unknown vector of length n, and b is the target vector of length m. The three fundamental questions are: does a solution exist, is it unique, and how do we find it?

    The shape of A tells us what to expect:

    • Square system (m = n): The classic case. If A is non-singular (det ≠ 0), there is exactly one solution.
    • Overdetermined (m > n): More equations than unknowns. Generically no exact solution exists; in CP these usually arise from redundant constraints.
    • Underdetermined (m < n): Fewer equations than unknowns. If a solution exists, infinitely many solutions exist (the solution space has free variables).

    The augmented matrix [A | b] packages the entire system into a single matrix that we can row-reduce:

    // Augmented matrix for 2x + 3y = 8, x - y = 1
// [ 2  3 | 8 ]
// [ 1 -1 | 1 ]

    We will always work with the augmented matrix: every row operation on it corresponds to a valid algebraic manipulation of the original equations.

    Solving with Gaussian Elimination

    Gaussian elimination transforms the augmented matrix into row echelon form (REF) via three elementary row operations: swap two rows, multiply a row by a non-zero scalar, and add a multiple of one row to another. Back substitution then extracts the solution.

    Algorithm Outline

    1. For each column (variable), find a row with a non-zero entry in that column (the pivot). Swap it to the current row.
    2. Eliminate all entries below the pivot by subtracting appropriate multiples of the pivot row.
    3. After all columns are processed, use back substitution starting from the last pivot row upward.
    Partial pivoting: Always swap in the row with the largest absolute value in the current column. This reduces floating-point error dramatically. For integer / modular problems, pick any non-zero entry.

    C++ Implementation (Floating-Point)

    #include <bits/stdc++.h>
using namespace std;

const double EPS = 1e-9;

// Returns the number of solutions: 0, 1, or 2 (infinity).
// If unique, x[] holds the solution.
int gauss(vector<vector<double>>& a, vector<double>& x) {
    int m = a.size(), n = (int)a[0].size() - 1;
    vector<int> pivot_col(m, -1);
    int col = 0, row = 0;

    // Forward elimination
    for (; col < n && row < m; col++) {
        // Partial pivoting: find best row
        int best = row;
        for (int i = row + 1; i < m; i++)
            if (abs(a[i][col]) > abs(a[best][col]))
                best = i;
        if (abs(a[best][col]) < EPS) continue;  // skip free column
        swap(a[row], a[best]);
        pivot_col[row] = col;

        // Eliminate below
        for (int i = row + 1; i < m; i++) {
            double factor = a[i][col] / a[row][col];
            for (int j = col; j <= n; j++)
                a[i][j] -= factor * a[row][j];
        }
        row++;
    }

    // Check for inconsistency: 0 = non-zero
    for (int i = row; i < m; i++)
        if (abs(a[i][n]) > EPS) return 0;

    // Back substitution
    x.assign(n, 0);
    for (int i = row - 1; i >= 0; i--) {
        int pc = pivot_col[i];
        x[pc] = a[i][n];
        for (int j = pc + 1; j < n; j++)
            x[pc] -= a[i][j] * x[j];
        x[pc] /= a[i][pc];
    }

    return (row == n) ? 1 : 2;  // 1 = unique, 2 = infinite
}

    Time complexity: O(min(m, n) · m · n), which is O(n³) for square systems. This is fast enough for n ≤ 500 in most CP judges.

    Free Variables and Solution Space

    After Gaussian elimination, each column either contains a pivot or it doesn't. Columns without pivots correspond to free variables — they can take any value, and the pivot variables are determined in terms of them.

    Identifying Free vs Pivot Variables

    // After forward elimination, determine which variables are free
vector<bool> is_pivot(n, false);
for (int i = 0; i < row; i++)
    if (pivot_col[i] != -1)
        is_pivot[pivot_col[i]] = true;

// Free variables: is_pivot[j] == false
int num_free = count(is_pivot.begin(), is_pivot.end(), false);
// Dimension of solution space = num_free

    Parametric Solutions

    When free variables exist, the general solution is:

    x = xparticular + t1v1 + t2v2 + …

    where each vi is a basis vector of the null space (one per free variable), and ti ranges over all reals (or all elements of the field). To build each vi: set the i-th free variable to 1, all other free variables to 0, and solve for the pivot variables via back substitution.

    Counting Solutions

    rank(A)rank([A|b])Solutions
    rr + 1None (inconsistent)
    r = nrExactly one
    r < nrInfinitely many (n − r free variables)

    No Solution vs Infinite Solutions

    Detecting these cases is critical in CP. After forward elimination:

    Inconsistency Detection

    A row of the form [0 0 … 0 | c] with c ≠ 0 means the system encodes the equation 0 = c, which is impossible. The system has no solution.

    // Check after forward elimination
bool inconsistent = false;
for (int i = row; i < m; i++) {
    if (abs(a[i][n]) > EPS) {
        inconsistent = true;
        break;
    }
}
if (inconsistent) cout << "No solution\n";

    Rank Conditions

    Let r = number of pivots found during elimination. Then:

    • r < rank([A|b]): No solution.
    • r = n: Unique solution (all variables are pivot variables).
    • r < n and consistent: Infinite solutions with (n − r) degrees of freedom.

    Complete Status Reporter

    enum SolveStatus { NO_SOLUTION, UNIQUE, INFINITE };

SolveStatus classify_system(int pivots, int n, bool consistent) {
    if (!consistent) return NO_SOLUTION;
    return (pivots == n) ? UNIQUE : INFINITE;
}

// Usage after gauss elimination:
// int pivots = row;  (number of pivots found)

    Modular Systems

    Many CP problems work modulo a prime p. Over Zp, every non-zero element has a multiplicative inverse (computed via Fermat's little theorem or the extended Euclidean algorithm), so Gaussian elimination works exactly — no floating-point issues.

    Modular Inverse

    long long mod_pow(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long mod_inv(long long a, long long mod) {
    return mod_pow(a, mod - 2, mod);  // Fermat's little theorem
}

    Gaussian Elimination mod p

    const long long MOD = 998244353;

// Returns 0 (no solution), 1 (unique), 2 (infinite).
// x[] holds solution when unique.
int gauss_mod(vector<vector<long long>>& a, vector<long long>& x) {
    int m = a.size(), n = (int)a[0].size() - 1;
    vector<int> pivot_col(m, -1);
    int col = 0, row = 0;

    for (; col < n && row < m; col++) {
        // Find a non-zero pivot
        int piv = -1;
        for (int i = row; i < m; i++)
            if (a[i][col] != 0) { piv = i; break; }
        if (piv == -1) continue;
        swap(a[row], a[piv]);
        pivot_col[row] = col;

        // Normalize pivot row
        long long inv = mod_inv(a[row][col], MOD);
        for (int j = col; j <= n; j++)
            a[row][j] = a[row][j] * inv % MOD;

        // Eliminate below
        for (int i = row + 1; i < m; i++) {
            if (a[i][col] == 0) continue;
            long long factor = a[i][col];
            for (int j = col; j <= n; j++)
                a[i][j] = (a[i][j] - factor * a[row][j] % MOD + MOD) % MOD;
        }
        row++;
    }

    // Check inconsistency
    for (int i = row; i < m; i++)
        if (a[i][n] != 0) return 0;

    // Back substitution
    x.assign(n, 0);
    for (int i = row - 1; i >= 0; i--) {
        int pc = pivot_col[i];
        x[pc] = a[i][n];
        for (int j = pc + 1; j < n; j++)
            x[pc] = (x[pc] - a[i][j] * x[j] % MOD + MOD) % MOD;
    }

    return (row == n) ? 1 : 2;
}
    Composite moduli: If the modulus is not prime, you cannot always compute inverses. For composite moduli, use the extended GCD and check that gcd(pivot, mod) = 1 before dividing. If the gcd divides the target, divide everything through; otherwise the system has no solution mod that modulus.

    Practical Solver Template

    Here is a complete, contest-ready solver that handles all three cases and reports status. It uses reduced row echelon form (RREF) for cleaner extraction of the solution and null space.

    #include <bits/stdc++.h>
using namespace std;
const double EPS = 1e-9;

struct GaussResult {
    int status;              // 0=no solution, 1=unique, 2=infinite
    vector<double> solution; // particular solution (if solvable)
    int rank;                // rank of coefficient matrix
    int free_vars;           // number of free variables
};

GaussResult solve(vector<vector<double>> a) {
    int m = a.size(), n = (int)a[0].size() - 1;
    vector<int> pcol(m, -1);
    int row = 0;

    // Forward elimination with partial pivoting
    for (int col = 0; col < n && row < m; col++) {
        int best = row;
        for (int i = row + 1; i < m; i++)
            if (abs(a[i][col]) > abs(a[best][col]))
                best = i;
        if (abs(a[best][col]) < EPS) continue;
        swap(a[row], a[best]);
        pcol[row] = col;

        // Normalize pivot row
        double piv = a[row][col];
        for (int j = col; j <= n; j++)
            a[row][j] /= piv;

        // Eliminate all other rows (full RREF)
        for (int i = 0; i < m; i++) {
            if (i == row || abs(a[i][col]) < EPS) continue;
            double f = a[i][col];
            for (int j = col; j <= n; j++)
                a[i][j] -= f * a[row][j];
        }
        row++;
    }

    // Check inconsistency
    for (int i = row; i < m; i++)
        if (abs(a[i][n]) > EPS)
            return {0, {}, row, n - row};

    // Extract particular solution
    vector<double> x(n, 0);
    for (int i = 0; i < row; i++)
        x[pcol[i]] = a[i][n];

    int status = (row == n) ? 1 : 2;
    return {status, x, row, n - row};
}

int main() {
    int m, n;
    cin >> m >> n;
    vector<vector<double>> a(m, vector<double>(n + 1));
    for (int i = 0; i < m; i++)
        for (int j = 0; j <= n; j++)
            cin >> a[i][j];

    auto res = solve(a);

    if (res.status == 0) {
        cout << "No solution\n";
    } else if (res.status == 1) {
        cout << "Unique solution:\n";
        for (int i = 0; i < n; i++)
            cout << fixed << setprecision(6) << res.solution[i] << " \n"[i==n-1];
    } else {
        cout << "Infinite solutions (rank=" << res.rank
             << ", free=" << res.free_vars << ")\n";
        cout << "Particular solution:\n";
        for (int i = 0; i < n; i++)
            cout << fixed << setprecision(6) << res.solution[i] << " \n"[i==n-1];
    }
}

    Usage Notes

    • Input format: first line is m n, then m rows of n + 1 values (the augmented matrix row by row).
    • For modular problems, swap the solver internals to gauss_mod from the previous section.
    • The RREF approach simplifies extraction: after elimination, the particular solution can be read directly from the augmented column.
    • For problems needing the null space basis, iterate over free columns and solve with the corresponding free variable set to 1.

    CP Applications

    Network Flow Setup

    Many network problems reduce to linear systems. At every node in a flow network, flow conservation gives an equation: sum of inflows equals sum of outflows. The unknowns are edge flows. If the graph has V vertices and E edges, you get V equations in E unknowns (one equation is redundant, so the rank is at most V − 1).

    // Build flow conservation system for a graph
// edges[i] = {from, to}, with unknown flow f_i
// For each node v: sum(f_i for edges into v) - sum(f_i for edges out of v) = demand[v]
void build_flow_system(int V, vector<pair<int,int>>& edges,
                       vector<double>& demand,
                       vector<vector<double>>& augmented) {
    int E = edges.size();
    augmented.assign(V, vector<double>(E + 1, 0));
    for (int j = 0; j < E; j++) {
        auto [u, v] = edges[j];
        augmented[u][j] -= 1;  // outflow from u
        augmented[v][j] += 1;  // inflow to v
    }
    for (int i = 0; i < V; i++)
        augmented[i][E] = demand[i];
}

    Chemical Equation Balancing

    Balancing a chemical equation is a system of linear equations: for each element, the total count on the left must equal the total count on the right. The unknowns are stoichiometric coefficients. The system is always homogeneous (b = 0), and we seek the smallest positive integer solution — which is found from the null space.

    // Example: balancing aFe + bO2 -> cFe2O3
// Element Fe: a = 2c      =>  a - 2c = 0
// Element O:  2b = 3c     =>  2b - 3c = 0
// System matrix (homogeneous):
// [ 1  0 -2 ]
// [ 0  2 -3 ]
// Null space basis: (2, 3/2, 1) -> scale to integers: (4, 3, 2)
// Answer: 4Fe + 3O2 -> 2Fe2O3

    Constraint Systems in CP

    Several classic problems encode constraints as linear equations:

    • Lights-out puzzles: Each button toggles a set of lights. Modeled as a system over GF(2) (XOR equations). Use Gaussian elimination mod 2.
    • Grid coloring: Constraints on row/column sums can be expressed as linear systems.
    • Difference constraints: Given xi − xj = d, build a system and check consistency. (Also solvable with union-find for special cases.)
    XOR systems (GF(2)): Replace addition with XOR and multiplication with AND. The Gaussian elimination code simplifies considerably — no need for division, and all arithmetic is bitwise. This handles lights-out, toggling puzzles, and linear recurrence over GF(2). 
    // Gaussian elimination over GF(2) using bitsets
// a[i] is a bitset representing row i of the augmented matrix
// Columns 0..n-1 are variables, column n is the RHS
int gauss_xor(vector<bitset<256>>& a, int m, int n, vector<int>& x) {
    vector<int> pcol(m, -1);
    int row = 0;
    for (int col = 0; col < n && row < m; col++) {
        int piv = -1;
        for (int i = row; i < m; i++)
            if (a[i][col]) { piv = i; break; }
        if (piv == -1) continue;
        swap(a[row], a[piv]);
        pcol[row] = col;
        for (int i = 0; i < m; i++)
            if (i != row && a[i][col])
                a[i] ^= a[row];
        row++;
    }
    for (int i = row; i < m; i++)
        if (a[i][n]) return 0;  // inconsistent
    x.assign(n, 0);
    for (int i = 0; i < row; i++)
        x[pcol[i]] = a[i][n] ? 1 : 0;
    return (row == n) ? 1 : 2;
}
