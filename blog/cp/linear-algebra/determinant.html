    Determinants

    June 2026 · 25 min read

    What Is a Determinant?

    The determinant is a scalar value associated with every square matrix. Geometrically it measures the signed volume of the parallelepiped spanned by the matrix’s column (or row) vectors. A zero determinant means the vectors are linearly dependent and the matrix is singular.

    2×2 Case

    For a 2×2 matrix the determinant is:

    | a  b |
| c  d |  =  a*d - b*c

    This equals the signed area of the parallelogram formed by the two row vectors (a,b) and (c,d).

    3×3 Case (Sarrus’ Rule)

    For a 3×3 matrix, the determinant expands as:

    det(A) = a(ei - fh) - b(di - fg) + c(dh - eg)

where A = | a  b  c |
          | d  e  f |
          | g  h  i |

    This is a special case of cofactor expansion along the first row and is sometimes called Sarrus’ rule. Note that Sarrus’ rule does not generalize to larger matrices.

    Key intuition. If you scale one row by factor k, the determinant scales by k. The determinant is an alternating multilinear function of the rows.

    Cofactor Expansion

    The Laplace expansion defines the determinant recursively. Pick any row i (or column j) and expand:

    det(A) = ∑ over j : (-1)^(i+j) * A[i][j] * det(M_ij)

    where M_ij is the (n−1)×(n−1) matrix obtained by deleting row i and column j. The factor (-1)^(i+j) · det(M_ij) is the cofactor C_ij.

    Complexity

    A naïve recursive implementation based on cofactor expansion runs in O(n · n!) time because it spawns n subproblems of size n−1 at each level. This is acceptable only for tiny matrices (n ≤ 8 or so).

    C++ Implementation — Naïve Cofactor Expansion

    // O(n * n!) cofactor expansion — only for small n
typedef long long ll;
typedef vector<vector<ll>> Mat;

ll cofactor_det(const Mat& A) {
    int n = A.size();
    if (n == 1) return A[0][0];
    if (n == 2) return A[0][0]*A[1][1] - A[0][1]*A[1][0];

    ll det = 0;
    for (int j = 0; j < n; j++) {
        // build minor M_{0,j}
        Mat minor(n-1, vector<ll>(n-1));
        for (int r = 1; r < n; r++) {
            int col = 0;
            for (int c = 0; c < n; c++) {
                if (c == j) continue;
                minor[r-1][col++] = A[r][c];
            }
        }
        ll sign = (j % 2 == 0) ? 1 : -1;
        det += sign * A[0][j] * cofactor_det(minor);
    }
    return det;
}

    Expanding along a row with many zeros can prune branches, but worst-case complexity remains factorial. For competitive programming we almost always want the Gaussian method instead.

    Gaussian Elimination Method

    Row-reducing a matrix to upper-triangular form lets us read off the determinant as the product of the diagonal entries. The key facts:

    • Swapping two rows multiplies the determinant by −1.
    • Scaling a row by k multiplies the determinant by k.
    • Adding a multiple of one row to another does not change the determinant.

    After O(n³) elimination the matrix is upper-triangular and det = (−1)^swaps · ∏ A[i][i].

    Algorithm Outline

    1. For each column j from 0 to n−1:
    2.  Find the pivot: the row r ≥ j with the largest absolute value in column j (partial pivoting).
    3.  If the pivot is zero, return det = 0.
    4.  Swap rows j and r if r ≠ j; increment swap counter.
    5.  Eliminate all entries below the pivot using row operations.

    C++ Implementation — Floating-Point

    // O(n^3) determinant via Gaussian elimination (doubles)
double gauss_det(vector<vector<double>> A) {
    int n = A.size();
    double det = 1.0;
    for (int col = 0; col < n; col++) {
        // partial pivoting
        int pivot = col;
        for (int row = col + 1; row < n; row++)
            if (fabs(A[row][col]) > fabs(A[pivot][col]))
                pivot = row;

        if (fabs(A[pivot][col]) < 1e-12) return 0.0;

        if (pivot != col) {
            swap(A[pivot], A[col]);
            det *= -1;
        }

        det *= A[col][col];

        for (int row = col + 1; row < n; row++) {
            double factor = A[row][col] / A[col][col];
            for (int k = col; k < n; k++)
                A[row][k] -= factor * A[col][k];
        }
    }
    return det;
}
    Partial pivoting is essential for numerical stability with floating-point. Without it, dividing by a near-zero pivot amplifies rounding errors dramatically.

    Properties of Determinants

    These properties appear constantly in proofs and problem-solving. Let A and B be n×n matrices.

    Row / Column Operations

    • Swap two rows (or columns): det flips sign.
    • Multiply a row by scalar k: det scales by k. Consequently det(kA) = kn det(A).
    • Add a multiple of one row to another: det is unchanged.
    • Two identical rows: det = 0.
    • Row of all zeros: det = 0.

    Fundamental Identities

    • Transpose: det(AT) = det(A). Row operations and column operations are interchangeable for determinant purposes.
    • Product formula: det(AB) = det(A) · det(B). This is one of the most powerful identities and holds for all square matrices.
    • Inverse: det(A−1) = 1 / det(A) when A is invertible.
    • Triangular matrix: det equals the product of diagonal entries. This is why Gaussian elimination works.
    • Block triangular: det([[A, B], [0, D]]) = det(A) · det(D).

    Less Obvious but Useful

    • Rank-1 update (Matrix Determinant Lemma): det(A + uvT) = (1 + vTA−1u) · det(A).
    • Cauchy–Binet formula: generalizes the product formula when the matrices are not square.
    • Vandermonde determinant: det(V) = ∏i<j(xj − xi) for the Vandermonde matrix. Appears in polynomial interpolation problems.

    Modular Determinant

    Many CP problems ask for the determinant modulo a prime p. Floating-point is useless here; we work entirely in Z/pZ. Since p is prime, every non-zero element has a modular inverse, so Gaussian elimination works perfectly.

    Key Differences from Floating-Point

    • Replace division by multiplication with the modular inverse: a / b ≡ a · bp−2 (mod p) via Fermat’s little theorem.
    • No need for partial pivoting for stability — just find any non-zero pivot.
    • All arithmetic is done modulo p.

    C++ Implementation — Determinant mod Prime

    // O(n^3) determinant modulo a prime p
typedef long long ll;

ll power(ll base, ll exp, ll mod) {
    ll result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

ll mod_inv(ll a, ll p) {
    return power(a % p + p, p - 2, p);
}

ll det_mod(vector<vector<ll>> A, ll p) {
    int n = A.size();
    ll det = 1;
    for (int col = 0; col < n; col++) {
        // find pivot
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (A[row][col] % p != 0) { pivot = row; break; }
        }
        if (pivot == -1) return 0;  // singular

        if (pivot != col) {
            swap(A[pivot], A[col]);
            det = det * (p - 1) % p;  // multiply by -1 mod p
        }

        det = det % p * (A[col][col] % p + p) % p;
        ll inv = mod_inv(A[col][col], p);

        for (int row = col + 1; row < n; row++) {
            ll factor = A[row][col] % p * inv % p;
            for (int k = col; k < n; k++) {
                A[row][k] = (A[row][k] - factor * A[col][k] % p + p * 2) % p;
            }
        }
    }
    return (det % p + p) % p;
}
    Composite modulus? If the modulus is not prime, you cannot always find inverses. One approach is to factor the modulus, compute determinants modulo each prime power, then combine via CRT. Alternatively, use fraction-free Gaussian elimination (Bareiss algorithm).

    Bareiss Algorithm (Integer Determinant, No Division)

    The Bareiss algorithm maintains exact integer arithmetic by performing “fraction-free” elimination. At each step it divides by a previous pivot, but this division is always exact (no remainder). This is useful when working modulo composite numbers or when you need the exact integer determinant.

    // Bareiss algorithm — exact integer determinant, O(n^3)
ll bareiss_det(vector<vector<ll>> A) {
    int n = A.size();
    int sign = 1;
    for (int col = 0; col < n; col++) {
        // find pivot
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (A[row][col] != 0) { pivot = row; break; }
        }
        if (pivot == -1) return 0;

        if (pivot != col) {
            swap(A[pivot], A[col]);
            sign = -sign;
        }

        for (int row = col + 1; row < n; row++) {
            for (int k = col + 1; k < n; k++) {
                A[row][k] = A[col][col] * A[row][k]
                           - A[row][col] * A[col][k];
                if (col > 0) A[row][k] /= A[col-1][col-1];
            }
            A[row][col] = 0;
        }
    }
    return sign * A[n-1][n-1];
}

    Applications in CP

    1. Area of a Polygon (Shoelace Formula)

    The signed area of a polygon with vertices (x0,y0), ..., (xn−1,yn−1) is:

    2A = ∑ det | x_i    y_i   |
             | x_{i+1} y_{i+1} |

   = ∑ (x_i * y_{i+1} - x_{i+1} * y_i)

    This is a sum of 2×2 determinants. The absolute value gives the area; the sign gives the orientation (CCW vs CW).

    // Shoelace formula — area of simple polygon
ll shoelace(vector<pair<ll,ll>>& pts) {
    int n = pts.size();
    ll area = 0;
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        area += pts[i].first * pts[j].second;
        area -= pts[j].first * pts[i].second;
    }
    return abs(area);  // 2 * actual area
}

    2. Collinearity Test

    Three points (x1,y1), (x2,y2), (x3,y3) are collinear if and only if:

    det | x1  y1  1 |
    | x2  y2  1 |  =  0
    | x3  y3  1 |

    Expanding gives the familiar cross-product formula: (x2−x1)(y3−y1) − (x3−x1)(y2−y1) = 0.

    3. Orientation Test

    The sign of the above 3×3 determinant tells you whether the triple is oriented counter-clockwise (+), clockwise (−), or collinear (0). This is the backbone of convex-hull algorithms and line-segment intersection tests.

    // Returns: +1 CCW, -1 CW, 0 collinear
int orientation(ll x1, ll y1, ll x2, ll y2, ll x3, ll y3) {
    ll cross = (x2 - x1) * (y3 - y1) - (x3 - x1) * (y2 - y1);
    if (cross > 0) return +1;
    if (cross < 0) return -1;
    return 0;
}

    4. System Solvability (Cramer’s Rule)

    A linear system Ax = b has a unique solution if and only if det(A) ≠ 0. Cramer’s rule gives:

    x_i = det(A_i) / det(A)

    where A_i is A with its i-th column replaced by b. This is O(n&sup4;) via cofactor expansion or O(n³) by precomputing the determinants via Gaussian elimination, so it’s mainly useful for small systems or theoretical arguments.

    5. Kirchhoff’s Matrix-Tree Theorem

    The number of spanning trees of a graph equals any cofactor of the Laplacian matrix L. The Laplacian is L = D − A where D is the degree matrix and A the adjacency matrix. Delete any one row and column from L and compute the determinant:

    // Count spanning trees of a graph via Kirchhoff's theorem
ll count_spanning_trees(vector<vector<int>>& adj, int n, ll mod) {
    // Build Laplacian
    vector<vector<ll>> L(n, vector<ll>(n, 0));
    for (int u = 0; u < n; u++) {
        for (int v : adj[u]) {
            L[u][u]++;
            L[u][v]--;
        }
    }
    // Delete last row and column
    vector<vector<ll>> M(n-1, vector<ll>(n-1));
    for (int i = 0; i < n-1; i++)
        for (int j = 0; j < n-1; j++)
            M[i][j] = (L[i][j] % mod + mod) % mod;

    return det_mod(M, mod);  // use our modular det
}
    Kirchhoff’s theorem is a favourite in CP for counting labelled spanning trees. Combined with modular determinant, it handles graphs up to n ≈ 2000 in O(n³).

    6. Volume of a Simplex

    The volume of an n-dimensional simplex with vertices v0, …, vn is:

    Volume = |det([v1-v0, v2-v0, ..., vn-v0])| / n!

    This generalizes the 2D area and 3D volume formulas, and occasionally appears in geometry problems.

    Common Pitfalls

    • Overflow: For integer Gaussian elimination, intermediate values can overflow 64-bit integers quickly. Use __int128 or switch to modular arithmetic.
    • Floating-point: Forgetting partial pivoting leads to catastrophic cancellation. Always pivot on the largest element.
    • Sign tracking: Every row swap flips the sign. Forgetting to track this is a classic bug.
    • Modular arithmetic: Ensure all intermediate values are taken mod p, especially after subtraction (add p before taking mod to avoid negatives).
    • Bareiss division: The exact division A[row][k] / A[col-1][col-1] must truly be exact. If the matrix has been modified incorrectly, this will produce wrong results silently.
