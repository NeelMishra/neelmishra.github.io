    Gaussian Elimination — Row Reduction, RREF & Pivoting

    July 2026 · 22 min read

    Gaussian elimination is the workhorse algorithm for solving systems of linear equations. In competitive programming it appears whenever you need to find the rank of a matrix, determine whether a system has a solution, compute determinants, or build an XOR basis. This post covers the algorithm from scratch, with full C++ implementations for both floating-point and modular-arithmetic variants.

    What Is Gaussian Elimination?

    Given a system of n linear equations in m unknowns, we can write it as an augmented matrix [A | b] where A is the coefficient matrix and b is the right-hand side vector. Gaussian elimination transforms this matrix into an upper-triangular (or staircase) form using three elementary row operations:

    • Swap two rows.
    • Scale a row by a nonzero constant.
    • Add a scalar multiple of one row to another.

    These operations do not change the solution set. Once the matrix is in row echelon form, we can read off solutions via back substitution.

    A Small Example

    Consider the system:

      2x + 1y - 1z =   8
 -3x - 1y + 2z = -11
 -2x + 1y + 2z =  -3

    We write the augmented matrix and apply row operations. First, use row 1 to eliminate x from rows 2 and 3. Then use the new row 2 to eliminate y from row 3. The result is upper-triangular, and back substitution gives x = 2, y = 3, z = -1.

    Key insight. The algorithm always terminates in O(n²m) operations for an n × m matrix. Each column contributes at most one pivot, and for each pivot we do O(nm) work to eliminate entries below (and above, for RREF).

    Row Echelon Form and RREF

    After the forward elimination phase, the matrix is in Row Echelon Form (REF). A matrix is in REF when:

    • All zero rows are at the bottom.
    • The leading (leftmost nonzero) entry of each nonzero row is strictly to the right of the leading entry of the row above it.

    Reduced Row Echelon Form (RREF) additionally requires:

    • Each leading entry is 1 (a pivot).
    • Each pivot is the only nonzero entry in its column.

    Back Substitution vs RREF

    In many CP problems, reaching REF is enough — you can solve by back substitution starting from the last nonzero row. However, computing the full RREF is sometimes cleaner: it directly tells you which variables are free and gives you the general solution in one pass.

    The choice depends on context. For rank computation you only need REF; for expressing the full solution space, RREF is more convenient. Performance-wise, both are O(n²m).

    Pivots and Rank

    The number of pivots equals the rank of the matrix. If you are solving Ax = b, a unique solution exists if and only if rank(A) = rank([A|b]) = m (number of unknowns). If rank(A) < m but rank(A) = rank([A|b]), there are infinitely many solutions. Otherwise the system is inconsistent.

    Implementation in C++

    Below is a complete Gaussian elimination implementation with partial pivoting (choosing the row with the largest absolute value in the pivot column). This version works with double and is suitable for problems where exact integer arithmetic is not required. 

    #include <bits/stdc++.h>
using namespace std;

const double EPS = 1e-9;

// Gaussian elimination with partial pivoting.
// a is the augmented matrix [A | b] of size n x (m+1).
// Returns the rank. Solution (if unique) is in a[i][m].
int gauss(vector<vector<double>>& a) {
    int n = a.size(), m = (int)a[0].size() - 1;
    int rank = 0;

    for (int col = 0; col < m && rank < n; col++) {
        // Partial pivoting: find row with largest |a[r][col]|
        int best = rank;
        for (int r = rank + 1; r < n; r++) {
            if (abs(a[r][col]) > abs(a[best][col]))
                best = r;
        }
        if (abs(a[best][col]) < EPS) continue; // no pivot in this column

        swap(a[rank], a[best]);

        // Scale pivot row so that a[rank][col] = 1
        double pivot = a[rank][col];
        for (int j = col; j <= m; j++)
            a[rank][j] /= pivot;

        // Eliminate col from all other rows
        for (int r = 0; r < n; r++) {
            if (r == rank || abs(a[r][col]) < EPS) continue;
            double factor = a[r][col];
            for (int j = col; j <= m; j++)
                a[r][j] -= factor * a[rank][j];
        }
        rank++;
    }
    return rank;
}

int main() {
    int n, m;
    cin >> n >> m;
    // n equations, m unknowns — augmented matrix is n x (m+1)
    vector<vector<double>> a(n, vector<double>(m + 1));
    for (int i = 0; i < n; i++)
        for (int j = 0; j <= m; j++)
            cin >> a[i][j];

    int rank = gauss(a);

    // Check consistency
    for (int i = rank; i < n; i++) {
        if (abs(a[i][m]) > EPS) {
            cout << "No solution\n";
            return 0;
        }
    }
    if (rank < m) {
        cout << "Infinitely many solutions (rank = "
             << rank << ", unknowns = " << m << ")\n";
    } else {
        cout << fixed << setprecision(6);
        for (int i = 0; i < m; i++)
            cout << "x" << i << " = " << a[i][m] << "\n";
    }
    return 0;
}

    Walking Through the Code

    The outer loop iterates over columns. For each column, we find the best pivot row (partial pivoting), swap it into position, scale to make the pivot 1, then zero out the entire column in all other rows. Because we eliminate both above and below the pivot, we directly produce RREF — no separate back-substitution pass is needed.

    Floating-point pitfall. We compare against EPS = 1e-9 rather than exact zero. For problems with integer coefficients up to ~10⁹, using long double and EPS = 1e-12 is safer. Better yet, use the modular version below.

    Gaussian Elimination Over Integers (Modular)

    Many CP problems specify that the answer should be computed modulo a prime p. In this case, all arithmetic is exact — no floating-point issues. We replace division by the modular inverse (computed via Fermat's little theorem since p is prime). 

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;

const ll MOD = 998244353;

ll power(ll base, ll exp, ll mod) {
    ll result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

ll modinv(ll a, ll mod) {
    return power(a, mod - 2, mod);
}

// Gaussian elimination mod prime.
// a is the augmented matrix [A | b] of size n x (m+1).
// All entries are in [0, MOD). Returns the rank.
int gauss_mod(vector<vector<ll>>& a) {
    int n = a.size(), m = (int)a[0].size() - 1;
    int rank = 0;

    for (int col = 0; col < m && rank < n; col++) {
        // Find a nonzero pivot in this column
        int pivot_row = -1;
        for (int r = rank; r < n; r++) {
            if (a[r][col] != 0) { pivot_row = r; break; }
        }
        if (pivot_row == -1) continue;

        swap(a[rank], a[pivot_row]);

        // Scale pivot row
        ll inv = modinv(a[rank][col], MOD);
        for (int j = col; j <= m; j++)
            a[rank][j] = a[rank][j] * inv % MOD;

        // Eliminate col from all other rows
        for (int r = 0; r < n; r++) {
            if (r == rank || a[r][col] == 0) continue;
            ll factor = a[r][col];
            for (int j = col; j <= m; j++) {
                a[r][j] = (a[r][j] - factor * a[rank][j] % MOD + MOD) % MOD;
            }
        }
        rank++;
    }
    return rank;
}

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<ll>> a(n, vector<ll>(m + 1));
    for (int i = 0; i < n; i++)
        for (int j = 0; j <= m; j++) {
            cin >> a[i][j];
            a[i][j] = (a[i][j] % MOD + MOD) % MOD;
        }

    int rank = gauss_mod(a);

    if (rank < m) {
        cout << "Underdetermined (rank = " << rank << ")\n";
    } else {
        for (int i = 0; i < m; i++)
            cout << "x" << i << " = " << a[i][m] << "\n";
    }
    return 0;
}

    Why Modular Is Preferred in CP

    The modular version is exact — no epsilon comparisons, no precision loss. When a problem asks for the answer modulo a prime (as most do), this is the way to go. The only overhead is computing modular inverses, which is O(log p) per pivot via fast exponentiation — negligible compared to the O(n²m) total work.

    Watch the subtraction. When computing a[r][j] - factor * a[rank][j] mod MOD, the result can go negative. Always add MOD before taking the remainder: (x - y % MOD + MOD) % MOD.

    Partial vs Full Pivoting

    Partial pivoting selects, for each column, the row (from the current row downward) whose entry in that column has the largest absolute value. This is the standard technique for numerical stability with floating-point arithmetic.

    Why Pivoting Matters

    Without pivoting, dividing by a very small number amplifies rounding errors catastrophically. Consider the system: 

      0.0001 x + y = 1
       x + y = 2

    Naively using 0.0001 as the pivot, the multiplier is 10000, and rounding errors explode. Swapping the rows so that 1 (the larger value) is the pivot keeps the multiplier ≤ 1 and preserves accuracy.

    Full Pivoting

    Full (complete) pivoting considers all entries in the remaining submatrix and swaps both rows and columns to place the largest entry on the diagonal. This is more stable but requires tracking column permutations — the unknowns get reordered.

    In competitive programming, full pivoting is rarely needed. Partial pivoting suffices for virtually all floating-point problems, and for modular arithmetic, any nonzero pivot works (there are no rounding errors).

    Pivoting in the Modular Version

    For gauss_mod, we simply pick the first nonzero entry in the column. All nonzero elements in ℤₚ have inverses, so the choice does not affect correctness or numerical stability. However, for the floating-point version, always use at least partial pivoting.

    Complexity Analysis

    Let the matrix have n rows and m columns (before augmentation).

    • Time: O(n · m · min(n, m)). For a square n×n system this is O(n³). Each of the min(n, m) pivot steps does O(nm) work to eliminate one column.
    • Space: O(nm) for the matrix itself. The algorithm works in-place.

    Practical Considerations

    For n = m = 500, the inner loop executes ~1.25 × 10⁸ iterations — tight for a 1-second time limit but usually feasible with simple operations (additions and multiplications). Key optimizations:

    • Skip zero rows early. If the pivot column is entirely zero, skip it without iterating over all rows for elimination.
    • Bitset for XOR systems. When all entries are 0 or 1 and the operation is XOR, use bitset to pack 64 columns per word. This reduces the constant factor by 64× and handles n = 4000 comfortably.
    • Cache-friendly access. The standard row-major layout is already cache-friendly since elimination sweeps across rows.

    Applications in CP

    Gaussian elimination is a building block for many CP techniques. Here are the most common applications:

    1. Solving Systems of Linear Equations

    The most direct application. Given n equations and m unknowns, build the augmented matrix and run Gauss. Check the rank to determine if the solution is unique, infinite, or nonexistent.

    2. Finding Matrix Rank

    The rank equals the number of pivots after elimination. This is useful for problems like "given a set of vectors, how many are linearly independent?" — just arrange them as rows and compute the rank.

    3. Computing Determinants

    After reducing to REF (without scaling rows to have leading 1s), the determinant is the product of the diagonal entries, times (−1)^(number of row swaps). Keep track of the swap parity and the product of original pivots before normalization. 

    // Determinant via Gaussian elimination (mod prime)
ll determinant(vector<vector<ll>> a) {
    int n = a.size();
    ll det = 1;
    for (int col = 0; col < n; col++) {
        int pivot_row = -1;
        for (int r = col; r < n; r++) {
            if (a[r][col] != 0) { pivot_row = r; break; }
        }
        if (pivot_row == -1) return 0; // singular

        if (pivot_row != col) {
            swap(a[col], a[pivot_row]);
            det = (MOD - det) % MOD; // flip sign
        }
        det = det * a[col][col] % MOD;

        ll inv = modinv(a[col][col], MOD);
        for (int j = col; j < n; j++)
            a[col][j] = a[col][j] * inv % MOD;

        for (int r = col + 1; r < n; r++) {
            if (a[r][col] == 0) continue;
            ll factor = a[r][col];
            for (int j = col; j < n; j++)
                a[r][j] = (a[r][j] - factor * a[col][j] % MOD + MOD) % MOD;
        }
    }
    return det;
}

    4. Checking Linear Independence

    To check if vectors v₁, v₂, …, vₖ are linearly independent, form a matrix with these vectors as rows and compute the rank. If rank = k, they are independent.

    5. XOR Basis (Gaussian Elimination over GF(2))

    When the "field" is {0, 1} with XOR as addition and AND as multiplication, Gaussian elimination gives you a linear basis for a set of integers. This is the foundation for problems like "maximum XOR subset" or "number of distinct XOR values." The algorithm is the same, but arithmetic is replaced by bitwise XOR. 

    // XOR basis using Gaussian elimination over GF(2)
struct XorBasis {
    vector<ll> basis;
    int sz = 0;

    void insert(ll val) {
        for (int i = 0; i < sz; i++)
            val = min(val, val ^ basis[i]);
        if (val > 0) {
            basis.push_back(val);
            sz++;
            // Maintain reduced form
            for (int i = sz - 1; i > 0 && basis[i] > basis[i-1]; i--)
                swap(basis[i], basis[i-1]);
        }
    }

    ll max_xor() {
        ll res = 0;
        for (int i = 0; i < sz; i++)
            res = max(res, res ^ basis[i]);
        return res;
    }

    int rank() { return sz; }
};
    Connection to standard Gauss. The XOR basis is exactly Gaussian elimination where each integer is a row vector over GF(2), with the bits as columns. The "reduce from highest bit" strategy mirrors column-by-column pivoting. We will explore this in detail in a later post on XOR linear algebra.

    6. Counting Solutions

    If the system has rank r and m unknowns, the free variables span an (m − r)-dimensional space. Over a finite field of size p, the number of solutions is p^(m−r) (assuming the system is consistent). This counting argument appears in several combinatorics problems.

    Summary

    • Gaussian elimination transforms a matrix to row echelon form using elementary row operations.
    • RREF goes further, making each pivot 1 and zeroing out entries above pivots too.
    • Partial pivoting (choosing the largest entry) is essential for floating-point stability.
    • The modular version avoids all precision issues — prefer it when the problem allows.
    • Time complexity is O(n²m) — feasible for n, m up to ~500 in most judges.
    • Applications span solving systems, computing rank, determinants, XOR bases, and counting solutions.
