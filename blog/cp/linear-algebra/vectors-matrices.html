    Vectors & Matrices — Basics & Representation in C++

    July 2026 · 18 min read

    Linear algebra is one of the most powerful mathematical tools in competitive programming. From computing Fibonacci numbers in O(log n) via matrix exponentiation, to solving systems of linear equations, to transforming geometry — vectors and matrices are everywhere once you start looking. This first post in the series lays the groundwork: what vectors and matrices are, how to represent them efficiently in C++, and the basic operations you will use in every problem that involves them.

    What Are Vectors and Matrices?

    A vector is an ordered list of numbers. In competitive programming we almost always work with column vectors of integers (or integers modulo a prime). A vector with n entries lives in ℝⁿ (or ℤₚⁿ when working modulo p).

    A matrix is a rectangular grid of numbers with r rows and c columns, denoted as an r × c matrix. You can think of a matrix as a collection of row vectors stacked vertically, or column vectors placed side by side.

    Key terminology you will see throughout this series:

    • Dimension — The size of a vector (n) or matrix (r × c).
    • Square matrix — A matrix where r = c. Most CP problems use square matrices.
    • Entry aᵢⱼ — The element at row i, column j (0-indexed in code, 1-indexed in math).
    • Row vector — A 1 × c matrix. Column vector — An r × 1 matrix.

    In contests the matrices are typically small (2×2, 3×3, rarely beyond 100×100) but the operations on them — especially exponentiation — are what save time.

    Representing Vectors in C++

    The simplest choice is std::vector<long long>. It is flexible, works with any dimension, and plays nicely with the matrix representations we will build later. For fixed small dimensions you can also use std::array or plain C arrays.

    Basic Vector Type

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;

// Dynamic-size vector
using Vec = vector<ll>;

// Create a zero vector of size n
Vec zero_vec(int n) {
    return Vec(n, 0);
}

// Print a vector
void print_vec(const Vec& v) {
    cout << "[";
    for (int i = 0; i < (int)v.size(); i++) {
        if (i) cout << ", ";
        cout << v[i];
    }
    cout << "]\n";
}

    Vector Operations

    Three operations come up constantly: addition, scalar multiplication, and the dot product. All run in O(n) time where n is the vector dimension. 

    const ll MOD = 1e9 + 7;

// Component-wise addition (mod p)
Vec vec_add(const Vec& a, const Vec& b) {
    int n = a.size();
    Vec c(n);
    for (int i = 0; i < n; i++)
        c[i] = (a[i] + b[i]) % MOD;
    return c;
}

// Scalar multiplication (mod p)
Vec vec_scale(const Vec& a, ll s) {
    int n = a.size();
    Vec c(n);
    for (int i = 0; i < n; i++)
        c[i] = a[i] % MOD * (s % MOD) % MOD;
    return c;
}

// Dot product (mod p)
ll dot(const Vec& a, const Vec& b) {
    ll res = 0;
    for (int i = 0; i < (int)a.size(); i++)
        res = (res + a[i] % MOD * (b[i] % MOD)) % MOD;
    return res;
}

    Why modular arithmetic from the start? In competitive programming, nearly every linear-algebra problem asks for the answer modulo 10⁹+7 (or similar). Building the habit of reducing at every step prevents overflow bugs. With long long and MOD ≈ 10⁹, the product of two entries fits in 64 bits (at most ~10¹⁸), so a single multiplication before reduction is safe.

    Matrix Representation

    The standard contest representation is a vector<vector<ll>> wrapped in a typedef. This balances readability, flexibility, and speed for the matrix sizes you encounter in CP (typically ≤ 100 × 100).

    The Matrix Typedef

    using Matrix = vector<vector<ll>>;

// Create an r x c zero matrix
Matrix zero_mat(int r, int c) {
    return Matrix(r, vector<ll>(c, 0));
}

// Create an n x n identity matrix
Matrix identity(int n) {
    Matrix I = zero_mat(n, n);
    for (int i = 0; i < n; i++) I[i][i] = 1;
    return I;
}

// Print a matrix
void print_mat(const Matrix& M) {
    for (auto& row : M) {
        for (int j = 0; j < (int)row.size(); j++) {
            if (j) cout << " ";
            cout << row[j];
        }
        cout << "\n";
    }
}

    Fixed-Size Alternative

    When the dimension is known at compile time and small (e.g., 2×2 for Fibonacci), a fixed-size struct can be faster due to stack allocation and no heap overhead: 

    template <int N>
struct Mat {
    ll a[N][N];
    Mat() { memset(a, 0, sizeof(a)); }

    static Mat identity() {
        Mat I;
        for (int i = 0; i < N; i++) I.a[i][i] = 1;
        return I;
    }
};

    For most contest problems the dynamic Matrix typedef is perfectly fine. Use the templated struct only when you need every microsecond of performance on tight time limits.

    Accessing Elements

    Matrix A = zero_mat(3, 3);

// Set individual entries
A[0][0] = 1;  A[0][1] = 2;  A[0][2] = 3;
A[1][0] = 4;  A[1][1] = 5;  A[1][2] = 6;
A[2][0] = 7;  A[2][1] = 8;  A[2][2] = 9;

// Read an entry
ll val = A[1][2];  // row 1, col 2 → 6

// Iterate over all entries
for (int i = 0; i < 3; i++)
    for (int j = 0; j < 3; j++)
        cout << A[i][j] << " \n"[j == 2];

    Basic Matrix Operations

    These four operations are the building blocks. Every one of them is O(r·c) for an r × c matrix — except transpose which is O(r·c) as well but produces a c × r matrix.

    Addition & Subtraction

    Matrix mat_add(const Matrix& A, const Matrix& B) {
    int r = A.size(), c = A[0].size();
    Matrix C = zero_mat(r, c);
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            C[i][j] = (A[i][j] + B[i][j]) % MOD;
    return C;
}

Matrix mat_sub(const Matrix& A, const Matrix& B) {
    int r = A.size(), c = A[0].size();
    Matrix C = zero_mat(r, c);
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            C[i][j] = (A[i][j] - B[i][j] % MOD + MOD) % MOD;
    return C;
}

    Scalar Multiplication

    Matrix mat_scale(const Matrix& A, ll s) {
    int r = A.size(), c = A[0].size();
    Matrix C = zero_mat(r, c);
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            C[i][j] = A[i][j] % MOD * (s % MOD) % MOD;
    return C;
}

    Transpose

    Matrix transpose(const Matrix& A) {
    int r = A.size(), c = A[0].size();
    Matrix T = zero_mat(c, r);
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            T[j][i] = A[i][j];
    return T;
}

    Subtraction pitfall: Notice the + MOD in mat_sub. When working modulo a prime, a - b can go negative. Always add MOD before taking the final mod to stay in [0, MOD).

    Matrix-Vector Multiplication

    This is the core operation that connects matrices to transformations. Given an r × c matrix A and a column vector v of size c, the product Av is a column vector of size r where the i-th entry is the dot product of the i-th row of A with v.

    Algorithm & Complexity

    For each of the r rows, we compute a dot product with the c-element vector. Each dot product is O(c), so the total complexity is O(r · c). For a square n × n matrix this is O(n²). 

    // Multiply r×c matrix A by c-dimensional vector v → r-dimensional result
Vec mat_vec_mul(const Matrix& A, const Vec& v) {
    int r = A.size(), c = A[0].size();
    Vec res(r, 0);
    for (int i = 0; i < r; i++)
        for (int j = 0; j < c; j++)
            res[i] = (res[i] + A[i][j] % MOD * (v[j] % MOD)) % MOD;
    return res;
}

    Example: Applying a Transformation

    int main() {
    // 2x2 matrix that rotates the state
    Matrix A = {{1, 1},
                {1, 0}};

    // Initial state vector [F(1), F(0)] = [1, 0]
    Vec v = {1, 0};

    // One step: A * v = [1*1+1*0, 1*1+0*0] = [1, 1]
    Vec result = mat_vec_mul(A, v);
    print_vec(result);  // [1, 1]

    // Another step gives [1+1, 1+0] = [2, 1] → F(3), F(2)
    result = mat_vec_mul(A, result);
    print_vec(result);  // [2, 1]

    return 0;
}

    This is the Fibonacci recurrence in matrix form. The matrix [[1,1],[1,0]] advances the pair (F(k), F(k-1)) to (F(k+1), F(k)). We will exploit this with matrix exponentiation in the next post.

    Common Patterns in Competitive Programming

    Identity Matrix

    The n × n identity matrix I has 1s on the diagonal and 0s elsewhere. It is the multiplicative identity: A · I = I · A = A. You need it as the base case in matrix exponentiation (just like result = 1 in scalar fast exponentiation).

    Zero Matrix

    All entries are 0. Acts as the additive identity. Useful as the initial accumulator when summing a series of matrix products.

    Diagonal Matrices

    Non-zero only on the main diagonal. Multiplying by a diagonal matrix simply scales each row (left-multiply) or column (right-multiply). Complexity reduces to O(n).

    Where Matrices Appear in CP

    • Linear recurrences — Any recurrence of the form f(n) = c₁f(n-1) + c₂f(n-2) + … + cₖf(n-k) can be expressed as a matrix-vector product and then solved in O(k³ log n) via exponentiation.
    • Graph path counting — The (i,j) entry of Aⁿ (adjacency matrix) counts the number of paths of length exactly n from node i to node j.
    • Systems of linear equations — Gaussian elimination on augmented matrices.
    • Geometry — Rotation, reflection, scaling via 2×2 or 3×3 transformation matrices.
    • DP optimization — Some DP transitions can be recast as matrix multiplication, enabling O(n³ log T) solutions instead of O(n² T).

    Summary

    • Vectors are ordered lists of numbers; matrices are 2D grids. Both are small in CP but powerful.
    • vector<vector<ll>> is the standard matrix representation — flexible and fast enough.
    • Always work modulo a prime from the start. Add + MOD before taking mod on subtractions.
    • Matrix-vector multiplication is O(n²) for square matrices — the key operation behind matrix exponentiation.
    • Identity matrix is to matrix exponentiation what 1 is to scalar fast power — your base case.
    • Matrices appear in recurrences, graph counting, geometry, and DP optimization.
