    Kirchhoff's Theorem — Counting Spanning Trees

    March 2026 · 22 min read

    The Problem

    Given an undirected graph with n vertices and m edges, how many distinct spanning trees does it contain? A spanning tree is a connected, acyclic subgraph that includes every vertex and exactly n−1 edges.

    For the complete graph Kn, Cayley's formula tells us the answer is nn−2. But for an arbitrary graph the combinatorial approach quickly becomes intractable. Kirchhoff's theorem reduces the problem to a single determinant computation — an O(n³) operation that fits perfectly inside CP time limits.

    Spanning-tree counting appears in:

    • Network reliability — the number of spanning trees measures how "well connected" a graph is.
    • Random spanning tree generation — Kirchhoff weights enable uniform sampling.
    • Competitive programming problems that ask for the count modulo a prime.

    Kirchhoff's Matrix-Tree Theorem

    Let G be an undirected graph on n vertices (possibly with multiple edges, but no self-loops). Define the Laplacian matrix L of G as described in the next section. Then:

    Theorem (Kirchhoff, 1847). The number of spanning trees of G equals any cofactor of L. Formally, for any index i,

    τ(G) = det(L[i])

    where L[i] is the (n−1)×(n−1) matrix obtained by deleting row i and column i from L.

    The remarkable property is that the result is the same regardless of which row/column you delete. This is because every row (and every column) of the Laplacian sums to zero, so L always has rank at most n−1. For a connected graph the rank is exactly n−1, and the cofactor is positive.

    For a disconnected graph, every (n−1)×(n−1) minor is zero, correctly reflecting that no spanning tree exists.

    Laplacian Matrix

    The Laplacian is defined as L = D − A, where:

    • D is the degree matrix: a diagonal matrix where D[i][i] = degree of vertex i.
    • A is the adjacency matrix: A[i][j] = number of edges between vertices i and j.

    Equivalently, the Laplacian can be built directly:

    • L[i][i] = degree of vertex i
    • L[i][j] = −(number of edges between i and j), for i ≠ j

    Example: consider a triangle graph on vertices {0, 1, 2} with edges {0-1, 1-2, 0-2}.

    Adjacency matrix A:          Degree matrix D:          Laplacian L = D - A:
  0  1  1                      2  0  0                    2 -1 -1
  1  0  1                      0  2  0                   -1  2 -1
  1  1  0                      0  0  2                   -1 -1  2

    Delete row 0, column 0 → the minor is:

     2 -1
-1  2

det = 2*2 - (-1)*(-1) = 4 - 1 = 3

    The triangle has 3 spanning trees (each formed by removing one of the three edges). ✓

    Building the Laplacian in C++

    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
typedef vector<vector<ll>> Matrix;

// Build the Laplacian for n vertices from an edge list.
// Vertices are 0-indexed.
Matrix build_laplacian(int n, const vector<pair<int,int>>& edges) {
    Matrix L(n, vector<ll>(n, 0));
    for (auto& [u, v] : edges) {
        L[u][u]++;
        L[v][v]++;
        L[u][v]--;
        L[v][u]--;
    }
    return L;
}

    Computing the Answer

    The algorithm is straightforward:

    1. Build the n×n Laplacian matrix L.
    2. Delete any one row and the corresponding column (typically row 0 and column 0).
    3. Compute the determinant of the resulting (n−1)×(n−1) matrix.

    For the determinant, Gaussian elimination runs in O(n³). Over the reals, we use row operations to reduce the matrix to upper-triangular form, then multiply the diagonal entries.

    Determinant via Gaussian Elimination (floating point)

    // Compute determinant of a square matrix using Gaussian elimination.
// Works over doubles — sufficient when the answer fits in ~10^15.
double determinant(vector<vector<double>> mat) {
    int n = mat.size();
    double det = 1.0;
    for (int col = 0; col < n; col++) {
        // Partial pivoting
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (abs(mat[row][col]) > 1e-9) {
                pivot = row;
                break;
            }
        }
        if (pivot == -1) return 0;  // singular
        if (pivot != col) {
            swap(mat[pivot], mat[col]);
            det *= -1;
        }
        det *= mat[col][col];
        double inv = 1.0 / mat[col][col];
        for (int row = col + 1; row < n; row++) {
            double factor = mat[row][col] * inv;
            for (int k = col; k < n; k++) {
                mat[row][k] -= factor * mat[col][k];
            }
        }
    }
    return det;
}

// Count spanning trees (floating-point version)
ll count_spanning_trees(int n, const vector<pair<int,int>>& edges) {
    auto L = build_laplacian(n, edges);
    // Build (n-1) x (n-1) minor by removing row 0 and col 0
    int m = n - 1;
    vector<vector<double>> minor(m, vector<double>(m));
    for (int i = 0; i < m; i++)
        for (int j = 0; j < m; j++)
            minor[i][j] = (double)L[i + 1][j + 1];
    return llround(determinant(minor));
}
    Precision warning. Floating-point determinant works for small graphs, but rounding errors accumulate for n > 20 or when the answer exceeds 1015. For CP, always prefer the modular version below.

    Modular Arithmetic Version

    Most CP problems ask for the spanning tree count modulo a prime p. Over Z/pZ, Gaussian elimination is exact — no floating-point issues. We need modular inverse (via Fermat's little theorem) instead of division.

    Complete C++ Solution (mod prime)

    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
const ll MOD = 1e9 + 7;

ll power(ll base, ll exp, ll mod) {
    ll result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

ll mod_inv(ll a, ll mod) {
    return power(a, mod - 2, mod);
}

// Determinant of an n×n matrix modulo a prime.
ll det_mod(vector<vector<ll>> mat, ll mod) {
    int n = mat.size();
    ll det = 1;
    for (int col = 0; col < n; col++) {
        // Find pivot
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (mat[row][col] % mod != 0) {
                pivot = row;
                break;
            }
        }
        if (pivot == -1) return 0;
        if (pivot != col) {
            swap(mat[pivot], mat[col]);
            det = (mod - det) % mod;  // negate
        }
        det = det % mod * (mat[col][col] % mod) % mod;
        ll inv = mod_inv(mat[col][col] % mod, mod);
        for (int row = col + 1; row < n; row++) {
            ll factor = mat[row][col] % mod * inv % mod;
            for (int k = col; k < n; k++) {
                mat[row][k] = (mat[row][k] - factor * mat[col][k] % mod + mod) % mod;
            }
        }
    }
    return (det % mod + mod) % mod;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    // Build Laplacian
    vector<vector<ll>> L(n, vector<ll>(n, 0));
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;  // 0-index
        L[u][u]++;
        L[v][v]++;
        L[u][v]--;
        L[v][u]--;
    }

    // Extract (n-1) x (n-1) minor, taking mod to stay positive
    int sz = n - 1;
    vector<vector<ll>> minor_mat(sz, vector<ll>(sz));
    for (int i = 0; i < sz; i++)
        for (int j = 0; j < sz; j++)
            minor_mat[i][j] = (L[i + 1][j + 1] % MOD + MOD) % MOD;

    cout << det_mod(minor_mat, MOD) << "\n";
    return 0;
}

    Complexity: Building the Laplacian is O(m). The determinant is O(n³). For n ≤ 500 and typical 2–3 second time limits, this runs comfortably.

    Handling n = 1

    A single-vertex graph has exactly one spanning tree (the trivial tree with no edges). When n = 1, the minor is a 0×0 matrix whose determinant is 1 by convention. Make sure your code handles this edge case:

    if (n == 1) {
    cout << 1 << "\n";
    return 0;
}

    Handling Multigraphs and Weighted Graphs

    Multigraphs

    Kirchhoff's theorem works directly on multigraphs (graphs with parallel edges). If there are w edges between vertices u and v, set:

    • L[u][v] = L[v][u] = −w
    • Add w to both L[u][u] and L[v][v]

    The theorem then counts the number of spanning trees, where each tree is counted once for each distinct combination of parallel edges it could use.

    Weighted Version

    For a weighted graph where edge (u, v) has weight w(u,v), define the weighted Laplacian:

    • L[u][v] = −w(u,v) for u ≠ v
    • L[u][u] = ∑v≠u w(u,v)

    The cofactor of this weighted Laplacian gives the sum over all spanning trees of the product of edge weights in each tree. When all weights are 1, this reduces to the ordinary count.

    // Weighted Laplacian construction
struct Edge { int u, v; ll w; };

vector<vector<ll>> build_weighted_laplacian(int n, const vector<Edge>& edges, ll mod) {
    vector<vector<ll>> L(n, vector<ll>(n, 0));
    for (auto& [u, v, w] : edges) {
        L[u][u] = (L[u][u] + w) % mod;
        L[v][v] = (L[v][v] + w) % mod;
        L[u][v] = (L[u][v] - w % mod + mod) % mod;
        L[v][u] = (L[v][u] - w % mod + mod) % mod;
    }
    return L;
}

    Directed Graphs (Bonus)

    The directed version of Kirchhoff's theorem counts arborescences (directed spanning trees rooted at a specific vertex). The Laplacian for a directed graph is defined differently:

    • L[i][j] = −(number of edges from j to i), for i ≠ j
    • L[i][i] = in-degree of vertex i

    Deleting row r and column r gives the number of arborescences rooted at r. Unlike the undirected case, the result depends on which row/column is deleted.

    Classic CP Problems

    Example 1: Direct Application

    Problem: Given an undirected graph with n vertices and m edges, compute the number of spanning trees modulo 109+7.

    Solution: Build the Laplacian, extract the minor, compute the modular determinant — exactly the template code above.

    Example 2: Complete Graph Verification

    For K4 (complete graph on 4 vertices with 6 edges), Cayley's formula gives 42 = 16 spanning trees. Verify:

    // K4: edges (0,1), (0,2), (0,3), (1,2), (1,3), (2,3)
Laplacian:
  3 -1 -1 -1
 -1  3 -1 -1
 -1 -1  3 -1
 -1 -1 -1  3

Minor (delete row 0, col 0):
  3 -1 -1
 -1  3 -1
 -1 -1  3

det = 3*(3*3 - (-1)*(-1)) - (-1)*((-1)*3 - (-1)*(-1)) + (-1)*((-1)*(-1) - 3*(-1))
    = 3*(9-1) - (-1)*(-3-1) + (-1)*(1+3)
    = 3*8 - (-1)*(-4) + (-1)*4
    = 24 - 4 - 4 = 16  ✓

    Example 3: Grid Graph

    For a 2×3 grid graph (6 vertices, 7 edges), the number of spanning trees is 15. These problems often appear in contests disguised as "count the number of ways to connect all cities with exactly n−1 roads."

    Example 4: SPOJ — MSTCNT

    Problem: Given an undirected weighted graph, count the number of minimum spanning trees.

    Approach: This is harder than pure Kirchhoff. The idea is to fix the MST edge-weight structure (using Kruskal's) and then for each weight class, count the number of ways to choose edges of that weight. Each weight class forms a subproblem solvable with Kirchhoff's theorem on a contracted graph.

    Example 5: Codeforces — Roads Not Only in Berland

    While not a direct Kirchhoff problem, connectivity problems on complete or near-complete graphs can sometimes be solved by counting spanning trees and applying the matrix-tree theorem to verify whether a subgraph has a unique spanning structure.

    Key Patterns to Recognize

    • "Count the number of spanning trees" → Direct Kirchhoff application.
    • "Count spanning trees mod p" → Modular Gaussian elimination on the Laplacian minor.
    • "Sum of products of edge weights over all spanning trees" → Weighted Kirchhoff.
    • "Count rooted spanning trees in a directed graph" → Directed Kirchhoff (arborescences).
    • "Count MSTs" → Kirchhoff per weight class after Kruskal contraction.
