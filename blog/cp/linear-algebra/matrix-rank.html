Matrix Rank — Rank Computation & Dimension Counting

June 2026 · 22 min read

What Is Matrix Rank?

The rank of an m × n matrix A is the maximum number of linearly independent rows (or, equivalently, columns). This single number encodes a surprising amount of information about the linear map that A represents.

Row Rank Equals Column Rank

A fundamental theorem in linear algebra states that the row rank (dimension of the row space) always equals the column rank (dimension of the column space). We simply call both “the rank” and denote it rank(A).

Intuitively: row reduction never changes the column space’s dimension, and transposing a matrix preserves rank. So the two notions coincide for every matrix over every field.

Key Properties at a Glance

  • 0 ≤ rank(A) ≤ min(m, n)
  • rank(A) = rank(AT)
  • rank(AB) ≤ min(rank(A), rank(B))
  • For a square n × n matrix, rank(A) = n iff A is invertible (full rank).
  • Adding a row that is a linear combination of existing rows does not increase rank.
CP intuition: Rank tells you the “effective dimension” of a set of vectors. If you have k vectors in ℝn and their matrix has rank r, then only r of them carry genuinely new information — the rest are redundant.

Computing Rank via Gaussian Elimination

The standard algorithm: bring the matrix to row echelon form (REF) using elementary row operations, then count the non-zero rows. Each non-zero row corresponds to a pivot column.

Algorithm Sketch

  1. For each column j (left to right), find a row with a non-zero entry in column j among the rows below the current pivot row.
  2. Swap that row into the pivot position.
  3. Use that pivot to eliminate all entries below it in column j.
  4. Move to the next row and next column. Repeat.
  5. The number of pivots found = rank(A).

Complexity

For an m × n matrix, Gaussian elimination runs in O(m · n · min(m, n)). For square matrices this is O(n³).

C++ Implementation (Reals with Epsilon)

// Rank of a real-valued matrix via Gaussian elimination
// O(m * n * min(m, n))
int matrix_rank(vector<vector<double>> a) {
    int m = a.size(), n = a[0].size();
    const double EPS = 1e-9;
    int rank = 0;

    for (int col = 0; col < n; col++) {
        // Find pivot
        int pivot = -1;
        for (int row = rank; row < m; row++) {
            if (abs(a[row][col]) > EPS) {
                pivot = row;
                break;
            }
        }
        if (pivot == -1) continue;

        swap(a[rank], a[pivot]);

        // Normalize pivot row
        double div = a[rank][col];
        for (int j = col; j < n; j++)
            a[rank][j] /= div;

        // Eliminate below
        for (int row = 0; row < m; row++) {
            if (row == rank || abs(a[row][col]) < EPS) continue;
            double factor = a[row][col];
            for (int j = col; j < n; j++)
                a[row][j] -= factor * a[rank][j];
        }
        rank++;
    }
    return rank;
}
Numerical pitfall: Choosing the first non-zero pivot is fragile. For better numerical stability, use partial pivoting — pick the row with the largest absolute value in the current column. Replace the pivot search with max_element by absolute value.

Partial Pivoting Variant

// Find best pivot (largest absolute value) for numerical stability
int pivot = rank;
for (int row = rank + 1; row < m; row++) {
    if (abs(a[row][col]) > abs(a[pivot][col]))
        pivot = row;
}
if (abs(a[pivot][col]) < EPS) continue;

Rank Over Different Fields

In CP, matrices don’t always live over the reals. The three most common settings are real numbers (with epsilon), modular arithmetic (mod a prime), and GF(2) (binary / XOR). Each requires a different implementation.

Rank Modulo a Prime

When working modulo a prime p, every non-zero element has a modular inverse, so Gaussian elimination works exactly — no epsilon needed.

// Rank of a matrix modulo a prime p
// O(m * n * min(m, n))
long long mod_pow(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

int matrix_rank_mod(vector<vector<long long>> a, long long p) {
    int m = a.size(), n = a[0].size();
    int rank = 0;

    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = rank; row < m; row++) {
            if (a[row][col] % p != 0) {
                pivot = row;
                break;
            }
        }
        if (pivot == -1) continue;

        swap(a[rank], a[pivot]);

        // Multiply pivot row by inverse of a[rank][col]
        long long inv = mod_pow(a[rank][col], p - 2, p);
        for (int j = col; j < n; j++)
            a[rank][j] = a[rank][j] % p * inv % p;

        // Eliminate all other rows
        for (int row = 0; row < m; row++) {
            if (row == rank || a[row][col] % p == 0) continue;
            long long factor = a[row][col] % p;
            for (int j = col; j < n; j++)
                a[row][j] = ((a[row][j] - factor * a[rank][j]) % p + p) % p;
        }
        rank++;
    }
    return rank;
}

Rank Over GF(2) — XOR Basis

Over GF(2), addition is XOR and multiplication is AND. This is by far the most common field in CP problems involving XOR. The algorithm simplifies dramatically because every non-zero element is 1, so no division is needed.

// Rank of a set of numbers over GF(2) — XOR basis / linear basis
// O(n * B) where B = number of bits
int xor_rank(vector<long long> v) {
    vector<long long> basis;
    for (long long x : v) {
        for (long long b : basis)
            x = min(x, x ^ b);
        if (x > 0)
            basis.push_back(x);
    }
    return basis.size();
}

// Cleaner version with fixed bit positions
int xor_rank_v2(vector<long long> v) {
    long long basis[60] = {};
    int rank = 0;
    for (long long x : v) {
        for (int bit = 59; bit >= 0; bit--) {
            if (!(x >> bit & 1)) continue;
            if (!basis[bit]) {
                basis[bit] = x;
                rank++;
                break;
            }
            x ^= basis[bit];
        }
    }
    return rank;
}
Why GF(2) is special in CP: The XOR basis is essentially Gaussian elimination over GF(2). Each element of the basis corresponds to a pivot. The rank tells you the dimension of the XOR-reachable subspace, which determines how many distinct XOR values you can form — exactly 2rank.

Comparison Summary

Real Numbers

  • Epsilon comparisons required
  • Use partial pivoting for stability
  • Rare in CP (precision issues)

Modular (mod prime)

  • Exact arithmetic, no epsilon
  • Use Fermat’s little theorem for inverse
  • Common in counting problems

GF(2) — XOR

  • No division needed (everything is 0 or 1)
  • XOR basis is the standard approach
  • Extremely common in CP

Choosing the Right Field

  • XOR operations → GF(2)
  • “Count mod 109+7” → mod prime
  • Geometry / continuous → reals

Rank-Nullity Theorem

For any m × n matrix A:

rank(A) + nullity(A) = n

where nullity(A) = dimension of the null space (kernel) of A — the set of all vectors x such that Ax = 0.

What This Means

  • rank(A) = number of pivot columns = number of “constrained” variables.
  • nullity(A) = number of free variables = n − rank(A).
  • The null space is spanned by exactly n - rank(A) independent vectors.

Example

Consider a 3 × 5 matrix with rank 2:

  • Nullity = 5 − 2 = 3 free variables.
  • The solution space of Ax = 0 is 3-dimensional.
  • If the system Ax = b has a solution, the set of all solutions is a 3-dimensional affine subspace.

CP Implications

The rank-nullity theorem is the key to counting solutions:

  • Over GF(2): if the system is consistent, the number of solutions is 2n − rank.
  • Over ℤp: the number of solutions to a consistent system is pn − rank.
  • This is how you solve problems like “count the number of subsets with XOR equal to k.”
// Count solutions to Ax = b over GF(2)
// Returns 0 if no solution, else 2^(n - rank)
long long count_xor_solutions(vector<long long> equations, long long target, int n) {
    // Gaussian elimination over GF(2)
    long long basis[60] = {};
    int rank = 0;
    // Augment: store target in a separate variable
    // that gets XOR'd in sync with elimination

    for (long long eq : equations) {
        long long cur = eq;
        for (int bit = 59; bit >= 0; bit--) {
            if (!(cur >> bit & 1)) continue;
            if (!basis[bit]) {
                basis[bit] = cur;
                rank++;
                break;
            }
            cur ^= basis[bit];
        }
    }

    // Check if target is in the span
    long long t = target;
    for (int bit = 59; bit >= 0; bit--) {
        if ((t >> bit & 1) && basis[bit])
            t ^= basis[bit];
    }
    if (t != 0) return 0; // no solution

    return 1LL << (n - rank);
}

Applications in CP

1. Linear Independence Check

Given k vectors, are they linearly independent? Compute the rank — it equals k iff the vectors are independent.

bool are_independent(vector<vector<long long>>& vecs, long long p) {
    int k = vecs.size();
    return matrix_rank_mod(vecs, p) == k;
}

2. Dimension of Span

The rank directly gives you the dimension of the subspace spanned by a set of vectors. This is needed when counting the size of a reachable set or the number of distinct linear combinations.

3. XOR Basis Size

The most common CP application: given an array of integers, find how many distinct XOR values can be formed using subsets. The answer is 2rank where rank is the GF(2) rank (XOR basis size).

// Classic problem: max XOR subset
// Given n numbers, find the maximum XOR obtainable from any subset
long long max_xor_subset(vector<long long> v) {
    long long basis[60] = {};
    for (long long x : v) {
        for (int bit = 59; bit >= 0; bit--) {
            if (!(x >> bit & 1)) continue;
            if (!basis[bit]) {
                basis[bit] = x;
                break;
            }
            x ^= basis[bit];
        }
    }

    long long ans = 0;
    for (int bit = 59; bit >= 0; bit--) {
        ans = max(ans, ans ^ basis[bit]);
    }
    return ans;
}

4. Number of Distinct XOR Sums

// How many distinct XOR values can be formed from subsets of v?
long long count_distinct_xor(vector<long long> v) {
    int r = xor_rank_v2(v);
    return 1LL << r;  // 2^rank values (including 0)
}

5. Minimum Cost Basis

Some problems assign costs to vectors and ask for the minimum-cost set of vectors that spans the same space. This is a weighted matroid intersection / greedy problem: sort by cost, then greedily add vectors that increase the rank.

// Greedy minimum-cost spanning set over GF(2)
long long min_cost_basis(vector<pair<long long, long long>> items) {
    // items[i] = {cost, value}
    sort(items.begin(), items.end());
    long long basis[60] = {}, total_cost = 0;
    for (auto [cost, val] : items) {
        long long x = val;
        for (int bit = 59; bit >= 0; bit--) {
            if (!(x >> bit & 1)) continue;
            if (!basis[bit]) {
                basis[bit] = x;
                total_cost += cost;
                break;
            }
            x ^= basis[bit];
        }
    }
    return total_cost;
}

Rank and System Solvability

Matrix rank is the key to determining whether a system of linear equations Ax = b has solutions, and if so, how many.

Rouché–Capelli Theorem

Let [A | b] denote the augmented matrix. The system Ax = b is:

ConditionResult
rank(A) < rank([A|b])No solution — the system is inconsistent.
rank(A) = rank([A|b]) = nUnique solution — every variable is determined.
rank(A) = rank([A|b]) < nInfinitely many solutions — n − rank free variables.

C++ — Full System Analysis

// Determine solvability of Ax = b (mod p)
// Returns: -1 = no solution, 0 = unique, else number of free variables
int system_analysis(vector<vector<long long>> A,
                    vector<long long> b, long long p) {
    int m = A.size(), n = A[0].size();

    // Build augmented matrix [A | b]
    vector<vector<long long>> aug(m, vector<long long>(n + 1));
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++)
            aug[i][j] = (A[i][j] % p + p) % p;
        aug[i][n] = (b[i] % p + p) % p;
    }

    int rank_A = matrix_rank_mod(A, p);
    int rank_aug = matrix_rank_mod(aug, p);

    if (rank_A < rank_aug) return -1;  // inconsistent
    if (rank_A == n) return 0;          // unique solution
    return n - rank_A;                  // number of free variables
}

Practical Checks in CP

Many CP problems reduce to checking one of these conditions:

  • “Is the target reachable?” — Does b lie in the column space of A? Check rank(A) == rank([A|b]).
  • “How many ways?” — If the system is consistent over ℤp, the answer is pn − rank.
  • “Unique assignment?” — Does rank(A) == n?
Common pattern: In XOR problems, “can we reach target t from some subset?” translates directly to “is t in the span of the XOR basis?” Reduce t by the basis; if it becomes 0, the answer is yes.
