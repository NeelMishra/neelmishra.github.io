Binary Matrices — GF(2) Operations & Light-Switching Puzzles

January 2026 · 20 min read

What Is GF(2)?

GF(2) (Galois Field of order 2) is the smallest finite field. It contains exactly two elements: {0, 1}. All arithmetic is performed modulo 2, which gives us beautifully simple operations that map directly to bitwise instructions on modern hardware.

Addition (= XOR)

+01
001
110

0 + 0 = 0, 0 + 1 = 1, 1 + 1 = 0. This is exactly the XOR operation. Note that subtraction = addition in GF(2) because −1 ≡ 1 (mod 2).

Multiplication (= AND)

×01
000
101

This is exactly the AND operation. Every nonzero element (just 1) has an inverse (also 1), satisfying the field axiom.

Why does GF(2) matter for CP? Many competitive-programming problems involve XOR operations on sets of numbers. These are secretly linear algebra over GF(2). Recognizing this lets you apply Gaussian elimination, rank computation, and solution counting — all with blazing-fast bitwise operations.

Key properties that make GF(2) special: every element is its own additive inverse (a + a = 0 for all a), there are no sign issues, and the characteristic is 2. Vectors over GF(2) are just bit-strings, and matrices over GF(2) are arrays of bit-strings — making std::bitset the perfect data structure.

Binary Matrix Operations

A binary matrix is a matrix where every entry is in {0, 1} and arithmetic follows GF(2) rules. The standard matrix operations translate as follows:

  • Addition: C[i][j] = A[i][j] XOR B[i][j]
  • Multiplication: C[i][j] = XOR over k of (A[i][k] AND B[k][j])
  • Scalar multiplication: trivial — multiplying by 0 gives zero, by 1 gives the same matrix

Because each element is a single bit, we can pack an entire row into a bitset. Row XOR becomes a single bitwise operation on the whole bitset, giving a factor-of-64 speedup on 64-bit machines.

C++ Binary Matrix with bitset

#include <bits/stdc++.h>
using namespace std;

const int MAXN = 512;

struct BinaryMatrix {
    int n, m;
    bitset<MAXN> row[MAXN];

    BinaryMatrix() : n(0), m(0) {}
    BinaryMatrix(int _n, int _m) : n(_n), m(_m) {}

    // Matrix addition over GF(2): XOR
    BinaryMatrix operator+(const BinaryMatrix& o) const {
        BinaryMatrix res(n, m);
        for (int i = 0; i < n; i++)
            res.row[i] = row[i] ^ o.row[i];
        return res;
    }

    // Matrix multiplication over GF(2): AND + XOR
    BinaryMatrix operator*(const BinaryMatrix& o) const {
        BinaryMatrix res(n, o.m);
        // Transpose o for efficient column access
        BinaryMatrix oT(o.m, o.n);
        for (int i = 0; i < o.n; i++)
            for (int j = 0; j < o.m; j++)
                if (o.row[i][j]) oT.row[j].set(i);

        for (int i = 0; i < n; i++)
            for (int j = 0; j < o.m; j++)
                res.row[i][j] = (row[i] & oT.row[j]).count() & 1;
        return res;
    }
};

The multiplication uses a transpose trick: to compute the dot product of row i of A with column j of B, we AND the bitsets and check the parity of set bits. The overall complexity is O(n²m / 64) thanks to bitset parallelism.

Bitset size. The template parameter of bitset must be a compile-time constant. Set MAXN to the largest dimension you need. For problems with n ≤ 500, MAXN = 512 is a safe choice.

Gaussian Elimination Over GF(2)

Gaussian elimination over GF(2) follows the same algorithm as over the reals, but every operation simplifies to XOR. There is no need for pivoting strategies to handle numerical stability — entries are either 0 or 1, and XOR is exact.

Algorithm

  1. For each column (left to right), find a row with a 1 in that column (the pivot).
  2. Swap the pivot row to the current position.
  3. XOR the pivot row into every other row that has a 1 in the pivot column.
  4. Move to the next column and repeat.

After elimination, the matrix is in reduced row echelon form (RREF). The number of nonzero rows is the rank.

C++ Implementation

// Gaussian elimination over GF(2) using bitset.
// A is an n x m augmented matrix (m includes the RHS column).
// Returns the rank. A is modified in place to RREF.
int gaussGF2(bitset<MAXN> A[], int n, int m) {
    int rank = 0;
    for (int col = 0; col < m; col++) {
        // Find pivot
        int pivot = -1;
        for (int r = rank; r < n; r++) {
            if (A[r][col]) { pivot = r; break; }
        }
        if (pivot == -1) continue;

        swap(A[rank], A[pivot]);

        // Eliminate all other rows
        for (int r = 0; r < n; r++) {
            if (r != rank && A[r][col])
                A[r] ^= A[rank];
        }
        rank++;
    }
    return rank;
}

Complexity: O(n · m · min(n, m) / 64). The inner XOR of bitsets processes 64 bits at a time, making this extremely fast in practice. For n = m = 500, this runs in a few milliseconds.

Solving Ax = b over GF(2)

To solve a linear system, augment A with the column vector b and run Gaussian elimination. After RREF:

  • If any row has all zeros in A-columns but a 1 in the b-column, the system is inconsistent (no solution).
  • Otherwise, free variables (columns without pivots) can be set to 0 or 1 freely, giving 2f solutions where f is the number of free variables.

Light-Switching Puzzles

The classic "Lights Out" puzzle is played on an n × n grid. Each cell has a light that is either on (1) or off (0). Pressing a cell toggles it and all its orthogonal neighbors. The goal is to turn all lights off. This is a textbook application of GF(2) linear algebra.

Formulation as Ax = b

Number the cells 0 to n² − 1. Define the adjacency matrix A where A[i][j] = 1 if pressing cell j affects cell i (including j itself). Let b be the initial state vector and x be the vector of moves (x[j] = 1 if we press cell j). Since pressing a cell twice cancels out, each x[j] ∈ {0, 1}. The condition "all lights off" becomes:

A · x = b   (over GF(2))

This is a standard GF(2) linear system. We solve it with the Gaussian elimination routine above. If no solution exists, the puzzle is unsolvable from the given initial state.

C++ Solution for Lights Out

// Solve the Lights Out puzzle on an n x n grid.
// grid[i][j] = initial state (0 or 1).
// Returns a vector of (row, col) pairs to press, or empty if unsolvable.
vector<pair<int,int>> solveLightsOut(int n, vector<vector<int>>& grid) {
    int N = n * n;
    // Build augmented matrix [A | b] of size N x (N+1)
    bitset<MAXN> A[MAXN];

    auto id = [&](int r, int c) { return r * n + c; };
    int dx[] = {0, 0, 0, 1, -1};
    int dy[] = {0, 1, -1, 0, 0};

    for (int r = 0; r < n; r++)
        for (int c = 0; c < n; c++) {
            int i = id(r, c);
            // Column j affects row i if pressing j toggles i
            for (int d = 0; d < 5; d++) {
                int nr = r + dx[d], nc = c + dy[d];
                if (nr >= 0 && nr < n && nc >= 0 && nc < n)
                    A[i].set(id(nr, nc));
            }
            // Augmented column = b
            if (grid[r][c]) A[i].set(N);
        }

    // Gaussian elimination on [A | b]
    int rank = gaussGF2(A, N, N + 1);

    // Check consistency
    for (int r = rank; r < N; r++)
        if (A[r][N]) return {};  // no solution

    // Extract solution (set free vars to 0)
    vector<pair<int,int>> moves;
    for (int r = 0; r < rank; r++) {
        // Find pivot column
        for (int c = 0; c < N; c++) {
            if (A[r][c]) {
                if (A[r][N]) moves.push_back({c / n, c % n});
                break;
            }
        }
    }
    return moves;
}
Generalization. The same approach works for any "toggling" puzzle: if pressing a button toggles a known subset of lights, just change the adjacency matrix accordingly. The framework Ax = b over GF(2) handles them all.

Variant: Minimum Presses

When there are free variables, each assignment gives a different valid solution with a different number of presses. To minimize the total presses, iterate over all 2f assignments of free variables (feasible when f is small) and pick the one with the smallest Hamming weight in x.

XOR Linear Basis

Given a set of integers S = {a1, a2, …, an}, a XOR linear basis is a minimal subset B ⊆ span(S) such that every value obtainable by XOR-ing elements of S can also be obtained by XOR-ing elements of B. This is exactly the concept of a basis for a vector space over GF(2), where each integer is viewed as a bit-vector.

Greedy Construction

We maintain an array basis[0..LOGMAX-1]. For each new value v, we try to "reduce" it by XOR-ing with existing basis elements (highest bit first). If v survives to a nonzero value, it is linearly independent and we insert it. 

struct XORBasis {
    static const int LOG = 60;  // for values up to 2^60
    long long basis[LOG];
    int sz;

    XORBasis() : sz(0) { memset(basis, 0, sizeof(basis)); }

    // Insert a value. Returns true if it was linearly independent.
    bool insert(long long v) {
        for (int i = LOG - 1; i >= 0; i--) {
            if (!(v >> i & 1)) continue;
            if (!basis[i]) {
                basis[i] = v;
                sz++;
                return true;
            }
            v ^= basis[i];
        }
        return false;  // linearly dependent
    }

    // Query: can we represent val using basis elements?
    bool canRepresent(long long val) {
        for (int i = LOG - 1; i >= 0; i--) {
            if (val >> i & 1) val ^= basis[i];
        }
        return val == 0;
    }

    // Maximum XOR obtainable from the basis
    long long maxXOR() {
        long long res = 0;
        for (int i = LOG - 1; i >= 0; i--)
            res = max(res, res ^ basis[i]);
        return res;
    }

    // Minimum nonzero XOR obtainable (after reducing to RREF)
    long long minXOR() {
        // First reduce to RREF
        for (int i = LOG - 1; i >= 0; i--)
            if (basis[i])
                for (int j = i + 1; j < LOG; j++)
                    if (basis[j] >> i & 1)
                        basis[j] ^= basis[i];
        for (int i = 0; i < LOG; i++)
            if (basis[i]) return basis[i];
        return 0;
    }
};

Complexity: Each insert call is O(LOG) = O(log max_val). Building a basis from n elements takes O(n · log max_val).

Connection to rank. The size of the XOR basis equals the rank of the matrix formed by stacking the input values as row vectors over GF(2). If you insert n values and the basis has size r, then n − r values were linearly dependent.

Key Operations

  • Maximum XOR subset: greedily set each bit to 1 if it increases the result.
  • Minimum nonzero XOR: reduce the basis to RREF, then the smallest basis element is the answer.
  • k-th smallest XOR: reduce to RREF, then interpret k in binary using the basis elements.
  • Count of distinct XOR values: 2r where r = basis size (including 0).

Counting Solutions Over GF(2)

One of the most powerful results in GF(2) linear algebra is the solution-counting theorem:

Number of solutions to Ax = b  =  2(n − rank(A))

(provided the system is consistent). Here n is the number of variables and rank(A) is computed via Gaussian elimination. This is because each of the n − rank(A) free variables can independently be 0 or 1.

Application: Counting Toggle Sequences

In a light-switching problem, the number of distinct ways to reach the goal state is 2(n − rank). This is often asked in counting variants of toggling problems on Codeforces and AtCoder. 

// Count solutions to Ax = b over GF(2).
// A is n x m augmented matrix (last column is b).
// Returns 0 if inconsistent, else 2^(m-1-rank) (mod MOD).
long long countSolutions(bitset<MAXN> A[], int n, int m, long long MOD) {
    int numVars = m - 1;  // exclude augmented column
    int rank = gaussGF2(A, n, m);

    // Check consistency
    for (int r = rank; r < n; r++)
        if (A[r][numVars]) return 0;

    int freeVars = numVars - rank;
    // 2^freeVars mod MOD
    long long ans = 1;
    for (int i = 0; i < freeVars; i++)
        ans = ans * 2 % MOD;
    return ans;
}

Example: XOR Subset Count

Given n integers, how many subsets (including empty) have XOR equal to a target T? Build the XOR basis of size r. If T is representable by the basis, the answer is 2(n − r) — the dependent elements can be freely included or excluded, and the basis elements adjust to compensate.

Advanced Applications

XOR Spanning Sets

Given a set of values, the XOR span is the set of all values obtainable by XOR-ing any subset. The span has exactly 2r elements, where r is the basis size. This is useful in problems like "how many distinct XOR values can be formed?"

Maximum XOR Subsets

To find the maximum XOR of any subset, build the basis and greedily compute: 

// Given an array, find the maximum XOR of any nonempty subset.
long long maxSubsetXOR(vector<long long>& arr) {
    XORBasis B;
    for (auto v : arr) B.insert(v);
    return B.maxXOR();
}

This runs in O(n · log max_val) and solves a family of classical problems (e.g., Codeforces "XOR on Segment", SPOJ XORX).

Linear Independence in XOR Problems

Two common problem patterns:

  • Minimum operations to reach target: if every operation is a XOR with some fixed value, the minimum number of operations equals the minimum-weight representation in the basis (often solvable with BFS over GF(2) span).
  • Dynamic XOR basis: maintain a basis under insertions. This supports online queries about the span. Deletions are harder and typically require offline techniques or a segment tree of bases.

Matrix Exponentiation Over GF(2)

Binary matrix exponentiation works exactly like standard matrix exponentiation: repeated squaring with O(log k) multiplications. Combined with the bitset representation, this gives O(n³ log k / 64) for computing Ak. Applications include:

  • Counting paths of length k in a graph where edges are XOR toggles.
  • Simulating k steps of a linear recurrence over GF(2) (e.g., LFSR sequences).
// Binary matrix exponentiation over GF(2).
BinaryMatrix matpow(BinaryMatrix base, long long exp) {
    int n = base.n;
    BinaryMatrix result(n, n);
    // Identity matrix
    for (int i = 0; i < n; i++) result.row[i].set(i);

    while (exp > 0) {
        if (exp & 1) result = result * base;
        base = base * base;
        exp >>= 1;
    }
    return result;
}

Segment Tree with XOR Basis

For range XOR-basis queries, store a basis at each segment tree node. Merging two bases takes O(LOG²) time — insert each element of one basis into the other. This enables queries like "what is the maximum XOR of any subset of elements in range [l, r]?" in O(LOG² · log n) per query.
