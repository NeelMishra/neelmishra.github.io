    Matrix Inverse

    July 2026 · 22 min read

    The inverse of an n × n matrix A is the unique matrix A−1 such that A · A−1 = A−1 · A = I, where I is the identity matrix. Not every matrix has an inverse—only those with a nonzero determinant (i.e., full rank). In competitive programming, matrix inverse shows up when solving linear systems, undoing matrix transformations, and computing closed-form expressions from recurrences. This post covers Gauss-Jordan elimination for both real and modular arithmetic, the two settings you will encounter in contests.

    When Do We Need Matrix Inverse?

    In most contest scenarios you will never explicitly compute an inverse just because you can. Instead, you reach for it when:

    • Solving a linear system A x = b. If you have A−1, the solution is simply x = A−1 b. This is useful when the same coefficient matrix appears with many different right-hand sides.
    • Undoing a transformation. If a state vector is repeatedly transformed by A, applying A−1 rolls the state back by one step. Problems that ask “what was the initial configuration?” often reduce to this.
    • Interpolation and recurrence extraction. Given several terms of a linear recurrence, you can set up a matrix equation and solve for the recurrence coefficients via inversion.
    • Matrix division in expressions. Some problems define a result as B · A−1 (mod p). Computing the inverse mod p is the standard approach.
    Rule of thumb. If you only need to solve A x = b once, prefer Gaussian elimination directly—it is faster by a constant factor. Compute the full inverse only when you will reuse it.

    Complexity note: inverting an n × n matrix via Gauss-Jordan takes O(n³) time. In contests, n is typically at most a few hundred for floating-point problems and can be larger for modular ones, depending on the time limit.

    Gauss-Jordan Method

    The Gauss-Jordan algorithm is the workhorse for computing matrix inverses. The idea is elegant: augment A with the identity matrix I to form the n × 2n block [A | I], then perform row operations until the left half becomes I. At that point the right half is A−1.

    Step-by-Step Procedure

    1. Augment. Form the n × 2n matrix [A | I].
    2. Forward elimination with partial pivoting. For column j = 0, 1, …, n−1:
      • Find the row r ≥ j with the largest absolute value in column j (the pivot).
      • If the pivot is zero (or below epsilon), the matrix is singular—stop.
      • Swap row r with row j.
      • Scale row j so the pivot becomes 1.
      • Eliminate all other entries in column j by subtracting appropriate multiples of row j.
    3. Read off the inverse. The right half of the augmented matrix is now A−1.

    Why it works: every row operation on [A | I] is equivalent to left-multiplying by an elementary matrix E. After all operations, the left side is Ek … E1 A = I, so the right side is Ek … E1 I = A−1.

    Worked Example

    Invert the 2 × 2 matrix A = [[2, 1], [5, 3]]:

    Augment:  [2  1 | 1  0]
          [5  3 | 0  1]

R1 /= 2:  [1  0.5 | 0.5  0]
           [5  3   | 0    1]

R2 -= 5*R1: [1  0.5  | 0.5   0]
            [0  0.5  | -2.5  1]

R2 /= 0.5: [1  0.5 | 0.5  0]
            [0  1   | -5   2]

R1 -= 0.5*R2: [1  0 | 3   -1]
              [0  1 | -5   2]

Inverse:  [[ 3, -1],
           [-5,  2]]

    Verification: [[2,1],[5,3]] · [[3,−1],[−5,2]] = [[1,0],[0,1]]. ✓

    C++ Implementation (Floating Point)

    Below is a clean Gauss-Jordan implementation using double. It returns an empty vector if the matrix is singular. 

    #include <bits/stdc++.h>
using namespace std;

using Matrix = vector<vector<double>>;
const double EPS = 1e-9;

// Returns the inverse of A, or an empty matrix if singular.
Matrix inverse(Matrix A) {
    int n = (int)A.size();
    // Augment with identity
    Matrix aug(n, vector<double>(2 * n, 0));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            aug[i][j] = A[i][j];
        aug[i][n + i] = 1.0;
    }

    for (int col = 0; col < n; col++) {
        // Partial pivoting: find row with largest |value| in this column
        int pivot = col;
        for (int row = col + 1; row < n; row++)
            if (abs(aug[row][col]) > abs(aug[pivot][col]))
                pivot = row;
        if (abs(aug[pivot][col]) < EPS)
            return {};  // singular
        swap(aug[col], aug[pivot]);

        // Scale pivot row so that aug[col][col] == 1
        double div = aug[col][col];
        for (int j = 0; j < 2 * n; j++)
            aug[col][j] /= div;

        // Eliminate column in all other rows
        for (int row = 0; row < n; row++) {
            if (row == col) continue;
            double factor = aug[row][col];
            for (int j = 0; j < 2 * n; j++)
                aug[row][j] -= factor * aug[col][j];
        }
    }

    // Extract the right half
    Matrix inv(n, vector<double>(n));
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            inv[i][j] = aug[i][n + j];
    return inv;
}

int main() {
    int n;
    cin >> n;
    Matrix A(n, vector<double>(n));
    for (auto &row : A)
        for (auto &x : row)
            cin >> x;

    Matrix inv = inverse(A);
    if (inv.empty()) {
        cout << "Singular matrix\n";
        return 0;
    }

    cout << fixed << setprecision(6);
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            cout << inv[i][j] << " \n"[j == n - 1];
    }
}
    Numerical stability. Partial pivoting (choosing the row with the largest absolute value) is critical for floating-point accuracy. Without it, tiny pivots can cause catastrophic cancellation. For contest purposes, an EPS of 10−9 works well with double.

    Testing Your Implementation

    A quick sanity check: multiply A by its computed inverse and verify each entry of the product is within EPS of the identity matrix. This is a useful debugging step during a contest. 

    bool verify(const Matrix &A, const Matrix &Ainv) {
    int n = (int)A.size();
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            double sum = 0;
            for (int k = 0; k < n; k++)
                sum += A[i][k] * Ainv[k][j];
            double expected = (i == j) ? 1.0 : 0.0;
            if (abs(sum - expected) > 1e-6) return false;
        }
    return true;
}

    Modular Matrix Inverse

    In many CP problems, all arithmetic is performed modulo a prime p (commonly 109 + 7). The Gauss-Jordan algorithm adapts seamlessly—the only change is that “dividing by x” becomes “multiplying by x−1 mod p”. Since p is prime, Fermat’s little theorem gives:

    x−1 ≡ xp−2 (mod p)

    computed via fast exponentiation in O(log p). This replaces floating-point division entirely, eliminating all precision concerns.

    Key Differences from Floating-Point Version

    AspectFloating PointModular (mod p)

    Zero check|value| < EPSvalue == 0
    PivotingPartial pivoting for stabilityAny nonzero pivot works (no stability issues)
    Data typedoublelong long

    Full C++ Code (Mod Prime)

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;

const ll MOD = 1e9 + 7;

ll power(ll base, ll exp, ll mod) {
    ll result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

ll mod_inv(ll a, ll mod) {
    return power(a % mod + mod, mod - 2, mod);
}

using Mat = vector<vector<ll>>;

// Returns inverse of A mod MOD, or empty matrix if singular.
Mat mat_inverse(Mat A) {
    int n = (int)A.size();
    Mat aug(n, vector<ll>(2 * n, 0));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            aug[i][j] = (A[i][j] % MOD + MOD) % MOD;
        aug[i][n + i] = 1;
    }

    for (int col = 0; col < n; col++) {
        // Find any nonzero pivot
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (aug[row][col] != 0) {
                pivot = row;
                break;
            }
        }
        if (pivot == -1) return {};  // singular

        swap(aug[col], aug[pivot]);

        // Scale pivot row: multiply by inverse of aug[col][col]
        ll inv_val = mod_inv(aug[col][col], MOD);
        for (int j = 0; j < 2 * n; j++)
            aug[col][j] = aug[col][j] * inv_val % MOD;

        // Eliminate column in all other rows
        for (int row = 0; row < n; row++) {
            if (row == col || aug[row][col] == 0) continue;
            ll factor = aug[row][col];
            for (int j = 0; j < 2 * n; j++)
                aug[row][j] = (aug[row][j] - factor * aug[col][j] % MOD + MOD) % MOD;
        }
    }

    Mat inv(n, vector<ll>(n));
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            inv[i][j] = aug[i][n + j];
    return inv;
}

// Helper: multiply two matrices mod MOD
Mat mat_mul(const Mat &A, const Mat &B) {
    int n = (int)A.size();
    Mat C(n, vector<ll>(n, 0));
    for (int i = 0; i < n; i++)
        for (int k = 0; k < n; k++) if (A[i][k])
            for (int j = 0; j < n; j++)
                C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % MOD;
    return C;
}

int main() {
    int n;
    cin >> n;
    Mat A(n, vector<ll>(n));
    for (auto &row : A)
        for (auto &x : row)
            cin >> x;

    Mat inv = mat_inverse(A);
    if (inv.empty()) {
        cout << "Singular\n";
        return 0;
    }

    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            cout << inv[i][j] << " \n"[j == n - 1];
}
    Pitfall: negative intermediates. When computing aug[row][j] - factor * aug[col][j], the result can be negative before taking mod. Always add MOD before the final % MOD to avoid undefined behavior with negative values.

    Complexity

    The modular version has the same O(n³) structure, but each “division” step costs O(log p) for the modular exponentiation. The total complexity is therefore O(n³ + n · log p)—the n³ term dominates in practice because modular inverses are computed only n times (once per column), not n³ times.

    Checking Invertibility

    A square matrix A is invertible if and only if:

    • det(A) ≠ 0
    • rank(A) = n (full rank)
    • The homogeneous system A x = 0 has only the trivial solution
    • All eigenvalues are nonzero

    In the Gauss-Jordan algorithm, singularity is detected naturally: if any column has no nonzero entry to pivot on, the matrix is singular and we return early.

    Relationship with Determinant

    The determinant and the inverse are deeply connected. During Gauss-Jordan elimination, we can track the determinant as a byproduct. Each row swap flips the sign, and each scaling by a factor c divides the determinant by c. If the determinant is zero (mod p), the matrix has no inverse. 

    // Combined: compute inverse and determinant simultaneously (mod p)
pair<Mat, ll> inverse_and_det(Mat A) {
    int n = (int)A.size();
    ll det = 1;
    Mat aug(n, vector<ll>(2 * n, 0));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            aug[i][j] = (A[i][j] % MOD + MOD) % MOD;
        aug[i][n + i] = 1;
    }

    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = col; row < n; row++)
            if (aug[row][col] != 0) { pivot = row; break; }
        if (pivot == -1) return {{}, 0};

        if (pivot != col) {
            swap(aug[col], aug[pivot]);
            det = (MOD - det) % MOD;  // flip sign
        }

        det = det * aug[col][col] % MOD;

        ll inv_val = mod_inv(aug[col][col], MOD);
        for (int j = 0; j < 2 * n; j++)
            aug[col][j] = aug[col][j] * inv_val % MOD;

        for (int row = 0; row < n; row++) {
            if (row == col || aug[row][col] == 0) continue;
            ll factor = aug[row][col];
            for (int j = 0; j < 2 * n; j++)
                aug[row][j] = (aug[row][j] - factor * aug[col][j] % MOD + MOD) % MOD;
        }
    }

    Mat inv(n, vector<ll>(n));
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            inv[i][j] = aug[i][n + j];
    return {inv, det};
}

    Singular Matrices in CP

    When a problem says “it is guaranteed that the answer exists” or “the matrix is invertible,” you can skip the singularity check. Otherwise, always handle the singular case gracefully—many problems require you to output −1 or “impossible” when the system has no unique solution.

    Applications in CP

    1. Solving Multiple Systems with the Same Coefficient Matrix

    If you need to solve A x = b1, A x = b2, …, A x = bk, computing A−1 once and then multiplying each bi is O(n³ + k · n²), versus O(k · n³) for solving each system independently with Gaussian elimination.

    2. Undoing Matrix Exponentiation

    A common pattern: the problem defines a transition matrix T and you know the state after m steps is Tm · s0. If you are given the final state and asked for the initial state: 

    // Given final_state = T^m * initial_state, recover initial_state:
Mat Tm = mat_power(T, m);        // O(n^3 log m)
Mat Tm_inv = mat_inverse(Tm);    // O(n^3)
// initial_state = Tm_inv * final_state

    Alternatively, T−m = (T−1)m, so you can invert T first, then exponentiate the inverse.

    3. Computing Expressions Involving Matrix “Division”

    Some problems reduce to computing B · A−1 (mod p). A typical example: you have a system of recurrences and the answer is a rational function of matrix entries. Modular inverse lets you express “division” exactly.

    4. Inverting Transformations in Geometry Problems

    Affine transformations (rotation, scaling, translation) in 2D/3D can be represented as matrix multiplications in homogeneous coordinates. To reverse a sequence of transformations, invert the combined transformation matrix. This appears in problems involving coordinate systems and camera models.

    5. Solving Recurrences Backwards

    Given a linear recurrence f(n) = a1 f(n−1) + … + ak f(n−k), the companion matrix T advances the state by one step. T−1 steps backward, letting you compute f(−1), f(−2), etc. Some problems give values at positive indices and ask for values at negative indices. 

    // Companion matrix for recurrence f(n) = c1*f(n-1) + c2*f(n-2) + c3*f(n-3)
// T = [[c1, c2, c3],
//      [ 1,  0,  0],
//      [ 0,  1,  0]]
// T^(-1) lets you go backward: state_{n-1} = T^{-1} * state_n

Mat build_companion(vector<ll> &coeff) {
    int k = (int)coeff.size();
    Mat T(k, vector<ll>(k, 0));
    for (int j = 0; j < k; j++)
        T[0][j] = (coeff[j] % MOD + MOD) % MOD;
    for (int i = 1; i < k; i++)
        T[i][i - 1] = 1;
    return T;
}

// To step backward m times from state vector v:
// v_prev = mat_power(mat_inverse(T), m) * v

    Template: Matrix Power with Inverse Support

    A complete utility combining exponentiation and inverse: 

    Mat mat_power(Mat A, ll p) {
    int n = (int)A.size();
    Mat result(n, vector<ll>(n, 0));
    for (int i = 0; i < n; i++) result[i][i] = 1;  // identity

    if (p < 0) {
        A = mat_inverse(A);
        p = -p;
    }

    while (p > 0) {
        if (p & 1) result = mat_mul(result, A);
        A = mat_mul(A, A);
        p >>= 1;
    }
    return result;
}

    Common Mistakes to Avoid

    • Forgetting partial pivoting in the floating-point version. Without it, even well-conditioned matrices can produce wildly inaccurate results.
    • Negative modular values. Always normalize: (x % MOD + MOD) % MOD. A single negative intermediate will corrupt the entire matrix.
    • Using inverse when Gaussian elimination suffices. If you only solve one system, the inverse is wasted work—use forward/back substitution instead.
    • Assuming invertibility. Always check whether the matrix is singular unless the problem guarantees it.
    • Integer overflow. In the modular version, factor * aug[col][j] can overflow if both values are up to 109. Use long long for all intermediate computations.
