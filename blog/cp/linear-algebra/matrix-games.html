    Matrix Games

    July 2026 · 20 min read

    Two of the most beautiful branches of discrete mathematics — combinatorial game theory and linear algebra — collide in a class of competitive programming problems we call matrix games. The idea is deceptively simple: represent game states as vectors, encode moves as matrices, and let the algebraic machinery of matrix multiplication, exponentiation, and XOR arithmetic solve problems that would otherwise require exponential brute-force. This post explores the intersection in depth, from Nim-value matrices to payoff matrices and full CP solutions.

    Games and Matrices

    Combinatorial game theory studies two-player, perfect-information games with no chance element. The Sprague-Grundy theorem tells us every such game position has a Grundy value (or Nim-value), and the game is equivalent to a Nim heap of that size. When a game decomposes into independent sub-games, the overall Grundy value is the XOR of individual values.

    Linear algebra enters the picture in several ways:

    • State as a vector. A game with n positions can encode its state distribution as a vector in ℤⁿ (or GF(2)ⁿ for XOR-based games).
    • Transitions as a matrix. Legal moves define an adjacency or transition matrix A where A[i][j] = 1 if position j is reachable from position i in one move.
    • k-step analysis via exponentiation. Ak tells us exactly which states are reachable in k moves — a classic matrix exponentiation pattern.
    • XOR arithmetic over GF(2). When Grundy values interact via XOR, we can do matrix operations over GF(2) — addition is XOR, multiplication is AND.

    The power of this approach: problems that seem to require game tree search become O(n³ log k) matrix exponentiation problems. Let's see how.

    Nim-Value Matrices

    A Nim-value matrix arranges Sprague-Grundy values of game positions into a structured grid. The most famous example is the Sprague-Grundy table for two-pile Nim variants, where entry (i, j) stores the Grundy value of the position with pile sizes i and j.

    XOR Matrices over GF(2)

    Many game problems reduce to linear algebra over GF(2) — the field with two elements {0, 1} where addition is XOR and multiplication is AND. A matrix over GF(2) has entries that are single bits, and matrix multiplication uses XOR instead of addition and AND instead of multiplication.

    This is enormously useful because:

    • The Nim-value of a combined game is XOR of components — a linear operation over GF(2).
    • Gaussian elimination over GF(2) finds the rank (basis) of a set of Nim-values.
    • Matrix exponentiation over GF(2) solves recurrences where the "sum" operation is XOR.

    Computing Grundy Values with Matrices

    Consider a game on a directed graph with n nodes. Define the n × n adjacency matrix A over GF(2). The Grundy value of each node can be computed bottom-up (from terminal positions). But when the game has a periodic or recurrent structure, we can encode the Grundy-value recurrence as a matrix equation and solve it with exponentiation. 

    // Matrix over GF(2): entries are bits, + is XOR, * is AND
struct GF2Matrix {
    int n;
    vector<vector<int>> a;

    GF2Matrix(int n) : n(n), a(n, vector<int>(n, 0)) {}

    GF2Matrix operator*(const GF2Matrix& o) const {
        GF2Matrix res(n);
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++) if (a[i][k])
                for (int j = 0; j < n; j++)
                    res.a[i][j] ^= o.a[k][j];
        return res;
    }
};

GF2Matrix power(GF2Matrix m, long long p) {
    int n = m.n;
    GF2Matrix res(n);
    for (int i = 0; i < n; i++) res.a[i][i] = 1; // identity
    while (p > 0) {
        if (p & 1) res = res * m;
        m = m * m;
        p >>= 1;
    }
    return res;
}

    Bitset Optimization

    Since GF(2) entries are single bits, we can pack each row into a bitset and use bitwise operations for an enormous constant-factor speedup — from O(n³) to O(n³/64): 

    const int MAXN = 512;
struct BitMatrix {
    int n;
    bitset<MAXN> row[MAXN];

    BitMatrix operator*(const BitMatrix& o) const {
        BitMatrix res;
        res.n = n;
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++)
                if (row[i][k])
                    res.row[i] ^= o.row[k];
        return res;
    }
};

int gf2_rank(BitMatrix& m) {
    int rank = 0;
    for (int col = 0; col < m.n; col++) {
        int pivot = -1;
        for (int r = rank; r < m.n; r++)
            if (m.row[r][col]) { pivot = r; break; }
        if (pivot == -1) continue;
        swap(m.row[rank], m.row[pivot]);
        for (int r = 0; r < m.n; r++)
            if (r != rank && m.row[r][col])
                m.row[r] ^= m.row[rank];
        rank++;
    }
    return rank;
}

    Game State Transitions as Matrices

    The core insight: if a game has a finite set of states {0, 1, …, n−1}, then every legal move can be represented as a directed edge. The entire move structure is captured in the transition matrix T.

    Boolean Reachability

    For the simplest case — "can the first player force a visit to state s within k moves?" — we compute Tk over boolean (OR/AND) arithmetic. Entry Tk[i][j] is 1 if and only if there exists a path of exactly k edges from i to j.

    Counting Paths

    Over the integers (or modulo a prime), Tk[i][j] counts the number of distinct paths of length k from state i to state j. This is the classic matrix exponentiation application. 

    using ll = long long;
const ll MOD = 1e9 + 7;

struct Matrix {
    int n;
    vector<vector<ll>> a;

    Matrix(int n) : n(n), a(n, vector<ll>(n, 0)) {}

    Matrix operator*(const Matrix& o) const {
        Matrix res(n);
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++) if (a[i][k])
                for (int j = 0; j < n; j++)
                    res.a[i][j] = (res.a[i][j] + a[i][k] * o.a[k][j]) % MOD;
        return res;
    }
};

Matrix mat_pow(Matrix m, ll p) {
    int n = m.n;
    Matrix res(n);
    for (int i = 0; i < n; i++) res.a[i][i] = 1;
    while (p > 0) {
        if (p & 1) res = res * m;
        m = m * m;
        p >>= 1;
    }
    return res;
}

    Encoding Game Rules

    Suppose a token starts at position 0 on a line of n cells. Each turn a player moves the token 1, 2, or 3 cells forward. The game ends when the token reaches or passes cell n−1. To count the number of distinct game sequences of exactly k moves:

    • Build an n × n matrix T where T[i][j] = 1 if j − i ∈ {1, 2, 3} and j < n.
    • Compute Tk and read off the answer from entry [0][n−1].

    When n is small (say ≤ 100) but k is enormous (up to 1018), matrix exponentiation is the only feasible approach.

    Matrix Exponentiation for Games

    Matrix exponentiation is the workhorse technique for game problems where the number of turns k is astronomically large. The idea is always the same:

    1. Identify the state space (size n).
    2. Build the transition matrix T (size n × n).
    3. Compute Tk via binary exponentiation in O(n³ log k).
    4. Multiply by the initial state vector to get the final state.

    Game on a Graph: Winning After Exactly k Moves

    Problem. A directed graph has n nodes. Two players alternate moving a token along edges. A player who cannot move loses. Given starting node s, determine whether the first player wins if the game lasts exactly k moves.

    Partition nodes into W (winning) and L (losing) positions using standard Sprague-Grundy analysis. Then define a vector v of size n where v[s] = 1 and all other entries 0. After k matrix-vector multiplications with the adjacency matrix, check which positions are reachable and whether they are winning or losing. 

    // Determine if first player can win starting from node s after exactly k moves
// adj[i] = list of neighbours of node i
bool can_first_player_win(int n, vector<vector<int>>& adj, int s, ll k) {
    // Compute Grundy values via BFS/DFS on the DAG
    vector<int> grundy(n, -1);
    vector<int> order; // topological order
    vector<int> indeg(n, 0);
    // ... (standard topo sort + mex computation) ...

    // Build transition matrix
    Matrix T(n);
    for (int i = 0; i < n; i++)
        for (int j : adj[i])
            T.a[i][j] = 1;

    Matrix Tk = mat_pow(T, k);

    // Check: is there any path of length k from s to a losing position?
    for (int j = 0; j < n; j++)
        if (Tk.a[s][j] > 0 && grundy[j] == 0)
            return true; // can reach a losing position for opponent
    return false;
}

    Periodicity Detection

    A subtle but powerful trick: if the game's Grundy sequence is eventually periodic with period p, we only need to compute Grundy values for the first pre + p positions, then answer queries by reducing k modulo p. Finding the period itself can sometimes be done by computing the order of the transition matrix — the smallest p such that Tp = I.

    Two-Player Zero-Sum Games

    While Sprague-Grundy theory handles combinatorial games, many CP problems involve payoff matrices — the domain of classical game theory. A two-player zero-sum game is defined by an m × n payoff matrix A: the row player picks row i, the column player picks column j, and the row player receives payoff A[i][j] (the column player loses the same).

    Minimax Theorem

    Von Neumann's minimax theorem guarantees that in a zero-sum game:

    maxi minj A[i][j] = minj maxi A[i][j] = v

    The value v is the game value. In pure strategies, the row player maximizes the minimum payoff (maximin), and the column player minimizes the maximum payoff (minimax). If the maximin equals the minimax, a saddle point exists and pure optimal strategies suffice.

    Mixed Strategies and Nash Equilibrium

    When no saddle point exists, players randomize. A mixed strategy is a probability distribution over actions. The Nash equilibrium for a 2×2 game with payoff matrix:

    A = [[a, b], [c, d]]

    has the row player mixing with probability p = (d − c) / (a − b − c + d) on the first row. The game value is v = (ad − bc) / (a − b − c + d). These formulas appear directly in CP problems.

    Finding Saddle Points

    // Find saddle point in a payoff matrix (if it exists)
// Returns {row, col} or {-1, -1} if no saddle point
pair<int,int> find_saddle_point(vector<vector<int>>& A) {
    int m = A.size(), n = A[0].size();
    for (int i = 0; i < m; i++) {
        int min_val = *min_element(A[i].begin(), A[i].end());
        for (int j = 0; j < n; j++) {
            if (A[i][j] != min_val) continue;
            int max_in_col = A[0][j];
            for (int r = 1; r < m; r++)
                max_in_col = max(max_in_col, A[r][j]);
            if (A[i][j] == max_in_col)
                return {i, j};
        }
    }
    return {-1, -1};
}

    LP Connection

    Computing the optimal mixed strategy for larger games reduces to a linear program. The row player's LP is:

    • Maximize v
    • Subject to: for each column j, Σᵢ pᵢ · A[i][j] ≥ v
    • Σᵢ pᵢ = 1, pᵢ ≥ 0

    In contests, you rarely need to implement a full LP solver for this. Most game theory problems are either 2×2 (closed-form) or reduce to Sprague-Grundy analysis. But knowing the LP formulation helps recognize the structure.

    CP Problem Walkthrough

    Problem: Game on a Circular Board

    A token sits on a circular board of n cells (0 to n−1). Two players alternate turns. On each turn, a player moves the token forward by 1, 2, or 3 cells (modulo n). A player who moves the token to cell 0 wins. Given the starting cell s and a constraint that the game must last exactly k turns, count the number of distinct game sequences where the first player wins (modulo 10⁹ + 7).

    Constraints: n ≤ 50, k ≤ 1018.

    Analysis

    The state is the current cell. The transition matrix T is n × n with T[i][(i+1) % n] = T[i][(i+2) % n] = T[i][(i+3) % n] = 1 for all i ≠ 0. Cell 0 is absorbing once reached — but we need to handle the "win" condition carefully.

    We split the problem into two layers: active states (token not yet at 0) and a terminal state (token at 0, game over). We modify the matrix so that cell 0 has no outgoing transitions (absorbing). Then Tk[s][0] counts paths that land on 0 for the first time at exactly step k... but that's not quite right — it counts paths that reach 0 at step k, possibly having visited 0 earlier.

    To count paths that reach 0 for the first time at step k: make cell 0 absorbing. Then Tk[s][0] counts all paths from s that reach 0 at some step ≤ k. Use subtraction: first-visit-at-k = Tk[s][0] − Tk−1[s][0].

    Since the first player wins by moving to 0 on their turn (odd steps 1, 3, 5, … for the first player), we sum over all odd k values. But with k fixed, we simply check: if k is odd, the answer is the number of paths where the first player lands on 0 at step k.

    Full Solution

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;
const ll MOD = 1e9 + 7;

struct Matrix {
    int n;
    vector<vector<ll>> a;
    Matrix(int n) : n(n), a(n, vector<ll>(n, 0)) {}
    Matrix operator*(const Matrix& o) const {
        Matrix res(n);
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++) if (a[i][k])
                for (int j = 0; j < n; j++)
                    res.a[i][j] = (res.a[i][j] + a[i][k] * o.a[k][j]) % MOD;
        return res;
    }
};

Matrix mat_pow(Matrix m, ll p) {
    int n = m.n;
    Matrix res(n);
    for (int i = 0; i < n; i++) res.a[i][i] = 1;
    while (p > 0) {
        if (p & 1) res = res * m;
        m = m * m;
        p >>= 1;
    }
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, s;
    ll k;
    cin >> n >> s >> k;

    // Build transition matrix: cell 0 is absorbing (no outgoing edges)
    Matrix T(n);
    for (int i = 1; i < n; i++) {
        T.a[i][(i + 1) % n] = 1;
        T.a[i][(i + 2) % n] = 1;
        T.a[i][(i + 3) % n] = 1;
    }

    // T^k[s][0] = number of paths from s reaching 0 by step k
    // T^(k-1)[s][0] = number reaching 0 by step k-1
    // Difference = paths that first reach 0 at exactly step k

    Matrix Tk = mat_pow(T, k);
    ll ans_k = Tk.a[s][0];

    if (k >= 1) {
        Matrix Tk1 = mat_pow(T, k - 1);
        ll ans_k1 = Tk1.a[s][0];
        ll first_visit = (ans_k - ans_k1 % MOD + MOD) % MOD;
        // First player wins if k is odd (they make the k-th move)
        if (k % 2 == 1)
            cout << first_visit << "\n";
        else
            cout << 0 << "\n";
    } else {
        cout << (s == 0 ? 1 : 0) << "\n";
    }

    return 0;
}

    Complexity: O(n³ log k) for each matrix exponentiation. With n ≤ 50 and k ≤ 1018, this runs in roughly 50³ × 60 ≈ 7.5 × 10⁶ operations per exponentiation — well within limits.

    Variations and Extensions

    • Multiple tokens: If several independent tokens move on separate boards, the combined Grundy value is the XOR of individual Grundy values. Use the GF(2) matrix approach from section 2.
    • Different move sets per cell: The transition matrix simply has different non-zero patterns per row.
    • Weighted games: Replace 0/1 entries with move costs and change the semiring (min-plus for shortest game, max-plus for longest).
    • Bidirectional moves: If both players share the same move set, the matrix is the same for both. If different, alternate multiplying by two different matrices: the answer uses T₁ · T₂ raised to the k/2 power.

    • Codeforces 1450E — Capitalism — Game theory on a graph with parity constraints. Think about how the transition matrix parity (odd/even path lengths) determines the winner.
    • AtCoder DP Contest — Walk — Count paths of length k in a directed graph using matrix exponentiation. The definitive introductory problem for this technique.

    Tip: When approaching a game problem, always ask: (1) Can I compute Grundy values? (2) Is the state space small enough for a transition matrix? (3) Is the number of turns large enough to need exponentiation? If all three answers are yes, you have a matrix game problem.
