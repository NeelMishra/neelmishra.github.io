    LU Decomposition

    July 2026 · 22 min read

    What Is LU Decomposition?

    LU Decomposition factors a square matrix A into the product of two triangular matrices:

    A = L · U

    where L is lower triangular (all entries above the diagonal are zero) and U is upper triangular (all entries below the diagonal are zero). For a 3×3 matrix the factorization looks like:

    A = | a11  a12  a13 |   | 1    0    0 |   | u11  u12  u13 |
    | a21  a22  a23 | = | l21  1    0 | * | 0    u22  u23 |
    | a31  a32  a33 |   | l31  l32  1 |   | 0    0    u33 |

    In the Doolittle form (the most common convention), the diagonal of L is all ones. The Crout form instead forces the diagonal of U to be all ones. Both are equivalent; we use Doolittle throughout this post.

    Why Factor a Matrix?

    The key insight is that triangular systems are cheap to solve. Given Ax = b, once we have A = LU:

    1. Solve Ly = b by forward substitution — O(n²).
    2. Solve Ux = y by back substitution — O(n²).

    The factorization itself costs O(n³), but it is done once. Each additional right-hand side b only costs O(n²). This is a massive win in competitive programming problems that ask you to solve many systems sharing the same coefficient matrix.

    Comparison with Gaussian Elimination. Gaussian elimination on Ax = b is O(n³) per system. LU decomposition is essentially Gaussian elimination with the intermediate steps recorded, so you pay O(n³) once and O(n²) per additional solve.

    Doolittle's Algorithm

    Doolittle's algorithm computes L and U in-place by processing column by column. For an n×n matrix the recurrences are:

    Upper triangular entries (row i ≤ column j):

    u[i][j] = a[i][j] - sum(l[i][k] * u[k][j], k = 0 .. i-1)

    Lower triangular entries (row i > column j):

    l[i][j] = (a[i][j] - sum(l[i][k] * u[k][j], k = 0 .. j-1)) / u[j][j]

    Because L has ones on its diagonal, we never need to compute or store l[i][i].

    Worked Example

    Decompose the matrix:

    A = | 2   1   1 |
    | 4   3   3 |
    | 8   7   9 |

    Column 0 of U: u[0][0] = 2, u[0][1] = 1, u[0][2] = 1 (first row copied).

    Column 0 of L: l[1][0] = 4/2 = 2, l[2][0] = 8/2 = 4.

    Row 1 of U: u[1][1] = 3 − 2·1 = 1, u[1][2] = 3 − 2·1 = 1.

    Column 1 of L: l[2][1] = (7 − 4·1)/1 = 3.

    Row 2 of U: u[2][2] = 9 − 4·1 − 3·1 = 2.

    L = | 1  0  0 |    U = | 2  1  1 |
    | 2  1  0 |        | 0  1  1 |
    | 4  3  1 |        | 0  0  2 |

    Verify: L · U gives back A. ✓

    C++ Implementation

    #include <bits/stdc++.h>
using namespace std;

using T = double;
const T EPS = 1e-9;

// Doolittle LU decomposition (no pivoting).
// Stores L and U in-place: L's strict lower triangle and U's upper triangle.
// Returns false if a zero pivot is encountered.
bool lu_decompose(vector<vector<T>>& A) {
    int n = A.size();
    for (int j = 0; j < n; j++) {
        // Upper triangular part: U[i][j] for i <= j
        for (int i = 0; i <= j; i++) {
            T sum = 0;
            for (int k = 0; k < i; k++)
                sum += A[i][k] * A[k][j];
            A[i][j] -= sum;
        }
        if (abs(A[j][j]) < EPS) return false;  // singular or needs pivoting
        // Lower triangular part: L[i][j] for i > j
        for (int i = j + 1; i < n; i++) {
            T sum = 0;
            for (int k = 0; k < j; k++)
                sum += A[i][k] * A[k][j];
            A[i][j] = (A[i][j] - sum) / A[j][j];
        }
    }
    return true;
}

    After calling lu_decompose(A), the matrix A holds both L and U compactly: the strict lower triangle stores L (the diagonal of L is implicitly 1), and the upper triangle (including diagonal) stores U.

    Complexity: O(n³) time and O(1) extra space (in-place in the input matrix).

    LU with Partial Pivoting (PA = LU)

    Plain Doolittle fails when a zero (or near-zero) pivot appears on the diagonal. Even when pivots are nonzero, small pivots amplify rounding errors. Partial pivoting fixes this by swapping rows so the largest element in the current column becomes the pivot.

    The decomposition becomes:

    PA = LU

    where P is a permutation matrix recording the row swaps. In code we store P as an integer array perm[] where perm[i] is the original row index that ended up in position i.

    Algorithm Outline

    1. For each column j, find the row with the largest absolute value in column j (among rows j..n−1).
    2. Swap that row with row j (in both the matrix and the permutation array).
    3. Proceed with the standard Doolittle step for column j.

    C++ Implementation

    // LU decomposition with partial pivoting.
// After the call:
//   A stores L (strict lower, diagonal implicitly 1) and U (upper incl. diagonal).
//   perm[i] = original row index in position i.
//   sign = +1 or -1 (parity of row swaps, useful for determinant sign).
// Returns false if the matrix is singular.
bool lu_pivot(vector<vector<T>>& A, vector<int>& perm, int& sign) {
    int n = A.size();
    perm.resize(n);
    iota(perm.begin(), perm.end(), 0);
    sign = 1;

    for (int j = 0; j < n; j++) {
        // Find pivot: row with max |A[i][j]| for i in [j, n)
        int best = j;
        for (int i = j + 1; i < n; i++)
            if (abs(A[i][j]) > abs(A[best][j]))
                best = i;

        if (abs(A[best][j]) < EPS) return false;  // singular

        if (best != j) {
            swap(A[j], A[best]);
            swap(perm[j], perm[best]);
            sign = -sign;
        }

        // Eliminate below pivot
        for (int i = j + 1; i < n; i++) {
            A[i][j] /= A[j][j];
            for (int k = j + 1; k < n; k++)
                A[i][k] -= A[i][j] * A[j][k];
        }
    }
    return true;
}

    This implementation performs the elimination in the standard Gaussian style (rather than the separate-loop Doolittle style) because it is more natural when pivoting is involved. The lower triangle of A stores the multipliers (i.e., L without the unit diagonal), and the upper triangle stores U.

    When Does Pivoting Matter?

    ScenarioWithout PivotingWith Pivoting
    Zero on diagonalDivision by zero — crashHandled by row swap
    Small pivot (e.g., 10−15)Catastrophic rounding errorNumerically stable
    Integer entries, exact arithmeticUsually fineStill recommended for robustness
    CP Tip: Always use partial pivoting unless the problem guarantees that the matrix is diagonally dominant or all pivots are nonzero. The overhead is negligible and it prevents wrong-answer verdicts from numerical issues.

    Solving Systems with LU

    Once we have PA = LU, solving Ax = b becomes two triangular solves:

    1. Permute b: Let b′ = Pb (reorder entries according to the permutation).
    2. Forward substitution (Ly = b′): Solve for y from top to bottom.
    3. Back substitution (Ux = y): Solve for x from bottom to top.

    Forward Substitution

    Since L is lower triangular with unit diagonal:

    y[0] = b'[0]
y[i] = b'[i] - sum(L[i][k] * y[k], k = 0 .. i-1)   for i = 1..n-1

    Back Substitution

    Since U is upper triangular:

    x[n-1] = y[n-1] / U[n-1][n-1]
x[i] = (y[i] - sum(U[i][k] * x[k], k = i+1 .. n-1)) / U[i][i]   for i = n-2..0

    C++ Implementation

    // Solve Ax = b given the LU decomposition (with pivoting) of A.
// A is the in-place LU matrix, perm is the permutation from lu_pivot.
// Returns the solution vector x.
vector<T> lu_solve(const vector<vector<T>>& A,
                    const vector<int>& perm,
                    const vector<T>& b) {
    int n = A.size();
    vector<T> y(n), x(n);

    // Forward substitution: Ly = Pb
    for (int i = 0; i < n; i++) {
        y[i] = b[perm[i]];
        for (int k = 0; k < i; k++)
            y[i] -= A[i][k] * y[k];
    }

    // Back substitution: Ux = y
    for (int i = n - 1; i >= 0; i--) {
        x[i] = y[i];
        for (int k = i + 1; k < n; k++)
            x[i] -= A[i][k] * x[k];
        x[i] /= A[i][i];
    }

    return x;
}

    Complete Workflow Example

    int main() {
    int n;
    cin >> n;
    vector<vector<T>> A(n, vector<T>(n));
    for (auto& row : A)
        for (auto& x : row)
            cin >> x;

    vector<int> perm;
    int sign;
    if (!lu_pivot(A, perm, sign)) {
        cout << "Singular matrix\n";
        return 0;
    }

    int q;
    cin >> q;
    while (q--) {
        vector<T> b(n);
        for (auto& x : b) cin >> x;

        vector<T> x = lu_solve(A, perm, b);
        for (int i = 0; i < n; i++)
            cout << fixed << setprecision(6) << x[i] << " \n"[i == n - 1];
    }
}

    This reads a matrix once, factors it, then answers q queries in O(n²) each. Total cost: O(n³ + q·n²).

    Multiple Right-Hand Sides. This is the primary competitive programming use case. Gaussian elimination would cost O(q·n³), while LU brings it down to O(n³ + q·n²) — a factor-of-n speedup when q is large.

    Computing Determinant from LU

    The determinant has a beautiful relationship with LU decomposition. Since PA = LU:

    det(P) · det(A) = det(L) · det(U)

    Because L has unit diagonal, det(L) = 1. The determinant of a triangular matrix is the product of its diagonal, so det(U) = ∏ U[i][i]. And det(P) = (−1)s where s is the number of row swaps.

    Therefore:

    det(A) = (−1)s · ∏i=0n−1 U[i][i]

    C++ Implementation

    // Compute determinant from the LU factorization.
// A is the in-place LU matrix from lu_pivot, sign is +1 or -1.
T lu_determinant(const vector<vector<T>>& A, int sign) {
    T det = sign;
    for (int i = 0; i < (int)A.size(); i++)
        det *= A[i][i];
    return det;
}

    Integer Determinant via LU

    For integer matrices, floating-point LU can lose precision. A common trick is to compute the LU decomposition modulo a prime, using modular arithmetic:

    const long long MOD = 998244353;

long long power(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long mod_inv(long long a, long long mod) {
    return power(a, mod - 2, mod);
}

// LU decomposition mod p.  Returns determinant mod p.
long long lu_det_mod(vector<vector<long long>> A, long long mod) {
    int n = A.size();
    long long det = 1;

    for (int j = 0; j < n; j++) {
        // Partial pivoting
        int pivot = -1;
        for (int i = j; i < n; i++) {
            if (A[i][j] % mod != 0) { pivot = i; break; }
        }
        if (pivot == -1) return 0;  // singular

        if (pivot != j) {
            swap(A[j], A[pivot]);
            det = (mod - det) % mod;  // flip sign
        }

        det = det % mod * (A[j][j] % mod) % mod;
        long long inv = mod_inv(A[j][j], mod);

        for (int i = j + 1; i < n; i++) {
            long long factor = A[i][j] % mod * inv % mod;
            for (int k = j; k < n; k++)
                A[i][k] = (A[i][k] - factor % mod * A[j][k] % mod + mod * 2) % mod;
        }
    }
    return (det % mod + mod) % mod;
}
    Note: Many CP problems ask for the determinant modulo a prime. The modular LU approach is exact and avoids all floating-point issues. Time complexity remains O(n³).

    When to Use LU in CP

    LU decomposition is not the most common tool in competitive programming — plain Gaussian elimination usually suffices for one-off systems. But LU shines in specific scenarios:

    1. Multiple Systems, Same Coefficient Matrix

    If a problem gives you a fixed matrix A and asks you to solve Ax = b for many different b vectors, LU is the way to go. Factorize once in O(n³), then each solve is O(n²).

    Example: A network flow problem where the topology is fixed but demands change across queries.

    2. Repeated Determinant Queries

    If you need det(A) after small updates to A (e.g., changing one row), you can use the matrix determinant lemma together with LU to update the determinant efficiently without refactoring from scratch.

    3. Computing the Inverse

    To compute A−1, solve Axi = ei for each standard basis vector ei. With LU, each column of the inverse costs O(n²), so the total inverse computation is O(n³) — the same as a single Gauss-Jordan elimination but with better numerical stability due to pivoting.

    4. Kirchhoff's Theorem Follow-Up

    In the previous post we saw that counting spanning trees requires computing the determinant of a Laplacian minor. LU decomposition (modular version) is the standard way to compute this determinant in O(n³).

    Complexity Comparison

    TaskGaussian EliminationLU Decomposition
    Solve 1 systemO(n³)O(n³)
    Solve q systems (same A)O(q·n³)O(n³ + q·n²)
    DeterminantO(n³)O(n³)
    Matrix inverseO(n³)O(n³), better stability

    Implementation Tips for Contests

    • Use long long with modular arithmetic for integer problems to avoid floating-point entirely.
    • Pre-allocate arrays if n is known at compile time — a fixed 2D array is faster than vector of vectors.
    • Combine LU + solve into one template. Keep a single struct that stores A, perm, and sign so you can solve and compute determinant from the same factorization.
    • Watch for singular matrices. If lu_pivot returns false, the system is either inconsistent or has infinitely many solutions. Handle this case explicitly.

    LU Decomposition Struct Template

    struct LU {
    int n, sign;
    vector<vector<T>> A;
    vector<int> perm;
    bool singular;

    LU(vector<vector<T>> mat) : n(mat.size()), A(move(mat)) {
        singular = !lu_pivot(A, perm, sign);
    }

    T det() const {
        if (singular) return 0;
        return lu_determinant(A, sign);
    }

    vector<T> solve(const vector<T>& b) const {
        assert(!singular);
        return lu_solve(A, perm, b);
    }
};
