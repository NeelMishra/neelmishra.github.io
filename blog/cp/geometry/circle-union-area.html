    Area of Union of Circles

    July 2025 · 20 min read

    Computing the area of the union of n circles is a classic computational geometry problem that appears in contests like ICPC and Codeforces. Unlike simple polygon union, circle union requires careful treatment of curved boundaries, arc integration, and containment detection. We'll build up from the two-circle case to a general O(n² log n) algorithm using angular sweep.

    Overview of approaches:

    • Inclusion-exclusion: Exact but exponential in the number of circles. Only practical for very small n (say, n ≤ 10).
    • Angular sweep: Process each circle's boundary to find arcs not covered by other circles. O(n² log n) time. The standard approach.
    • Green's theorem: Integrate along the union boundary using the line integral formula for area. Elegant and combines well with angular sweep.
    • Monte Carlo: Random sampling for approximate area. Useful for verification but not for exact answers.

    Two-Circle Intersection Area

    Before tackling the general case, let's solve the foundation: the area of intersection of exactly two circles. Given circles (O₁, r₁) and (O₂, r₂) with center distance d = |O₂ − O₁|:

    • If d ≥ r₁ + r₂: intersection area is 0.
    • If d + min(r₁, r₂) ≤ max(r₁, r₂): one circle contains the other; intersection area is π · min(r₁, r₂)².
    • Otherwise: the intersection is a lens-shaped region formed by two circular segments.
    double circleIntersectionArea(Point O1, double r1, Point O2, double r2) {
    double d = (O2 - O1).norm();
    if (d >= r1 + r2) return 0.0;
    if (d + r2 <= r1) return M_PI * r2 * r2;
    if (d + r1 <= r2) return M_PI * r1 * r1;

    double alpha = acos((r1*r1 + d*d - r2*r2) / (2.0*r1*d));
    double beta  = acos((r2*r2 + d*d - r1*r1) / (2.0*r2*d));

    // Each circular segment: sector area minus triangle area
    double area1 = r1*r1 * (alpha - sin(2*alpha)/2.0);
    double area2 = r2*r2 * (beta  - sin(2*beta)/2.0);

    return area1 + area2;
}

    The angles α and β are the half-angles subtended at each center by the intersection chord. The area of each circular segment equals the sector area minus the triangle area.

    For the union of two circles: Aunion = πr₁² + πr₂² − Aintersection.

    Angular Sweep Method

    For n circles, inclusion-exclusion becomes exponential. Instead, we use an angular sweep approach that processes each circle independently: for each circle Ci, we determine which arcs of its boundary are on the outer boundary of the union (not covered by any other circle).

    The algorithm for each circle Ci:

    1. For each other circle Cj, compute the arc of Ci that is covered by Cj.
    2. Merge all covered arcs (as angular intervals on [−π, π)).
    3. The uncovered arcs contribute to the union boundary.
    4. For each uncovered arc, add the corresponding circular segment area and the signed polygon area contribution.
    struct Circle {
    Point o;
    double r;
};

// Compute covered angular intervals on circle i by circle j
// Returns {start_angle, end_angle} or empty if no overlap
pair<double,double> coveredArc(const Circle& ci, const Circle& cj) {
    Point d = cj.o - ci.o;
    double dist = d.norm();

    if (dist >= ci.r + cj.r) return {0, 0};         // no overlap
    if (dist + ci.r <= cj.r) return {-M_PI, M_PI};   // ci fully inside cj

    double ang = atan2(d.y, d.x);
    double delta = acos((ci.r*ci.r + dist*dist - cj.r*cj.r) / (2*ci.r*dist));

    return {ang - delta, ang + delta};
}

    Containment Cases

    Special care is needed when one circle is entirely contained within another:

    • If Ci is fully contained in some Cj, then Ci's entire boundary is covered. It contributes no area to the union boundary.
    • If Cj is fully contained in Ci, then Cj does not cover any arc of Ci.
    • Identical circles should be deduplicated: keep only one copy.
    // Remove circles fully contained by others
vector<Circle> removeContained(vector<Circle> circles) {
    int n = circles.size();
    vector<bool> removed(n, false);

    // Sort by radius descending for efficiency
    sort(circles.begin(), circles.end(),
         [](const Circle& a, const Circle& b) { return a.r > b.r; });

    for (int i = 0; i < n; i++) {
        if (removed[i]) continue;
        for (int j = i + 1; j < n; j++) {
            if (removed[j]) continue;
            double d = (circles[i].o - circles[j].o).norm();
            if (d + circles[j].r <= circles[i].r + 1e-9) {
                removed[j] = true;
            }
        }
    }

    vector<Circle> result;
    for (int i = 0; i < n; i++)
        if (!removed[i]) result.push_back(circles[i]);
    return result;
}

    Tip: Always perform containment removal as a preprocessing step. This reduces the number of circles and eliminates many edge cases.

    Green's Theorem Approach

    The most elegant method for computing the area of the union uses Green's theorem. The area enclosed by a closed curve is:

    A = ½ ∮ (x dy − y dx)

    The union boundary consists of line segments (chords between intersection points) and circular arcs. For each piece, we compute the signed area contribution.

    Arc contribution: For an arc of circle (O, r) from angle θ₁ to θ₂:

    // Signed area contribution of circular arc from angle a1 to a2
double arcArea(Point O, double r, double a1, double a2) {
    // Parametric: x = O.x + r*cos(t), y = O.y + r*sin(t)
    // integral of (x dy - y dx) / 2
    double dtheta = a2 - a1;
    // Sector part: r^2 * dtheta / 2
    double sector = r * r * dtheta / 2.0;
    // Cross term with center offset
    double cross_term = r * (O.x * (sin(a2) - sin(a1))
                            - O.y * (cos(a2) - cos(a1)));
    return sector + cross_term;
}

    Chord contribution: For a line segment from point P to point Q:

    // Signed area contribution of line segment P -> Q
double segmentArea(Point P, Point Q) {
    return (P.x * Q.y - Q.x * P.y) / 2.0;
}

    Full Implementation

    Putting it all together, here is the complete algorithm for computing the area of the union of n circles: 

    double circleUnionArea(vector<Circle> circles) {
    circles = removeContained(circles);
    int n = circles.size();
    if (n == 0) return 0.0;

    double totalArea = 0.0;

    for (int i = 0; i < n; i++) {
        // Collect all covered intervals on circle i
        vector<pair<double,double>> intervals;

        for (int j = 0; j < n; j++) {
            if (i == j) continue;
            auto [a, b] = coveredArc(circles[i], circles[j]);
            if (fabs(a - b) < 1e-12) continue;

            // Normalize to [-pi, pi)
            if (a < -M_PI) {
                intervals.push_back({a + 2*M_PI, M_PI});
                intervals.push_back({-M_PI, b});
            } else if (b > M_PI) {
                intervals.push_back({a, M_PI});
                intervals.push_back({-M_PI, b - 2*M_PI});
            } else {
                intervals.push_back({a, b});
            }
        }

        // Merge intervals
        sort(intervals.begin(), intervals.end());
        vector<pair<double,double>> merged;
        for (auto& seg : intervals) {
            if (!merged.empty() && seg.first <= merged.back().second + 1e-12)
                merged.back().second = max(merged.back().second, seg.second);
            else
                merged.push_back(seg);
        }

        // Process free arcs
        double cur = -M_PI;
        auto& c = circles[i];

        for (auto& seg : merged) {
            if (cur < seg.first - 1e-12) {
                // Free arc from cur to seg.first
                totalArea += arcArea(c.o, c.r, cur, seg.first);
                // Add chord contribution
                Point P = {c.o.x + c.r*cos(cur), c.o.y + c.r*sin(cur)};
                Point Q = {c.o.x + c.r*cos(seg.first),
                           c.o.y + c.r*sin(seg.first)};
                totalArea += segmentArea(Q, P);  // reversed for correct sign
            }
            cur = max(cur, seg.second);
        }

        // Final free arc from cur to pi
        if (cur < M_PI - 1e-12) {
            totalArea += arcArea(c.o, c.r, cur, M_PI);
            // Complete the loop back to -pi
            totalArea += arcArea(c.o, c.r, -M_PI, -M_PI);
            Point P = {c.o.x + c.r*cos(cur), c.o.y + c.r*sin(cur)};
            Point Q = {c.o.x + c.r*cos(-M_PI), c.o.y + c.r*sin(-M_PI)};
            totalArea += segmentArea(Q, P);
        }
    }

    return fabs(totalArea);
}

    Complexity: O(n² log n) — for each of the n circles, we process O(n) intervals and sort them in O(n log n).

    Complexity: The overall algorithm is O(n² log n). For most contest problems with n ≤ 1000, this is efficient enough. For very large n, specialized approaches (e.g., plane sweep on circle events) can reduce this, but they are rarely needed in contests.

    Testing & Verification

    Circle union area is notoriously tricky to implement correctly. Here are strategies for verifying your solution during a contest:

    Test case 1: Single circle. Area should be πr².

    // Single circle: r = 5
// Expected: pi * 25 ≈ 78.5398
vector<Circle> test1 = {{{0, 0}, 5}};
assert(fabs(circleUnionArea(test1) - M_PI * 25) < 1e-6);

    Test case 2: Two identical circles. Area should be πr² (not doubled).

    // Two identical circles at same position
vector<Circle> test2 = {{{0, 0}, 3}, {{0, 0}, 3}};
assert(fabs(circleUnionArea(test2) - M_PI * 9) < 1e-6);

    Test case 3: Two non-overlapping circles. Area should be the sum of individual areas.

    // Two circles far apart
vector<Circle> test3 = {{{0, 0}, 1}, {{10, 0}, 1}};
assert(fabs(circleUnionArea(test3) - 2 * M_PI) < 1e-6);

    Test case 4: One circle inside another. Area should be the larger circle’s area.

    // Small circle inside large circle
vector<Circle> test4 = {{{0, 0}, 5}, {{1, 0}, 2}};
assert(fabs(circleUnionArea(test4) - M_PI * 25) < 1e-6);

    Test case 5: Two touching circles. Good for checking the tangent boundary case.

    // Externally tangent circles
vector<Circle> test5 = {{{0, 0}, 3}, {{5, 0}, 2}};
// Should equal pi*9 + pi*4 = 13*pi ≈ 40.8407
assert(fabs(circleUnionArea(test5) - 13 * M_PI) < 1e-6);

    Common pitfalls:

    • Angle wrapping around −π and π — always handle intervals that cross the boundary.
    • Forgetting to handle the case where a circle is fully covered (all arcs are covered intervals spanning [−π, π)).
    • Sign errors in Green’s theorem — make sure the boundary is traversed counterclockwise for positive area.
    • Identical circles at the same position — deduplicate before processing.
    • When the number of circles is 0, return 0 immediately.
    • Floating-point precision in acos calls — always clamp the argument to [-1, 1].
