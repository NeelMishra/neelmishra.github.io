    Closest Pair of Points

    July 2025 · 22 min read

    Problem Definition

    Given n points in the plane, find the pair with the smallest Euclidean distance. This classic problem appears in computational geometry courses and competitive programming contests alike.

    • Input: n points (xi, yi)
    • Output: The minimum distance between any two distinct points
    • Constraint: n can be up to 10⁵ or 10⁶, so O(n²) is too slow

    We'll explore three approaches: brute force O(n²), divide & conquer O(n log n), and sweep line O(n log n).

    Among all pairs, the green pair has the smallest Euclidean distance.

    Brute Force: O(n²)

    Check every pair of points. Simple but slow for large inputs.

    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
struct Point { ll x, y; };

ll dist2(Point a, Point b) {
    return (a.x - b.x) * (a.x - b.x)
         + (a.y - b.y) * (a.y - b.y);
}

// Returns the squared minimum distance
ll brute_force(vector<Point>& pts) {
    int n = pts.size();
    ll best = LLONG_MAX;
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            best = min(best, dist2(pts[i], pts[j]));
    return best;
}

    We use squared distances to avoid floating-point issues and unnecessary square roots. This is a standard competitive programming trick.

    When brute force is fine: For n ≤ 2000, O(n²) runs in under a second. Always consider the constraints before implementing a complex algorithm.

    Divide and Conquer: O(n log n)

    The classic algorithm by Shamos and Hoey (1975):

    Algorithm Overview

    1. Sort all points by x-coordinate.
    2. Divide the point set into two halves at the median x-coordinate.
    3. Conquer: recursively find the closest pair in each half. Let δ = min(δleft, δright).
    4. Combine: check if there's a closer pair that spans the dividing line. Only points within distance δ of the dividing line need to be checked.

    The strip around the dividing line (width 2δ) is where cross-half closest pairs might exist.

    The Combine Step

    Collect all points within distance δ of the dividing line into a strip. Sort them by y-coordinate. For each point in the strip, compare it only with the next few points in y-order.

    double closest_pair(vector<Point>& pts, int lo, int hi,
                    vector<Point>& strip) {
    if (hi - lo <= 3) {
        // Brute force for small sets
        double best = 1e18;
        for (int i = lo; i < hi; i++)
            for (int j = i + 1; j < hi; j++)
                best = min(best, dist(pts[i], pts[j]));
        sort(pts.begin() + lo, pts.begin() + hi,
             [](const Point& a, const Point& b){ return a.y < b.y; });
        return best;
    }

    int mid = (lo + hi) / 2;
    ll midx = pts[mid].x;
    double dl = closest_pair(pts, lo, mid, strip);
    double dr = closest_pair(pts, mid, hi, strip);
    double d = min(dl, dr);

    // Merge the two halves by y-coordinate (merge sort style)
    merge(pts.begin() + lo, pts.begin() + mid,
          pts.begin() + mid, pts.begin() + hi,
          strip.begin(),
          [](const Point& a, const Point& b){ return a.y < b.y; });
    copy(strip.begin(), strip.begin() + (hi - lo), pts.begin() + lo);

    // Build the strip
    int cnt = 0;
    for (int i = lo; i < hi; i++)
        if (abs(pts[i].x - midx) < d)
            strip[cnt++] = pts[i];

    // Check pairs in strip
    for (int i = 0; i < cnt; i++)
        for (int j = i + 1; j < cnt && strip[j].y - strip[i].y < d; j++)
            d = min(d, dist(strip[i], strip[j]));

    return d;
}

    The Strip Proof: Why Only 7 Comparisons?

    The key insight that makes the combine step O(n) is this: for each point in the strip, we need to check at most 7 other points (sorted by y-coordinate).

    The Packing Argument

    Consider a δ × 2δ rectangle centered at a strip point. Any pair within this rectangle is within distance δ of the dividing line. Since all points are at least distance δ apart (from the recursive solutions), we can fit at most 8 points in this rectangle.

    A δ × 2δ rectangle around the dividing line can contain at most 8 points (4 on each side), since each side's points are at least δ apart.

    Why This Matters

    This packing argument proves that the inner loop of the combine step iterates at most 7 times for each point. Therefore:

    • The combine step is O(n), not O(n²).
    • The total recurrence is T(n) = 2T(n/2) + O(n), giving O(n log n).
    • In practice, the constant is small — typically only 1–3 comparisons per point.

    Sweep Line Approach

    An alternative O(n log n) algorithm uses a sweep line with an ordered set. This avoids the complexity of the divide & conquer merge step.

    Algorithm

    1. Sort points by x-coordinate.
    2. Maintain a set of "active" points, ordered by y-coordinate.
    3. For each new point, remove points from the set whose x-distance exceeds the current best distance δ.
    4. Query the set for all points with y-coordinate in [y − δ, y + δ] and update δ.
    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
struct Point { ll x, y; int idx; };

ll dist2(Point a, Point b) {
    return (a.x - b.x) * (a.x - b.x)
         + (a.y - b.y) * (a.y - b.y);
}

ll closest_pair_sweep(vector<Point>& pts) {
    int n = pts.size();
    sort(pts.begin(), pts.end(),
         [](const Point& a, const Point& b) {
             return a.x < b.x || (a.x == b.x && a.y < b.y);
         });

    // Ordered set: (y, x, index) for y-ordered lookup
    auto cmp = [](const Point& a, const Point& b) {
        return a.y < b.y || (a.y == b.y && a.x < b.x)
            || (a.y == b.y && a.x == b.x && a.idx < b.idx);
    };
    set<Point, decltype(cmp)> active(cmp);

    ll best = LLONG_MAX;
    int left = 0;

    for (int i = 0; i < n; i++) {
        ll d = (ll)ceil(sqrt((double)best));

        // Remove points too far left
        while (left < i && pts[i].x - pts[left].x >= d) {
            active.erase(pts[left]);
            left++;
        }

        // Query range [y - d, y + d]
        Point lo_pt = {0, pts[i].y - d, -1};
        Point hi_pt = {0, pts[i].y + d, INT_MAX};
        auto it = active.lower_bound(lo_pt);
        while (it != active.end() &&
               it->y <= pts[i].y + d) {
            ll dd = dist2(pts[i], *it);
            if (dd < best) best = dd;
            ++it;
        }

        active.insert(pts[i]);
    }
    return best;
}

    Sweep Line vs Divide & Conquer

    Aspect Divide & Conquer Sweep Line
    TimeO(n log n)O(n log n)
    ImplementationMore complex (merge sort)Cleaner (set operations)
    Constant factorBetter in practiceSlightly worse (set overhead)
    Ease of debuggingHarder (recursion)Easier (iterative)

    Full Contest Implementation

    Here is a clean, contest-ready divide & conquer implementation. It uses the merge sort trick to maintain y-sorted order across recursive calls.

    #include <bits/stdc++.h>
using namespace std;

struct Point { long long x, y; };

double dist(Point a, Point b) {
    double dx = a.x - b.x, dy = a.y - b.y;
    return sqrt(dx * dx + dy * dy);
}

double solve(vector<Point>& pts, int lo, int hi, vector<Point>& tmp) {
    if (hi - lo <= 3) {
        double best = 1e18;
        for (int i = lo; i < hi; i++)
            for (int j = i + 1; j < hi; j++)
                best = min(best, dist(pts[i], pts[j]));
        sort(pts.begin() + lo, pts.begin() + hi,
             [](const Point& a, const Point& b){ return a.y < b.y; });
        return best;
    }

    int mid = (lo + hi) / 2;
    long long midx = pts[mid].x;
    double d = min(solve(pts, lo, mid, tmp),
                   solve(pts, mid, hi, tmp));

    // Merge by y
    merge(pts.begin() + lo, pts.begin() + mid,
          pts.begin() + mid, pts.begin() + hi,
          tmp.begin(),
          [](const Point& a, const Point& b){ return a.y < b.y; });
    copy(tmp.begin(), tmp.begin() + (hi - lo), pts.begin() + lo);

    // Build strip
    int cnt = 0;
    for (int i = lo; i < hi; i++)
        if (abs(pts[i].x - midx) < d)
            tmp[cnt++] = pts[i];

    // Check strip pairs
    for (int i = 0; i < cnt; i++)
        for (int j = i + 1; j < cnt && tmp[j].y - tmp[i].y < d; j++)
            d = min(d, dist(tmp[i], tmp[j]));

    return d;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<Point> pts(n);
    for (int i = 0; i < n; i++) cin >> pts[i].x >> pts[i].y;

    sort(pts.begin(), pts.end(),
         [](const Point& a, const Point& b) {
             return a.x < b.x || (a.x == b.x && a.y < b.y);
         });

    vector<Point> tmp(n);
    cout << fixed << setprecision(6) << solve(pts, 0, n, tmp) << "\n";
}

    Complexity Recap

    • Sorting: O(n log n)
    • Divide & Conquer: T(n) = 2T(n/2) + O(n) = O(n log n)
    • Total: O(n log n) time, O(n) extra space

    Practice & Tips

    Practice Problems

    • CSES — Minimum Euclidean Distance: The standard closest pair problem.
    • SPOJ CLOPPAIR: Find the closest pair and output the indices.
    • Codeforces 429D — Tricky Function: Reduces to closest pair after transformation.
    • UVa 10245 — The Closest Pair Problem: Classic problem with floating-point output.

    Competition Tips

    • Use squared distances when possible to avoid precision loss from sqrt.
    • Base case size matters: Using brute force for n ≤ 3 (not n ≤ 1) avoids off-by-one bugs.
    • The sweep line approach is easier to code under time pressure. Consider it first.
    • For Manhattan distance closest pair, rotate coordinates by 45° to reduce to Chebyshev distance, which is easier to handle with sweep line.
    • Watch for duplicate points: the closest pair distance can be 0. Don't initialize best distance to 0!

    Variants & Extensions

    The closest pair problem has many important variants that appear in competitions:

    Manhattan Distance Closest Pair

    For Manhattan distance |x₁−x₂| + |y₁−y₂|, rotate coordinates by 45°: transform (x, y) to (x+y, x−y). Manhattan distance in the original space becomes Chebyshev distance (max of coordinate differences) in the new space, which is easier to handle with sweep line.

    // Transform for Manhattan distance closest pair
// After this, Chebyshev distance = original Manhattan distance
for (auto& p : pts) {
    ll nx = p.x + p.y;
    ll ny = p.x - p.y;
    p.x = nx;
    p.y = ny;
}
// Now find min of max(|xi-xj|, |yi-yj|) using sweep line

    K-Closest Pairs

    Sometimes you need the k smallest pairwise distances. One approach: use a priority queue with the divide-and-conquer structure, expanding candidates lazily.

    Closest Pair in Higher Dimensions

    The divide-and-conquer approach generalizes to d dimensions with time O(n logd−1 n). For d=3, the strip becomes a slab and the packing argument still works (constant points per cell).

    Online / Dynamic Closest Pair

    If points are added one at a time, maintain a grid of cell size δ (current closest pair distance). When a new point arrives, check only the O(1) neighboring cells. If a closer pair is found, rebuild the grid. The amortized expected time is O(n) for random insertions.

    Rotating coordinates 45° transforms the Manhattan diamond into a Chebyshev square, simplifying sweep line approaches.

    Key insight: The closest pair problem is O(n log n) because the strip in the combine step has a constant number of candidate pairs per point. This packing argument is the heart of the algorithm.
