Convex Hull: Graham Scan & Monotone Chain

July 2025 · 15 min read

The convex hull is the most classical problem in computational geometry and appears frequently in competitive programming contests. We'll study two O(n log n) algorithms and their C++ implementations.

What is a Convex Hull?

The convex hull of a set of points is the smallest convex polygon that contains all the points. Imagine stretching a rubber band around all the points and letting it snap tight—that's the hull.

  • The hull vertices are a subset of the input points.
  • All other points lie inside or on the boundary of the hull.
  • The hull is unique (assuming no three collinear hull vertices, or if we define a canonical form).

Common applications:

  1. Finding the diameter (farthest pair) of a point set.
  2. Determining if a point is inside a convex region (binary search on hull).
  3. Computing the minimum enclosing rectangle.
  4. Solving half-plane intersection problems.

Graham Scan

Graham Scan was one of the first O(n log n) convex hull algorithms, published in 1972.

Algorithm

  1. Find the point with the lowest y-coordinate (break ties by x). Call it the pivot.
  2. Sort all other points by polar angle with respect to the pivot.
  3. Process points in sorted order, maintaining a stack. For each new point, pop the stack while the top two points and the new point make a clockwise (or collinear) turn.
  4. Push the new point onto the stack.
typedef pair<long long, long long> pll;

long long cross(pll O, pll A, pll B) {
    return (long long)(A.first - O.first) * (B.second - O.second)
         - (long long)(A.second - O.second) * (B.first - O.first);
}

long long dist2(pll a, pll b) {
    long long dx = a.first - b.first, dy = a.second - b.second;
    return dx * dx + dy * dy;
}

vector<pll> grahamScan(vector<pll> pts) {
    int n = pts.size();
    if (n < 3) return pts;

    // Step 1: Find pivot (lowest y, then leftmost x)
    int pivot = 0;
    for (int i = 1; i < n; i++) {
        if (pts[i].second < pts[pivot].second ||
            (pts[i].second == pts[pivot].second &&
             pts[i].first < pts[pivot].first))
            pivot = i;
    }
    swap(pts[0], pts[pivot]);
    pll p0 = pts[0];

    // Step 2: Sort by polar angle
    sort(pts.begin() + 1, pts.end(), [&](pll &a, pll &b) {
        long long v = cross(p0, a, b);
        if (v == 0) return dist2(p0, a) < dist2(p0, b);
        return v > 0;
    });

    // Step 3: Build hull with stack
    vector<pll> hull;
    for (auto &p : pts) {
        while (hull.size() >= 2 &&
               cross(hull[hull.size()-2], hull[hull.size()-1], p) <= 0)
            hull.pop_back();
        hull.push_back(p);
    }
    return hull;
}

Complexity: O(n log n) due to sorting. The stack processing is amortized O(n) since each point is pushed and popped at most once.

Monotone Chain

Andrew's Monotone Chain (1979) is the preferred algorithm in competitive programming due to its simplicity and robust handling of collinear points.

Algorithm

  1. Sort points lexicographically (by x, then y).
  2. Build the lower hull: process points left to right, maintaining the CCW invariant.
  3. Build the upper hull: process points right to left.
  4. Concatenate (removing duplicate endpoints).
vector<pll> convexHull(vector<pll> pts) {
    int n = pts.size(), k = 0;
    if (n < 3) return pts;
    sort(pts.begin(), pts.end());

    vector<pll> hull(2 * n);

    // Lower hull
    for (int i = 0; i < n; i++) {
        while (k >= 2 && cross(hull[k-2], hull[k-1], pts[i]) <= 0)
            k--;
        hull[k++] = pts[i];
    }

    // Upper hull
    int lower_size = k + 1;
    for (int i = n - 2; i >= 0; i--) {
        while (k >= lower_size &&
               cross(hull[k-2], hull[k-1], pts[i]) <= 0)
            k--;
        hull[k++] = pts[i];
    }

    hull.resize(k - 1); // remove last point (duplicate of first)
    return hull;
}

Complexity: O(n log n) for sorting, O(n) for the two passes.

Why prefer Monotone Chain?

  • No need for atan2 or polar-angle sorting.
  • Lexicographic sorting is faster and more robust.
  • Handles collinear points naturally (use < 0 vs <= 0 to include or exclude collinear hull points).

Comparison

Feature Graham Scan Monotone Chain
Complexity O(n log n) O(n log n)
Sort type Polar angle Lexicographic
Collinear handling Requires care Straightforward
Implementation Moderate Simple
Contest preference Less common More common

Verdict: Use Monotone Chain as your default. Only switch to Graham Scan if the problem specifically benefits from polar-angle ordering (rare).

Edge Cases

Convex hull problems are full of edge cases. Here's a checklist:

  • n ≤ 2: The hull is just the points themselves (a point or a segment).
  • All points collinear: The hull degenerates to a line segment. Your code should return the two endpoints.
  • Duplicate points: Remove duplicates before running the algorithm, or ensure your comparator handles them.
  • Collinear points on the hull boundary: Use <= 0 in the cross-product check to exclude them, or < 0 to include them. Be consistent and read the problem statement carefully.
  • Integer overflow: Coordinates up to 109 mean products up to 1018. Use long long.
// Include collinear points on hull boundary:
// Change cross(...) <= 0 to cross(...) < 0
// But also sort collinear points on the last segment in reverse order

// Exclude collinear points (standard):
while (k >= 2 && cross(hull[k-2], hull[k-1], pts[i]) <= 0)
    k--;

// Include collinear points:
while (k >= 2 && cross(hull[k-2], hull[k-1], pts[i]) < 0)
    k--;

Convex Hull Applications

Once you have the convex hull, many problems become tractable:

Diameter (Farthest Pair)

The diameter of a point set is the maximum distance between any two points, and this pair always lies on the convex hull. Use rotating calipers to find it in O(n) after hull computation: 

// Rotating calipers — O(n) after hull is built
long long diameter2(vector<pll> &hull) {
    int n = hull.size();
    if (n == 1) return 0;
    if (n == 2) return dist2(hull[0], hull[1]);
    long long best = 0;
    int j = 1;
    for (int i = 0; i < n; i++) {
        int ni = (i + 1) % n;
        while (true) {
            int nj = (j + 1) % n;
            // Check if rotating j further increases the distance
            pll edge = {hull[ni].first - hull[i].first,
                        hull[ni].second - hull[i].second};
            pll diag = {hull[nj].first - hull[j].first,
                        hull[nj].second - hull[j].second};
            if (cross_val(edge, diag) > 0) j = nj;
            else break;
        }
        best = max(best, dist2(hull[i], hull[j]));
    }
    return best;
}

Minimum Enclosing Rectangle

The minimum-area enclosing rectangle of a point set always has one side flush with a hull edge. This is also solved with rotating calipers in O(n).

Point-in-Convex-Polygon

Testing whether a point lies inside a convex hull can be done in O(log n) using binary search on the hull vertices (instead of O(n) for general polygons). 

// O(log n) point-in-convex-polygon
bool inConvexPoly(vector<pll> &hull, pll p) {
    int n = hull.size();
    if (n < 3) return false;
    if (cross(hull[0], hull[1], p) < 0) return false;
    if (cross(hull[0], hull[n-1], p) > 0) return false;
    int lo = 1, hi = n - 1;
    while (hi - lo > 1) {
        int mid = (lo + hi) / 2;
        if (cross(hull[0], hull[mid], p) >= 0) lo = mid;
        else hi = mid;
    }
    return cross(hull[lo], hull[lo+1], p) >= 0;
}
