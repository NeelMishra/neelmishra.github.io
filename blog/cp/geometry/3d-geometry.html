    3D Geometry Basics

    July 2025 · 15 min read

    Moving from 2D to 3D geometry introduces new challenges and tools. The cross product becomes a vector (not a scalar), planes replace lines as the fundamental boundaries, and many 2D intuitions need careful extension. This post covers the essentials: 3D point/vector operations, planes, distances, and intersections.

    3D Point and Vector Class

    A 3D point or vector is a triple (x, y, z). We use the same struct for both, as in 2D geometry. The key operations are addition, subtraction, scalar multiplication, dot product, and cross product. 

    // 3D Point / Vector class
struct Vec3 {
    double x, y, z;
    Vec3() : x(0), y(0), z(0) {}
    Vec3(double x, double y, double z) : x(x), y(y), z(z) {}

    Vec3 operator+(const Vec3& o) const { return {x+o.x, y+o.y, z+o.z}; }
    Vec3 operator-(const Vec3& o) const { return {x-o.x, y-o.y, z-o.z}; }
    Vec3 operator*(double t) const { return {x*t, y*t, z*t}; }
    Vec3 operator/(double t) const { return {x/t, y/t, z/t}; }

    double dot(const Vec3& o) const { return x*o.x + y*o.y + z*o.z; }
    Vec3 cross(const Vec3& o) const {
        return {y*o.z - z*o.y, z*o.x - x*o.z, x*o.y - y*o.x};
    }

    double norm() const { return sqrt(x*x + y*y + z*z); }
    double norm2() const { return x*x + y*y + z*z; }
    Vec3 unit() const { double n = norm(); return {x/n, y/n, z/n}; }
};

    Dot product in 3D: a · b = aₓbₓ + aᵧbᵧ + a_z b_z. It still gives |a||b|cos θ, where θ is the angle between the vectors. A zero dot product means perpendicularity.

    Competition note: For integer problems, use long long for coordinates and avoid the norm() and unit() functions (which involve sqrt). Work with squared distances and raw cross products instead.

    Cross Product in 3D

    Unlike in 2D where the cross product is a scalar, the 3D cross product a × b is a vector perpendicular to both a and b, with magnitude |a||b|sin θ.

    The formula expands as:

    a × b = (aᵧb_z − a_z bᵧ, a_z bₓ − aₓb_z, aₓbᵧ − aᵧbₓ)

    Key properties:

    • Anti-commutative: a × b = -(b × a).
    • Magnitude = area: |a × b| equals the area of the parallelogram spanned by a and b.
    • Direction: Follows the right-hand rule — curl fingers from a to b, thumb points in the direction of a × b.
    • Zero cross product: a × b = 0 if and only if a and b are parallel (or one is zero).

    The diagram shows vectors a (blue) and b (pink) in 3D, with their cross product a × b (orange) pointing upward. The green parallelogram's area equals |a × b|. 

    // Triangle area in 3D using cross product
double triangleArea3D(Vec3 a, Vec3 b, Vec3 c) {
    return (b - a).cross(c - a).norm() / 2.0;
}

// Check if three 3D points are collinear
bool collinear3D(Vec3 a, Vec3 b, Vec3 c) {
    return (b - a).cross(c - a).norm2() < 1e-18;
}

    Planes

    A plane in 3D is defined by the equation ax + by + cz + d = 0, where n = (a, b, c) is the normal vector to the plane and d is the signed distance offset from the origin.

    Equivalent representations:

    • Normal + point: The plane through point p₀ with normal n: n · (p - p₀) = 0.
    • Three points: Given non-collinear points a, b, c, the normal is n = (b - a) × (c - a).
    • Equation form: ax + by + cz + d = 0 where d = -n · p₀.
    // Plane from three points
struct Plane {
    Vec3 n; // normal
    double d; // n.x*x + n.y*y + n.z*z + d = 0

    // Construct from 3 non-collinear points
    Plane(Vec3 a, Vec3 b, Vec3 c) {
        n = (b - a).cross(c - a);
        d = -n.dot(a);
    }

    // Signed distance from point to plane (positive = same side as normal)
    double signedDist(Vec3 p) const {
        return (n.dot(p) + d) / n.norm();
    }

    // Which side of the plane is point p on? (+1, -1, or 0)
    int side(Vec3 p) const {
        double val = n.dot(p) + d;
        if (val > 1e-9) return 1;
        if (val < -1e-9) return -1;
        return 0;
    }
};

    Tip: When using integer coordinates, avoid division in signedDist. Instead, compare the raw dot product n · p + d (which has the same sign as the signed distance) to avoid floating-point errors.

    Point-to-Plane Distance

    The signed distance from point p to plane ax + by + cz + d = 0 is:

    dist(p, π) = (n · p + d) / |n|

    The unsigned distance is |dist(p, π)|. The sign indicates which side of the plane the point lies on.

    The projection (closest point on the plane) is:

    proj(p) = p − dist(p, π) · n̂

    where n̂ = n / |n| is the unit normal.

    // Point-to-plane distance and projection
double pointPlaneDist(Vec3 p, Plane& pi) {
    return abs(pi.n.dot(p) + pi.d) / pi.n.norm();
}

Vec3 projectOntoPlane(Vec3 p, Plane& pi) {
    double t = (pi.n.dot(p) + pi.d) / pi.n.norm2();
    return p - pi.n * t;
}

    Integer arithmetic version: When coordinates are integers up to 10⁶, n · p + d fits in a long long. To compare distances without sqrt, compare (n · p + d)² / |n|². Since |n|² is the same for all points relative to one plane, you only need to compare numerators. 

    // Integer-friendly distance comparison
typedef long long ll;
struct Vec3i { ll x, y, z; };

struct Planei {
    Vec3i n;
    ll d;
    // Compare which of p1, p2 is farther from the plane (no sqrt)
    bool farther(Vec3i p1, Vec3i p2) {
        ll d1 = n.x*p1.x + n.y*p1.y + n.z*p1.z + d;
        ll d2 = n.x*p2.x + n.y*p2.y + n.z*p2.z + d;
        return abs(d1) > abs(d2);
    }
};

    Line-Plane Intersection

    A line in 3D can be parametrised as L(t) = p + t·v, where p is a point on the line and v is the direction vector. To find where it intersects plane n · x + d = 0:

    Substitute into the plane equation: n · (p + t·v) + d = 0, giving:

    t = -(n · p + d) / (n · v)

    Three cases:

    • n · v ≠ 0: The line intersects the plane at exactly one point L(t).
    • n · v = 0 and n · p + d = 0: The line lies entirely in the plane.
    • n · v = 0 and n · p + d ≠ 0: The line is parallel to the plane (no intersection).
    // Line-plane intersection
// Returns: 0 = no intersection (parallel), 1 = single point, 2 = line in plane
int linePlaneIntersect(Vec3 p, Vec3 v, Plane& pi, Vec3& hit) {
    double denom = pi.n.dot(v);
    double numer = pi.n.dot(p) + pi.d;
    if (abs(denom) < 1e-9) {
        return abs(numer) < 1e-9 ? 2 : 0;
    }
    double t = -numer / denom;
    hit = p + v * t;
    return 1;
}

// Segment-plane intersection (t must be in [0, 1])
int segPlaneIntersect(Vec3 a, Vec3 b, Plane& pi, Vec3& hit) {
    Vec3 v = b - a;
    double denom = pi.n.dot(v);
    double numer = pi.n.dot(a) + pi.d;
    if (abs(denom) < 1e-9) {
        return abs(numer) < 1e-9 ? 2 : 0;
    }
    double t = -numer / denom;
    if (t < -1e-9 || t > 1.0 + 1e-9) return 0;
    hit = a + v * t;
    return 1;
}

    Applications: Line-plane intersection is used in ray tracing, clipping polygons against half-spaces (Sutherland-Hodgman in 3D), and computing cross-sections of 3D objects.

    Implementation & Competition Tips

    Working in 3D brings extra pitfalls. Here are practical tips for contests:

    • Always test with simple cases: Use axis-aligned planes and cardinal direction vectors to verify your formulas before testing general cases.
    • Avoid sqrt when possible: Compare squared distances and squared norms. Only take square roots for final output.
    • Orientation matters: The direction of the cross product depends on operand order. a × b = -(b × a). Swap the order if your normal points the wrong way.
    • Degenerate cases: Lines parallel to planes, points on planes, collinear input points — always handle these explicitly.
    • Floating-point precision: In 3D, numerical errors compound faster. Use an epsilon of 1e-9 and be prepared to increase it for problems with large coordinates.

    Additional useful 3D operations:

    // Distance between two 3D lines
// Line 1: p1 + t*v1, Line 2: p2 + s*v2
double lineLineDist(Vec3 p1, Vec3 v1, Vec3 p2, Vec3 v2) {
    Vec3 w = p1 - p2;
    Vec3 n = v1.cross(v2);
    double n2 = n.norm2();
    if (n2 < 1e-18) {
        // lines are parallel: distance = point-to-line distance
        return v1.cross(w).norm() / v1.norm();
    }
    return abs(n.dot(w)) / sqrt(n2);
}

// Angle between two planes (dihedral angle)
double dihedralAngle(Plane& p1, Plane& p2) {
    double cosA = p1.n.dot(p2.n) / (p1.n.norm() * p2.n.norm());
    cosA = max(-1.0, min(1.0, cosA)); // clamp for acos safety
    return acos(cosA);
}

// Volume of tetrahedron with vertices a, b, c, d
double tetraVolume(Vec3 a, Vec3 b, Vec3 c, Vec3 d) {
    return abs((b-a).cross(c-a).dot(d-a)) / 6.0;
}

    Summary

    3D geometry extends 2D concepts with the full vector cross product, planes as boundary elements, and an extra dimension of freedom. The Vec3 class with dot, cross, and norm operations forms the foundation. Planes are defined by a normal and offset, enabling efficient distance and intersection computations. Line-plane intersection reduces to a single division after a dot product. For competitive programming, favour integer arithmetic, avoid unnecessary square roots, and always handle degenerate cases explicitly.
