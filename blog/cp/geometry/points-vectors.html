Points & Vectors: The Foundation

July 2025 · 14 min read

Every computational geometry algorithm builds on two primitives: points and vectors. Getting their implementation right—including floating-point tolerance—saves hours of debugging in contests. This post establishes the Point struct we'll reuse throughout the entire 25-part series.

Point Representation

In competitive programming we typically represent a 2-D point as a pair (x, y). Depending on the problem, coordinates can be integers or floating-point. We use a single struct for both by templating or simply using double. 

struct Point {
    double x, y;
    Point() : x(0), y(0) {}
    Point(double x, double y) : x(x), y(y) {}
    Point operator+(const Point &p) const { return {x + p.x, y + p.y}; }
    Point operator-(const Point &p) const { return {x - p.x, y - p.y}; }
    Point operator*(double t) const { return {x * t, y * t}; }
    Point operator/(double t) const { return {x / t, y / t}; }
    bool operator<(const Point &p) const {
        if (x != p.x) return x < p.x;
        return y < p.y;
    }
    bool operator==(const Point &p) const {
        return x == p.x && y == p.y;
    }
};

Tip: When coordinates are guaranteed to be integers and fit in long long, prefer integer arithmetic to avoid precision issues entirely. Many top competitive programmers keep two versions of the struct. 

struct Pt {
    long long x, y;
    Pt() : x(0), y(0) {}
    Pt(long long x, long long y) : x(x), y(y) {}
    Pt operator+(const Pt &p) const { return {x + p.x, y + p.y}; }
    Pt operator-(const Pt &p) const { return {x - p.x, y - p.y}; }
    long long cross(const Pt &p) const { return x * p.y - y * p.x; }
    long long dot(const Pt &p) const { return x * p.x + y * p.y; }
};

Vector Operations

A vector is represented exactly like a point—a pair (x, y)—but its meaning is a displacement, not a position. Given two points A and B, the vector from A to B is simply B − A.

  • Addition: u + v = (u.x + v.x, u.y + v.y)
  • Subtraction: u - v = (u.x - v.x, u.y - v.y)
  • Scalar multiplication: t * u = (t * u.x, t * u.y)
  • Magnitude: |u| = sqrt(u.x² + u.y²)
  • Normalization: u / |u| gives the unit vector.
double mag(Point v) { return sqrt(v.x * v.x + v.y * v.y); }
Point normalize(Point v) {
    double m = mag(v);
    return {v.x / m, v.y / m};
}

Normalization is useful when you need direction without magnitude, for example when projecting onto a unit vector.

Dot Product

The dot product of two 2-D vectors u and v is:

u · v = u.x * v.x + u.y * v.y = |u| * |v| * cos(θ)

Key properties used in CP:

  1. Sign test: positive ⇒ acute angle, zero ⇒ perpendicular, negative ⇒ obtuse.
  2. Projection length: proj = (u · v) / |v|.
  3. Squared distance: |u|² = u · u avoids a costly sqrt.
double dot(Point u, Point v) { return u.x * v.x + u.y * v.y; }
double dist2(Point a, Point b) {
    Point d = b - a;
    return dot(d, d);
}
double dist(Point a, Point b) { return sqrt(dist2(a, b)); }

Cross Product

The 2-D cross product yields a scalar (the z-component of the 3-D cross product):

u × v = u.x * v.y - u.y * v.x = |u| * |v| * sin(θ)

Key properties:

  • Positive ⇒ v is counter-clockwise from u.
  • Negative ⇒ v is clockwise from u.
  • Zero ⇒ u and v are collinear.
  • Its absolute value equals twice the area of the triangle formed by the origin, u, and v.
double cross(Point u, Point v) { return u.x * v.y - u.y * v.x; }

// Orientation of ordered triplet (a, b, c)
// Returns +1 (CCW), -1 (CW), 0 (collinear)
int orient(Point a, Point b, Point c) {
    double v = cross(b - a, c - a);
    if (v > 0) return +1;
    if (v < 0) return -1;
    return 0;
}

The cross product is arguably the single most important primitive in 2-D computational geometry. We'll see it used in convex hulls, polygon area, line intersection, and many more algorithms.

EPS Handling

Floating-point arithmetic is imprecise. Comparing doubles with == is almost always wrong. Instead, we use a small tolerance EPS. 

const double EPS = 1e-9;

int sign(double x) {
    if (x > EPS) return 1;
    if (x < -EPS) return -1;
    return 0;
}

bool eq(double a, double b) { return fabs(a - b) < EPS; }

// EPS-aware point comparison
bool ptEq(Point a, Point b) {
    return eq(a.x, b.x) && eq(a.y, b.y);
}

// EPS-aware less-than for sorting
bool ptLt(Point a, Point b) {
    if (!eq(a.x, b.x)) return a.x < b.x;
    return a.y < b.y;
}

Choosing EPS:

  • For coordinates up to 106, EPS = 1e-9 is standard.
  • For coordinates up to 109, use long double or consider EPS = 1e-7.
  • When intermediate values are products of two coordinates, precision loss doubles; adjust EPS accordingly.
  • Best practice: Use integer arithmetic whenever the problem allows. If all coordinates are integers and you only need the sign of a cross product, exact integer computation is superior.

Angle Computation

Computing angles precisely is a frequent source of bugs. Use atan2, which returns the angle in (-π, π] and handles all quadrants correctly. 

// Angle of vector (p.x, p.y) relative to positive x-axis
double angle(Point p) { return atan2(p.y, p.x); }

// Angle at vertex B in triangle A-B-C
double angleBetween(Point a, Point b, Point c) {
    Point u = a - b, v = c - b;
    return acos(dot(u, v) / (mag(u) * mag(v)));
}

// Signed angle from u to v (positive = CCW)
double signedAngle(Point u, Point v) {
    return atan2(cross(u, v), dot(u, v));
}

Competition tip: Avoid angles when possible! The cross product gives orientation without trigonometry, and atan2 is slow compared to integer cross products. Only compute angles when the problem explicitly requires them or when you need to sort by polar angle (and even then, a cross-product comparator often works).

Complete Template

Here is the full Point struct we'll use throughout this series. Copy this into your competitive programming template. 

#include <bits/stdc++.h>
using namespace std;
const double EPS = 1e-9;

struct Point {
    double x, y;
    Point() : x(0), y(0) {}
    Point(double x, double y) : x(x), y(y) {}

    Point operator+(const Point &p) const { return {x + p.x, y + p.y}; }
    Point operator-(const Point &p) const { return {x - p.x, y - p.y}; }
    Point operator*(double t) const { return {x * t, y * t}; }
    Point operator/(double t) const { return {x / t, y / t}; }

    double dot(const Point &p) const { return x * p.x + y * p.y; }
    double cross(const Point &p) const { return x * p.y - y * p.x; }
    double norm() const { return sqrt(x * x + y * y); }
    double norm2() const { return x * x + y * y; }
    Point unit() const { double n = norm(); return {x / n, y / n}; }
    Point perp() const { return {-y, x}; }
    double angle() const { return atan2(y, x); }

    bool operator<(const Point &p) const {
        if (fabs(x - p.x) > EPS) return x < p.x;
        return y < p.y;
    }
    bool operator==(const Point &p) const {
        return fabs(x - p.x) < EPS && fabs(y - p.y) < EPS;
    }
};

double dist(Point a, Point b) { return (b - a).norm(); }
double dist2(Point a, Point b) { return (b - a).norm2(); }

int sign(double x) {
    if (x > EPS) return 1;
    if (x < -EPS) return -1;
    return 0;
}

// Orientation: +1 CCW, -1 CW, 0 collinear
int orient(Point a, Point b, Point c) {
    return sign((b - a).cross(c - a));
}

Design choices explained:

  • perp() returns the perpendicular vector (-y, x), which is a 90° CCW rotation. Useful for computing normals.
  • norm2() avoids the costly sqrt. Always prefer squared distances when comparing magnitudes.
  • The operator< uses EPS-aware comparison, critical for algorithms that sort points (e.g., sweep line).
  • We keep dot and cross as member functions so you can write (b - a).cross(c - a) naturally.

Integer variant

For problems with integer coordinates, use this version to avoid all floating-point: 

struct Pt {
    long long x, y;
    Pt() : x(0), y(0) {}
    Pt(long long x, long long y) : x(x), y(y) {}
    Pt operator+(const Pt &p) const { return {x + p.x, y + p.y}; }
    Pt operator-(const Pt &p) const { return {x - p.x, y - p.y}; }
    long long dot(const Pt &p) const { return x * p.x + y * p.y; }
    long long cross(const Pt &p) const { return x * p.y - y * p.x; }
    long long norm2() const { return x * x + y * y; }
    bool operator<(const Pt &p) const {
        return x < p.x || (x == p.x && y < p.y);
    }
    bool operator==(const Pt &p) const { return x == p.x && y == p.y; }
};
