    Pick's Theorem: Counting Lattice Points

    July 2025 · 16 min read

    Pick's theorem is an elegant formula connecting the area of a polygon with integer-coordinate vertices to the number of lattice points it contains. Despite its simplicity, it is remarkably powerful in competitive programming — enabling you to count interior lattice points, compute areas from point counts, and solve polygon problems with integer arithmetic alone.

    Prerequisites: You should be familiar with the shoelace formula for polygon area and basic GCD computation. Both are essential building blocks for applying Pick's theorem efficiently.

    When to use Pick's theorem:

    • When the problem involves lattice points (integer coordinates) and polygon areas.
    • When you need to count points inside a polygon without iterating over all grid cells.
    • When a problem gives you area and boundary information and asks for interior counts.
    • When constructing polygons with specific lattice point properties.

    Pick's Theorem Statement

    For a simple polygon (no self-intersections) whose vertices are all at integer coordinates (lattice points):

    A = I + B/2 − 1

    Where:

    • A = area of the polygon
    • I = number of lattice points strictly inside the polygon
    • B = number of lattice points on the boundary of the polygon

    Rearranging, we can compute any one of the three quantities given the other two. Most commonly in contests:

    • Given polygon vertices, compute A via the shoelace formula and B via GCD, then find I = A − B/2 + 1.
    • Given I and B, compute A = I + B/2 − 1.

    Boundary Points via GCD

    The number of lattice points on the segment from (x₁, y₁) to (x₂, y₂) (including both endpoints) is:

    gcd(|x₂ − x₁|, |y₂ − y₁|) + 1

    To count boundary points B of a polygon, sum the lattice points on each edge, but subtract the number of vertices (each vertex is shared by two edges): 

    long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

// Count lattice points on segment from (x1,y1) to (x2,y2), excluding endpoint
long long edgeLatticePoints(long long x1, long long y1,
                             long long x2, long long y2) {
    return gcd(abs(x2 - x1), abs(y2 - y1));
}

// Total boundary points of polygon with integer vertices
long long boundaryPoints(const vector<pair<long long,long long>>& poly) {
    long long B = 0;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        B += edgeLatticePoints(poly[i].first, poly[i].second,
                                poly[j].first, poly[j].second);
    }
    return B;
}

    Why GCD? Consider the segment from (0, 0) to (dx, dy). A lattice point (kx, ky) lies on this segment iff there exists t = k/g where g = gcd(dx, dy) and 0 ≤ k ≤ g. So there are exactly g + 1 lattice points including both endpoints.

    Important: When summing over all edges of a polygon, each edge contributes gcd(|dx|, |dy|) lattice points excluding the endpoint. Since each vertex is counted exactly once this way, the total gives exactly B.

    Interior Points

    To count interior lattice points, combine Pick's theorem with the shoelace formula for area: 

    // Shoelace formula: returns 2*A to keep everything in integers
long long twiceArea(const vector<pair<long long,long long>>& poly) {
    long long area2 = 0;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        area2 += poly[i].first * poly[j].second;
        area2 -= poly[j].first * poly[i].second;
    }
    return abs(area2);
}

// Count interior lattice points using Pick's theorem
// A = I + B/2 - 1  =>  I = A - B/2 + 1  =>  I = (2A - B + 2) / 2
long long interiorPoints(const vector<pair<long long,long long>>& poly) {
    long long area2 = twiceArea(poly);
    long long B = boundaryPoints(poly);
    return (area2 - B + 2) / 2;
}

    Key insight: By computing 2A as an integer (the shoelace formula gives an integer when coordinates are integers), we can compute I = (2A − B + 2) / 2 using only integer arithmetic. The numerator is always even, so the division is exact.

    Why is 2A − B + 2 always even? The shoelace formula gives 2A as an integer. From Pick's theorem, 2A = 2I + B − 2, so 2A − B + 2 = 2I, which is always even.

    Proof Sketch

    Pick's theorem can be proved by triangulation. Every simple polygon with lattice vertices can be decomposed into lattice triangles (triangles with lattice vertices and no interior lattice points), each of which has area exactly 1/2.

    Step 1: Primitive triangles. A triangle with lattice vertices and no interior lattice points and no boundary lattice points other than vertices has area exactly 1/2. This follows from the fact that such a triangle can be transformed by a unimodular transformation to the standard triangle (0,0), (1,0), (0,1).

    Step 2: Triangulation. Any simple lattice polygon can be triangulated into primitive triangles (by adding lattice points on edges and in the interior as extra vertices).

    Step 3: Euler's formula. For a triangulation with V vertices, E edges, and F faces (including the outer face): V − E + F = 2. The number of interior faces (triangles) is F − 1. Each has area 1/2, so A = (F − 1)/2.

    Step 4: Counting. The vertices are: I interior points + B boundary points = V. Through careful edge and face counting using Euler's formula, we arrive at A = I + B/2 − 1.

    Applications

    Pick's theorem is a versatile tool in competitive programming:

    • Counting lattice points in polygons: The most direct application. Given polygon vertices, compute interior and boundary counts.
    • Area from lattice counts: If you know the interior and boundary lattice points (e.g., from a grid scan), compute the area without the shoelace formula.
    • Lattice polygons with given area: Construct a lattice polygon with area exactly A. For example, a triangle (0,0), (2A, 0), (0, 1) has area A, boundary 2A + 2, and interior A − A − 1 + 1 = 0 when gcd(2A, 1) = 1.
    • Rectangles and grids: For axis-aligned rectangles with lattice corners, the formula simplifies nicely and can verify grid-based solutions.
    • Polygon with holes: Apply Pick's theorem to the outer polygon, then subtract the interior area of each hole. Lattice points in holes require separate counting.
    // Complete solution: given polygon vertices, output I and B
void solve() {
    int n;
    cin >> n;
    vector<pair<long long,long long>> poly(n);
    for (int i = 0; i < n; i++)
        cin >> poly[i].first >> poly[i].second;

    long long B = boundaryPoints(poly);
    long long I = interiorPoints(poly);
    long long area2 = twiceArea(poly);

    // Output area as fraction: area2 / 2
    cout << "Area: " << area2 / 2;
    if (area2 % 2) cout << ".5";
    cout << "\n";

    cout << "Boundary points: " << B << "\n";
    cout << "Interior points: " << I << "\n";
    cout << "Total lattice points: " << I + B << "\n";
}

    Extended example: Find the number of lattice points inside a triangle with vertices (0, 0), (10, 0), (3, 7).

// 2A = |0*0 - 10*0 + 10*7 - 3*0 + 3*0 - 0*7| = 70
// I = (70 - 18 + 2) / 2 = 27
// Verify: A = 35, I + B/2 - 1 = 27 + 9 - 1 = 35 ✓

    Common pitfalls:

    • Forgetting that Pick's theorem only works for simple polygons (no self-intersections).
    • Using floating-point area instead of the integer 2A formula — always use long long for exact results.
    • Off-by-one errors in boundary counting: gcd(0, 0) = 0 for a zero-length edge (degenerate vertex), which should not occur in valid input.
    • Not handling the case where the polygon is given in clockwise order — the shoelace formula gives a negative value, so take abs().
    • Overflow: with coordinates up to 10⁹, the shoelace cross products can reach 10¹⁸, requiring long long or even __int128.

    Generalizations:

    • Polygons with holes: For a polygon with h holes, A = I + B/2 − 1 + h. Each hole adds 1 to the formula.
    • Higher dimensions: Pick's theorem does not directly generalize to 3D. The 3D analogue involves Ehrhart polynomials for counting lattice points in polytopes.
    • Lattice width: The minimum number of lattice lines needed to cover a convex lattice polygon relates to its width in the lattice direction.
    // Count lattice points in a triangle with vertices at origin,
// (a, b), and (c, d) using Pick's theorem
long long latticePointsInTriangle(long long a, long long b,
                                    long long c, long long d) {
    long long area2 = abs(a * d - b * c);
    long long B = gcd(abs(a), abs(b)) + gcd(abs(c), abs(d))
                + gcd(abs(a - c), abs(b - d));
    long long I = (area2 - B + 2) / 2;
    return I + B;  // total lattice points
}

    Pick's theorem beautifully bridges combinatorics and geometry. In contests, it often converts a seemingly difficult geometric problem into simple integer arithmetic. Remember the formula: A = I + B/2 − 1.
