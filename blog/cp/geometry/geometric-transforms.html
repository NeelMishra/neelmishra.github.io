    Geometric Transformations

    July 2025 · 17 min read

    Geometric transformations — translation, rotation, reflection, and scaling — are fundamental tools in computational geometry and competitive programming. Many problems become dramatically simpler when you transform the coordinate system to exploit symmetry or align shapes with axes.

    This post covers each transformation, its matrix representation, and practical C++ implementations. We finish with the powerful technique of composing transformations using affine matrices to apply complex sequences of operations efficiently.

    Translation

    Translation moves every point by a fixed offset vector (dx, dy): 

    // (x, y) → (x + dx, y + dy)
struct Vec2 { double x, y; };

Vec2 translate(Vec2 p, double dx, double dy) {
    return {p.x + dx, p.y + dy};
}

    Translation preserves distances, angles, and orientation. It is an isometry.

    CP applications:

    • Centering a polygon at the origin before rotation simplifies the math.
    • Translating a key point to (0,0) to reduce coordinate magnitudes and improve precision.
    • Converting “relative to point P” problems to “relative to origin” problems.

    Rotation

    Rotation by angle θ around the origin: 

    // (x, y) → (x·cos θ − y·sin θ,  x·sin θ + y·cos θ)
Vec2 rotate(Vec2 p, double theta) {
    double c = cos(theta), s = sin(theta);
    return {p.x * c - p.y * s, p.x * s + p.y * c};
}

// Rotation around an arbitrary center point
Vec2 rotateAround(Vec2 p, Vec2 center, double theta) {
    Vec2 shifted = {p.x - center.x, p.y - center.y};
    Vec2 rotated = rotate(shifted, theta);
    return {rotated.x + center.x, rotated.y + center.y};
}

    Rotation preserves distances and angles but changes orientation when the angle is negative (clockwise). It is also an isometry.

    CP applications:

    • Rotating the coordinate system so that a given line becomes the x-axis.
    • Rotating all points by 45° to convert Chebyshev distance to Manhattan distance (and vice versa).
    • Simulating physical rotations in robotics/geometry problems.

    The 45° rotation trick: Given points with Chebyshev distance max(|x1−x2|, |y1−y2|), apply the transformation (x, y) → (x+y, x−y). In the new coordinates, Chebyshev distance becomes Manhattan distance, which is often easier to work with. 

    // Chebyshev to Manhattan transformation
pair<long long, long long> chebToManhattan(long long x, long long y) {
    return {x + y, x - y};
}

    Reflection

    Reflection over a line through the origin with direction (dx, dy): 

    // Reflect point p over line through origin with direction (dx, dy)
Vec2 reflect(Vec2 p, double dx, double dy) {
    double d2 = dx * dx + dy * dy;
    double dot = p.x * dx + p.y * dy;
    return {2 * dot * dx / d2 - p.x, 2 * dot * dy / d2 - p.y};
}

// Reflect over x-axis: (x, y) → (x, -y)
Vec2 reflectX(Vec2 p) { return {p.x, -p.y}; }

// Reflect over y-axis: (x, y) → (-x, y)
Vec2 reflectY(Vec2 p) { return {-p.x, p.y}; }

// Reflect over line y = x: (x, y) → (y, x)
Vec2 reflectDiag(Vec2 p) { return {p.y, p.x}; }

    Reflection preserves distances and angles but reverses orientation (clockwise becomes counter-clockwise). It is an improper isometry.

    CP applications:

    • Reflecting across an axis to halve the problem size by symmetry.
    • Unfolding reflections in “billiard” or “light reflection” problems to turn bouncing paths into straight lines.
    • Mirror symmetry detection in polygon problems.

    Affine Transforms

    All 2D affine transformations (translation, rotation, reflection, scaling, shear) can be represented as a 3×3 matrix using homogeneous coordinates. A point (x, y) is represented as (x, y, 1): 

    // Affine transformation as 3x3 matrix
// [a  b  tx]   [x]   [a·x + b·y + tx]
// [c  d  ty] · [y] = [c·x + d·y + ty]
// [0  0   1]   [1]   [       1       ]

struct Affine {
    double m[3][3];

    Affine() {
        memset(m, 0, sizeof(m));
        m[0][0] = m[1][1] = m[2][2] = 1;  // identity
    }

    Vec2 apply(Vec2 p) const {
        return {
            m[0][0]*p.x + m[0][1]*p.y + m[0][2],
            m[1][0]*p.x + m[1][1]*p.y + m[1][2]
        };
    }

    Affine operator*(const Affine &o) const {
        Affine res;
        for (int i = 0; i < 3; i++)
            for (int j = 0; j < 3; j++) {
                res.m[i][j] = 0;
                for (int k = 0; k < 3; k++)
                    res.m[i][j] += m[i][k] * o.m[k][j];
            }
        return res;
    }
};

    Individual transformation matrices: 

    Affine translationMatrix(double dx, double dy) {
    Affine t;
    t.m[0][2] = dx;
    t.m[1][2] = dy;
    return t;
}

Affine rotationMatrix(double theta) {
    Affine r;
    double c = cos(theta), s = sin(theta);
    r.m[0][0] = c;  r.m[0][1] = -s;
    r.m[1][0] = s;  r.m[1][1] = c;
    return r;
}

Affine scaleMatrix(double sx, double sy) {
    Affine s;
    s.m[0][0] = sx;
    s.m[1][1] = sy;
    return s;
}

Affine reflectXMatrix() {
    Affine r;
    r.m[1][1] = -1;
    return r;
}

    Composition

    The true power of affine matrices is composition: you can combine any sequence of transformations into a single matrix multiplication. To apply transformations T1, then T2, then T3, compute M = T3 × T2 × T1 (right-to-left order). 

    // Rotate shape 45° around point (100, 200):
// 1. Translate center to origin
// 2. Rotate
// 3. Translate back

double theta = M_PI / 4;
Affine step1 = translationMatrix(-100, -200);
Affine step2 = rotationMatrix(theta);
Affine step3 = translationMatrix(100, 200);
Affine combined = step3 * step2 * step1;

// Apply to all points in O(1) per point
for (auto &p : points) {
    p = combined.apply(p);
}

    Benefits of composition:

    • Efficiency: Precompute the composed matrix once, apply to each point in O(1). Especially useful when transforming millions of points or applying the same transform to many queries.
    • Matrix exponentiation: If the same transformation is applied k times, use matrix exponentiation to compute Mk in O(log k). This appears in problems involving repeated rotations/translations.
    • Inverse transforms: The inverse of a composition is the composition of inverses in reverse order. For orthogonal transforms (rotation, reflection), the inverse is the transpose. 
    // Matrix exponentiation for repeated transforms
Affine power(Affine base, long long exp) {
    Affine result;  // identity
    while (exp > 0) {
        if (exp & 1) result = result * base;
        base = base * base;
        exp >>= 1;
    }
    return result;
}

// Apply rotation by θ exactly k times:
Affine singleRotation = rotationMatrix(theta);
Affine kRotations = power(singleRotation, k);
// Equivalent to rotating by k·θ, but computed in O(log k)

    Implementation Tips

    Practical tips for using geometric transforms in contests:

    1. Rotation direction: In standard math coordinates (y up), positive θ is counter-clockwise. In screen coordinates (y down), it is clockwise. Know which convention your problem uses.
    2. Avoid repeated sin/cos: Compute sin(θ) and cos(θ) once and reuse. Trigonometric functions are expensive.
    3. Integer-friendly transforms: 90° rotations ((x,y) → (-y,x)) are exact with integers. The 45° Chebyshev trick ((x,y) → (x+y, x-y)) is also exact.
    4. Precision: Each matrix multiplication introduces rounding error. For long chains, consider using long double or algebraic simplification.
    5. Special cases first: Before reaching for general affine transforms, check if the problem only needs translation, 90° rotation, or reflection — these have simple, exact integer implementations.

    Quick reference — common transforms:

    • (x, y) → (x+dx, y+dy) — Translation
    • (x, y) → (-y, x) — 90° CCW rotation
    • (x, y) → (y, -x) — 90° CW rotation
    • (x, y) → (-x, -y) — 180° rotation
    • (x, y) → (x, -y) — Reflect over x-axis
    • (x, y) → (-x, y) — Reflect over y-axis
    • (x, y) → (y, x) — Reflect over y = x
    • (x, y) → (x+y, x-y) — 45° rotation + scale (Chebyshev trick)

    Complexity: All individual transforms are O(1) per point. Composing two 3×3 matrices is O(27) = O(1). Matrix exponentiation is O(27 · log k). Applying a composed transform to n points is O(n).

    Scaling is another useful transform that multiplies coordinates by a factor. Uniform scaling preserves angles but changes distances. Non-uniform scaling (different factors for x and y) distorts shapes but can simplify certain problems. 

    // Uniform scaling by factor k around the origin
Vec2 scaleUniform(Vec2 p, double k) {
    return {p.x * k, p.y * k};
}

// Scaling around an arbitrary center
Vec2 scaleAround(Vec2 p, Vec2 center, double kx, double ky) {
    return {
        center.x + (p.x - center.x) * kx,
        center.y + (p.y - center.y) * ky
    };
}

    CP example — normalizing coordinates: In problems where you need to compare shapes regardless of size, translate the centroid to the origin and scale so that the farthest point is at distance 1. This normalizes the shape for comparison. 

    void normalize(vector<Vec2> &pts) {
    // Translate centroid to origin
    double cx = 0, cy = 0;
    for (auto &p : pts) { cx += p.x; cy += p.y; }
    cx /= pts.size(); cy /= pts.size();
    double maxDist = 0;
    for (auto &p : pts) {
        p.x -= cx; p.y -= cy;
        maxDist = max(maxDist, sqrt(p.x*p.x + p.y*p.y));
    }
    // Scale to unit size
    if (maxDist > 1e-9) {
        for (auto &p : pts) {
            p.x /= maxDist;
            p.y /= maxDist;
        }
    }
}

    Shear transformation: Less common in CP but occasionally useful. A horizontal shear maps (x, y) → (x + ky, y). This can convert parallelograms into rectangles, simplifying area and intersection calculations. 

    // Shear to convert a parallelogram into a rectangle
// If the parallelogram has one side along the x-axis,
// shear by -cot(angle) to make it rectangular
Vec2 shearX(Vec2 p, double k) {
    return {p.x + k * p.y, p.y};
}

Affine shearXMatrix(double k) {
    Affine s;
    s.m[0][1] = k;
    return s;
}

    Summary: Geometric transformations are not just theoretical tools — they are practical problem-solving weapons. By transforming the coordinate system, you can often reduce a complex problem to a simple one. The affine matrix framework unifies all these transforms and enables efficient composition, exponentiation, and inversion. Keep the integer-friendly transforms (90° rotation, reflection, Chebyshev trick) in your mental toolkit for instant use in contests.
