Lines & Segments: Representation and Intersection

July 2025 · 12 min read

Lines and segments are the next building block after points and vectors. Choosing the right representation determines whether your intersection code is clean or an unreadable nest of special cases. We'll cover three standard representations and a robust segment-intersection algorithm.

Line Representations

There are three common ways to represent a line in 2-D:

1. General form: ax + by = c

Given two points P and Q, the coefficients are: 

struct Line {
    double a, b, c;
    // From two points
    Line(Point p, Point q) {
        a = q.y - p.y;
        b = p.x - q.x;
        c = a * p.x + b * p.y;
    }
};

The normal vector of the line is (a, b), and the direction vector is (-b, a).

2. Two-point form

Simply store the two points. This is the most compact representation and pairs well with vector operations. 

struct Line2 {
    Point p, q;
    Point dir() const { return q - p; }
};

3. Slope-intercept: y = mx + b

Avoid this form in CP—it cannot represent vertical lines and introduces special cases everywhere.

Parametric Form

The parametric form expresses every point on a line (or segment) as:

P(t) = A + t · (B − A)

  • t = 0 ⇒ point A
  • t = 1 ⇒ point B
  • 0 ≤ t ≤ 1 ⇒ segment AB
  • t ∈ (−∞, +∞) ⇒ full line through A, B

This form is extremely powerful for intersection: find the value of t where two parameterized entities meet, then check whether t is in the valid range. 

// Point on segment/line at parameter t
Point at(Point a, Point b, double t) {
    return a + (b - a) * t;
}

Segment Intersection

Given segments AB and CD, we want to determine whether they intersect and, if so, find the intersection point. The standard approach uses the cross product.

Algorithm overview:

  1. Compute the orientation of C and D with respect to line AB.
  2. Compute the orientation of A and B with respect to line CD.
  3. If the orientations differ on each side, the segments cross.
  4. Handle the degenerate case where a point is exactly on the other segment (orientation = 0).
double cross(Point o, Point a, Point b) {
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x);
}

bool onSegment(Point p, Point a, Point b) {
    return min(a.x, b.x) <= p.x + EPS && p.x <= max(a.x, b.x) + EPS
        && min(a.y, b.y) <= p.y + EPS && p.y <= max(a.y, b.y) + EPS;
}

bool segmentsIntersect(Point a, Point b, Point c, Point d) {
    double d1 = cross(c, d, a);
    double d2 = cross(c, d, b);
    double d3 = cross(a, b, c);
    double d4 = cross(a, b, d);
    if (((d1 > 0 && d2 < 0) || (d1 < 0 && d2 > 0)) &&
        ((d3 > 0 && d4 < 0) || (d3 < 0 && d4 > 0)))
        return true;
    if (fabs(d1) < EPS && onSegment(a, c, d)) return true;
    if (fabs(d2) < EPS && onSegment(b, c, d)) return true;
    if (fabs(d3) < EPS && onSegment(c, a, b)) return true;
    if (fabs(d4) < EPS && onSegment(d, a, b)) return true;
    return false;
}

Complexity: O(1) per pair of segments.

Special Cases

Segment intersection is notorious for edge cases. Here is a summary:

  • Proper intersection: Segments cross in their interiors.
  • T-touch: An endpoint of one segment lies on the other. The onSegment check catches this.
  • Collinear overlap: Both segments lie on the same line and share a sub-segment. This requires a bounding-box check.
  • Parallel / collinear disjoint: No intersection.

Full Implementation

Here is a complete function that returns the intersection point (if unique) using the parametric approach with Cramer's rule: 

// Returns true if segments AB and CD intersect.
// If they do and the intersection is unique, sets 'out' to the point.
bool segIntersection(Point a, Point b, Point c, Point d, Point &out) {
    Point ab = b - a, cd = d - c, ac = c - a;
    double denom = cross(ab, cd);
    if (fabs(denom) < EPS) {
        // Parallel or collinear
        return false; // extend for overlap if needed
    }
    double t = cross(ac, cd) / denom;
    double u = cross(ac, ab) / denom;
    if (t >= -EPS && t <= 1 + EPS &&
        u >= -EPS && u <= 1 + EPS) {
        out = a + ab * t;
        return true;
    }
    return false;
}

// Line-line intersection (infinite lines through a-b and c-d)
Point lineIntersection(Point a, Point b, Point c, Point d) {
    Point ab = b - a, cd = d - c, ac = c - a;
    double t = cross(ac, cd) / cross(ab, cd);
    return a + ab * t;
}

Complexity: O(1). The function uses two cross products and a division. For integer coordinates, multiply through to avoid floating-point when you only need to test existence.

Point-to-Line Distance

The distance from point P to the line through A, B is:

dist = |cross(B − A, P − A)| / |B − A| 

double pointToLine(Point p, Point a, Point b) {
    return fabs(cross(b - a, p - a)) / mag(b - a);
}

// Closest point on segment AB to point P
Point closestOnSegment(Point p, Point a, Point b) {
    Point ab = b - a;
    double t = dot(p - a, ab) / dot(ab, ab);
    t = max(0.0, min(1.0, t));
    return a + ab * t;
}

double pointToSegment(Point p, Point a, Point b) {
    return mag(p - closestOnSegment(p, a, b));
}

The closestOnSegment function clamps the parameter t to [0, 1], ensuring we stay on the segment.

Segment-to-Segment Distance

A harder variant: find the minimum distance between two non-intersecting segments. The closest pair must be either an endpoint of one segment projected onto the other, or two endpoints. 

double segSegDist(Point a, Point b, Point c, Point d) {
    if (segmentsIntersect(a, b, c, d)) return 0;
    return min({pointToSegment(a, c, d),
                pointToSegment(b, c, d),
                pointToSegment(c, a, b),
                pointToSegment(d, a, b)});
}

Complexity: O(1). Each pointToSegment call is O(1), and we check at most four candidates plus the intersection test.

Common pitfall: Don't forget the endpoint-to-endpoint cases! The minimum distance isn't always achieved at a projection point; sometimes the nearest pair is simply two endpoints.

Line Reflection

Reflecting a point across a line is another common operation. Given a line through points A and B and a point P, the reflection P' is: 

Point reflect(Point p, Point a, Point b) {
    Point ab = b - a;
    Point ap = p - a;
    // Project ap onto ab
    double t = dot(ap, ab) / dot(ab, ab);
    Point foot = a + ab * t; // foot of perpendicular
    return foot * 2 - p;    // reflection = 2*foot - P
}

This is useful in billiard-style problems and light-reflection geometry tasks that appear in ICPC regionals.
