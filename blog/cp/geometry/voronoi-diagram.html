    Voronoi Diagrams

    July 2025 · 16 min read

    Given a set of sites (points) in the plane, the Voronoi diagram partitions the plane into cells, one per site, such that every point in a cell is closer to its site than to any other. This structure is foundational in computational geometry and appears in diverse applications from nearest-neighbor search to mesh generation.

    Definition and Properties

    Formally, for a set S = {s₁, s₂, …, sₙ} of sites, the Voronoi cell of site sᵢ is:

    V(sᵢ) = { p ∈ ℝ² : d(p, sᵢ) ≤ d(p, sⱼ) for all j ≠ i }

    Key properties:

    • Each Voronoi cell is a convex polygon (possibly unbounded).
    • Each Voronoi edge lies on the perpendicular bisector of two sites.
    • Each Voronoi vertex is equidistant from exactly three (or more, in degenerate cases) sites.
    • The diagram has at most 2n - 5 vertices and 3n - 6 edges for n ≥ 3 sites.
    • The dual of the Voronoi diagram is the Delaunay triangulation.

    The diagram shows six sites (pink) with their Voronoi edges (blue) and vertices (orange). Each cell Cᵢ contains all points nearest to site sᵢ.

    Construction Ideas

    There are several ways to construct a Voronoi diagram:

    • Incremental insertion: Add sites one at a time, splitting cells. O(n²) worst case.
    • Divide and conquer: Split sites by x-median, recurse, merge. O(n log n).
    • Fortune's sweep line: The most popular O(n log n) algorithm, described below.
    • Via Delaunay: Build the Delaunay triangulation first, then take its dual.

    In competitive programming, Voronoi diagrams are rarely constructed directly. Instead, contestants exploit the Voronoi-Delaunay duality: compute the Delaunay triangulation (using a library or known algorithm), then derive the Voronoi diagram if needed.

    Fortune's Algorithm

    Fortune's algorithm builds the Voronoi diagram with a sweep line moving from top to bottom (or left to right). It maintains a beach line — a curve made up of parabolic arcs, one per site above the sweep line.

    Two event types:

    1. Site event: The sweep line reaches a new site. A new parabolic arc is inserted into the beach line.
    2. Circle event: Three consecutive arcs on the beach line become co-circular. The middle arc shrinks to zero, creating a Voronoi vertex.

    In the diagram, the orange dashed line is the sweep line, the blue curve is the beach line (composed of parabolic arcs), and the dashed blue verticals show where breakpoints trace out Voronoi edges.

    Complexity: O(n log n) time and O(n) space. The event queue has at most 2n events (n site + at most n circle events active simultaneously), and each event is processed in O(log n) time.

        pq.push({sites[i].first, 0, nullptr});

    }
}

    Applications

    Voronoi diagrams appear in many practical and competitive programming contexts:

    • Nearest-neighbor queries: Point-location in the Voronoi diagram answers nearest-site queries.
    • Largest empty circle: The center of the largest circle containing no sites is a Voronoi vertex or lies on a Voronoi edge.
    • Facility location: The Voronoi diagram partitions customers into regions served by the nearest facility.
    • Minimum spanning tree: The Euclidean MST is a subgraph of the Delaunay triangulation (the dual of the Voronoi diagram).
    • Motion planning: The Voronoi diagram of obstacles provides the “safest” path (maximising clearance).

    Nearest Neighbor Queries

    Once the Voronoi diagram is built, answering “which site is closest to query point q?” reduces to point location: find which Voronoi cell contains q.

    Point location in a planar subdivision can be done in O(log n) per query after O(n log n) preprocessing (e.g., using a persistent balanced BST or a trapezoidal decomposition). In practice, however, competitive programmers rarely implement this. Instead:

    • Use a k-d tree for nearest-neighbor queries directly (O(√n) expected per query).
    • Use the Delaunay triangulation: start at any triangle and “walk” toward the query point using the adjacency structure.
    // Simple brute-force nearest neighbour (for reference)
int nearest(vector<pair<double,double>>& sites, double qx, double qy) {
    int best = 0;
    double bestDist = 1e18;
    for (int i = 0; i < (int)sites.size(); i++) {
        double dx = sites[i].first - qx;
        double dy = sites[i].second - qy;
        double d = dx*dx + dy*dy;
        if (d < bestDist) { bestDist = d; best = i; }
    }
    return best;
}

    Complexity: O(n) per query. For competitive programming, this is acceptable when q (number of queries) is small. For many queries, use a k-d tree.

    Implementation Notes

    Full implementation of Fortune's algorithm is rarely required in contests. Here are practical tips:

    • Use a library: If the contest allows, use CGAL (C++) or scipy.spatial.Voronoi (Python) for Voronoi construction.
    • Duality shortcut: Build the Delaunay triangulation (often simpler to implement) and derive the Voronoi diagram from the circumcenters of the Delaunay triangles.
    • Epsilon handling: Degenerate cases (four co-circular points, collinear sites) require careful epsilon comparisons or exact arithmetic.
    • Bounded diagrams: If you need a bounded Voronoi diagram, clip infinite edges against a bounding box.
    // Voronoi vertex from Delaunay triangle circumcenter
struct Point { double x, y; };

Point circumcenter(Point a, Point b, Point c) {
    double D = 2.0 * (a.x * (b.y - c.y) + b.x * (c.y - a.y) + c.x * (a.y - b.y));
    double ux = ((a.x*a.x + a.y*a.y) * (b.y - c.y) +
                 (b.x*b.x + b.y*b.y) * (c.y - a.y) +
                 (c.x*c.x + c.y*c.y) * (a.y - b.y)) / D;
    double uy = ((a.x*a.x + a.y*a.y) * (c.x - b.x) +
                 (b.x*b.x + b.y*b.y) * (a.x - c.x) +
                 (c.x*c.x + c.y*c.y) * (b.x - a.x)) / D;
    return {ux, uy};
}

    Weighted and Higher-Order Voronoi Diagrams

    Generalisations of the standard Voronoi diagram appear in advanced problems:

    • Additively weighted Voronoi: Each site sᵢ has a weight wᵢ. The cell of sᵢ contains points p where d(p, sᵢ) - wᵢ ≤ d(p, sⱼ) - wⱼ for all j. The boundaries become hyperbolic arcs instead of straight lines.
    • Multiplicatively weighted Voronoi: Cell boundaries are circles (Apollonius circles). Used in facility location with varying service radii.
    • k-th order Voronoi: Each cell corresponds to a set of k nearest sites. The first-order diagram is the standard Voronoi diagram. The second-order diagram partitions the plane into regions sharing the same two nearest sites. 
    // Additively weighted distance
struct WeightedSite {
    double x, y, w;
    double dist(double px, double py) const {
        double dx = x - px, dy = y - py;
        return sqrt(dx*dx + dy*dy) - w;
    }
};

// Brute-force nearest weighted site
int nearestWeighted(vector<WeightedSite>& sites, double qx, double qy) {
    int best = 0;
    double bestD = sites[0].dist(qx, qy);
    for (int i = 1; i < (int)sites.size(); i++) {
        double d = sites[i].dist(qx, qy);
        if (d < bestD) { bestD = d; best = i; }
    }
    return best;
}

    Competition context: While weighted Voronoi diagrams rarely appear explicitly, the underlying concepts (bisector types, distance functions) show up in problems involving facilities with different ranges or priorities.

    Common Competition Problems

    Here are typical problem patterns involving Voronoi-like reasoning:

    1. “Which site is closest?” — Direct Voronoi query. In practice, use brute force for small n or a k-d tree for larger n.
    2. “Largest empty circle” — The center of the largest empty circle (containing no sites) is either a Voronoi vertex or a point on the convex hull boundary. Check all Voronoi vertices and all projections of sites onto hull edges.
    3. “Minimum enclosing circle of nearest-neighbour regions” — Uses the Voronoi cell boundaries.
    4. “Euclidean MST” — Build Delaunay triangulation (dual of Voronoi), extract edges, and run Kruskal's. 
    // Largest empty circle center candidates: Voronoi vertices
// After computing Voronoi vertices as circumcenters of Delaunay triangles:
double largestEmptyCircle(vector<pair<double,double>>& sites,
                          vector<pair<double,double>>& voronoiVerts,
                          double xmin, double ymin, double xmax, double ymax) {
    double best = 0;
    for (auto& v : voronoiVerts) {
        if (v.first < xmin || v.first > xmax) continue;
        if (v.second < ymin || v.second > ymax) continue;
        double minD = 1e18;
        for (auto& s : sites) {
            double dx = s.first - v.first;
            double dy = s.second - v.second;
            minD = min(minD, sqrt(dx*dx + dy*dy));
        }
        best = max(best, minD);
    }
    return best;
}

    Summary

    The Voronoi diagram encodes the nearest-site relationship for an entire plane. Its edges lie on perpendicular bisectors, and its dual is the Delaunay triangulation. Fortune's sweep-line algorithm constructs it in O(n log n) time. While full implementation is complex, understanding the structure and its duality with Delaunay triangulation is invaluable for solving geometry problems in competitions. Weighted and higher-order variants extend the basic concept to more complex distance functions and multi-nearest-site queries.
