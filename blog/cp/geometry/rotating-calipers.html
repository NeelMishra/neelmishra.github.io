    Rotating Calipers: Diameter and Beyond

    July 2025 · 19 min read

    Antipodal Pairs

    Two vertices of a convex polygon are antipodal if there exist two parallel lines of support (tangent lines) that pass through them. Informally, they are "opposite" vertices that can simultaneously touch two parallel rulers.

    Antipodal pairs are the foundation of the rotating calipers technique. Key properties:

    • A convex polygon with n vertices has at most n antipodal pairs.
    • The diameter (maximum distance between any two vertices) is always achieved by an antipodal pair.
    • Antipodal pairs can be enumerated in O(n) time using the rotating calipers.

    An antipodal pair (vi, vj) with parallel lines of support. The diameter is the longest such pair.

    The Rotating Calipers Technique

    Imagine holding a pair of calipers (two parallel jaws) against the convex polygon. As you rotate the calipers around the polygon, the pair of vertices touching the jaws traces out all antipodal pairs.

    How It Works

    1. Find the topmost and bottommost vertices — this is the initial antipodal pair and the initial caliper orientation (horizontal).
    2. At each step, consider the edges adjacent to each caliper vertex. Rotate the calipers to align with whichever edge makes a smaller angle with the current orientation.
    3. Advance the corresponding pointer. Record the new antipodal pair.
    4. Stop after a full rotation (total angle ≥ π).

    The key insight: we advance two pointers around the polygon, and each pointer advances a total of at most n steps, giving O(n) total.

    The rotating calipers sweep around the polygon, identifying each antipodal pair in O(n) total time.

    Angle Comparison via Cross Product

    Instead of computing actual angles, we compare which edge "turns more" using the cross product. This avoids trigonometric functions and floating-point issues:

    typedef long long ll;
struct Point { ll x, y; };

Point sub(Point a, Point b) { return {a.x - b.x, a.y - b.y}; }
ll cross(Point a, Point b) { return a.x * b.y - a.y * b.x; }
ll dot(Point a, Point b) { return a.x * b.x + a.y * b.y; }

ll dist2(Point a, Point b) {
    ll dx = a.x - b.x, dy = a.y - b.y;
    return dx * dx + dy * dy;
}

    Computing the Diameter

    The diameter of a convex polygon is the maximum distance between any pair of vertices. Since this maximum is always achieved by an antipodal pair, we just need to enumerate all antipodal pairs and track the maximum distance.

    // Convex polygon diameter in O(n)
// poly must be convex, counter-clockwise, n >= 2
ll diameter_squared(vector<Point>& poly) {
    int n = poly.size();
    if (n == 1) return 0;
    if (n == 2) return dist2(poly[0], poly[1]);

    // Find the vertex farthest from edge (poly[0], poly[1])
    int j = 1;
    while (cross(sub(poly[(j+1)%n], poly[j]),
                 sub(poly[1], poly[0]))
           > cross(sub(poly[j], poly[(j-1+n)%n]),
                   sub(poly[1], poly[0]))) {
        j = (j + 1) % n;
    }
    // Actually simpler: find the point farthest from poly[0]
    // by iterating until distance starts decreasing
    j = 0;
    for (int i = 1; i < n; i++)
        if (poly[i].y < poly[j].y ||
            (poly[i].y == poly[j].y && poly[i].x > poly[j].x))
            j = i;

    ll best = 0;
    for (int i = 0, k = j; i <= j; i++) {
        best = max(best, dist2(poly[i], poly[k]));
        while (k < n - 1) {
            Point edge_i = sub(poly[(i+1)%n], poly[i]);
            Point edge_k = sub(poly[(k+1)%n], poly[k]);
            if (cross(edge_i, edge_k) > 0) {
                k++;
                best = max(best, dist2(poly[i], poly[k]));
            } else break;
        }
    }
    return best;
}

    A cleaner implementation uses the standard two-pointer approach:

    // Cleaner rotating calipers for diameter
ll diameter_sq(vector<Point>& poly) {
    int n = poly.size();
    if (n <= 2) {
        if (n == 2) return dist2(poly[0], poly[1]);
        return 0;
    }

    // Find bottom-most and top-most vertices
    int bot = 0, top = 0;
    for (int i = 1; i < n; i++) {
        if (poly[i].y < poly[bot].y) bot = i;
        if (poly[i].y > poly[top].y) top = i;
    }

    ll best = dist2(poly[bot], poly[top]);
    int i = bot, j = top;
    int rotations = 0;

    while (rotations < 2 * n) {
        Point ei = sub(poly[(i + 1) % n], poly[i]);
        Point ej = sub(poly[(j + 1) % n], poly[j]);

        if (cross(ei, ej) > 0) {
            i = (i + 1) % n;
        } else if (cross(ei, ej) < 0) {
            j = (j + 1) % n;
        } else {
            // Both edges are parallel — advance both
            i = (i + 1) % n;
            j = (j + 1) % n;
        }
        best = max(best, dist2(poly[i], poly[j]));
        rotations++;
    }
    return best;
}
    Squared distance: We compute dist2 instead of dist to avoid floating-point. Return the squared distance and take the square root only at the very end if needed.

    Width of a Convex Polygon

    The width is the minimum distance between any pair of parallel support lines. While the diameter finds the maximum antipodal distance, the width finds the minimum distance from any edge to its antipodal vertex.

    Algorithm

    For each edge of the polygon, find the vertex farthest from it (the antipodal vertex for that edge). The distance from the edge to this vertex gives the "width" in that direction. The polygon width is the minimum over all edges.

    // Width of convex polygon (minimum distance between parallel support lines)
double polygon_width(vector<Point>& poly) {
    int n = poly.size();
    if (n <= 2) return 0;

    int j = 1;
    double min_width = 1e18;

    for (int i = 0; i < n; i++) {
        Point edge = sub(poly[(i + 1) % n], poly[i]);
        // Advance j to the farthest vertex from edge i
        while (true) {
            Point next_edge = sub(poly[(j + 1) % n], poly[j]);
            // The cross product gives twice the triangle area
            // which is proportional to distance from the line
            ll cur  = cross(edge, sub(poly[j], poly[i]));
            ll nxt  = cross(edge, sub(poly[(j+1)%n], poly[i]));
            if (nxt > cur)
                j = (j + 1) % n;
            else
                break;
        }
        // Distance from poly[j] to line through poly[i]-poly[(i+1)%n]
        ll area2 = abs(cross(edge, sub(poly[j], poly[i])));
        double edge_len = sqrt((double)dot(edge, edge));
        double width = (double)area2 / edge_len;
        min_width = min(min_width, width);
    }
    return min_width;
}

    The width is measured as the perpendicular distance between a polygon edge and its antipodal vertex.

    Minimum Enclosing Rectangle

    The minimum-area enclosing rectangle of a convex polygon always has one side flush with a polygon edge (a result from the rotating calipers theorem). This means we only need to check n candidate rectangles.

    Algorithm

    1. For each edge of the polygon, find:
      • The farthest vertex (antipodal) — gives the rectangle height.
      • The leftmost and rightmost vertices when projected onto the edge direction — gives the rectangle width.
    2. Compute the rectangle area for each edge and take the minimum.
    3. All three "extreme" pointers advance monotonically, so total time is O(n).
    // Minimum area enclosing rectangle of a convex polygon
// Returns the area
double min_enclosing_rect(vector<Point>& poly) {
    int n = poly.size();
    if (n <= 2) return 0;

    int top = 0, left = 0, right = 0;
    double min_area = 1e18;

    for (int i = 0; i < n; i++) {
        Point edge = sub(poly[(i + 1) % n], poly[i]);

        // Advance top (farthest from edge)
        while (cross(edge, sub(poly[(top+1)%n], poly[i]))
             > cross(edge, sub(poly[top], poly[i])))
            top = (top + 1) % n;

        // Initialize right on first iteration
        if (i == 0) right = top;

        // Advance right (largest projection along edge)
        while (dot(edge, sub(poly[(right+1)%n], poly[i]))
             > dot(edge, sub(poly[right], poly[i])))
            right = (right + 1) % n;

        // Advance left (smallest projection along edge)
        while (dot(edge, sub(poly[(left+1)%n], poly[i]))
             < dot(edge, sub(poly[left], poly[i])))
            left = (left + 1) % n;

        // Compute rectangle dimensions
        double edge_len2 = (double)dot(edge, edge);
        double height = (double)abs(cross(edge,
                        sub(poly[top], poly[i]))) / sqrt(edge_len2);
        double width  = (double)(dot(edge, sub(poly[right], poly[i]))
                      - dot(edge, sub(poly[left], poly[i])))
                      / sqrt(edge_len2);
        min_area = min(min_area, height * width);
    }
    return min_area;
}

    Complexity

    • O(n) for the rotating calipers (three pointers, each advancing at most n steps).
    • Requires a convex polygon as input. Compute the convex hull first if needed.

    Full Implementation & Summary

    Here is a complete implementation that computes diameter, width, and minimum enclosing rectangle:

    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
struct Point { ll x, y; };

Point sub(Point a, Point b) { return {a.x - b.x, a.y - b.y}; }
ll cross(Point a, Point b) { return a.x * b.y - a.y * b.x; }
ll dot(Point a, Point b) { return a.x * b.x + a.y * b.y; }
ll dist2(Point a, Point b) {
    ll dx = a.x - b.x, dy = a.y - b.y;
    return dx * dx + dy * dy;
}

// Diameter (squared) of convex polygon
ll diameter(vector<Point>& p) {
    int n = p.size();
    if (n == 1) return 0;
    if (n == 2) return dist2(p[0], p[1]);

    int j = 1;
    ll best = 0;
    for (int i = 0; i < n; i++) {
        Point e = sub(p[(i+1)%n], p[i]);
        while (cross(e, sub(p[(j+1)%n], p[i]))
             > cross(e, sub(p[j], p[i])))
            j = (j + 1) % n;
        best = max(best, max(dist2(p[i], p[j]),
                             dist2(p[(i+1)%n], p[j])));
    }
    return best;
}

// Width of convex polygon
double width(vector<Point>& p) {
    int n = p.size();
    if (n <= 2) return 0;
    int j = 1;
    double mn = 1e18;
    for (int i = 0; i < n; i++) {
        Point e = sub(p[(i+1)%n], p[i]);
        while (cross(e, sub(p[(j+1)%n], p[i]))
             > cross(e, sub(p[j], p[i])))
            j = (j + 1) % n;
        ll a2 = abs(cross(e, sub(p[j], p[i])));
        double el = sqrt((double)dot(e, e));
        mn = min(mn, (double)a2 / el);
    }
    return mn;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<Point> poly(n);
    for (int i = 0; i < n; i++) cin >> poly[i].x >> poly[i].y;

    cout << fixed << setprecision(9);
    cout << "Diameter: " << sqrt((double)diameter(poly)) << "\n";
    cout << "Width: " << width(poly) << "\n";
}

    Competition Tips

    • Always compute the convex hull first. Rotating calipers only work on convex polygons.
    • Use squared distances and integer arithmetic where possible.
    • Watch for collinear points on the hull — they can cause the caliper pointers to stall. Use >= instead of > when advancing to handle ties.
    • For the minimum enclosing rectangle, remember to initialize the right pointer at the top vertex, not at 0.
    • The diameter of a point set = convex hull diameter. Points inside the hull are never part of the diameter pair.
    Key takeaway: Rotating calipers transform O(n²) problems on convex polygons into O(n) by exploiting the monotone advancement of antipodal pointers. Master the two-pointer pattern — it appears in diameter, width, enclosing rectangle, and inter-polygon distance problems.
