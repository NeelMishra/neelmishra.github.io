    Line Intersection: All Cases Covered

    July 2025 · 20 min read

    Line–Line Intersection

    Two infinite lines in 2D either intersect at exactly one point, are parallel (no intersection), or are coincident (infinite intersections). We solve for the intersection using Cramer's rule.

    Parametric Form

    Represent each line parametrically. Line 1 passes through points A and B; line 2 passes through C and D:

    • Line 1: P = A + t(B − A)
    • Line 2: P = C + u(D − C)

    Setting them equal and solving for t and u using the cross product:

    // Cross product of 2D vectors
double cross(double ax, double ay, double bx, double by) {
    return ax * by - ay * bx;
}

// Line-line intersection
// Lines: A-B and C-D
// Returns true if they intersect; sets (px, py) to intersection point
bool line_line(double ax, double ay, double bx, double by,
               double cx, double cy, double dx, double dy,
               double &px, double &py) {
    double d1x = bx - ax, d1y = by - ay;
    double d2x = dx - cx, d2y = dy - cy;
    double denom = cross(d1x, d1y, d2x, d2y);
    if (abs(denom) < 1e-9) return false;  // parallel or coincident
    double t = cross(cx - ax, cy - ay, d2x, d2y) / denom;
    px = ax + t * d1x;
    py = ay + t * d1y;
    return true;
}

    Two non-parallel lines always intersect at exactly one point.

    Cramer's Rule Derivation

    The system of equations is:

    • t·d1x − u·d2x = cx − ax
    • t·d1y − u·d2y = cy − ay

    The denominator is the cross product d1 × d2. When this is zero, the lines are parallel. The numerator for t is (C−A) × d2.

    Line–Segment Intersection

    Given an infinite line (through A, B) and a segment (from C to D), we need the intersection point to lie on the segment. Compute t and u as before, then check 0 ≤ u ≤ 1.

    // Line (A-B) with segment (C-D)
// Returns true if they intersect; sets (px,py)
bool line_segment(double ax, double ay, double bx, double by,
                  double cx, double cy, double dx, double dy,
                  double &px, double &py) {
    double d1x = bx - ax, d1y = by - ay;
    double d2x = dx - cx, d2y = dy - cy;
    double denom = cross(d1x, d1y, d2x, d2y);
    if (abs(denom) < 1e-9) return false;
    double t = cross(cx - ax, cy - ay, d2x, d2y) / denom;
    double u = cross(cx - ax, cy - ay, d1x, d1y) / denom;
    if (u < -1e-9 || u > 1 + 1e-9) return false;  // outside segment
    px = ax + t * d1x;
    py = ay + t * d1y;
    return true;
}

    The parameter u must lie in [0, 1] for the intersection to be on the segment.

    Segment–Segment Intersection

    For two segments (A–B and C–D), both parameters must satisfy 0 ≤ t ≤ 1 and 0 ≤ u ≤ 1. This is the most common case in competitive programming.

    Cross Product Approach

    An alternative to computing the actual intersection: just test whether the segments cross using orientation tests.

    typedef long long ll;
struct Point { ll x, y; };

ll cross(Point O, Point A, Point B) {
    return (A.x - O.x) * (ll)(B.y - O.y)
         - (A.y - O.y) * (ll)(B.x - O.x);
}

bool on_seg(Point p, Point a, Point b) {
    return cross(a, b, p) == 0
        && min(a.x, b.x) <= p.x && p.x <= max(a.x, b.x)
        && min(a.y, b.y) <= p.y && p.y <= max(a.y, b.y);
}

// Returns true if segments AB and CD intersect (including endpoints)
bool segments_intersect(Point a, Point b, Point c, Point d) {
    ll d1 = cross(c, d, a);
    ll d2 = cross(c, d, b);
    ll d3 = cross(a, b, c);
    ll d4 = cross(a, b, d);

    // Standard crossing test
    if (((d1 > 0 && d2 < 0) || (d1 < 0 && d2 > 0)) &&
        ((d3 > 0 && d4 < 0) || (d3 < 0 && d4 > 0)))
        return true;

    // Collinear / endpoint cases
    if (d1 == 0 && on_seg(a, c, d)) return true;
    if (d2 == 0 && on_seg(b, c, d)) return true;
    if (d3 == 0 && on_seg(c, a, b)) return true;
    if (d4 == 0 && on_seg(d, a, b)) return true;

    return false;
}

    This is the standard approach for segment intersection in competitive programming. It uses only integer arithmetic and handles all degenerate cases.

    The four cases of segment–segment interaction: crossing, endpoint touch, parallel, and collinear overlap.

    Parallel & Overlapping Cases

    When the denominator (cross product of direction vectors) is zero, the lines are parallel. But they might be coincident — the same line — and the segments might overlap.

    Detecting Collinear Overlap

    If the segments are collinear, project them onto the axis with the larger span and check if the intervals overlap:

    // Returns true if collinear segments AB and CD overlap
// Assumes they ARE collinear (cross product is 0)
bool collinear_overlap(Point a, Point b, Point c, Point d) {
    // Project onto x-axis (or y-axis if vertical)
    if (a.x == b.x) {
        ll lo1 = min(a.y, b.y), hi1 = max(a.y, b.y);
        ll lo2 = min(c.y, d.y), hi2 = max(c.y, d.y);
        return lo1 <= hi2 && lo2 <= hi1;
    }
    ll lo1 = min(a.x, b.x), hi1 = max(a.x, b.x);
    ll lo2 = min(c.x, d.x), hi2 = max(c.x, d.x);
    return lo1 <= hi2 && lo2 <= hi1;
}

    The Full Decision Tree

    1. Compute denom = d1 × d2
    2. If denom ≠ 0: compute t, u. Check bounds → intersection or no intersection.
    3. If denom = 0: lines are parallel.
      • Check if they're the same line (is C on line AB?).
      • If same line: check interval overlap → collinear overlap or no overlap.
      • If different lines: no intersection.
    Integer vs floating-point: The boolean intersection test (do they cross?) can be done entirely with long long arithmetic. Computing the actual intersection coordinates requires division, so use double or return as a fraction.

    Full Implementation

    Here is a comprehensive implementation that handles all cases and returns detailed intersection information.

    #include <bits/stdc++.h>
using namespace std;

typedef long long ll;
typedef pair<double,double> pdd;

struct Point { ll x, y; };

ll cross(Point O, Point A, Point B) {
    return (A.x - O.x) * (ll)(B.y - O.y)
         - (A.y - O.y) * (ll)(B.x - O.x);
}

bool on_seg(Point p, Point a, Point b) {
    return cross(a, b, p) == 0
        && min(a.x, b.x) <= p.x && p.x <= max(a.x, b.x)
        && min(a.y, b.y) <= p.y && p.y <= max(a.y, b.y);
}

// Result codes:
// 0 = no intersection
// 1 = single intersection point (stored in res)
// 2 = collinear overlap (segment)
int seg_intersect(Point a, Point b, Point c, Point d, pdd &res) {
    ll d1 = cross(c, d, a);
    ll d2 = cross(c, d, b);
    ll d3 = cross(a, b, c);
    ll d4 = cross(a, b, d);

    if (((d1 > 0 && d2 < 0) || (d1 < 0 && d2 > 0)) &&
        ((d3 > 0 && d4 < 0) || (d3 < 0 && d4 > 0))) {
        // Proper intersection — compute coordinates
        double t = (double)d1 / (d1 - d2);
        res.first  = a.x + t * (b.x - a.x);
        res.second = a.y + t * (b.y - a.y);
        return 1;
    }

    // Check endpoint cases
    if (d1 == 0 && on_seg(a, c, d)) { res = {(double)a.x, (double)a.y}; return 1; }
    if (d2 == 0 && on_seg(b, c, d)) { res = {(double)b.x, (double)b.y}; return 1; }
    if (d3 == 0 && on_seg(c, a, b)) { res = {(double)c.x, (double)c.y}; return 1; }
    if (d4 == 0 && on_seg(d, a, b)) { res = {(double)d.x, (double)d.y}; return 1; }

    // Check collinear overlap
    if (d1 == 0 && d2 == 0) {
        // Segments are collinear — check overlap
        auto proj = [](Point p) -> pair<ll,ll> { return {p.x, p.y}; };
        // Sort endpoints on the line
        auto cmp = [](Point p, Point q) {
            return p.x != q.x ? p.x < q.x : p.y < q.y;
        };
        Point pts[4] = {a, b, c, d};
        if (cmp(b, a)) swap(pts[0], pts[1]);
        if (cmp(d, c)) swap(pts[2], pts[3]);
        Point lo = max(pts[0], pts[2], cmp);
        Point hi = min(pts[1], pts[3], cmp);
        if (!cmp(hi, lo)) return 2;  // overlap exists
    }

    return 0;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        Point a, b, c, d;
        cin >> a.x >> a.y >> b.x >> b.y;
        cin >> c.x >> c.y >> d.x >> d.y;
        pdd res;
        int code = seg_intersect(a, b, c, d, res);
        if (code == 0) cout << "NO\n";
        else if (code == 1)
            cout << fixed << setprecision(9) << res.first << " " << res.second << "\n";
        else cout << "OVERLAP\n";
    }
}

    Complexity

    O(1) per intersection test. Each test involves a constant number of cross product computations and comparisons.

    Competition Tips & Common Mistakes

    Pitfalls to Watch

    • Forgetting endpoint intersections: The "proper crossing" test misses cases where segments touch at endpoints. Always add the collinear checks.
    • Overflow with long long: Cross products of points with coordinates up to 10⁹ produce values up to ~10¹⁸, which fits in long long. But if you multiply two cross products, you overflow — use __int128 or restructure.
    • Division for coordinates: When you need the actual intersection point (not just yes/no), you must divide. Use double or return a rational number as a pair of integers.
    • Collinear segments touching at a single point: This is the sneakiest case. Two collinear segments might share exactly one endpoint — this should return "intersect" with that point.

    Applications in Competitive Programming

    Line and segment intersection is a building block for many advanced geometry problems:

    Polygon Intersection Area

    To compute the intersection area of two convex polygons, find all edge–edge intersections and all vertices of one polygon inside the other. The intersection vertices form a new convex polygon whose area can be computed via the shoelace formula.

    Line Sweep Problems

    Many problems involve counting intersections among n segments. The Bentley–Ottmann sweep line algorithm solves this in O((n + k) log n) where k is the number of intersections. The core primitive is the segment–segment intersection test.

    Visibility and Ray Tracing

    Given a point source and a set of line segments (walls), compute the visible area. This requires testing ray–segment intersections for each wall and finding the nearest hit.

    // Find nearest intersection of ray from origin in direction d
// with a set of walls (segments)
struct Ray { double ox, oy, dx, dy; };
struct Wall { Point a, b; };

double nearest_hit(Ray r, vector<Wall>& walls) {
    double best_t = 1e18;
    for (auto& w : walls) {
        double wx = w.b.x - w.a.x, wy = w.b.y - w.a.y;
        double denom = r.dx * wy - r.dy * wx;
        if (abs(denom) < 1e-9) continue;
        double t = ((w.a.x - r.ox) * wy - (w.a.y - r.oy) * wx) / denom;
        double u = ((w.a.x - r.ox) * r.dy - (w.a.y - r.oy) * r.dx) / denom;
        if (t > 1e-9 && u > -1e-9 && u < 1 + 1e-9)
            best_t = min(best_t, t);
    }
    return best_t;
}

    Art Gallery and Triangulation

    In polygon triangulation problems (e.g., ear clipping), you must check that proposed diagonals don't intersect existing edges. This is a direct application of segment–segment intersection.

    A light source casts rays that intersect wall segments. The nearest hit for each ray determines visibility.

    Summary: Master the cross-product based boolean test for segments first. Add the parametric form when you need actual coordinates. Always handle the four cases: proper crossing, endpoint touch, parallel miss, and collinear overlap.
