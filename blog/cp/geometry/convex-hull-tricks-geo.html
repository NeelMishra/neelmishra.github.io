    Convex Hull Tricks in Geometry

    July 2025 · 19 min read

    Convex hulls are not just for finding boundaries — they enable powerful geometric operations. In this post we cover the Minkowski sum of convex polygons, distance queries between convex shapes, the support function, and farthest point queries. These techniques appear in ICPC, Codeforces, and advanced optimization problems.

    Minkowski Sum Basics

    The Minkowski sum of two sets A and B is defined as:

    A ⊕ B = { a + b : a ∈ A, b ∈ B }

    For two convex polygons P (with n vertices) and Q (with m vertices), the Minkowski sum is also a convex polygon with at most n + m vertices.

    Key properties:

    • The Minkowski sum of convex sets is convex.
    • The edges of P ⊕ Q are exactly the edges of P and Q sorted by angle.
    • A point x is within distance d of a convex polygon P if and only if x lies in P ⊕ D, where D is a disk of radius d.

    Construction Algorithm

    To compute the Minkowski sum of two convex polygons in O(n + m):

    1. Start both polygons at their bottom-most (then left-most) vertex.
    2. Compute the edge vectors for both polygons.
    3. Merge the edge vectors in sorted angular order (like merge sort).
    4. Accumulate the edge vectors starting from the sum of the two starting vertices.
    typedef vector<Point> Polygon;

// Ensure polygon is counterclockwise and starts at bottom-left vertex
void reorder(Polygon& P) {
    int idx = 0;
    for (int i = 1; i < (int)P.size(); i++) {
        if (P[i].y < P[idx].y ||
            (P[i].y == P[idx].y && P[i].x < P[idx].x))
            idx = i;
    }
    rotate(P.begin(), P.begin() + idx, P.end());
}

Polygon minkowskiSum(Polygon P, Polygon Q) {
    reorder(P);
    reorder(Q);

    int n = P.size(), m = Q.size();
    // Compute edge vectors
    vector<Point> edgesP(n), edgesQ(m);
    for (int i = 0; i < n; i++)
        edgesP[i] = P[(i+1)%n] - P[i];
    for (int i = 0; i < m; i++)
        edgesQ[i] = Q[(i+1)%m] - Q[i];

    Polygon result;
    result.push_back(P[0] + Q[0]);

    int i = 0, j = 0;
    while (i < n || j < m) {
        Point cur = result.back();
        if (i < n && j < m) {
            double cr = edgesP[i].cross(edgesQ[j]);
            if (cr > 0) {
                result.push_back(cur + edgesP[i++]);
            } else if (cr < 0) {
                result.push_back(cur + edgesQ[j++]);
            } else {
                result.push_back(cur + edgesP[i++] + edgesQ[j++]);
            }
        } else if (i < n) {
            result.push_back(cur + edgesP[i++]);
        } else {
            result.push_back(cur + edgesQ[j++]);
        }
    }
    result.pop_back();  // last vertex equals first
    return result;
}

    Complexity: O(n + m) after both polygons are already in convex hull form. If you need to compute hulls first, add O(n log n).

    Important detail: When two edge vectors have the same angle (cross product is zero), we combine them into a single step. This is the merge case that produces the n + m upper bound on output vertices.

    Distance Between Convex Polygons

    The minimum distance between two convex polygons P and Q can be computed using the Minkowski difference. Define:

    P ⊖ Q = P ⊕ (−Q) = { p − q : p ∈ P, q ∈ Q }

    Then:

    • P and Q intersect iff the origin lies inside P ⊖ Q.
    • The minimum distance between P and Q equals the distance from the origin to P ⊖ Q.
    // Negate a polygon (reflect through origin)
Polygon negatePolygon(const Polygon& Q) {
    Polygon neg(Q.size());
    for (int i = 0; i < (int)Q.size(); i++)
        neg[i] = {-Q[i].x, -Q[i].y};
    return neg;
}

// Minimum distance from origin to convex polygon
double distToOrigin(const Polygon& poly) {
    int n = poly.size();
    double best = 1e18;
    for (int i = 0; i < n; i++) {
        Point A = poly[i], B = poly[(i+1)%n];
        // Distance from origin to segment AB
        Point d = B - A;
        double t = max(0.0, min(1.0, -(A.dot(d)) / d.dot(d)));
        Point closest = A + d * t;
        best = min(best, closest.norm());
    }
    return best;
}

double minDistConvex(Polygon P, Polygon Q) {
    Polygon diff = minkowskiSum(P, negatePolygon(Q));
    return distToOrigin(diff);
}

    This elegantly reduces the two-polygon distance problem to a point-polygon distance problem. The same technique works for collision detection: two convex shapes collide iff the Minkowski difference contains the origin.

    Support Function

    The support function of a convex set S in direction d is:

    hS(d) = max { d · p : p ∈ S }

    It gives the signed distance from the origin to the supporting hyperplane of S perpendicular to d. The support function completely characterizes a convex body.

    Key properties:

    • hP⊕Q(d) = hP(d) + hQ(d) — additivity under Minkowski sum.
    • The width of P in direction d is hP(d) + hP(−d).
    • Two convex polygons are separated in direction d iff hP(d) + hQ(−d) < 0.
    // Support function: max dot product with direction d
// For a convex polygon, can be computed in O(log n) with binary search
double support(const Polygon& P, Point d) {
    int n = P.size();
    int lo = 0, hi = n;

    auto eval = [&](int i) { return d.dot(P[i % n]); };

    // Ternary search on the convex function
    while (hi - lo > 2) {
        int m1 = lo + (hi - lo) / 3;
        int m2 = hi - (hi - lo) / 3;
        if (eval(m1) < eval(m2))
            lo = m1;
        else
            hi = m2;
    }

    double best = -1e18;
    for (int i = lo; i <= hi; i++)
        best = max(best, eval(i));
    return best;
}

// O(n) version for simplicity
double supportLinear(const Polygon& P, Point d) {
    double best = -1e18;
    for (auto& p : P)
        best = max(best, d.dot(p));
    return best;
}

    The O(log n) version uses the fact that the dot product with a fixed direction is unimodal when evaluated along the vertices of a convex polygon in order.

    Applications

    These convex hull tricks have wide applications in competitive programming:

    • Robot motion planning: The configuration space obstacle for a convex robot moving among convex obstacles is computed via Minkowski sums. The robot can reach a target iff a point path exists in the complement of these obstacles.
    • Collision detection: Two convex shapes collide iff the origin lies in their Minkowski difference. Combined with GJK algorithm for real-time physics.
    • Polygon offsetting: Expanding a convex polygon by distance r is the Minkowski sum with a disk of radius r. For polygonal approximation, use the Minkowski sum with a regular polygon.
    • Maximum overlap of translated polygons: Given two convex polygons, find the translation that maximizes their overlap area. The Minkowski difference encodes all feasible translations.
    • Rotating calipers: Find the maximum distance between two convex polygons (diameter of the Minkowski sum) in O(n + m).

    Minkowski sum with queries: A common pattern in contests is to precompute the Minkowski sum and then answer multiple point-in-polygon queries on it. Each query checks if a point lies in the Minkowski sum, which determines some geometric property of the original objects.

    // Point in convex polygon test (O(log n))
bool inConvexPolygon(const Polygon& P, Point q) {
    int n = P.size();
    if (n < 3) return false;

    // Check if on the correct side of first and last edges
    if ((P[1] - P[0]).cross(q - P[0]) < -1e-9) return false;
    if ((P[n-1] - P[0]).cross(q - P[0]) > 1e-9) return false;

    // Binary search for the sector containing q
    int lo = 1, hi = n - 1;
    while (hi - lo > 1) {
        int mid = (lo + hi) / 2;
        if ((P[mid] - P[0]).cross(q - P[0]) >= 0)
            lo = mid;
        else
            hi = mid;
    }

    return (P[hi] - P[lo]).cross(q - P[lo]) >= -1e-9;
}
