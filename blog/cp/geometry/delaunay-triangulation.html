    Delaunay Triangulation

    July 2025 · 16 min read

    The Delaunay triangulation is the “nicest” way to triangulate a point set: it maximises the minimum angle across all triangles, avoids skinny triangles, and is the dual of the Voronoi diagram. In competitive programming, it provides the key structure for problems involving Euclidean minimum spanning trees, nearest-neighbour graphs, and more.

    Definition

    Given a set P of n points in the plane, a triangulation is a maximal set of non-crossing edges connecting points of P such that every bounded face is a triangle.

    The Delaunay triangulation DT(P) is the unique triangulation (assuming no four points are co-circular) satisfying the empty circumcircle property: the circumscribed circle of every triangle contains no other point of P in its interior.

    Properties of DT(P):

    • It maximises the minimum angle among all possible triangulations of P.
    • It has at most 2n - 2 - h triangles, where h is the number of convex hull vertices.
    • The Euclidean minimum spanning tree (EMST) is always a subgraph of DT(P).
    • The nearest-neighbour graph is a subgraph of DT(P).

    The diagram shows a Delaunay triangulation of 8 points. The orange dashed circle is the circumcircle of triangle (p₁, p₂, p₅) — note that no other point lies inside it.

    Voronoi Duality

    The Delaunay triangulation and Voronoi diagram are dual structures:

    • Each Voronoi vertex (equidistant from three sites) corresponds to a Delaunay triangle.
    • Each Voronoi edge (on the perpendicular bisector of two sites) corresponds to a Delaunay edge.
    • Each Voronoi cell (region closest to a site) corresponds to a Delaunay vertex (the site itself).

    This duality means that building one structure immediately gives the other. The circumcenter of each Delaunay triangle is a Voronoi vertex, and Voronoi edges connect adjacent circumcenters. 

    // Convert Delaunay triangle to Voronoi vertex
struct Point { double x, y; };

Point circumcenter(Point a, Point b, Point c) {
    double ax = a.x, ay = a.y;
    double bx = b.x, by = b.y;
    double cx = c.x, cy = c.y;
    double D = 2.0 * (ax * (by - cy) + bx * (cy - ay) + cx * (ay - by));
    double ux = ((ax*ax + ay*ay) * (by - cy) +
                 (bx*bx + by*by) * (cy - ay) +
                 (cx*cx + cy*cy) * (ay - by)) / D;
    double uy = ((ax*ax + ay*ay) * (cx - bx) +
                 (bx*bx + by*by) * (ax - cx) +
                 (cx*cx + cy*cy) * (bx - ax)) / D;
    return {ux, uy};
}

    Empty Circumcircle Property

    The defining property of Delaunay triangulation is: for every triangle in the triangulation, the circumscribed circle contains no other points from the set in its interior.

    Equivalently, an edge ab is a Delaunay edge if and only if there exists a circle through a and b with no other points inside. This leads to the in-circle test, the fundamental predicate for Delaunay algorithms: 

    // In-circle test: returns > 0 if d is inside circumcircle of (a, b, c)
// Assumes a, b, c are in counter-clockwise order
double inCircle(Point a, Point b, Point c, Point d) {
    double ax = a.x - d.x, ay = a.y - d.y;
    double bx = b.x - d.x, by = b.y - d.y;
    double cx = c.x - d.x, cy = c.y - d.y;
    return ax * (by * (cx*cx + cy*cy) - cy * (bx*bx + by*by))
         - ay * (bx * (cx*cx + cy*cy) - cx * (bx*bx + by*by))
         + (ax*ax + ay*ay) * (bx * cy - by * cx);
}

    Numerical precision: For integer coordinates up to 10⁵, the determinant values can reach 10¹⁵, so use long long or __int128 for exact computation. For floating-point inputs, use robust predicates (Shewchuk's predicates are the gold standard).

    Flip Algorithm

    The edge flip operation is the local repair mechanism for Delaunay triangulation. Given two triangles sharing an edge, if the shared edge violates the empty circumcircle property (the opposite vertex is inside the circumcircle), we flip the diagonal.

    The red edge is the non-Delaunay diagonal. After the flip, the green edge replaces it, and both resulting triangles satisfy the empty circumcircle property. 

    // Edge flip in a triangulation
// tri[i] = {a, b, c} indices; adj[i][k] = adjacent triangle across edge k
void flip(int t1, int t2, int sharedEdge1, int sharedEdge2,
          vector<array<int,3>>& tri, vector<array<int,3>>& adj) {
    int a = tri[t1][(sharedEdge1 + 2) % 3]; // opposite vertex in t1
    int b = tri[t2][(sharedEdge2 + 2) % 3]; // opposite vertex in t2
    int c = tri[t1][sharedEdge1];
    int d = tri[t1][(sharedEdge1 + 1) % 3];
    // Replace t1 = (a, c, d) and t2 = (b, d, c) with
    //         t1 = (a, b, d) and t2 = (a, c, b)
    tri[t1] = {a, b, d};
    tri[t2] = {a, c, b};
    // Update adjacency (omitted for brevity)
}

    Key theorem: Starting from any triangulation, repeatedly flipping non-Delaunay edges converges to the Delaunay triangulation in O(n²) flips.

    Incremental Construction

    The most practical algorithm for competitive programming is randomised incremental construction:

    1. Start with a large “super-triangle” enclosing all points.
    2. Insert points one at a time in random order.
    3. For each inserted point, locate the triangle containing it.
    4. Split that triangle into 2 or 3 new triangles.
    5. Flip edges as needed to restore the Delaunay property.
    6. Remove the super-triangle at the end.
    // Randomised incremental Delaunay triangulation (outline)

    // walk from a known triangle toward p
    // O(sqrt(n)) expected with random insertion order

        for (int i = 0; i < 3; i++) {

            if (cross < -1e-9) { cur = tris[cur].adj[i]; found = true; break; }

    Applications in Competitive Programming

    The Delaunay triangulation is a workhorse for several classic problems:

    • Euclidean MST: The EMST is a subgraph of DT(P). Build the Delaunay triangulation (O(n log n)), extract edges, sort by weight, and run Kruskal's algorithm.
    • Nearest-neighbour graph: For each point, its nearest neighbour is connected by a Delaunay edge. Just check all Delaunay edges.
    • Largest empty circle: The center of the largest circle containing no points is either a Voronoi vertex or lies on the convex hull. Compute via the dual Voronoi diagram.
    • Minimum-weight triangulation: Although NP-hard in general, the Delaunay triangulation gives a good approximation.
    // Euclidean MST via Delaunay triangulation
#include <bits/stdc++.h>
using namespace std;

struct Edge { int u, v; double w; };

double euclideanMST(vector<Pt>& pts) {
    // 1. Build Delaunay triangulation
    // 2. Extract all edges with their Euclidean weights
    vector<Edge> edges;
    for (auto& tri : tris) {
        if (tri.dead) continue;
        for (int i = 0; i < 3; i++) {
            int u = tri.v[i], v = tri.v[(i+1)%3];
            if (u < v) {
                double dx = pts[u].x - pts[v].x;
                double dy = pts[u].y - pts[v].y;
                edges.push_back({u, v, sqrt(dx*dx + dy*dy)});
            }
        }
    }
    // 3. Sort edges and run Kruskal's

    Complexity and Practical Considerations

    Understanding the complexity of different Delaunay algorithms helps choose the right approach for a given contest constraint:

    • Randomised incremental: O(n log n) expected. Simple to implement. Most popular in contests.
    • Divide and conquer: O(n log n) worst-case. More complex to code but deterministic.
    • Flip-based from arbitrary triangulation: O(n²) worst-case flips. Simple but slow.
    • Fortune's algorithm (via Voronoi dual): O(n log n). Useful if you also need the Voronoi diagram.

    For competitive programming with n ≤ 10⁵, the randomised incremental algorithm is the standard choice. For n ≤ 10³, even the O(n²) flip approach works. 

    // Practical Delaunay: using __int128 for exact in-circle test
// Avoids floating-point issues with integer coordinates
typedef long long ll;
typedef __int128 lll;

struct Pt2 { ll x, y; };

lll inCircleExact(Pt2 a, Pt2 b, Pt2 c, Pt2 d) {
    lll ax = a.x - d.x, ay = a.y - d.y;
    lll bx = b.x - d.x, by = b.y - d.y;
    lll cx = c.x - d.x, cy = c.y - d.y;
    lll a2 = ax*ax + ay*ay;
    lll b2 = bx*bx + by*by;
    lll c2 = cx*cx + cy*cy;
    return ax*(by*c2 - cy*b2) - ay*(bx*c2 - cx*b2) + a2*(bx*cy - by*cx);
}
// Returns > 0 if d is inside circumcircle of CCW triangle (a,b,c)

    Tip: When coordinates are up to 10⁶, the in-circle determinant can reach 10¹⁸, which overflows long long in intermediate products. Using __int128 (supported by GCC) avoids this. Alternatively, restructure the computation to avoid overflow.

    Common Pitfalls

    Implementing Delaunay triangulation has several tricky aspects:

    1. Super-triangle size: The initial super-triangle must be large enough to contain all points. Using coordinates like ±3×10⁹ works for most contest constraints. Too small a super-triangle causes missing triangles.
    2. Removing super-triangle: After construction, delete all triangles incident to the three super-triangle vertices. Forgetting this step leaves artificial triangles in the output.
    3. Degenerate inputs: Four or more co-circular points create ambiguity in the triangulation. The in-circle test returns zero, and the flip direction is arbitrary. Add a small symbolic perturbation or break ties consistently.
    4. Point location walk: The walk from a random triangle to the target may loop if the triangulation has errors. Add a visited-triangle check as a safeguard.
    5. Adjacency maintenance: The most bug-prone part is updating the adjacency structure after splits and flips. Draw diagrams and label edges carefully. 
    // Safe point location with loop detection
int locateTriangleSafe(Pt p, vector<Triangle>& tris, vector<Pt>& pts) {
    int cur = 0, steps = 0;
    int maxSteps = (int)tris.size() + 10;
    while (steps++ < maxSteps) {
        bool found = false;
        for (int i = 0; i < 3; i++) {
            Pt a = pts[tris[cur].v[i]];
            Pt b = pts[tris[cur].v[(i+1)%3]];
            double cross = (b.x-a.x)*(p.y-a.y) - (b.y-a.y)*(p.x-a.x);
            if (cross < -1e-9) {
                cur = tris[cur].adj[i];
                found = true;
                break;
            }
        }
        if (!found) return cur;
    }
    return -1; // should not happen with correct triangulation
}

    Summary

    The Delaunay triangulation maximises minimum angles, satisfies the empty circumcircle property, and is dual to the Voronoi diagram. The flip algorithm provides local repair, while randomised incremental construction achieves O(n log n) expected time. Its role as a supergraph of the Euclidean MST and nearest-neighbour graph makes it indispensable for geometric optimization problems in competitive programming. Use __int128 for exact predicates and handle degenerate cases (co-circular points, super-triangle removal) carefully.
