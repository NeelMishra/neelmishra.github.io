    Sweep Line in Geometry

    July 2025 · 18 min read

    The sweep line (or plane sweep) paradigm is one of the most versatile techniques in computational geometry. Instead of examining every pair of objects, we sweep an imaginary vertical line from left to right, processing events in sorted order. This lets us reduce many O(n²) problems to O(n log n) or even O((n + k) log n) where k is the output size.

    The Sweep Line Paradigm

    The core idea is simple: imagine a vertical line sweeping from x = -∞ to x = +∞. As the line moves, it maintains a status structure — a balanced BST or ordered set — that stores the geometric objects currently intersecting the sweep line, sorted by their y-coordinate at the current x.

    Key ingredients of every sweep-line algorithm:

    • Event queue — a priority queue (or sorted array) of x-coordinates where something interesting happens (segment starts, ends, or two segments swap order).
    • Status structure — the set of objects currently crossing the sweep line, sorted by y at the current x.
    • Event handling — at each event we update the status and possibly discover new events.

    In the diagram above, segments are coloured differently. The orange sweep line sits at an x-coordinate that has just discovered an intersection event (orange dot).

    Event Processing

    For segment-intersection problems, there are three event types:

    1. Left endpoint — a segment starts. Insert it into the status structure and check for intersections with its immediate neighbours.
    2. Right endpoint — a segment ends. Remove it and check if its former neighbours now intersect each other.
    3. Intersection — two segments swap their vertical order. Swap them in the status structure and check new neighbour pairs for further intersections.

    A critical implementation detail: the comparison function for the status structure must evaluate each segment's y-coordinate at the current sweep x. This means the comparator is dynamic and depends on global state. 

    // Dynamic comparator for segments in the sweep status
double currentX; // global sweep x

struct Seg {
    double x1, y1, x2, y2;
    int id;
    double yAtX(double x) const {
        if (abs(x2 - x1) < 1e-9) return y1;
        return y1 + (y2 - y1) * (x - x1) / (x2 - x1);
    }
};

struct CmpSeg {
    bool operator()(const Seg& a, const Seg& b) const {
        double ya = a.yAtX(currentX);
        double yb = b.yAtX(currentX);
        if (abs(ya - yb) > 1e-9) return ya < yb;
        return a.id < b.id;
    }
};

    Tip: In competitive programming you can often avoid a full Bentley-Ottmann implementation. For problems that only ask whether any intersection exists among n segments, a simplified sweep without the intersection-event queue suffices and runs in O(n log n).

    Bentley-Ottmann Algorithm

    The Bentley-Ottmann algorithm finds all k pairwise intersections of n line segments in O((n + k) log n) time and O(n + k) space. The key insight is that two segments can only begin intersecting when they become adjacent in the sweep-line status.

    Algorithm outline:

    1. Insert all left and right endpoints into the event queue.
    2. While the event queue is non-empty, extract the next event.
    3. For a left endpoint: insert the segment, check above/below neighbours.
    4. For a right endpoint: check if the neighbours above/below now intersect; remove the segment.
    5. For an intersection: report it, swap the two segments, and check new neighbours.
    // Simplified Bentley-Ottmann skeleton
#include <bits/stdc++.h>
using namespace std;

struct Event {
    double x;
    int type; // 0=left, 1=right, 2=intersection
    int s1, s2;
    bool operator>(const Event& o) const {
        if (abs(x - o.x) > 1e-9) return x > o.x;
        return type > o.type;
    }
};

vector<Seg> segs;
set<Seg, CmpSeg> status;
priority_queue<Event, vector<Event>, greater<Event>> pq;

        pq.push({lx, 0, s.id, -1});
        pq.push({rx, 1, s.id, -1});

            if (next(it) != status.end()) checkIntersection(*it, *next(it));

    Competition note: Bentley-Ottmann is rarely implemented from scratch in contests due to its complexity. However, understanding the paradigm is essential because many simpler sweep-line problems use the same ideas.

    Rectangle Union Area

    A classic sweep-line application: given n axis-aligned rectangles, find the total area of their union. This is solved in O(n log n) with a sweep line and a segment tree that tracks coverage.

    Approach:

    1. Create events for left and right edges of each rectangle.
    2. Coordinate-compress the y-values.
    3. Sweep from left to right. At each event, update a segment tree that tracks how many rectangles cover each y-interval.
    4. Between consecutive x-events, add covered_length × Δx to the answer.
    // Rectangle union area via sweep line + segment tree
#include <bits/stdc++.h>
using namespace std;

struct Event {
    int x, y1, y2, type; // type: +1 open, -1 close
    bool operator<(const Event& o) const { return x < o.x; }
};

int cnt[4 * 800]; // coverage count
double len[4 * 800]; // covered length
vector<int> ys; // compressed y-values

void update(int node, int lo, int hi, int l, int r, int val) {
    if (r < lo || hi < l) return;
    if (l <= lo && hi <= r) { cnt[node] += val; }
    else {
        int mid = (lo + hi) / 2;
        update(2*node, lo, mid, l, r, val);
        update(2*node+1, mid+1, hi, l, r, val);
    }
    if (cnt[node] > 0) len[node] = ys[hi+1] - ys[lo];
    else if (lo == hi) len[node] = 0;
    else len[node] = len[2*node] + len[2*node+1];
}

double rectangleUnion(vector<array<int,4>>& rects) {
    vector<Event> events;
    for (auto& r : rects) {
        ys.push_back(r[1]); ys.push_back(r[3]);
        events.push_back({r[0], r[1], r[3], +1});
        events.push_back({r[2], r[1], r[3], -1});
    }
    sort(ys.begin(), ys.end());
    ys.erase(unique(ys.begin(), ys.end()), ys.end());
    sort(events.begin(), events.end());
    int m = ys.size() - 1;
    double area = 0;
    for (int i = 0; i < (int)events.size(); i++) {
        if (i > 0) area += len[1] * (events[i].x - events[i-1].x);
        int ly = lower_bound(ys.begin(), ys.end(), events[i].y1) - ys.begin();
        int ry = lower_bound(ys.begin(), ys.end(), events[i].y2) - ys.begin() - 1;
        update(1, 0, m - 1, ly, ry, events[i].type);
    }
    return area;
}

    Complexity: O(n log n) for sorting events and y-compression, plus O(n log n) for segment-tree updates. Total: O(n log n).

    Tip: A common mistake is to forget that the segment tree here does not support lazy propagation in the usual sense — the cnt array acts as its own form of deferred state. Never push down the count; only propagate the computed length upward.

    Closest Pair via Sweep Line

    The closest pair problem — finding two points with minimum Euclidean distance — has a well-known O(n log n) divide-and-conquer solution. The sweep-line approach is an elegant alternative.

    Idea:

    1. Sort points by x-coordinate.
    2. Maintain a set of “active” points sorted by y, containing all points whose x-coordinate is within distance d of the current point (where d is the best distance found so far).
    3. For each new point, remove expired points from the set, then check only the O(1) amortised neighbours within y-range [y - d, y + d].
    // Closest pair of points via sweep line
#include <bits/stdc++.h>
using namespace std;

typedef pair<double,double> pt;

double closestPair(vector<pt>& pts) {
    int n = pts.size();
    sort(pts.begin(), pts.end());
    set<pair<double,int>> active; // {y, index}
    double best = 1e18;
    int left = 0;
    for (int i = 0; i < n; i++) {
        double d = sqrt(best);
        // remove points too far left
        while (left < i && pts[i].first - pts[left].first > d) {
            active.erase({pts[left].second, left});
            left++;
        }
        // check nearby points in y range
        auto it = active.lower_bound({pts[i].second - d, -1});
        for (; it != active.end() && it->first <= pts[i].second + d; ++it) {
            double dx = pts[i].first - pts[it->second].first;
            double dy = pts[i].second - it->first;
            best = min(best, dx*dx + dy*dy);
        }
        active.insert({pts[i].second, i});
    }
    return sqrt(best);
}

    Complexity: O(n log n). Each point is inserted and removed from the set exactly once. The inner loop checks at most O(1) amortised neighbours due to the packing argument (at most 6 points can lie within a d × 2d strip).

    Implementation & Competition Tips

    Practical advice for sweep-line problems in competitions:

    • Coordinate compression: When values are large but the number of distinct values is small, compress coordinates before building segment trees.
    • Epsilon handling: Floating-point sweep lines are error-prone. Prefer integer arithmetic or rational numbers where possible.
    • Event tie-breaking: Always define a clear tie-breaking rule when multiple events share the same x-coordinate. Typically, openings before closings, and closings before intersections.
    • Debugging: Print the status structure after each event during development. Verify that the invariant (sorted by y at current x) is maintained.
    • STL pitfalls: std::set with a dynamic comparator can behave unpredictably if the comparator changes while elements are in the set. Some contestants prefer a hand-rolled balanced BST or a policy-based tree.

    Common competitive programming problems solvable with sweep line:

    • Segment intersection detection / counting
    • Rectangle union area / perimeter
    • Closest pair of points
    • Visible segments from a viewpoint
    • Maximum overlap of intervals (1D sweep)

    Visible Segments Problem

    A classic sweep-line application: given n horizontal segments at different heights, determine which segments are visible when looking from above (or from y = +∞). A segment is visible at x-coordinate x if no other segment with a higher y-coordinate covers the same x.

    Approach:

    1. Create left-endpoint and right-endpoint events for each segment.
    2. Maintain an ordered set of active segments sorted by y-coordinate (highest first).
    3. At each event, update the active set. The topmost segment in the set at any x is the visible one.
    4. Track which segments ever appear as the topmost segment.
    // Visible segments from above (simplified version)
#include <bits/stdc++.h>
using namespace std;

struct Seg { int xl, xr, y, id; };

set<int> visibleSegments(vector<Seg>& segs) {
    vector<pair<int,pair<int,int>>> events; // {x, {type, idx}}
    for (int i = 0; i < (int)segs.size(); i++) {
        events.push_back({segs[i].xl, {0, i}}); // open
        events.push_back({segs[i].xr, {1, i}}); // close
    }
    sort(events.begin(), events.end());
    set<pair<int,int>> active; // {-y, id} for max-y ordering
    set<int> visible;
    for (auto& [x, ev] : events) {
        auto [type, idx] = ev;
        if (type == 0) {
            active.insert({-segs[idx].y, segs[idx].id});
        } else {
            active.erase({-segs[idx].y, segs[idx].id});
        }
        if (!active.empty()) {
            visible.insert(active.begin()->second);
        }
    }
    return visible;
}

    Complexity: O(n log n) for sorting and set operations. This pattern generalises to any “which object is on top at each x” query.

    Sweep Line Variations

    The sweep-line paradigm extends far beyond the examples above:

    • Angular sweep: Instead of sweeping a vertical line, rotate a ray around a fixed point. Useful for visibility problems, finding closest pair in angular order, and polar-angle sorting.
    • Radial sweep: Sweep a circle of increasing radius from a centre point. Used in some closest-pair and clustering algorithms.
    • Dual sweep: Sweep in the dual plane (where points become lines and vice versa). Powerful for problems involving line arrangements.
    • 3D sweep: Sweep a plane through 3D space. Reduces 3D problems to sequences of 2D problems, analogous to how 2D sweep reduces to 1D.

    In all cases, the core idea remains: process events in a sorted order, maintain a dynamic data structure encoding the current state, and extract answers from the structure. 

    // Angular sweep: count pairs of points within angle α
// Sort points by polar angle, use two pointers
int countPairsWithinAngle(vector<pair<double,double>>& pts,
                          double cx, double cy, double alpha) {
    int n = pts.size();
    vector<double> angles(n);
    for (int i = 0; i < n; i++)
        angles[i] = atan2(pts[i].second - cy, pts[i].first - cx);
    sort(angles.begin(), angles.end());
    // Duplicate for circular handling
    for (int i = 0; i < n; i++)
        angles.push_back(angles[i] + 2 * M_PI);
    int count = 0, j = 0;
    for (int i = 0; i < n; i++) {
        while (j < 2*n && angles[j] - angles[i] <= alpha) j++;
        count += (j - i - 1);
    }
    return count;
}

    Summary

    The sweep-line paradigm converts 2D problems into a sequence of 1D problems managed by a dynamic data structure. The Bentley-Ottmann algorithm extends this to handle segment intersections in output-sensitive time. Rectangle union area is a textbook application combining sweep with segment trees. The closest-pair sweep demonstrates that even classic problems can be solved elegantly with this technique. Angular and radial sweeps generalise the idea to rotational settings.

    Master the sweep line and you will have a powerful tool for a wide range of geometry problems in competitive programming.
