Polygon Area: The Shoelace Formula

July 2025 · 11 min read

Computing the area of a polygon given its vertices is one of the most common geometry tasks in competitive programming. The Shoelace Formula (also called the Surveyor's Formula or Gauss's area formula) does it in O(n) using cross products. It also reveals the polygon's orientation (CW vs CCW).

The Shoelace Formula

Given a simple polygon with vertices P0, P1, …, Pn-1 listed in order (either CW or CCW), the signed area is:

2A = ∑ (xi · yi+1 − xi+1 · yi)

where indices are taken modulo n (so Pn = P0). The name "shoelace" comes from the criss-cross pattern of multiplications that resembles lacing a shoe. 

// Basic shoelace — returns SIGNED area (positive if CCW)
double shoelace(vector<pair<double,double>> &poly) {
    double area = 0;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        area += poly[i].first * poly[j].second;
        area -= poly[j].first * poly[i].second;
    }
    return area / 2.0;
}

Complexity: O(n) time, O(1) extra space.

Derivation

The shoelace formula is derived by decomposing the polygon into triangles with a common vertex at the origin. For each edge PiPi+1, the triangle O–Pi–Pi+1 has signed area:

½(xi · yi+1 − xi+1 · yi)

This is just ½ cross(Pi, Pi+1) where the cross product is taken with respect to the origin. Summing over all edges, the interior triangles cancel out (due to opposite signs), leaving the polygon's area.

The green triangles contribute positive area and the red triangles contribute negative area. Their algebraic sum gives the polygon area regardless of where the origin is.

Signed vs. Unsigned Area

The shoelace formula returns a signed area:

  • Positive if vertices are listed in counter-clockwise (CCW) order.
  • Negative if vertices are listed in clockwise (CW) order.

This gives us a free orientation test:

// Determine polygon orientation
string orientation(vector<pair<double,double>> &poly) {
    double a = shoelace(poly);
    if (a > 0) return "CCW";
    if (a < 0) return "CW";
    return "Degenerate";
}

// Ensure CCW ordering
void makeCCW(vector<pair<double,double>> &poly) {
    if (shoelace(poly) < 0)
        reverse(poly.begin(), poly.end());
}

// Unsigned area
double polygonArea(vector<pair<double,double>> &poly) {
    return fabs(shoelace(poly));
}

Competition tip: Many problems expect output to a certain precision. For integer coordinates, 2A is always an integer (since each cross product term xiyj − xjyi is an integer). So you can output an integer and handle the "divided by 2" with a special format like "X.0" or "X.5". 

// Integer version: returns 2 * signed area (exact)
long long twiceShoelace(vector<pair<long long,long long>> &poly) {
    long long area = 0;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        area += poly[i].first * poly[j].second;
        area -= poly[j].first * poly[i].second;
    }
    return area;
}

// Print area with one decimal place
void printArea(long long twice) {
    twice = abs(twice);
    printf("%lld.%d\n", twice / 2, (int)(twice % 2) * 5);
}

Centroid of a Polygon

The centroid (center of mass, assuming uniform density) of a simple polygon can also be computed using cross products:

Cx = &frac{1}{6A} ∑ (xi + xi+1)(xiyi+1 − xi+1yi)

Cy = &frac{1}{6A} ∑ (yi + yi+1)(xiyi+1 − xi+1yi) 

pair<double,double> centroid(vector<pair<double,double>> &poly) {
    double cx = 0, cy = 0, area = 0;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        double cross_ij = poly[i].first * poly[j].second
                        - poly[j].first * poly[i].second;
        cx += (poly[i].first + poly[j].first) * cross_ij;
        cy += (poly[i].second + poly[j].second) * cross_ij;
        area += cross_ij;
    }
    area /= 2.0;
    cx /= (6.0 * area);
    cy /= (6.0 * area);
    return {cx, cy};
}

Note: The centroid of the vertices (simple average) is not the same as the centroid of the polygon region unless it's a triangle. The formula above computes the area-weighted centroid.

Complete Implementation

Here is a self-contained competitive-programming template that combines area, orientation, and centroid computation: 

#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
typedef pair<ll,ll> pll;

ll twiceSignedArea(vector<pll> &p) {
    ll area = 0;
    int n = p.size();
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        area += p[i].first * p[j].second;
        area -= p[j].first * p[i].second;
    }
    return area;
}

bool isCCW(vector<pll> &p) {
    return twiceSignedArea(p) > 0;
}

void makeCCW(vector<pll> &p) {
    if (!isCCW(p)) reverse(p.begin(), p.end());
}

// For problems that need exact "X.Y" output
void printArea(vector<pll> &p) {
    ll t = abs(twiceSignedArea(p));
    printf("%lld.%d\n", t / 2, (int)(t % 2) * 5);
}

int main() {
    int n;
    scanf("%d", &n);
    vector<pll> poly(n);
    for (int i = 0; i < n; i++)
        scanf("%lld %lld", &poly[i].first, &poly[i].second);
    printArea(poly);
    return 0;
}

This handles the CSES "Polygon Area" problem directly.
