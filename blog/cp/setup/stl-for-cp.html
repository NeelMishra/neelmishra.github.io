Order Statistics Tree

The order statistics tree from __gnu_pbds is arguably the most useful non-standard data structure in competitive programming. It provides an augmented balanced BST with two extra operations in O(log n):

  • find_by_order(k) — returns an iterator to the k-th element (0-indexed)
  • order_of_key(x) — returns the number of elements strictly less than x

Without this, you'd need a segment tree or BIT with coordinate compression—much more code for the same result.

#include <ext/pb_ds/assoc_container.hpp> #include <ext/pb_ds/tree_policy.hpp> using namespace __gnu_pbds; typedef tree<int, null_type, less<int>, rb_tree_tag, tree_order_statistics_node_update> ordered_set; int main() { ordered_set os; os.insert(1); os.insert(5); os.insert(3); os.insert(8); // find_by_order(k): iterator to k-th element (0-indexed) cout << *os.find_by_order(2) << '\n'; // prints 5 (3rd smallest) // order_of_key(x): count of elements < x cout << os.order_of_key(4) << '\n'; // prints 2 (1,3 are < 4) return 0; }

Handling Duplicates

The ordered set doesn't allow duplicates (like std::set). To handle duplicates, use pair<int, int> where the second element is a unique index:

typedef tree<pair<int,int>, null_type, less<pair<int,int>>, rb_tree_tag, tree_order_statistics_node_update> ordered_multiset; ordered_multiset oms; int uid = 0; // Insert value v: oms.insert({v, uid++}); // unique pair ensures no collision // Count of elements < v: oms.order_of_key({v, 0}); // Count of elements <= v: oms.order_of_key({v + 1, 0});
Classic use case: Count inversions in an array. Insert elements left to right; for each element a[i], inversions = i − order_of_key(a[i]). Total time: O(n log n).

Policy-Based Sets & Maps

The pb_ds library offers more than just order statistics. You can create associative containers with different underlying data structures:

// Red-black tree (default, balanced BST) typedef tree<int, int, less<int>, rb_tree_tag, tree_order_statistics_node_update> rb_map; // Splay tree (good for access patterns with locality) typedef tree<int, int, less<int>, splay_tree_tag, tree_order_statistics_node_update> splay_map; // Hash table with chaining #include <ext/pb_ds/hash_policy.hpp> typedef cc_hash_table<int, int> cc_map; // Hash table with probing typedef gp_hash_table<int, int> gp_map;

The gp_hash_table is particularly useful—it's often 2–3× faster than unordered_map for integer keys because it uses open addressing with a simpler hash function and less overhead.

Portability warning: All pb_ds structures are GCC-specific extensions. They work on Codeforces (GCC) and AtCoder but may not work on judges using MSVC or Clang. Always verify on your target judge.

Rope Data Structure

The __gnu_cxx::rope is an immutable sequence with O(log n) insert, erase, and concatenation. Think of it as a balanced BST of characters/elements. It's useful for problems involving heavy string manipulation.

#include <ext/rope> using namespace __gnu_cxx; int main() { rope<char> r = "hello"; r.insert(5, " world"); // "hello world" r.erase(0, 5); // " world" r = r.substr(1, 5); // "world" r.append("!"); // "world!" cout << r << '\n'; // Also works with other types: rope<int> ir; ir.push_back(42); ir.insert(0, 17); // insert at position 0 return 0; }

Custom Hash Maps

The default unordered_map has a known vulnerability: adversarial input can force O(n) per operation by exploiting the default hash function. This is a real attack on Codeforces (called "blowing up unordered_map"). The fix: use a custom hash.

struct custom_hash { static uint64_t splitmix64(uint64_t x) { x += 0x9e3779b97f4a7c15; x = (x ^ (x >> 30)) * 0xbf58476d1ce4e5b9; x = (x ^ (x >> 27)) * 0x94d049bb133111eb; return x ^ (x >> 31); } size_t operator()(uint64_t x) const { static const uint64_t FIXED_RANDOM = chrono::steady_clock::now() .time_since_epoch().count(); return splitmix64(x + FIXED_RANDOM); } }; // Usage: hack-proof unordered_map unordered_map<long long, int, custom_hash> mp;

The key insight: by XOR-ing with a random seed based on the current time, the hash function becomes unpredictable to adversaries. The splitmix64 function ensures good distribution.

Hashing Pairs and Tuples

struct pair_hash { custom_hash h; size_t operator()(pair<int,int> const& p) const { auto h1 = h(p.first); auto h2 = h(p.second); return h1 ^ (h2 << 32); } }; unordered_map<pair<int,int>, int, pair_hash> mp2;
Codeforces hack warning: If you use unordered_map with the default hash, your solution can be hacked during the open hacking phase. Always use custom_hash or switch to map (O(log n) but safe). Several high-rated Codeforces rounds have seen mass hacks exploiting this.

Bitset Tricks

std::bitset is criminally underused in competitive programming. It provides bit-parallel operations that can speed up certain algorithms by a factor of 64 (the word size).

const int N = 100000; bitset<N> a, b; // Set operations in O(N/64) a |= b; // union a &= b; // intersection a ^= b; // symmetric difference a <<= k; // shift left by k bits // Count in O(N/64) using popcount int cnt = a.count(); // Find first/any set bit int first = a._Find_first(); // GCC extension int next = a._Find_next(first); // next set bit after 'first'

Bitset DP Optimization

The classic application is subset sum with bitset. Instead of O(n·S) DP, use bitset shift to achieve O(n·S/64):

// Subset sum: can we make sum S from array a[]? const int MAXS = 1000000; bitset<MAXS + 1> dp; dp[0] = 1; // base case: sum 0 is achievable for (int i = 0; i < n; i++) dp |= (dp << a[i]); // O(MAXS/64) per element // dp[s] == 1 iff sum s is achievable if (dp[target]) cout << "YES" << '\n';
Other bitset applications: Transitive closure of a graph (Floyd-Warshall with bitset rows), reachability queries, pattern matching with mismatches, and Gaussian elimination over GF(2).

Practical Patterns

Here are battle-tested patterns combining these structures for common CP scenarios.

Dynamic Rank Queries

Given online queries: insert x, delete x, find k-th smallest, count elements < x. Use the ordered_set directly:

ordered_multiset oms; int uid = 0; map<int, vector<int>> ids; // track UIDs per value void insert(int x) { int id = uid++; oms.insert({x, id}); ids[x].push_back(id); } void erase(int x) { int id = ids[x].back(); ids[x].pop_back(); oms.erase({x, id}); } int kth(int k) { return oms.find_by_order(k)->first; } int countLess(int x) { return oms.order_of_key({x, 0}); }

Fast Frequency Map

For problems requiring many lookups/insertions with integer keys:

// Prefer gp_hash_table over unordered_map for speed gp_hash_table<int, int, custom_hash> freq; for (int i = 0; i < n; i++) { auto it = freq.find(a[i]); if (it != freq.end()) it->second++; else freq.insert({a[i], 1}); } // Note: gp_hash_table doesn't support operator[] // Use find + insert pattern instead
