Multiplicative Order & Primitive Roots

CP Math Series — Post 17Number Theory

The multiplicative group modulo n underpins much of competitive-programming number theory. Two concepts sit at its core: the multiplicative order of an element and primitive roots—generators of the entire group. Mastering these unlocks efficient solutions to problems involving cyclic structures, discrete logarithms, and index calculus.

Multiplicative Order

Let n ≥ 1 and gcd(a, n) = 1. The multiplicative order of a modulo n, written ordn(a), is the smallest positive integer k such that

ak ≡ 1 (mod n)

By Euler’s theorem, aφ(n) ≡ 1 (mod n), so the order always exists and divides φ(n). This divisibility property is the key to computing the order efficiently.

Key Properties

  • ordn(a) divides φ(n).
  • aj ≡ 1 (mod n) if and only if ordn(a) divides j.
  • The elements a0, a1, …, aord−1 are all distinct modulo n.
  • ordn(at) = ordn(a) / gcd(t, ordn(a)).

Computing the Order

A naïve approach iterates k = 1, 2, … until ak ≡ 1, costing O(φ(n)) multiplications. We can do much better by exploiting the divisibility property: ordn(a) divides φ(n), so we only need to check divisors of φ(n).

Efficient Algorithm

  1. Compute φ(n) and factorise it: φ(n) = p1e1 · … · pkek.
  2. Start with result = φ(n).
  3. For each prime factor pi, while result is divisible by pi and aresult/pi ≡ 1, divide result by pi.
  4. Return result.
// Compute multiplicative order of a modulo n long long mult_order(long long a, long long n) { long long phi = euler_phi(n); auto factors = factorise(phi); // returns vector<pair<ll,int>> long long ord = phi; for (auto& [p, e] : factors) { while (ord % p == 0) { if (power_mod(a, ord / p, n) == 1) ord /= p; else break; } } return ord; }

Complexity: The outer loop runs once per distinct prime factor of φ(n), and the inner while runs at most ei times for factor pi. Each iteration does one modular exponentiation in O(log φ(n)). Total: O(k · log2 φ(n)) where k is the number of distinct prime factors—very fast in practice.

Tip: Precompute a prime factorisation function using your sieve. For contest use, factorising numbers up to 1012 via trial division up to 106 is fast enough.

Primitive Roots

An integer g with gcd(g, n) = 1 is a primitive root modulo n if ordn(g) = φ(n). Equivalently, g generates the entire multiplicative group (ℤ/nℤ)×.

When a primitive root g exists, every element a coprime to n can be written as a ≡ gk for a unique 0 ≤ k < φ(n). This exponent k is the discrete logarithm of a to base g.

Has Primitive Root

Group is cyclic. Every element is a power of one generator g. Structure is isomorphic to ℤ/φ(n)ℤ.

No Primitive Root

Group is not cyclic. No single generator exists. The group decomposes as a direct product of cyclic groups of smaller order.

Existence Proof for Primes

Theorem. For every prime p, there exists a primitive root modulo p.

Proof.

We need to show that the multiplicative group (ℤ/pℤ)× is cyclic, i.e., there exists g with ordp(g) = p − 1.

Step 1 (Polynomial root bound). Over ℤ/pℤ, a polynomial of degree d has at most d roots. This is because ℤ/pℤ is a field, so the factor theorem applies.

Step 2 (Counting elements of each order). For each divisor d of p − 1, define ψ(d) = number of elements of order exactly d in (ℤ/pℤ)×. Since every element has order dividing p − 1, we have:

∑d | (p−1) ψ(d) = p − 1

Step 3 (Upper bound via roots). If ψ(d) > 0, pick an element a of order d. Then a, a2, …, ad are d distinct roots of xd − 1. By Step 1 these are all roots. Among them, those with order exactly d are precisely aj with gcd(j, d) = 1. There are φ(d) such values. So ψ(d) ≤ φ(d), with equality when ψ(d) > 0.

Step 4 (Conclusion). Using the identity ∑d | n φ(d) = n, we get:

p − 1 = ∑ ψ(d) ≤ ∑ φ(d) = p − 1

Equality forces ψ(d) = φ(d) for every divisor d. In particular, ψ(p − 1) = φ(p − 1) ≥ 1, so a primitive root exists. ▮

Corollary: There are exactly φ(p − 1) primitive roots modulo any prime p. For p = 109 + 7, this is roughly 2 × 108, so a random guess has ≈ 20% chance of hitting one.

Finding Primitive Roots

Given the proof above, the algorithm is straightforward: test candidates g = 2, 3, … and verify that g has order p − 1. The verification uses the fact that ordp(g) = p − 1 if and only if g(p−1)/q ¬≡ 1 (mod p) for every prime factor q of p − 1.

Algorithm

  1. Factorise p − 1 to get its distinct prime factors q1, …, qk.
  2. For g = 2, 3, …: check g(p−1)/qi ¬≡ 1 (mod p) for all i.
  3. Return the first g that passes all checks.
// Find the smallest primitive root modulo prime p long long primitive_root(long long p) { long long phi = p - 1; auto factors = factorise(phi); // distinct prime factors for (long long g = 2; g < p; g++) { bool ok = true; for (auto& [q, _] : factors) { if (power_mod(g, phi / q, p) == 1) { ok = false; break; } } if (ok) return g; } return -1; // should never reach here for prime p }

Complexity analysis: The smallest primitive root of a prime p is conjectured (and conditionally proved under GRH) to be O(log6 p). In practice, it is almost always very small—usually below 10 for primes used in CP. Each candidate test costs O(k log p) where k is the number of distinct prime factors of p − 1. Overall the algorithm is extremely fast.

Pitfall: Don’t forget to check all prime factors of p − 1, not just a few. Missing even one factor can cause a non-generator to be accepted. For p = 109 + 7, we have p − 1 = 2 × 500000003, so you need to check both prime factors.

Helper: Modular Exponentiation and Factorisation

long long power_mod(long long base, long long exp, long long mod) { long long result = 1; base %= mod; while (exp > 0) { if (exp & 1) result = (__int128)result * base % mod; base = (__int128)base * base % mod; exp >>= 1; } return result; } vector<pair<long long, int>> factorise(long long n) { vector<pair<long long, int>> res; for (long long d = 2; d * d <= n; d++) { if (n % d == 0) { int cnt = 0; while (n % d == 0) { n /= d; cnt++; } res.push_back({d, cnt}); } } if (n > 1) res.push_back({n, 1}); return res; }

Which Moduli Have Primitive Roots

Not every modulus admits a primitive root. The complete characterisation is:

Theorem: A primitive root modulo n exists if and only if n is one of: 1, 2, 4, pk, or 2pk, where p is an odd prime and k ≥ 1.

Here is why other moduli fail:

  • n = 2k for k ≥ 3: The group (ℤ/2kℤ)× is isomorphic to ℤ/2 × ℤ/2k−2, which is not cyclic.
  • n = pa qb with distinct odd primes p, q: By CRT, the group decomposes as a product, and since both factors have even order, the product cannot be cyclic.

Primitive Root Exists

  • n = 1, 2, 4
  • n = pk (odd prime p)
  • n = 2pk (odd prime p)

No Primitive Root

  • n = 2k for k ≥ 3
  • n with 2+ distinct odd prime factors
  • n = 4m for odd m > 1

CP Applications

Application 1: NTT-Friendly Primes

The Number Theoretic Transform (NTT) requires a primitive root of the working prime. Common NTT primes like 998244353 = 223 × 7 × 17 + 1 have primitive root g = 3. Knowing how to verify this is essential when writing or debugging NTT code.

Application 2: Discrete Logarithm Setup

Many problems reduce to solving gx ≡ a (mod p). Finding a primitive root gives us the base g for the BSGS algorithm (covered in the next post). Without a generator, the discrete log problem becomes harder to set up correctly.

Application 3: Counting Elements of a Given Order

Once you know g, every element of order d (where d | p−1) is g(p−1)j/d with gcd(j, d) = 1. The count is φ(d). This is useful in problems asking “how many x satisfy xk ≡ 1?”—the answer is gcd(k, p−1).

Template: Full Primitive Root Finder

#include <bits/stdc++.h> using namespace std; typedef long long ll; ll power_mod(ll b, ll e, ll m) { ll r = 1; b %= m; while (e > 0) { if (e & 1) r = (__int128)r * b % m; b = (__int128)b * b % m; e >>= 1; } return r; } ll primitive_root(ll p) { ll phi = p - 1, n = phi; vector<ll> pf; for (ll d = 2; d * d <= n; d++) { if (n % d == 0) { pf.push_back(d); while (n % d == 0) n /= d; } } if (n > 1) pf.push_back(n); for (ll g = 2; g < p; g++) { bool ok = true; for (ll q : pf) if (power_mod(g, phi / q, p) == 1) { ok = false; break; } if (ok) return g; } return -1; } int main() { ll p = 1000000007; cout << primitive_root(p) << "\n"; // outputs 5 }
Contest tip: For well-known primes, memorise the primitive root: g = 5 for 109+7, g = 3 for 998244353. This saves precious contest time.
