Multiplicative Functions — φ, σ, τ, μ & Dirichlet Convolution

CP Math SeriesNumber Theory

Multiplicative functions are the backbone of analytic number theory and appear constantly in competitive programming — from counting divisors to Möbius inversion. This post builds the theory from scratch, proves key properties, and gives O(n log log n) sieve code ready for contests.

Multiplicative Function Definition

An arithmetic function f : ℤ⁺ → ℂ is called multiplicative if

  1. f(1) = 1, and
  2. f(mn) = f(m)·f(n) whenever gcd(m, n) = 1.

It is completely multiplicative if condition 2 holds for all m, n (not just coprime pairs). The distinction matters: Euler's φ is multiplicative but not completely multiplicative.

Why it matters: For a multiplicative f, knowing f at prime powers f(pk) determines f everywhere. This is what makes sieve computation possible.

Common Multiplicative Functions

Here is a reference grid of the most important multiplicative functions you will encounter in competitive programming.

φ(n) — Euler's Totient

Counts integers in [1, n] coprime to n.

Formula: φ(pk) = pk − pk−1

Product: φ(n) = n · ∏p|n(1 − 1/p)

τ(n) — Number of Divisors

τ(n) = |{d : d | n}|

Formula: τ(pk) = k + 1

Product: τ(n) = ∏(aᵢ + 1)

σ(n) — Sum of Divisors

σ(n) = Σd|n d

Formula: σ(pk) = (pk+1 − 1)/(p − 1)

Generalizes to σk(n) = Σ dk.

μ(n) — Möbius Function

μ(1)=1; μ(n)=0 if p²|n; else μ(n)=(−1)ω(n)

At primes: μ(p) = −1

Key identity: Σd|n μ(d) = [n = 1]

λ(n) — Liouville Function

λ(n) = (−1)Ω(n) where Ω counts prime factors with multiplicity. Completely multiplicative.

Id(n), 1(n), ε(n)

Id(n) = n (identity), 1(n) = 1 (constant), ε(n) = [n=1] (Dirichlet identity). All completely multiplicative.

Proof That φ Is Multiplicative

We prove: if gcd(m, n) = 1 then φ(mn) = φ(m)·φ(n).

Proof via CRT bijection

Consider the ring ℤ/mnℤ. By the Chinese Remainder Theorem (since gcd(m,n)=1) there is a ring isomorphism

ℤ/mnℤ  ≅  ℤ/mℤ × ℤ/nℤ

An element a is a unit in ℤ/mnℤ iff its image (a mod m, a mod n) is a unit in the product ring, which happens iff gcd(a, m) = 1 and gcd(a, n) = 1. Counting units on each side:

φ(mn) = |(ℤ/mnℤ)×| = |(ℤ/mℤ)×| · |(ℤ/nℤ)×| = φ(m) · φ(n)   □

Why CRT? The same bijection argument proves that any function defined by counting solutions modulo n tends to be multiplicative — this is the template for proving τ, σ, etc.

Deriving the product formula

Write n = p₁a₁ · … · pₖaₖ. By multiplicativity:

φ(n) = ∏ φ(pᵢaᵢ) = ∏ (pᵢaᵢ − pᵢaᵢ−1) = n · ∏(1 − 1/pᵢ)

Dirichlet Convolution

The Dirichlet convolution of two arithmetic functions f and g is

(f ∗ g)(n) = Σd|n f(d) · g(n/d)

This operation is commutative, associative, and its identity element is ε(n) = [n=1].

Key identities

Convolution Result
1 ∗ 1τ  (number of divisors)
Id ∗ 1σ  (sum of divisors)
φ ∗ 1Id  (Gauss's identity: Σd|n φ(d) = n)
μ ∗ 1ε  (Möbius is the inverse of 1)
μ ∗ Idφ  (Euler from Möbius)

Closure under convolution

Theorem: If f and g are both multiplicative, so is f ∗ g.

This is why the table above works — convolving multiplicative functions yields another multiplicative function.

Pitfall: Dirichlet convolution is not the same as pointwise multiplication. f(n)·g(n) is also multiplicative if f and g are, but it's a different operation with different identities.

Möbius Inversion Preview

The Möbius function μ is the Dirichlet inverse of the constant function 1. This gives us the Möbius inversion formula:

If g(n) = Σd|n f(d), then f(n) = Σd|n μ(d) · g(n/d).

We will prove this and apply it in the dedicated Möbius function post, but the key insight is: once you express a summation over divisors as a Dirichlet convolution, you can "undo" it by convolving with μ.

Sieve Computation

For CP, we typically need all values f(1), f(2), …, f(N) at once. A multiplicative sieve computes them in O(N log log N) by iterating over primes and updating prime-power entries.

Linear sieve for φ, μ, τ simultaneously

const int MAXN = 1e7 + 5; int phi[MAXN], mu[MAXN], tau[MAXN]; int sp[MAXN]; // smallest prime factor vector<int> primes; void linear_sieve(int N) { phi[1] = mu[1] = tau[1] = 1; for (int i = 2; i <= N; ++i) { if (!sp[i]) { // i is prime sp[i] = i; primes.push_back(i); phi[i] = i - 1; mu[i] = -1; tau[i] = 2; } for (int p : primes) { if (p > sp[i] || (long long)i * p > N) break; sp[i * p] = p; if (i % p == 0) { // p^2 | i*p ⟹ handle carefully phi[i * p] = phi[i] * p; mu[i * p] = 0; // For tau: find exponent of p in i int tmp = i, e = 0; while (tmp % p == 0) { tmp /= p; ++e; } tau[i * p] = tau[i] / (e + 1) * (e + 2); break; } phi[i * p] = phi[i] * (p - 1); mu[i * p] = -mu[i]; tau[i * p] = tau[i] * 2; } } }

Complexity

The linear sieve visits each composite exactly once, giving O(N) time — better than the O(N log log N) of a traditional Eratosthenes sieve. Memory is O(N) for the arrays.

Pitfall: The τ update when p | i requires dividing out the old exponent contribution and multiplying the new one. Getting this wrong is the #1 bug in sieve-based τ computation.

Simpler O(N log N) sieve (divisor-style)

// O(N log N) sieve for any additive-over-divisors function void divisor_sieve(int N) { fill(tau, tau + N + 1, 0); for (int d = 1; d <= N; ++d) for (int m = d; m <= N; m += d) tau[m]++; }

CP Patterns

Pattern 1 — Sieve + point query

Precompute all values up to N, then answer Q queries in O(1) each. Works for N ≤ 107 easily.

Pattern 2 — Summation via Dirichlet hyperbola method

To compute Σn≤N (f∗g)(n) in sub-linear time, split the sum at √N. This reduces O(N) to O(√N) for many standard functions.

Pattern 3 — GCD sums

Σi,j≤N f(gcd(i,j)) can be rewritten using Möbius inversion or the identity φ ∗ 1 = Id. This pattern appears in many CF/AtCoder problems.

Contest wisdom: If a problem asks you to sum something over gcd or divisors, think "Dirichlet convolution + Möbius inversion" first. It's almost always the intended approach.

Complexity cheat-sheet

Precomputation

Linear sieve: O(N)
Divisor sieve: O(N log N)
Eratosthenes: O(N log log N)

Query

Point query: O(1)
Prefix sum: O(1) after O(N) precomp
Hyperbola method: O(√N) per query
