Quadratic Residues & Tonelli-Shanks

CP Math Series — Post 19Number Theory

Can you take a square root modulo a prime? This deceptively simple question leads to rich theory: the Legendre symbol, Euler’s criterion, the law of quadratic reciprocity, and efficient algorithms like Tonelli-Shanks. These tools appear frequently in CP problems involving quadratic equations, counting solutions, and number-theoretic constructions.

Quadratic Residues

Let p be an odd prime. An integer a ≠ 0 (mod p) is a quadratic residue (QR) modulo p if there exists x such that x2 ≡ a (mod p). Otherwise, a is a quadratic non-residue (QNR).

Basic Counting

Among the p−1 non-zero residues modulo p, exactly (p−1)/2 are QRs and (p−1)/2 are QNRs. This is because the squaring map x ↦ x2 is exactly 2-to-1 on (ℤ/pℤ)× (since x2 = (−x)2 and x ≠ −x for p odd).

Example: p = 13

The squares mod 13 are: 12=1, 22=4, 32=9, 42=3, 52=12, 62=10. So QR = {1, 3, 4, 9, 10, 12} and QNR = {2, 5, 6, 7, 8, 11}. Each set has exactly 6 = (13−1)/2 elements.

The Legendre Symbol

The Legendre symbol (a/p) encodes the quadratic residuosity of a modulo an odd prime p:

  • (a/p) = 0 if p | a
  • (a/p) = 1 if a is a QR mod p
  • (a/p) = −1 if a is a QNR mod p

Multiplicativity

The Legendre symbol is completely multiplicative: (ab/p) = (a/p)(b/p). This means:

  • QR × QR = QR
  • QR × QNR = QNR
  • QNR × QNR = QR

This mirrors the sign rule for multiplication, and is one of many analogies between the Legendre symbol and the ±1 character.

Euler’s Criterion

Theorem (Euler’s Criterion). For odd prime p and gcd(a, p) = 1:

(a/p) ≡ a(p−1)/2 (mod p)

Proof.

Case 1: a is a QR. Then a = x2 for some x. So a(p−1)/2 = xp−1 ≡ 1 (mod p) by Fermat’s little theorem. This matches (a/p) = 1.

Case 2: a is a QNR. Consider the polynomial f(t) = t(p−1)/2 − 1 over ℤ/pℤ. It has degree (p−1)/2, so at most (p−1)/2 roots. By Case 1, all (p−1)/2 quadratic residues are roots. Thus the QRs account for all roots, and a(p−1)/2 ≠ 1.

But ap−1 ≡ 1, so (a(p−1)/2)2 ≡ 1, meaning a(p−1)/2 ≡ ±1. Since it’s not 1, it must be −1 ≡ p−1 (mod p). This matches (a/p) = −1. ▮

// Compute Legendre symbol (a/p) using Euler's criterion int legendre(long long a, long long p) { a %= p; if (a < 0) a += p; if (a == 0) return 0; ll val = power_mod(a, (p - 1) / 2, p); return val == 1 ? 1 : -1; }
Tip: In CP, Euler’s criterion is the fastest way to test whether a is a QR. It’s a single modular exponentiation—O(log p). No need to enumerate squares.

Quadratic Reciprocity

The law of quadratic reciprocity is one of the crown jewels of number theory, first proved by Gauss. For distinct odd primes p and q:

(p/q) · (q/p) = (−1)(p−1)(q−1)/4

In plain terms: (p/q) = (q/p) unless both p ≡ q ≡ 3 (mod 4), in which case they differ in sign.

Supplement Laws

  • (−1/p) = (−1)(p−1)/2: −1 is a QR iff p ≡ 1 (mod 4).
  • (2/p) = (−1)(p2−1)/8: 2 is a QR iff p ≡ ±1 (mod 8).

−1 is a QR mod p

p ≡ 1 (mod 4)

Examples: 5, 13, 17, 29, 37, 41

These primes split in ℤ[i]

−1 is a QNR mod p

p ≡ 3 (mod 4)

Examples: 3, 7, 11, 19, 23, 31

These primes stay inert in ℤ[i]

While quadratic reciprocity is rarely needed directly in CP, it’s the theoretical backbone that makes efficient Jacobi/Legendre symbol computation possible for large numbers. It also appears in olympiad number theory problems.

Tonelli-Shanks Algorithm

Given that a is a QR modulo an odd prime p, the Tonelli-Shanks algorithm finds r such that r2 ≡ a (mod p). The other root is p − r.

Special Case: p ≡ 3 (mod 4)

When p ≡ 3 (mod 4), the square root is simply r = a(p+1)/4 mod p. This works because r2 = a(p+1)/2 = a · a(p−1)/2 = a · 1 = a.

General Case: Tonelli-Shanks

Write p−1 = 2s · q with q odd. The algorithm maintains an invariant and reduces s step by step:

  1. Find a QNR z modulo p (random search, expected 2 tries).
  2. Initialise: M = s, c = zq, t = aq, r = a(q+1)/2.
  3. Loop: if t = 1, return r. Otherwise find the smallest i such that t2i ≡ 1. Set b = c2(M−i−1), then update M=i, c=b2, t=t·b2, r=r·b.

Complexity: O(s · log p) worst case, where s = v2(p−1). For p up to 1018, s ≤ 60, so this is very fast.

Full Implementation

typedef long long ll; ll power_mod(ll b, ll e, ll m) { ll r = 1; b %= m; while (e > 0) { if (e & 1) r = (__int128)r * b % m; b = (__int128)b * b % m; e >>= 1; } return r; } // Tonelli-Shanks: find r with r^2 ≡ a (mod p), p odd prime // Returns -1 if a is not a QR ll tonelli_shanks(ll a, ll p) { a %= p; if (a < 0) a += p; if (a == 0) return 0; if (power_mod(a, (p - 1) / 2, p) != 1) return -1; // not a QR if (p % 4 == 3) return power_mod(a, (p + 1) / 4, p); // Factor out powers of 2: p-1 = 2^s * q ll s = 0, q = p - 1; while (q % 2 == 0) { s++; q /= 2; } // Find a QNR z ll z = 2; while (power_mod(z, (p - 1) / 2, p) != p - 1) z++; ll M = s; ll c = power_mod(z, q, p); ll t = power_mod(a, q, p); ll r = power_mod(a, (q + 1) / 2, p); while (true) { if (t == 1) return r; // Find smallest i with t^{2^i} = 1 ll i = 0, tmp = t; while (tmp != 1) { tmp = (__int128)tmp * tmp % p; i++; } ll b = c; for (ll j = 0; j < M - i - 1; j++) b = (__int128)b * b % p; M = i; c = (__int128)b * b % p; t = (__int128)t * c % p; r = (__int128)r * b % p; } }
Pitfall: Don’t forget to handle a = 0 separately (the square root of 0 is 0). Also, when p = 2, the answer is just a mod 2. The Tonelli-Shanks algorithm is designed for odd primes only.

Cipolla’s Algorithm

An alternative to Tonelli-Shanks is Cipolla’s algorithm, which works by extending to ℤ/pℤ[√d] (an “imaginary” extension field) where d is a chosen non-residue.

How It Works

  1. Pick random t until t2 − a is a QNR. Call this d = t2 − a.
  2. Work in the ring ℤ/pℤ[x] / (x2 − d). Elements are pairs (u, v) representing u + v√d.
  3. Compute (t + √d)(p+1)/2 in this ring. The result will have v = 0, and u is the desired square root.
When to use Cipolla vs Tonelli-Shanks: Cipolla is simpler to implement (just complex-number-style multiplication with a custom “i2” value) and has the same O(log p) complexity. Some competitive programmers prefer it for its elegance. Tonelli-Shanks is slightly more cache-friendly and commonly found in reference libraries.
// Cipolla's algorithm: find r with r^2 ≡ a (mod p) ll n1_cipolla_w2; // global: w^2 mod p (the non-residue) ll n1_cipolla_p; pair<ll, ll> mul_ext(pair<ll,ll> a, pair<ll,ll> b) { ll p = n1_cipolla_p, w2 = n1_cipolla_w2; return { ((__int128)a.first * b.first + (__int128)a.second * b.second % p * w2) % p, ((__int128)a.first * b.second + (__int128)a.second * b.first) % p }; } ll cipolla(ll a, ll p) { a %= p; if (a < 0) a += p; if (a == 0) return 0; if (power_mod(a, (p - 1) / 2, p) != 1) return -1; n1_cipolla_p = p; ll t; do { t = rand() % p; n1_cipolla_w2 = ((__int128)t * t - a % p + p) % p; } while (power_mod(n1_cipolla_w2, (p - 1) / 2, p) != p - 1); pair<ll,ll> res = {1, 0}, base = {t, 1}; ll exp = (p + 1) / 2; while (exp > 0) { if (exp & 1) res = mul_ext(res, base); base = mul_ext(base, base); exp >>= 1; } return res.first; // the imaginary part will be 0 }

CP Applications

Application 1: Modular Square Root Queries

Given q queries, each asking for √a mod p: use Tonelli-Shanks (or Cipolla) for each query. Total time: O(q log p).

Application 2: Counting Solutions of x2 + y2 ≡ n

For each x, check if n − x2 is a QR using Euler’s criterion. Each QR contributes 2 solutions (except when the root is 0). This gives an O(p log p) approach. For better results, use character sum techniques.

Application 3: Constructing Quadratic Extensions

When a problem involves arithmetic in ℤ/pℤ[√d], you need to verify that d is indeed a QNR (so the extension is a proper field). Euler’s criterion does this in O(log p).

Application 4: Cryptographic Context

The quadratic residuosity problem is the basis of the Goldwasser-Micali cryptosystem. Given a composite n = pq, deciding whether a is a QR mod n (without knowing the factorisation) is believed to be hard. This is distinct from the CP setting where p is known and prime—there, Euler’s criterion solves it instantly.

Contest tip: Most CP problems use primes p ≡ 3 (mod 4) (e.g., 109 + 7 ≡ 3 mod 4). In this case, the square root is just power_mod(a, (p+1)/4, p)—no need for Tonelli-Shanks at all!
Pitfall: When a problem asks for “the” square root, remember there are two roots: r and p − r. Always check which one the problem expects (e.g., smallest positive, or both).
