The Möbius Function — Inversion, Square-Free & Coprime Counting

CP Math SeriesNumber Theory

The Möbius function μ is the Swiss army knife of number-theoretic summation. Whenever a problem asks you to count integers satisfying a divisibility condition — square-free numbers, coprime pairs, GCD sums — Möbius inversion is likely the intended tool. We prove every identity from scratch and build contest-ready C++ code.

μ(n) Definition

The Möbius function is defined as:

  • μ(1) = 1
  • μ(n) = (−1)k if n = p₁ p₂ … pₖ is a product of k distinct primes (i.e. n is square-free)
  • μ(n) = 0 if p² | n for some prime p

Equivalently, at prime powers: μ(p) = −1 and μ(pk) = 0 for k ≥ 2. Since μ is multiplicative, this determines it everywhere.

Computing μ via sieve

const int MAXN = 1e7 + 5; int mu[MAXN], sp[MAXN]; vector<int> primes; void sieve_mu(int N) { mu[1] = 1; for (int i = 2; i <= N; ++i) { if (!sp[i]) { sp[i] = i; primes.push_back(i); mu[i] = -1; } for (int p : primes) { if (p > sp[i] || (long long)i * p > N) break; sp[i * p] = p; if (i % p == 0) { mu[i * p] = 0; break; } mu[i * p] = -mu[i]; } } }

The Fundamental Identity

The single most important property of μ is:

Σd|n μ(d) = [n = 1]

where [P] is the Iverson bracket: 1 if P is true, 0 otherwise.

Proof

For n = 1, the only divisor is 1, and μ(1) = 1 = [1=1]. ✓

For n > 1, write n = p₁a₁ … pₖaₖ. A divisor d contributes μ(d) ≠ 0 only if d is square-free, so d must be a product of a subset S of {p₁, …, pₖ}. For such d, μ(d) = (−1)|S|. Summing over all subsets:

Σd|n μ(d) = Σj=0k C(k,j)·(−1)j = (1 − 1)k = 0

by the binomial theorem (k ≥ 1). □

Interpretation: This identity says μ ∗ 1 = ε in the Dirichlet convolution algebra. In other words, μ is the Dirichlet inverse of the constant function 1.

Möbius Inversion Formula

Theorem (Möbius Inversion): Let f, g be arithmetic functions. If

g(n) = Σd|n f(d)

then

f(n) = Σd|n μ(d) · g(n/d)

and conversely.

Proof

In Dirichlet convolution notation, g = f ∗ 1. Since μ ∗ 1 = ε (the identity proved above), convolving both sides with μ:

μ ∗ g = μ ∗ (f ∗ 1) = f ∗ (μ ∗ 1) = f ∗ ε = f

using associativity of Dirichlet convolution. □

Direct verification

Alternatively, substitute and swap summation:

Σd|n μ(d)·g(n/d) = Σd|n μ(d) Σe|(n/d) f(e) = Σe|n f(e) Σd|(n/e) μ(d) = f(n)

because the inner sum equals [n/e = 1] = [e = n]. □

Summatory version: If G(x) = Σn≤x f(n), there is an analogous inversion: F(x) = Σd≤x μ(d)·G(x/d). This is the version used in sub-linear algorithms.
Pitfall: Make sure your summation indices are correct. The formula sums μ(d)·g(n/d), not μ(n/d)·g(d) — the latter is a different (but related) formula.

Square-Free Counting

Problem: Count square-free integers in [1, N].

An integer is square-free if no perfect square > 1 divides it. We can detect this using μ: n is square-free iff μ(n) ≠ 0, but there's a more efficient counting formula.

Formula via inclusion-exclusion

Let Q(N) = number of square-free integers ≤ N. By Möbius inversion:

Q(N) = Σd=1⌊√N⌋ μ(d) · ⌊N/d²⌋

Proof sketch

An integer n is not square-free iff there exists a prime p with p² | n. By inclusion-exclusion over the primes, the count of non-square-free numbers involves sums over products of primes — which is exactly what μ encodes. The square-free count becomes:

Q(N) = Σd²≤N μ(d) · ⌊N/d²⌋

// Count square-free integers in [1, N] // Requires mu[] sieved up to sqrt(N) long long count_squarefree(long long N) { int sq = (int)sqrt((double)N); long long ans = 0; for (int d = 1; d <= sq; ++d) ans += mu[d] * (N / ((long long)d * d)); return ans; }

Complexity

O(√N) time after O(√N) sieve precomputation. For N = 1018 this is about 109 — still feasible with careful optimization, or use sub-linear μ prefix-sum techniques to bring it down further.

Inclusion-exclusion connection: μ(d) for square-free d is exactly the inclusion-exclusion coefficient over the prime factors of d. Möbius inversion is a generalized inclusion-exclusion.

Coprime Pair Counting

Problem: Count pairs (i, j) with 1 ≤ i ≤ A, 1 ≤ j ≤ B, and gcd(i, j) = 1.

Derivation

Using the fundamental identity [gcd(i,j)=1] = Σd|gcd(i,j) μ(d), swap the order of summation:

Σi≤A Σj≤B [gcd(i,j)=1] = Σd=1min(A,B) μ(d) · ⌊A/d⌋ · ⌊B/d⌋

// Count coprime pairs (i,j) with 1≤i≤A, 1≤j≤B long long count_coprime_pairs(long long A, long long B) { long long ans = 0; long long lim = min(A, B); for (long long d = 1; d <= lim; ++d) ans += mu[d] * (A / d) * (B / d); return ans; }

Optimization — block decomposition

Since ⌊A/d⌋ and ⌊B/d⌋ each take O(√A) and O(√B) distinct values, we can group consecutive d-values with the same quotient pair and evaluate each block in O(1) using a prefix sum of μ.

// Optimized coprime count with block decomposition // Pre: mu_prefix[d] = μ(1) + μ(2) + ... + μ(d) long long count_coprime_fast(long long A, long long B) { long long ans = 0; long long lim = min(A, B); for (long long d = 1, nd; d <= lim; d = nd + 1) { long long qa = A / d, qb = B / d; nd = min({A / qa, B / qb, lim}); ans += (mu_prefix[nd] - mu_prefix[d - 1]) * qa * qb; } return ans; }

Complexity

Naive: O(min(A, B)). Optimized: O(√A + √B) per query, which handles A, B up to 107 easily with sieve precomputation.

Pitfall: When computing nd (the end of a block), take the minimum of A/qa, B/qb, and lim. Forgetting the lim bound causes out-of-range access in the μ prefix array.

Generalization — counting pairs with gcd = g

Replace the condition gcd(i,j)=1 with gcd(i,j)=g by substituting i→ig, j→jg. Then count_coprime_pairs(⌊A/g⌋, ⌊B/g⌋) gives the answer. Summing over g gives the full GCD distribution.

CP Applications

Pattern 1 — GCD sums

Compute Σi=1N Σj=1N gcd(i,j). Use the identity gcd(i,j) = Σd|gcd φ(d) and swap summation to get Σd=1N φ(d) · ⌊N/d⌋². Apply block decomposition for O(√N) per query.

Pattern 2 — "Visible lattice points"

How many lattice points (i, j) with 1 ≤ i, j ≤ N are visible from the origin? A point is visible iff gcd(i, j) = 1. The answer is count_coprime_pairs(N, N) = Σ μ(d)·⌊N/d⌋².

Naive approach

Compute gcd(i,j) for all pairs: O(N² log N). Hopelessly slow for N > 10⁴.

Möbius + blocks

Sieve μ in O(N), answer each query in O(√N). Handles N up to 10⁷ in milliseconds.

Pattern 3 — Euler's product via μ

The identity φ = μ ∗ Id lets you compute φ from μ (or vice versa) via Dirichlet convolution. This is rarely faster than a direct sieve, but understanding the relationship helps in deriving new identities.

Pattern 4 — Problem: "For each k, count pairs with gcd exactly k"

Let f(k) = #{(i,j) : gcd(i,j) = k}. Then f(k) = Σd=1⌊N/k⌋ μ(d) · ⌊N/(kd)⌋². Precomputing all f(k) for k = 1..N takes O(N√N) using block decomposition inside each k.

Key takeaway: Möbius inversion transforms hard divisibility conditions into tractable sums. Master the three-step recipe: (1) insert Σ μ(d), (2) swap summation order, (3) apply block decomposition.
