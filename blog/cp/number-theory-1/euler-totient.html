Euler's Totient Function

CP Math SeriesNumber Theory

Definition of φ(n)

Euler's totient function φ(n) counts the number of integers in {1, 2, …, n} that are coprime to n (i.e., gcd(k, n) = 1).

Examples:

  • φ(1) = 1 (by convention, gcd(1,1) = 1)
  • φ(6) = 2 — the integers 1, 5 are coprime to 6
  • φ(12) = 4 — the integers 1, 5, 7, 11 are coprime to 12
  • φ(p) = p − 1 for any prime p (all integers 1, …, p−1 are coprime to p)

Computing φ from Prime Factorization

The product formula for Euler's totient is one of the most important identities in number theory:

φ(n) = n × ∏p | n (1 − 1/p)

where the product runs over all distinct prime factors p of n. Equivalently, if n = p₁a₁ × p₂a₂ × … × pₖaₖ, then:

φ(n) = n × (1 − 1/p₁) × (1 − 1/p₂) × … × (1 − 1/pₖ)

Example: φ(60) = φ(2² × 3 × 5) = 60 × (1 − 1/2) × (1 − 1/3) × (1 − 1/5) = 60 × 1/2 × 2/3 × 4/5 = 16.

Direct Computation via Trial Division

int euler_phi(int n) { int result = n; for (int p = 2; p * p <= n; p++) { if (n % p == 0) { while (n % p == 0) n /= p; result -= result / p; // multiply by (1 - 1/p) } } if (n > 1) result -= result / n; // remaining prime factor return result; }

Complexity: O(√n) per call — same as trial-division factorization.

Why result -= result / p? This is equivalent to result = result * (p-1) / p but avoids potential overflow for large values. Since p divides the current result (we've confirmed n has factor p), the division is exact.

Sieve of Euler's Totient

When we need φ(k) for all k ≤ N, we use a sieve analogous to the Sieve of Eratosthenes. The idea is to initialize φ(k) = k and then for each prime p, multiply all multiples of p by (1 − 1/p).

const int MAXN = 1e6 + 5; int phi[MAXN]; void sieve_phi() { for (int i = 0; i < MAXN; i++) phi[i] = i; // initialize φ(i) = i for (int p = 2; p < MAXN; p++) { if (phi[p] == p) { // p is prime (unchanged) for (int j = p; j < MAXN; j += p) { phi[j] -= phi[j] / p; // multiply by (1 - 1/p) } } } }

Complexity: O(N log log N) — same as the Sieve of Eratosthenes.

Alternative: Linear Sieve for φ

Using the linear sieve (Euler's sieve), we can compute φ for all numbers up to N in O(N) time by leveraging the multiplicative property of φ.

const int MAXN = 1e6 + 5; int phi[MAXN], primes[MAXN], cnt = 0; bool is_composite[MAXN]; void linear_sieve_phi() { phi[1] = 1; for (int i = 2; i < MAXN; i++) { if (!is_composite[i]) { primes[cnt++] = i; phi[i] = i - 1; // φ(p) = p - 1 } for (int j = 0; j < cnt && (long long)i * primes[j] < MAXN; j++) { is_composite[i * primes[j]] = true; if (i % primes[j] == 0) { phi[i * primes[j]] = phi[i] * primes[j]; break; } else { phi[i * primes[j]] = phi[i] * (primes[j] - 1); } } } }

Standard Sieve — O(N log log N)

  • Simpler to implement
  • Uses Eratosthenes-style iteration
  • Good enough for N ≤ 107
  • Slightly higher constant factor

Linear Sieve — O(N)

  • Optimal time complexity
  • Also computes smallest prime factor
  • Better for N ≤ 108
  • More complex implementation
⚠ Memory: The linear sieve uses additional arrays (primes, is_composite). For N = 10⁸, this can use ~400MB. In tight memory constraints, prefer the standard sieve.

Euler's Theorem & Proof

Euler's theorem is the generalization of Fermat's Little Theorem to composite moduli.

Theorem. If gcd(a, n) = 1, then:

aφ(n) ≡ 1 (mod n)

Note: When n = p (prime), φ(p) = p − 1, and this reduces exactly to Fermat's Little Theorem.

Proof

Consider the set R = {r₁, r₂, …, rφ(n)} of all integers in {1, …, n} coprime to n. This is called a reduced residue system modulo n.

Step 1: If gcd(a, n) = 1, then {a·r₁, a·r₂, …, a·rφ(n)} modulo n is a permutation of R. This is because:

  • Each a·rᵢ is coprime to n (since gcd(a,n) = 1 and gcd(rᵢ,n) = 1).
  • All a·rᵢ mod n are distinct (if a·rᵢ ≡ a·rⱼ mod n, then rᵢ ≡ rⱼ mod n since gcd(a,n) = 1).
  • There are φ(n) of them, same as |R|.

Step 2: Take the product of all elements:

(a·r₁)(a·r₂)…(a·rφ(n)) ≡ r₁·r₂·…·rφ(n) (mod n)

Step 3: The left side equals aφ(n) × (r₁·r₂·…·rφ(n)). Since each rᵢ is coprime to n, their product P = r₁·r₂·…·rφ(n) is also coprime to n. Dividing both sides by P:

aφ(n) ≡ 1 (mod n)   QED.

Key Properties of φ

Understanding these properties is essential for CP problems involving the totient function.

Multiplicativity

If gcd(a, b) = 1, then φ(a·b) = φ(a)·φ(b). This makes φ a multiplicative function.

Divisor Sum Identity

A beautiful result: the sum of φ(d) over all divisors d of n equals n itself.

∑d | n φ(d) = n

Proof sketch: Partition {1, 2, …, n} by gcd(k, n). For each divisor d of n, exactly φ(n/d) integers k satisfy gcd(k, n) = d. Summing over all d gives n. ∎

Power of a Prime

For prime p and k ≥ 1: φ(pk) = pk − pk−1 = pk−1(p − 1).

Iterating φ Reaches 1

Repeatedly applying φ always reaches 1. For any n ≥ 3, φ(n) is even (since the group (Z/nZ)* has even order for n ≥ 3). So the sequence n, φ(n), φ(φ(n)), … strictly decreases and hits 1.

Useful identity: For n > 1, ∑ᵢ₌₁ⁿ [gcd(i,n) = 1] · i = n · φ(n) / 2. This sum of coprime residues appears in many CP problems.

CP Applications

Application 1: Modular Inverse for Composite Modulus

By Euler's theorem, a−1 ≡ aφ(n)−1 (mod n) when gcd(a, n) = 1. This generalizes the FLT-based inverse to non-prime moduli.

// Modular inverse for any modulus (when gcd(a, mod) = 1) long long mod_inverse_euler(long long a, long long mod) { return power(a, euler_phi(mod) - 1, mod); }

Application 2: Counting Coprime Pairs

Count pairs (i, j) with 1 ≤ i < j ≤ n and gcd(i, j) = 1. Answer: (∑ φ(k) for k=2..n) — which the sieve computes efficiently.

Application 3: Discrete Logarithm Reduction

To compute ab mod m where b is extremely large (given as a string), use:

  • If gcd(a, m) = 1: ab mod m = ab mod φ(m) mod m
  • If gcd(a, m) ≠ 1 and b ≥ log₂(m): ab mod m = a(b mod φ(m)) + φ(m) mod m
// Compute a^b mod m where b is given as a string long long big_power_mod(long long a, string &b, long long m) { long long ph = euler_phi(m); // Compute b mod φ(m) long long exp_mod = 0; bool b_ge_phi = false; for (char c : b) { exp_mod = exp_mod * 10 + (c - '0'); if (exp_mod >= ph) { b_ge_phi = true; exp_mod %= ph; } } if (b_ge_phi) exp_mod += ph; // handles gcd(a,m) != 1 return power(a, exp_mod, m); }
⚠ Common mistake: The formula ab mod φ(m) mod m only works when gcd(a, m) = 1. For the general case, you must add φ(m) to the exponent when b ≥ φ(m). This is often called the "generalized Euler's theorem" or "lifting the exponent."

Application 4: Primitive Roots Count

If n has a primitive root, the number of primitive roots modulo n is φ(φ(n)). This connects two layers of the totient function and appears in problems about cyclic groups.
