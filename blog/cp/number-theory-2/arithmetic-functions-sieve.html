    Arithmetic Functions & Sieves

    July 2026 · 35 min read

    Multiplicative Functions Review

    An arithmetic function f : ℤ+ → ℂ is multiplicative if f(1) = 1 and f(mn) = f(m)f(n) whenever gcd(m, n) = 1. It is completely multiplicative if f(mn) = f(m)f(n) for all m, n (no coprimality requirement).

    Key Multiplicative Functions

    φ(n) — Euler's Totient

    Counts integers in [1,n] coprime to n.

    φ(pa) = pa − pa−1 = pa−1(p − 1)

    μ(n) — Möbius Function

    μ(1)=1; μ(n)=(−1)k if n is a product of k distinct primes; μ(n)=0 if n has a squared prime factor.

    σk(n) — Divisor Sum

    σk(n) = ∑d|n dk. Special cases: σ0(n) = τ(n) (number of divisors), σ1(n) = σ(n) (sum of divisors).

    λ(n) — Liouville Function

    λ(n) = (−1)Ω(n) where Ω(n) is the number of prime factors counted with multiplicity. Completely multiplicative.

    Key property. Because multiplicative functions are determined by their values at prime powers, we need only compute f(pa) for each prime power in the factorisation. The sieve computes these building blocks.

    Formulas at Prime Powers

    For prime p and exponent a ≥ 1:

    • φ(pa) = pa−1(p − 1)
    • μ(pa) = −1 if a=1, else 0
    • τ(pa) = a + 1
    • σ(pa) = (pa+1 − 1) / (p − 1)
    • λ(pa) = (−1)a

    Linear Sieve

    The linear sieve (Euler's sieve) finds all primes up to N in O(N) time, visiting each composite exactly once. The key idea: each composite m is “killed” by its smallest prime factor spf(m).

    Algorithm

    linear_sieve(N): spf[1..N] = 0 // smallest prime factor primes = [] for i = 2 to N: if spf[i] == 0: // i is prime spf[i] = i primes.append(i) for p in primes: if p > spf[i] or i * p > N: break spf[i * p] = p return spf, primes

    Why O(N)?

    Each composite c = spf(c) · (c / spf(c)) is produced exactly once: when i = c / spf(c) and p = spf(c). The inner loop breaks when p > spf(i), ensuring that spf(i·p) = p ≤ spf(i), so every composite is generated with its true smallest prime factor. Since each composite is created once, the total work is Θ(N).

    C++ Implementation

    const int MAXN = 10'000'001;
int spf[MAXN];
vector<int> primes;

void linear_sieve() {
    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == 0) {
            spf[i] = i;
            primes.push_back(i);
        }
        for (int p : primes) {
            if (p > spf[i] || (long long)i * p >= MAXN)
                break;
            spf[i * p] = p;
        }
    }
}
    Overflow guard. The product i * p can overflow int when MAXN approaches 2·109. Cast to long long for the comparison.

    Computing Functions from SPF

    Once we have spf[], we can compute any multiplicative function f in O(N) total by maintaining auxiliary arrays during the sieve. The key insight: when processing composite i·p where p = spf(i·p), we can extract the full prime-power contribution of p.

    Euler's Totient φ

    int phi[MAXN];

void compute_phi() {
    phi[1] = 1;
    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == i) {  // i is prime
            phi[i] = i - 1;
        } else {
            int p = spf[i], q = i / p;
            if (q % p == 0) {
                // p^2 | i, so phi(i) = p * phi(i/p)
                phi[i] = p * phi[q];
            } else {
                // gcd(p, i/p) = 1, so phi(i) = phi(p)*phi(i/p)
                phi[i] = (p - 1) * phi[q];
            }
        }
    }
}

    Correctness. If p2 | i, then i/p shares the prime p, so we can't use multiplicativity directly. Instead, φ(pa+1) = p · φ(pa), giving φ(i) = p · φ(i/p). If p ∤ (i/p), then gcd(p, i/p) = 1 and multiplicativity gives φ(i) = φ(p) · φ(i/p) = (p−1) · φ(i/p).

    Möbius Function μ

    int mu[MAXN];

void compute_mu() {
    mu[1] = 1;
    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == i) {
            mu[i] = -1;
        } else {
            int p = spf[i], q = i / p;
            if (q % p == 0) {
                mu[i] = 0;  // p^2 | i
            } else {
                mu[i] = -mu[q];
            }
        }
    }
}

    Number of Divisors τ

    For τ we need to track the exponent of the smallest prime factor. Maintain an auxiliary array e[] where e[n] is the exponent of spf(n) in n. 

    int tau[MAXN], e[MAXN];

void compute_tau() {
    tau[1] = 1; e[1] = 0;
    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == i) {
            tau[i] = 2; e[i] = 1;
        } else {
            int p = spf[i], q = i / p;
            if (spf[q] == p) {
                e[i] = e[q] + 1;
                tau[i] = tau[q] / (e[q] + 1) * (e[i] + 1);
            } else {
                e[i] = 1;
                tau[i] = tau[q] * 2;
            }
        }
    }
}

    Sum of Divisors σ

    long long sigma[MAXN], pw[MAXN];

void compute_sigma() {
    sigma[1] = 1; pw[1] = 1;
    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == i) {
            sigma[i] = 1 + i;
            pw[i] = i;  // p^1
        } else {
            int p = spf[i], q = i / p;
            if (spf[q] == p) {
                pw[i] = pw[q] * p;
                // sigma(p^a) = (p^{a+1}-1)/(p-1)
                sigma[i] = sigma[q] / sigma[pw[q]] *
                           (pw[i] * (long long)p - 1) / (p - 1);
            } else {
                pw[i] = p;
                sigma[i] = sigma[q] * (1 + p);
            }
        }
    }
}

    Dirichlet Convolution

    The Dirichlet convolution of arithmetic functions f and g is:

    (f * g)(n) = ∑d|n f(d) · g(n/d)

    Properties

    • Associative & commutative: f * g = g * f, (f * g) * h = f * (g * h).
    • Identity: The function ε(n) = [n=1] is the identity: f * ε = f.
    • Closure: If f and g are multiplicative, then f * g is multiplicative.

    Key Identities

    • μ * 1 = ε — the Möbius function is the Dirichlet inverse of the constant function 1.
    • φ * 1 = id — ∑d|n φ(d) = n.
    • μ * id = φ — φ(n) = ∑d|n μ(d) · (n/d).
    • 1 * 1 = τ — ∑d|n 1 = τ(n).
    • id * 1 = σ — ∑d|n d = σ(n).

    Möbius Inversion

    Möbius Inversion Formula. If g(n) = ∑d|n f(d) for all n, then f(n) = ∑d|n μ(d) · g(n/d). Equivalently: g = f * 1 ⇒ f = g * μ.

    Proof. If g = f * 1, then g * μ = f * 1 * μ = f * ε = f. □

    Computational Möbius Inversion

    Given an array g[1..N], compute f[1..N] where f = g * μ. Naively this is O(N log N) using the harmonic-series iteration: 

    // Compute f[n] = sum_{d|n} mu(d) * g[n/d]
for (int d = 1; d <= N; d++) {
    for (int m = d; m <= N; m += d) {
        f[m] += mu[d] * g[m / d];
    }
}

    The ∑d=1N N/d = O(N log N) bound makes this efficient enough for N up to ~107.

    Prefix Sums of Multiplicative Functions

    A common CP task: compute F(N) = ∑i=1N f(i) for a multiplicative function f, where N can be up to 109 or even 1012. The linear sieve handles N ≤ 107, but for larger N we need sublinear methods.

    The Hyperbola Method

    For f = g * h (Dirichlet convolution), the prefix sum satisfies:

    F(N) = ∑d=1N g(d) · H(N/d) = ∑d=1N h(d) · G(N/d)

    where G and H are prefix sums of g and h. By splitting the sum at √N:

    F(N) = ∑d ≤ √N g(d) · H(N/d) + ∑d ≤ √N h(d) · G(N/d) − G(√N) · H(√N)

    Meissel–Mertens approach. If G and H can each be computed in O(1), the hyperbola method computes F(N) in O(√N) time. This is the backbone of many sublinear summation techniques.

    Lucy DP (for sum of primes)

    To compute ∑p ≤ N f(p) (sum over primes), we use the “Lucy DP” approach: define S(v, j) = ∑ f(i) for i ≤ v where i is either 1 or has smallest prime factor > pj. Sieve out primes one by one. This naturally leads to the Min-25 sieve (covered in the next post).

    Computing ∑φ(i) for i=1..N

    Using φ * 1 = id, i.e., ∑d|n φ(d) = n, and summing:

    ∑n=1N n = ∑n=1N ∑d|n φ(d) = ∑d=1N φ(d) · ⌊N/d⌋

    Let Φ(N) = ∑i=1N φ(i). Then:

    N(N+1)/2 = ∑d=1N Φ(N/d) ⇒ Φ(N) = N(N+1)/2 − ∑d=2N Φ(N/d)

    The values ⌊N/d⌋ take only O(√N) distinct values, so with memoisation this runs in O(N2/3) time (with appropriate sieve precomputation for small values). 

    unordered_map<long long, long long> memo_phi;
long long phi_prefix[SMALL];  // precomputed for i ≤ SMALL

long long sum_phi(long long n) {
    if (n < SMALL) return phi_prefix[n];
    if (memo_phi.count(n)) return memo_phi[n];

    long long res = n * (n + 1) / 2;
    for (long long d = 2, nd; d <= n; d = nd + 1) {
        long long q = n / d;
        nd = n / q;  // largest d' with floor(n/d') = q
        res -= (nd - d + 1) * sum_phi(q);
    }
    return memo_phi[n] = res;
}

    Implementation

    Complete Linear Sieve with All Functions

    The following code computes spf, φ, μ, τ, and σ simultaneously in a single O(N) pass. 

    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 10'000'001;
int spf[MAXN], phi_arr[MAXN], mu_arr[MAXN];
int tau_arr[MAXN], e_arr[MAXN];
long long sigma_arr[MAXN], pw_arr[MAXN];
vector<int> primes;

void full_sieve() {
    phi_arr[1] = mu_arr[1] = tau_arr[1] = sigma_arr[1] = 1;
    pw_arr[1] = 1;

    for (int i = 2; i < MAXN; i++) {
        if (spf[i] == 0) {
            spf[i] = i;
            primes.push_back(i);
            phi_arr[i] = i - 1;
            mu_arr[i] = -1;
            tau_arr[i] = 2;
            e_arr[i] = 1;
            sigma_arr[i] = 1 + i;
            pw_arr[i] = i;
        }
        for (int p : primes) {
            if (p > spf[i] || (long long)i * p >= MAXN)
                break;
            int ip = i * p;
            spf[ip] = p;

            if (i % p == 0) {
                // p divides i: p^2 | ip
                phi_arr[ip] = phi_arr[i] * p;
                mu_arr[ip] = 0;
                e_arr[ip] = e_arr[i] + 1;
                tau_arr[ip] = tau_arr[i] / (e_arr[i] + 1)
                            * (e_arr[ip] + 1);
                pw_arr[ip] = pw_arr[i] * p;
                sigma_arr[ip] = sigma_arr[i / pw_arr[i]]
                    * (pw_arr[ip] * (long long)p - 1) / (p - 1);
            } else {
                // gcd(p, i) = 1
                phi_arr[ip] = phi_arr[i] * (p - 1);
                mu_arr[ip] = -mu_arr[i];
                e_arr[ip] = 1;
                tau_arr[ip] = tau_arr[i] * 2;
                pw_arr[ip] = p;
                sigma_arr[ip] = sigma_arr[i] * (1 + p);
            }
        }
    }
}
    Memory note. This sieve uses ≈80 bytes per entry (7 arrays). For MAXN = 107, that's ≈760 MB. In contests with 256 MB memory limits, compute only the functions you need.

    Factorisation from SPF

    // O(log n) factorisation using precomputed spf[]
vector<pair<int,int>> factorize(int n) {
    vector<pair<int,int>> factors;
    while (n > 1) {
        int p = spf[n], cnt = 0;
        while (n % p == 0) {
            n /= p;
            cnt++;
        }
        factors.push_back({p, cnt});
    }
    return factors;
}

    Tips for Contests

    When to Use Linear Sieve

    N ≤ 107. Need multiple functions. Memory is not extremely tight.

    When to Use Sublinear Methods

    N > 107. Only need prefix sums, not pointwise values. Typically combined with the Meissel/Lucy DP approach.
