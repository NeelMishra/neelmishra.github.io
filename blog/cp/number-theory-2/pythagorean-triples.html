    Pythagorean Triples

    July 2026 · 30 min read

    Primitive Pythagorean Triples

    A Pythagorean triple is an ordered triple of positive integers (a, b, c) satisfying a² + b² = c². The triple is primitive when gcd(a, b, c) = 1 — equivalently, when gcd(a, b) = 1 (the third condition follows automatically).

    Key fact. Every non-primitive triple is a positive integer multiple of a primitive one: if (a, b, c) satisfies a² + b² = c² with d = gcd(a, b, c) > 1, then (a/d, b/d, c/d) is primitive.

    The Parametrization

    Every primitive Pythagorean triple (with b even) can be written as:

    a = m² − n²,   b = 2mn,   c = m² + n²

    where m and n are positive integers with:

    • m > n > 0
    • gcd(m, n) = 1
    • m − n is odd (i.e., m and n have opposite parity)

    The first few primitive triples generated this way are:

    m n a b c
    21345
    3251213
    4115817
    4372425
    52212029
    5494041

    Note that every primitive triple has exactly one even leg (always b = 2mn) and the hypotenuse c is always odd.

    Proof of Completeness

    We must show that the parametrization above captures every primitive Pythagorean triple. Suppose (a, b, c) is primitive with a² + b² = c². We first observe:

    Step 1: Exactly One of a, b is Even

    If both a and b are even, then c² is divisible by 4 so c is even, contradicting gcd(a,b,c) = 1. If both are odd, then a² + b² ≡ 1 + 1 = 2 (mod 4), but a perfect square is ≡ 0 or 1 mod 4 — contradiction. So exactly one is even; WLOG let b be even.

    Step 2: Factor the Equation

    Write b² = c² − a² = (c − a)(c + a). Since a and c are both odd, c − a and c + a are both even. Set c − a = 2u, c + a = 2v, so b² = 4uv and (b/2)² = uv.

    Step 3: u and v are Coprime

    If a prime p divides both u and v, then p divides u + v = c and v − u = a, hence p | gcd(a, c). But gcd(a, c) | gcd(a², c²) and b² = c² − a², so p | b, contradicting primitivity.

    Step 4: Conclude the Parametrization

    Since uv = (b/2)² and gcd(u, v) = 1, both u and v must individually be perfect squares. Write u = n², v = m². Then:

    • c = u + v = m² + n²
    • a = v − u = m² − n²
    • b = 2√(uv) = 2mn

    Since gcd(u, v) = 1 we get gcd(m, n) = 1, and since a is odd, m² − n² is odd, forcing m and n to have opposite parity. ∎

    Corollary. The map (m, n) → (m² − n², 2mn, m² + n²) is a bijection from pairs satisfying the three conditions to primitive triples with even second component.

    Generating Triples

    In 1934, Berggren discovered that every primitive Pythagorean triple can be obtained from the seed triple (3, 4, 5) by repeatedly applying three linear transformations. Write a triple as a column vector v = (a, b, c)T. The three Berggren matrices are:

    // Berggren matrices A, B, C ┌ 1 -2 2 ┐ ┌ 1 2 2 ┐ ┌ -1 2 2 ┐ A = │ 2 -1 2 │ B = │ 2 1 2 │ C = │ -2 1 2 │ └ 2 -2 3 ┘ └ 2 2 3 ┘ └ -2 2 3 ┘

    Each matrix, when applied to a primitive triple, produces another primitive triple. Moreover, every primitive triple is reached exactly once, forming an infinite ternary tree rooted at (3, 4, 5).

    Tree Structure

    The first few levels of the tree:

    // Level 0 (3, 4, 5) ├── A: (5, 12, 13) │ ├── (7, 24, 25) │ ├── (55, 48, 73) │ └── (45, 28, 53) ├── B: (21, 20, 29) │ ├── (39, 80, 89) │ ├── (119, 120, 169) │ └── (77, 36, 85) └── C: (15, 8, 17) ├── (33, 56, 65) ├── (65, 72, 97) └── (35, 12, 37)
    Traversal order. A BFS of the Berggren tree enumerates all primitive triples. By sorting each level, we can enumerate them in increasing order of hypotenuse, which is useful for contest problems that ask "all triples up to N."

    Direct Enumeration by (m, n)

    Alternatively, iterate over all valid (m, n) pairs with m > n ≥ 1, gcd(m, n) = 1, m − n odd. For all triples with hypotenuse ≤ N, we need m² + n² ≤ N, so m ≤ √N. This gives an O(N) algorithm (summing over valid n for each m).

    Counting Triples

    Let P(N) be the number of primitive Pythagorean triples with hypotenuse c ≤ N. The asymptotic count is:

    P(N) ∼ N / (2π)

    This is a classical result. The total number of triples (including non-primitive) with hypotenuse ≤ N is:

    T(N) ∼ N / (2π) · ln N

    Connection to Sum of Two Squares

    A positive integer c is the hypotenuse of a primitive triple if and only if c has at least one prime factor of the form 4k + 1. More precisely, the number of representations of c as a sum of two squares is connected to r₂(c²), the number of ways to write c² as a² + b².

    Theorem (Jacobi). The number of representations of n as a sum of two squares (counting order and signs) is r₂(n) = 4(d₁(n) − d₃(n)), where d₁(n) counts divisors ≡ 1 mod 4 and d₃(n) counts divisors ≡ 3 mod 4.

    For a prime p ≡ 1 (mod 4), r₂(p) = 8, giving exactly one essentially distinct representation p = a² + b² (up to order and signs). This connects to Fermat's theorem on sums of two squares.

    Counting via Multiplicative Functions

    Define χ(n) as the non-principal character mod 4: χ(n) = 0 if n is even, χ(n) = 1 if n ≡ 1 (mod 4), χ(n) = −1 if n ≡ 3 (mod 4). Then:

    r₂(n) = 4 · Σd|n χ(d)

    The number of primitive triples with hypotenuse exactly c equals 2ω₁(c)−1 where ω₁(c) is the number of distinct prime factors of c that are ≡ 1 (mod 4), provided c has no prime factors ≡ 3 (mod 4) appearing to an odd power (and c is odd).

    Implementation

    Generating All Primitive Triples up to N

    // Generate all primitive Pythagorean triples (a, b, c) with c <= N
#include <bits/stdc++.h>
using namespace std;

struct Triple { long long a, b, c; };

vector<Triple> genPrimitive(long long N) {
    vector<Triple> res;
    for (long long m = 2; m * m <= N; m++) {
        for (long long n = 1; n < m; n++) {
            if ((m - n) % 2 == 0) continue;  // same parity
            if (__gcd(m, n) != 1) continue;    // not coprime
            long long a = m*m - n*n;
            long long b = 2*m*n;
            long long c = m*m + n*n;
            if (c > N) break;
            if (a > b) swap(a, b);
            res.push_back({a, b, c});
        }
    }
    sort(res.begin(), res.end(),
         [](const Triple& x, const Triple& y) {
             return x.c < y.c;
         });
    return res;
}

    Berggren Tree BFS

    // BFS on Berggren tree — enumerate all primitive triples with c <= N
using Vec3 = array<long long, 3>;

const int M[3][3][3] = {
    {{ 1,-2, 2},{ 2,-1, 2},{ 2,-2, 3}},  // A
    {{ 1, 2, 2},{ 2, 1, 2},{ 2, 2, 3}},  // B
    {{-1, 2, 2},{-2, 1, 2},{-2, 2, 3}}   // C
};

vector<Vec3> berggren(long long N) {
    vector<Vec3> result;
    queue<Vec3> q;
    q.push({3, 4, 5});
    while (!q.empty()) {
        auto [a, b, c] = q.front(); q.pop();
        if (c > N) continue;
        result.push_back({a, b, c});
        for (int k = 0; k < 3; k++) {
            Vec3 nxt;
            for (int i = 0; i < 3; i++)
                nxt[i] = M[k][i][0]*a + M[k][i][1]*b + M[k][i][2]*c;
            if (nxt[2] <= N) q.push(nxt);
        }
    }
    return result;
}

    Counting All Triples (Primitive + Non-Primitive)

    // Count ALL triples (a, b, c) with c <= N
long long countAll(long long N) {
    long long cnt = 0;
    for (long long m = 2; m * m <= N; m++) {
        for (long long n = 1; n < m; n++) {
            if ((m - n) % 2 == 0) continue;
            if (__gcd(m, n) != 1) continue;
            long long c0 = m*m + n*n;
            if (c0 > N) break;
            cnt += N / c0;  // multiples k·c0 <= N
        }
    }
    return cnt;
}
    Complexity. Both the parametric generator and the Berggren BFS run in O(N) total work to enumerate all primitive triples with hypotenuse ≤ N, since the number of primitive triples itself is Θ(N). For counting only (without enumeration), the parametric approach runs in the same bound.

    CP Applications

    Lattice Points on Circles

    A classic problem: count the number of lattice points on the circle x² + y² = R². This equals r₂(R²), which can be computed using the factorization of R and Jacobi's formula. Pythagorean triples give the positive-quadrant lattice points directly.

    // Lattice points on circle x² + y² = R² (first quadrant, on axes)
int latticeOnCircle(long long R) {
    int cnt = 0;
    for (long long a = 0; a <= R; a++) {
        long long bsq = R*R - a*a;
        long long b = (long long)sqrtl(bsq);
        if (b*b == bsq) cnt++;
    }
    return cnt;  // multiply by 4 for all quadrants, adjust for axes
}

    Right Triangles with Integer Coordinates

    Given a grid of size W × H, count right triangles with vertices at lattice points where one side is along a grid line. Every such triangle corresponds to a Pythagorean triple scaled to fit the grid.

    Contest Problem: Counting Right Triangles

    Problem. Given N ≤ 106, count the number of Pythagorean triples (a, b, c) with a ≤ b < c ≤ N.

    function countTriples(N): count ← 0 for m ← 2 to √N: for n ← 1 to m − 1: if (m − n) is even: continue if gcd(m, n) ≠ 1: continue c₀ ← m² + n² if c₀ > N: break count ← count + ⌊N / c₀⌋ return count

    Generating Triples with Fixed Leg

    Sometimes a problem fixes one leg: "find all triples with a = k." From the parametrization, if a = m² − n² is odd, factor a = (m−n)(m+n). Each factorization a = d₁ · d₂ with d₁ < d₂, both odd, gives n = (d₂ − d₁)/2, m = (d₁ + d₂)/2. Check gcd(m, n) = 1.

    // Find all primitive triples with odd leg = a
vector<Triple> triplesWithLeg(long long a) {
    vector<Triple> res;
    for (long long d1 = 1; d1 * d1 < a; d1++) {
        if (a % d1 != 0) continue;
        long long d2 = a / d1;
        if ((d1 % 2) != (d2 % 2)) continue;  // need same parity (both odd)
        long long m = (d1 + d2) / 2, n = (d2 - d1) / 2;
        if (__gcd(m, n) != 1) continue;
        long long b = 2*m*n, c = m*m + n*n;
        res.push_back({a, b, c});
    }
    return res;
}

    Parametric Enumeration

    Time: O(N) for all triples with c ≤ N

    Best for: ordered enumeration, counting

    Pros: simple, cache-friendly iteration

    Berggren BFS

    Time: O(N) — same asymptotic

    Best for: tree-structured problems, DFS pruning

    Pros: natural DFS for bounded search

    Common pitfall. Don't forget that the parametrization gives a = m² − n² as odd and b = 2mn as even. If a problem asks for triples with a < b, you may need to swap. Also, for non-primitive triples, multiply each primitive triple by all integers k with k · c ≤ N.
