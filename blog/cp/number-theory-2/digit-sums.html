    Digit Sums & Digital Root

    July 2026 · 25 min read

    Digit Sum Basics

    Given a positive integer n with decimal representation n = dk dk−1 … d1 d0, the digit sum is defined as

    S(n) = dk + dk−1 + … + d1 + d0

    Because n equals ∑ di · 10i and 10 ≡ 1 (mod 9), every power 10i ≡ 1 (mod 9). Consequently:

    n ≡ dk + dk−1 + … + d0 ≡ S(n) (mod 9)

    Core identity. For every non-negative integer n, n ≡ S(n) (mod 9). This is the foundation of the casting-out-nines technique used since antiquity to verify arithmetic.

    Proof

    Write n = ∑i=0k di · 10i. Since 10 ≡ 1 (mod 9), we have 10i ≡ 1i = 1 (mod 9). Therefore n ≡ ∑ di · 1 = S(n) (mod 9). □

    Basic Properties of S(n)

    • Sub-additivity: S(a + b) ≤ S(a) + S(b). Equality holds iff no carry occurs in the addition.
    • Carries reduce digit sums: Each carry decreases the digit sum by exactly 9. If addition of a and b produces c carries, then S(a+b) = S(a) + S(b) − 9c.
    • Multiplicative congruence: S(ab) ≡ S(a)·S(b) (mod 9), following directly from ab ≡ S(ab) and a ≡ S(a), b ≡ S(b) (mod 9).
    • Bound: For an n-digit number, 1 ≤ S(n) ≤ 9n. The maximum is achieved by the repdigit 999…9.

    Computing S(n) — O(log n)

    int digit_sum(long long n) {
    int s = 0;
    while (n > 0) {
        s += n % 10;
        n /= 10;
    }
    return s;
}

    Digital Root

    The digital root dr(n) is obtained by repeatedly taking the digit sum until a single digit remains:

    dr(n) = S(S(…S(n)…))

    Closed-Form Formula

    For n ≥ 1 the digital root has the elegant closed form:

    dr(n) = 1 + ((n − 1) mod 9)

    Equivalently, dr(n) = n mod 9 when n mod 9 ≠ 0, and dr(n) = 9 when n mod 9 = 0 (for n ≥ 1). We also define dr(0) = 0.

    Proof

    Since S(n) ≡ n (mod 9) and repeated application preserves the congruence, dr(n) ≡ n (mod 9). The digital root is a single digit in {1, 2, …, 9} (for n ≥ 1), and the unique value in that range congruent to n mod 9 is exactly 1 + ((n−1) mod 9). □

    O(1) computation. The digital root never requires iterating over digits. Use the formula dr(n) = 1 + (n - 1) % 9 directly. 
    int digital_root(long long n) {
    if (n == 0) return 0;
    return 1 + (n - 1) % 9;
}

    Properties of Digital Root

    • Periodicity: dr(n) is periodic with period 9 for n ≥ 1: dr(1)=1, dr(2)=2, …, dr(9)=9, dr(10)=1, …
    • Multiplicative: dr(a · b) = dr(dr(a) · dr(b)). This makes digital root a ring homomorphism from (ℤ, +, ·) onto (ℤ/9ℤ, +, ·) (with relabelling 0 → 9).
    • Perfect squares: dr(n²) ∈ {1, 4, 7, 9}. If dr(n) ∉ {1, 4, 7, 9}, then n cannot be a perfect square.
    • Triangular numbers: dr(Tk) ∈ {1, 3, 6, 9}.

    Application: Quick Square Test

    Before running an expensive integer square-root check, verify that dr(n) ∈ {1, 4, 7, 9}. This filters out roughly 5/9 ≈ 55.6% of non-squares in O(1). 

    bool maybe_square(long long n) {
    int d = digital_root(n);
    return d == 1 || d == 4 || d == 7 || d == 9;
}

    Digit Sum in Other Bases

    The digit sum generalises naturally. For base b ≥ 2, write n = ∑ di · bi, and define:

    Sb(n) = ∑ di

    By the same argument (b ≡ 1 mod (b−1)), we have n ≡ Sb(n) (mod b−1).

    Legendre's Formula Revisited

    The exact power of prime p dividing n! is given by Legendre's formula:

    vp(n!) = ∑i=1∞ ⌊n / pi⌋ = (n − Sp(n)) / (p − 1)

    Legendre ↔ digit sum. The p-adic valuation of n! depends only on the base-p digit sum of n. This is extremely useful for computing vp(n!) when n is huge.

    Kummer's Theorem & Carries

    Kummer's Theorem. The exact power of prime p dividing C(m+n, m) equals the number of carries when adding m and n in base p.

    Proof sketch.

    vp(C(m+n,m)) = vp((m+n)!) − vp(m!) − vp(n!) = (Sp(m) + Sp(n) − Sp(m+n)) / (p−1).

    Each carry in base-p addition of m and n decreases the digit sum by exactly (p−1). So the number of carries c satisfies Sp(m) + Sp(n) − Sp(m+n) = c(p−1), giving vp(C(m+n,m)) = c. □ 

    // Count carries when adding a and b in base p
int count_carries(long long a, long long b, int p) {
    int carries = 0, carry = 0;
    while (a > 0 || b > 0 || carry) {
        int sum = a % p + b % p + carry;
        carry = sum / p;
        carries += carry;
        a /= p;
        b /= p;
    }
    return carries;
}

    Lucas' Theorem Connection

    For prime p, C(n, k) ≡ ∏ C(ni, ki) (mod p), where ni and ki are the base-p digits of n and k. In particular, C(n,k) ≡ 0 (mod p) iff any digit of k exceeds the corresponding digit of n in base p. This pairs with Kummer's theorem to give a complete picture of p-adic valuations of binomial coefficients.

    Digit DP

    Digit DP is the standard technique for counting integers in [0, N] satisfying digit-based constraints. The idea: process digits from the most significant to least significant, tracking whether we are still “tight” (equal to N's prefix) or “free” (already strictly less).

    General Framework

    // dp[pos][tight][state] = count of valid numbers // pos : current digit position (0 = MSB) // tight : are we still bounded by N? // state : problem-specific (e.g., digit sum mod m) solve(pos, tight, state): if pos == num_digits: return is_valid(state) if memo[pos][tight][state] != -1: return memo[pos][tight][state] limit = digits[pos] if tight else 9 result = 0 for d = 0 to limit: new_tight = tight AND (d == limit) new_state = transition(state, d) result += solve(pos + 1, new_tight, new_state) memo[pos][tight][state] = result return result

    State Design for Digit-Sum Problems

    When counting integers with digit sum ≡ r (mod m), the state is simply (current digit-sum mod m). The DP has O(D · 2 · m) states where D is the number of digits, each with up to 10 transitions ⇒ total O(10 · D · m) time.

    Complexity. Digit DP runs in O(D · B · |State|) where D is the number of digits, B is the base (10 for decimal), and |State| is the state-space size. For digit-sum problems with modular constraints, |State| = m, giving O(10 · D · m).

    Counting Numbers with Exact Digit Sum

    Problem: Count integers in [1, N] with digit sum exactly equal to S.

    Set the DP state as the running digit sum (capped at S+1 to save memory, since any partial sum exceeding S is already invalid). Base case: at pos = D, return 1 iff running sum equals S.

    Range Queries: [L, R]

    To count valid numbers in [L, R], compute f(R) − f(L−1), where f(X) counts valid numbers in [0, X]. This is the standard inclusion-exclusion trick for digit DP.

    Implementation

    C++ Digit DP Template

    The following template counts integers in [0, N] whose digit sum is divisible by a given modulus m. 

    #include <bits/stdc++.h>
using namespace std;

string num;
int m;
long long dp[20][2][200];  // [pos][tight][sum_mod_m]
bool vis[20][2][200];

long long solve(int pos, bool tight, int sum_mod) {
    if (pos == (int)num.size()) {
        return sum_mod == 0 ? 1 : 0;
    }
    if (vis[pos][tight][sum_mod]) return dp[pos][tight][sum_mod];
    vis[pos][tight][sum_mod] = true;

    int limit = tight ? (num[pos] - '0') : 9;
    long long res = 0;
    for (int d = 0; d <= limit; d++) {
        res += solve(pos + 1,
                     tight && (d == limit),
                     (sum_mod + d) % m);
    }
    return dp[pos][tight][sum_mod] = res;
}

long long count_up_to(long long N) {
    if (N < 0) return 0;
    num = to_string(N);
    memset(vis, false, sizeof vis);
    return solve(0, true, 0);
}

int main() {
    long long L, R;
    cin >> L >> R >> m;
    cout << count_up_to(R) - count_up_to(L - 1) << "\n";
}

    Digit DP with Exact Digit Sum

    A slight modification counts numbers with digit sum exactly equal to S: 

    int target_sum;
long long dp2[20][2][200];
bool vis2[20][2][200];

long long solve2(int pos, bool tight, int cur_sum) {
    if (cur_sum > target_sum) return 0;  // prune
    if (pos == (int)num.size()) {
        return cur_sum == target_sum ? 1 : 0;
    }
    if (vis2[pos][tight][cur_sum])
        return dp2[pos][tight][cur_sum];
    vis2[pos][tight][cur_sum] = true;

    int limit = tight ? (num[pos] - '0') : 9;
    long long res = 0;
    for (int d = 0; d <= limit; d++) {
        res += solve2(pos + 1,
                      tight && (d == limit),
                      cur_sum + d);
    }
    return dp2[pos][tight][cur_sum] = res;
}

    Handling Leading Zeros

    Some problems require distinguishing “the number 007” (three digits) from “7” (one digit). Add a boolean started flag to the state. If started is false and the current digit is 0, don't transition into “started” mode. This adds a factor of 2 to the state space but is necessary for problems like “count numbers with exactly k non-zero digits.”

    CP Applications

    Casting Out Nines

    A classic verification technique: to check whether a + b = c, verify S(a) + S(b) ≡ S(c) (mod 9). This catches ≈88.9% of errors (all errors except those that are multiples of 9). In contests, this is useful as a sanity check for big-number arithmetic.

    Divisibility Rules via Digit Sums

    Divisibility by 3

    3 | n ⇔ 3 | S(n). Since 10 ≡ 1 (mod 3).

    Divisibility by 9

    9 | n ⇔ 9 | S(n). Same reasoning with mod 9.

    Divisibility by 11

    11 | n ⇔ 11 divides the alternating digit sum d0 − d1 + d2 − … since 10 ≡ −1 (mod 11).

    Divisibility by 7

    No simple digit-sum rule exists. Use the weighted cycle {1,3,2,6,4,5} repeating on digits from the right.

    Contest Problem: Digit Sum Queries

    Problem. Given N ≤ 1018 and M ≤ 200, count integers in [1, N] whose digit sum is divisible by M.

    This is a direct application of the digit DP template above with state = (digit sum mod M). Complexity: O(19 · 2 · M · 10) = O(38000) for M = 200. Extremely fast.

    Contest Problem: Minimum Operations

    Problem. Given n, repeatedly replace n with n − S(n). How many steps until n = 0?

    Key insight: n − S(n) is always divisible by 9 (since n ≡ S(n) mod 9). After the first step, n becomes a multiple of 9, and all subsequent values remain multiples of 9. The sequence decreases rapidly — each step removes at least Ω(log n) from n. Total steps: O(n / log n) in the worst case, but in practice much faster for large n.

    Trap. Don't confuse S(n) with dr(n) in contest problems. S(1234) = 10 but dr(1234) = 1. Many problems ask for digit sum (one-level), not digital root (iterated to single digit).

    Advanced: Sum of Digit Sums

    Compute ∑i=1N S(i). This can be done in O(log N) by counting the contribution of each digit position independently. Position j (0-indexed from the right) contributes: 

    long long sum_digit_sums(long long N) {
    long long result = 0, pw = 1;
    while (N >= pw) {
        long long full = N / (pw * 10);
        long long d = (N / pw) % 10;
        long long rem = N % pw;
        // Contribution of this position
        result += full * 45 * pw;
        result += d * (d - 1) / 2 * pw;
        result += d * (rem + 1);
        pw *= 10;
    }
    return result;
}

    The idea: for each position, digits 0 through d−1 each appear in a complete cycle of length pw, contributing full * 45 * pw (where 45 = 0+1+…+9 for all 10 digits). Digit d appears for the partial cycle of length rem + 1.
