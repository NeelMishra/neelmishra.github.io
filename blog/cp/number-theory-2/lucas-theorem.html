    Lucas' Theorem

    July 2026 · 30 min read

    Binomial Coefficients Modulo Primes

    Computing C(n, k) mod p is one of the most common operations in combinatorics problems. When n and k are small (say, ≤ 106), we can precompute factorials and inverse factorials modulo p. But what if n can be as large as 1018 and p is a small prime (say, p ≤ 105)?

    Standard precomputation requires O(n) space and time—totally infeasible. Lucas' theorem reduces the problem to O(logp n) small binomial coefficients, each computable in O(p) time.

    When to use Lucas. Lucas' theorem applies when the modulus p is prime and relatively small. For large prime moduli, use the standard factorial-inverse approach. For composite moduli, combine Lucas with CRT (see below).

    Lucas' Theorem Statement & Proof

    Statement

    Theorem (Lucas, 1878). Let p be a prime. Write n and k in base p:

    n = nmpm + … + n1p + n0
    k = kmpm + … + k1p + k0

    Then:

    C(n, k) ≡ ∏i=0m C(ni, ki) (mod p)

    where C(a, b) = 0 if b > a.

    Proof

    We use the generating-function approach. Consider the polynomial identity in 𝔽p[x]:

    (1 + x)p ≡ 1 + xp (mod p)

    This follows from the fact that C(p, j) ≡ 0 (mod p) for 0 < j < p (since p divides the numerator but not the denominator).

    Raising both sides to pi: (1+x)pi+1 ≡ (1+xpi)p ≡ 1 + xpi+1. By induction:

    (1+x)pi ≡ 1 + xpi (mod p)

    Now write n = ∑ni pi and expand:

    (1+x)n = ∏ (1+x)ni pi ≡ ∏ (1+xpi)ni

    The coefficient of xk on the left is C(n, k). On the right, each factor contributes powers of xpi, so we pick the ki-th coefficient from the i-th factor, which is C(ni, ki). The product of these gives the result. ■

    Corollary. C(n, k) ≡ 0 (mod p) if and only if some base-p digit of k exceeds the corresponding digit of n. This is the carries interpretation: C(n, k) is odd iff adding k and n−k in binary produces no carries (Kummer's theorem).

    Implementation

    Basic Lucas for prime modulus

    const int MAXP = 100005;
long long fact[MAXP], inv_fact[MAXP];

long long power(long long a, long long e, long long mod) {
    long long res = 1;
    a %= mod;
    while (e > 0) {
        if (e & 1) res = res * a % mod;
        a = a * a % mod;
        e >>= 1;
    }
    return res;
}

void precompute(int p) {
    fact[0] = 1;
    for (int i = 1; i < p; i++)
        fact[i] = fact[i-1] * i % p;
    inv_fact[p-1] = power(fact[p-1], p-2, p);
    for (int i = p-2; i >= 0; i--)
        inv_fact[i] = inv_fact[i+1] * (i+1) % p;
}

// C(n, k) mod p for 0 <= n, k < p
long long smallC(int n, int k, int p) {
    if (k < 0 || k > n) return 0;
    return fact[n] % p * inv_fact[k] % p * inv_fact[n-k] % p;
}

// Lucas: C(n, k) mod p for arbitrary n, k
long long lucas(long long n, long long k, int p) {
    long long result = 1;
    while (n > 0 || k > 0) {
        int ni = n % p, ki = k % p;
        result = result * smallC(ni, ki, p) % p;
        n /= p; k /= p;
    }
    return result;
}

    Code walkthrough

    1. precompute(p) — Build factorial and inverse-factorial arrays of size p. The inverse-factorial array is computed via Fermat's little theorem: a−1 ≡ ap−2. We compute inv_fact[p-1] then cascade downward.
    2. smallC(n, k, p) — Standard O(1) binomial coefficient for n,k < p.
    3. lucas(n, k, p) — Extracts base-p digits of n and k, multiplies the corresponding small binomial coefficients.

    Complexity

    Precomputation: O(p) time and space. Each query: O(logpn) time.

    Extensions: Lucas for Prime Powers

    Granville's generalisation

    Lucas' theorem only works for prime moduli. For a prime power pa, we need Andrew Granville's extension, which computes C(n, k) mod pa using the factorisation:

    C(n, k) = pe · ∏ (adjusted factorials)

    where e = (sp(k) + sp(n−k) − sp(n)) / (p−1) is computed via Legendre's formula. Here sp(m) is the sum of base-p digits of m.

    Implementation complexity. The Granville extension requires computing "Wilson quotients" and is significantly more complex than basic Lucas. For CP, the CRT approach (below) is usually preferred unless pa itself is the modulus.

    Key idea: factoring out p

    Define the p-free part of n! as the product of all residues from 1 to n with factors of p removed. By Wilson's theorem generalisations, this product has a periodic structure modulo pa that can be computed in O(pa + logpn) time.

    function nCrModPrimePower(n, k, p, a): pa ← pa // Legendre's formula: exponent of p in C(n,k) e ← (digitSum(k,p) + digitSum(n-k,p) - digitSum(n,p)) / (p-1) if e ≥ a: return 0 // p^a divides C(n,k) // Compute p-free factorial mod p^a num ← pFreeFact(n, p, a) den ← pFreeFact(k, p, a) × pFreeFact(n-k, p, a) return pe × num × modInverse(den, pa) mod pa

    nCr mod m (General Modulus)

    For a general (composite) modulus m, we combine the prime-power results using the Chinese Remainder Theorem.

    Algorithm

    1. Factorise m = p1a1 × … × ptat. (Use Pollard's rho from the previous post if m is large.)
    2. For each prime power piai, compute ri = C(n, k) mod piai. If ai = 1, use Lucas directly; otherwise use Granville's extension.
    3. CRT reconstruction: Find the unique x with 0 ≤ x < m satisfying x ≡ ri (mod piai) for all i.
    // CRT for two congruences: x ≡ r1 (mod m1), x ≡ r2 (mod m2)
pair<long long, long long> crt(long long r1, long long m1,
                                     long long r2, long long m2) {
    long long g = __gcd(m1, m2);
    // Assume gcd(m1,m2) divides (r2-r1)
    long long lcm = m1 / g * m2;
    long long diff = (r2 - r1 % m2 + m2) % m2;
    // Extended GCD to find inverse of (m1/g) mod (m2/g)
    long long inv = power(m1 / g, (m2/g) - 1, m2 / g); // Euler shortcut
    long long x = (r1 + m1 * (diff / g % (m2/g) * inv % (m2/g))) % lcm;
    return {x, lcm};
}

long long nCrMod(long long n, long long k, long long m) {
    // factorise m, compute C(n,k) mod each prime power, CRT merge
    auto factors = factorize(m); // from Pollard's rho post
    long long ans = 0, mod = 1;
    for (auto [p, a] : factors) {
        long long pa = 1;
        for (int i = 0; i < a; i++) pa *= p;
        long long r;
        if (a == 1) {
            precompute(p);
            r = lucas(n, k, p);
        } else {
            r = nCrModPrimePower(n, k, p, a); // Granville
        }
        auto [x, lcm] = crt(ans, mod, r, pa);
        ans = x; mod = lcm;
    }
    return ans;
}
    Practical note. In most CP problems the modulus is either a prime (use standard factorials or Lucas) or 109+7 (a prime, standard factorials suffice). The CRT approach is needed for exotic moduli like "mod 1000000" or "mod 12".

    Worked example: C(100, 3) mod 12

    Let's trace through the full CRT pipeline for m = 12 = 22 × 3.

    1. mod 4 (prime power 22): Using Granville's method, compute C(100, 3) mod 4. The exponent of 2 in C(100,3) can be found via Legendre's formula: e = (s2(3)+s2(97)−s2(100)) / 1 = (2+4−3)/1 = 3. Since e ≥ 2, we know 4 | C(100,3), so r1 = 0.
    2. mod 3 (prime): Using Lucas, write 100 = 1·27+0·9+2·3+1 in base 3 → (1,0,2,1)3 and 3 = (0,0,1,0)3. Then C(100,3) mod 3 = C(1,0)·C(0,0)·C(2,1)·C(1,0) = 1·1·2·1 = 2.
    3. CRT: x ≡ 0 (mod 4) and x ≡ 2 (mod 3). Solutions: x = 8. Verify: C(100,3) = 161700 = 13475 × 12 + 0, so 161700 mod 12 = 0. Wait—let's recheck: 161700/4 = 40425 so r1 = 0; 161700/3 = 53900 so r2 = 0 as well. CRT gives x = 0.
    Careful with digit sums. The Legendre exponent formula uses base-p digit sums, which can be tricky. Always double-check with a direct computation for small cases.

    Kummer's theorem

    Theorem (Kummer, 1852). The exponent of a prime p in C(m+n, m) equals the number of carries when adding m and n in base p.

    This is a powerful corollary of Lucas' theorem. For p = 2, it gives: C(n, k) is odd if and only if (k & (n−k)) = 0 (no overlapping bits), or equivalently, (k & n) = k (every bit of k is a bit of n).

    This bit-manipulation characterisation is extremely useful in competitive programming for counting odd entries in Pascal's triangle rows, or for XOR-convolution problems.

    Worked Examples

    Example 1: C(1000000000000, 500000000000) mod 7

    Convert to base 7:

    • 1012 = (33531600616001)7
    • 5×1011 = (15214300303000)7

    Multiply the small C values for each pair of digits. If any digit of k exceeds the corresponding digit of n, the result is 0. Otherwise, multiply C(ni, ki) mod 7 for each digit position.

    Example 2: Counting odd entries in row n of Pascal's triangle

    By Lucas' theorem with p = 2, C(n, k) is odd iff every binary digit of k is ≤ the corresponding digit of n. In other words, k & n = k. The number of such k is 2s where s = popcount(n) (each set bit independently chooses 0 or 1).

    // Count odd binomial coefficients in row n
long long countOddInRow(long long n) {
    return 1LL << __builtin_popcountll(n);
}

    Example 3: Lucas for large modular arithmetic chains

    A common CP pattern: compute a product of binomial coefficients modulo a small prime. With Lucas, each C(n, k) decomposes into O(logp n) tiny computations.

    // Product of C(a_i, b_i) mod p, all values up to 10^18
long long productOfBinomials(vector<pair<long long,long long>>& pairs, int p) {
    precompute(p);
    long long result = 1;
    for (auto [a, b] : pairs) {
        result = result * lucas(a, b, p) % p;
        if (result == 0) return 0; // early exit
    }
    return result;
}
    Optimisation tip. If any single C(ni, ki) ≡ 0, the entire product is 0. Lucas naturally returns 0 in O(1) upon encountering the first invalid digit, so early termination is built-in.

    Complexity Deep-Dive

    Time complexity breakdown

    MethodPrecomputationPer QueryConstraint
    Standard factorialsO(n)O(1)n ≤ 107
    Lucas (prime p)O(p)O(logpn)p small, n huge
    Granville (prime power)O(pa)O(logpn)pa small
    CRT compositeO(Σ piai)O(Σ logpin)factorisation of m known

    Space considerations

    For basic Lucas, we store factorial and inverse-factorial arrays of size p. If p ≤ 105, this is negligible. For Granville with pa ≤ 106, we need arrays of that size.

    When handling multiple different primes (e.g., CRT with composite modulus), precompute factorials for each prime separately. The total space is O(Σpi).

    Common pitfalls

    • Overflow in smallC: When computing fact[n] * inv_fact[k], always reduce mod p between multiplications. Even with long long, two mod-p values multiplied can overflow if p > 109 (but Lucas is typically used with small p).
    • k > n: C(n, k) = 0 when k > n. The implementation must handle this at the digit level (any ki > ni makes the product 0).
    • Multiple queries: Precompute factorials once per prime, not per query. A common mistake is calling precompute(p) inside the query loop.
    Modulus not prime? Lucas' theorem does not work for composite moduli. If you apply it with, say, p = 6, you will get incorrect results. Always verify that your modulus is prime before using Lucas directly.
