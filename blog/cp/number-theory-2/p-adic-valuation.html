    p-adic Valuation

    July 2026 · 30 min read

    Definition of vp

    The p-adic valuation (or p-adic order) of an integer n with respect to a prime p, denoted vp(n), is the largest exponent e such that pe divides n.

    Formal Definition. For a prime p and integer n ≠ 0:
    vp(n) = max{e ≥ 0 : pe | n}
    By convention, vp(0) = +∞.

    Basic Properties

    • Multiplicativity: vp(ab) = vp(a) + vp(b) for all a, b ≠ 0.
    • Non-archimedean triangle inequality: vp(a + b) ≥ min(vp(a), vp(b)), with equality when vp(a) ≠ vp(b).
    • Division: vp(a/b) = vp(a) − vp(b) when b | a.
    • GCD/LCM: vp(gcd(a,b)) = min(vp(a), vp(b)) and vp(lcm(a,b)) = max(vp(a), vp(b)).

    These properties make vp a valuation in the algebraic sense: it turns multiplication into addition and provides an ultrametric structure. The p-adic valuation is the foundation of p-adic number theory, but for competitive programming, we focus on its combinatorial applications.

    Computing vp(n)

    int vp(long long n, int p) {
    int e = 0;
    while (n % p == 0) { n /= p; e++; }
    return e;
}

    Complexity: O(logp n) per call. For n up to 1018 and p = 2, this is at most 60 iterations.

    Legendre’s Formula

    One of the most useful results in elementary number theory: the exact power of a prime p dividing n!.

    Legendre’s Formula. For a prime p and positive integer n:
    vp(n!) = ∑i=1∞ ⌊n / pi⌋ = (n − sp(n)) / (p − 1)
    where sp(n) is the sum of digits of n in base p.

    Proof

    Among the integers 1, 2, …, n:

    • Exactly ⌊n/p⌋ are divisible by p (contributing at least one factor of p each).
    • Exactly ⌊n/p2⌋ are divisible by p2 (contributing a second factor of p each).
    • Exactly ⌊n/p3⌋ are divisible by p3 (contributing a third factor).
    • And so on…

    Each integer m in [1, n] with vp(m) = e is counted exactly e times across these sums (once in ⌊n/p⌋, once in ⌊n/p2⌋, …, once in ⌊n/pe⌋). Therefore the total count equals vp(n!).

    The Digit-Sum Form

    Write n in base p: n = akpk + ak−1pk−1 + … + a1p + a0. Then:

    ⌊n/p⌋ = akpk−1 + … + a1

    ⌊n/p2⌋ = akpk−2 + … + a2

    Summing all these telescopes to (n − sp(n)) / (p − 1), where sp(n) = a0 + a1 + … + ak.

    Classic Application: Trailing Zeros in n!

    The number of trailing zeros of n! in base 10 is v2(n!) and v5(n!) — we take the minimum, which is always v5(n!) since factors of 2 are more abundant:

    long long trailingZeros(long long n) {
    long long count = 0;
    for (long long p = 5; p <= n; p *= 5)
        count += n / p;
    return count;
}

    Complexity: O(log5 n). For n = 1018, only about 26 iterations.

    Generalization. Trailing zeros of n! in base b: compute vp(n!) for each prime factor p of b, then take min(vp(n!) / ep) where b = ∏ pep.

    Kummer’s Theorem

    Kummer’s theorem gives the exact p-adic valuation of binomial coefficients in terms of carries in base-p arithmetic.

    Kummer’s Theorem (1852). For a prime p and non-negative integers m, n:
    vp(C(m+n, m)) = (number of carries when adding m and n in base p)

    Proof Sketch

    Using Legendre’s formula:

    vp(C(m+n, m)) = vp((m+n)!) − vp(m!) − vp(n!)

    = (m+n − sp(m+n)) / (p−1) − (m − sp(m)) / (p−1) − (n − sp(n)) / (p−1)

    = (sp(m) + sp(n) − sp(m+n)) / (p−1)

    Now observe: sp(m) + sp(n) − sp(m+n) equals exactly (p−1) times the number of carries when adding m and n in base p. This is because each carry reduces the digit sum by exactly (p−1): a carry from position i means the digits at position i sum to ≥ p, producing a digit sum contribution of (sum − p) + 1 at positions i and i+1, losing p − 1 in total.

    Immediate Corollary: Lucas’ Theorem

    C(m+n, m) is not divisible by p if and only if there are no carries, i.e., each pair of digits (mi, ni) in base p satisfies mi + ni < p. This is precisely the condition in Lucas’ theorem: C(a, b) ≡ ∏ C(ai, bi) (mod p).

    Example

    Compute v3(C(14, 5)):

    • 14 in base 3: 112 (1·9 + 1·3 + 2)
    • 5 in base 3: 012 (0·9 + 1·3 + 2)
    • 9 in base 3: 100 (1·9 + 0·3 + 0)
    • Adding 012 + 100 in base 3: rightmost digits 2+0=2 (no carry), next 1+0=1 (no carry), next 0+1=1 (no carry). But wait — we need 5 + 9 = 14, and 14 in base 3 is 112. Let’s recheck: the carries approach uses m=5, n=9, m+n=14.
    • 5 = 0123, 9 = 1003. Adding: 2+0=2, 1+0=1, 0+1=1. Result = 1123 = 14. Zero carries.
    • So v3(C(14,5)) = 0. Indeed, C(14,5) = 2002 = 2 · 7 · 11 · 13, not divisible by 3. ✓

    Lifting the Exponent Lemma

    The Lifting the Exponent (LTE) lemma gives vp(an − bn) and vp(an + bn) under certain conditions. It is one of the most powerful tools in olympiad number theory.

    LTE Lemma (p odd, p | a − b, p ∤ a, p ∤ b).
    vp(an − bn) = vp(a − b) + vp(n)

    Proof for p Odd

    Write an − bn = (a − b)(an−1 + an−2b + … + bn−1). The second factor has n terms. Since a ≡ b (mod p), each term an−1−kbk ≡ an−1 (mod p), so the sum ≡ n · an−1 (mod p).

    Since p ∤ a, we have p ∤ an−1, so vp of the second factor equals vp(n). Combined with the first factor, we get vp(an − bn) = vp(a − b) + vp(n).

    The full proof requires showing there are no “unexpected” carries of p-powers between the two factors, which uses the identity:

    an−1 + an−2b + … + bn−1 = ∑k=0n−1 akbn−1−k

    and a careful analysis modulo higher powers of p.

    LTE for p = 2

    Warning: p = 2 is different!
    For p = 2, a and b both odd, and n even:
    v2(an − bn) = v2(a − b) + v2(a + b) + v2(n) − 1

    The extra v2(a + b) term arises because for p = 2, the factorization a2 − b2 = (a−b)(a+b) contributes valuations from both factors, unlike odd primes where only a−b matters.

    LTE for an + bn

    LTE (sum version, p odd, n odd, p | a + b, p ∤ a, p ∤ b).
    vp(an + bn) = vp(a + b) + vp(n)

    This follows by substituting b → −b in the difference version: an + bn = an − (−b)n when n is odd.

    When LTE Applies: Conditions Summary

    vp(an − bn), p odd

    Requires: p | (a−b), p ∤ a, p ∤ b

    Result: vp(a−b) + vp(n)

    vp(an + bn), p odd, n odd

    Requires: p | (a+b), p ∤ a, p ∤ b

    Result: vp(a+b) + vp(n)

    v2(an − bn), n even

    Requires: 2 | (a−b), a,b odd

    Result: v2(a−b) + v2(a+b) + v2(n) − 1

    v2(an − bn), n odd

    Requires: 2 | (a−b), a,b odd

    Result: v2(a−b)

    Implementation

    vp(n!) via Legendre

    long long vpFactorial(long long n, int p) {
    long long result = 0;
    long long pk = p;
    while (pk <= n) {
        result += n / pk;
        if (pk > n / p) break;  // prevent overflow
        pk *= p;
    }
    return result;
}

    vp(C(n, k)) via Kummer’s Carries

    // Count carries when adding k and (n-k) in base p
int vpBinom(long long n, long long k, int p) {
    long long a = k, b = n - k;
    int carries = 0, carry = 0;
    while (a > 0 || b > 0) {
        int da = a % p, db = b % p;
        if (da + db + carry >= p) {
            carries++;
            carry = 1;
        } else {
            carry = 0;
        }
        a /= p; b /= p;
    }
    if (carry) carries++;
    return carries;
}

    Alternatively, using Legendre’s formula directly:

    long long vpBinomLegendre(long long n, long long k, int p) {
    return vpFactorial(n, p) - vpFactorial(k, p) - vpFactorial(n - k, p);
}

    Lifting the Exponent Implementation

    // v_p(a^n - b^n) for odd prime p where p | (a-b), p does not divide a or b
long long lteDiff(long long a, long long b, long long n, int p) {
    assert(p > 2);
    assert((a - b) % p == 0);
    assert(a % p != 0 && b % p != 0);
    return vp(a - b, p) + vp(n, p);
}

// v_2(a^n - b^n) for p=2, a,b odd, n even
long long lteDiff2(long long a, long long b, long long n) {
    assert(a % 2 == 1 && b % 2 == 1);
    assert(n % 2 == 0);
    return vp(a - b, 2) + vp(a + b, 2) + vp(n, 2) - 1;
}

    Digit Sum in Base p

    long long digitSum(long long n, int p) {
    long long s = 0;
    while (n > 0) { s += n % p; n /= p; }
    return s;
}

// Alternative Legendre via digit sum
long long vpFactorial2(long long n, int p) {
    return (n - digitSum(n, p)) / (p - 1);
}

    CP Applications

    Application 1: Divisibility of Factorials

    Problem: Given n and m, does n! divide m!? Equivalently, is vp(n!) ≤ vp(m!) for all primes p?

    Since n ≤ m implies vp(n!) ≤ vp(m!) by Legendre’s formula (each term ⌊n/pi⌋ ≤ ⌊m/pi⌋), the answer is simply “yes iff n ≤ m.” But more interesting: does (n!)k divide m!?

    // Does (n!)^k divide m!?
bool factorialPowerDivides(long long n, long long k, long long m) {
    // Check for all primes p ≤ n: k * v_p(n!) ≤ v_p(m!)
    // Sieve primes up to n, then check Legendre for each
    vector<bool> is_prime(n + 1, true);
    is_prime[0] = is_prime[1] = false;
    for (long long i = 2; i * i <= n; i++)
        if (is_prime[i])
            for (long long j = i*i; j <= n; j += i)
                is_prime[j] = false;
    for (int p = 2; p <= n; p++) {
        if (!is_prime[p]) continue;
        if (k * vpFactorial(n, p) > vpFactorial(m, p))
            return false;
    }
    return true;
}

    Application 2: Largest Power of p Dividing a Product

    Problem: Given an array a[1..n], find vp(a[1] · a[2] · … · a[n]) for a given p.

    By multiplicativity: vp(∏ a[i]) = ∑ vp(a[i]). Just sum the valuations.

    Application 3: When Does pk | C(n, m)?

    Using Kummer’s theorem: C(n, m) is divisible by pk iff adding m and n−m in base p produces ≥ k carries. This characterization is useful when you need to check divisibility for many (n, m) pairs efficiently.

    Application 4: LTE in Olympiad Problems

    Classic Problem: Prove that 3 | (4n − 1) for all n ≥ 1, and find v3(4n − 1).

    By LTE with p = 3, a = 4, b = 1: since 3 | (4 − 1) = 3, and 3 ∤ 4, 3 ∤ 1:

    v3(4n − 1) = v3(4 − 1) + v3(n) = 1 + v3(n)

    Contest Tip. Whenever you see an − bn or an + bn in a divisibility problem, immediately think LTE. It converts the problem from reasoning about exponential expressions to simple valuation arithmetic.

    Application 5: Trailing Zeros in Different Bases

    Problem: How many trailing zeros does n! have when written in base b?

    Factor b = p1e1 · p2e2 · … · pkek. The answer is:

    mini ⌊vpi(n!) / ei⌋

    long long trailingZerosBase(long long n, long long b) {
    long long result = 1e18;
    for (long long p = 2; p * p <= b; p++) {
        if (b % p != 0) continue;
        int e = 0;
        while (b % p == 0) { b /= p; e++; }
        result = min(result, vpFactorial(n, p) / e);
    }
    if (b > 1) result = min(result, vpFactorial(n, b));
    return result;
}

    Application 6: Finding Minimum n with vp(n!) ≥ k

    Problem: Given p and k, find the smallest n such that pk | n!.

    Binary search on n, checking vpFactorial(n, p) ≥ k. Since vpFactorial is non-decreasing, binary search works. The answer is at most k · p (a loose upper bound).

    long long minFactorialDivisible(int p, long long k) {
    long long lo = 0, hi = k * p;
    while (lo < hi) {
        long long mid = (lo + hi) / 2;
        if (vpFactorial(mid, p) >= k) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}
    Complexity. O(log(kp) · logp(kp)) for the binary search, which simplifies to O(log2(kp)).
