    Floor & Ceiling Sums

    July 2026 · 30 min read

    The Divisor Sum Problem

    Consider the sum:

    S(n) = Σi=1n ⌊n/i⌋

    This looks innocent, but it appears constantly in number theory and competitive programming. It equals the total number of divisor pairs — that is, S(n) = Σd=1n d(d) in a different guise — because ⌊n/i⌋ counts how many multiples of i are ≤ n.

    Interpretation. S(n) = Σi=1n ⌊n/i⌋ = Σk=1n d(k) where d(k) is the number of divisors of k. This is because the pair (i, j) with i · j ≤ n is counted once on each side: ⌊n/i⌋ counts the valid j values for each i, and d(k) counts the valid i values for each product k = i·j.

    Naive Approach

    The straightforward O(n) approach:

    long long naiveFloorSum(long long n) {
    long long s = 0;
    for (long long i = 1; i <= n; i++)
        s += n / i;
    return s;
}

    For n = 1012, this is far too slow. We need to exploit the structure of the floor function.

    Key Observation

    The function f(i) = ⌊n/i⌋ is a decreasing staircase function that takes at most O(√n) distinct values. For large i (specifically i > √n), we have ⌊n/i⌋ < √n, so there are at most √n distinct values. For small i ≤ √n, there are trivially at most √n terms. Combined: at most 2√n distinct values.

    Block Decomposition

    The block decomposition (also called the "floor sum trick" or "integer block decomposition") groups consecutive values of i that give the same value of ⌊n/i⌋ into a single block.

    Block Boundaries

    For a given i, let q = ⌊n/i⌋. Then all indices j with ⌊n/j⌋ = q form a contiguous range [i, j_max] where:

    jmax = ⌊n / q⌋ = ⌊n / ⌊n/i⌋⌋

    Proof. If ⌊n/j⌋ = q, then q ≤ n/j < q + 1, which gives n/(q+1) < j ≤ n/q. The largest integer in this range is ⌊n/q⌋. Since q = ⌊n/i⌋, we get jmax = ⌊n/q⌋ = ⌊n/⌊n/i⌋⌋. ∎

    Number of Blocks

    Each block has a distinct value of q = ⌊n/i⌋. As argued above, the number of distinct values is at most 2⌊√n⌋. More precisely:

    • For q ≤ √n, each value of q corresponds to a block. There are ⌊√n⌋ such values.
    • For q > √n, we have i = ⌊n/q⌋ ≤ √n, so each such block corresponds to a single (or few) index i ≤ √n. There are at most ⌊√n⌋ such blocks.

    Total: at most 2⌊√n⌋ blocks, giving an O(√n) algorithm.

    The Algorithm

    function floorSum(n): result ← 0 i ← 1 while i ≤ n: q ← ⌊n / i⌋ j ← ⌊n / q⌋ // last index with same floor value result ← result + q · (j − i + 1) i ← j + 1 // jump to next block return result

    Each iteration processes one block: the value q is constant for indices i through j, so the contribution is q × (j − i + 1).

    Implementation

    // Compute Σ_{i=1}^{n} floor(n/i) in O(√n)
long long floorSum(long long n) {
    long long result = 0;
    for (long long i = 1, j; i <= n; i = j + 1) {
        long long q = n / i;
        j = n / q;  // end of block
        result += q * (j - i + 1);
    }
    return result;
}

    Worked Example

    Let n = 20. The blocks are:

    i range q = ⌊20/i⌋ j = ⌊20/q⌋ count contribution
    1201120
    2102110
    36316
    45415
    5–64528
    7–1021048

    Total: 20 + 10 + 6 + 5 + 8 + 8 + 10 = 66. (Note: the block for q=3 has ⌊20/3⌋ = 6, but i range 4 gives ⌊20/4⌋ = 5 ≠ 6, so that block is just {3}. The block [5,6] unifies i=5 and i=6 since both give ⌊20/i⌋ = 4.)

    Common mistake. When computing j = ⌊n/q⌋, make sure q ≠ 0. This is guaranteed when i ≤ n, but be careful in generalized versions where the range starts at i = 0.

    Generalizations

    Weighted Floor Sum: Σ f(i) · ⌊n/i⌋

    If we need Σi=1n f(i) · ⌊n/i⌋ where f has a computable prefix sum F(x) = Σi=1x f(i), then we can still use block decomposition:

    // Compute Σ_{i=1}^{n} f(i) · floor(n/i) given prefix sum F
long long weightedFloorSum(long long n,
                            function<long long(long long)> F) {
    long long result = 0;
    for (long long i = 1, j; i <= n; i = j + 1) {
        long long q = n / i;
        j = n / q;
        result += q * (F(j) - F(i - 1));
    }
    return result;
}

    Important special cases:

    f(i) = 1

    The basic divisor sum. F(x) = x.

    Computes Σ d(k) for k = 1..n.

    f(i) = i

    Σ i · ⌊n/i⌋. F(x) = x(x+1)/2.

    Computes Σ σ(k) — sum-of-divisors sum.

    f(i) = μ(i)

    Σ μ(i) · ⌊n/i⌋. Requires Mertens function prefix sums.

    Useful in Möbius inversion problems.

    f(i) = φ(i)

    Σ φ(i) · ⌊n/i⌋. F(x) = Σ φ(i) via sieve.

    Counts coprime pairs in range.

    Sum of ⌊n/i⌋²

    For Σ ⌊n/i⌋², the same block decomposition works — within each block, ⌊n/i⌋ is constant at value q, so the contribution is q² · (j − i + 1):

    long long floorSumSquared(long long n) {
    long long result = 0;
    for (long long i = 1, j; i <= n; i = j + 1) {
        long long q = n / i;
        j = n / q;
        result += q * q * (j - i + 1);
    }
    return result;
}

    Two-Variable Floor Sums

    A harder generalization: Σi=1n ⌊(ai + b)/m⌋ for constants a, b, m. This is the floor sum computed by the famous "floor sum" algorithm (related to the Euclidean algorithm). It runs in O(log m) time:

    // Compute Σ_{i=0}^{n-1} floor((a·i + b) / m)
// Based on the algorithm from competitive programming references
long long floorSumGeneral(long long n, long long a,
                          long long b, long long m) {
    long long ans = 0;
    if (a >= m) {
        ans += n * (n - 1) / 2 * (a / m);
        a %= m;
    }
    if (b >= m) {
        ans += n * (b / m);
        b %= m;
    }
    long long y_max = (a * n + b) / m;
    if (y_max == 0) return ans;
    long long x_max = y_max * m - b;
    ans += (n - 1) * y_max;
    ans -= floorSumGeneral(y_max, m, m - b - 1 + a, a);
    return ans;
}
// WARNING: ensure a, b, m > 0 and handle edge cases for n = 0.
    Complexity. The recursive floorSumGeneral mirrors the Euclidean algorithm: each recursive call reduces the parameters analogously to the GCD computation, giving O(log(max(a, m))) time.

    Implementation

    Template for Block Decomposition

    Here is a reusable C++ template that handles the core pattern:

    // Block decomposition template for floor(n/i)
// Calls process(q, lo, hi) for each block where floor(n/i) = q for i ∈ [lo, hi]
template<typename F>
void floorBlocks(long long n, F process) {
    for (long long i = 1, j; i <= n; i = j + 1) {
        long long q = n / i;
        j = n / q;
        process(q, i, j);
    }
}

// Usage: count divisors sum
long long divisorSum(long long n) {
    long long ans = 0;
    floorBlocks(n, [&](long long q, long long lo, long long hi) {
        ans += q * (hi - lo + 1);
    });
    return ans;
}

// Usage: sum of sigma(k) for k = 1..n
long long sigmaSumTotal(long long n) {
    long long ans = 0;
    floorBlocks(n, [&](long long q, long long lo, long long hi) {
        // Σ_{i=lo}^{hi} i = (hi*(hi+1)/2) - ((lo-1)*lo/2)
        long long sumI = hi*(hi+1)/2 - (lo-1)*lo/2;
        ans += q * sumI;
    });
    return ans;
}

    Modular Version

    // Floor sum mod p
const long long MOD = 1e9 + 7;

long long floorSumMod(long long n) {
    long long result = 0;
    for (long long i = 1, j; i <= n; i = j + 1) {
        long long q = n / i;
        j = n / q;
        long long cnt = (j - i + 1) % MOD;
        result = (result + (q % MOD) * cnt % MOD) % MOD;
    }
    return result;
}

    Connections to Number Theory

    Dirichlet Hyperbola Method

    The Dirichlet hyperbola method computes sums of the form Σn≤N (f * g)(n) where * denotes Dirichlet convolution and f, g are arithmetic functions. The key identity is:

    Σn≤N (f*g)(n) = Σd≤U f(d) · G(⌊N/d⌋) + Σe≤V g(e) · F(⌊N/e⌋) − F(U) · G(V)

    where U · V = N (typically U = V = √N), F and G are prefix sums of f and g.

    Application: counting divisors. Setting f = g = 1 (the constant function), (f*g)(n) = d(n) (divisor count), and F(x) = G(x) = x. The formula becomes:

    Σk≤N d(k) = 2 · Σd≤√N ⌊N/d⌋ − (⌊√N⌋)²

    This reduces the floor sum to only √N terms (instead of N), and combined with block decomposition, gives an O(N1/3) algorithm in some formulations.

    Euler's Totient Summatory Function

    The sum Φ(N) = Σk=1N φ(k) can be computed using the identity:

    Σd|n φ(d) = n  ⟹  Σk=1N k = Σd=1N φ(d) · ⌊N/d⌋

    Rearranging: Φ(N) = N(N+1)/2 − Σd=2N Φ(⌊N/d⌋). With memoization and block decomposition, this gives an O(N2/3) algorithm for computing Φ(N).

    // Compute Σ φ(k) for k=1..n using the hyperbola method
using ll = long long;
unordered_map<ll, ll> n2_memo;
vector<ll> n2_phi_prefix;  // precomputed for small values

ll totientSum(ll n) {
    if (n < (ll)n2_phi_prefix.size())
        return n2_phi_prefix[n];
    auto it = n2_memo.find(n);
    if (it != n2_memo.end()) return it->second;

    ll result = n * (n + 1) / 2;
    for (ll i = 2, j; i <= n; i = j + 1) {
        ll q = n / i;
        j = n / q;
        result -= (j - i + 1) * totientSum(q);
    }
    return n2_memo[n] = result;
}

    Counting Lattice Points Under a Hyperbola

    The sum Σ ⌊n/i⌋ counts lattice points (i, j) with 1 ≤ i, j and ij ≤ n — i.e., lattice points in the first quadrant under the hyperbola xy = n. This geometric interpretation connects floor sums to problems in analytic number theory.

    Problem Template: GCD Pair Counting

    Problem. Count pairs (i, j) with 1 ≤ i < j ≤ N and gcd(i, j) = 1.

    Solution. The answer is (Φ(N) − 1) / 1 where Φ(N) = Σ φ(k)… wait, more precisely:

    coprimePairs(N): // Number of pairs with gcd = 1 is (2·Φ(N) - 1) / 2 // where Φ(N) = Σ_{k=1}^{N} φ(k) // But we can also use Möbius inversion: count ← 0 for d ← 1 to N: count ← count + μ(d) · ⌊N/d⌋ · (⌊N/d⌋ − 1) / 2 return count // Optimize with block decomposition: O(√N) with precomputed prefix μ
    Summary of complexities.
    • Σ ⌊n/i⌋: O(√n) via block decomposition
    • Σ f(i)⌊n/i⌋: O(√n) if prefix sums of f are O(1)
    • Σ ⌊(ai+b)/m⌋: O(log m) via Euclidean-like recursion
    • Σ d(k) for k≤N: O(√N) via hyperbola method
    • Σ φ(k) for k≤N: O(N^{2/3}) with sieve + memoization
