    Continued Fractions

    July 2026 · 35 min read

    Simple Continued Fractions

    A simple continued fraction (SCF) is an expression of the form:

    α = a₀ + 1/(a₁ + 1/(a₂ + 1/(a₃ + ⋯)))

    We write this compactly as [a₀; a₁, a₂, a₃, …] where a₀ is an integer (possibly zero or negative) and a₁, a₂, … ≥ 1 are positive integers called partial quotients.

    Fundamental dichotomy.
    • Finite CF: [a₀; a₁, …, aₙ] — represents a rational number. Every rational has exactly two CF representations: one ending in aₙ ≥ 2 and one ending in (aₙ − 1), 1.
    • Infinite CF: [a₀; a₁, a₂, …] — represents an irrational number. The correspondence is bijective.

    Examples

    Number CF
    22/7[3; 7]
    355/113[3; 7, 16]
    √2[1; 2, 2, 2, …] = [1; 2̄]
    √3[1; 1, 2, 1, 2, …] = [1; 1, 2̄]
    φ = (1+√5)/2[1; 1, 1, 1, …] = [1; 1̄]
    e[2; 1, 2, 1, 1, 4, 1, 1, 6, …]
    π[3; 7, 15, 1, 292, …]

    The golden ratio φ has the simplest possible infinite CF — all partial quotients equal to 1. This makes it the "hardest" number to approximate by rationals, a fact we'll quantify shortly.

    Computing a CF from a Rational

    For a rational p/q, the Euclidean algorithm directly yields the CF:

    function rationalCF(p, q): coeffs ← [] while q ≠ 0: a ← ⌊p / q⌋ coeffs.append(a) p, q ← q, p − a · q return coeffs

    For example, 67/29: 67 = 2·29 + 9, 29 = 3·9 + 2, 9 = 4·2 + 1, 2 = 2·1. So 67/29 = [2; 3, 4, 2].

    Computing Convergents

    The k-th convergent of the CF [a₀; a₁, a₂, …] is the rational number obtained by truncating at the k-th term:

    C₀ = a₀/1,   C₁ = (a₁ · a₀ + 1)/a₁,   Cₖ = hₖ/kₖ

    The Fundamental Recurrence

    The numerators hₖ and denominators kₖ satisfy:

    h−1 = 1,   h₀ = a₀,   hₖ = aₖ · hk−1 + hk−2
    k−1 = 0,   k₀ = 1,   kₖ = aₖ · kk−1 + kk−2

    Key identity. For all k ≥ 0:
    hk · kk−1 − hk−1 · kk = (−1)k−1
    This implies that consecutive convergents are always in lowest terms and satisfy gcd(hₖ, kₖ) = 1.

    Matrix Interpretation

    The recurrence can be expressed as a matrix product:

    ┌ hₖ h_{k-1} ┐ ┌ a₀ 1 ┐ ┌ a₁ 1 ┐ ┌ aₖ 1 ┐ │ │ = │ │ · │ │ · ··· · │ │ └ kₖ k_{k-1} ┘ └ 1 0 ┘ └ 1 0 ┘ └ 1 0 ┘

    This matrix form is computationally useful: to find the convergent at position k, we can use matrix exponentiation if the partial quotients follow a pattern (as with periodic CFs).

    Convergent Properties

    • Even-indexed convergents C₀, C₂, C₄, … form an increasing sequence approaching α from below.
    • Odd-indexed convergents C₁, C₃, C₅, … form a decreasing sequence approaching α from above.
    • Every even convergent is less than every odd convergent.
    • The denominators satisfy kₖ ≥ Fk+1 where F is the Fibonacci sequence. In particular, kₖ grows at least exponentially.
    // Compute convergents of [a0; a1, a2, ..., an]
using ll = long long;

vector<pair<ll,ll>> convergents(const vector<ll>& a) {
    vector<pair<ll,ll>> conv;
    ll h_prev = 1, h_curr = a[0];
    ll k_prev = 0, k_curr = 1;
    conv.push_back({h_curr, k_curr});
    for (int i = 1; i < (int)a.size(); i++) {
        ll h_next = a[i] * h_curr + h_prev;
        ll k_next = a[i] * k_curr + k_prev;
        conv.push_back({h_next, k_next});
        h_prev = h_curr; h_curr = h_next;
        k_prev = k_curr; k_curr = k_next;
    }
    return conv;
}

    Best Rational Approximation

    Convergents give the best rational approximations in a precise sense. There are two notions:

    Best Approximation of the 1st Kind

    p/q is a best approx. of the 1st kind to α if no fraction with smaller denominator is closer: for all p'/q' with 0 < q' < q, |α − p/q| < |α − p'/q'|.

    Best Approximation of the 2nd Kind

    p/q is a best approx. of the 2nd kind to α if: for all p'/q' with 0 < q' ≤ q, |q · α − p| ≤ |q' · α − p'|, with strict inequality when q' < q.

    Theorem. Every convergent of α is a best approximation of the 2nd kind. Conversely, every best approximation of the 2nd kind is a convergent. Best approximations of the 1st kind are either convergents or intermediate fractions (mediants formed during the CF expansion).

    Approximation Quality

    The error in approximating α by its k-th convergent satisfies:

    1/(kk(kk+1 + kk)) < |α − hk/kk| < 1/(kk · kk+1)

    Since kk+1 ≥ ak+1 · kk, a large next partial quotient means an exceptionally good approximation. For π, the partial quotient 292 after [3; 7, 15, 1] explains why 355/113 is such an excellent approximation (error < 3 × 10⁻⁷).

    The Hurwitz Bound

    For any irrational α, infinitely many convergents satisfy:

    |α − hₖ/kₖ| < 1/(√5 · kₖ²)

    The constant √5 is best possible — it is achieved exactly by the golden ratio φ. This is Hurwitz's theorem, and it quantifies why the golden ratio is the "most irrational" number.

    Euclidean Algorithm Connection

    The Euclidean algorithm for computing gcd(a, b) is exactly the CF expansion of a/b. Each step of the algorithm produces one partial quotient:

    // Euclidean algorithm = CF expansion of a/b
// gcd(67, 29):
//   67 = 2·29 + 9    →  a₀ = 2
//   29 = 3·9  + 2    →  a₁ = 3
//   9  = 4·2  + 1    →  a₂ = 4
//   2  = 2·1  + 0    →  a₃ = 2
// CF of 67/29 = [2; 3, 4, 2]
// gcd = 1

    Lamé's Theorem

    Lamé's Theorem (1844). The number of steps in the Euclidean algorithm for gcd(a, b) with a > b ≥ 1 is at most 5 · log₁₀(b) + 1. Equivalently, the worst case occurs when a and b are consecutive Fibonacci numbers, giving exactly ⌊logφ(b√5)⌋ steps.

    This is historically the first non-trivial complexity bound for any algorithm. The proof follows directly from the CF perspective: each partial quotient is at least 1, so the denominators of convergents grow at least as fast as Fibonacci numbers.

    Extended GCD via Convergents

    The extended Euclidean algorithm finds x, y such that ax + by = gcd(a, b). These coefficients are exactly the convergent numerators/denominators: if a/b = [a₀; a₁, …, aₙ], then x = (−1)n−1 kn−1 and y = (−1)n hn−1 (up to signs depending on the gcd value).

    Quadratic Irrationals

    A quadratic irrational is a number of the form (a + b√D)/c where a, b, c are integers and D is a positive non-square integer.

    Lagrange's Theorem. A real number has a periodic continued fraction expansion if and only if it is a quadratic irrational. Furthermore:
    • The CF is purely periodic (period starts at a₀) iff the quadratic irrational is a reduced quadratic irrational (greater than 1 with conjugate between −1 and 0).
    • The CF of √D always has the form [a₀; a₁, a₂, …, a₂, a₁, 2a₀] — the period is palindromic except for the last term, which is 2a₀.

    Computing the CF of √D

    We use the standard algorithm. Define the "complete quotients" αk = (mk + √D) / dk with:

    function sqrtCF(D): a₀ ← ⌊√D⌋ period ← [] m ← 0, d ← 1 loop: m ← a · d − m d ← (D − m²) / d // always divides exactly a ← ⌊(a₀ + m) / d⌋ period.append(a) if a = 2 · a₀: break // period complete return (a₀, period)

    Examples of √D Expansions

    D CF Period Length
    2[1; 2̄]1
    3[1; 1, 2̄]2
    5[2; 4̄]1
    7[2; 1, 1, 1, 4̄]4
    13[3; 1, 1, 1, 1, 6̄]5
    19[4; 2, 1, 3, 1, 2, 8̄]6
    23[4; 1, 3, 1, 8̄]4

    Notice the palindromic structure: for D = 7, the period is [1, 1, 1, 4] — ignoring the last element (4 = 2·2), the sequence [1, 1, 1] is a palindrome. For D = 19, the period is [2, 1, 3, 1, 2, 8] — the core [2, 1, 3, 1, 2] is a palindrome.

    Implementation

    Full CF Expansion and Convergent Computation

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;

struct CFExpansion {
    ll a0;
    vector<ll> period;
};

// CF expansion of √D
CFExpansion sqrtCF(ll D) {
    ll a0 = (ll)sqrtl(D);
    if (a0 * a0 == D) return {a0, {}};  // perfect square

    CFExpansion cf;
    cf.a0 = a0;
    ll m = 0, d = 1, a = a0;
    do {
        m = a * d - m;
        d = (D - m * m) / d;
        a = (a0 + m) / d;
        cf.period.push_back(a);
    } while (a != 2 * a0);
    return cf;
}

// Compute convergent h_k / k_k from CF coefficients
pair<ll,ll> convergent(const vector<ll>& a, int k) {
    ll h_prev = 1, h_curr = a[0];
    ll k_prev = 0, k_curr = 1;
    for (int i = 1; i <= k; i++) {
        ll h_next = a[i] * h_curr + h_prev;
        ll k_next = a[i] * k_curr + k_prev;
        h_prev = h_curr; h_curr = h_next;
        k_prev = k_curr; k_curr = k_next;
    }
    return {h_curr, k_curr};
}

    Rational CF and Reconstruction

    // CF expansion of rational p/q
vector<ll> rationalCF(ll p, ll q) {
    vector<ll> cf;
    while (q != 0) {
        cf.push_back(p / q);
        ll tmp = p % q;
        p = q;
        q = tmp;
    }
    return cf;
}

// Reconstruct rational from CF
pair<ll,ll> fromCF(const vector<ll>& cf) {
    ll h = 0, k = 1;
    for (int i = cf.size() - 1; i >= 0; i--) {
        // h/k = cf[i] + k/h (but swap h,k first)
        swap(h, k);
        h += cf[i] * k;
    }
    return {h, k};
}

    Best Approximation with Bounded Denominator

    // Find best rational p/q ≈ a/b with q ≤ maxQ
pair<ll,ll> bestApprox(ll a, ll b, ll maxQ) {
    vector<ll> cf = rationalCF(a, b);
    ll h_prev = 1, h_curr = cf[0];
    ll k_prev = 0, k_curr = 1;
    pair<ll,ll> best = {h_curr, k_curr};

    for (int i = 1; i < (int)cf.size(); i++) {
        // Try the full convergent
        ll h_next = cf[i] * h_curr + h_prev;
        ll k_next = cf[i] * k_curr + k_prev;
        if (k_next > maxQ) {
            // Use largest valid intermediate: t = ⌊(maxQ - k_prev) / k_curr⌋
            ll t = (maxQ - k_prev) / k_curr;
            if (t >= 1) {
                ll h_int = t * h_curr + h_prev;
                ll k_int = t * k_curr + k_prev;
                // Check if this is actually closer than current best
                if (abs(h_int * b - a * k_int) * best.second <
                    abs(best.first * b - a * best.second) * k_int)
                    best = {h_int, k_int};
            }
            break;
        }
        best = {h_next, k_next};
        h_prev = h_curr; h_curr = h_next;
        k_prev = k_curr; k_curr = k_next;
    }
    return best;
}

    CP Applications

    Problem 1: Fraction Reconstruction

    Problem. You are given a decimal approximation of a fraction p/q where q ≤ N. Recover p and q.

    Solution. Compute the CF expansion of the decimal, and check each convergent. The first convergent with denominator ≤ N that matches the decimal to sufficient precision is the answer.

    Problem 2: Approximation with Bounded Denominator

    Problem (Codeforces). Given a fraction a/b and an upper bound Q, find p/q with q ≤ Q minimizing |a/b − p/q|.

    Solution. Use the bestApprox function above. The key insight: we scan convergents until the denominator exceeds Q, then use a partial convergent (semi-convergent) with the largest valid coefficient.

    Semi-convergents. Between convergent hₖ/kₖ and hk+1/kk+1, the semi-convergents (t · hₖ + hk−1) / (t · kₖ + kk−1) for t = 1, 2, …, ak+1 − 1 are also best approximations of the 1st kind (though not always of the 2nd kind).

    Problem 3: Stern-Brocot Search

    The Stern-Brocot tree is intimately connected to CFs. The path from the root to p/q in the Stern-Brocot tree encodes the CF: going Left a₀ times, Right a₁ times, Left a₂ times, etc. corresponds to the expansion [0; a₀, a₁, a₂, …] (or with the initial integer part if p/q ≥ 1).

    Problem 4: Beating Floating Point

    In some contest problems, you need to compare a/b vs c/d without floating-point error. If a, b, c, d are huge (128-bit), compute the CFs of both fractions and compare lexicographically. This gives exact comparison in O(log(min(b, d))) time without big-integer multiplication.

    Implementation detail. When comparing CFs lexicographically, remember that odd-indexed partial quotients have reversed comparison direction. That is, [a₀; a₁, a₂, …] < [a₀; a₁, b₂, …] iff a₂ < b₂ when the differing position is even, but a₂ > b₂ when it's odd (because deeper levels alternate the direction of the inequality).
