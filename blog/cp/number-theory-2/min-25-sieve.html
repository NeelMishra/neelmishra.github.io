    Min-25 Sieve

    July 2026 · 40 min read

    The Problem

    We want to compute F(n) = ∑i=1n f(i) where f is a multiplicative function and n can be as large as 1010 or even 1013. The linear sieve computes all values in O(n), which is infeasible for such large n. The Min-25 sieve (named after its inventor’s handle on Project Euler) computes F(n) in O(n2/3 / log n) time using O(√n) space.

    Requirements on f. The Min-25 sieve requires: (1) f is multiplicative, (2) f(p) can be expressed as a polynomial in p (e.g., f(p) = p − 1 for φ), (3) f(pe) can be computed quickly for each prime power.

    Motivating Examples

    • Prime counting: π(n) = ∑p ≤ n 1. Here f(p) = 1 (constant polynomial).
    • Sum of primes: ∑p ≤ n p. Here f(p) = p.
    • ∑φ(i): φ(p) = p − 1. Polynomial in p of degree 1.
    • ∑μ2(i): μ2(p) = 1. Requires a different approach since μ2 is not easily handled by Min-25.

    Key Idea: Prime Sums

    The Min-25 sieve operates in two phases. The first phase computes sums of f over primes; the second phase extends to all integers.

    Notation

    Let p1 = 2, p2 = 3, p3 = 5, … be the primes in increasing order. Define:

    Fprime(v, j) = ∑ fpoly(i) for 2 ≤ i ≤ v where i is either prime or has smallest prime factor > pj.

    Here fpoly is the polynomial that agrees with f on primes. For example, if f = φ, then fpoly(x) = x − 1, even though this polynomial is applied to composite numbers too during the sieve.

    Base Case and Transition

    Base case (j = 0): Fprime(v, 0) = ∑i=2v fpoly(i). This is a sum over all integers from 2 to v.

    Transition: Going from j−1 to j, we remove composite numbers whose smallest prime factor is pj:

    Fprime(v, j) = Fprime(v, j−1) − fpoly(pj) · [Fprime(v/pj, j−1) − Fprime(pj−1, j−1)]

    The term in brackets represents the sum over numbers whose smallest prime factor is ≥ pj (excluding the primes smaller than pj).

    When to stop. The sieve only runs for j up to π(√n), since any remaining composite ≤ n must have a prime factor ≤ √n. After sieving out all primes p ≤ √n, only actual primes remain.

    Phase 1: Computing Prime Sums

    When fpoly(x) = xk, the base case is a power sum: Fprime(v, 0) = ∑i=2v ik. For k=0, this is v − 1. For k=1, this is v(v+1)/2 − 1.

    Algorithm

    // Phase 1: compute Fp[v] = F_prime(v, pi(sqrt(n))) for all relevant v // Start with Fp[v] = sum_{i=2}^{v} f_poly(i) [base case] for j = 1 to pi(sqrt(n)): p = primes[j] for each relevant v where v >= p*p: Fp[v] -= f_poly(p) * (Fp[v/p] - Fp[p-1])

    The condition v ≥ p2 is crucial: if v < p2, there are no composites with smallest prime factor p that are ≤ v.

    The ⌊n/i⌋ Trick

    We only need Fprime(v, ·) for values v of the form ⌊n/i⌋ for some integer i. There are only O(√n) such distinct values. We store them in two arrays:

    • small[v] for v ≤ √n (indexed directly by v)
    • large[i] for v = ⌊n/i⌋ where i ≤ √n

    To look up Fprime(v): if v ≤ √n, use small[v]; otherwise v = ⌊n/i⌋ for some i ≤ √n, so use large[i].

    C++ Phase 1 for Sum of Primes

    long long n;
int sqn;
vector<long long> smalls, larges;
vector<int> primes;

auto idx(long long v) {
    return v <= sqn ? v : sqn + n / v;
}

// Fp[v] = sum of primes ≤ v  (Phase 1 result)
void phase1() {
    sqn = (int)sqrt((double)n);
    int sz = sqn + sqn + 2;
    vector<long long> Fp(sz), Fc(sz);

    // Base case: Fp[v] = v*(v+1)/2 - 1, Fc[v] = v - 1
    for (long long i = 1; i <= sqn; i++) {
        Fp[idx(i)] = i * (i + 1) / 2 - 1;
        Fc[idx(i)] = i - 1;
        long long v = n / i;
        Fp[idx(v)] = v * (v + 1) / 2 - 1;
        Fc[idx(v)] = v - 1;
    }

    // Sieve: remove composites with smallest prime factor = p
    for (int p : primes) {
        if ((long long)p * p > n) break;
        long long p2 = (long long)p * p;
        long long fp_prev = Fp[idx(p - 1)];
        long long fc_prev = Fc[idx(p - 1)];

        // Update large values first, then small
        for (long long i = 1; i <= sqn && n / i >= p2; i++) {
            long long v = n / i;
            Fp[idx(v)] -= p * (Fp[idx(v / p)] - fp_prev);
            Fc[idx(v)] -= (Fc[idx(v / p)] - fc_prev);
        }
        for (long long v = sqn; v >= p2; v--) {
            Fp[idx(v)] -= p * (Fp[idx(v / p)] - fp_prev);
            Fc[idx(v)] -= (Fc[idx(v / p)] - fc_prev);
        }
    }
}

    Phase 2: Extending to All Integers

    Phase 1 gives us the sum of f over primes. Phase 2 extends to all integers by adding contributions from prime powers and composites.

    Definition

    Define S(v, j) = ∑ f(i) for 2 ≤ i ≤ v where the smallest prime factor of i is > pj, plus the sum over primes > pj that are ≤ v.

    Recurrence

    S(v, j) = [Fprime(v) − Fprime(pj)] + ∑k > j ∑e ≥ 1, pke ≤ v f(pke) · [S(v / pke, k) + [e > 1]]

    The first term accounts for primes. The double sum accounts for composites: for each prime pk and exponent e, the contribution is f(pke) times the sum over the remaining part (which has smallest prime factor > pk). The [e > 1] term handles pure prime powers pke (which contribute f(pke) by themselves when the cofactor is 1).

    Final Answer

    F(n) = S(n, 0) + f(1) = S(n, 0) + 1.

    Recursive Implementation

    // Phase 2: recursive computation
// Fp[] and Fc[] from Phase 1 are available globally
long long S(long long v, int j) {
    // Sum of f(i) for i in (p_j, v] where i is prime
    long long res = Fp[idx(v)] - Fp[idx(primes[j])];

    for (int k = j + 1; k < (int)primes.size(); k++) {
        long long p = primes[k];
        if (p * p > v) break;

        long long pe = p;
        for (int e = 1; pe <= v; e++, pe *= p) {
            long long fpe = f_prime_power(p, e);
            if (e >= 2)
                res += fpe;  // pure prime power
            if (pe * p <= v)
                res += fpe * S(v / pe, k);
        }
    }
    return res;
}
    Overflow alert. When computing pe *= p, check for overflow before the multiplication. Use pe ≤ v / p instead of pe * p ≤ v to avoid 64-bit overflow.

    Index Compression

    The key to O(√n) space is observing that the only values of v we ever query are of the form ⌊n/i⌋. We use a simple indexing function: 

    // Map v to array index in O(1)
int idx(long long v) {
    if (v <= sqn)
        return (int)v;
    else
        return sqn + (int)(n / v);
}

    Proof of Correctness

    For v ≤ √n, we store directly at index v. For v > √n, we have v = ⌊n/i⌋ for some i ≤ √n, and we store at index √n + i. Since all small v's map to [1, √n] and all large v's map to [√n+1, 2√n], these ranges don't overlap. Total array size: 2√n + O(1).

    Distinct Values of ⌊n/i⌋

    Lemma. The set {⌊n/i⌋ : 1 ≤ i ≤ n} has at most 2⌊√n⌋ distinct elements.

    Proof. For i ≤ √n, there are at most √n values of ⌊n/i⌋. For i > √n, ⌊n/i⌋ < √n, so at most √n distinct values. Total ≤ 2√n. □

    Implementation

    Complete C++ Min-25 Sieve: Prime Counting

    The simplest application: compute π(n), the number of primes ≤ n. 

    #include <bits/stdc++.h>
using namespace std;

long long n;
int sqn;
vector<long long> vals;
vector<long long> Fc;  // count of "primes" after sieving
vector<int> primes;

int id(long long v) {
    return v <= sqn ? (int)v : sqn + (int)(n / v);
}

long long prime_count(long long N) {
    n = N;
    sqn = (int)sqrt((double)n);
    while ((long long)(sqn + 1) * (sqn + 1) <= n) sqn++;

    int sz = 2 * sqn + 2;
    Fc.assign(sz, 0);

    // Generate small primes up to sqrt(n)
    primes.clear();
    vector<bool> is_p(sqn + 1, true);
    for (int i = 2; i <= sqn; i++) {
        if (is_p[i]) {
            primes.push_back(i);
            for (long long j = (long long)i * i; j <= sqn; j += i)
                is_p[j] = false;
        }
    }

    // Base case: Fc[v] = v - 1  (all integers 2..v)
    for (long long i = 1; i <= sqn; i++) {
        Fc[id(i)] = i - 1;
        Fc[id(n / i)] = n / i - 1;
    }

    // Sieve out composites
    for (int p : primes) {
        long long p2 = (long long)p * p;
        if (p2 > n) break;
        long long prev = Fc[id(p - 1)];

        for (long long i = 1; i <= sqn && n / i >= p2; i++)
            Fc[id(n / i)] -= Fc[id(n / i / p)] - prev;
        for (long long v = sqn; v >= p2; v--)
            Fc[id(v)] -= Fc[id(v / p)] - prev;
    }

    return Fc[id(n)];
}

int main() {
    long long n;
    cin >> n;
    cout << prime_count(n) << "\n";
}

    Complete C++ Min-25 Sieve: Sum of Euler Totient

    // Compute sum of phi(i) for i = 1..n using Min-25 sieve
// phi(p) = p - 1, phi(p^e) = p^{e-1}(p-1)

long long Fp_sum[MAXSZ], Fp_cnt[MAXSZ];  // sum of p, count

void phase1_phi() {
    // Base: Fp_sum[v] = v(v+1)/2 - 1, Fp_cnt[v] = v - 1
    for (long long i = 1; i <= sqn; i++) {
        Fp_sum[id(i)] = i * (i + 1) / 2 - 1;
        Fp_cnt[id(i)] = i - 1;
        long long v = n / i;
        Fp_sum[id(v)] = v * (v + 1) / 2 - 1;
        Fp_cnt[id(v)] = v - 1;
    }

    for (int p : primes) {
        if ((long long)p * p > n) break;
        long long sp = Fp_sum[id(p - 1)];
        long long cp = Fp_cnt[id(p - 1)];

        for (long long i = 1; i <= sqn && n / i >= (long long)p * p; i++) {
            long long v = n / i;
            Fp_sum[id(v)] -= p * (Fp_sum[id(v / p)] - sp);
            Fp_cnt[id(v)] -= (Fp_cnt[id(v / p)] - cp);
        }
        for (long long v = sqn; v >= (long long)p * p; v--) {
            Fp_sum[id(v)] -= p * (Fp_sum[id(v / p)] - sp);
            Fp_cnt[id(v)] -= (Fp_cnt[id(v / p)] - cp);
        }
    }
    // After phase 1: Fp_sum[v] = sum of primes ≤ v
    //                Fp_cnt[v] = count of primes ≤ v
    // phi at primes: sum of phi(p) = Fp_sum[v] - Fp_cnt[v]
}

long long S_phi(long long v, int j) {
    // sum of phi(p) for primes p in (p_j, v]
    long long res = (Fp_sum[id(v)] - Fp_cnt[id(v)])
                  - (Fp_sum[id(primes[j])] - Fp_cnt[id(primes[j])]);

    for (int k = j + 1; k < (int)primes.size(); k++) {
        long long p = primes[k];
        if (p * p > v) break;

        long long pe = p, ppow = 1;
        for (int e = 1; pe <= v; e++) {
            // phi(p^e) = p^{e-1}(p-1)
            long long phi_pe = ppow * (p - 1);
            if (e >= 2)
                res += phi_pe;
            if (pe <= v / p)
                res += phi_pe * S_phi(v / pe, k);
            ppow = pe;
            pe = pe <= v / p ? pe * p : v + 1;
        }
    }
    return res;
}

    Complexity Analysis

    Phase 1 Complexity

    Phase 1 iterates over primes p ≤ √n and for each, over O(√n) values of v. The total number of operations is:

    ∑p ≤ √n O(√n / p) = O(√n · ∑p ≤ √n 1/p) = O(√n · log log √n)

    Using the Mertens estimate ∑ 1/p ≈ log log x. More careful analysis using partial summation gives O(n2/3 / log n) as the tight bound.

    Tight bound proof sketch. The number of (p, v) pairs where p2 ≤ v and v is of the form ⌊n/i⌋ is Θ(n2/3 / log n). This follows from summing π(√v) over all O(√n) values of v, applying the Prime Number Theorem, and evaluating the resulting integral.

    Phase 2 Complexity

    Phase 2 visits each integer m ≤ n exactly once through its canonical representation m = pke · (m / pke). The recursion tree has O(n / log n) leaves (the primes) plus the composites. The total work in Phase 2 is O(n1−ε) for any ε > 0, dominated by Phase 1.

    Space Complexity

    The arrays Fp, Fc use O(√n) entries each. Phase 2 uses O(log n) recursion stack depth. Total space: O(√n).

    Min-25 Sieve

    Time: O(n2/3 / log n)
    Space: O(√n)
    Handles n up to 1013

    Linear Sieve

    Time: O(n)
    Space: O(n)
    Handles n up to 108

    Lucy DP (primes only)

    Time: O(n3/4 / log n)
    Space: O(√n)
    Only gives prime sums
