    Smooth Numbers

    July 2026 · 25 min read

    Definition & Examples

    A positive integer n is called B-smooth (or y-smooth) if every prime factor of n is at most B. Formally:

    Definition. An integer n ≥ 1 is B-smooth if P+(n) ≤ B, where P+(n) denotes the largest prime factor of n. By convention, P+(1) = 1.

    Some immediate examples make the definition concrete:

    • 2-smooth numbers are exactly the powers of 2: 1, 2, 4, 8, 16, 32, … These are sometimes called regular numbers in certain contexts.
    • 3-smooth numbers have the form 2a · 3b: 1, 2, 3, 4, 6, 8, 9, 12, 16, 18, 24, …
    • 5-smooth numbers (also called Hamming numbers) have the form 2a · 3b · 5c: 1, 2, 3, 4, 5, 6, 8, 9, 10, 12, 15, 16, …
    • 7-smooth numbers are also known as humble numbers.

    The concept is central to computational number theory: the running time of the most powerful integer factorization algorithms (quadratic sieve, number field sieve) depends critically on how many smooth numbers fall in certain ranges.

    The term friable (French: easily crumbled) is used synonymously with “smooth” in much of the analytic number theory literature, especially in work by Hildebrand, Tenenbaum, and de la Bretèche.

    Key Properties

    • If n is B-smooth and m | n, then m is also B-smooth. (Divisors inherit smoothness.)
    • If a and b are both B-smooth, then ab is B-smooth. (Smooth numbers are closed under multiplication.)
    • A B-smooth number less than or equal to x can have at most ⌊log x / log 2⌋ prime factors counted with multiplicity.

    Counting Smooth Numbers

    The function Ψ(x, y) counts how many integers in [1, x] are y-smooth:

    Notation. Ψ(x, y) = |{n ≤ x : P+(n) ≤ y}|

    This is one of the central objects in analytic number theory. Understanding Ψ(x, y) precisely is fundamental to the analysis of sub-exponential factoring algorithms.

    Dickman’s Function

    For u = (log x) / (log y), the Dickman function ρ(u) governs the asymptotic density of smooth numbers. It is defined by the delay-differential equation:

    Dickman’s function ρ(u):
    ρ(u) = 1 for 0 ≤ u ≤ 1
    u · ρ′(u) = −ρ(u − 1) for u > 1

    The key asymptotic result (due to Dickman, 1930; refined by many) is:

    Ψ(x, y) ∼ x · ρ(u)   as x → ∞ with u = (log x)/(log y) bounded.

    Some useful values of ρ(u):

    uρ(u)Approx
    111.000
    21 − ln 20.307

    The rapid decay of ρ(u) explains why smooth numbers become exponentially rare as u grows. For u = 2, about 30.7% of numbers up to x are x1/2-smooth. For u = 10, the fraction drops to roughly 10−11.

    De Bruijn’s Refinement

    For a wider range of y (specifically y ≥ (log x)1+ε), de Bruijn (1951) proved:

    Ψ(x, y) = x · ρ(u) · (1 + O(log(u+1) / log y))

    This is the workhorse estimate used in cryptographic security analysis.

    Hildebrand–Tenenbaum Saddle-Point Method

    For the widest useful range (y ≥ exp((log log x)5/3+ε)), Hildebrand and Tenenbaum (1986) gave the precise estimate:

    Ψ(x, y) = x · ρ(u) · exp(O(u · exp(−(log y)3/5−ε)))

    Computing Smooth Numbers

    Sieve of Eratosthenes Variant

    The simplest approach: run a sieve but instead of marking composites, store the largest prime factor. Then a number is B-smooth iff its largest prime factor entry is ≤ B.

    function sieveLargestPrimeFactor(N): lpf[1..N] ← 0 for p = 2 to N: if lpf[p] == 0: // p is prime for m = p to N step p: lpf[m] ← p // overwrite: last prime to divide m is the largest return lpf // n is B-smooth iff lpf[n] ≤ B

    Time: O(N log log N) — same as classic Eratosthenes.
    Space: O(N).

    Canfield–Erdős–Pomerance Theorem

    This foundational result (1983) describes the expected size of the largest prime factor of a random integer near x. It shows that for factoring algorithms based on smooth-number detection, the optimal smoothness bound is:

    CEP Theorem. The probability that a random integer near x is L(x)1/√2-smooth is L(x)−1/√2 + o(1), where L(x) = exp(√(ln x · ln ln x)).

    This L-notation governs the sub-exponential complexity L(x)c+o(1) of the quadratic sieve (c = 1) and number field sieve (c ≈ 1.923 in the exponent for the cubic root variant).

    Smooth Number Generation via Repeated Squaring

    In the quadratic sieve, we don’t enumerate all smooth numbers — instead, we search for values of a polynomial Q(x) = (x + ⌊√N⌋)2 − N that happen to be B-smooth. Trial division over the factor base {p ≤ B : (N/p) = 1} tests smoothness.

    Friable Numbers in Practice

    Connection to Factoring Algorithms

    Quadratic Sieve (QS)

    Finds B-smooth values of (x + ⌊√N⌋)2 mod N. Complexity: L_N[1/2, 1].

    Optimal B ≈ L_N[1/2, 1/√2].

    Number Field Sieve (NFS)

    Searches for smooth norms in a number field. Complexity: L_N[1/3, (64/9)1/3].

    The improvement over QS comes from smoothness in two factor bases simultaneously.

    CFRAC

    Continued fraction factoring: the convergents of √N yield values that are often smooth. Historical precursor to QS.

    Pollard’s p−1

    Works when p−1 is B-smooth for some factor p of N. Choose B, compute aB! mod N, then gcd(aB! − 1, N).

    RSA Security Assumptions

    RSA’s security relies on the difficulty of factoring N = pq. Pollard’s p−1 attack succeeds when p−1 is smooth, so cryptographic prime generation specifically checks that p−1 has a large prime factor (often requiring p = 2q + 1 for a prime q, making p a “safe prime”).

    Cryptographic Note. Never use primes p where p−1 is B-smooth for small B. The Pollard p−1 and Williams p+1 methods can factor N = pq in O(B · log N) time when p−1 (or p+1) is B-smooth.

    Lenstra’s ECM

    The elliptic curve method generalizes Pollard’s p−1: instead of requiring p−1 to be smooth, it tries random elliptic curves E and succeeds when |E(F_p)| is smooth. Since the group orders are distributed near p (by Hasse’s theorem), many random curves give a smooth order, making ECM effective for finding moderate-sized prime factors regardless of the structure of p−1.

    Implementation

    Checking B-Smoothness

    Given a number n and a bound B, we can check B-smoothness by trial division over primes up to B:

    #include <bits/stdc++.h>
using namespace std;

bool isBSmooth(long long n, int B) {
    if (n <= 1) return true;
    for (int p = 2; p <= B; p++) {
        if (p * (long long)p > n && n > 1) break;
        while (n % p == 0) n /= p;
    }
    // If n == 1, all factors were ≤ B
    // If n > 1 but n ≤ B, the remaining factor is a prime ≤ B
    return n <= B;
}

    Complexity: O(B) per query in the worst case. For small B this is fast; for large B we need a sieve.

    Counting B-Smooth Numbers via Sieve

    int countBSmooth(int N, int B) {
    // lpf[i] = largest prime factor of i
    vector<int> lpf(N + 1, 0);
    for (int p = 2; p <= N; p++) {
        if (lpf[p] == 0) { // p is prime
            for (int m = p; m <= N; m += p)
                lpf[m] = p;
        }
    }
    int cnt = 0;
    for (int i = 1; i <= N; i++)
        if (lpf[i] <= B) cnt++;
    return cnt;
}

    Enumerating All B-Smooth Numbers up to N

    // Generate all B-smooth numbers up to N in sorted order
// Uses a priority-queue merge of geometric sequences
vector<long long> generateSmooth(int N, vector<int>& primes) {
    set<long long> seen;
    priority_queue<long long, vector<long long>, greater<>> pq;
    pq.push(1);
    seen.insert(1);
    vector<long long> result;

    while (!pq.empty()) {
        long long cur = pq.top(); pq.pop();
        if (cur > N) break;
        result.push_back(cur);
        for (int p : primes) {
            long long nxt = cur * p;
            if (nxt <= N && !seen.count(nxt)) {
                seen.insert(nxt);
                pq.push(nxt);
            }
        }
    }
    return result;
}
    Performance. The heap-based approach generates Ψ(N, B) smooth numbers in O(Ψ(N,B) · π(B) · log Ψ(N,B)) time. For the sieve approach, the time is O(N log log N) regardless of B.

    Approximating ρ(u) Numerically

    // Compute Dickman's rho via Euler method on the DDE
// u*rho'(u) = -rho(u-1), rho(u) = 1 for 0 ≤ u ≤ 1
double dickmanRho(double u, int steps = 10000) {
    if (u <= 1.0) return 1.0;
    double dt = u / steps;
    vector<double> rho(steps + 1);
    for (int i = 0; i <= steps; i++) {
        double t = i * dt;
        if (t <= 1.0) { rho[i] = 1.0; continue; }
        // rho'(t) = -rho(t-1)/t
        int j = (int)((t - 1.0) / dt);
        j = min(j, i - 1);
        rho[i] = rho[i - 1] - dt * rho[j] / t;
    }
    return rho[steps];
}

    CP Applications

    Problem Type 1: Counting Smooth Numbers in a Range

    A classic contest problem asks: “How many integers in [L, R] have all prime factors ≤ K?” When R is small enough (≤ 107), the sieve approach is direct. For larger ranges, a segmented sieve or inclusion-exclusion over the primes > K can work.

    Problem Type 2: Hamming Numbers / Regular Numbers

    Many problems ask for the n-th number whose only prime factors are from a given set S. This is exactly the n-th S-smooth number. The priority queue generation method works well here, and for S = {2, 3, 5} there is the classic O(n) DP solution:

    long long nthHamming(int n) {
    vector<long long> h(n);
    h[0] = 1;
    int i2 = 0, i3 = 0, i5 = 0;
    for (int i = 1; i < n; i++) {
        long long v2 = h[i2] * 2, v3 = h[i3] * 3, v5 = h[i5] * 5;
        h[i] = min({v2, v3, v5});
        if (h[i] == v2) i2++;
        if (h[i] == v3) i3++;
        if (h[i] == v5) i5++;
    }
    return h[n - 1];
}
    Key Insight. The DP uses one pointer per prime: each pointer tracks the smallest Hamming number whose product with that prime hasn’t been generated yet. This avoids duplicates and runs in O(n) time with O(n) space.

    Problem Type 3: Pollard p−1 Style Problems

    Some contest problems give you N and ask to find a non-trivial factor. If N has a factor p such that p−1 is B-smooth, you can implement Pollard’s p−1:

    long long pollardPm1(long long N, int B) {
    long long a = 2;
    for (int p = 2; p <= B; p++) {
        // Raise to highest power of p ≤ N
        long long pk = p;
        while (pk * p <= N) pk *= p;
        a = modpow(a, pk, N);
    }
    long long g = __gcd(a - 1, N);
    if (g > 1 && g < N) return g;
    return -1; // failed
}

    Problem Type 4: GCD-Smooth Decomposition

    Given an array of integers, find the B-smooth part of each: repeatedly divide by all primes ≤ B until the remaining cofactor has no factor ≤ B. The smooth part captures the “easy” structure of the number.

    Complexity Cheat Sheet

    B-smooth check (trial division)

    O(B) per number

    Sieve largest prime factor

    O(N log log N) preprocessing, O(1) per query

    Enumerate Ψ(N, B) smooth numbers

    O(Ψ · π(B) · log Ψ) via heap

    n-th Hamming number

    O(n) DP with |S| pointers
