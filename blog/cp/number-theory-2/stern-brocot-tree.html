    Stern–Brocot Tree & Farey Sequences

    July 2026 · 35 min read

    Mediants and the Stern–Brocot Tree

    The mediant of two fractions a/b and c/d is defined as (a+c)/(b+d). Note that this is not the arithmetic mean—it's a "freshman sum" that produces a new fraction strictly between the two parents (when they are Farey neighbours).

    Construction

    Start with the "sentinel" fractions 0/1 and 1/0 (representing 0 and ∞). At each level, insert the mediant between every pair of adjacent fractions:

    Level Fractions
    0 0/1   1/0
    1 0/1   1/1   1/0
    2 0/1   1/2   1/1   2/1   1/0
    3 0/1   1/3   1/2   2/3   1/1   3/2   2/1   3/1   1/0

    The resulting infinite binary tree is the Stern–Brocot tree. Its key properties:

    Properties

    • Completeness: Every positive rational number appears exactly once (in lowest terms).
    • Ordering: An in-order traversal gives all positive rationals in increasing order.
    • Neighbour property: If a/b and c/d are adjacent in the tree at any level, then |ad − bc| = 1.

    Proof of the neighbour property. By induction. At level 0, |0·0 − 1·1| = 1. When we insert the mediant (a+c)/(b+d) between a/b and c/d, the new pairs satisfy:

    a(b+d) − b(a+c) = ad − bc = ±1

    and similarly for the right pair. The determinant is preserved. ■

    Connection to continued fractions. The path from the root to a fraction p/q in the Stern–Brocot tree encodes its continued fraction expansion: each "left" turn corresponds to a partial quotient, and each "right" turn adds to the next partial quotient.

    Farey Sequences

    The Farey sequence Fn is the ascending sequence of all reduced fractions a/b with 0 ≤ a/b ≤ 1 and b ≤ n.

    n Fn
    1 0/1, 1/1
    2 0/1, 1/2, 1/1
    3 0/1, 1/3, 1/2, 2/3, 1/1
    4 0/1, 1/4, 1/3, 1/2, 2/3, 3/4, 1/1
    5 0/1, 1/5, 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, 3/4, 4/5, 1/1

    Key theorems

    Neighbour theorem. If a/b and c/d are consecutive in Fn, then bc − ad = 1.

    Mediant property. If a/b and c/d are consecutive in Fn and b+d ≤ n+1, the next fraction to be inserted (in Fb+d) is their mediant.

    Counting. |Fn| = 1 + ∑k=1nφ(k), where φ is Euler's totient. Asymptotically, |Fn| ∼ 3n2/π2.

    Farey vs. Stern–Brocot. The Farey sequence Fn is the subset of the Stern–Brocot tree restricted to fractions in [0,1] with denominator ≤ n. The tree structure gives a natural way to enumerate or search Farey fractions.

    Searching the Stern–Brocot Tree

    Given a target real number x (or a fraction p/q), we can binary search the Stern–Brocot tree to find it. The search descends the tree, going left or right at each node.

    function sbSearch(p, q): // Find p/q in the Stern-Brocot tree // Boundaries: left = a/b, right = c/d a, b ← 0, 1 // left sentinel c, d ← 1, 0 // right sentinel path ← "" loop: m_num ← a + c // mediant numerator m_den ← b + d // mediant denominator if m_num × q = m_den × p: return (m_num, m_den, path) // found else if m_num × q < m_den × p: a, b ← m_num, m_den // go right path ← path + "R" else: c, d ← m_num, m_den // go left path ← path + "L"

    Complexity: O(p + q) in the worst case (for fractions like 1/q that require many left turns). However, with the shortcut optimization below, this drops to O(log(max(p,q))).

    Shortcut: batch turns

    Instead of taking one step at a time, when we need k consecutive turns in the same direction, we can jump by k steps at once. The number of direction changes equals the length of the continued fraction of p/q, which is O(log(max(p,q))).

    function sbSearchFast(p, q): a, b ← 0, 1 c, d ← 1, 0 while true: m_num, m_den ← a + c, b + d if m_num × q = m_den × p: return (m_num, m_den) if m_num × q < m_den × p: // Need to go right. How many times? k ← (p × b − q × a) / (q × c − p × d) // floor division a ← a + k × c b ← b + k × d else: k ← (q × c − p × d) / (p × b − q × a) c ← c + k × a d ← d + k × b

    Best Rational Approximation

    A best rational approximation of a real number x with denominator ≤ N is a fraction p/q with q ≤ N that minimises |x − p/q|. The Stern–Brocot tree gives a clean algorithm for this.

    Algorithm

    Search the Stern–Brocot tree towards x. At each node, if the mediant's denominator exceeds N, stop. The answer is the last visited node whose denominator was ≤ N—either the left or right boundary, whichever is closer to x.

    function bestApprox(x_num, x_den, N): a, b ← 0, 1 c, d ← 1, 0 best_p, best_q ← 0, 1 while true: m_num, m_den ← a + c, b + d if m_den > N: // Can't go deeper; pick best among a/b and c/d // (both have denom ≤ N if they're valid) break if m_num × x_den < m_den × x_num: a, b ← m_num, m_den else if m_num × x_den > m_den × x_num: c, d ← m_num, m_den else: return (m_num, m_den) // exact match // Compare |a/b - x| vs |c/d - x| (cross-multiply) if b > 0 and |a × x_den − b × x_num| × d ≤ |c × x_den − d × x_num| × b: return (a, b) return (c, d)

    Connection to continued fractions. The convergents of the continued-fraction expansion of x are exactly the ancestors on the path to x in the Stern–Brocot tree where the direction changes. These are the best approximations of the second kind: no fraction with smaller denominator is closer to x.

    Theorem (Hurwitz). For any irrational x, there are infinitely many fractions p/q with |x − p/q| < 1/(√5 q2). The golden ratio (√5+1)/2 is the "hardest to approximate" irrational—its convergents (Fibonacci ratios) achieve this bound.

    Implementation

    Stern–Brocot tree search

    #include <bits/stdc++.h>
using namespace std;
typedef long long ll;
typedef pair<ll,ll> frac; // (numerator, denominator)

// Find p/q in the Stern-Brocot tree (fast version)
// Returns the path as a string of 'L' and 'R'
string sbPath(ll p, ll q) {
    string path;
    ll a = 0, b = 1, c = 1, d = 0;
    while (true) {
        ll mn = a + c, md = b + d;
        if (mn == p && md == q) break;
        if ((__int128)mn * q < (__int128)md * p) {
            // go right, compute how many steps
            ll k = max(1LL, (p * b - q * a) / (q * c - p * d));
            a += k * c; b += k * d;
            for (ll i = 0; i < k; i++) path += 'R';
        } else {
            ll k = max(1LL, (q * c - p * d) / (p * b - q * a));
            c += k * a; d += k * b;
            for (ll i = 0; i < k; i++) path += 'L';
        }
    }
    return path;
}

    Best rational approximation

    // Find best p/q with q <= N approximating target a/b
frac bestApprox(ll a, ll b, ll N) {
    ll lp = 0, lq = 1; // left boundary
    ll rp = 1, rq = 0;  // right boundary
    frac best = {0, 1};
    while (true) {
        ll mp = lp + rp, mq = lq + rq;
        if (mq > N) break;
        // Compare mediant mp/mq with target a/b
        if ((__int128)mp * b < (__int128)mq * a) {
            lp = mp; lq = mq;
        } else if ((__int128)mp * b > (__int128)mq * a) {
            rp = mp; rq = mq;
        } else {
            return {mp, mq}; // exact
        }
        best = {mp, mq};
    }
    // Also consider jumping to the boundary max times
    if (lq > 0 && lq <= N) {
        ll k = (N - lq) / rq; // max right jumps from left boundary
        frac cand = {lp + k * rp, lq + k * rq};
        if (cand.second <= N) {
            // Check if candidate is closer to a/b than best
            auto dist = [](auto p1, auto q1, auto a, auto b) {
                return abs((__int128)p1*b - (__int128)q1*a);
            };
            if (dist(cand.first,cand.second,a,b) * best.second
                <= dist(best.first,best.second,a,b) * cand.second)
                best = cand;
        }
    }
    return best;
}

    Farey sequence generation

    // Generate Farey sequence F_n using the mediant property
vector<frac> farey(int n) {
    vector<frac> seq;
    ll a = 0, b = 1, c = 1, d = n;
    seq.push_back({a, b});
    while (c <= n) {
        seq.push_back({c, d});
        ll k = (n + b) / d; // floor((n + b) / d)
        ll nc = k * c - a, nd = k * d - b;
        a = c; b = d; c = nc; d = nd;
    }
    return seq;
}

    Complexity. The Farey generation runs in O(|Fn|) = O(n2) time with O(1) working space per step (using the mediant recurrence). The SB-tree search with shortcuts runs in O(log(max(p,q))) time.

    CP Applications

    Problem patterns

    Fraction with bounded denominator

    "Find p/q closest to x with q ≤ N."

    Direct SB-tree search with denominator cutoff.

    Counting fractions

    "Count reduced fractions in [0,1] with denominator ≤ N."

    Answer: ∑φ(k) for k=1..N. Use sieve for φ.

    Rank of a fraction

    "Given p/q, find its position in Fn."

    Use the SB-tree path + counting formula.

    Classic problems

    • Codeforces 305C — "Ivan and Powers of Two": counting in the SB-tree context.
    • SPOJ FRACTION — Direct best-approximation problem.
    • Codeforces 1354D — Farey sequence + binary search on fractions.
    • ICPC 2017 Asia Dhaka Regional — Problem G — Stern–Brocot tree search.
    • Project Euler 71–73 — Counting Farey fractions in ranges; use the neighbour theorem and Euler's totient.

    The mediant trick in binary search

    Some problems require binary-searching over rationals (not integers). Instead of using floating point—which introduces precision errors—use the Stern–Brocot tree: at each step, the mediant is the exact "midpoint" between the two boundary fractions, and the search converges in O(log(answer)) steps with exact arithmetic.

    Overflow warning. Numerators and denominators grow rapidly in the SB-tree. For fractions with p,q ≤ 109, use __int128 for cross-multiplication comparisons (a×d vs. b×c). The intermediate products can reach ~1018.
