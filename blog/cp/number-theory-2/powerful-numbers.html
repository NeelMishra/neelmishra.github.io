    Powerful Numbers

    July 2026 · 25 min read

    Definition

    A positive integer n is called powerful (also called squareful) if for every prime p dividing n, we also have p2 | n. Equivalently, in the prime factorisation n = p1a1 · p2a2 · … · pkak, every exponent ai ≥ 2.

    Equivalent characterisation. n is powerful if and only if n can be written as n = a2 b3 for some positive integers a, b. This representation is the foundation for efficient enumeration.

    First Few Powerful Numbers

    1, 4, 8, 9, 16, 25, 27, 32, 36, 49, 64, 72, 81, 100, 108, 121, 125, 128, …

    The number 1 is powerful (vacuously — it has no prime factors). The number 72 = 23 · 32 is powerful because both exponents are ≥ 2. The number 12 = 22 · 3 is not powerful because the exponent of 3 is 1.

    Proof of the a²b³ Characterisation

    (⇒) If n is powerful, write n = ∏ piai with each ai ≥ 2. Write ai = 2qi + 3ri where ri ∈ {0, 1} and qi ≥ 0 (since ai ≥ 2, we can always find non-negative qi, ri). Then n = (∏ piqi)2 · (∏ piri)3.

    (⇐) If n = a2b3 and p | n, then p | a2b3. If p | a, then p2 | a2 | n. If p ∤ a, then p | b3, so p | b, and p3 | b3 | n. Either way, p2 | n. □

    Decomposition Algorithm

    Given that every ai ≥ 2, write ai = 2qi + 3ri:

    • If ai is even: qi = ai/2, ri = 0
    • If ai is odd (and ≥ 3): qi = (ai − 3)/2, ri = 1

    Counting Powerful Numbers

    Let P(x) denote the count of powerful numbers ≤ x. Golomb (1970) proved:

    P(x) = ∑b=1⌊x1/3⌋ ⌊√(x / b3)⌋ · μ2(b)

    Derivation

    Using n = a2b3, the count of powerful numbers ≤ x is the number of pairs (a, b) with a2b3 ≤ x and b squarefree (to avoid double-counting — if b had a squared factor, we could absorb it into a).

    For each squarefree b, a ranges from 1 to ⌊√(x/b3)⌋. And b ranges from 1 to ⌊x1/3⌋ (since b3 ≤ x).

    Asymptotic. P(x) ~ c · √x where c = ζ(3/2) / ζ(3) ≈ 2.173. The powerful numbers have density zero but grow like √x.

    Why μ²(b)?

    The squarefree condition on b is essential to avoid overcounting. If b = 2d where d has a squared factor p2, then b3 contains p6, and we could write p6 = (p3)2, absorbing p3 into a. The Möbius-squared function μ2(b) equals 1 iff b is squarefree, exactly filtering these duplicates.

    Computing μ²

    Recall μ2(n) = ∑d2|n μ(d). Alternatively, μ2(n) = 1 iff n is squarefree, which can be checked by trial division or from the SPF sieve. 

    // Check if b is squarefree using SPF array
bool is_squarefree(int b) {
    while (b > 1) {
        int p = spf[b];
        b /= p;
        if (b % p == 0) return false;
    }
    return true;
}

    Representation

    The n = a2b3 representation (with b squarefree) is unique. This gives us a canonical way to enumerate all powerful numbers.

    Proof of Uniqueness

    Suppose a12 b13 = a22 b23 with b1, b2 squarefree. For each prime p, comparing p-adic valuations: 2vp(a1) + 3vp(b1) = 2vp(a2) + 3vp(b2). Since bi are squarefree, vp(bi) ∈ {0, 1}. If vp(b1) = vp(b2), then vp(a1) = vp(a2). If they differ (say 0 and 1), then 2vp(a1) = 2vp(a2) + 3, which is impossible since the left side is even and the right is odd. So b1 = b2 and a1 = a2. □

    Enumeration Algorithm

    enumerate_powerful(x): result = [] for b = 1 to x^{1/3}: if not squarefree(b): continue b3 = b * b * b for a = 1 to sqrt(x / b3): result.append(a * a * b3) sort(result) return result

    The total number of powerful numbers ≤ x is O(√x), so the enumeration produces O(√x) elements. Sorting takes O(√x · log x).

    Sum of Multiplicative Functions over Powerful Numbers

    The powerful number sieve is a technique for computing ∑i=1n f(i) for a multiplicative function f, by decomposing the sum using a multiplicative function g that agrees with f on powerful numbers.

    The Technique

    Suppose we can find multiplicative functions g and h such that f = g * h (Dirichlet convolution) and h is “sparse” (h(n) = 0 for most n). Then:

    ∑i=1n f(i) = ∑d=1n h(d) · G(⌊n/d⌋)

    where G(x) = ∑i=1x g(i). If h is supported only on powerful numbers, then the sum over d has only O(√n) terms!

    Key insight. The powerful number sieve reduces the computation of ∑f(i) to computing O(√n) evaluations of a prefix sum G(x). If G(x) can be computed in sublinear time (e.g., using the Min-25 sieve), the overall complexity is O(n1/2+ε) or better.

    Finding g and h

    Given f, we want f = g * h where:

    • g is easy to sum (e.g., g = id, g = φ, g = μ)
    • h(p) = 0 for all primes p (so h is supported on powerful numbers)

    Since f = g * h implies f(p) = g(p)h(1) + g(1)h(p) = g(p) + h(p), setting h(p) = 0 forces g(p) = f(p). For prime powers pe with e ≥ 2:

    h(pe) = f(pe) − ∑j=0e−1 g(pj) · h(pe−j)

    Example: Counting Squarefree Numbers

    Let f(n) = μ2(n) (indicator of squarefree). We know μ2 = |μ| and ∑ μ2(i) for i ≤ n counts squarefree numbers. Take g(n) = 1 (trivially summable: G(x) = x). Then h(p) = f(p) − g(p) = 1 − 1 = 0. Good: h is supported on powerful numbers. We get h(pe) = −1 for e = 2 and 0 for e ≥ 3.

    This recovers the well-known formula: ∑i=1n μ2(i) = ∑d=1⌊√n⌋ μ(d) · ⌊n/d2⌋.

    General Framework

    // Powerful number sieve: compute sum f(i) for i = 1..n
// Requires: prefix_G(x) computes G(x) = sum g(i) for i=1..x
// Requires: enumerate powerful numbers with their h-values

long long sum_f(long long n) {
    long long result = 0;
    // For each powerful number d with h(d) != 0:
    for (auto [d, hd] : powerful_with_h_values(n)) {
        result += hd * prefix_G(n / d);
    }

}

    Implementation

    Counting Powerful Numbers up to x

    #include <bits/stdc++.h>
using namespace std;

// Precompute mu^2 (squarefree indicator) via sieve
const int MAXB = 1'000'001;
bool sqfree[MAXB];
int spf[MAXB];

void sieve_squarefree() {
    fill(sqfree, sqfree + MAXB, true);
    for (int i = 2; (long long)i * i < MAXB; i++) {
        long long i2 = (long long)i * i;
        for (long long j = i2; j < MAXB; j += i2)
            sqfree[j] = false;
    }
}

long long count_powerful(long long x) {
    long long cbrt_x = (long long)cbrt((double)x);
    while ((cbrt_x + 1) * (cbrt_x + 1) * (cbrt_x + 1) <= x)
        cbrt_x++;
    while (cbrt_x * cbrt_x * cbrt_x > x)
        cbrt_x--;

    long long count = 0;
    for (long long b = 1; b <= cbrt_x; b++) {
        if (!sqfree[b]) continue;
        long long b3 = b * b * b;
        long long lim = (long long)sqrt((double)(x / b3));
        // Adjust for floating-point errors
        while ((lim + 1) * (lim + 1) * b3 <= x) lim++;
        while (lim * lim * b3 > x) lim--;
        count += lim;
    }
    return count;
}

int main() {
    sieve_squarefree();
    long long x;
    cin >> x;
    cout << count_powerful(x) << "\n";
}
    Floating-point caution. Always adjust the cube root and square root with while-loops. Functions like cbrt() and sqrt() can be off by 1 for large inputs. Never trust floating point for exact integer results.

    Enumerating Powerful Numbers with Values

    // Enumerate all powerful numbers ≤ x as (value, a, b) triples
vector<tuple<long long, long long, long long>>
enumerate_powerful(long long x) {
    vector<tuple<long long, long long, long long>> result;
    long long cbrt_x = (long long)cbrt((double)x);
    while (cbrt_x * cbrt_x * cbrt_x > x) cbrt_x--;

    for (long long b = 1; b <= cbrt_x; b++) {
        if (!sqfree[b]) continue;
        long long b3 = b * b * b;
        long long a_max = (long long)sqrt((double)(x / b3));
        while (a_max * a_max * b3 > x) a_max--;

        for (long long a = 1; a <= a_max; a++) {
            result.emplace_back(a * a * b3, a, b);
        }
    }
    sort(result.begin(), result.end());
    return result;
}

    Powerful Number Sieve: Computing h-values

    When enumerating powerful numbers for the sieve technique, we also need the h-values. Since h is multiplicative and supported on powerful numbers, we compute h(d) during enumeration by factoring d via its (a, b) representation. 

    // DFS enumeration of powerful numbers with h-values
// primes[] is a list of small primes up to sqrt(n)
vector<pair<long long, long long>> powerful_h;

void dfs_powerful(int idx, long long d, long long hd,
                   long long n) {
    powerful_h.emplace_back(d, hd);

    for (int i = idx; i < (int)primes.size(); i++) {
        long long p = primes[i];
        if (p * p > n / d) break;

        long long pe = p * p, ppow = p * p;
        for (int e = 2; d * pe <= n; e++, pe *= p) {
            long long h_pe = compute_h_prime_power(p, e);
            if (h_pe != 0)
                dfs_powerful(i + 1, d * pe, hd * h_pe, n);
        }
    }
}

    CP Applications

    Application 1: Counting Squarefree Numbers

    The number of squarefree integers in [1, n] equals:

    Q(n) = ∑d=1⌊√n⌋ μ(d) · ⌊n / d2⌋

    This is a special case of the powerful number sieve with g = 1 and h supported on squares. Complexity: O(√n) with precomputed μ. 

    long long count_squarefree(long long n) {
    int sq = (int)sqrt((double)n);
    while ((long long)(sq + 1) * (sq + 1) <= n) sq++;
    long long res = 0;
    for (int d = 1; d <= sq; d++) {
        res += mu_arr[d] * (n / ((long long)d * d));
    }
    return res;
}

    Application 2: Sum of Euler Totient via Powerful Numbers

    To compute ∑i=1n φ(i), set g = id (so G(x) = x(x+1)/2) and solve for h. Then h(p) = φ(p) − p = −1, which is non-zero. So the direct powerful-number approach doesn't apply for φ with g = id.

    Instead, use g = φ itself (with G computed via the hyperbola method or Min-25 sieve). The powerful number sieve is most useful when g has a simple closed-form prefix sum.

    Application 3: Enumerating Powerful Numbers in Contests

    Some contest problems ask to process all powerful numbers up to N. Since there are only O(√N) of them, explicit enumeration is feasible even for N = 1018 (about 109 powerful numbers, but for N = 1012 we get about 106 powerful numbers).

    N = 106

    P(N) ≈ 2,173 powerful numbers

    N = 109

    P(N) ≈ 68,726 powerful numbers

    N = 1012

    P(N) ≈ 2,173,243 powerful numbers

    N = 1018

    P(N) ≈ 2.17 × 109 powerful numbers

    Application 4: Powerful Number Sieve for σk

    For σk(n) = ∑d|n dk, we have σk = idk * 1 (Dirichlet convolution). Set g = idk, then G(x) = ∑ ik (computable using Faulhaber formulas in O(k) per query). Then h(p) = σk(p) − pk = 1 + pk − pk = 1 ≠ 0. So this decomposition doesn't give h supported on powerful numbers.

    Instead, take g = σk (computed via Min-25 for the prefix sum) and h = ε (identity of Dirichlet convolution). Then only d = 1 contributes, giving the trivial ∑f = G(n). The power of the technique emerges when a natural decomposition makes h sparse.

    When to use the powerful number sieve. It shines when you can decompose f = g * h where (1) G(x) has a sublinear prefix sum, and (2) h is naturally supported on powerful numbers (h(p) = 0 for all primes). The classic example is μ2 = |μ|.
