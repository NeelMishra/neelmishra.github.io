    Pell's Equation

    July 2026 · 30 min read

    What is Pell's Equation?

    Pell's equation is the Diophantine equation:

    x² − D · y² = 1

    where D is a positive integer that is not a perfect square. We seek solutions in positive integers (x, y). When D is a perfect square, say D = k², the equation (x − ky)(x + ky) = 1 forces x − ky = x + ky = 1, giving only the trivial solution (1, 0).

    Historical note. Despite the name, the equation was studied by Indian mathematicians (Brahmagupta, 628 CE; Bhaskara II, 1150 CE) long before Pell. Euler incorrectly attributed the work to John Pell, and the name stuck. Brahmagupta's chakravala method solves it efficiently.

    Why It Matters in CP

    Pell's equation appears in competitive programming whenever a problem reduces to finding integer points on a hyperbola, computing fundamental units in quadratic number fields, or solving large-scale Diophantine problems. Common reductions include:

    • Finding integers n such that n(n+1)/2 is a perfect square (triangular-square numbers)
    • Almost-square rectangles: n × (n+1) grids with area near a square
    • Balancing numbers: 1 + 2 + … + (n−1) = (n+1) + … + (n+r)

    Fundamental Solution

    The fundamental solution (x₁, y₁) of x² − Dy² = 1 is the solution with the smallest positive y (equivalently, smallest positive x > 1).

    Theorem. For every non-square D > 0, Pell's equation has infinitely many positive solutions. Moreover, if (x₁, y₁) is the fundamental solution, then all positive solutions (xₖ, yₖ) are given by:

    xₖ + yₖ√D = (x₁ + y₁√D)k,   k = 1, 2, 3, …

    Notice how the fundamental solution for D = 61 is enormous. The size of the fundamental solution can grow exponentially in D, which is why a systematic method (continued fractions) is essential.

    Proof Sketch of Existence

    By the pigeonhole principle: consider the numbers ⌊k√D⌋ for k = 0, 1, …, N. The fractional parts {k√D} lie in [0, 1). Partitioning into N intervals of length 1/N, two values k₁, k₂ fall in the same interval, giving |k₁√D − k₂√D − (m₁ − m₂)| < 1/N. Setting x = m₁ − m₂, y = k₁ − k₂, we get |x² − Dy²| < 2√D + 1. Infinitely many such pairs exist but only finitely many possible values of x² − Dy², so some value r recurs infinitely often. A congruence argument (mod |r|) then yields a solution to x² − Dy² = 1.

    Continued Fraction Method

    The practical algorithm for finding the fundamental solution uses the continued fraction expansion of √D. The key facts are:

    1. The continued fraction of √D is eventually periodic: √D = [a₀; a₁, a₂, …, aₜ, a₁, a₂, …, aₜ, …] where a₀ = ⌊√D⌋ and the period has length t.
    2. The fundamental solution comes from the convergent ht−1/kt−1 at the end of the first period: (x₁, y₁) = (ht−1, kt−1) if t is even, or (x₁, y₁) = (h2t−1, k2t−1) if t is odd.

    Algorithm: CF Expansion of √D

    function sqrtCF(D): a₀ ← ⌊√D⌋ if a₀² = D: return "D is a perfect square" coefficients ← [a₀] m ← 0, d ← 1, a ← a₀ repeat: m ← a · d − m d ← (D − m²) / d a ← ⌊(a₀ + m) / d⌋ append a to coefficients until a = 2 · a₀ // period ends when a = 2a₀ return coefficients

    Computing Convergents

    Given the CF coefficients [a₀; a₁, a₂, …], the convergents hₖ/kₖ satisfy the recurrence:

    h−1 = 1, h₀ = a₀, hₖ = aₖ · hk−1 + hk−2
    k−1 = 0, k₀ = 1, kₖ = aₖ · kk−1 + kk−2

    We compute convergents until we find h² − D · k² = 1.

    Complexity. The period length t of the CF of √D is O(√D log D) in the worst case. Each step does constant work (with big-integer arithmetic if needed), so finding the fundamental solution is polynomial in log D if we count bit operations.

    Generating All Solutions

    Once we have the fundamental solution (x₁, y₁), all positive solutions come from the recurrence:

    xk+1 = x₁ · xₖ + D · y₁ · yₖ
    yk+1 = y₁ · xₖ + x₁ · yₖ

    Matrix Form

    This can be expressed as matrix exponentiation:

    ┌ x_k ┐ ┌ x₁ D·y₁ ┐ᵏ ┌ 1 ┐ │ │ = │ │ · │ │ └ y_k ┘ └ y₁ x₁ ┘ └ 0 ┘

    To find the k-th solution quickly, use fast matrix exponentiation in O(log k) matrix multiplications. If we need the solution modulo some prime p, this is especially efficient.

    Growth Rate

    The solutions grow exponentially: xₖ ≈ (x₁ + y₁√D)k / 2. For D = 2, the fundamental solution is (3, 2) and 3 + 2√2 ≈ 5.83, so the solutions roughly quintuple each step.

    // Generate k-th solution of x² - D·y² = 1 using matrix exponentiation
using ll = long long;
using Mat = array<array<ll, 2>, 2>;

Mat matMul(const Mat& A, const Mat& B, ll mod) {
    Mat C{};
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++)
            for (int k = 0; k < 2; k++)
                C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % mod;
    return C;
}

Mat matPow(Mat A, ll p, ll mod) {
    Mat R = {{{1,0},{0,1}}};
    while (p > 0) {
        if (p & 1) R = matMul(R, A, mod);
        A = matMul(A, A, mod);
        p >>= 1;
    }
    return R;
}

pair<ll,ll> pellKth(ll x1, ll y1, ll D, ll k, ll mod) {
    Mat M = {{{x1 % mod, (D % mod) * (y1 % mod) % mod},
              {y1 % mod, x1 % mod}}};
    Mat R = matPow(M, k, mod);
    return {R[0][0], R[1][0]};
}

    The Negative Pell Equation

    The negative Pell equation is:

    x² − D · y² = −1

    Unlike the positive equation, this does not always have a solution.

    Criterion. The negative Pell equation x² − Dy² = −1 has a solution if and only if the period length t of the continued fraction of √D is odd.

    When t is Odd

    If the period is odd, the convergent ht−1/kt−1 gives a solution to the negative equation. The next convergent (at index 2t − 1) gives the fundamental solution to the positive equation.

    Examples

    D Period t x²−Dy²=−1?
    2[1; 2̄]1 (odd)Yes: (1,1)
    5[2; 4̄]1 (odd)Yes: (2,1)
    10[3; 6̄]1 (odd)Yes: (3,1)
    3[1; 1,2̄]2 (even)No
    6[2; 2,4̄]2 (even)No

    A necessary condition for solvability: D must not be divisible by 4, and no prime p ≡ 3 (mod 4) can divide D. But this is not sufficient — D = 34 has all prime factors ≡ 1 mod 4 yet the negative equation has no solution.

    Implementation

    Finding the Fundamental Solution via CF

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;
using pll = pair<ll,ll>;

// Returns fundamental solution (x1, y1) of x²-Dy²=1.
// D must not be a perfect square.
pll pellFundamental(ll D) {
    ll a0 = (ll)sqrtl(D);
    if (a0 * a0 == D) return {-1, -1};  // perfect square

    ll m = 0, d = 1, a = a0;
    // convergent numerators/denominators
    ll h_prev = 1, h_curr = a0;
    ll k_prev = 0, k_curr = 1;

    while (true) {
        m = a * d - m;
        d = (D - m * m) / d;
        a = (a0 + m) / d;

        ll h_next = a * h_curr + h_prev;
        ll k_next = a * k_curr + k_prev;

        h_prev = h_curr; h_curr = h_next;
        k_prev = k_curr; k_curr = k_next;

        if (h_curr * h_curr - D * k_curr * k_curr == 1)
            return {h_curr, k_curr};
    }
}
    Overflow warning. For large D, the fundamental solution may exceed 64-bit integers. Use __int128 or Python for D up to ~109. For contest problems, check the constraints carefully — often the answer is requested modulo a prime, allowing matrix exponentiation in modular arithmetic.

    Complete Solver with Negative Pell

    // Solve x²-Dy²=c where c ∈ {1, -1}.
// Returns (x, y) for the fundamental solution, or (-1,-1) if none.
pll pellGeneral(ll D, int c) {
    ll a0 = (ll)sqrtl(D);
    if (a0 * a0 == D) return {-1, -1};

    ll m = 0, d = 1, a = a0;
    ll h_prev = 1, h_curr = a0;
    ll k_prev = 0, k_curr = 1;

    for (int iter = 0; iter < 100000; iter++) {
        m = a * d - m;
        d = (D - m * m) / d;
        a = (a0 + m) / d;

        ll h_next = a * h_curr + h_prev;
        ll k_next = a * k_curr + k_prev;
        h_prev = h_curr; h_curr = h_next;
        k_prev = k_curr; k_curr = k_next;

        ll val = h_curr * h_curr - D * k_curr * k_curr;
        if (val == c) return {h_curr, k_curr};
        if (a == 2 * a0 && c == -1)
            return {-1, -1};  // period complete, no -1 solution
    }
    return {-1, -1};
}

    CP Applications

    Problem 1: Triangular-Square Numbers

    Find all numbers that are both triangular and perfect squares. A number T is triangular if T = n(n+1)/2 and square if T = k². So n(n+1)/2 = k², i.e., n² + n = 2k², which gives (2n+1)² − 2(2k)² = 1, i.e., x² − 2y² = 1 with x = 2n+1, y = 2k.

    // Find first K triangular-square numbers
void triSquare(int K) {
    ll x = 3, y = 2;  // fundamental solution of x²-2y²=1
    ll xk = 3, yk = 2;
    for (int i = 0; i < K; i++) {
        ll n = (xk - 1) / 2;
        ll T = n * (n + 1) / 2;
        cout << T << "\n";
        ll nx = x * xk + 2 * y * yk;
        ll ny = y * xk + x * yk;
        xk = nx; yk = ny;
    }
}

    Problem 2: Near-Square Rectangles

    Find the smallest rectangle with integer dimensions whose area is within 1 of a perfect square: m · n = k² ± 1 with m ≈ n. Setting m = k + a, n = k − a (approximately), this leads to a Pell-like equation.

    Problem 3: Generalized Pell

    Many CP problems reduce to x² − Dy² = N for small N. The approach: find one solution (by brute force for small residues), then generate all solutions using the homogeneous solution (x₁, y₁) from the standard Pell equation.

    Key technique. Given one solution (x₀, y₀) to x² − Dy² = N, all solutions come from multiplying (x₀ + y₀√D) by (x₁ + y₁√D)k for all integers k. In practice, there may be multiple "base" solutions modulo the fundamental solution — enumerate them by checking small candidates.

    Summary of Pell Solver Workflow

    function solvePellProblem(D, targetN): // Step 1: find fundamental solution of x²-Dy²=1 (x₁, y₁) ← pellFundamental(D) // Step 2: find base solutions of x²-Dy²=targetN bases ← [] for y ← 0 to √(|targetN|/D + some bound): x² ← D·y² + targetN if x² ≥ 0 and isSquare(x²): bases.append( (√x², y) ) // Step 3: generate all solutions from each base for each (x₀, y₀) in bases: yield solutions via (x₀+y₀√D)·(x₁+y₁√D)ᵏ
