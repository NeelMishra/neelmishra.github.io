DP Approaches to Game Problems

June 2026 · 30 min read

Game Theory Meets DP

Many two-player combinatorial games exhibit optimal substructure and overlapping subproblems—the two hallmarks of dynamic programming. The connection is natural: a game state encodes the current position, whose value depends on the values of states reachable through legal moves.

Formally, define a game as a tuple $(S, s_0, T, M)$ where:

  • $S$ is the set of all game states,
  • $s_0 \in S$ is the initial state,
  • $T \subseteq S$ is the set of terminal states,
  • $M : S \to 2^S$ maps each non-terminal state to its successors (legal moves).

A utility function $u : T \to \mathbb{R}$ assigns a value to each terminal state from the perspective of the maximizing player. The goal is to compute:

$$V(s) = \begin{cases} u(s) & \text{if } s \in T \\ \max_{s' \in M(s)} V(s') & \text{if it is the maximizer's turn} \\ \min_{s' \in M(s)} V(s') & \text{if it is the minimizer's turn} \end{cases}$$

This recurrence is exactly a DP recurrence. Each state $s$ is a DP subproblem, each move is a transition, and the game tree is the dependency DAG. When the same position can be reached via multiple move sequences, we have overlapping subproblems—making memoization or tabulation essential.

Key insight: Every finite two-player zero-sum game with perfect information can be solved by DP on the game DAG. The game value $V(s_0)$ tells us the outcome under optimal play from both sides.

Why DP over brute-force game trees? A game tree can have exponentially many nodes, but the number of distinct positions is often much smaller. For example, Nim with $n$ piles of sizes $\le k$ has $O(k^n)$ distinct positions, while the game tree can be vastly larger due to transpositions.

Minimax Algorithm

The minimax algorithm is the direct recursive evaluation of the game value recurrence. Two players alternate: the maximizer picks the move that maximizes the value, and the minimizer picks the move that minimizes it.

Formal Definition

For a zero-sum game with utility function $u$ evaluated at terminal nodes:

$$\text{minimax}(s, \text{depth}, \text{isMax}) = \begin{cases} u(s) & \text{if } s \in T \text{ or depth} = 0 \\ \max_{s' \in M(s)} \text{minimax}(s', \text{depth}-1, \text{false}) & \text{if isMax} \\ \min_{s' \in M(s)} \text{minimax}(s', \text{depth}-1, \text{true}) & \text{if } \neg\text{isMax} \end{cases}$$

Basic C++ Implementation

#include <bits/stdc++.h>
using namespace std;

// State encodes the game position; moves() returns successors.
struct State {
    // game-specific fields
    bool is_terminal() const;
    int utility() const;            // value at terminal state
    vector<State> moves() const;    // legal successors
};

int minimax(const State& s, bool is_maximizer) {
    if (s.is_terminal())
        return s.utility();

    if (is_maximizer) {
        int best = INT_MIN;
        for (const auto& next : s.moves())
            best = max(best, minimax(next, false));
        return best;
    } else {
        int best = INT_MAX;
        for (const auto& next : s.moves())
            best = min(best, minimax(next, true));
        return best;
    }
}

Correctness Proof

Theorem (Zermelo, 1913): In any finite two-player zero-sum game with perfect information, the minimax algorithm computes the game-theoretic value, and both players have optimal strategies.

Proof sketch by strong induction on game depth $d$:

Base case ($d = 0$): The state is terminal, so $\text{minimax}(s) = u(s)$, which is trivially the game value.

Inductive step: Assume minimax correctly computes the value for all states at depth $\le d-1$. At depth $d$, the maximizer can guarantee value $\max_{s'} V(s')$ by choosing the best successor (whose value is correctly computed by induction). The minimizer cannot force a worse outcome for the maximizer than $\min_{s'} V(s')$. Since both play optimally, the value at depth $d$ is:

$$V(s) = \begin{cases} \max_{s'} V(s') & \text{maximizer's turn} \\ \min_{s'} V(s') & \text{minimizer's turn} \end{cases}$$

which matches the minimax recurrence. $\blacksquare$

Complexity Analysis

Let $b$ be the branching factor and $d$ the depth. Without memoization, minimax explores the entire game tree:

$$\text{Time: } O(b^d), \qquad \text{Space: } O(d) \text{ (recursion stack)}$$

This is prohibitive for most games. With memoization, we visit each distinct state once: $O(|S| \cdot b)$ time and $O(|S|)$ space.

Memoized Game DP

When the game graph has many transpositions (different move sequences reaching the same position), memoization transforms exponential tree search into polynomial DP.

Top-Down Framework

The general pattern: encode the state as a hashable key, cache computed values, and return cached results on re-visits.

#include <bits/stdc++.h>
using namespace std;

// Example: stone game — n stones, can take 1..k stones per turn.
// Player who takes the last stone wins.
// State: number of stones remaining.
// Value: +1 if current player wins, -1 if current player loses.

unordered_map<int, int> memo;

int solve(int n, int k) {
    if (n == 0) return -1;  // no stones left — current player loses
    if (memo.count(n)) return memo[n];

    int best = -1;  // assume worst case
    for (int take = 1; take <= min(n, k); take++) {
        // Opponent plays next; negate their value
        best = max(best, -solve(n - take, k));
    }
    return memo[n] = best;
}

int main() {
    int n, k;
    cin >> n >> k;
    cout << (solve(n, k) == 1 ? "First" : "Second") << " player wins\n";
}

Complexity: $O(n \cdot k)$ time, $O(n)$ space. Each of the $n$ states is computed once, and each requires checking $k$ moves.

Coin Game on an Array

A classic game DP: given an array $a[0..n{-}1]$ of coin values, two players alternately pick a coin from either end. Maximize your total.

#include <bits/stdc++.h>
using namespace std;

int a[1001];
int dp[1001][1001];  // dp[i][j] = max advantage for current player on a[i..j]
bool vis[1001][1001];

// Returns net advantage of current player on subarray a[i..j]
int solve(int i, int j) {
    if (i > j) return 0;
    if (vis[i][j]) return dp[i][j];
    vis[i][j] = true;

    // Pick left coin: gain a[i], opponent plays optimally on a[i+1..j]
    int pick_left  = a[i] - solve(i + 1, j);
    // Pick right coin: gain a[j], opponent plays optimally on a[i..j-1]
    int pick_right = a[j] - solve(i, j - 1);

    return dp[i][j] = max(pick_left, pick_right);
}

int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];

    int total = 0;
    for (int i = 0; i < n; i++) total += a[i];

    int advantage = solve(0, n - 1);
    // First player's score = (total + advantage) / 2
    cout << "First player score: " << (total + advantage) / 2 << "\n";
}

The key trick is the negamax formulation: $\text{dp}[i][j]$ stores the current player's advantage (own score minus opponent's score) on the subarray $a[i..j]$. When you pick a coin worth $a[i]$, the opponent then gets advantage $\text{solve}(i{+}1, j)$, so your net advantage is $a[i] - \text{solve}(i{+}1, j)$.

Complexity: $O(n^2)$ states, $O(1)$ transitions each → $O(n^2)$ time and space.

Encoding Complex States

For games with more complex states, use tuples or hash maps:

// Multi-pile Nim with bounded moves: piles (a, b, c), take 1..k from any pile
map<tuple<int,int,int>, int> memo;

int solve(int a, int b, int c, int k) {
    if (a == 0 && b == 0 && c == 0) return -1;
    auto key = make_tuple(a, b, c);
    if (memo.count(key)) return memo[key];

    int best = -1;
    for (int t = 1; t <= k; t++) {
        if (a >= t) best = max(best, -solve(a - t, b, c, k));
        if (b >= t) best = max(best, -solve(a, b - t, c, k));
        if (c >= t) best = max(best, -solve(a, b, c - t, k));
    }
    return memo[key] = best;
}

Bottom-Up Game DP

Bottom-up (tabulation) fills the DP table iteratively from base cases to the answer. It avoids recursion overhead and stack overflow risks.

When to Prefer Bottom-Up

  • All states reachable: If most states are visited anyway, bottom-up avoids hash-map overhead.
  • State space is a grid or array: Easy to enumerate in order.
  • Memory optimization: Bottom-up often allows space compression (e.g., rolling arrays).
  • Deep recursion: When the game depth exceeds the stack limit.

Stone Game — Bottom-Up

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;

    // dp[i] = 1 if current player wins with i stones, -1 otherwise
    vector<int> dp(n + 1, -1);
    dp[0] = -1;  // no stones: current player loses

    for (int i = 1; i <= n; i++) {
        dp[i] = -1;
        for (int take = 1; take <= min(i, k); take++) {
            if (dp[i - take] == -1) {
                dp[i] = 1;  // opponent is in a losing position
                break;
            }
        }
    }

    cout << (dp[n] == 1 ? "First" : "Second") << " player wins\n";
}

Complexity: $O(nk)$ time, $O(n)$ space.

Coin Game — Bottom-Up Tabulation

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];

    // dp[i][j] = current player's advantage on a[i..j]
    vector<vector<int>> dp(n, vector<int>(n, 0));

    // Base cases: single elements
    for (int i = 0; i < n; i++)
        dp[i][i] = a[i];

    // Fill by increasing interval length
    for (int len = 2; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            dp[i][j] = max(a[i] - dp[i + 1][j],
                           a[j] - dp[i][j - 1]);
        }
    }

    int total = 0;
    for (int x : a) total += x;
    cout << "First player score: " << (total + dp[0][n - 1]) / 2 << "\n";
}

Complexity: $O(n^2)$ time and space. Space can be reduced to $O(n)$ with a rolling-array trick since each row only depends on the previous diagonal.

General Bottom-Up Pattern for Games

The key observation is that states form a DAG. Process them in reverse topological order (from terminal states backward):

  1. Initialize all terminal states with their utility values.
  2. For each non-terminal state in reverse topological order, compute its value using the recurrence.
  3. The answer is the value of the initial state.

For interval DP games, the natural order is by increasing interval length. For graph-based games, perform a topological sort of the game DAG first.

Alpha-Beta Pruning

Alpha-beta pruning is a branch-and-bound optimization of minimax that eliminates subtrees which cannot influence the final decision. It maintains two bounds:

  • $\alpha$: the best (highest) value the maximizer can guarantee so far along the path,
  • $\beta$: the best (lowest) value the minimizer can guarantee so far along the path.

When $\alpha \ge \beta$, we can prune: the current branch will never be selected by the opponent.

Formal Pruning Rules

At a maximizer node: If we find a child with value $v \ge \beta$, the minimizer (parent) would never allow this branch. Prune remaining children.

At a minimizer node: If we find a child with value $v \le \alpha$, the maximizer (grandparent) already has a better option. Prune remaining children.

Theorem: Alpha-beta pruning returns the same result as minimax.

Proof: We show that pruned branches cannot affect the minimax value. At a maximizer node with parent bound $\beta$: if a child yields $v \ge \beta$, the maximizer would choose $v$, but the minimizer parent already has an option $\le \beta \le v$, so it will not pick this branch. Hence the subtree is irrelevant. The symmetric argument holds for minimizer nodes. Since we only prune irrelevant subtrees, the computed value equals the true minimax value. $\blacksquare$

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

struct State {
    bool is_terminal() const;
    int utility() const;
    vector<State> moves() const;
};

int alpha_beta(const State& s, int alpha, int beta, bool is_max) {
    if (s.is_terminal())
        return s.utility();

    if (is_max) {
        int value = INT_MIN;
        for (const auto& next : s.moves()) {
            value = max(value, alpha_beta(next, alpha, beta, false));
            alpha = max(alpha, value);
            if (alpha >= beta)
                break;  // beta cutoff
        }
        return value;
    } else {
        int value = INT_MAX;
        for (const auto& next : s.moves()) {
            value = min(value, alpha_beta(next, alpha, beta, true));
            beta = min(beta, value);
            if (alpha >= beta)
                break;  // alpha cutoff
        }
        return value;
    }
}

// Usage:
// int game_value = alpha_beta(initial_state, INT_MIN, INT_MAX, true);

Complexity Analysis

In the best case (optimal move ordering), alpha-beta examines only $O(b^{d/2})$ nodes—an effective branching factor of $\sqrt{b}$. This means it can search twice as deep as plain minimax in the same time.

$$\text{Best case: } O(b^{d/2}), \qquad \text{Worst case: } O(b^d)$$

The best case is achieved when at every node, the best move is examined first. In practice, move ordering heuristics (e.g., examining captures first in chess, or ordering by heuristic evaluation) get close to this ideal.

Negamax with Alpha-Beta

The negamax formulation simplifies the code by always maximizing, and negating the returned value:

int negamax_ab(const State& s, int alpha, int beta) {
    if (s.is_terminal())
        return s.utility();  // from current player's perspective

    int value = INT_MIN;
    for (const auto& next : s.moves()) {
        value = max(value, -negamax_ab(next, -beta, -alpha));
        alpha = max(alpha, value);
        if (alpha >= beta)
            break;  // cutoff
    }
    return value;
}

// Usage:
// int val = negamax_ab(initial_state, INT_MIN, INT_MAX);

Negamax works because in a zero-sum game, one player's gain is the other's loss: $V_{\text{min}}(s) = -V_{\text{max}}(s)$. The alpha-beta window is negated and swapped when passing to the child.

Classic Game DP Problems

Problem 1: Stone Game (Leetcode 877)

Piles of stones $a[0..n{-}1]$ (even $n$). Two players pick from either end. Determine who wins with optimal play.

#include <bits/stdc++.h>
using namespace std;

bool stoneGame(vector<int>& piles) {
    int n = piles.size();
    // dp[i][j] = max net score advantage for current player on piles[i..j]
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int i = 0; i < n; i++)
        dp[i][i] = piles[i];

    for (int len = 2; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            dp[i][j] = max(piles[i] - dp[i + 1][j],
                           piles[j] - dp[i][j - 1]);
        }
    }
    return dp[0][n - 1] > 0;
}

int main() {
    int n;
    cin >> n;
    vector<int> piles(n);
    for (int& x : piles) cin >> x;
    cout << (stoneGame(piles) ? "First" : "Second") << " wins\n";
}

Complexity: $O(n^2)$ time and space.

Problem 2: Predict the Winner (Leetcode 486)

Given $n$ scores, two players alternately pick from either end. Determine if Player 1 can win. This uses the same advantage-based interval DP:

bool predictTheWinner(vector<int>& nums) {
    int n = nums.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));
    for (int i = 0; i < n; i++) dp[i][i] = nums[i];
    for (int len = 2; len <= n; len++)
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            dp[i][j] = max(nums[i] - dp[i+1][j], nums[j] - dp[i][j-1]);
        }
    return dp[0][n-1] >= 0;
}

Complexity: $O(n^2)$ time and space.

Bitmask DP for Games

When the game state involves a set of items (stones, cards, elements) and the order of removal doesn't matter (only which items remain), we can represent the state as a bitmask.

State Representation

For $n$ items, the state is an $n$-bit integer where bit $i$ is 1 if item $i$ is still in the game. This gives $2^n$ possible states.

$$\text{mask} = \sum_{i \in \text{remaining}} 2^i$$

Subset Enumeration

To enumerate all subsets of a bitmask (useful for partition games), we use the classic trick:

// Enumerate all non-empty subsets of mask
for (int sub = mask; sub > 0; sub = (sub - 1) & mask) {
    // sub is a subset of mask
}

// Enumerate all submasks of size exactly k
// (use Gosper's hack to enumerate k-element subsets of {0..n-1},
//  then intersect with mask)

Full Bitmask Game DP Template

#include <bits/stdc++.h>
using namespace std;

const int MAXN = 20;
int a[MAXN], n;
int dp[1 << MAXN];
bool vis[1 << MAXN];

// Game: n items, players alternate picking one item.
// Player wants to maximize own total (or equivalently, advantage).
// dp[mask] = advantage of current player given remaining items = mask
int solve(int mask) {
    if (mask == 0) return 0;
    if (vis[mask]) return dp[mask];
    vis[mask] = true;

    int best = INT_MIN;
    int tmp = mask;
    while (tmp) {
        int bit = tmp & (-tmp);       // lowest set bit
        int idx = __builtin_ctz(bit);  // index of that bit
        best = max(best, a[idx] - solve(mask ^ bit));
        tmp ^= bit;                    // remove that bit
    }
    return dp[mask] = best;
}

int main() {
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];

    int full = (1 << n) - 1;
    int total = 0;
    for (int i = 0; i < n; i++) total += a[i];

    int adv = solve(full);
    cout << "First: " << (total + adv) / 2 << "\n";
    cout << "Second: " << (total - adv) / 2 << "\n";
}

Complexity: $O(2^n \cdot n)$ time — each of $2^n$ masks processes up to $n$ set bits. Space is $O(2^n)$.

Worked Example: Divisor Game Variant

Given numbers $1, 2, \ldots, n$ on a board. Players alternately pick a number $x$ and remove $x$ along with all its divisors. Player who cannot move loses.

#include <bits/stdc++.h>
using namespace std;

int n;
int div_mask[21];  // bitmask of divisors of x
int dp[1 << 21];
bool vis[1 << 21];

void precompute() {
    for (int x = 1; x <= n; x++) {
        div_mask[x] = 0;
        for (int d = 1; d <= x; d++)
            if (x % d == 0) div_mask[x] |= (1 << d);
    }
}

int solve(int mask) {
    if (vis[mask]) return dp[mask];
    vis[mask] = true;
    dp[mask] = 0;
    for (int x = 1; x <= n; x++) {
        if (!(mask & (1 << x))) continue;
        if ((mask & div_mask[x]) != div_mask[x]) continue;
        if (!solve(mask ^ div_mask[x])) { dp[mask] = 1; break; }
    }
    return dp[mask];
}

int main() {
    cin >> n;
    precompute();
    int full = 0;
    for (int i = 1; i <= n; i++) full |= (1 << i);
    cout << (solve(full) ? "First" : "Second") << " player wins\n";
}

Complexity: $O(2^n \cdot n)$ time, $O(2^n)$ space. Practical for $n \le 20$.
