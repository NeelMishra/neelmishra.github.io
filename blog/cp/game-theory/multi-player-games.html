Multi-Player Games — Coalitions, Non-Zero-Sum & Beyond Two Players

June 2026 · 25 min read

Throughout this series we have built a powerful toolkit — Sprague-Grundy theory, Nim values, surreal numbers — but every result so far assumes exactly two players taking turns in a zero-sum, perfect-information game. In competitive programming (and in the real world) many situations involve three or more players, the possibility of coalitions, or payoffs that don't sum to zero. This post explores when the two-player theory extends cleanly, when it breaks, and what new tools we need.

Beyond Two Players

Why Sprague-Grundy Breaks

The Sprague-Grundy theorem rests on a key dichotomy: every position is either a win for the current player (N-position) or a loss (P-position). With two players, "my loss" is "your win" — the game is strictly antagonistic. With three or more players this dichotomy collapses.

Theorem (Failure of SG with 3+ Players)

In a combinatorial game with $n \geq 3$ players, a position cannot in general be classified as winning or losing for the current player independently of future players' strategies, because those strategies depend on relative standings and potential alliances.

Argument. Consider a 3-player game where Player A is about to lose. Players B and C each prefer to eliminate A first, but they disagree on moves that affect each other. B might prefer a move that hurts C, and C might prefer the opposite. Player A can exploit this disagreement: by choosing moves that change the B-vs-C dynamic, A can make one of them prefer to keep A alive. The "value" of A's position thus depends on B and C's relationship — not just on the game graph.

This means there is no function $\mathcal{G}: \text{Positions} \to \text{Ordinals}$ that determines the outcome independently of opponents' play, unlike the two-player Sprague-Grundy value. ■

What Still Works

Some structural properties do survive:

  • Backward induction still applies — we can compute optimal strategies from terminal positions, but the "optimal" strategy for each player may depend on others' strategies (leading to Nash equilibrium concepts).
  • Impartial games with alliance-free play — if every player plays purely to maximize their own outcome with no coordination, some Nim-like results extend.
  • Sum decomposition — games that decompose into independent components can still be analyzed component-by-component in restricted cases.

The Three Key Challenges

Challenge Two-Player Multi-Player
Win/loss classification Binary: W or L Outcome depends on alliances
Grundy values Single ordinal per position Needs vector of values
Nim-value XOR XOR of components = 0 iff P-position XOR theory breaks in general

Multi-Player Nim

Setup

Classic Nim has two players alternating turns, removing stones from heaps. The multi-player extension has $k \geq 3$ players taking turns in a fixed cyclic order: Player 1, Player 2, …, Player $k$, Player 1, …. The last player to move wins (normal play convention), or equivalently, the player who cannot move loses.

When XOR Theory Still Applies

Theorem (Li, 1978 — Multi-Player Nim with Alliance-Free Play)

In $k$-player Nim under the alliance-free assumption (each player independently tries to avoid being the loser), the XOR characterization extends if and only if $k = 2$. For $k \geq 3$, the XOR of heap sizes does not determine the outcome.

Proof sketch. For $k = 2$, the standard proof applies: XOR = 0 is maintained by the losing player's responses. For $k = 3$, consider heaps $(1, 1, 1)$. XOR = $1 \oplus 1 \oplus 1 = 1 \neq 0$. Player 1 takes the stone from heap 1, leaving $(0, 1, 1)$. Player 2 takes from heap 2, leaving $(0, 0, 1)$. Player 3 takes the last stone and wins. But if Player 1 had instead taken from heap 3, the dynamics shift entirely depending on Player 2's choice. The outcome is not a function of XOR alone. ■

Propp's Multi-Player Nim Analysis

James Propp showed that for 3-player Nim, defining "rational play" requires a notion of stable alliances. His key insight: in a 3-player game, two weaker players often have incentive to gang up on the strongest. The solution concept shifts from minimax to Nash equilibrium.

CP Relevance. Multi-player Nim rarely appears directly in contests, but the concept of analyzing positions with more than two outcomes appears in problems where you must determine which of several players wins, or in scoring games where you maximize your own score.

Vector Grundy Values

For $k$-player games, we replace the single Grundy number with a Grundy vector $\mathbf{g}(P) \in \{0, 1, \ldots\}^k$ where entry $i$ indicates the "value" of position $P$ from Player $i$'s perspective. Formally:

$$ \mathbf{g}(P) = (g_1(P), g_2(P), \ldots, g_k(P)) $$

where $g_i(P) = 0$ if Player $i$ loses under optimal play from all players, and $g_i(P) > 0$ otherwise. The difficulty is that "optimal play" is no longer well-defined without specifying a solution concept (minimax generalizes to maximin for each player, but alliances complicate this).

Coalition Formation

Why Coalitions Matter

In any game with $k \geq 3$ players, there is an incentive for subsets of players to coordinate their moves. A coalition is a subset $S \subseteq \{1, 2, \ldots, k\}$ of players who agree to coordinate their strategies to improve their joint outcome. Coalition formation is central to cooperative game theory.

The Core and Stability

Definition (Characteristic Function)

A characteristic function game is a pair $(N, v)$ where $N = \{1, \ldots, k\}$ is the set of players and $v: 2^N \to \mathbb{R}$ assigns a value to each coalition $S \subseteq N$, with $v(\emptyset) = 0$. The value $v(S)$ represents the total payoff that coalition $S$ can guarantee regardless of the actions of $N \setminus S$.

An imputation is a payoff vector $\mathbf{x} = (x_1, \ldots, x_k)$ such that:

  • Individual rationality: $x_i \geq v(\{i\})$ for all $i$ — no player gets less than they could guarantee alone.
  • Efficiency: $\sum_{i=1}^k x_i = v(N)$ — the grand coalition distributes all value.

The core of the game is the set of imputations where no coalition has incentive to deviate:

$$ \text{Core}(N, v) = \left\{ \mathbf{x} \in \mathbb{R}^k : \sum_{i \in S} x_i \geq v(S) \;\forall\; S \subseteq N, \; \sum_{i=1}^k x_i = v(N) \right\} $$
The core can be empty! Not every game has a stable allocation. For example, the 3-player majority game where any two players can win has an empty core — for any allocation, some pair of players can do better by deviating.

Nash Equilibrium in Multi-Player Games

A Nash equilibrium is a strategy profile where no single player can improve their payoff by unilaterally changing their strategy. Formally, a strategy profile $\sigma^* = (\sigma_1^*, \ldots, \sigma_k^*)$ is a Nash equilibrium if for all players $i$ and all alternative strategies $\sigma_i'$:

$$ u_i(\sigma_i^*, \sigma_{-i}^*) \geq u_i(\sigma_i', \sigma_{-i}^*) $$

where $\sigma_{-i}^*$ denotes all other players' strategies and $u_i$ is player $i$'s payoff function.

Theorem (Nash, 1950)

Every finite game (finite players, finite strategies each) has at least one Nash equilibrium in mixed strategies.

Proof sketch. Define best-response correspondences $B_i: \Delta_{-i} \to \Delta_i$ mapping opponents' mixed strategy profiles to the set of player $i$'s best responses. The product $B = B_1 \times \cdots \times B_k$ maps the product simplex to itself. By Kakutani's fixed-point theorem (B is upper hemicontinuous with convex values on a compact convex set), $B$ has a fixed point. A fixed point of $B$ is precisely a Nash equilibrium. ■

Computing Nash Equilibria in Small Games

For $k$-player games, the support enumeration method checks all possible support combinations: (1) enumerate supports $S_i \subseteq \{1, \ldots, n_i\}$ per player, (2) solve indifference equations for each combination, (3) verify no player benefits from switching to an out-of-support strategy. The complexity is exponential but feasible for small games in CP ($\leq 10$ strategies per player).

Non-Zero-Sum Games

Zero-Sum vs. Non-Zero-Sum

In a zero-sum game, the total payoff across all players is constant (usually zero): one player's gain is another's loss. In a non-zero-sum game, the total payoff varies by outcome — cooperation can create value, and conflict can destroy it.

Prisoner's Dilemma (2-Player Non-Zero-Sum)
B: CooperateB: Defect
A: Cooperate(-1, -1)(-3, 0)
A: Defect(0, -3)(-2, -2)

The unique Nash equilibrium is (Defect, Defect) with payoff $(-2, -2)$, even though (Cooperate, Cooperate) yields a better joint outcome $(-1, -1)$. The total payoff is $-4$ at equilibrium vs. $-2$ at the cooperative outcome — hence non-zero-sum.

Formal Definition

A normal-form game is a tuple $(N, (A_i)_{i \in N}, (u_i)_{i \in N})$ where:

  • $N = \{1, \ldots, k\}$ is the set of players.
  • $A_i$ is the set of actions (pure strategies) for player $i$.
  • $u_i: A_1 \times \cdots \times A_k \to \mathbb{R}$ is player $i$'s payoff function.

The game is zero-sum if $\sum_{i=1}^k u_i(a) = 0$ for all $a \in A_1 \times \cdots \times A_k$. It is constant-sum if $\sum_{i=1}^k u_i(a) = c$ for some constant $c$.

Multi-Player Prisoner's Dilemma

The $k$-player generalization: each player simultaneously chooses Cooperate or Defect. The payoff to player $i$ depends on their own choice and the number of cooperators $c$:

$$ u_i = \begin{cases} b \cdot c - d & \text{if player } i \text{ cooperates} \\ b \cdot c & \text{if player } i \text{ defects} \end{cases} $$

where $b > 0$ is the benefit per cooperator and $d > b$ is the cost of cooperating. Since $d > b$, defecting is always a dominant strategy (regardless of what others do, defecting gives $d$ more utility). The unique Nash equilibrium is all-defect, even though all-cooperate yields total payoff $k(b \cdot k - d)$ vs. all-defect total of $0$.

Proposition (Dominant Strategy in k-Player PD)

In the $k$-player Prisoner's Dilemma with $d > b > 0$, Defect is a strictly dominant strategy for every player.

Proof. Fix any strategy profile for all players except $i$, with $c'$ cooperators among them. If $i$ cooperates, $u_i = b(c' + 1) - d$. If $i$ defects, $u_i = b \cdot c'$. The difference is $u_i(\text{Defect}) - u_i(\text{Cooperate}) = d - b > 0$. Since this holds for all $c'$, Defect strictly dominates Cooperate. ■

Relevance to CP

Non-zero-sum aspects appear in competitive programming when:

  • Scoring problems: Multiple agents maximize their own scores; total score varies by play sequence.
  • Resource allocation: Players distribute resources; cooperation can increase total output.
  • Sequential games with payoffs: Problems where you compute the optimal strategy vector for each player via backward induction on a game tree.

Scoring Games

Definition and Motivation

A scoring game is a combinatorial game where each player accumulates points during play, and the winner is the player with the highest score at the end (rather than the last player to move). This is fundamentally different from standard combinatorial game theory:

  • There is no "last player wins" — all positions eventually terminate.
  • The objective is to maximize your own score, not just force the opponent to lose.
  • With $k$ players, each has their own score accumulator.

Formal Model

A $k$-player scoring game on a directed acyclic graph $G = (V, E)$ is defined by:

  • A turn function $\tau: V \to \{1, \ldots, k\}$ assigning each non-terminal vertex to a player.
  • A scoring function $s: E \to \mathbb{R}^k$ where $s(e)_i$ is the score added to Player $i$'s total when edge $e$ is traversed.

At vertex $v$ with $\tau(v) = j$, Player $j$ chooses an outgoing edge $e = (v, w)$, scores $s(e)$ are added to all players' totals, and play moves to $w$. The game ends at terminal vertices (those with no outgoing edges).

Backward Induction for Scoring Games

We compute the value vector $\mathbf{V}(v) = (V_1(v), \ldots, V_k(v))$ for each vertex, where $V_i(v)$ is Player $i$'s total score from $v$ onward under optimal play:

$$ \mathbf{V}(v) = \begin{cases} \mathbf{0} & \text{if } v \text{ is terminal} \\ \displaystyle\arg\max_{(v,w) \in E} \left[ s((v,w))_{\tau(v)} + V_{\tau(v)}(w) \right] & \text{choosing the edge maximizing current player's total} \end{cases} $$

More precisely: at vertex $v$ where Player $j = \tau(v)$ moves, Player $j$ picks the edge $(v, w^*)$ that maximizes $s((v,w))_j + V_j(w)$. The value vector is then:

$$ \mathbf{V}(v) = s((v, w^*)) + \mathbf{V}(w^*) $$
Key insight: Even though Player $j$ chooses to maximize only their own score, the value vector tracks all players' scores. This is critical — in a scoring game, a move that maximizes your score might also change what your opponents accumulate.

Example: 3-Player Token Game

Three players take turns picking tokens from a shared pool. Each token has a value visible to all players. On your turn, you take one token. Goal: maximize your total.

With $n$ tokens and values $a_1 \geq a_2 \geq \cdots \geq a_n$ (sorted descending), the greedy strategy is optimal: each player takes the highest-value remaining token. Player 1 gets tokens at positions $1, 4, 7, \ldots$, Player 2 gets $2, 5, 8, \ldots$, and Player 3 gets $3, 6, 9, \ldots$.

Proposition (Greedy Optimality in Token Game)

In the $k$-player token game with visible values, the greedy strategy (always pick the highest remaining value) is a Nash equilibrium.

Proof. Suppose all players play greedily except Player $i$, who deviates at some turn by picking token $a_j$ instead of the maximum available $a_m$ where $a_m > a_j$. Player $i$'s total decreases by at least $a_m - a_j > 0$ on this turn. The remaining tokens available to Player $i$ in subsequent turns are at most as good (since $a_m$ is now available for others to take instead of $a_j$). Therefore the deviation is not profitable. ■

C++ Solutions

Multi-Player Game Tree Solver

The following implementation solves a general $k$-player scoring game on a DAG using backward induction. Each node stores which player moves, and each edge stores a score vector. 

#include <bits/stdc++.h>
using namespace std;

struct Edge {
    int to;
    vector<double> score; // score[i] = score added to player i
};

struct Node {
    int player;           // which player moves at this node (-1 if terminal)
    vector<Edge> edges;
};

// Returns value vector for each node under optimal play (backward induction)
// Nodes must be in topological order (0..n-1), with terminals having no edges
vector<vector<double>> solve_scoring_game(const vector<Node>& nodes, int k) {
    int n = nodes.size();
    vector<vector<double>> val(n, vector<double>(k, 0.0));

    // Process in reverse topological order
    for (int v = n - 1; v >= 0; v--) {
        if (nodes[v].edges.empty()) continue; // terminal: val = 0

        int p = nodes[v].player;
        double best = -1e18;
        int best_edge = -1;

        for (int i = 0; i < (int)nodes[v].edges.size(); i++) {
            const Edge& e = nodes[v].edges[i];
            double candidate = e.score[p] + val[e.to][p];
            if (candidate > best) {
                best = candidate;
                best_edge = i;
            }
        }

        const Edge& chosen = nodes[v].edges[best_edge];
        for (int i = 0; i < k; i++) {
            val[v][i] = chosen.score[i] + val[chosen.to][i];
        }
    }
    return val;
}

Nash Equilibrium Finder (2-Player Bimatrix)

For small normal-form games, we can find all pure-strategy Nash equilibria by checking each strategy profile: 

// Find all pure-strategy Nash equilibria of a 2-player game
// payoff_a[i][j] = Player A's payoff when A plays i, B plays j
// payoff_b[i][j] = Player B's payoff
vector<pair<int,int>> find_pure_nash(
    const vector<vector<double>>& payoff_a,
    const vector<vector<double>>& payoff_b)
{
    int m = payoff_a.size();    // A's strategies
    int n = payoff_a[0].size(); // B's strategies
    vector<pair<int,int>> equilibria;

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            // Check if i is A's best response to j
            bool a_best = true;
            for (int i2 = 0; i2 < m; i2++) {
                if (payoff_a[i2][j] > payoff_a[i][j]) {
                    a_best = false;
                    break;
                }
            }
            // Check if j is B's best response to i
            bool b_best = true;
            for (int j2 = 0; j2 < n; j2++) {
                if (payoff_b[i][j2] > payoff_b[i][j]) {
                    b_best = false;
                    break;
                }
            }
            if (a_best && b_best) {
                equilibria.push_back({i, j});
            }
        }
    }
    return equilibria;
}

k-Player Token Game (Greedy)

// Solve the k-player token picking game
// tokens: values of tokens (unsorted)
// k: number of players
// Returns score[i] = total for player i
vector<long long> token_game(vector<int> tokens, int k) {
    sort(tokens.rbegin(), tokens.rend());
    vector<long long> score(k, 0);
    for (int i = 0; i < (int)tokens.size(); i++) {
        score[i % k] += tokens[i];
    }
    return score;
}

Coalition Value Computation

Given $k$ players and a value function over subsets, compute the core constraints and check if a given allocation is in the core: 

// Check if allocation x is in the core of game (N, v)
// v[mask] = value of coalition represented by bitmask
// x[i] = allocation to player i
bool is_in_core(const vector<double>& v, const vector<double>& x, int k) {
    int full = (1 << k) - 1;

    // Check efficiency: sum(x) == v(N)
    double total = 0;
    for (int i = 0; i < k; i++) total += x[i];
    if (abs(total - v[full]) > 1e-9) return false;

    // Check coalition rationality
    for (int mask = 1; mask <= full; mask++) {
        double coalition_sum = 0;
        for (int i = 0; i < k; i++) {
            if (mask & (1 << i)) coalition_sum += x[i];
        }
        if (coalition_sum < v[mask] - 1e-9) return false;
    }
    return true;
}

// Compute Shapley value for each player
vector<double> shapley_value(const vector<double>& v, int k) {
    vector<double> phi(k, 0.0);
    // For each permutation, compute marginal contributions
    vector<int> perm(k);
    iota(perm.begin(), perm.end(), 0);

    int count = 0;
    do {
        int mask = 0;
        for (int pos = 0; pos < k; pos++) {
            int i = perm[pos];
            int prev_mask = mask;
            mask |= (1 << i);
            phi[i] += v[mask] - v[prev_mask]; // marginal contribution
        }
        count++;
    } while (next_permutation(perm.begin(), perm.end()));

    for (int i = 0; i < k; i++) phi[i] /= count;
    return phi;
}
Complexity note. The Shapley value computation above is $O(k! \cdot k)$ which is feasible for $k \leq 12$. For larger $k$, use the formula: $$\phi_i(v) = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|! \cdot (k - |S| - 1)!}{k!} \left[ v(S \cup \{i\}) - v(S) \right]$$ which iterates over $2^{k-1}$ subsets — still exponential but avoids generating permutations.

Hints and Approaches

  1. Three-player Nim: Enumerate small cases. Show that XOR does not determine the outcome by finding positions where the first player wins with XOR = 0.
  2. Arpa and a game: Model each number's game independently, compute Grundy values, and handle the multi-outcome structure.
  3. Balance: Formulate as a non-zero-sum optimization; use DP to track both players' accumulated scores.
  4. Game in Momotetsu World: Classic grid DP where two players alternately pick directions, tracking score difference.
  5. Tree-Game: Backward induction on a tree structure; each node's value is computed from children.
  6. Removing Graph: Analyze which coalitions can form based on graph structure; reduce to a cooperative game.
  7. Shiritori: Bitmask DP over the game state; determine each player's optimal play in a multi-round word game.
