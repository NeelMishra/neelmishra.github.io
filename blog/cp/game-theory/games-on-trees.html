Games on Trees

June 2026 · 25 min read

Tree Games Overview

Trees are the most natural setting for combinatorial games. Their acyclic structure gives every game a finite horizon, and their recursive decomposition makes Sprague–Grundy analysis tractable. In this post we study Green Hackenbush on trees — a foundational game that connects tree structure to Nim values via the elegant Colon Principle.

A combinatorial game on a tree is defined by:

  • A rooted tree $T$ with a distinguished ground vertex (the root).
  • Edges represent game tokens. Players alternate removing (deleting) an edge.
  • When an edge is removed, every edge no longer connected to the ground is also removed.
  • The player who cannot move (no edges remain) loses — this is the normal play convention.

Why trees? Three key reasons:

1. Finite and acyclic. Every path from a leaf to the ground is finite, guaranteeing termination. No cycles mean no infinite loops of play.
2. Recursive decomposition. Removing the root edge splits a tree into independent subtrees — directly yielding a sum of games, to which Sprague–Grundy theory applies.
3. Efficient computation. Grundy values on trees can be computed in $O(n)$ time with a single DFS, making them ideal for competitive programming.

Before diving into analysis, let's establish the basic structure. Consider a tree rooted at the ground vertex $g$:

Figure 1: A tree rooted at the ground (GND). Six edges, each a valid move.

Removing edge $(v, a)$ disconnects vertices $a, c, d$ — all three edges incident to them vanish. This cascading removal is what makes Hackenbush games on trees structurally rich and strategically deep.

Green Hackenbush on Trees

Green Hackenbush is the impartial variant of Hackenbush — both players can remove any edge. On a general graph this can be complex, but on trees the analysis is remarkably clean.

Definition. Given a tree $T$ rooted at the ground, the Grundy value of vertex $v$ (written $\mathcal{G}(v)$) is defined recursively:

$$\mathcal{G}(v) = \begin{cases} 0 & \text{if } v \text{ is a leaf (no children)} \\ \bigoplus_{u \in \text{children}(v)} (\mathcal{G}(u) + 1) & \text{otherwise} \end{cases}$$

Here $\oplus$ denotes XOR (nim-sum) and $+1$ accounts for the edge from $v$ to each child $u$. Each child subtree rooted at $u$ contributes a nim-heap of size $\mathcal{G}(u) + 1$.

Intuition. A single path of length $k$ from a vertex to the ground is equivalent to a Nim heap of size $k$. A vertex with multiple branches combines them via XOR — exactly as in a sum of Nim heaps.

Example. Consider a path of three edges (a "bamboo stalk" of length 3):

Figure 2: Bamboo stalk of length 3. Grundy values bottom-up: leaf = 0, then 0+1 = 1, then 1+1 = 2.

The Grundy value of the entire game (at the ground's child) is $\mathcal{G} + 1 = 2 + 1 = 3$, which equals the path length — confirming that a bamboo stalk of length $k$ is a Nim heap $*k$.

Branching example. Consider vertex $a$ with two children: a leaf $c$ and a leaf $d$.

  • $\mathcal{G}(c) = 0$ (leaf), so child $c$ contributes $0 + 1 = 1$.
  • $\mathcal{G}(d) = 0$ (leaf), so child $d$ contributes $0 + 1 = 1$.
  • $\mathcal{G}(a) = 1 \oplus 1 = 0$.

Now vertex $a$ contributes $\mathcal{G}(a) + 1 = 1$ to its parent — the entire subtree rooted at $a$ with two leaf children is equivalent to a single edge!

Key insight: A vertex with two children, each a single leaf, has Grundy value $1 \oplus 1 = 0$. The whole subtree collapses to a Nim heap $*1$ (one edge). This is the Colon Principle in action.

The Colon Principle

The Colon Principle (due to Berlekamp, Conway, and Guy) is the central theorem for Green Hackenbush on trees. It lets us reduce any tree to a single edge (a single Nim heap), making game evaluation trivial.

Theorem (Colon Principle)

Let $v$ be a vertex in a Green Hackenbush position (connected to the ground), and let $B_1, B_2, \ldots, B_k$ be the branches at $v$ (each $B_i$ is a subtree rooted at a child of $v$, including the edge from $v$ to that child). Suppose branch $B_i$ has Grundy value $g_i$. Then the entire collection of branches at $v$ can be replaced by a single edge of Grundy value:

$$g_1 \oplus g_2 \oplus \cdots \oplus g_k$$

without changing the Grundy value of the overall game. Formally, if $G$ is the original game and $G'$ is obtained by replacing all branches at $v$ with one bamboo stalk of length $g_1 \oplus g_2 \oplus \cdots \oplus g_k$, then $\mathcal{G}(G) = \mathcal{G}(G')$.

Before the proof, let's see what this means visually:

Figure 3: Colon Principle — three branches at v with Grundy values 2, 1, 2 collapse to a single edge of value 2⊕1⊕2 = 1.

Proof of the Colon Principle

We prove this by showing that the game $G$ (original) and $G'$ (reduced) have the same Grundy value. We proceed by strong induction on the total number of edges in the branches at $v$.

Base case. If $v$ has a single branch $B_1$ that is a single edge (bamboo of length 1), then $g_1 = 1$ and the replacement is also a single edge of value 1. The games are identical. ✓

Inductive step. Assume the Colon Principle holds for all configurations with fewer total branch edges. Let $v$ have branches $B_1, \ldots, B_k$ with Grundy values $g_1, \ldots, g_k$, and let $g = g_1 \oplus \cdots \oplus g_k$.

We need to show $\mathcal{G}(G) = \mathcal{G}(G')$ where $G'$ replaces all branches with a bamboo stalk of length $g$ attached at $v$.

Strategy: We exhibit a winning strategy for the second player in the game $G - G'$ (the difference game), proving $G - G' = 0$ (a P-position).

Case 1: First player moves in the branches at $v$ in $G$.

Suppose the first player removes an edge in branch $B_i$, changing its Grundy value from $g_i$ to some $g_i'$. The new XOR of branches is $g' = g_1 \oplus \cdots \oplus g_i' \oplus \cdots \oplus g_k$. By the inductive hypothesis (the total edges decreased), the branches can now be replaced by a bamboo of length $g'$. The second player's response in $G'$ is to change the bamboo from length $g$ to length $g'$. This is always possible since $g' \neq g$ (the first player changed something) and $g' < g$ or we can find an appropriate edge to cut in the bamboo of length $g$ to leave length $g'$.

More precisely: the bamboo of length $g$ in $G'$ is a Nim heap $*g$. From $*g$, any value $*g'$ with $g' < g$ is reachable by removing the edge at depth $g' + 1$. If $g' > g$, then by standard Nim theory applied to the difference game, the second player can always mirror to maintain the invariant.

Case 2: First player moves in the bamboo in $G'$.

The first player cuts the bamboo from length $g$ to some length $g'$ where $0 \leq g' < g$. The second player must adjust the branches in $G$ so their XOR becomes $g'$. Since $g' \oplus g \neq 0$, let $d = g' \oplus g$. There exists some branch $B_i$ such that $g_i \oplus d < g_i$ (by the standard Nim XOR argument — the highest set bit of $d$ must appear in some $g_i$). The second player moves in $B_i$ to change its Grundy value to $g_i \oplus d$. The new XOR is:

$$g_1 \oplus \cdots \oplus (g_i \oplus d) \oplus \cdots \oplus g_k = g \oplus d = g \oplus g' \oplus g = g'$$

The move in $B_i$ from $g_i$ to $g_i \oplus d < g_i$ is achievable because $\mathcal{G}(B_i) = g_i$ means all values $0, 1, \ldots$ up to $g_i - 1$ are reachable from $B_i$ (by definition of Grundy values).

Case 3: First player cuts the edge connecting $v$ to the ground (in either component).

This removes the entire tree/bamboo above $v$. The second player mirrors by cutting the same edge in the other component, leaving both components empty.

In all cases the second player has a legal reply maintaining $G - G' = 0$. Therefore $\mathcal{G}(G) = \mathcal{G}(G')$.

∎

Corollary. The Grundy value of any rooted Green Hackenbush tree equals the Grundy value computed by the bottom-up formula:

$$\mathcal{G}(v) = \bigoplus_{u \in \text{children}(v)} \big(\mathcal{G}(u) + 1\big)$$

This is because we can apply the Colon Principle repeatedly from the leaves upward, reducing each internal vertex's branches to a single edge, until the entire tree becomes a single bamboo stalk.

Tree Reduction Algorithm

The Colon Principle gives us a systematic algorithm to reduce any tree to a single Nim heap. Here's the step-by-step procedure:

Step 1: Identify leaves

Find all leaf vertices (vertices with no children). Each leaf has $\mathcal{G} = 0$.

Step 2: Process leaves bottom-up

For each internal vertex $v$ whose children are all leaves (or already processed), compute $\mathcal{G}(v) = \bigoplus (\mathcal{G}(\text{child}) + 1)$. By the Colon Principle, replace all branches at $v$ with a single bamboo of length $\mathcal{G}(v)$.

Step 3: Repeat until root

Continue upward until only the root's child vertex remains. The tree is now a bamboo of length $\mathcal{G}(\text{root's child}) + 1$, which is the game value.

Bamboo Stalks and Flowers

Certain tree shapes appear so frequently in Hackenbush problems that they deserve special names and known Grundy values.

Bamboo Stalks

A bamboo stalk is a path — a tree with no branching. A bamboo of $k$ edges has Grundy value $k$:

$$\text{Bamboo}(k) \equiv *k$$

This is trivially a Nim heap. The first player wins iff $k > 0$ (by removing the edge closest to the ground).

Stars (Flowers)

A star $S_k$ (also called a flower) is a vertex connected to $k$ leaves — a tree of depth 1 with $k$ children:

Figure 6: Stars S₁, S₂, S₃. Each leaf contributes (0+1) = 1, so the central vertex XORs k ones.

The Grundy value of $S_k$ (at the central vertex) is:

$$\mathcal{G}(S_k) = \underbrace{1 \oplus 1 \oplus \cdots \oplus 1}_{k \text{ times}} = \begin{cases} 0 & \text{if } k \text{ is even} \\ 1 & \text{if } k \text{ is odd} \end{cases}$$

The edge from the center to the ground adds 1, so the game value is $\mathcal{G}(S_k) + 1$:

Caterpillars

A caterpillar is a tree where all vertices are within distance 1 of a central path (the "spine"). Each spine vertex $v_i$ may have $d_i$ additional leaf children. By the Colon Principle, each spine vertex first resolves its leaves:

$$\mathcal{G}(v_i) = \underbrace{1 \oplus \cdots \oplus 1}_{d_i} \oplus (\text{contribution from next spine vertex})$$

The leaves at $v_i$ contribute $d_i \bmod 2$ to the XOR, effectively toggling the parity. This means a caterpillar's Grundy value depends on the spine length and the parity of leaf counts at each spine vertex.

Paths with Pendant Edges

A common pattern: a path of length $\ell$ with single pendant edges attached at various points. Each pendant edge contributes an extra XOR of 1 at its attachment point. The key formula:

$$\mathcal{G} = \ell + \sum_{i : v_i \text{ has pendant}} (\text{parity adjustment at } v_i)$$

But it's usually easier to just apply the DFS algorithm directly rather than deriving closed forms for complex tree shapes.

C++ Implementation

Here's a complete implementation that computes the Grundy value of a Green Hackenbush tree game. The algorithm runs a single DFS in $O(n)$ time.

#include <bits/stdc++.h>
using namespace std;

// Compute Grundy value of the subtree rooted at 'u'
// parent = -1 for the root call
int grundy(int u, int parent, const vector<vector<int>>& adj) {
    int g = 0;
    for (int v : adj[u]) {
        if (v == parent) continue;
        // Each child subtree contributes grundy(child) + 1
        g ^= (grundy(v, u, adj) + 1);
    }
    return g;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;  // 0-indexed
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // Ground is vertex 0; game value is grundy(0, -1, adj)
    int g = grundy(0, -1, adj);
    cout << (g ? "First" : "Second") << " player wins\n";
    cout << "Grundy value: " << g << "\n";

    return 0;
}

Handling forests. For a forest (multiple trees sharing the same ground), the total Grundy value is the XOR of individual tree values — the grundy function above handles this automatically since each child of the ground root is an independent subtree.

Iterative version (avoids stack overflow for $n > 10^5$):

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // BFS to find processing order (leaves first)
    vector<int> parent(n, -1), order;
    vector<bool> visited(n, false);
    queue<int> q;
    q.push(0);
    visited[0] = true;

    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                parent[v] = u;
                q.push(v);
            }
        }
    }

    // Process in reverse BFS order (leaves first)
    vector<int> g(n, 0);
    for (int i = n - 1; i >= 0; i--) {
        int u = order[i];
        g[u] = 0;
        for (int v : adj[u]) {
            if (v != parent[u]) {
                g[u] ^= (g[v] + 1);
            }
        }
    }

    cout << (g[0] ? "First" : "Second") << " player wins\n";
    cout << "Nim value: " << g[0] << "\n";

    return 0;
}
Complexity. Both versions run in $O(n)$ time and $O(n)$ space. The iterative version is preferred for $n > 10^5$ to avoid recursion depth issues.

Finding the winning move. If $g \neq 0$, find a child $c$ of the root where $(g[c]+1) \oplus g[\text{root}] < g[c]+1$. Cut in that subtree to reduce its contribution to the target value:

void find_move(int root, const vector<vector<int>>& adj,
               const vector<int>& g) {
    for (int c : adj[root]) {
        int old_val = g[c] + 1;
        int target = g[root] ^ old_val;
        if (target < old_val) {
            cout << "Cut in subtree of " << c + 1
                 << ": reduce value from " << old_val
                 << " to " << target << "\n";
            return;
        }
    }
}

The Colon Principle transforms trees — which could have exponentially many positions — into single Nim heaps computable in linear time. Combined with DFS, it gives competitive programmers a powerful tool for solving Hackenbush and related tree games efficiently.
