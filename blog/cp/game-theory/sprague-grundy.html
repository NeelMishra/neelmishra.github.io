    Sprague-Grundy Theorem

    June 2026 · 35 min read

    The Sprague-Grundy theorem is one of the most powerful results in combinatorial game theory. It shows that every impartial game under normal play is equivalent to a single Nim heap. This transforms the analysis of complex games into a single number — the Grundy value (or nimber) — and reduces multi-component games to a XOR computation.

    We assume familiarity with impartial games (both players have the same moves available) under normal play convention (the last player to move wins). We also assume all games are short: every sequence of moves eventually terminates.

    The Mex Function

    Definition

    The minimum excludant (mex) of a set $S \subseteq \mathbb{N}_0$ of non-negative integers is the smallest non-negative integer not in $S$:

    $$\operatorname{mex}(S) = \min\{n \in \mathbb{N}_0 : n \notin S\}$$

    Examples

    • $\operatorname{mex}(\emptyset) = 0$ — the empty set excludes $0$.
    • $\operatorname{mex}(\{0\}) = 1$.
    • $\operatorname{mex}(\{0, 1, 2\}) = 3$.
    • $\operatorname{mex}(\{1, 2, 5\}) = 0$ — $0$ is missing, so it is the mex.
    • $\operatorname{mex}(\{0, 1, 3, 4\}) = 2$.
    • $\operatorname{mex}(\{0, 2, 4, 6\}) = 1$.

    Properties

    Property 1. $\operatorname{mex}(S) \leq |S|$. The mex of a finite set of size $n$ is at most $n$, since we can have at most the values $\{0, 1, \ldots, n-1\}$ in $S$.

    Property 2. $\operatorname{mex}(S) = 0$ if and only if $0 \notin S$.

    Property 3. For any $k < \operatorname{mex}(S)$, we have $k \in S$. That is, every value smaller than the mex belongs to $S$.

    C++ implementation:

    int mex(const vector<int>& s) {
    unordered_set<int> st(s.begin(), s.end());
    int m = 0;
    while (st.count(m)) ++m;
    return m;
}

// Optimized for small values (common in game theory)
int mex_small(const vector<int>& s) {
    vector<bool> present(s.size() + 1, false);
    for (int x : s)
        if (x <= (int)s.size()) present[x] = true;
    for (int i = 0; i <= (int)s.size(); i++)
        if (!present[i]) return i;
    return (int)s.size() + 1; // unreachable
}

    Grundy Values

    Definition

    Consider an impartial game represented as a directed acyclic graph (DAG), where each node is a game position and edges represent legal moves. We define the Grundy value (or Sprague-Grundy value, or nimber) $\mathcal{G}(x)$ of a position $x$ recursively:

    $$\mathcal{G}(x) = \operatorname{mex}\bigl(\{\mathcal{G}(y) : y \in \text{moves}(x)\}\bigr)$$

    where $\text{moves}(x)$ is the set of positions reachable from $x$ in one move.

    Base case: A terminal position $x$ (no moves available) has $\text{moves}(x) = \emptyset$, so $\mathcal{G}(x) = \operatorname{mex}(\emptyset) = 0$.

    Key Observations

    • $\mathcal{G}(x) = 0$ if and only if $x$ is a losing position (P-position) for the player about to move.
    • $\mathcal{G}(x) > 0$ if and only if $x$ is a winning position (N-position) — the current player can force a win.
    • From any position with $\mathcal{G}(x) = g > 0$, there exists a move to a position with Grundy value $k$ for every $0 \leq k < g$. (This follows directly from Property 3 of the mex.)
    • From any position with $\mathcal{G}(x) = 0$, every move leads to a position with Grundy value $> 0$.

    Grundy Values of Nim Heaps

    In a single Nim heap of size $n$, a player can remove any number of stones (at least 1), leaving a heap of size $0, 1, \ldots, n-1$. By induction:

    $$\mathcal{G}(\text{Nim}_n) = \operatorname{mex}(\{0, 1, \ldots, n-1\}) = n$$

    So the Grundy value of a Nim heap of size $n$ is exactly $n$. This is why Grundy values are also called nimbers.

    Computation Algorithm

    To compute Grundy values on a game DAG, we process positions in reverse topological order (terminals first):

    1. Initialize $\mathcal{G}(x) = 0$ for all terminal positions.
    2. For each non-terminal position $x$ (processed after all its successors), compute $\mathcal{G}(x) = \operatorname{mex}\bigl(\{\mathcal{G}(y) : y \in \text{moves}(x)\}\bigr)$.

    For games with a natural parameter (like heap size), we typically compute bottom-up from small to large.

    The Sprague-Grundy Theorem

    Statement

    Theorem (Sprague-Grundy). Every impartial game $G$ under normal play convention is equivalent to a Nim heap of size $\mathcal{G}(G)$, where $\mathcal{G}(G)$ is the Grundy value of the starting position.

    More precisely, two games $G_1$ and $G_2$ are equivalent (written $G_1 \equiv G_2$) if for every game $H$, the compound game $G_1 + H$ has the same outcome (first-player win or second-player win) as $G_2 + H$. The theorem states:

    $$G \equiv *\mathcal{G}(G)$$

    where $*n$ denotes a Nim heap of size $n$.

    Proof

    We prove that a game position $x$ with $\mathcal{G}(x) = g$ is equivalent to a Nim heap of size $g$, i.e., the game $x + *g$ is a P-position (second-player win). This suffices because $*g + *g$ is always a P-position (both heaps XOR to $0$), so replacing one copy of $*g$ with $x$ preserves the outcome if and only if $x \equiv *g$.

    We proceed by strong induction on the game tree (well-founded since all games terminate).

    Claim: $\mathcal{G}(x + *g) = 0$, where $g = \mathcal{G}(x)$.

    Proof that $x + *g$ is a P-position: We show that from position $(x, *g)$, every move leads to an N-position, and from any N-position, there exists a move to a P-position.

    Direction 1: Every move from $(x, *g)$ leads to a position with nonzero Grundy value.

    A move in $x + *g$ is a move in exactly one component. Consider two cases:

    Case A: The move is in $x$, going to some $x'$ with $\mathcal{G}(x') = g' \neq g$.

    Since $g = \mathcal{G}(x) = \operatorname{mex}(\ldots)$, and $x'$ is a successor of $x$, we know $g' \neq g$ (if $g' = g$ were possible, the mex would still be $g$ only if $g$ is not among the successor values, but $g'$ is a successor value; however, $g' = g$ is indeed possible — the key is what happens next). Actually, $g'$ can be anything in $\{\mathcal{G}(y) : y \in \text{moves}(x)\}$, so $g'$ can equal $g$ or not.

    Let us use a cleaner approach. We show directly that $(x, *g)$ is a P-position using the characterization: a position is P if and only if every move leads to an N-position.

    Revised proof via the Sprague-Grundy recursion:

    We prove by structural induction that $\mathcal{G}(x) = g$ implies $x \equiv *g$. This means: for any game $H$, the first player wins $x + H$ if and only if the first player wins $*g + H$.

    Base case: If $x$ is terminal, then $\mathcal{G}(x) = 0$ and $x \equiv *0$ (the zero game, which is a loss for the mover). This is immediate since $x$ has no moves, just like $*0$.

    Inductive step: Assume the result holds for all successors of $x$. Let $\mathcal{G}(x) = g$. We need to show that $x + *g$ is a P-position, i.e., $\mathcal{G}(x + *g) = 0$.

    Consider any move from $(x, *g)$:

    Move in $x$ to some $x'$: We have $\mathcal{G}(x') = g'$ for some $g' \neq g$ or $g' = g$. But by the mex definition, $g \notin \{\mathcal{G}(y) : y \in \text{moves}(x)\}$ is false — wait, $g = \operatorname{mex}(\{\mathcal{G}(y)\})$ means $g$ is the smallest value not in the set. So indeed $g' \neq g$ for all successors $y$... no, that's not right either: it means $g$ itself is not in the set, but the successors can have any values; some may repeat.

    Let's be precise. Since $\mathcal{G}(x) = \operatorname{mex}(S)$ where $S = \{\mathcal{G}(y) : y \in \text{moves}(x)\}$, we know:

    • $g \notin S$ — no successor of $x$ has Grundy value $g$.
    • For every $0 \leq k < g$, $k \in S$ — there exists a successor of $x$ with Grundy value $k$.

    Now consider any move from $(x, *g)$:

    Case 1: Move in $x$ to $x'$ with $\mathcal{G}(x') = g'$. Since $g \notin S$, we have $g' \neq g$. The resulting position is $(x', *g)$. By the inductive hypothesis, $x' \equiv *g'$, so $(x', *g) \equiv (*g', *g)$. Since $g' \neq g$, the Nim position $(*g', *g)$ has Grundy value $g' \oplus g \neq 0$, so this is an N-position. ✓

    Case 2: Move in $*g$ to $*g'$ where $0 \leq g' < g$. The resulting position is $(x, *g')$. By the inductive hypothesis, $x \equiv *g$. But wait — we're trying to prove $x \equiv *g$, so we can't use that here directly. Instead, note that the position is $(x, *g')$. Since $g' < g$ and $g' \in S$, there exists a successor $x''$ of $x$ with $\mathcal{G}(x'') = g'$. But this is about showing the second player has a response, not that the position is losing for the mover.

    Let us instead use the standard clean proof:

    Theorem. For any position $x$ in an impartial game, $x + *\mathcal{G}(x)$ is a P-position.

    Proof by strong induction on the game tree. Let $g = \mathcal{G}(x)$. We must show that from $(x, *g)$, the second player (Player 2) has a winning strategy.

    Showing every move from $(x, *g)$ can be countered:

    If Player 1 moves in $x$, reaching $(x', *g)$ with $\mathcal{G}(x') = g'$:

    Since $g \notin \{\mathcal{G}(y) : y \in \text{moves}(x)\}$, we have $g' \neq g$. Player 2 responds by moving in $*g$ to $*g'$ (which is legal since either $g' < g$ or $g' > g$; in Nim, we can only reduce, so if $g' > g$ this doesn't work). Hmm — we need a subtler argument.

    Here is the standard proof presented cleanly:

    Theorem (Sprague-Grundy). For any impartial game position $x$, the Grundy value $\mathcal{G}(x)$ determines the outcome: $x$ is a P-position (previous player wins / current player loses) if and only if $\mathcal{G}(x) = 0$.

    Moreover, $x$ is equivalent to the Nim heap $*\mathcal{G}(x)$, meaning: for every game $H$, the position $x + H$ is a P-position iff $*\mathcal{G}(x) + H$ is a P-position.

    Proof. We prove this by well-founded induction on the game DAG. The key insight is to show that positions with Grundy value $g$ have exactly the same "reachability structure" as a Nim heap of size $g$.

    What does $\mathcal{G}(x) = g$ tell us?

    1. For every $0 \leq k < g$, there exists a move from $x$ to some $x_k$ with $\mathcal{G}(x_k) = k$.
    2. There is no move from $x$ to any position with Grundy value $g$.

    Compare with a Nim heap $*g$: from $*g$ you can move to $*k$ for any $0 \leq k < g$, and you cannot stay at $*g$. The reachable Grundy values are identical.

    Proof that $x + *g$ is a P-position (second player wins):

    We use strong induction. Consider any move by Player 1 from $(x, *g)$:

    Case 1: Player 1 moves $x \to x'$ where $\mathcal{G}(x') = g' \neq g$.

    • If $g' < g$: Player 2 moves $*g \to *g'$. Now both components have Grundy value $g'$. By induction, $x' + *g'$ is a P-position (since $x'$ is a smaller game position and $\mathcal{G}(x') = g'$). It's now Player 1's turn in a P-position, so Player 2 is winning.
    • If $g' > g$: Since $\mathcal{G}(x') = g' > g$, there exists a move from $x'$ to some $x''$ with $\mathcal{G}(x'') = g$ (by property (1) of Grundy values, since $g < g'$). Player 2 moves $x' \to x''$. Now we have $(x'', *g)$ with $\mathcal{G}(x'') = g$. By induction, this is a P-position for the mover (Player 1), so Player 2 wins.

    Case 2: Player 1 moves $*g \to *g'$ where $0 \leq g' < g$.

    • Since $g' < g = \mathcal{G}(x)$, there exists a move $x \to x'$ with $\mathcal{G}(x') = g'$ (property (1)). Player 2 makes this move. Now we have $(x', *g')$ with $\mathcal{G}(x') = g'$. By induction, $x' + *g'$ is a P-position for the mover (Player 1). Player 2 wins.

    In every case, Player 2 can respond to reach a position of the form $y + *\mathcal{G}(y)$ on a strictly smaller game, which is a P-position by the inductive hypothesis. Since the game is finite, the induction is well-founded. $\square$

    Converse Direction

    We also verify the converse: if $x + *h$ is a P-position for some $h$, then $h = \mathcal{G}(x)$.

    Proof. Suppose $h \neq g = \mathcal{G}(x)$. Then $*g + *h$ has Grundy value $g \oplus h \neq 0$ (since $g \neq h$), so $*g + *h$ is an N-position. If $x \equiv *g$ (which we proved above), then $x + *h \equiv *g + *h$ is also an N-position, contradicting the assumption that $x + *h$ is a P-position. $\square$

    Game Sums

    Definition

    The sum of games $G_1, G_2, \ldots, G_k$ is the game $G_1 + G_2 + \cdots + G_k$ where:

    • A position is a tuple $(x_1, x_2, \ldots, x_k)$ with $x_i$ a position in $G_i$.
    • A move consists of choosing exactly one component $i$ and making a legal move in $G_i$, leaving all other components unchanged.
    • The game ends when no moves are available in any component (all components are terminal).

    This is precisely the structure of multi-heap Nim and many other combinatorial games.

    The XOR Theorem for Game Sums

    Theorem. The Grundy value of a sum of games equals the XOR (nim-sum) of the individual Grundy values:

    $$\mathcal{G}(G_1 + G_2 + \cdots + G_k) = \mathcal{G}(G_1) \oplus \mathcal{G}(G_2) \oplus \cdots \oplus \mathcal{G}(G_k)$$

    Proof for Two Games

    We prove $\mathcal{G}(G_1 + G_2) = \mathcal{G}(G_1) \oplus \mathcal{G}(G_2)$ by strong induction. Let $a = \mathcal{G}(G_1)$ and $b = \mathcal{G}(G_2)$, and let $c = a \oplus b$. We need to show $\mathcal{G}(G_1 + G_2) = c$.

    By the definition of Grundy values, it suffices to show:

    1. (Reachability) For every $0 \leq k < c$, there is a move from $(G_1, G_2)$ to a position with Grundy value $k$.
    2. (Non-reachability) There is no move from $(G_1, G_2)$ to a position with Grundy value $c$.

    Proof of (1): Let $0 \leq k < c = a \oplus b$. Consider $d = k \oplus b$. Then $d \oplus b = k$, and $d \neq a$ (since $k \neq c = a \oplus b$ would give $d = k \oplus b \neq a \oplus b \oplus b = a$... let's be more careful).

    We want to reach Grundy value $k$ from $(G_1, G_2)$ in one move. We move in whichever component allows us to "fix" the XOR.

    Consider $k \oplus b$. If $k \oplus b < a$: move $G_1$ to a position $G_1'$ with $\mathcal{G}(G_1') = k \oplus b$. Then by induction, $\mathcal{G}(G_1' + G_2) = (k \oplus b) \oplus b = k$. ✓

    Otherwise, consider $a \oplus k$. If $a \oplus k < b$: move $G_2$ to a position $G_2'$ with $\mathcal{G}(G_2') = a \oplus k$. Then $\mathcal{G}(G_1 + G_2') = a \oplus (a \oplus k) = k$. ✓

    Claim: At least one of $k \oplus b < a$ or $a \oplus k < b$ holds whenever $k < a \oplus b$.

    Proof of claim: Let $k < c = a \oplus b$. Look at the highest bit where $k$ and $c$ differ. At that bit position, $c$ has a $1$ and $k$ has a $0$. Since $c = a \oplus b$, exactly one of $a, b$ has a $1$ at this bit. Without loss of generality, say $a$ has a $1$ at this bit. Then $k \oplus b$ agrees with $a$ on all higher bits, and at this bit, $k \oplus b$ has a $0$ while $a$ has a $1$. Therefore $k \oplus b < a$ (when compared from the most significant differing bit). ✓

    Proof of (2): We must show there is no move to a position with Grundy value $c = a \oplus b$.

    Suppose we move in $G_1$ to $G_1'$ with $\mathcal{G}(G_1') = a'$. By induction, the resulting Grundy value is $a' \oplus b$. For this to equal $c = a \oplus b$, we need $a' = a$. But $a = \mathcal{G}(G_1)$, and by the mex property, no successor of $G_1$ has Grundy value $a$. ✓

    Similarly, moving in $G_2$ to $G_2'$ with $\mathcal{G}(G_2') = b'$ gives Grundy value $a \oplus b'$, which equals $c = a \oplus b$ only if $b' = b$. Again, no successor of $G_2$ has Grundy value $b$. ✓

    This completes the proof. The extension to $k$ games follows by induction on $k$, using the two-game case and associativity of XOR. $\square$

    Consequences

    This immediately recovers the Bouton's theorem for multi-heap Nim: the position $(n_1, n_2, \ldots, n_k)$ is a P-position iff $n_1 \oplus n_2 \oplus \cdots \oplus n_k = 0$, since $\mathcal{G}(*n_i) = n_i$.

    More importantly, it extends to any combination of impartial games. If you can compute the Grundy value of each independent component, you can determine the winner of the whole game by XOR-ing them together.

    Computing Grundy Values

    DP / Memoization Approach

    For games parameterized by a single integer (heap size, board position, etc.), we compute Grundy values bottom-up:

    #include <bits/stdc++.h>
using namespace std;

// Generic Grundy value computation for a game with positions 0..maxN.
// moves(x) returns the set of positions reachable from x.
vector<int> computeGrundy(int maxN, function<vector<int>(int)> moves) {
    vector<int> grundy(maxN + 1, 0);
    for (int x = 0; x <= maxN; x++) {
        vector<int> reachable = moves(x);
        // Collect Grundy values of successors
        vector<int> successorG;
        for (int y : reachable)
            successorG.push_back(grundy[y]);
        // Compute mex
        unordered_set<int> st(successorG.begin(), successorG.end());
        int g = 0;
        while (st.count(g)) ++g;
        grundy[x] = g;
    }
    return grundy;
}

    Example: Subtraction Game

    In a subtraction game with set $S = \{s_1, s_2, \ldots, s_m\}$, a player may remove exactly $s_i$ stones from a heap of size $n$ (for any $s_i \in S$ with $s_i \leq n$).

    // Subtraction game: from heap of size n, remove s in S (where s <= n).
vector<int> subtraction_grundy(int maxN, const vector<int>& S) {
    vector<int> grundy(maxN + 1, 0);
    for (int n = 0; n <= maxN; n++) {
        set<int> reachable;
        for (int s : S)
            if (n >= s) reachable.insert(grundy[n - s]);
        int g = 0;
        while (reachable.count(g)) ++g;
        grundy[n] = g;
    }
    return grundy;
}

// Example: S = {1, 3, 4}
// grundy[0] = 0  (terminal)

// grundy[2] = mex({G(1)}) = mex({1}) = 0
// grundy[3] = mex({G(2), G(0)}) = mex({0, 0}) = mex({0}) = 1

// grundy[7] = mex({G(6), G(4), G(3)}) = mex({2, 2, 1}) = mex({1,2}) = 0

    General Game on a DAG

    When the game is given as an explicit directed graph, compute Grundy values via topological sort:

    // Grundy values on an explicit game DAG.
// adj[u] = list of positions reachable from u.
// Positions 0..n-1, DAG (no cycles).
vector<int> grundy_dag(int n, const vector<vector<int>>& adj) {
    // Compute in-degree for reverse topological ordering
    vector<int> indeg(n, 0);
    // We need topological order: process terminals first.
    // Use Kahn's algorithm on the reverse graph.
    vector<vector<int>> radj(n);
    for (int u = 0; u < n; u++)
        for (int v : adj[u]) {
            radj[v].push_back(u);
            indeg[u]++; // u depends on v being computed first
        }

    // Actually, we need to process nodes whose successors
    // are all computed. Terminals have no outgoing edges.
    vector<int> outdeg(n, 0);
    for (int u = 0; u < n; u++)
        outdeg[u] = (int)adj[u].size();

    queue<int> q;
    for (int u = 0; u < n; u++)
        if (outdeg[u] == 0) q.push(u);

    vector<int> grundy(n, 0);
    vector<int> remaining(n);
    for (int u = 0; u < n; u++)
        remaining[u] = outdeg[u];

    // Process in reverse topological order
    // But simpler: just do DFS with memoization
    vector<int> memo(n, -1);
    function<int(int)> dfs = [&](int u) -> int {
        if (memo[u] != -1) return memo[u];
        set<int> child_grundy;
        for (int v : adj[u])
            child_grundy.insert(dfs(v));
        int g = 0;
        while (child_grundy.count(g)) ++g;
        return memo[u] = g;
    };

    for (int u = 0; u < n; u++)
        if (memo[u] == -1) dfs(u);

    return memo;
}

    Multi-Component Game Solver

    Putting it all together — solve a game that is a sum of independent components:

    // Solve: given k independent game components, determine if
// the first player wins and find the optimal first move.
struct GameResult {
    bool first_player_wins;
    int move_component; // which component to move in (-1 if losing)
    int move_to;        // new Grundy value in that component
};

GameResult solve_game_sum(const vector<int>& grundy_values) {
    int xor_sum = 0;
    for (int g : grundy_values) xor_sum ^= g;

    GameResult res;
    res.first_player_wins = (xor_sum != 0);
    res.move_component = -1;
    res.move_to = -1;

    if (xor_sum != 0) {
        // Find a component to move in to make XOR = 0
        for (int i = 0; i < (int)grundy_values.size(); i++) {
            int target = grundy_values[i] ^ xor_sum;
            if (target < grundy_values[i]) {
                res.move_component = i;
                res.move_to = target;
                break;
            }
        }
    }
    return res;
}

    Worked Examples

    Example 1: Subtraction Game $S = \{1, 3, 4\}$

    We computed the Grundy table earlier. Let's extend it and look for a pattern:

    $n$ 01 23 45 67 89 1011 1213
    $\mathcal{G}(n)$ 01 01 23 20 10 12 32

    The Grundy values are periodic with period $p = 7$: $\mathcal{G}(n) = \mathcal{G}(n \bmod 7)$ for $n \geq 0$.

    Fact: For a subtraction game with $S = \{s_1, \ldots, s_m\}$, the Grundy sequence is eventually periodic with period dividing $\text{lcm}(s_1, \ldots, s_m) + \max(S)$. This is useful for handling very large $n$.

    Application: Suppose we have 3 heaps of sizes $n_1 = 10, n_2 = 15, n_3 = 22$ with $S = \{1,3,4\}$. Then:

    • $\mathcal{G}(10) = \mathcal{G}(10 \bmod 7) = \mathcal{G}(3) = 1$
    • $\mathcal{G}(15) = \mathcal{G}(15 \bmod 7) = \mathcal{G}(1) = 1$
    • $\mathcal{G}(22) = \mathcal{G}(22 \bmod 7) = \mathcal{G}(1) = 1$

    XOR: $1 \oplus 1 \oplus 1 = 1 \neq 0$, so the first player wins.

    Example 2: Staircase Nim

    In Staircase Nim, there are $k$ stairs numbered $0, 1, \ldots, k-1$. Stair $i$ has $a_i$ stones. A move consists of choosing a stair $i > 0$ and moving any positive number of stones from stair $i$ to stair $i-1$. The game ends when all stones are on stair $0$ (no moves possible from stair $0$).

    Analysis: This game is equivalent to Nim on the odd-indexed stairs. The key insight is that moves involving even stairs are "wasted" — the opponent can mirror them.

    Grundy value: $\mathcal{G}(a_0, a_1, \ldots, a_{k-1}) = a_1 \oplus a_3 \oplus a_5 \oplus \cdots$ (XOR of odd-indexed stairs only).

    bool staircase_nim_first_wins(const vector<int>& stairs) {
    int xor_sum = 0;
    for (int i = 1; i < (int)stairs.size(); i += 2)
        xor_sum ^= stairs[i];
    return xor_sum != 0;
}

    Example 3: Mock Turtles (Turning Turtles)

    Consider a row of $n$ coins, each either heads (H) or tails (T). A move consists of flipping exactly one coin from H to T, optionally also flipping one other coin to its left. The game ends when all coins are T.

    Each coin at position $i$ (0-indexed from left) that is H contributes a Grundy value. By the Sprague-Grundy theorem, the game's Grundy value is the XOR of Grundy values of all heads-up coins.

    Analysis shows that coin $i$ in isolation has $\mathcal{G} = i + 1$ (it can move to any configuration reachable by flipping itself down and optionally flipping a coin to its left). Therefore:

    $$\mathcal{G} = \bigoplus_{i : \text{coin}_i = H} (i + 1)$$ 
    bool turning_turtles_first_wins(const string& coins) {
    // coins[i] = 'H' or 'T'
    int xor_sum = 0;
    for (int i = 0; i < (int)coins.size(); i++)
        if (coins[i] == 'H') xor_sum ^= (i + 1);
    return xor_sum != 0;
}

    Example 4: Green Hackenbush on a Path

    Consider a graph game where edges form a single path of length $n$ rooted at the ground. A move removes an edge, which also removes all edges no longer connected to the ground.

    For a single path of length $n$:

    $$\mathcal{G}(\text{path}_n) = n \bmod 2$$

    This can be verified by induction. Removing the $k$-th edge from the ground leaves a connected path of length $k-1$, so:

    $$\mathcal{G}(n) = \operatorname{mex}\bigl(\{\mathcal{G}(0), \mathcal{G}(1), \ldots, \mathcal{G}(n-1)\}\bigr)$$

    With $\mathcal{G}(0) = 0$ (no edge = terminal), $\mathcal{G}(1) = \operatorname{mex}(\{0\}) = 1$, $\mathcal{G}(2) = \operatorname{mex}(\{0, 1\}) = 2$... actually $\mathcal{G}(n) = n$ here. Each edge can be removed independently (cutting a path at position $k$ removes all edges above it), giving reachability to all positions $0, \ldots, n-1$.

    For a graph with multiple paths joined at the ground, the Grundy value is the XOR of the individual path Grundy values (by the game sum theorem).

    // Green Hackenbush: multiple paths from ground.
// paths[i] = length of i-th path.
int hackenbush_paths(const vector<int>& paths) {
    int xor_sum = 0;
    for (int len : paths) xor_sum ^= len;
    return xor_sum; // 0 means second player wins
}

    Tips for Game Theory Problems

    1. Identify the game type. Is it impartial? Is it a sum of independent games?
    2. Compute small cases. Build a Grundy table for small inputs and look for patterns.
    3. Look for periodicity. Many games have eventually periodic Grundy sequences. Compute enough values to spot the period, then verify.
    4. Decompose into components. If the game is a sum, compute Grundy values per component and XOR.
    5. Watch for game reductions. Some games can be reduced to Nim or known games via clever bijections.
