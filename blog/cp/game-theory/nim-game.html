The Nim Game

June 2026 · 22 min read

Nim is the cornerstone of combinatorial game theory. Every impartial game — where both players have the same set of moves — can be reduced to Nim through the Sprague–Grundy theorem (covered later in this series). Understanding Nim deeply is therefore not optional; it is foundational.

In this post we develop the complete theory from scratch: formal rules, the XOR characterization, Bouton's 1901 theorem with full proof, and efficient C++ implementations ready for competitive programming.

The Game of Nim

Rules

Nim is a two-player game played with $n$ piles of objects (stones, coins, etc.). The players alternate turns. On each turn a player must:

  1. Choose exactly one non-empty pile.
  2. Remove at least one object from that pile (they may remove all objects in the pile).

Under normal play convention (the standard in competitive programming), the player who takes the last object wins. Equivalently, the player who cannot move (faces all-empty piles) loses.

Formal Definition

A Nim position is a tuple $(a_1, a_2, \dots, a_n)$ where $a_i \geq 0$ is the size of the $i$-th pile. A move from position $(a_1, \dots, a_i, \dots, a_n)$ consists of choosing index $i$ and a value $a_i'$ with $0 \leq a_i' < a_i$, producing the new position $(a_1, \dots, a_i', \dots, a_n)$.

The terminal position is $(0, 0, \dots, 0)$, from which no move is possible.

Position Classification

Following standard game theory notation, every position is either:

  • P-position (Previous player wins): the player who just moved is in a winning state — equivalently, the player about to move is losing.
  • N-position (Next player wins): the player about to move has a winning strategy.
Example: Three-Pile Nim

Consider the position $(3, 4, 5)$. Three piles with 3, 4, and 5 stones:

Pile A = 3
Pile B = 4
Pile C = 5

We compute $3 \oplus 4 \oplus 5 = 011_2 \oplus 100_2 \oplus 101_2 = 010_2 = 2 \neq 0$. Since the XOR is nonzero, this is an N-position — the first player wins with optimal play. (We will prove exactly why shortly.)

Example: A Losing Position

The position $(1, 2, 3)$: we have $1 \oplus 2 \oplus 3 = 01_2 \oplus 10_2 \oplus 11_2 = 00_2 = 0$. This is a P-position. No matter what the first player does, the second player can always win.

Nim-Sum and XOR

The nim-sum of two non-negative integers $a$ and $b$, written $a \oplus b$, is their bitwise exclusive OR (XOR). If $a = \sum_{k} a_k 2^k$ and $b = \sum_{k} b_k 2^k$ are the binary representations, then:

$$a \oplus b = \sum_{k} (a_k \oplus b_k) \cdot 2^k$$

where $a_k \oplus b_k$ is addition modulo 2 for each bit position.

Fundamental XOR Properties

The XOR operation over $\mathbb{Z}_{\geq 0}$ satisfies these algebraic properties:

PropertyStatementSignificance
Commutativity $a \oplus b = b \oplus a$ Order of piles doesn't matter
Associativity $(a \oplus b) \oplus c = a \oplus (b \oplus c)$ We can XOR any number of piles
Identity $a \oplus 0 = a$ Empty piles don't affect the nim-sum
Self-inverse $a \oplus a = 0$ Every element is its own inverse
Cancellation $a \oplus b = c \iff a = b \oplus c$ XOR equations are freely rearrangeable
Proof of Cancellation Law

Suppose $a \oplus b = c$. XOR both sides by $b$:

$$(a \oplus b) \oplus b = c \oplus b$$ $$a \oplus (b \oplus b) = c \oplus b \quad \text{(associativity)}$$ $$a \oplus 0 = c \oplus b \quad \text{(self-inverse)}$$ $$a = b \oplus c \quad \text{(identity)}$$

The converse follows identically. $\blacksquare$

Binary Representation Intuition

Think of XOR as addition without carries in binary. This is why nim-sum is sometimes called "binary addition mod 2." For example:

ValueBinary
$5$101
$3$011
$5 \oplus 3 = 6$110

Crucially, $(\mathbb{Z}_{\geq 0}, \oplus)$ forms an abelian group. This algebraic structure is what makes the entire Nim theory work.

Bouton's Theorem

Theorem (Bouton, 1901)

A Nim position $(a_1, a_2, \dots, a_n)$ is a P-position (losing for the player to move) if and only if the nim-sum of all pile sizes is zero:

$$a_1 \oplus a_2 \oplus \cdots \oplus a_n = 0$$

Equivalently, it is an N-position (winning for the player to move) if and only if $a_1 \oplus a_2 \oplus \cdots \oplus a_n \neq 0$.

The proof proceeds by verifying the three conditions that characterize P-positions and N-positions in combinatorial game theory.

Proof Setup

Let $S = a_1 \oplus a_2 \oplus \cdots \oplus a_n$. We call a position zero if $S = 0$ and nonzero if $S \neq 0$. We must show:

  1. Terminal condition: The terminal position $(0, 0, \dots, 0)$ is a zero position.
  2. N → P: From every nonzero position, there exists a move to a zero position.
  3. P → N: From every zero position, every move leads to a nonzero position.
Proof — Part 1: Terminal Condition

The terminal position is $(0, 0, \dots, 0)$, and $0 \oplus 0 \oplus \cdots \oplus 0 = 0$. ✓

Proof — Part 2: From Nonzero to Zero (N → P)

Suppose $S = a_1 \oplus a_2 \oplus \cdots \oplus a_n \neq 0$. Let $d$ be the position of the highest set bit in $S$ (i.e., $2^d$ is the largest power of 2 dividing some term in the binary expansion of $S$).

There must exist some pile $a_i$ whose bit $d$ is set (otherwise bit $d$ of $S$ would be 0, contradicting our choice). Consider $a_i' = a_i \oplus S$. We claim:

Claim 1: $a_i' < a_i$.

Proof of Claim 1: Since bit $d$ of both $a_i$ and $S$ is 1, bit $d$ of $a_i' = a_i \oplus S$ is 0. So $a_i'$ and $a_i$ agree on all bits above position $d$, and $a_i$ has bit $d$ set while $a_i'$ does not. Therefore $a_i' < a_i$.

Claim 2: After replacing $a_i$ with $a_i'$, the new nim-sum is zero.

Proof of Claim 2: The new nim-sum is:

$$a_1 \oplus \cdots \oplus a_i' \oplus \cdots \oplus a_n = a_1 \oplus \cdots \oplus (a_i \oplus S) \oplus \cdots \oplus a_n$$ $$= (a_1 \oplus \cdots \oplus a_i \oplus \cdots \oplus a_n) \oplus S = S \oplus S = 0$$

So the move of reducing pile $i$ from $a_i$ to $a_i' = a_i \oplus S$ is valid (since $a_i' < a_i$) and produces a zero position. ✓

Proof — Part 3: From Zero to Nonzero (P → N)

Suppose $S = a_1 \oplus a_2 \oplus \cdots \oplus a_n = 0$. Consider any move that changes pile $i$ from $a_i$ to $a_i'$ where $0 \leq a_i' < a_i$. The new nim-sum is:

$$S' = a_1 \oplus \cdots \oplus a_i' \oplus \cdots \oplus a_n$$ $$= S \oplus a_i \oplus a_i' = 0 \oplus a_i \oplus a_i' = a_i \oplus a_i'$$

Since $a_i' \neq a_i$ (we must remove at least one stone), we have $a_i \oplus a_i' \neq 0$ by the property that $x \oplus y = 0 \iff x = y$. Therefore $S' \neq 0$, and we reach a nonzero position. ✓

$\blacksquare$

This proof is constructive: Part 2 doesn't just assert a winning move exists — it tells us exactly how to find one. This is the basis of the winning strategy algorithm.

Winning Strategy

Algorithm

Given a Nim position $(a_1, a_2, \dots, a_n)$ with nim-sum $S = a_1 \oplus \cdots \oplus a_n \neq 0$, the winning move is:

  1. Compute $S$.
  2. Find the highest bit $d$ of $S$.
  3. Find any pile $a_i$ with bit $d$ set.
  4. Replace $a_i$ with $a_i \oplus S$ (this is guaranteed to be smaller than $a_i$).
Worked Example: $(3, 4, 5)$

Step 1: $S = 3 \oplus 4 \oplus 5 = 011 \oplus 100 \oplus 101 = 010 = 2$.

Step 2: Highest bit of $S = 2$ is bit 1 (zero-indexed).

Step 3: Which piles have bit 1 set?

  • $3 = 011_2$ → bit 1 is set ✓
  • $4 = 100_2$ → bit 1 is not set ✗
  • $5 = 101_2$ → bit 1 is not set ✗

Step 4: Choose pile A ($a_1 = 3$). Set $a_1' = 3 \oplus 2 = 1$.

Move: Remove 2 stones from pile A, leaving position $(1, 4, 5)$.

Verify: $1 \oplus 4 \oplus 5 = 001 \oplus 100 \oplus 101 = 000 = 0$ ✓

Worked Example: $(7, 11, 8, 4)$

Step 1: $S = 7 \oplus 11 \oplus 8 \oplus 4$

$S = 10$1010

Step 2: Highest bit of $S = 10 = 1010_2$ is bit 3.

Step 3: Piles with bit 3 set: $11 = 1011_2$ ✓ and $8 = 1000_2$ ✓. Pick pile 2 ($a_2 = 11$).

Step 4: $a_2' = 11 \oplus 10 = 1011 \oplus 1010 = 0001 = 1$.

Verify: $7 \oplus 1 \oplus 8 \oplus 4 = 0$ ✓

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

// Determine winner and find optimal move for Nim
// Returns: {winning pile index, new pile size} or {-1, -1} if losing
pair<int,int> nim_strategy(vector<int>& piles) {
    int xor_sum = 0;
    for (int x : piles) xor_sum ^= x;

    if (xor_sum == 0) return {-1, -1}; // P-position: no winning move

    // Find highest set bit of xor_sum
    int highest_bit = 31 - __builtin_clz(xor_sum);

    // Find a pile with that bit set
    for (int i = 0; i < (int)piles.size(); i++) {
        if ((piles[i] >> highest_bit) & 1) {
            int new_size = piles[i] ^ xor_sum;
            return {i, new_size};
        }
    }
    return {-1, -1}; // unreachable
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<int> piles(n);
    for (int& x : piles) cin >> x;

    auto [pile_idx, new_size] = nim_strategy(piles);
    if (pile_idx == -1) {
        cout << "Second player wins\n";
    } else {
        cout << "First player wins\n";
        cout << "Remove " << (piles[pile_idx] - new_size)
             << " from pile " << (pile_idx + 1) << "\n";
    }
    return 0;
}

Time complexity: $O(n)$ — a single pass to compute the XOR sum, then at most $n$ comparisons to find the target pile.

Space complexity: $O(1)$ extra space (beyond storing the piles).

Multi-Pile Nim

The beauty of Bouton's theorem is that it works for any number of piles. There is no increase in algorithmic complexity — the XOR of $n$ pile sizes is computed in $O(n)$ regardless of individual pile sizes.

Competitive Programming Formulation

The classic Nim problem on competitive programming judges is:

Given $n$ piles of sizes $a_1, a_2, \dots, a_n$, two players take turns removing any positive number of stones from a single pile. The player who removes the last stone wins. Determine the winner assuming optimal play.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        int xor_sum = 0;
        for (int i = 0; i < n; i++) {
            int a;
            cin >> a;
            xor_sum ^= a;
        }
        // xor_sum != 0 ⟹ first player wins
        cout << (xor_sum ? "First" : "Second") << "\n";
    }
    return 0;
}

This handles up to $n \leq 10^6$ piles with $a_i \leq 10^9$ efficiently — XOR on 32-bit integers is a single CPU instruction.

Important Edge Cases

  • Single pile ($n = 1$): First player takes everything and wins (unless $a_1 = 0$).
  • All piles equal ($a_i = c$ for all $i$): If $n$ is even, XOR sum is $0$ (P-position). If $n$ is odd, XOR sum is $c$ (N-position if $c > 0$).
  • Two piles ($n = 2$): $a \oplus b = 0 \iff a = b$. So $(a, b)$ is losing iff both piles are equal — a result that's easy to verify by intuition.

Complexity Analysis

Computing the nim-sum and finding the optimal move both run in $O(n)$ time and $O(1)$ extra space. Playing a full game takes $O(n \cdot M)$ where $M = \max(a_i)$, since the game lasts at most $O(M)$ moves and each requires $O(n)$ to recompute the XOR.

XOR Properties Deep Dive

The XOR operation appears throughout game theory and competitive programming. Let's formalize the properties that make it so powerful.

Algebraic Structure

$(\mathbb{Z}_{\geq 0}, \oplus)$ is an abelian group where every element has order 2 (i.e., $a \oplus a = 0$ for all $a$). In abstract algebra terms, it is isomorphic to a direct sum of countably many copies of $\mathbb{Z}/2\mathbb{Z}$:

$$(\mathbb{Z}_{\geq 0}, \oplus) \cong \bigoplus_{k=0}^{\infty} \mathbb{Z}/2\mathbb{Z}$$

Each bit position operates independently — this is the deepest reason why XOR works so cleanly for Nim.

Key XOR Identities

Identity 1: Uniqueness of Solutions

For any $a, c \in \mathbb{Z}_{\geq 0}$, the equation $a \oplus x = c$ has exactly one solution: $x = a \oplus c$.

Proof: Existence: $a \oplus (a \oplus c) = (a \oplus a) \oplus c = 0 \oplus c = c$. Uniqueness: if $a \oplus x_1 = c$ and $a \oplus x_2 = c$, then $x_1 = a \oplus c = x_2$. $\blacksquare$

Identity 2: XOR Preserves Bit Count Parity

For any bit position $k$: the number of values among $a_1, \dots, a_n$ with bit $k$ set is odd if and only if bit $k$ of $a_1 \oplus \cdots \oplus a_n$ is 1.

Proof: XOR at each bit position is addition modulo 2. The sum $\sum_{i=1}^n (a_i)_k \pmod{2}$ is 1 iff an odd number of summands are 1. $\blacksquare$

Identity 3: Prefix XOR Trick

Given an array $a[0..n-1]$, define prefix XOR $p_i = a_0 \oplus a_1 \oplus \cdots \oplus a_{i}$. Then the XOR of any subarray $a[l..r]$ is:

$$a_l \oplus a_{l+1} \oplus \cdots \oplus a_r = p_r \oplus p_{l-1}$$

where $p_{-1} = 0$. This is analogous to prefix sums but with XOR instead of addition.

Bit Manipulation Tricks for Game Theory

// Useful XOR / bit tricks in competitive programming

// 1. Lowest set bit of x
int lowest_bit = x & (-x);

// 2. Highest set bit position (0-indexed)
int highest_pos = 31 - __builtin_clz(x); // undefined for x=0

// 3. Check if bit k is set
bool is_set = (x >> k) & 1;

// 4. Toggle bit k
x ^= (1 << k);

// 5. XOR of range [0, n]: has a pattern with period 4
//    xor_0_to_n(n) based on n % 4:
//    0 -> n, 1 -> 1, 2 -> n+1, 3 -> 0
int xor_0_to_n(int n) {
    switch (n % 4) {
        case 0: return n;
        case 1: return 1;
        case 2: return n + 1;
        case 3: return 0;
    }
    return 0; // unreachable
}

// 6. XOR of range [l, r]
int xor_range(int l, int r) {
    return xor_0_to_n(r) ^ (l ? xor_0_to_n(l - 1) : 0);
}

Why XOR and Not Some Other Operation?

One might ask: why does XOR characterize Nim positions, rather than addition, multiplication, or some other operation? The answer lies in the independence of bit positions.

Each bit of the pile sizes can be analyzed independently. At bit position $k$, the "game" is equivalent to a single-bit Nim: you can turn a 1 into a 0 (by removing $2^k$ stones from a pile), but this might affect lower bits too. The key insight is that affecting lower bits only helps — it provides additional degrees of freedom, never fewer. The constraint propagates upward from the highest bit, which is exactly how XOR operates.

More formally, the Sprague–Grundy theorem (Part 4 of this series) shows that the Grundy value of a Nim pile of size $a$ is exactly $a$, and Grundy values combine with XOR. Bouton's theorem is thus a special case of the Sprague–Grundy theorem, though historically it came first.

Tips for Nim Problems in Contests

  1. Always check if the game reduces to Nim. Many seemingly complex games are Nim in disguise.
  2. Misère Nim: Under misère rules (last player to move loses), the strategy changes slightly: if all piles have size $\leq 1$, the player facing an odd number of size-1 piles loses. Otherwise, play as in normal Nim but aim for a position where all remaining piles have size $\leq 1$ and there's an odd number of them.
  3. XOR is fast. Don't overthink complexity — XOR of $10^6$ numbers takes microseconds.
  4. Watch for the Sprague–Grundy connection. If each pile is an independent subgame, compute Grundy values and XOR them (covered in Part 4).
Misère Nim Theorem

Under misère play (the player who takes the last stone loses), the first player wins if and only if:

  • The XOR of all pile sizes is nonzero and at least one pile has size $> 1$, OR
  • The XOR of all pile sizes is zero and all piles have size $\leq 1$.

The strategy is: play normal Nim until all piles have size $\leq 1$, then leave an odd number of size-1 piles for your opponent.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        int xor_sum = 0;
        bool all_one = true;
        for (int i = 0; i < n; i++) {
            int a;
            cin >> a;
            xor_sum ^= a;
            if (a > 1) all_one = false;
        }

        // Misère Nim: flip result when all piles ≤ 1
        if (all_one) {
            // Odd number of piles → second player wins (they take last)
            cout << (n % 2 == 0 ? "First" : "Second") << "\n";
        } else {
            // Play like normal Nim
            cout << (xor_sum ? "First" : "Second") << "\n";
        }
    }
    return 0;
}

Summary

Nim is deceptively simple in its rules but extraordinarily deep in theory. The key takeaways:

  • A Nim position is losing iff the XOR of all pile sizes is 0 (Bouton's theorem).
  • The winning move reduces some pile $a_i$ to $a_i \oplus S$, where $S$ is the total XOR.
  • Everything runs in $O(n)$ time — XOR is your best friend in game theory.
