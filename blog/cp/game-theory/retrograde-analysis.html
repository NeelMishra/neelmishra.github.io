Retrograde Analysis

June 2026 · 30 min read

Most game-solving algorithms work forward: from the starting position, explore moves until a terminal state is reached. Retrograde analysis reverses this entirely — it starts from every terminal position and works backward, labeling each game state as a win (N-position) or loss (P-position) for the player to move. This technique is the backbone of endgame databases in chess, the complete solution of checkers, and a powerful tool in competitive programming.

Backward Induction

The Core Idea

Consider a two-player, zero-sum game with perfect information and no chance moves. The game can be modeled as a directed acyclic graph (DAG) — or more generally a directed graph — where each node is a game state and each edge is a legal move. Some nodes are terminal: the game has ended, and the outcome is determined.

Backward induction assigns a value to every non-terminal state by reasoning from the terminal states upward. The key insight is:

  • A terminal losing position (the player to move has lost) is a P-position (Previous player wins).
  • A terminal winning position (the player to move has won) is an N-position (Next player wins).
  • A non-terminal state is a P-position if all of its successors are N-positions (every move leads to a win for the opponent).
  • A non-terminal state is an N-position if at least one of its successors is a P-position (there exists a move leading to a loss for the opponent).

Formal Definition

Let $G = (V, E)$ be the game graph. Let $T \subseteq V$ be the set of terminal positions with a known outcome function $\text{outcome}: T \to \{W, L, D\}$ (win, loss, draw for the player to move). Define the labeling function $\lambda: V \to \{N, P, D\}$ as:

$$\lambda(v) = \begin{cases} N & \text{if } v \in T \text{ and } \text{outcome}(v) = W \\ P & \text{if } v \in T \text{ and } \text{outcome}(v) = L \\ D & \text{if } v \in T \text{ and } \text{outcome}(v) = D \\ N & \text{if } v \notin T \text{ and } \exists (v, u) \in E : \lambda(u) = P \\ P & \text{if } v \notin T \text{ and } \forall (v, u) \in E : \lambda(u) = N \\ D & \text{otherwise (no move leads to } P \text{, at least one leads to } D\text{)} \end{cases}$$

The Algorithm (Informal)

  1. Label all terminal states with their known outcomes.
  2. For each labeled terminal state, look at its predecessors (states that can move to it).
  3. If a predecessor can reach a P-position, label it N (the player to move can win).
  4. If all successors of a predecessor are labeled N, label it P (the player to move must lose).
  5. Repeat until no more states can be labeled.

States that remain unlabeled after the algorithm terminates are draws (in games that allow infinite play) or are unreachable from terminal states.

The Retrograde Algorithm

BFS from Terminal Positions

The retrograde algorithm implements backward induction using a BFS (breadth-first search) that propagates labels from terminal positions through the reverse game graph. We need:

  • The game graph $G = (V, E)$ and its reverse $G^R = (V, E^R)$ where $(u, v) \in E^R \iff (v, u) \in E$.
  • For each vertex $v$, its out-degree $\text{deg}^+(v) = |\{u : (v, u) \in E\}|$.
  • A counter $c(v)$ initialized to $\text{deg}^+(v)$, tracking how many successors remain unlabeled as N.

Propagation Rules

The BFS processes vertices in waves. When a vertex $u$ is labeled:

Rule 1 (N-to-P propagation): If $\lambda(u) = P$, then for every predecessor $v$ such that $(v, u) \in E$: label $\lambda(v) = N$ (if not already labeled). Rationale: the player at $v$ can move to $u$, which is losing for the opponent.

Rule 2 (P-to-N counting): If $\lambda(u) = N$, then for every predecessor $v$ such that $(v, u) \in E$: decrement $c(v)$. If $c(v) = 0$, then all successors of $v$ are N-positions, so label $\lambda(v) = P$.

Pseudocode

function RetrogradeAnalysis(G, G_reverse, terminals):
    queue ← empty
    label ← array of UNKNOWN for all vertices
    counter ← array where counter[v] = out_degree(v) for all v

    // Phase 1: Seed terminal positions
    for each t in terminals:
        if outcome(t) == LOSS:
            label[t] ← P
            queue.push(t)
        else if outcome(t) == WIN:
            label[t] ← N
            queue.push(t)

    // Phase 2: BFS propagation
    while queue is not empty:
        u ← queue.pop()
        for each v in G_reverse[u]:       // predecessors of u
            if label[v] != UNKNOWN:
                continue
            if label[u] == P:             // Rule 1
                label[v] ← N
                queue.push(v)
            else if label[u] == N:        // Rule 2
                counter[v] ← counter[v] - 1
                if counter[v] == 0:
                    label[v] ← P
                    queue.push(v)

    return label

Correctness Proof

Theorem. The retrograde algorithm correctly labels every reachable position: $\lambda(v) = N$ if and only if the player to move at $v$ has a winning strategy, and $\lambda(v) = P$ if and only if the player to move at $v$ loses under optimal play.

Proof. We prove both directions by strong induction on the "distance" from terminal states, defined as the minimum number of BFS waves needed to label a vertex.

Base case. Terminal positions are labeled directly from the outcome function. By definition, a terminal position where the player to move has lost is a P-position, and one where the player to move has won is an N-position. These labels are correct. $\checkmark$

Inductive step (N-labeling). Suppose vertex $v$ is labeled $N$ during the BFS because some successor $u$ was labeled $P$. By the inductive hypothesis, $u$ is correctly labeled: the player to move at $u$ loses under optimal play. Since $(v, u) \in E$, the player at $v$ can move to $u$, forcing the opponent into a losing position. Therefore $v$ is correctly an N-position. $\checkmark$

Inductive step (P-labeling). Suppose vertex $v$ is labeled $P$ because $c(v)$ reached $0$, meaning all $\text{deg}^+(v)$ successors have been labeled $N$. By the inductive hypothesis, every successor $u_i$ is correctly an N-position: the player to move at $u_i$ wins. Since every move from $v$ leads to an N-position (a state where the opponent wins), the player at $v$ loses under optimal play. Therefore $v$ is correctly a P-position. $\checkmark$

Completeness. We must show that every position with a determined outcome is eventually labeled. Consider any position $v$ that is a true N-position. Then there exists a successor $u$ that is a true P-position. By induction on the longest path in the game tree, $u$ will be labeled $P$ before or at the same BFS wave. When $u$ is labeled $P$, Rule 1 labels $v$ as $N$. Similarly, if $v$ is a true P-position, all its successors are true N-positions and will be labeled $N$ by the inductive hypothesis. Each labeling decrements $c(v)$; when $c(v) = 0$, Rule 2 labels $v$ as $P$. $\square$

Termination. Each vertex is enqueued at most once (we skip already-labeled vertices), and each edge is traversed at most once in the reverse graph. The BFS terminates in $O(|V| + |E|)$ steps.

Endgame Databases

Motivation: Chess Endgames

The most famous application of retrograde analysis is chess endgame tablebases. In a chess endgame with few pieces remaining, the game graph is small enough to solve completely. For example, with King + Queen vs. King, there are roughly $64 \times 64 \times 64 \approx 2.6 \times 10^5$ positions (before symmetry reduction). A retrograde solver can label every position with the exact outcome: win in $n$ moves, loss, or draw.

Building a Complete Database

The construction proceeds as follows:

  1. Enumerate all legal positions for the given piece combination. Apply symmetry reductions (reflections, rotations) to shrink the state space.
  2. Identify terminal states: checkmate positions (P-positions for the side to move) and stalemate positions (draws).
  3. Run retrograde analysis: BFS from terminal states, labeling each position with its value and the distance-to-terminal (DTM) — the number of moves to checkmate under optimal play.
  4. Store the database: For each position, store the outcome and DTM value. This allows perfect play: at any position, look up the best move by choosing the successor that minimizes DTM (for the winning side) or maximizes it (for the losing side).

Distance-to-Mate Extension

To compute DTM, we extend the labeling. Process BFS in waves: terminal positions get DTM $= 0$. A vertex labeled N in wave $k$ (because it has a P-successor labeled in wave $k-1$) gets $\text{DTM}(v) = k$. A vertex labeled P in wave $k$ gets $\text{DTM}(v) = k$. This naturally falls out of the BFS structure:

$$\text{DTM}(v) = \begin{cases} 0 & \text{if } v \text{ is terminal} \\ 1 + \min_{(v,u) \in E,\, \lambda(u) = P} \text{DTM}(u) & \text{if } \lambda(v) = N \\ 1 + \max_{(v,u) \in E} \text{DTM}(u) & \text{if } \lambda(v) = P \end{cases}$$

The BFS naturally computes this: N-positions are labeled as soon as any P-successor is found (minimum), while P-positions are labeled only when all successors are resolved (maximum).

Complexity Analysis

Time Complexity

The retrograde BFS visits each vertex at most once and traverses each reverse edge at most once. Let $n = |V|$ and $m = |E|$.

Theorem. The retrograde algorithm runs in $O(n + m)$ time.

Proof.

  • Initialization: Setting up the counter array and seeding terminal states takes $O(n + |T|) \subseteq O(n)$.
  • BFS loop: Each vertex is enqueued at most once (the label check prevents re-processing), contributing $O(n)$ total dequeue operations.
  • Edge traversal: For each dequeued vertex $u$, we iterate over its predecessors in $G^R$. Each edge $(v, u) \in E$ is processed at most once (when $u$ is dequeued), contributing $O(m)$ total.
  • Counter updates: Each counter decrement corresponds to one reverse edge traversal, so the total number of decrements is $\leq m$.

Total: $O(n + m)$. This is optimal — any algorithm must read the entire graph. $\square$

Space Complexity

We need to store:

  • The game graph $G$ and its reverse $G^R$: $O(n + m)$ using adjacency lists.
  • The label array: $O(n)$, requiring 2 bits per position (P, N, D, or Unknown).
  • The counter array: $O(n)$, each counter needs $\lceil \log_2 (\Delta^+) \rceil$ bits where $\Delta^+$ is the maximum out-degree.
  • The BFS queue: $O(n)$ in the worst case.

Total space: $O(n + m)$.

State Space Explosion

The primary challenge is that $n$ and $m$ grow exponentially with the game's "complexity parameter." For chess endgames:

Pieces Approximate States Storage (2 bits/state)
3 (e.g., KQK)$\sim 10^5$~25 KB
4$\sim 10^7$~2.5 MB
5$\sim 10^9$~250 MB
6$\sim 10^{11}$~25 GB
7$\sim 10^{13}$~2.5 TB

Compression Techniques

Several strategies reduce the storage requirements:

  • Symmetry reduction: Exploit board symmetries (8-fold for chess with kings on the a1–d1–d4 triangle) to reduce position count by a constant factor.
  • Encoding schemes: Instead of indexing positions by raw coordinates, use a compact indexing function that maps each legal position to a consecutive integer. For chess, this avoids wasting space on illegal positions (e.g., kings adjacent).
  • DTM compression: Store the outcome in 2 bits and the DTM separately. Since DTM values cluster heavily (most positions are short wins), Huffman or run-length encoding achieves significant compression.
  • Block compression: Divide the table into blocks and compress each block with general-purpose algorithms (LZ4, zstd). Decompress on access. The Syzygy tablebases use this approach and compress 6-piece tables from ~1 TB to ~150 GB.
  • DTZ (Distance-to-Zeroing): Instead of DTM, store the distance to the next irreversible move (pawn push or capture). This metric compresses better because it resets at conversion points.

BFS Implementation

We implement retrograde analysis for a general game graph. The input is the game graph (forward edges), and the algorithm builds the reverse graph, identifies terminal positions, and runs BFS.

Full C++ Implementation

#include <bits/stdc++.h>
using namespace std;

enum Label { UNKNOWN = 0, P_POS = 1, N_POS = 2 };

struct RetrogradeResult {
    vector<Label> label;   // P, N, or UNKNOWN (draw/unreachable)
    vector<int>   dtm;     // distance-to-terminal (-1 if unknown)
};

// Retrograde analysis on a game graph.
// adj[v] = list of successors of v (forward moves).
// is_terminal[v] = true if v is a terminal position.
// terminal_label[v] = P_POS if the player to move at v has lost,
//                     N_POS if the player to move at v has won.
RetrogradeResult retrograde(
    int n,
    const vector<vector<int>>& adj,
    const vector<bool>& is_terminal,
    const vector<Label>& terminal_label)
{
    // Build reverse graph
    vector<vector<int>> radj(n);
    vector<int> out_deg(n, 0);
    for (int v = 0; v < n; v++) {
        out_deg[v] = (int)adj[v].size();
        for (int u : adj[v]) {
            radj[u].push_back(v);
        }
    }

    vector<Label> label(n, UNKNOWN);
    vector<int> dtm(n, -1);
    vector<int> counter(n); // tracks remaining unlabeled-N successors
    for (int v = 0; v < n; v++) {
        counter[v] = out_deg[v];
    }

    queue<int> bfs;

    // Phase 1: Seed terminal positions
    for (int v = 0; v < n; v++) {
        if (!is_terminal[v]) continue;
        label[v] = terminal_label[v];
        dtm[v] = 0;
        bfs.push(v);
    }

    // Phase 2: BFS propagation
    while (!bfs.empty()) {
        int u = bfs.front();
        bfs.pop();

        for (int v : radj[u]) {
            if (label[v] != UNKNOWN) continue;

            if (label[u] == P_POS) {
                // Rule 1: predecessor can move to a P-position → it's N
                label[v] = N_POS;
                dtm[v] = dtm[u] + 1;
                bfs.push(v);
            } else if (label[u] == N_POS) {
                // Rule 2: decrement counter
                counter[v]--;
                if (counter[v] == 0) {
                    // All successors are N → this is P
                    label[v] = P_POS;
                    // DTM = 1 + max DTM among successors
                    int max_dtm = 0;
                    for (int w : adj[v]) {
                        max_dtm = max(max_dtm, dtm[w]);
                    }
                    dtm[v] = max_dtm + 1;
                    bfs.push(v);
                }
            }
        }
    }

    return {label, dtm};
}

// ─── Example: Subtraction Game ───────────────────────────────
// Two players alternate removing 1, 2, or 3 stones from a pile
// of n stones. The player who takes the last stone wins.
// States: 0..n, Terminal: state 0 (player to move loses → P).
void solve_subtraction_game(int n) {
    int states = n + 1;
    vector<vector<int>> adj(states);
    vector<bool> is_terminal(states, false);
    vector<Label> terminal_label(states, UNKNOWN);

    // Build game graph
    for (int s = 1; s <= n; s++) {
        for (int take = 1; take <= 3 && take <= s; take++) {
            adj[s].push_back(s - take);
        }
    }

    // Terminal: state 0 → player to move loses
    is_terminal[0] = true;
    terminal_label[0] = P_POS;

    auto result = retrograde(states, adj, is_terminal, terminal_label);

    cout << "Subtraction Game (take 1-3 from " << n << " stones):\n";
    cout << "State | Label | DTM\n";
    cout << "------+-------+----\n";
    for (int s = 0; s <= n; s++) {
        const char* lbl = (result.label[s] == P_POS) ? "P" :
                          (result.label[s] == N_POS) ? "N" : "?";
        cout << setw(5) << s << " | "
             << setw(5) << lbl << " | "
             << setw(3) << result.dtm[s] << "\n";
    }
}

int main() {
    solve_subtraction_game(16);
    return 0;
}

Expected output (states 0–16): P, N, N, N, P, N, N, N, P, N, N, N, P, N, N, N, P — the classic $\text{mod } 4$ pattern. The retrograde solver discovers this pattern automatically from the terminal condition alone.

Extension: Games with Draws

In games that can draw (e.g., chess with the 50-move rule or repetition), we extend the algorithm. After the BFS terminates, all positions still labeled UNKNOWN are draws — neither player can force a win. 

// After retrograde() returns:
// UNKNOWN positions are draws (neither player can force a win).
// This is correct because:
//   - If v were truly a win for the mover, some path to a P-position
//     exists, and the BFS would have labeled v as N.
//   - If v were truly a loss, all successors lead to N-positions,
//     and the BFS would have labeled v as P.
//   - Therefore UNKNOWN ⟹ Draw.

void classify_draws(RetrogradeResult& result, int n) {
    for (int v = 0; v < n; v++) {
        if (result.label[v] == UNKNOWN) {
            // Optionally mark as DRAW in a richer label set
            result.dtm[v] = 0; // draws have no DTM
        }
    }
}

Optimized Version with Bit-Packed Labels

For large state spaces, we can pack labels into 2 bits per position, reducing memory by 16x compared to using int labels: 

// Compact label storage: 2 bits per position
// 00 = UNKNOWN, 01 = P, 10 = N
struct CompactLabels {
    vector<uint64_t> data;
    int n;

    CompactLabels(int n) : n(n), data((n + 31) / 32, 0ULL) {}

    Label get(int i) const {
        int word = i / 32;
        int bit  = (i % 32) * 2;
        return static_cast<Label>((data[word] >> bit) & 3);
    }

    void set(int i, Label lb) {
        int word = i / 32;
        int bit  = (i % 32) * 2;
        data[word] &= ~(3ULL << bit);
        data[word] |= (static_cast<uint64_t>(lb) << bit);
    }
};

Applications

Chess Endgame Tablebases

Retrograde analysis was pioneered by Ken Thompson in the 1980s for building chess endgame databases. Today, the Syzygy tablebases (by Ronald de Man) cover all endgames up to 7 pieces and are used by every top chess engine (Stockfish, Leela Chess Zero). Key facts:

  • 7-piece tables were computed by the Lomonosov team (Moscow State University) in 2012, requiring a supercomputer.
  • Syzygy tables use DTZ (distance-to-zeroing) for compression, fitting 6-piece tables in ~150 GB.
  • Online probing: Lichess and other platforms provide real-time tablebase lookups via APIs.

Checkers: The Chinook Project

In 2007, Jonathan Schaeffer's team at the University of Alberta weakly solved checkers using retrograde analysis combined with forward search. The result: checkers is a draw under optimal play from both sides. This was a landmark achievement:

  • The endgame database covered all positions with $\leq 10$ pieces (39 trillion positions).
  • Forward search from the initial position, using the endgame database as an oracle, proved the draw.
  • Computation took 18 years of continuous effort (1989–2007).

Competitive Programming Problems

Retrograde analysis appears in contest problems where the game graph is explicitly given or can be constructed. Typical patterns:

  • Explicit game graph: Given a directed graph where two players alternate moving a token. The player who cannot move loses. This is a direct application of retrograde BFS on the P/N classification.
  • Grid/board games: Two players move pieces on a grid. Terminal states are known (e.g., reaching a corner, being trapped). Build the state graph and run retrograde analysis.
  • Nim variants with traps: Modified Nim games where certain positions are "poisoned" (immediately losing). Standard Sprague-Grundy may not apply cleanly, but retrograde analysis always works.

CP Example: Token Game on a DAG

// Problem: Two players alternate moving a token on a DAG.
// The player who cannot move loses.
// Given: directed graph with n nodes, m edges, token starts at node s.
// Output: "First" if the first player wins, "Second" otherwise.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m, s;
    cin >> n >> m >> s;

    vector<vector<int>> adj(n), radj(n);
    vector<int> out_deg(n, 0);

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        radj[v].push_back(u);
        out_deg[u]++;
    }

    // Terminal nodes: out_deg == 0 → player to move loses → P-position
    vector<int> label(n, 0); // 0=unknown, 1=P, 2=N
    vector<int> counter(n);
    queue<int> bfs;

    for (int v = 0; v < n; v++) {
        counter[v] = out_deg[v];
        if (out_deg[v] == 0) {
            label[v] = 1; // P-position
            bfs.push(v);
        }
    }

    while (!bfs.empty()) {
        int u = bfs.front();
        bfs.pop();
        for (int v : radj[u]) {
            if (label[v] != 0) continue;
            if (label[u] == 1) { // P → predecessor is N
                label[v] = 2;
                bfs.push(v);
            } else { // N → decrement counter
                counter[v]--;
                if (counter[v] == 0) {
                    label[v] = 1; // all successors N → P
                    bfs.push(v);
                }
            }
        }
    }

    cout << (label[s] == 2 ? "First" : "Second") << "\n";
    return 0;
}
Implementation Note. In the competitive programming version, we identify terminal positions as nodes with out-degree 0. This is the most common convention: a player who cannot move loses. If the problem uses a different terminal condition (e.g., reaching a specific node), adjust the seeding phase accordingly.

Handling Cycles

The retrograde algorithm handles cycles naturally. Positions in cycles with no path to a terminal state remain UNKNOWN (draws). For games with cycle-avoidance rules, extend the state space to include history — though this is only practical for small games.
