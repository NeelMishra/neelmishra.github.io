Turning Turtles — Mock Turtles & Coin-Turning Games

June 2026 · 24 min read

Coin-turning games form an elegant family of combinatorial games where positions are rows of coins showing heads or tails. A move turns the rightmost heads coin to tails and optionally flips coins to its left. The game ends when every coin shows tails — last mover wins.

The Mock Turtles theorem connects these games to Nim via odious numbers (numbers with an odd popcount). We develop the full theory, prove the key results, and implement efficient C++ solvers.

Coin-Turning Games

Setup

The terminal position (all tails) has Grundy value 0 — the player to move loses.

General Move Rules

A coin-turning game on a line is specified by a turning set $\mathcal{T}$. A legal move consists of:

  1. Choose a coin at position $x$ that currently shows H.
  2. Turn coin $x$ from H to T (this is mandatory).
  3. For each position $y < x$ with $x - y \in \mathcal{T}$, you may optionally flip coin $y$ (H→T or T→H).

Different choices of $\mathcal{T}$ yield different games. The game always terminates since each move strictly reduces the largest heads-index.

Termination. Every coin-turning game terminates, so by Sprague–Grundy theory every position has a well-defined Grundy value.

Important Special Cases

Game Turning Set $\mathcal{T}$ Description
Mock Turtles $\{0, 1, 2, \ldots\}$ (all offsets) Flip any subset of coins to the left
Ruler Game $\{0\}$ (only the coin itself) Just turn one coin H→T, no other flips
Turning Turtles $\{0, 1, 2, \ldots\}$ but flip at most 1 extra Turn rightmost H→T, optionally flip one coin to the left
Ruler + 1 $\{1\}$ Must also flip the coin immediately left

Turning Turtles

Classic Formulation

The Turning Turtles game has a row of turtles on a line, each either face-up (heads) or face-down (tails). On each turn, a player must:

  1. Choose a face-up turtle at some position $x$.
  2. Turn it face-down (H → T).
  3. Optionally, flip exactly one turtle at some position $y < x$ (either direction: H→T or T→H).

The game ends when all turtles are face-down. Under normal play convention (last player to move wins), this is a classic impartial game.

Reduction to Nim

Each heads-coin at position $x$ behaves like a Nim pile. Turning coin $x$ H→T corresponds to removing a pile. Turning $x$ and flipping $y < x$ (T→H) corresponds to reducing the pile from $x$ to $y$.

Theorem (Turning Turtles = Nim)

The Grundy value of a single heads-coin at position $x$ in the Turning Turtles game (1-indexed) is $\mathcal{G}(x) = x$. A position with heads at positions $x_1, x_2, \ldots, x_k$ has Grundy value $x_1 \oplus x_2 \oplus \cdots \oplus x_k$.

Note: With 0-indexed coins and the more general "flip any subset" rule (Mock Turtles), the Grundy values change to odious numbers — see the Mock Turtles section below.

Worked Example

Consider 1-indexed positions with heads at $\{2, 5, 7\}$:

$$\mathcal{G} = 2 \oplus 5 \oplus 7 = (010) \oplus (101) \oplus (111) = (000) = 0$$

Grundy value 0 → P-position (current player loses with optimal play). Now consider heads at $\{1, 4, 6\}$:

$$\mathcal{G} = 1 \oplus 4 \oplus 6 = (001) \oplus (100) \oplus (110) = (011) = 3 \ne 0$$

N-position (first player wins). Winning move: turn coin 6 off, flip coin 5 on → new heads $\{1, 4, 5\}$, giving $1 \oplus 4 \oplus 5 = 0$. ✓

Grundy Analysis of Turning Games

Sprague–Grundy Framework

Recall: the Grundy value of position $P$ is:

$$\mathcal{G}(P) = \text{mex}\{\mathcal{G}(Q) : Q \in \text{options}(P)\}$$

For a sum of independent games, the combined Grundy value is the XOR of the individual values.

Coin Games as Sums

A key property: in Turning Turtles, each heads-coin at position $x$ is an independent game component. A move either removes it or replaces it with a component at some $y < x$. This decomposition lets us apply the Sprague–Grundy XOR theorem.

Grundy Values for Turning Turtles (1-Indexed)

Using 1-indexed positions ($x = 1, 2, 3, \ldots$), we compute Grundy values for a lone heads-coin at position $x$:

$x$ Options (Grundy values) $\mathcal{G}(x)$
1 $\{0\}$ (just turn off) 1
2 $\{0, 1\}$ (turn off, or turn off + flip pos 1) 2
3 $\{0, 1, 2, 3\}$ — wait, $\text{mex}$ = ? 3

Let us be precise for $x = 3$. Starting from $\{3\}$ (only coin 3 is H), the moves are:

  • Turn coin 3 off → $\emptyset$ → Grundy 0
  • Turn coin 3 off, flip coin 1 on → $\{1\}$ → Grundy 1
  • Turn coin 3 off, flip coin 2 on → $\{2\}$ → Grundy 2

So $\mathcal{G}(3) = \text{mex}\{0, 1, 2\} = 3$. By induction, starting from $\{x\}$, we can reach any position $\{y\}$ for $0 \le y < x$ (by flipping on coin $y$) plus $\emptyset$. Hence the options have Grundy values $\{0, 1, \ldots, x-1\}$ and $\mathcal{G}(x) = \text{mex}\{0, 1, \ldots, x-1\} = x$.

Theorem (Grundy Values for Turning Turtles)

In the Turning Turtles game with 1-indexed positions, the Grundy value of a single heads-coin at position $x$ is:

$$\mathcal{G}(x) = x$$

Proof (by strong induction on $x$).

Base case: $\mathcal{G}(1) = \text{mex}\{0\} = 1$ ✓

Inductive step: Assume $\mathcal{G}(y) = y$ for all $1 \le y < x$. From position $\{x\}$, the available moves are:

  • Turn $x$ off → terminal position with $\mathcal{G} = 0$.
  • For each $1 \le y < x$: turn $x$ off, flip $y$ on → position $\{y\}$ with $\mathcal{G} = y$.

The option Grundy values are $\{0, 1, 2, \ldots, x-1\}$, so $\mathcal{G}(x) = \text{mex}\{0, 1, \ldots, x-1\} = x$. $\square$

Connection to Nim

Since $\mathcal{G}(x) = x$, a position with heads at $\{x_1, \ldots, x_k\}$ has Grundy value $x_1 \oplus \cdots \oplus x_k$ — identical to Nim with piles $x_1, \ldots, x_k$. The winning strategy is the same: find a move making XOR zero.

Mock Turtles Theorem

The Mock Turtles Game

The Mock Turtles game generalizes Turning Turtles. Using 0-indexed positions, a move turns coin $x$ to tails and flips any subset of coins at positions $0, 1, \ldots, x-1$.

Statement of the Mock Turtles Theorem

Mock Turtles Theorem (Berlekamp, 1988)

In the Mock Turtles game (0-indexed, flip any subset left of the turned coin), the Grundy value of a single heads-coin at position $x$ is the $x$-th odious number:

$$\mathcal{G}(x) = o(x)$$

Equivalently, define $\hat{x}$ as the number obtained by prepending a parity bit to $x$ such that $\hat{x}$ has an odd number of 1-bits. Then $\mathcal{G}(x) = \hat{x}$.

A position with heads at $\{x_1, \ldots, x_k\}$ has Grundy value $o(x_1) \oplus o(x_2) \oplus \cdots \oplus o(x_k)$. The position is a P-position if and only if this XOR is zero.

The mapping $o(x)$: prepend a parity bit to $x$ making the total popcount odd:

$$o(x) = \begin{cases} 2x & \text{if popcount}(x) \text{ odd} \\ 2x + 1 & \text{if popcount}(x) \text{ even} \end{cases}$$

Proof of the Mock Turtles Theorem

Proof (via Sprague–Grundy)

We must verify two conditions for the claimed Grundy values $g(x) = o(x)$:

(1) Every option has a different Grundy value. From position $\{x\}$ with any subset $S \subseteq \{0, \ldots, x-1\}$ flipped, the resulting Grundy value is $o(x) \oplus \bigoplus_{s \in S} o(s)$... but this doesn't directly apply since we're modifying multiple coins simultaneously.

The correct approach uses the superposition principle for coin-turning games. Define the "Nim-value" of the position as $N = \bigoplus_{i \in H} o(i)$ where $H$ is the set of heads positions.

Claim: We can reach any Nim-value $N' < N$ (interpreting $<$ as "can be obtained by XOR-cancellation in a specific bit") from $N$, and we cannot reach $N$ from $N$. This follows from two key properties of odious numbers:

Property A: The XOR of any non-empty subset of distinct odious numbers is non-zero. This is because the leading (prepended parity) bits of the odious representations maintain a linear independence structure over GF(2).

Property B: For any $x$ and any target odious value $v < o(x)$, we can find a subset $S \subseteq \{0, \ldots, x-1\}$ such that $o(x) \oplus \bigoplus_{s \in S} o(s) = v$. This follows from the fact that the odious numbers $\{o(0), o(1), \ldots, o(x-1)\}$ span all $(x)$-bit values in GF(2)$^{n+1}$.

The full proof is by induction on $x$, verifying the mex condition. The parity-bit construction ensures that the Grundy values $o(0), o(1), \ldots$ form a complete system — every non-negative integer appears as the mex of the appropriate option set. $\square$

Intuition. The prepended parity bit in $o(x)$ acts as a "checksum" that prevents the XOR of distinct odious numbers from accidentally cancelling to zero. This is precisely what makes the Sprague–Grundy analysis work: each position's Grundy value is distinct from all reachable positions' Grundy values.

Example: Mock Turtles in Action

Consider heads at 0-indexed positions $\{0, 3, 5\}$:

$$\mathcal{G} = o(0) \oplus o(3) \oplus o(5) = 1 \oplus 7 \oplus 11 = (0001) \oplus (0111) \oplus (1011) = (1101) = 13 \ne 0$$

This is an N-position (first player wins). To find the winning move, compute the XOR without each coin and find a subset flip that cancels the remainder — this is where the C++ implementation below becomes essential.

Ruler Game

Definition

The Ruler Game is the simplest coin-turning game: the only legal move is to pick a heads-coin and turn it to tails. No other coins are affected ($\mathcal{T} = \emptyset$).

Grundy Values

Each heads-coin has exactly one option (turn to T), so:

$$\mathcal{G}(x) = \text{mex}\{0\} = 1 \quad \forall\, x$$

With $k$ heads-coins: Grundy value is $k \bmod 2$. P-position iff heads count is even.

Generalized Ruler Function

More interesting: the game where a move at position $x$ turns coin $x$ H→T and mandatorily flips coin $x - 1$. The Grundy values follow the ruler function:

$$\rho(n) = \nu_2(n) + 1$$

where $\nu_2(n)$ is the 2-adic valuation (largest power of 2 dividing $n$):

$n$ 12345678 910111213141516
$\nu_2(n)$ 01020103 01020104
$\mathcal{G}(n)$ 121 312 141 213 121 5

This pattern — the "binary carry sequence" — gives the game its name.

Theorem (Ruler Function Grundy Values)

In the game where a move at 1-indexed position $n$ turns coin $n$ to T and mandatorily flips coin $n-1$, the Grundy value is:

$$\mathcal{G}(n) = \nu_2(n) + 1$$

Proof sketch. Model the state as a vector in GF(2)$^n$; a move at position $i$ adds $e_i + e_{i-1}$. When $n$ is odd ($\nu_2(n) = 0$), the move simply swaps two adjacent coins, giving $\mathcal{G} = 1$. When $n = 2^k \cdot m$ ($m$ odd), the cascade resolves after $k$ steps, yielding $\mathcal{G} = k + 1$. $\square$

Computing the Ruler Function

In C++, the 2-adic valuation uses bit tricks:

// 2-adic valuation: largest k such that 2^k | n
int nu2(int n) {
    return __builtin_ctz(n); // count trailing zeros
}

// Ruler-game Grundy value (1-indexed)
int ruler_grundy(int n) {
    return __builtin_ctz(n) + 1;
}

C++ Implementations

Odious Number / Mock Turtles Grundy

#include <bits/stdc++.h>
using namespace std;

// Compute the x-th odious number (0-indexed).
// Prepend a parity bit to x so that the result has odd popcount.
long long odious(long long x) {
    int bits = __builtin_popcountll(x);
    if (bits % 2 == 1) return 2 * x;       // prepend 0 → already odd popcount
    else               return 2 * x + 1;   // prepend 1 → make popcount odd
}

// Determine winner of Mock Turtles game.
// heads[] = 0-indexed positions of heads coins.
bool mock_turtles_first_wins(const vector<int>& heads) {
    long long xor_sum = 0;
    for (int h : heads)
        xor_sum ^= odious(h);
    return xor_sum != 0;
}

Turning Turtles Solver (1-Indexed)

#include <bits/stdc++.h>
using namespace std;

// Turning Turtles: Grundy(x) = x for 1-indexed position x.
// Position is a P-position iff XOR of all heads positions = 0.
bool turning_turtles_first_wins(const vector<int>& heads) {
    int xor_sum = 0;
    for (int h : heads)
        xor_sum ^= h;
    return xor_sum != 0;
}

// Find a winning move: returns {x, y} where we turn x off and flip y on.
// y = -1 means we just turn x off.
pair<int, int> find_winning_move(const vector<int>& heads) {
    int xor_sum = 0;
    for (int h : heads) xor_sum ^= h;
    if (xor_sum == 0) return {-1, -1}; // losing position

    set<int> head_set(heads.begin(), heads.end());

    for (int x : heads) {
        // Option 1: just turn x off
        int new_xor = xor_sum ^ x; // remove x's contribution
        if (new_xor == 0) return {x, -1};

        // Option 2: turn x off and flip some y < x
        for (int y = 1; y < x; y++) {
            int after;
            if (head_set.count(y)) {
                // y was H, flipping makes it T → remove y
                after = xor_sum ^ x ^ y;
            } else {
                // y was T, flipping makes it H → add y
                after = xor_sum ^ x ^ y;
            }
            if (after == 0) return {x, y};
        }
    }
    return {-1, -1}; // should not reach here if xor_sum != 0
}

int main() {
    int n;
    cin >> n;
    vector<int> heads(n);
    for (int& h : heads) cin >> h;

    if (turning_turtles_first_wins(heads)) {
        cout << "First player wins\n";
        auto [x, y] = find_winning_move(heads);
        if (y == -1) cout << "Turn off coin " << x << "\n";
        else cout << "Turn off coin " << x << ", flip coin " << y << "\n";
    } else {
        cout << "Second player wins\n";
    }
}

Mock Turtles: Efficient Solution for Large Inputs

#include <bits/stdc++.h>
using namespace std;

long long odious(long long x) {
    return (__builtin_popcountll(x) & 1) ? 2 * x : 2 * x + 1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        long long xor_val = 0;
        for (int i = 0; i < n; i++) {
            long long x;
            cin >> x;
            xor_val ^= odious(x);
        }
        cout << (xor_val ? "First" : "Second") << "\n";
    }
}
Complexity. The odious-number mapping is O(1) per coin (just a popcount and a shift). The overall Mock Turtles solver runs in O($k$) per test case where $k$ is the number of heads-coins, handling positions up to $10^{18}$ with ease.

Practice Problems

  1. Basic Turning Turtles. Given $n$ heads positions (1-indexed), determine if the first player wins and output a winning move. Hint: XOR all positions; find a move that zeroes the XOR.
  2. Mock Turtles. Given $n$ heads positions (0-indexed) where any subset of left coins can be flipped, determine the winner. Hint: Compute odious numbers, XOR them. — CF Game Theory Tutorial
  3. Ruler Game Variant. Move at position $n$ turns it off and flips coin $n-1$. Determine the winner. Hint: XOR of $\nu_2(n) + 1$ values.
  4. Counting P-Positions. Count subsets of $\{0, \ldots, n-1\}$ forming a P-position in Mock Turtles. Hint: Linear algebra over GF(2); answer is $2^{n - \text{rank}}$.
  5. 2D Coin Turning. On a grid, move at $(r,c)$ can flip coins in $[0,r] \times [0,c]$. Grundy value is $o(r) \cdot o(c)$ (Nim-product).
Indexing pitfall. 0-indexed → Mock Turtles (odious numbers). 1-indexed → Turning Turtles ($\mathcal{G}(x) = x$). Always check the problem's convention before applying the formula.
