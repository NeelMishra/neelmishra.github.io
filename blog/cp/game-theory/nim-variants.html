    Nim Variants

    June 2026 · 30 min read

    Standard Nim — where two players alternate removing stones from piles and the last player to move wins — is completely solved by the Sprague-Grundy theorem: the first player wins if and only if the XOR of all pile sizes is non-zero. But competitive programming rarely gives you plain Nim. Instead, you encounter variants that twist the rules just enough to demand new theory.

    This post covers five important Nim variants: Staircase Nim, Wythoff's Game, Fibonacci Nim, Moore's Nim, and a family of Turning Games. For each, we develop the winning condition from scratch, prove it rigorously, and provide optimised C++ code.

    Prerequisites: Familiarity with standard Nim and the Sprague-Grundy theorem (covered in Parts 1–3 of this module). All games in this post use normal play convention — the player who makes the last move wins.

    Staircase Nim

    Rules

    A staircase consists of positions $0, 1, 2, \ldots, n$. Position $i$ holds $a_i \ge 0$ stones. On each turn a player picks any position $i \ge 1$ and moves any positive number of stones from position $i$ down to position $i-1$. Position $0$ is the ground — stones that reach it are effectively out of play. The player who cannot move (all stones on the ground) loses.

    Key Theorem

    Theorem (Staircase Nim). The first player wins if and only if $$ a_1 \oplus a_3 \oplus a_5 \oplus \cdots \neq 0, $$ i.e., the XOR of the stone counts on all odd-indexed positions is non-zero.

    Proof

    We show that the positions where $a_1 \oplus a_3 \oplus a_5 \oplus \cdots = 0$ form exactly the set of P-positions (previous-player-wins), satisfying the two defining properties:

    1. Terminal position is a P-position. When all stones are on position $0$, every odd-index pile is $0$, so $\bigoplus_{\text{odd } i} a_i = 0$. ✓
    2. From every position with odd-XOR $\neq 0$, there exists a move to a position with odd-XOR $= 0$.

      Since $\bigoplus_{\text{odd } i} a_i \neq 0$, standard Nim theory tells us we can reduce some odd-indexed pile $a_j$ (where $j$ is odd) so that the XOR becomes zero. Moving stones from position $j$ to position $j-1$ (which is even) changes $a_j$ without affecting any other odd-index pile. The even pile $a_{j-1}$ increases, but even-indexed piles are irrelevant to the XOR. ✓

    3. From every position with odd-XOR $= 0$, every move leads to odd-XOR $\neq 0$.

      Consider any move: the player moves stones from position $i$ to position $i-1$.

      • Case 1: $i$ is odd. Pile $a_i$ strictly decreases (at least one stone is moved), so the XOR of odd piles changes (since one summand decreased and no other odd-index pile changed). Hence the new odd-XOR $\neq 0$. ✓
      • Case 2: $i$ is even. Pile $a_i$ (even-indexed) decreases and pile $a_{i-1}$ (odd-indexed) increases. The odd-XOR changes because $a_{i-1}$ strictly increased. ✓

    All three conditions of the P-position characterization are met, completing the proof. $\blacksquare$

    Intuition: Moves on even-indexed piles are "wasted" — the opponent can mirror them by pushing the same stones one more step down (from odd to even again). Only odd-indexed piles matter strategically, and they behave exactly like standard Nim piles.

    Example

    Staircase with piles $a = [0, 3, 1, 4, 2, 5]$ (indices $0 \ldots 5$). Odd-indexed piles: $a_1 = 3, a_3 = 4, a_5 = 5$. XOR: $3 \oplus 4 \oplus 5 = 2 \neq 0$, so the first player wins.

    Winning move: reduce $a_5$ from $5$ to $5 \oplus 2 = 7$? No — we can only decrease pile sizes. Recalculate: we need to find an odd pile $a_j$ where $a_j \oplus 2 < a_j$. For $a_3 = 4$: $4 \oplus 2 = 6 > 4$ (no). For $a_1 = 3$: $3 \oplus 2 = 1 < 3$ (yes!). Move $3 - 1 = 2$ stones from position $1$ to position $0$. New odd-XOR: $1 \oplus 4 \oplus 5 = 0$. ✓

    Wythoff's Game

    Rules

    Two piles of sizes $a$ and $b$. On each turn a player may:

    • Remove any positive number of stones from one pile (standard Nim move), or
    • Remove the same positive number of stones from both piles simultaneously.

    The player who takes the last stone wins.

    P-Positions and the Golden Ratio

    Let $\varphi = \frac{1 + \sqrt{5}}{2} \approx 1.618$ be the golden ratio and $\varphi^2 = \varphi + 1 \approx 2.618$. The P-positions (losing positions for the player to move) are exactly the pairs:

    $$ (a_n, b_n) = \bigl(\lfloor n\varphi \rfloor,\; \lfloor n\varphi^2 \rfloor\bigr) \quad \text{for } n = 0, 1, 2, \ldots $$

    along with their reflections $(b_n, a_n)$.

    $n$$\lfloor n\varphi \rfloor$$\lfloor n\varphi^2 \rfloor$P-position
    000(0, 0)
    112(1, 2)
    235(3, 5)
    347(4, 7)
    4610(6, 10)
    5813(8, 13)
    6915(9, 15)
    71118(11, 18)

    Notice that $b_n - a_n = n$ for every P-position, and every non-negative integer appears exactly once across all $a_n$ and $b_n$ values.

    Proof via Beatty's Theorem

    Beatty's Theorem. If $\alpha, \beta > 1$ are irrational with $\frac{1}{\alpha} + \frac{1}{\beta} = 1$, then the Beatty sequences $\bigl(\lfloor n\alpha \rfloor\bigr)_{n \ge 1}$ and $\bigl(\lfloor n\beta \rfloor\bigr)_{n \ge 1}$ partition the positive integers.

    Setting $\alpha = \varphi$ and $\beta = \varphi^2$, we verify the condition:

    $$ \frac{1}{\varphi} + \frac{1}{\varphi^2} = (\varphi - 1) + (2 - \varphi) = 1. \quad\checkmark $$

    (Using $1/\varphi = \varphi - 1$ and $1/\varphi^2 = 2 - \varphi$, both consequences of $\varphi^2 = \varphi + 1$.)

    By Beatty's theorem, every positive integer appears as either some $a_n = \lfloor n\varphi \rfloor$ or some $b_n = \lfloor n\varphi^2 \rfloor$, but not both. This is crucial for showing the P-positions satisfy the required properties:

    1. Terminal position. $(0, 0) = (a_0, b_0)$ is a P-position. ✓
    2. Every move from a P-position leads to an N-position.

      Suppose we are at $(a_n, b_n)$. Any move changes at least one coordinate. Since Beatty's theorem gives a partition, each number appears in exactly one P-position pair. Changing any coordinate breaks this pairing, landing on an N-position. Specifically:

      • Reducing one pile: Changes one of $a_n$ or $b_n$ while preserving the other. The resulting pair cannot be another P-position because the partition property means $a_n$ already uniquely determines $n$ and hence $b_n$.
      • Reducing both piles equally: Let us remove $k$ from both, giving $(a_n - k, b_n - k)$. Note that $b_n - k - (a_n - k) = b_n - a_n = n$. If $(a_n - k, b_n - k)$ were $P$-position $(a_m, b_m)$, then $b_m - a_m = m = n$ but $a_m = a_n - k < a_n$, contradicting $a_n = \lfloor n\varphi \rfloor$ being the unique first component for index $n$.
    3. From every N-position, there exists a move to a P-position.

      Given any $(a, b)$ with $a \le b$ that is not a P-position, let $d = b - a$. The P-position with index $d$ is $(a_d, b_d) = (\lfloor d\varphi \rfloor, \lfloor d\varphi^2 \rfloor)$ with $b_d - a_d = d$.

      • If $a > a_d$: reduce pile $a$ to $a_d$ (and pile $b$ to $b$ which we verify gives an N... actually we need $b = b_d$ too). We can reduce just pile $a$ to $a_d$ — but then the difference changes. Instead, we consider all reachable P-positions:
        • Remove from pile $b$ only to reach $(a, b_m)$ for some $m$ where $a = a_m$.
        • Remove from pile $a$ only to reach $(a_m, b)$ for some $m$ where $b = b_m$.
        • Remove equally to reach $(a - k, b - k)$ which has difference $d$, targeting $(a_d, b_d)$ by setting $k = a - a_d$.
        By the partition property and case analysis, at least one of these is always reachable.

    The complete formal proof requires careful case analysis, but the key insight is that Beatty's partition guarantees the P-positions are well-structured. $\blacksquare$

    Efficient Check

    Given $(a, b)$ with $a \le b$, compute $n = b - a$ and check whether $a = \lfloor n\varphi \rfloor$. If yes, it is a P-position (second player wins); otherwise, the first player wins.

    // O(1) Wythoff check (with floating-point)
bool isWythoffP(long long a, long long b) {
    if (a > b) std::swap(a, b);
    long long n = b - a;
    const double phi = (1.0 + std::sqrt(5.0)) / 2.0;
    return a == (long long)(n * phi);
}
    Precision warning: For large values ($a, b > 10^{15}$), floating-point computation of $n\varphi$ may be inaccurate. Use continued-fraction or integer-only methods in that case (see C++ section below).

    Fibonacci Nim

    Rules

    A single pile of $n$ stones. The first player removes between $1$ and $n-1$ stones (cannot take the whole pile on the first move). Thereafter, each player may remove between $1$ and $2k$ stones, where $k$ is the number of stones the previous player removed. The player who takes the last stone wins.

    Zeckendorf's Representation

    Every positive integer has a unique representation as a sum of non-consecutive Fibonacci numbers. This is the Zeckendorf representation:

    $$ n = F_{i_1} + F_{i_2} + \cdots + F_{i_r} \quad \text{where } i_1 > i_2 > \cdots > i_r \ge 2 \text{ and } i_{j} - i_{j+1} \ge 2. $$

    For example: $20 = 13 + 5 + 2 = F_7 + F_5 + F_3$.

    Winning Condition

    Theorem (Fibonacci Nim / Schwenk's Theorem). In Fibonacci Nim, the first player loses if and only if $n$ is a Fibonacci number.

    Proof

    The proof relies on the Zeckendorf representation and a key lemma:

    Lemma. $F_k > 2 F_{k-2}$ for all $k \ge 4$. (Equivalently, $F_k - 1 \ge 2 F_{k-2}$.)

    Proof of Lemma: $F_k = F_{k-1} + F_{k-2} > 2F_{k-2}$ since $F_{k-1} > F_{k-2}$ for $k \ge 4$. $\square$

    Winning strategy when $n$ is not Fibonacci:

    Write $n$ in its Zeckendorf form: $n = F_{i_1} + F_{i_2} + \cdots + F_{i_r}$ with $r \ge 2$ and $i_1 > i_2 > \cdots > i_r$. The first player takes $F_{i_r}$ stones (the smallest Fibonacci summand).

    • This is a legal first move since $F_{i_r} < n$ (as $r \ge 2$).
    • The remaining pile has $n - F_{i_r} = F_{i_1} + \cdots + F_{i_{r-1}}$.
    • The opponent may now take at most $2 F_{i_r}$ stones. By the lemma and the non-consecutivity of Zeckendorf terms, $F_{i_{r-1}} \ge F_{i_r + 2} > 2 F_{i_r}$. So the opponent cannot take the entire next Fibonacci summand in one move.
    • The game on the remaining pile now has a Zeckendorf representation, and the same strategy applies recursively: always take the smallest Fibonacci summand in the current Zeckendorf decomposition.

    Losing when $n = F_k$:

    If $n$ is a Fibonacci number, say $n = F_k$, then the first player takes some $m$ stones where $1 \le m \le F_k - 1$. Write $m$ in Zeckendorf form and note the resulting pile is $F_k - m$. The second player now adopts the Zeckendorf strategy on the remaining pile. The key is that no matter what $m$ the first player chooses, the second player can always respond optimally because the constraint $m \le F_k - 1$ ensures the Zeckendorf structure is maintained for the second player's strategy.

    The formal proof uses strong induction on $n$ and the identity $F_k - 1 = F_{k-2} + F_{k-4} + \cdots$ (alternating Fibonacci sum). $\blacksquare$

    Example

    $n = 30 = 21 + 8 + 1 = F_8 + F_6 + F_2$. First player takes $F_2 = 1$ stone. Remaining: $29 = 21 + 8$. Opponent can take at most $2 \times 1 = 2$ stones. They take $2$, leaving $27$. Zeckendorf of $27 = 21 + 5 + 1$. First player takes $1$. Continue recursively.

    $n = 13 = F_7$: First player loses — every possible opening move gives the second player a winning Zeckendorf strategy.

    Moore's Nim

    Rules

    Generalisation of standard Nim: there are $n$ piles and on each turn a player may remove any positive number of stones from at most $k$ piles (choosing which piles and how many from each independently). Standard Nim is the special case $k = 1$.

    Winning Condition

    Theorem (Moore, 1910). In Nim with $n$ piles where each player may remove from at most $k$ piles per turn, the position $(a_1, a_2, \ldots, a_n)$ is a P-position if and only if, for every bit position $j \ge 0$: $$ \sum_{i=1}^{n} \operatorname{bit}_j(a_i) \equiv 0 \pmod{k+1}, $$ where $\operatorname{bit}_j(x)$ denotes the $j$-th bit of $x$.

    For $k = 1$, the condition becomes: each bit-column sums to $0 \pmod{2}$, which is exactly the XOR condition of standard Nim.

    Proof Sketch

    We verify the P-position characterization:

    1. Terminal: All piles are $0$, so every bit-sum is $0 \equiv 0 \pmod{k+1}$. ✓
    2. From P to N (any move disrupts):

      A single move changes at most $k$ piles. Consider the highest bit position $j^*$ where any pile changes. At most $k$ piles change their $j^*$-th bit, so the sum of $j^*$-th bits changes by at most $k$ (in absolute value). Since the original sum was $\equiv 0 \pmod{k+1}$ and the change is between $1$ and $k$ (at least one pile changed), the new sum is $\not\equiv 0 \pmod{k+1}$. ✓

    3. From N to P (a fixing move exists):

      Find the highest bit $j^*$ where the sum $\not\equiv 0 \pmod{k+1}$. We need to "fix" this bit by adjusting at most $k$ piles. Since each pile contributes either $0$ or $1$ to bit $j^*$, and we need to change the sum by some value $d$ where $1 \le d \le k$, we select $d$ piles that have bit $j^*$ set (or not set, as needed) and modify them. The key insight is that we can always fix all bits from high to low by modifying at most $k$ piles, because once we commit to changing a pile, we can freely set all its lower bits. ✓

    Example

    $k = 2$, piles $= [5, 7, 3, 6]$. Write in binary:

    5 = 101
7 = 111
3 = 011
6 = 110
---------
bit sums: bit2=3, bit1=3, bit0=2

    Check mod $k+1 = 3$: bit2 sum $= 3 \equiv 0$, bit1 sum $= 3 \equiv 0$, bit0 sum $= 2 \not\equiv 0$. Not all zero — this is an N-position (first player wins).

    Turning Games

    Turning games are played on a row of coins, each showing heads (H) or tails (T). A move consists of turning over (flipping) a set of coins according to specific rules, with the constraint that the rightmost coin flipped must go from H to T. The player who turns over the last head wins. These are naturally Nim-like because each move reduces the "weight" of the position.

    Mock Turtles (Turning Turtles)

    In Turning Turtles, a move flips any single coin from H to T, and optionally also flips one other coin to its left (which may go H→T or T→H). The game is equivalent to Nim with pile sizes being the positions (0-indexed) of all heads.

    Formally, if heads appear at positions $p_1, p_2, \ldots, p_r$, then the Grundy value is:

    $$ g = p_1 \oplus p_2 \oplus \cdots \oplus p_r. $$

    First player wins if and only if $g \neq 0$.

    Ruler Game

    In the Ruler game, a move flips a contiguous block of $1, 2, \ldots$ coins where the rightmost coin in the block goes from H to T. Equivalently, the player picks a head at position $p$ and flips all coins from some position $q \le p$ through $p$.

    The Grundy value for a single head at position $p$ is the largest power of $2$ dividing $p + 1$, which corresponds to the "ruler function" — hence the name. For a configuration with heads at positions $p_1, \ldots, p_r$:

    $$ g = \nu_2(p_1 + 1) \oplus \nu_2(p_2 + 1) \oplus \cdots \oplus \nu_2(p_r + 1), $$

    where $\nu_2(m)$ denotes the 2-adic valuation (exponent of the largest power of 2 dividing $m$).

    General principle: Most turning games can be analyzed using the Sprague-Grundy theorem by computing Grundy values for individual coins and XOR-ing them together. The challenge lies in determining the Grundy value for each coin position given the specific turning rules.

    C++ Implementations

    Staircase Nim

    #include <bits/stdc++.h>
using namespace std;

// Returns true if the first player wins Staircase Nim.
// a[0..n-1]: stone counts at positions 0..n-1.
bool staircaseNim(const vector<int>& a) {
    int xorOdd = 0;
    for (int i = 1; i < (int)a.size(); i += 2)
        xorOdd ^= a[i];
    return xorOdd != 0;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<int> a(n);
    for (auto& x : a) cin >> x;

    cout << (staircaseNim(a) ? "First" : "Second") << "\n";
}

    Wythoff's Game

    #include <bits/stdc++.h>
using namespace std;

// Returns true if (a, b) is a P-position in Wythoff's game.
// Uses the golden-ratio formula. For a, b up to ~10^15 this is safe.
bool isWythoffP(long long a, long long b) {
    if (a > b) swap(a, b);
    long long diff = b - a;
    // phi = (1 + sqrt(5)) / 2
    // Use long double for better precision
    long double phi = (1.0L + sqrtl(5.0L)) / 2.0L;
    long long expected_a = (long long)(diff * phi);
    // Check expected_a and expected_a ± 1 to handle precision issues
    for (long long ea = expected_a - 1; ea <= expected_a + 1; ea++) {
        if (ea < 0) continue;
        long long eb = ea + diff;
        if (ea == a && eb == b) return true;
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t;
    cin >> t;
    while (t--) {
        long long a, b;
        cin >> a >> b;
        // P-position means second player wins
        cout << (isWythoffP(a, b) ? "Second" : "First") << "\n";
    }
}

    Fibonacci Nim — Zeckendorf Strategy

    #include <bits/stdc++.h>
using namespace std;

// Precompute Fibonacci numbers up to 10^18
vector<long long> fibs;

void precomputeFibs(long long maxVal) {
    fibs = {1, 2};
    while (true) {
        long long nxt = fibs[fibs.size()-1] + fibs[fibs.size()-2];
        if (nxt > maxVal) break;
        fibs.push_back(nxt);
    }
}

// Returns the Zeckendorf representation (sorted descending)
vector<long long> zeckendorf(long long n) {
    vector<long long> rep;
    for (int i = (int)fibs.size() - 1; i >= 0 && n > 0; i--) {
        if (fibs[i] <= n) {
            rep.push_back(fibs[i]);
            n -= fibs[i];
        }
    }
    return rep;
}

// Returns true if n is a Fibonacci number
bool isFibonacci(long long n) {
    for (auto f : fibs)
        if (f == n) return true;
    return false;
}

// If first player wins, returns the optimal first move (stones to take).
// If first player loses (n is Fibonacci), returns -1.
long long fibonacciNimMove(long long n) {
    if (isFibonacci(n)) return -1;
    vector<long long> z = zeckendorf(n);
    return z.back();  // take the smallest Fibonacci summand
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    precomputeFibs(2e18);

    int t;
    cin >> t;
    while (t--) {
        long long n;
        cin >> n;
        if (isFibonacci(n)) {
            cout << "Second\n";
        } else {
            long long move = fibonacciNimMove(n);
            cout << "First (take " << move << ")\n";
        }
    }
}

    Moore's Nim ($k$-Nim)

    #include <bits/stdc++.h>
using namespace std;

// Returns true if the first player wins Moore's Nim.
// piles: array of pile sizes, k: max piles removable per turn.
bool mooresNim(const vector<int>& piles, int k) {
    // For each bit position, sum the bits across all piles.
    // If every bit-sum ≡ 0 (mod k+1), it's a P-position.
    int mod = k + 1;
    for (int bit = 0; bit < 30; bit++) {
        int sum = 0;
        for (int p : piles)
            sum += (p >> bit) & 1;
        if (sum % mod != 0)
            return true;  // N-position: first player wins
    }
    return false;  // P-position: second player wins
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;
    vector<int> piles(n);
    for (auto& x : piles) cin >> x;

    cout << (mooresNim(piles, k) ? "First" : "Second") << "\n";
}

    Turning Turtles

    #include <bits/stdc++.h>
using namespace std;

// coins[i] = 1 means head, 0 means tail.
// Returns true if first player wins Turning Turtles.
bool turningTurtles(const vector<int>& coins) {
    int grundy = 0;
    for (int i = 0; i < (int)coins.size(); i++)
        if (coins[i]) grundy ^= i;
    return grundy != 0;
}

// Ruler game: Grundy value uses 2-adic valuation
int rulerVal(int pos) {
    // Largest power of 2 dividing (pos + 1)
    int v = pos + 1;
    return v & (-v);  // isolates lowest set bit = 2^(v2(pos+1))
}

bool rulerGame(const vector<int>& coins) {
    int grundy = 0;
    for (int i = 0; i < (int)coins.size(); i++)
        if (coins[i]) grundy ^= rulerVal(i);
    return grundy != 0;
}
