Subtraction Games

June 2026 · 25 min read

Subtraction games are one of the cleanest families of combinatorial games. A single pile of stones, a fixed set of allowed moves, and a surprisingly elegant periodicity in the resulting Grundy values — they form the perfect bridge between the abstract Sprague-Grundy framework and concrete competitive programming problems. In this post we define the game, derive Grundy values via dynamic programming, prove the periodicity theorem, work through detailed examples, extend to multi-pile variants, and build optimized C++ code ready for contest use.

Definition of Subtraction Games

A subtraction game is a two-player, impartial, normal-play combinatorial game defined by:

  • A single pile of \(n\) stones (\(n \ge 0\)).
  • A finite subtraction set \(S = \{s_1, s_2, \ldots, s_k\}\) where each \(s_i \in \mathbb{Z}^+\).
  • Players alternate turns. On each turn a player removes exactly \(s\) stones from the pile for some \(s \in S\) with \(s \le n\).
  • The player who takes the last stone wins (normal play convention). Equivalently, the player unable to move (facing \(n = 0\)) loses.

Formally, the set of positions is \(\mathbb{N}_0 = \{0, 1, 2, \ldots\}\). From position \(n\), the set of options (positions reachable in one move) is:

$$\text{opt}(n) = \{ n - s \mid s \in S,\ s \le n \}$$

Position \(n = 0\) is a terminal P-position (previous player wins, i.e., the player who just moved wins). Whether a general position \(n\) is a P-position or N-position (next player wins) is determined by the Sprague-Grundy theory.

Convention. We assume \(S\) is given in sorted order \(s_1 < s_2 < \cdots < s_k\). The game is fully determined by \(S\); changing \(S\) produces a different game.

Example. The classic game of Nim with maximum removal \(m\) is a subtraction game with \(S = \{1, 2, \ldots, m\}\). The game "remove 1 or 3 stones" uses \(S = \{1, 3\}\).

Computing Grundy Values

The Grundy value (or nimber) of position \(n\) in a subtraction game is computed recursively via the minimum excludant (mex) function:

$$\mathcal{G}(n) = \operatorname{mex}\bigl(\{\mathcal{G}(n - s) \mid s \in S,\ s \le n\}\bigr)$$

where \(\operatorname{mex}(A)\) is the smallest non-negative integer not in the set \(A\).

Base case: \(\mathcal{G}(0) = \operatorname{mex}(\emptyset) = 0\), since no moves are available from an empty pile.

DP Approach

We compute \(\mathcal{G}(0), \mathcal{G}(1), \ldots, \mathcal{G}(N)\) iteratively. For each position \(n\), we collect the Grundy values of all reachable positions and take their mex. The mex of a set of at most \(k\) values can be computed in \(O(k)\) time using a boolean array. 

// Build Grundy table for subtraction game with set S
// Time: O(N * |S|), Space: O(N)
vector<int> grundy_table(int N, const vector<int>& S) {
    vector<int> G(N + 1, 0);
    for (int n = 1; n <= N; n++) {
        // Collect Grundy values of reachable positions
        vector<bool> seen(S.size() + 1, false);
        for (int s : S) {
            if (s <= n && G[n - s] < (int)seen.size())
                seen[G[n - s]] = true;
        }
        // mex: smallest non-negative integer not in 'seen'
        int mex = 0;
        while (mex < (int)seen.size() && seen[mex]) mex++;
        G[n] = mex;
    }
    return G;
}

Detailed computation:

  • \(\mathcal{G}(0) = 0\) (base case)
  • \(\mathcal{G}(1) = \operatorname{mex}(\{\mathcal{G}(0)\}) = \operatorname{mex}(\{0\}) = 1\)
  • \(\mathcal{G}(2) = \operatorname{mex}(\{\mathcal{G}(1)\}) = \operatorname{mex}(\{1\}) = 0\)   (only \(s=1\) applies)
  • \(\mathcal{G}(3) = \operatorname{mex}(\{\mathcal{G}(2), \mathcal{G}(0)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
  • \(\mathcal{G}(4) = \operatorname{mex}(\{\mathcal{G}(3), \mathcal{G}(1), \mathcal{G}(0)\}) = \operatorname{mex}(\{1, 1, 0\}) = 2\)
  • \(\mathcal{G}(5) = \operatorname{mex}(\{\mathcal{G}(4), \mathcal{G}(2), \mathcal{G}(1)\}) = \operatorname{mex}(\{2, 0, 1\}) = 3\)
  • \(\mathcal{G}(6) = \operatorname{mex}(\{\mathcal{G}(5), \mathcal{G}(3), \mathcal{G}(2)\}) = \operatorname{mex}(\{3, 1, 0\}) = 2\)

Actually, for \(S = \{1, 3, 4\}\) and \(n = 6\): moves go to positions \(5, 3, 2\) with Grundy values \(3, 1, 0\). So \(\operatorname{mex}(\{3, 1, 0\}) = 2\). Continuing carefully, \(\mathcal{G}(7) = \operatorname{mex}(\{G(6),G(4),G(3)\}) = \operatorname{mex}(\{2,2,1\}) = 0\). The full corrected table:

n 01 23 45 67 89 1011
G(n) 01 01 23 20 10 12

We verify computationally that this sequence becomes periodic with period 7 after preperiod 0: the block \([0, 1, 0, 1, 2, 3, 2]\) repeats. This illustrates a general phenomenon.

Periodicity Theorem

The most important structural result about subtraction games is that their Grundy sequences are eventually periodic.

Theorem (Sprague-Grundy Periodicity for Subtraction Games). Let \(S = \{s_1, s_2, \ldots, s_k\}\) be a subtraction set with \(s_k = \max(S)\). Then the Grundy sequence \(\mathcal{G}(0), \mathcal{G}(1), \mathcal{G}(2), \ldots\) is eventually periodic. That is, there exist integers \(n_0 \ge 0\) (the preperiod) and \(p \ge 1\) (the period) such that for all \(n \ge n_0\): $$\mathcal{G}(n + p) = \mathcal{G}(n)$$

Proof Sketch

The key insight is that \(\mathcal{G}(n)\) depends only on the values \(\mathcal{G}(n - s_1), \mathcal{G}(n - s_2), \ldots, \mathcal{G}(n - s_k)\). Define the state vector:

$$\vec{v}(n) = \bigl(\mathcal{G}(n - s_1),\ \mathcal{G}(n - s_2),\ \ldots,\ \mathcal{G}(n - s_k)\bigr) \in \mathbb{N}_0^k$$

Since \(\mathcal{G}(n) = \operatorname{mex}(\vec{v}(n))\), the value \(\mathcal{G}(n)\) is a function of \(\vec{v}(n)\) alone.

Step 1: Bounded Grundy values.

We claim \(\mathcal{G}(n) \le k\) for all \(n\). The mex of a set with at most \(k\) elements is at most \(k\), since a set of \(k\) non-negative integers can exclude at most the values \(\{0, 1, \ldots, k-1\}\), giving mex \(= k\). Thus each component of \(\vec{v}(n)\) lies in \(\{0, 1, \ldots, k\}\).

Step 2: Finite state space.

The vector \(\vec{v}(n)\) takes values in a finite set of size at most \((k+1)^k\). By the pigeonhole principle, as \(n\) increases, eventually two indices \(n_1 < n_2\) must share the same state vector: \(\vec{v}(n_1) = \vec{v}(n_2)\).

Step 3: Periodicity propagates.

Let \(p = n_2 - n_1\). We show by induction that \(\mathcal{G}(n + p) = \mathcal{G}(n)\) for all \(n \ge n_1\).

  • Base: \(\vec{v}(n_1) = \vec{v}(n_2)\) implies \(\mathcal{G}(n_1) = \operatorname{mex}(\vec{v}(n_1)) = \operatorname{mex}(\vec{v}(n_2)) = \mathcal{G}(n_2) = \mathcal{G}(n_1 + p)\).
  • Inductive step: Suppose \(\mathcal{G}(m + p) = \mathcal{G}(m)\) for all \(n_1 \le m \le n\). Then for \(n+1\): $$\vec{v}(n+1+p) = \bigl(\mathcal{G}(n+1+p - s_i)\bigr)_{i=1}^k = \bigl(\mathcal{G}(n+1 - s_i)\bigr)_{i=1}^k = \vec{v}(n+1)$$ where the second equality uses the induction hypothesis (each \(n+1+p - s_i \ge n_1 + p\) and \(n+1 - s_i \ge n_1\), since \(s_i \le s_k\) and \(n \ge n_1 + s_k - 1\) for sufficiently large \(n\)). Therefore \(\mathcal{G}(n+1+p) = \operatorname{mex}(\vec{v}(n+1+p)) = \operatorname{mex}(\vec{v}(n+1)) = \mathcal{G}(n+1)\).  \(\blacksquare\)

Bounds on the Period

The state space argument gives a crude upper bound of \((k+1)^k\) on the period. In practice, periods are typically much smaller. For many natural subtraction sets, the period divides \(\text{lcm}(s_1, s_2, \ldots, s_k)\) or equals \(s_k + s_1\) or similar small expressions. Experimentally:

  • \(S = \{1, 2, \ldots, m\}\): period = \(m + 1\), preperiod = 0.
  • \(S = \{1, s\}\) for \(s\) odd: period = \(s + 1\), preperiod = 0.
  • \(S = \{a, b\}\) with \(\gcd(a, b) = 1\): period is typically \(a + b\).
Computational strategy. In competitive programming, we often compute Grundy values up to \(2 \cdot \max(S)\) or a bit more, detect the period, and then answer queries for arbitrary \(n\) in \(O(1)\) using \(\mathcal{G}(n) = \mathcal{G}(n_0 + ((n - n_0) \bmod p))\).

Examples and Patterns

Example 1: S = {1, 2, 3} — Bounded Nim

Players can remove 1, 2, or 3 stones. Since \(|S| = 3\), Grundy values are at most 3. Computing:

n 01 23 45 67 89 1011
G(n) 01 23 01 23 01 23

Pattern: \(\mathcal{G}(n) = n \bmod 4\). Period = 4 = \(\max(S) + 1\), preperiod = 0.

This generalizes: for \(S = \{1, 2, \ldots, m\}\), we get \(\mathcal{G}(n) = n \bmod (m+1)\). The proof is straightforward by induction: the set of reachable Grundy values from position \(n\) is \(\{(n-1)\bmod(m+1),\ldots,(n-m)\bmod(m+1)\}\), which is exactly \(\{0,\ldots,m\} \setminus \{n\bmod(m+1)\}\) when the values wrap around modulo \(m+1\). Hence \(\operatorname{mex} = n \bmod (m+1)\).

Example 2: S = {1, 3} — Detailed Analysis

Compute step by step:

  • \(\mathcal{G}(0) = 0\)
  • \(\mathcal{G}(1) = \operatorname{mex}(\{G(0)\}) = \operatorname{mex}(\{0\}) = 1\)
  • \(\mathcal{G}(2) = \operatorname{mex}(\{G(1)\}) = \operatorname{mex}(\{1\}) = 0\)
  • \(\mathcal{G}(3) = \operatorname{mex}(\{G(2), G(0)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
  • \(\mathcal{G}(4) = \operatorname{mex}(\{G(3), G(1)\}) = \operatorname{mex}(\{1, 1\}) = 0\)
  • \(\mathcal{G}(5) = \operatorname{mex}(\{G(4), G(2)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
  • \(\mathcal{G}(6) = \operatorname{mex}(\{G(5), G(3)\}) = \operatorname{mex}(\{1, 1\}) = 0\)
  • \(\mathcal{G}(7) = \operatorname{mex}(\{G(6), G(4)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
n 01 23 45 67
G(n) 01 01 01 01

Pattern: \(\mathcal{G}(n) = n \bmod 2\). Period = 2, preperiod = 0. Since both 1 and 3 are odd, removing any element from \(S\) changes the parity of the pile. So even positions always move to odd positions and vice versa, giving the alternating pattern.

Game-theoretic meaning: P-positions (losing for the player to move) are exactly the even pile sizes. First player wins iff \(n\) is odd.

Example 3: S = {2, 5} — Non-trivial Period

When \(1 \notin S\), the structure becomes more interesting because some positions have no moves (specifically, position 1 is a P-position since neither 2 nor 5 can be subtracted from 1).

  • \(\mathcal{G}(0) = \operatorname{mex}(\emptyset) = 0\)
  • \(\mathcal{G}(1) = \operatorname{mex}(\emptyset) = 0\)  (no valid moves)
  • \(\mathcal{G}(2) = \operatorname{mex}(\{G(0)\}) = \operatorname{mex}(\{0\}) = 1\)
  • \(\mathcal{G}(3) = \operatorname{mex}(\{G(1)\}) = \operatorname{mex}(\{0\}) = 1\)
  • \(\mathcal{G}(4) = \operatorname{mex}(\{G(2)\}) = \operatorname{mex}(\{1\}) = 0\)
  • \(\mathcal{G}(5) = \operatorname{mex}(\{G(3), G(0)\}) = \operatorname{mex}(\{1, 0\}) = 2\)
  • \(\mathcal{G}(6) = \operatorname{mex}(\{G(4), G(1)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
  • \(\mathcal{G}(7) = \operatorname{mex}(\{G(5), G(2)\}) = \operatorname{mex}(\{2, 1\}) = 0\)
  • \(\mathcal{G}(8) = \operatorname{mex}(\{G(6), G(3)\}) = \operatorname{mex}(\{1, 1\}) = 0\)
  • \(\mathcal{G}(9) = \operatorname{mex}(\{G(7), G(4)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
  • \(\mathcal{G}(10) = \operatorname{mex}(\{G(8), G(5)\}) = \operatorname{mex}(\{0, 2\}) = 1\)
  • \(\mathcal{G}(11) = \operatorname{mex}(\{G(9), G(6)\}) = \operatorname{mex}(\{1, 1\}) = 0\)
  • \(\mathcal{G}(12) = \operatorname{mex}(\{G(10), G(7)\}) = \operatorname{mex}(\{1, 0\}) = 2\)
  • \(\mathcal{G}(13) = \operatorname{mex}(\{G(11), G(8)\}) = \operatorname{mex}(\{0, 0\}) = 1\)
n 01 23 45 67 89 1011 1213
G(n) 00 11 02 10 01 10 21

Pattern: After a preperiod of \(n_0 = 0\), the sequence is periodic with period \(p = 7\): \([0, 0, 1, 1, 0, 2, 1]\) repeating. Note \(p = 2 + 5 = \max(S) + \min(S)\). So \(\mathcal{G}(n) = \mathcal{G}(n \bmod 7)\) for all \(n \ge 0\).

Key observation. When \(1 \notin S\), some positions may have Grundy value 0 even though \(n > 0\) (dead positions where no move is valid). This makes the game richer and the periodicity pattern less obvious.

Multi-Pile Subtraction Games

In competitive programming, subtraction games frequently appear in multi-pile form. We have \(m\) independent piles of sizes \(n_1, n_2, \ldots, n_m\), each governed by the same subtraction set \(S\). On each turn, a player chooses one pile and removes \(s \in S\) stones from it.

By the Sprague-Grundy theorem, the combined game's Grundy value is the XOR of individual Grundy values:

$$\mathcal{G}(n_1, n_2, \ldots, n_m) = \mathcal{G}(n_1) \oplus \mathcal{G}(n_2) \oplus \cdots \oplus \mathcal{G}(n_m)$$

The first player wins if and only if this XOR is non-zero.

Algorithm

  1. Compute the Grundy table for the subtraction game up to \(\max(n_i)\) (or detect the period and use modular lookup).
  2. For each pile \(n_i\), look up \(\mathcal{G}(n_i)\).
  3. XOR all values. If the result is non-zero, first player wins.

Finding a Winning Move

If the overall XOR \(X = \mathcal{G}(n_1) \oplus \cdots \oplus \mathcal{G}(n_m) \neq 0\), we need to find a pile \(i\) and a move \(s \in S\) such that after the move, the new XOR is 0. That is, we need:

$$\mathcal{G}(n_i - s) = \mathcal{G}(n_i) \oplus X$$

We iterate over piles and moves, checking if any \(s \in S\) with \(s \le n_i\) satisfies \(\mathcal{G}(n_i - s) = \mathcal{G}(n_i) \oplus X\). By the Sprague-Grundy theorem, at least one such move must exist. 

// Returns {pile_index, stones_to_remove} or {-1, -1} if losing
pair<int,int> find_winning_move(
    const vector<int>& piles,
    const vector<int>& G,    // precomputed Grundy table
    const vector<int>& S,
    int period, int preperiod)
{
    auto grundy = [&](int n) -> int {
        if (n < (int)G.size()) return G[n];
        return G[preperiod + (n - preperiod) % period];
    };

    int xor_sum = 0;
    for (int pile : piles) xor_sum ^= grundy(pile);
    if (xor_sum == 0) return {-1, -1}; // losing position

    for (int i = 0; i < (int)piles.size(); i++) {
        int target = grundy(piles[i]) ^ xor_sum;
        for (int s : S) {
            if (s <= piles[i] && grundy(piles[i] - s) == target)
                return {i, s};
        }
    }
    return {-1, -1}; // should never reach here
}

Variant: Different Subtraction Sets per Pile

If each pile \(i\) has its own subtraction set \(S_i\), the Sprague-Grundy theorem still applies. We simply compute separate Grundy sequences for each \(S_i\) and XOR them:

$$\mathcal{G}_{\text{total}} = \mathcal{G}_{S_1}(n_1) \oplus \mathcal{G}_{S_2}(n_2) \oplus \cdots \oplus \mathcal{G}_{S_m}(n_m)$$

C++ Implementation

Below is a complete, contest-ready C++ solution that handles subtraction games efficiently. It computes the Grundy table, automatically detects the periodic pattern, and answers multi-pile game queries in \(O(1)\) per pile. 

#include <bits/stdc++.h>
using namespace std;

struct SubtractionGame {
    vector<int> S;         // subtraction set (sorted)
    vector<int> G;         // Grundy values
    int period = 0;
    int preperiod = 0;

    SubtractionGame(vector<int> sub_set) : S(move(sub_set)) {
        sort(S.begin(), S.end());
        compute();
    }

    void compute() {
        int maxS = S.back();
        // Compute enough values to detect period
        // Upper bound: compute 3 * maxS + maxS^2 values
        int limit = max(3 * maxS, maxS * maxS) + maxS;
        G.assign(limit + 1, 0);

        for (int n = 1; n <= limit; n++) {
            vector<bool> seen((int)S.size() + 2, false);
            for (int s : S) {
                if (s <= n && G[n - s] < (int)seen.size())
                    seen[G[n - s]] = true;
            }
            int m = 0;
            while (m < (int)seen.size() && seen[m]) m++;
            G[n] = m;
        }

        detect_period();
    }

    void detect_period() {
        int maxS = S.back();
        int n = (int)G.size();
        // Try each possible period p from 1 to n/3
        for (int p = 1; p <= n / 3; p++) {
            // Check if G[i] == G[i+p] for all i in [n/3 .. 2n/3]
            int start = n / 3;
            bool ok = true;
            for (int i = start; i < start + p + maxS; i++) {
                if (i + p < n && G[i] != G[i + p]) {
                    ok = false;
                    break;
                }
            }
            if (ok) {
                period = p;
                // Find earliest preperiod
                preperiod = 0;
                for (int i = 0; i < start; i++) {
                    if (G[i] != G[i + p]) {
                        preperiod = i + 1;
                    }
                }
                return;
            }
        }
    }

    int grundy(long long n) const {
        if (n < (int)G.size()) return G[n];
        if (period == 0) {
            // No period detected; should not happen for subtraction games
            return -1;
        }
        return G[preperiod + (int)((n - preperiod) % period)];
    }

    // Returns true if first player wins with piles of given sizes
    bool first_wins(const vector<long long>& piles) const {
        int xor_sum = 0;
        for (long long pile : piles)
            xor_sum ^= grundy(pile);
        return xor_sum != 0;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int k;
    cin >> k;
    vector<int> S(k);
    for (int& s : S) cin >> s;

    SubtractionGame game(S);

    int q;
    cin >> q;
    while (q--) {
        int m;
        cin >> m;
        vector<long long> piles(m);
        for (long long& p : piles) cin >> p;

        cout << (game.first_wins(piles) ? "First" : "Second") << "\n";
    }

    return 0;
}

Complexity Analysis

  • Preprocessing: \(O(L \cdot k)\) where \(L\) is the table size and \(k = |S|\). With \(L = O((\max S)^2)\), this is \(O(k \cdot (\max S)^2)\).
  • Period detection: \(O(L)\) time.
  • Per query: \(O(m)\) where \(m\) is the number of piles (one Grundy lookup per pile, each \(O(1)\)).
  • Memory: \(O(L)\) for the Grundy table.

Optimized Period Detection

A cleaner approach to detect the period uses the suffix matching idea: check if the last \(2p\) values of the computed table form two identical blocks of length \(p\). 

// Detect smallest period p such that G[n-2p..n-p-1] == G[n-p..n-1]
int detect_period_suffix(const vector<int>& G, int maxS) {
    int n = (int)G.size();
    for (int p = 1; p <= n / 3; p++) {
        bool match = true;
        // Verify over a window larger than maxS to ensure true periodicity
        for (int i = n - 2 * p - maxS; i < n - p; i++) {
            if (i < 0) continue;
            if (G[i] != G[i + p]) { match = false; break; }
        }
        if (match) return p;
    }
    return 0; // no period found in computed range
}
