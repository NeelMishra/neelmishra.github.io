    Stirling Numbers: First & Second Kind

    July 2026 · 20 min read

    Stirling Numbers of the Second Kind S(n, k)

    The Stirling number of the second kind, denoted S(n, k) or {n \brace k}, counts the number of ways to partition a set of n labelled elements into exactly k non-empty, unordered subsets.

    For example, S(4, 2) = 7: the set {a, b, c, d} can be split into 2 non-empty subsets in 7 ways: 

    {a} | {b,c,d}     {b} | {a,c,d}     {c} | {a,b,d}     {d} | {a,b,c}
{a,b} | {c,d}     {a,c} | {b,d}     {a,d} | {b,c}

    The first few values form a triangle: 

    S(0,0) = 1
S(1,1) = 1
S(2,1) = 1   S(2,2) = 1
S(3,1) = 1   S(3,2) = 3   S(3,3) = 1
S(4,1) = 1   S(4,2) = 7   S(4,3) = 6   S(4,4) = 1
S(5,1) = 1   S(5,2) = 15  S(5,3) = 25  S(5,4) = 10  S(5,5) = 1
    Boundary conditions: S(n, 0) = 0 for n ≥ 1 (can't partition non-empty set into 0 parts), S(0, 0) = 1 (empty partition of empty set), S(n, 1) = 1 (everything in one subset), S(n, n) = 1 (each element alone).

    Recurrence for S(n, k)

    The Stirling numbers of the second kind satisfy:

    S(n, k) = k · S(n − 1, k) + S(n − 1, k − 1)

    Combinatorial proof: Consider element n. It either:

    1. Joins an existing subset: Remove element n from its subset. We have a partition of {1, …, n−1} into k subsets, and element n can join any of the k subsets. This gives k · S(n−1, k).
    2. Forms a new singleton subset: The remaining n−1 elements are partitioned into k−1 subsets. This gives S(n−1, k−1). 
    // DP for Stirling numbers of the second kind
const int MOD = 1e9 + 7;
long long S[5001][5001];

void compute_stirling2(int n) {
    S[0][0] = 1;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= i; j++) {
            S[i][j] = (j * S[i-1][j] % MOD + S[i-1][j-1]) % MOD;
        }
    }
}
    Memory warning: A 5001 × 5001 array of long long uses about 200 MB, which may exceed memory limits. For large n, use a rolling array (only keep two rows) or the PIE-based explicit formula.

    Stirling Numbers of the First Kind s(n, k)

    The (unsigned) Stirling number of the first kind, denoted |s(n, k)| or [n \atop k], counts the number of permutations of n elements that have exactly k cycles.

    For example, |s(4, 2)| = 11: there are 11 permutations of {1, 2, 3, 4} with exactly 2 cycles. The permutation (1 3)(2 4) has 2 cycles: (1 → 3 → 1) and (2 → 4 → 2).

    The first few values:

    |s(0,0)| = 1
|s(1,1)| = 1
|s(2,1)| = 1   |s(2,2)| = 1
|s(3,1)| = 2   |s(3,2)| = 3   |s(3,3)| = 1
|s(4,1)| = 6   |s(4,2)| = 11  |s(4,3)| = 6   |s(4,4)| = 1
    Signed vs unsigned: The signed Stirling numbers of the first kind are s(n, k) = (−1)n−k |s(n, k)|. In CP, we almost always work with the unsigned version |s(n, k)|.

    Note that the sum across a row gives the total number of permutations: ∑k |s(n, k)| = n! (since every permutation has some number of cycles).

    Recurrence for s(n, k)

    The unsigned Stirling numbers of the first kind satisfy:

    |s(n, k)| = (n − 1) · |s(n − 1, k)| + |s(n − 1, k − 1)|

    Combinatorial proof: Consider element n in a permutation. It either:

    1. Inserts into an existing cycle: Take a permutation of {1, …, n−1} with k cycles. Element n can be inserted after any of the n−1 elements in the permutation, giving (n−1) · |s(n−1, k)|.
    2. Forms a new fixed-point cycle (n): The remaining n−1 elements form a permutation with k−1 cycles, giving |s(n−1, k−1)|. 
    // DP for unsigned Stirling numbers of the first kind
const int MOD = 1e9 + 7;
long long s1[5001][5001];

void compute_stirling1(int n) {
    s1[0][0] = 1;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= i; j++) {
            s1[i][j] = ((i - 1) * s1[i-1][j] % MOD + s1[i-1][j-1]) % MOD;
        }
    }
}
    Key difference from second kind: The coefficient in S(n,k) is k (number of existing subsets to join), while in |s(n,k)| it is n−1 (number of positions in existing cycles where n can be inserted).

    Connection to Factorials and Polynomials

    Stirling numbers form the bridge between ordinary powers xn and falling/rising factorials. These are the change-of-basis formulas for polynomial representation.

    Falling Factorials

    Define the falling factorial x(k) = x(x−1)(x−2)…(x−k+1). Then:

    xn = ∑k=0n S(n, k) · x(k)

    This expresses ordinary powers in terms of falling factorials. The Stirling numbers of the second kind are the conversion coefficients.

    Rising Factorials

    Define the rising factorial x(k) = x(x+1)(x+2)…(x+k−1). Then:

    x(n) = ∑k=0n |s(n, k)| · xk

    This expresses rising factorials in terms of ordinary powers. The unsigned Stirling numbers of the first kind are these conversion coefficients.

    Inverse Relationship

    The falling factorial expansion gives:

    x(n) = ∑k=0n s(n, k) · xk

    where s(n, k) are the signed Stirling numbers of the first kind. In matrix form, the Stirling number matrices of the first and second kind are inverses of each other!

    Why this matters: Many CP problems involve sums of the form ∑ f(x) where f is a polynomial. Converting between the xn basis and the falling factorial basis (using Stirling numbers) often simplifies these sums dramatically, because ∑x=0m x(k) has a clean closed form.

    Explicit Formula via PIE

    The Stirling number of the second kind has an explicit formula derived from the Principle of Inclusion-Exclusion:

    S(n, k) = (1/k!) ∑j=0k (−1)j C(k, j) (k − j)n

    Derivation sketch:

    1. Count surjections from an n-set to a k-set using PIE: the number of onto functions is ∑j=0k (−1)j C(k, j) (k−j)n.
    2. Each partition into k unordered subsets corresponds to k! surjections (one for each labelling of the subsets). So S(n, k) = surjections / k!. 
    // Compute a single S(n, k) using PIE formula — O(k log n)
const int MOD = 1e9 + 7;

long long power(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long stirling2_pie(int n, int k) {
    // Precompute factorials and inverse factorials
    vector<long long> fac(k + 1), inv_fac(k + 1);
    fac[0] = 1;
    for (int i = 1; i <= k; i++) fac[i] = fac[i-1] * i % MOD;
    inv_fac[k] = power(fac[k], MOD - 2, MOD);
    for (int i = k - 1; i >= 0; i--) inv_fac[i] = inv_fac[i+1] * (i+1) % MOD;

    long long ans = 0;
    for (int j = 0; j <= k; j++) {
        long long term = inv_fac[j] % MOD * inv_fac[k - j] % MOD * power(k - j, n, MOD) % MOD;
        if (j & 1) ans = (ans - term + MOD) % MOD;
        else ans = (ans + term) % MOD;
    }
    return ans;
}
    Advantage of PIE formula: It computes a single value S(n, k) in O(k log n) time without building the entire triangle. This is essential when n is very large (e.g., n = 109) but k is small.

    Surjections and Onto Functions

    The number of surjective functions from an n-set to a k-set is k! · S(n, k). This is a very common CP pattern:

    • Distributing n distinct balls into k distinct boxes, each box non-empty: k! · S(n, k).
    • Distributing n distinct balls into k identical boxes, each non-empty: S(n, k).

    Computing Stirling Numbers

    Summary of computational approaches:

    DP (full triangle)
    Time: O(n · k)
    Space: O(n · k) or O(k) with rolling array
    Best when you need many values
    PIE formula (single value)
    Time: O(k log n)
    Space: O(k)
    Best for large n, small k

    Rolling Array Optimization

    // Compute row n of Stirling second kind using O(n) space
vector<long long> stirling2_row(int n) {
    vector<long long> dp(n + 1, 0);
    dp[0] = (n == 0) ? 1 : 0;
    for (int i = 1; i <= n; i++) {
        // Traverse right to left to avoid overwriting
        for (int j = i; j >= 1; j--) {
            dp[j] = (j * dp[j] % MOD + dp[j-1]) % MOD;
        }
        dp[0] = 0;
    }
    return dp;
}

    NTT-Based O(n log n) Computation

    For advanced problems, the entire row S(n, 0), S(n, 1), …, S(n, n) can be computed in O(n log n) using the convolution form of the PIE formula:

    S(n, k) = ∑j [(−1)j/j!] · [(k−j)n/(k−j)!]

    This is a convolution of two sequences, computable via NTT (Number Theoretic Transform).

    When to use NTT: Only when the problem explicitly requires the entire row for large n (e.g., n up to 2 × 105). For most contest problems, the DP or PIE approach suffices.

    Practice Problems

    1. CF 961G — Partitions — Requires computing sums involving Stirling numbers of the second kind.
    2. CF 1342E — Placing Rooks — Place n rooks on an n × n board so that every cell is attacked, with exactly k pairs of rooks attacking each other. (Reduces to Stirling numbers.)
    3. Surjection Counting — Count the number of onto functions from {1, …, n} to {1, …, k}. (Direct application: k! · S(n, k).)
    4. CF 1872F — Balls and Boxes — Distribute n balls into k boxes with constraints. (Stirling + PIE.)
    5. Polynomial Conversion — Convert a polynomial from the xn basis to the falling factorial basis.
    6. AtCoder ABC 282 Ex — Stirling Number of the First Kind (row) — Compute |s(n, k)| for all k, given n up to 2 × 105. (Requires NTT or divide-and-conquer.)
    Summary: S(n, k) counts set partitions (second kind), |s(n, k)| counts permutations by cycle count (first kind). Both satisfy similar-looking recurrences but with different coefficients (k vs n−1). The PIE formula gives S(n, k) in O(k log n). Together, they form the basis-conversion machinery between ordinary powers and falling/rising factorials — a powerful technique in advanced counting problems.
