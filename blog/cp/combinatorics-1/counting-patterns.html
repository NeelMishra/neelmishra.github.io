    Counting Patterns: Problems from CF, ICPC & Beyond

    September 2026 · 25 min read

    Recognizing Counting Problems

    After 21 posts of theory, the final challenge is recognition: given a contest problem, which technique applies? The answer usually falls into one of five patterns. Before diving in, here is a quick mental checklist:

    1. Can I write a direct closed-form? (factorials, binomial coefficients, Catalan numbers)
    2. Is there overcounting I need to subtract? (inclusion-exclusion)
    3. Is the state space exponential but small? (bitmask DP / subset DP)
    4. Can I model it as a polynomial product? (generating functions / NTT)
    5. Can I biject to a known sequence? (Catalan, Stirling, partition numbers)
    Golden rule: Always start by computing small cases by hand. If the sequence matches OEIS, you often immediately know the formula or recurrence.

    Pattern 1: Direct Formula

    The simplest (and fastest!) pattern. The problem reduces to a product of binomial coefficients, factorials, or other classical quantities.

    Signals

    • "How many ways to choose…" → C(n, k)
    • "How many arrangements…" → P(n, k) or n!
    • "How many binary strings of length n with exactly k ones…" → C(n, k)
    • The answer involves Catalan numbers, derangements, or multinomials.

    Problem: Colorful Bracelet Segments

    Given n beads arranged in a line (not a cycle), paint exactly k of them red. No two red beads are adjacent. Count the arrangements modulo 109+7.

    Solution: Place k red beads among n positions such that no two are adjacent. First place n − k blue beads in a line, creating n − k + 1 gaps (including the two ends). Choose k of these gaps: the answer is C(n − k + 1, k).

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;
const int MAXN = 2000005;
long long fac[MAXN], ifac[MAXN];

long long power(long long a, long long b) {
    long long r = 1; a %= MOD;
    while (b > 0) { if (b & 1) r = r * a % MOD; a = a * a % MOD; b >>= 1; }
    return r;
}

void precompute() {
    fac[0] = 1;
    for (int i = 1; i < MAXN; i++) fac[i] = fac[i - 1] * i % MOD;
    ifac[MAXN - 1] = power(fac[MAXN - 1], MOD - 2);
    for (int i = MAXN - 2; i >= 0; i--) ifac[i] = ifac[i + 1] * (i + 1) % MOD;
}

long long C(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fac[n] % MOD * ifac[r] % MOD * ifac[n - r] % MOD;
}

int main() {
    precompute();
    int n, k;
    cin >> n >> k;
    cout << C(n - k + 1, k) << "\n";
    return 0;
}
    Tip: Always precompute factorials and inverse factorials up to the maximum needed value. This gives O(1) per binomial coefficient query and is the single most important utility in combinatorics contests.

    Pattern 2: Inclusion-Exclusion

    When direct counting overcounts or when you need to subtract "bad" configurations, inclusion-exclusion (PIE) is the tool.

    Signals

    • "At least one of…" or "none of…" conditions
    • Derangement-style "no element in its original position"
    • Surjection counting (every box non-empty)
    • "Not divisible by any prime in a set" (Euler's totient generalizations)

    Problem: Grid Paths Avoiding Obstacles

    Count lattice paths from (0,0) to (n,m) that avoid a set of k forbidden cells. Moves are right (+1,0) or up (0,+1). k ≤ 20.

    Solution: Sort the forbidden cells in topological order (by x+y). Let f(i) = number of paths from (0,0) to obstacle i that don't pass through any earlier obstacle. Use PIE / DP:

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;
const int MAXN = 2000005;
long long fac[MAXN], ifac[MAXN];

long long power(long long a, long long b) {
    long long r = 1; a %= MOD;
    while (b > 0) { if (b & 1) r = r * a % MOD; a = a * a % MOD; b >>= 1; }
    return r;
}

void precompute() {
    fac[0] = 1;
    for (int i = 1; i < MAXN; i++) fac[i] = fac[i-1] * i % MOD;
    ifac[MAXN-1] = power(fac[MAXN-1], MOD-2);
    for (int i = MAXN-2; i >= 0; i--) ifac[i] = ifac[i+1] * (i+1) % MOD;
}

long long C(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fac[n] % MOD * ifac[r] % MOD * ifac[n-r] % MOD;
}

long long paths(int dx, int dy) {
    if (dx < 0 || dy < 0) return 0;
    return C(dx + dy, dx);
}

int main() {
    precompute();
    int n, m, k;
    cin >> n >> m >> k;

    vector<pair<int,int>> obs(k);
    for (auto &[x, y] : obs) cin >> x >> y;
    // Add the destination as a "virtual obstacle"
    obs.push_back({n, m});
    k++;

    // Sort by x+y, then by x
    sort(obs.begin(), obs.end(), [](auto &a, auto &b){
        return (a.first + a.second) < (b.first + b.second);
    });

    vector<long long> dp(k);
    for (int i = 0; i < k; i++) {
        dp[i] = paths(obs[i].first, obs[i].second);
        for (int j = 0; j < i; j++) {
            int dx = obs[i].first - obs[j].first;
            int dy = obs[i].second - obs[j].second;
            if (dx >= 0 && dy >= 0)
                dp[i] = (dp[i] - dp[j] % MOD * paths(dx, dy) % MOD + MOD) % MOD;
        }
    }
    cout << dp[k - 1] << "\n";
    return 0;
}
    Pitfall: When subtracting in modular arithmetic, always add MOD before taking the remainder to avoid negative values. The pattern (a - b % MOD + MOD) % MOD is essential.

    Pattern 3: DP on Subsets

    When the number of "categories" or "slots" is small (≤ 20–23), we can iterate over all 2k subsets using bitmask DP.

    Signals

    • n ≤ 20 in the constraint
    • "Assign n items to n slots with constraints" (generalized permanent)
    • Hamiltonian path / TSP counting
    • Profile DP on grids (broken profile)

    Problem: Counting Perfect Matchings

    Given a bipartite graph with n vertices on each side (n ≤ 20), count the number of perfect matchings modulo 109+7.

    Solution: This is the permanent of the biadjacency matrix. Use bitmask DP where the state is the set of matched right-side vertices.

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

int main() {
    int n;
    cin >> n;
    // adj[i] = bitmask of right-side vertices that left vertex i can match
    vector<int> adj(n);
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            int e; cin >> e;
            if (e) adj[i] |= (1 << j);
        }

    // dp[mask] = number of ways to match left vertices 0..popcount(mask)-1
    //            to the set of right vertices indicated by mask
    vector<long long> dp(1 << n, 0);
    dp[0] = 1;

    for (int mask = 0; mask < (1 << n); mask++) {
        if (dp[mask] == 0) continue;
        int i = __builtin_popcount(mask); // next left vertex to match
        if (i >= n) continue;
        int available = adj[i] & ~mask; // right vertices not yet used
        for (int sub = available; sub; sub = (sub - 1) & available) {
            int j = __builtin_ctz(sub); // pick lowest bit
            dp[mask | (1 << j)] = (dp[mask | (1 << j)] + dp[mask]) % MOD;
            // We only want single bits, so:
            if (__builtin_popcount(sub) == 1) continue;
            // Actually, iterate over single bits only:
        }
        // Cleaner: iterate over individual bits of available
        for (int tmp = available; tmp; tmp &= tmp - 1) {
            int j = __builtin_ctz(tmp);
            dp[mask | (1 << j)] = (dp[mask | (1 << j)] + dp[mask]) % MOD;
        }
    }
    cout << dp[(1 << n) - 1] << "\n";
    return 0;
}
    Complexity: O(n · 2n) time and O(2n) memory. For n = 20, 220 ≈ 106, well within limits. For n up to 23, you may need to optimize constants.

    Pattern 4: Generating Functions

    When the answer involves combining independent choices multiplicatively, generating functions and polynomial multiplication (via NTT) are the right tool.

    Signals

    • "Choose items from multiple independent groups, count total value/weight combinations"
    • Partition function variants
    • Convolutions of sequences
    • The constraint involves a sum: "a1 + a2 + … + ak = n with conditions on each ai"

    Problem: Coin Change Counting

    You have coins of denominations d1, d2, …, dk (unlimited supply). How many ways to make change for amount n?

    Solution: The generating function for each coin type d is 1/(1 − xd). The answer is [xn] in the product Π 1/(1 − xdi). We compute this with a simple DP:

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

int main() {
    int n, k;
    cin >> n >> k;
    vector<int> d(k);
    for (auto &x : d) cin >> x;

    vector<long long> dp(n + 1, 0);
    dp[0] = 1;
    for (int coin : d)
        for (int j = coin; j <= n; j++)
            dp[j] = (dp[j] + dp[j - coin]) % MOD;

    cout << dp[n] << "\n";
    return 0;
}

    For more complex GF problems where you need to multiply arbitrary polynomials, use NTT (Number Theoretic Transform) for O(n log n) convolution as covered in Post 18.

    Pattern 5: Bijection to Known Sequence

    Sometimes the hardest part is seeing that your problem is equivalent to a well-known one. Bijections are the key.

    Signals

    • Dyck paths, balanced parentheses, binary trees → Catalan numbers
    • Set partitions → Bell / Stirling numbers
    • Non-crossing structures → often Catalan
    • Lattice paths that stay above a line → ballot problem / reflection principle

    Problem: Valid Bracket Sequences with Colors

    Count the number of valid bracket sequences of length 2n where each pair of brackets is assigned one of c colors. Answer modulo 109+7.

    Solution: The number of valid bracket sequences of length 2n is the n-th Catalan number Cn = C(2n, n) / (n + 1). Each of the n pairs independently gets a color, so the answer is Cn · cn.

    long long solve(int n, int c) {
    // Catalan(n) * c^n
    long long catalan = C(2 * n, n) % MOD * power(n + 1, MOD - 2) % MOD;
    return catalan % MOD * power(c, n) % MOD;
}
    Bijection toolkit: Keep a mental library of ~10 bijections: Dyck paths ↔ binary trees ↔ triangulations ↔ non-crossing partitions (all Catalan), lattice paths ↔ ballot sequences (reflection), permutations ↔ inversion tables, etc.

    Common Pitfalls

    After solving hundreds of counting problems, here are the mistakes I see most frequently:

    1. Modular Arithmetic Bugs

    • Forgetting to mod intermediate results. Even 109+7 squared overflows long long? No — (109+7)² ≈ 1018, which fits. But triple products like a*b*c can overflow. Always mod after each multiplication.
    • Negative modular values. In C++, -3 % MOD is -3, not MOD - 3. Always add MOD before the final mod: (a - b % MOD + MOD) % MOD.
    • Using modular inverse when MOD is not prime. Fermat's little theorem requires MOD to be prime. For composite moduli, use extended GCD.

    2. Off-by-One Errors

    • Stars and bars: is the lower bound 0 or 1 per bin? C(n+k-1, k-1) vs. C(n-1, k-1).
    • Stirling numbers: S(n, 0) = [n = 0], not 1. Make sure your base cases are correct.
    • Boundary: does "length n" mean n characters or n pairs? Read the problem statement twice.

    3. Overcounting / Undercounting

    • Symmetry overcounting: If the problem says "unordered groups," divide by the number of permutations of identical groups (or use Stirling/partition numbers instead of PIE directly).
    • Missed constraints: "Non-empty" is easy to miss. Always check if boxes/groups must be non-empty (surjective) or can be empty (any f).
    Debugging strategy: When your answer is wrong, compute the first 5–10 values by brute force and compare with your formula. This catches almost every combinatorial bug.

    Problem Walkthroughs

    Problem A: Painting Fences (CF-style, 1700)

    You have n fence posts in a row and c colors. Paint each post so that no more than 2 consecutive posts have the same color. How many valid colorings?

    Analysis: Let f(i) = number of valid colorings of posts 1..i. At post i, either it differs from post i−1 (contributing (c−1) · f(i−1) ways), or it matches post i−1 but differs from post i−2 (contributing (c−1) · f(i−2) ways). So:

    f(n) = (c−1) · f(n−1) + (c−1) · f(n−2)

    Base cases: f(1) = c, f(2) = c². This is a linear recurrence — for large n, use matrix exponentiation!

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

int main() {
    long long n, c;
    cin >> n >> c;
    if (n == 1) { cout << c % MOD; return 0; }
    if (n == 2) { cout << c % MOD * c % MOD; return 0; }

    long long a = c % MOD, b = c % MOD * c % MOD;
    for (long long i = 3; i <= n; i++) {
        long long nxt = (c - 1) % MOD * ((a + b) % MOD) % MOD;
        a = b;
        b = nxt;
    }
    cout << b << "\n";
    return 0;
}

    Problem B: Derangement Subsequences (CF-style, 2100)

    Given a permutation of [1..n], count the number of subsequences of length k that are derangements of their index set. Answer modulo 109+7.

    Analysis: Choose a k-element subset S ⊂ [n]. The subsequence formed by positions in S is a derangement if psi ≠ si for all i. This is a PIE problem over the "bad" positions where pj = j.

    Let B = {j : pj = j} (fixed points), |B| = b. A k-subset avoids all fixed points iff it doesn't contain any element of B. The number of derangement subsequences is C(n − b, k) times the number of derangements? No — we need subsequences that form derangements of their own index set, which is more subtle.

    Key insight: After choosing positions S = {s1 < … < sk}, we need psi ≠ si for all i. This means we avoid positions where the element equals the position: exactly the fixed points. So the answer is simply C(n − b, k) where b is the number of fixed points of the permutation.

    int main() {
    precompute();
    int n, k;
    cin >> n >> k;
    int b = 0;
    for (int i = 1; i <= n; i++) {
        int p; cin >> p;
        if (p == i) b++;
    }
    cout << C(n - b, k) << "\n";
    return 0;
}

    Problem C: Colored Grid Tilings (ICPC-style, Hard)

    Tile a 2 × n grid with 1 × 2 dominoes such that each domino is one of c colors and no two adjacent dominoes (sharing an edge) have the same color. Count the configurations modulo 109+7.

    Analysis: First, count the tilings (Fibonacci: T(n) = T(n−1) + T(n−2)). Each tiling defines an adjacency structure among dominoes. For vertical dominoes, the color must differ from left and right neighbors. For horizontal pairs, they must differ from each other and from vertical neighbors above/below.

    The key observation: the domino adjacency graph is a path (linear chain). The number of proper c-colorings of a path of m vertices is c · (c−1)m−1. But the number of dominoes m depends on the tiling.

    A tiling of a 2×n grid uses: all vertical → n dominoes; each horizontal pair replaces 2 verticals → one fewer domino. If there are h horizontal pairs, the number of dominoes is n − h + h = n — wait, each horizontal pair uses 2 dominoes covering 2 columns. So h horizontal pairs and (n − 2h) vertical pairs give h·2 + (n − 2h) = n dominoes total. The number of dominoes is always n!

    But the adjacency structure varies. We need to sum over all tilings, each weighted by the chromatic polynomial of its adjacency graph. This becomes a transfer-matrix problem solvable with matrix exponentiation.

    Problem D: Subsequence Pairs (CF 1462F-style, 2200)

    Given an array of n integers, count the number of pairs (i, j) with i < j such that ai + aj is divisible by k. n ≤ 2 × 105, k ≤ 109.

    Analysis: Group elements by their remainder mod k. For remainder r, it can pair with remainder k − r. Count: if cnt[r] = number of elements with remainder r, then:

    • Pairs (0, 0): C(cnt[0], 2)
    • Pairs (r, k−r) for r ≠ 0 and r ≠ k−r: cnt[r] × cnt[k−r]
    • If k is even, pairs (k/2, k/2): C(cnt[k/2], 2)
    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    long long k;
    cin >> n >> k;

    map<long long, long long> cnt;
    for (int i = 0; i < n; i++) {
        long long a; cin >> a;
        cnt[((a % k) + k) % k]++;
    }

    long long ans = 0;
    // Pairs with remainder 0
    ans += cnt[0] * (cnt[0] - 1) / 2;

    set<long long> visited;
    for (auto &[r, c] : cnt) {
        if (r == 0 || visited.count(r)) continue;
        long long comp = (k - r) % k;
        if (comp == r) {
            ans += c * (c - 1) / 2;
        } else {
            ans += c * cnt[comp];
        }
        visited.insert(r);
        visited.insert(comp);
    }
    cout << ans << "\n";
    return 0;
}
