    Generating Functions: OGF Basics

    August 2026 · 25 min read

    What Are Generating Functions?

    A generating function encodes an infinite sequence (a₀, a₁, a₂, …) as a single formal power series. Instead of reasoning about individual terms we manipulate the entire series algebraically — addition, multiplication, composition — and read off the answer from the coefficient we need.

    Formally, given a sequence {aₙ} we write: 

    A(x) = a₀ + a₁·x + a₂·x² + a₃·x³ + …
     = Σ (n ≥ 0) aₙ · xⁿ

    The variable x is a formal indeterminate: we never substitute a real number into it (convergence is irrelevant). Two power series are equal exactly when every coefficient matches.

    Why bother? Many counting problems reduce to multiplying or composing power series. The algebra is mechanical, and once you learn the dictionary between "combinatorial operation" and "algebraic operation" you can solve problems on autopilot.

    There are several flavours — ordinary (OGF), exponential (EGF), Dirichlet, etc. This post covers ordinary generating functions, the workhorse for problems involving unlabelled objects (coins, tiles, subsets).

    Ordinary Generating Functions (OGF)

    An ordinary generating function (OGF) for {aₙ} is simply: 

    A(x) = Σ (n ≥ 0) aₙ · xⁿ

    Some classic OGFs you should memorise:

    SequenceaₙOGF
    All ones11 / (1 − x)
    Powers of ccⁿ1 / (1 − cx)
    Binomial C(n+k−1,k−1)C(n+k−1,k−1)1 / (1 − x)ᵏ
    FibonacciFₙx / (1 − x − x²)
    CatalanCₙ(1 − √(1−4x)) / (2x)
    Tip: Memorise the geometric-series OGF 1/(1 − x) = 1 + x + x² + …. Almost every other OGF is derived from it by substitution, differentiation, or multiplication.

    Operations on OGFs

    The power of generating functions comes from the dictionary between combinatorial operations on sequences and algebraic operations on power series.

    Addition — disjoint union

    If A(x) counts one type of object and B(x) counts another, then A(x) + B(x) counts either/or. Coefficient-wise: cₙ = aₙ + bₙ.

    Multiplication — convolution

    C(x) = A(x) · B(x) gives cₙ = Σ (k=0..n) aₖ · bₙ₋ₖ. Combinatorially this counts ordered pairs where the total "size" is n — exactly the idea behind distributing items into two groups.

    Differentiation — index shift

    A′(x) = Σ (n ≥ 0) (n+1)·aₙ₊₁ · xⁿ. Differentiating an OGF shifts the sequence left and multiplies by the old index. Useful for extracting n · aₙ or building recurrences.

    Multiplication by x — right shift

    x · A(x) shifts every coefficient one position to the right (prepends a 0). This is handy for initial-condition tweaks.

    Combinatorial
    Choose one object from set A and one from set B such that sizes sum to n.
    Algebraic
    Multiply A(x) · B(x) and read off [xⁿ].

    Coin Change via OGF

    Problem. How many ways can you make change for n cents using coins of denominations d₁, d₂, …, dₖ (unlimited supply)?

    Each denomination dᵢ contributes a factor 1 / (1 − x^{dᵢ}) because you can use 0, 1, 2, … coins of that value: 

    1 + x^d + x^{2d} + x^{3d} + … = 1 / (1 − x^d)

    The full OGF is the product over all denominations: 

    G(x) = ∏ (i=1..k) 1 / (1 − x^{dᵢ})

    The answer for n cents is [xⁿ] G(x).

    In competitive programming we truncate the series at degree n and multiply polynomials iteratively: 

    #include <bits/stdc++.h>
using namespace std;

// Coin-change via polynomial multiplication (OGF approach).
// coins[] = denominations, n = target sum.
// Returns the number of ways modulo MOD.
const int MOD = 998244353;

long long coinChangeOGF(vector<int>& coins, int n) {
    // poly[j] = coefficient of x^j in the running product
    vector<long long> poly(n + 1, 0);
    poly[0] = 1; // empty product = 1

    for (int d : coins) {
        // Multiply current poly by 1/(1 - x^d) truncated to degree n.
        // Equivalent to: for j = d..n, poly[j] += poly[j - d]
        for (int j = d; j <= n; j++) {
            poly[j] = (poly[j] + poly[j - d]) % MOD;
        }
    }
    return poly[n];
}

int main() {
    int k, n;
    cin >> k >> n;
    vector<int> coins(k);
    for (int& c : coins) cin >> c;
    cout << coinChangeOGF(coins, n) << "\n";
}
    Complexity: The naïve approach above is O(k · n). For large n with few denominations this is already fast. When k is large and denominations are arbitrary you might need NTT-based polynomial multiplication — covered later in this post.

    Fibonacci via OGF

    The Fibonacci sequence F₀ = 0, F₁ = 1, Fₙ = Fₙ₋₁ + Fₙ₋₂ has OGF: 

    F(x) = Σ Fₙ xⁿ

    From the recurrence we derive:

    F(x) − x·F(x) − x²·F(x) = F₀ + (F₁ − F₀)·x = x
⟹  F(x) · (1 − x − x²) = x
⟹  F(x) = x / (1 − x − x²)

    This rational form is the starting point for extracting a closed-form for Fₙ. Factor the denominator: 

    1 − x − x² = −(x − φ⁻¹)(x − ψ⁻¹)
where φ = (1+√5)/2,  ψ = (1−√5)/2

    Apply partial fractions (see the next post for the full technique) to obtain Binet's formula: 

    Fₙ = (φⁿ − ψⁿ) / √5
    Contest shortcut: You rarely need Binet's formula in a contest (it has floating-point issues). The real win is recognising that any linear recurrence yields a rational OGF, and from there you can use matrix exponentiation to compute the n-th term in O(k³ log n).

    Closed-Form Extraction

    Given a rational OGF P(x) / Q(x) where deg P < deg Q, we can always find a closed form for the coefficients:

    1. Factor Q(x) into linear factors (1 − rᵢ x).
    2. Partial-fraction decompose: P(x)/Q(x) = Σ Aᵢ / (1 − rᵢ x).
    3. Expand each term as a geometric series: Aᵢ / (1 − rᵢ x) = Σ Aᵢ rᵢⁿ xⁿ.
    4. Collect: aₙ = Σ Aᵢ rᵢⁿ.

    For repeated roots the expansion involves polynomial factors in n — e.g., 1/(1−x)² = Σ (n+1) xⁿ.

    Example. Suppose A(x) = 1 / ((1−x)(1−2x)). Partial fractions give A(x) = −1/(1−x) + 2/(1−2x), so aₙ = 2ⁿ⁺¹ − 1.

    Computing Coefficients with NTT

    When the OGF is a product of many polynomials (not just geometric series), we multiply them with the Number Theoretic Transform (NTT) — the modular analogue of FFT.

    Below is a compact NTT-based polynomial multiplication that works modulo 998244353 (a common CP prime with primitive root 3): 

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353, g = 3;

long long power(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}

void ntt(vector<long long>& a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        long long w = inv ? power(g, MOD - 1 - (MOD - 1) / len, MOD)
                         : power(g, (MOD - 1) / len, MOD);
        for (int i = 0; i < n; i += len) {
            long long wn = 1;
            for (int j = 0; j < len / 2; j++) {
                long long u = a[i + j];
                long long v = a[i + j + len / 2] * wn % MOD;
                a[i + j] = (u + v) % MOD;
                a[i + j + len / 2] = (u - v + MOD) % MOD;
                wn = wn * w % MOD;
            }
        }
    }
    if (inv) {
        long long n_inv = power(n, MOD - 2, MOD);
        for (auto& x : a) x = x * n_inv % MOD;
    }
}

vector<long long> multiply(vector<long long> a, vector<long long> b) {
    int result_size = a.size() + b.size() - 1;
    int n = 1;
    while (n < result_size) n <<= 1;
    a.resize(n); b.resize(n);
    ntt(a, false); ntt(b, false);
    for (int i = 0; i < n; i++) a[i] = a[i] * b[i] % MOD;
    ntt(a, true);
    a.resize(result_size);
    return a;
}

int main() {
    // Example: multiply (1 + 2x + 3x²) by (4 + 5x)
    vector<long long> A = {1, 2, 3};
    vector<long long> B = {4, 5};
    auto C = multiply(A, B);
    // C = {4, 13, 22, 15}  →  4 + 13x + 22x² + 15x³
    for (auto v : C) cout << v << " ";
    cout << "\n";
}
    Complexity: Each NTT call is O(n log n). Multiplying two degree-d polynomials costs O(d log d). For multiplying k polynomials whose total degree is D, use a divide-and-conquer merge for O(D log² D) total.

    A common competitive-programming pattern is to represent a counting problem as a product of small polynomials and then batch-multiply them with NTT. For example, the number of ways to select one item from each of k groups (with group sizes encoded as polynomials) is the product of all group polynomials evaluated at the right coefficient.

    Divide-and-conquer polynomial multiplication

    When you need to multiply many polynomials, the order matters for performance. A balanced merge-tree approach keeps intermediate sizes small: 

    // Multiply a list of polynomials using D&C + NTT.
vector<long long> multiplyAll(vector<vector<long long>>& polys, int lo, int hi) {
    if (lo == hi) return polys[lo];
    int mid = (lo + hi) / 2;
    auto L = multiplyAll(polys, lo, mid);
    auto R = multiplyAll(polys, mid + 1, hi);
    return multiply(L, R);
}

// Usage: multiplyAll(polys, 0, polys.size() - 1)

    Practice Problems

    Try these to solidify your understanding of OGF basics:

    1. CSES — Dice Combinations
      Count the number of ways to reach sum n by throwing a die. Model as the OGF (x + x² + … + x⁶)ⁿ and extract [xⁿ] — or just use DP.
    2. CSES — Coin Combinations I
      Classic ordered coin change. The OGF is a product of geometric series.
    3. CSES — Coin Combinations II
      Unordered variant — same OGF product, different DP loop order. Make sure you understand why the OGF is identical.
    4. Codeforces 1548C — The Three Bags
      Requires recognising a convolution structure. Build the polynomial and multiply with NTT.
    5. Codeforces 632E — Thief in a Shop
      Given item values, find all achievable sums using exactly k items. This is [polynomial]^k computed efficiently.
    6. Atcoder ABC315 — Typical Stairs
      Fibonacci-style recurrence; derive the OGF and compute with matrix exponentiation for large n.
