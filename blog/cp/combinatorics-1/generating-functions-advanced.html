    Advanced Generating Functions: Composition & Partial Fractions

    August 2026 · 25 min read

    Composition of Generating Functions

    If A(x) = Σ aₙ xⁿ and B(x) = Σ bₙ xⁿ with b₀ = 0, the composition A(B(x)) substitutes B(x) for every occurrence of x in A: 

    A(B(x)) = Σ (n ≥ 0) aₙ · [B(x)]ⁿ

    Combinatorially, composition models substitution of structures. If A(x) counts "outer" structures of size n and B(x) counts "inner" structures, then A(B(x)) counts structures where each atom of the outer structure is replaced by an inner structure.

    Example — Set partitions. A partition of a set is a collection of non-empty subsets. The OGF for "non-empty subset" is B(x) = x + x² + x³ + … = x/(1−x). The OGF for "collection of things" (any number, unordered) uses the exponential formula (more in the EGF post), but for OGFs over integer compositions the idea is the same: compose the "collection" GF with the "part" GF.

    Key rule: Composition requires b₀ = 0. If the inner GF has a constant term, [B(x)]ⁿ has contributions to every coefficient and the series doesn't converge formally.

    Computing composition in CP

    Naïvely computing A(B(x)) truncated to degree N costs O(N³). With Brent–Kung's algorithm or careful baby-step/giant-step splitting, this drops to O(N^{1.5} log N) using NTT. In practice, for CP constraints (N ≤ 10⁵), a direct power-table approach often suffices: 

    #include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

// Forward-declare NTT multiply (same as previous post).
vector<long long> multiply(vector<long long> a, vector<long long> b);

// Compute A(B(x)) mod x^N.
// a = coefficients of A, b = coefficients of B (b[0] must be 0).
vector<long long> compose(vector<long long>& a, vector<long long>& b, int N) {
    vector<long long> result(N, 0);
    vector<long long> bPow(N, 0); // B(x)^k, starts at B^0 = 1
    bPow[0] = 1;

    for (int k = 0; k < (int)a.size() && k < N; k++) {
        // result += a[k] * bPow
        for (int j = 0; j < N; j++)
            result[j] = (result[j] + a[k] % MOD * bPow[j]) % MOD;
        // bPow *= b, truncated to degree N
        auto tmp = multiply(bPow, b);
        tmp.resize(N);
        bPow = tmp;
    }
    return result;
}

    Partial Fraction Decomposition

    Partial fractions split a rational GF into a sum of simpler terms whose coefficients we already know. Given: 

    P(x) / Q(x)   where  deg P < deg Q

    and Q(x) = (1 − r₁x)(1 − r₂x)…(1 − rₘx) with distinct roots, we write: 

    P(x)/Q(x) = A₁/(1 − r₁x) + A₂/(1 − r₂x) + … + Aₘ/(1 − rₘx)

    Each constant Aᵢ can be found by the cover-up method: multiply both sides by (1 − rᵢx) and set x = 1/rᵢ. 

    Aᵢ = P(1/rᵢ) / ∏ (j ≠ i) (1 − rⱼ/rᵢ)

    Once decomposed, the n-th coefficient is simply: 

    [xⁿ] P(x)/Q(x) = A₁·r₁ⁿ + A₂·r₂ⁿ + … + Aₘ·rₘⁿ
    Repeated roots: If rᵢ appears with multiplicity s, the corresponding terms become Σ (j=0..s−1) Aᵢⱼ / (1 − rᵢ x)^{j+1}. Each term expands as Aᵢⱼ · C(n+j, j) · rᵢⁿ.

    Modular partial fractions in C++

    In CP, roots are usually elements of ℤ/pℤ. We factor Q(x) modulo the prime and compute each Aᵢ with modular inverses: 

    #include <bits/stdc++.h>
using namespace std;
const long long MOD = 998244353;

long long power(long long a, long long b, long long mod) {
    long long r = 1; a %= mod;
    while (b > 0) { if (b & 1) r = r * a % mod; a = a * a % mod; b >>= 1; }
    return r;
}
long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Given P(x) and roots r[] of Q(x) (all distinct), compute partial fraction
// coefficients A[].  P is stored as a vector of coefficients.
// Returns A[] such that P(x)/Q(x) = Σ A[i]/(1 - r[i]*x).
vector<long long> partialFractions(vector<long long>& P, vector<long long>& r) {
    int m = r.size();
    vector<long long> A(m);
    for (int i = 0; i < m; i++) {
        // Evaluate P at x = inv(r[i])
        long long xi = inv(r[i]);
        long long pVal = 0, xPow = 1;
        for (auto c : P) {
            pVal = (pVal + c % MOD * xPow) % MOD;
            xPow = xPow * xi % MOD;
        }
        // Evaluate ∏(j≠i)(1 - r[j]*xi)
        long long denom = 1;
        for (int j = 0; j < m; j++) {
            if (j == i) continue;
            denom = denom % MOD * ((1 - r[j] % MOD * xi % MOD + MOD) % MOD) % MOD;
        }
        A[i] = pVal % MOD * inv(denom) % MOD;
    }
    return A;
}

// [x^n] P(x)/Q(x) = Σ A[i] * r[i]^n
long long extractCoeff(vector<long long>& A, vector<long long>& r, long long n) {
    long long ans = 0;
    for (int i = 0; i < (int)A.size(); i++)
        ans = (ans + A[i] % MOD * power(r[i], n, MOD)) % MOD;
    return ans;
}

    Coefficient Extraction Techniques

    The notation [xⁿ] F(x) means "the coefficient of xⁿ in F(x)". Several identities simplify extraction:

    Useful identities

    [xⁿ] 1/(1−x)ᵏ   = C(n+k−1, k−1)           (stars-and-bars)
[xⁿ] xᵐ·F(x)    = [xⁿ⁻ᵐ] F(x)             (shift)
[xⁿ] F(cx)       = cⁿ · [xⁿ] F(x)          (scaling)
[xⁿ] F′(x)       = (n+1) · [xⁿ⁺¹] F(x)     (differentiation)

    Residue method

    From complex analysis, [xⁿ] F(x) = (1/2πi) ∮ F(x)/x^{n+1} dx. While rarely used directly in CP, this viewpoint motivates many algebraic manipulations — especially Lagrange inversion.

    CP tip: When a problem asks for the n-th coefficient of a known rational GF, first try partial fractions. If the denominator doesn't factor nicely, use Berlekamp–Massey to find the minimal recurrence, then Kitamasa or matrix exponentiation.

    Lagrange Inversion

    Lagrange Inversion Theorem lets you extract coefficients of the compositional inverse of a power series. If F(x) satisfies F(x) = x · φ(F(x)) for some known function φ, then: 

    [xⁿ] F(x) = (1/n) · [xⁿ⁻¹] φ(x)ⁿ

    More generally, for any function G: 

    [xⁿ] G(F(x)) = (1/n) · [xⁿ⁻¹] G′(x) · φ(x)ⁿ

    Classic application: Catalan numbers

    The Catalan OGF satisfies C(x) = 1 + x·C(x)², or equivalently C(x) = x · φ(C(x)) doesn't quite fit. Rewrite: let F(x) = x·C(x), then F(x) = x · (1 + F(x))², giving F(x) = x · φ(F(x)) with φ(u) = (1+u)². 

    [xⁿ] F(x) = (1/n) · [xⁿ⁻¹] (1+x)^{2n}
           = (1/n) · C(2n, n−1)
           = (1/(n+1)) · C(2n, n)    // the Catalan number!
    When to use Lagrange inversion: Any time a GF is defined implicitly by a functional equation F = x · φ(F) — common in tree-counting, lattice-path, and map-enumeration problems.

    Snake Oil Method

    The Snake Oil method (due to Wilf) is a systematic technique for evaluating combinatorial sums of the form: 

    Σ (k) f(n, k) · g(k)

    The recipe:

    1. Identify the "free" variable (usually the one being summed over, say k).
    2. Introduce a generating function in the other variable (say n): S(x) = Σₙ [Σₖ f(n,k) g(k)] xⁿ.
    3. Interchange the order of summation.
    4. Recognise the inner sum as a known GF.
    5. Simplify and extract [xⁿ].

    Example — Vandermonde's identity

    Prove that Σ (k=0..r) C(m,k)·C(n,r−k) = C(m+n,r).

    LHS = [xʳ] (Σ C(m,k) xᵏ) · (Σ C(n,j) xʲ)
    = [xʳ] (1+x)ᵐ · (1+x)ⁿ
    = [xʳ] (1+x)^{m+n}
    = C(m+n, r)   ✓

    The "Snake Oil" is the act of recognising the inner sum as a coefficient in a product of known generating functions. With practice, this becomes almost reflexive.

    Tip: When a contest problem gives a complicated double sum with binomial coefficients, try the Snake Oil method before attempting algebraic manipulation. It often reveals the identity in two lines.

    Polynomial Operations in CP

    Many advanced GF techniques boil down to performing polynomial arithmetic modulo xⁿ. Here is a summary of what's achievable with NTT:

    OperationComplexity
    MultiplicationO(n log n)
    Inverse (1/F mod xⁿ)O(n log n)
    DivisionO(n log n)
    Logarithm ln(F) mod xⁿO(n log n)
    Exponential exp(F) mod xⁿO(n log n)
    Power Fᵏ mod xⁿO(n log n)
    Composition F(G(x)) mod xⁿO(n^{1.5} log n) or O(n²)
    Multi-point evaluationO(n log² n)

    Polynomial inverse via Newton's method

    To compute 1/F(x) mod xⁿ, we use Newton iteration. Start with G₀ = 1/f₀ (the inverse of the constant term) and iterate: 

    Gₖ₊₁ = Gₖ · (2 − F · Gₖ)   mod x^{2ᵏ⁺¹}

    Each step doubles the number of correct coefficients.

    #include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

long long power(long long a, long long b, long long mod);
void ntt(vector<long long>& a, bool inv);
vector<long long> multiply(vector<long long> a, vector<long long> b);

// Compute 1/F(x) mod x^n.  F[0] must be nonzero.
vector<long long> polyInverse(vector<long long> F, int n) {
    vector<long long> G = { power(F[0], MOD - 2, MOD) };
    for (int len = 2; len < 2 * n; len <<= 1) {
        // G = G * (2 - F * G) mod x^len
        vector<long long> tmp(F.begin(), F.begin() + min((int)F.size(), len));
        tmp.resize(len);
        auto fg = multiply(tmp, G);
        fg.resize(len);
        for (auto& v : fg) v = (MOD - v) % MOD;
        fg[0] = (fg[0] + 2) % MOD;
        G = multiply(G, fg);
        G.resize(len);
    }
    G.resize(n);
    return G;
}

// Polynomial division: compute P(x) / Q(x) mod x^n
vector<long long> polyDivide(vector<long long>& P, vector<long long>& Q, int n) {
    auto Qinv = polyInverse(Q, n);
    auto res = multiply(P, Qinv);
    res.resize(n);
    return res;
}
    Warning: Newton iteration requires F[0] ≠ 0. If the constant term is zero, factor out xᵏ first, invert the remaining polynomial, then shift back.
