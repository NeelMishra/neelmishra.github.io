    Double Counting & Combinatorial Proofs

    August 2026 · 18 min read

    What Is Double Counting?

    Double counting is a proof technique where we count the same quantity in two different ways, then equate the results. If counting method A gives expression X and counting method B gives expression Y, then X = Y. Despite its simplicity, this technique is incredibly powerful and often produces elegant, insightful proofs of combinatorial identities.

    The idea dates back to Euler and has been a staple of combinatorics ever since. In competitive programming, double counting helps us:

    • Derive formulas from scratch during a contest
    • Verify that two expressions are equivalent
    • Transform a hard counting problem into an easier one
    • Prove identities that are needed to simplify a solution
    The meta-principle: If you can count something two ways and one way is easy to compute while the other gives you the quantity you need, you’ve solved the problem. Always ask: “Can I count this from a different perspective?”

    A combinatorial proof is a proof that establishes an identity by showing both sides count the same thing. This is in contrast to algebraic proofs that manipulate symbols. Combinatorial proofs are often more intuitive and reveal why an identity is true, not just that it is true.

    The Handshaking Lemma

    Perhaps the most famous example of double counting in graph theory:

    ∑v ∈ V deg(v) = 2|E|

    The sum of all vertex degrees equals twice the number of edges.

    Proof by Double Counting

    Count the number of (vertex, edge) incidence pairs — pairs (v, e) where vertex v is an endpoint of edge e.

    • Counting by vertices: Each vertex v contributes deg(v) pairs. Total = ∑ deg(v).
    • Counting by edges: Each edge has exactly 2 endpoints. Total = 2|E|.

    Both count the same set, so ∑ deg(v) = 2|E|. □

    Consequences

    • The number of vertices with odd degree is always even.
    • In a tournament (complete directed graph), ∑ out-deg(v) = C(n, 2).
    • The average degree in a graph is 2|E|/|V|.
    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    vector<int> deg(n + 1, 0);
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        deg[u]++;
        deg[v]++;
    }

    long long sum_deg = 0;
    for (int i = 1; i <= n; i++) sum_deg += deg[i];

    // Verify handshaking lemma
    assert(sum_deg == 2LL * m);

    // Count vertices with odd degree
    int odd_count = 0;
    for (int i = 1; i <= n; i++)
        if (deg[i] & 1) odd_count++;

    // odd_count is always even
    cout << "Odd-degree vertices: " << odd_count << "\n";
    return 0;
}
    CP usage: The handshaking lemma is used in Eulerian path/circuit conditions, network flow arguments, and any problem where you need to relate degrees to edge counts. It’s also a quick sanity check for your graph data.

    Proving Vandermonde’s Identity

    Vandermonde’s identity states:

    C(m + n, r) = ∑k=0r C(m, k) · C(n, r − k)

    Combinatorial Proof

    Consider a group of m men and n women. We want to choose a committee of r people.

    • Left side: Choose r people from the combined group of m + n. Answer: C(m+n, r).
    • Right side: Choose k men (from m) and r−k women (from n), then sum over all valid k. Answer: ∑ C(m,k) · C(n, r−k).

    Both sides count the same set of committees, so they’re equal. □

    Verification in Code

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;
const int MAXN = 1005;
long long fact[MAXN], inv_fact[MAXN];

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

void precompute() {
    fact[0] = 1;
    for (int i = 1; i < MAXN; i++)
        fact[i] = fact[i-1] * i % MOD;
    inv_fact[MAXN-1] = power(fact[MAXN-1], MOD-2, MOD);
    for (int i = MAXN-2; i >= 0; i--)
        inv_fact[i] = inv_fact[i+1] * (i+1) % MOD;
}

long long C(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fact[n] % MOD * inv_fact[r] % MOD * inv_fact[n-r] % MOD;
}

int main() {
    precompute();

    // Verify Vandermonde's identity for specific values
    int m = 10, n = 8, r = 7;

    long long lhs = C(m + n, r);
    long long rhs = 0;
    for (int k = 0; k <= r; k++) {
        rhs = (rhs + C(m, k) * C(n, r - k)) % MOD;
    }

    cout << "LHS: " << lhs << "\n";
    cout << "RHS: " << rhs << "\n";
    assert(lhs == rhs);
    cout << "Vandermonde verified!\n";
    return 0;
}
    When Vandermonde appears in CP: Whenever you see a convolution of binomial coefficients — summing C(a, k) · C(b, r−k) — recognize it as C(a+b, r). This can reduce an O(r) sum to an O(1) computation.

    Proving the Hockey Stick Identity

    The hockey stick identity (also called the Christmas stocking identity) states:

    ∑i=rn C(i, r) = C(n + 1, r + 1)

    Visually in Pascal’s triangle, if you sum entries along a diagonal, the result is the entry one row below and one position to the right — forming a hockey-stick shape.

    Combinatorial Proof

    Count the ways to choose r + 1 elements from {1, 2, …, n + 1}.

    • Left side: Condition on the largest element chosen. If the largest element is i + 1 (where r ≤ i ≤ n), then the remaining r elements must come from {1, …, i}, giving C(i, r) ways. Sum over all possible largest elements: ∑i=rn C(i, r).
    • Right side: Directly choose r + 1 elements from {1, …, n+1}: C(n+1, r+1) ways.

    Both sides count the same thing. □

    Application: Prefix Sums of Binomial Coefficients

    The hockey stick identity lets us compute partial sums of a column of Pascal’s triangle in O(1):

    // Sum of C(r,r) + C(r+1,r) + ... + C(n,r) = C(n+1, r+1)
// This is extremely useful for range queries on Pascal's triangle

// Example: sum of C(i, 3) for i = 3..100
long long answer = C(101, 4);  // Done! No loop needed.
    Contest insight: If your solution requires summing C(i, r) for i in a range [a, b], rewrite it as C(b+1, r+1) − C(a, r+1). This telescopes a linear sum into two binomial coefficient evaluations.

    Counting Edges in Complete Graphs

    The complete graph Kn has an edge between every pair of vertices. How many edges?

    Double Counting Approach

    Method 1 (pairs): Each edge is a pair of vertices. Answer: C(n, 2) = n(n−1)/2.

    Method 2 (degrees): Each vertex has degree n − 1. By the handshaking lemma: |E| = n(n−1)/2. ✓

    Counting Triangles

    How many triangles in Kn? We can count by different methods:

    Method 1: Choose 3 vertices: C(n, 3) = n(n−1)(n−2)/6.

    Method 2 (double counting edges in triangles): Each triangle has 3 edges. Each edge (u,v) is in exactly n−2 triangles (one for each remaining vertex). So the total edge-triangle incidences = 3T = C(n,2)(n−2), giving T = C(n,3). ✓

    // Count triangles in a general graph by double counting
// For each edge (u,v), count common neighbors
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;

    vector<set<int>> adj(n + 1);
    vector<pair<int,int>> edges(m);

    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].insert(v);
        adj[v].insert(u);
        edges[i] = {u, v};
    }

    long long triangles = 0;
    for (auto& [u, v] : edges) {
        // Count common neighbors of u and v
        for (int w : adj[u]) {
            if (adj[v].count(w)) {
                triangles++;
            }
        }
    }
    // Each triangle counted 3 times (once per edge)
    triangles /= 3;

    cout << "Triangles: " << triangles << "\n";
    return 0;
}
    Complexity note: The naive triangle counting above is O(m · Δ) where Δ is the max degree. For sparse graphs, orient edges from lower to higher degree and count wedges — this gives O(m√m) complexity.

    The Committee Selection Principle

    A powerful double-counting technique involves counting (element, subset) incidence pairs. Consider choosing a committee of size r from n people and then selecting a chair from the committee.

    • Committee first, then chair: Choose the committee (C(n, r) ways), then the chair (r ways). Total: r · C(n, r).
    • Chair first, then committee: Choose the chair (n ways), then choose the remaining r−1 members from the other n−1 people (C(n−1, r−1) ways). Total: n · C(n−1, r−1).

    Equating: r · C(n, r) = n · C(n−1, r−1)

    This is the absorption identity and is extremely useful for simplifying expressions with binomial coefficients.

    Generalization: Choosing a Sub-Committee

    Choose a committee of r and then a sub-committee of s from it. Double counting gives:

    C(n, r) · C(r, s) = C(n, s) · C(n − s, r − s)

    Proof:

    • Left: Choose r people, then choose s of them for the sub-committee.
    • Right: Choose s people for the sub-committee first, then choose the remaining r − s from the other n − s people.

    This identity (sometimes called the triple-binomial identity) is useful when you encounter nested binomial coefficients in a sum.

    More Combinatorial Identities

    Sum of All Binomial Coefficients

    ∑k=0n C(n, k) = 2n

    Proof: Both sides count the total number of subsets of an n-element set.

    Alternating Sum

    ∑k=0n (−1)k C(n, k) = 0   (for n ≥ 1)

    Proof: Expand (1 − 1)n = 0 using the binomial theorem. Alternatively: the number of even-sized subsets equals the number of odd-sized subsets (biject by toggling membership of element 1).

    Weighted Sum

    ∑k=0n k · C(n, k) = n · 2n−1

    Proof by double counting: Count (element, subset) pairs where the element is in the subset. Left: for each subset of size k, there are k such pairs, total = ∑ k · C(n,k). Right: for each element (n choices), it appears in 2n−1 subsets.

    Sum of Squares of Binomial Coefficients

    ∑k=0n C(n, k)2 = C(2n, n)

    Proof: Apply Vandermonde with m = n, r = n: C(2n, n) = ∑ C(n,k) · C(n, n−k) = ∑ C(n,k)2. □

    // Verify sum of squares identity
void verify_sum_of_squares(int n) {
    long long lhs = 0;
    for (int k = 0; k <= n; k++) {
        long long ck = C(n, k);
        lhs = (lhs + ck * ck) % MOD;
    }
    long long rhs = C(2 * n, n);
    assert(lhs == rhs);
}

    Diagonal Sum in Pascal’s Triangle

    ∑k=0⌊n/2⌋ C(n − k, k) = Fn+1

    where Fn is the n-th Fibonacci number. This beautiful identity connects Pascal’s triangle to the Fibonacci sequence via the “shallow diagonals.”

    Identity toolkit for CP: Knowing these identities by heart saves precious contest time. When you see a sum of binomial coefficients, try to match it with a known identity before resorting to brute force computation.

    Practice Problems

    1. Handshaking verification: Given a graph, verify ∑ deg(v) = 2|E| and count odd-degree vertices.
    2. Vandermonde application: Compute ∑k=0r C(a, k) · C(b, r−k) mod 109+7 in O(1) using Vandermonde.
    3. Hockey stick range query: Given queries (a, b, r), compute ∑i=ab C(i, r) mod 109+7 in O(1) per query.
    4. Triangle counting: Given a graph with n ≤ 2000 vertices, count all triangles in O(n · m) time.
    5. Subset sum of sizes: Compute ∑k=0n k2 · C(n, k). (Hint: differentiate the binomial theorem, or double count pairs of elements in subsets.)
    6. Combinatorial proof challenge: Prove that ∑k=0n C(n, k) · C(n, n−k) = C(2n, n) by constructing a bijection.
    7. Double counting on bipartite graphs: Given a bipartite graph, prove that the number of paths of length 2 equals ∑ C(deg(v), 2) summed over vertices on one side.
    8. Generalized committee: n people, form a committee of r, then choose a sub-committee of s, then choose a leader from the sub-committee. Express the answer in closed form.
