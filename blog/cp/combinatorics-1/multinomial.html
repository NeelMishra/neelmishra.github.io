    Multinomial Coefficients & the Multinomial Theorem

    June 2026 · 18 min read

    From Binomial to Multinomial

    You already know the binomial coefficient C(n, k) counts the ways to split n objects into two groups of sizes k and n − k. But what if you need to split them into more than two groups?

    Imagine distributing n distinct objects into m labeled groups of sizes k₁, k₂, …, kₘ where k₁ + k₂ + … + kₘ = n. The number of ways is the multinomial coefficient:

    n! / (k₁! · k₂! · … · kₘ!)

    The binomial coefficient C(n, k) = n! / (k! · (n−k)!) is just the special case m = 2. Every binomial identity you know has a multinomial generalization.

    Notation: The multinomial coefficient is sometimes written as C(n; k₁, k₂, …, kₘ) or with the "multi-choose" parentheses. In this post we write it explicitly as the factorial ratio.

    Multinomial Coefficients

    Definition and Intuition

    Given non-negative integers k₁, k₂, …, kₘ with k₁ + k₂ + … + kₘ = n, the multinomial coefficient is: 

    multinomial(n; k₁, k₂, …, kₘ) = n! / (k₁! · k₂! · … · kₘ!)

    Why does this work? Start with n! permutations of all objects. Within each group, order doesn't matter, so we divide by the factorial of each group size to remove the internal orderings.

    Successive Binomial Decomposition

    A multinomial coefficient can be computed as a product of binomial coefficients: 

    n!/(k₁!·k₂!·…·kₘ!) = C(n, k₁) · C(n−k₁, k₂) · C(n−k₁−k₂, k₃) · …

    This is the "choose sequentially" approach: first choose k₁ items from n, then k₂ from the remaining n − k₁, and so on.

    Example: Distributing Playing Cards

    Deal a standard 52-card deck to 4 players (13 cards each). The number of possible deals is: 

    52! / (13! · 13! · 13! · 13!) ≈ 5.36 × 10²⁸
    CP Tip: When a problem says "arrange n objects where k₁ are of type 1, k₂ of type 2, etc." — the answer is a multinomial coefficient.

    The Multinomial Theorem

    The binomial theorem states: 

    (x + y)ⁿ = Σ_{k=0}^{n} C(n, k) · xᵏ · yⁿ⁻ᵏ

    The multinomial theorem generalizes this to m variables: 

    (x₁ + x₂ + … + xₘ)ⁿ =   Σ         n!/(k₁!·k₂!·…·kₘ!) · x₁^k₁ · x₂^k₂ · … · xₘ^kₘ
                          k₁+…+kₘ=n

    The sum ranges over all non-negative integer tuples (k₁, …, kₘ) summing to n.

    Proof Sketch

    Expand (x₁ + x₂ + … + xₘ)ⁿ by choosing one term from each of the n factors. The term x₁^k₁ · x₂^k₂ · … · xₘ^kₘ arises whenever we choose x₁ from exactly k₁ factors, x₂ from k₂ factors, etc. The number of such choices is n! / (k₁! · k₂! · … · kₘ!).

    Special Case: Counting Terms

    Setting all xᵢ = 1 gives: 

    mⁿ = Σ_{k₁+…+kₘ=n}  n!/(k₁!·k₂!·…·kₘ!)

    This makes sense: mⁿ is the total number of functions from an n-set to an m-set, and each such function partitions the domain into m groups.

    Identity: The sum of all multinomial coefficients for a given n and m is mⁿ.

    Counting Arrangements with Repetitions

    One of the most direct applications of multinomial coefficients is counting distinct permutations of a multiset — often phrased as "anagram counting."

    Anagram Counting

    How many distinct arrangements of the letters in MISSISSIPPI are there?

    • Total letters: 11
    • M: 1, I: 4, S: 4, P: 2
    Answer = 11! / (1! · 4! · 4! · 2!) = 39916800 / (1 · 24 · 24 · 2) = 34650

    Grid Path Counting

    Count lattice paths from (0, 0) to (a, b) using only right (R) and up (U) moves. A path is a sequence of a R's and b U's, so: 

    Paths = (a + b)! / (a! · b!) = C(a + b, a)

    For 3D paths to (a, b, c) with moves in three directions: 

    Paths = (a + b + c)! / (a! · b! · c!)

    Distributing Labeled Objects into Labeled Bins

    Assign n students to m project teams of specified sizes. This is exactly the multinomial coefficient. 

    // Count distinct permutations of a string
#include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;
const int MAXN = 1e6 + 5;
long long fac[MAXN], inv_fac[MAXN];

long long power(long long a, long long b, long long mod) {
    long long res = 1; a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod; b >>= 1;
    }
    return res;
}

void precompute() {
    fac[0] = 1;
    for (int i = 1; i < MAXN; i++)
        fac[i] = fac[i - 1] * i % MOD;
    inv_fac[MAXN - 1] = power(fac[MAXN - 1], MOD - 2, MOD);
    for (int i = MAXN - 2; i >= 0; i--)
        inv_fac[i] = inv_fac[i + 1] * (i + 1) % MOD;
}

long long count_anagrams(const string& s) {
    int freq[26] = {};
    for (char c : s) freq[c - 'a']++;
    int n = s.size();
    long long ans = fac[n];
    for (int i = 0; i < 26; i++)
        ans = ans % MOD * inv_fac[freq[i]] % MOD;
    return ans;
}

    Computing Multinomials Modularly

    In competitive programming, we almost always compute multinomial coefficients modulo a prime. There are several strategies.

    Method 1: Factorial + Inverse Factorial

    Precompute fac[] and inv_fac[]. Then: 

    multinomial(n; k₁, …, kₘ) = fac[n] · inv_fac[k₁] · inv_fac[k₂] · … · inv_fac[kₘ]  (mod p)

    This is O(n) preprocessing and O(m) per query. It works when n < p. 

    long long multinomial(int n, vector<int>& k) {
    long long ans = fac[n];
    for (int ki : k)
        ans = ans % MOD * inv_fac[ki] % MOD;
    return ans;
}

    Method 2: Successive Binomial Products

    Compute as a chain of binomial coefficients. Useful when you don't want to precompute a large factorial table: 

    long long multinomial_via_binom(int n, vector<int>& k) {
    long long ans = 1;
    int remaining = n;
    for (int ki : k) {
        ans = ans % MOD * C(remaining, ki) % MOD;
        remaining -= ki;
    }
    return ans;
}

    Method 3: Lucas' Theorem for Large n

    When n ≥ p (the modulus), Fermat's inverse doesn't directly help because fac[n] ≡ 0 (mod p) for n ≥ p. In this case, use Lucas' theorem to decompose each binomial coefficient in the chain. 

    // Lucas' theorem for C(n, r) mod small prime p
long long lucas(long long n, long long r, long long p) {
    if (r == 0) return 1;
    return C(n % p, r % p) % p * lucas(n / p, r / p, p) % p;
}
    Pitfall: If the modulus is not prime, you cannot directly use Fermat's little theorem for inverse factorials. Use the Chinese Remainder Theorem combined with Lucas' theorem for prime-power factors, or use a different approach altogether.

    Complexity Summary

    Factorial Table
    Precompute: O(n)
    Per query: O(m)
    Constraint: n < p
    Lucas' Theorem
    No precompute needed
    Per query: O(m · log_p(n))
    Works for any n, small p

    Practice Problems

    Work through these to reinforce multinomial coefficient concepts:

    1. Creating Strings (CSES 1715) — Count distinct permutations of a given string. Direct multinomial application.
    2. Grid Paths in 3D — Count lattice paths from origin to (a, b, c) using axis-aligned unit steps.
    3. Committee Formation — From n people, form committees of specified sizes. The number of ways is a multinomial coefficient.
    4. MISSISSIPPI Variants — Count arrangements of ABRACADABRA (A:5, B:2, R:2, C:1, D:1). Answer: 11! / (5!·2!·2!·1!·1!) = 83160.
    5. Polynomial Expansion — Find the coefficient of x²y³z in (x + y + z)⁶. Answer: 6! / (2!·3!·1!) = 60.
    Mental Checklist: Whenever you see "distinct arrangements of objects where some are identical," think multinomial. If objects are all distinct and you're partitioning into groups, it's also a multinomial coefficient.

    Complete Working Example

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;
const int MAXN = 1e6 + 5;
long long fac[MAXN], inv_fac[MAXN];

long long power(long long a, long long b, long long mod) {
    long long res = 1; a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod; b >>= 1;
    }
    return res;
}

void precompute() {
    fac[0] = 1;
    for (int i = 1; i < MAXN; i++)
        fac[i] = fac[i - 1] * i % MOD;
    inv_fac[MAXN - 1] = power(fac[MAXN - 1], MOD - 2, MOD);
    for (int i = MAXN - 2; i >= 0; i--)
        inv_fac[i] = inv_fac[i + 1] * (i + 1) % MOD;
}

int main() {
    precompute();
    string s;
    cin >> s;

    int freq[26] = {};
    for (char c : s) freq[c - 'a']++;

    long long ans = fac[s.size()];
    for (int i = 0; i < 26; i++)
        ans = ans % MOD * inv_fac[freq[i]] % MOD;

    cout << ans << "\n";
    return 0;
}
