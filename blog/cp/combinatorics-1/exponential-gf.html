    Exponential Generating Functions & Labeled Structures

    September 2026 · 22 min read

    From OGF to EGF

    In the previous two posts we used ordinary generating functions (OGFs) where A(x) = Σ aₙ xⁿ. OGFs are perfect for unlabelled objects — coins, tiles, integer compositions — because multiplying two OGFs gives a convolution that counts unordered pairings.

    But many combinatorial objects are labelled: the elements carry distinct identities (e.g., people sitting in a row, edges of a graph, labelled trees). When we merge two labelled structures of sizes a and b into one of size a + b, we must choose which a of the a + b labels go to the first structure — a factor of C(a+b, a).

    The exponential generating function absorbs this binomial coefficient automatically by dividing each term by n!: 

    Â(x) = Σ (n ≥ 0) aₙ · xⁿ / n!

    Now multiplication of two EGFs produces exactly the right binomial convolution for labelled merging. This single change in normalisation unlocks a beautiful theory.

    Rule of thumb: Use OGFs for unlabelled objects, EGFs for labelled objects. If the problem says "distinguishable" or "labelled" or "permutation", reach for EGFs.

    Exponential Generating Functions Defined

    Formally, the EGF of a sequence {aₙ} is: 

    Â(x) = Σ (n ≥ 0) aₙ · xⁿ / n!

    The product rule for EGFs is: 

    If Ĉ(x) = Â(x) · B̂(x),  then  cₙ = Σ (k=0..n) C(n,k) · aₖ · bₙ₋ₖ

    This is the binomial convolution (also called the exponential convolution). Combinatorially: split the label set {1,…,n} into a subset of size k (for the A-structure) and the complementary subset of size n−k (for the B-structure), in all C(n,k) ways.

    Some fundamental EGFs:

    StructureaₙEGF
    Single element[n=1]x
    Set (any size)1eˣ
    Non-empty set[n≥1]eˣ − 1
    Permutationsn!1/(1−x)
    DerangementsDₙe⁻ˣ/(1−x)
    InvolutionsIₙexp(x + x²/2)
    Labelled rooted treesnⁿ⁻¹T(x) where T = x·exp(T)

    Labeled vs Unlabeled Counting

    The distinction matters enormously. Consider arranging objects into groups:

    Unlabelled (OGF)
    Partition the integer n into parts from a given set. Use OGF product: ∏ 1/(1 − xᵈ).
    Example: coin change, integer partitions.
    Labelled (EGF)
    Partition the set {1,…,n} into non-empty subsets. Use EGF composition: exp(eˣ − 1) → Bell numbers.
    Example: set partitions, graph coloring.

    A common mistake in contests is applying OGF multiplication when the objects are labelled. If you find yourself multiplying OGFs and getting answers that are too small by a factor of C(n,k), you probably need EGFs.

    Warning: The same combinatorial structure can have very different OGF and EGF. For example, the number of labelled graphs on n vertices is 2^{C(n,2)}, but the number of unlabelled graphs (up to isomorphism) is much smaller and has no nice closed form.

    EGF for Permutations

    A permutation of {1,…,n} is an ordered arrangement. There are n! permutations, so: 

    Â(x) = Σ n! · xⁿ/n! = Σ xⁿ = 1/(1−x)

    Wait — that's the same as the OGF of the all-ones sequence! Yes, and this is no coincidence: the EGF of n! simplifies to a geometric series.

    Derangements

    A derangement is a permutation with no fixed points. By inclusion-exclusion: 

    Dₙ = n! · Σ (k=0..n) (−1)ᵏ / k!

    The EGF of derangements is:

    D̂(x) = Σ Dₙ xⁿ/n! = e⁻ˣ / (1 − x)

    This follows because D̂(x) = (EGF of "not a fixed point") composed appropriately, or more directly from the formula Dₙ/n! = Σ (−1)ᵏ/k! whose EGF is e⁻ˣ times 1/(1−x).

    Here is a C++ snippet to compute derangement counts modulo a prime using the recurrence Dₙ = (n−1)(Dₙ₋₁ + Dₙ₋₂): 

    #include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

// Compute D[0..n] where D[i] = number of derangements of i elements.
vector<long long> derangements(int n) {
    vector<long long> D(n + 1);
    D[0] = 1; // empty permutation is trivially a derangement
    if (n >= 1) D[1] = 0;
    for (int i = 2; i <= n; i++)
        D[i] = (long long)(i - 1) % MOD * ((D[i-1] + D[i-2]) % MOD) % MOD;
    return D;
}

int main() {
    int n;
    cin >> n;
    auto D = derangements(n);
    cout << D[n] << "\n";
}
    Fun fact: Dₙ / n! converges to 1/e ≈ 0.3679 extremely quickly. For n ≥ 12, Dₙ = round(n!/e).

    EGF for Set Partitions

    A set partition of {1,…,n} divides the elements into non-empty, non-overlapping subsets (called blocks). The number of such partitions is the Bell number Bₙ.

    The EGF for a single non-empty block is eˣ − 1 (a set of any positive size). A partition is a set of blocks, so by the exponential formula: 

    B̂(x) = exp(eˣ − 1)

    This means Bₙ = n! · [xⁿ] exp(eˣ − 1). The first few Bell numbers are 1, 1, 2, 5, 15, 52, 203, …

    Stirling numbers of the second kind

    S(n,k) counts partitions of {1,…,n} into exactly k blocks. Its EGF (in n, with k fixed) is: 

    Ŝₖ(x) = (eˣ − 1)ᵏ / k!

    This is just the k-th term in the Taylor expansion of exp(eˣ − 1). We can compute S(n,k) via inclusion-exclusion: 

    S(n,k) = (1/k!) · Σ (j=0..k) (−1)^{k−j} · C(k,j) · jⁿ
    #include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

long long power(long long a, long long b, long long mod) {
    long long r = 1; a %= mod;
    while (b > 0) { if (b & 1) r = r * a % mod; a = a * a % mod; b >>= 1; }
    return r;
}
long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Compute S(n, k) = Stirling number of the second kind.
long long stirling2(int n, int k) {
    // Precompute factorials and inverse factorials.
    vector<long long> fact(k + 1), ifact(k + 1);
    fact[0] = 1;
    for (int i = 1; i <= k; i++) fact[i] = fact[i-1] * i % MOD;
    ifact[k] = inv(fact[k]);
    for (int i = k - 1; i >= 0; i--) ifact[i] = ifact[i+1] * (i+1) % MOD;

    long long ans = 0;
    for (int j = 0; j <= k; j++) {
        long long term = ifact[j] % MOD * ifact[k - j] % MOD * power(j, n, MOD) % MOD;
        if ((k - j) & 1) ans = (ans - term + MOD) % MOD;
        else ans = (ans + term) % MOD;
    }
    return ans;
}

int main() {
    int n, k;
    cin >> n >> k;
    cout << stirling2(n, k) << "\n";
}

    EGF Composition (Exponential Formula)

    The exponential formula is one of the most powerful tools in enumerative combinatorics. It states: 

    If Ĉ(x) is the EGF of connected structures,
then  exp(Ĉ(x))  is the EGF of sets of connected structures.

    More precisely, if every labelled structure on {1,…,n} decomposes uniquely into connected components, then the EGF of all structures equals exp of the EGF of connected structures.

    Applications

    • Labelled graphs: The EGF of connected labelled graphs Ĉ(x) satisfies exp(Ĉ(x)) = Σ 2^{C(n,2)} xⁿ/n!. Taking the logarithm gives Ĉ(x).
    • Permutations → cycles: A permutation decomposes into cycles. The EGF of a single cycle of length n is (n−1)! · xⁿ/n! = xⁿ/n, so the cycle EGF is ln(1/(1−x)). Then exp(ln(1/(1−x))) = 1/(1−x) — the EGF of all permutations. ✓
    • Set partitions: A partition = set of non-empty blocks. Block EGF = eˣ − 1. Partition EGF = exp(eˣ − 1) — the Bell number EGF we saw above. ✓
    The exponential formula in one line: Structures = exp(Connected). This single identity encodes dozens of classical results.

    Labeled rooted trees (Cayley's formula)

    The EGF T(x) of labelled rooted trees satisfies the functional equation: 

    T(x) = x · exp(T(x))

    By Lagrange inversion (from the previous post), with φ(u) = exp(u): 

    [xⁿ] T(x) = (1/n) · [uⁿ⁻¹] exp(nu) = (1/n) · nⁿ⁻¹/(n−1)! = nⁿ⁻¹/n!

    So the number of labelled rooted trees on n vertices is n! · nⁿ⁻¹/n! = nⁿ⁻¹. Dividing by n (un-rooting) gives Cayley's formula: nⁿ⁻² labelled unrooted trees.

    Computing with EGFs

    In competitive programming, EGF computations boil down to polynomial arithmetic with a twist: we store aₙ/n! as the n-th coefficient, multiply polynomials with NTT, and recover aₙ by multiplying back by n!.

    EGF multiplication template

    #include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

long long power(long long a, long long b, long long mod) {
    long long r = 1; a %= mod;
    while (b > 0) { if (b & 1) r = r * a % mod; a = a * a % mod; b >>= 1; }
    return r;
}
long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Precompute factorial and inverse factorial tables.
vector<long long> fact, ifact;
void initFact(int n) {
    fact.resize(n + 1); ifact.resize(n + 1);
    fact[0] = 1;
    for (int i = 1; i <= n; i++) fact[i] = fact[i-1] * i % MOD;
    ifact[n] = inv(fact[n]);
    for (int i = n - 1; i >= 0; i--) ifact[i] = ifact[i+1] * (i+1) % MOD;
}

// Convert sequence a[] to EGF coefficients: egf[i] = a[i] / i!
vector<long long> toEGF(vector<long long>& a) {
    int n = a.size();
    vector<long long> egf(n);
    for (int i = 0; i < n; i++)
        egf[i] = a[i] % MOD * ifact[i] % MOD;
    return egf;
}

// Convert EGF coefficients back to sequence: a[i] = egf[i] * i!
vector<long long> fromEGF(vector<long long>& egf) {
    int n = egf.size();
    vector<long long> a(n);
    for (int i = 0; i < n; i++)
        a[i] = egf[i] % MOD * fact[i] % MOD;
    return a;
}

// Forward-declare NTT multiply (same as previous posts).
vector<long long> multiply(vector<long long> a, vector<long long> b);

// Multiply two EGFs: the result's n-th coeff encodes
// cₙ = Σ C(n,k) · aₖ · bₙ₋ₖ  (binomial convolution).
vector<long long> multiplyEGF(vector<long long>& A, vector<long long>& B, int N) {
    // A and B are already in EGF form (divided by factorials).
    auto C = multiply(A, B);
    C.resize(N);
    return C;
}

    Computing exp(F(x)) for the exponential formula

    To apply the exponential formula we need exp(F(x)) mod xⁿ. This can be computed via Newton iteration (similar to polynomial inverse). The key recurrence is: 

    Gₖ₊₁ = Gₖ · (1 + F − ln(Gₖ))   mod x^{2ᵏ⁺¹}

    where ln is the formal logarithm ln(G) = ∫ G'/G. Each doubling step uses O(n log n) NTT operations, giving O(n log n) total. 

    // Polynomial exp: compute exp(F(x)) mod x^n.
// Requires F[0] == 0.  Uses polyInverse and NTT (defined in earlier posts).
vector<long long> polyLn(vector<long long> F, int n);   // ln(F) mod x^n
vector<long long> polyInverse(vector<long long> F, int n);

vector<long long> polyExp(vector<long long> F, int n) {
    // Newton iteration: G = G * (1 + F - ln(G))
    vector<long long> G = {1};
    for (int len = 2; len < 2 * n; len <<= 1) {
        auto lnG = polyLn(G, len);
        vector<long long> tmp(len, 0);
        tmp[0] = 1; // the "1" in (1 + F - ln(G))
        for (int i = 0; i < len; i++) {
            long long fi = (i < (int)F.size()) ? F[i] : 0;
            tmp[i] = (tmp[i] + fi - lnG[i] + MOD) % MOD;
        }
        G = multiply(G, tmp);
        G.resize(len);
    }
    G.resize(n);
    return G;
}
    Complexity: Both polyLn and polyExp run in O(n log n). Together with NTT multiplication, this gives a complete toolkit for EGF computations in competitive programming.

    Practice Problems

    These problems will test your understanding of EGFs and the exponential formula:

    1. CSES — Counting Necklaces (Labeled version)
      Count labelled necklaces (cyclic permutations). The cycle EGF is −ln(1−x); extract the appropriate coefficient.
    2. Codeforces 1549F — Derangements
      Count derangements with additional constraints. Start from the EGF e⁻ˣ/(1−x) and modify.
    3. Atcoder ABC253 Ex — Set Partitions
      Count set partitions with bounded block sizes. The block EGF is a truncated exponential; apply the exponential formula.
    4. Library Checker — Exp of Formal Power Series
      Implement polyExp and verify against the online judge.
    5. Codeforces 1824B1/B2 — Labeled Trees
      Apply Cayley's formula via the functional equation T(x) = x · exp(T(x)) and Lagrange inversion.
    6. Codeforces 960G — Bandit Blues
      Uses Stirling numbers and the connection between permutation statistics and EGFs. Compute via NTT-based convolution.
    7. Atcoder ARC158 E — Connected Labeled Graphs
      The exponential formula in action: total graphs = exp(connected graphs). Take the log of the total-graph EGF to extract the connected count.
