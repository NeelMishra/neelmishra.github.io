    Bell Numbers & Set Partitions

    July 2026 · 18 min read

    What Are Bell Numbers?

    The n-th Bell number B(n) counts the total number of ways to partition a set of n labelled elements into non-empty, unordered subsets (where the number of subsets can be anything from 1 to n).

    The first few Bell numbers are:

    B(0) = 1,   B(1) = 1,   B(2) = 2,   B(3) = 5,   B(4) = 15,   B(5) = 52,   B(6) = 203,   B(7) = 877

    For example, the set {a, b, c} has B(3) = 5 partitions: 

    {a, b, c}                   — 1 subset
{a, b} | {c}                — 2 subsets
{a, c} | {b}                — 2 subsets
{b, c} | {a}                — 2 subsets
{a} | {b} | {c}             — 3 subsets
    Growth rate: Bell numbers grow super-exponentially. B(25) is already over 4.6 × 1018, exceeding the long long limit. As with other counting sequences, CP problems ask for B(n) mod a prime.

    Bell numbers appear in problems involving equivalence relations (since there is a bijection between partitions of a set and equivalence relations on it), in counting rhyme schemes, in database query optimization (number of join orderings), and in various combinatorial identities.

    The Bell Triangle

    The Bell triangle (also called the Aitken's array or Peirce triangle) is an elegant construction for computing Bell numbers. It works as follows:

    1. Start with B(0) = 1 in the top-left corner.
    2. For each new row, the leftmost entry equals the rightmost entry of the previous row.
    3. Each subsequent entry in a row is the sum of the entry to its left and the entry directly above that entry.
    Row 0:  1
Row 1:  1   2
Row 2:  2   3   5
Row 3:  5   7  10  15
Row 4: 15  20  27  37  52
Row 5: 52  67  87 114 151 203

    The first entry of each row gives the Bell numbers: 1, 1, 2, 5, 15, 52, 203, …

    More precisely, if we denote the triangle entries as T(n, k) where n is the row (0-indexed) and k is the column (0-indexed), then:

    • T(0, 0) = 1
    • T(n, 0) = T(n − 1, n − 1)   (wrap from end of previous row)
    • T(n, k) = T(n, k − 1) + T(n − 1, k − 1)   for k ≥ 1
    • B(n) = T(n, 0)
    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

// Compute Bell numbers B(0) .. B(n) using the Bell triangle
vector<long long> bell_numbers(int n) {
    // T[i][j] = Bell triangle entry
    // Only need previous row and current row
    vector<long long> prev(1, 1); // Row 0: {1}
    vector<long long> bell(n + 1);
    bell[0] = 1;

    for (int i = 1; i <= n; i++) {
        vector<long long> curr(i + 1);
        curr[0] = prev.back(); // wrap from end of previous row
        for (int j = 1; j <= i; j++) {
            curr[j] = (curr[j - 1] + prev[j - 1]) % MOD;
        }
        bell[i] = curr[0];
        prev = curr;
    }
    return bell;
}

int main() {
    int n;
    cin >> n;
    auto b = bell_numbers(n);
    for (int i = 0; i <= n; i++)
        cout << "B(" << i << ") = " << b[i] << "\n";
    return 0;
}
    Complexity: The Bell triangle approach computes B(0) through B(n) in O(n2) time and O(n) space (with rolling arrays). This is the standard method for n up to about 5000.

    Relation to Stirling Numbers

    Bell numbers are directly related to Stirling numbers of the second kind:

    B(n) = ∑k=0n S(n, k)

    This follows immediately from the definitions: S(n, k) counts partitions into exactly k subsets, and B(n) counts all partitions regardless of the number of subsets. Summing over all possible k gives B(n).

    Practical implication: If you've already computed the Stirling triangle S(n, k), getting Bell numbers is a simple row sum. Conversely, if you only need B(n) and not individual S(n, k) values, the Bell triangle is more efficient since it avoids the full 2D Stirling table.

    Bell Number Recurrence

    There is also a direct recurrence for Bell numbers:

    B(n + 1) = ∑k=0n C(n, k) · B(k)

    Proof: Consider element n + 1 in a partition of {1, …, n + 1}. The subset containing n + 1 has some k other elements chosen from {1, …, n} (there are C(n, k) choices). The remaining n − k elements are partitioned in B(n − k) ways. Substituting j = n − k gives the formula above. 

    // Bell numbers via the binomial recurrence
vector<long long> bell_via_recurrence(int n) {
    // Precompute binomial coefficients
    vector<vector<long long>> C(n + 2, vector<long long>(n + 2, 0));
    for (int i = 0; i <= n + 1; i++) {
        C[i][0] = 1;
        for (int j = 1; j <= i; j++)
            C[i][j] = (C[i-1][j-1] + C[i-1][j]) % MOD;
    }

    vector<long long> B(n + 1);
    B[0] = 1;
    for (int i = 0; i < n; i++) {
        B[i + 1] = 0;
        for (int k = 0; k <= i; k++)
            B[i + 1] = (B[i + 1] + C[i][k] % MOD * B[k]) % MOD;
    }
    return B;
}

    Exponential Generating Function

    The exponential generating function (EGF) for the Bell numbers is remarkably clean:

    ∑n≥0 B(n) · xn / n! = eex − 1

    Why? The EGF for the number of ways to partition a set into non-empty subsets is the "set of non-empty sets" construction. The EGF for a single non-empty set is ex − 1. The EGF for a "set of" objects is exp(EGF of the object). So:

    EGF of partitions = exp(ex − 1)

    This is a beautiful example of the exponential formula in combinatorics.

    Advanced: The EGF can be used to derive asymptotic estimates and to prove identities. For instance, expanding eex − 1 around x = 0 and extracting the coefficient of xn/n! recovers B(n). In contest settings, the EGF is rarely needed directly, but understanding it helps build intuition for "compound counting" problems.

    Connection to Poisson Distribution

    The EGF eex − 1 arises naturally from the composition of exponential generating functions. It also connects Bell numbers to the Poisson distribution: B(n) is the n-th moment of a Poisson(1) random variable.

    Dobinski's Formula

    Dobinski's formula expresses Bell numbers as an infinite series:

    B(n) = (1/e) · ∑k=0∞ kn / k!

    Since kn / k! decreases rapidly for k > n, in practice we only need a finite number of terms for exact computation modulo a prime.

    Using Dobinski's Formula Modulo a Prime

    To compute B(n) mod p, note that for k ≥ p, we have k! ≡ 0 (mod p), so those terms vanish. This gives a finite sum:

    B(n) mod p ≡ (1/e mod p) · ∑k=0p−1 kn / k!   (mod p)

    But what is "1/e mod p"? We need the modular inverse of e evaluated mod p. Since e mod p = ∑k=0p−1 1/k! mod p, we can compute this.

    Caution: Dobinski's formula is theoretically elegant but computationally tricky modulo a prime. The Bell triangle method is usually more straightforward for contests. Use Dobinski's formula when the problem structure specifically calls for it (e.g., when n is huge but p is small). 
    // Dobinski's formula for B(n) mod p, where p is prime
// O(p log n) time
long long bell_dobinski(long long n, long long p) {
    // Compute inv_fac[0..p-1] and e_mod_p
    vector<long long> inv_fac(p);
    vector<long long> fac(p);
    fac[0] = 1;
    for (long long i = 1; i < p; i++) fac[i] = fac[i-1] * i % p;
    inv_fac[p-1] = power(fac[p-1], p - 2, p);
    for (long long i = p - 2; i >= 0; i--) inv_fac[i] = inv_fac[i+1] * (i+1) % p;

    // e mod p = sum 1/k! for k=0..p-1
    long long e_mod_p = 0;
    for (long long k = 0; k < p; k++)
        e_mod_p = (e_mod_p + inv_fac[k]) % p;

    long long inv_e = power(e_mod_p, p - 2, p);

    // B(n) = inv_e * sum k^n / k! for k=0..p-1
    long long ans = 0;
    for (long long k = 0; k < p; k++) {
        long long term = power(k, n, p) % p * inv_fac[k] % p;
        ans = (ans + term) % p;
    }
    ans = ans % p * inv_e % p;
    return ans;
}

    Computing Bell Numbers — Method Summary

    Bell Triangle
    Time: O(n2)
    Space: O(n)
    Computes B(0)…B(n)
    Best general-purpose method
    Sum of Stirling
    Time: O(n2)
    Space: O(n2) or O(n)
    Gives individual S(n,k) too
    When you need S(n,k) as well
    Dobinski's Formula
    Time: O(p log n)
    Space: O(p)
    Works for huge n, small p
    Specialized use cases
    EGF + NTT
    Time: O(n log n)
    Space: O(n)
    Full row in quasi-linear time
    Advanced / large n

    EGF + Polynomial Exponentiation

    For very large n (e.g., n up to 105 or 106), we can compute all Bell numbers B(0) through B(n) using the EGF:

    1. Compute the polynomial ex − 1 truncated to n + 1 terms.
    2. Compute exp(ex − 1) using polynomial exponentiation (via polynomial log and exp, each O(n log n) with NTT).
    3. Extract coefficients: B(k) = k! · [xk] result.
    Contest reality: The Bell triangle covers 99% of contest problems. NTT-based methods are for the hardest problems at the national/international level.

    CP Applications

    Counting Equivalence Relations

    Problem: Given n elements, how many equivalence relations can be defined on them? Answer: B(n), since equivalence relations correspond bijectively to set partitions.

    Counting Rhyme Schemes

    Problem: A poem has n lines. Two lines rhyme if they end with the same sound. How many distinct rhyme schemes are there? Answer: B(n). Each rhyme scheme is a partition of the lines into groups of rhyming lines.

    Distributing Identical Roles

    Problem: Assign n people to groups where groups are unordered and unlabelled (only group membership matters). Answer: B(n).

    Variant: if groups are labelled (ordered), the answer is the ordered Bell number (Fubini number), which is ∑k k! · S(n, k).

    Graph Colouring with Unlimited Colours

    The number of ways to properly colour the vertices of a graph G on n vertices using any number of colours (where two colourings are the same if they induce the same partition of vertices) relates to Bell numbers and the chromatic polynomial.

    Set Partition Lattice Problems

    Some advanced problems operate on the partition lattice — the partial order on set partitions ordered by refinement. Möbius inversion on this lattice involves Bell numbers. 

    // Example: count set partitions with exactly k non-empty parts
// This is just S(n, k), but here we compute B(n) and verify the sum
int main() {
    int n;
    cin >> n;
    auto b = bell_numbers(n);
    cout << "B(" << n << ") = " << b[n] << "\n";

    // Verify: B(n) = sum S(n, k)
    // (Assuming compute_stirling2 is available)
    compute_stirling2(n);
    long long check = 0;
    for (int k = 0; k <= n; k++)
        check = (check + S[n][k]) % MOD;
    assert(check == b[n]);
    cout << "Verified!\n";
    return 0;
}
    Recognition tip: If a problem asks "in how many ways can you group n objects" without specifying the number of groups, and the groups are unordered, the answer is B(n). If groups are ordered, multiply by k! to get Fubini numbers.

    Properties and Identities

    Bell Numbers Modulo a Prime

    The Touchard congruence states that for a prime p:

    B(n + p) ≡ B(n) + B(n + 1)   (mod p)

    This gives a way to reduce B(n) mod p for large n by repeatedly subtracting p from the index.

    Exponential Bell Numbers

    The complementary Bell numbers or Rao Uppuluri–Carpenter numbers are defined by the alternating sum:

    B̃(n) = ∑k=0n (−1)k S(n, k)

    These count the difference between partitions with an even number of parts and those with an odd number of parts.

    Spivey's Identity

    A beautiful generalization combining Bell and Stirling numbers:

    B(n + m) = ∑k=0n ∑j=0m S(n, k) · C(m, j) · km−j · B(j)

    Practice Problems

    1. Set Partitions — Given n, compute B(n) mod 109+7. (Direct Bell triangle application.)
    2. CF 1185G2 — Playlist — Counting problems involving grouping with constraints, reducible to Stirling/Bell numbers.
    3. Rhyme Schemes — Count distinct rhyme schemes for an n-line poem. (Direct: B(n).)
    4. Fubini Numbers — Count the number of ordered set partitions (surjections to any-sized set). (Sum k! · S(n, k) over k.)
    5. Touchard Congruence — Given very large n and a prime p, compute B(n) mod p. (Apply B(n + p) ≡ B(n) + B(n + 1) repeatedly.)
    6. AtCoder ABC 310 — Set Partition DP — Partition elements into groups with cost functions. Enumerate subsets or use Bell-number-based bounds.
    7. CF 932E — Team Work — Compute ∑k=1n C(n,k) · km. (Convert km using Stirling numbers, then simplify with Bell-like sums.)
    Summary: Bell numbers B(n) count the total number of set partitions of an n-element set, equal to the sum of Stirling numbers ∑ S(n,k). The Bell triangle computes them in O(n2). Dobinski's formula and the EGF eex−1 provide alternative approaches for special cases. In contests, recognise "unordered grouping" as a Bell number problem and "ordered grouping" as a Fubini number problem.
