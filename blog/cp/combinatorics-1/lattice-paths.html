    Lattice Paths, Dyck Paths & the Ballot Problem

    August 2026 · 22 min read

    Counting Lattice Paths

    A lattice path on a grid moves from one lattice point to another using a fixed set of allowed steps. The most common variant: paths from (0, 0) to (m, n) using only right (R) and up (U) steps.

    Such a path consists of m right steps and n up steps, in some order. The total number of steps is m + n, and we simply choose which m of them are right steps:

    Number of paths from (0,0) to (m,n) = C(m + n, n)

    For example, paths from (0,0) to (3,2) = C(5,2) = 10. Each path corresponds to a binary string of length 5 with exactly 3 R’s and 2 U’s (e.g., RRUРУ, RURUR, …).

    Grid interpretation: If you think of an m × n grid, the number of shortest paths from the bottom-left corner to the top-right corner is C(m+n, n). This is one of the most fundamental facts in combinatorics and appears constantly in CP.

    Paths through a Specific Point

    The number of paths from (0,0) to (m,n) that pass through an intermediate point (a,b) is:

    C(a + b, b) · C((m − a) + (n − b), n − b)

    We split the journey into two independent segments and multiply by the multiplication principle.

    Paths with Obstacles

    Now suppose certain grid cells are blocked. We want to count paths from (0,0) to (m,n) that avoid all blocked cells.

    Single Obstacle

    If cell (a,b) is blocked, the number of valid paths is:

    C(m+n, n) − C(a+b, b) · C((m−a)+(n−b), n−b)

    Multiple Obstacles via Inclusion-Exclusion

    For obstacles at positions p1, p2, …, pk, we apply PIE over subsets of obstacles. However, this is exponential in k. When obstacles lie on a line or have special structure, we can do better.

    DP Approach for General Obstacles

    For arbitrary obstacles, a DP on the grid works well:

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

int main() {
    int m, n, k;
    cin >> m >> n >> k;

    vector<vector<long long>> dp(m + 1, vector<long long>(n + 1, 0));
    set<pair<int,int>> blocked;

    for (int i = 0; i < k; i++) {
        int a, b;
        cin >> a >> b;
        blocked.insert({a, b});
    }

    dp[0][0] = 1;
    for (int i = 0; i <= m; i++) {
        for (int j = 0; j <= n; j++) {
            if (blocked.count({i, j})) {
                dp[i][j] = 0;
                continue;
            }
            if (i > 0) dp[i][j] = (dp[i][j] + dp[i-1][j]) % MOD;
            if (j > 0) dp[i][j] = (dp[i][j] + dp[i][j-1]) % MOD;
        }
    }

    cout << dp[m][n] << "\n";
    return 0;
}

    This runs in O(mn) time, which is efficient when the grid is not too large.

    Large grids: If m, n are up to 106 but there are only a few obstacles, the DP approach is too slow. Use the inclusion-exclusion or LGV approach instead (covered below).

    Dyck Paths and the Catalan Connection

    A Dyck path of length 2n is a lattice path from (0,0) to (2n, 0) using steps (+1, +1) and (+1, −1) that never goes below the x-axis. Equivalently, it’s a path from (0,0) to (n,n) using R and U steps that never crosses above the diagonal y = x (staying on or below it depends on convention).

    The number of Dyck paths of length 2n is the n-th Catalan number:

    Cn = C(2n, n) / (n + 1) = C(2n, n) − C(2n, n + 1)

    This is one of the most important connections in combinatorics. We derive it using the reflection principle (next section).

    Dyck paths encode many combinatorial objects:

    • Balanced parenthesisations of n pairs
    • Binary trees with n internal nodes
    • Triangulations of an (n+2)-gon
    • Non-crossing partition structures
    Catalan recap: C0=1, C1=1, C2=2, C3=5, C4=14, C5=42. We covered Catalan numbers in depth in an earlier post; here we focus on the lattice-path perspective.

    The Reflection Principle

    The reflection principle (also called André’s reflection) is a powerful technique for counting lattice paths that avoid a boundary. The core idea:

    Every “bad” path (one that crosses the forbidden boundary) can be uniquely mapped to a path to a reflected endpoint, creating a bijection.

    Derivation of Catalan Numbers

    Count paths from (0,0) to (n,n) that stay weakly below the diagonal. A “bad” path must touch the line y = x + 1 at some point. Reflect the portion of the path after the first touch across this line. The reflected endpoint is (n − 1, n + 1).

    This creates a bijection between bad paths from (0,0) to (n,n) and all paths from (0,0) to (n−1, n+1). Therefore:

    Good paths = Total paths - Bad paths
           = C(2n, n) - C(2n, n+1)
           = C(2n, n) / (n + 1)
           = Catalan(n)

    General Reflection Formula

    For paths from (0,0) to (a,b) with a ≥ b that must stay strictly below y = x + c, the bad paths (touching y = x + c) biject with all paths to the reflected point. After reflection across y = x + c:

    (a, b) reflects to (b − c, a + c) 

    // Count paths from (0,0) to (a,b) staying below y = x + c
// where a >= b (more right steps than up steps)
long long count_restricted_paths(int a, int b, int c) {
    // Total paths
    long long total = comb(a + b, b);
    // Reflected endpoint: (b - c, a + c)
    int ra = b - c, rb = a + c;
    if (ra < 0 || rb < 0) return total; // no bad paths
    long long bad = comb(ra + rb, rb);
    return (total - bad + MOD) % MOD;
}
    When to use reflection: The reflection principle works beautifully when the boundary is a straight line (y = x + c). For more complex boundaries, you may need the Cycle Lemma, Lindström–Gessel–Viennot, or full inclusion-exclusion.

    The Ballot Problem

    The ballot problem (Bertrand, 1887): In an election, candidate A receives a votes and candidate B receives b votes, with a > b. If votes are counted in random order, what is the probability that A is strictly ahead throughout the entire counting?

    P(A always strictly ahead) = (a − b) / (a + b)

    This elegant result can be proved using the reflection principle or the Cycle Lemma.

    Proof Sketch via Reflection

    Model the vote counting as a lattice path: +1 for each A vote, −1 for each B vote. We want paths from (0,0) to (a + b, a − b) that remain strictly positive after step 0. By the reflection principle:

    Total paths:  C(a+b, a)
Bad paths (touch or cross 0 after start):
  Reflect across y = 0 at first touch.
  These biject with paths to (a+b, -(a-b)-2)
  ... but the cleaner approach uses the Cycle Lemma.

Via Cycle Lemma: exactly (a-b)/(a+b) fraction of all
  sequences have A strictly ahead at every step.

    Variant: Non-strict Lead

    If we require A to be ahead or tied (never behind), the probability becomes:

    P(A ≥ B throughout) = (a − b + 1) / (a + 1)    [when a ≥ b]

    CP application: The ballot problem appears in problems about random walks, queue waiting times, and stock price models. Whenever you see “one quantity always exceeds another during a sequential process,” think ballot problem.

    Lindström–Gessel–Viennot Lemma

    The LGV Lemma is a powerful tool for counting tuples of non-intersecting lattice paths. Given n source points and n destination points, it expresses the count as a determinant.

    Statement

    Let A1, …, An be source points and B1, …, Bn be destination points on a directed acyclic graph. Define e(Ai, Bj) = number of paths from Ai to Bj. Then the number of n-tuples of vertex-disjoint paths (P1: A1→B1, …, Pn: An→Bn) is:

    det [e(Ai, Bj)]n×n

    (with a sign convention for the “correct” pairing).

    Example: Two Non-Intersecting Paths

    Count pairs of non-intersecting paths where P1 goes from (0,0) to (a,b) and P2 goes from (1,−1) to (a+1, b−1) (both using R and U steps).

    // e(A_i, B_j) = C(dx + dy, dy) where dx, dy are differences
long long e00_ab = comb(a + b, b);
long long e00_a1b1 = comb(a + b, b - 1);     // (0,0) to (a+1,b-1)
long long e1m1_ab = comb(a + b, b + 1);      // (1,-1) to (a,b)
long long e1m1_a1b1 = comb(a + b, b);        // (1,-1) to (a+1,b-1)

// LGV determinant
long long ans = (e00_ab * e1m1_a1b1 - e00_a1b1 * e1m1_ab) % MOD;
ans = (ans + MOD) % MOD;

    Implementation for k Paths

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 1e9 + 7;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

// Precomputed fact[] and inv_fact[]
long long comb(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fact[n] % MOD * inv_fact[r] % MOD * inv_fact[n - r] % MOD;
}

// Compute determinant of matrix M (mod MOD) via Gaussian elimination
long long determinant(vector<vector<long long>>& M) {
    int n = M.size();
    long long det = 1;
    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (M[row][col] != 0) { pivot = row; break; }
        }
        if (pivot == -1) return 0;
        if (pivot != col) {
            swap(M[pivot], M[col]);
            det = (MOD - det) % MOD;
        }
        det = det * M[col][col] % MOD;
        long long inv = power(M[col][col], MOD - 2, MOD);
        for (int row = col + 1; row < n; row++) {
            long long factor = M[row][col] * inv % MOD;
            for (int j = col; j < n; j++) {
                M[row][j] = (M[row][j] - factor * M[col][j] % MOD + MOD) % MOD;
            }
        }
    }
    return det;
}

// Count non-intersecting path tuples using LGV
long long lgv_count(vector<pair<int,int>>& A,
                    vector<pair<int,int>>& B) {
    int k = A.size();
    vector<vector<long long>> M(k, vector<long long>(k));
    for (int i = 0; i < k; i++)
        for (int j = 0; j < k; j++) {
            int dx = B[j].first - A[i].first;
            int dy = B[j].second - A[i].second;
            M[i][j] = comb(dx + dy, dy);
        }
    return determinant(M);
}
    LGV power: The LGV lemma reduces a seemingly intractable non-intersecting path problem to a determinant computation. For k paths, it’s O(k3) after building the matrix. This is a favourite technique in hard CP problems.

    CP Applications

    Grid Path Counting

    The most direct application: count shortest paths on a grid from source to destination, possibly avoiding obstacles. Use C(m+n, n) for unrestricted, DP for obstacles, reflection for boundary constraints.

    Parenthesis Sequences

    Counting valid parenthesisations = counting Dyck paths = Catalan numbers. If the sequence must stay above −k (allowing up to k unmatched closing brackets), use the reflection principle with the appropriate boundary.

    Non-Crossing Structures

    Problems about non-crossing partitions, non-intersecting chords, or non-overlapping intervals often reduce to non-intersecting lattice paths via the LGV lemma.

    Random Walk Problems

    A 1D random walk with +1 and −1 steps is equivalent to a lattice path. The ballot problem gives the probability of staying positive. First-passage times, returns to origin, and maximum displacement all have lattice path interpretations.

    Common mistake: When using the reflection principle, be careful about strict vs. non-strict inequalities. Reflecting across y = x is different from reflecting across y = x + 1. Off-by-one errors here cause wrong answers.

    Practice Problems

    1. Basic grid paths: Count paths from (0,0) to (n,m) using R and U steps. Input: n, m ≤ 106.
    2. Grid with obstacles: Given a grid with k ≤ 200 blocked cells, count paths from (0,0) to (n,m). (Hint: sort obstacles and apply PIE or DP on obstacles.)
    3. Ballot problem: Given a > b, compute (a−b)/(a+b) mod p. (Hint: modular inverse.)
    4. Monotone paths: Count paths from (0,0) to (n,n) that stay below the diagonal. Verify the answer is the n-th Catalan number.
    5. Non-intersecting paths: Given k = 2 source-destination pairs on a grid, count pairs of non-intersecting paths using LGV.
    6. Dyck path variations: Count paths from (0,0) to (2n, 0) with steps (+1,+1) and (+1,−1) that stay above y = −k.
    7. Three non-intersecting paths: Given 3 sources and 3 destinations, compute the 3×3 LGV determinant to count non-intersecting triples.
    8. Staircase walk: Count paths on a staircase-shaped grid (each row has a different width). Model as a restricted lattice path problem.
