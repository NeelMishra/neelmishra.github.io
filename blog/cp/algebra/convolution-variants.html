    Convolution Variants

    July 2026 · 28 min read

    1. Bitwise Convolution — The Unifying Idea

    Throughout this series we have multiplied polynomials: given arrays a and b of length n, compute c where ck = ∑i+j=k ai · bj. That was ordinary (additive) convolution, accelerated by FFT in O(n log n).

    Competitive programming frequently requires a different family of convolutions where the “+” relating i, j, k is replaced by a bitwise operator ⊕ ∈ {AND, OR, XOR}:

    ck = ∑i ⊕ j = k ai · bj

    All three share the same algorithmic skeleton:

    1. Transform both arrays into a “spectral” domain where convolution becomes pointwise multiplication.
    2. Pointwise multiply the transformed arrays.
    3. Inverse-transform the product back to recover c.

    The differences lie only in the choice of transform. Each transform runs in O(n log n) where n = 2m is the universe size, giving an overall O(n log n) algorithm for every bitwise convolution — a dramatic speedup over the naïve O(n2).

    Convention. Arrays are indexed by bitmasks in {0, 1, …, 2m−1}. We write n = 2m. All arithmetic is over a ring (typically Z or Z/pZ).

    2. OR Convolution — Zeta and Möbius on Subsets

    We want ck = ∑i | j = k ai · bj. The key observation is that i | j = k implies both i ⊆ k and j ⊆ k (viewing masks as sets). Define the subset-sum (zeta) transform:

    âS = ∑T ⊆ S aT

    2.1 Proof that Zeta Diagonalises OR Convolution

    We must show ĉS = âS · &bcirc;S where ĉ denotes the zeta transform of c.

    Expand the left side:

    ĉS = ∑K ⊆ S cK = ∑K ⊆ S ∑i | j = K ai bj.

    Since i | j = K and K ⊆ S, we know i ⊆ K ⊆ S and j ⊆ K ⊆ S. Conversely, for any i, j ⊆ S the mask K = i | j satisfies K ⊆ S. So summing over all K ⊆ S with i | j = K is the same as summing over all pairs (i, j) with i ⊆ S and j ⊆ S:

    ĉS = ∑i ⊆ S ai · ∑j ⊆ S bj = âS · &bcirc;S.   ■

    2.2 Fast Zeta & Möbius Transforms

    The naïve zeta transform iterates over all subsets — O(3m). A much faster method processes one bit at a time (the sum-over-subsets DP):

    // OR-convolution zeta transform (subset sum)
// f[mask] becomes sum_{sub ⊆ mask} f[sub]
void zeta_or(vector<long long> &f) {
    int n = f.size();          // must be power of 2
    for (int bit = 1; bit < n; bit <<= 1)
        for (int mask = 0; mask < n; mask++)
            if (mask & bit)
                f[mask] += f[mask ^ bit];
}

// Inverse (Möbius) — subtract instead of add
void mobius_or(vector<long long> &f) {
    int n = f.size();
    for (int bit = 1; bit < n; bit <<= 1)
        for (int mask = 0; mask < n; mask++)
            if (mask & bit)
                f[mask] -= f[mask ^ bit];
}

    Complexity. Each of m passes processes n elements, giving O(n log n) total (here log n = m).

    Correctness of the inverse. Denote the zeta transform as a matrix Z with Z[S][T] = [T ⊆ S]. The Möbius matrix M satisfies M[S][T] = (−1)|S\T| [T ⊆ S], and one verifies MZ = I by inclusion–exclusion. The iterative code above implements M by reversing every additive step with a subtraction.

    2.3 Full OR Convolution

    vector<long long> conv_or(vector<long long> a, vector<long long> b) {
    int n = a.size();
    zeta_or(a);
    zeta_or(b);
    vector<long long> c(n);
    for (int i = 0; i < n; i++) c[i] = a[i] * b[i];
    mobius_or(c);
    return c;
}

    3. AND Convolution — Superset Sums

    Now ck = ∑i & j = k ai · bj. Dually to OR, if i & j = k then k ⊆ i and k ⊆ j. Define the superset-sum transform:

    âS = ∑T ⊇ S aT

    3.1 Proof of Diagonalisation

    Identical reasoning to OR with subset/superset roles swapped. Expanding ĉS = ∑K ⊇ S cK and noting i & j = K ⊇ S forces i ⊇ S, j ⊇ S, we obtain:

    ĉS = ∑i ⊇ S ai · ∑j ⊇ S bj = âS · &bcirc;S.   ■

    3.2 Implementation

    // AND-convolution zeta transform (superset sum)
void zeta_and(vector<long long> &f) {
    int n = f.size();
    for (int bit = 1; bit < n; bit <<= 1)
        for (int mask = 0; mask < n; mask++)
            if (!(mask & bit))           // mask does NOT have this bit
                f[mask] += f[mask | bit]; // add the superset
}

// Inverse (Möbius for superset sums)
void mobius_and(vector<long long> &f) {
    int n = f.size();
    for (int bit = 1; bit < n; bit <<= 1)
        for (int mask = 0; mask < n; mask++)
            if (!(mask & bit))
                f[mask] -= f[mask | bit];
}

vector<long long> conv_and(vector<long long> a, vector<long long> b) {
    int n = a.size();
    zeta_and(a);
    zeta_and(b);
    vector<long long> c(n);
    for (int i = 0; i < n; i++) c[i] = a[i] * b[i];
    mobius_and(c);
    return c;
}

    Complexity. Exactly the same O(n log n) as OR convolution. The only structural change is which element gets added to which.

    4. XOR Convolution — Walsh–Hadamard Transform

    We want ck = ∑i ⊕ j = k ai · bj where ⊕ is bitwise XOR. Unlike OR/AND, the lattice structure of subsets does not help; we need a different transform.

    4.1 Deriving the Transform

    For each bitmask S, define:

    âS = ∑T (−1)popcount(S & T) aT

    This is the Walsh–Hadamard Transform (WHT). The kernel (−1)|S∩T| is a character of the group (Z/2Z)m.

    Claim: ĉS = âS · &bcirc;S.

    Proof. Expand the right side:

    âS · &bcirc;S = ∑i (−1)|S∩i| ai · ∑j (−1)|S∩j| bj = ∑i,j (−1)|S∩i| + |S∩j| ai bj.

    The crucial identity is |S ∩ i| + |S ∩ j| ≡ |S ∩ (i ⊕ j)| (mod 2), which follows because for every bit position, the parity of the sum of the two indicator values equals the XOR of those values ANDed with S. Therefore:

    âS · &bcirc;S = ∑k (−1)|S∩k| ∑i ⊕ j = k ai bj = ∑k (−1)|S∩k| ck = ĉS.   ■

    4.2 The Inverse Transform

    Since the WHT matrix H satisfies H2 = nI (each entry ±1, H is symmetric), the inverse is simply H/n: apply the same transform and divide every entry by n.

    4.3 Fast WHT (Butterfly)

    The transform admits a radix-2 butterfly decomposition, identical in structure to the Cooley–Tukey FFT. At each level, pairs of elements at distance bit are combined with a sum/difference:

    // Walsh-Hadamard Transform (in-place)
// inverse = false  =>  forward transform
// inverse = true   =>  inverse (divides by n)
void wht(vector<long long> &f, bool inverse) {
    int n = f.size();
    for (int len = 1; len < n; len <<= 1) {
        for (int i = 0; i < n; i += len << 1) {
            for (int j = 0; j < len; j++) {
                long long u = f[i + j];
                long long v = f[i + j + len];
                f[i + j]       = u + v;   // "even" part
                f[i + j + len] = u - v;   // "odd"  part
            }
        }
    }
    if (inverse)
        for (int i = 0; i < n; i++) f[i] /= n;
}

vector<long long> conv_xor(vector<long long> a, vector<long long> b) {
    int n = a.size();
    wht(a, false);
    wht(b, false);
    vector<long long> c(n);
    for (int i = 0; i < n; i++) c[i] = a[i] * b[i];
    wht(c, true);
    return c;
}

    Complexity. O(n log n) with very small constant — no trig, no modular division until the final step. In practice WHT is ~3× faster than NTT for the same n.

    Modular WHT. When working modulo a prime p, replace the final division by n with multiplication by the modular inverse n−1 mod p. Ensure p is odd so the inverse exists.

    5. Subset Convolution

    Subset convolution is arguably the most elegant — and the most useful — variant. Given functions a, b on subsets of {1,…,m}, compute:

    cS = ∑T ⊆ S aT · bS \ T

    Unlike OR convolution, the two summands must partition S: T and S \ T are disjoint and their union is S. This makes the problem substantially harder — naïve enumeration costs O(3m).

    5.1 Ranked Representation

    The trick is to stratify by popcount. Define ranked versions of a and b:

    a(k)[S] = a[S] if popcount(S) = k, else 0.

    Then a = ∑k a(k). Because T and S \ T must be disjoint:

    popcount(T) + popcount(S \ T) = popcount(S).

    So the subset convolution equals:

    c(s) = ∑k=0s a(k) *OR b(s−k)

    where *OR denotes OR convolution. Each OR convolution costs O(n log n), and we do O(m2) of them (over all s, k pairs), giving a total of O(m2 · 2m).

    5.2 Why This Works — Full Proof

    Fix a target set S with |S| = s. In the ranked OR convolution:

    (a(k) *OR b(s−k))[S] = ∑i | j = S a(k)[i] · b(s−k)[j].

    The non-zero terms require popcount(i) = k and popcount(j) = s − k. Combined with i | j = S and i, j ⊆ S:

    • popcount(i) + popcount(j) = k + (s−k) = s = popcount(S).
    • Since i | j = S and popcount(i) + popcount(j) = popcount(i | j) + popcount(i & j), we get popcount(i & j) = 0, meaning i ∩ j = ∅.

    Therefore j = S \ i, and each OR convolution recovers exactly the disjoint partition terms.   ■

    5.3 Complete Implementation

    // Subset convolution in O(m^2 * 2^m)
// a, b: arrays of length 2^m indexed by bitmask
vector<long long> subset_conv(vector<long long> a, vector<long long> b, int m) {
    int n = 1 << m;

    // Step 1: Build ranked arrays  ranked_a[k][mask], ranked_b[k][mask]
    vector<vector<long long>> ra(m + 1, vector<long long>(n, 0));
    vector<vector<long long>> rb(m + 1, vector<long long>(n, 0));
    for (int mask = 0; mask < n; mask++) {
        int pc = __builtin_popcount(mask);
        ra[pc][mask] = a[mask];
        rb[pc][mask] = b[mask];
    }

    // Step 2: Zeta (OR) transform each rank
    for (int k = 0; k <= m; k++) {
        zeta_or(ra[k]);
        zeta_or(rb[k]);
    }

    // Step 3: Pointwise multiply with rank convolution
    // rc[s][mask] = sum_{k=0}^{s} ra[k][mask] * rb[s-k][mask]
    vector<vector<long long>> rc(m + 1, vector<long long>(n, 0));
    for (int s = 0; s <= m; s++)
        for (int k = 0; k <= s; k++)
            for (int mask = 0; mask < n; mask++)
                rc[s][mask] += ra[k][mask] * rb[s - k][mask];

    // Step 4: Möbius (inverse OR) transform each rank
    for (int s = 0; s <= m; s++)
        mobius_or(rc[s]);

    // Step 5: Extract results — c[S] = rc[popcount(S)][S]
    vector<long long> c(n);
    for (int mask = 0; mask < n; mask++)
        c[mask] = rc[__builtin_popcount(mask)][mask];

    return c;
}

    Complexity analysis. Steps 2 & 4 each do (m+1) transforms of size n: O(m · n · m) = O(m2 n). Step 3 loops over O(m2) pairs and n masks: O(m2 n). Total: O(m2 · 2m).

    Typical constraint. m ≤ 20 gives n = 220 ≈ 106 and the algorithm performs roughly 202 · 106 = 4 × 108 additions — tight but feasible in ~1.5 s with simple operations.

    6. Möbius Inversion on the Subset Lattice

    All OR/AND transforms are instances of Möbius inversion on the Boolean lattice 2[m]. This section formalises the framework.

    6.1 Zeta and Möbius Functions

    Let (P, ≤) be a locally finite poset. The zeta function is ζ(x, y) = 1 if x ≤ y, else 0. The Möbius function μ is its inverse in the incidence algebra:

    ∑x ≤ z ≤ y μ(x, z) = [x = y].

    For the subset lattice with ≤ = ⊆, the Möbius function is:

    μ(T, S) = (−1)|S \ T|   if T ⊆ S.

    6.2 Deriving OR/AND from the General Framework

    OR transform (zeta): âS = ∑T ≤ S ζ(T, S) aT = ∑T ⊆ S aT.

    OR inverse (Möbius): aS = ∑T ⊆ S μ(T, S) âT = ∑T ⊆ S (−1)|S\T| âT.

    This is precisely inclusion–exclusion. The AND case uses the dual poset (⊇).

    6.3 Application: Counting Colourings

    A classic application: given a graph on m vertices, count the number of ways to partition the vertex set into independent sets. For each subset S, let f(S) = 1 if S is independent, 0 otherwise. Then the answer for using exactly k colours is the coefficient of xk in the subset-convolution power f *subset f *subset … *subset f (k times), evaluated at S = [m].

    With the ranked OR approach, each multiplication costs O(m2 2m) and we need k ≤ m multiplications, giving O(m3 2m) overall.

    7. Unified C++ Template

    Here is a production-ready template that implements all four convolutions in a single header.

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;

// ---------- OR ----------
void zeta_or(vector<ll> &f) {
    int n = f.size();
    for (int b = 1; b < n; b <<= 1)
        for (int i = 0; i < n; i++)
            if (i & b) f[i] += f[i ^ b];
}
void mobius_or(vector<ll> &f) {
    int n = f.size();
    for (int b = 1; b < n; b <<= 1)
        for (int i = 0; i < n; i++)
            if (i & b) f[i] -= f[i ^ b];
}

// ---------- AND ----------
void zeta_and(vector<ll> &f) {
    int n = f.size();
    for (int b = 1; b < n; b <<= 1)
        for (int i = 0; i < n; i++)
            if (!(i & b)) f[i] += f[i | b];
}
void mobius_and(vector<ll> &f) {
    int n = f.size();
    for (int b = 1; b < n; b <<= 1)
        for (int i = 0; i < n; i++)
            if (!(i & b)) f[i] -= f[i | b];
}

// ---------- XOR (Walsh-Hadamard) ----------
void wht(vector<ll> &f, bool inv) {
    int n = f.size();
    for (int len = 1; len < n; len <<= 1)
        for (int i = 0; i < n; i += len << 1)
            for (int j = 0; j < len; j++) {
                ll u = f[i+j], v = f[i+j+len];
                f[i+j] = u + v;
                f[i+j+len] = u - v;
            }
    if (inv) for (auto &x : f) x /= n;
}

// ---------- Generic convolution ----------
enum ConvType { OR, AND, XOR };

vector<ll> bitwise_conv(vector<ll> a, vector<ll> b, ConvType type) {
    int n = a.size();
    auto fwd = [&](vector<ll> &f) {
        if (type == OR)  zeta_or(f);
        if (type == AND) zeta_and(f);
        if (type == XOR) wht(f, false);
    };
    auto inv = [&](vector<ll> &f) {
        if (type == OR)  mobius_or(f);
        if (type == AND) mobius_and(f);
        if (type == XOR) wht(f, true);
    };
    fwd(a); fwd(b);
    vector<ll> c(n);
    for (int i = 0; i < n; i++) c[i] = a[i] * b[i];
    inv(c);
    return c;
}

// ---------- Subset convolution ----------
vector<ll> subset_conv(vector<ll> &a, vector<ll> &b, int m) {
    int n = 1 << m;
    vector<vector<ll>> ra(m+1, vector<ll>(n)), rb(m+1, vector<ll>(n));
    for (int i = 0; i < n; i++) {
        int pc = __builtin_popcount(i);
        ra[pc][i] = a[i];
        rb[pc][i] = b[i];
    }
    for (int k = 0; k <= m; k++) { zeta_or(ra[k]); zeta_or(rb[k]); }
    vector<vector<ll>> rc(m+1, vector<ll>(n));
    for (int s = 0; s <= m; s++)
        for (int k = 0; k <= s; k++)
            for (int i = 0; i < n; i++)
                rc[s][i] += ra[k][i] * rb[s-k][i];
    for (int s = 0; s <= m; s++) mobius_or(rc[s]);
    vector<ll> c(n);
    for (int i = 0; i < n; i++) c[i] = rc[__builtin_popcount(i)][i];
    return c;
}

    8. Comparison & Choosing the Right Variant

    OR Convolution

    Condition: ck = ∑i|j=k aibj

    Transform: Subset sums (zeta)

    Inverse: Inclusion–exclusion (Möbius)

    Time: O(n log n)

    AND Convolution

    Condition: ck = ∑i&j=k aibj

    Transform: Superset sums

    Inverse: Superset Möbius

    Time: O(n log n)

    XOR Convolution

    Condition: ck = ∑i⊕j=k aibj

    Transform: Walsh–Hadamard

    Inverse: Same transform, divide by n

    Time: O(n log n)

    Subset Convolution

    Condition: cS = ∑T⊆S aTbS\T

    Transform: Ranked OR + popcount trick

    Inverse: Ranked Möbius

    Time: O(m² 2m)

    Rule of thumb. If the problem says “subsets must be disjoint” or “partition into parts”, use subset convolution. If the combination rule is purely bitwise (OR, AND, XOR), use the corresponding transform directly.
