    Formal Power Series

    July 2026 · 30 min read

    1. What Are Formal Power Series?

    A formal power series (FPS) over a ring R is an infinite sequence of coefficients (a0, a1, a2, …) written as f(x) = ∑i≥0 ai xi. Unlike polynomials, an FPS has infinitely many terms. Unlike analytic functions, convergence is irrelevant — we never "evaluate" an FPS at a number. All operations are purely algebraic.

    The ring of formal power series R[[x]] inherits addition and multiplication from R: the product of f = ∑ aixi and g = ∑ bixi is h = ∑ ckxk where ck = ∑i+j=k aibj.

    In competitive programming, we work with FPS truncated to n terms (modulo xn), turning all operations into polynomial operations of degree < n. The challenge is performing these operations in O(n log n) time.

    The four fundamental operations we will derive are:

    1. Inverse: Given f, compute g such that f · g ≡ 1 (mod xn).
    2. Logarithm: Given f with f(0) = 1, compute ln(f) mod xn.
    3. Exponential: Given f with f(0) = 0, compute exp(f) mod xn.
    4. Square root: Given f with f(0) = 1, compute √f mod xn.
    Prerequisite: This post assumes you are comfortable with NTT-based polynomial multiplication (see Post 5). All operations are performed modulo 998244353.

    2. FPS Inverse via Newton's Method

    We want to find g(x) such that f(x) · g(x) ≡ 1 (mod xn). This exists if and only if f(0) ≠ 0 (i.e., a0 is invertible in R).

    Derivation of Newton's iteration

    We seek the root of the equation h(g) = 1/g − f = 0, or equivalently h(g) = f · g − 1 = 0. Newton's method for power series lifting says: if gk satisfies f · gk ≡ 1 (mod x2k), then:

    gk+1 = gk − h(gk) / h'(gk)

    For h(g) = f · g − 1, we have h'(g) = f. So:

    gk+1 = gk − (f · gk − 1) / f = gk − gk + 1/f

    This is circular! Instead, use the formulation h(g) = 1/g − f, so h'(g) = −1/g2:

    gk+1 = gk − (1/gk − f) / (−1/gk2) = gk + gk2(1/gk − f) = gk(2 − f · gk)

    This is the Newton's iteration for FPS inverse:

    gk+1 = gk · (2 − f · gk)  mod x2k+1

    Proof of quadratic convergence

    Theorem. If f · gk ≡ 1 (mod x2k), then f · gk+1 ≡ 1 (mod x2k+1).

    Proof. Let ek = 1 − f · gk. By assumption, ek ≡ 0 (mod x2k), so ek2 ≡ 0 (mod x2k+1). Now:

    f · gk+1 = f · gk · (2 − f · gk) = (1 − ek)(1 + ek) = 1 − ek2 ≡ 1  (mod x2k+1) ■

    The number of correct terms doubles at each step (quadratic convergence), so after O(log n) iterations starting from g0 = a0−1, we have n correct terms.

    Complexity analysis

    At iteration k, we work with polynomials of degree < 2k+1, so each step costs O(2k log 2k) for two NTT multiplications. Summing over all iterations:

    T(n) = ∑k=0log n O(2k · k) = O(n log n)

    The geometric growth means the last iteration dominates, giving overall O(n log n).

    Implementation

    const int MOD = 998244353, G = 3;

long long power(long long a, long long b, long long m = MOD) {
    long long r = 1; a %= m;
    for (; b > 0; b >>= 1) {
        if (b & 1) r = r * a % m;
        a = a * a % m;
    }
    return r;
}

void ntt(vector<long long> &a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        long long w = inv ? power(G, MOD - 1 - (MOD - 1) / len)
                         : power(G, (MOD - 1) / len);
        for (int i = 0; i < n; i += len) {
            long long wn = 1;
            for (int j = 0; j < len / 2; j++) {
                long long u = a[i+j], v = a[i+j+len/2] * wn % MOD;
                a[i+j] = (u + v) % MOD;
                a[i+j+len/2] = (u - v + MOD) % MOD;
                wn = wn * w % MOD;
            }
        }
    }
    if (inv) {
        long long ni = power(n, MOD - 2);
        for (auto &x : a) x = x * ni % MOD;
    }
}

vector<long long> poly_mul(vector<long long> a, vector<long long> b) {
    int res = a.size() + b.size() - 1, n = 1;
    while (n < res) n <<= 1;
    a.resize(n); b.resize(n);
    ntt(a, false); ntt(b, false);
    for (int i = 0; i < n; i++) a[i] = a[i] * b[i] % MOD;
    ntt(a, true);
    a.resize(res);
    return a;
}

vector<long long> fps_inv(vector<long long> f, int n) {
    assert(f[0] != 0);
    vector<long long> g = {power(f[0], MOD - 2)};

    for (int len = 2; len < 2 * n; len <<= 1) {
        int cur = min(len, (int)f.size());
        vector<long long> tmp(f.begin(), f.begin() + cur);
        tmp.resize(len); g.resize(len);

        int sz = 1;
        while (sz < 2 * len) sz <<= 1;
        tmp.resize(sz); g.resize(sz);
        ntt(tmp, false); ntt(g, false);
        for (int i = 0; i < sz; i++)
            g[i] = g[i] * (2 - tmp[i] * g[i] % MOD + MOD) % MOD;
        ntt(g, true);
        g.resize(len);
    }
    g.resize(n);
    return g;
}

    3. FPS Logarithm

    Given f(x) with f(0) = 1, compute ln(f(x)) mod xn.

    Derivation

    Formally, the logarithm of a power series is defined by:

    ln(f) = ∫ f'/f · dx

    where f' is the formal derivative, f'/f is computed via FPS inverse, and ∫ denotes formal integration (integrating term by term: ∫ ∑ akxk dx = ∑ ak/(k+1) · xk+1).

    Why does this work? In calculus, (ln f)' = f'/f. The same identity holds formally in R[[x]] when f(0) = 1. We can also verify by checking that exp(ln(f)) = f (which we will derive in the next section).

    Algorithm:

    1. Compute f'(x) — the formal derivative (O(n) time).
    2. Compute 1/f(x) mod xn — using fps_inv (O(n log n) time).
    3. Multiply f' · (1/f) mod xn — using NTT (O(n log n) time).
    4. Integrate the result — formal integration (O(n) time).

    Total: O(n log n).

    Implementation

    vector<long long> derivative(vector<long long> f) {
    int n = f.size();
    if (n <= 1) return {0};
    vector<long long> g(n - 1);
    for (int i = 1; i < n; i++)
        g[i - 1] = f[i] * i % MOD;
    return g;
}

vector<long long> integrate(vector<long long> f) {
    int n = f.size();
    vector<long long> g(n + 1, 0);
    // Precompute modular inverses of 1..n
    vector<long long> inv(n + 1);
    inv[1] = 1;
    for (int i = 2; i <= n; i++)
        inv[i] = (MOD - MOD / i) * inv[MOD % i] % MOD;
    for (int i = 0; i < n; i++)
        g[i + 1] = f[i] * inv[i + 1] % MOD;
    return g;
}

vector<long long> fps_log(vector<long long> f, int n) {
    assert(f[0] == 1);
    // ln(f) = integral( f' / f )
    auto df = derivative(f);
    auto inv_f = fps_inv(f, n);
    auto quotient = poly_mul(df, inv_f);
    quotient.resize(n - 1);
    auto result = integrate(quotient);
    result.resize(n);
    return result;
}
    Note on modular inverses: The integration step requires dividing by 1, 2, …, n. We precompute these modular inverses in O(n) using the recurrence inv[i] = −(p/i) · inv[p mod i] mod p.

    4. FPS Exponential via Newton's Method

    Given f(x) with f(0) = 0, compute g(x) = exp(f(x)) mod xn. That is, g satisfies ln(g) = f.

    Derivation

    We seek the root of h(g) = ln(g) − f = 0. Apply Newton's method for power series lifting:

    gk+1 = gk − h(gk) / h'(gk)

    Since h(g) = ln(g) − f, we have h'(g) = 1/g (the formal derivative of ln(g) with respect to g). Therefore:

    gk+1 = gk − (ln(gk) − f) · gk = gk · (1 − ln(gk) + f)

    gk+1 = gk · (1 + f − ln(gk))  mod x2k+1

    Proof of quadratic convergence

    Theorem. If ln(gk) ≡ f (mod x2k), then ln(gk+1) ≡ f (mod x2k+1).

    Proof. Let ek = f − ln(gk). By assumption, ek ≡ 0 (mod x2k). Then gk+1 = gk · (1 + ek). Taking logarithms:

    ln(gk+1) = ln(gk) + ln(1 + ek) = ln(gk) + ek − ek2/2 + …

    Since ek ≡ 0 (mod x2k), all terms from ek2 onward vanish modulo x2k+1. So:

    ln(gk+1) ≡ ln(gk) + ek = ln(gk) + f − ln(gk) = f  (mod x2k+1) ■

    Complexity

    Each iteration requires: one fps_log (O(len log len)), one polynomial multiplication (O(len log len)). The recurrence T(n) = T(n/2) + O(n log n) solves to O(n log n) (again, the last step dominates).

    Implementation

    vector<long long> fps_exp(vector<long long> f, int n) {
    assert(f.empty() || f[0] == 0);
    f.resize(n);
    vector<long long> g = {1};

    for (int len = 2; len < 2 * n; len <<= 1) {
        g.resize(len);
        // Compute ln(g) mod x^len
        auto lng = fps_log(g, len);
        // g = g * (1 + f - ln(g)) mod x^len
        vector<long long> h(len);
        h[0] = 1;
        for (int i = 0; i < len && i < n; i++)
            h[i] = (h[i] + f[i]) % MOD;
        for (int i = 0; i < len; i++)
            h[i] = (h[i] - lng[i] + MOD) % MOD;
        g = poly_mul(g, h);
        g.resize(len);
    }
    g.resize(n);
    return g;
}
    Common pitfall: The fps_exp function calls fps_log internally, which calls fps_inv. Make sure your fps_inv handles the case where g is growing in size correctly. Off-by-one errors in the truncation length are the #1 source of bugs.

    5. FPS Square Root

    Given f(x) with f(0) = 1, compute g(x) = √f(x) mod xn, i.e., g2 ≡ f (mod xn).

    Derivation via Newton's method

    We seek the root of h(g) = g2 − f = 0. Newton's iteration gives:

    gk+1 = gk − (gk2 − f) / (2gk) = (gk + f/gk) / 2 = (gk + f · gk−1) / 2

    gk+1 = (gk + f · gk−1) / 2  mod x2k+1

    Proof of convergence. Let ek = gk2 − f. If ek ≡ 0 (mod x2k), then:

    gk+12 = ((gk + f · gk−1) / 2)2 = (gk2 + 2f + f2/gk2) / 4

    gk+12 − f = (gk2 − 2f + f2/gk2) / 4 = (gk2 − f)2 / (4gk2) = ek2 / (4gk2)

    Since ek ≡ 0 (mod x2k), ek2 ≡ 0 (mod x2k+1), confirming quadratic convergence. ■

    Implementation

    vector<long long> fps_sqrt(vector<long long> f, int n) {
    assert(f[0] == 1); // general case requires Tonelli-Shanks
    f.resize(n);
    long long inv2 = power(2, MOD - 2);
    vector<long long> g = {1};

    for (int len = 2; len < 2 * n; len <<= 1) {
        g.resize(len);
        auto ig = fps_inv(g, len);
        // Extract first 'len' coefficients of f
        vector<long long> ftrunc(f.begin(),
            f.begin() + min(len, (int)f.size()));
        ftrunc.resize(len);
        auto fig = poly_mul(ftrunc, ig);
        fig.resize(len);
        // g = (g + f * g^{-1}) / 2
        for (int i = 0; i < len; i++)
            g[i] = (g[i] + fig[i]) % MOD * inv2 % MOD;
    }
    g.resize(n);
    return g;
}

    Complexity: Each iteration calls fps_inv (O(len log len)) and one multiplication (O(len log len)). Total: T(n) = T(n/2) + O(n log n) = O(n log n).

    When f(0) ≠ 1

    If f(0) = c ≠ 1, we need to check whether c is a quadratic residue modulo p. If it is, compute its square root using the Tonelli-Shanks algorithm, then factor out: √f = √c · √(f/c).

    6. Putting It All Together

    Here is a summary of the FPS operations, their requirements, and their complexities:

    Operation Requirement Formula Complexity
    Inverse f−1f(0) ≠ 0g ← g(2 − fg)O(n log n)
    Logarithm ln(f)f(0) = 1∫ f'/fO(n log n)
    Exponential exp(f)f(0) = 0g ← g(1 + f − ln g)O(n log n)
    Square root √ff(0) = 1g ← (g + f/g)/2O(n log n)

    Dependency graph

    The operations build on each other:

    • Inverse is the base — it uses only NTT.
    • Log uses inverse + NTT + derivative/integrate.
    • Exp uses log + NTT.
    • Sqrt uses inverse + NTT (or alternatively, exp + log: √f = exp(ln(f)/2)).
    Alternative for sqrt: Since √f = exp(ln(f)/2), you can implement sqrt without a separate Newton's iteration — just use fps_log and fps_exp. This is fewer lines of code but slightly slower due to the overhead of exp calling log internally.

    Complete example: computing exp(f) mod xn

    Suppose f(x) = x + x2/2 + x3/3 + … = −ln(1 − x). Then exp(f) = 1/(1 − x) = 1 + x + x2 + …, and our algorithm should produce [1, 1, 1, …, 1]. This is a useful sanity check.

    int main() {
    int n = 8;
    // f = -ln(1 - x) = x + x^2/2 + x^3/3 + ...
    vector<long long> f(n, 0);
    for (int i = 1; i < n; i++)
        f[i] = power(i, MOD - 2); // f[i] = 1/i mod p

    auto g = fps_exp(f, n);
    // Expected: [1, 1, 1, 1, 1, 1, 1, 1]
    for (int i = 0; i < n; i++)
        printf("%lld ", g[i]);
    printf("\n");
    return 0;
}

    7. Applications in Competitive Programming

    FPS operations unlock a wide range of combinatorial identities and generating function manipulations:

    Counting labelled structures

    The exponential generating function (EGF) for labelled rooted trees is T(x) = ∑ nn−1 xn/n!. By the Lagrange inversion formula, T satisfies T = x · exp(T). Using fps_exp, we can compute the first n terms of T in O(n log n).

    Partition function

    The generating function for integer partitions is ∏k=1∞ 1/(1 − xk). Taking the logarithm: ln(∏ 1/(1−xk)) = −∑ ln(1 − xk) = ∑k ∑j≥1 xjk/j. This inner sum can be computed in O(n log n) via a harmonic-sum sieve. Then exponentiate to get the partition function.

    Polynomial exponentiation

    To compute f(x)k mod xn for large k, use: fk = exp(k · ln(f)). This requires f(0) = 1 (factor out xv if needed). Time: O(n log n), regardless of k.

    Template tip: In competitive programming, maintain a single FPS library with ntt, poly_mul, fps_inv, fps_log, fps_exp, and fps_sqrt. This ~100-line template handles nearly all polynomial problems you will encounter.
