    Polynomial Division

    July 2026 · 22 min read

    1. The Division Algorithm for Polynomials

    Just as integers admit a division algorithm, so do polynomials over a field. Given polynomials A(x) of degree n and B(x) of degree m ≤ n with B ≠ 0, there exist unique polynomials Q(x) (quotient) of degree n − m and R(x) (remainder) of degree < m such that:

    A(x) = B(x) · Q(x) + R(x)

    Theorem (Polynomial Division Algorithm). Let F be a field, and let A, B ∈ F[x] with B ≠ 0. Then there exist unique Q, R ∈ F[x] with deg(R) < deg(B) such that A = BQ + R.

    Proof (Existence). By induction on n = deg(A). If n < m = deg(B), take Q = 0 and R = A. Otherwise, let an and bm be the leading coefficients of A and B. Define:

    A'(x) = A(x) − (an / bm) · xn−m · B(x)

    Then deg(A') < n (the leading terms cancel). By induction, A' = BQ' + R with deg(R) < m. So A = B(Q' + (an/bm)xn−m) + R.

    Proof (Uniqueness). Suppose A = BQ1 + R1 = BQ2 + R2. Then B(Q1 − Q2) = R2 − R1. If Q1 ≠ Q2, the left side has degree ≥ m, but the right side has degree < m, a contradiction. So Q1 = Q2 and R1 = R2. ■

    2. Naive Division (Synthetic Division)

    The constructive proof above directly gives an O(nm) algorithm: repeatedly subtract multiples of B from A to eliminate the leading term.

    // Naive polynomial division: A = B * Q + R
// Returns {Q, R}. All polynomials stored as coefficient vectors (index = degree).
pair<vector<long long>, vector<long long>>
naive_div(vector<long long> A, const vector<long long> &B, long long mod) {
    int n = A.size() - 1; // deg A
    int m = B.size() - 1; // deg B
    if (n < m) return {{0}, A};

    long long inv_bm = power(B[m], mod - 2, mod);
    vector<long long> Q(n - m + 1, 0);

    for (int i = n; i >= m; i--) {
        long long coeff = A[i] % mod * inv_bm % mod;
        Q[i - m] = coeff;
        for (int j = 0; j <= m; j++) {
            A[i - m + j] = (A[i - m + j] - coeff * B[j] % mod + mod) % mod;
        }
    }
    // Remainder is A[0..m-1]
    A.resize(m);
    while (A.size() > 1 && A.back() == 0) A.pop_back();
    return {Q, A};
}

    Complexity: O((n − m + 1) · m) = O(nm) in the worst case. This is adequate for small polynomials but far too slow when n reaches 105.

    3. The Reverse Polynomial Trick

    The key insight for fast polynomial division is the reversal operation. Define the reversal of a polynomial f(x) of degree d as:

    revd(f) = xd · f(1/x)

    Concretely, if f(x) = a0 + a1x + … + adxd, then revd(f) = ad + ad−1x + … + a0xd. It simply reverses the coefficient vector.

    Theorem (Reversal and Division). If A = BQ + R with deg(A) = n, deg(B) = m, deg(Q) = n − m, and deg(R) < m, then:

    revn(A) ≡ revm(B) · revn−m(Q)  (mod xn−m+1)

    Proof. Start with A(x) = B(x) · Q(x) + R(x). Substitute x → 1/x and multiply both sides by xn:

    xn A(1/x) = xn B(1/x) · Q(1/x) + xn R(1/x)

    The left side is revn(A). On the right:

    • xn B(1/x) Q(1/x) = [xm B(1/x)] · [xn−m Q(1/x)] = revm(B) · revn−m(Q)
    • xn R(1/x) = xn−deg(R) · [xdeg(R) R(1/x)] = xn−deg(R) · revdeg(R)(R)

    Since deg(R) < m, the exponent n − deg(R) > n − m, so xn R(1/x) is divisible by xn−m+1. Therefore, modulo xn−m+1, the remainder term vanishes:

    revn(A) ≡ revm(B) · revn−m(Q)  (mod xn−m+1) ■

    Why this matters: The quotient Q has degree n − m, so revn−m(Q) is fully determined by its coefficients of x0 through xn−m. The congruence modulo xn−m+1 retains exactly these coefficients. So we can solve for Q using modular polynomial inversion!

    4. Fast Division via Polynomial Inverse

    From the reversal theorem, we can compute Q in three steps:

    1. Compute revn(A) and revm(B) by reversing the coefficient arrays.
    2. Compute the modular inverse of revm(B) modulo xn−m+1. Call it B*.
    3. Compute revn−m(Q) = revn(A) · B* mod xn−m+1, then reverse to get Q.

    The remainder is then R = A − B · Q.

    Polynomial modular inverse (computing f−1 mod xk) is done via Newton's method, which we will cover in full detail in the next post on Formal Power Series. For now, we state the result: given f with f(0) ≠ 0, we can compute g ≡ f−1 (mod xk) in O(k log k) time using the iteration:

    gi+1 = gi · (2 − f · gi)  mod x2i+1

    starting from g0 = (f(0))−1.

    Total complexity for division:

    • Polynomial inverse: O(n log n)
    • Multiplication revn(A) · B*: O(n log n)
    • Multiplication B · Q: O(n log n)
    • Subtraction A − BQ: O(n)

    Overall: O(n log n).

    5. C++ Implementation

    Below is a complete implementation of fast polynomial division modulo 998244353. It uses NTT for multiplication and Newton's method for the modular inverse.

    const int MOD = 998244353, G = 3;

long long power(long long a, long long b, long long m = MOD) {
    long long r = 1; a %= m;
    for (; b > 0; b >>= 1) {
        if (b & 1) r = r * a % m;
        a = a * a % m;
    }
    return r;
}

void ntt(vector<long long> &a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        long long w = inv ? power(G, MOD - 1 - (MOD - 1) / len)
                         : power(G, (MOD - 1) / len);
        for (int i = 0; i < n; i += len) {
            long long wn = 1;
            for (int j = 0; j < len / 2; j++) {
                long long u = a[i+j], v = a[i+j+len/2] * wn % MOD;
                a[i+j] = (u + v) % MOD;
                a[i+j+len/2] = (u - v + MOD) % MOD;
                wn = wn * w % MOD;
            }
        }
    }
    if (inv) {
        long long ni = power(n, MOD - 2);
        for (auto &x : a) x = x * ni % MOD;
    }
}

vector<long long> poly_mul(vector<long long> a, vector<long long> b) {
    int res = a.size() + b.size() - 1, n = 1;
    while (n < res) n <<= 1;
    a.resize(n); b.resize(n);
    ntt(a, false); ntt(b, false);
    for (int i = 0; i < n; i++) a[i] = a[i] * b[i] % MOD;
    ntt(a, true);
    a.resize(res);
    return a;
}

// Compute f^{-1} mod x^k using Newton's method
vector<long long> poly_inv(vector<long long> f, int k) {
    assert(f[0] != 0);
    vector<long long> g = {power(f[0], MOD - 2)};
    for (int len = 2; len < 2 * k; len <<= 1) {
        // g = g * (2 - f * g) mod x^len
        int cur = min(len, (int)f.size());
        vector<long long> tmp(f.begin(), f.begin() + cur);
        tmp.resize(len); g.resize(len);

        int n = 1;
        while (n < 2 * len) n <<= 1;
        tmp.resize(n); g.resize(n);
        ntt(tmp, false); ntt(g, false);
        for (int i = 0; i < n; i++)
            g[i] = g[i] * (2 - tmp[i] * g[i] % MOD + MOD) % MOD;
        ntt(g, true);
        g.resize(len);
    }
    g.resize(k);
    return g;
}

// Reverse a polynomial (reverse the coefficient vector)
vector<long long> poly_rev(vector<long long> a) {
    reverse(a.begin(), a.end());
    return a;
}

// Polynomial division: A = B * Q + R
// Returns {Q, R}
pair<vector<long long>, vector<long long>>
poly_div(vector<long long> A, vector<long long> B) {
    int n = A.size() - 1; // deg A
    int m = B.size() - 1; // deg B
    if (n < m) return {{0}, A};

    int qlen = n - m + 1;

    // Step 1: Reverse A and B
    auto rA = poly_rev(A);
    auto rB = poly_rev(B);

    // Step 2: Compute inverse of rB mod x^{qlen}
    rB.resize(qlen);
    auto inv_rB = poly_inv(rB, qlen);

    // Step 3: Multiply rA * inv_rB, take mod x^{qlen}
    rA.resize(qlen);
    auto rQ = poly_mul(rA, inv_rB);
    rQ.resize(qlen);

    // Step 4: Reverse to get Q
    auto Q = poly_rev(rQ);

    // Step 5: R = A - B * Q
    auto BQ = poly_mul(B, Q);
    vector<long long> R(m);
    for (int i = 0; i < m; i++)
        R[i] = (A[i] - (i < (int)BQ.size() ? BQ[i] : 0) + MOD) % MOD;
    while (R.size() > 1 && R.back() == 0) R.pop_back();

    return {Q, R};
}

    Correctness verification. After computing Q and R, one should check that B · Q + R = A. In contest settings this check is usually omitted, but it is useful during debugging.

    Degree bookkeeping: The most common bug in polynomial division is off-by-one errors in sizes. Remember: deg(Q) = n − m (so Q has n − m + 1 coefficients), and deg(R) < m (so R has at most m coefficients). Always resize carefully after each step.

    6. Applications and Complexity Summary

    Polynomial division is a building block for several advanced algorithms:

    • Polynomial modular reduction: Computing f(x) mod g(x) is just the remainder from division. Used in minimal polynomial computation and linear recurrence solving.
    • Multipoint evaluation: Evaluating f at points a1, …, an by building a subproduct tree and reducing f modulo each subtree polynomial. Each level involves polynomial division. Total: O(n log2 n).
    • Polynomial GCD: The Euclidean algorithm for polynomials uses repeated division. With fast division, each step is O(n log n), and there are O(n) steps, giving O(n2 log n) total — or O(n log2 n) with the half-GCD algorithm.
    • Berlekamp-Massey + Bostan-Mori: Finding the minimal linear recurrence and then computing its n-th term in O(d log d log n) using polynomial division modulo the characteristic polynomial.

    Naive Division

    • Time: O(n · m)
    • Simple to implement
    • Good for m ≤ 1000

    Fast Division (NTT)

    • Time: O(n log n)
    • Requires polynomial inverse
    • Necessary for n ≥ 104
