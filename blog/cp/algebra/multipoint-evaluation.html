    Multipoint Evaluation

    July 2026 · 25 min read

    1. Problem Statement & Motivation

    Problem. Given a polynomial P(x) of degree n−1 and n evaluation points x0, x1, …, xn−1, compute all values P(x0), P(x1), …, P(xn−1).

    The naïve approach evaluates each point independently via Horner’s method in O(n), giving O(n2) total. We will achieve O(n log2 n) using a technique based on polynomial remainders and a data structure called the subproduct tree.

    Where does this appear? Multipoint evaluation is a building block for fast interpolation, fast polynomial GCD (via the Half-GCD algorithm), and decoding of Reed–Solomon codes. In competitive programming, it enables efficient solutions when a polynomial must be evaluated at many modular residues.

    The Key Identity

    The entire algorithm rests on one simple fact from polynomial algebra:

    P(xi) = P(x) mod (x − xi).

    Proof. By the division algorithm, P(x) = Q(x)(x − xi) + r where r is a constant (degree 0). Substituting x = xi: P(xi) = 0 + r = r. But r = P(x) mod (x − xi). ∎

    This generalizes: if M(x) = ∏i ∈ S (x − xi) for some subset S, then P(x) mod M(x) preserves all evaluations P(xi) for i ∈ S, because reducing mod M “forgets” no information about those evaluation points.

    2. The Subproduct Tree

    The subproduct tree is a complete binary tree built over the evaluation points.

    • Leaves: Leaf i stores the linear polynomial (x − xi).
    • Internal nodes: Each internal node stores the product of its two children’s polynomials.
    • Root: The root stores M(x) = ∏i=0n−1 (x − xi), the full node polynomial.

    Construction

    Build the tree bottom-up. At each level, multiply pairs of adjacent polynomials. Level k contains polynomials of degree 2k, and there are n / 2k of them. The multiplication at level k costs O(2k log 2k) per pair, and there are n / 2k+1 pairs, so the total cost at level k is O(n log 2k) = O(nk). Summing over k = 0, …, log n:

    Total = O(n ∑k=0log n k) = O(n log2 n). 

    // Subproduct tree construction
// tree[k] = vector of polynomials at level k
// tree[0] = leaves: {[-x_0, 1], [-x_1, 1], ...}
struct SubproductTree {
    int n, levels;
    vector<vector<poly>> tree;

    void build(const vector<long long> &pts) {
        n = pts.size();
        levels = 0;
        int sz = 1;
        while (sz < n) { sz *= 2; levels++; }
        levels++;
        tree.resize(levels);

        // Level 0: leaves
        tree[0].resize(sz);
        for (int i = 0; i < n; i++)
            tree[0][i] = {(MOD - pts[i]) % MOD, 1}; // (x - x_i)
        for (int i = n; i < sz; i++)
            tree[0][i] = {1}; // placeholder: polynomial "1"

        // Build upward
        for (int k = 1; k < levels; k++) {
            int cnt = (int)tree[k - 1].size() / 2;
            tree[k].resize(cnt);
            for (int i = 0; i < cnt; i++)
                tree[k][i] = poly_mul(tree[k - 1][2 * i],
                                      tree[k - 1][2 * i + 1]);
        }
    }
};

    3. Top-Down Remainder Evaluation

    With the subproduct tree built, we evaluate P at all points using a top-down traversal. At each node, we reduce the current polynomial modulo the node’s subproduct.

    Algorithm

    function evaluate(P, node):
      if node is leaf i:
        return P mod (x - x_i) // = P(x_i), a constant
      R_left = P mod tree[left_child]
      R_right = P mod tree[right_child]
      evaluate(R_left, left_child)
      evaluate(R_right, right_child)

    Correctness Proof

    Claim: At each node v of the subproduct tree with associated polynomial Mv(x), the remainder P(x) mod Mv(x) preserves all evaluations at points in the subtree of v.

    Proof by induction on tree height.

    Base case. At a leaf with Mv = (x − xi), the remainder is P(xi) by the polynomial remainder theorem.

    Inductive step. Let v have children u, w with Mv = Mu · Mw. For any point xi in u’s subtree, Mu(xi) = 0 (since (x − xi) divides Mu). Write P = Q · Mv + R where R = P mod Mv. Then P(xi) = R(xi), so R preserves evaluations. When we compute R mod Mu, by inductive hypothesis, this preserves evaluations at u’s points. Similarly for w. ∎

    Complexity

    At level k (counting from root = level log n), we perform polynomial divisions where the dividend has degree O(2k) and the divisor has degree O(2k−1). Each such division costs O(2k log 2k) using NTT-based division. There are n/2k divisions at level k. Total:

    ∑k (n/2k) · O(2k · k) = O(n ∑ k) = O(n log2 n). 

    // Multipoint evaluation using subproduct tree
// Returns vector of P(x_0), P(x_1), ..., P(x_{n-1})
vector<long long> multipoint_eval(const poly &P,
                                   const SubproductTree &st) {
    int n = st.n;
    vector<long long> results(n);

    // Recursive top-down evaluation
    // node_poly = P mod (subproduct at this node)
    function<void(int, int, int, const poly&)> solve;
    solve = [&](int level, int idx, int sz, const poly &cur) {
        if (sz == 1) {
            // leaf: cur is a constant = P(x_{idx})
            results[idx] = cur.empty() ? 0 : cur[0] % MOD;
            return;
        }
        int half = sz / 2;
        int child_level = level - 1;

        // Reduce mod left and right subproducts
        poly left_rem = poly_mod(cur, st.tree[child_level][2 * idx]);
        poly right_rem = poly_mod(cur, st.tree[child_level][2 * idx + 1]);

        solve(child_level, 2 * idx, half, left_rem);
        solve(child_level, 2 * idx + 1, half, right_rem);
    };

    // Start from root: reduce P mod root polynomial first
    int top_level = st.levels - 1;
    poly root_rem = poly_mod(P, st.tree[top_level][0]);
    solve(top_level, 0, 1 << top_level, root_rem);

    return results;
}

    4. Worked Example

    Let P(x) = 2x3 + x2 − 3x + 5 and evaluation points x0 = 1, x1 = 2, x2 = 3, x3 = 4.

    Step 1: Build Subproduct Tree

    Level 0 (leaves):

    • M0 = (x − 1)
    • M1 = (x − 2)
    • M2 = (x − 3)
    • M3 = (x − 4)

    Level 1:

    • M01 = (x−1)(x−2) = x2 − 3x + 2
    • M23 = (x−3)(x−4) = x2 − 7x + 12

    Level 2 (root):

    • M = (x−1)(x−2)(x−3)(x−4) = x4 − 10x3 + 35x2 − 50x + 24

    Step 2: Top-Down Remainders

    Root: P mod M. Since deg(P) = 3 < deg(M) = 4, the remainder is P itself.

    Left child: P mod M01 = P mod (x2 − 3x + 2).
    Divide: 2x3 + x2 − 3x + 5 = (2x + 7)(x2 − 3x + 2) + (10x − 9).
    So Rleft = 10x − 9.

    Right child: P mod M23 = P mod (x2 − 7x + 12).
    Divide: 2x3 + x2 − 3x + 5 = (2x + 15)(x2 − 7x + 12) + (88x − 175).
    So Rright = 88x − 175.

    Leaves:

    • P(1) = (10·1 − 9) mod (x−1) = 10 − 9 = 1.   Verification: 2+1−3+5 = 5. Wait — let me recompute.
    Arithmetic check. P(1) = 2(1) + 1 − 3 + 5 = 5. Rleft = 10(1) − 9 = 1. There’s a discrepancy — I made an arithmetic error in the division. Let’s redo: 2x3 + x2 − 3x + 5 divided by x2 − 3x + 2. First term: 2x · (x2−3x+2) = 2x3−6x2+4x. Remainder: 7x2−7x+5. Next: 7(x2−3x+2) = 7x2−21x+14. Remainder: 14x−9. So Rleft = 14x − 9. P(1) = 14−9 = 5 ✓. P(2) = 28−9 = 19 = 2(8)+4−6+5 = 19 ✓.

    This example illustrates that careful polynomial division is critical. The correctness follows automatically from the remainder theorem, but hand computation is error-prone.

    5. Fast Interpolation as the Inverse

    Multipoint evaluation takes (coefficients, points) → values. The inverse operation is interpolation: (points, values) → coefficients. Fast interpolation also runs in O(n log2 n) and reuses the subproduct tree.

    Fast Interpolation Algorithm

    Step 1. Build the subproduct tree from the evaluation points xi.

    Step 2. Compute the derivative of the root polynomial: M′(x) = d/dx [∏(x − xi)].

    Step 3. Use multipoint evaluation to compute M′(xi) for all i. These are the denominators: M′(xi) = ∏j≠i(xi − xj).

    Step 4. Compute the scaled values ci = yi / M′(xi).

    Step 5. Build the interpolating polynomial via divide and conquer:

    function build_interp(lo, hi):
      if lo == hi: return c[lo] // constant polynomial
      mid = (lo + hi) / 2
      L = build_interp(lo, mid)
      R = build_interp(mid + 1, hi)
      return L * M[mid+1..hi] + R * M[lo..mid]

    Here M[lo..hi] = ∏i=lohi(x − xi) is taken from the subproduct tree. The total cost is O(n log2 n).

    Connection to Lagrange’s Formula

    This algorithm is exactly computing P(x) = ∑ ci ∏j≠i(x − xj) = ∑ (yi / M′(xi)) ∏j≠i(x − xj), which is the Lagrange interpolation formula in its “coefficient form.”

    Practical Considerations

    • Memory. The subproduct tree stores O(n log n) coefficients total. For n = 218, this is about 5 × 106 integers — manageable.
    • Constant factor. The O(n log2 n) algorithms have large constants (due to polynomial division via NTT). The naïve O(n2) is faster for n ≤ 1000.
    • Polynomial division. To compute P mod Q, use: P mod Q = P − ⌊P/Q⌋ · Q, where ⌊P/Q⌋ is computed via reverse-polynomial inversion and multiplication.
    • NTT-friendly moduli. The modulus must support NTT (e.g., 998244353 = 119 · 223 + 1). If the problem uses a different modulus, use arbitrary-mod NTT or CRT.
    // Polynomial division and modular reduction
// Returns P mod Q (assumes deg P >= deg Q)
poly poly_mod(const poly &P, const poly &Q) {
    if (P.size() < Q.size()) return P;

    // Reverse polynomials
    poly Pr = P, Qr = Q;
    reverse(Pr.begin(), Pr.end());
    reverse(Qr.begin(), Qr.end());

    int quot_deg = P.size() - Q.size();
    Pr.resize(quot_deg + 1);
    Qr.resize(quot_deg + 1);

    // Quotient = Pr * Qr^{-1} mod x^{quot_deg+1}
    poly Qr_inv = poly_inverse(Qr, quot_deg + 1);
    poly quot = poly_mul(Pr, Qr_inv);
    quot.resize(quot_deg + 1);
    reverse(quot.begin(), quot.end());

    // Remainder = P - quot * Q
    poly product = poly_mul(quot, Q);
    poly rem(Q.size() - 1);
    for (int i = 0; i < (int)rem.size(); i++)
        rem[i] = (P[i] - product[i] % MOD + MOD) % MOD;

    // Remove trailing zeros
    while (rem.size() > 1 && rem.back() == 0)
        rem.pop_back();
    return rem;
}
    Contest tip. Multipoint evaluation and fast interpolation are rarely needed in standard competitive programming due to the high constant factor. However, they appear in “library checker” type problems and are essential building blocks for algorithms like Half-GCD (next post). For contest use, the naïve O(n2) methods or the O(n) consecutive-point trick from the previous post are usually sufficient.
