    Lagrange Interpolation

    July 2026 · 25 min read

    1. The Interpolation Problem

    Given n + 1 distinct points (x0, y0), (x1, y1), …, (xn, yn), there exists a unique polynomial P(x) of degree ≤ n such that P(xi) = yi for all i.

    Uniqueness Proof

    Theorem. The interpolating polynomial is unique.

    Proof. Suppose P and Q are both polynomials of degree ≤ n passing through all n + 1 points. Then D(x) = P(x) − Q(x) is a polynomial of degree ≤ n with at least n + 1 roots (at x0, …, xn). By the fundamental theorem of algebra, a nonzero polynomial of degree ≤ n has at most n roots. Therefore D ≡ 0, and P = Q. ∎

    Why this matters. Uniqueness means any method that produces a degree-≤n polynomial passing through the given points will give the same answer: Lagrange, Newton, Vandermonde — they all yield the identical polynomial.

    2. The Lagrange Interpolation Formula

    The Lagrange basis polynomials are defined as

    Li(x) = ∏j ≠ i (x − xj) / (xi − xj).

    Each Li satisfies Li(xj) = δij (Kronecker delta). The interpolating polynomial is then

    P(x) = ∑i=0n yi · Li(x).

    Proof of Correctness

    We verify: (1) each Li is a polynomial of degree exactly n (it is a product of n linear factors), so P has degree ≤ n; (2) P(xk) = ∑ yi Li(xk) = yk · 1 + ∑i≠k yi · 0 = yk. By uniqueness, this is the interpolating polynomial. ∎

    Direct O(n²) Evaluation

    To evaluate P at a single point t (not one of the xi), compute:

    P(t) = ∑i=0n yi ∏j≠i (t − xj) / (xi − xj).

    Each term requires O(n) work (computing the product over j), and there are n + 1 terms, giving O(n2) total. 

    // Lagrange interpolation: evaluate P(t) given points (x[], y[])
// O(n^2) where n = number of points
long long lagrange_eval(const vector<long long> &x,
                        const vector<long long> &y,
                        long long t, long long mod) {
    int n = x.size();
    long long result = 0;
    for (int i = 0; i < n; i++) {
        long long num = y[i] % mod;
        long long den = 1;
        for (int j = 0; j < n; j++) {
            if (i == j) continue;
            num = num % mod * ((t - x[j]) % mod + mod) % mod;
            den = den % mod * ((x[i] - x[j]) % mod + mod) % mod;
        }
        result = (result + num % mod * mod_pow(den, mod - 2, mod)) % mod;
    }
    return result;
}

    3. Barycentric Lagrange Interpolation

    The barycentric form restructures the Lagrange formula so that, once we precompute certain weights, evaluating at a new point costs only O(n).

    Derivation

    Define the barycentric weights

    wi = 1 / ∏j≠i (xi − xj)

    and the node polynomial ℓ(x) = ∏j=0n (x − xj). Then Li(x) = ℓ(x) · wi / (x − xi), and

    P(x) = ℓ(x) ∑i=0n wi yi / (x − xi).

    Second barycentric form. Since ∑ Li(x) = 1 for any x (interpolating the constant function 1), we have ℓ(x) = 1 / ∑ wi / (x − xi), giving the ratio

    P(x) = ∑i [wi yi / (x − xi)] ⁄ ∑i [wi / (x − xi)].

    Once the weights wi are precomputed (O(n2) or O(n log2 n)), each new evaluation is O(n). 

    // Barycentric Lagrange interpolation
// Precompute weights in O(n^2), then evaluate in O(n)
struct BarycentricInterp {
    int n;
    vector<long long> x, y, w;
    long long mod;

    void precompute() {
        n = x.size();
        w.resize(n);
        for (int i = 0; i < n; i++) {
            w[i] = 1;
            for (int j = 0; j < n; j++) {
                if (i == j) continue;
                w[i] = w[i] % mod * ((x[i] - x[j]) % mod + mod) % mod;
            }
            w[i] = mod_pow(w[i], mod - 2, mod);
        }
    }

    long long eval(long long t) {
        // Check if t is one of the nodes
        for (int i = 0; i < n; i++)
            if (t == x[i]) return y[i];

        long long num = 0, den = 0;
        for (int i = 0; i < n; i++) {
            long long diff_inv = mod_pow((t - x[i] + mod) % mod, mod - 2, mod);
            long long term = w[i] % mod * diff_inv % mod;
            num = (num + term % mod * y[i]) % mod;
            den = (den + term) % mod;
        }
        return num % mod * mod_pow(den, mod - 2, mod) % mod;
    }
};

    4. Interpolation on Consecutive Points

    A particularly important special case: the points are xi = i for i = 0, 1, …, n. This arises constantly in competitive programming (e.g., computing sum of k-th powers).

    Simplified Formula

    When xi = i, the barycentric weight becomes

    wi = 1 / ∏j≠i (i − j) = (−1)n−i / (i! (n−i)!).

    Proof. The denominator is ∏j=0, j≠in (i − j) = (i)(i−1)…(1) · (−1)(−2)…(i−n) = i! · (−1)n−i (n−i)!. ∎

    Substituting into the barycentric formula and evaluating at a point t:

    P(t) = ∑i=0n yi · (−1)n−i / (i!(n−i)!) · ∏j≠i (t − j).

    The product ∏j≠i (t − j) = [∏j=0n (t − j)] / (t − i). Precomputing the full product and prefix/suffix products gives an O(n) evaluation. 

    // Lagrange interpolation on consecutive points 0, 1, ..., n
// Given y[0..n], evaluate P(t) in O(n)
long long interp_consecutive(const vector<long long> &y,
                             long long t, long long mod) {
    int n = y.size() - 1;

    // Precompute factorials and inverse factorials
    vector<long long> fac(n + 1), ifac(n + 1);
    fac[0] = 1;
    for (int i = 1; i <= n; i++)
        fac[i] = fac[i - 1] * i % mod;
    ifac[n] = mod_pow(fac[n], mod - 2, mod);
    for (int i = n - 1; i >= 0; i--)
        ifac[i] = ifac[i + 1] * (i + 1) % mod;

    // Precompute prefix and suffix products of (t - j)
    vector<long long> pre(n + 2, 1), suf(n + 2, 1);
    for (int i = 0; i <= n; i++)
        pre[i + 1] = pre[i] % mod * ((t - i) % mod + mod) % mod;
    for (int i = n; i >= 0; i--)
        suf[i] = suf[i + 1] % mod * ((t - i) % mod + mod) % mod;

    long long result = 0;
    for (int i = 0; i <= n; i++) {
        long long num = pre[i] % mod * suf[i + 1] % mod;
        long long den = ifac[i] % mod * ifac[n - i] % mod;
        if ((n - i) % 2 == 1) den = (mod - den) % mod;
        result = (result + y[i] % mod * num % mod * den) % mod;
    }
    return result;
}

    5. Fast Interpolation: O(n log² n)

    For general (non-consecutive) points, we can reconstruct the polynomial P(x) in coefficient form in O(n log2 n) using divide and conquer.

    Algorithm Overview

    Step 1: Build the subproduct tree. Construct a binary tree where leaf i stores (x − xi), and each internal node stores the product of its children. The root contains ∏(x − xi).

    Step 2: Multi-point evaluation of ∏j≠i(x − xj). Compute the derivative of the node polynomial: ℓ′(x) = ∑i ∏j≠i(x − xj). Evaluate ℓ′ at all xi to get ℓ′(xi) = ∏j≠i(xi − xj) = 1/wi.

    Step 3: Divide and conquer interpolation. With the weights wi = 1/ℓ′(xi), define ci = yi wi. Then P(x) = ∑ ci Li(x). Split the points into two halves and recursively interpolate:

    function interpolate(points, c_values):
      if |points| == 1: return c[0]
      split into left half L, right half R
      P_L = interpolate(L, c_L)
      P_R = interpolate(R, c_R)
      // M_R = prod_{j in R} (x - x_j), M_L = prod_{j in L} (x - x_j)
      return P_L * M_R + P_R * M_L

    Each level of recursion performs O(n log n) work (polynomial multiplications), and there are O(log n) levels, yielding O(n log2 n) total.

    6. Contest Applications

    Application 1: Sum of k-th Powers

    Problem. Compute Sk(n) = ∑i=0n ik mod p.

    Key insight. Sk(n) is a polynomial in n of degree k + 1. So we only need k + 2 points to determine it completely. Evaluate Sk(0) = 0, Sk(1) = 1, Sk(2) = 1 + 2k, …, Sk(k+1). Then use the O(k) consecutive-point interpolation to evaluate Sk(n). 

    // Compute sum_{i=0}^{n} i^k mod p
// O(k log k) for computing powers, O(k) for interpolation
long long sum_of_kth_powers(long long n, int k, long long mod) {
    // Compute y[i] = sum_{j=0}^{i} j^k for i = 0..k+1
    vector<long long> y(k + 2);
    y[0] = 0;
    for (int i = 1; i <= k + 1; i++)
        y[i] = (y[i - 1] + mod_pow(i, k, mod)) % mod;

    // If n <= k+1, we already computed it
    if (n <= k + 1) return y[n];

    // Interpolate at t = n using consecutive points
    return interp_consecutive(y, n % mod, mod);
}

    Application 2: Prefix Sums of Polynomial Functions

    If f(x) is a polynomial of degree d, then ∑i=0n f(i) is a polynomial in n of degree d + 1. Evaluate at d + 2 small points and interpolate to get the answer for arbitrary n in O(d).

    Application 3: Recovering a Hidden Polynomial

    Some interactive problems give you an oracle for P(x) mod p (for unknown P of known degree d). Query d + 1 points, then interpolate to recover P completely.

    Contest tip. Whenever you notice that an answer is a polynomial in the input parameter, compute it for small values and interpolate. This “guess the polynomial” trick solves many problems that seem to require complex closed-form derivations.

    7. Complexity Comparison

    Method
    Naïve Lagrange
    Barycentric (precomputed)
    Consecutive points
    D&C interpolation
    Complexity
    O(n2) per evaluation
    O(n) per evaluation
    O(n) total
    O(n log2 n) for coefficients

    In competitive programming, the consecutive-points O(n) method is by far the most common. The full D&C interpolation is needed when points are not consecutive or when you need the polynomial in coefficient form for further operations.
