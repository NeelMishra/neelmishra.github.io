    Vieta's Formulas

    July 2026 · 20 min read

    1. Statement of Vieta's Formulas

    Let f(x) = anxn + an−1xn−1 + … + a1x + a0 be a polynomial of degree n with roots r1, r2, …, rn (counted with multiplicity, over the algebraic closure). Then:

    ek(r1, …, rn) = (−1)k · an−k / an

    where ek denotes the k-th elementary symmetric polynomial:

    • e1 = r1 + r2 + … + rn = −an−1/an
    • e2 = ∑i<j rirj = an−2/an
    • e3 = ∑i<j<k rirjrk = −an−3/an
    • …
    • en = r1r2…rn = (−1)n a0/an
    Special case (quadratic): For ax² + bx + c = 0 with roots r, s:
    r + s = −b/a,    r·s = c/a.

    2. Proof from the Factored Form

    Since f has roots r1, …, rn and leading coefficient an:

    f(x) = an(x − r1)(x − r2)…(x − rn)

    Expand the product (x − r1)…(x − rn) by choosing either x or −ri from each factor. If we choose −ri from exactly k factors and x from the remaining n−k factors, we get a term (−1)k · xn−k · (product of k roots). Summing over all subsets of size k:

    ∏i=1..n(x − ri) = ∑k=0..n (−1)k ek(r1,…,rn) · xn−k

    Multiplying by an and matching coefficients with f(x) = ∑ ajxj:

    an−k = an · (−1)k · ek

    Rearranging gives ek = (−1)k an−k/an. ▮

    This proof is completely elementary — it is just the distributive law applied to the product of linear factors. The beauty of Vieta's formulas is that they provide exact algebraic relations between coefficients and roots without needing to solve the polynomial.

    3. Constructing a Polynomial from Root Constraints

    A frequent contest pattern is: "Find a monic polynomial of degree n whose roots satisfy certain symmetric conditions." Vieta's formulas let you translate root conditions into coefficient equations.

    3.1 Example: Sum and Product Given

    Problem: Find a quadratic with roots summing to 7 and product equal to 12.

    By Vieta's: −a1/a2 = 7 and a0/a2 = 12. Taking a2 = 1 (monic): f(x) = x² − 7x + 12 = (x−3)(x−4).

    3.2 Example: Constructing from All Root Sums

    Problem: Given e1, e2, …, en, construct the monic polynomial.

    Answer: f(x) = xn − e1xn−1 + e2xn−2 − … + (−1)nen.

    This is used in problems where you are given information about the roots (often as values of symmetric functions) and must reconstruct the polynomial.

    4. C++ Implementation

    4.1 Reconstructing a Polynomial from Its Roots

    #include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long mod(long long x) {
    return ((x % MOD) + MOD) % MOD;
}

// Given roots r[0..n-1], compute the monic polynomial coefficients
// Returns c[0..n] where c[n]=1 and poly = c[n]*x^n + c[n-1]*x^{n-1} + ... + c[0]
vector<long long> poly_from_roots(const vector<long long>& roots) {
    int n = (int)roots.size();
    // Start with polynomial = 1 (constant)
    vector<long long> poly = {1};

    for (int i = 0; i < n; i++) {
        // Multiply poly by (x - roots[i])
        vector<long long> next(poly.size() + 1, 0);
        for (int j = (int)poly.size() - 1; j >= 0; j--) {
            // x * poly[j] contributes to next[j+1]
            next[j + 1] = (next[j + 1] + poly[j]) % MOD;
            // -roots[i] * poly[j] contributes to next[j]
            next[j] = (next[j] - roots[i] % MOD * poly[j] % MOD + MOD * 2) % MOD;
        }
        poly = next;
    }
    return poly;  // poly[k] = coefficient of x^k
}

// Verify Vieta's formulas: extract e_k from coefficients
void verify_vietas(const vector<long long>& poly, const vector<long long>& roots) {
    int n = (int)roots.size();
    // poly[n] = a_n = 1 (monic)
    cout << "Polynomial coefficients (x^0 to x^n): ";
    for (auto c : poly) cout << c << " ";
    cout << "\n";

    // e_1 = -a_{n-1}/a_n = -poly[n-1]
    cout << "e1 (sum of roots) = " << mod(-poly[n - 1]) << "\n";

    // Direct computation of sum of roots
    long long sum = 0;
    for (auto r : roots) sum = (sum + r) % MOD;
    cout << "Direct sum = " << sum << "\n";

    // e_n = (-1)^n * a_0 / a_n = (-1)^n * poly[0]
    long long prod = 1;
    for (auto r : roots) prod = prod % MOD * (r % MOD) % MOD;
    cout << "Product of roots = " << mod(prod) << "\n";

    long long en = poly[0];
    if (n % 2 == 1) en = mod(-en);
    cout << "e_n from coefficients = " << en << "\n";
}

int main() {
    // Roots: 1, 2, 3, 4
    vector<long long> roots = {1, 2, 3, 4};
    auto poly = poly_from_roots(roots);
    verify_vietas(poly, roots);
    // Expected: x^4 - 10x^3 + 35x^2 - 50x + 24
    return 0;
}

    4.2 Computing All Elementary Symmetric Polynomials

    Sometimes we need all ek values explicitly (not just as polynomial coefficients). The following computes them in O(n²) via dynamic programming: 

    // Compute e[0..n] from values v[0..n-1]
// e[k] = sum of all products of k distinct elements from v
vector<long long> elementary_symmetric(const vector<long long>& v) {
    int n = (int)v.size();
    // dp[j] = e_j after processing first i elements
    vector<long long> dp(n + 1, 0);
    dp[0] = 1;

    for (int i = 0; i < n; i++) {
        // Process in reverse to avoid using v[i] twice
        for (int j = min(i + 1, n); j >= 1; j--) {
            dp[j] = (dp[j] + v[i] % MOD * dp[j - 1]) % MOD;
        }
    }
    return dp;
}

    5. Contest Applications

    5.1 Bounding Roots

    Vieta's formulas give immediate constraints on the roots. For a monic polynomial with integer coefficients:

    • The sum of roots is an integer (= −an−1).
    • The product of roots is an integer (= (−1)na0).
    • If the polynomial is over Z and a0 ≠ 0, every rational root divides a0 (Rational Root Theorem, which is a direct consequence of Vieta's).

    This is used in contest problems that ask "how many integer-coefficient polynomials of degree n have roots in a given set?" or "prove that a polynomial has no rational roots."

    5.2 Relating Coefficient Conditions to Root Conditions

    Problem (contest-style): Prove that if all roots of xn + an−1xn−1 + … + a0 are real and positive, then an−1 < 0.

    Proof: By Vieta's, an−1 = −(r1 + … + rn). Since all ri > 0, the sum is positive, so an−1 < 0. ▮

    5.3 Transforming Polynomials

    Given f(x) with roots r1, …, rn, find a polynomial with roots r1², …, rn². The elementary symmetric polynomials of the squared roots can be expressed in terms of those of the original roots using Newton's identities (from the previous post). Vieta's formulas then give the coefficients of the new polynomial.

    6. Worked Contest Examples

    6.1 USAMO-style Problem

    Problem: Let a, b, c be roots of x³ − 6x² + 11x − 6 = 0. Compute a² + b² + c² and a³ + b³ + c³.

    By Vieta's: e1 = 6, e2 = 11, e3 = 6.

    • p1 = e1 = 6
    • p2 = e1² − 2e2 = 36 − 22 = 14   (so a²+b²+c² = 14)
    • p3 = e1³ − 3e1e2 + 3e3 = 216 − 198 + 18 = 36   (so a³+b³+c³ = 36)

    Verification: roots are 1, 2, 3. 1+4+9=14 ✓, 1+8+27=36 ✓.

    6.2 Codeforces-style Problem

    Problem: You are given n integers. Determine if there exists a monic polynomial of degree n with integer coefficients whose roots are exactly those integers.

    Solution: Just check that the integers form a valid multiset. The polynomial is uniquely determined: multiply out ∏(x − ri). The key insight is that by Vieta's formulas, all coefficients are automatically integers if all roots are integers. This is because ek is a sum of products of integers.

    7. Vieta's Formulas Modulo a Prime

    When working in Z/pZ, Vieta's formulas still hold: for f(x) in Z/pZ[x], the elementary symmetric polynomials of the roots (in the splitting field) satisfy the same relations. This is particularly useful in competitive programming where computations are done modulo a large prime.

    However, a subtlety arises: a polynomial of degree n over Z/pZ may have fewer than n roots in Z/pZ (though it always has exactly n roots in the algebraic closure). Vieta's formulas always refer to all roots (including multiplicities) in the splitting field.

    Warning: If you are working over Z/pZ and the polynomial does not split completely (i.e., has irreducible factors of degree > 1), you cannot directly "see" all roots. Nevertheless, the coefficient relationships from Vieta's still hold — they just relate to roots in an extension field.

    In practice, for contest problems, the polynomial usually splits completely (e.g., the roots are given explicitly, or you are asked to construct a polynomial from given root values modulo p), so this subtlety rarely causes issues.

    8. Summary & Connections

    Vieta's Formulas
    Connect coefficients of f to elementary symmetric polynomials of roots. ek = (−1)k an−k/an.
    Newton's Identities
    Connect elementary symmetric polynomials ek to power sums pk = ∑ rik.

    Together, these two tools form a complete pipeline:

    Coefficients ⟹Vieta ek ⟹Newton pk

    This pipeline is reversible: given power sums, Newton's identities recover the ek, and Vieta's formulas then give the polynomial coefficients.

    Key takeaway. Vieta's formulas are the fundamental bridge between a polynomial's coefficients and its roots. They are indispensable in competitive programming for constructing polynomials, verifying root properties, and setting up equations that constrain roots via their symmetric functions.
