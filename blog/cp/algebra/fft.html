    Fast Fourier Transform

    July 2026 · 30 min read

    1 — The Big Picture

    Multiplying two degree-n polynomials takes Оҳ(nВІ) in coefficient form. But in point-value form, multiplication is Оҳ(n)—just multiply corresponding values. The strategy is:

    1. Evaluate both polynomials at N вүҘ 2n + 1 carefully chosen points — O(n log n)
    2. Multiply the point values pairwise — O(n)
    3. Interpolate back to coefficients — O(n log n)

    The “carefully chosen points” are the N-th roots of unity, and the conversion is the Discrete Fourier Transform (DFT). The Fast Fourier Transform (FFT) computes the DFT in O(n log n) instead of O(nВІ).

    Historical note. The algorithm is often attributed to Cooley and Tukey (1965), but Gauss knew it in 1805. It was rediscovered independently by several researchers in the 20th century.

    2 — Roots of Unity

    The N-th roots of unity are the N complex numbers satisfying z^N = 1. They are

    ПүвӮҷбөҸ = e^{2ПҖik/N} = cos(2ПҖk/N) + iВ·sin(2ПҖk/N)     k = 0, 1, вҖҰ, NвҲ’1

    where ПүвӮҷ = e^{2ПҖi/N} is the primitive N-th root of unity. Key properties:

    Property 1: Cancellation lemma

    Пү_{dN}^{dk} = Пү_N^k     for any positive integer d

    Proof: Пү_{dN}^{dk} = e^{2ПҖiВ·dk/(dN)} = e^{2ПҖik/N} = Пү_N^k. вҲҺ

    Property 2: Halving lemma

    The squares of the N-th roots of unity (N even) are the (N/2)-th roots of unity.

    Proof: (Пү_N^k)ВІ = Пү_N^{2k} = Пү_{N/2}^k (by cancellation with d=2). Also, (Пү_N^{k+N/2})ВІ = Пү_N^{2k+N} = Пү_N^{2k} В· Пү_N^N = Пү_N^{2k} В· 1 = (Пү_N^k)ВІ. So the N squares collapse to N/2 distinct values. вҲҺ

    Property 3: Summation lemma

    ОЈ_{k=0}^{NвҲ’1} (Пү_N^j)^k = 0     for j вүў 0 (mod N)

    Proof: This is a geometric series with ratio r = Пү_N^j вү  1. Sum = (r^N вҲ’ 1)/(r вҲ’ 1) = ((Пү_N^N)^j вҲ’ 1)/(r вҲ’ 1) = (1 вҲ’ 1)/(r вҲ’ 1) = 0. вҲҺ

    When j вүЎ 0 (mod N), every term is 1, so the sum is N.

    3 — The Discrete Fourier Transform

    3.1 Definition

    Given a coefficient vector a = (aвӮҖ, aвӮҒ, вҖҰ, a_{NвҲ’1}), the DFT produces Е· = (yвӮҖ, yвӮҒ, вҖҰ, y_{NвҲ’1}) where

    yвӮ– = ОЈ_{j=0}^{NвҲ’1} aвұј В· ПүвӮҷ^{jk}     k = 0, 1, вҖҰ, NвҲ’1

    Equivalently, yвӮ– = A(ПүвӮҷбөҸ)—we are evaluating the polynomial A(x) = ОЈ aвұјxКІ at each N-th root of unity.

    3.2 Matrix form

    Let FвӮҷ be the NГ—N matrix with entry F_{k,j} = Пү_N^{jk}. Then

    Е· = FвӮҷ В· a

    The matrix FвӮҷ is the DFT matrix. It is a Vandermonde matrix evaluated at the roots of unity.

    3.3 The DFT matrix is unitary (up to scaling)

    Theorem: FМ„вӮҷбөҖ В· FвӮҷ = N В· IвӮҷ, or equivalently (1/вҲҡN)FвӮҷ is unitary.

    Proof: The (r, s) entry of FМ„вӮҷбөҖ В· FвӮҷ is

    (FМ„вӮҷбөҖ В· FвӮҷ)_{r,s} = ОЈ_{k=0}^{NвҲ’1} FМ„_{k,r} В· F_{k,s}
                    = ОЈ_{k=0}^{NвҲ’1} Пү_N^{вҲ’kr} В· Пү_N^{ks}
                    = ОЈ_{k=0}^{NвҲ’1} Пү_N^{k(sвҲ’r)}

    By the summation lemma:

    • If s = r: the sum equals N.
    • If s вү  r: the sum equals 0.

    Therefore FМ„вӮҷбөҖ В· FвӮҷ = N В· IвӮҷ. вҲҺ

    4 — Inverse DFT

    Since FвӮҷвҒ»В№ = (1/N) FМ„вӮҷбөҖ, the inverse transform is

    aвұј = (1/N) ОЈ_{k=0}^{NвҲ’1} yвӮ– В· Пү_N^{вҲ’jk}     j = 0, 1, вҖҰ, NвҲ’1
    Key observation. The IDFT has exactly the same structure as the DFT, with two differences: (1) we use Пү_N^{вҲ’1} instead of Пү_N, and (2) we divide by N at the end. This means the same code computes both DFT and IDFT!

    Proof of the IDFT formula: Substitute the DFT definition into the right-hand side:

    (1/N) ОЈ_{k=0}^{NвҲ’1} yвӮ– В· Пү_N^{вҲ’jk}
= (1/N) ОЈ_{k=0}^{NвҲ’1} [ОЈ_{m=0}^{NвҲ’1} aвӮҳ Пү_N^{mk}] В· Пү_N^{вҲ’jk}
= (1/N) ОЈ_{m=0}^{NвҲ’1} aвӮҳ В· [ОЈ_{k=0}^{NвҲ’1} Пү_N^{(mвҲ’j)k}]

    By the summation lemma, the inner sum is N if m = j and 0 otherwise. So the entire expression reduces to aвұј. вҲҺ

    5 — Cooley-Tukey: From O(nВІ) to O(n log n)

    5.1 Divide and conquer

    Assume N is a power of 2. Split the polynomial into even- and odd-indexed coefficients:

    A(x) = A_even(xВІ) + x В· A_odd(xВІ)
where
  A_even(y) = aвӮҖ + aвӮӮy + aвӮ„yВІ + вҖҰ
  A_odd(y)  = aвӮҒ + aвӮғy + aвӮ…yВІ + вҖҰ

    To compute A(Пү_N^k) for k = 0, вҖҰ, NвҲ’1:

    A(Пү_N^k) = A_even(Пү_N^{2k}) + Пү_N^k В· A_odd(Пү_N^{2k})
         = A_even(Пү_{N/2}^k) + Пү_N^k В· A_odd(Пү_{N/2}^k)

    Here we used the halving lemma: Пү_N^{2k} = Пү_{N/2}^k.

    5.2 The butterfly

    For k = 0, вҖҰ, N/2 вҲ’ 1, define:

    e_k = A_even(Пү_{N/2}^k)
o_k = A_odd(Пү_{N/2}^k)

    Then we get both halves of the output from the same sub-results:

    y_k       = e_k + Пү_N^k В· o_k       (first half)
y_{k+N/2} = e_k вҲ’ Пү_N^k В· o_k       (second half)

    Proof of the second half:

    A(Пү_N^{k+N/2}) = A_even(Пү_N^{2k+N}) + Пү_N^{k+N/2} В· A_odd(Пү_N^{2k+N})
               = A_even(Пү_N^{2k} В· Пү_N^N) + Пү_N^k В· Пү_N^{N/2} В· A_odd(Пү_N^{2k} В· Пү_N^N)
               = A_even(Пү_{N/2}^k) В· 1  +  Пү_N^k В· (вҲ’1) В· A_odd(Пү_{N/2}^k)
               = e_k вҲ’ Пү_N^k В· o_k   вҲҺ

    We used Пү_N^N = 1 and Пү_N^{N/2} = e^{ПҖi} = вҲ’1.

    5.3 Recurrence and complexity

    Each level of recursion halves the problem size and does O(N) butterfly operations:

    T(N) = 2T(N/2) + O(N)   вҶ’   T(N) = O(N log N)

    6 — Polynomial Multiplication via FFT

    6.1 The algorithm

    1. Choose N = 2вҢҲlogвӮӮ(m+n+1)вҢү (next power of 2 вүҘ m+n+1).
    2. Pad both polynomials to length N with zeros.
    3. Compute Е·_A = DFT(A) and Е·_B = DFT(B).
    4. Compute Е·_C[k] = Е·_A[k] В· Е·_B[k] for each k.
    5. Compute C = IDFT(Е·_C). The real parts of C are the product coefficients.

    Total: O(n log n) — two DFTs, one IDFT, and O(n) pointwise multiplications.

    6.2 Correctness

    By the convolution theorem, pointwise multiplication in the frequency domain corresponds to convolution in the time domain:

    DFT(A * B) = DFT(A) вҠҷ DFT(B)

    where вҠҷ is pointwise multiplication and * is polynomial multiplication (convolution). Applying IDFT to both sides gives A * B = IDFT(DFT(A) вҠҷ DFT(B)). вҲҺ

    6.3 C++ implementation (recursive)

    #include <vector>
#include <complex>
#include <cmath>
using cd = std::complex<double>;
const double PI = acos(-1.0);

void fft(std::vector<cd>& a, bool invert) {
    int n = (int)a.size();
    if (n == 1) return;

    std::vector<cd> a0(n / 2), a1(n / 2);
    for (int i = 0; i < n / 2; ++i) {
        a0[i] = a[2 * i];
        a1[i] = a[2 * i + 1];
    }
    fft(a0, invert);
    fft(a1, invert);

    double angle = 2 * PI / n * (invert ? -1 : 1);
    cd w(1), wn(cos(angle), sin(angle));

    for (int i = 0; i < n / 2; ++i) {
        cd t = w * a1[i];         // twiddle factor
        a[i]         = a0[i] + t; // butterfly
        a[i + n / 2] = a0[i] - t;
        w *= wn;
    }

    if (invert) {
        for (auto& x : a) x /= n;
        // Note: division by n should only happen at
        // the top level. This recursive version divides
        // at each level but the net effect is dividing
        // by n once (each element is divided by 2 at
        // each of log(n) levels: 2^log(n) = n).
    }
}

std::vector<long long> multiply(std::vector<int> const& a,
                                 std::vector<int> const& b) {
    std::vector<cd> fa(a.begin(), a.end());
    std::vector<cd> fb(b.begin(), b.end());

    int n = 1;
    while (n < (int)(a.size() + b.size()))
        n <<= 1;
    fa.resize(n);
    fb.resize(n);

    fft(fa, false);   // DFT
    fft(fb, false);

    for (int i = 0; i < n; ++i)
        fa[i] *= fb[i]; // pointwise multiply

    fft(fa, true);    // IDFT

    std::vector<long long> result(n);
    for (int i = 0; i < n; ++i)
        result[i] = llround(fa[i].real());

    // trim trailing zeros
    while (result.size() > 1 && result.back() == 0)
        result.pop_back();
    return result;
}
    Precision warning. Floating-point FFT introduces rounding errors. For integer coefficients up to ~10вҒ№ and n up to ~10вҒ¶, llround usually gives the correct answer. Beyond that, use the Number Theoretic Transform (Part 5) for exact arithmetic.

    Key takeaways:

    • Roots of unity provide the magic evaluation points that make FFT work.
    • The halving lemma is what enables the divide-and-conquer: the squares of N roots collapse to N/2 roots.
    • IDFT is structurally identical to DFT, so one function handles both.
    • The recursive version is clean but allocates O(n log n) memory. The next post presents the iterative version, which is faster in practice.
