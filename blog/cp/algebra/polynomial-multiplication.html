    Polynomial Multiplication

    July 2026 · 22 min read

    1 — The Convolution Problem

    Given two polynomials A(x) = Σ aᵢxⁱ of degree m and B(x) = Σ bⱼxʲ of degree n, their product C = A · B has degree m + n and coefficients

    cₖ = Σ_{i=0}^{k} aᵢ · b_{k−i}       k = 0, 1, …, m + n

    This is the discrete convolution of the coefficient sequences. Computing all m + n + 1 values of cₖ by the definition requires Θ(mn) multiplications. When m ≈ n, this is Θ(n²).

    Why is polynomial multiplication everywhere? Counting problems, generating-function manipulations, string matching, and number-theoretic transforms all reduce to polynomial multiplication. Speeding it up from O(n²) to O(n log n) is one of the most impactful algorithmic breakthroughs.

    2 — Naïve O(n²) Multiplication

    The straightforward approach iterates over all pairs (i, j):

    #include <vector>
using ll = long long;
using Poly = std::vector<ll>;

Poly naiveMul(const Poly& a, const Poly& b) {
    if (a.empty() || b.empty()) return {};
    int n = (int)a.size() + (int)b.size() - 1;
    Poly c(n, 0);
    for (int i = 0; i < (int)a.size(); ++i)
        for (int j = 0; j < (int)b.size(); ++j)
            c[i + j] += a[i] * b[j];
    return c;
}

    Analysis. The inner loop executes (m+1)(n+1) times. Each iteration does O(1) work (one multiplication, one addition). Total: Θ(mn).

    This is perfectly fine when n ≤ 3000 or so. Beyond that, we need something faster.

    3 — Karatsuba’s Algorithm

    3.1 The key insight

    Split each polynomial into two halves at the midpoint ⌈n/2⌉. Write

    A(x) = A₀ + x^m · A₁       (low and high halves)
B(x) = B₀ + x^m · B₁       where m = ⌈n/2⌉

    Then the product is:

    A · B = A₀B₀ + x^m (A₀B₁ + A₁B₀) + x^{2m} A₁B₁

    This appears to need four sub-multiplications: A₀B₀, A₀B₁, A₁B₀, A₁B₁. Karatsuba’s trick reduces this to three.

    3.2 The Karatsuba identity

    Define three products:

    z₀ = A₀ · B₀
z₂ = A₁ · B₁
z₁ = (A₀ + A₁)(B₀ + B₁) − z₀ − z₂

    Claim: z₁ = A₀B₁ + A₁B₀.

    Proof:

    z₁ = (A₀ + A₁)(B₀ + B₁) − z₀ − z₂
   = A₀B₀ + A₀B₁ + A₁B₀ + A₁B₁ − A₀B₀ − A₁B₁
   = A₀B₁ + A₁B₀   ∎

    So the final result is:

    A · B = z₀ + x^m · z₁ + x^{2m} · z₂

    We used only 3 recursive multiplications instead of 4, plus O(n) additions/subtractions.

    4 — Complexity Analysis

    4.1 Recurrence

    Let T(n) be the time to multiply two degree-n polynomials with Karatsuba. Each call makes 3 recursive calls on polynomials of size ≈ n/2 and does O(n) additional work:

    T(n) = 3 T(n/2) + O(n)

    4.2 Master Theorem solution

    We apply the Master Theorem with a = 3, b = 2, f(n) = O(n). The critical exponent is

    log_b(a) = log₂(3) ≈ 1.585

    Since f(n) = O(n) = O(n^{log₂3 − ε}) for ε ≈ 0.585, we fall into Case 1 of the Master Theorem:

    T(n) = Θ(n^{log₂3}) ≈ Θ(n^{1.585})

    4.3 Detailed derivation (recursion tree)

    At depth d of the recursion tree there are 3d sub-problems, each of size n/2d. The work at depth d is

    W(d) = 3^d · c · (n / 2^d) = cn · (3/2)^d

    The tree has depth log₂ n. Total work:

    T(n) = cn · Σ_{d=0}^{log₂ n} (3/2)^d
     = cn · [(3/2)^{log₂ n + 1} − 1] / (3/2 − 1)
     = O(n · (3/2)^{log₂ n})
     = O(n · n^{log₂(3/2)})
     = O(n^{1 + log₂(3) − 1})
     = O(n^{log₂ 3})   ∎

    Naïve multiplication

    • Time: Θ(n²)
    • Space: Θ(n) output
    • Constant factor: Very small
    • Best for: n ≤ 64

    Karatsuba

    • Time: Θ(n1.585)
    • Space: Θ(n log n) recursion
    • Constant factor: Moderate
    • Best for: 64 < n ≤ 10⁴

    5 — C++ Implementation

    #include <vector>
#include <algorithm>
using ll = long long;
using Poly = std::vector<ll>;

// Pad to equal length, rounding up to facilitate splitting
void padTo(Poly& p, int len) {
    p.resize(len, 0);
}

Poly addPoly(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] += b[i];
    return c;
}

Poly subPoly(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] -= b[i];
    return c;
}

Poly naiveMul(const Poly& a, const Poly& b) {
    if (a.empty() || b.empty()) return {};
    Poly c(a.size() + b.size() - 1, 0);
    for (size_t i = 0; i < a.size(); ++i)
        for (size_t j = 0; j < b.size(); ++j)
            c[i + j] += a[i] * b[j];
    return c;
}

Poly karatsuba(Poly a, Poly b) {
    int n = (int)std::max(a.size(), b.size());
    if (n <= 64) return naiveMul(a, b);

    // Make lengths equal and even
    int m = (n + 1) / 2;
    padTo(a, 2 * m);
    padTo(b, 2 * m);

    // Split: A = A0 + x^m * A1
    Poly a0(a.begin(), a.begin() + m);
    Poly a1(a.begin() + m, a.end());
    Poly b0(b.begin(), b.begin() + m);
    Poly b1(b.begin() + m, b.end());

    Poly z0 = karatsuba(a0, b0);          // A0 * B0
    Poly z2 = karatsuba(a1, b1);          // A1 * B1
    Poly z1 = karatsuba(addPoly(a0, a1),  // (A0+A1)(B0+B1) - z0 - z2
                         addPoly(b0, b1));
    z1 = subPoly(subPoly(z1, z0), z2);

    // Combine: result = z0 + x^m * z1 + x^{2m} * z2
    int resultLen = (int)a.size() + (int)b.size() - 1;
    Poly result(resultLen, 0);
    for (size_t i = 0; i < z0.size(); ++i) result[i] += z0[i];
    for (size_t i = 0; i < z1.size(); ++i) result[i + m] += z1[i];
    for (size_t i = 0; i < z2.size(); ++i) result[i + 2*m] += z2[i];

    while (result.size() > 1 && result.back() == 0)
        result.pop_back();
    return result;
}
    Base-case threshold. We fall back to naïve multiplication when n ≤ 64. This avoids the overhead of recursion and memory allocation for small inputs. The optimal threshold depends on hardware; 32–128 is typical.

    Worked example

    Multiply A(x) = 1 + 2x + 3x² by B(x) = 4 + 5x:

    Pad to length 4: A = {1, 2, 3, 0}, B = {4, 5, 0, 0}
m = 2
A₀ = {1, 2}, A₁ = {3, 0}
B₀ = {4, 5}, B₁ = {0, 0}

z₀ = A₀·B₀ = {1,2}·{4,5} = {4, 13, 10}
z₂ = A₁·B₁ = {3,0}·{0,0} = {0}
z₁ = (A₀+A₁)·(B₀+B₁) − z₀ − z₂
   = {4,2}·{4,5} − {4,13,10} − {0}
   = {16, 28, 10} − {4, 13, 10}
   = {12, 15, 0}

C = z₀ + x²·z₁ + x⁴·z₂
  = {4, 13, 10, 0, 0} + {0, 0, 12, 15, 0} + {0}
  = {4, 13, 22, 15}

Verify: (1+2x+3x²)(4+5x) = 4 + 13x + 22x² + 15x³  ✓

    6 — Beyond Karatsuba

    Karatsuba is a stepping stone. The same “reduce sub-problems” idea generalizes:

    Algorithm Complexity Year
    NaïveO(n²)—
    KaratsubaO(n1.585)1960
    Toom-Cook-3O(n1.465)1963
    FFT / NTTO(n log n)1965
    Harvey–van der HoevenO(n log n)2019

    For competitive programming, FFT is the workhorse for large n. We derive it fully in the next part. Karatsuba remains useful as a practical alternative when you need exact integer arithmetic and can’t afford FFT’s floating-point overhead (or as a component inside big-integer libraries).

    Correctness of the full Karatsuba algorithm (by induction)

    Theorem: The karatsuba function correctly computes the product of two polynomials A and B.

    Base case: When n ≤ 64, we use naiveMul, which is correct by the definition of polynomial multiplication (computing each coefficient as a sum of products).

    Inductive step: Assume karatsuba correctly multiplies polynomials of degree < n. For degree n:

    • We split A = A₀ + xmA₁ and B = B₀ + xmB₁. This is correct because the coefficients of A₀ are exactly a[0..m−1] and A₁ holds a[m..2m−1].
    • By the inductive hypothesis, z₀ = A₀B₀, z₂ = A₁B₁, and (A₀+A₁)(B₀+B₁) are all computed correctly.
    • z₁ = (A₀+A₁)(B₀+B₁) − z₀ − z₂ = A₀B₁ + A₁B₀ (proved algebraically above).
    • The recombination z₀ + xmz₁ + x2mz₂ = A₀B₀ + xm(A₀B₁+A₁B₀) + x2mA₁B₁ = AB. ∎
