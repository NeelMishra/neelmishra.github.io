    Polynomial GCD & Half-GCD

    July 2026 · 28 min read

    1. Polynomial GCD: Foundations

    The greatest common divisor of two polynomials A(x) and B(x) over a field F is the monic polynomial G(x) of highest degree that divides both A and B. The polynomial ring F[x] is a Euclidean domain, so GCD is well-defined and can be computed by the Euclidean algorithm, just as for integers.

    The Division Algorithm for Polynomials

    Theorem. For polynomials A, B ∈ F[x] with B ≠ 0, there exist unique Q, R ∈ F[x] such that A = BQ + R and deg(R) < deg(B).

    Proof. If deg(A) < deg(B), set Q = 0, R = A. Otherwise, let a = lc(A), b = lc(B) (leading coefficients), and d = deg(A) − deg(B). Then A1 = A − (a/b) xd B has degree < deg(A). By induction, A1 = BQ1 + R with deg(R) < deg(B). Thus A = B(Q1 + (a/b)xd) + R. Uniqueness follows by assuming two decompositions and showing their difference is zero. ∎

    Properties of Polynomial GCD

    • Associativity: gcd(A, gcd(B, C)) = gcd(gcd(A, B), C).
    • Bézout’s identity: There exist polynomials U, V such that AU + BV = gcd(A, B).
    • Coprime criterion: gcd(A, B) = 1 ⇔ ∃ U, V with AU + BV = 1.
    • Common roots: Over an algebraically closed field, gcd(A, B) encodes exactly the common roots of A and B (with appropriate multiplicities).

    2. The Euclidean Algorithm for Polynomials

    The Euclidean algorithm computes gcd(A, B) by repeatedly replacing (A, B) with (B, A mod B) until the remainder is zero.

    function poly_gcd(A, B):
      while B ≠ 0:
        (A, B) = (B, A mod B)
      return A / lc(A) // make monic

    Termination and Correctness

    Termination: Each step strictly decreases deg(B), and degrees are non-negative, so the algorithm terminates in at most deg(B) + 1 steps.

    Correctness: gcd(A, B) = gcd(B, A mod B) because: if G divides both A and B, it divides A − QB = R; conversely if G divides B and R, it divides QB + R = A.

    Complexity Analysis

    Let n = max(deg A, deg B). Each step performs a polynomial division where the dividend has degree ≤ n and the divisor has degree ≤ n, costing O(n) with schoolbook division. In the worst case, degrees decrease by 1 each step (n steps), giving:

    Total = n + (n−1) + … + 1 = O(n2). 

    // Euclidean algorithm for polynomial GCD over Z/pZ
// Polynomials represented as vector<long long> (ascending powers)
// O(n^2) where n = max degree

poly poly_gcd(poly A, poly B) {
    auto normalize = [&](poly &p) {
        while (p.size() > 1 && p.back() == 0)
            p.pop_back();
    };

    normalize(A);
    normalize(B);

    while (B.size() > 1 || (B.size() == 1 && B[0] != 0)) {
        // A = Q * B + R
        poly Q, R;
        poly_divmod(A, B, Q, R);
        A = B;
        B = R;
        normalize(A);
        normalize(B);
    }

    // Make A monic
    if (!A.empty() && A.back() != 0) {
        long long inv_lc = mod_pow(A.back(), MOD - 2, MOD);
        for (auto &c : A)
            c = c * inv_lc % MOD;
    }
    return A;
}

// Helper: polynomial division A = Q*B + R
void poly_divmod(const poly &A, const poly &B,
                 poly &Q, poly &R) {
    int n = A.size(), m = B.size();
    if (n < m) { Q = {0}; R = A; return; }

    Q.assign(n - m + 1, 0);
    R = A;
    long long inv_lc = mod_pow(B.back(), MOD - 2, MOD);

    for (int i = n - m; i >= 0; i--) {
        Q[i] = R[i + m - 1] * inv_lc % MOD;
        for (int j = 0; j < m; j++)
            R[i + j] = (R[i + j] - Q[i] * B[j] % MOD + MOD) % MOD;
    }
    R.resize(m - 1);
    while (R.size() > 1 && R.back() == 0)
        R.pop_back();
}

    3. Extended Euclidean Algorithm

    The extended GCD computes not just G = gcd(A, B) but also the Bézout coefficients U, V satisfying AU + BV = G.

    Matrix Formulation

    Each step of the Euclidean algorithm can be encoded as a 2×2 matrix multiplication. If A = QB + R, then:

    [B, R]T = [[0, 1], [1, −Q]] · [A, B]T.

    Let Mi = [[0, 1], [1, −Qi]] be the matrix for step i. After k steps, the accumulated transformation is T = Mk · Mk−1 ·…· M1, and [G, 0]T = T [A, B]T. The Bézout coefficients are the first row of T. 

    // Extended Euclidean algorithm for polynomials
// Returns {G, U, V} where A*U + B*V = G
struct ExtGCDResult {
    poly G, U, V;
};

ExtGCDResult poly_ext_gcd(poly A, poly B) {
    // u0*A_orig + v0*B_orig = A
    // u1*A_orig + v1*B_orig = B
    poly u0 = {1}, v0 = {0};
    poly u1 = {0}, v1 = {1};

    auto normalize = [](poly &p) {
        while (p.size() > 1 && p.back() == 0)
            p.pop_back();
    };

    while (B.size() > 1 || (B.size() == 1 && B[0] != 0)) {
        poly Q, R;
        poly_divmod(A, B, Q, R);

        // Update Bezout coefficients
        // new_u = u0 - Q * u1, new_v = v0 - Q * v1
        poly new_u = poly_sub(u0, poly_mul(Q, u1));
        poly new_v = poly_sub(v0, poly_mul(Q, v1));

        A = B; B = R;
        u0 = u1; u1 = new_u;
        v0 = v1; v1 = new_v;
        normalize(A); normalize(B);
    }

    // Make A monic
    if (!A.empty() && A.back() != 0) {
        long long inv_lc = mod_pow(A.back(), MOD - 2, MOD);
        for (auto &c : A) c = c * inv_lc % MOD;
        for (auto &c : u0) c = c * inv_lc % MOD;
        for (auto &c : v0) c = c * inv_lc % MOD;
    }
    return {A, u0, v0};
}

    Application: Modular Inverse of Polynomials

    To compute A−1 mod M (where M is a fixed modulus polynomial and gcd(A, M) = 1), run the extended GCD: AU + MV = 1. Then A−1 ≡ U (mod M).

    4. The Half-GCD Algorithm

    The Euclidean algorithm is O(n2) because it performs up to n division steps, each costing O(n). The Half-GCD (HGCD) algorithm reduces this to O(n log2 n) by exploiting a key structural insight: the first ~n/2 quotients of the Euclidean algorithm for (A, B) depend only on the top ~n/2 coefficients of A and B.

    The Key Insight

    Lemma. Let A, B have degrees n, n−1 respectively. Consider the Euclidean algorithm on (A, B). The quotients Q1, Q2, … are determined by the remainder sequence. The first quotients that reduce the degree by a total of m depend only on the top 2m coefficients of A and B.

    Intuition. Think of long division: to determine the first quotient digit, you only look at the leading terms. More generally, the first ~n/2 quotients are determined by the top ~n coefficients — and since A has n+1 coefficients, this is essentially all of them, but the point is we can recurse on a half-sized problem.

    Algorithm Structure

    HGCD(A, B) returns a 2×2 polynomial matrix T such that T · [A, B]T = [A′, B′] where deg(A′) has dropped by about half (from n to about n/2).

    function HGCD(A, B):
      n = deg(A)
      if deg(B) < n/2: return identity matrix I

      m = floor(n / 2)

      // Step 1: Recurse on top halves
      A_top = A div x^m // top coefficients
      B_top = B div x^m
      T1 = HGCD(A_top, B_top)

      // Step 2: Apply T1 to full (A, B)
      [A', B'] = T1 * [A, B]

      if deg(B') < m: return T1

      // Step 3: One Euclidean step
      Q, R = divmod(A', B')
      T_step = [[0, 1], [1, -Q]]

      // Step 4: Recurse on top part again
      m2 = floor((deg(A') + m) / 2) // or similar threshold
      B'_top = B' div x^m2
      R_top = R div x^m2
      T2 = HGCD(B'_top, R_top)

      return T2 * T_step * T1

    Complexity Derivation

    Let T(n) be the cost of HGCD on polynomials of degree n. The algorithm makes:

    • Two recursive calls on half-sized inputs: 2T(n/2)
    • Constant many polynomial multiplications of degree-n polynomials: O(M(n)) = O(n log n)
    • One polynomial division: O(M(n))

    The recurrence is

    T(n) = 2T(n/2) + O(n log n).

    By the Master Theorem (Case 2 with f(n) = n log n and a = 2, b = 2): nlogb a = n1, and f(n) = n log n = n1 · log n, so T(n) = O(n log2 n).

    Comparison with integer GCD. The analogous algorithm for integer GCD is Schönhage’s algorithm, which also runs in O(n log2 n log log n) for n-digit integers. The polynomial version is cleaner because polynomial division is exact (no carries).

    5. Computing Full GCD via Half-GCD

    HGCD reduces the degree by half. To compute the full GCD, we apply HGCD iteratively:

    function fast_gcd(A, B):
      while deg(B) > 0:
        T = HGCD(A, B)
        [A, B] = T * [A, B]
        if B == 0: break
        Q, R = divmod(A, B)
        A = B; B = R
      return A / lc(A)

    Each call to HGCD halves the degree, and the while loop runs O(log n) times (since after HGCD + one division step, the total degree drops by at least a constant fraction). But the dominant cost is the HGCD calls themselves, giving O(n log2 n) overall.

    Correctness

    HGCD produces a matrix T such that T[A, B]T has reduced degrees, and det(T) is a nonzero constant (since each elementary step matrix has determinant ±1 or a constant). The transformation is invertible, so gcd(A, B) = gcd(A′, B′) up to scalar multiples. The extra division step ensures progress even if HGCD didn’t quite reduce to the halfway point.

    6. Applications & Contest Relevance

    Application 1: Finding Common Factors

    If two polynomials A and B share a nontrivial GCD, then gcd(A, B) gives their common factor. This is used in:

    • Squarefree decomposition: gcd(A, A′) finds repeated roots of A.
    • Rational function simplification: cancel common factors in P/Q.
    • Error-correcting codes: finding error locator polynomials.

    Application 2: Padé Approximation

    Given a power series S(x) mod xn, find polynomials P(x), Q(x) with deg P ≤ k, deg Q ≤ n − k − 1 such that P ≡ Q · S (mod xn). This is equivalent to running the extended Euclidean algorithm on (xn, S(x)) and stopping when the remainder has degree ≤ k.

    Application 3: Berlekamp–Massey Connection

    The Berlekamp–Massey algorithm for finding the minimal linear recurrence of a sequence is closely related to the extended Euclidean algorithm on polynomials. Specifically, running the extended GCD on (x2n, S(x)) where S encodes the sequence, and stopping at the right point, yields the minimal polynomial. The Half-GCD algorithm thus gives an O(n log2 n) version of Berlekamp–Massey.

    Contest tip. The Half-GCD algorithm is one of the most complex polynomial algorithms in competitive programming. In practice, O(n2) polynomial GCD suffices for n ≤ 5000. Implement Half-GCD only when n ≥ 105 and polynomial GCD is explicitly required. The matrix formulation is error-prone — test extensively against the naïve version before submitting.

    Summary

    Polynomial GCD is the workhorse of algebraic algorithms. The Euclidean algorithm gives a simple O(n2) solution. The Half-GCD algorithm, by exploiting the locality of quotient computation, achieves the near-optimal O(n log2 n) bound. Together with multipoint evaluation (previous post), these tools form the foundation for advanced polynomial algorithms in competitive programming.
