    Polynomial Basics

    July 2026 · 25 min read

    1 — What Is a Polynomial?

    A polynomial in a single indeterminate x over a ring R is a formal expression

    P(x) = a₀ + a₁x + a₂x² + … + aₙxⁿ

    where every coefficient aᵢ ∈ R and n is a non-negative integer. The degree of P, written deg(P), is the largest index k with aₖ ≠ 0. By convention, the zero polynomial has degree −∞.

    In competitive programming we almost always work with R = ℤ (integers) or R = ℤ/pℤ (integers modulo a prime). Operations in these rings are exact, which avoids the floating-point headaches we will encounter later with FFT.

    Key convention. We store a degree-n polynomial as an array of length n + 1. Index i holds the coefficient of xⁱ. This “coefficient representation” is the starting point for every algorithm in this series.

    Degree arithmetic

    Two rules govern how degree behaves under operations:

    • Addition: deg(P + Q) ≤ max(deg P, deg Q), with equality when the leading coefficients don’t cancel.
    • Multiplication: deg(P · Q) = deg P + deg Q (assuming the coefficient ring has no zero divisors).

    2 — Representations

    2.1 Coefficient representation

    Store the polynomial as a vector a[0..n] where a[i] is the coefficient of xⁱ. This is the most natural form and the one we use for input/output in nearly every problem.

    // C++ coefficient representation
#include <vector>
using Poly = std::vector<long long>;

// Example: 3 + 2x + 5x^2  →  {3, 2, 5}
Poly p = {3, 2, 5};

    2.2 Point-value representation

    A degree-n polynomial is uniquely determined by its values at any n + 1 distinct points (this follows from the fact that a non-zero polynomial of degree n has at most n roots). So we can also represent P as a set of pairs:

    { (x₀, P(x₀)), (x₁, P(x₁)), …, (xₙ, P(xₙ)) }

    Why care? Multiplication in point-value form is O(n)—just multiply corresponding y-values. The expensive step is converting between the two representations. The Fast Fourier Transform (Part 3) makes that conversion O(n log n).

    Coefficient form

    • Addition: O(n)
    • Evaluation at one point: O(n)
    • Multiplication: O(n²) naïve

    Point-value form

    • Addition: O(n)
    • Evaluation: not directly available
    • Multiplication: O(n)

    3 — Horner’s Method

    Evaluating P(x₀) by computing each power x₀ⁱ separately uses 2n − 1 multiplications and n additions. Horner’s method rewrites the polynomial as a nested chain and needs only n multiplications and n additions:

    P(x) = a₀ + x(a₁ + x(a₂ + … + x(aₙ₋₁ + x·aₙ)…))

    Algorithm

    function Horner(a[0..n], x₀):
      result ← aₙ
      for i ← n−1 down to 0:
        result ← result · x₀ + aᵢ
      return result

    Proof of correctness

    Define the partial result after processing coefficient aₖ as

    rₖ = aₖ + aₖ₊₁x₀ + aₖ₊₂x₀² + … + aₙx₀ⁿ⁻ᵏ

    Base case (k = n): r_n = a_n. ✓

    Inductive step: Assume r_{k+1} = Σ_{j=k+1}^{n} a_j · x₀^{j−k−1}. The algorithm computes

    rₖ = r_{k+1} · x₀ + aₖ
   = x₀ · Σ_{j=k+1}^{n} a_j x₀^{j-k-1}  +  aₖ
   = Σ_{j=k+1}^{n} a_j x₀^{j-k}  +  aₖ
   = Σ_{j=k}^{n} a_j x₀^{j-k}         ✓

    At termination, r₀ = Σ_{j=0}^{n} a_j x₀^j = P(x₀). ∎

    Complexity

    The loop runs n iterations, each performing one multiplication and one addition. Total: n multiplications + n additions = Θ(n). This is optimal—any algorithm evaluating a degree-n polynomial must read all n + 1 coefficients, so Ω(n) is a lower bound.

    Interactive visualization — Horner’s evaluation

    Watch Horner’s method evaluate P(x) = 2 + 3x + x² + 4x³ at x₀ = 2, step by step.

    C++ implementation

    #include <vector>
using ll = long long;

ll horner(const std::vector<ll>& a, ll x0) {
    ll result = 0;
    for (int i = (int)a.size() - 1; i >= 0; --i)
        result = result * x0 + a[i];
    return result;
}
// Usage: horner({2, 3, 1, 4}, 2)  →  40

    4 — Basic Operations in Coefficient Form

    4.1 Addition & subtraction

    Given P of degree m and Q of degree n, the sum P + Q has degree at most max(m, n). We simply add (or subtract) coefficients index-by-index:

    Poly add(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] += b[i];
    while (c.size() > 1 && c.back() == 0) c.pop_back();
    return c;
}

Poly sub(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] -= b[i];
    while (c.size() > 1 && c.back() == 0) c.pop_back();
    return c;
}

    Complexity: Θ(max(m, n)). We also strip trailing zeros to maintain the invariant that the stored length equals degree + 1.

    4.2 Scalar multiplication

    Poly scalarMul(Poly a, ll k) {
    for (auto& c : a) c *= k;
    return a;
}

    This is Θ(n) and doesn’t change the degree (unless k = 0).

    4.3 Shift (multiply by xk)

    Multiplying by x^k shifts all coefficients up by k positions:

    Poly shift(const Poly& a, int k) {
    Poly c(a.size() + k, 0);
    for (size_t i = 0; i < a.size(); ++i) c[i + k] = a[i];
    return c;
}

    5 — Naive Polynomial Multiplication (Preview)

    Given P of degree m and Q of degree n, the product R = P · Q has degree m + n and coefficients:

    rₖ = Σ_{i+j=k} aᵢ · bⱼ     for k = 0, 1, …, m+n

    This is precisely the convolution of the coefficient sequences. The naïve implementation runs in Θ(mn):

    Poly multiply(const Poly& a, const Poly& b) {
    if (a.empty() || b.empty()) return {};
    int n = (int)a.size() + (int)b.size() - 1;
    Poly c(n, 0);
    for (size_t i = 0; i < a.size(); ++i)
        for (size_t j = 0; j < b.size(); ++j)
            c[i + j] += a[i] * b[j];
    return c;
}

    When both polynomials have degree n, this is Θ(n²)—too slow for competitive programming when n exceeds ~10⁴. In the next part we study Karatsuba, and in Part 3 the FFT brings us down to Θ(n log n).

    6 — Putting It All Together

    Here is a complete minimal polynomial library suitable for problems that only need basic operations:

    #include <bits/stdc++.h>
using ll = long long;
using Poly = std::vector<ll>;

// Strip leading zeros
void normalize(Poly& p) {
    while (p.size() > 1 && p.back() == 0) p.pop_back();
}

int deg(const Poly& p) {
    for (int i = (int)p.size() - 1; i >= 0; --i)
        if (p[i] != 0) return i;
    return -1; // zero polynomial
}

ll eval(const Poly& p, ll x) {
    ll r = 0;
    for (int i = (int)p.size() - 1; i >= 0; --i)
        r = r * x + p[i];
    return r;
}

Poly add(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] += b[i];
    normalize(c);
    return c;
}

Poly sub(const Poly& a, const Poly& b) {
    Poly c(std::max(a.size(), b.size()), 0);
    for (size_t i = 0; i < a.size(); ++i) c[i] += a[i];
    for (size_t i = 0; i < b.size(); ++i) c[i] -= b[i];
    normalize(c);
    return c;
}

Poly scalarMul(Poly a, ll k) {
    for (auto& c : a) c *= k;
    normalize(a);
    return a;
}

Poly naiveMul(const Poly& a, const Poly& b) {
    if (a.empty() || b.empty()) return {};
    Poly c((int)a.size() + (int)b.size() - 1, 0);
    for (size_t i = 0; i < a.size(); ++i)
        for (size_t j = 0; j < b.size(); ++j)
            c[i + j] += a[i] * b[j];
    return c;
}
    Overflow note. When working modulo a prime p ≈ 10⁹, a single product aᵢ · bⱼ fits in a long long but accumulated sums can overflow. Always reduce mod p inside the inner loop, or use __int128.
