    Number Theoretic Transform

    July 2026 · 28 min read

    1. Motivation: Why Avoid Floating Point?

    The FFT we developed in the previous posts computes the Discrete Fourier Transform over the complex numbers. While this is elegant and fast, it introduces floating-point rounding errors that accumulate during the butterfly operations. For many competitive-programming problems the answer must be computed modulo a prime p, and we need the coefficients to be exact integers.

    Consider multiplying two polynomials whose coefficient products can reach 1018. With double-precision FFT, the accumulated error in a size-220 transform can exceed 0.5, causing the rounded result to differ from the true answer by 1 or more. Long doubles help but are not portable, and they slow down SIMD vectorisation.

    The Number Theoretic Transform (NTT) solves this by replacing the complex n-th roots of unity with primitive roots of unity modulo a prime p. Every arithmetic operation becomes an exact integer operation in ℤ/pℤ, eliminating rounding entirely.

    Key insight: The FFT algorithm never uses any special property of complex numbers beyond the algebraic properties of n-th roots of unity (cancellation, halving, summation). As long as our ring contains an element satisfying these same properties, the algorithm works unchanged.

    2. Roots of Unity in ℤ/pℤ

    Let p be a prime. The multiplicative group (ℤ/pℤ)* is cyclic of order p − 1. If n divides p − 1, then there exists an element ω of order exactly n in (ℤ/pℤ)*. Such an ω is called a primitive n-th root of unity modulo p.

    Theorem. Let p be prime, n | (p − 1), and let g be a primitive root modulo p (a generator of (ℤ/pℤ)*). Then ω = g(p−1)/n mod p is a primitive n-th root of unity, and the following properties hold:

    1. Unity: ωn ≡ 1 (mod p).
    2. Primitivity: ωk ≠ 1 for 0 < k < n.
    3. Cancellation: ω2n2 = ωn, where ωm denotes a primitive m-th root.
    4. Halving: ωnn/2 ≡ −1 (mod p) when n is even.
    5. Summation: ∑j=0n−1 ωjk ≡ 0 (mod p) for k ≢ 0 (mod n).

    Proof of Summation. Let S = ∑j=0n−1 (ωk)j. Since ωk ≠ 1, the geometric series gives S = (ωkn − 1) / (ωk − 1) = (1 − 1)/(ωk − 1) = 0. All arithmetic is in ℤ/pℤ and the denominator is invertible because ωk ≠ 1. ■

    Proof of Halving. We have (ωn/2)2 = ωn = 1, so ωn/2 is a square root of 1 modulo p. The only square roots of 1 modulo a prime are ±1. Since ω has order n, ωn/2 ≠ 1, hence ωn/2 ≡ −1 (mod p). ■

    3. NTT-Friendly Primes & the Magic of 998244353

    For the Cooley-Tukey FFT, we need n to be a power of two, so we need 2k | (p − 1) for a large enough k. A prime of the form p = c · 2k + 1 with large k is called an NTT-friendly prime.

    Derivation for 998244353. We want a prime close to 109 (fits in 32-bit int) with a large power-of-two factor in p − 1:

    998244353 − 1 = 998244352 = 119 × 223

    Since 223 = 8388608, we can perform NTT of size up to 223 ≈ 8 × 106, which is more than enough for most competitive-programming problems. Moreover, 998244353 is prime (easily verified), and the product of two values modulo this prime fits in a 64-bit integer (9982443522 < 260), so we never need 128-bit arithmetic for a single multiplication.

    Why 998244353 specifically? It is the smallest prime of the form c · 223 + 1 that lies in the convenient range near 109. Other popular NTT primes include 985661441 = 235 · 222 + 1 and 754974721 = 45 · 224 + 1.

    A primitive root modulo 998244353 is g = 3 (this can be verified by checking that 3(p−1)/q ≢ 1 for each prime factor q of p − 1). Therefore, for an NTT of size n = 2k, we use:

    ωn = 3(998244353 − 1) / n mod 998244353

    4. Finding Primitive Roots

    Given a prime p, a primitive root g is a generator of (ℤ/pℤ)*. To find one, we factor p − 1 = q1a1 · q2a2 · … · qsas and check candidates g = 2, 3, 5, … until none of g(p−1)/qi ≡ 1 (mod p).

    Theorem. An integer g is a primitive root modulo p if and only if g(p−1)/q ≢ 1 (mod p) for every prime divisor q of p − 1.

    Proof. (⇒) If g has order p − 1 and g(p−1)/q ≡ 1, then ord(g) | (p−1)/q < p−1, contradiction. (⇐) If g is not a primitive root, then ord(g) | (p−1) and ord(g) < p−1. So there exists a prime q | (p−1) such that ord(g) | (p−1)/q, giving g(p−1)/q ≡ 1, contradicting our assumption. ■

    C++ implementation to find a primitive root:

    long long power(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}

int find_primitive_root(int p) {
    int phi = p - 1, n = phi;
    vector<int> factors;
    for (int i = 2; (long long)i * i <= n; i++) {
        if (n % i == 0) {
            factors.push_back(i);
            while (n % i == 0) n /= i;
        }
    }
    if (n > 1) factors.push_back(n);

    for (int g = 2; g <= p - 1; g++) {
        bool ok = true;
        for (int q : factors) {
            if (power(g, phi / q, p) == 1) {
                ok = false;
                break;
            }
        }
        if (ok) return g;
    }
    return -1; // should not happen for prime p
}

    For p = 998244353, we have p − 1 = 223 · 7 · 17, so we only need to check g(p−1)/2, g(p−1)/7, and g(p−1)/17. The candidate g = 3 passes all three checks.

    5. NTT Implementation

    The NTT follows the same iterative Cooley-Tukey structure as the FFT. The only differences are: (1) we use modular arithmetic, and (2) our root of unity is computed via modular exponentiation.

    const int MOD = 998244353;
const int PRIM_ROOT = 3;

long long power(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}

void ntt(vector<long long> &a, bool inverse) {
    int n = a.size();
    // bit-reversal permutation
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }

    for (int len = 2; len <= n; len <<= 1) {
        long long w = inverse
            ? power(PRIM_ROOT, MOD - 1 - (MOD - 1) / len, MOD)
            : power(PRIM_ROOT, (MOD - 1) / len, MOD);
        for (int i = 0; i < n; i += len) {
            long long wn = 1;
            for (int j = 0; j < len / 2; j++) {
                long long u = a[i + j];
                long long v = a[i + j + len / 2] * wn % MOD;
                a[i + j] = (u + v) % MOD;
                a[i + j + len / 2] = (u - v + MOD) % MOD;
                wn = wn * w % MOD;
            }
        }
    }

    if (inverse) {
        long long n_inv = power(n, MOD - 2, MOD);
        for (auto &x : a) x = x * n_inv % MOD;
    }
}

    Complexity. Exactly the same as FFT: O(n log n) operations, but each operation is a modular multiplication (one 64-bit multiply + one modular reduction), which is about 2–3× slower than a floating-point multiply on modern CPUs. In practice NTT runs in roughly the same time as FFT for n ≤ 220.

    Inverse NTT. The inverse root is ω−1 = ωn−1 = g(p−1)(n−1)/n. Equivalently, we can compute it as gp−1−(p−1)/n (Fermat's little theorem). After the inverse transform we divide every element by n (i.e., multiply by n−1 mod p).

    6. Modular Polynomial Multiplication

    With NTT in hand, multiplying two polynomials modulo 998244353 is straightforward:

    vector<long long> multiply(vector<long long> a, vector<long long> b) {
    int result_size = a.size() + b.size() - 1;
    int n = 1;
    while (n < result_size) n <<= 1;
    a.resize(n); b.resize(n);

    ntt(a, false);
    ntt(b, false);

    vector<long long> c(n);
    for (int i = 0; i < n; i++)
        c[i] = a[i] * b[i] % MOD;

    ntt(c, true);
    c.resize(result_size);
    return c;
}

    This gives the exact coefficients of A(x) · B(x) modulo 998244353 in O(n log n) time with zero floating-point error.

    When the modulus is not NTT-friendly

    If the problem modulus m is not NTT-friendly (e.g., 109 + 7), we can use one of two strategies:

    CRT with three NTT primes

    Pick three NTT-friendly primes p1, p2, p3 such that p1 · p2 · p3 exceeds the maximum possible coefficient. Multiply using NTT modulo each prime, then reconstruct the true coefficient via the Chinese Remainder Theorem, and finally reduce modulo m.

    Splitting coefficients

    Write each coefficient ai = ai,hi · B + ai,lo where B ≈ √m. Then A · B decomposes into three convolutions of values ≤ B · n, which fit in double-precision FFT. This avoids NTT entirely but is trickier to implement.

    Warning: When using the CRT approach with three primes, the intermediate product p1 · p2 can overflow 64 bits. Use __int128 or careful modular arithmetic to handle the reconstruction step.

    7. NTT vs FFT: A Practical Comparison

    FFT (Complex)

    • Works over any ring (no prime needed)
    • Faster per-operation (FP multiply)
    • Rounding errors accumulate
    • Safe for coefficients up to ~1014
    • Cannot do exact modular arithmetic

    NTT (Modular)

    • Requires NTT-friendly prime
    • Slower per-operation (modular multiply)
    • Exact — zero rounding error
    • Coefficients exact modulo p
    • Native for "answer mod p" problems

    Rule of thumb for contests: if the problem says "output answer modulo 998244353", use NTT. If the problem asks for exact integer results or uses a non-NTT-friendly modulus, consider FFT with the splitting trick or CRT-based NTT.

    In terms of constant factors, a well-optimised NTT over 998244353 runs at roughly 150–200 ms for n = 220 on Codeforces, compared to ~100 ms for double-precision FFT. The difference is rarely the bottleneck.

    Complete Example: Polynomial Multiplication mod 998244353

    #include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353, g = 3;

long long power(long long a, long long b, long long m = MOD) {
    long long r = 1; a %= m;
    for (; b > 0; b >>= 1) {
        if (b & 1) r = r * a % m;
        a = a * a % m;
    }
    return r;
}

void ntt(vector<long long> &a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        long long w = inv ? power(g, MOD - 1 - (MOD - 1) / len)
                         : power(g, (MOD - 1) / len);
        for (int i = 0; i < n; i += len) {
            long long wn = 1;
            for (int j = 0; j < len / 2; j++) {
                long long u = a[i+j], v = a[i+j+len/2] * wn % MOD;
                a[i+j] = (u+v) % MOD;
                a[i+j+len/2] = (u-v+MOD) % MOD;
                wn = wn * w % MOD;
            }
        }
    }
    if (inv) {
        long long ni = power(n, MOD - 2);
        for (auto &x : a) x = x * ni % MOD;
    }
}

int main() {
    int n, m;
    scanf("%d%d", &n, &m);
    int sz = 1;
    while (sz < n + m + 1) sz <<= 1;
    vector<long long> A(sz), B(sz);
    for (int i = 0; i <= n; i++) scanf("%lld", &A[i]);
    for (int i = 0; i <= m; i++) scanf("%lld", &B[i]);

    ntt(A, false); ntt(B, false);
    for (int i = 0; i < sz; i++) A[i] = A[i] * B[i] % MOD;
    ntt(A, true);

    for (int i = 0; i <= n + m; i++)
        printf("%lld%c", A[i], " \n"[i == n + m]);
    return 0;
}

    This implementation reads two polynomials of degrees n and m, computes their product modulo 998244353, and prints the result. It runs in O((n + m) log(n + m)) time and uses O(n + m) memory.
