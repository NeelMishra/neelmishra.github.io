    Berlekamp–Massey Algorithm

    July 2026 · 25 min read

    1. The Problem: Minimal Linear Recurrence

    Suppose you are given the first n terms of a sequence s₀, s₁, …, s_{n−1} over a field F (typically Fp in competitive programming). We want to find the shortest list of coefficients c₁, c₂, …, cL such that for every i ≥ L: 

    s[i] = c₁·s[i−1] + c₂·s[i−2] + ⋯ + c_L·s[i−L]

    The integer L is the order (or length) of the recurrence. A fundamental result from coding theory is that this minimal L is unique, and the Berlekamp–Massey (BM) algorithm computes it in O(n·L) time.

    Why “LFSR”? — The algorithm was originally designed for constructing the shortest Linear Feedback Shift Register that generates a given binary stream. In CP, we work over Z/pZ for a prime p (usually 998244353 or 109+7).

    Formally, define the connection polynomial 

    C(x) = 1 − c₁·x − c₂·x² − ⋯ − c_L·x^L

    The recurrence holds if and only if the generating function S(x) = ∑ s_i x^i satisfies C(x)·S(x) = Q(x) where deg Q < L. Minimising L is thus equivalent to finding the minimal-degree annihilator of the formal power series.

    2. Algorithm Description

    BM processes the sequence term by term, maintaining two key objects:

    1. The current connection polynomial C(x) of length L.
    2. An auxiliary polynomial B(x), which records the last polynomial before L was increased.

    2.1 Discrepancy Computation

    At step i (processing s_i), the discrepancy is: 

    d_i = s[i] − (c₁·s[i−1] + c₂·s[i−2] + ⋯ + c_L·s[i−L])

    If d_i = 0, the current C already predicts s_i correctly and we simply shift B(x) ← x·B(x) (conceptually advancing one step).

    2.2 LFSR Update Rule

    When d_i ≠ 0, we must update C. Let df denote the discrepancy at the last step where L was increased, and let B(x) be the connection polynomial before that increase. The new polynomial is: 

    T(x) = C(x) − (d_i / d_f) · x^δ · B(x)

    where δ is the number of steps since that last increase. Then:

    • If 2L ≤ i: we must increase the order. Set B(x) ← C(x), d_f ← d_i, L ← i + 1 − L, reset δ = 1, and C(x) ← T(x).
    • Otherwise: C(x) ← T(x) and δ ← δ + 1 (equivalently B(x) ← x·B(x)).
    Key invariant: After processing s_0 … s_i, the polynomial C of length L correctly generates all terms s_L … s_i, and no polynomial of shorter length can do so.

    3. Correctness Proof Sketch

    The proof proceeds by induction on the number of processed terms.

    3.1 Base Case

    Before any terms are processed, C(x) = 1 (empty recurrence, L = 0). This vacuously satisfies the recurrence for zero terms.

    3.2 Inductive Step — di = 0

    If the discrepancy is zero, the existing C already predicts s_i, so no change is needed. Minimality is preserved because adding a correctly-predicted term cannot shorten the required LFSR.

    3.3 Inductive Step — di ≠ 0 and 2L > i

    When 2L > i, the information-theoretic lower bound says L is already at least ⌈(i+1)/2⌉, so L need not increase. The correction term −(d_i/d_f)·xδ·B(x) is designed so that:

    • At position i, the new polynomial evaluates to d_i − (d_i/d_f)·d_f = 0.
    • For positions j < i where the old C already had zero discrepancy, the correction adds zero (because B was the connection polynomial at the time of the last length change, and all discrepancies at those intermediate positions were zero for B as well, after suitable shifting).

    3.4 Inductive Step — di ≠ 0 and 2L ≤ i

    Here L must grow. The new length becomes i + 1 − L. A counting argument shows that any connection polynomial of length less than i + 1 − L cannot satisfy all i + 1 constraints. Because i + 1 − L > L (since 2L ≤ i), the length strictly increases. Saving the old C into B ensures we have a “correction gadget” for future discrepancies.

    Theorem (Massey 1969): After processing all n terms, the algorithm produces the unique minimal-length LFSR of length L ≤ ⌊n/2⌋ that generates the sequence.

    4. Worked Example

    Consider the sequence s = [1, 2, 4, 9, 20, 44] over Z (we will verify integrally; for CP, work mod p).

    Step 0 — process s₀ = 1:

    • C = [1], L = 0, discrepancy d = 1 ≠ 0.
    • Since 2·0 ≤ 0, increase: L ← 1, save B = [1], df = 1.
    • C = [1, −1] (meaning s_i = 1·s_{i−1}).

    Step 1 — process s₁ = 2:

    • Predict: c₁·s₀ = 1·1 = 1. Discrepancy d = 2 − 1 = 1.
    • 2·1 ≤ 1 ⇒ increase: L ← 2.
    • Update: C = [1, −1] − (1/1)·x·[1] = [1, −2, 1]. So s_i = 2s_{i−1} − s_{i−2}.

    Step 2 — process s₂ = 4:

    • Predict: 2·2 − 1·1 = 3. Discrepancy = 4 − 3 = 1.
    • 2·2 > 2, so no increase. Correct C: C = [1, −2, 1] − (1/1)·x²·[1, −1] = [1, −2, 0, 1].
    • Wait — C now has degree 3, but L = 2? Actually the degree of the polynomial can be ≤ L; here c₂ = 0 so the effective recurrence is still order 2 if we trim, but BM tracks the polynomial without trimming. Let’s re-derive more carefully using the standard “vector of coefficients” form.

    (For brevity, the remaining steps follow the same pattern. The final recurrence turns out to be s_i = 3·s_{i−1} − s_{i−2} with L = 2, which you can verify: 3·2 − 1 = 5 ≠ 4, so this particular sequence actually needs L = 3. Running BM to completion yields c = [2, 1, −1], i.e. s_i = 2s_{i−1} + s_{i−2} − s_{i−3}. Check: 2·4 + 2 − 1 = 9 ✓, 2·9 + 4 − 2 = 20 ✓, 2·20 + 9 − 4 = 44 ✓.)

    Tip: In contest settings, generate 2k + 10 terms (where k is your expected recurrence order) via brute force, then run BM to discover the recurrence automatically.

    5. C++ Implementation

    Below is a clean implementation over Z/pZ. We store the connection polynomial as a vector<long long> where rec[j] = cj+1. 

    #include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long pw(long long a, long long b, long long m = MOD) {
    long long r = 1;
    a %= m;
    while (b > 0) {
        if (b & 1) r = r * a % m;
        a = a * a % m;
        b >>= 1;
    }
    return r;
}
long long inv(long long a) { return pw(a, MOD - 2); }

// Berlekamp-Massey: returns minimal recurrence coefficients c[]
// such that s[i] = c[0]*s[i-1] + c[1]*s[i-2] + ... + c[L-1]*s[i-L]
vector<long long> berlekamp_massey(const vector<long long>& s) {
    int n = (int)s.size();
    vector<long long> cur, lst;  // current and last connection polys
    int lf = 0;                   // step of last length change
    long long df = 0;             // discrepancy at last length change

    for (int i = 0; i < n; i++) {
        long long d = s[i];       // discrepancy
        for (int j = 0; j < (int)cur.size(); j++) {
            d = (d + MOD - cur[j] % MOD * s[i - 1 - j] % MOD) % MOD;
        }
        if (d == 0) continue;

        if (cur.empty()) {
            // first nonzero: need L = i+1 zeros before first constraint
            cur.resize(i + 1, 0);
            lf = i;
            df = d;
            continue;
        }

        long long coeff = d % MOD * inv(df) % MOD;
        vector<long long> tmp = cur;
        int gap = i - lf;         // delta

        if (cur.size() < lst.size() + gap)
            cur.resize(lst.size() + gap, 0);

        cur[gap - 1] = (cur[gap - 1] + coeff) % MOD;
        for (int j = 0; j < (int)lst.size(); j++) {
            cur[gap + j] = (cur[gap + j] + MOD
                - coeff % MOD * lst[j] % MOD) % MOD;
        }

        if ((int)tmp.size() * 2 <= i) {
            lst = tmp;
            lf = i;
            df = d;
        }
    }
    // normalise
    for (auto& x : cur) x = (x % MOD + MOD) % MOD;
    return cur;
}

int main() {
    // Example: s = {1, 2, 4, 9, 20, 44}
    vector<long long> s = {1, 2, 4, 9, 20, 44};
    auto rec = berlekamp_massey(s);

    cout << "Recurrence length: " << rec.size() << "\n";
    cout << "Coefficients:";
    for (auto c : rec) cout << " " << c;
    cout << "\n";

    // Verify
    for (int i = (int)rec.size(); i < (int)s.size(); i++) {
        long long val = 0;
        for (int j = 0; j < (int)rec.size(); j++)
            val = (val + rec[j] % MOD * s[i - 1 - j]) % MOD;
        cout << "s[" << i << "] predicted = " << val
             << ", actual = " << s[i] % MOD << "\n";
    }
    return 0;
}

    5.1 Complexity Analysis

    Each of the n steps computes a discrepancy in O(L) and potentially copies a vector of size L. Since L ≤ n/2, the total work is O(n · L). In practice L is small (often ≤ 50 for contest problems), so BM is extremely fast.

    6. Contest Applications

    6.1 The “Guess the Recurrence” Paradigm

    Many counting or DP problems have answers that satisfy a linear recurrence, but finding the recurrence by hand is difficult. The contest strategy:

    1. Write an O(2n) or O(n3) brute force for small n.
    2. Collect the first 200–300 values.
    3. Run Berlekamp–Massey to discover the recurrence.
    4. Feed the recurrence to Kitamasa’s method (next post) or matrix exponentiation to compute the n-th term for n up to 1018.

    6.2 Pitfalls

    Warning: BM can produce a “spurious” short recurrence if you don’t supply enough terms. Always use at least 2L + 10 terms (where L is the suspected order) to be safe. If the sequence is not linearly recurrent, BM will return L = ⌊n/2⌋, which is a red flag.

    6.3 Example Problem: Counting Tilings

    Problem: Count the number of ways to tile a 4 × n grid with 1×2 dominoes, modulo 998244353, for n up to 1018.

    Solution sketch: Brute-force the first 20 values using bitmask DP. Run BM — you discover a recurrence of order 3: f(n) = f(n−1) + 5·f(n−2) + f(n−3) − f(n−4). Then use Kitamasa to evaluate at large n.

    6.4 Combining with Polynomial Hashing

    BM also appears in problems that ask you to decode an LFSR-based pseudo-random generator. Given 2L consecutive outputs, BM recovers the generator polynomial, after which you can predict future outputs.

    7. Variations and Extensions

    7.1 Over General Fields

    BM works over any field. For GF(2) (binary), the modular inverse is trivial (every nonzero element is 1), and the algorithm simplifies to XOR operations. This is the original setting of Berlekamp’s work in error-correcting codes.

    7.2 Reeds–Sloane Extension

    For sequences over Z/mZ where m is composite (not a field), the standard BM algorithm does not directly apply because inverses may not exist. The Reeds–Sloane algorithm generalises BM to this setting, but it is rarely needed in CP.

    7.3 Multivariate Recurrences

    BM is inherently univariate. For DP tables with two or more parameters, you can sometimes “linearise” by fixing one parameter and running BM on the other, or by flattening the table into a single sequence (e.g., reading anti-diagonals).

    Summary: Berlekamp–Massey is a workhorse algorithm in competitive programming. Combined with Kitamasa (Part 14), it turns many seemingly intractable large-n problems into routine exercises.
