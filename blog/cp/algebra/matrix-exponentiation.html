    Matrix Exponentiation

    July 2026 · 25 min read

    1. The Core Idea

    Any system of linear recurrences can be written as a single matrix–vector multiplication. If the recurrence has constant coefficients, then computing the n-th term reduces to raising the transition matrix to the n-th power — which we can do in O(k³ log n) via binary exponentiation.

    This technique is one of the most versatile tools in competitive programming. It applies to:

    • Linear recurrences (Fibonacci, Tribonacci, general k-th order)
    • Counting paths of length n in a graph
    • Tiling problems with profile DP
    • Expected value recurrences with constant transition probabilities
    • Systems of coupled recurrences

    2. Companion Matrix Construction

    Consider the k-th order recurrence: 

    s[n] = c₁·s[n−1] + c₂·s[n−2] + ⋯ + c_k·s[n−k]

    Define the state vector at step n as: 

    v[n] = [ s[n], s[n−1], s[n−2], …, s[n−k+1] ]ᵀ

    Then v[n] = M · v[n−1] where M is the k × k companion matrix: 

        ┌ c₁  c₂  c₃  ⋯  c_{k-1}  c_k ┐
    │  1   0   0  ⋯    0       0  │
M = │  0   1   0  ⋯    0       0  │
    │  ⋮       ⋱            ⋮     │
    └  0   0   0  ⋯    1       0  ┘

    The first row holds the recurrence coefficients. Rows 2 through k form a shifted identity that “slides the window” forward by one step.

    2.1 Proof: Companion Matrix Correctness

    Claim: v[n] = Mn−k+1 · v[k−1] for all n ≥ k − 1.

    Proof by induction on n:

    • Base (n = k−1): M0 = I, so v[k−1] = I · v[k−1]. ✓
    • Inductive step: Assume v[n−1] = Mn−k · v[k−1]. Then v[n] = M · v[n−1] = Mn−k+1 · v[k−1]. The first component of M · v[n−1] is c₁·s[n−1] + c₂·s[n−2] + … + c_k·s[n−k] = s[n], and components 2–k are s[n−1], …, s[n−k+1], matching the definition of v[n]. □

    3. Binary Exponentiation of Matrices

    Matrix binary exponentiation follows the same pattern as integer modular exponentiation:

    function matpow(M, n):
      R ← I   // identity matrix
      while n > 0:
        if n is odd:
          R ← R × M
        M ← M × M
        n ← ⌊n / 2⌋
      return R

    Each matrix multiplication of k × k matrices is O(k³), and there are O(log n) iterations, giving O(k³ log n) total.

    Complexity note: For a fixed small k (e.g., k = 2 for Fibonacci), this is effectively O(log n) with a small constant. For large k, consider Kitamasa’s method (Part 14) which is O(k² log n) or O(k log k log n).

    4. C++ Template

    #include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

struct Matrix {
    int n;
    vector<vector<long long>> a;

    Matrix(int _n) : n(_n), a(_n, vector<long long>(_n, 0)) {}

    static Matrix identity(int n) {
        Matrix I(n);
        for (int i = 0; i < n; i++) I.a[i][i] = 1;
        return I;
    }

    Matrix operator*(const Matrix& o) const {
        Matrix res(n);
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++) {
                if (a[i][k] == 0) continue;
                for (int j = 0; j < n; j++)
                    res.a[i][j] = (res.a[i][j]
                        + a[i][k] * o.a[k][j]) % MOD;
            }
        return res;
    }
};

Matrix matpow(Matrix M, long long p) {
    Matrix R = Matrix::identity(M.n);
    while (p > 0) {
        if (p & 1) R = R * M;
        M = M * M;
        p >>= 1;
    }
    return R;
}

// Solve: s[n] = c[0]*s[n-1] + c[1]*s[n-2] + ... + c[k-1]*s[n-k]
// init[0] = s[0], init[1] = s[1], ..., init[k-1] = s[k-1]
long long solve_recurrence(long long n, const vector<long long>& c,
                           const vector<long long>& init) {
    int k = (int)c.size();
    if (n < k) return init[n] % MOD;

    // Build companion matrix
    Matrix M(k);
    for (int j = 0; j < k; j++) M.a[0][j] = c[j] % MOD;
    for (int i = 1; i < k; i++) M.a[i][i - 1] = 1;

    Matrix R = matpow(M, n - k + 1);

    // Multiply R * v[k-1]
    // v[k-1] = [s[k-1], s[k-2], ..., s[0]]
    long long ans = 0;
    for (int j = 0; j < k; j++)
        ans = (ans + R.a[0][j] % MOD * init[k - 1 - j]) % MOD;
    return ans;
}

int main() {
    // Example 1: Fibonacci — s[n] = s[n-1] + s[n-2]
    cout << "Fib(10) = "
         << solve_recurrence(10, {1, 1}, {0, 1}) << "\n";

    // Example 2: s[n] = 2*s[n-1] + 3*s[n-2], s[0]=1, s[1]=2
    cout << "Example2(20) = "
         << solve_recurrence(20, {2, 3}, {1, 2}) << "\n";

    return 0;
}

    5. Example: Fibonacci Numbers

    The Fibonacci recurrence F(n) = F(n−1) + F(n−2) with F(0) = 0, F(1) = 1 gives the companion matrix: 

    M = ┌ 1  1 ┐
    └ 1  0 ┘

    State vector: v[n] = [F(n), F(n−1)]ᵀ. Then:

    ┌ F(n)   ┐   ┌ 1  1 ┐ⁿ⁻¹   ┌ F(1) ┐   ┌ 1  1 ┐ⁿ⁻¹   ┌ 1 ┐
│        │ = │      │     · │      │ = │      │     · │   │
└ F(n−1) ┘   └ 1  0 ┘       └ F(0) ┘   └ 1  0 ┘       └ 0 ┘

    For n = 10: F(10) = 55. The computation involves about 4 matrix multiplications (since log₂(9) ≈ 3.17), each of which is a 2×2 multiply — essentially 8 multiplications per step.

    Fun fact: The identity F(m+n) = F(m)·F(n+1) + F(m−1)·F(n) follows directly from the matrix factorisation Mm+n = Mm · Mn.

    6. Application: Counting Paths in Graphs

    Theorem: If A is the adjacency matrix of a (directed or undirected) graph, then (An)ij equals the number of walks of length n from vertex i to vertex j.

    Proof: By induction on n.

    • Base (n = 0): A0 = I. There is exactly one walk of length 0 from i to i (the empty walk) and zero walks from i to j ≠ i. ✓
    • Inductive step: (An)ij = ∑_k (An−1)ik · Akj. By hypothesis, (An−1)ik counts walks of length n−1 from i to k. Multiplying by Akj appends one edge from k to j (if it exists), and summing over all k counts all walks of length n from i to j. □

    6.1 Example: Paths in a Small Graph

    Graph with 3 vertices and edges 0→1, 1→2, 2→0, 0→2: 

        ┌ 0  1  1 ┐
A = │ 0  0  1 │
    └ 1  0  0 ┘

    Number of walks of length 5 from vertex 0 to vertex 0 = (A⁵)[0][0]. Computing A⁵ via binary exponentiation (3×3 matrices): 

    Matrix A(3);
A.a = {{0,1,1},{0,0,1},{1,0,0}};
Matrix R = matpow(A, 5);
cout << R.a[0][0];  // number of walks of length 5 from 0 to 0

    7. Application: Tiling Problems

    Problem: Count the number of ways to tile a 3 × n grid with 1×2 dominoes (modulo 109+7).

    The standard approach uses profile DP: the state is a bitmask of size 3 representing which cells in the current column are already occupied by dominoes from the previous column. There are 2³ = 8 possible profiles.

    Build an 8×8 transition matrix T where T[i][j] = 1 if profile i can transition to profile j when placing dominoes in one column. Then the answer for width n is (Tn)[0][0] (starting and ending with an empty profile). 

    // Build transition matrix for 3×n tiling
// mask represents which cells protrude into the next column
int rows = 3;
int sz = 1 << rows;
Matrix T(sz);

function<void(int, int, int, int)> fill = [&](int row, int prev, int cur, int nxt) {
    if (row == rows) {
        T.a[prev][nxt] = (T.a[prev][nxt] + 1) % MOD;
        return;
    }
    if ((cur >> row) & 1) {
        // cell already filled from previous column
        fill(row + 1, prev, cur, nxt);
    } else {
        // place horizontal domino (protrudes into next column)
        fill(row + 1, prev, cur, nxt | (1 << row));
        // place vertical domino (if room)
        if (row + 1 < rows && !((cur >> (row + 1)) & 1))
            fill(row + 2, prev, cur, nxt);
    }
};

for (int mask = 0; mask < sz; mask++)
    fill(0, mask, mask, 0);

// Answer for 3 × n: (T^n)[0][0]
long long n;
cin >> n;
Matrix R = matpow(T, n);
cout << R.a[0][0] << "\n";

    8. Application: Expected Value Recurrences

    Consider a Markov chain with k states and constant transition probabilities. If the state vector at time n is π(n), then π(n) = P · π(n−1) where P is the transition matrix. Hence π(n) = Pn · π(0).

    More generally, if you have an expected-value DP of the form: 

    E[i][n] = Σ_j  p_{ij} · E[j][n−1] + b_i

    you can encode this as a matrix recurrence by augmenting the state with a constant “1” term to handle the additive constants b_i: 

    ┌ E[0][n] ┐   ┌ p₀₀  p₀₁  ⋯  b₀ ┐   ┌ E[0][n−1] ┐
│ E[1][n] │ = │ p₁₀  p₁₁  ⋯  b₁ │ · │ E[1][n−1] │
│    ⋮     │   │  ⋮              ⋮  │   │     ⋮      │
│ E[k][n] │   │ p_{k0} ⋯     b_k │   │ E[k][n−1] │
└    1     ┘   └  0    0   ⋯   1  ┘   └     1      ┘

    This “affine-to-linear” trick is extremely useful and appears frequently in probability problems on platforms like Codeforces and AtCoder.

    Tip: Whenever your recurrence has the form f(n) = A·f(n−1) + b (vector equation with constant b), augment the state with an extra “1” dimension. This lets you use matrix exponentiation directly without solving for a fixed point first.

    9. Optimisation Tips

    9.1 Cache-Friendly Multiplication

    In the triple loop for matrix multiplication, the order i → k → j is significantly faster than i → j → k due to cache locality (the inner loop accesses contiguous memory in the result and in o.a[k]). Our template above already uses this order.

    9.2 Skip Zero Entries

    The if (a[i][k] == 0) continue; line in our template skips unnecessary work when the matrix is sparse. For companion matrices (where only the first row and subdiagonal are nonzero), this can give a significant speedup.

    9.3 Modular Arithmetic

    Use long long to avoid overflow. With MOD ≈ 10&sup9;, the product of two entries fits in a 64-bit integer (10&sup9; × 10&sup9; = 1018 < 9.2 × 1018). With accumulation, be careful: for k additions of products, you need k · 1018 to fit. For k ≤ 9000 this is fine; for larger k, take modulus inside the inner loop.
