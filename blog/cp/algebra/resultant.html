    Polynomial Resultant

    July 2026 · 22 min read

    1. Motivation & Definition

    Given two univariate polynomials f(x) = amxm + am−1xm−1 + … + a0 and g(x) = bnxn + bn−1xn−1 + … + b0, a fundamental question in algebra is: do f and g share a common root? Computing the GCD via the Euclidean algorithm answers this, but the resultant provides a single scalar whose vanishing is equivalent to the existence of a common factor, and it carries additional algebraic information that the GCD does not.

    Definition. The resultant Res(f, g) is the determinant of the Sylvester matrix S(f, g), an (m + n) × (m + n) matrix built from the coefficients of f and g. We have Res(f, g) = 0 if and only if f and g share a common root (over the algebraic closure) or both leading coefficients vanish.

    The resultant appears throughout competitive programming whenever we need to eliminate a variable from a system of polynomial equations, test coprimality, or compute discriminants (the discriminant of f is essentially Res(f, f′) up to a sign and scaling factor).

    2. The Sylvester Matrix

    Let deg f = m and deg g = n. The Sylvester matrix S(f, g) is the (m + n) × (m + n) matrix formed by writing the coefficients of f in n rows (each shifted by one column) and the coefficients of g in m rows (each shifted by one column): 

             c0   c1   c2   ...  c_{m+n-1}
Row 1:   a_m  a_{m-1} ... a_0  0  ...  0
Row 2:   0    a_m  a_{m-1} ... a_0  0  ...
  ...
Row n:   0  ...  0  a_m  a_{m-1} ... a_0
Row n+1: b_n  b_{n-1} ... b_0  0  ...  0
Row n+2: 0    b_n  b_{n-1} ... b_0  0  ...
  ...
Row n+m: 0  ...  0  b_n  b_{n-1} ... b_0

    Concretely, row i (1 ≤ i ≤ n) of the first block contains the coefficients of xn−i · f(x), and row j (1 ≤ j ≤ m) of the second block contains the coefficients of xm−j · g(x). Thus the matrix encodes the vector space of all combinations u·f + v·g where deg u < n and deg v < m.

    Example. Let f(x) = x² + 3x + 2 and g(x) = x + 1. Then m = 2, n = 1, and the Sylvester matrix is 3 × 3:

    S = [[1, 3, 2], [1, 1, 0], [0, 1, 1]]

    det S = 1·(1·1 − 0·1) − 3·(1·1 − 0·0) + 2·(1·1 − 1·0) = 1 − 3 + 2 = 0.
    Indeed f and g share the root x = −1.

    3. Key Properties & Proofs

    3.1 Anti-symmetry

    Res(f, g) = (−1)mn · Res(g, f).

    Proof. Swapping the block of n rows of f with the block of m rows of g requires m·n transpositions of adjacent rows. Each transposition multiplies the determinant by −1, giving the factor (−1)mn. ▮

    3.2 Product Formula

    If f(x) = am ∏i=1..m(x − αi), then:

    Res(f, g) = amn · ∏i=1..m g(αi)

    Proof sketch. Consider Res(f, g) as a polynomial in the coefficients of g. Fixing f and varying g, Res(f, g) vanishes whenever g shares a root αi with f — i.e., whenever g(αi) = 0. Since the resultant has degree m in the coefficients of g (there are m rows of g-coefficients), and it vanishes at each αi, we get Res(f, g) = c · ∏ g(αi) for some constant c. Checking the leading coefficient by setting g(x) = bnxn shows c = amn. ▮

    3.3 Vanishing Criterion

    Theorem. Res(f, g) = 0 if and only if f and g have a common factor of positive degree (equivalently, they share a root over the algebraic closure of the coefficient field).

    Proof. (⇒) If Res(f, g) = det S = 0, there exists a nonzero vector (un−1, …, u0, vm−1, …, v0) in the kernel of S. This means u(x)f(x) + v(x)g(x) = 0 where deg u < n and deg v < m. Since u · f = −v · g and f, g have positive degree, any irreducible factor of f that does not divide g must divide v. But deg v < m = deg f, so f cannot entirely divide v; hence f and g must share at least one irreducible factor.

    (⇐) If f = d · f̃ and g = d · g̃ where deg d ≥ 1, set u = g̃ and v = −f̃. Then u · f + v · g = g̃ d f̃ − f̃ d g̃ = 0, and deg u = deg g − deg d < n, deg v < m. This gives a nonzero kernel vector, so det S = 0. ▮

    3.4 Multiplicativity

    For fixed g: Res(f1 · f2, g) = Res(f1, g) · Res(f2, g). This follows directly from the product formula.

    4. Applications in Competitive Programming

    4.1 Variable Elimination

    Given a system of two polynomial equations in two unknowns x, y — say f(x, y) = 0 and g(x, y) = 0 — treat both as polynomials in y with coefficients in the ring of polynomials in x. The resultant Resy(f, g) is a polynomial in x alone. Its roots are exactly the x-coordinates of intersection points (plus possibly some spurious points from leading coefficient vanishing). This technique reduces a two-variable system to a single-variable problem.

    4.2 Testing Coprimality

    Two polynomials f, g are coprime (gcd = 1) if and only if Res(f, g) ≠ 0. This is useful when working modulo a prime p: rather than running the Euclidean algorithm over Z/pZ[x], we can compute a single determinant.

    4.3 Discriminant

    The discriminant of f(x) = anxn + … + a0 is:

    Δ(f) = (−1)n(n−1)/2 · (1/an) · Res(f, f′)

    Δ(f) = 0 iff f has a repeated root. Many contest problems reduce to checking whether a discriminant is zero or computing it modulo a prime.

    5. Building the Sylvester Matrix in C++

    Below is a clean implementation that constructs the Sylvester matrix from two coefficient vectors and computes its determinant via Gaussian elimination over a modular field. 

    #include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}

long long inv(long long a, long long mod) {
    return power(a, mod - 2, mod);
}

// f: coefficients [a_m, a_{m-1}, ..., a_0] (length m+1)
// g: coefficients [b_n, b_{n-1}, ..., b_0] (length n+1)
// Returns the (m+n) x (m+n) Sylvester matrix
vector<vector<long long>> sylvester(const vector<long long>& f,
                                      const vector<long long>& g) {
    int m = (int)f.size() - 1;  // deg f
    int n = (int)g.size() - 1;  // deg g
    int sz = m + n;
    vector<vector<long long>> S(sz, vector<long long>(sz, 0));

    // n rows from f
    for (int i = 0; i < n; i++) {
        for (int j = 0; j <= m; j++) {
            S[i][i + j] = f[j];
        }
    }
    // m rows from g
    for (int i = 0; i < m; i++) {
        for (int j = 0; j <= n; j++) {
            S[n + i][i + j] = g[j];
        }
    }
    return S;
}

// Determinant mod p via Gaussian elimination
long long determinant(vector<vector<long long>> mat, long long mod) {
    int n = (int)mat.size();
    long long det = 1;
    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (mat[row][col] != 0) { pivot = row; break; }
        }
        if (pivot == -1) return 0;
        if (pivot != col) {
            swap(mat[pivot], mat[col]);
            det = (mod - det) % mod;  // negate
        }
        det = det % mod * (mat[col][col] % mod) % mod;
        long long iv = inv(mat[col][col], mod);
        for (int row = col + 1; row < n; row++) {
            long long factor = mat[row][col] % mod * iv % mod;
            for (int k = col; k < n; k++) {
                mat[row][k] = (mat[row][k] - factor % mod * mat[col][k] % mod
                               + mod * 2) % mod;
            }
        }
    }
    return (det % mod + mod) % mod;
}

long long resultant(const vector<long long>& f, const vector<long long>& g) {
    auto S = sylvester(f, g);
    return determinant(S, MOD);
}

int main() {
    // f(x) = x^2 + 3x + 2 = (x+1)(x+2)
    // g(x) = x + 1
    // Common root x = -1, so resultant should be 0
    vector<long long> f = {1, 3, 2};
    vector<long long> g = {1, 1};
    cout << "Res(f, g) = " << resultant(f, g) << "\n";

    // h(x) = x + 3  -- no common root with f
    vector<long long> h = {1, 3};
    cout << "Res(f, h) = " << resultant(f, h) << "\n";
    // Should be f(-3) = 9 - 9 + 2 = 2
    return 0;
}

    Complexity: Building the matrix is O((m+n)²). Gaussian elimination is O((m+n)³). For polynomials of degree up to a few hundred this is perfectly adequate. For larger degrees, sub-cubic algorithms exist based on the extended Euclidean algorithm for polynomials, running in O(n log² n) via half-GCD, but these are rarely needed in contests.

    6. Worked Example: Two Conics

    Suppose we need to find the x-coordinates of intersections of the two conics:

    • f: x² + y² − 4 = 0
    • g: x² − y + 1 = 0, i.e., y = x² + 1

    Rewriting as polynomials in y:

    • f(y) = y² + (x² − 4)  —  degree 2 in y
    • g(y) = −y + (x² + 1)  —  degree 1 in y

    The Sylvester matrix (3 × 3) in y is: 

    [ 1,         0,       x^2 - 4   ]
[ -1,        x^2+1,   0         ]
[ 0,         -1,      x^2+1     ]

    Computing the determinant:

    Resy(f, g) = 1 · [(x²+1)² − 0] − 0 + (x²−4) · [1] = (x²+1)² + (x²−4) = x4 + 2x² + 1 + x² − 4 = x4 + 3x² − 3.

    The roots of this degree-4 polynomial in x give the x-coordinates of the intersection points. We have eliminated y entirely using the resultant.

    7. Computing Resultant via Polynomial GCD

    An alternative O(n²) algorithm avoids building the full matrix. It uses the sub-resultant pseudo-remainder sequence, which mirrors the Euclidean algorithm but tracks the determinantal scaling carefully.

    The key recurrence is: if the Euclidean algorithm produces remainders r0 = f, r1 = g, r2, …, rk, then:

    Res(f, g) = (−1)∑ didi+1 · ∏ lc(ri)di-1 − di+1

    where di = deg ri and lc denotes the leading coefficient. In practice, the sub-resultant PRS avoids coefficient explosion over the integers and keeps intermediate results bounded. 

    // Simplified resultant via Euclidean algorithm (mod p)
long long resultant_euclidean(vector<long long> f, vector<long long> g,
                              long long mod) {
    // Remove leading zeros
    auto trim = [](vector<long long>& v) {
        while (v.size() > 1 && v[0] == 0) v.erase(v.begin());
    };
    trim(f); trim(g);

    int m = (int)f.size() - 1, n = (int)g.size() - 1;
    if (n == 0) return power(g[0], m, mod);
    if (m == 0) return power(f[0], n, mod);

    long long res = 1;
    while (true) {
        m = (int)f.size() - 1;
        n = (int)g.size() - 1;
        if (n == 0) {
            res = res % mod * power(g[0], m, mod) % mod;
            return (res % mod + mod) % mod;
        }
        // res *= lc(g)^(m - deg(f mod g)) but we compute step by step
        // Pseudo-remainder: f mod g
        vector<long long> r(f);
        while ((int)r.size() - 1 >= n) {
            long long coeff = r[0] % mod * inv(g[0], mod) % mod;
            for (int i = 0; i <= n; i++) {
                r[i] = (r[i] - coeff % mod * g[i] % mod + mod * 2) % mod;
            }
            r.erase(r.begin());
        }
        trim(r);
        // Res(f,g) = (-1)^(mn) * lc(g)^(m - deg r) * Res(g, r)
        int dr = (int)r.size() - 1;
        if (r.size() == 1 && r[0] == 0) return 0;

        long long sign = ((long long)m * n % 2 == 1) ? mod - 1 : 1;
        res = res % mod * sign % mod;
        res = res % mod * power(g[0], m - dr, mod) % mod;

        f = g;
        g = r;
    }
}

    This runs in O(m · n) time in the worst case, which is typically faster than the O((m+n)³) Gaussian elimination approach for large polynomials.

    8. Practice Pointers & Summary

    • When to use the resultant: Whenever a problem requires eliminating a variable from a system of polynomial equations, or testing whether two polynomials share a root modulo a prime.
    • Discriminant shortcut: Δ(f) = (−1)n(n−1)/2 · Res(f, f′) / an. Check for repeated roots quickly.
    • Modular arithmetic: Always work mod a prime to avoid big-integer overhead. The Sylvester matrix approach is the simplest to implement correctly.
    • Verification: For small examples, verify by computing amn ∏ g(αi) directly.
    Key takeaway. The resultant Res(f, g) is a single number that encodes whether f and g share a common root. It equals the determinant of the Sylvester matrix and also equals amn ∏ g(αi). In competitive programming, it is most useful for variable elimination and coprimality testing.
