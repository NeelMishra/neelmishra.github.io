    FFT Applications

    July 2026 · 25 min read

    1. Large Integer Multiplication

    The most classical application of FFT is multiplying two large integers stored as digit arrays. If A has n digits and B has m digits, the schoolbook algorithm is O(nm). Using FFT, we can do it in O((n+m) log(n+m)).

    Key idea. Represent an integer A = an−1 · 10n−1 + … + a1 · 10 + a0 as the polynomial PA(x) = a0 + a1x + … + an−1xn−1. Then A = PA(10). The product A · B corresponds to evaluating PA(x) · PB(x) at x = 10, after propagating carries through the coefficient array.

    Derivation of carry propagation. After multiplying the polynomials, coefficient ck = ∑i+j=k ai · bj can be as large as 81 · min(n, m) (if using base 10). We process from c0 upward: keep ck mod 10 as the k-th digit, carry ck / 10 to ck+1.

    #include <bits/stdc++.h>
using namespace std;
typedef complex<double> cd;
const double PI = acos(-1);

void fft(vector<cd> &a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        double ang = 2 * PI / len * (inv ? -1 : 1);
        cd wlen(cos(ang), sin(ang));
        for (int i = 0; i < n; i += len) {
            cd w(1);
            for (int j = 0; j < len / 2; j++) {
                cd u = a[i+j], v = a[i+j+len/2] * w;
                a[i+j] = u + v;
                a[i+j+len/2] = u - v;
                w *= wlen;
            }
        }
    }
    if (inv) for (auto &x : a) x /= n;
}

string multiply_big(const string &num1, const string &num2) {
    int n = num1.size(), m = num2.size();
    int sz = 1;
    while (sz < n + m) sz <<= 1;
    vector<cd> fa(sz), fb(sz);
    for (int i = 0; i < n; i++) fa[i] = num1[n-1-i] - '0';
    for (int i = 0; i < m; i++) fb[i] = num2[m-1-i] - '0';

    fft(fa, false); fft(fb, false);
    for (int i = 0; i < sz; i++) fa[i] *= fb[i];
    fft(fa, true);

    vector<int> result(n + m, 0);
    for (int i = 0; i < n + m; i++)
        result[i] = (int)round(fa[i].real());

    // carry propagation
    for (int i = 0; i < n + m - 1; i++) {
        result[i+1] += result[i] / 10;
        result[i] %= 10;
    }

    // build answer string
    int top = n + m - 1;
    while (top > 0 && result[top] == 0) top--;
    string ans;
    for (int i = top; i >= 0; i--)
        ans += (char)('0' + result[i]);
    return ans;
}

int main() {
    string a, b;
    cin >> a >> b;
    cout << multiply_big(a, b) << "\n";
    return 0;
}

    Complexity: O((n + m) log(n + m)) time and O(n + m) memory. This is the same asymptotic complexity as Karatsuba for very large numbers, but with a much smaller constant for n > 104.

    2. String Matching via Convolution

    FFT can solve pattern-matching problems that would otherwise require O(nm) comparisons. The key technique is encoding match/mismatch as a polynomial product.

    Exact matching with wildcards

    Problem. Given a text T of length n and a pattern P of length m, both over the alphabet {a–z, *} where * matches any character, find all positions where P matches T.

    Approach. Define character encodings: let ti be the ASCII code of T[i] (0 if wildcard), and pj similarly. Define the match function:

    M(i) = ∑j=0m−1 pj · ti+j · (pj − ti+j)2

    Then M(i) = 0 if and only if P matches T starting at position i. Each factor is zero when either character is a wildcard (encoded as 0) or when the characters are equal.

    Expanding M(i), we get three sums of the form ∑ f(j) · g(i+j), which are convolutions. By reversing one of the arrays, each becomes a polynomial multiplication computable via FFT.

    Complexity: O(n log n) for three FFT convolutions, versus O(nm) naive.

    Reversal trick for convolution: A correlation ∑ f(j) · g(i+j) becomes a convolution by defining f'(j) = f(m−1−j). Then (∑ f' · g)[i + m − 1] gives the desired value.

    3. Counting Problems via Polynomial Multiplication

    Many counting problems reduce to polynomial multiplication. The general pattern: if f(x) = ∑ ai xi encodes "number of ways to achieve value i with one object", then f(x)k encodes "number of ways to achieve value i using k objects".

    Example: Coin Change Counting

    Problem. Given coin denominations d1, d2, …, dk (each usable once), count the number of subsets summing to each value 0, 1, …, S.

    For each coin of denomination d, create the polynomial (1 + xd). The product of all such polynomials is ∏i=1k (1 + xdi), and its coefficient of xs is the number of subsets summing to s.

    Using a divide-and-conquer approach with FFT: split the k polynomials into two halves, recursively multiply each half, then multiply the two results. Total work is O(S log2 k) using FFT at each level.

    Example: Dice Sum Distribution

    Problem. You roll n dice, each with faces 1 to 6. Compute the number of ways to get each sum s from n to 6n.

    Each die is the polynomial f(x) = x + x2 + x3 + x4 + x5 + x6. The answer is the coefficient vector of f(x)n. Using repeated squaring with FFT-based multiplication:

    vector<long long> poly_mult(vector<long long> &a, vector<long long> &b);

vector<long long> poly_power(vector<long long> base, int exp) {
    vector<long long> result = {1};
    while (exp > 0) {
        if (exp & 1) result = poly_mult(result, base);
        base = poly_mult(base, base);
        exp >>= 1;
    }
    return result;
}

// Usage: f = {0, 1, 1, 1, 1, 1, 1} for a 6-sided die
// poly_power(f, n) gives the distribution

    Complexity: Each multiplication involves polynomials of degree up to 6n, so each FFT costs O(n log n). We do O(log n) multiplications, giving O(n log2 n) total.

    4. Worked Contest Problems

    Problem A: Counting Subarray Sums

    Statement. Given an array of n positive integers (each ≤ 104), count how many pairs of non-empty contiguous subarrays have the same sum. (n ≤ 2 × 105)

    Solution. First compute all subarray sums using prefix sums. Let cnt[s] be the number of subarrays with sum s. The answer is ∑ C(cnt[s], 2). But computing cnt[s] naively takes O(n2) time.

    FFT approach. Let P[0], P[1], …, P[n] be the prefix sums. A subarray sum equals P[j] − P[i] for i < j. We need the number of pairs (i, j) with i < j and P[j] − P[i] = s for each s. This is a convolution of the prefix-sum frequency array with its reversal.

    Define freq[v] = number of indices i such that P[i] = v. Then cnt[s] = ∑v freq[v] · freq[v + s] = (freq ⋆ rev(freq))[s], adjusting for i < j.

    #include <bits/stdc++.h>
using namespace std;
typedef complex<double> cd;
const double PI = acos(-1);

void fft(vector<cd> &a, bool inv) {
    int n = a.size();
    for (int i = 1, j = 0; i < n; i++) {
        int bit = n >> 1;
        for (; j & bit; bit >>= 1) j ^= bit;
        j ^= bit;
        if (i < j) swap(a[i], a[j]);
    }
    for (int len = 2; len <= n; len <<= 1) {
        double ang = 2 * PI / len * (inv ? -1 : 1);
        cd wlen(cos(ang), sin(ang));
        for (int i = 0; i < n; i += len) {
            cd w(1);
            for (int j = 0; j < len / 2; j++) {
                cd u = a[i+j], v = a[i+j+len/2] * w;
                a[i+j] = u + v;
                a[i+j+len/2] = u - v;
                w *= wlen;
            }
        }
    }
    if (inv) for (auto &x : a) x /= n;
}

int main() {
    int n;
    scanf("%d", &n);
    vector<int> a(n);
    for (int i = 0; i < n; i++) scanf("%d", &a[i]);

    vector<long long> prefix(n + 1, 0);
    for (int i = 0; i < n; i++) prefix[i+1] = prefix[i] + a[i];

    long long maxP = *max_element(prefix.begin(), prefix.end());
    int sz = 1;
    while (sz <= 2 * maxP) sz <<= 1;

    vector<cd> fa(sz);
    for (int i = 0; i <= n; i++) fa[(int)prefix[i]] += 1.0;

    // Self-convolution via squaring in frequency domain
    fft(fa, false);
    for (int i = 0; i < sz; i++) fa[i] *= fa[i];
    fft(fa, true);

    // fa[s + maxP] would give cross-correlation at shift s
    // The answer: sum of C(cnt, 2) = (sum of cnt^2 - sum of cnt) / 2
    // sum of cnt^2 = coefficient at offset 0 in the auto-correlation
    long long total_pairs = (long long)round(fa[0].real());
    // subtract self-pairs: each prefix index pairs with itself
    total_pairs -= (n + 1);
    total_pairs /= 2;

    printf("%lld\n", total_pairs);
    return 0;
}

    Problem B: Polynomial Multiplication for Subset Convolution Setup

    Statement. Given two arrays A and B of length n (≤ 105), each element in [0, 104], compute array C where C[k] = number of pairs (i, j) such that A[i] + B[j] = k.

    Solution. This is a direct polynomial multiplication. Let fA(x) = ∑ xA[i] and fB(x) = ∑ xB[j]. Then C[k] is the coefficient of xk in fA · fB.

    int main() {
    int n;
    scanf("%d", &n);
    int maxA = 0, maxB = 0;
    vector<int> freqA(10001, 0), freqB(10001, 0);
    for (int i = 0; i < n; i++) {
        int x; scanf("%d", &x);
        freqA[x]++;
        maxA = max(maxA, x);
    }
    for (int i = 0; i < n; i++) {
        int x; scanf("%d", &x);
        freqB[x]++;
        maxB = max(maxB, x);
    }

    int sz = 1;
    while (sz <= maxA + maxB) sz <<= 1;
    vector<cd> fa(sz), fb(sz);
    for (int i = 0; i <= maxA; i++) fa[i] = freqA[i];
    for (int i = 0; i <= maxB; i++) fb[i] = freqB[i];

    fft(fa, false); fft(fb, false);
    for (int i = 0; i < sz; i++) fa[i] *= fb[i];
    fft(fa, true);

    for (int k = 0; k <= maxA + maxB; k++)
        printf("%lld\n", (long long)round(fa[k].real()));
    return 0;
}

    Problem C: Bitstring Autocorrelation

    Statement. Given a binary string S of length n, for each shift d (0 ≤ d < n), compute the number of positions i where S[i] = S[i + d]. (n ≤ 106)

    Solution. Encode S as a polynomial f(x) = ∑ si xi where si ∈ {0, 1}. The number of matching positions at shift d involves the correlation ∑ si · si+d. This equals the (n−1+d)-th coefficient of f(x) · rev(f(x)). Additionally, we must count positions where both are zero, which equals (n − d − ones) − (mismatches at ones), but the simpler approach is:

    Let match(d) = (number of i where S[i] = 1 and S[i+d] = 1) + (number of i where S[i] = 0 and S[i+d] = 0). The first term is the correlation of f with itself, computable via FFT. The second term equals (n − d) − (ones in shifted window) − (ones in base window) + 2 · (first term). This can be simplified with careful algebra to a single FFT convolution.

    Pattern recognition: Whenever a problem asks "for each offset d, count something involving pairs at distance d", think correlation/convolution via FFT.

    5. Frequency Domain Tricks

    Beyond straightforward polynomial multiplication, the frequency domain offers several powerful techniques:

    Or-convolution via FFT

    Given arrays A and B indexed by integers in [0, M], compute C where C[k] = ∑i|j=k A[i] · B[j] (bitwise OR). This uses the subset-sum (Möbius) transform, not FFT directly, but the structure is analogous: transform, pointwise multiply, inverse transform. Complexity: O(M log M).

    Power spectrum and periodicity

    The autocorrelation of a sequence reveals periodicities. If a string has period p, then the autocorrelation at shift p will be close to n. This can be used to efficiently find all periods of a string in O(n log n) time.

    Multipoint evaluation preview

    Evaluating a polynomial at many points simultaneously can be done in O(n log2 n) using a technique based on polynomial division and FFT. This is covered in detail in later posts, but the key insight is that evaluation at a point a is equivalent to computing f(x) mod (x − a).
