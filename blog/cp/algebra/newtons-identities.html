    Newton's Identities

    July 2026 · 22 min read

    1. Power Sums & Symmetric Polynomials

    Let α1, α2, …, αn be the roots of a monic polynomial f(x) = xn − e1xn−1 + e2xn−2 − … + (−1)nen. The elementary symmetric polynomials are:

    • e1 = ∑ αi
    • e2 = ∑i<j αiαj
    • ek = ∑i1<…<ik αi1 … αik

    The power sums are:

    pk = α1k + α2k + … + αnk

    Newton's identities provide a systematic way to convert between these two families. This is extremely useful because the ek are immediately readable from the polynomial's coefficients (via Vieta's formulas), while pk appear naturally in many counting and optimization problems.

    2. Statement of Newton's Identities

    For 1 ≤ k ≤ n:

    pk − e1·pk−1 + e2·pk−2 − … + (−1)k−1·k·ek = 0

    For k > n:

    pk − e1·pk−1 + e2·pk−2 − … + (−1)n·en·pk−n = 0

    Equivalently, using the convention e0 = 1 and ej = 0 for j > n:

    pk = ∑i=1..min(k,n) (−1)i−1 ei · pk−i + (−1)k−1 k · ek · [k ≤ n]

    First few identities (expanded):
    k=1: p1 = e1
    k=2: p2 = e1·p1 − 2e2 = e1² − 2e2
    k=3: p3 = e1·p2 − e2·p1 + 3e3 = e1³ − 3e1e2 + 3e3

    3. Proof via Logarithmic Derivative

    Consider the polynomial in factored form:

    f(x) = ∏i=1..n(x − αi)

    Take the logarithmic derivative:

    f′(x) / f(x) = ∑i=1..n 1/(x − αi)

    Now expand each term as a geometric series for |x| > |αi|:

    1/(x − αi) = (1/x) · 1/(1 − αi/x) = ∑k≥0 αik / xk+1

    Summing over all roots:

    f′(x) / f(x) = ∑k≥0 pk / xk+1

    On the other hand, from the coefficient form f(x) = xn − e1xn−1 + …, we have:

    f′(x) = nxn−1 − (n−1)e1xn−2 + …

    Writing f′(x) = f(x) · ∑ pk/xk+1 and comparing coefficients of xn−k−1 on both sides yields exactly Newton's identities. Specifically, multiplying through by f(x):

    f′(x) = f(x) · ∑k≥0 pk x−k−1

    The coefficient of xn−1 on the left is n; on the right it is p0 = n. The coefficient of xn−2 on the left is −(n−1)e1; on the right it is p1 − e1 · p0 = p1 − ne1. Equating: −(n−1)e1 = p1 − ne1, so p1 = e1. Continuing this pattern for each power of x produces the full set of identities. ▮

    4. Alternative Proof: Direct Algebraic Manipulation

    For a more elementary proof (not using formal power series), we can proceed by induction. We prove the identity for k ≤ n; the case k > n is similar.

    Base case (k = 1): p1 = ∑ αi = e1. ✓

    Inductive step: Since f(αi) = 0 for each root, we have:

    αin = e1αin−1 − e2αin−2 + … + (−1)n+1en

    Multiply both sides by αik−n (for k ≥ n) and sum over i = 1, …, n:

    pk = e1pk−1 − e2pk−2 + … + (−1)n+1enpk−n

    which is exactly Newton's identity for k > n. For k ≤ n, a more careful argument handles the boundary term involving k·ek. The idea is to consider αik = (partial sum of the relation) + remainder involving products that sum to k·ek. ▮

    5. Converting Between Power Sums and Elementary Symmetric Polynomials

    5.1 Power Sums from ek

    Given e1, …, en, we can compute p1, p2, … iteratively: 

    #include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

// Given e[1..n], compute p[1..maxK]
// e[0] is unused (or set to 1 conceptually)
vector<long long> power_sums_from_e(const vector<long long>& e, int maxK) {
    int n = (int)e.size() - 1;  // e[1..n]
    vector<long long> p(maxK + 1, 0);

    for (int k = 1; k <= maxK; k++) {
        long long val = 0;
        // p_k = sum_{i=1}^{min(k,n)} (-1)^{i-1} * e_i * p_{k-i}
        for (int i = 1; i <= min(k, n); i++) {
            long long term = e[i] % MOD * p[k - i] % MOD;
            if (i % 2 == 1)
                val = (val + term) % MOD;
            else
                val = (val - term + MOD) % MOD;
        }
        // Add the boundary term for k <= n
        if (k <= n) {
            long long boundary = (long long)k % MOD * e[k] % MOD;
            if (k % 2 == 1)
                val = (val + boundary) % MOD;
            else
                val = (val - boundary + MOD) % MOD;
        }
        p[k] = val;
    }
    return p;
}

int main() {
    // f(x) = (x-1)(x-2)(x-3) = x^3 - 6x^2 + 11x - 6
    // e1=6, e2=11, e3=6
    // Roots: 1, 2, 3
    // p1 = 1+2+3 = 6, p2 = 1+4+9 = 14, p3 = 1+8+27 = 36
    vector<long long> e = {0, 6, 11, 6};
    auto p = power_sums_from_e(e, 5);
    for (int k = 1; k <= 5; k++) {
        cout << "p_" << k << " = " << p[k] << "\n";
    }
    return 0;
}

    5.2 Elementary Symmetric Polynomials from pk

    Conversely, given p1, …, pn, we can recover e1, …, en. Rearranging Newton's identity for the case k ≤ n:

    k · ek = (−1)k−1 · [pk − e1pk−1 + e2pk−2 − … + (−1)k−1ek−1p1]

    So ek can be computed as long as k is invertible (which holds modulo a prime > n). 

    // Given p[1..n], compute e[1..n]
vector<long long> e_from_power_sums(const vector<long long>& p) {
    int n = (int)p.size() - 1;  // p[1..n]
    vector<long long> e(n + 1, 0);

    auto inv = [&](long long a) -> long long {
        long long res = 1, b = MOD - 2;
        a %= MOD;
        while (b > 0) {
            if (b & 1) res = res * a % MOD;
            a = a * a % MOD;
            b >>= 1;
        }
        return res;
    };

    for (int k = 1; k <= n; k++) {
        long long val = p[k];
        for (int i = 1; i < k; i++) {
            long long term = e[i] % MOD * p[k - i] % MOD;
            if (i % 2 == 1)
                val = (val - term + MOD) % MOD;
            else
                val = (val + term) % MOD;
        }
        // e_k = (-1)^{k-1} * val / k
        if (k % 2 == 0) val = (MOD - val) % MOD;
        e[k] = val % MOD * inv(k) % MOD;
    }
    return e;
}

    6. Applications in Competitive Programming

    6.1 Computing Sums of Powers of Roots

    A classic problem: given the coefficients of a polynomial of degree n, compute ∑ αik for each k = 1, 2, …, K. By Vieta's formulas the coefficients give us e1, …, en directly, and then Newton's identities let us compute each pk in O(n) time, for a total of O(nK).

    6.2 Counting with Characteristic Polynomials

    If A is an n × n matrix and we want ∑ λik (the sum of k-th powers of eigenvalues), this equals tr(Ak). Newton's identities connect this to the coefficients of the characteristic polynomial det(xI − A). For small n, this is much faster than matrix exponentiation.

    6.3 Generating Functions & Partition Problems

    Newton's identities establish the formal power series identity:

    ∑k≥1 pk tk/k = −ln(∑k≥0 (−1)k ek tk)

    This is essentially the exp/ln conversion between the generating functions of power sums and elementary symmetric polynomials. It appears in problems involving cycle index polynomials, Burnside's lemma computations, and partition enumeration.

    7. Worked Example

    Problem: Given f(x) = x4 − 10x3 + 35x² − 50x + 24, compute p1 through p6.

    The coefficients give e1=10, e2=35, e3=50, e4=24. (These are the roots 1,2,3,4.)

    • p1 = e1 = 10
    • p2 = e1·p1 − 2e2 = 100 − 70 = 30
    • p3 = e1·p2 − e2·p1 + 3e3 = 300 − 350 + 150 = 100
    • p4 = e1·p3 − e2·p2 + e3·p1 − 4e4 = 1000 − 1050 + 500 − 96 = 354
    • p5 = e1·p4 − e2·p3 + e3·p2 − e4·p1 = 3540 − 3500 + 1500 − 240 = 1300
    • p6 = e1·p5 − e2·p4 + e3·p3 − e4·p2 = 13000 − 12390 + 5000 − 720 = 4890

    Verification: 16+26+36+46 = 1+64+729+4096 = 4890. ✓

    8. Complexity & Summary

    ek → pk
    Given e1..n, compute p1..K in O(n·K) time. Each pk requires summing min(k, n) terms.
    pk → ek
    Given p1..n, compute e1..n in O(n²) time. Requires modular inverse of k for each step.

    Both conversions can be sped up to O(n log² n) using the connection to polynomial exp/ln (since pk/k are the coefficients of −ln E(t) where E(t) = ∑ (−1)k ek tk). This is useful when n is in the tens of thousands.

    Key takeaway. Newton's identities let you freely convert between the elementary symmetric polynomials (which come from polynomial coefficients via Vieta's) and power sums (which appear in many combinatorial and algebraic problems). The naive conversion is O(n²); the fast version uses polynomial exp/ln in O(n log² n).
