    FPS Composition & Inverse

    July 2026 · 25 min read

    1. What Is FPS Composition?

    Given two formal power series (FPS) f(x) = ∑k≥0 ak xk and g(x) = ∑k≥0 bk xk with b0 = 0, the composition h(x) = f(g(x)) is defined as

    h(x) = ∑k≥0 ak g(x)k.

    The requirement b0 = 0 ensures that every coefficient of h depends on only finitely many terms, so the composition is well-defined in the ring of formal power series.

    Why does b0 = 0 matter? If g(0) ≠ 0, then g(x)k contributes to the constant term for every k, making the coefficient of x0 in h an infinite sum — undefined in the formal-power-series world (unless we work over a topological ring).

    Example. Let f(x) = 1/(1−x) = ∑ xk and g(x) = x/(1−x). Then f(g(x)) = 1/(1 − x/(1−x)) = (1−x)/(1−2x) = ∑n≥0 2n−1xn for n ≥ 1, with constant term 1.

    Naïve Algorithm

    The most straightforward approach computes g(x)k for k = 0, 1, …, n−1 and accumulates ak g(x)k. Each multiplication costs O(n) using the truncated product mod xn, so the total complexity is O(n2). 

    // Naive FPS composition: compute f(g(x)) mod x^n in O(n^2)
// poly is vector<long long>, MOD is a prime
poly compose_naive(const poly &f, const poly &g, int n) {
    poly result(n, 0);
    poly gpow(n, 0);   // g(x)^k mod x^n
    gpow[0] = 1;       // g(x)^0 = 1

    for (int k = 0; k < (int)f.size() && k < n; k++) {
        // result += f[k] * gpow
        for (int j = 0; j < n; j++)
            result[j] = (result[j] + f[k] * gpow[j]) % MOD;
        // gpow = gpow * g mod x^n
        poly tmp(n, 0);
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n && i + j < n; j++)
                tmp[i + j] = (tmp[i + j] + gpow[i] * g[j]) % MOD;
        gpow = tmp;
    }
    return result;
}

    This O(n2) routine is useful for small n (≤ 5000) or as a reference implementation to verify faster algorithms.

    2. The Brent–Kung Algorithm

    In 1978, Brent and Kung showed that FPS composition can be done in O(n1/2 · M(n)) operations, where M(n) is the cost of multiplying two degree-n polynomials (i.e., O(n log n) with NTT). The idea uses a “baby-step / giant-step” decomposition.

    Baby-Step / Giant-Step Decomposition

    Choose a parameter m = ⌈√n⌉. Write f as

    f(x) = ∑i=0m−1 Fi(x) · xim

    where each Fi is a polynomial of degree < m (the “baby-step” blocks). Now substitute g for x:

    f(g(x)) = ∑i=0m−1 Fi(g(x)) · g(x)im.

    Step 1 — Baby Steps

    Compute g(x)1, g(x)2, …, g(x)m mod xn. This requires m multiplications, each costing M(n), for a total of m · M(n) = O(√n · M(n)).

    Step 2 — Evaluate Baby Blocks

    For each block Fi, compute Fi(g(x)) mod xn. Since deg(Fi) < m, this is a linear combination of g(x)0, …, g(x)m−1. Viewing this as a matrix–vector product (with an m × n coefficient matrix times the m baby-step vectors), we can batch all m blocks into a single matrix product costing O(m · n) = O(n3/2) scalar operations — or, more carefully, O(m) polynomial additions.

    Step 3 — Giant Steps

    Set G = g(x)m. Accumulate the sum ∑ Fi(g(x)) · Gi with m multiplications by G, costing another m · M(n).

    Combining all steps: Total = O(m · M(n)) = O(√n · n log n), which is significantly better than O(n2) for large n.

    Practical note. Implementing Brent–Kung efficiently requires careful handling of polynomial truncation and NTT-friendly primes. In competitive programming, the naïve O(n2) or O(n√n) methods often suffice for n ≤ 105.

    3. Compositional Inverse

    Given an FPS f(x) = ∑ ak xk with a0 = 0 and a1 ≠ 0, the compositional inverse is the unique FPS g(x) satisfying

    f(g(x)) = g(f(x)) = x.

    Existence and Uniqueness

    Theorem. A formal power series f(x) = a1x + a2x2 + … has a compositional inverse if and only if a1 ≠ 0. The inverse is unique.

    Proof sketch. We construct g(x) = b1x + b2x2 + … coefficient by coefficient. From f(g(x)) = x, comparing coefficients of x1: a1 b1 = 1, so b1 = a1−1 (requires a1 ≠ 0). For xn with n ≥ 2, we get a1 bn plus a polynomial in b1, …, bn−1 equals 0, which uniquely determines bn. By induction, all coefficients are determined. ∎

    Naïve Computation

    Extracting coefficients one by one from the relation f(g(x)) = x gives an O(n2) algorithm. For competitive programming, this is often sufficient. The key recurrence is:

    bn = −a1−1 ∑k=2n ak [xn] g(x)k. 

    // Compositional inverse: given f with f[0]=0, f[1]!=0
// find g such that f(g(x)) = x, mod x^n
// O(n^2) implementation
poly comp_inverse_naive(const poly &f, int n) {
    long long a1_inv = mod_pow(f[1], MOD - 2, MOD);
    poly g(n, 0);
    g[1] = a1_inv;

    // gpow[k] = g(x)^k mod x^n
    vector<poly> gpow(n, poly(n, 0));
    gpow[0][0] = 1;
    gpow[1] = g;

    for (int coef = 2; coef < n; coef++) {
        // update gpow up to power coef
        gpow[coef] = multiply_trunc(gpow[coef - 1], g, n);

        // solve for g[coef]:
        // sum_{k=1}^{coef} f[k] * [x^coef] g^k = 0  (for coef >= 2)
        long long sum = 0;
        for (int k = 2; k <= coef && k < (int)f.size(); k++)
            sum = (sum + f[k] % MOD * gpow[k][coef]) % MOD;
        g[coef] = (MOD - sum % MOD) % MOD * a1_inv % MOD;

        // recompute gpow with updated g
        for (int k = 2; k <= coef; k++)
            gpow[k] = multiply_trunc(gpow[k - 1], g, n);
    }
    return g;
}
    Warning. The above naïve code recomputes gpow every iteration and is actually O(n3). A cleaner O(n2) approach avoids storing all powers. In practice, use the Lagrange inversion formula (next section) for an O(n log n) method.

    4. The Lagrange Inversion Formula

    The crown jewel of FPS composition theory is the Lagrange inversion formula, which gives a closed-form expression for the coefficients of a compositional inverse.

    Statement

    Theorem (Lagrange Inversion). Let f(x) = x/φ(x) where φ(0) ≠ 0, and let g be the compositional inverse of f (so f(g(x)) = x). Then for n ≥ 1:

    [xn] g(x) = (1/n) [xn−1] φ(x)n.

    Equivalently, if f(x) = a1x + a2x2 + … and g is its compositional inverse, then

    [xn] g(x) = (1/n) [xn−1] (x / f(x))n.

    Proof

    We derive the formula using the formal residue. Let y = g(x), so f(y) = x and dy/dx exists as a formal power series. By the chain rule, f′(y) · dy/dx = 1, giving dy/dx = 1/f′(y).

    For any formal power series H, the coefficient extraction [xn] H(g(x)) can be computed via the formal residue:

    [xn] H(g(x)) = (1/n) [yn−1] H′(y) · (y/f(y))n.

    Setting H(y) = y (so H′(y) = 1), we get

    [xn] g(x) = (1/n) [yn−1] (y/f(y))n.

    Writing f(x) = x/φ(x), we have y/f(y) = φ(y), and thus

    [xn] g(x) = (1/n) [yn−1] φ(y)n. ∎

    Algorithmic Consequence

    To compute all coefficients of g(x) mod xn, we need [xk−1] φ(x)k for k = 1, …, n−1. Computing φ(x)k for each k separately is O(n2), but there are faster approaches:

    • O(n log n): When φ has a nice closed form, use the relation g(x) = x · φ(g(x)) and Newton’s method on the equation y − xφ(y) = 0. Each Newton step doubles the number of correct coefficients, and each step costs O(n log n), yielding O(n log n) total.
    • O(n√n): Use the Brent–Kung trick on φ(x)k.
    // Lagrange inversion via Newton's method
// Given phi with phi[0] != 0, find g s.t. g = x * phi(g)
// i.e., f(x) = x/phi(x) and f(g(x)) = x
// O(n log n) using Newton iteration on F(y) = y - x*phi(y)
poly lagrange_inversion_newton(const poly &phi, int n) {
    // Start: g = phi[0] * x
    poly g = {0, phi[0]};

    for (int len = 2; len < 2 * n; len *= 2) {
        int cur = min(len, n);
        // Newton step: g_{new} = g - (g - x*phi(g)) / (1 - x*phi'(g))
        poly pg = compose_trunc(phi, g, cur);  // phi(g) mod x^cur
        poly xpg = shift_right(pg, cur);       // x * phi(g)

        poly residual = sub_poly(g, xpg, cur); // g - x*phi(g)

        poly dphi = derivative(phi);
        poly dpg = compose_trunc(dphi, g, cur); // phi'(g)
        poly xdpg = shift_right(dpg, cur);      // x * phi'(g)
        poly denom = sub_poly(one_poly(cur), xdpg, cur); // 1 - x*phi'(g)

        poly inv_denom = poly_inverse(denom, cur);
        poly correction = multiply_trunc(residual, inv_denom, cur);
        g = sub_poly(g, correction, cur);
        g.resize(cur);
    }
    g.resize(n);
    return g;
}

    5. Applications in Combinatorics

    The Lagrange inversion formula is a powerhouse in enumerative combinatorics. Here are the classic applications.

    Counting Labeled Rooted Trees (Cayley’s Formula)

    The exponential generating function for labeled rooted trees satisfies T(x) = x · eT(x). Here φ(y) = ey, and by Lagrange inversion:

    [xn] T(x) = (1/n) [yn−1] eny = (1/n) · nn−1/(n−1)! = nn−1/n!.

    Since T is an EGF, the number of labeled rooted trees on n vertices is n! · [xn] T(x) = nn−1, which is Cayley’s formula.

    Catalan Numbers

    The generating function for Catalan numbers satisfies C(x) = 1 + x C(x)2, or equivalently C(x) = (1 + x C(x)2). Rearranging: if we set g(x) = x C(x), then g = x(1 + g2/x2 · x2) — but more directly, Lagrange inversion on f(y) = y/(1+y)2 (since C = 1/(1 − xC), yielding f compositional inverse) gives

    Cn = (1/(n+1)) C(2n, n).

    General Tree Counting

    For many species of trees, the generating function satisfies an equation of the form T(x) = x · Φ(T(x)) where Φ encodes the allowed branching structure. Lagrange inversion immediately gives coefficients of T.

    Binary Trees
    T = x(1 + T)2
    φ(y) = (1 + y)2
    [xn]T = (1/n)C(2n−2, n−1)
    Plane Trees
    T = x/(1 − T)
    φ(y) = 1/(1−y)
    [xn]T = (1/n)C(2n−2, n−1)

    6. Complexity Summary & Contest Tips

    Operation
    Naïve composition
    Brent–Kung composition
    Compositional inverse (naïve)
    Lagrange inv. + Newton
    Complexity
    O(n2)
    O(√n · n log n)
    O(n2)
    O(n log n)

    When to use what in contests:

    • If n ≤ 5000, the naïve O(n2) composition is simplest and fast enough.
    • If the problem reduces to an equation T = xφ(T), recognize Lagrange inversion immediately and use Newton’s method for O(n log n).
    • The Brent–Kung algorithm is rarely needed in contests but essential for library problems with n ∼ 105 and arbitrary f, g.
    • Always check whether a combinatorial identity (Catalan, Cayley, etc.) short-circuits the computation entirely.
    Key takeaway. FPS composition is inherently harder than multiplication — no O(n log n) algorithm is known for general composition. The Brent–Kung O(n3/2 log n) bound is essentially the best we have. But compositional inverse via Lagrange inversion is O(n log n), which is the key insight for competitive programming.
