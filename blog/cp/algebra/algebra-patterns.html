    Algebra Problem Patterns

    July 2026 · 25 min read

    1. Overview — Recognising the Pattern

    After twenty-one posts building algebraic tools, this capstone distils everything into recognisable patterns. In contests you rarely think “I need to implement NTT”; instead you see a counting problem and must decide which algebraic lens to use. We group recurring themes into five pattern families:

    1. Reduce to polynomial multiplication — spot a convolution hiding in a counting formula.
    2. Generating-function approach — encode a sequence as a formal power series and manipulate algebraically.
    3. Linear recurrence detection — recognise that a sequence satisfies a short recurrence; apply Berlekamp–Massey + Kitamasa.
    4. Polynomial hashing tricks — exploit algebraic structure in string or array problems.
    5. Bitwise convolution & subset DP — optimise exponential DP using OR/AND/XOR/subset convolution.

    For each pattern we present the signature (how to recognise it), 2–3 example problems with solution sketches, and compact C++ implementations.

    2. Pattern 1 — Reduce to Polynomial Multiplication

    2.1 Signature

    You have two discrete distributions (or frequency arrays) a[0..n] and b[0..n], and you need the distribution of their sum: c[k] = ∑i+j=k a[i]·b[j]. This is textbook polynomial multiplication, solvable in O(n log n) via FFT/NTT.

    Key trigger. Whenever you count the number of ways to combine two independent choices whose “costs” add, the answer is a convolution.

    2.2 Example — Dice Sums (CF 553E style)

    Problem. Roll k dice, each with faces 1…d. How many ways to get each possible total?

    Solution. Let p(x) = x + x2 + … + xd. The answer array is the coefficient vector of p(x)k. Compute via repeated squaring: O(kd · log k) total using O(log k) NTT multiplications, each on polynomials of degree ≤ kd.

    // Compute p(x)^k mod x^{maxn} using NTT-based multiply
// ntt_multiply(a, b) returns the product of two polynomials mod prime
vector<long long> poly_power(vector<long long> base, int k, int maxn) {
    vector<long long> result = {1};  // polynomial "1"
    while (k > 0) {
        if (k & 1) {
            result = ntt_multiply(result, base);
            if ((int)result.size() > maxn) result.resize(maxn);
        }
        base = ntt_multiply(base, base);
        if ((int)base.size() > maxn) base.resize(maxn);
        k >>= 1;
    }
    return result;
}

    2.3 Example — Count Arrays with Given Element Sums

    Problem. Given arrays A and B of length n, for each k count the number of pairs (i, j) with A[i] + B[j] = k.

    Solution. Build frequency arrays freqA and freqB, convolve them. O(V log V) where V is the value range.

    2.4 Example — Wildcard String Matching (CF 528D)

    Problem. Given text T and pattern P (both with wildcards ‘?’), find all positions where P matches T.

    Solution sketch. Define weight functions: wT(i) = 0 if T[i] = ‘?’, else T[i], and similarly for P. The match score at shift s is:

    M(s) = ∑j wT(s+j) · wP(j) · (T[s+j] − P[j])2

    Expanding the square yields three convolutions, each computable via FFT. Positions where M(s) = 0 are matches. Total: O(n log n).

    3. Pattern 2 — Generating Function Approach

    3.1 Signature

    The problem asks for a sequence a0, a1, … satisfying some combinatorial definition. You encode it as A(x) = ∑ an xn and derive a closed-form or functional equation. Then extract coefficients using FPS operations (inverse, log, exp, etc.).

    3.2 Example — Counting Labelled Trees by Degree Sequence

    Problem. Count labelled rooted forests on n vertices where vertex i has exactly di children.

    Solution. By the exponential formula, if T(x) is the EGF for rooted labelled trees, then the EGF for forests is exp(T(x)). Since T(x) = x · exp(T(x)) (Cayley’s functional equation), we compute T(x) via Newton’s method on formal power series in O(n log n):

    // Newton's method to solve T = x * exp(T)
// Given: we want T(x) mod x^n
// F(T) = T - x*exp(T) = 0
// T_{k+1} = T_k - F(T_k) / F'(T_k)
//         = T_k - (T_k - x*exp(T_k)) / (1 - x*exp(T_k))
// Each step doubles precision; O(n log n) total.
vector<long long> solve_tree_egf(int n) {
    vector<long long> T = {0, 1}; // T = x initially
    for (int sz = 2; sz < n; sz <<= 1) {
        // Compute exp(T) mod x^sz
        auto eT = poly_exp(T, sz);
        // x * exp(T)
        vector<long long> xeT(sz + 1, 0);
        for (int i = 0; i < sz; i++) xeT[i + 1] = eT[i];
        xeT.resize(sz);
        // F = T - x*exp(T), F' = 1 - x*exp(T) (derivative w.r.t. T)
        // Newton update: T = T - F/F' = (x*exp(T) * (1 + T) - T) / ... simplified
        // Practical: use the identity T = x*exp(T) and iterate
        T = xeT;
        T.resize(sz);
    }
    T.resize(n);
    return T;
}

    3.3 Example — Partition Function (CF 1553I style)

    Problem. Compute p(n) = number of integer partitions of n, for all n ≤ N.

    Solution. The generating function is P(x) = ∏k≥1 1/(1 − xk). Taking log: ln P(x) = ∑k≥1 ∑j≥1 xkj/j. This is a harmonic sum computable in O(N log N). Then P(x) = exp(ln P(x)) via FPS exp in O(N log N).

    // Partition numbers mod p in O(N log N)
vector<long long> partition_numbers(int N, long long mod) {
    // Step 1: compute ln P(x) = sum_{k>=1} sum_{j>=1} x^{kj}/j
    vector<long long> logP(N + 1, 0);
    for (int k = 1; k <= N; k++)
        for (int j = 1; (long long)k * j <= N; j++)
            logP[k * j] = (logP[k * j] + mod_inv(j, mod)) % mod;

    // Step 2: P = exp(logP)
    return poly_exp_mod(logP, mod);
}

    3.4 Example — Catalan Convolutions

    Problem. Compute the k-th convolution of Catalan numbers: C(k)n = [xn] C(x)k where C(x) = (1 − √(1−4x))/(2x).

    Solution. Use C(x)k = exp(k · ln C(x)). Compute ln C(x) via FPS log, scale by k, then FPS exp. Total: O(n log n).

    4. Pattern 3 — Linear Recurrence Detection

    4.1 Signature

    A sequence a0, a1, … satisfies a linear recurrence of order d: an = ∑i=1d ri an−i. You are asked for aN where N can be up to 1018. Direct computation is O(Nd) — far too slow.

    Pipeline. (1) Compute the first ~2d terms of the sequence. (2) Use Berlekamp–Massey to recover the minimal recurrence. (3) Use Kitamasa (polynomial modular exponentiation) to find aN in O(d log d log N).

    4.2 Example — Fibonacci Variant

    Problem. f(n) = 2f(n−1) + 3f(n−2) − f(n−3), with f(0)=1, f(1)=2, f(2)=7. Find f(1018) mod 998244353.

    Solution. The characteristic polynomial is x3 − 2x2 − 3x + 1. Kitamasa computes xN mod char_poly using repeated squaring on polynomials of degree < 3:

    // Kitamasa: compute a[N] for a linear recurrence
// rec: coefficients r[1..d] such that a[n] = sum r[i]*a[n-i]
// init: initial values a[0..d-1]
long long kitamasa(vector<long long> rec, vector<long long> init, long long N, long long mod) {
    int d = rec.size();
    if (N < d) return init[N] % mod;

    // Characteristic polynomial: x^d - r[1]*x^{d-1} - ... - r[d]
    vector<long long> char_poly(d + 1, 0);
    char_poly[d] = 1;
    for (int i = 0; i < d; i++)
        char_poly[d - 1 - i] = (mod - rec[i]) % mod;

    // Compute x^N mod char_poly via repeated squaring
    vector<long long> cur = {1};    // x^0
    vector<long long> base = {0, 1}; // x^1
    long long e = N;
    while (e > 0) {
        if (e & 1) {
            cur = poly_mul_mod(cur, base, mod);
            cur = poly_mod(cur, char_poly, mod);
        }
        base = poly_mul_mod(base, base, mod);
        base = poly_mod(base, char_poly, mod);
        e >>= 1;
    }

    // a[N] = sum_{i=0}^{d-1} cur[i] * init[i]
    long long ans = 0;
    for (int i = 0; i < d && i < (int)cur.size(); i++)
        ans = (ans + cur[i] % mod * (init[i] % mod)) % mod;
    return ans;
}

    Complexity: O(d log d · log N) using NTT for polynomial multiplication, or O(d2 log N) with naïve multiplication (fine for small d).

    4.3 Example — Unknown Recurrence (CF 506E style)

    Problem. You can compute an for small n (say via DP), but the recurrence order is unknown. Find aN for large N.

    Solution. Compute the first 2D values where D is an upper bound on the recurrence order. Run Berlekamp–Massey to find the minimal linear recurrence, then apply Kitamasa.

    // Berlekamp-Massey: find shortest LFSR generating sequence s
vector<long long> berlekamp_massey(vector<long long> s, long long mod) {
    vector<long long> cur, prev;
    long long delta_prev = 1;
    int shift = 1;
    for (int i = 0; i < (int)s.size(); i++) {
        long long delta = s[i];
        for (int j = 0; j < (int)cur.size(); j++)
            delta = (delta + cur[j] % mod * s[i - 1 - j]) % mod;
        delta = (delta % mod + mod) % mod;
        if (delta == 0) { shift++; continue; }
        if (cur.empty()) {
            cur.resize(i + 1, 0);
            prev = cur;
            delta_prev = delta;
            shift = 1;
            continue;
        }
        long long coeff = delta % mod * mod_inv(delta_prev, mod) % mod;
        vector<long long> tmp = cur;
        if (cur.size() < prev.size() + shift)
            cur.resize(prev.size() + shift, 0);
        for (int j = 0; j < (int)prev.size(); j++)
            cur[j + shift] = (cur[j + shift] - coeff % mod * prev[j] % mod + mod) % mod;
        // Keep the longer sequence
        if ((int)tmp.size() <= (int)prev.size() + shift - 1) {
            prev = tmp;
            delta_prev = delta;
            shift = 1;
        } else {
            shift++;
        }
    }
    return cur; // cur[i] are negated coefficients of the recurrence
}

    4.4 Example — Grid Path Counting

    Problem. Count paths on a grid with forbidden cells. For small grids the answer satisfies a linear recurrence in the grid width. Compute initial terms via DP, apply BM + Kitamasa.

    5. Pattern 4 — Polynomial Hashing Tricks

    5.1 Signature

    String or array problems where you need to compare substructures efficiently. Polynomial (Rabin–Karp) hashing provides O(1) substring comparison after O(n) preprocessing. Algebraic tricks extend it to more complex queries.

    5.2 Example — Longest Common Substring

    Problem. Given strings S and T, find the longest common substring.

    Solution. Binary search on the length L. For each L, compute all rolling hash values of length-L substrings of both S and T; check for a common hash using a hash set. Total: O((|S|+|T|) log min(|S|,|T|)).

    // Rolling polynomial hash for substring matching
struct PolyHash {
    vector<long long> h, pw;
    long long base, mod;

    PolyHash(const string &s, long long b, long long m) : base(b), mod(m) {
        int n = s.size();
        h.resize(n + 1); pw.resize(n + 1);
        pw[0] = 1;
        for (int i = 0; i < n; i++) {
            h[i + 1] = (h[i] * base + s[i]) % mod;
            pw[i + 1] = pw[i] * base % mod;
        }
    }

    // Hash of s[l..r) in O(1)
    long long query(int l, int r) {
        return (h[r] - h[l] * pw[r - l] % mod + mod * 2) % mod;
    }
};

// Binary search for longest common substring
int longest_common_substr(const string &S, const string &T) {
    int lo = 0, hi = min(S.size(), T.size());
    long long B = 131, M = (1LL << 61) - 1;
    PolyHash hS(S, B, M), hT(T, B, M);
    while (lo < hi) {
        int mid = (lo + hi + 1) / 2;
        unordered_set<long long> seen;
        for (int i = 0; i + mid <= (int)S.size(); i++)
            seen.insert(hS.query(i, i + mid));
        bool found = false;
        for (int i = 0; i + mid <= (int)T.size(); i++)
            if (seen.count(hT.query(i, i + mid))) { found = true; break; }
        if (found) lo = mid; else hi = mid - 1;
    }
    return lo;
}

    5.3 Example — Period Detection via Hashing

    Problem. Find the shortest period of a string S of length n.

    Solution. A period p divides n and satisfies S[0..n−p) = S[p..n). Check each divisor of n in increasing order using O(1) hash comparison. Total: O(n + d(n)) where d(n) is the number of divisors.

    5.4 Example — 2D Pattern Matching

    Problem. Find all occurrences of a small 2D pattern P (r × c) inside a large grid G (R × C).

    Solution. Hash each row of G with a rolling hash, obtaining a 1D array of hashes for each row. Then hash vertically over r consecutive row-hashes to get a 2D hash. Compare with the 2D hash of P. Total: O(RC).

    6. Pattern 5 — Bitwise Convolution & Subset DP

    6.1 Signature

    An exponential DP over subsets that looks like:

    dp[S] = ∑T ⊆ S f(T) · g(S \ T)

    or more generally a sum over pairs whose bitwise combination equals the target. These are subset convolutions (or OR/AND/XOR convolutions) in disguise.

    6.2 Example — Chromatic Number (CF 914G approach)

    Problem. Given a graph with n ≤ 20 vertices, find the chromatic number.

    Solution. Let I(S) = 1 if S is an independent set, 0 otherwise. The number of proper k-colourings is: P(k) = ∑partition S1⊔…⊔Sk = V ∏ I(Si). This is the k-fold subset convolution of I with itself, evaluated at V. Compute for k = 1, 2, … until non-zero.

    // Chromatic number via subset convolution
// adj[v] = bitmask of neighbours of vertex v
int chromatic_number(int n, vector<int> &adj) {
    int full = (1 << n) - 1;

    // Precompute independent-set indicator
    vector<long long> indep(1 << n, 1);
    for (int S = 1; S < (1 << n); S++) {
        int v = __builtin_ctz(S); // lowest set bit
        // S is independent iff S\{v} is independent
        // and no neighbour of v is in S\{v}
        int rest = S ^ (1 << v);
        if (!indep[rest] || (rest & adj[v]))
            indep[S] = 0;
    }

    // Iterated subset convolution
    vector<long long> cur = indep; // k=1: single colour
    for (int k = 1; k <= n; k++) {
        if (cur[full] > 0) return k;
        if (k < n) cur = subset_conv(cur, indep, n);
    }
    return n;
}

    6.3 Example — Steiner Tree

    Problem. Given a weighted graph and a set of k terminal vertices, find the minimum-weight connected subgraph spanning all terminals (k ≤ 10).

    Solution. DP over subsets of terminals: dp[S][v] = minimum cost of a tree spanning terminal set S rooted at v. The key transition is a subset convolution:

    dp[S][v] = minT ⊂ S, T≠∅ (dp[T][v] + dp[S\T][v])

    Then relax with shortest paths (Dijkstra) for each S. Total: O(3k · n + 2k · n log n). The subset enumeration uses the standard “enumerate submasks” trick:

for (int T = (S - 1) & S; T > 0; T = (T - 1) & S) {

    6.4 Example — Counting Hamiltonian Paths

    Problem. Count Hamiltonian paths in a graph with n ≤ 20 vertices.

    Solution. Classic bitmask DP: dp[S][v] = number of paths visiting exactly vertex set S ending at v. This is O(2n · n2). With profile optimisation and the inclusion–exclusion principle, subset convolution can reduce constant factors.

    7. Decision Table — Which Technique to Use

    The following table maps problem characteristics to the recommended algebraic tool.

    Clue in Problem Statement Technique Complexity
    “Count pairs with a[i] + b[j] = k” FFT / NTT convolution O(n log n)
    “Product of generating functions”, “exponential formula” FPS (log, exp, inverse) O(n log n) per operation
    “Find aN” where N ≤ 1018 BM + Kitamasa O(d2 log N) or O(d log d log N)
    “Compare substrings”, “find pattern in text” Polynomial hashing O(n) preprocess, O(1) query
    “i OR j = k” or “sum over subsets” OR convolution (zeta/Möbius) O(n log n)
    “i AND j = k” or “sum over supersets” AND convolution O(n log n)
    “i XOR j = k” Walsh–Hadamard (XOR conv) O(n log n)
    “Partition S into disjoint parts” Subset convolution O(m² 2m)
    “Minimum Steiner tree”, “bitmask DP with combine” Subset DP + submask enumeration O(3m)
    Meta-strategy. When stuck, ask: “Is the answer a coefficient of a product of generating functions?” If yes, identify which convolution type (additive, OR, AND, XOR, subset) and apply the corresponding transform. If the answer satisfies a short recurrence, compute initial terms and use BM + Kitamasa.

    8. Series Recap & Further Reading

    This concludes the Algebra & Polynomials module. Across 22 posts we covered:

    • Foundations: modular arithmetic, FFT, NTT, arbitrary-mod convolution.
    • FPS operations: inverse, division, log, exp, square root, Newton’s method.
    • Applications: polynomial interpolation, multipoint evaluation, compositional inverse.
    • Linear recurrences: Berlekamp–Massey, Kitamasa, Cayley–Hamilton speedup.
    • Hashing: polynomial hashing, anti-hash attacks, multi-hash strategies.
    • Bitwise convolutions: OR, AND, XOR (Walsh–Hadamard), subset convolution, Möbius inversion.

    The unifying theme is that algebraic structure turns brute force into elegance. Whenever a problem asks you to combine distributions, count structured objects, or evaluate sequences at extreme indices, there is usually an algebraic approach that slashes the complexity from polynomial (or exponential) to near-linear.
