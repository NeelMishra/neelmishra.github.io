Meet in the Middle — Split-and-Merge & Subset Sum

July 2025 · 20 min read

The Meet-in-the-Middle Idea

Many brute-force problems enumerate an exponential search space — typically O(2n) for subset-style problems or O(kn) for combinatorial ones. When n is around 30–40, a full enumeration is hopeless (~1012 operations), but half that exponent (~220 ≈ 106) is very comfortable.

Meet in the Middle (MITM) exploits exactly this observation. The technique follows three steps:

  1. Split the input into two roughly equal halves, A and B.
  2. Enumerate all possibilities for each half independently.
  3. Merge the two lists of partial results to form valid full solutions.

The merge step is where the cleverness lies — it typically uses sorting + binary search, two pointers, or hashing so that the combination runs in O(2n/2 · log(2n/2)) = O(n · 2n/2) total time. This is a dramatic improvement over O(2n).

When to reach for MITM: The problem has exponential search space, n is between ~30 and ~45, and the partial results from each half can be combined efficiently (usually via addition, XOR, or some invertible operation).

The generic pseudocode skeleton looks like this:

// Generic MITM skeleton
void solve(vector<int>& items, int target) {
    int n = items.size();
    int half = n / 2;

    // Step 1 — enumerate left half
    vector<long long> left;
    for (int mask = 0; mask < (1 << half); mask++) {
        long long sum = 0;
        for (int i = 0; i < half; i++)
            if (mask & (1 << i)) sum += items[i];
        left.push_back(sum);
    }

    // Step 2 — sort left for binary search
    sort(left.begin(), left.end());

    // Step 3 — enumerate right half, merge
    int right_size = n - half;
    for (int mask = 0; mask < (1 << right_size); mask++) {
        long long sum = 0;
        for (int i = 0; i < right_size; i++)
            if (mask & (1 << i)) sum += items[half + i];
        long long need = target - sum;
        // binary search for 'need' in left[]
        if (binary_search(left.begin(), left.end(), need)) {
            cout << "Found subset with sum " << target << "\n";
            return;
        }
    }
    cout << "No solution\n";
}

Subset Sum

The most classic MITM application: given n integers (n ≤ 40) and a target T, determine whether any subset sums exactly to T.

Why not DP?

The standard DP approach runs in O(n · T) time. When element values can be up to 1015, the target is far too large for a DP table. MITM doesn't care about the magnitude of values — only about n.

Complete Implementation

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long T;
    cin >> n >> T;
    vector<long long> a(n);
    for (auto& x : a) cin >> x;

    int half = n / 2;
    int left_sz = half, right_sz = n - half;

    // Enumerate all subset sums of the left half
    vector<long long> left(1 << left_sz);
    for (int mask = 0; mask < (1 << left_sz); mask++) {
        long long s = 0;
        for (int i = 0; i < left_sz; i++)
            if (mask & (1 << i)) s += a[i];
        left[mask] = s;
    }

    sort(left.begin(), left.end());
    // Remove duplicates for counting variants — keep all for existence check
    left.erase(unique(left.begin(), left.end()), left.end());

    // Enumerate right half and binary search
    bool found = false;
    for (int mask = 0; mask < (1 << right_sz); mask++) {
        long long s = 0;
        for (int i = 0; i < right_sz; i++)
            if (mask & (1 << i)) s += a[half + i];
        long long need = T - s;
        if (binary_search(left.begin(), left.end(), need)) {
            found = true;
            break;
        }
    }

    cout << (found ? "YES" : "NO") << "\n";
    return 0;
}
Complexity: O(2n/2 · (n/2)) for enumeration, plus O(2n/2 · log(2n/2)) for the merge. Overall O(n · 2n/2). Memory is O(2n/2).

Counting the Number of Subsets

If the problem asks how many subsets sum to T, don't deduplicate. Use lower_bound / upper_bound to count occurrences: 

// Count subsets that sum to T
sort(left.begin(), left.end());
long long count = 0;
for (int mask = 0; mask < (1 << right_sz); mask++) {
    long long s = 0;
    for (int i = 0; i < right_sz; i++)
        if (mask & (1 << i)) s += a[half + i];
    long long need = T - s;
    count += upper_bound(left.begin(), left.end(), need)
           - lower_bound(left.begin(), left.end(), need);
}
cout << count << "\n";

Closest Subset Sum

Problem: Given n integers (n ≤ 40) and a target T, find the subset whose sum is closest to T.

The merge step changes from an exact binary search to finding the nearest value using lower_bound. 

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long T;
    cin >> n >> T;
    vector<long long> a(n);
    for (auto& x : a) cin >> x;

    int half = n / 2;
    int left_sz = half, right_sz = n - half;

    vector<long long> left(1 << left_sz);
    for (int mask = 0; mask < (1 << left_sz); mask++) {
        long long s = 0;
        for (int i = 0; i < left_sz; i++)
            if (mask & (1 << i)) s += a[i];
        left[mask] = s;
    }

    sort(left.begin(), left.end());
    left.erase(unique(left.begin(), left.end()), left.end());

    long long best = LLONG_MAX;  // best |sum - T|

    for (int mask = 0; mask < (1 << right_sz); mask++) {
        long long s = 0;
        for (int i = 0; i < right_sz; i++)
            if (mask & (1 << i)) s += a[half + i];
        long long need = T - s;

        // Find closest value in left[] to 'need'
        auto it = lower_bound(left.begin(), left.end(), need);
        if (it != left.end())
            best = min(best, abs(*it - need));
        if (it != left.begin()) {
            --it;
            best = min(best, abs(*it - need));
        }
    }

    cout << "Minimum difference: " << best << "\n";
    cout << "Closest sum: " << T - best << " or " << T + best << "\n";
    return 0;
}
Key insight: When looking for the closest value with lower_bound, always check both the iterator returned and the one before it. lower_bound gives the first element ≥ target, so the closest could be either that element or its predecessor.

Variant: Closest to T with Sum ≤ T

If the sum must not exceed T, use upper_bound and step back one position: 

auto it = upper_bound(left.begin(), left.end(), need);
if (it != left.begin()) {
    --it;
    best = min(best, need - *it);  // need - *it >= 0 guaranteed
}

4-SUM & Generalized Problems

MITM is the standard approach for k-SUM problems when k is even. The classic 4-SUM problem: given an array of n integers, find four elements that sum to a target T.

4-SUM in O(n² log n)

Split the four indices into two groups of two. Enumerate all O(n²) pairwise sums, then for each pair sum s, search for T − s in the sorted list. 

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long T;
    cin >> n >> T;
    vector<long long> a(n);
    for (auto& x : a) cin >> x;

    // Store all pair sums with their indices
    vector<pair<long long, pair<int,int>>> pairs;
    pairs.reserve((long long)n * (n - 1) / 2);
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            pairs.push_back({a[i] + a[j], {i, j}});

    sort(pairs.begin(), pairs.end());

    // For each pair (i,j), binary-search for T - a[i] - a[j]
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            long long need = T - a[i] - a[j];
            auto lo = lower_bound(pairs.begin(), pairs.end(),
                                  make_pair(need, make_pair(0, 0)));
            while (lo != pairs.end() && lo->first == need) {
                auto [pi, pj] = lo->second;
                // Ensure all four indices are distinct
                if (pi != i && pi != j && pj != i && pj != j) {
                    cout << a[i] << " " << a[j] << " "
                         << a[pi] << " " << a[pj] << "\n";
                    return 0;
                }
                ++lo;
            }
        }
    }

    cout << "No 4-SUM solution\n";
    return 0;
}

Generalization to k-SUM

For even k, split into k/2 + k/2. Enumerate all (k/2)-element sums in O(nk/2), then merge. For odd k, split into ⌊k/2⌋ and ⌈k/2⌉.

Watch out for duplicate indices. When merging pair sums, always verify that the selected elements don't overlap. This is the most common source of wrong answers in MITM k-SUM problems.

MITM with Constraints

Real contest problems rarely give a pure subset-sum. Common additional constraints include:

Cardinality Constraint

Problem: Find a subset of exactly k elements that sums to T.

Store pairs (sum, count) instead of just sums. In the merge, search for entries where left_count + right_count == k. 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    long long T;
    cin >> n >> k >> T;
    vector<long long> a(n);
    for (auto& x : a) cin >> x;

    int half = n / 2;
    int left_sz = half, right_sz = n - half;

    // left[c] = sorted list of sums using exactly c elements
    map<int, vector<long long>> left;
    for (int mask = 0; mask < (1 << left_sz); mask++) {
        long long s = 0;
        int cnt = __builtin_popcount(mask);
        for (int i = 0; i < left_sz; i++)
            if (mask & (1 << i)) s += a[i];
        left[cnt].push_back(s);
    }
    for (auto& [c, v] : left)
        sort(v.begin(), v.end());

    bool found = false;
    for (int mask = 0; mask < (1 << right_sz) && !found; mask++) {
        long long s = 0;
        int cnt = __builtin_popcount(mask);
        if (cnt > k) continue;
        for (int i = 0; i < right_sz; i++)
            if (mask & (1 << i)) s += a[half + i];

        int need_cnt = k - cnt;
        if (left.count(need_cnt)) {
            long long need = T - s;
            if (binary_search(left[need_cnt].begin(),
                              left[need_cnt].end(), need)) {
                found = true;
            }
        }
    }

    cout << (found ? "YES" : "NO") << "\n";
    return 0;
}

Modular Constraint

Problem: Count subsets whose sum ≡ r (mod m).

Store left sums grouped by their value mod m. For each right sum with remainder rr, look up left sums with remainder (r − rr + m) mod m. 

// Group left sums by residue mod m
map<int, long long> left_cnt;  // residue -> count
for (int mask = 0; mask < (1 << left_sz); mask++) {
    long long s = 0;
    for (int i = 0; i < left_sz; i++)
        if (mask & (1 << i)) s += a[i];
    left_cnt[((s % m) + m) % m]++;
}

long long answer = 0;
for (int mask = 0; mask < (1 << right_sz); mask++) {
    long long s = 0;
    for (int i = 0; i < right_sz; i++)
        if (mask & (1 << i)) s += a[half + i];
    int need_r = ((r - s % m) % m + m) % m;
    answer += left_cnt[need_r];
}
cout << answer << "\n";

XOR-based MITM

If the combination operation is XOR instead of addition, the technique works identically — enumerate XOR values of each half and search for the complement: 

// XOR variant: find subset with XOR == T
sort(left.begin(), left.end());
for (int mask = 0; mask < (1 << right_sz); mask++) {
    long long x = 0;
    for (int i = 0; i < right_sz; i++)
        if (mask & (1 << i)) x ^= a[half + i];
    long long need = T ^ x;  // since x ^ need == T
    if (binary_search(left.begin(), left.end(), need))
        found = true;
}
General principle: MITM works whenever the combination function is "invertible" — meaning given the total and one half's contribution, you can compute what the other half must contribute. Addition, XOR, multiplication (mod prime), and string concatenation all qualify.

MITM on Graphs

Meet in the Middle isn't limited to subset enumeration. The same principle applies to graph search — bidirectional BFS/DFS is MITM applied to shortest-path problems.

Bidirectional BFS

Standard BFS from source s to target t in an unweighted graph explores O(bd) nodes, where b is the branching factor and d is the distance. Bidirectional BFS runs two BFS instances — one from s, one from t — and stops when they meet. Each explores O(bd/2) nodes, for a combined O(bd/2) total. 

#include <bits/stdc++.h>
using namespace std;

int bidirectional_bfs(vector<vector<int>>& adj, int s, int t) {
    if (s == t) return 0;
    int n = adj.size();

    vector<int> dist_s(n, -1), dist_t(n, -1);
    queue<int> qs, qt;

    dist_s[s] = 0; qs.push(s);
    dist_t[t] = 0; qt.push(t);

    while (!qs.empty() || !qt.empty()) {
        // Expand the smaller frontier
        auto expand = [&](queue<int>& q, vector<int>& dist_from,
                           vector<int>& dist_other) -> int {
            int sz = q.size();
            while (sz--) {
                int u = q.front(); q.pop();
                for (int v : adj[u]) {
                    if (dist_from[v] != -1) continue;
                    dist_from[v] = dist_from[u] + 1;
                    if (dist_other[v] != -1)
                        return dist_from[v] + dist_other[v];
                    q.push(v);
                }
            }
            return -1;
        };

        int res;
        if (qs.size() <= qt.size())
            res = expand(qs, dist_s, dist_t);
        else
            res = expand(qt, dist_t, dist_s);

        if (res != -1) return res;
    }

    return -1;  // no path
}
Subtlety: Bidirectional BFS can give incorrect results on directed graphs if edges aren't reversible. For directed graphs, you must reverse the edge directions for the backward search — i.e., build a separate reverse adjacency list.

Worked Example: CF 888E — Maximum Subsequence

Statement: Given n ≤ 35 integers and a modulus m, find a non-empty subsequence whose sum mod m is maximized.

Split the array in half. For each half, enumerate all 2n/2 subset sums mod m. Sort the left list. For each right sum r, the best left value is the largest l such that (l + r) mod m is maximized. This equals finding the largest l < m − r (sum doesn't wrap) or the largest l overall (sum wraps around). 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    long long m;
    cin >> n >> m;
    vector<long long> a(n);
    for (auto& x : a) cin >> x;

    int half = n / 2;
    int left_sz = half, right_sz = n - half;

    auto enumerate = [&](int start, int len) {
        vector<long long> sums(1 << len);
        for (int mask = 0; mask < (1 << len); mask++) {
            long long s = 0;
            for (int i = 0; i < len; i++)
                if (mask & (1 << i))
                    s = (s + a[start + i]) % m;
            sums[mask] = s;
        }
        sort(sums.begin(), sums.end());
        sums.erase(unique(sums.begin(), sums.end()), sums.end());
        return sums;
    };

    auto left = enumerate(0, left_sz);
    auto right_sums = enumerate(half, right_sz);

    long long ans = 0;
    for (long long r : right_sums) {
        // Case 1: no wraparound — find largest l < m - r
        long long need = m - r - 1;
        auto it = upper_bound(left.begin(), left.end(), need);
        if (it != left.begin()) {
            --it;
            ans = max(ans, (*it + r) % m);
        }
        // Case 2: wraparound — use the largest l overall
        ans = max(ans, (left.back() + r) % m);
    }

    cout << ans << "\n";
    return 0;
}
Summary checklist for MITM problems:
  1. Verify n is in the ~30–45 range (signals MITM).
  2. Identify the combination operation (sum, XOR, mod, etc.).
  3. Split the input into two halves.
  4. Enumerate each half exhaustively — O(2n/2) or O(nk/2).
  5. Sort one half; binary-search or two-pointer merge from the other.
  6. Handle edge cases: empty subset, duplicate indices, overflow.
