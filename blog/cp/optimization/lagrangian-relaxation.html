    Lagrangian Relaxation

    July 2025 · 20 min read

    Many competitive-programming problems boil down to “optimise an objective subject to several constraints.” When one of those constraints makes the problem intractable, Lagrangian relaxation offers a principled way to remove it, replace it with a penalty in the objective, and still obtain strong bounds — or even exact solutions. This post develops the theory from scratch, then shows how to implement it in C++ and apply it to classic CP problems.

    What Is Lagrangian Relaxation?

    Consider a generic optimisation problem in its most common CP form:

    minimise   f(x)
subject to g(x) ≤ 0          (hard constraint)
           x ∈ S             (easy constraints)

    The set S captures constraints you can handle efficiently (shortest path, minimum spanning tree, etc.), while g(x) ≤ 0 is the troublesome one — for instance a budget limit on a shortest-path problem, or a deadline in a scheduling problem.

    Lagrangian relaxation removes the hard constraint and adds a penalty term to the objective:

    L(λ) = min { f(x) + λ · g(x) : x ∈ S }     (λ ≥ 0)

    Here λ (the Lagrange multiplier) controls how severely we penalise violations of g(x) ≤ 0. If λ is large, any solution with g(x) > 0 becomes expensive; if λ = 0, the constraint is ignored entirely.

    Key insight: For every λ ≥ 0, the value L(λ) is a lower bound on the optimal value of the original problem. This is because we are minimising over a larger feasible set (we dropped a constraint) and the penalty term is non-positive for any feasible solution.

    The idea works because any feasible solution x* of the original problem satisfies g(x*) ≤ 0, so λ · g(x*) ≤ 0 for λ ≥ 0. Therefore:

    L(λ) ≤ f(x*) + λ · g(x*) ≤ f(x*)

    This means every Lagrangian evaluation gives us a valid lower bound.

    The Lagrangian Dual

    Since every λ ≥ 0 yields a lower bound, we naturally want the best (tightest) lower bound:

    Lagrangian dual:   maximise  L(λ)   over λ ≥ 0

    This is the Lagrangian dual problem. Denote its optimal value by d* and the primal optimum by p*.

    Weak Duality

    By construction, d* ≤ p*. This always holds regardless of the structure of f, g, or S. The gap p* − d* is the duality gap.

    Strong Duality

    When does the gap vanish? For linear programs, strong duality (d* = p*) holds whenever the primal is feasible and bounded. For combinatorial problems over discrete sets, we get strong duality when the convex hull of the feasible set (after projecting out the relaxed constraint) is described exactly by the remaining constraints — in other words, when the LP relaxation of the subproblem has an integral polyhedron.

    In practice: Even when strong duality does not hold, the Lagrangian bound is at least as tight as the LP relaxation bound, and often significantly tighter. This makes it extremely useful for bounding in branch-and-bound and for generating near-optimal solutions.

    Properties of L(λ)

    The function L(λ) is a point-wise minimum of affine functions of λ (one affine function per x ∈ S). Therefore L(λ) is concave and piecewise linear, even when the original problem is combinatorial. This concavity is what makes the dual tractable.

    However, L(λ) is generally non-differentiable — it can have kinks where the optimal x changes. This rules out gradient descent and motivates subgradient methods.

    Subgradient Optimization

    Because L(λ) is concave but non-smooth, we maximise it using the subgradient method. A subgradient of L at λk is simply g(xk), where xk is the minimiser of the Lagrangian at λk.

    The Algorithm

    1. Initialise λ° ≥ 0, set best_lb = −∞, best_ub = +∞.
2. For k = 0, 1, 2, …
   a. Solve the inner problem:  x<sup>k</sup> = argmin { f(x) + λ<sup>k</sup> g(x) : x ∈ S }
   b. Update lower bound:  best_lb = max(best_lb, L(λ<sup>k</sup>))
   c. If x<sup>k</sup> is feasible for the original problem:
        best_ub = min(best_ub, f(x<sup>k</sup>))
   d. Compute subgradient:  s<sup>k</sup> = g(x<sup>k</sup>)
   e. Update multiplier:  λ<sup>k+1</sup> = max(0, λ<sup>k</sup> + t<sub>k</sub> · s<sup>k</sup>)
3. Stop when best_ub − best_lb < ε or after max iterations.

    Step-Size Rules

    The step size tk is critical. The most popular rule in CP/OR is the Polyak step size:

    tk = αk · (best_ub − L(λk)) / ||sk||²

    where αk ∈ (0, 2] is a scaling factor, typically starting at 2 and halved every N iterations without improvement. This converges to the optimal dual value under mild conditions.

    Alternative rules:

    • Constant step: tk = c — simple but converges only to a neighbourhood of the optimum.
    • Diminishing step: tk = a / (b + k) — converges, but slowly.
    • Polyak with target: Most effective in practice when a good upper bound is available.
    Tip for CP: When you know the optimal value is an integer, round best_ub down and best_lb up. If they meet, you have proven optimality. Also, if the inner problem always returns integer costs, ceil(best_lb) is a valid lower bound.

    Implementation

    Let us implement Lagrangian relaxation for the Resource-Constrained Shortest Path problem: find the shortest path from s to t in a directed graph where each edge has both a cost and a weight, and the total weight must not exceed a budget W.

    Problem Formulation

    minimise   ∑ ce xe          (total cost)
subject to ∑ we xe ≤ W      (weight budget)
           x is an s-t path        (network constraint)

    The weight budget is the hard constraint. We relax it:

    L(λ) = min { ∑ (ce + λ we) xe − λ W : x is an s-t path }
       = −λ W + shortest-path with edge costs (ce + λ we)

    So the inner problem is just a standard shortest path with modified edge weights — solvable by Dijkstra or Bellman-Ford.

    C++ Code

    #include <bits/stdc++.h>
using namespace std;

struct Edge {
    int to;
    double cost, weight;
};

struct State {
    double dist;
    int node;
    bool operator>(const State& o) const { return dist > o.dist; }
};

// Dijkstra with modified edge costs: cost_e + lambda * weight_e
// Returns {shortest_path_cost, total_weight_on_path, parent_array}
tuple<double, double, vector<int>>
dijkstra(const vector<vector<Edge>>& g, int s, int t, double lam) {
    int n = g.size();
    vector<double> dist(n, 1e18);
    vector<int> par(n, -1);
    vector<double> wt(n, 0);   // total weight to reach node
    vector<double> ct(n, 0);   // total original cost to reach node
    priority_queue<State, vector<State>, greater<State>> pq;

    dist[s] = 0;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u] + 1e-9) continue;
        for (auto& e : g[u]) {
            double nd = d + e.cost + lam * e.weight;
            if (nd < dist[e.to] - 1e-9) {
                dist[e.to] = nd;
                par[e.to] = u;
                wt[e.to] = wt[u] + e.weight;
                ct[e.to] = ct[u] + e.cost;
                pq.push({nd, e.to});
            }
        }
    }
    // Return original cost, total weight, and parent array
    return {ct[t], wt[t], par};
}

struct LagrangianResult {
    double lower_bound;
    double best_feasible_cost;
    vector<int> best_path;
};

LagrangianResult lagrangian_shortest_path(
    const vector<vector<Edge>>& g, int s, int t, double W,
    int max_iter = 500, double eps = 1e-6)
{
    double lam = 0.0;           // initial multiplier
    double best_lb = -1e18;
    double best_ub = 1e18;
    vector<int> best_path;
    double alpha = 2.0;
    int no_improve = 0;

    for (int iter = 0; iter < max_iter; iter++) {
        auto [orig_cost, path_weight, par] = dijkstra(g, s, t, lam);

        // Lagrangian value: original_cost + lambda * (weight - W)
        // But Dijkstra minimised (cost + lambda * weight), so:
        // L(lambda) = dijkstra_dist[t] - lambda * W
        //           = orig_cost + lam * path_weight - lam * W
        double L_val = orig_cost + lam * (path_weight - W);
        best_lb = max(best_lb, L_val);

        // Check feasibility
        if (path_weight <= W + eps) {
            if (orig_cost < best_ub) {
                best_ub = orig_cost;
                best_path.clear();
                for (int v = t; v != -1; v = par[v])
                    best_path.push_back(v);
                reverse(best_path.begin(), best_path.end());
            }
        }

        // Convergence check
        if (best_ub - best_lb < eps) break;

        // Subgradient: g(x) = path_weight - W
        double sg = path_weight - W;
        if (abs(sg) < eps) break;

        // Polyak step size
        double step = alpha * (best_ub - L_val) / (sg * sg);
        lam = max(0.0, lam + step * sg);

        // Halve alpha if no improvement
        if (L_val <= best_lb - eps) {
            no_improve++;
            if (no_improve >= 30) {
                alpha *= 0.5;
                no_improve = 0;
            }
        } else {
            no_improve = 0;
        }
    }

    return {best_lb, best_ub, best_path};
}

int main() {
    int n, m, s, t;
    double W;
    scanf("%d %d %d %d %lf", &n, &m, &s, &t, &W);

    vector<vector<Edge>> g(n);
    for (int i = 0; i < m; i++) {
        int u, v;
        double c, w;
        scanf("%d %d %lf %lf", &u, &v, &c, &w);
        g[u].push_back({v, c, w});
    }

    auto res = lagrangian_shortest_path(g, s, t, W);

    printf("Lower bound: %.4f\n", res.lower_bound);
    printf("Best feasible cost: %.4f\n", res.best_feasible_cost);
    printf("Path:");
    for (int v : res.best_path) printf(" %d", v);
    printf("\n");
    return 0;
}
    Implementation note: We track both the original cost and the modified cost separately. The Lagrangian value is computed from the original cost plus the penalty. The Dijkstra routine uses the modified costs for edge selection but records original costs for bound computation.

    Applications in CP

    Lagrangian relaxation appears in many competitive-programming and optimisation contexts. Here are the most common ones.

    1. Constrained Minimum Spanning Tree

    Given a graph where each edge has a cost and a colour, find a minimum-cost spanning tree using at most k edges of a given colour. Relax the colour constraint:

    L(λ) = min { ∑ ce xe + λ(∑ coloure xe − k) : x is a spanning tree }

    The inner problem is a standard MST with modified costs ce + λ · coloure. Solve with Kruskal’s algorithm.

    // Lagrangian relaxation for constrained MST
// Edges: {u, v, cost, is_coloured}
// Relax: at most k coloured edges
double constrained_mst_lagrangian(
    int n, vector<array<int,4>>& edges, int k)
{
    double lam = 0.0, best_lb = -1e18, best_ub = 1e18;
    double alpha = 2.0;
    int no_improve = 0;

    for (int iter = 0; iter < 300; iter++) {
        // Build modified costs
        int m = edges.size();
        vector<pair<double,int>> order(m);
        for (int i = 0; i < m; i++)
            order[i] = {edges[i][2] + lam * edges[i][3], i};
        sort(order.begin(), order.end());

        // Kruskal's MST
        vector<int> dsu(n);
        iota(dsu.begin(), dsu.end(), 0);
        function<int(int)> find = [&](int x) {
            return dsu[x] == x ? x : dsu[x] = find(dsu[x]);
        };

        double mst_cost = 0, orig_cost = 0;
        int coloured_count = 0;
        for (auto& [w, i] : order) {
            int a = find(edges[i][0]), b = find(edges[i][1]);
            if (a != b) {
                dsu[a] = b;
                mst_cost += w;
                orig_cost += edges[i][2];
                coloured_count += edges[i][3];
            }
        }

        double L_val = mst_cost - lam * k;
        best_lb = max(best_lb, L_val);

        if (coloured_count <= k)
            best_ub = min(best_ub, orig_cost);

        if (best_ub - best_lb < 1e-6) break;

        double sg = coloured_count - k;
        if (abs(sg) < 1e-9) break;

        double step = alpha * (best_ub - L_val) / (sg * sg);
        lam = max(0.0, lam + step * sg);

        if (++no_improve >= 30) { alpha *= 0.5; no_improve = 0; }
    }
    return best_lb;
}

    2. Scheduling with Deadlines

    Given n jobs with processing times, profits, and deadlines on a single machine, maximise profit while meeting deadlines. Relax the deadline constraints:

    L(λ) = max { ∑ pj xj − ∑ λj (Cj − dj) : schedule x }

    The inner problem decomposes into a weighted job scheduling problem with modified profits.

    3. Generalised Knapsack / Multi-Constraint Knapsack

    A knapsack with two capacity constraints (weight and volume). Relax the volume constraint:

    L(λ) = max { ∑ (vi − λ voli) xi + λ V : ∑ wi xi ≤ W }

    The inner problem is a standard 0/1 knapsack, solvable by DP. The Lagrangian bound is typically very tight for knapsack variants.

    4. Degree-Constrained Spanning Tree

    Find a minimum spanning tree where a specific vertex has degree at most d. Relax the degree constraint and solve a standard MST at each iteration. This is a classic application studied by Held and Karp.

    Duality Gap and Convergence

    Understanding when Lagrangian relaxation works well (and when it doesn’t) requires understanding the duality gap.

    When Is the Relaxation Tight?

    • Linear programs: The gap is always zero (strong duality).
    • Network flow problems: The constraint matrix is totally unimodular, so the LP relaxation is integral and the Lagrangian bound matches the LP bound exactly.
    • Spanning tree with one side constraint: Empirically very tight, often zero gap.
    • General integer programs: The Lagrangian bound is at least as good as the LP relaxation, and often strictly better when the relaxed constraints have special structure.

    Convergence of the Subgradient Method

    With the Polyak step-size rule and an exact upper bound, the subgradient method converges to the optimal dual value. In practice:

    • Convergence is typically fast in the first few iterations (the bound improves quickly).
    • Fine-tuning near the optimum is slow — subgradient methods do not converge linearly.
    • For CP, 100–500 iterations are usually sufficient since we need only integer bounds.

    Strengthening the Bound

    Several techniques can tighten the Lagrangian bound further:

    • Primal recovery: Combine Lagrangian solutions from multiple iterations to construct good feasible solutions.
    • Cut generation: Add violated cuts to the subproblem to tighten the formulation.
    • Variable fixing: If an item appears in every Lagrangian solution across many iterations with very different multipliers, it likely belongs in the optimal solution.
    Bounding quality: If the duality gap is zero and the subgradient method converges, you have proven optimality. Even with a nonzero gap, the Lagrangian bound combined with a feasible solution gives you a certificate of the solution quality: the true optimum lies between the lower and upper bounds.

    Worked Example

    Let us trace through a complete example to solidify the ideas. Consider finding the cheapest path from node 0 to node 3 in this graph with a weight budget of W = 5:

    Edges:  (from, to, cost, weight)
        (0, 1, 1, 4)
        (0, 2, 3, 1)
        (1, 3, 1, 3)
        (2, 3, 2, 2)

    Two s-t paths exist:

    • Path A: 0 → 1 → 3 — cost = 2, weight = 7 (infeasible, exceeds budget 5)
    • Path B: 0 → 2 → 3 — cost = 5, weight = 3 (feasible)

    The optimal solution is Path B with cost 5. Let us see how Lagrangian relaxation finds this.

    Iteration 0: λ = 0

    Modified edge costs: original costs (since λ = 0). Shortest path is A with cost 2. But weight = 7 > 5, so it is infeasible.

    • L(0) = 2 + 0 · (7 − 5) = 2
    • best_lb = 2, best_ub = +∞ (no feasible solution found yet… actually Path B has cost 5, but Dijkstra only returns the shortest)

    Subgradient: s = 7 − 5 = 2. Since best_ub is unknown, we use a heuristic target. Suppose we set best_ub = 5 (from a feasibility heuristic).

    Step: t = 2 · (5 − 2) / 4 = 1.5

    New λ = max(0, 0 + 1.5 · 2) = 3.0

    Iteration 1: λ = 3.0

    Modified edge costs:

    • (0,1): 1 + 3·4 = 13
    • (0,2): 3 + 3·1 = 6
    • (1,3): 1 + 3·3 = 10
    • (2,3): 2 + 3·2 = 8

    Shortest path: 0 → 2 → 3 with modified cost 14. Original cost = 5, weight = 3.

    • L(3) = 5 + 3 · (3 − 5) = 5 − 6 = −1
    • best_lb = max(2, −1) = 2
    • best_ub = 5 (path is feasible)

    Subgradient: s = 3 − 5 = −2 (negative — we overshot the constraint).

    Step: t = 2 · (5 − (−1)) / 4 = 3.0

    New λ = max(0, 3.0 + 3.0 · (−2)) = max(0, −3) = 0

    Iteration 2: λ = 0 (again)

    The method oscillates. In practice, reducing α stabilises the iterations. After several rounds with decreasing α, the multiplier converges near λ ≈ 1.5, yielding:

    L(1.5) = min-path cost with edges (c + 1.5w) − 1.5 · 5

Modified costs: (0,1)=7, (0,2)=4.5, (1,3)=5.5, (2,3)=5
Path B: modified cost = 9.5
L(1.5) = 5 + 1.5(3-5) = 5 - 3 = 2

Path A: modified cost = 12.5
L(1.5) via A = 2 + 1.5(7-5) = 2 + 3 = 5... but we take min, so path B.

    At λ = 1.5 both paths map to L = 2 via path B. The best lower bound remains 2 and the best feasible upper bound is 5. The duality gap is 3 here — which is expected since there is no path with cost between 2 and 5 in this small example.

    Takeaway: The Lagrangian bound (2) proves that no feasible path costs less than 2. Since we found a feasible path of cost 5 and the integrality gap is unavoidable in this tiny example, the bound is still useful — it certifies that our solution is at most 3 units from optimal. In larger instances, the gap is typically much smaller relative to the optimal value.

    Complete Trace in C++

    // Build the example graph and run Lagrangian relaxation
int main() {
    int n = 4;
    vector<vector<Edge>> g(n);
    g[0].push_back({1, 1.0, 4.0});
    g[0].push_back({2, 3.0, 1.0});
    g[1].push_back({3, 1.0, 3.0});
    g[2].push_back({3, 2.0, 2.0});

    auto res = lagrangian_shortest_path(g, 0, 3, 5.0);

    printf("Lower bound: %.2f\n", res.lower_bound);      // ~2.00
    printf("Upper bound: %.2f\n", res.best_feasible_cost); // 5.00
    printf("Path:");
    for (int v : res.best_path) printf(" %d", v);          // 0 2 3
    printf("\n");

    return 0;
}

    Summary

    Lagrangian relaxation is one of the most versatile tools for constrained optimisation in competitive programming:

    • Remove hard constraints by adding penalty terms with multipliers λ.
    • Each evaluation of the Lagrangian subproblem yields a lower bound (for minimisation).
    • The Lagrangian dual finds the tightest such bound by maximising over λ.
    • The subgradient method with Polyak step sizes solves the dual iteratively.
    • The inner subproblem should be easy (shortest path, MST, knapsack DP).
    • The bound is at least as strong as the LP relaxation, often strictly stronger.
    • Even when the duality gap is nonzero, you get a quality certificate for your feasible solution.

    Master this technique and you gain a principled approach to problems where a single constraint turns a polynomial problem into an NP-hard one — exactly the kind of problems that appear in advanced CP contests.
