    Golden Section Search & Fibonacci Search

    July 2025 · 22 min read

    Introduction to Golden Section Search

    When optimizing a unimodal function over a continuous interval—one that has exactly one local minimum (or maximum)—we need a strategy that narrows the search interval as efficiently as possible. Ternary search is the most common approach in competitive programming, but it isn't optimal. Golden section search achieves the same goal with fewer function evaluations per iteration by exploiting a remarkable property of the golden ratio.

    The Golden Ratio φ

    The golden ratio is defined as:

    φ = (1 + √5) / 2 ≈ 1.6180339887...

Its conjugate:
ψ = (φ - 1) = 1/φ = (√5 - 1) / 2 ≈ 0.6180339887...

    The key algebraic identity is φ² = φ + 1, which equivalently gives us 1/φ = φ - 1. This self-similar property is what makes golden section search work: when we place two probe points inside an interval using the golden ratio, discarding one side leaves us with an interval where one probe is already correctly positioned for the next iteration.

    Unimodal Functions

    A function f is unimodal on [a, b] if there exists a point x* in [a, b] such that f is strictly decreasing on [a, x*] and strictly increasing on [x*, b] (for minimization). Many CP problems reduce to finding the minimum or maximum of such a function: optimal position problems, minimizing cost functions, or maximizing concave utility functions.

    Key Insight: For any unimodal function, if we evaluate f at two interior points x1 < x2 in [a, b], then:
    • If f(x1) < f(x2), the minimum lies in [a, x2]
    • If f(x1) > f(x2), the minimum lies in [x1, b]
    This lets us shrink the interval each iteration. The question is: where should we place x1 and x2 to shrink it fastest?

    Algorithm and Implementation

    How It Works

    Golden section search places two interior points at fractions ψ and ψ² of the interval width from the left endpoint (equivalently, at 1 - ψ and ψ from the right). Because ψ² = 1 - ψ, these two placements are symmetric in a specific way:

    Interval: [a ────── x1 ────── x2 ────── b]

x1 = a + (1 - ψ) * (b - a)   = a + ψ² * (b - a)
x2 = a + ψ * (b - a)

where ψ = (√5 - 1) / 2 ≈ 0.618

    After comparing f(x1) and f(x2):

    • If f(x1) < f(x2): The new interval becomes [a, x2]. The old x1 becomes the new x2 of the smaller interval, so we only need to evaluate f at one new point.
    • If f(x1) ≥ f(x2): The new interval becomes [x1, b]. The old x2 becomes the new x1 of the smaller interval, again requiring only one new evaluation.

    Each iteration shrinks the interval by a factor of ψ ≈ 0.618 and requires only one new function evaluation (after the first iteration which needs two). This is the fundamental advantage over ternary search, which requires two new evaluations per iteration.

    C++ Implementation — Finding the Minimum

    #include <bits/stdc++.h>
using namespace std;

// Golden section search: finds x in [a, b] that minimizes f(x)
// f must be unimodal (single minimum) on [a, b]
template <typename F>
double golden_section_min(F f, double a, double b, double eps = 1e-9) {
    const double phi = (sqrt(5.0) - 1.0) / 2.0; // ≈ 0.618

    double x1 = b - phi * (b - a);
    double x2 = a + phi * (b - a);
    double f1 = f(x1);
    double f2 = f(x2);

    while (b - a > eps) {
        if (f1 < f2) {
            b = x2;
            x2 = x1;
            f2 = f1;
            x1 = b - phi * (b - a);
            f1 = f(x1);
        } else {
            a = x1;
            x1 = x2;
            f1 = f2;
            x2 = a + phi * (b - a);
            f2 = f(x2);
        }
    }
    return (a + b) / 2.0;
}

int main() {
    // Example: minimize f(x) = (x - 3)^2 + 1 on [0, 10]
    auto f = [](double x) { return (x - 3) * (x - 3) + 1; };

    double ans = golden_section_min(f, 0.0, 10.0);
    printf("Minimum at x = %.10f\n", ans);
    printf("f(x) = %.10f\n", f(ans));
    // Output: Minimum at x = 3.0000000000, f(x) = 1.0000000000
}

    Iteration-Count Version

    In competitive programming, you sometimes want a fixed number of iterations rather than an epsilon-based termination. With N iterations, the interval shrinks to (b - a) * ψ^N:

    template <typename F>
double golden_section_fixed(F f, double a, double b, int iterations = 200) {
    const double phi = (sqrt(5.0) - 1.0) / 2.0;

    double x1 = b - phi * (b - a);
    double x2 = a + phi * (b - a);
    double f1 = f(x1);
    double f2 = f(x2);

    for (int i = 0; i < iterations; i++) {
        if (f1 < f2) {
            b = x2;
            x2 = x1;
            f2 = f1;
            x1 = b - phi * (b - a);
            f1 = f(x1);
        } else {
            a = x1;
            x1 = x2;
            f1 = f2;
            x2 = a + phi * (b - a);
            f2 = f(x2);
        }
    }
    return (a + b) / 2.0;
}
    Practical tip: 200 iterations of golden section search shrinks the interval by a factor of 0.618^200 ≈ 10^{-42}, which is far beyond the precision of double. For most CP problems, 100 iterations is more than enough.

    Finding the Maximum

    To find the maximum of a unimodal function (single peak), simply negate the function or flip the comparison:

    template <typename F>
double golden_section_max(F f, double a, double b, double eps = 1e-9) {
    return golden_section_min([&](double x) { return -f(x); }, a, b, eps);
}

    Why the Golden Ratio Is Optimal

    The Reuse Property

    The crucial advantage of golden section search is that it reuses one function evaluation from the previous iteration. Let's see why the golden ratio is the unique solution that makes this possible.

    Suppose we place probe points at fractions r and 1 - r of the interval (measured from the left), where 0 < r < 0.5. After eliminating one side, the interval shrinks by factor (1 - r). For the surviving probe point to land at the correct position in the new interval, we need:

    Position of surviving probe in new interval = r / (1 - r)
This must equal (1 - r)  [the "far" probe position in the new interval]

So: r / (1 - r) = 1 - r
    r = (1 - r)²
    r = 1 - 2r + r²
    r² - 3r + 1 = 0

Solving: r = (3 - √5) / 2 = 1 - φ ≈ 0.382

And 1 - r = (√5 - 1) / 2 = ψ ≈ 0.618

    This is exactly the golden ratio conjugate! No other ratio has this reuse property. This makes golden section search the optimal method among all strategies that use symmetric probe placement in each step.

    Comparison with Ternary Search

    Ternary search divides the interval into three equal parts, placing probes at 1/3 and 2/3. Each iteration:

    • Ternary search: Shrinks interval to 2/3 ≈ 0.667 of its size, uses 2 evaluations. Per evaluation: 0.667^(1/2) ≈ 0.816 shrinkage.
    • Golden section: Shrinks interval to 0.618 of its size, uses 1 evaluation (after startup). Per evaluation: 0.618 shrinkage.

    Golden section search is about 24% more efficient per function evaluation. After N evaluations:

    Ternary search interval:  (2/3)^(N/2)    [2 evals per iteration]
Golden section interval:  0.618^(N-1)    [1 eval per iteration, +1 startup]

After 50 evaluations:
  Ternary:  (2/3)^25  ≈ 1.2 × 10^{-5}
  Golden:   0.618^49  ≈ 2.3 × 10^{-11}

Golden section is ~6 orders of magnitude tighter!
    Why does CP still use ternary search? Because the difference rarely matters. Both converge exponentially, and with floating-point arithmetic, ~100–200 iterations of either method reaches machine precision. Ternary search is simpler to remember and implement under contest pressure. Use golden section when function evaluations are expensive (e.g., each call runs an O(N log N) subroutine).

    Information-Theoretic View

    Each function evaluation gives us one bit of information (which side to discard). The maximum shrinkage per bit of information is achieved when each evaluation reduces the interval by the same ratio—which is exactly what golden section search does. The shrinkage factor ψ ≈ 0.618 is optimal: it is the unique fixed point of the equation r = 1 - r² arising from the reuse constraint.

    Fibonacci Search

    Motivation

    Golden section search uses the same ratio ψ at every step, which is elegant but not quite optimal when the total number of evaluations is known in advance. Fibonacci search is the truly optimal method for a fixed evaluation budget—it uses Fibonacci numbers to determine probe placement and achieves the maximum possible interval reduction.

    The relationship is natural: the ratio of consecutive Fibonacci numbers converges to φ, so Fibonacci search is essentially golden section search with slightly adjusted ratios in the early iterations.

    Algorithm

    Given an interval [a, b] and a budget of n function evaluations, Fibonacci search proceeds as follows:

    1. Precompute Fibonacci numbers F[0], F[1], ..., F[n+1].
    2. Set initial probes at distances F[n-1]/F[n+1] and F[n]/F[n+1] from the left endpoint.
    3. At each step k, after discarding one side, the surviving probe is at the correct Fibonacci ratio for the remaining budget n - k.
    4. The interval shrinks by a factor of F[n] / F[n+1] in the first step, F[n-1] / F[n] in the second, etc.

    C++ Implementation

    #include <bits/stdc++.h>
using namespace std;

// Fibonacci search: finds x in [a, b] that minimizes f(x)
// Uses exactly `n` function evaluations (n >= 2)
template <typename F>
double fibonacci_search(F f, double a, double b, int n) {
    // Precompute Fibonacci numbers
    vector<double> fib(n + 2);
    fib[0] = 1;
    fib[1] = 1;
    for (int i = 2; i < n + 2; i++)
        fib[i] = fib[i - 1] + fib[i - 2];

    // Initial probe points
    double x1 = a + (fib[n - 1] / fib[n + 1]) * (b - a);
    double x2 = a + (fib[n] / fib[n + 1]) * (b - a);
    double f1 = f(x1);
    double f2 = f(x2);

    for (int k = 0; k < n - 2; k++) {
        if (f1 < f2) {
            b = x2;
            x2 = x1;
            f2 = f1;
            // New x1 using Fibonacci ratio for remaining budget
            x1 = a + (fib[n - k - 2] / fib[n - k]) * (b - a);
            f1 = f(x1);
        } else {
            a = x1;
            x1 = x2;
            f1 = f2;
            x2 = a + (fib[n - k - 1] / fib[n - k]) * (b - a);
            f2 = f(x2);
        }
    }

    // Final step: x1 and x2 coincide (or are very close)
    // Use a small perturbation for the last comparison
    if (f1 < f2)
        b = x2;
    else
        a = x1;

    return (a + b) / 2.0;
}

int main() {
    auto f = [](double x) { return (x - 2.5) * (x - 2.5) + 3; };

    // Use exactly 30 function evaluations
    double ans = fibonacci_search(f, 0.0, 10.0, 30);
    printf("Minimum at x = %.10f\n", ans);
    printf("f(x) = %.10f\n", f(ans));
}

    Why Fibonacci Search Is Optimal

    Kiefer (1953) proved that among all sequential search strategies for a unimodal function with a fixed budget of n evaluations, Fibonacci search achieves the smallest possible final interval:

    Final interval = (b - a) / F[n+1]

where F[n+1] is the (n+1)-th Fibonacci number.

For n = 30: F[31] = 1346269, so the interval shrinks to ~7.4 × 10^{-7}
For n = 50: F[51] ≈ 2 × 10^{10}, shrinkage to ~5 × 10^{-11}
    Golden section as limit of Fibonacci search: As the evaluation budget n → ∞, the Fibonacci ratios F[k]/F[k+1] → 1/φ = ψ, and Fibonacci search converges to golden section search. In practice, for large budgets (> 20 evaluations), the difference is negligible.

    When to Use Fibonacci Search

    Fibonacci search shines when:

    • Function evaluations are extremely expensive (physical experiments, complex simulations).
    • You have a strict, known budget of evaluations.
    • You need provably optimal worst-case performance.

    In CP, golden section search is almost always preferred for its simplicity. Fibonacci search is mainly of theoretical interest, but understanding it deepens your grasp of why the golden ratio appears in optimization.

    Comparison: Ternary vs Golden Section vs Fibonacci

    Evaluation Efficiency

    The following table compares the three methods for minimizing a unimodal function on [a, b] to within tolerance ε:

    Method Evals/Iteration Shrinkage/Iter Shrinkage/Eval Evals for 10-9
    Ternary Search 2 0.667 0.816 ~102
    Golden Section 1* 0.618 0.618 ~43
    Fibonacci Search 1* ~0.618 ~0.618 ~43

    * After the initial 2 evaluations; Fibonacci search is marginally better in the first few steps.

    Convergence Rate Analysis

    Let's verify the evaluation counts with a concrete example. Starting from an interval of width 1, how many evaluations does each method need to reach width ≤ 10^{-9}?

    #include <bits/stdc++.h>
using namespace std;

int main() {
    double target = 1e-9;

    // Ternary search: 2 evals per iteration, shrinks by 2/3
    int ternary_evals = 0;
    double width = 1.0;
    while (width > target) {
        width *= 2.0 / 3.0;
        ternary_evals += 2;
    }
    printf("Ternary search: %d evaluations\n", ternary_evals);

    // Golden section: 1 eval per iteration (after first 2)
    int golden_evals = 2;
    width = 1.0;
    double phi = (sqrt(5.0) - 1.0) / 2.0;
    width *= phi;  // first iteration
    while (width > target) {
        width *= phi;
        golden_evals += 1;
    }
    printf("Golden section: %d evaluations\n", golden_evals);

    // Fibonacci search: count Fibonacci numbers until F[n+1] > 1/target
    vector<double> fib = {1, 1};
    while (fib.back() < 1.0 / target)
        fib.push_back(fib[fib.size()-1] + fib[fib.size()-2]);
    printf("Fibonacci search: %d evaluations\n", (int)fib.size() - 2);

    return 0;
}
// Output:
// Ternary search: 102 evaluations
// Golden section: 45 evaluations
// Fibonacci search: 43 evaluations

    A Complete CP Template

    Here is a ready-to-use competitive programming template that supports both minimization and maximization, with configurable precision:

    #include <bits/stdc++.h>
using namespace std;

// Minimize f on [lo, hi]. Returns the x that minimizes f.
template <typename F>
double gss_min(F f, double lo, double hi) {
    const double gr = (sqrt(5.0) - 1.0) / 2.0;
    double a = hi - gr * (hi - lo);
    double b = lo + gr * (hi - lo);
    double fa = f(a), fb = f(b);
    for (int i = 0; i < 200; i++) {
        if (fa < fb) {
            hi = b; b = a; fb = fa;
            a = hi - gr * (hi - lo);
            fa = f(a);
        } else {
            lo = a; a = b; fa = fb;
            b = lo + gr * (hi - lo);
            fb = f(b);
        }
    }
    return (lo + hi) / 2.0;
}

// Maximize f on [lo, hi]
template <typename F>
double gss_max(F f, double lo, double hi) {
    return gss_min([&](double x) { return -f(x); }, lo, hi);
}

int main() {
    // Example: maximize f(x) = -x^2 + 4x + 5 on [0, 5]
    auto f = [](double x) { return -x * x + 4 * x + 5; };
    double x_max = gss_max(f, 0.0, 5.0);
    printf("Max at x = %.6f, f(x) = %.6f\n", x_max, f(x_max));
    // Output: Max at x = 2.000000, f(x) = 9.000000
}

    Advanced Techniques

    Golden Section on Integers

    Some CP problems require finding the optimal integer argument. We can adapt golden section search for integer domains:

    // Golden section search on integers: minimize f on [lo, hi]
template <typename F>
long long gss_int_min(F f, long long lo, long long hi) {
    while (hi - lo > 2) {
        long long m1 = lo + (hi - lo) / 3;
        long long m2 = hi - (hi - lo) / 3;
        if (f(m1) < f(m2))
            hi = m2;
        else
            lo = m1;
    }
    // Check remaining candidates
    long long best = lo;
    auto best_val = f(lo);
    for (long long x = lo + 1; x <= hi; x++) {
        auto val = f(x);
        if (val < best_val) {
            best_val = val;
            best = x;
        }
    }
    return best;
}
    Note: For integer domains, the golden ratio advantage is less pronounced since we can't place probes at arbitrary positions. The above code falls back to ternary-style division, which is fine for integers. The real advantage of golden section shows up in continuous optimization.

    Multidimensional Optimization via Coordinate Descent

    Golden section search can be used as a building block for multidimensional optimization through coordinate descent: optimize one variable at a time while holding others fixed.

    // 2D optimization using coordinate descent with golden section
// Finds (x, y) that minimizes f(x, y) in [x_lo, x_hi] x [y_lo, y_hi]
template <typename F>
pair<double,double> coord_descent_2d(
    F f, double x_lo, double x_hi, double y_lo, double y_hi, int rounds = 50
) {
    double x = (x_lo + x_hi) / 2.0, y = (y_lo + y_hi) / 2.0;

    for (int r = 0; r < rounds; r++) {
        // Optimize x with y fixed
        x = gss_min([&](double xi) { return f(xi, y); }, x_lo, x_hi);
        // Optimize y with x fixed
        y = gss_min([&](double yi) { return f(x, yi); }, y_lo, y_hi);
    }
    return {x, y};
}

// Example: minimize (x-1)^2 + (y-2)^2 + xy
int main() {
    auto f = [](double x, double y) {
        return (x-1)*(x-1) + (y-2)*(y-2) + x*y;
    };
    auto [x, y] = coord_descent_2d(f, -10, 10, -10, 10);
    printf("Min at (%.6f, %.6f), f = %.6f\n", x, y, f(x, y));
}

    Handling Numerical Precision

    A common pitfall in golden section search is floating-point error accumulation. Here are robust practices:

    • Use fixed iterations (100–200) rather than epsilon-based termination to avoid infinite loops near machine precision.
    • Recompute probe points from a and b each iteration rather than updating incrementally, to prevent drift.
    • Avoid comparing f(x1) == f(x2): when values are equal, pick either branch consistently (e.g., always shrink from the right).

    Example Walkthrough: Minimum Distance to a Parabola

    Given a point (px, py) and a parabola y = ax² + bx + c, find the closest point on the parabola:

    #include <bits/stdc++.h>
using namespace std;

int main() {
    double a, b, c, px, py;
    scanf("%lf %lf %lf %lf %lf", &a, &b, &c, &px, &py);

    // Distance squared from (px, py) to point (x, ax^2+bx+c)
    // is unimodal in x (for convex parabolas, a > 0)
    auto dist_sq = [&](double x) {
        double y = a * x * x + b * x + c;
        return (x - px) * (x - px) + (y - py) * (y - py);
    };

    // Search over a generous range
    double lo = -1e6, hi = 1e6;
    const double gr = (sqrt(5.0) - 1.0) / 2.0;
    double x1 = hi - gr * (hi - lo);
    double x2 = lo + gr * (hi - lo);
    double f1 = dist_sq(x1), f2 = dist_sq(x2);

    for (int i = 0; i < 300; i++) {
        if (f1 < f2) {
            hi = x2; x2 = x1; f2 = f1;
            x1 = hi - gr * (hi - lo);
            f1 = dist_sq(x1);
        } else {
            lo = x1; x1 = x2; f1 = f2;
            x2 = lo + gr * (hi - lo);
            f2 = dist_sq(x2);
        }
    }

    double best_x = (lo + hi) / 2.0;
    printf("%.10f\n", sqrt(dist_sq(best_x)));
    return 0;
}

    Tips for CP Contests

    Golden Section Search Checklist for Contests:
    1. Verify unimodality — is the function truly single-peaked/single-valley on your interval?
    2. Choose the right range — ensure the optimum lies strictly inside [a, b].
    3. Use fixed iterations — 200 iterations is safe for any double precision problem.
    4. Watch for flat regions — if f(x1) == f(x2) due to precision, shrink from either side consistently.
    5. Consider ternary search first — if evaluations are cheap, ternary search is simpler and sufficient.
