Random Walk on a Tree

A random walk on a tree is among the most elegant objects in probabilistic combinatorics. At each step, the walker at node v moves to a uniformly random neighbor of v. Unlike random walks on general graphs, the tree structure gives us powerful tools for exact analysis.

Consider an unweighted tree T = (V, E) with n nodes. A walker starts at node s. At each discrete time step, the walker at node v moves to one of v’s neighbors, chosen uniformly at random. The key quantity we want to compute is the hitting time H(s, t) — the expected number of steps to first reach node t starting from node s.

On a tree, the path between any two nodes is unique. This means the random walk must eventually traverse every edge on the s-t path, though it will likely detour into subtrees along the way. These detours are what make the hitting time larger than the simple graph distance.

Fundamental Property: On a tree, the expected time to traverse a single edge (u, v) from u to v equals 2 · |Tv| − 1, where |Tv| is the size of the subtree on v’s side when edge (u, v) is removed. This is because the walk must “escape” the subtree on u’s side before crossing to v.

The Edge-Crossing Formula

Remove edge (u, v) to split the tree into components of sizes a and b = n − a. The hitting time decomposes as a sum over edges on the s-t path. For each such edge, the contribution is 2a − 1 (where a is the component size on the starting side). This leads to the commute time formula: H(s, t) + H(t, s) = 2|E|.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // Compute subtree sizes rooted at node 0
    vector<int> sz(n, 1), par(n, -1);
    vector<int> order;
    // BFS to build order and compute subtree sizes
    vector<bool> visited(n, false);
    queue<int> bfs;
    bfs.push(0);
    visited[0] = true;
    while (!bfs.empty()) {
        int u = bfs.front(); bfs.pop();
        order.push_back(u);
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                par[v] = u;
                bfs.push(v);
            }
        }
    }
    // Compute subtree sizes in reverse BFS order
    for (int i = n - 1; i >= 1; i--) {
        sz[par[order[i]]] += sz[order[i]];
    }

    // Hitting time from s to t: sum over edges on the path
    // For edge (u, parent[u]) on the path, the contribution depends
    // on direction and subtree sizes
    int s, t;
    cin >> s >> t;
    s--; t--;

    // Find path from s to t using LCA
    vector<int> depth(n, 0);
    for (int i = 1; i < n; i++) {
        depth[order[i]] = depth[par[order[i]]] + 1;
    }

    // Compute hitting time (simplified for path queries)
    // H(s,t) = sum over edges (u,v) on s-t path of (2*size_away - 1)
    // where size_away is subtree size on the side away from t
    auto getPath = [&](int u, int v) -> vector<int> {
        vector<int> pu, pv;
        int a = u, b = v;
        while (a != b) {
            if (depth[a] > depth[b]) { pu.push_back(a); a = par[a]; }
            else { pv.push_back(b); b = par[b]; }
        }
        pu.push_back(a);
        for (int i = (int)pv.size() - 1; i >= 0; i--)
            pu.push_back(pv[i]);
        return pu;
    };

    auto path = getPath(s, t);
    long long hitTime = 0;
    for (int i = 0; i + 1 < (int)path.size(); i++) {
        int u = path[i], v = path[i + 1];
        // Size of component on u's side when edge (u,v) removed
        int sizeU;
        if (par[v] == u) {
            sizeU = n - sz[v];
        } else {
            sizeU = sz[u];
        }
        hitTime += 2LL * sizeU - 1;
    }

    cout << hitTime << "\n";
    return 0;
}

Expected Hitting Time & Cover Time

Beyond point-to-point hitting time, two other quantities are fundamental for random walks on trees:

Hitting Time via System of Equations

For a general formulation, define H[v] = expected steps to reach target t starting from v. Then H[t] = 0 and for v ≠ t:

H[v] = 1 + (1/deg(v)) · Σu ∈ N(v) H[u]

On a tree, this system has a unique solution. Since the tree is acyclic (relative to a rooted DFS), we can solve it efficiently. If t is the root, we process nodes in post-order: leaves first, then internal nodes.

Watch Out: Even on a tree, the random walk equations have “backward” dependencies (a child’s expected value depends on its parent). The trick is to express H[v] = αv · H[parent(v)] + βv, propagate (α, β) from leaves to root, and then resolve top-down.

Cover Time

The cover time C(T) — expected steps to visit every node — has a clean formula for trees:

C(T) = Σe=(u,v) ∈ E 2 · size(u) · size(v)

For a path Pn the cover time is Θ(n2); for a star Sn it is Θ(n log n) (coupon collector on leaves).

Computing Cover Time

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // Root at 0, compute subtree sizes
    vector<int> sz(n, 1), par(n, -1);
    vector<int> order;
    vector<bool> vis(n, false);
    queue<int> q;
    q.push(0); vis[0] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u]) if (!vis[v]) {
            vis[v] = true;
            par[v] = u;
            q.push(v);
        }
    }
    for (int i = n - 1; i >= 1; i--)
        sz[par[order[i]]] += sz[order[i]];

    // Cover time = sum over edges of 2 * size_a * size_b
    // For edge (v, par[v]): size_a = sz[v], size_b = n - sz[v]
    long long coverTime = 0;
    for (int v = 1; v < n; v++) {
        coverTime += 2LL * sz[v] * (n - sz[v]);
    }

    cout << coverTime << "\n";
    return 0;
}

Random Root Technique

The random root technique is a powerful analytical tool: instead of analyzing the tree with a fixed root, we choose the root uniformly at random from all n nodes. This often dramatically simplifies the analysis of expected values on trees.

Why Random Rooting Works

Many tree problems ask for the expected value of some quantity over all possible inputs or random choices. By choosing a random root, we can convert sums over pairs of nodes into sums over ancestor-descendant relationships, which are much easier to compute.

Key identity: node u is a proper ancestor of v when the root lies on u’s side of the u-v path. If removing u splits off a component of size sv|u containing v, then:

P(u is proper ancestor of v) = (n − sv|u) / n

Application Pattern: To compute Σv f(depth(v)) over all rootings, apply the random root technique: each rooting is equally likely, so the sum is n · Erandom root[f(depth(v))]. Convert depth to ancestor counts and use the ancestor probability formula.

Expected Depth of a Random Node

Root the tree at a uniformly random node. What is the expected depth of a uniformly random node v? By linearity, E[depth(v)] = Σu ≠ v P(u is proper ancestor of v). Using the formula above, this sum can be computed in O(n) after a single DFS.

For problems asking “compute Σ depth(v) over all rootings,” the technique converts an O(n2) brute force into an O(n) computation by counting ancestor contributions via subtree sizes.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // Sum of depths over all rootings
    // = sum over all ordered pairs (u,v) of [u is ancestor of v]
    // = sum over edges (parent, child) of sz[child] * (n - sz[child])
    // (each edge contributes when root is on parent side)
    vector<int> sz(n, 1), par(n, -1);
    vector<int> order;
    vector<bool> vis(n, false);
    queue<int> q;
    q.push(0); vis[0] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u]) if (!vis[v]) {
            vis[v] = true;
            par[v] = u;
            q.push(v);
        }
    }
    for (int i = n - 1; i >= 1; i--)
        sz[par[order[i]]] += sz[order[i]];

    // Total depth summed over all rootings
    long long totalDepth = 0;
    for (int v = 1; v < n; v++) {
        // Edge (v, par[v]) contributes: nodes in child subtree
        // become deeper for roots on the other side
        totalDepth += (long long)sz[v] * (n - sz[v]);
    }

    // Expected depth for random root and random node:
    // totalDepth / (n * n)
    cout << fixed << setprecision(9)
         << (double)totalDepth / ((long long)n * n) << "\n";
    return 0;
}

Probabilistic Tree DP

Probabilistic tree DP combines the standard tree DP framework with probability calculations. For each node v, we compute either dp[v] = probability of some event for the subtree rooted at v, or dp[v] = expected value of some quantity for the subtree rooted at v.

Pattern: Event Probability in Subtree

Given a rooted tree where each edge (v, parent(v)) fails independently with probability pv, compute for each node v the probability that v remains connected to the root.

Define dp[v] = probability that v is connected to the root. Then dp[root] = 1 and for each child c of v:

dp[c] = dp[v] · (1 − pc)

This is a simple top-down DP. Now consider a harder variant: each node v independently “activates” with probability qv. What is the expected number of active connected components?

By linearity of expectation, E[components] = Σv active P(v is a component root). Node v is a component root if v is active and either v is the tree root or the edge from v to parent(v) is missing (parent not active, or not connected). This decomposes into a product of independent events.

Pattern: Expected Subtree Size

Each node v is included independently with probability qv. What is the expected size of the connected component containing the root (considering only included nodes)?

Define dp[v] = expected number of included nodes in v’s subtree that are connected to v through a path of included nodes. Then:

dp[v] = qv · (1 + Σc child of v dp[c])

If v is not included (probability 1 − qv), it contributes 0 and blocks all descendants. If v is included, it contributes 1 plus the contributions from all children subtrees.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    vector<double> q(n);
    for (int i = 0; i < n; i++) cin >> q[i];

    // dp[v] = expected connected component size from v downward
    vector<double> dp(n, 0.0);

    function<void(int, int)> dfs = [&](int u, int par) {
        double childSum = 0.0;
        for (int v : adj[u]) {
            if (v == par) continue;
            dfs(v, u);
            childSum += dp[v];
        }
        dp[u] = q[u] * (1.0 + childSum);
    };

    dfs(0, -1);
    cout << fixed << setprecision(9) << dp[0] << "\n";
    return 0;
}
Key Technique — Contribution Counting: In probabilistic tree DP, think about what each node contributes to the answer, and compute its expected contribution. By linearity of expectation, the answer is the sum of all expected contributions. This avoids having to enumerate all possible outcomes.

Rerooting Technique for Expected Values

To compute dp[v] for every root v in O(n) total, use rerooting: first pass (post-order) computes dpdown[v]; second pass (pre-order) computes dpup[v] from siblings; combine for the full answer. Identical to deterministic rerooting, applied to expected values.

Random Ancestor Queries

A surprisingly useful concept in CP is the random ancestor: for a rooted tree, pick a uniformly random ancestor of node v (including v itself). Several problems reduce to computing expected values over random ancestors.

Expected LCA Depth

Given a rooted tree, pick two nodes u and v uniformly at random. What is the expected depth of their LCA?

Node w is the LCA of (u,v) when u and v are in different child subtrees, or one equals w. The count of ordered pairs with LCA exactly w is:

count(w) = sz[w]2 − Σc child of w sz[c]2

Then E[depth(LCA)] = (1/n2) Σw count(w) · depth(w).

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    vector<int> sz(n, 1), depth(n, 0);

    // DFS to compute sz and depth
    function<void(int, int)> dfs = [&](int u, int par) {
        for (int v : adj[u]) {
            if (v == par) continue;
            depth[v] = depth[u] + 1;
            dfs(v, u);
            sz[u] += sz[v];
        }
    };
    dfs(0, -1);

    // E[depth(LCA)] for random pair (u,v)
    // count(w) = sz[w]^2 - sum(sz[c]^2) for children c of w
    double ans = 0.0;
    function<void(int, int)> solve = [&](int u, int par) {
        long long childSqSum = 0;
        for (int v : adj[u]) {
            if (v == par) continue;
            childSqSum += (long long)sz[v] * sz[v];
            solve(v, u);
        }
        long long cnt = (long long)sz[u] * sz[u] - childSqSum;
        ans += (double)cnt * depth[u];
    };
    solve(0, -1);

    ans /= (double)n * n;
    cout << fixed << setprecision(9) << ans << "\n";
    return 0;
}
Precision Warning: When computing expected values on trees with large n, the intermediate sums can be very large (up to O(n3) for LCA depth problems). Use long long for intermediate computations and only convert to double at the final division. Alternatively, work modulo a prime if the problem requires exact rational output.

Random Walk to Random Ancestor

Starting at node v, the expected steps to reach the parent via random walk is 2 · sz[v] − 1 (the walk must escape v’s subtree). This follows directly from the edge-crossing formula of Section 1.

CP Applications

Probabilistic tree problems appear frequently in competitive programming. Here are the most important patterns and how to recognize them.

Pattern 1: Random Edge Deletion

Each edge is deleted independently with some probability. Common questions include: expected number of connected components, expected size of the component containing a specific node, or probability that two nodes remain connected.

For expected components: E[components] = 1 + Σedges pe (each deleted edge increases the count by 1).

For connectivity probability: P(u connected to v) = Πe on u-v path (1 − pe). This is because on a tree, the unique path must be entirely intact.

Pattern 2: Random Subtree Selection

Each node is selected independently. Compute expected values of various subtree properties. Use probabilistic tree DP (Section 4) with the contribution technique.

Pattern 3: Random Walk Games on Trees

Two-player games where players move tokens on a tree according to random rules. Common in problems that combine game theory with probability. Analyze using hitting times and optional stopping theorems.

Competition Strategy: When you see a probability problem on trees, first check if linearity of expectation applies. If you can decompose the answer into independent per-node or per-edge contributions, the problem usually becomes straightforward tree DP. Only set up systems of equations if the random process has cycles (random walks).

Example: Expected Distance Between Random Nodes

Given a weighted tree, pick two nodes u, v uniformly at random. What is the expected distance dist(u, v)?

By linearity: E[dist] = (1/n2) Σu,v dist(u,v). Each edge e with weight we contributes we to dist(u,v) whenever u and v are on opposite sides. So:

E[dist] = (1/n2) Σe we · 2 · sze · (n − sze)

This is computable in O(n) with a single DFS — a beautiful application of edge contribution counting.

Warmup

  • Codeforces 1092F – Tree with Maximum Cost: Rerooting DP. While not explicitly probabilistic, the technique is identical to computing expected values for all roots.
  • Codeforces 600E – Lomsat gelral: Small-to-large merging on trees. Good practice for the subtree contribution technique.

Intermediate

  • Codeforces 1540B – Tree Array: Expected value on trees using BFS and probabilistic DP. For each pair, compute the probability that one node appears before the other in a random BFS order.
  • AtCoder ABC298F – Rook Score: Expected value tree DP combined with contribution technique and linearity of expectation.
  • Codeforces 963B – Destruction of a Tree: Probability of connectivity after random edge deletions. Apply the path product formula.

Advanced

  • Codeforces 1778E – The Tree Has Fallen: Random walk on a tree with expected hitting time queries. Combine LCA with the edge-crossing formula for efficient query answering.
  • AtCoder AGC018E – Sightseeing Plan: Advanced counting on trees with probabilistic interpretation. Requires deep understanding of path contribution and inclusion-exclusion on trees.
  • Codeforces 827D – Best Edge Weight: While primarily a MST problem, the analysis involves expected values over random spanning trees and edge replacement probabilities.
Progression Tip: Master the subtree-size edge contribution formula first — it appears in nearly every tree probability problem. Then learn the rerooting technique for computing expected values for all roots. Finally, tackle random walk problems where you need systems of equations or the (α, β) coefficient trick.
