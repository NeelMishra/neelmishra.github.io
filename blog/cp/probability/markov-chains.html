Markov Chains for Competitive Programming

July 2026 · 25 min read

The Markov Property

A stochastic process {X0, X1, X2, …} on a finite state space S satisfies the Markov property if the future depends on the past only through the present:

P(Xn+1 = j | Xn = i, Xn−1, …, X0) = P(Xn+1 = j | Xn = i).

In plain English: where you go next depends only on where you are now, not on how you got here. This “memoryless” property is both the definition and the source of power of Markov chains.

Why CP cares: Whenever a problem describes a system whose state transitions depend only on the current state (dice games, board games, random processes on graphs), you are looking at a Markov chain. Recognising this lets you apply matrix exponentiation, Gaussian elimination, and other standard techniques.

We restrict attention to time-homogeneous chains: the transition probabilities do not change with time. That is, P(Xn+1 = j | Xn = i) = pij for all n.

Classifying States

A state i is absorbing if pii = 1 (once you enter, you never leave). A state is transient if there is a positive probability of never returning to it. A state is recurrent if the chain returns to it with probability 1. In finite chains, every non-transient state is recurrent.

Transition Matrices

The entire behaviour of a finite Markov chain with m states is captured by an m × m transition matrix P:

Pij = P(go from state i to state j).

Each row sums to 1 (it is a stochastic matrix). The initial state is described by a row vector π0, and after one step the distribution is π1 = π0 P.

Example: Weather Model

Suppose weather is either Sunny (S) or Rainy (R), with transitions:

S R
S 0.8 0.2
R 0.4 0.6

If today is sunny (π0 = [1, 0]), then tomorrow’s distribution is [0.8, 0.2] and the day after is [0.8, 0.2]·P = [0.72, 0.28].

Convention alert: Some textbooks use column stochastic matrices (columns sum to 1) with right-multiplication. In CP and most programming contexts, row stochastic matrices with left-multiplication are standard. Make sure you know which convention a problem uses.

n-Step Transitions

The probability of moving from state i to state j in exactly n steps is given by the (i, j) entry of Pn. This is the key connection to matrix exponentiation, one of the most powerful techniques in CP.

P(Xn = j | X0 = i) = (Pn)ij.

Computing Pn by repeated squaring takes O(m3 log n), where m = |S|. This is efficient enough for m up to a few hundred and n up to 1018.

C++ — Matrix Exponentiation for Markov Chains

#include <bits/stdc++.h>
using namespace std;

typedef vector<vector<double>> Mat;

Mat multiply(const Mat &A, const Mat &B){
    int m = A.size();
    Mat C(m, vector<double>(m, 0));
    for(int i = 0; i < m; i++)
        for(int k = 0; k < m; k++) if(A[i][k] != 0)
            for(int j = 0; j < m; j++)
                C[i][j] += A[i][k] * B[k][j];
    return C;
}

Mat mat_pow(Mat A, long long n){
    int m = A.size();
    Mat R(m, vector<double>(m, 0));
    for(int i = 0; i < m; i++) R[i][i] = 1; // identity
    while(n > 0){
        if(n & 1) R = multiply(R, A);
        A = multiply(A, A);
        n >>= 1;
    }
    return R;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int m; long long n;
    cin >> m >> n;
    Mat P(m, vector<double>(m));
    for(int i = 0; i < m; i++)
        for(int j = 0; j < m; j++)
            cin >> P[i][j];

    int start; cin >> start;
    Mat Pn = mat_pow(P, n);
    for(int j = 0; j < m; j++)
        cout << fixed << setprecision(9)
             << Pn[start][j] << " \n"[j==m-1];
    return 0;
}
Modular arithmetic variant: If the problem asks for probabilities modulo a prime (common in CP), replace double with modular integers and use modular inverse for divisions. The matrix exponentiation structure is identical.

Stationary Distributions

A probability vector π is a stationary distribution if it is unchanged by the transition:

π = π P,    Σj πj = 1.

In other words, π is a left eigenvector of P with eigenvalue 1.

Existence & Uniqueness

For a finite, irreducible (all states communicate) and aperiodic (gcd of return times is 1) chain, there is a unique stationary distribution, and Pn converges to a matrix whose every row is π. This is the fundamental theorem of Markov chains.

Computing π

Rewrite π = πP as π(P − I) = 0, giving a homogeneous linear system. Replace any one equation with Σπj = 1 and solve. For the weather example:

πS = 0.8 πS + 0.4 πR,   πS + πR = 1.

Solving: πS = 2/3, πR = 1/3. In the long run, it is sunny about 66.7% of the time.

CP shortcut: For doubly-stochastic matrices (both rows and columns sum to 1), the stationary distribution is uniform: πj = 1/m. Random walks on regular graphs have this property.

PageRank Connection

Google’s original PageRank algorithm models a “random surfer” as a Markov chain on the web graph. The stationary distribution gives the importance of each page. The damping factor ensures irreducibility and aperiodicity.

Absorbing Markov Chains

A Markov chain is absorbing if (1) it has at least one absorbing state, and (2) from every state, there is a positive probability of eventually reaching an absorbing state. These are the chains that “eventually stop”.

Canonical Form

Reorder states so that transient states come first, absorbing states last. The transition matrix takes the block form:

P = | Q R |
    | 0 I |

Q = transitions among transient states (t × t)
R = transitions from transient to absorbing states (t × a)
I = identity (absorbing states stay put)

Here t is the number of transient states and a is the number of absorbing states.

Key Questions

  • What is the expected number of steps before absorption?
  • What is the probability of being absorbed in each absorbing state?
  • What is the expected number of times we visit each transient state?

All are answered by the fundamental matrix.

The Fundamental Matrix

The fundamental matrix of an absorbing chain is:

N = (I − Q)−1

where Q is the transient-to-transient submatrix and I is the identity of the same size.

Interpretation of N

Nij = expected number of times the chain visits transient state j before absorption, given it started in transient state i.

Expected Absorption Time

The expected number of steps before absorption, starting from state i, is:

ti = Σj Nij   (sum of row i of N).

In vector form: t = N 1, where 1 is the all-ones column vector.

Absorption Probabilities

The probability of being absorbed in absorbing state j, starting from transient state i, is:

B = N · R

where R is the transient-to-absorbing submatrix.

C++ — Fundamental Matrix via Gaussian Elimination

#include <bits/stdc++.h>
using namespace std;

typedef vector<vector<double>> Mat;

// Solve AX = B where A is n x n, B is n x m
Mat solve_system(Mat A, Mat B){
    int n = A.size(), m = B[0].size();
    // Augment A with B
    for(int i = 0; i < n; i++)
        for(int j = 0; j < m; j++)
            A[i].push_back(B[i][j]);

    for(int col = 0; col < n; col++){
        int pivot = -1;
        for(int row = col; row < n; row++)
            if(abs(A[row][col]) > 1e-9){ pivot = row; break; }
        swap(A[col], A[pivot]);
        double div = A[col][col];
        for(auto &v : A[col]) v /= div;
        for(int row = 0; row < n; row++){
            if(row == col) continue;
            double f = A[row][col];
            for(int j = 0; j < n + m; j++)
                A[row][j] -= f * A[col][j];
        }
    }
    Mat X(n, vector<double>(m));
    for(int i = 0; i < n; i++)
        for(int j = 0; j < m; j++)
            X[i][j] = A[i][n + j];
    return X;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t, a; // transient states, absorbing states
    cin >> t >> a;

    Mat Q(t, vector<double>(t));
    Mat R(t, vector<double>(a));
    for(int i = 0; i < t; i++){
        for(int j = 0; j < t; j++) cin >> Q[i][j];
        for(int j = 0; j < a; j++) cin >> R[i][j];
    }

    // N = (I - Q)^{-1}  <==>  (I - Q) N = I
    Mat IQ(t, vector<double>(t, 0));
    Mat I(t, vector<double>(t, 0));
    for(int i = 0; i < t; i++){
        I[i][i] = 1;
        for(int j = 0; j < t; j++)
            IQ[i][j] = (i == j ? 1 : 0) - Q[i][j];
    }
    Mat N = solve_system(IQ, I);

    // Expected absorption times
    cout << "Expected steps:\n";
    for(int i = 0; i < t; i++){
        double sum = 0;
        for(int j = 0; j < t; j++) sum += N[i][j];
        cout << "  State " << i << ": "
             << fixed << setprecision(4) << sum << "\n";
    }

    // Absorption probabilities B = N * R
    cout << "Absorption probabilities:\n";
    for(int i = 0; i < t; i++){
        cout << "  From " << i << ": ";
        for(int j = 0; j < a; j++){
            double val = 0;
            for(int k = 0; k < t; k++) val += N[i][k] * R[k][j];
            cout << fixed << setprecision(6) << val << " ";
        }
        cout << "\n";
    }
    return 0;
}
Precision: Gaussian elimination on doubles gives errors around 10−9 for moderate matrix sizes. If the problem requires exact answers modulo a prime, use modular inverse instead of floating-point division.

CP Applications

1. Matrix Exponentiation for Large n

“After 1018 steps, what is the probability of being in state j?” Use fast matrix exponentiation. The state space must be small (m ≤ 200 or so) but n can be astronomically large. This appears frequently in Codeforces Div 1 D/E problems.

2. Expected Value via Absorbing Chains

Many “expected number of steps” problems reduce to absorbing Markov chains. Model the goal state as absorbing, compute the fundamental matrix, and read off the answer. Classic example: expected number of steps for a random walk to reach a target on a graph.

3. Dice and Board Games

Snakes-and-ladders, Monopoly-style problems, and general dice-on-board problems are Markov chains. Each board cell is a state, and the transition probabilities come from the dice distribution. Snakes and ladders modify the transitions.

State compression: If the full state space is too large, look for symmetries or equivalences. For instance, in a circular board game, the state might be the position modulo the board length. In a game with dice, you might only need the current cell and a small amount of additional info (like the last roll).

4. Coupling & Mixing Time

Some advanced CP problems ask “how many steps until the chain is close to stationary?” The mixing time Tmix is the first time the distribution is within total-variation distance ε of π. For CP, matrix exponentiation gives exact distributions; checking convergence is then straightforward.

5. System of Equations Approach

For problems where you need E[steps from state i to some target], set up equations:

E(i) = 1 + Σj ≠ target pij · E(j),    E(target) = 0.

This gives a system of linear equations solvable with Gaussian elimination in O(m3). This is the method to use when the state space is small enough (m ≤ ~500). 

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, target;
    cin >> n >> target;

    // Read transition matrix
    vector<vector<double>> P(n, vector<double>(n));
    for(int i = 0; i < n; i++)
        for(int j = 0; j < n; j++)
            cin >> P[i][j];

    // Set up: E(i) - sum_{j != target} p_ij * E(j) = 1
    // E(target) = 0
    vector<vector<double>> A(n, vector<double>(n + 1, 0));
    for(int i = 0; i < n; i++){
        if(i == target){
            A[i][i] = 1; A[i][n] = 0;
        } else {
            A[i][i] = 1;
            for(int j = 0; j < n; j++)
                if(j != target) A[i][j] -= P[i][j];
            A[i][n] = 1;
        }
    }
    // Gaussian elimination
    for(int col = 0; col < n; col++){
        int pivot = col;
        for(int row = col + 1; row < n; row++)
            if(abs(A[row][col]) > abs(A[pivot][col])) pivot = row;
        swap(A[col], A[pivot]);
        for(int row = 0; row < n; row++){
            if(row == col) continue;
            double f = A[row][col] / A[col][col];
            for(int j = col; j <= n; j++)
                A[row][j] -= f * A[col][j];
        }
    }
    cout << "Expected steps to reach state " << target << ":\n";
    for(int i = 0; i < n; i++)
        cout << "  State " << i << ": "
             << fixed << setprecision(4)
             << A[i][n] / A[i][i] << "\n";
    return 0;
}

Practice Problems

Here are problems to solidify your Markov chain skills:

  1. Weather Prediction (easy): Given a 2-state weather model with known transition probabilities and an initial state, compute the probability of each state after n days using matrix exponentiation.
  2. Dice Board Game (medium): A player starts at cell 0 on a linear board of N cells. Each turn they roll a fair die and advance. Cell N is absorbing (the goal). Compute the expected number of turns to reach cell N.
  3. Random Walk on a Graph (medium): Given an undirected graph, a start node, and a target node, compute the expected number of steps for a random walk to reach the target. Set up the linear system and solve.
  4. Ant on a Cube (medium): An ant starts at a vertex of a cube and at each step moves to a uniformly random adjacent vertex. What is the expected number of steps to reach the opposite vertex? Hint: exploit symmetry to reduce to 4 states.
  5. Snakes and Ladders (hard): Given a board with snakes and ladders, model as a Markov chain and compute the expected number of dice rolls to reach the final cell.
  6. Probability After 1018 Steps (hard): Given a small Markov chain (m ≤ 50) and an astronomically large n, compute the distribution after n steps modulo 109 + 7. Use matrix exponentiation with modular arithmetic.
Summary: Markov chains provide a unified framework for analysing memoryless stochastic processes. The key tools are (1) matrix exponentiation for n-step distributions, (2) the fundamental matrix for absorbing chains, and (3) systems of linear equations for expected values and hitting probabilities.
