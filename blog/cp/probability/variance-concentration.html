Variance & Concentration Inequalities

July 2026 · 24 min read

Variance & Standard Deviation

While the expected value tells us the “center” of a distribution, it says nothing about how spread out the values are. Two random variables can have the same mean but wildly different behaviors. Variance quantifies this spread.

Definition: The variance of a random variable X is:

Var(X) = E[(X − E[X])²] = E[(X − μ)²]

This is the expected squared deviation from the mean. Squaring ensures positive and negative deviations don’t cancel, and penalizes large deviations more than small ones.

The standard deviation is σ = √Var(X), which has the same units as X (variance has squared units). By convention, σ² denotes variance and σ denotes standard deviation.

Alternative formula (computational form): Expanding the square gives the most useful computational identity:

Var(X) = E[X²] − (E[X])²

Proof:

Var(X) = E[(X − μ)²]
      = E[X² − 2μX + μ²]
      = E[X²] − 2μ · E[X] + μ²
      = E[X²] − 2μ² + μ²
      = E[X²] − μ²   ✓
Mnemonic: “The variance is the mean of the square minus the square of the mean.” In code: var = E_of_X_sq - E_of_X * E_of_X. This formula is almost always easier to compute than the definition.

Key Properties of Variance:

  • Var(X) ≥ 0 always (it’s an expected square).
  • Var(X) = 0 if and only if X is a constant (X = μ with probability 1).
  • Var(aX + b) = a² · Var(X) — adding a constant doesn’t change spread; scaling by a multiplies variance by a².
  • If X and Y are independent: Var(X + Y) = Var(X) + Var(Y).
  • In general (not necessarily independent): Var(X + Y) = Var(X) + Var(Y) + 2 · Cov(X, Y).
Independence Required! Unlike expectation, variance does not always add. Var(X + Y) = Var(X) + Var(Y) requires independence (or at minimum, uncorrelated variables). This is the most common mistake in probability computations. Always verify independence before summing variances.

Computing Variance: Examples

Example 1: Bernoulli Variable

Let X ∼ Bernoulli(p), so X = 1 with probability p and X = 0 with probability 1−p.

E[X] = p
E[X²] = 1² · p + 0² · (1−p) = p
Var(X) = p − p² = p(1 − p)

Maximum variance occurs at p = 1/2: Var = 1/4. The more “uncertain” the outcome, the higher the variance.

Example 2: Sum of Independent Bernoullis (Binomial)

If X = X1 + … + Xn where Xi ∼ Bernoulli(p) independently:

E[X] = np
Var(X) = Σ Var(Xi) = n · p(1 − p)    (by independence)
σ = √(np(1−p))

Example 3: Uniform Distribution on {1, …, n}

E[X] = (n + 1) / 2
E[X²] = (1/n) · Σk=1..n k² = (n+1)(2n+1) / 6
Var(X) = (n+1)(2n+1)/6 − (n+1)²/4 = (n² − 1) / 12

Example 4: Geometric Distribution

If X ∼ Geometric(p):

E[X] = 1/p
Var(X) = (1 − p) / p² 
#include <bits/stdc++.h>
using namespace std;

// Compute variance of a discrete distribution given as array of (value, probability)
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<double> val(n), prob(n);
    for (int i = 0; i < n; i++) cin >> val[i] >> prob[i];

    double ex = 0, ex2 = 0;
    for (int i = 0; i < n; i++) {
        ex += val[i] * prob[i];
        ex2 += val[i] * val[i] * prob[i];
    }

    double variance = ex2 - ex * ex;
    cout << fixed << setprecision(6);
    cout << "E[X]   = " << ex << "\n";
    cout << "E[X^2] = " << ex2 << "\n";
    cout << "Var(X) = " << variance << "\n";
    cout << "SD(X)  = " << sqrt(variance) << "\n";
    return 0;
}

Chebyshev’s Inequality

Chebyshev’s inequality is the first tool that converts variance information into probability bounds. It tells us that a random variable is unlikely to be far from its mean, quantified by the variance.

Statement: For any random variable X with finite mean μ and variance σ²:

P(|X − μ| ≥ kσ) ≤ 1/k²    for any k > 0

Equivalently, for any t > 0:

P(|X − μ| ≥ t) ≤ σ² / t²

Proof (from Markov’s inequality, which we prove in the next section):

P(|X − μ| ≥ t) = P((X − μ)² ≥ t²)
   ≤ E[(X − μ)²] / t²    (Markov on the non-negative variable (X−μ)²)
   = Var(X) / t²
   = σ² / t²    ✓
What Chebyshev tells us: At least 75% of any distribution lies within 2 standard deviations of the mean (k=2: 1/4 tail). At least 89% lies within 3 standard deviations (k=3: 1/9 tail). These bounds hold for any distribution — no normality assumption needed!

Application: Weak Law of Large Numbers.

Let X1, …, Xn be i.i.d. with mean μ and variance σ². Let X̄ = (X1 + … + Xn) / n be the sample mean. Then:

E[X̄] = μ
Var(X̄) = σ² / n

P(|X̄ − μ| ≥ ε) ≤ σ² / (nε²) → 0 as n → ∞

This is the weak law of large numbers: the sample mean converges in probability to the true mean. Chebyshev provides the rate: O(1/n) probability bound.

Markov’s Inequality

Markov’s inequality is the most basic concentration inequality. It applies to any non-negative random variable and requires only knowledge of the mean.

Statement: If X ≥ 0 and a > 0, then:

P(X ≥ a) ≤ E[X] / a

Proof:

E[X] = Σx x · P(X = x)
     ≥ Σx ≥ a x · P(X = x)    (dropped non-negative terms)
     ≥ Σx ≥ a a · P(X = x)    (x ≥ a in this sum)
     = a · P(X ≥ a)

Therefore: P(X ≥ a) ≤ E[X] / a    ✓
Markov’s Limitation: Markov only applies to non-negative random variables. It gives loose bounds in general — if E[X] = 10 and a = 20, the bound is only P(X ≥ 20) ≤ 1/2. But its simplicity and generality make it the foundation for stronger inequalities (Chebyshev, Chernoff).

Why Markov is tight: Consider X = a with probability μ/a and X = 0 otherwise. Then E[X] = μ and P(X ≥ a) = μ/a, achieving equality. So the bound cannot be improved without additional information.

Useful corollary for tail probabilities:

P(X ≥ c · E[X]) ≤ 1/c

A non-negative random variable exceeds c times its mean with probability at most 1/c. For c = 10: at most 10% probability of being 10× the mean.

Chernoff Bounds

Markov and Chebyshev give polynomial tail bounds (1/k and 1/k²). For sums of independent random variables, we can do exponentially better using Chernoff bounds.

Setting: Let X = X1 + X2 + … + Xn, where each Xi ∼ Bernoulli(pi) independently. Let μ = E[X] = Σ pi.

Upper Tail Bound

P(X ≥ (1 + δ)μ) ≤ (eδ / (1+δ)(1+δ))μ

A simpler (weaker) form for 0 < δ ≤ 1:

P(X ≥ (1 + δ)μ) ≤ e−μδ²/3

Lower Tail Bound

P(X ≤ (1 − δ)μ) ≤ e−μδ²/2    for 0 < δ < 1

Two-Sided Bound

P(|X − μ| ≥ δμ) ≤ 2 · e−μδ²/3

Proof sketch (Chernoff method for upper tail):

// Step 1: Exponential Markov trick
P(X ≥ t) = P(esX ≥ est) ≤ E[esX] / est   (Markov, s > 0)

// Step 2: Factor the MGF by independence
E[esX] = Πi E[esXi]
         = Πi (1 − pi + pi · es)
         ≤ Πi epi(es−1)    (using 1+x ≤ ex)
         = eμ(es−1)

// Step 3: Optimize over s
P(X ≥ t) ≤ eμ(es−1) − st
Set t = (1+δ)μ, optimize s = ln(1+δ) to get the result. ✓
Power of Chernoff: Chernoff gives exponential decay in the deviation. Flipping 1000 fair coins, the probability of getting ≥ 600 heads is: Chebyshev gives ≤ 1/16 ≈ 6.25%. Chernoff gives ≤ e−500·0.04/3 ≈ e−6.67 ≈ 0.13%. The real probability is about 0.0000027%. Chernoff is tight up to constant factors in the exponent. 
#include <bits/stdc++.h>
using namespace std;

// Compare Markov, Chebyshev, and Chernoff bounds
// for the sum of n independent Bernoulli(p) variables
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n = 1000;
    double p = 0.5;
    double mu = n * p;              // E[X] = 500
    double var = n * p * (1 - p);   // Var(X) = 250
    double sigma = sqrt(var);

    // Probability of X >= 600 (i.e., delta = 0.2, t = 600)
    double t = 600;
    double delta = (t - mu) / mu;   // 0.2

    // Markov: P(X >= t) <= mu / t
    double markov = mu / t;

    // Chebyshev: P(|X - mu| >= t - mu) <= var / (t - mu)^2
    double chebyshev = var / ((t - mu) * (t - mu));

    // Chernoff: P(X >= (1+d)mu) <= exp(-mu * d^2 / 3)
    double chernoff = exp(-mu * delta * delta / 3.0);

    // Simulation
    mt19937 rng(42);
    int trials = 10000000, hits = 0;
    for (int tr = 0; tr < trials; tr++) {
        int cnt = 0;
        for (int i = 0; i < n; i++) {
            if (rng() % 2 == 0) cnt++;
        }
        if (cnt >= (int)t) hits++;
    }

    cout << fixed << setprecision(8);
    cout << "P(X >= " << (int)t << ") bounds for Bin(" << n << ", " << p << "):\n";
    cout << "Markov:     " << markov << "\n";
    cout << "Chebyshev:  " << chebyshev << "\n";
    cout << "Chernoff:   " << chernoff << "\n";
    cout << "Simulated:  " << (double)hits / trials << "\n";
    return 0;
}

Applications in CP

Variance and concentration inequalities are essential in competitive programming for two main reasons: (1) analyzing randomized algorithms, and (2) solving problems that ask for variance or higher moments.

Application 1: Proving Randomized Algorithms Work

Many randomized algorithms have a “good” event that occurs with probability p per trial. Running k independent trials, the expected successes are kp. By Chernoff, the probability of zero successes in k trials is at most e−kp/2 (using the lower tail with δ = 1). Setting k = O(log n / p) gives failure probability 1/poly(n).

Application 2: Randomized Hashing Security

When using polynomial hashing with a random base, the probability of a collision between two distinct strings of length L is at most L/M (where M is the modulus). Hashing n strings gives C(n,2) · L/M expected collisions. By Markov, the probability of any collision is at most n²L/(2M). Choosing M ≥ n²L makes this small.

Application 3: Monte Carlo Estimation

To estimate a quantity μ = E[f(X)] by averaging N samples, the error satisfies:

P(|X̄ − μ| ≥ ε) ≤ σ² / (Nε²)    (Chebyshev)

For 99% confidence (δ = 0.01):
N ≥ σ² / (ε² · 0.01) = 100 · σ² / ε²

Application 4: Variance in Expected Value Problems

Some CP problems explicitly ask for Var(X) mod p. The approach: compute E[X] and E[X²] separately using linearity, then Var(X) = E[X²] − (E[X])². Computing E[X²] often requires analyzing pairs of indicators.

E[X²] = E[(Σ Xi)²]
       = E[Σi Xi² + Σi≠j XiXj]
       = Σi E[Xi²] + Σi≠j E[XiXj]
       = Σi P(event i) + Σi≠j P(event i ∩ event j)

For indicators, Xi² = Xi, and XiXj is the indicator for the joint event. The second sum requires computing pairwise joint probabilities — this is where the real work is. 

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) { if (e & 1) r = r * b % m; b = b * b % m; e >>= 1; }
    return r;
}

long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Example: Variance of the number of fixed points in a random permutation
// X = sum X_i where X_i = 1 if pi(i) = i
// E[X] = n * (1/n) = 1
// E[X^2] = sum E[Xi^2] + sum_{i!=j} E[Xi*Xj]
//        = n*(1/n) + n*(n-1)*(1/(n*(n-1)))
//        = 1 + 1 = 2
// Var(X) = 2 - 1 = 1
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long n;
    cin >> n;

    // E[X] = 1
    long long ex = 1;

    // E[X^2] = sum P(Xi=1) + sum_{i!=j} P(Xi=1 and Xj=1)
    // P(Xi=1) = 1/n, so sum = n * 1/n = 1
    // P(Xi=1 and Xj=1) = 1/(n*(n-1)), pairs = n*(n-1)
    // so pairwise sum = n*(n-1) * 1/(n*(n-1)) = 1
    long long ex2 = 2 % MOD;  // 1 + 1

    // Var(X) = E[X^2] - (E[X])^2 = 2 - 1 = 1
    long long variance = (ex2 - ex * ex % MOD + MOD) % MOD;

    cout << "E[X] = " << ex << "\n";
    cout << "E[X^2] = " << ex2 << "\n";
    cout << "Var(X) = " << variance << "\n";
    return 0;
}
Remarkable Fact: The number of fixed points in a random permutation has variance exactly 1, regardless of n. The distribution is approximately Poisson(1) for large n, which is why the mean and variance are both 1. This is a beautiful consequence of the indicator decomposition.

Application 5: Confidence in Competitive Programming Heuristics

Sometimes in contests, you use a randomized approach (e.g., random restarts for optimization, random sampling for approximation). Chernoff bounds let you calculate how many trials you need. If each trial succeeds with probability p ≥ 0.01 and you need at least one success:

P(all k trials fail) = (1−p)k ≤ e−pk
Want failure < 10−6: k ≥ 6 · ln(10) / p ≈ 1382 trials for p = 0.01 
#include <bits/stdc++.h>
using namespace std;

// Monte Carlo confidence estimation
// Estimate P(event) by sampling, report confidence interval
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int N = 1000000;

    // Example: estimate probability that sum of 10 dice >= 40
    int hits = 0;
    for (int t = 0; t < N; t++) {
        int s = 0;
        for (int d = 0; d < 10; d++) s += rng() % 6 + 1;
        if (s >= 40) hits++;
    }

    double p_hat = (double)hits / N;
    double se = sqrt(p_hat * (1 - p_hat) / N);  // standard error

    cout << fixed << setprecision(6);
    cout << "Estimated P: " << p_hat << "\n";
    cout << "Std Error:   " << se << "\n";
    cout << "95% CI:      [" << p_hat - 1.96 * se
         << ", " << p_hat + 1.96 * se << "]\n";

    // Chebyshev 95% CI (wider): k = sqrt(1/0.05) ≈ 4.47
    double k = sqrt(1.0 / 0.05);
    cout << "Cheby 95% CI: [" << p_hat - k * se
         << ", " << p_hat + k * se << "]\n";
    return 0;
}

Practice Problems

Build your mastery of variance and concentration inequalities:

  1. Variance of Inversions: Compute Var(inversions) for a random permutation of n elements. Decompose into indicator pairs. The diagonal terms give Σ Var(Xij) = C(n,2) · 1/4. The cross terms require computing P(Xij = 1 & Xkl = 1) for overlapping and non-overlapping pairs. Final answer: n(n−1)(2n+5)/72.
  2. Median Finding Confidence: Using random sampling, you pick k elements from an array of n. Show that with k = O(n2/3) samples, the median of the sample is within n1/3 positions of the true median with high probability. Use Chernoff bounds.
  3. Randomized Min-Cut (Karger’s): Karger’s algorithm finds a min-cut with probability ≥ 2/(n(n−1)). Running it t = n² ln(n)/2 times, use Chernoff to show the probability of missing the min-cut is at most 1/n.
  4. Balls into Bins — Maximum Load: Throw n balls into n bins uniformly. Use Chernoff + union bound to show the maximum bin load is O(log n / log log n) with high probability.
  5. Hash Collision Bound: n strings hashed with polynomial hashing mod M. Using Markov on the number of collisions, determine the minimum M such that the probability of any collision is at most 1/n. (Answer: M ≥ n³/2 suffices.)
  6. Variance of Coupon Collector: Compute the exact variance of the coupon collector time T for n types. (Hint: Var(T) = Σ (1−pi)/pi² since phases are independent geometric variables.)
  7. Randomized Quickselect Analysis: Show that the expected number of comparisons in randomized quickselect is O(n). Then use Chebyshev to show that the probability of needing more than cn comparisons is O(1/c²).
Hierarchy of Bounds: Remember the power hierarchy: Markov (weakest, most general) → Chebyshev (needs variance) → Chernoff (needs independence, gives exponential tails). Always try the strongest bound that your problem’s structure supports. In contests, if you see “independent trials,” immediately reach for Chernoff.
Common Contest Trap: Problems that ask “what is the probability this randomized approach fails?” almost always require Chernoff or a union bound. Don’t waste time trying exact computation — bound it. And remember: union bound (P(A ∪ B) ≤ P(A) + P(B)) combines perfectly with Chernoff to handle “bad events” across multiple independent components.
