Probability Patterns from CF & ICPC

October 2026 · 26 min read

Pattern Overview

After solving hundreds of probability problems on Codeforces and ICPC, clear patterns emerge. Recognizing these patterns in contests is the difference between solving in 15 minutes and being stuck for hours. This post catalogs the five most important patterns and gives you ready-to-use templates for each.

The Five Core Patterns

# Pattern Key Idea Frequency
1 Linearity of Expectation E[X] = Σ E[Xi] Very High
2 Contribution Technique Each element's expected contribution High
3 Probability DP State-based probability computation High
4 Expected Moves to Target System of equations / recursive EV Medium
5 Random Graph/Permutation Expected cycles, components, inversions Medium
Contest strategy: When you see a probability problem, first check if linearity of expectation applies (Pattern 1). It works ~60% of the time and gives the simplest solution. If not, check for DP structure (Pattern 3). Only resort to complex techniques when the simpler patterns don't fit.

Pattern 1: Linearity of Expectation

The single most powerful technique in probability for CP. It states:

E[X1 + X2 + … + Xn] = E[X1] + E[X2] + … + E[Xn]

This holds regardless of dependence between the Xi. Even if the random variables are heavily correlated, we can still decompose the expected value into a sum.

Recipe

  1. Identify indicator random variables: Xi = 1 if event i occurs, 0 otherwise.
  2. Compute P(Xi = 1) for each i.
  3. Sum: E[X] = Σ P(Xi = 1).

Example: Expected Inversions in a Random Permutation

Problem: A random permutation of [1..n] is generated. Find the expected number of inversions (pairs i < j where π(i) > π(j)).

Solution: Let Xij = 1 if (i, j) is an inversion. For any pair, P(Xij = 1) = 1/2 (by symmetry). Number of pairs = C(n, 2). So E[inversions] = C(n, 2) / 2 = n(n−1)/4.

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long n;
    cin >> n;

    // E[inversions] = n*(n-1)/4  mod p
    long long num = n % MOD * ((n - 1) % MOD) % MOD;
    long long ans = num % MOD * power(4, MOD - 2, MOD) % MOD;
    cout << ans << "\n";
    return 0;
}

Example: Expected Distinct Values

Problem: Pick k elements uniformly at random (with replacement) from {1, 2, …, n}. Expected number of distinct values?

Solution: Let Xi = 1 if value i appears at least once. P(Xi = 1) = 1 − ((n−1)/n)k. E[distinct] = n · (1 − ((n−1)/n)k).

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long n, k;
    cin >> n >> k;

    long long inv_n = power(n, MOD - 2, MOD);
    long long p_miss = (n - 1) % MOD * inv_n % MOD;
    long long p_miss_k = power(p_miss, k, MOD);
    long long p_hit = (1 - p_miss_k + MOD) % MOD;
    long long ans = n % MOD * p_hit % MOD;
    cout << ans << "\n";
    return 0;
}
Tip: Linearity of expectation works even when the indicator variables are dependent! You don't need independence to decompose the sum. This is what makes it so powerful.

Pattern 2: Contribution Technique

A refinement of linearity: instead of thinking about the total, think about each element's contribution to the answer.

Recipe

  1. For each element e in the input, compute its expected contribution to the answer.
  2. Sum over all elements.

Example: Expected Sum of Subarray Maximums

Problem: Given array a[1..n], a subarray [l, r] is chosen uniformly at random. Find E[max(a[l..r])].

Solution: For each element a[i], compute the number of subarrays where a[i] is the maximum. Let left[i] = distance to the nearest element > a[i] on the left, right[i] = same for the right. Then a[i] is the max in left[i] × right[i] subarrays. Contribution of a[i] = a[i] · left[i] · right[i] / C(n,2)+n).

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<long long> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];

    // Monotone stack for left[i] and right[i]
    vector<int> left_dist(n), right_dist(n);
    stack<int> st;

    for (int i = 0; i < n; i++) {
        while (!st.empty() && a[st.top()] <= a[i]) st.pop();
        left_dist[i] = st.empty() ? i + 1 : i - st.top();
        st.push(i);
    }
    while (!st.empty()) st.pop();

    for (int i = n - 1; i >= 0; i--) {
        while (!st.empty() && a[st.top()] < a[i]) st.pop();
        right_dist[i] = st.empty() ? n - i : st.top() - i;
        st.push(i);
    }

    // Total subarrays = n*(n+1)/2
    long long total_sub = (long long)n * (n + 1) / 2 % MOD;
    long long inv_total = power(total_sub, MOD - 2, MOD);

    long long ans = 0;
    for (int i = 0; i < n; i++) {
        long long cnt = (long long)left_dist[i] * right_dist[i] % MOD;
        ans = (ans + a[i] % MOD * cnt % MOD) % MOD;
    }
    ans = ans % MOD * inv_total % MOD;
    cout << ans << "\n";
    return 0;
}
Warning: When using monotone stacks for the contribution technique, be careful with equal elements. Use strict inequality on one side and non-strict on the other to avoid double-counting. The code above uses <= on the left and < on the right.

When Contribution Technique Shines

  • Expected sum of subarray functions (max, min, gcd, etc.)
  • Expected number of elements satisfying a property in a random subset
  • Expected weight of a random spanning tree (each edge's contribution)

Pattern 3: Probability DP

When the process has a state that evolves over time, model it as a DP where dp[state] = probability (or expected value) of reaching or being in that state.

Forward Probability DP

dp[state] = probability of being in this state. Start with dp[initial] = 1. Distribute probability forward along transitions.

// Example: random walk on [0..n], start at 0
// Each step: +1 with prob p, -1 with prob 1-p
// dp[t][x] = prob of being at position x after t steps

const int T = 1000, N = 500;
long long dp[T + 1][2 * N + 1];
long long p, q; // p = prob of +1 (mod MOD), q = 1-p

void solve() {
    memset(dp, 0, sizeof dp);
    dp[0][N] = 1; // start at position 0 (offset by N)

    for (int t = 0; t < T; t++) {
        for (int x = 1; x < 2 * N; x++) {
            if (dp[t][x] == 0) continue;
            dp[t + 1][x + 1] = (dp[t + 1][x + 1] + dp[t][x] % MOD * p) % MOD;
            dp[t + 1][x - 1] = (dp[t + 1][x - 1] + dp[t][x] % MOD * q) % MOD;
        }
    }
}

Backward Expected Value DP

dp[state] = expected value from this state onward. Start from terminal states and work backward.

// Example: expected steps to reach position n from position 0
// Each step: +1 with prob p, stay with prob 1-p
// dp[i] = expected steps from position i

vector<long long> dp(n + 1, 0);
// dp[n] = 0 (already at target)
long long inv_p = power(p, MOD - 2, MOD);

for (int i = n - 1; i >= 0; i--) {
    // dp[i] = 1 + p*dp[i+1] + (1-p)*dp[i]
    // => dp[i] - (1-p)*dp[i] = 1 + p*dp[i+1]
    // => p*dp[i] = 1 + p*dp[i+1]
    // => dp[i] = (1 + p*dp[i+1]) * inv_p
    dp[i] = (1 + p % MOD * dp[i + 1] % MOD) % MOD * inv_p % MOD;
}
Key distinction: Forward DP computes probabilities (where are you now?). Backward DP computes expected values (what happens from here?). Most CP problems use backward DP because they ask for expected value.

Absorbing States and Self-Loops

When a state transitions to itself with some probability (e.g., "retry if you fail"), you get a self-loop in the recurrence. Solve algebraically:

E[s] = 1 + p · E[next] + (1−p) · E[s]  ⇒  E[s] = (1 + p · E[next]) / p

Pattern 4: Expected Moves to Target

A very common pattern: "Starting from state S, what is the expected number of moves/steps to reach state T?"

The System of Equations Approach

When the state graph has cycles, backward DP alone doesn't work because states depend on each other circularly. Instead, set up a system of linear equations and solve with Gaussian elimination.

Example: Expected Steps in a Random Walk on a Cycle

Problem: You're on a cycle of n nodes. Each step, move to a random neighbor (left or right with equal probability). Expected steps to return to start?

Solution: By symmetry this is a classic result: E = n2/2 for even n, but let's derive it via the equation approach for general graphs.

#include <bits/stdc++.h>
using namespace std;

// Gaussian elimination for expected values on a graph
// E[i] = 1 + (1/deg(i)) * sum_{j neighbor of i} E[j]
// E[target] = 0

const int MAXN = 305;
const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

long long mat[MAXN][MAXN + 1]; // augmented matrix
int n;

void gauss() {
    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = col; row < n; row++)
            if (mat[row][col] != 0) { pivot = row; break; }
        if (pivot == -1) continue;
        swap(mat[col], mat[pivot]);

        long long inv = power(mat[col][col], MOD - 2, MOD);
        for (int j = col; j <= n; j++)
            mat[col][j] = mat[col][j] * inv % MOD;

        for (int row = 0; row < n; row++) {
            if (row == col || mat[row][col] == 0) continue;
            long long factor = mat[row][col];
            for (int j = col; j <= n; j++)
                mat[row][j] = (mat[row][j] - factor * mat[col][j] % MOD + MOD) % MOD;
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n;
    int target = 0;

    // Build equations: E[i] - (1/2)*E[left] - (1/2)*E[right] = 1
    // E[target] = 0
    long long inv2 = power(2, MOD - 2, MOD);
    memset(mat, 0, sizeof mat);

    for (int i = 0; i < n; i++) {
        if (i == target) {
            mat[i][i] = 1;
            mat[i][n] = 0;
        } else {
            mat[i][i] = 1;
            int left = (i - 1 + n) % n;
            int right = (i + 1) % n;
            mat[i][left] = (MOD - inv2) % MOD;
            mat[i][right] = (MOD - inv2) % MOD;
            mat[i][n] = 1;
        }
    }

    gauss();

    // E[start] where start = 1 (or any non-target node)
    cout << mat[1][n] << "\n";
    return 0;
}
Warning: Gaussian elimination is O(n3) and works for n ≤ ~300. For larger state spaces, look for structure that allows the equations to be solved in closed form or via sparse methods.

Common "Expected Moves" Variants

  • Coupon collector: E = n · H(n) = n · Σ(1/i)
  • Gambler's ruin: Closed-form based on p and starting position
  • Random walk to boundary: E = distance2 for unbiased walk
  • Absorbing Markov chain: E = (I − Q)−1 · 1 where Q is the transient submatrix

Pattern 5: Random Graph Properties

Problems about random permutations, random graphs, or random structures often ask for expected values of structural properties.

Random Permutation Facts

  • Expected number of fixed points: 1 (for any n)
  • Expected number of cycles: H(n) = Σi=1..n 1/i
  • Expected length of the longest cycle: ~0.6243·n (Golomb–Dickman constant)
  • Expected number of inversions: n(n−1)/4
  • Expected number of descents: (n−1)/2

Example: Expected Cycles in Random Permutation

Problem: Find the expected number of cycles in a random permutation of [1..n], modulo 998244353.

Solution: Element i starts a new cycle with probability 1/i (it's the smallest element in its cycle with prob 1/i... actually, by a clever argument, the probability that position i is a cycle leader is 1/i in a uniform random permutation). So E[cycles] = Σi=1..n 1/i = H(n).

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    long long ans = 0;
    for (int i = 1; i <= n; i++)
        ans = (ans + power(i, MOD - 2, MOD)) % MOD;

    cout << ans << "\n";
    return 0;
}

Random Graph: Expected Components

In an Erdős–Rényi random graph G(n, p), the expected number of isolated vertices is n · (1−p)n−1. More generally, the expected number of components of size k can be computed via the contribution technique.

Pattern recognition: When a problem says "random permutation" or "random graph," immediately think about known expected values for structural properties. Many of these have clean closed-form answers involving harmonic numbers, factorials, or simple fractions.

Modular Probability Template

Here is a complete contest-ready template that includes all the utilities you need for probability problems modulo a prime:

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;
const int MAXN = 2000005;

long long power(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    if (base < 0) base += mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long inv(long long x) { return power(x, MOD - 2, MOD); }

long long fact[MAXN], inv_fact[MAXN], inv_arr[MAXN];

void precompute() {
    fact[0] = 1;
    for (int i = 1; i < MAXN; i++)
        fact[i] = fact[i - 1] * i % MOD;

    inv_fact[MAXN - 1] = power(fact[MAXN - 1], MOD - 2, MOD);
    for (int i = MAXN - 2; i >= 0; i--)
        inv_fact[i] = inv_fact[i + 1] * (i + 1) % MOD;

    inv_arr[1] = 1;
    for (int i = 2; i < MAXN; i++)
        inv_arr[i] = (MOD - MOD / i) * inv_arr[MOD % i] % MOD;
}

long long C(int n, int k) {
    if (k < 0 || k > n) return 0;
    return fact[n] % MOD * inv_fact[k] % MOD * inv_fact[n - k] % MOD;
}

long long P(int n, int k) {
    if (k < 0 || k > n) return 0;
    return fact[n] % MOD * inv_fact[n - k] % MOD;
}

// Modular fraction: a/b mod p
long long frac(long long a, long long b) {
    return (a % MOD + MOD) % MOD * inv(b) % MOD;
}

// Harmonic number H(n) = sum 1/i for i=1..n, mod p
long long harmonic(int n) {
    long long h = 0;
    for (int i = 1; i <= n; i++)
        h = (h + inv_arr[i]) % MOD;
    return h;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    precompute();

    // Your solution here

    return 0;
}
Usage notes: This template precomputes factorials and inverses up to 2×106. Adjust MAXN for your problem's constraints. The precompute() function runs in O(MAXN) time.

Cheat Sheet

Quick reference for the most commonly needed formulas in CP probability problems. All formulas assume modular arithmetic with a prime p.

  • P = a/b mod p: a · bp−2 mod p
  • Linearity: E[aX + bY] = a·E[X] + b·E[Y] (even dependent!)
  • Indicator: E[1A] = P(A)
  • Geometric: E[trials] = 1/p → inv(p) mod prime
  • Coupon collector: n·H(n) = n·Σinv[i]
  • Permutation: Fixed pts = 1, Cycles = H(n), Inversions = n(n−1)/4
// DP-A: Forward probability dp[next] += dp[cur] * P(transition) // DP-B: Backward E[X] dp[s] = 1 + Σ P(s→t) * dp[t] // DP-C: Self-loop dp[s] = c / (1 - p_self) // use modular inverse
