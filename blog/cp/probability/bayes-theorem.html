Bayes’ Theorem & Total Probability

June 2026 · 20 min read

In Part 1 we introduced conditional probability: P(A | B) = P(A ∩ B) / P(B). Bayes’ theorem takes the idea further, letting us reverse the conditioning. Given how likely the evidence is under each hypothesis, Bayes tells us how to update our belief about which hypothesis is true. This pattern crops up in contest problems involving urns, random graphs, interactive queries, and game theory.

Law of Total Probability

Before deriving Bayes’ rule we need its companion result. Suppose A1, A2, …, An form a partition of Ω — they are pairwise disjoint and their union is Ω. Then for any event B:

P(B) = ∑i=1n P(B | Ai) · P(Ai)

Proof. Since {Ai} is a partition, B = B ∩ Ω = B ∩ (A1 ∪ … ∪ An) = (B ∩ A1) ∪ … ∪ (B ∩ An). The pieces are disjoint (since the Ai are), so by axiom 3: P(B) = ∑ P(B ∩ Ai) = ∑ P(B | Ai) P(Ai). ▮

Intuition. Think of “paths to B.” The universe is split into exhaustive, non-overlapping scenarios Ai. To find the total probability of B you add up the contribution from each scenario.

Example – two urns. Urn 1 has 3 red and 7 blue balls. Urn 2 has 6 red and 4 blue balls. You pick an urn uniformly at random and draw a ball. What is the probability the ball is red?

Let A1 = “urn 1 chosen”, A2 = “urn 2 chosen”, B = “red ball drawn.” P(B) = P(B|A1)P(A1) + P(B|A2)P(A2) = (3/10)(1/2) + (6/10)(1/2) = 9/20.

CP Tip: In many contest problems the partition is over “types” or “states”, and the law of total probability becomes a DP recurrence: the probability of being in a particular state at time t+1 is the weighted sum over states at time t.

Bayes’ Rule

Bayes’ theorem follows immediately from the definition of conditional probability:

P(Ai | B) = P(B | Ai) · P(Ai) / P(B)

When combined with the law of total probability for the denominator we get the full “Bayes recipe”:

P(Ai | B) = P(B | Ai) P(Ai) / ∑j P(B | Aj) P(Aj)

Terminology.

  • Prior P(Ai) — your belief before seeing evidence.
  • Likelihood P(B | Ai) — how likely the evidence is under hypothesis i.
  • Posterior P(Ai | B) — updated belief after seeing the evidence.
  • Marginal likelihood P(B) — normalization constant.
Warning: Bayes’ theorem is trivially true (it’s just algebra of conditional probability). The difficulty in CP is correctly identifying the hypotheses, the evidence, and computing the likelihoods. Always write down the partition first.

Derivation & Proof

Full derivation.
Start with the definition of conditional probability applied in two ways:

(1)  P(A ∩ B) = P(A | B) · P(B)
(2)  P(A ∩ B) = P(B | A) · P(A)

Equating (1) and (2):
P(A | B) · P(B) = P(B | A) · P(A)

Dividing both sides by P(B) (assuming P(B) > 0):
P(A | B) = P(B | A) · P(A) / P(B) ▮

The general multi-hypothesis version follows by replacing P(B) with the law of total probability expansion.

Why is this useful? Often P(A | B) is what we want but P(B | A) is what we can compute. For instance, given that a patient tested positive (B), what is the probability they have the disease (A)? The likelihood P(positive | disease) is determined by the test’s sensitivity, and the prior P(disease) is the prevalence. Bayes combines these into the posterior.

Medical Test & Monty Hall Examples

Medical test

A disease affects 1 in 1000 people. A test has 99% sensitivity (true positive rate) and 95% specificity (true negative rate — 5% false positive rate). If a person tests positive, what is the probability they actually have the disease?

Let D = “has disease”, + = “tests positive.”

  • P(D) = 0.001,   P(Dc) = 0.999
  • P(+ | D) = 0.99,   P(+ | Dc) = 0.05

Total probability of positive: P(+) = 0.99 × 0.001 + 0.05 × 0.999 = 0.00099 + 0.04995 = 0.05094.

Bayes: P(D | +) = 0.00099 / 0.05094 ≈ 1.94%.

Despite the test being 99% sensitive, the posterior is under 2%. This is because the disease is rare, so false positives vastly outnumber true positives. This “base rate fallacy” appears in many probability contest problems.

Key insight: When the prior is very low, even a highly accurate test produces many false positives relative to true positives. Always account for base rates.

The Monty Hall problem

You pick door 1 out of three. The host (who knows where the prize is) opens door 3, revealing a goat. Should you switch to door 2?

Let Ci = “prize behind door i.” Prior: P(Ci) = 1/3 for each. Let H3 = “host opens door 3.”

  • P(H3 | C1) = 1/2 — host can open either 2 or 3.
  • P(H3 | C2) = 1 — host must open 3 (can’t open 2, can’t open your door).
  • P(H3 | C3) = 0 — host won’t open the prize door.

P(H3) = (1/2)(1/3) + (1)(1/3) + (0)(1/3) = 1/2.

P(C2 | H3) = (1 · 1/3) / (1/2) = 2/3. Switching wins with probability 2/3.

C++ – Bayesian probability computation

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // n hypotheses, each with prior P(H_i) and likelihood P(E | H_i)
    int n;
    cin >> n;
    vector<double> prior(n), likelihood(n);
    for(int i = 0; i < n; i++) cin >> prior[i];
    for(int i = 0; i < n; i++) cin >> likelihood[i];

    // P(E) via total probability
    double pE = 0;
    for(int i = 0; i < n; i++)
        pE += likelihood[i] * prior[i];

    // Posterior via Bayes
    cout << fixed << setprecision(9);
    for(int i = 0; i < n; i++){
        double posterior = likelihood[i] * prior[i] / pE;
        cout << "P(H" << i+1 << " | E) = " << posterior << "\n";
    }
    return 0;
}

CP Applications

Application 1: Urn selection with observations

Problem. There are k urns. Urn i contains ri red and bi blue balls. An urn is chosen uniformly at random, and you draw m balls (with replacement). You observe the sequence of colours. Output the posterior probability of each urn.

This is a direct application of Bayes with sequential updates. After each draw, the posterior from the previous step becomes the new prior. 

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int k, m;
    cin >> k >> m;
    vector<double> r(k), b(k), post(k);
    for(int i = 0; i < k; i++){
        cin >> r[i] >> b[i];
        post[i] = 1.0 / k;  // uniform prior
    }

    for(int j = 0; j < m; j++){
        char c;
        cin >> c;  // 'R' or 'B'
        double pE = 0;
        vector<double> lik(k);
        for(int i = 0; i < k; i++){
            double total_i = r[i] + b[i];
            lik[i] = (c == 'R') ? r[i] / total_i : b[i] / total_i;
            pE += lik[i] * post[i];
        }
        for(int i = 0; i < k; i++){
            post[i] = lik[i] * post[i] / pE;
        }
    }

    cout << fixed << setprecision(9);
    for(int i = 0; i < k; i++){
        cout << "P(Urn " << i+1 << ") = " << post[i] << "\n";
    }
    return 0;
}

Application 2: Conditional probability DP

Problem. You have a biased coin with P(H) = p. Flip it n times. Given that you observed exactly k heads, what is the probability the first flip was heads?

Let A = “first flip is heads”, B = “exactly k heads total.” By Bayes:

P(A | B) = P(B | A) · P(A) / P(B)

P(B | A) = C(n−1, k−1) · pk−1 · (1−p)n−k. P(B) = C(n, k) · pk · (1−p)n−k. P(A) = p.

Simplifying: P(A | B) = C(n−1, k−1) / C(n, k) = k/n.

Elegant result: Given k heads in n flips, the probability any specific flip was heads is k/n — regardless of the coin’s bias! This symmetry argument generalizes beautifully and is a favourite in CP.

Application 3: Random graphs and edge existence

In random graph problems (Erdős–Rényi G(n,p)), you often need to compute the probability a graph has certain properties (e.g., is connected). Bayes’ theorem is useful when conditioning on the existence of a specific edge or set of edges. For example: “Given that node 1 and node 2 are in the same connected component, what is the probability edge (1,2) exists?”

This uses Bayes with A = “edge (1,2) exists” and B = “1 and 2 are connected.” The likelihood P(B | A) = 1 (if the edge exists, they are definitely connected), while P(B | Ac) is the probability they are connected via other paths. 

#include <bits/stdc++.h>
using namespace std;

// Monte Carlo: P(edge(0,1) exists | 0 and 1 connected) in G(n, p)
int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n = 6;
    double p = 0.3;
    int trials = 1000000;
    mt19937_64 rng(42);
    uniform_real_distribution<double> dist(0.0, 1.0);

    long long connected_count = 0, edge_and_connected = 0;
    for(int t = 0; t < trials; t++){
        // generate adjacency
        vector<vector<int>> adj(n);
        bool edge01 = false;
        for(int i = 0; i < n; i++)
            for(int j = i+1; j < n; j++)
                if(dist(rng) < p){
                    adj[i].push_back(j);
                    adj[j].push_back(i);
                    if(i == 0 && j == 1) edge01 = true;
                }
        // BFS to check connectivity of 0 and 1
        vector<bool> vis(n, false);
        queue<int> q;
        q.push(0); vis[0] = true;
        while(!q.empty()){
            int u = q.front(); q.pop();
            for(int v : adj[u]) if(!vis[v]){
                vis[v] = true; q.push(v);
            }
        }
        if(vis[1]){
            connected_count++;
            if(edge01) edge_and_connected++;
        }
    }
    cout << fixed << setprecision(6);
    cout << "P(edge | connected) = "
         << (double)edge_and_connected / connected_count << "\n";
    return 0;
}
Warning: In CP problems with modular arithmetic, Bayes’ formula P(A|B) = P(A∩B) / P(B) becomes a modular division. Compute P(A∩B) · P(B)−1 mod p using Fermat’s little theorem. Make sure P(B) ≠ 0 mod p!

Practice Problems

Test your Bayes skills with these exercises:

  1. Three urns. Urn A has 5 red, 3 blue; Urn B has 2 red, 6 blue; Urn C has 4 red, 4 blue. An urn is chosen uniformly at random and one ball is drawn. It is red. Find the posterior probability for each urn.
  2. Sequential updates. Using the three urns above, suppose you draw 3 balls (with replacement) and get Red, Blue, Red. Compute the posterior for each urn after all three observations.
  3. False positive puzzle. A factory produces widgets. 2% are defective. Inspection catches 90% of defectives but also flags 5% of good widgets. If a widget is flagged, what is P(defective)?
  4. Monty Hall variant. There are 4 doors. You pick door 1. The host opens 2 doors revealing goats (he always can, since at most 1 door has the prize). Should you switch? What is your win probability if you switch vs. stay?
  5. Coin identification (CF-style). You have two coins: coin A with P(H) = 0.6 and coin B with P(H) = 0.4. A coin is picked uniformly at random and flipped 10 times. You observe 7 heads. What is the posterior probability it is coin A?

Solution sketch for Problem 3

D = “defective”, F = “flagged.” P(D) = 0.02, P(F|D) = 0.90, P(F|Dc) = 0.05.

P(F) = 0.90×0.02 + 0.05×0.98 = 0.018 + 0.049 = 0.067.

P(D|F) = 0.018 / 0.067 ≈ 26.87%. Much higher than the 2% base rate, but still far from certain.

Solution sketch for Problem 5

Let X = number of heads. X | coin A ~ Bin(10, 0.6), X | coin B ~ Bin(10, 0.4). 

#include <bits/stdc++.h>
using namespace std;

double binomial_pmf(int n, int k, double p){
    // C(n,k) * p^k * (1-p)^(n-k)
    double res = 1.0;
    for(int i = 0; i < k; i++)
        res *= (double)(n - i) / (i + 1) * p;
    for(int i = 0; i < n - k; i++)
        res *= (1 - p);
    // fix: need to separate C(n,k) and powers
    return res;  // simplified for illustration
}

int main(){
    double pA = 0.5, pB = 0.5;  // uniform prior
    double likA = 1, likB = 1;
    double pa = 0.6, pb = 0.4;
    int n = 10, k = 7;

    // C(10,7) * 0.6^7 * 0.4^3
    // C(10,7) * 0.4^7 * 0.6^3
    // C(10,7) cancels in Bayes ratio!
    double numA = pow(pa, k) * pow(1-pa, n-k);
    double numB = pow(pb, k) * pow(1-pb, n-k);

    double postA = numA * pA / (numA * pA + numB * pB);
    cout << fixed << setprecision(6);
    cout << "P(coin A | 7 heads) = " << postA << "\n";
    // Expected: ≈ 0.8670
    return 0;
}
Key takeaway: Bayes’ theorem is a mechanical recipe. The hard part in CP is (1) identifying the right partition, (2) computing likelihoods efficiently (often via DP or combinatorics), and (3) handling modular arithmetic when probabilities must be expressed as fractions mod a prime.
