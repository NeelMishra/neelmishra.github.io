DP on Probability Distributions

August 2026 · 24 min read

Probability Meets DP

Dynamic programming is the workhorse of competitive programming. When we combine it with probability, a new family of problems opens up: instead of counting ways or optimising values, we propagate probability mass through a state graph.

The core idea is simple but powerful:

dp[state] = probability of being in that state.

Transitions take the form dp[next_state] += dp[current_state] × P(transition). Because probabilities sum to 1 at every stage, the total probability mass is conserved — a useful sanity check during implementation.

Two flavours of probability DP:
1. Forward DP: start from the initial state and propagate probability forward through time/stages.
2. Backward DP: define dp[state] as the probability of eventually reaching the goal from that state, and solve backwards (or via a system of equations).

In this post we focus on the forward style, as it is the most natural extension of classical DP. The backward style was covered in the Markov chains post (using linear equations).

When to Use Probability DP

  • The process has a clear sequence of stages (rolls, rounds, steps).
  • The state space is finite and not too large.
  • Transitions from each state depend only on the current state (Markov property).
  • You need the exact probability of one or more outcomes.

Dice Sum Probability

Problem: You roll a fair 6-sided die n times. What is the probability that the sum of all rolls equals exactly s?

State Definition

Let dp[i][j] = probability that after i rolls the sum is j. The answer is dp[n][s].

Base case: dp[0][0] = 1 (before any roll, the sum is 0 with certainty).

Transition

For each roll i (1 ≤ i ≤ n) and each current sum j, for each face value f ∈ {1, 2, 3, 4, 5, 6}:

dp[i][j + f] += dp[i−1][j] × (1/6).

Complexity

O(n × s × 6) time, O(s) space with rolling array optimisation. For n ≤ 1000 and s ≤ 6000, this runs in milliseconds.

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, s;
    cin >> n >> s;

    // dp[j] = probability that current sum is j
    vector<double> dp(s + 1, 0);
    dp[0] = 1.0;

    for(int i = 0; i < n; i++){
        vector<double> ndp(s + 1, 0);
        for(int j = 0; j <= s; j++){
            if(dp[j] == 0) continue;
            for(int f = 1; f <= 6; f++){
                if(j + f <= s)
                    ndp[j + f] += dp[j] / 6.0;
            }
        }
        dp = ndp;
    }
    cout << fixed << setprecision(9) << dp[s] << "\n";
    return 0;
}
Modular version: If the problem asks for the probability as a fraction p/q modulo 109+7, count the number of ways to achieve sum s (standard DP) and divide by 6n using modular inverse: answer = ways × modinv(6n, MOD).

Extension: Non-uniform Dice

If face f has probability wf (not necessarily 1/6), the transition becomes dp[i][j+f] += dp[i−1][j] × wf. The structure is identical; only the weights change.

Probability Knapsack

Problem: You have n items, each with a weight wi. Each item is included independently with probability pi. What is the probability that the total weight is exactly W?

State Definition

dp[i][j] = probability that among the first i items, the total included weight is j.

Base case: dp[0][0] = 1.

Transition

For item i (weight wi, inclusion probability pi):

dp[i][j] = dp[i−1][j] × (1 − pi) + dp[i−1][j − wi] × pi

The first term corresponds to not including item i; the second to including it.

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, W;
    cin >> n >> W;
    vector<int> w(n);
    vector<double> p(n);
    for(int i = 0; i < n; i++) cin >> w[i] >> p[i];

    vector<double> dp(W + 1, 0);
    dp[0] = 1.0;

    for(int i = 0; i < n; i++){
        // Process in reverse to avoid using same item twice
        for(int j = W; j >= 0; j--){
            if(j + w[i] <= W)
                dp[j + w[i]] += dp[j] * p[i];
            dp[j] *= (1.0 - p[i]);
        }
    }
    cout << fixed << setprecision(9) << dp[W] << "\n";
    return 0;
}
Order matters! When using a 1-D rolling array, process weights in reverse (from W down to 0) to avoid counting the same item twice — exactly as in the classical 0-1 knapsack. But here we must also scale dp[j] by (1 − pi) to account for the “skip” case. Process the “include” case before scaling, or accumulate into a separate array.

Variant: At-Least-W Probability

To find P(total weight ≥ W), sum dp[j] for j = W, W+1, …, max_weight. Or equivalently, compute 1 − Σj=0W−1 dp[j].

General Probability Transitions

The dice and knapsack examples are special cases of a general pattern. Let us formalise:

The General Framework

  1. Define the state space: what information do you track?
  2. Define the stages: what sequential process drives the system?
  3. For each state and each possible event, write the transition: dp[next] += dp[curr] × P(event).
  4. The answer is dp[target_state] after the final stage.

Example: Coin Flipping Sequences

Problem: Flip a biased coin (P(H) = p) repeatedly. What is the probability of getting k consecutive heads within n flips?

State: dp[i][j] = probability that after i flips, the current run of consecutive heads is j and we have not yet achieved k consecutive heads.

Transitions:

  • Flip heads: dp[i+1][j+1] += dp[i][j] × p (if j+1 < k).
  • Flip tails: dp[i+1][0] += dp[i][j] × (1−p).
  • If j+1 = k, the probability mass is “absorbed” into the success state.

The answer is 1 − Σj=0k−1 dp[n][j], i.e. 1 minus the probability of never achieving k consecutive heads. 

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    double p;
    cin >> n >> k >> p;

    // dp[j] = prob current run of heads is j, no k-run yet
    vector<double> dp(k, 0);
    dp[0] = 1.0;

    for(int i = 0; i < n; i++){
        vector<double> ndp(k, 0);
        for(int j = 0; j < k; j++){
            if(dp[j] == 0) continue;
            // Heads: extend run
            if(j + 1 < k)
                ndp[j + 1] += dp[j] * p;
            // If j+1 == k, mass is absorbed (success)

            // Tails: reset run
            ndp[0] += dp[j] * (1.0 - p);
        }
        dp = ndp;
    }
    double no_success = 0;
    for(int j = 0; j < k; j++) no_success += dp[j];
    cout << fixed << setprecision(9) << 1.0 - no_success << "\n";
    return 0;
}
Sanity check: At every stage, the total probability mass (sum of all dp values plus absorbed mass) should equal 1. If it doesn’t, there is a bug in your transitions. Add an assertion during debugging.

Bitmask Probability DP

When the state involves a subset of items (visited cities, completed tasks), bitmask DP is the natural representation. Combined with probability, this gives a powerful technique for problems like “probability of visiting all nodes in a random process”.

Problem: Random TSP Completion

There are n cities (n ≤ 20). Starting from city 0, at each step you move to a random unvisited city (each equally likely). What is the probability that the total distance is at most D?

State Definition

dp[mask][v] = probability of having visited exactly the cities in mask and currently being at city v, with total distance ≤ D.

Base case: dp[1][0] = 1 (visited only city 0, at city 0, distance 0).

Transition

From state (mask, v), consider each unvisited city u (bit u not set in mask). If dist(v, u) + accumulated_distance ≤ D:

dp[mask | (1 << u)][u] += dp[mask][v] / (number of unvisited cities).

C++ Implementation (Simplified)

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> dist(n, vector<int>(n));
    for(int i = 0; i < n; i++)
        for(int j = 0; j < n; j++)
            cin >> dist[i][j];

    int full = (1 << n) - 1;
    // dp[mask][v] = probability of being in state (mask, v)
    vector<vector<double>> dp(1 << n, vector<double>(n, 0));
    dp[1][0] = 1.0;

    for(int mask = 1; mask <= full; mask++){
        int visited = __builtin_popcount(mask);
        int unvisited = n - visited;
        if(unvisited == 0) continue;
        for(int v = 0; v < n; v++){
            if(dp[mask][v] == 0) continue;
            if(!(mask & (1 << v))) continue;
            for(int u = 0; u < n; u++){
                if(mask & (1 << u)) continue;
                int nmask = mask | (1 << u);
                dp[nmask][u] += dp[mask][v] / unvisited;
            }
        }
    }

    // Answer: probability of visiting all cities
    double ans = 0;
    for(int v = 0; v < n; v++) ans += dp[full][v];
    cout << fixed << setprecision(9) << ans << "\n";
    return 0;
}
Memory: Bitmask DP uses O(2n × n) space. For n = 20 and double precision, that is 20 × 220 × 8 bytes ≈ 160 MB. Keep n ≤ 20 (preferably ≤ 16) or use space-saving tricks like processing masks in order and discarding old layers.

Expected Value Variant

To compute the expected total distance, track dp[mask][v] = (probability, expected_distance) pairs and propagate both. Or use linearity of expectation when applicable.

Gaussian Elimination for Probabilities

Sometimes the probability DP has cycles: from state A you can reach state B, and from B you can return to A. A straightforward forward DP cannot handle this because there is no topological ordering. Instead, we set up a system of linear equations and solve with Gaussian elimination.

Setup

For each non-absorbing state i: P(i) = Σj pij · P(j), where P(goal) = 1 and P(lose) = 0. Rearranging gives a linear system: P(i) − Σj pij P(j) = 0.

Expected Value Version

For expected steps to a goal: E(i) = 1 + Σj pij · E(j), with E(goal) = 0. Same structure, different right-hand side.

C++ — Gaussian Elimination Template

#include <bits/stdc++.h>
using namespace std;

const double EPS = 1e-9;

// Solves Ax = b, returns x. A is n x n, b is n x 1.
vector<double> gauss(vector<vector<double>> A, vector<double> b){
    int n = A.size();
    for(int i = 0; i < n; i++) A[i].push_back(b[i]);

    for(int col = 0; col < n; col++){
        int best = col;
        for(int row = col + 1; row < n; row++)
            if(abs(A[row][col]) > abs(A[best][col]))
                best = row;
        swap(A[col], A[best]);
        if(abs(A[col][col]) < EPS) continue; // singular
        double pivot = A[col][col];
        for(int j = col; j <= n; j++) A[col][j] /= pivot;
        for(int row = 0; row < n; row++){
            if(row == col) continue;
            double f = A[row][col];
            for(int j = col; j <= n; j++)
                A[row][j] -= f * A[col][j];
        }
    }
    vector<double> x(n);
    for(int i = 0; i < n; i++) x[i] = A[i][n];
    return x;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, goal;
    cin >> n >> goal;

    vector<vector<pair<int,double>>> adj(n);
    for(int i = 0; i < n; i++){
        int deg; cin >> deg;
        for(int j = 0; j < deg; j++){
            int to; double prob;
            cin >> to >> prob;
            adj[i].push_back({to, prob});
        }
    }

    // E(i) - sum_j p_ij * E(j) = 1 for i != goal
    // E(goal) = 0
    vector<vector<double>> A(n, vector<double>(n, 0));
    vector<double> b(n, 0);
    for(int i = 0; i < n; i++){
        if(i == goal){
            A[i][i] = 1; b[i] = 0;
        } else {
            A[i][i] = 1;
            for(auto &[j, p] : adj[i])
                A[i][j] -= p;
            b[i] = 1;
        }
    }
    vector<double> E = gauss(A, b);
    for(int i = 0; i < n; i++)
        cout << "E[" << i << "] = "
             << fixed << setprecision(6) << E[i] << "\n";
    return 0;
}
When to use which: DAG → forward DP. Cyclic graph → Gaussian elimination O(n3). Large n with small state space → matrix exponentiation O(m3 log n).

CP Applications

1. Dice Games with Target Scores

“Roll dice until your score reaches T. What is the probability your score lands exactly on T?” This is a direct dice-sum DP. If exceeding T is also a valid ending condition, adjust the transitions to absorb probability mass at states ≥ T.

2. Random Graph Connectivity

Given n nodes and m potential edges, each present independently with probability p. What is the probability the graph is connected? Use bitmask DP over subsets of nodes, tracking which components form. For small n (≤ 15), this is tractable.

3. Coupon Collector via DP

There are n distinct coupons. At each step you receive a uniformly random coupon. What is the probability of collecting all n within k steps? State: dp[i][mask] where mask tracks which coupons are collected. For small n, bitmask DP; for larger n, use the inclusion-exclusion formula instead.

4. Probability on Trees & Expected Value Preview

A particle starts at the root and moves to a random child each step. Since trees are DAGs, forward DP works: dp[child] += dp[parent] / (children count). The companion technique — Expected Value DP — uses transitions E[state] = Σ P(transition) × (cost + E[next_state]) and will be the focus of the next post.

Common mistake: When the state graph has cycles, do NOT use forward DP naively. Always verify the state graph is a DAG; if not, use Gaussian elimination.

5. Combining Probability DP with Modular Arithmetic

Many CP problems ask for the answer modulo 109+7 or 998244353. Store dp values as modular integers and replace 1/6 with modinv(6). The DP structure is unchanged — only the arithmetic layer differs. 

#include <bits/stdc++.h>
using namespace std;
const int MOD = 998244353;

long long power(long long a, long long b, long long mod){
    long long res = 1; a %= mod;
    while(b > 0){
        if(b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}
long long modinv(long long a, long long mod){ return power(a, mod - 2, mod); }

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, s;
    cin >> n >> s;

    vector<long long> dp(s + 1, 0);
    dp[0] = 1;
    long long inv6 = modinv(6, MOD);

    for(int i = 0; i < n; i++){
        vector<long long> ndp(s + 1, 0);
        for(int j = 0; j <= s; j++){
            if(dp[j] == 0) continue;
            for(int f = 1; f <= 6; f++){
                if(j + f <= s)
                    ndp[j + f] = (ndp[j + f] + dp[j] % MOD * inv6) % MOD;
            }
        }
        dp = ndp;
    }
    cout << dp[s] << "\n";
    return 0;
}

Practice Problems

Sharpen your probability DP skills with these problems:

  1. Dice Sum (easy): Roll n fair dice. Compute the probability that the sum is exactly s.
  2. Coin Streak (easy–medium): Flip a fair coin n times. What is the probability of getting at least k consecutive heads?
  3. Probabilistic Knapsack (medium): Given n items each included independently with probability pi, compute P(total weight ≥ W).
  4. Random Walk DP (medium): A particle at 0 on [0, N] moves ±1 with probabilities p and 1−p. Compute P(reach N before 0) using DP.
  5. Coupon Collector Probability (medium–hard): n coupon types (n ≤ 15), each step yields type i with probability pi. Compute P(all collected within k steps) via bitmask DP.
  6. Random Graph Connectivity (hard): n nodes (n ≤ 15), each pair connected independently with probability p. Compute P(connected) using subset DP.
  7. Cyclic Expected Value (hard): Directed graph with transition probabilities; compute expected steps to a target via Gaussian elimination.
Summary: Probability DP propagates probability mass through a state graph. For DAGs use forward DP; for cyclic graphs use Gaussian elimination. Next up: Expected Value DP.
