Expected Value & Linearity of Expectation

June 2026 · 25 min read

If you learn only one technique from this entire probability module, make it linearity of expectation. It is, without exaggeration, the most frequently useful probabilistic tool in competitive programming. Problems that look intractable become one-liners once you decompose the random variable into indicator sums and apply linearity. In this post we build from the definition of expected value all the way to computing E[X] mod p with modular inverses.

Definition & Intuition

The expected value (or expectation, or mean) of a discrete random variable X is:

E[X] = ∑x x · P(X = x)

The sum is over all values x in the range of X, weighted by their probabilities. Intuitively, E[X] is the “long-run average” — if you repeated the experiment many times, the average outcome converges to E[X] (law of large numbers).

Example – fair die. X = result of rolling a fair die.

E[X] = 1·(1/6) + 2·(1/6) + 3·(1/6) + 4·(1/6) + 5·(1/6) + 6·(1/6) = 21/6 = 3.5

Note that E[X] = 3.5 is not even a possible outcome. That’s fine — expected value is an average, not a prediction of any single trial.

Example – Bernoulli. X is 1 with probability p and 0 with probability 1−p. Then E[X] = 1·p + 0·(1−p) = p. This makes indicator random variables extremely convenient: E[1A] = P(A).

CP Tip: Whenever you see “find the expected number of…” in a problem, immediately think: can I write this as a sum of indicator random variables? If yes, linearity of expectation will let you compute each piece independently.

Properties of expectation

  • E[c] = c for any constant c.
  • E[cX] = c·E[X] — constants pull out.
  • E[X + Y] = E[X] + E[Y] — linearity (see next section).
  • If X ≥ 0 always, then E[X] ≥ 0.
  • If X and Y are independent, E[XY] = E[X]·E[Y].

Linearity of Expectation

The single most important property:

E[X1 + X2 + … + Xn] = E[X1] + E[X2] + … + E[Xn]

This holds regardless of whether the Xi are independent or not. That is the magic. You do not need to know how the variables interact — you can compute each expectation separately and add them up.

Quick example – sum of dice. Roll n fair dice. Let S be the sum. S = X1 + … + Xn where each Xi is the result of die i. By linearity, E[S] = n · E[X1] = n · 3.5 = 3.5n. No need to enumerate all 6n outcomes!

Warning: Linearity works for sums, not products. E[XY] ≠ E[X]·E[Y] in general (only when X and Y are independent). Similarly, E[1/X] ≠ 1/E[X]. These are common mistakes in contests.

Proof of Linearity

We prove it for two variables; the general case follows by induction. Let X and Y be discrete random variables defined on the same probability space.

Theorem. E[X + Y] = E[X] + E[Y].

Proof. By definition of expectation over the joint distribution:

E[X + Y] = ∑x ∑y (x + y) · P(X=x, Y=y)

Split the sum:
= ∑x ∑y x · P(X=x, Y=y) + ∑x ∑y y · P(X=x, Y=y)

In the first double sum, x does not depend on y, so factor it out of the inner sum:
= ∑x x · ∑y P(X=x, Y=y) + ∑y y · ∑x P(X=x, Y=y)

Now ∑y P(X=x, Y=y) = P(X=x)  (marginalisation), and similarly for the other sum. Therefore:

= ∑x x · P(X=x) + ∑y y · P(Y=y) = E[X] + E[Y]. ▮

The key step is marginalisation: summing the joint probability over one variable yields the marginal probability of the other. This is why independence is not needed — we never assumed anything about how X and Y relate.

Why it matters in CP: In problems like “expected number of inversions in a random permutation,” the indicator variables 1{π(i)>π(j)} are heavily dependent (knowing some inversions affects others). Yet linearity lets us simply sum E[1{ij}] = 1/2 over all C(n,2) pairs to get E = n(n−1)/4.

Classic Problems

1. Expected fixed points in a random permutation

A fixed point of a permutation π is an index i where π(i) = i. Let X = number of fixed points. Define Xi = 1 if π(i) = i. Then X = ∑ Xi.

E[Xi] = P(π(i) = i) = (n−1)! / n! = 1/n.

By linearity: E[X] = n · (1/n) = 1.

Remarkably, no matter how large n is, the expected number of fixed points is exactly 1. This beautiful result would be very hard to derive without linearity.

2. Expected record elements

Element π(i) is a record (or left-to-right maximum) if π(i) > π(j) for all j < i. Define Ri = 1 if π(i) is a record.

P(π(i) is a record) = 1/i, because among the first i elements, each is equally likely to be the largest (by symmetry of random permutations).

E[records] = ∑i=1n 1/i = Hn — the n-th harmonic number. For n = 106 this is about 14.4.

3. The hat-check problem (derangements)

n people check hats, and hats are returned randomly. What is the expected number of people who get their own hat back? This is identical to the fixed-point problem: E = 1.

But what about the probability that nobody gets their hat? That’s P(X = 0) = Dn/n!, where Dn is the number of derangements. Using inclusion-exclusion:

Dn = n! ∑k=0n (−1)k/k!  ≈ n!/e

So P(no fixed points) ≈ 1/e ≈ 0.3679. This converges incredibly fast — for n ≥ 5 it is accurate to many decimal places.

Pattern: Linearity of expectation + indicator variables is the standard approach for expected-value problems in CP. The typical workflow is: (1) decompose X = ∑ Xi, (2) compute E[Xi] for each indicator, (3) sum.

4. Expected number of tosses to see all faces

The coupon collector problem: how many times must you roll a fair die to see all 6 faces? Let Ti be the number of rolls to go from i−1 distinct faces to i distinct faces. At stage i, the probability of seeing a new face is (6−i+1)/6. So Ti ~ Geometric((6−i+1)/6), meaning E[Ti] = 6/(6−i+1).

Total expected rolls: E[T] = ∑i=16 6/(6−i+1) = 6(1 + 1/2 + 1/3 + 1/4 + 1/5 + 1/6) = 6 · H6 = 14.7.

For n coupons: E[T] = n · Hn ~ n ln n. This result is a favourite in CP and appears in problems about random exploration, covering sets, and birthday-style waiting.

Modular Expected Value

Many CP problems ask for the expected value modulo a prime p (usually 109+7 or 998244353). Since expected values are rational numbers a/b, we compute a · b−1 mod p where b−1 is the modular inverse.

Fermat’s little theorem for modular inverse

If p is prime and gcd(b, p) = 1 then bp−1 ≡ 1 (mod p), so b−1 ≡ bp−2 (mod p). Compute this with fast exponentiation in O(log p). 

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long base, long long exp, long long mod){
    long long result = 1;
    base %= mod;
    while(exp > 0){
        if(exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long modinv(long long a, long long mod){
    return power(a, mod - 2, mod);
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // Example: E[X] = 7/3, output mod 998244353
    long long numerator = 7;
    long long denominator = 3;
    long long ans = numerator % MOD * modinv(denominator, MOD) % MOD;
    cout << ans << "\n";
    // Verify: ans * 3 mod MOD should equal 7
    cout << "Check: " << ans * 3 % MOD << "\n";
    return 0;
}
Warning: Modular inverse only exists when gcd(b, p) = 1. Since p is prime, this fails only when b ≡ 0 (mod p). If your denominator can be 0 mod p, the problem is ill-posed (or you need to restructure).

Working with fractions mod p

In DP-on-probability problems, you maintain each state’s probability as a modular integer. Addition works normally. For “multiply by probability p/q”, compute p · modinv(q) mod MOD and multiply. Example: multiplying by 1/2 means multiplying by modinv(2) = (MOD+1)/2.

Precomputing inverses of 1 through N

When you need many inverse values (e.g., for harmonic sums), precompute them in O(N): 

#include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353;
const int MAXN = 1000001;
long long inv[MAXN], fact[MAXN], ifact[MAXN];

void precompute(){
    inv[1] = 1;
    for(int i = 2; i < MAXN; i++)
        inv[i] = (MOD - MOD / i) * inv[MOD % i] % MOD;

    fact[0] = ifact[0] = 1;
    for(int i = 1; i < MAXN; i++){
        fact[i] = fact[i-1] * i % MOD;
        ifact[i] = ifact[i-1] * inv[i] % MOD;
    }
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    precompute();

    // Expected fixed points = sum of 1/i for i=1..n? No!
    // Fixed points: E = n * (1/n) = 1
    int n;
    cin >> n;
    // E[fixed points] = 1, always
    cout << 1 << "\n";

    // Harmonic number H_n mod p
    long long Hn = 0;
    for(int i = 1; i <= n; i++)
        Hn = (Hn + inv[i]) % MOD;
    cout << "H_" << n << " mod p = " << Hn << "\n";
    return 0;
}
The inverse recurrence: inv[i] = −(p/i) · inv[p%i] mod p. This comes from writing p = q·i + r, then q·i + r ≡ 0 (mod p), so i−1 ≡ −q · r−1 (mod p). It’s O(1) per element and is a standard CP technique.

CP Applications

Application 1: Expected inversions in a random permutation

An inversion is a pair (i, j) with i < j and π(i) > π(j). For a uniformly random permutation of [n]:

Define Xij = 1 if (i,j) is an inversion. By symmetry, P(π(i) > π(j)) = 1/2. There are C(n,2) pairs, so:

E[inversions] = C(n,2) · (1/2) = n(n−1)/4

Application 2: DP with expected value

Problem. You are at position 0. Each second, with probability p you move right by 1, else you stay. Find the expected time to reach position n.

Let Ei = expected steps from position i to n. Base case: En = 0. Recurrence:

Ei = 1 + p · Ei+1 + (1−p) · Ei

Rearranging: p · Ei = 1 + p · Ei+1, so Ei = 1/p + Ei+1. Therefore E0 = n/p. 

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m){
    long long r = 1; b %= m;
    while(e > 0){
        if(e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // n = target, p = a/b (probability of moving right)
    long long n, a, b;
    cin >> n >> a >> b;

    // E[0] = n / p = n * b / a, mod MOD
    long long ans = n % MOD * (b % MOD) % MOD;
    ans = ans * power(a, MOD - 2, MOD) % MOD;
    cout << ans << "\n";
    return 0;
}

Application 3: Expected value on trees

Problem. A tree with n nodes. Starting at the root, each step you move to a uniformly random neighbour. Find the expected number of steps to visit all nodes (tree cover time).

This is a random walk problem. For each edge (u,v) in the tree, the expected time for the walk to traverse the edge in each direction is related to the sizes of the two subtrees. Specifically, for an edge separating subtrees of size s and n−s:

E[traverse(u→v)] = 2(n−s)−1   (where s = size of subtree on v's side)

The total expected cover time is the sum of traversal times over all edges (in both directions). By linearity, we just sum over edges.

Application 4: Randomised algorithms – expected running time

Linearity also helps analyse randomised algorithms. For example, in randomised QuickSort, the expected number of comparisons is:

E[comparisons] = ∑i<j P(ai and aj are compared) = ∑i<j 2/(j−i+1)

This simplifies to 2n·Hn − 2n, confirming the O(n log n) expected time. The proof uses linearity over all pairs of elements.

Strategy summary: For expected value in CP, your first instinct should always be: (1) try linearity with indicators, (2) if that doesn’t work, try DP with recurrences of the form Estate = 1 + ∑ pnext · Enext, (3) for output, use modular arithmetic.

Practice Problems

Put your expected value skills to the test:

  1. Dice sum expected value. Roll 100 fair six-sided dice. What is E[sum]? (Trivial with linearity: 100 × 3.5 = 350.)
  2. Random graph edges. G(n,p) is a random graph where each of the C(n,2) possible edges exists independently with probability p. What is the expected number of edges? What is the expected number of triangles?
  3. Coupon collector. There are n types of coupons. Each day you receive one uniformly at random. Compute E[days to collect all n] modulo 998244353. Your code should handle n up to 106.
  4. Expected max. Two fair dice are rolled. What is E[max(die1, die2)]? Hint: P(max ≤ k) = (k/6)2.
  5. Hat-check modular. There are n = 105 people. Compute the expected number of fixed points modulo 109+7. (It is always 1, but verify by computing ∑ (n−1)! / n! = ∑ 1/n using modular inverse.)
  6. Random walk on line (CF-style). Start at 0. Each step: +1 with probability p (given as a/b), −1 with probability 1−p. Find the expected number of steps to reach +n. Output mod 998244353. Hint: gambler’s ruin expected hitting time.
  7. Inversions after partial shuffle. You have a sorted array of n elements. You perform one random swap (choose two distinct indices uniformly at random and swap them). What is the expected number of inversions after the swap?

Solution sketch for Problem 2 (triangles)

The number of potential triangles is C(n,3). For each triple {i,j,k}, the probability all three edges exist is p3 (by independence). By linearity:

E[triangles] = C(n,3) · p3

For n = 100, p = 0.5: E = 161700 · 0.125 = 20212.5.

Solution sketch for Problem 4

P(max = k) = P(max ≤ k) − P(max ≤ k−1) = k2/36 − (k−1)2/36 = (2k−1)/36.

E[max] = ∑k=16 k·(2k−1)/36 = (1+6+15+28+45+66)/36 = 161/36 ≈ 4.472. 

#include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353;

long long power(long long b, long long e, long long m){
    long long r = 1; b %= m;
    while(e > 0){
        if(e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

long long modinv(long long a, long long m){ return power(a, m-2, m); }

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // Coupon collector: E = n * H_n mod p
    int n;
    cin >> n;
    long long inv_val[1000001];
    inv_val[1] = 1;
    for(int i = 2; i <= n; i++)
        inv_val[i] = (MOD - MOD/i) * inv_val[MOD % i] % MOD;

    long long Hn = 0;
    for(int i = 1; i <= n; i++)
        Hn = (Hn + inv_val[i]) % MOD;

    long long ans = (long long)n % MOD * Hn % MOD;
    cout << ans << "\n";
    return 0;
}
