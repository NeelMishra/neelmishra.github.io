Martingales in Competitive Programming

September 2026 · 24 min read

Martingale Definition

A martingale is a sequence of random variables X0, X1, X2, … that models a “fair game”—one where, on average, your future value equals your current value, regardless of the past. Formally:

E[Xn+1 | X1, X2, …, Xn] = Xn

In words: given everything that has happened up to time n, the expected value at time n + 1 is exactly the current value Xn. There is no drift—no systematic tendency to go up or down.

More precisely, we require three conditions for {Xn} to be a martingale with respect to a filtration {Fn}:

  1. Adapted: Xn is determined by the information available at time n (Xn is Fn-measurable)
  2. Integrable: E[|Xn|] < ∞ for all n
  3. Fair game: E[Xn+1 | Fn] = Xn for all n

If instead E[Xn+1 | Fn] ≥ Xn, we have a submartingale (favorable game). If E[Xn+1 | Fn] ≤ Xn, it is a supermartingale (unfavorable game).

Intuition: Think of Xn as your wealth after n rounds of a game. A martingale means the game is perfectly fair—your expected future wealth is always your current wealth. The powerful theorems about martingales let us derive surprising results about when and where fair games end.

Why should competitive programmers care? Many problems involve random walks, stopping conditions, and expected values of random processes. Martingale theory gives us powerful tools—especially the Optional Stopping Theorem—that can turn a complex expected-value computation into an elegant one-line argument.

Examples of Martingales

Let us build intuition through concrete examples that appear frequently in competitive programming.

Example 1: Simple Random Walk

Let Sn = S0 + Z1 + Z2 + … + Zn, where each Zi is +1 or −1 with equal probability 1/2. Then {Sn} is a martingale:

E[Sn+1 | S0, …, Sn] = Sn + E[Zn+1] = Sn + 0 = Sn ✓

Example 2: Squared Random Walk minus Time

Define Mn = Sn2 − n. Then Mn is also a martingale! This is because:

E[Sn+12 | Fn] = E[(Sn + Zn+1)2 | Fn] = Sn2 + 2Sn · E[Zn+1] + E[Zn+12] = Sn2 + 0 + 1 = Sn2 + 1

So E[Mn+1 | Fn] = E[Sn+12 − (n+1) | Fn] = (Sn2 + 1) − (n+1) = Sn2 − n = Mn ✓

This martingale is incredibly useful for computing the expected hitting time of a random walk.

Example 3: Exponential Martingale

For a random walk with step Zi ∈ {+1, −1} equally likely, define Mn = rSn for any r > 0 satisfying E[rZ] = (r + 1/r)/2 = 1, i.e., r = 1 (trivial) or using hyperbolic functions. More generally, for a biased walk with P(Z = +1) = p, the choice r = (1 − p)/p makes {rSn} a martingale.

This exponential martingale is the key tool for the elegant martingale proof of the Gambler’s Ruin theorem.

Common Pitfall: Not every “fair-looking” process is a martingale. For instance, if Zi are i.i.d. with E[Zi] = 0 but the steps are not independent of the current state, the process may not be a martingale. Always verify the conditional expectation property. 
#include <bits/stdc++.h>
using namespace std;

// Verify martingale property by simulation
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int trials = 1000000, steps = 100;

    // Verify: E[S_{n+1} | S_n = v] = v
    int v = 5;
    double sum_next = 0;
    for (int t = 0; t < trials; t++)
        sum_next += v + ((rng() % 2 == 0) ? 1 : -1);
    cout << "E[S_{n+1} | S_n=" << v << "] = "
         << fixed << setprecision(4) << sum_next / trials << "\n";

    // Verify: E[S_n^2 - n] = 0 (squared walk martingale)
    double sum_M = 0;
    for (int t = 0; t < trials; t++) {
        int S = 0;
        for (int i = 0; i < steps; i++)
            S += (rng() % 2 == 0) ? 1 : -1;
        sum_M += (long long)S * S - steps;
    }
    cout << "E[S_" << steps << "^2 - " << steps << "] = "
         << sum_M / trials << " (should be ~0)\n";
    return 0;
}

Optional Stopping Theorem

The Optional Stopping Theorem (OST) is perhaps the most useful result in martingale theory for competitive programming. It states:

If {Xn} is a martingale and τ is a stopping time, then under suitable conditions:

E[Xτ] = E[X0]

In words: the expected value when you stop a fair game equals the expected value when you started, regardless of your stopping strategy. You cannot “beat” a fair game by choosing when to stop.

Sufficient conditions for OST: The theorem holds if any one of these is satisfied:

  1. τ is bounded: there exists N such that τ ≤ N almost surely
  2. E[τ] < ∞ and the increments |Xn+1 − Xn| are bounded
  3. |Xn∧τ| is bounded by an integrable random variable (dominated convergence)
Power of OST: Many expected-value problems in CP can be solved by: (1) identifying a martingale related to the process, (2) defining the right stopping time, and (3) applying OST to get E[Xτ] = E[X0], then solving for the unknown.

Classic Application: Expected steps for random walk to reach a boundary

A random walk starts at position a (0 < a < b). At each step, it moves +1 or −1 with equal probability. What is the expected number of steps to reach 0 or b?

Use the martingale Mn = Sn2 − n. Let τ be the hitting time of {0, b}. By OST: E[Mτ] = M0 = a2.

At time τ, Sτ is either 0 or b. Let p = P(Sτ = b). From the simple random walk martingale: E[Sτ] = a, so p · b + (1−p) · 0 = a, giving p = a/b.

Now: E[Mτ] = E[Sτ2] − E[τ] = a2. And E[Sτ2] = (a/b) · b2 + (1 − a/b) · 0 = ab.

Therefore: E[τ] = ab − a2 = a(b − a).

This elegant result—the expected hitting time is a(b − a)—is derived in just a few lines using martingale theory, compared to the much longer approach of solving recurrence relations.

Wald’s Equation

Wald’s equation (also called Wald’s identity) is a fundamental result that connects the expected value of a random sum to the expected number of terms:

If X1, X2, … are i.i.d. random variables with E[|Xi|] < ∞, and N is a stopping time with E[N] < ∞, then:

E[X1 + X2 + … + XN] = E[N] · E[X1]

This is remarkably useful: it says that even though N is random and depends on the Xi’s, the expected total is simply the product of the expected count and the expected step size.

Proof sketch: Define Mn = Sn − n · E[X1] (a martingale). By OST with stopping time N: E[MN] = 0, giving E[SN] = E[N] · E[X1].

Warning: Wald’s equation requires E[N] < ∞. If the stopping time has infinite expectation, the result can fail. For example, the doubling strategy in gambling (double your bet after each loss) has E[N] = ∞ and Wald’s equation does not apply.

Application: Expected cost of a randomized algorithm

Suppose each iteration of a randomized algorithm costs Ci time (i.i.d., E[C] = c), and the algorithm runs for a random number N of iterations (a stopping time). By Wald’s equation, the expected total cost is E[N] · c. This lets us analyze randomized algorithms by separately analyzing the expected number of iterations and the expected cost per iteration. 

#include <bits/stdc++.h>
using namespace std;

// Verify Wald's equation: flip coins until 3 heads in a row
// E[total flips] = E[N] * E[cost per flip] = E[N] * 1
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int trials = 1000000;
    long long total_flips = 0;

    for (int t = 0; t < trials; t++) {
        int consec = 0, flips = 0;
        while (consec < 3) {
            flips++;
            consec = (rng() % 2 == 0) ? consec + 1 : 0;
        }
        total_flips += flips;
    }

    cout << "Empirical E[N]: " << fixed << setprecision(2)
         << (double)total_flips / trials << "\n";
    cout << "Theoretical E[N]: 14.00\n"; // 2^4 - 2
    return 0;
}

Gambler’s Ruin via Martingales

The Gambler’s Ruin problem is a classic in probability: a gambler starts with $a and plays a fair game, winning or losing $1 each round. The game ends when the gambler reaches $0 (ruin) or $b (target). What is the probability of ruin?

The traditional approach solves a system of linear recurrence equations. The martingale approach is far more elegant and generalizes easily to biased games.

Fair Game (p = 1/2)

The wealth process Sn is a martingale (fair random walk). Let τ = min{n : Sn ∈ {0, b}}. By OST: E[Sτ] = S0 = a.

Let q = P(ruin) = P(Sτ = 0). Then E[Sτ] = q · 0 + (1 − q) · b = a. Solving: q = 1 − a/b. Probability of reaching b is a/b.

Biased Game (p ≠ 1/2)

When P(win) = p ≠ 1/2, the wealth is not a martingale. But Mn = ((1−p)/p)Sn is a martingale (the exponential martingale). Let r = (1−p)/p.

E[Mn+1 | Fn] = rSn · (p · r + (1−p) · r−1) = rSn · (p · (1−p)/p + (1−p) · p/(1−p)) = rSn · ((1−p) + p) = Mn ✓

By OST: E[Mτ] = M0 = ra. So q · r0 + (1−q) · rb = ra. Solving: q = (ra − rb) / (1 − rb).

Elegance of Martingales: The traditional recurrence approach requires solving f(a) = p · f(a+1) + (1−p) · f(a−1) with boundary conditions. The martingale approach gets the same answer in 3 lines by choosing the right martingale and applying OST. This is the power of the method. 
#include <bits/stdc++.h>
using namespace std;

// Simulate Gambler's Ruin and compare with martingale formula
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int trials = 500000;

    int a = 30;   // starting wealth
    int b = 100;  // target wealth
    double p = 0.48; // probability of winning each round

    // Simulation
    int ruin_count = 0;
    long long total_steps = 0;

    for (int t = 0; t < trials; t++) {
        int wealth = a;
        int steps = 0;
        while (wealth > 0 && wealth < b) {
            double roll = uniform_real_distribution<double>(0.0, 1.0)(rng);
            wealth += (roll < p) ? 1 : -1;
            steps++;
        }
        if (wealth == 0) ruin_count++;
        total_steps += steps;
    }

    double sim_ruin = (double)ruin_count / trials;
    double sim_steps = (double)total_steps / trials;

    // Martingale formula for biased game
    double r = (1.0 - p) / p;
    double ra = pow(r, a), rb = pow(r, b);
    double theory_ruin = (ra - rb) / (1.0 - rb);

    // For fair game (p = 0.5): ruin prob = 1 - a/b
    double fair_ruin = 1.0 - (double)a / b;

    cout << "Gambler's Ruin: a=" << a << ", b=" << b
         << ", p=" << p << "\n";
    cout << "Simulated ruin probability: " << fixed << setprecision(4)
         << sim_ruin << "\n";
    cout << "Martingale formula:         " << theory_ruin << "\n";
    cout << "Fair game formula (p=0.5):  " << fair_ruin << "\n";
    cout << "Average steps to absorb:    " << setprecision(1)
         << sim_steps << "\n";
    return 0;
}

The Ballot Problem

The ballot problem is another classic that connects beautifully to martingale theory. The problem: candidate A receives ‘a’ votes and candidate B receives ‘b’ votes, with a > b. If votes are counted in random order, what is the probability that A is strictly ahead throughout the entire counting?

The answer is the elegant ballot theorem:

P(A strictly ahead throughout) = (a − b) / (a + b)

Connection to Martingales

Define the vote-counting process: after counting i votes, let Si = (votes for A so far) − (votes for B so far). This is a random walk on the set of vote orderings.

The ballot problem asks for P(Si > 0 for all i = 1, …, a + b). While Si itself is not a martingale (votes are drawn without replacement), we can use the cycle lemma, which has a martingale-flavored proof.

Cycle Lemma argument: Consider the a + b votes arranged in a circle. There are exactly a − b cyclic shifts where A is strictly ahead at every point. Since all (a + b) shifts are equally likely, the probability is (a − b)/(a + b).

Generalization (Catalan Numbers): When a = b (tied election), the number of monotone paths that never go below zero is the Catalan number Cn = C(2n, n)/(n + 1). This connects to the ballot problem via the reflection principle. 
#include <bits/stdc++.h>
using namespace std;

// Verify ballot theorem by simulation
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int trials = 1000000;

    int a = 7, b = 3; // a votes for A, b votes for B, a > b

    int success = 0;
    for (int t = 0; t < trials; t++) {
        // Create vote sequence: a ones and b negative ones
        vector<int> votes(a + b);
        fill(votes.begin(), votes.begin() + a, 1);
        fill(votes.begin() + a, votes.end(), -1);
        shuffle(votes.begin(), votes.end(), rng);

        // Check if A is strictly ahead throughout
        int prefix = 0;
        bool always_ahead = true;
        for (int v : votes) {
            prefix += v;
            if (prefix <= 0) {
                always_ahead = false;
                break;
            }
        }
        if (always_ahead) success++;
    }

    double sim_prob = (double)success / trials;
    double theory_prob = (double)(a - b) / (a + b);

    cout << "Ballot problem: a=" << a << ", b=" << b << "\n";
    cout << "Simulated: " << fixed << setprecision(4) << sim_prob << "\n";
    cout << "Theorem:   " << theory_prob << "\n";
    return 0;
}

CP Applications

Martingale theory provides elegant solutions and analysis tools for several competitive programming problem types:

1. Random Walk Expected Hitting Times

As shown earlier, the martingale Mn = Sn2 − n gives us E[τ] = a(b − a) for the time to hit {0, b} starting from a. More generally, for any random walk problem asking “expected time to reach some state,” look for a martingale and apply OST.

2. Expected Value of Random Processes

Many CP problems define a random process and ask for the expected value at termination. If you can identify a martingale (or transform the process into one), OST immediately gives you the answer. Common patterns: wealth in gambling games, score in random contests, position in random walks.

3. Probability of Reaching a Target

The Gambler’s Ruin analysis generalizes: whenever you have a random process that stops at one of several absorbing states, a martingale + OST gives the absorption probabilities. This is cleaner than setting up and solving recurrence equations, especially for biased processes.

4. Analyzing Randomized Algorithms

Define a potential function Φ(state) such that {Φ(staten) − cn} is a supermartingale. Then OST gives E[τ] ≤ Φ(initial) / c, bounding expected runtime.

5. Concentration Inequalities

Azuma’s inequality for martingales with bounded differences |Xn+1 − Xn| ≤ cn: P(|Xn − X0| ≥ t) ≤ 2 exp(−t2 / (2 ∑ ci2)). This proves randomized algorithms work “with high probability.” 

#include <bits/stdc++.h>
using namespace std;

// Verify E[tau] = a*(b-a) for random walk from a to {0,b}
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int trials = 500000;
    int a = 20, b = 50;

    long long total_steps = 0;
    for (int t = 0; t < trials; t++) {
        int pos = a;
        int steps = 0;
        while (pos > 0 && pos < b) {
            pos += (rng() % 2 == 0) ? 1 : -1;
            steps++;
        }
        total_steps += steps;
    }

    cout << "Simulated E[tau]: " << fixed << setprecision(1)
         << (double)total_steps / trials << "\n";
    cout << "Martingale E[tau] = a*(b-a) = " << (double)a * (b - a) << "\n";
    return 0;
}

Practice Problems

Here are problems to practice martingale theory and related techniques in competitive programming:

  1. Codeforces 850F – Arpa and a game with Mojtaba
    Game theory problem with random elements. Martingale analysis helps bound expected outcomes.
  2. Codeforces 1025G – Company Acquisitions
    Define a potential function that forms a martingale. The answer follows directly from the invariant property. A beautiful problem with an elegant martingale-based solution.
  3. Codeforces 1392H – ZS Shuffles Cards
    Random shuffling process with a stopping condition. Use linearity of expectation combined with martingale-style arguments to compute the expected value.
  4. AtCoder ABC276 G – Count Sequences
    Counting sequences with properties related to random walks and ballot-type conditions. The reflection principle (ballot problem connection) is key.
  5. Codeforces 1540B – Tree Array
    Expected value problem on trees involving a random process. Define the right martingale for the BFS/DFS process and apply optional stopping.
  6. USACO 2020 February Platinum – Help Yourself
    Expected value of a random process over intervals. Martingale-style decomposition of the expected value leads to an efficient solution.
  7. Codeforces 1479E – School Clubs
    Advanced expected value problem requiring careful probabilistic analysis. The exponential generating function approach has connections to exponential martingales.
  8. Project Euler 232 – The Race
    Optimal stopping problem that can be analyzed via martingale theory and dynamic programming. A good bridge between theory and implementation.
Study Strategy: Start with Codeforces 1025G (Company Acquisitions)—it is the cleanest example of a martingale invariant in competitive programming. Then work through the random walk problems to build intuition for OST. The ballot problem exercises will strengthen your combinatorial intuition for path-counting arguments.
Exam Tip: In contest settings, you rarely need to formally prove a martingale. Instead, look for invariants—quantities whose expected value does not change over time. If you find such an invariant, you have found a martingale, and OST gives you the answer at the stopping time for free.
