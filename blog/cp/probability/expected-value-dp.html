Expected Value via DP

August 2026 · 25 min read

DP Setup for Expected Value

Expected value is one of the most powerful concepts in competitive programming, and dynamic programming gives us a systematic way to compute it. The core idea is deceptively simple: decompose the expected value into states, and express the expected value of each state in terms of its successors.

Recall the law of total expectation: for any partition of the sample space into events B1, B2, …, Bn:

E[X] = Σi E[X | Bi] · P(Bi)

In a DP context, each Bi corresponds to a state. We define dp[s] = expected value (or expected cost) starting from state s. Then each transition from state s to state s′ with probability p gives us:

dp[s] = cost(s) + Σs′ P(s → s′) · dp[s′]

Key Insight: Unlike traditional DP where we take min or max, expected value DP takes a weighted sum over all successor states. Every possible transition contributes to the answer, weighted by its probability.

The general recipe: (1) Define states — position, count, bitmask, etc. (2) Identify terminal states with known values (often 0). (3) Write the recurrence for each non-terminal state. (4) Solve — topological order for DAGs, system of equations for cycles.

Warning: Many expected value problems have cyclic state transitions — you can return to a state you have already visited. Standard memoization will not work! You must solve a system of equations or reformulate to eliminate cycles.

Simple EV DP: Coin Flips

Let us start with the classic warmup: what is the expected number of fair coin flips to get heads?

Define E = expected number of flips. On each flip, with probability 1/2 we get heads (done, used 1 flip) and with probability 1/2 we get tails (used 1 flip, back to the same state):

E = 1 + (1/2) · 0 + (1/2) · E

Solving: E − E/2 = 1, so E = 2. This self-referential equation is the hallmark of expected value problems with cycles.

Coupon Collector Problem

A more interesting example: there are n distinct coupons. Each day you receive one uniformly at random. What is the expected number of days to collect all n coupons?

Define dp[i] = expected additional days to go from having i distinct coupons to having all n. Base case: dp[n] = 0. When you have i coupons:

  • Probability of getting a new coupon: (n − i) / n
  • Probability of a duplicate: i / n

dp[i] = 1 + (i/n) · dp[i] + ((n−i)/n) · dp[i+1]

Rearranging: dp[i] − (i/n) · dp[i] = 1 + ((n−i)/n) · dp[i+1], so dp[i] · (n−i)/n = 1 + ((n−i)/n) · dp[i+1], giving us:

dp[i] = n/(n−i) + dp[i+1]

Therefore E[total] = dp[0] = Σi=0n−1 n/(n−i) = n · Hn, where Hn is the n-th harmonic number. For n = 100, this is approximately 518.7 days.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // dp[i] = expected days to collect all n, having i so far
    vector<double> dp(n + 1, 0.0);
    for (int i = n - 1; i >= 0; i--) {
        dp[i] = (double)n / (n - i) + dp[i + 1];
    }

    cout << fixed << setprecision(9) << dp[0] << "\n";
    return 0;
}

Absorbing Markov Chains via DP

An absorbing Markov chain is a Markov chain in which every state can eventually reach an absorbing state — a state that, once entered, is never left. Many expected value problems in CP are secretly absorbing Markov chains.

The fundamental question is: starting from state i, what is the expected number of steps to reach an absorbing state?

For each non-absorbing (transient) state i, we write:

E[i] = 1 + Σj P(i → j) · E[j]

where E[j] = 0 for all absorbing states j. This gives us a system of linear equations in the unknowns E[i] for transient states.

Example: Random Walk on a Line

Consider a random walk on positions 0, 1, 2, …, n. At each step, you move left with probability p and right with probability q = 1 − p. States 0 and n are absorbing. Starting from position k, what is the expected number of steps to be absorbed?

For 1 ≤ k ≤ n−1:

E[k] = 1 + p · E[k−1] + q · E[k+1]

with E[0] = E[n] = 0. For the symmetric case p = q = 1/2, the solution is E[k] = k(n − k). This elegant closed form emerges from solving the recurrence directly.

Pattern Recognition: Whenever a problem says “at each step, randomly choose one of several actions” and asks for the expected number of steps to reach a goal, you are looking at an absorbing Markov chain. Write the E[i] = 1 + Σ pij E[j] recurrence and solve.

Expected Steps to Absorbing State — Implementation

When the state graph is a DAG (no cycles among transient states), we can solve with simple DP in reverse topological order. Here is a template for a grid-based absorbing chain:

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    // Grid with absorbing cells marked 'A', start at (0,0)
    vector<string> grid(n);
    for (auto &row : grid) cin >> row;

    // dp[i][j] = expected steps from (i,j) to any absorbing cell
    vector<vector<double>> dp(n, vector<double>(m, 0.0));
    vector<vector<bool>> vis(n, vector<bool>(m, false));

    // BFS/topological order from absorbing states backward
    // For DAG structure: process states in reverse BFS order
    queue<pair<int,int>> q;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == 'A') {
                dp[i][j] = 0.0;
                vis[i][j] = true;
                q.push({i, j});
            }

    int dx[] = {0, 0, 1, -1};
    int dy[] = {1, -1, 0, 0};

    // For a DAG, process in reverse topological order
    // For general case, need Gaussian elimination (see next section)
    // This simplified version assumes movement only toward absorbing states
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        // Propagate to predecessors...
    }

    cout << fixed << setprecision(9) << dp[0][0] << "\n";
    return 0;
}

System of Equations for E[X]

When the Markov chain has cycles among transient states, we cannot simply compute dp values in a fixed order. Instead, we must set up and solve a system of linear equations.

Consider a general Markov chain with states 0, 1, …, n where state n is absorbing. For each transient state i (0 ≤ i < n):

E[i] = 1 + Σj=0n P(i → j) · E[j]

Since E[n] = 0, we rearrange each equation to standard form:

E[i] − Σj=0n−1 P(i → j) · E[j] = 1

This is a system of n equations in n unknowns. In matrix form: (I − Q) · e = 1, where Q is the sub-matrix of transition probabilities among transient states, I is the identity matrix, and 1 is the all-ones vector.

Dice Game Example

You roll a fair six-sided die repeatedly. You keep a running sum. If the sum reaches exactly n, you win. If it exceeds n, you lose and restart from 0. What is the expected number of rolls?

States: 0, 1, 2, …, n (sum so far). State n is absorbing (win). For state i < n:

E[i] = 1 + (1/6) · Σk=16 E[(i+k) if i+k ≤ n, else 0]

Notice the cycle: overshooting sends you back to state 0. We need to solve the resulting system of equations.

Common Pitfall: When setting up the system, make sure to correctly handle self-loops. If state i transitions to itself with probability p, the equation becomes E[i] = 1 + p · E[i] + …, which you can simplify to (1−p) · E[i] = 1 + … before adding to the matrix. This avoids numerical issues and reduces the system size.

Building the Linear System

Here is the pattern for constructing the augmented matrix [A | b] from the recurrence E[i] = 1 + Σ pij E[j]:

#include <bits/stdc++.h>
using namespace std;

// Build augmented matrix for E[i] = 1 + sum(p[i][j] * E[j])
// n = number of transient states
// p[i][j] = transition probability from i to j (among transient states)
void buildSystem(int n, vector<vector<double>> &p,
                 vector<vector<double>> &aug) {
    aug.assign(n, vector<double>(n + 1, 0.0));
    for (int i = 0; i < n; i++) {
        aug[i][i] = 1.0;  // coefficient of E[i]
        for (int j = 0; j < n; j++) {
            aug[i][j] -= p[i][j];  // subtract p[i][j] * E[j]
        }
        aug[i][n] = 1.0;  // right-hand side = 1 (one step cost)
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<vector<double>> p(n, vector<double>(n, 0.0));
    // ... fill transition probabilities ...

    vector<vector<double>> aug;
    buildSystem(n, p, aug);

    // Solve with Gaussian elimination (next section)
    return 0;
}

Matrix & Gaussian Elimination

Gaussian elimination is the workhorse for solving expected value systems in competitive programming. Given the augmented matrix [A | b], we reduce it to row echelon form and back-substitute.

The algorithm runs in O(n3) time, which is fast enough for n up to about 500–1000 in most CP time limits. For larger systems, we need special structure (banded matrices, sparse methods, or matrix exponentiation).

Gaussian Elimination Implementation

#include <bits/stdc++.h>
using namespace std;

const double EPS = 1e-9;

// Solves Ax = b given augmented matrix [A|b] of size n x (n+1)
// Returns solution vector x, or empty vector if no unique solution
vector<double> gaussianElimination(vector<vector<double>> aug) {
    int n = aug.size();

    for (int col = 0; col < n; col++) {
        // Partial pivoting: find row with largest absolute value
        int pivot = col;
        for (int row = col + 1; row < n; row++) {
            if (abs(aug[row][col]) > abs(aug[pivot][col]))
                pivot = row;
        }
        swap(aug[col], aug[pivot]);

        if (abs(aug[col][col]) < EPS) return {};  // singular

        // Eliminate below
        for (int row = col + 1; row < n; row++) {
            double factor = aug[row][col] / aug[col][col];
            for (int j = col; j <= n; j++) {
                aug[row][j] -= factor * aug[col][j];
            }
        }
    }

    // Back substitution
    vector<double> x(n);
    for (int i = n - 1; i >= 0; i--) {
        x[i] = aug[i][n];
        for (int j = i + 1; j < n; j++) {
            x[i] -= aug[i][j] * x[j];
        }
        x[i] /= aug[i][i];
    }
    return x;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // Example: dice game with target sum n
    int n;
    cin >> n;

    // States: 0..n-1 are transient, state n is absorbing
    vector<vector<double>> aug(n, vector<double>(n + 1, 0.0));

    for (int i = 0; i < n; i++) {
        aug[i][i] = 1.0;
        for (int k = 1; k <= 6; k++) {
            int nxt = i + k;
            if (nxt == n) {
                // Absorbing state, E[n] = 0, nothing to subtract
            } else if (nxt > n) {
                aug[i][0] -= 1.0 / 6.0;  // restart at state 0
            } else {
                aug[i][nxt] -= 1.0 / 6.0;
            }
        }
        aug[i][n] = 1.0;  // RHS
    }

    auto sol = gaussianElimination(aug);
    cout << fixed << setprecision(9) << sol[0] << "\n";
    return 0;
}

Matrix Exponentiation for Expected Values

When the number of steps is fixed, use matrix exponentiation. The state distribution after k steps is vk = Tk · v0, computable in O(n3 log k). Essential when k ≤ 1018 but the state space is small (n ≤ 100).

When to use which method:
• Acyclic DP: state graph is a DAG — solve in topological order, O(states).
• Gaussian elimination: cyclic transitions, need expected time to absorption — O(n3).
• Matrix exponentiation: need distribution or expected value after exactly k steps — O(n3 log k).

Expected Value DP on Trees

Trees offer a natural DAG structure (rooted tree, parent → children), making expected value DP particularly clean. A common pattern is computing the expected value of some property over all paths or subtrees.

Expected Depth of a Random Walk on a Tree

Consider a random walk starting at the root of a tree. At each step, you move to a uniformly random neighbor. What is the expected number of steps to reach a leaf?

For a leaf v: E[v] = 0. For an internal node v with degree d(v) (including parent):

E[v] = 1 + (1/d(v)) · E[parent(v)] + (1/d(v)) · Σc ∈ children(v) E[c]

The parent term introduces a “backward” dependency, requiring us to solve a system of equations along each root-to-leaf path. However, on a tree this system is tridiagonal and can be solved in O(n) time.

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // dp[v] = expected value for subtree rooted at v
    // For problems without backward edges, simple DFS suffices
    vector<double> dp(n, 0.0);
    vector<bool> visited(n, false);

    // DFS computing expected subtree values
    function<void(int, int)> dfs = [&](int u, int par) {
        visited[u] = true;
        int childCount = 0;
        double childSum = 0.0;
        for (int v : adj[u]) {
            if (v == par) continue;
            dfs(v, u);
            childSum += dp[v];
            childCount++;
        }
        if (childCount == 0) {
            dp[u] = 0.0;  // leaf
        } else {
            // Each child equally likely to be chosen
            dp[u] = 1.0 + childSum / childCount;
        }
    };

    dfs(0, -1);
    cout << fixed << setprecision(9) << dp[0] << "\n";
    return 0;
}
Tree DP Trick: When the random walk can go back to the parent (creating cycles), use the “contribution” technique: express E[v] = av · E[parent(v)] + bv in a single DFS pass. Each node computes its (a, b) coefficients from its children’s coefficients, then the root (with no parent dependency) gives E[root] = broot / (1 − aroot).

CP Applications

Expected value DP appears in many competition problems. Here are the most common patterns to recognize.

Pattern 1: Bitmask DP for Expected Collecting

Given n items (n ≤ 20), each step gives you a random item. Expected steps to collect all? Use dp[mask] where mask tracks collected items:

dp[mask] = 1 + (popcount(mask)/n) · dp[mask] + Σi ∉ mask (pi) · dp[mask | (1<<i)]

Rearranging to eliminate the self-loop: dp[mask] = (n / (n − popcount(mask))) + Σ …

Pattern 2: Grid Random Walk

Robot starts at (0,0) on an n×m grid, moves to random adjacent cell each step. Expected steps to reach (n−1, m−1)? Set up E[i][j] equations and solve with Gaussian elimination. State space is n·m, so Gaussian elimination is O(n3m3).

Pattern 3: Expected Value with Modular Arithmetic

Some problems ask for expected values modulo a prime p. Since E[X] is rational, express it as a/b and compute a · b−1 mod p using modular inverse. This requires adapting Gaussian elimination to work in modular arithmetic — replace division by multiplication with modular inverse.

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Gaussian elimination in Z_p
vector<long long> gaussMod(vector<vector<long long>> aug) {
    int n = aug.size();
    for (int col = 0; col < n; col++) {
        int pivot = -1;
        for (int row = col; row < n; row++) {
            if (aug[row][col] != 0) { pivot = row; break; }
        }
        if (pivot == -1) return {};
        swap(aug[col], aug[pivot]);
        long long iv = inv(aug[col][col]);
        for (int j = col; j <= n; j++)
            aug[col][j] = aug[col][j] % MOD * iv % MOD;
        for (int row = 0; row < n; row++) {
            if (row == col || aug[row][col] == 0) continue;
            long long factor = aug[row][col];
            for (int j = col; j <= n; j++)
                aug[row][j] = (aug[row][j] - factor % MOD * aug[col][j] % MOD + MOD * 2) % MOD;
        }
    }
    vector<long long> x(n);
    for (int i = 0; i < n; i++) x[i] = aug[i][n];
    return x;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    // ... set up system modulo MOD and solve ...
    return 0;
}

Practice Problems

Here is a curated set of problems ordered by difficulty to solidify your understanding of expected value DP techniques.

Warmup

  • Codeforces 518D – Ilya and Escalator: Direct expected value DP on a queue. States represent number of people on the escalator.
  • Codeforces 235B – Let's Play Osu!: Expected value with linearity of expectation. Define the contribution of each position independently.

Intermediate

  • Codeforces 1392D – Omkar and Bed Wars: Expected value DP with bitmask states and cyclic transitions requiring equation solving.
  • Codeforces 1265E – Beautiful Mirrors: Expected value with a restart mechanism. Classic system-of-equations setup with back-edges to state 0.
  • AtCoder ABC263E – Sugoroku 3: Expected steps on a number line with random jumps. Direct application of the rearrangement trick to eliminate self-loops.

Advanced

  • Codeforces 123D – String: Expected value combined with suffix structures. Requires careful state definition and Gaussian elimination.
  • Codeforces 605E – Intergalaxy Trips: Expected value DP where edges appear with probability. Greedy ordering by expected value enables efficient computation.
  • AtCoder ARC126E – Ascending Slope: Matrix exponentiation for expected value after many steps in a small-state Markov chain.
Study Strategy: Start with the warmup problems to get comfortable with the recurrence pattern. Then tackle intermediate problems where you must handle cycles (system of equations). Finally, attempt advanced problems combining EV DP with other techniques (bitmasks, trees, matrix exponentiation).
