Random Walks & Gambler's Ruin

July 2026 · 22 min read

What Is a Random Walk?

A random walk is one of the most fundamental objects in probability theory. Imagine you are standing on the number line at position 0. At every discrete time step you flip a (possibly biased) coin and move one unit to the right or one unit to the left. The sequence of positions you visit is a random walk.

Formally, let X1, X2, … be independent, identically distributed random variables taking values +1 (with probability p) and −1 (with probability q = 1 − p). The random walk is the sequence of partial sums:

S0 = 0,   Sn = X1 + X2 + … + Xn.

When p = 1/2 the walk is called simple symmetric; otherwise it is asymmetric (or biased). Despite its simplicity, the random walk underpins Brownian motion, stock-price models, diffusion, and many CP problems involving grid movement or stochastic games.

Key intuition: A symmetric random walk in 1-D will visit every integer infinitely often with probability 1, yet the expected time to reach any particular value is infinite. This counterintuitive fact lies at the heart of many tricky problems.

The 1-D Random Walk

Let us pin down the basic properties of the one-dimensional walk with step probabilities p (right) and q = 1 − p (left).

Expected Position & Variance

Because E[Xi] = p − q = 2p − 1, we have:

E[Sn] = n(2p − 1),    Var(Sn) = 4npq.

For the symmetric case (p = 1/2) the expected position stays at 0 and the standard deviation grows as √n. After n = 10 000 steps the walk is typically within ±100 of the origin — far smaller than n itself.

The Reflection Principle

A powerful combinatorial trick: the number of walks from (0, 0) to (n, k) that touch or cross the x-axis at some point can be counted by reflecting the portion of the walk before the first touch. This gives the classic ballot problem result and is used to count lattice paths that stay strictly positive.

Specifically, the number of paths from (0, a) to (n, b) that touch 0 equals the total number of paths from (0, −a) to (n, b). Using binomial coefficients, this lets us derive closed-form hitting probabilities.

CP Trap: Many grid-walk counting problems silently rely on the reflection principle. If a problem says “count paths that never go below y = 0”, think reflection + Catalan numbers.

Simulation in C++

A quick Monte-Carlo simulation to estimate the probability that a symmetric random walk starting at 0 is positive after n steps: 

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(42);
    int n, trials;
    cin >> n >> trials;

    int pos_count = 0;
    for(int t = 0; t < trials; t++){
        int pos = 0;
        for(int i = 0; i < n; i++){
            pos += (rng() % 2) ? 1 : -1;
        }
        if(pos > 0) pos_count++;
    }
    cout << fixed << setprecision(6)
         << (double)pos_count / trials << "\n";
    return 0;
}

Return to the Origin

One of the most celebrated results in probability is Pólya’s recurrence theorem. It answers: “If a random walk starts at the origin, will it ever return?”

Dimension 1 (d = 1)

For the simple symmetric walk on ℤ the return probability is 1. In fact, the walk returns to the origin infinitely often with probability 1. We say the walk is recurrent.

The probability of being at the origin after 2n steps is:

P(S2n = 0) = C(2n, n) / 4n ≈ 1 / √(πn).

Because Σn 1/√n diverges, the walk returns infinitely often (by the second Borel–Cantelli lemma argument combined with the renewal structure).

Dimension 2 (d = 2)

Also recurrent! A 2-D symmetric walk on the integer lattice returns to the origin with probability 1. The return probability at step 2n is proportional to 1/n, and Σ 1/n diverges.

Dimension 3 and above (d ≥ 3)

The walk is transient: it escapes to infinity. For d = 3, the return probability is approximately 0.3405. As Shizuo Kakutani famously said: “A drunk man will find his way home, but a drunk bird may get lost forever.”

CP relevance: On a 1-D or 2-D grid with absorbing barriers, the walk will always eventually be absorbed. This guarantees termination of many stochastic processes you encounter in competitive programming.

Gambler's Ruin Problem

The Gambler’s Ruin is the oldest and most important random-walk problem. A gambler starts with i dollars and plays a series of fair (or biased) bets against an infinitely wealthy casino (or another player with N − i dollars).

At each round, the gambler wins $1 with probability p and loses $1 with probability q = 1 − p. The game ends when the gambler’s fortune reaches 0 (ruin) or N (victory).

Define P(i) = probability of reaching N starting from fortune i. We want to compute P(i) for each 0 ≤ i ≤ N.

Setting Up the Recurrence

By the law of total probability, conditioning on the first step:

P(i) = p · P(i + 1) + q · P(i − 1),   1 ≤ i ≤ N − 1.

Boundary conditions: P(0) = 0 (ruin), P(N) = 1 (victory).

This is a second-order linear recurrence with constant coefficients — exactly the kind of equation we know how to solve.

Deriving the Ruin Probability

Step 1: Rewrite the Recurrence

Since p + q = 1, rearrange:

p · P(i+1) − P(i) + q · P(i−1) = 0.

Equivalently: p[P(i+1) − P(i)] = q[P(i) − P(i−1)].

Let D(i) = P(i+1) − P(i). Then D(i) = (q/p) · D(i−1). This is a geometric sequence with ratio r = q/p.

Step 2: Telescoping

D(i) = (q/p)i · D(0). Summing from 0 to N−1:

P(N) − P(0) = D(0) · Σi=0N−1 (q/p)i.

Using P(0) = 0 and P(N) = 1:

1 = D(0) · Σi=0N−1 (q/p)i.

Step 3: Case Analysis

Case 1: p ≠ q (biased walk).

The geometric sum equals (1 − (q/p)N) / (1 − q/p). Solving for D(0), then summing the first i terms of the geometric series gives:

P(i) = (1 − (q/p)i) / (1 − (q/p)N)

Case 2: p = q = 1/2 (fair game).

The ratio q/p = 1, so the geometric series reduces to N. D(0) = 1/N and:

P(i) = i / N

Takeaway: In a fair game starting with i out of N total dollars, your probability of winning is exactly i/N. If you have $20 and the casino has $980 (N = 1000), you win with probability 0.02. Even a fair game is brutal for the smaller player.

C++ — Computing Gambler's Ruin Probabilities

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, start;
    double p;
    cin >> N >> start >> p;
    double q = 1.0 - p;

    if(abs(p - 0.5) < 1e-9){
        // Fair game
        cout << fixed << setprecision(9)
             << (double)start / N << "\n";
    } else {
        double r = q / p;
        double ri = pow(r, start);
        double rN = pow(r, N);
        double prob = (1.0 - ri) / (1.0 - rN);
        cout << fixed << setprecision(9)
             << prob << "\n";
    }
    return 0;
}
Numerical warning: When q/p is close to 1, direct computation of (q/p)N can overflow or lose precision for large N. Use logarithms or handle the fair-game case separately (as above).

Expected Duration of the Game

How long does a gambler’s ruin game last on average? Define E(i) = expected number of steps until the game ends, starting from fortune i.

The Recurrence

E(i) = 1 + p · E(i+1) + q · E(i−1),   E(0) = E(N) = 0.

The “1 +” accounts for the step just taken. This is a non-homogeneous version of the ruin recurrence.

Solution: Fair Game (p = 1/2)

E(i) = i · (N − i)

Remarkably elegant! Starting from the middle (i = N/2), the expected duration is N2/4, which is Θ(N2).

Solution: Biased Game (p ≠ 1/2)

Let r = q/p. Then:

E(i) = [i / (q − p)] − [N / (q − p)] · [(1 − ri) / (1 − rN)]

When the game is biased against you (p < 1/2), the expected duration is actually shorter than the fair game — you lose faster.

C++ — Expected Duration Computation

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, start;
    double p;
    cin >> N >> start >> p;
    double q = 1.0 - p;

    if(abs(p - 0.5) < 1e-9){
        // Fair game: E(i) = i * (N - i)
        cout << fixed << setprecision(4)
             << (double)start * (N - start) << "\n";
    } else {
        double r = q / p;
        double ri = pow(r, start);
        double rN = pow(r, N);
        double coeff = 1.0 / (q - p);
        double E = coeff * start
                 - coeff * N * (1.0 - ri) / (1.0 - rN);
        cout << fixed << setprecision(4)
             << E << "\n";
    }
    return 0;
}

CP Applications

1. Grid Random Walks with Absorbing Barriers

Many CP problems place a token on a 1-D line segment [0, N] and ask for the probability of reaching the left or right boundary. This is a direct gambler’s ruin instance. Map states to positions and check whether the transition probabilities are uniform or position-dependent.

2. Expected Steps to Reach a Target

Problems like “expected number of coin flips to get k consecutive heads” can be modelled as random walks on a state graph with absorbing barriers. Set up the expected-value recurrence exactly as in the gambler’s ruin duration formula.

3. 2-D Grid Walks

A robot moves on a grid and needs to reach a target cell. If the movement is random, you can decompose the 2-D walk into two independent 1-D walks (one per axis) when the axes are decoupled. The total probability/expected time is the product/max of the individual components.

Technique: When a random walk lives on a graph (not just a line), convert to a system of linear equations. Each node gives one equation of the form P(v) = Σu p(v,u) · P(u) (for absorbing-probability problems) or E(v) = 1 + Σu p(v,u) · E(u) (for expected-time problems). Solve with Gaussian elimination in O(n3).

4. Random Walks on Trees

A particle starts at the root of a tree and at each step moves to a uniformly random neighbour. The expected time to reach a leaf (or any target node) can be set up as a system of linear equations, one per node. For trees this system is tridiagonal and solvable in O(n).

Monte-Carlo Verification Template

When in doubt, verify your analytic formula with a simulation: 

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937 rng(12345);
    int N = 10, start = 3, trials = 1000000;
    double p = 0.4;

    int wins = 0;
    long long total_steps = 0;
    for(int t = 0; t < trials; t++){
        int pos = start;
        int steps = 0;
        while(pos > 0 && pos < N){
            double u = uniform_real_distribution<double>(0,1)(rng);
            pos += (u < p) ? 1 : -1;
            steps++;
        }
        if(pos == N) wins++;
        total_steps += steps;
    }
    cout << "Win prob: " << (double)wins / trials << "\n";
    cout << "Avg steps: " << (double)total_steps / trials << "\n";
    return 0;
}

Practice Problems

Test your understanding of random walks and gambler’s ruin with these problems, ordered roughly by difficulty:

  1. Drunk Walk (easy): A person starts at position 0 on a line of length N. At each step they move ±1 with equal probability. What is the expected number of steps to reach either boundary? Answer: use E(i) = i(N−i).
  2. Biased Coin Ruin (medium): A gambler has $5 and plays against someone with $15 (N = 20). P(win each round) = 0.45. Compute the probability the gambler wins. Apply the biased gambler’s ruin formula.
  3. Absorbing Random Walk on a Graph (medium): Given a graph with two absorbing nodes, compute the probability of being absorbed at each. Set up the system of linear equations and solve.
  4. 2-D Grid Escape (hard): A token starts at (r, c) on an H×W grid and moves in one of four directions uniformly at random. Compute the expected time to reach any boundary cell.
  5. Random Walk on a Tree (hard): A particle starts at the root of a tree. At each step it moves to a uniform random neighbour. Compute the expected number of steps to reach a designated leaf.
Summary: Random walks and gambler’s ruin give you a powerful toolkit: direct formulas for 1-D problems, the reflection principle for counting, and a systematic approach (linear equations) for arbitrary graphs.
