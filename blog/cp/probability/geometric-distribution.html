Geometric Distribution & Coupon Collector

July 2026 · 22 min read

Geometric Distribution

The geometric distribution models the number of independent Bernoulli trials needed to get the first success. If each trial succeeds with probability p (and fails with probability q = 1 − p), then:

P(X = k) = (1 − p)k−1 · p    for k = 1, 2, 3, …

The intuition is clear: we need k − 1 failures followed by one success.

Verifying it’s a valid PMF:

Σk=1..∞ P(X = k) = p · Σk=0..∞ qk = p · 1/(1 − q) = p · 1/p = 1   ✓

Expected value derivation:

E[X] = Σk=1..∞ k · qk−1 · p
     = p · Σk=1..∞ k · qk−1
     = p · d/dq [Σk=0..∞ qk] evaluated carefully
     = p · 1/(1 − q)²
     = p / p²
     = 1/p
Key Result: If each trial succeeds with probability p, the expected number of trials until the first success is 1/p. Flipping a fair coin? Expect 2 flips for heads. Rolling a 6 on a die? Expect 6 rolls. This is the single most important formula in waiting-time problems.

Variance: Var(X) = (1 − p) / p² = q / p². The variance grows as p shrinks — rare events have highly variable waiting times.

CDF (Cumulative Distribution Function): P(X ≤ k) = 1 − (1 − p)k. This is useful for computing the probability of success within a given number of trials.

Memoryless Property

The geometric distribution is the only discrete distribution with the memoryless property:

P(X > s + t | X > s) = P(X > t)    for all s, t ≥ 0

In words: given that you’ve already waited s trials without success, the conditional distribution of additional waiting time is identical to starting fresh. Past failures give zero information about future success.

Proof:

P(X > k) = (1 − p)k    (all k trials fail)

P(X > s + t | X > s) = P(X > s + t) / P(X > s)
   = (1 − p)s+t / (1 − p)s
   = (1 − p)t
   = P(X > t)    ✓
Gambler’s Fallacy Warning: Many people intuitively feel that after many failures, success must be “due.” The memoryless property says this intuition is mathematically wrong for independent trials. A coin doesn’t remember its past flips. In CP, this means you cannot optimize waiting-time computations by conditioning on past failures — always start fresh from the current state.

Uniqueness: A proof by induction shows that if P(X > s + t) = P(X > s) · P(X > t) for all non-negative integers s, t, then X must be geometric. The continuous analog is the exponential distribution (the only continuous memoryless distribution).

Waiting Time Interpretation

The geometric distribution naturally models waiting times in probabilistic processes. Here are common scenarios in competitive programming:

Scenario 1: Random Walk Absorption

A particle at position 0 moves right with probability p and left with probability 1−p at each step. The time to first reach position +1 is not exactly geometric, but the number of “net right” successes needed can be decomposed using geometric waiting times.

Scenario 2: Collecting a Specific Item

A gacha game drops a specific item with probability 1/100 each pull. The expected pulls to get it: E[X] = 100. The probability of needing more than 200 pulls: P(X > 200) = (99/100)200 ≈ 0.134. About 13.4% of players will need 200+ pulls.

Scenario 3: Randomized Algorithms

Many randomized algorithms repeat a procedure that succeeds with probability p. By the geometric distribution, the expected number of repetitions is 1/p. If p ≥ 1/2, we expect at most 2 repetitions on average — this is why “repeat until success” is practical when success probability is bounded away from zero.

Sum of Geometric Variables

If X1, X2, …, Xr are independent geometric random variables each with parameter p, their sum X1 + … + Xr follows a negative binomial distribution, representing the number of trials needed for r successes. Its expectation is r/p by linearity. 

#include <bits/stdc++.h>
using namespace std;

// Simulate geometric distribution and verify E[X] = 1/p
int main() {
    mt19937 rng(42);
    double p = 0.3;
    int trials = 1000000;
    long long total = 0;

    for (int t = 0; t < trials; t++) {
        int count = 0;
        while (true) {
            count++;
            if ((rng() % 10000) < (int)(p * 10000)) break;
        }
        total += count;
    }

    cout << fixed << setprecision(4);
    cout << "Empirical E[X]: " << (double)total / trials << "\n";
    cout << "Theoretical:    " << 1.0 / p << "\n";
    return 0;
}

Coupon Collector Problem

The coupon collector problem is one of the most important applications of the geometric distribution. It asks: if there are n distinct coupon types and each purchase gives a uniformly random coupon, how many purchases are needed to collect all n types?

This problem appears everywhere: completing a sticker album, hashing into all buckets, random graph connectivity, and many CP problems.

Setup: Let T be the total number of purchases needed. We decompose T into phases. Phase i begins when you have exactly i − 1 distinct coupons and ends when you get a new one (the i-th distinct coupon).

T = T1 + T2 + … + Tn

During phase i, you already have i − 1 coupons, so each purchase yields a new coupon with probability:

pi = (n − (i − 1)) / n = (n − i + 1) / n

Since each purchase is independent and success probability is pi, the waiting time Ti follows a geometric distribution with parameter pi. Note that T1 = 1 always (the first coupon is always new), consistent with p1 = n/n = 1.

Full Proof: E[T] = n · Hn

By linearity of expectation:

E[T] = E[T1] + E[T2] + … + E[Tn]
     = 1/p1 + 1/p2 + … + 1/pn
     = n/n + n/(n−1) + n/(n−2) + … + n/1
     = n · (1/n + 1/(n−1) + … + 1/1)
     = n · (1 + 1/2 + 1/3 + … + 1/n)
     = n · Hn
The Coupon Collector Formula: E[T] = n · Hn ≈ n · ln(n) + γ · n. For n = 100 coupons, you need about 100 · 5.187 ≈ 519 purchases. For n = 1000, about 7485 purchases. The n · ln(n) growth is the key takeaway.

Variance of the coupon collector:

Var(T) = Σi=1..n Var(Ti) = Σi=1..n (1 − pi) / pi²

= n² · Σi=1..n (i − 1) / (n − i + 1)²

≤ n² · Σj=1..n 1/j² ≤ n² · π²/6

So the standard deviation is O(n), meaning the coupon collector time concentrates reasonably well around its mean.

Tail bound: P(T > n · ln(n) + cn) ≤ e−c. This exponential tail bound shows that collecting all coupons is extremely unlikely to take much more than n · ln(n) + O(n) purchases.

Expected Trials & Variants

Variant 1: Generalized Coupon Collector

What if coupons don’t appear with equal probability? Suppose coupon i appears with probability pi (where Σ pi = 1). The analysis becomes harder because phases don’t have clean geometric distributions.

Using inclusion-exclusion, the expected time to collect all n coupons is:

E[T] = ΣS ≠ ∅ (−1)|S|+1 / (1 − Σi∉S pi)
     = ΣS ≠ ∅ (−1)|S|+1 / (Σi∈S pi)

For equal probabilities pi = 1/n, this reduces to our formula since the sum over subsets of size k contributes C(n,k) · (−1)k+1 · n/k.

Variant 2: Collect at Least k Copies

If you need k copies of each coupon type (not just one), the expected time scales as n · Hn + (k−1) · n · ln(ln(n)) + O(n) for large n. The first copy of each coupon dominates the waiting time.

Variant 3: Collect Any m out of n Types

If you only need m distinct types out of n, the expected time is:

E[Tm] = n · (1/(n) + 1/(n−1) + … + 1/(n−m+1))
         = n · (Hn − Hn−m)
Implementation Note: When computing coupon collector answers modulo a prime, you must compute n · Hn mod p. This means computing n · Σi=1..n modinv(i, p) mod p. Pre-computing modular inverses is essential for large n. Use the linear sieve for inverses in O(n) time. 
#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) { if (e & 1) r = r * b % m; b = b * b % m; e >>= 1; }
    return r;
}

// Compute E[coupon collector] = n * H_n mod p
// Using linear sieve for modular inverses
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // Linear sieve for modular inverses: inv[i] = -(p/i) * inv[p%i] mod p
    vector<long long> inv(n + 1);
    inv[1] = 1;
    for (int i = 2; i <= n; i++) {
        inv[i] = (MOD - (MOD / i) * inv[MOD % i] % MOD) % MOD;
    }

    // H_n = sum of inv[1..n]
    long long hn = 0;
    for (int i = 1; i <= n; i++) {
        hn = (hn + inv[i]) % MOD;
    }

    long long ans = (long long)n % MOD * hn % MOD;
    cout << ans << "\n";

    return 0;
}

Simulation for verification: The following code simulates the coupon collector problem and compares with the theoretical formula: 

#include <bits/stdc++.h>
using namespace std;

int main() {
    mt19937 rng(42);
    int n = 50;
    int trials = 200000;
    long long total = 0;

    for (int t = 0; t < trials; t++) {
        vector<bool> seen(n, false);
        int distinct = 0, steps = 0;
        while (distinct < n) {
            int c = rng() % n;
            if (!seen[c]) { seen[c] = true; distinct++; }
            steps++;
        }
        total += steps;
    }

    // Compute H_n
    double hn = 0;
    for (int i = 1; i <= n; i++) hn += 1.0 / i;

    cout << fixed << setprecision(2);
    cout << "Empirical E[T]: " << (double)total / trials << "\n";
    cout << "Theoretical:    " << n * hn << "\n";
    return 0;
}

CP Applications

Geometric distributions and coupon collector arguments appear in many contest problems. Here are the key patterns:

Pattern 1: Expected Steps in Markov Chains

Many CP problems define a random process: from state s, you transition to various states with given probabilities. If from state s the process “advances” with probability ps, the expected time in state s before advancing is 1/ps (geometric waiting). The total expected time is Σ 1/ps over all states.

Pattern 2: Random Operations Until Condition Met

“Each second, a random element is deleted. What is the expected time until the array is sorted?” Decompose into phases based on which “bad” element gets removed. Each phase is geometric with a computable success probability.

Pattern 3: Randomized Algorithm Analysis

Treaps, randomized quicksort, and skip lists all rely on geometric distribution analysis. In a treap, the expected depth of a node is O(log n) because each ancestor’s priority must beat the node’s priority — a sequence of geometric waiting times.

Pattern 4: Balls into Bins

Throwing n balls uniformly into n bins: the expected number of throws to fill every bin is exactly the coupon collector problem. The expected number of empty bins after n throws is n · (1 − 1/n)n ≈ n/e. 

#include <bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

long long power(long long b, long long e, long long m) {
    long long r = 1; b %= m;
    while (e > 0) { if (e & 1) r = r * b % m; b = b * b % m; e >>= 1; }
    return r;
}

long long inv(long long a) { return power(a, MOD - 2, MOD); }

// Problem: n states, from state i you advance with probability p_i = i/n.
// Expected total steps = sum 1/p_i = sum n/i = n * H_n.
// But what if p_i are arbitrary rationals a_i/b_i?
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // Read probabilities as fractions a_i / b_i
    long long ans = 0;
    for (int i = 0; i < n; i++) {
        long long a, b;
        cin >> a >> b;
        // E[time in state i] = b/a mod MOD
        ans = (ans + b % MOD * inv(a) % MOD) % MOD;
    }

    cout << ans << "\n";
    return 0;
}
Modular Harmonic Numbers: Many problems boil down to computing Hn mod p. Pre-compute modular inverses using the linear recurrence: inv[i] = −(p / i) · inv[p mod i] mod p. This runs in O(n) and avoids n calls to fast exponentiation.

Practice Problems

Build your fluency with geometric distributions and coupon collector arguments:

  1. Die Rolling Game: You roll a fair 6-sided die repeatedly. What is the expected number of rolls until you’ve seen all 6 faces? (Answer: 6 · H6 = 6(1 + 1/2 + 1/3 + 1/4 + 1/5 + 1/6) = 14.7)
  2. Collecting Cards: A cereal box contains one of n cards uniformly at random. You need all n. If each box costs $1, what is the expected cost? (Answer: n · Hn dollars)
  3. First Repeated Birthday: People enter a room one by one, birthdays uniform over m = 365 days. The expected number of people until a repeated birthday is ≈ √(π · m / 2) ≈ 24.0. Derive using waiting-time analysis.
  4. Random Sorting: Given a permutation, you pick a random adjacent pair and swap them if they’re out of order. Expected steps to sort? Model each inversion as needing a geometric number of swaps to be resolved.
  5. CF 1Expected Geometric (CF 1Expected): Various Codeforces problems where the answer involves summing 1/pi for geometric phases. Practice identifying the phase decomposition.
  6. Pokémon Collector: There are n Pokémon species, but each species has a different catch probability pi. Write an O(2n) solution using inclusion-exclusion to find the expected time to catch all species.
  7. Partial Coupon Collector: With n coupon types, how many purchases (in expectation) to collect exactly n/2 distinct types? Use the partial sum formula E[Tm] = n · (Hn − Hn−m).
Contest Tip: When a problem says “perform random operations until some condition is met,” immediately think: can I decompose into geometric phases? Identify the phases, compute each success probability, and sum the reciprocals. This pattern solves at least 50% of expected-value problems in competitive programming.
