    Pólya Enumeration Theorem

    July 2026 · 28 min read

    From Burnside to Pólya

    In the previous post we met Burnside’s lemma: the number of distinct colourings of a set X under a group G of symmetries is

    |X/G|  =  (1/|G|)  Σg∈G |Fix(g)|

    Burnside tells us how many distinct colourings exist, but nothing about their structure. What if we want to know how many distinct necklaces use exactly 3 red beads and 2 blue beads? Burnside alone cannot answer that—it only yields a single count.

    Pólya’s Enumeration Theorem (PET) extends Burnside’s lemma by introducing a weight for each colour. Instead of merely counting fixed points, we substitute colour–weight polynomials into the cycle index of the group and obtain a generating function whose coefficients tell us the number of distinct colourings by weight distribution.

    Key insight. Burnside is the special case of Pólya where every colour has weight 1 (i.e. we only care about the total count, not the colour breakdown). PET keeps track of which colours go where by working with formal power series.

    The central object that makes PET work is the cycle index of the symmetry group. Once you can compute the cycle index, the rest is pure substitution. Let’s define it precisely.

    Cycle Index of a Permutation Group

    Definition

    Let G be a permutation group acting on a set of n elements. Every permutation σ∈G can be decomposed into disjoint cycles. If σ has jk cycles of length k (for k = 1, 2, …, n), we write its cycle type as (j1, j2, …, jn). Note that Σk k·jk = n.

    The cycle index of G is the polynomial in indeterminates s1, s2, …, sn:

    Z(G; s1, s2, …, sn)  =  (1/|G|)  Σσ∈G  s1j1(σ)  s2j2(σ)  …  snjn(σ)

    In other words, each permutation contributes a monomial that encodes its cycle structure, and the cycle index is the average of all these monomials.

    Cycle index of the cyclic group Cn

    The cyclic group Cn consists of the n rotations r0, r1, …, rn−1 of n objects arranged in a circle. The rotation ri has gcd(i, n) cycles, each of length n/gcd(i, n). Grouping rotations by gcd with n and applying Euler’s totient function φ gives the elegant closed form:

    Z(Cn)  =  (1/n) Σd|n φ(d) · sdn/d

    Derivation. Consider rotation ri. Let d = gcd(i, n). This permutation consists of exactly d cycles, each of length n/d, so it contributes sn/dd. For a given divisor d of n, how many values of i (0 ≤ i < n) satisfy gcd(i, n) = d? There are φ(n/d) such values. Substituting d’ = n/d and relabelling:

    Z(Cn) = (1/n) Σd|n φ(n/d) · sn/dd = (1/n) Σd’|n φ(d’) · sd’n/d’

    Cycle index of the dihedral group Dn

    The dihedral group Dn adds n reflections to Cn, giving |Dn| = 2n. The reflection cycle types depend on parity:

    CaseReflection typeContribution to Z
    n oddEach reflection fixes 1 bead and swaps (n−1)/2 pairsn · s1 s2(n−1)/2
    n evenAxis through 2 beads: fixes 2, swaps (n−2)/2 pairs(n/2) · s12 s2(n−2)/2
    Axis between beads: swaps n/2 pairs(n/2) · s2n/2

    So the full cycle index of Dn is:

    Z(Dn)  =  ½ Z(Cn)  +  (reflection terms) / (2n)

    Common mistake. Don’t forget to divide by 2n (not just n) when combining rotations and reflections. The dihedral group has 2n elements, so the cycle index averages over all 2n of them.

    Worked example: Z(C4) and Z(D4)

    For n = 4 the divisors are 1, 2, 4.

    Rotations (C4):

    • r0 (identity): 4 fixed points → s14
    • r1: one 4-cycle → s4
    • r2: two 2-cycles → s22
    • r3: one 4-cycle → s4

    Z(C4) = (1/4)(s14 + s22 + 2s4)

    Using the φ-formula: φ(1)·s14 + φ(2)·s22 + φ(4)·s41 = 1·s14 + 1·s22 + 2·s4 ✔

    Reflections (n = 4, even):

    • 2 reflections through vertices: each contributes s12 s2
    • 2 reflections through edges: each contributes s22

    Z(D4) = (1/8)(s14 + 2s4 + 3s22 + 2s12 s2)

    The Pólya Enumeration Theorem

    Unweighted version (counting distinct colourings)

    Suppose we want to colour n positions using k colours under a symmetry group G. The total number of distinct colourings is obtained by substituting si = k for all i:

    |colourings|  =  Z(G; k, k, …, k)

    Why si = k? A cycle of length i in permutation σ fixes a colouring only if all positions in that cycle share the same colour. There are k choices for each cycle. If σ has j1 + j2 + … + jn total cycles, the number of colourings fixed by σ is k(total cycles). Replacing si = k in s1j1…snjn gives exactly kj1+…+jn. Averaging over G recovers Burnside.

    Weighted (inventory) version

    Now let each colour c have a formal weight w(c). Define the figure-counting series:

    f(x)  =  Σc w(c)

    The Pólya Enumeration Theorem states that the pattern inventory (generating function counting distinct colourings by weight) is:

    PET(G)  =  Z(G; s1→f(x), s2→f(x2), …, sn→f(xn))

    More explicitly, for each indeterminate si, substitute:

    si  →  Σc w(c)i

    Example. Two colours {R, B} with weights w(R) = r, w(B) = b. Then si → ri + bi. The resulting polynomial in r and b enumerates distinct colourings: the coefficient of rabc is the number of distinct colourings using exactly a reds and c blues.

    Connection to Burnside

    Setting every w(c) = 1 collapses the inventory to a single number. Each si becomes 1i + 1i + … = k (for k colours), and PET reduces to the unweighted formula, which is precisely Burnside’s lemma. So PET is a strict generalisation.

    Proof sketch

    The proof follows the same orbit-counting idea as Burnside. A colouring f: X → C is fixed by σ iff f is constant on every cycle of σ. The weight of such a colouring is the product of w(ccycle)|cycle| over all cycles. Summing these weights over all fixed colourings and averaging over G yields the substitution formula above.

    Necklace and Bracelet Counting

    The most classic application of Pólya’s theorem in competitive programming is counting necklaces (rotational equivalence) and bracelets (rotational + reflectional equivalence).

    Necklaces — cyclic group Cn

    A necklace is an equivalence class of colourings under rotation. Using the cycle index of Cn with sd = k:

    N(n, k)  =  (1/n) Σd|n φ(d) · kn/d

    Example: n = 6, k = 2.

    Divisors of 6: 1, 2, 3, 6.

    • φ(1)·26 = 1 · 64 = 64
    • φ(2)·23 = 1 · 8 = 8
    • φ(3)·22 = 2 · 4 = 8
    • φ(6)·21 = 2 · 2 = 4

    N(6, 2) = (64 + 8 + 8 + 4) / 6 = 84 / 6 = 14 distinct necklaces.

    Bracelets — dihedral group Dn

    A bracelet is an equivalence class under both rotation and reflection (you can flip the necklace over). We add the reflection contributions:

    B(n, k)  =  ½ N(n, k)  +  { (k⌈n/2⌉)/2 if n is odd;  (kn/2+1 + kn/2)/4 if n is even }

    Equivalently, for n odd:

    B(n, k) = ½ N(n, k) + ½ k(n+1)/2

    For n even:

    B(n, k) = ½ N(n, k) + ¼ (kn/2 + 1 + kn/2)

    Example: n = 6, k = 2.

    N(6, 2) = 14 (from above). n is even, so reflection term = (24 + 23)/4 = (16 + 8)/4 = 6.

    B(6, 2) = 14/2 + 6 = 7 + 6 = 13.

    Wait—let’s double-check with the explicit D6 cycle index.

    Sanity check your formulas. For small cases, verify by hand or enumerate all colourings. A very common error is confusing N(n,k) (necklaces, rotations only) with B(n,k) (bracelets, rotations + reflections). In CP problems, read carefully whether the problem says “rotations only” or “rotations and reflections.”

    Verification for n = 6, k = 2:

    D6 has 12 elements: 6 rotations + 6 reflections.

    Rotation contributions (sum of kcycles): 26 + 21 + 22 + 23 + 22 + 21 = 64 + 2 + 4 + 8 + 4 + 2 = 84.

    Reflection contributions (n = 6, even): 3 reflections through vertices give 24 = 16 each → 48; 3 reflections through edges give 23 = 8 each → 24. Total reflections: 48 + 24 = 72.

    B(6, 2) = (84 + 72) / 12 = 156 / 12 = 13. ✔

    Necklaces with fixed colour counts

    Use the weighted PET. For a necklace of n = 4 beads with 2 colours {R, B} where we want exactly 2 reds and 2 blues, substitute si → ri + bi into Z(C4):

    Z(C4) = (1/4)(s14 + s22 + 2s4)

    = (1/4)((r+b)4 + (r2+b2)2 + 2(r4+b4))

    The coefficient of r2b2 in this expansion:

    • From (r+b)4: C(4,2) = 6
    • From (r2+b2)2 = r4 + 2r2b2 + b4: coefficient is 2
    • From 2(r4+b4): coefficient of r2b2 is 0

    Total = (6 + 2 + 0)/4 = 2 necklaces with exactly 2R, 2B. They are RRBB and RBRB (up to rotation).

    C++ Implementation

    Let’s build a complete modular-arithmetic toolkit for necklace and bracelet counting. All computations are mod a prime p to handle the huge numbers that arise when n and k are large (up to 109).

    Core utilities: modular exponentiation, inverse, Euler totient

    #include <bits/stdc++.h>
using namespace std;

using ll = long long;
const ll MOD = 1e9 + 7;

ll power(ll base, ll exp, ll mod = MOD) {
    ll result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) result = result * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

ll modinv(ll a, ll mod = MOD) {
    return power(a, mod - 2, mod); // Fermat's little theorem, mod must be prime
}

ll euler_totient(ll n) {
    ll result = n;
    for (ll p = 2; p * p <= n; p++) {
        if (n % p == 0) {
            while (n % p == 0) n /= p;
            result -= result / p;
        }
    }
    if (n > 1) result -= result / n;
    return result;
}
    Euler totient in O(√n). The function above computes φ(n) by trial division. This is fast enough because we only call it for each divisor of n, and the sum of √d over all divisors d of n is manageable for n ≤ 109.

    Counting necklaces mod p

    // Returns the number of distinct necklaces with n beads
// and k colours, modulo MOD.
ll count_necklaces(ll n, ll k) {
    // N(n,k) = (1/n) * sum_{d|n} phi(d) * k^{n/d}
    ll inv_n = modinv(n);
    ll total = 0;

    // Enumerate divisors of n
    for (ll d = 1; d * d <= n; d++) {
        if (n % d != 0) continue;

        // d is a divisor
        ll phi_d = euler_totient(d);
        total = (total + phi_d % MOD * power(k, n / d)) % MOD;

        // n/d is also a divisor (if different from d)
        if (d != n / d) {
            ll d2 = n / d;
            ll phi_d2 = euler_totient(d2);
            total = (total + phi_d2 % MOD * power(k, d)) % MOD;
        }
    }

    return total % MOD * inv_n % MOD;
}

// Example usage
int main() {
    ll n, k;
    cin >> n >> k;
    cout << count_necklaces(n, k) << "\n";
    // n=6, k=2 => 14
}
    Watch out for modular division. We compute 1/n as modinv(n) using Fermat’s little theorem. This requires MOD to be prime and gcd(n, MOD) = 1. If n is a multiple of MOD (extremely rare for MOD = 109+7 and n ≤ 109), you’d need a different approach.

    Counting bracelets mod p

    ll count_bracelets(ll n, ll k) {
    // B(n,k) = (1/2) * N(n,k) + reflection_term
    // where reflection_term depends on parity of n.

    ll necklaces = count_necklaces(n, k);
    ll inv2 = modinv(2);
    ll reflection;

    if (n % 2 == 1) {
        // n odd: n reflections, each with 1 fixed point
        // and (n-1)/2 transpositions => k^{(n+1)/2} per reflection
        // reflection_term = (1/2) * k^{(n+1)/2}
        reflection = inv2 % MOD * power(k, (n + 1) / 2) % MOD;
    } else {
        // n even: n/2 reflections through vertices (k^{n/2+1})
        //       + n/2 reflections through edges   (k^{n/2})
        // reflection_term = (1/4)(k^{n/2+1} + k^{n/2})
        ll inv4 = modinv(4);
        ll a = power(k, n / 2 + 1);
        ll b = power(k, n / 2);
        reflection = inv4 % MOD * ((a + b) % MOD) % MOD;
    }

    return (necklaces % MOD * inv2 % MOD + reflection) % MOD;
}

// Example: n=6, k=2 => 13

    Weighted Pólya counting

    When we need the full pattern inventory (e.g. how many necklaces have exactly a beads of colour 1 and b beads of colour 2), we must work with polynomials. Here is a simplified version for two colours under Cn:

    // Polynomial multiplication (truncated to degree n)
using Poly = vector<ll>;

Poly poly_mult(const Poly& a, const Poly& b, int max_deg) {
    Poly c(max_deg + 1, 0);
    for (int i = 0; i < (int)a.size() && i <= max_deg; i++)
        for (int j = 0; j < (int)b.size() && i + j <= max_deg; j++)
            c[i + j] = (c[i + j] + a[i] * b[j]) % MOD;
    return c;
}

// Raise polynomial p to the e-th power, keeping terms up to max_deg
Poly poly_pow(Poly p, int e, int max_deg) {
    Poly result(max_deg + 1, 0);
    result[0] = 1;
    while (e > 0) {
        if (e & 1) result = poly_mult(result, p, max_deg);
        p = poly_mult(p, p, max_deg);
        e >>= 1;
    }
    return result;
}

// Weighted necklace count for 2 colours, n beads.
// Returns polynomial where coeff[a] = number of distinct necklaces
// with exactly a beads of colour 1 (and n-a of colour 2).
Poly weighted_necklaces(int n) {
    ll inv_n = modinv(n);
    Poly total(n + 1, 0);

    for (int d = 1; d * d <= n; d++) {
        if (n % d != 0) continue;

        auto process = [&](int div) {
            ll phi_val = euler_totient(div) % MOD;
            int cycle_len = n / div;
            int num_cycles = div;
            // s_cycle_len -> (x^cycle_len + 1) represents
            // colour1^cycle_len + colour2^cycle_len
            Poly base(cycle_len + 1, 0);
            base[0] = 1;                // colour 2 contributes x^0
            base[cycle_len] = 1;        // colour 1 contributes x^cycle_len
            Poly contrib = poly_pow(base, num_cycles, n);
            for (int i = 0; i <= n; i++)
                total[i] = (total[i] + phi_val * contrib[i]) % MOD;
        };

        process(d);
        if (d != n / d) process(n / d);
    }

    for (int i = 0; i <= n; i++)
        total[i] = total[i] * inv_n % MOD;

    return total;
}

    For n = 4 the output polynomial is [1, 1, 2, 1, 1], meaning 1 necklace with 0 reds, 1 with 1 red, 2 with 2 reds, 1 with 3 reds, 1 with 4 reds. Total = 6 = N(4, 2). ✔

    Generating Functions with Pólya

    One of the most powerful features of PET is that it naturally produces generating functions. By assigning each colour a formal variable, the cycle index substitution yields a multivariate polynomial whose coefficients enumerate distinct structures by their “weight profile.”

    Setup

    Given colours C = {c1, c2, …, ck} with formal weights w(ci) = xi, the pattern inventory is:

    P(x1, …, xk)  =  Z(G; si → x1i + x2i + … + xki)

    The coefficient of x1a1 x2a2 … xkak counts the number of distinct colourings using exactly ai copies of colour ci.

    Example: graph colouring (cube faces)

    Consider colouring the 6 faces of a cube with colours {R, B, G}. The rotation group of the cube has 24 elements. Its cycle index (acting on faces) is:

    Z = (1/24)(s16 + 6s12s4 + 3s12s22 + 8s32 + 6s23)

    Substituting si → 3 (for k = 3 colours, unweighted): Z(3, 3, 3, 3, 3, 3) = (729 + 6·9·3 + 3·9·9 + 8·9 + 6·27)/24 = (729 + 162 + 243 + 72 + 162)/24 = 1368/24 = 57.

    For the weighted version (tracking colour counts), substitute si → ri + bi + gi. The resulting polynomial in r, b, g gives the complete breakdown. For example, the coefficient of r3b2g tells you how many cubes have 3 red, 2 blue, and 1 green face.

    Chemical isomers

    PET was originally motivated by chemistry. To count distinct chemical isomers of a substituted benzene ring C6H6−mXm (e.g. how many dichlorobenzenes exist), we use Z(D6) and substitute with two “colours” (H and X). The answer for m = 2 is the coefficient of x2 in the pattern inventory, which equals 3 (ortho, meta, para).

    PET in competitive programming. While full multivariate generating functions are rare in contests, the technique of substituting si → sum of i-th powers is extremely common. Many CP problems reduce to: (1) identify the symmetry group, (2) compute its cycle index, (3) substitute.

    CP Problem Walkthrough

    Problem: Coloured Necklace (CSES-style)

    Statement. Given n (1 ≤ n ≤ 109) and k (1 ≤ k ≤ 109), compute the number of distinct necklaces with n beads and k colours, modulo 109+7. Two necklaces are identical if one can be obtained from the other by rotation.

    Analysis

    This is a direct application of the necklace formula:

    N(n, k) = (1/n) Σd|n φ(d) · kn/d  (mod 109+7)

    The key algorithmic challenges are:

    1. Enumerating divisors of n in O(√n) time.
    2. Computing φ(d) for each divisor d. Since d can be up to 109, we use the O(√d) trial-division totient.
    3. Modular exponentiation kn/d mod p in O(log(n/d)) time.
    4. Modular inverse of n using Fermat’s little theorem.

    The total number of divisors of n is at most about 1344 for n ≤ 109 (a well-known bound), and computing φ for each costs O(√d), so the overall complexity is roughly O(√n · d(n)) which is very fast.

    Full solution

    #include <bits/stdc++.h>
using namespace std;
using ll = long long;
const ll MOD = 1e9 + 7;

ll power(ll b, ll e, ll m = MOD) {
    ll r = 1; b %= m;
    for (; e; e >>= 1) {
        if (e & 1) r = r * b % m;
        b = b * b % m;
    }
    return r;
}

ll modinv(ll a) { return power(a, MOD - 2); }

ll euler_phi(ll n) {
    ll res = n;
    for (ll p = 2; p * p <= n; p++) {
        if (n % p == 0) {
            while (n % p == 0) n /= p;
            res -= res / p;
        }
    }
    if (n > 1) res -= res / n;
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n, k;
    cin >> n >> k;

    // Collect all divisors of n
    vector<ll> divs;
    for (ll d = 1; d * d <= n; d++) {
        if (n % d == 0) {
            divs.push_back(d);
            if (d != n / d) divs.push_back(n / d);
        }
    }

    ll ans = 0;
    for (ll d : divs) {
        ll phi = euler_phi(d) % MOD;
        ll contrib = phi * power(k, n / d) % MOD;
        ans = (ans + contrib) % MOD;
    }

    ans = ans % MOD * modinv(n) % MOD;
    cout << ans << "\n";
}

    Variant: bracelet counting

    If the problem instead asks for bracelets (rotation + reflection), simply add the reflection terms as shown in the count_bracelets function above. Many contest problems (e.g. “NECKLACE” on SPOJ) explicitly state whether flipping is allowed—read carefully!

    Variant: colourings with forbidden patterns

    Some harder problems restrict which colours may be adjacent. PET alone does not handle adjacency constraints. For those, combine PET with transfer matrix / DP approaches: use the matrix to count valid colourings fixed by each group element, then average via Burnside.

    Summary

    Burnside’s Lemma

    • Counts total distinct colourings
    • |X/G| = (1/|G|) Σ |Fix(g)|
    • No weight information
    • Sufficient when you only need a count

    Pólya Enumeration

    • Gives a generating function over colour weights
    • Substitute si → Σ w(c)i in cycle index
    • Recovers Burnside as special case (all weights = 1)
    • Needed when colour counts matter

    Key formulas to memorise for contests:

    • Cycle index of Cn: Z(Cn) = (1/n) Σd|n φ(d) · sdn/d
    • Necklaces: N(n, k) = (1/n) Σd|n φ(d) · kn/d
    • Bracelets (odd n): B = N/2 + k(n+1)/2/2
    • Bracelets (even n): B = N/2 + (kn/2+1 + kn/2)/4
    • Weighted PET: substitute si → Σc w(c)i

    Implementation checklist:

    1. Enumerate divisors of n in O(√n)
    2. Compute φ(d) for each divisor via trial division
    3. Use fast modular exponentiation for kn/d
    4. Use Fermat inverse for 1/n (mod prime)
    5. For bracelets, add reflection terms and divide by 2
