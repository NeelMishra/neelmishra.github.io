    Graph Construction

    July 2026

    Constructive graph problems ask you to build a graph that satisfies a set of constraints — a given degree sequence, a target diameter, a required connectivity, or some combination of all three. These problems are a staple of Codeforces Div 1 and ICPC regionals, and they reward a blend of graph theory knowledge and careful algorithmic thinking.

    In this post we cover the essential theory and practical algorithms you need to construct graphs on demand during a contest.

    Degree Sequences & the Erdős–Gallai Theorem

    A degree sequence is the sorted (non-increasing) list of vertex degrees in a simple undirected graph. The central question is: given a sequence d1 ≥ d2 ≥ … ≥ dn, does a simple graph with this degree sequence exist? If yes, the sequence is called graphical.

    Erdős–Gallai Theorem

    A non-increasing sequence of non-negative integers d1 ≥ d2 ≥ … ≥ dn is graphical if and only if:

    1. The sum ∑ di is even.
    2. For every k = 1, 2, …, n:
      ∑i=1..k di ≤ k(k−1) + ∑i=k+1..n min(di, k)
    Intuition: Condition 2 says that the top-k vertices cannot demand more edges among themselves than the complete graph on k vertices allows, plus whatever edges the remaining vertices can supply to those k vertices.

    The check runs in O(n) time if you precompute prefix sums:

    #include <bits/stdc++.h>
using namespace std;

// Returns true if the degree sequence is graphical (Erdos-Gallai).
// Input: d must be sorted in non-increasing order.
bool erdosGallai(vector<int>& d) {
    int n = d.size();
    long long total = 0;
    for (int x : d) total += x;
    if (total % 2 != 0) return false;

    // prefix[i] = d[0] + d[1] + ... + d[i-1]
    vector<long long> prefix(n + 1, 0);
    for (int i = 0; i < n; i++)
        prefix[i + 1] = prefix[i] + d[i];

    for (int k = 1; k <= n; k++) {
        long long lhs = prefix[k];
        long long rhs = (long long)k * (k - 1);
        // sum of min(d[i], k) for i = k..n-1
        // Since d is sorted, find first index where d[i] < k
        int pos = lower_bound(d.begin() + k, d.end(), k,
                              greater<int>()) - d.begin();
        // d[k..pos-1] >= k, so min(d[i],k) = k for those
        // d[pos..n-1] < k, so min(d[i],k) = d[i] for those
        rhs += (long long)k * (pos - k);
        rhs += prefix[n] - prefix[pos];

        if (lhs > rhs) return false;
    }
    return true;
}

    Realizing a Degree Sequence: The Hakimi Algorithm

    Once we know a sequence is graphical, we need to actually build the graph. The Erdős–Gallai / Hakimi algorithm is the classic greedy approach:

    1. Sort vertices by remaining degree (descending).
    2. Take the vertex v with the highest remaining degree d.
    3. Connect v to the next d vertices with the highest remaining degrees.
    4. Decrease those degrees by 1, remove v from the list.
    5. Repeat until all degrees are zero.

    If at any step the highest-degree vertex demands more connections than remain available, the sequence is not graphical (but we already checked that with Erdős–Gallai).

    Complexity: A naive implementation is O(n2 log n) due to repeated sorting. Using a priority queue brings it to O(n2) or O(m log n) where m is the number of edges. For contest purposes O(n2) usually suffices since n ≤ 105 is rare in construction problems. 
    #include <bits/stdc++.h>
using namespace std;

// Given a graphical degree sequence d[0..n-1], construct the graph.
// Returns the edge list. d is destroyed in the process.
vector<pair<int,int>> hakimi(vector<int> d) {
    int n = d.size();
    vector<pair<int,int>> edges;

    // pairs of (remaining_degree, original_index)
    vector<pair<int,int>> verts(n);
    for (int i = 0; i < n; i++) verts[i] = {d[i], i};

    for (int step = 0; step < n; step++) {
        sort(verts.begin(), verts.end(),
             [](auto& a, auto& b){ return a.first > b.first; });

        // remove vertices with degree 0 from the back
        while (!verts.empty() && verts.back().first == 0)
            verts.pop_back();
        if (verts.empty()) break;

        auto [deg, u] = verts[0];
        if (deg > (int)verts.size() - 1) {
            // should not happen if Erdos-Gallai passed
            return {};
        }
        for (int j = 1; j <= deg; j++) {
            edges.push_back({u, verts[j].second});
            verts[j].first--;
        }
        verts[0].first = 0;
    }
    return edges;
}

int main() {
    int n;
    cin >> n;
    vector<int> d(n);
    for (int i = 0; i < n; i++) cin >> d[i];

    sort(d.begin(), d.end(), greater<int>());
    if (!erdosGallai(d)) {          // reuse from previous section
        cout << "IMPOSSIBLE\n";
        return 0;
    }

    auto edges = hakimi(d);
    cout << edges.size() << "\n";
    for (auto [u, v] : edges)
        cout << u + 1 << " " << v + 1 << "\n";
}

    Key pitfall: when the problem gives vertex labels along with degrees, you must track the original indices through the sorting steps (as we do with the verts array above).

    Prüfer Sequences & Labeled Tree Construction

    A Prüfer sequence is a unique encoding of a labeled tree on n vertices as a sequence of n − 2 integers, each in [1, n]. This bijection between labeled trees and sequences has powerful consequences:

    • Cayley’s formula: The number of labeled trees on n vertices is nn−2, since there are nn−2 possible Prüfer sequences.
    • Degree control: Vertex v appears exactly deg(v) − 1 times in the Prüfer sequence. So to build a tree where vertex v has degree dv, include v exactly dv − 1 times.
    • Random trees: Generate a random Prüfer sequence to get a uniformly random labeled tree.

    Decoding a Prüfer Sequence

    Given a Prüfer sequence a[0..n-3], reconstruct the tree in O(n):

    1. Count how many times each vertex appears in the sequence — call this cnt[v].
    2. Find the smallest vertex leaf with cnt[leaf] == 0.
    3. For each element a[i]: add edge (leaf, a[i]), decrement cnt[a[i]]. If a[i] < leaf_ptr and cnt[a[i]] == 0, set leaf = a[i]; otherwise advance leaf_ptr to find the next zero-count vertex.
    4. The last edge connects the two remaining vertices with cnt == 0.
    #include <bits/stdc++.h>
using namespace std;

// Decode Prufer sequence to tree edges. O(n) time.
// seq has length n-2, vertices are 1-indexed [1..n].
vector<pair<int,int>> pruferDecode(const vector<int>& seq, int n) {
    vector<int> cnt(n + 1, 0);
    for (int x : seq) cnt[x]++;

    vector<pair<int,int>> edges;
    int leaf = -1;
    for (int i = 1; i <= n; i++) {
        if (cnt[i] == 0) { leaf = i; break; }
    }

    int ptr = leaf;
    for (int x : seq) {
        edges.push_back({leaf, x});
        cnt[x]--;
        if (cnt[x] == 0 && x < ptr) {
            leaf = x;
        } else {
            ptr++;
            while (ptr <= n && cnt[ptr] != 0) ptr++;
            leaf = ptr;
        }
    }
    // last edge: connect the two remaining vertices
    // ptr should be at one of them, and n is always the other
    edges.push_back({leaf, n});
    return edges;
}

int main() {
    int n;
    cin >> n;
    if (n == 1) { cout << "0\n"; return 0; }
    if (n == 2) { cout << "1\n1 2\n"; return 0; }

    vector<int> seq(n - 2);
    for (int i = 0; i < n - 2; i++) cin >> seq[i];

    auto edges = pruferDecode(seq, n);
    for (auto [u, v] : edges)
        cout << u << " " << v << "\n";
}

    Encoding: Tree → Prüfer Sequence

    The reverse direction is also O(n): repeatedly remove the smallest-label leaf and append its neighbour to the sequence. The implementation mirrors the decode — use a pointer that scans upward to find the next leaf efficiently.

    Contest tip: If a problem asks you to construct a labeled tree where vertex v has degree dv, just build the Prüfer sequence with vertex v appearing dv − 1 times, then decode. The necessary and sufficient condition is ∑ (dv − 1) = n − 2, i.e. ∑ dv = 2(n − 1).

    Constructing Graphs with Given Diameter

    A common problem type: construct a tree (or graph) on n vertices with diameter exactly D. Here is the standard strategy for trees:

    1. Create a path of length D — that is, D+1 vertices v0, v1, …, vD connected in a chain. This path is the diameter.
    2. Attach the remaining n − D − 1 vertices to the interior of the path (not to the endpoints!) in a way that does not increase the diameter.

    Attaching a vertex to vi (where 1 ≤ i ≤ D−1) creates a new leaf whose eccentricity is max(i + 1, D − i + 1). This is at most D when i is “near the middle” of the path. Specifically:

    • For D ≥ 2, attach extra vertices to v⌊D/2⌋ (the center). Each extra leaf has eccentricity ⌈D/2⌉ + 1 ≤ D (as long as D ≥ 2).
    • For D = 1, you need n = 2 (a single edge). If n > 2 and D = 1, it is impossible for a tree.
    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n, D;
    cin >> n >> D;

    if (D >= n) {
        cout << "IMPOSSIBLE\n";
        return 0;
    }
    if (D == 1 && n > 2) {
        cout << "IMPOSSIBLE\n";
        return 0;
    }

    vector<pair<int,int>> edges;

    // 1) Build the backbone path: 1 -- 2 -- ... -- (D+1)
    for (int i = 1; i <= D; i++)
        edges.push_back({i, i + 1});

    // 2) Attach remaining vertices to the center of the path
    int center = D / 2 + 1;  // 1-indexed center vertex
    for (int v = D + 2; v <= n; v++)
        edges.push_back({center, v});

    for (auto [u, v] : edges)
        cout << u << " " << v << "\n";
}

    Graphs (Non-Trees) with Given Diameter

    For general graphs you have more freedom. Build the diameter path, connect remaining vertices to the center, then add extra edges between non-endpoint vertices. Adding edges can only decrease distances, so verify with BFS that the diameter was not reduced below D.

    Watch out: If the problem says “exactly diameter D”, you must verify that adding extra edges did not reduce the actual diameter below D. In trees this is never an issue; in general graphs you may need a BFS check.

    Constructing Eulerian & Hamiltonian Graphs

    Eulerian Graphs

    Recall the characterisation:

    • An undirected graph has an Eulerian circuit iff it is connected and every vertex has even degree.
    • It has an Eulerian path iff it is connected and has exactly two vertices of odd degree.

    To construct a connected Eulerian graph on n vertices, start with a Hamiltonian cycle (every vertex gets degree 2) and add shortcut edges as needed, ensuring all degree increments stay even.

    Directed Eulerian Graphs

    A directed graph has an Eulerian circuit iff it is strongly connected and in-degree(v) = out-degree(v) for every vertex. Construction: 

    // Build a directed Eulerian graph on n vertices with
// in-deg = out-deg = k for every vertex.
// Strategy: n vertices on a "ring", each vertex has k outgoing
// edges to the next k vertices (mod n).
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;
    // k must be < n for simple graph
    if (k >= n) { cout << "IMPOSSIBLE\n"; return 0; }

    vector<pair<int,int>> edges;
    for (int v = 0; v < n; v++)
        for (int j = 1; j <= k; j++)
            edges.push_back({v, (v + j) % n});

    cout << edges.size() << "\n";
    for (auto [u, v] : edges)
        cout << u + 1 << " " << v + 1 << "\n";
}

    Hamiltonian Graphs

    There is no efficient general test for Hamiltonicity, but construction is often tractable when you get to choose the graph. Useful facts:

    • Dirac’s theorem: If n ≥ 3 and every vertex has degree ≥ n/2, the graph is Hamiltonian.
    • Ore’s theorem: If for every pair of non-adjacent vertices u, v we have deg(u) + deg(v) ≥ n, the graph is Hamiltonian.

    When a problem asks you to construct a graph that is Hamiltonian, the safest approach is to start with a Hamiltonian cycle and then add or remove edges as needed to satisfy additional constraints.

    Constructing Graphs with Target Connectivity

    k-vertex-connected means the graph stays connected after removing any k−1 vertices. The Harary graph Hk,n achieves k-connectivity with the minimum number of edges:

    • Place n vertices on a circle.
    • If k is even: connect each vertex to its nearest k/2 neighbours on each side.
    • If k is odd and n is even: start with Hk−1,n and add all diametrically opposite edges.
    • If both k and n are odd: start with Hk−1,n and add edges from vertex i to vertex i + (n−1)/2 and i + (n+1)/2 for i = 0, …, (n−1)/2.
    #include <bits/stdc++.h>
using namespace std;

// Construct Harary graph H_{k,n}: k-connected graph on n vertices
// with the minimum number of edges.
vector<pair<int,int>> harary(int k, int n) {
    set<pair<int,int>> edgeSet;
    auto addEdge = [&](int u, int v) {
        if (u > v) swap(u, v);
        if (u != v) edgeSet.insert({u, v});
    };

    // Even part: connect each vertex to nearest floor(k/2) on each side
    int half = k / 2;
    for (int i = 0; i < n; i++)
        for (int j = 1; j <= half; j++)
            addEdge(i, (i + j) % n);

    // If k is odd
    if (k % 2 == 1) {
        if (n % 2 == 0) {
            // add diametrically opposite edges
            for (int i = 0; i < n / 2; i++)
                addEdge(i, i + n / 2);
        } else {
            // add edges from i to i+(n-1)/2 and i+(n+1)/2
            for (int i = 0; i <= (n - 1) / 2; i++) {
                addEdge(i, (i + (n - 1) / 2) % n);
                addEdge(i, (i + (n + 1) / 2) % n);
            }
        }
    }

    return vector<pair<int,int>>(edgeSet.begin(), edgeSet.end());
}

int main() {
    int k, n;
    cin >> k >> n;
    auto edges = harary(k, n);
    cout << edges.size() << "\n";
    for (auto [u, v] : edges)
        cout << u + 1 << " " << v + 1 << "\n";
}
    Edge count: Hk,n has exactly ⌈kn/2⌉ edges, which is the proven minimum for a k-connected graph on n vertices.

    Constructing Trees with Specific Properties

    Trees are the most common target of construction problems. Here are the key patterns:

    Tree with Given Degree Sequence

    A sequence d1, …, dn (each ≥ 1) is the degree sequence of a tree iff ∑ di = 2(n − 1). The construction via Prüfer sequences is the most elegant:

    1. For each vertex v, it must appear dv − 1 times in the Prüfer sequence.
    2. Arrange those appearances in any order (e.g., smallest label first).
    3. Decode the Prüfer sequence to recover the tree.

    Tree with Given Number of Leaves

    Build a “caterpillar”: take a path of length n − leaves, then attach the remaining leaves vertices as leaves to interior nodes (distributing them to keep the tree valid).

    Competitive Programming Problem Patterns

    Let’s look at several common problem archetypes and their solutions.

    Problem 1: Construct a Connected Graph with Given Degrees

    Given n and degree sequence d1, …, dn, construct a connected simple graph with these degrees, or report it’s impossible.

    Approach: First check Erdős–Gallai. Then use Hakimi to build the graph. Finally, check connectivity with BFS/DFS. A subtlety: a graphical sequence may be realisable but only as a disconnected graph. One way to ensure connectivity is to first build a spanning tree (possible iff all degrees ≥ 1 and n − 1 ≤ m), then use the remaining degree budget for extra edges.

    Problem 2: Construct a Tree with Exactly k Leaves (Full Solution)

    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;

    // Edge cases
    if (k >= n || k < 2) {
        // k < 2: tree with <2 leaves needs n<=1
        // k >= n: impossible for n>=3 (need at least 1 internal node)
        if (n == 1 && k == 0) { cout << "0\n"; return 0; }
        if (n == 2 && k == 2) { cout << "1 2\n"; return 0; }
        cout << "IMPOSSIBLE\n";
        return 0;
    }

    // Strategy: build a path of length (n - k) using (n - k + 1) vertices
    // These form the "spine". Then attach k - 2 extra leaves to internal
    // nodes of the spine. (The two endpoints of the spine are already leaves.)
    // Spine: 1 -- 2 -- 3 -- ... -- (n - k + 1)
    // Extra leaves: (n-k+2), (n-k+3), ..., n  attached to vertex 2

    int spine_len = n - k + 1;  // number of vertices in the spine
    vector<pair<int,int>> edges;

    for (int i = 1; i < spine_len; i++)
        edges.push_back({i, i + 1});

    // Attach extra leaves to vertex 2 (an internal node)
    for (int v = spine_len + 1; v <= n; v++)
        edges.push_back({2, v});

    for (auto [u, v] : edges)
        cout << u << " " << v << "\n";
}

    Problem 3: Construct a Graph with n Vertices, m Edges, and Diameter D

    Build the diameter path (D edges), attach remaining vertices to the center, then add m − (n−1) extra edges between non-endpoint internal vertices. Verify with BFS from both endpoints that the diameter was not reduced below D.

    Problem 4: Construct a Regular Graph

    A k-regular graph on n vertices has every vertex with degree exactly k. Existence requires kn to be even. Construction: 

    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;

    if ((long long)n * k % 2 != 0 || k >= n) {
        cout << "IMPOSSIBLE\n";
        return 0;
    }

    vector<pair<int,int>> edges;

    // Circulant graph: vertex i connects to i+1, i+2, ..., i+floor(k/2) mod n
    for (int i = 0; i < n; i++)
        for (int j = 1; j <= k / 2; j++)
            edges.push_back({i, (i + j) % n});

    // If k is odd, n must be even. Add edges between opposite vertices.
    if (k % 2 == 1) {
        for (int i = 0; i < n / 2; i++)
            edges.push_back({i, i + n / 2});
    }

    cout << edges.size() << "\n";
    for (auto [u, v] : edges)
        cout << u + 1 << " " << v + 1 << "\n";
}

    Tips & Summary

    Task Key Tool Complexity
    Check if degree sequence is graphical Erdős–Gallai O(n)
    Realise a degree sequence Hakimi algorithm O(n2)
    Build a labeled tree with given degrees Prüfer sequence O(n)
    Tree with diameter D Path + attach to center O(n)
    k-connected graph Harary graph O(kn)
    k-regular graph Circulant graph O(kn)
    General contest strategy for graph construction:
    1. Determine the necessary conditions for existence (parity, edge counts, degree sums). Print “IMPOSSIBLE” early.
    2. Start with the simplest structure that meets the hardest constraint (path for diameter, cycle for Eulerian, star for high connectivity from one vertex).
    3. Incrementally add edges/vertices to satisfy remaining constraints.
    4. Verify the construction with BFS/DFS if time permits — especially for diameter and connectivity checks.

    Graph construction is one of those topics where knowing a handful of classical results — Erdős–Gallai, Prüfer, Harary, Dirac — gives you a ready-made toolkit for the majority of contest problems. The key insight is always the same: start from the most constrained property, build the minimal structure that satisfies it, and then carefully layer on additional constraints without breaking what you already have.
