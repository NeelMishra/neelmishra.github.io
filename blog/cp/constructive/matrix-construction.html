Matrix Construction

July 2026

Matrix construction problems are a staple of competitive programming. They ask you to fill an n × m grid subject to constraints — every row a permutation, every column summing to a target, XOR across rows equalling zero. Unlike search problems, here you must build a valid object, and the key insight is structural: once you see the right pattern, the code writes itself.

This post covers the most important matrix construction families: Latin squares, magic squares, matrices with prescribed row/column sums, constrained binary matrices, and XOR constructions. For each we develop theory and write concrete C++ solutions.

1. Latin Squares

A Latin square of order n is an n × n matrix filled with n distinct symbols such that each symbol occurs exactly once in every row and exactly once in every column. The simplest example for n = 3:

1 2 3
2 3 1
3 1 2

Cyclic Construction

The most contest-friendly construction is the cyclic shift method. Set a[i][j] = (i + j) mod n. Row i is just the sequence 0, 1, …, n−1 shifted left by i positions. Because addition modulo n is a group operation, every row and every column is a permutation of {0, 1, …, n−1}. 

// Construct an n x n Latin square using cyclic shifts.
// Output uses 1-indexed symbols [1..n].
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n;
    cin >> n;
    for(int i = 0; i < n; i++){
        for(int j = 0; j < n; j++){
            cout << (i + j) % n + 1;
            if(j + 1 < n) cout << ' ';
        }
        cout << '\n';
    }
}

Mutually Orthogonal Latin Squares (MOLS)

Two Latin squares L1 and L2 of order n are orthogonal if, when superimposed, every ordered pair (L1[i][j], L2[i][j]) is distinct. When n is prime or a prime power, we can build up to n − 1 MOLS using the formula:

Lk[i][j] = (k · i + j) mod n, for k = 1, 2, …, n − 1.

Contest tip: If a problem asks you to colour an n × n grid with n colours so that no colour repeats in any row or column, the answer is simply a Latin square. The cyclic construction above works for every n.

Reduced Latin Squares

A Latin square is reduced if the first row and first column are in natural order: 1, 2, …, n. Any Latin square can be converted to reduced form by permuting rows and columns. Rarely needed in contests unless explicitly required.

2. Magic Squares

A magic square of order n is an n × n matrix filled with the integers 1, 2, …, n2 such that every row, every column, and both main diagonals sum to the same magic constant M = n(n2 + 1) / 2. Construction splits into three cases depending on n modulo 4.

Odd Order — The Siamese (De La Loubère) Method

For odd n, start by placing 1 in the middle of the top row. Then repeatedly move one step up and one step right (wrapping around), placing the next number. If the target cell is already occupied, move one step down from the current position instead. 

// Siamese method for odd-order magic squares.
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n;
    cin >> n;
    assert(n % 2 == 1);
    vector<vector<int>> sq(n, vector<int>(n, 0));

    int r = 0, c = n / 2;
    for(int val = 1; val <= n * n; val++){
        sq[r][c] = val;
        int nr = (r - 1 + n) % n;
        int nc = (c + 1) % n;
        if(sq[nr][nc] != 0){
            // Cell occupied: drop down one row instead
            nr = (r + 1) % n;
            nc = c;
        }
        r = nr;
        c = nc;
    }

    for(int i = 0; i < n; i++){
        for(int j = 0; j < n; j++){
            cout << sq[i][j];
            if(j + 1 < n) cout << '\t';
        }
        cout << '\n';
    }
}

For n = 5, this produces:

17   24    1    8   15
23    5    7   14   16
 4    6   13   20   22
10   12   19   21    3
11   18   25    2    9

Every row, column, and main diagonal sums to M = 65.

Doubly Even Order (n ≡ 0 mod 4)

When n is divisible by 4, the classical approach is:

  1. Fill the grid with numbers 1 to n2 in row-major order.
  2. Divide the grid into n/4 × n/4 blocks of size 4 × 4. In each 4 × 4 block, mark the cells on both diagonals.
  3. Replace every marked cell a[i][j] with n2 + 1 − a[i][j].

This works because swapping symmetric pairs preserves row, column, and diagonal sums. For n = 4 the magic constant is 34 and the result is:

16    2    3   13
 5   11   10    8
 9    7    6   12
 4   14   15    1

Singly Even Order (n ≡ 2 mod 4, n ≥ 6)

The hardest case. Write n = 2m where m is odd. The Strachey method builds four copies of an odd-order magic square of size m with offsets 0, m2, 2m2, 3m2 placed in the four quadrants, then swaps select columns between the top-left and bottom-left quadrants to fix diagonal sums.

Warning: The singly-even case is rarely asked in CP because the construction is fiddly. Most contest problems either restrict n to be odd (Siamese method) or divisible by 4 (doubly-even method), or they only ask for existence. Know the high-level idea but don’t memorise the full algorithm.

3. Matrices with Given Row/Column Sums

A very common CP motif: “Construct an n × m matrix of non-negative integers whose i-th row sums to ri and j-th column sums to cj.” A necessary condition is that the total row sum equals the total column sum: ∑ ri = ∑ cj. For non-negative integers this condition is also sufficient — we can always build such a matrix greedily.

Greedy Algorithm

Process cells in row-major order. For cell (i, j), assign min(remaining_row[i], remaining_col[j]). This never violates any constraint because the total of remaining row demands always equals the total of remaining column demands. 

// Construct a matrix with given row sums r[] and column sums c[].
// Assumes sum(r) == sum(c).
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, m;
    cin >> n >> m;
    vector<long long> r(n), c(m);
    for(auto &x : r) cin >> x;
    for(auto &x : c) cin >> x;

    vector<vector<long long>> mat(n, vector<long long>(m));
    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            long long val = min(r[i], c[j]);
            mat[i][j] = val;
            r[i] -= val;
            c[j] -= val;
        }
    }

    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            cout << mat[i][j];
            if(j + 1 < m) cout << ' ';
        }
        cout << '\n';
    }
}

The Gale–Ryser Theorem (0/1 Matrices)

Given sorted row sums r1 ≥ … ≥ rn and column sums c1 ≥ … ≥ cm, a binary matrix with these margins exists iff:

  1. ∑ ri = ∑ cj, and
  2. For every k = 1, 2, …, m: ∑j=1k cj ≤ ∑i=1n min(ri, k).

This is the Gale–Ryser theorem. The constructive proof gives an algorithm: sort rows by decreasing sum, then greedily assign 1s in each row to the columns with the largest remaining column sum.

Key insight: The Gale–Ryser condition generalises the Erdős–Gallai theorem for degree sequences. A degree sequence is graphical if and only if a symmetric 0/1 matrix with zero diagonal and those row sums exists.

Constructive Algorithm for Binary Matrices

Sort rows by decreasing ri. For each row, place 1s in the ri columns with largest remaining column sums, then update. 

// Construct a binary matrix with given row sums and column sums.
// Returns "NO" if impossible (Gale-Ryser violated).
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, m;
    cin >> n >> m;
    vector<int> r(n), c(m);
    for(auto &x : r) cin >> x;
    for(auto &x : c) cin >> x;

    if(accumulate(r.begin(), r.end(), 0LL) !=
       accumulate(c.begin(), c.end(), 0LL)){
        cout << "NO\n";
        return 0;
    }

    // Store original indices for output
    vector<pair<int,int>> rows(n), cols(m);
    for(int i = 0; i < n; i++) rows[i] = {r[i], i};
    for(int j = 0; j < m; j++) cols[j] = {c[j], j};
    sort(rows.rbegin(), rows.rend());

    vector<vector<int>> mat(n, vector<int>(m, 0));

    for(int ii = 0; ii < n; ii++){
        auto [ri, orig_i] = rows[ii];
        // Pick the ri columns with largest remaining sums
        sort(cols.rbegin(), cols.rend());
        if(ri > m || (ri > 0 && cols[ri - 1].first <= 0)){
            cout << "NO\n";
            return 0;
        }
        for(int k = 0; k < ri; k++){
            mat[orig_i][cols[k].second] = 1;
            cols[k].first--;
        }
    }

    cout << "YES\n";
    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            cout << mat[i][j];
            if(j + 1 < m) cout << ' ';
        }
        cout << '\n';
    }
}

Time complexity: O(n · m log m) due to sorting columns per row.

4. Binary Matrices with Constraints

Many CP problems ask you to fill a 0/1 matrix satisfying more exotic constraints than just row and column sums. Here are the most common patterns.

No Two Adjacent 1s

Construct an n × m binary matrix with exactly k ones such that no two 1s share an edge. The maximum number of non-adjacent 1s in an n × m grid is ⌈n · m / 2⌉ (checkerboard pattern). So the problem is solvable iff k ≤ ⌈nm/2⌉.

Strategy: place 1s on the checkerboard positions (i, j) where (i + j) % 2 == 0, stopping after k placements. Iterate in row-major order to get a deterministic construction.

Each Row and Column Has Even Sum

Fill the first (n−1) × (m−1) submatrix freely, then set each entry in the last column so that its row has even sum, and each entry in the last row so that its column has even sum. The bottom-right corner is determined by either constraint (they agree because the total sum of all entries is even). This is a parity argument.

Exactly k Inversions in Each Row

Construct a binary matrix where each row has exactly k pairs (a, b) with a 1 before a 0. The inversion count of a binary row with p ones depends on how far right the ones are shifted. This reduces to choosing p and placing 1s greedily to hit the target count.

5. XOR Matrix Construction

XOR-based matrix problems appear frequently in Codeforces Div 1/2 contests. The typical setup: construct an n × m matrix where the XOR of each row equals x and the XOR of each column equals y.

Existence Condition

If n is even, XORing all rows gives x ⊕ x ⊕ … = 0. Column-wise, if m is even, the total is also 0. These must agree:

  • n even ⇒ need x = 0 (XOR of n copies of x is 0 only if x = 0)
  • m even ⇒ need y = 0 (same reasoning for columns)
  • n odd and m odd ⇒ need x = y (total XOR from rows must equal total from columns)
Summary of existence:
• n even, m even: possible iff x = y = 0.
• n even, m odd: possible iff x = 0.
• n odd, m even: possible iff y = 0.
• n odd, m odd: possible iff x = y.

Construction Strategy

When the existence condition is satisfied, the following construction works:

  1. Fill the top-left (n−1) × (m−1) submatrix with all zeros.
  2. Set the last column (except bottom-right) so that each of the first n−1 rows XORs to x. Since the rest of the row is 0, just put x in column m−1.
  3. Set the last row (except bottom-right) so that each of the first m−1 columns XORs to y. Since the rest of the column is 0, just put y in row n−1.
  4. The bottom-right corner must satisfy both its row and column constraint. Compute it as the XOR of the rest of row n−1 with x, or equivalently the XOR of the rest of column m−1 with y. These agree precisely when the existence condition holds.
// Construct n x m matrix where each row XORs to x, each column XORs to y.
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, m;
    long long x, y;
    cin >> n >> m >> x >> y;

    // Check existence
    bool ok = true;
    if(n % 2 == 0 && x != 0) ok = false;
    if(m % 2 == 0 && y != 0) ok = false;
    if(n % 2 == 1 && m % 2 == 1 && x != y) ok = false;
    if(!ok){
        cout << "NO\n";
        return 0;
    }

    cout << "YES\n";
    vector<vector<long long>> mat(n, vector<long long>(m, 0));

    // Last column of first n-1 rows = x
    for(int i = 0; i < n - 1; i++)
        mat[i][m - 1] = x;

    // Last row of first m-1 columns = y
    for(int j = 0; j < m - 1; j++)
        mat[n - 1][j] = y;

    // Bottom-right corner: must make last row XOR to x and last col XOR to y.
    // Last row currently: (m-1) copies of y, then corner.
    // XOR of row = y XOR'd (m-1) times, XOR corner = x.
    // If m is odd: (m-1) even copies of y XOR to 0, so corner = x.
    // If m is even: (m-1) odd copies of y XOR to y, so corner = x XOR y.
    //   But m even => y = 0, so corner = x XOR 0 = x.  And n even => x = 0.
    //   So corner = 0.  Consistent.
    long long row_xor = 0;
    for(int j = 0; j < m - 1; j++) row_xor ^= mat[n - 1][j];
    mat[n - 1][m - 1] = row_xor ^ x;

    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            cout << mat[i][j];
            if(j + 1 < m) cout << ' ';
        }
        cout << '\n';
    }
}

XOR Basis Technique

For problems asking “construct a matrix where no two rows have the same XOR over any subset of columns,” build rows from a linear basis over GF(2). Row i gets value 1 << i — these are linearly independent, so all 2n subset-XORs are distinct.

6. Competitive Programming Problems

Let’s solve a few representative problems that tie together the techniques from this post.

Problem A: Matrix with Given Row XOR and Column XOR

Task (Codeforces style): Given arrays R[0..n−1] and C[0..m−1], construct an n × m matrix of non-negative integers where row i XORs to R[i] and column j XORs to C[j]. The XOR of all R[i] must equal the XOR of all C[j] (necessary and sufficient condition).

Construction: Place R[i] in position (i, 0) for the first n−1 rows, and C[j] in position (0, j) for the first m−1 columns, with all other cells as 0. Then fix (0, 0) so both constraints are met. 

// Matrix with prescribed row-XOR and column-XOR.
// R[] = target XOR for each row, C[] = target XOR for each column.
// Condition: XOR(R) == XOR(C).
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, m;
    cin >> n >> m;
    vector<long long> R(n), C(m);
    for(auto &x : R) cin >> x;
    for(auto &x : C) cin >> x;

    long long xr = 0, xc = 0;
    for(auto v : R) xr ^= v;
    for(auto v : C) xc ^= v;
    if(xr != xc){
        cout << "NO\n";
        return 0;
    }
    cout << "YES\n";

    vector<vector<long long>> mat(n, vector<long long>(m, 0));

    // Row 0: place C[j] in each column j for j >= 1
    for(int j = 1; j < m; j++)
        mat[0][j] = C[j];

    // Column 0: place R[i] in each row i for i >= 1
    for(int i = 1; i < n; i++)
        mat[i][0] = R[i];

    // Fix (0,0): row 0 should XOR to R[0].
    // Currently row 0 XOR = C[1] ^ C[2] ^ ... ^ C[m-1].
    // We need mat[0][0] ^ (C[1]^...^C[m-1]) = R[0].
    long long row0_xor = 0;
    for(int j = 1; j < m; j++) row0_xor ^= mat[0][j];
    mat[0][0] = row0_xor ^ R[0];

    // Verify column 0: should XOR to C[0].
    // col0 = mat[0][0] ^ R[1] ^ R[2] ^ ... ^ R[n-1]
    //       = (row0_xor ^ R[0]) ^ (R[1] ^ ... ^ R[n-1])
    //       = row0_xor ^ XOR(R)
    //       = (C[1]^...^C[m-1]) ^ XOR(R)
    //       = (C[1]^...^C[m-1]) ^ XOR(C)  [since XOR(R)=XOR(C)]
    //       = C[0].  Correct!

    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            cout << mat[i][j];
            if(j + 1 < m) cout << ' ';
        }
        cout << '\n';
    }
}

Problem B: Anti-Magic Square

Task: Fill an n × n grid with distinct positive integers 1 to n2 such that all row sums, column sums, and diagonal sums are distinct. This is the opposite of a magic square.

Key observation: Start with any arrangement (e.g., row-major: a[i][j] = i*n + j + 1). The row sums form an arithmetic progression with common difference n2. The column sums also form an arithmetic progression. The two diagonals have specific sums. If any collision occurs, swap two elements in different rows and columns to break ties. For n ≥ 3, a single swap of elements in the top-left 2 × 2 submatrix typically suffices.

General strategy for “construct X such that all Y are distinct”: start with a structured arrangement where the quantities are easy to compute, then apply a small perturbation (one or two swaps) to break any accidental equalities.

Practice Problems

Here are recommended problems to solidify these techniques:

  • Codeforces 1500C — Matrix Filling: Construct a matrix where each row and column is sorted. Greedy diagonal filling.
  • Codeforces 1294E — Obtain a Permutation: Transform a matrix so each column is a permutation. Minimum swaps per column.
  • Codeforces 1333D — Challenges in School: Constructive matrix simulation with bubble-sort-like passes.
  • Codeforces 1391D — 505 (Binary Matrix XOR): Determine if a binary matrix can have all 2×2 submatrices with odd sum.
  • Codeforces 1504C — Balance the Bits: Construct two bracket sequences with constraints resembling row/column balance.
  • AtCoder ABC 265 F — Manhattan Cafe: Matrix DP with row constraints, illustrating how construction ideas guide DP state design.

Matrix construction is one of the most satisfying CP families. The pattern is nearly always the same: identify the structural invariant, then build around it with a simple formula or greedy fill. Practice the problems above to develop intuition for decomposing matrix constraints into manageable pieces.
