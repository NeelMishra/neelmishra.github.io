Interactive Sorting — Minimum Comparisons & Information-Theoretic Bounds

July 2026 · 25 min read

Introduction

Every programmer learns that comparison-based sorting takes Θ(n log n) comparisons. But how many comparisons exactly? In interactive problems the judge limits you to a precise number of queries — sometimes exactly ⌈log2(n!)⌉, the absolute theoretical minimum. Going even one comparison over means Wrong Answer.

We develop the theory from first principles: the information-theoretic lower bound, optimal algorithms for small n, the celebrated Ford-Johnson algorithm, and how these ideas manifest in competitive programming.

Why care about exact counts? In interactive problems the judge acts as an oracle: you ask “Is a[i] < a[j]?” and it answers yes or no. Knowing the precise lower bound tells you whether a given query budget Q is feasible and guides your algorithm design.

Information-Theoretic Lower Bound

A comparison-based sorting algorithm on n elements is modelled as a binary decision tree. Each internal node is a comparison “a[i] < a[j]?”, each leaf is a permutation. Since there are n! permutations, the tree has at least n! leaves, giving height at least:

C(n) ≥ ⌈log2(n!)⌉

By Stirling’s approximation: log2(n!) ≈ n log2 n − 1.4427n. Here are exact values for small n:

Notice merge sort is not always optimal — for n = 5 it uses 8 comparisons while the optimum is 7. This gap motivates the algorithms below. 

#include <cmath>
#include <iostream>
using namespace std;

// Compute ceil(log2(n!)) via direct summation
int log2_nfact(int n) {
    double logfact = 0;
    for (int i = 2; i <= n; i++) logfact += log2((double)i);
    return (int)ceil(logfact);
}

int main() {
    for (int n = 2; n <= 20; n++)
        cout << "n=" << n << " : ceil(log2(n!)) = " << log2_nfact(n) << "\n";
}

Sorting Small Sets Optimally

For small n, hand-crafted comparison networks match the information-theoretic bound. These appear constantly in interactive problems with tight query budgets.

Sorting 5 elements in 7 comparisons

Standard merge sort uses 8 comparisons for n=5. To reach 7, we exploit partial order information:

  1. Compare a1 vs a2 and a3 vs a4 (2 comparisons). WLOG a1 < a2 and a3 < a4.
  2. Compare a2 vs a4 (1 comparison). WLOG a2 < a4. Now: a1 < a2 < a4 and a3 < a4.
  3. Insert a5 into {a1, a2, a4} by binary insertion (2 comparisons).
  4. Insert a3 before a4 — binary search over at most 3 elements preceding a4 (2 comparisons).

Total: 2 + 1 + 2 + 2 = 7 comparisons, matching the bound.

#include <iostream>
#include <functional>
using namespace std;

// Sort 5 elements in at most 7 comparisons (worst case).
void sort5(int idx[5], function<bool(int,int)> less) {
    if (!less(idx[0], idx[1])) swap(idx[0], idx[1]);
    if (!less(idx[2], idx[3])) swap(idx[2], idx[3]);
    if (!less(idx[1], idx[3])) { swap(idx[0], idx[2]); swap(idx[1], idx[3]); }
    // Now: idx[0] < idx[1] < idx[3], idx[2] < idx[3]

    // Insert idx[4] into chain {idx[0], idx[1], idx[3]}
    if (less(idx[4], idx[1])) {
        if (less(idx[4], idx[0])) {
            int t = idx[4]; idx[4] = idx[3]; idx[3] = idx[1]; idx[1] = idx[0]; idx[0] = t;
        } else {
            int t = idx[4]; idx[4] = idx[3]; idx[3] = idx[1]; idx[1] = t;
        }
    } else {
        if (less(idx[4], idx[3])) {
            int t = idx[4]; idx[4] = idx[3]; idx[3] = t;
        }
    }

    // Insert idx[2] into chain before idx[3]
    int e = idx[2];
    if (less(e, idx[1])) {
        if (less(e, idx[0])) { idx[2] = idx[1]; idx[1] = idx[0]; idx[0] = e; }
        else                 { idx[2] = idx[1]; idx[1] = e; }
    }
}

int main() {
    int a[] = {5, 2, 8, 1, 4};
    int idx[] = {0, 1, 2, 3, 4};
    int cnt = 0;
    auto cmp = [&](int i, int j) { cnt++; return a[i] < a[j]; };
    sort5(idx, cmp);
    cout << "Sorted: ";
    for (int i = 0; i < 5; i++) cout << a[idx[i]] << " ";
    cout << "\nComparisons: " << cnt << "\n";
}
Key insight: Build a partial order (DAG of known relations), then use binary insertion to place remaining elements. Binary search over known chains minimises new comparisons.

Merge-Insertion Sort (Ford-Johnson Algorithm)

The Ford-Johnson algorithm (1959) generalises the above to arbitrary n. It achieves ⌈log2(n!)⌉ comparisons for all n ≤ 47 (verified computationally).

Algorithm outline

  1. Pair up. Compare ⌊n/2⌋ pairs. The larger is the “winner,” the smaller is the “loser.”
  2. Recurse on winners. Sort the ⌊n/2⌋ winners. We get chain w1 < … < wm, each wi paired with loser li < wi.
  3. Insert losers in Jacobsthal order. The Jacobsthal numbers (0, 1, 1, 3, 5, 11, 21, 43, …) dictate the insertion order. This ensures each binary search is over a chain of length 2t−1, using exactly t comparisons.
Why Jacobsthal order? When inserting lk, we know lk < wk, so we only search before wk in the chain. The Jacobsthal order makes this search region always a power-of-2-minus-1 in length, so binary search never wastes a comparison on an awkwardly sized region. 
#include <bits/stdc++.h>
using namespace std;

struct Cmp {
    int count = 0;
    vector<int>& arr;
    Cmp(vector<int>& a) : arr(a) {}
    bool operator()(int i, int j) { count++; return arr[i] < arr[j]; }
};

void bin_insert(vector<int>& chain, int elem, int hi, Cmp& cmp) {
    int lo = 0;
    while (lo < hi) {
        int mid = (lo + hi) / 2;
        if (cmp(elem, chain[mid])) hi = mid; else lo = mid + 1;
    }
    chain.insert(chain.begin() + lo, elem);
}

vector<int> jacobsthal_seq(int n) {
    vector<int> J = {0, 1};
    while (J.back() < n) J.push_back(J[J.size()-1] + 2*J[J.size()-2]);
    return J;
}

vector<int> ford_johnson(vector<int> idx, Cmp& cmp) {
    int n = idx.size();
    if (n <= 1) return idx;

    // Pair up and compare
    vector<pair<int,int>> pairs;
    int extra = -1;
    for (int i = 0; i+1 < n; i += 2) {
        if (cmp(idx[i], idx[i+1])) pairs.push_back({idx[i+1], idx[i]});
        else                        pairs.push_back({idx[i], idx[i+1]});
    }
    if (n & 1) extra = idx[n-1];

    // Recursively sort winners
    vector<int> winners;
    for (auto& p : pairs) winners.push_back(p.first);
    auto sw = ford_johnson(winners, cmp);

    map<int,int> w2l;
    for (auto& p : pairs) w2l[p.first] = p.second;

    // Build chain: l1 goes before w1
    vector<int> chain = {w2l[sw[0]]};
    for (int w : sw) chain.push_back(w);

    // Remaining losers + extra
    vector<int> losers;
    for (int i = 1; i < (int)sw.size(); i++) losers.push_back(w2l[sw[i]]);
    if (extra != -1) losers.push_back(extra);

    // Insert in Jacobsthal order
    auto J = jacobsthal_seq(losers.size() + 2);
    vector<bool> done(losers.size(), false);
    for (int k = 2; k < (int)J.size(); k++) {
        int hi = min((int)losers.size(), J[k]);
        for (int idx2 = hi; idx2 > J[k-1]; idx2--) {
            int i = idx2 - 1;
            if (i >= (int)losers.size() || done[i]) continue;
            done[i] = true;
            int ub = (int)chain.size();
            if (i < (int)sw.size()-1) {
                for (int p = 0; p < (int)chain.size(); p++)
                    if (chain[p] == sw[i+1]) { ub = p; break; }
            }
            bin_insert(chain, losers[i], ub, cmp);
        }
        if (hi >= (int)losers.size()) break;
    }
    return chain;
}

int main() {
    vector<int> arr = {38, 27, 43, 3, 9, 82, 10};
    int n = arr.size();
    vector<int> indices(n);
    iota(indices.begin(), indices.end(), 0);
    Cmp cmp(arr);
    auto res = ford_johnson(indices, cmp);
    cout << "Sorted: ";
    for (int i : res) cout << arr[i] << " ";
    double lg = 0;
    for (int i = 2; i <= n; i++) lg += log2((double)i);
    cout << "\nComparisons: " << cmp.count
         << "  (lower bound: " << (int)ceil(lg) << ")\n";
}

Selection & Median Finding

A related class of problems: find the k-th smallest with fewest comparisons.

  • Minimum: exactly n−1 comparisons (each non-min must lose once).
  • Min and max together: ⌈3n/2⌉−2 comparisons — process pairs, compare smaller with running min and larger with running max.
  • Second smallest: n + ⌈log2 n⌉ − 2 comparisons — run a tournament to find the min (n−1), then find the minimum among the ⌈log2 n⌉ elements that lost directly to it.
  • Median: at least 3(n−1)/2 comparisons by an adversary argument. Practically, nth_element gives O(n).

Tournament Trees & Selection

A tournament tree is a complete binary tree where each node stores the winner (smaller element) of its children. Building costs n−1 comparisons. The second smallest must have lost to the minimum at some round — there are ⌈log2 n⌉ such candidates. 

#include <bits/stdc++.h>
using namespace std;

struct Tournament {
    int n;
    vector<int> tree;
    vector<vector<int>> lost_to;
    int cmp_count = 0;
    function<bool(int,int)> cmp;

    Tournament(int n_, function<bool(int,int)> cmp_) : n(n_), cmp(cmp_) {
        int sz = 1;
        while (sz < n) sz *= 2;
        tree.assign(2*sz, -1);
        lost_to.resize(n);
        for (int i = 0; i < n; i++) tree[sz+i] = i;
        for (int i = sz-1; i >= 1; i--) {
            int l = tree[2*i], r = tree[2*i+1];
            if (l == -1) { tree[i] = r; continue; }
            if (r == -1) { tree[i] = l; continue; }
            cmp_count++;
            if (cmp(l, r)) { tree[i] = l; lost_to[l].push_back(r); }
            else            { tree[i] = r; lost_to[r].push_back(l); }
        }
    }

    int find_second() {
        int w = tree[1];
        auto& L = lost_to[w];
        if (L.empty()) return -1;
        int best = L[0];
        for (int i = 1; i < (int)L.size(); i++) {
            cmp_count++;
            if (cmp(L[i], best)) best = L[i];
        }
        return best;
    }
};

int main() {
    vector<int> a = {5, 3, 8, 1, 9, 2, 7, 4, 6};
    int n = a.size();
    auto cmp = [&](int i, int j) { return a[i] < a[j]; };
    Tournament t(n, cmp);
    cout << "Min: " << a[t.tree[1]]
         << "\nSecond: " << a[t.find_second()]
         << "\nComparisons: " << t.cmp_count
         << "  (optimal: " << n + (int)ceil(log2(n)) - 2 << ")\n";
}

Partial order tracking

Maintain a DAG of known relations. Before asking a query, check if the answer is already implied by transitivity — this saves precious queries. 

struct PartialOrder {
    int n;
    vector<vector<bool>> reach; // reach[i][j] = true means i < j

    PartialOrder(int n_) : n(n_), reach(n_, vector<bool>(n_, false)) {}

    void add_less(int a, int b) {
        if (reach[a][b]) return;
        for (int x = 0; x < n; x++)
            if (x == a || reach[x][a])
                for (int y = 0; y < n; y++)
                    if (y == b || reach[b][y])
                        reach[x][y] = true;
    }

    // 1 if a<b known, -1 if a>b known, 0 if unknown
    int query(int a, int b) {
        if (reach[a][b]) return 1;
        if (reach[b][a]) return -1;
        return 0;
    }
};
Complexity note: Transitive closure is O(n²) per update. Fine for interactive problems where n ≤ 1000 and the bottleneck is query count, not CPU time.

Strategy guide

Sort n ≤ 20 optimally

Use Ford-Johnson. Budget is exactly ⌈log2(n!)⌉. For n=20: 62 queries.

Find median of n elements

Tournament + median-of-medians. Budget typically O(n) or ⌈3n/2⌉−2.

Sort with lies (noisy comparator)

Compare each pair O(log n) times, take majority. Then sort normally.

Merge k sorted runs

k-way tournament tree: O(n log k) comparisons total.

Tips for interactive sorting problems

  • Flush output. Always cout << endl or cout.flush() after each query. Forgetting this is the #1 cause of TLE on interactive problems.
  • Count queries. Add a counter and assert it stays within budget during local testing.
  • Exploit transitivity. If a < b and b < c, never query a vs c.
  • Test locally. Write an interactor that simulates the judge with adversarial inputs maximising query count.
  • Stirling check: Q ≈ n log2 n − 1.44n gives a quick sanity check on the budget.
Summary: ⌈log2(n!)⌉ is the fundamental lower bound for comparison-based sorting. Ford-Johnson’s merge-insertion sort achieves this for practical n. Tournament trees solve selection problems optimally. In interactive problems, combine these techniques with partial order tracking to stay within tight query budgets.
