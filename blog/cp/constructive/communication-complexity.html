Communication Complexity — Lower Bounds & Interactive Query Strategies

July 2026

Introduction

Interactive problems on Codeforces and similar platforms ask you to discover a hidden object by querying an oracle, with a strict limit on the number of queries. To solve these problems you need two skills: designing a strategy that uses few queries, and proving that no strategy can do better. The second skill draws on communication complexity and information theory — the mathematical study of how much information must be exchanged to compute a function.

In this post we build the theory from the ground up: entropy-based counting, decision-tree lower bounds, the Alice-Bob communication model, fooling sets, and rectangle techniques. Along the way we solve three full interactive problems in C++, proving each query count is optimal.

Prerequisites. Basic probability, logarithms, familiarity with interactive-problem I/O on Codeforces (flushing cout), and comfort reading C++17.

Information-Theoretic Lower Bounds

The most elementary lower-bound argument comes straight from Shannon’s information theory. If the hidden answer can be any one of N equally likely possibilities and each query returns one of k possible answers, then the number of queries q must satisfy:

kq ≥ N   ⇒   q ≥ ⌈logk N⌉

This is the entropy argument. Each query produces at most log2 k bits of information; you need at least log2 N bits total, so you need at least ⌈log2 N / log2 k⌉ queries.

Example: Guessing a Number

Classic problem: find a hidden integer x ∈ [1, n] using yes/no queries of the form “Is x ≤ m?”. Here N = n and k = 2, so we need at least ⌈log2 n⌉ queries. Binary search matches this bound exactly, so it is optimal.

Entropy with Non-Uniform Distributions

When the prior distribution over answers is not uniform, we use Shannon entropy. If the answer is drawn from distribution p, the expected number of binary queries is at least:

H(p) = −∑i pi log2 pi

In competitive programming the hidden answer is usually adversarial, so we use the uniform bound log2 N. However, entropy reasoning becomes essential in expected-query problems and randomized strategies.

Ternary and Multi-Valued Responses

Some interactive problems let the judge reply with more than two values. A “compare” oracle returning { <, =, > } gives three outcomes per query. Finding one element among N then requires at least ⌈log3 N⌉ queries.

Key takeaway. Always start a lower-bound analysis by asking: “How many distinguishable outcomes exist, and how many bits does each query provide?” This gives an immediate, often tight, lower bound.

Complete Example: Binary Search Interactive

Here is a complete C++ solution for the standard “guess the number” interactive problem, with exactly ⌈log2 n⌉ queries — matching the information-theoretic lower bound. 

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int lo = 1, hi = n;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        cout << "? " << mid << endl;   // query: is x <= mid?

        string resp;
        cin >> resp;

        if (resp == "YES" || resp == "<=") {
            hi = mid;
        } else {
            lo = mid + 1;
        }
    }
    cout << "! " << lo << endl;
    return 0;
}

Query count: exactly ⌈log2 n⌉. Lower bound: ⌈log2 n⌉ (entropy argument). The strategy is optimal.

Decision Tree Lower Bounds

A decision tree is a rooted tree where each internal node is a query, each edge a response, and each leaf an output. The worst-case query count equals the tree’s depth. To prove a lower bound of d, show every valid decision tree has depth ≥ d.

Adversary Argument

The adversary method is a powerful technique: instead of analyzing a specific algorithm, we design an adversary — a strategy for answering queries — that forces any algorithm to use many queries.

The adversary maintains a set S of inputs consistent with all answers so far. On each query, it answers to keep |S| as large as possible. If after q queries |S| ≥ 2 always, then the lower bound is q + 1.

Example: Finding the Maximum

Given an array of n distinct elements, find the maximum using comparisons. The entropy bound gives ⌈log2 n⌉, but the true lower bound is n − 1: every non-maximum element must lose at least one comparison, and each comparison produces at most one loser.

When entropy is not enough. The find-maximum example shows that information-theoretic bounds can be loose. Decision-tree and adversary arguments capture structural constraints that entropy misses.

Tournament Trees and Tight Bounds

A tournament tree organizes n elements into a single-elimination bracket with n − 1 internal nodes (comparisons). For finding both the max and the second max, the lower bound is n + ⌈log2 n⌉ − 2, achieved by the tournament method.

Communication Complexity Basics

Communication complexity, introduced by Andrew Yao in 1979: Alice holds input x, Bob holds input y, and they want to compute f(x, y) by exchanging bits. The communication complexity D(f) is the minimum worst-case number of bits exchanged by the best deterministic protocol.

A deterministic protocol forms a binary tree. At each internal node, one player sends a bit. The set of input pairs reaching any node forms a combinatorial rectangle Av × Bv — this structural property is the foundation of all communication lower-bound techniques.

Connecting to Interactive Problems

An interactive competitive-programming problem maps naturally to this framework. “Alice” is the judge (who knows the hidden answer), and “Bob” is your program. Each query you send is a message from Bob to Alice; each response is a message from Alice to Bob.

Equality: A Foundational Example

The EQUALITY function: Alice has an n-bit string x, Bob has an n-bit string y, decide if x = y. Deterministically, D(EQ) = n — Bob must effectively learn x to compare.

The rectangle argument formalizes this: any 1-rectangle containing (x, x) must be {x} × {x}, so every n-bit string requires its own leaf — 2n leaves, requiring depth ≥ n.

Fooling Sets & Rectangle Arguments

Two of the most widely used techniques for proving communication lower bounds are fooling sets and monochromatic rectangle arguments.

Fooling Sets

A fooling set for a function f with target value z is a set of input pairs F = {(x1, y1), …, (xm, ym)} such that:

  1. f(xi, yi) = z for all i.
  2. For all i ≠ j, either f(xi, yj) ≠ z or f(xj, yi) ≠ z (or both).

Theorem. If f has a fooling set of size m, then D(f) ≥ ⌈log2 m⌉.

Proof sketch. In any protocol, each leaf is a monochromatic rectangle. A z-monochromatic rectangle can contain at most one element of the fooling set (otherwise a rectangle containing (xi, yi) and (xj, yj) also contains (xi, yj), contradicting condition 2). So the protocol needs ≥ m leaves labelled z, hence depth ≥ ⌈log2 m⌉.

Monochromatic Rectangle Bound

More generally, let Cz(f) be the minimum number of z-monochromatic rectangles needed to cover all (x, y) with f(x, y) = z (a “z-cover”). Then:

D(f) ≥ ⌈log2 Cz(f)⌉

The fooling-set bound is a corollary: if a fooling set has m elements, then Cz(f) ≥ m.

Example: Disjointness

The DISJOINTNESS function: Alice and Bob each hold a subset of {1, …, n}; they must decide if the sets are disjoint. A simple fooling set gives a log n bound, but the true bound D(DISJ) = Ω(n) requires heavier machinery (the rank method or information-complexity arguments).

Technique hierarchy. In order of strength (and difficulty):
1. Entropy / counting: ⌈log2 N⌉
2. Fooling set: ⌈log2 |F|⌉
3. Rectangle cover / partition number
4. Rank method (over reals or finite fields)
5. Information complexity / corruption bounds
For CP, levels 1–3 cover almost every problem you will encounter.

Applying Fooling Sets to Interactive Problems

For an interactive problem with N hidden configurations, build a fooling set of size m where any two configurations differ on some query’s “cross term.” This yields a ⌈log2 m⌉ query lower bound.

Query vs Communication Complexity

In competitive programming, we usually work in the query model rather than the full communication model. The key difference:

  • Query complexity: one party (your program) asks questions; the other (the judge) answers. Messages are structured — each query has a fixed format and each answer has a fixed number of possible values.
  • Communication complexity: both parties send arbitrary bit strings. The content of each message can depend on the full transcript.

Query complexity is a special case of communication complexity where one party asks structured questions and the other answers. A query lower bound of q implies a communication lower bound of q, but not vice versa.

Adaptive vs Non-Adaptive Queries

A strategy is non-adaptive if all queries are decided in advance; adaptive if later queries depend on earlier answers. Adaptive is at least as powerful — binary search needs ⌈log2 n⌉ adaptive queries vs n non-adaptive.

Complete Example: Finding Two Secret Numbers

Problem. The judge has two distinct secret numbers a, b ∈ [1, n] with a < b. In each query you give a number m, and the judge tells you the count of secrets that are ≤ m (possible replies: 0, 1, or 2). Find both a and b using at most ⌈log3 n⌉ + ⌈log2 n⌉ queries.

Lower bound. C(n, 2) possible pairs, ternary responses ⇒ q ≥ ⌈log3(n(n−1)/2)⌉ ≈ 2⌈log3 n⌉. Strategy: ternary search to locate a, then binary search for b. 

#include <bits/stdc++.h>
using namespace std;

int query(int m) {
    cout << "? " << m << endl;
    int resp;
    cin >> resp;
    return resp;   // 0, 1, or 2
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // Phase 1: ternary search for a (smallest secret)
    // Find smallest m such that query(m) >= 1
    int lo = 1, hi = n;
    while (lo < hi) {
        int third = (hi - lo) / 3;
        int m1 = lo + third;
        int m2 = hi - third;

        int r1 = query(m1);
        if (r1 >= 1) {
            // a <= m1
            hi = m1;
        } else {
            int r2 = query(m2);
            if (r2 >= 1) {
                lo = m1 + 1;
                hi = m2;
            } else {
                lo = m2 + 1;
            }
        }
    }
    int a = lo;

    // Phase 2: binary search for b in [a+1, n]
    lo = a + 1;
    hi = n;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        int r = query(mid);
        if (r == 2) {
            hi = mid;
        } else {
            lo = mid + 1;
        }
    }
    int b = lo;

    cout << "! " << a << " " << b << endl;
    return 0;
}

The total query count is at most ⌈log3 n⌉ + ⌈log2 n⌉, close to the information-theoretic optimum.

Randomized Communication Complexity

Deterministic communication complexity can be much larger than its randomized counterpart. The randomized communication complexity R(f) allows the protocol to use random coins and must compute f correctly with probability ≥ 2/3 for every input.

The most dramatic separation is EQUALITY: D(EQ) = n, but R(EQ) = O(log n). Alice picks a random prime p and sends x mod p. Bob checks if x mod p = y mod p. By a number-theoretic argument the error is O(1/n), well below 1/3.

Fingerprinting Technique

Compress a large input into a small hash that preserves equality with high probability. In CP, randomized interactive problems often allow small error and fingerprinting is the go-to approach.

Yao’s Minimax Principle

Yao’s Minimax Theorem connects randomized and distributional complexity: R(f) = maxμ Dμ(f), where Dμ(f) is the cost of the best deterministic protocol that errs on at most 1/3 of inputs under distribution μ. To prove a randomized lower bound, find a hard distribution and show every deterministic protocol fails on a constant fraction.

Yao’s principle in CP. When a problem says “the judge is adaptive” (adversarial), you cannot exploit randomization to beat the deterministic bound. When the judge is oblivious (fixes the answer before your queries), randomization can help exponentially, as the EQUALITY example shows.

Public vs Private Coins

In the public-coin model both parties share randomness; in the private-coin model each has its own. Newman’s theorem shows the gap is at most O(log n). In CP you generate your own seed, so the distinction rarely matters.

Competitive Programming Applications

Let’s apply our theory to realistic competitive-programming interactive problems.

Problem 1: Guess the Permutation Element

Problem. The judge has a hidden permutation P of [1, n]. You can query “? i j” and the judge answers which of P[i] and P[j] is larger (returns 1 if P[i] > P[j], else 2). Find the index of the maximum element in at most n − 1 queries.

Lower bound (adversary argument). Every index except the winner must lose at least one comparison. Each query eliminates at most one index. Therefore n − 1 queries are necessary. This matches the tournament bound from our decision-tree section. 

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int best = 1;
    for (int i = 2; i <= n; i++) {
        cout << "? " << best << " " << i << endl;
        int resp;
        cin >> resp;
        if (resp == 2) {
            // P[i] > P[best]
            best = i;
        }
    }
    cout << "! " << best << endl;
    return 0;
}

Exactly n − 1 queries, matching the lower bound. Optimal.

Problem 2: Find a Threshold in a Sorted Binary Array

Problem. A hidden array A[1..n] is monotone non-decreasing (0s then 1s). Find the first index i where A[i] = 1, or report none exists. Each query “? i” returns A[i]. Limit: ⌈log2(n + 1)⌉ queries.

Lower bound. N + 1 possible answers, binary queries ⇒ ≥ ⌈log2(n + 1)⌉ queries. Binary search matches: 

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // lo-1 is guaranteed 0, hi+1 is guaranteed 1
    int lo = 1, hi = n;
    int ans = -1;  // -1 means all zeros

    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        cout << "? " << mid << endl;
        int val;
        cin >> val;

        if (val == 1) {
            ans = mid;
            hi = mid - 1;
        } else {
            lo = mid + 1;
        }
    }

    if (ans == -1) {
        cout << "! -1" << endl;
    } else {
        cout << "! " << ans << endl;
    }
    return 0;
}

The loop runs exactly ⌈log2(n + 1)⌉ times, matching the lower bound.

Problem 3: Identify a Hidden Graph Edge (Fooling-Set Lower Bound)

Problem. The judge has a hidden edge (u, v) in the complete graph Kn. You query a vertex w, and the judge tells you the distance from w to the closest endpoint of the edge: min(dist(w, u), dist(w, v)), where dist(a, b) = |a − b| (vertices labelled 1 to n). Find the hidden edge.

Fooling-set lower bound. Consider edges F = {(i, i+1) : i = 1,…,n−1}. For i ≠ j, querying vertex i gives distance 0 for (i,i+1) but positive for (j,j+1). This fooling set of size n − 1 gives ≥ ⌈log2(n − 1)⌉ queries.

The total candidate count C(n,2) gives an entropy bound of ⌈log2 C(n,2)⌉ ≈ 2 log2 n. We match this with a two-phase binary search: 

#include <bits/stdc++.h>
using namespace std;

int query(int w) {
    cout << "? " << w << endl;
    int resp;
    cin >> resp;
    return resp;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // Phase 1: binary search for u (left endpoint).
    // As w goes from 1 toward u, query(w) = u - w decreases to 0.
    int lo = 1, hi = n;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        int r = query(mid);
        if (r == 0) {
            hi = mid;
        } else {
            lo = mid + 1;
        }
    }
    int u = lo;

    // Phase 2: binary search for v in [u+1, n].
    lo = u + 1; hi = n;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        int r = query(mid);
        if (r == 0) {
            hi = mid;
        } else if (r == mid - u) {
            lo = mid + 1;   // closer to u, so v is further right
        } else {
            hi = mid;
        }
    }
    int v = lo;

    cout << "! " << u << " " << v << endl;
    return 0;
}

Each phase uses ⌈log2 n⌉ queries, for a total of 2⌈log2 n⌉ — matching the entropy bound ⌈log2 C(n,2)⌉ up to a small constant.

Summary of Proof Techniques for CP

When you solve an interactive problem, prove your solution is optimal using this checklist:

  1. Count outcomes. If there are N possible hidden objects and each query has k outcomes, you need ≥ ⌈logk N⌉ queries.
  2. Adversary argument. Design a strategy for the judge that forces your algorithm to use many queries. Show that after q queries, at least 2 candidates remain consistent.
  3. Fooling set. Find a set of m inputs where any two are “confused” by at least one cross-term. This gives a ⌈log2 m⌉ bound.
  4. Reduction. Reduce a known-hard communication problem (EQUALITY, DISJOINTNESS) to your problem. The lower bound carries over.
Practical tip. On Codeforces, if the query limit is exactly ⌈log2 n⌉ or 2n or n − 1, that is a strong hint about which lower-bound argument applies. Match the limit to the technique: logarithmic → entropy/binary search; linear → adversary/tournament; sub-linear → randomized fingerprinting.

Practice Problems

Test your understanding of communication complexity ideas with these problems, roughly ordered by difficulty:

  1. CF 1503B — Guess the Permutation: find a swap in a permutation with limited queries. Apply the entropy argument to justify query count.
  2. CF 1698D — Guess the Maximums: find maximums of subsets interactively. Use binary search with a careful adversary analysis.
  3. CF 1762D — GCD Queries: determine two special positions using GCD queries. The fooling-set technique applies directly.
  4. CF 1464C — Poman forlife: an adaptive problem where the judge responds adversarially. Randomized techniques can beat the deterministic lower bound here.
  5. CF 843B — Interactive LowerBound: O(√n) random queries suffice (birthday paradox meets communication complexity).
  6. CF 1675F — Vlad and Unfinished Business: reconstruct a tree with limited queries using encoding arguments.
When practicing. For each problem, write down: (1) how many possible answers exist, (2) how many bits each query provides, (3) the entropy lower bound. Compare to the problem’s query limit to guide your approach.
