    K-Armed Bandits & Action Values

    June 2026 · 14 min read · Coursera: Fundamentals of Reinforcement Learning (Univ. of Alberta), Week 1

    These are my worked-up notes from Week 1 of the University of Alberta's Fundamentals of Reinforcement Learning course (the first course in the RL Specialization). Bandits are the simplest setting that still captures the central tension of RL: learning purely from evaluative feedback, where you only find out how good the action you took was, never what the best action would have been.

    Sequential decision making with evaluative feedback

    In supervised learning the environment tells you the correct label: the feedback is instructive, it does not depend on the action you took. Reinforcement learning is different. The agent generates all of its own training data by interacting with the world, and it learns the consequences of its choices through trial and error. The feedback is evaluative: it tells you how good your action was, but not whether it was the best one available.

    The running example from the course is a doctor choosing treatments. A doctor (the agent) can prescribe one of several treatments to a patient. Each treatment yields an outcome, mapped to a numeric reward (patient gets worse, no change, or recovers). The doctor only observes the result of the treatment they actually gave, never the counterfactual outcome of the treatments they withheld.

    The k-armed bandit problem

    In a k-armed bandit, an agent repeatedly chooses between $k$ different actions and receives a reward based on the action it chose. The name comes from a row of slot machines ("one-armed bandits"): each arm pays out according to its own unknown probability distribution, and you want to find the arm that pays best.

    Formally, on each time step $t$ the agent selects an action $A_t \in \{1, 2, \dots, k\}$ and receives a numeric reward $R_t$. The rewards for each action are drawn from a fixed (but unknown) probability distribution. The doctor's treatments, an ad to display, a move in a simplified game: all of these are bandit arms.

    Action values

    We define the value of selecting an action as the expected reward we receive when we take that action. This is the action-value function $q_*$:

    $$ q_*(a) \;\doteq\; \mathbb{E}[\,R_t \mid A_t = a\,], \qquad \forall\, a \in \{1, \dots, k\} $$

    The symbol $\doteq$ means "is defined as". Reading it piece by piece: $q_*(a)$ is the expected reward given that we picked action $a$, and it is defined for every one of the $k$ possible actions. Expanding the expectation for a discrete reward gives a sum over possible reward values weighted by their probabilities:

    $$ q_*(a) = \sum_{r} p(r \mid a)\, r $$

    where $p(r \mid a)$ is the probability of observing reward $r$ after taking action $a$. In the continuous-reward case the sum becomes an integral, $q_*(a) = \int p(r \mid a)\, r \, dr$, but the idea is identical: average every reward the action can produce, weighted by how likely it is.

    The goal: maximize expected reward

    The goal of the agent is to choose the action that maximizes the expected reward. If we knew the true values $q_*(a)$, the problem would be trivial: always pick the action with the highest value. We call that the optimal action $a_*$:

    $$ a_* = \arg\max_{a} \, q_*(a) $$

    The catch, of course, is that we do not know $q_*$. The entire problem reduces to estimating these values well enough, and acting on those estimates, while we learn.

    Learning action values

    Because the true value $q_*(a)$ is unknown, we estimate it. Call the estimate at time $t$ $Q_t(a)$. The most natural estimate is the sample-average method: the value of an action is just the mean of the rewards we have actually received from it so far.

    $$ Q_t(a) \;=\; \frac{\text{sum of rewards when } a \text{ taken prior to } t}{\text{number of times } a \text{ taken prior to } t} \;=\; \frac{\sum_{i=1}^{t-1} R_i \cdot \mathbb{1}[A_i = a]}{\sum_{i=1}^{t-1} \mathbb{1}[A_i = a]} $$

    Here $\mathbb{1}[A_i = a]$ is the indicator function: it is $1$ when the action taken on step $i$ was $a$, and $0$ otherwise. So the numerator adds up only the rewards that came from action $a$, and the denominator counts only the steps on which $a$ was chosen. By the law of large numbers, as an action is taken more and more times, $Q_t(a) \to q_*(a)$.

    Greedy actions, exploration and exploitation

    Once we have estimates, how do we act? If we choose the action with the largest estimated value, that action is called the greedy action:

    $$ A_t = \arg\max_{a} \, Q_t(a) $$

    Selecting the greedy action means the agent is exploiting its current knowledge: it is cashing in on what it already believes to be best. But the agent may instead choose a non-greedy action. When it does, it is exploring: sacrificing immediate reward in order to gain more information about the other actions, which might turn out to be better than its current estimates suggest.

    The exploration–exploitation dilemma. You cannot do both on the same step. Exploit too much and you may get stuck on a mediocre action whose early rewards happened to look good. Explore too much and you waste reward sampling actions you already know are bad. Every bandit algorithm is some recipe for balancing the two.

    The simplest balance is $\varepsilon$-greedy: with probability $1 - \varepsilon$ take the greedy action, and with small probability $\varepsilon$ pick a random action instead. That tiny bit of forced exploration guarantees every action keeps getting sampled, so the estimates for all arms keep improving.

    Estimating action values incrementally

    The sample-average formula as written requires storing every past reward and re-summing them on every step. That is wasteful. We can transform the equation into an incremental update that needs only the current estimate and a counter. Start from the average of the first $n$ rewards for an action and peel off the most recent one:

    $$ \begin{aligned} Q_{n+1} &= \frac{1}{n} \sum_{i=1}^{n} R_i \\[4pt] &= \frac{1}{n}\left( R_n + \sum_{i=1}^{n-1} R_i \right) \\[4pt] &= \frac{1}{n}\left( R_n + (n-1)\, Q_n \right) \\[4pt] &= \frac{1}{n}\left( R_n + n\,Q_n - Q_n \right) \\[4pt] &= Q_n + \frac{1}{n}\left( R_n - Q_n \right) \end{aligned} $$

    The third line uses the fact that $\sum_{i=1}^{n-1} R_i = (n-1) Q_n$, since $Q_n$ is by definition the average of the first $n-1$ rewards. The result is one of the most important update rules in all of RL:

    $$ \boxed{\,Q_{n+1} = Q_n + \frac{1}{n}\left( R_n - Q_n \right)\,} $$

    This has a beautiful and very general shape:

    $$ \text{NewEstimate} \leftarrow \text{OldEstimate} + \text{StepSize} \times \big(\text{Target} - \text{OldEstimate}\big) $$

    The term $\big(R_n - Q_n\big)$ is the error in the estimate: how far the latest reward (the target) was from what we predicted. We nudge the estimate a fraction of the way toward the target, and that fraction is the step size. For the plain sample average the step size is $\tfrac{1}{n}$, which shrinks as we collect more data, so each new reward matters less and the estimate settles down.

    Non-stationary bandit problems

    The shrinking $\tfrac{1}{n}$ step size is exactly what you want when the true values are fixed. But many real problems are non-stationary: the reward distributions drift over time (a patient's physiology changes, user tastes shift, a market moves). If we keep using $\tfrac{1}{n}$, the estimate becomes so sluggish that it can never track the moving target. The fix is to replace the decaying step size with a small constant step size $\alpha$:

    $$ Q_{n+1} = Q_n + \alpha \left( R_n - Q_n \right), \qquad \alpha \in (0, 1] $$

    With a constant $\alpha$, recent rewards are weighted more heavily than old ones. Expanding the recursion shows it is an exponentially weighted moving average: the influence of a reward received $k$ steps ago decays like $(1-\alpha)^k$. This keeps the estimate responsive, which is precisely what a drifting environment demands. The trade-off is that the estimate never fully stops fluctuating, since the most recent reward always retains weight $\alpha$.

    Rule of thumb. Use the sample-average ($\tfrac{1}{n}$) step size when the world is stationary and you want estimates that converge. Use a constant $\alpha$ step size when the world changes over time and you need to keep tracking it.

    Key takeaways

    • Evaluative feedback is the heart of RL: you learn only from the reward of the action you took, never the counterfactual.
    • A k-armed bandit repeatedly picks one of $k$ actions; each has an unknown reward distribution.
    • The action value $q_*(a) = \mathbb{E}[R_t \mid A_t = a]$ is the expected reward of an action; the optimal action is $a_* = \arg\max_a q_*(a)$.
    • We estimate values with the sample average, and act greedily to exploit or non-greedily to explore; $\varepsilon$-greedy is the simplest balance.
    • The incremental update $Q_{n+1} = Q_n + \tfrac{1}{n}(R_n - Q_n)$ generalises to $\text{New} \leftarrow \text{Old} + \text{StepSize}\,(\text{Target} - \text{Old})$, a pattern you will see again and again.
    • For non-stationary problems, swap $\tfrac{1}{n}$ for a constant step size $\alpha$ to keep tracking the moving target.
