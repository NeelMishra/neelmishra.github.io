    The Encoder Block

    ML Series · March 2026

    Anatomy of One Encoder Block

    Each encoder block in the Transformer performs two main operations on its input: multi-head self-attention and a position-wise feed-forward network. Each operation is wrapped in a residual connection and followed by layer normalization. The block takes a tensor of shape [batch, seq_len, d_model] and produces an output of the same shape.

    The original Transformer stacks N=6 of these blocks. The output of one block becomes the input to the next. No parameters are shared between blocks. Each block has its own attention weights and feed-forward weights.

    Data Flow Through a Single Encoder Block

    The input tensor X enters from the bottom. It is projected three times to produce queries, keys, and values. These feed into multi-head attention, whose output is added back to the original input (residual connection) and normalized. The result passes through the feed-forward network, and again a residual connection and normalization are applied. The output has the same shape as the input.

    Multi-Head Self-Attention

    In self-attention, every position in the sequence attends to every other position. The term "self" means that the queries, keys, and values all come from the same input (as opposed to cross-attention, where keys and values come from a different source).

    The computation for a single attention head:

    # For one head (head i): Q_i = X @ W_Q_i # [batch, seq, d_k] where d_k = d_model / h = 64 K_i = X @ W_K_i # [batch, seq, d_k] V_i = X @ W_V_i # [batch, seq, d_v] (d_v = d_k = 64) scores = Q_i @ K_i.T / sqrt(d_k) # [batch, seq, seq] weights = softmax(scores, dim=-1) # [batch, seq, seq] head_i = weights @ V_i # [batch, seq, d_v]

    With 8 heads, we get 8 separate [batch, seq, 64] outputs. These are concatenated along the last dimension to produce [batch, seq, 512], then projected through a final weight matrix W_O of shape [512, 512].

    # Multi-head combination: multi_head = concat(head_1, head_2, ..., head_8) # [batch, seq, 512] output = multi_head @ W_O # [batch, seq, 512]
    Why multiple heads? A single attention head can only learn one type of relationship between positions. Multiple heads allow the model to simultaneously attend to different aspects: one head might track syntactic dependencies, another might capture coreference, a third might focus on adjacent tokens. The concatenation and projection lets the model combine these perspectives.

    Residual Connections: The Gradient Highway

    Every sub-layer in the encoder (attention and FFN) is wrapped in a residual connection:

    output = LayerNorm(x + SubLayer(x))

    The identity path (x) provides a direct route for gradients during backpropagation. Without it, gradients must flow through the attention computation and the feed-forward network, where they risk vanishing or exploding. The residual connection ensures that at minimum, the gradient of the identity function (which is 1) is always present.

    This idea comes from ResNet (He et al., 2015). The insight is the same: identity mappings make deep networks trainable. A 6-layer Transformer encoder without residual connections would be extremely difficult to train. With them, gradients flow freely through all 6 blocks.

    Layer Normalization

    After each residual addition, the result is passed through layer normalization. Unlike batch normalization (which normalizes across the batch dimension), layer norm normalizes across the feature dimension for each individual example and position:

    # For a single position's vector x of length d_model: mean = sum(x) / d_model var = sum((x - mean)^2) / d_model x_norm = (x - mean) / sqrt(var + epsilon) output = gamma * x_norm + beta # learnable scale and shift

    Layer norm is preferred over batch norm in Transformers because it works identically regardless of batch size and does not require running statistics. Each position is normalized independently, which matches the position-wise nature of the architecture.

    Pre-norm vs post-norm: The original paper uses post-norm (normalize after the residual addition). Many later models (GPT-2, LLaMA) use pre-norm (normalize before the sublayer). Pre-norm tends to be more stable during training, especially for very deep models, because the residual stream is not modified by the normalization.

    The Feed-Forward Network

    The second sub-layer in each encoder block is a simple two-layer fully connected network. It is applied to each position independently and identically:

    FFN(x) = Linear(ReLU(Linear(x))) # Dimensions: # x: [batch, seq, 512] # Linear_1: [512, 2048] (expand) # ReLU: element-wise # Linear_2: [2048, 512] (project back) # output: [batch, seq, 512]

    The expansion factor is typically 4x. The inner dimension of 2048 gives the network more capacity to learn complex transformations, while the outer dimension stays at d_model to maintain the residual stream width.

    What does the FFN actually do?

    Recent research suggests the FFN layers act as key-value memories. The first linear layer's rows act as "keys" that match input patterns. The second linear layer's columns act as "values" that are retrieved when a key matches. The ReLU activation provides sparsity: most neurons are inactive for any given input, so only a small subset of "memories" are retrieved.

    In the original paper, ReLU is used as the activation. Many modern architectures have switched to GELU (Gaussian Error Linear Unit) or SwiGLU, which provide smoother gradients and slightly better performance.

    Parameter count

    The FFN contains the majority of parameters in each encoder block:

    • Attention: 4 weight matrices of [512, 512] = 4 × 262,144 = 1,048,576 parameters
    • FFN: [512, 2048] + [2048, 512] = 2 × 1,048,576 = 2,097,152 parameters

    The FFN has exactly twice as many parameters as the attention sub-layer. This ratio is consistent across most Transformer variants.

    Stacking N=6 Encoder Blocks

    The encoder is not one block but a stack of N identical blocks. Each block takes the previous block's output and refines it further.

    Early layers tend to capture local, syntactic information. Later layers capture more abstract, semantic information. This is not a hard rule, but a pattern observed across many probing studies. The residual connections ensure that raw token information remains accessible even at the top layer.

    Concrete Dimension Walkthrough

    Let us trace exact tensor shapes through one encoder block with concrete values: batch size 32, sequence length 128, model dimension 512, 8 attention heads.

    StageTensor ShapeNotes
    Input X [32, 128, 512] Embeddings + positional encoding
    Q = X @ W_Q [32, 128, 512] W_Q is [512, 512]
    Reshape Q for heads [32, 8, 128, 64] Split d_model into h heads of d_k
    Attention scores [32, 8, 128, 128] Q @ K^T; each head has its own 128x128 matrix
    Attention weights [32, 8, 128, 128] After softmax; each row sums to 1
    Weighted values [32, 8, 128, 64] weights @ V
    Concat heads [32, 128, 512] Reshape back: 8 heads of 64 = 512
    Output projection [32, 128, 512] @ W_O where W_O is [512, 512]
    Add & Norm [32, 128, 512] X + attention_output, then normalize
    FFN Linear 1 [32, 128, 2048] Expansion: 512 to 2048
    ReLU [32, 128, 2048] Element-wise, shape unchanged
    FFN Linear 2 [32, 128, 512] Projection back: 2048 to 512
    Add & Norm [32, 128, 512] Residual + normalize. Same as input shape.

    The key observation: the tensor shape entering and leaving each encoder block is always [batch, seq_len, d_model]. This uniformity is what makes stacking possible. You can add more blocks without changing any interfaces.

    Memory cost: The attention score matrix [32, 8, 128, 128] requires 32 × 8 × 128 × 128 × 4 bytes (float32) = roughly 16 MB. For sequence length 2048, this grows to roughly 4 GB. This quadratic scaling in sequence length is the primary memory bottleneck in Transformers.

    Summary

    The encoder block is conceptually simple: self-attention lets each token see the full context, the FFN transforms each position independently, and residual connections keep gradients healthy. The power comes from stacking these blocks and training them end-to-end on large data.

    • Multi-head attention: parallel attention in h subspaces, concatenated and projected
    • Residual connections: gradient highway, makes deep stacking feasible
    • Layer normalization: stabilizes training without batch dependence
    • Feed-forward network: 4x expansion with nonlinearity, most parameters live here
    • All dimensions are preserved through the block: input shape = output shape
