    Multi-Head Attention: Splitting Representation Space

    January 2025 · 11 min read

    Scaled-dot-product attention works, but a single head has one representation capacity. Multi-Head Attention (Vaswani et al., 2017) runs $h$ parallel attention heads, each operating on a compressed subspace of dimension $d_k = d_{\mathrm{model}} / h$. This allows the model to attend to different semantic/syntactic patterns simultaneously. Each head learns its own query/key/value projections, runs scaled-dot-product attention independently, and the outputs are concatenated and projected back. This design is the foundation of all modern transformers.

    Motivation: Redundancy and Specialization

    Suppose we have a 768-dimensional embedding. A single scaled-dot-product attention head operating on this space can attend to one pattern very well. But language has multiple aspects: some tokens might need to attend to grammatical subjects, others to semantic arguments, others to nearby function words. A single head cannot excel at all patterns simultaneously. By splitting into $h$ independent heads, we get $h$ "slots," each learned to specialize. Head 1 might learn to attend to verb objects; Head 2 might learn nearby context. During backprop, each head gradient flows independently, allowing rapid specialization.

    Formulation and Shapes

    Given input $X \in \mathbb{R}^{n \times d_{\mathrm{model}}}$:

    1. Project to Q, K, V: $Q = XW^Q$, $K = XW^K$, $V = XW^V$, where $W^Q, W^K, W^V \in \mathbb{R}^{d_{\mathrm{model}} \times d_{\mathrm{model}}}$. Shapes: $(n, d_{\mathrm{model}})$ each.
    2. Reshape for heads: Split each of $Q, K, V$ into $h$ heads of size $d_k = d_{\mathrm{model}} / h$: $$Q_i = \mathrm{reshape}(Q)[i, :, :] \in \mathbb{R}^{n \times d_k}, \quad K_i, V_i \text{ similarly}$$ Equivalently (and commonly in code), separate projection matrices $W^Q_i, W^K_i, W^V_i \in \mathbb{R}^{d_{\mathrm{model}} \times d_k}$ for each head.
    3. Apply attention per head: For each head $i = 1, \ldots, h$: $$\mathrm{head}_i = \mathrm{Attention}(Q_i, K_i, V_i) = \mathrm{Softmax}\left(\frac{Q_i K_i^T}{\sqrt{d_k}}\right) V_i \in \mathbb{R}^{n \times d_k}$$
    4. Concatenate: $\mathrm{concat} = [{\mathrm{head}_1 \| \mathrm{head}_2 \| \ldots \| \mathrm{head}_h}] \in \mathbb{R}^{n \times d_{\mathrm{model}}}$.
    5. Output projection: $\mathrm{MultiHeadAttention}(Q, K, V) = \mathrm{concat} \cdot W^O$, where $W^O \in \mathbb{R}^{d_{\mathrm{model}} \times d_{\mathrm{model}}}$.

    Why This Works

    The capacity argument: a single head with $d_{\mathrm{model}} = 768$ and one query-key dot product can attend to one global pattern. But with $h = 12$ heads of $d_k = 64$, each head can learn a distinct pattern. The model can simultaneously attend to multiple semantic relationships. During backpropagation, gradients for different patterns flow independently, avoiding the bottleneck of forcing all gradients through a single attention mechanism. The output projection $W^O$ learns to combine the heads adaptively.

    Empirically, analyses of attention heads in trained models show clear specialization: some heads consistently focus on local syntax, others on document structure, others on discourse markers. This supports the design.

    Implementation Sketch

    def multihead_attention(Q, K, V, d_model, num_heads):
    """
    Q, K, V: (batch, seq_len, d_model)
    d_model: total dimension (e.g., 768)
    num_heads: number of heads (e.g., 12)
    """
    batch, seq_len, d = Q.shape
    d_k = d_model // num_heads
    
    # Project to Q, K, V (full dimension)
    Q = linear_Q(Q)  # (batch, seq_len, d_model)
    K = linear_K(K)  # (batch, seq_len, d_model)
    V = linear_V(V)  # (batch, seq_len, d_model)
    
    # Reshape for multiple heads
    Q = Q.view(batch, seq_len, num_heads, d_k).transpose(1, 2)  # (batch, num_heads, seq_len, d_k)
    K = K.view(batch, seq_len, num_heads, d_k).transpose(1, 2)  # (batch, num_heads, seq_len, d_k)
    V = V.view(batch, seq_len, num_heads, d_k).transpose(1, 2)  # (batch, num_heads, seq_len, d_k)
    
    # Scaled dot-product attention per head (batched)
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)  # (batch, num_heads, seq_len, seq_len)
    attn_weights = softmax(scores, dim=-1)  # (batch, num_heads, seq_len, seq_len)
    output_heads = attn_weights @ V  # (batch, num_heads, seq_len, d_k)
    
    # Concatenate heads
    output = output_heads.transpose(1, 2)  # (batch, seq_len, num_heads, d_k)
    output = output.contiguous().view(batch, seq_len, d_model)  # (batch, seq_len, d_model)
    
    # Output projection
    output = linear_O(output)  # (batch, seq_len, d_model)
    return output

    Complexity Analysis

    Aspect Complexity
    Time (seq len $n$, dim $d_{\mathrm{model}}$) $O(n^2 d_{\mathrm{model}})$ (same as single-head attention)
    Parameters (per layer) $3 d_{\mathrm{model}}^2 + d_{\mathrm{model}}^2 = 4 d_{\mathrm{model}}^2$ for $W^Q, W^K, W^V, W^O$
    Memory (intermediate attention matrix) $O(n^2 / h)$ per head; total $O(n^2)$ across heads

    Variants and Scaling

    Number of heads: Typical choices are 8, 12, or 16 for base models. Larger models (e.g., GPT-3 with 96 layers) use 96 heads. The total parameter count is independent of the number of heads (all heads share the same total $d_{\mathrm{model}}$), so more heads is "free" in terms of parameters, though it adds modest computational overhead.

    Grouped projections vs. separate: In practice, a single large projection matrix is often used for efficiency: $W^Q, W^K, W^V$ concatenated, then reshaped. The math is identical but memory layout is better.

    Cross-attention: When $Q$ comes from one sequence and $K, V$ from another (e.g., decoder attending to encoder), the mechanics are the same; only the input sources differ.

    Empirical Head Analysis

    Post-hoc analysis of trained transformers (e.g., "Attention Is All You Need" Fig. 5, or Vig & Belinkov 2019) shows that heads specialize:

    • Positional heads: Attend to nearby positions (adjacent tokens).
    • Syntactic heads: Attend to tokens by grammatical role (subjects, verbs, objects).
    • Discourse heads: Attend to distant antecedents or discourse markers.

    This empirical evidence justifies the design: the parameter overhead of maintaining $h$ separate heads is rewarded with emergent specialization.
