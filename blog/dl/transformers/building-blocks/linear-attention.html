    Linear Attention: The Kernel Trick

    January 2026 · 12 min read

    Parts 5–6 traded quadratic cost for sparsity (local context). Linear attention takes a different approach: it eliminates the $n \times n$ attention matrix entirely via the kernel trick, achieving $O(nd^2)$ complexity. Instead of computing pairwise similarities and then aggregating, we compute feature maps and accumulate online. This is used in Performer, Linear Transformer, and recent efficient models.

    The Kernel Trick: Reordering Matmuls

    Standard attention is:

    $$\text{Attn}(Q, K, V) = \text{softmax}(QK^T) V$$

    The $n \times n$ matrix $QK^T$ is the bottleneck. But we can rewrite this using associativity of matrix multiplication:

    $$\text{Attn}(Q, K, V) = (Q (K^T V)) \text{ ... if we remove softmax}$$

    This is linear in $n$! But softmax breaks the factorization. Enter: kernel feature maps. Replace softmax with a feature map function $\phi$:

    $$\text{Attn}(Q, K, V) \approx \text{diag}(D)^{-1} \phi(Q) (\phi(K)^T V)$$ where $D$ is a normalization vector. Now we compute $\phi(K)^T V$ (a $d \times d$ matrix, expensive in $d$ but cheap in $n$) once, then multiply by $\phi(Q)$ (queries see the aggregated result).

    Feature Maps: Approximating Softmax

    Softmax is $\text{softmax}(x)_i = \frac{e^{x_i}}{\sum_j e^{x_j}}$. We approximate it with:

    $$\phi(x) \approx e^x \text{ (before normalization)}$$

    Then $\phi(q) \cdot \phi(k)^T = e^{q} e^{k^T} = e^{q + k^T}$, which approximates the softmax numerator. Common choices:

    • Exponential: $\phi(x) = e^x$ (naive; numerically unstable).
    • Elu+1: $\phi(x) = \text{elu}(x) + 1$ (stable, element-wise).
    • Random features: $\phi(x) = e^{i \omega \cdot x}$ for random $\omega$ (approximates exp kernel via Fourier).
    Key insight: We sacrifice exact softmax for computational efficiency. The approximation error is usually acceptable because transformers are robust to attention distribution perturbations. The paper "Rethinking Attention with Performers" shows perplexity drops are minimal for many tasks.

    Online Accumulation

    The algorithm processes queries and keys sequentially without storing the full attention matrix:

    def linear_attention(Q, K, V, phi):
    """
    Q, K, V: (n, d)
    phi: feature map function
    """
    n, d = Q.shape
    state = torch.zeros(d, d)  # accumulator for phi(K)^T V
    norm = torch.zeros(d)      # normalization
    
    outputs = []
    for i in range(n):
        phi_k_i = phi(K[i])    # (d,)
        # Update state: state += phi(K[i]) * V[i]^T
        state += torch.outer(phi_k_i, V[i])
        norm += phi_k_i
        
        # Query: compute attention for position i
        phi_q_i = phi(Q[i])    # (d,)
        # out_i = (phi_q_i . state) / (phi_q_i . norm)
        out_i = (phi_q_i @ state) / (phi_q_i @ norm + eps)
        outputs.append(out_i)
    
    return torch.stack(outputs)

    Complexity: $O(nd^2)$ total—no $n^2$ term!

    Quality vs. Speed Trade-Off

    MethodComplexityApproximationUse
    Standard Softmax$O(n^2 d)$ExactShort sequences
    Sliding-Window$O(nwd)$Local onlyLong sequences, local context sufficient
    Linear (feature map)$O(nd^2)$ApproximateVery long sequences, approximation acceptable

    For $n=100K, d=64$: $O(n^2 d) = 640M$ vs. $O(nd^2) = 410M$. But as $n \to \infty$, linear wins decisively.

    Limitations

    • Approximation error: Not exact softmax. Can degrade performance on tasks requiring precise attention patterns.
    • Dimension dependence: Still quadratic in $d$. If $d = 512$ (large), overhead is significant.
    • Normalization: Computing $\text{diag}(D)^{-1}$ online requires care. Division by small numbers can be numerically unstable.

    Performer: A Practical Linear Transformer

    Performer (Choromanski et al., 2020) uses random Fourier features to approximate softmax:

    $$\phi(x) = e^{-\|x\|^2/2} [\cos(\omega_1 \cdot x), \sin(\omega_1 \cdot x), \ldots, \cos(\omega_m \cdot x), \sin(\omega_m \cdot x)]$$

    where $\omega_i$ are random vectors. Theoretically, this approximates the RBF kernel, which itself approximates softmax. Performer achieves comparable performance to BERT on many tasks with much faster inference on long sequences.

    Numerical Stability and Online Normalization

    Computing $\text{diag}(D)^{-1}$ requires tracking normalization denominators. A small error in the normalization accumulator can cause division by near-zero. Modern implementations use:

    • Double precision for accumulation.
    • Stabilized feature maps (e.g., $\phi(x) = (elu(x) + 1)^2$) to keep values positive.
    • Layernorm or other stochastic stabilization.
