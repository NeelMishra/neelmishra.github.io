    Sliding-Window Attention: Linear-Cost Localization

    January 2026 · 11 min read

    The quadratic complexity of full attention ($O(n^2)$) is problematic for long sequences. Sliding-window attention restricts each token to attend only to a fixed window of neighbors. The result: $O(nw)$ instead of $O(n^2)$, where $w$ is the window size. This is used in Longformer, Mistral, and Llama2. This part explores the trade-off and the intuition.

    Motivation: Locality Bias

    Empirically, transformers learn that nearby tokens are often more relevant than distant ones. A word is more likely to be modified by its immediate neighbors than by a word ten positions away. So why compute full attention? We can apply a locality inductive bias: each position attends to only $w$ tokens to its left (and optionally right).

    Window size $w$: Typically $w$ is $64$, $512$, or $4096$. For Longformer, $w = 64$ or $512$. For Mistral, $w = 4096$. The choice depends on expected token dependencies and memory budget.

    Definition: The Sliding Window

    For position $i$ with window size $w$, we compute attention only over positions in range $[\max(0, i-w), i]$ (left-looking) or $[i, \min(n-1, i+w)]$ (right-looking). In causal models, we use left-looking to stay autoregressive.

    $$\text{SlidingWindowAttn}(i) = \sum_{j \in [i-w, i]} \text{softmax}(s_{ij}) \cdot v_j$$

    The mask is now a band: entries outside the window are set to $-\infty$ before softmax, exactly like causal masking in Part 3.

    Attention Matrix Structure

    The full attention matrix becomes banded. For a window size $w = 3$:

    • Row 0: attends to column 0 only (no history)
    • Row 1: attends to columns 0-1
    • Row 2: attends to columns 0-2 (full window)
    • Row 3 onwards: attends to current and $w-1$ previous positions

    This is a lower-triangular band. Storage is $O(nw)$ instead of $O(n^2)$.

    Complexity Analysis

    VariantTimeSpaceWhen to Use
    Full Self-Attention$O(n^2 d)$$O(n^2)$Short sequences (<512 tokens)
    Sliding-Window$O(nwd)$$O(nw)$Long sequences, local context sufficient
    Linear Attention (Part 7)$O(nd^2)$$O(nd)$Very long, approximation acceptable

    For $w << n$, this is a massive speedup. E.g., $n=4096, w=128$: $nw d = 524K d$ vs. $n^2 d = 16M d$, a 30x reduction.

    Two-Sided Windows

    In non-causal models (encoders), you can attend to both left and right neighbors, using a center window of size $2w$. This improves context but doubles the memory. Longformer supports both modes.

    PyTorch Implementation

    def sliding_window_attention(Q, K, V, window_size):
    """
    Q, K, V: (batch, n, d)
    window_size: w
    Returns: (batch, n, d)
    """
    n = Q.shape[1]
    scores = torch.matmul(Q, K.transpose(-1, -2)) / (Q.shape[-1] ** 0.5)
    
    # Create sliding window mask
    mask = torch.tril(torch.ones(n, n)) - torch.tril(torch.ones(n, n), 
                                                      offset=-window_size-1)
    scores = scores.masked_fill(mask == 0, float('-inf'))
    
    weights = F.softmax(scores, dim=-1)
    output = torch.matmul(weights, V)
    return output

    Limitations: Information Bottleneck

    Sliding-window attention cannot capture long-range dependencies. If you need to relate tokens 1000 positions apart, a window of size 512 won't see them. Solutions include:

    • Stacked layers: Multiple attention layers can compose to reach further context (receptive field grows exponentially with depth).
    • Hybrid (Part 6): Add a few "global" tokens that attend to the full sequence.
    • Hierarchical: Aggregate local windows into larger contexts.

    Window size w=3. Each position attends only to its recent past.

    Empirical Results

    Longformer on BERT tasks with $w=512$: achieves $\sim 92\%$ of full attention performance while being 10x faster. For document QA, the gap is even smaller. This suggests that in practice, most dependencies are indeed local.
