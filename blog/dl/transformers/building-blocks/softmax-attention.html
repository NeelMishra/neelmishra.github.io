    Softmax-Attention: Why Softmax Wins

    January 2026 · 11 min read

    Part 1 introduced self-attention. But why apply *softmax* to the scores? Why not normalize linearly, or use a sparse weighting? This post derives why softmax is the natural and optimal choice: it gives probabilistic semantics, handles unbounded scores, sharpens the attention distribution, and is numerically stable.

    The Weighting Function Design Choice

    After computing scores $s_{ij} = \langle q_i, k_j \rangle / \sqrt{d_k}$, we need to convert them into weights $w_{ij}$ such that:

    • $w_{ij} \geq 0$ (non-negative)
    • $\sum_j w_{ij} = 1$ per query (normalization)
    • Large scores $\Rightarrow$ large weights (monotone increasing)

    Three candidates come to mind: linear normalization, sparsemax, and softmax. Softmax is standard. Let's understand why.

    Softmax Definition and Properties

    The softmax function is:

    $$\text{softmax}(s)_i = \frac{e^{s_i}}{\sum_j e^{s_j}}$$

    Key properties:

    • Probabilistic: Output is a probability distribution (non-negative, sums to 1).
    • Differentiable: Easy to backpropagate through. The derivative of softmax has a clean form.
    • Exponential sharpening: Large scores become much larger relative to small ones. A score of 5 contributes $e^5 \approx 148$ vs. score 0 contributes $e^0 = 1$.
    • Unbounded range: Scores can be any real number; softmax always normalizes.
    Intuition: Softmax turns scores into a "winner-take-most" distribution. If one key is much more similar to the query, it gets most of the attention weight. This is often what we want: focus on the most relevant tokens.

    Numerical Stability: The Subtract-Max Trick

    Computing $e^{s_i}$ directly can overflow for large $s_i$. The solution is the log-sum-exp trick:

    $$\text{softmax}(s)_i = \frac{e^{s_i - m}}{\sum_j e^{s_j - m}}$$ where $m = \max_j s_j$. Subtracting the max before exponentiation keeps values in a safe numerical range without changing the result (it cancels in the ratio).

    import numpy as np

def softmax_stable(scores):
    """Numerically stable softmax."""
    m = np.max(scores)
    exp_scores = np.exp(scores - m)
    return exp_scores / np.sum(exp_scores)

# Example
scores = np.array([1000, 1001, 999])  # Large scores
weights = softmax_stable(scores)
print(weights)  # ~[0.09, 0.81, 0.02]

    Gradient Flow and Learning

    Why softmax helps optimization: consider a score $s_i$ that is too low. In gradient descent, we want to increase it to improve the match. The gradient of softmax is:

    $$\frac{\partial \text{softmax}(s)_i}{\partial s_k} = \text{softmax}(s)_i (\delta_{ik} - \text{softmax}(s)_k)$$

    Even if $w_i$ is small, the gradient remains non-zero, allowing the model to learn to increase it. With linear normalization (where $w_i \propto \max(0, s_i)$), scores at 0 have zero gradient—no learning signal when you're outside the ReLU.

    Entropic Interpretation

    Softmax can be derived as the solution to: maximize entropy subject to fixed expected score. Given a budget of "probability mass" and a collection of scores, softmax allocates mass to maximize surprise (entropy), weighted by scores. This is a principled information-theoretic choice.

    ▶ Softmax vs. Linear Normalization

    Same logits, two normalizations. Softmax sharpens; linear mostly ignores small scores.

    Attention Entropy

    The entropy of attention weights is a diagnostic: $H = -\sum_j w_j \log w_j$. High entropy means the model attends to many positions (distributed attention). Low entropy means focused. During training, attention entropy often decreases as the model learns to specialize. Softmax allows this gradient signal; some alternatives do not.
