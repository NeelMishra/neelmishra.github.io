    Training Transformers

    March 2026 · ML Series · Training & Optimization

    Overview

    Building a transformer architecture is only the first step. Training one to produce useful outputs requires careful choices about the loss function, optimizer, learning rate schedule, numerical precision, and regularization. Each decision interacts with the others. A learning rate that works with one batch size may diverge with another. Dropout that improves a small model may hurt a large one. This article covers the core training recipe used by most modern transformers, from the original "Attention Is All You Need" paper through to current large language models.

    Cross-Entropy Loss for Language Modeling

    Language models are trained to predict the next token given the preceding context. At each position in the sequence, the model outputs a probability distribution over the entire vocabulary. The training objective minimizes the cross-entropy between the predicted distribution and the true distribution (a one-hot vector where the correct token has probability 1).

    L = −(1/T) ∑t=1T log P(xt | x1, ..., xt−1)

    Here T is the sequence length and P is the model's predicted probability for the correct token xt. This loss has a direct relationship to perplexity: PPL = exp(L). Lower cross-entropy means lower perplexity, which means the model assigns higher probability to the correct continuations.

    In practice, the model's final layer produces logits z (unnormalized scores), which are passed through softmax to get probabilities. The cross-entropy loss is computed efficiently by combining the log-softmax and negative log-likelihood steps. Most frameworks provide a fused operation (e.g., torch.nn.CrossEntropyLoss) that is numerically stable and avoids computing the full softmax explicitly.

    Label Smoothing

    Standard cross-entropy pushes the model to assign all probability mass to the correct token. This encourages the logits for the correct class to grow without bound relative to the others, which has two problems:

    • The model becomes overconfident. It assigns near-zero probability to plausible alternative tokens, even when multiple continuations are valid.
    • The gradients become very small for well-classified examples, slowing down learning on hard cases.

    Label smoothing addresses this by replacing the hard one-hot target with a soft distribution. Instead of target = [0, 0, 1, 0, ...], we use:

    ysmooth(k) = (1 − ε) · y(k) + ε / V

    where ε is typically 0.1 and V is the vocabulary size. The correct class gets probability (1 − ε + ε/V) and every other class gets ε/V. This prevents the logits from becoming too extreme and acts as a form of regularization. The original transformer paper used ε = 0.1 and reported improved BLEU scores on translation tasks.

    Key insight: Label smoothing does not change which token has the highest predicted probability. It changes how aggressively the model is penalized for assigning low (but nonzero) probability to incorrect tokens. The model still learns to rank the correct token first, but it maintains a more calibrated distribution.

    The Adam Optimizer

    Transformers are almost universally trained with Adam or its variant AdamW. Adam maintains two exponential moving averages for each parameter: the first moment (mean of gradients, m) and the second moment (mean of squared gradients, v). These serve different purposes:

    • First moment (m) provides momentum. It smooths out gradient noise by averaging over recent steps. Parameters that consistently receive gradients in the same direction move faster.
    • Second moment (v) provides adaptive learning rates. Parameters with historically large gradients get smaller effective learning rates, and parameters with small gradients get larger ones.

    The update rule at step t:

    // Adam update with bias correction
mt = β1 · mt−1 + (1 − β1) · gt
vt = β2 · vt−1 + (1 − β2) · gt2

// Bias correction (accounts for zero initialization)
m̂t = mt / (1 − β1t)
v̂t = vt / (1 − β2t)

// Parameter update
θt = θt−1 − η · m̂t / (√v̂t + ε)

    The original transformer uses β1 = 0.9, β2 = 0.98, and ε = 10−9. The choice of β2 = 0.98 (rather than the default 0.999) is significant: it makes the second moment estimate adapt faster to changing gradient magnitudes, which is important during the early phase of training when the loss landscape is changing rapidly.

    AdamW decouples weight decay from the gradient update. Standard Adam applies weight decay through the gradient (L2 regularization), but this interacts poorly with the adaptive learning rate. AdamW applies weight decay directly to the parameters after the Adam step, which produces better generalization in practice.

    Learning Rate Warmup Schedule

    The learning rate schedule is one of the most important hyperparameters for training transformers. The original paper introduced a specific schedule that combines linear warmup with inverse square root decay:

    lr(step) = dmodel−0.5 · min(step−0.5, step · warmup_steps−1.5)

    During warmup (step < warmup_steps), the learning rate increases linearly from near zero. After warmup, it decays proportionally to the inverse square root of the step number. The warmup_steps hyperparameter is typically set to 4000.

    Why Warmup Matters

    At the beginning of training, all parameters are randomly initialized and the model's outputs are essentially random. The gradients in this regime have two problematic properties:

    • High variance. The gradients are noisy because the model has not yet learned any useful representations. Large steps in random directions waste computation and can push parameters into bad regions of the loss landscape.
    • Adam's second moment estimates are unreliable. The moving average vt is initialized to zero and needs several hundred steps to converge to an accurate estimate of the gradient variance. During this period, the bias-corrected estimate can be very small, causing the effective learning rate to be much larger than intended.

    Warmup addresses both issues. By starting with a tiny learning rate and increasing it gradually, the model takes small, safe steps while the optimizer's statistics stabilize. By the time the learning rate reaches its peak, the first and second moment estimates are reliable, and the model has begun to learn basic patterns that produce more informative gradients.

    Without warmup, training often diverges in the first few hundred steps. The loss spikes and never recovers. This is especially true for large models, where the interaction between high-dimensional parameter spaces and adaptive optimizers creates instability.

    Key insight: Warmup is not about being conservative. It is about giving the optimizer time to calibrate its internal state. Once the moving averages are accurate, the optimizer can safely use the full learning rate.

    Batch Size and Gradient Accumulation

    The effective batch size during training affects both the optimization dynamics and the final model quality. Larger batches produce more accurate gradient estimates (lower variance), which allows for larger learning rates and faster convergence in terms of wall-clock time. However, there are limits: very large batches can lead to sharp minima that generalize poorly.

    Transformer training often uses large effective batch sizes (32k to 4M tokens per step for language models). Since a single GPU cannot fit this many tokens in memory, gradient accumulation is used:

    // Gradient accumulation over K micro-batches
for step in training_steps:
    optimizer.zero_grad()
    for k in range(K):
        micro_batch = get_next_batch()
        loss = model(micro_batch) / K
        loss.backward()       // gradients accumulate
    optimizer.step()

    Each micro-batch fits in GPU memory. The gradients are accumulated (summed) across K micro-batches before the optimizer takes a step. Dividing the loss by K ensures that the accumulated gradient is the average over the full batch, matching what you would get if you processed all K micro-batches at once.

    The relationship between batch size and learning rate follows a rough rule: when you double the batch size, you can increase the learning rate by up to a factor of √2 (linear scaling also works in some regimes). This is because the gradient variance decreases with batch size, making each step more reliable.

    Mixed Precision Training

    Modern transformer training uses mixed precision: most computations run in half precision (FP16 or BF16) while a master copy of the weights is kept in FP32. This provides two benefits:

    • Speed. Half-precision matrix multiplications run 2 to 8 times faster on modern GPUs (tensor cores are designed for FP16/BF16).
    • Memory. Activations stored in FP16 use half the memory, allowing larger batch sizes or longer sequences.

    The challenge is that FP16 has limited dynamic range (roughly 6 x 10−8 to 65504). Small gradients can underflow to zero, and large values can overflow to infinity. Two techniques handle this:

    Loss scaling. Before the backward pass, the loss is multiplied by a large factor (e.g., 1024). This scales all gradients up, preventing underflow. After the backward pass, the gradients are divided by the same factor before the optimizer step. Dynamic loss scaling starts with a large scale and reduces it whenever an overflow (NaN or Inf) is detected.

    BF16 (Brain Float 16). This format uses the same exponent range as FP32 (8 exponent bits) but with reduced mantissa precision (7 bits instead of 23). It avoids the overflow and underflow issues of FP16 and typically does not require loss scaling. BF16 is the preferred format for training on hardware that supports it (TPUs, A100 and newer NVIDIA GPUs).

    // PyTorch mixed precision training
scaler = GradScaler()

for batch in dataloader:
    optimizer.zero_grad()
    with autocast(dtype=torch.float16):
        output = model(batch)
        loss = criterion(output, target)
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()

    Dropout in Transformers

    Dropout is applied at several specific locations within the transformer architecture. During training, a fraction p of activations are randomly set to zero. During inference, dropout is disabled and all activations are used (scaled by 1/(1-p) during training to maintain expected values).

    The standard dropout locations in a transformer block:

    • Attention weights. After the softmax that produces attention scores, dropout is applied to the attention matrix. This randomly prevents some tokens from attending to others, encouraging the model to spread attention across multiple positions.
    • Residual connection outputs. After the multi-head attention sublayer and after the feed-forward sublayer, dropout is applied to the output before it is added to the residual. This is the primary regularization point.
    • Embedding layer. Dropout is applied to the sum of token embeddings and positional embeddings. This prevents the model from relying too heavily on the exact embedding values.

    The original transformer used p = 0.1 for all dropout locations. Larger models sometimes use lower dropout (0.0 to 0.05) because they have enough parameters to memorize training data less and benefit more from seeing all activations. Some recent architectures (GPT-3, PaLM) drop residual dropout entirely for very large models and rely on other regularization (data diversity, weight decay).

    Key insight: Dropout in the attention matrix is conceptually different from standard dropout. It randomly masks entire attention connections, which is equivalent to randomly removing edges in the token interaction graph during training. This forces each token to gather information from multiple source positions rather than relying on a single attended token.

    Putting It All Together

    A typical transformer training recipe combines all of the elements above:

    • Loss: cross-entropy with label smoothing (ε = 0.1)
    • Optimizer: AdamW with β1 = 0.9, β2 = 0.98, weight decay 0.01 to 0.1
    • Learning rate: warmup over 4000 steps, then inverse square root or cosine decay
    • Batch size: as large as memory allows, using gradient accumulation
    • Precision: BF16 mixed precision with FP32 master weights
    • Regularization: dropout (0.1 for small models, lower for large ones) and weight decay

    The interaction between these choices matters. Increasing batch size allows a higher learning rate. Using mixed precision frees memory for larger batches. Longer warmup may be needed for larger learning rates. Finding the right combination requires systematic experimentation, but the recipe above serves as a strong starting point backed by years of practice across the research community.
