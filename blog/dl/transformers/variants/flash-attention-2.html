Flash Attention 2

ML Series · Transformers · Variants

1 · Flash Attention 1 Recap

Flash Attention 1 introduced tiled, IO-aware attention: instead of materializing the full N×N attention matrix in HBM, it processes Q and K/V in blocks that fit in SRAM, using the online softmax trick to accumulate results without a second pass. This reduces memory from O(N²) to O(N) and avoids the HBM bandwidth bottleneck.

However, FA1 left significant performance on the table. On an A100 GPU, FA1 achieved only 25–40% of theoretical peak FLOP/s. Flash Attention 2 closes this gap with three algorithmic improvements.

2 · Limitations of FA1

  • Non-matmul FLOPs dominate: FA1 performs many element-wise operations (rescaling, softmax normalization) that don't use GPU tensor cores. On A100, tensor cores are 16× faster than non-tensor-core math. Even though non-matmul ops are a small fraction of total FLOPs, they consume a disproportionate amount of wall-clock time.
  • Suboptimal parallelism: FA1's outer loop iterates over K/V blocks, inner loop over Q blocks. This means the output accumulator for each Q block is updated by multiple K/V iterations, requiring shared-memory synchronization. It also limits parallelism to the batch × heads dimension.
  • Work partitioning within warps: FA1 splits work across warps such that they need to read/write shared memory to communicate partial results, adding latency.

3 · The Three Key Changes

Swap the loop order. Make Q the outer loop and K/V the inner loop. Each thread block processes one Q block against all K/V blocks, writing the final output once. No cross-iteration accumulation conflicts.

Defer rescaling. Instead of rescaling the running output after each K/V block, track the log-sum-exp and apply the final correction once at the end. This eliminates most non-matmul FLOPs from the inner loop.

Parallelize over sequence length. With Q as the outer loop, each Q block is independent → can be processed by a different thread block. For long sequences, this provides N/B_q additional parallelism beyond batch × heads.

4 · Loop Swap: Outer on Q, Inner on K/V

In FA1, the outer loop is over K/V blocks and the inner over Q blocks. Each Q block's output is updated across multiple outer iterations, requiring careful synchronization:

FA1 loop structure: for j = 1 to T_kv: ← outer (K/V blocks) load K_j, V_j to SRAM for i = 1 to T_q: ← inner (Q blocks) load Q_i, O_i to SRAM O_i ← update(O_i, Q_i, K_j, V_j) ← writes O_i multiple times store O_i back to HBM

FA2 swaps this:

FA2 loop structure: for i = 1 to T_q: ← outer (Q blocks) — parallelize here! load Q_i to SRAM O_i ← 0, ℓ_i ← 0, m_i ← -∞ for j = 1 to T_kv: ← inner (K/V blocks) load K_j, V_j to SRAM O_i ← update(O_i, Q_i, K_j, V_j) ← one thread block owns O_i store O_i to HBM ← single write at the end
Key insight: Each Q block's output is now computed entirely within one thread block. No shared-memory conflicts, no synchronization between blocks, and O_i is written to HBM exactly once instead of T_kv times.

5 · Reducing Non-Matmul FLOPs

On A100, matmul instructions (tensor cores) achieve 312 TFLOP/s in FP16, while non-matmul (element-wise, reductions) only get ~19.5 TFLOP/s — a 16× gap. FA1's inner loop rescales the output after every K/V block:

FA1 inner loop (per K/V block j): S_ij = Q_i · K_j^T / √d ← matmul ✓ m_new = max(m_old, rowmax(S_ij)) ← non-matmul ✗ P_ij = exp(S_ij - m_new) ← non-matmul ✗ ℓ_new = exp(m_old - m_new) · ℓ_old + rowsum(P_ij) ← non-matmul ✗ O_i = diag(exp(m_old - m_new)) · O_old + P_ij · V_j ← matmul + non-matmul O_i = diag(1/ℓ_new) · O_i ← non-matmul ✗ (RESCALE)

FA2 removes the final rescaling from the inner loop. Instead, it maintains unnormalized O and applies the 1/ℓ correction once after the loop:

FA2 inner loop (per K/V block j): S_ij = Q_i · K_j^T / √d ← matmul ✓ m_new = max(m_old, rowmax(S_ij)) ← non-matmul (kept) P_ij = exp(S_ij - m_new) ← non-matmul (kept) ℓ_new = exp(m_old - m_new) · ℓ_old + rowsum(P_ij) ← non-matmul (kept) O_i = diag(exp(m_old - m_new)) · O_old + P_ij · V_j ← matmul ✓ After all K/V blocks: O_i = diag(1/ℓ_final) · O_i ← one-time correction
Impact: This eliminates one element-wise division per inner iteration. For a sequence of 4096 tokens with block size 128, that's 32 iterations × N elements worth of non-tensor-core work saved per Q block.

6 · Parallelism over Sequence Length

FA1 parallelizes over batch size (B) and number of heads (H). For small batches (B=1 during inference), this means only H thread blocks, which can underutilize the GPU:

FA1 parallelism: B × H thread blocks A100 has 108 SMs → need ≥ 108 blocks for full utilization Inference: B=1, H=32 → only 32 blocks (30% utilization) FA2 parallelism: B × H × (N / B_q) thread blocks Each Q block is an independent unit of work B=1, H=32, N=4096, B_q=128 → 32 × 32 = 1024 blocks (full utilization!)

This is especially important for long-context inference and small-batch scenarios. With sequences of 8K–128K tokens, FA2 can keep all GPU SMs busy even at batch size 1.

7 · Full Algorithm

# Flash Attention 2 — Forward Pass def flash_attn_2_forward(Q, K, V, B_q, B_kv): N, d = Q.shape O = zeros(N, d) L = zeros(N) # log-sum-exp for backward # Outer loop: parallelize over Q blocks (one thread block each) parallel for i = 0 to N/B_q - 1: Q_i = Q[i*B_q : (i+1)*B_q] # load Q block to SRAM O_i = zeros(B_q, d) ℓ_i = zeros(B_q) m_i = full(B_q, -∞) # Inner loop: sequential over K/V blocks for j = 0 to N/B_kv - 1: K_j = K[j*B_kv : (j+1)*B_kv] # load K block to SRAM V_j = V[j*B_kv : (j+1)*B_kv] # load V block to SRAM S_ij = Q_i @ K_j.T * (1/√d) # B_q × B_kv (matmul) m_new = max(m_i, rowmax(S_ij)) P_ij = exp(S_ij - m_new) ℓ_new = exp(m_i - m_new) * ℓ_i + rowsum(P_ij) O_i = diag(exp(m_i - m_new)) @ O_i + P_ij @ V_j m_i = m_new ℓ_i = ℓ_new # Final normalization (once, outside inner loop) O_i = diag(1/ℓ_i) @ O_i L[i*B_q:(i+1)*B_q] = m_i + log(ℓ_i) # save LSE for backward O[i*B_q:(i+1)*B_q] = O_i # single HBM write return O, L

9 · Backward Pass Improvements

FA2 also optimizes the backward pass. Key changes:

  • No Q/K/V recomputation overlap: FA1's backward recomputed S from Q and K in the inner loop with complex bookkeeping. FA2's loop swap makes this cleaner — each thread block owns one Q block and recomputes S_ij as it iterates over K/V blocks.
  • Atomic-free gradient accumulation: In FA1, multiple thread blocks write to dK and dV of the same block, requiring atomics. FA2 restructures so that each K/V block's gradient is computed by reducing across Q blocks, avoiding atomics.
  • Better warp specialization: Warps within a thread block are partitioned into "producer" warps (that load data from HBM to SRAM) and "consumer" warps (that compute matmuls). This hides memory latency behind compute.
Backward pass speedup: FA1 backward: ~50% of theoretical FLOP/s FA2 backward: ~63% of theoretical FLOP/s (1.26× faster)

11 · Causal Masking Optimization

For causal (autoregressive) attention, FA2 skips entire K/V blocks that are fully masked out. When Q block i starts at row position i × B_q, any K/V block j where j × B_kv > (i+1) × B_q is entirely in the future and can be skipped:

Causal masking savings: Without optimization: each Q block processes all T_kv K/V blocks With optimization: Q block i processes only ceil((i+1) × B_q / B_kv) blocks For N=4096, B_q=B_kv=128 (32 blocks): Total work without: 32 × 32 = 1024 block pairs Total work with: 32 × 33 / 2 = 528 block pairs (1.94× speedup)

This nearly halves the work for causal attention — exactly matching the theoretical savings from the triangular mask.

12 · Summary

  • Loop swap (Q outer, K/V inner): each Q block writes output once, enables parallel Q-block processing.
  • Deferred rescaling: removes per-iteration normalization, keeping non-matmul ops out of the inner loop.
  • Sequence-level parallelism: N/B_q additional thread blocks beyond batch × heads, crucial for long-context inference.
  • Result: 2× speedup over FA1 on forward pass, 73% of A100 peak, same IO complexity and O(N) memory.
  • Combined with causal masking skip, FA2 achieves ~3.8× over standard attention.
One-line summary: FA2 squeezes 2× more FLOP/s out of FA1's IO-optimal algorithm by restructuring loops, minimizing non-matmul ops, and parallelizing over sequence length.
