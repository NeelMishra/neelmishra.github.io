Multi-Head Latent Attention (MLA)

ML Series · Transformers · Variants

1 · Beyond GQA

GQA reduces the KV cache by sharing KV heads across groups of query heads. But it still stores the full K and V vectors for each KV group. MLA (introduced in DeepSeek-V2) takes a fundamentally different approach: instead of reducing the number of KV heads, it reduces the dimensionality of what's cached.

GQA approach: reduce number of KV heads (H → G) cache size = 2 × G × seq_len × d_head MLA approach: compress ALL KV information into a low-rank latent cache size = seq_len × d_c (where d_c ≪ 2 × H × d_head)

The key insight: the keys and values across all heads are highly correlated. Instead of caching them separately, project them into a shared low-rank latent space, and reconstruct head-specific K/V at attention time.

2 · The MLA Idea

For each token, compute and cache H separate K vectors and H separate V vectors. Cache stores 2 × H × d_head values per token per layer.

For each token, compute a single latent vector c ∈ ℝ^{d_c} by projecting the input through a down-projection: c = W_down · x. Cache only c — which is much smaller than 2H separate K/V vectors.

At attention time, reconstruct head-specific K and V by up-projecting from the latent: K_i = W_UK_i · c, V_i = W_UV_i · c. This reconstruction is absorbed into the attention computation, so it adds no extra memory overhead.

Analogy: Think of MLA as JPEG for KV cache. JPEG compresses an image into fewer bytes by exploiting spatial correlations, then decompresses on the fly. MLA compresses KV information into a lower-dimensional latent, then decompresses each head's K/V on the fly during attention.

3 · Low-Rank KV Compression

MLA performs joint KV compression — a single latent vector encodes information for both keys and values across all heads simultaneously:

Standard MHA per token: K = [K₀; K₁; ...; K_{H-1}] ∈ ℝ^{H × d_head} (cached) V = [V₀; V₁; ...; V_{H-1}] ∈ ℝ^{H × d_head} (cached) Cache per token: 2 × H × d_head values MLA per token: c_KV = W_DKV · x ∈ ℝ^{d_c} (cached) Cache per token: d_c values At attention time, reconstruct: K_i = W_UK_i · c_KV (d_c → d_head, per head) V_i = W_UV_i · c_KV (d_c → d_head, per head) Compression ratio: 2 × H × d_head / d_c DeepSeek-V2: H=128, d_head=128, d_c=512 Ratio: 2 × 128 × 128 / 512 = 64× compression!

4 · The Math

MLA forward pass: 1. Compress KV: c_KV = W_DKV · x where W_DKV ∈ ℝ^{d_c × d_model} 2. Compress Q (optional, for reducing activation memory during training): c_Q = W_DQ · x where W_DQ ∈ ℝ^{d_c' × d_model} 3. Reconstruct per-head Q, K, V: Q_i = W_UQ_i · c_Q where W_UQ_i ∈ ℝ^{d_head × d_c'} K_i = W_UK_i · c_KV where W_UK_i ∈ ℝ^{d_head × d_c} V_i = W_UV_i · c_KV where W_UV_i ∈ ℝ^{d_head × d_c} 4. Standard attention: Attn_i = softmax(Q_i · K_i^T / √d_head) · V_i
Key optimization — absorb up-projection into attention: Q_i · K_i^T = (W_UQ_i · c_Q) · (W_UK_i · c_KV)^T = c_Q^T · W_UQ_i^T · W_UK_i · c_KV = c_Q^T · W_QK_i · c_KV Where W_QK_i = W_UQ_i^T · W_UK_i (precomputed, d_c' × d_c) → We never explicitly reconstruct K_i! Just compute c_Q^T · W_QK_i · c_KV → Cache only c_KV per token, compute attention directly on latents

5 · The RoPE Challenge

There's a problem: RoPE (Rotary Position Embeddings) applies position-dependent rotations to Q and K. This breaks the absorption trick because the rotation depends on the position of each token:

With RoPE: Q_i^rope = RoPE(Q_i, pos_q) = R(pos_q) · Q_i K_i^rope = RoPE(K_i, pos_k) = R(pos_k) · K_i Q^rope · K^rope_T = Q · R(pos_q)^T · R(pos_k) · K^T = Q · R(pos_q - pos_k) · K^T ← position-dependent! Problem: R(pos_k) is entangled with K_i = W_UK_i · c_KV → Cannot absorb W_UK into the attention weight → Would need to reconstruct full K_i from cache, defeating MLA's purpose

6 · Decoupled RoPE

DeepSeek's solution: decouple the RoPE-carrying component from the content component. Split each key into two parts:

Decoupled key for head i: K_i = [K_i^content ; K^rope] K_i^content = W_UK_i · c_KV (head-specific, from latent, NO RoPE) K^rope = RoPE(W_KR · x, pos) (shared across heads, HAS RoPE) Cache per token: c_KV ∈ ℝ^{d_c} (latent for content K and all V) k^rope ∈ ℝ^{d_rope} (small RoPE component) Total cache: d_c + d_rope per token DeepSeek-V2: 512 + 64 = 576 values per token (vs 2 × 128 × 128 = 32768 for MHA)
Why this works: The content part of attention (what to attend to) is compressed in the latent and absorbed into attention weights. The positional part (where to attend) uses a small separate vector with RoPE applied normally. The two are concatenated in the attention score computation.

8 · Cache Size Analysis

Method Cache per token DeepSeek-V2 (d=5120, H=128, d_h=128) Compression vs MHA
MHA 2 × H × d_head 32,768 1×
GQA-8 2 × G × d_head 2,048 16×
MQA 2 × d_head 256 128×
MLA d_c + d_rope 576 57×
MLA vs MQA: MLA caches slightly more than MQA (576 vs 256) but achieves significantly better quality because it preserves per-head diversity through the up-projection. MQA forces all heads to share the exact same K/V; MLA reconstructs different K/V per head from the shared latent.

9 · Implementation

class MultiHeadLatentAttention: def __init__(self, d_model, n_heads, d_c, d_rope): self.n_heads = n_heads self.d_head = d_model // n_heads self.d_c = d_c self.d_rope = d_rope # Down-projections (compress) self.W_DKV = Linear(d_model, d_c) # KV latent self.W_DQ = Linear(d_model, d_c) # Q latent (optional) # Up-projections (per-head reconstruct) self.W_UK = Linear(d_c, n_heads * self.d_head) self.W_UV = Linear(d_c, n_heads * self.d_head) self.W_UQ = Linear(d_c, n_heads * self.d_head) # Decoupled RoPE projection self.W_KR = Linear(d_model, d_rope) # shared RoPE key self.W_QR = Linear(d_c, n_heads * d_rope) # per-head RoPE query self.W_O = Linear(d_model, d_model) def forward(self, x, rope_fn, cache=None): B, S, D = x.shape # Compress c_kv = self.W_DKV(x) # (B, S, d_c) — CACHED c_q = self.W_DQ(x) # (B, S, d_c) # RoPE component k_rope = rope_fn(self.W_KR(x)) # (B, S, d_rope) — CACHED # Update cache if cache is not None: c_kv = cat([cache['c_kv'], c_kv], dim=1) k_rope = cat([cache['k_rope'], k_rope], dim=1) # Reconstruct per-head K, V from latent K_content = self.W_UK(c_kv).view(B,-1,self.n_heads,self.d_head) V = self.W_UV(c_kv).view(B,-1,self.n_heads,self.d_head) # Reconstruct per-head Q Q_content = self.W_UQ(c_q).view(B,S,self.n_heads,self.d_head) Q_rope = rope_fn(self.W_QR(c_q)).view(B,S,self.n_heads,self.d_rope) # Attention with decoupled RoPE score_content = (Q_content @ K_content.transpose(-2,-1)) score_rope = (Q_rope @ k_rope.unsqueeze(2).transpose(-2,-1)) scores = (score_content + score_rope) / sqrt(self.d_head + self.d_rope) weights = softmax(scores, dim=-1) out = (weights @ V).reshape(B, S, -1) return self.W_O(out), {'c_kv': c_kv, 'k_rope': k_rope}

10 · Comparison with GQA/MQA

Aspect GQA MQA MLA
Strategy Share KV across head groups Single KV for all heads Compress KV to latent space
Per-head diversity Partial (within-group shared) None (all heads same KV) Full (reconstructed per head)
Cache compression H/G× (e.g., 4–8×) H× (e.g., 32×) Arbitrary (e.g., 57×)
RoPE compatible Yes (native) Yes (native) Requires decoupled RoPE
Extra compute None None Up-projection at attention time
Quality Near MHA Slight degradation Near MHA (or better)

11 · DeepSeek Models Using MLA

  • DeepSeek-V2 (236B total, 21B active) — first model using MLA. Combined with MoE (160 experts, top-6). Matched LLaMA-2 70B quality with 5× less inference cost.
  • DeepSeek-V2-Lite (16B) — smaller model validating MLA at reduced scale.
  • DeepSeek-V3 (671B total, 37B active) — scaled up MLA + MoE. State-of-the-art open model on many benchmarks.
  • DeepSeek-R1 — reasoning model built on V3's MLA architecture, demonstrating MLA's compatibility with chain-of-thought training.
MLA's impact: DeepSeek's models serve inference at a fraction of the cost of comparably-capable models, largely because MLA's extreme KV cache compression allows much larger batch sizes per GPU. This economic advantage is a key reason for DeepSeek's competitiveness.

12 · Summary

  • MLA compresses the KV cache into a low-rank latent vector (d_c ≪ 2·H·d_head), achieving 57× compression in DeepSeek-V2.
  • Per-head K and V are reconstructed from the shared latent via up-projections, preserving per-head diversity.
  • The up-projection can be absorbed into the attention weight matrix, avoiding explicit K/V reconstruction.
  • Decoupled RoPE separates positional information into a small cached component, maintaining RoPE compatibility.
  • MLA achieves better quality than MQA (57× compression) and comparable quality to GQA (16× compression) at much higher compression.
One-line summary: MLA compresses all KV information into a tiny latent vector per token, then reconstructs per-head K/V on the fly — extreme cache compression with full head diversity.
