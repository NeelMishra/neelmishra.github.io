Linear Attention

ML Series · Transformers · Variants

1 · The Softmax Bottleneck

In standard attention, the softmax function creates the O(N²) bottleneck. We must compute all N² scores before applying softmax, because softmax's denominator requires the sum across all positions:

Standard attention: Attn(Q, K, V) = softmax(Q · K^T / √d) · V softmax(s_i) = exp(s_i) / Σⱼ exp(s_j) ← needs ALL scores before normalizing This forces us to: 1. Compute full N×N score matrix (O(N²d) compute, O(N²) memory) 2. Apply softmax row-wise 3. Multiply by V

What if we could remove softmax and still get a meaningful attention mechanism — one where we never need to form the N×N matrix?

2 · The Kernel Trick

The key insight: softmax attention can be viewed as a kernel function. If we replace exp(q·k/√d) with a decomposable kernel φ(q)·φ(k), we can change the order of operations:

Softmax attention as a kernel: A_ij = exp(q_i · k_j / √d) / Σₘ exp(q_i · k_m / √d) ≈ φ(q_i) · φ(k_j) / Σₘ φ(q_i) · φ(k_m) Where φ: ℝ^d → ℝ^D is a feature map such that: φ(q) · φ(k) ≈ exp(q · k / √d) or just a positive kernel

With this decomposition, the attention output for token i becomes:

o_i = Σⱼ A_ij · v_j = Σⱼ [φ(q_i) · φ(k_j) / Σₘ φ(q_i) · φ(k_m)] · v_j = φ(q_i) · [Σⱼ φ(k_j) ⊗ v_j] / [φ(q_i) · Σₘ φ(k_m)] ───────── ────────────────── ───────────────────── 1×D query D×d "KV state" 1×1 normalizer

3 · The Associativity Trick

The magic is in the order of multiplication. Standard attention computes (Q·K^T)·V — the N×N matrix first. Linear attention computes Q·(K^T·V) — the d×d matrix first:

Standard: O = (Q · K^T) · V (N×d) · (d×N) → (N×N) · (N×d) → (N×d) Cost: O(N²d) — the N×N intermediate dominates Linear: O = Q · (K^T · V) (d×N) · (N×d) → (d×d) then (N×d) · (d×d) → (N×d) Cost: O(Nd²) — no N×N matrix! When d ≪ N (e.g., d=128, N=32768): Standard: O(N²d) = O(32768² × 128) ≈ 1.4 × 10¹¹ Linear: O(Nd²) = O(32768 × 128²) ≈ 5.4 × 10⁸ (260× fewer ops!)
This is just matrix multiplication associativity. (AB)C = A(BC). But it changes the complexity from O(N²d) to O(Nd²). Since d is fixed (typically 64–128) and N grows, this makes attention linear in sequence length.

4 · The Math

Linear attention: Given feature map φ: ℝ^d → ℝ^D (often D = d) Φ_Q = φ(Q) (N × D) Φ_K = φ(K) (N × D) KV state: S = Φ_K^T · V (D × d) — precomputed once Normalizer: z = Φ_K^T · 𝟏 (D × 1) — precomputed once Output: O_i = (Φ_Q[i] · S) / (Φ_Q[i] · z) for each token i Total cost: O(NDd) for S, O(ND) for z, O(NDd) for O → O(NDd) With D = d: O(Nd²) — linear in N!

5 · Causal Linear Attention

For autoregressive models, token i can only attend to tokens 1..i. The causal version maintains a running state:

Causal linear attention (recurrent form): S₀ = 0 (D × d zero matrix) z₀ = 0 (D × 1 zero vector) For i = 1 to N: S_i = S_{i-1} + φ(k_i) ⊗ v_i (update KV state) z_i = z_{i-1} + φ(k_i) (update normalizer) o_i = (φ(q_i) · S_i) / (φ(q_i) · z_i) (query the state) Each step: O(Dd) to update S, O(D) for z, O(Dd) for o Total: O(NDd) = O(Nd²) for D = d

This is remarkable: causal linear attention is equivalent to an RNN with a matrix-valued hidden state S ∈ ℝ^{D×d}. Each new token updates the state, and the query reads from it.

Inference advantage: During autoregressive decoding, standard attention recomputes attention over all past tokens (O(N) per step, O(N²) total). Causal linear attention just updates the D×d state — O(d²) per step, O(Nd²) total. For long sequences, this is dramatically faster.

7 · Feature Map Choices

Feature Map φ Formula Properties Used in
ELU + 1 φ(x) = elu(x) + 1 Simple, fast, always positive Linear Transformer
Random Fourier Features φ(x) = exp(Wx + b) / √D Approximates softmax kernel Performer
FAVOR+ φ(x) = exp(Wx - ||x||²/2) / √D Unbiased softmax approximation Performer
ReLU φ(x) = max(0, x) Sparse, hardware-friendly cosFormer
1 + cos φ(x) = 1 + cos(Wx) Bounded, stable cosFormer
Identity (none) φ(x) = x (no transform) Simplest, can be negative Some SSMs
Quality gap: No feature map has fully closed the quality gap with softmax attention. The best linear attention variants (RetNet, RWKV, Mamba) add decay factors, gating, or data-dependent state transitions to improve quality, moving closer to state-space models than pure linear attention.

8 · Connection to RNNs and State-Space Models

Causal linear attention's recurrent form reveals it as a special case of a linear RNN:

Causal linear attention: S_i = S_{i-1} + φ(k_i) ⊗ v_i Linear RNN (general): S_i = A · S_{i-1} + B · x_i State-Space Model (S4/Mamba): S_i = Ā · S_{i-1} + B̄ · x_i, y_i = C · S_i Linear attention = Linear RNN with A = I (identity, no decay) Adding decay: S_i = γ · S_{i-1} + φ(k_i) ⊗ v_i (0 < γ < 1) → This is RetNet's retention mechanism! → Exponentially forgets old information (like RNN forget gate)

This connection has driven a convergence of architectures: RetNet, RWKV, Mamba, and GLA (Gated Linear Attention) all live on the spectrum between linear attention and state-space models.

9 · Implementation

def linear_attention(Q, K, V, feature_map=elu_plus_1): # Non-causal version (bidirectional) phi_Q = feature_map(Q) # (B, N, D) phi_K = feature_map(K) # (B, N, D) KV = phi_K.transpose(-2,-1) @ V # (B, D, d) — the key step! z = phi_K.sum(dim=-2) # (B, D) numer = phi_Q @ KV # (B, N, d) denom = (phi_Q * z.unsqueeze(-2)).sum(-1, keepdim=True) return numer / denom def causal_linear_attention(Q, K, V, feature_map=elu_plus_1): # Causal version (recurrent) phi_Q = feature_map(Q) phi_K = feature_map(K) B, N, D = phi_Q.shape d = V.shape[-1] S = zeros(B, D, d) # running KV state z = zeros(B, D) # running normalizer outputs = [] for i in range(N): S = S + phi_K[:,i,:,None] * V[:,i,None,:] # outer product update z = z + phi_K[:,i,:] o_i = (phi_Q[:,i,:] @ S) / (phi_Q[:,i,:] @ z).clamp(min=1e-6) outputs.append(o_i) return stack(outputs, dim=1) def elu_plus_1(x): return elu(x) + 1 # ensures positivity

10 · Models & Architectures

  • Linear Transformer (Katharopoulos et al., 2020) — first pure linear attention with ELU+1 feature map. Showed viability but quality gap on language.
  • Performer (Choromanski et al., 2021) — FAVOR+ random feature maps to approximate softmax. Better quality but still below standard attention.
  • RetNet (Sun et al., 2023) — linear attention + exponential decay (retention). Competitive with Transformers on language modeling.
  • RWKV (Peng et al., 2023) — "Receptance Weighted Key Value" — linear attention variant trained at scale (14B params). Competitive with same-size Transformers.
  • Mamba (Gu & Dao, 2023) — selective state-space model with data-dependent transitions. Matches Transformer quality with linear-time inference.
  • GLA (Yang et al., 2024) — Gated Linear Attention with data-dependent gating. Bridges linear attention and Mamba.

11 · Limitations

  • Quality gap: Pure linear attention (without decay/gating) underperforms softmax on tasks requiring precise token-to-token comparison (retrieval, copying).
  • State size: The D×d state can be large. For D=d=128, that's 16K values per layer — fine for one sequence, but grows with head count and layers.
  • Training efficiency: The recurrent form is sequential (O(N) steps). Training uses the parallel form (chunk-wise), but this adds implementation complexity.
  • In-context learning: Softmax attention's ability to form sharp, sparse patterns (one-hot-like) seems important for in-context learning. Linear attention's softer patterns may limit this.

12 · Summary

  • Linear attention replaces softmax with a decomposable kernel φ(q)·φ(k), enabling the associativity trick: Q·(K^T·V) instead of (Q·K^T)·V.
  • Complexity drops from O(N²d) to O(Nd²) — linear in sequence length.
  • Causal linear attention is equivalent to a linear RNN with a matrix-valued state, enabling O(1) per-token inference.
  • Adding decay and gating (RetNet, RWKV, GLA, Mamba) bridges linear attention with state-space models and closes the quality gap.
  • Linear attention is now competitive with softmax transformers for language modeling, though softmax still wins on retrieval-heavy tasks.
One-line summary: Linear attention swaps softmax for a decomposable kernel, enabling O(N) attention via the associativity trick — and when combined with decay/gating, matches Transformer quality.
