Sparse Attention

ML Series · Transformers · Variants

1 · The O(N²) Problem

Standard attention computes pairwise scores between all N tokens, creating an N×N attention matrix. For a sequence of 4096 tokens, that's 16.7 million entries. For 128K tokens, it's 16.4 billion. Both memory and compute scale as O(N²).

Standard attention complexity: Compute: O(N² · d) — N² dot products, each of dimension d Memory: O(N²) — store the full attention matrix HBM: O(N² + Nd) — read/write attention matrix + Q,K,V For N = 128,000, d = 128: Attention matrix: 128K × 128K × 2 bytes = 32 GB (just one head, one layer!)

The insight behind sparse attention: most attention weights are near zero. In a trained transformer, each token attends strongly to only a few other tokens — nearby tokens, special tokens (CLS, period), and a handful of semantically related tokens. The O(N²) computation mostly produces zeros.

2 · The Sparse Attention Idea

Instead of computing all N² attention scores, sparse attention computes only a predetermined subset of entries. The rest are implicitly treated as -∞ (zero after softmax).

Standard: Attn(Q, K, V) = softmax(Q · K^T / √d) · V (N² scores) Sparse: Attn(Q, K, V) = softmax(M ⊙ (Q · K^T / √d)) · V (S scores, S ≪ N²) Where M is a sparse binary mask: M[i,j] = 1 if token i attends to token j

If each token attends to only k other tokens (where k ≪ N), complexity drops to O(N·k·d) — linear in N when k is constant.

3 · Fixed Sparsity Patterns

Each token attends to its w nearest neighbors. Captures local context efficiently. Complexity: O(N·w·d). This is the basis for Mistral's sliding window attention.

Each token attends to every s-th token across the full sequence. Provides long-range connectivity with O(N·(N/s)·d) cost. Combined with local attention, this forms the Sparse Transformer pattern.

Designate a small set of tokens (e.g., [CLS], first token, every 512th token) as "global" — they attend to and are attended by all other tokens. Acts as information hubs.

Each token randomly attends to a fixed number of tokens. Surprisingly effective because random connections provide short paths between any two tokens in the graph.

4 · Strided (Factorized) Attention

The original Sparse Transformer (Child et al., 2019) introduced factorized attention: split the attention into two patterns applied in alternating heads:

Pattern A — Local: token i attends to tokens [i-w, ..., i] Pattern B — Strided: token i attends to tokens {j : j mod s = i mod s} Combined receptive field after 2 layers: Layer 1 (local): sees w neighbors Layer 2 (strided): each of those w neighbors connects to every s-th token → Full sequence coverage in just 2 layers Complexity per pattern: O(N · √N) (with s = √N) Total: O(N · √N · d) vs O(N² · d) for standard attention
Why √N? If you set the stride s = √N and window w = √N, each pattern computes √N scores per token. Two patterns give √N × √N = N coverage — matching full attention's reach with O(N√N) cost.

5 · Local + Global Attention

Longformer and BigBird refine the approach by combining local sliding-window attention with a small number of global tokens:

Longformer attention pattern for token i: Local: attend to tokens [i-w, ..., i+w] (window of 2w+1) Global: attend to/from designated global tokens (first token, task token) BigBird adds random attention: Local: sliding window of size w Global: g global tokens Random: r random connections per token Complexity: O(N · (w + g + r) · d) — linear in N! Example (Longformer-4096): w = 256, g = 2 Scores per token: 514 (vs 4096 for full attention) Savings: 8× less compute

7 · The Math

Sparse attention with mask M: S = Q · K^T / √d (compute only where M[i,j]=1) S[i,j] = -∞ where M[i,j] = 0 (mask out non-attended pairs) A = softmax(S, dim=-1) (softmax over attended positions only) O = A · V
Pattern Scores per token Complexity Full-sequence coverage
Full attention N O(N²d) 1 layer
Local (window w) w O(Nwd) N/w layers
Strided (stride s) N/s O(N²d/s) 2 layers (with local)
Local + Global (g global) w + g O(N(w+g)d) 2 layers
BigBird (w+g+r) w + g + r O(N(w+g+r)d) 2 layers (theoretical)

8 · Models & Architectures

  • Sparse Transformer (OpenAI, 2019) — factorized attention (local + strided). Applied to images and text generation up to 12K tokens.
  • Longformer (Allen AI, 2020) — local + global attention for documents up to 16K tokens. Used for long-document QA and summarization.
  • BigBird (Google, 2020) — local + global + random. Proved that sparse attention with random connections is a universal approximator of full attention.
  • LED (Longformer Encoder-Decoder) — Longformer applied to seq2seq tasks (summarization).
  • Mistral 7B — sliding window attention (local only, w=4096) combined with rolling KV cache for efficient inference.
  • LongT5 (Google) — transient global attention where global tokens are computed as summaries of local blocks.

9 · Implementation

def sparse_attention(Q, K, V, window_size, global_indices): N, d = Q.shape O = zeros(N, d) # Local attention (sliding window) for i in range(N): start = max(0, i - window_size) end = min(N, i + window_size + 1) # Local scores local_K = K[start:end] local_V = V[start:end] scores = Q[i] @ local_K.T / sqrt(d) # Global scores (attend to global tokens too) global_K = K[global_indices] global_V = V[global_indices] global_scores = Q[i] @ global_K.T / sqrt(d) # Combine and softmax all_scores = cat([scores, global_scores]) all_V = cat([local_V, global_V]) weights = softmax(all_scores) O[i] = weights @ all_V return O # In practice, use block-sparse matrix operations (e.g., Triton kernels) # for GPU-efficient sparse attention computation
GPU efficiency: Naive sparse attention implementations are slower than dense attention despite fewer FLOPs, because sparse memory access patterns defeat GPU parallelism. Efficient implementations use block-sparse formats (e.g., Triton's block-sparse matmul) where sparsity is at the block level (e.g., 64×64 blocks), not individual elements.

10 · Sparse Attention vs Flash Attention

Aspect Sparse Attention Flash Attention
Approach Reduce which scores to compute Reduce memory IO for all scores
FLOPs O(N·k·d) (fewer ops) O(N²·d) (same ops, faster IO)
Memory O(N·k) attention matrix O(N) (never materializes full matrix)
Quality Approximation (some info lost) Exact (numerically identical)
Best for Very long sequences (>32K) Moderate sequences (up to ~128K)
Combinable? Yes! Block-sparse Flash Attention exists (best of both worlds)
Modern trend: Flash Attention has largely displaced sparse attention for sequences up to 128K. For sequences beyond that, block-sparse Flash Attention combines both: sparse patterns at the block level with IO-efficient tiled computation within each block.

11 · Summary

  • Sparse attention reduces the O(N²) cost by computing only a subset of attention scores, determined by a fixed or learned mask.
  • Local (sliding window), strided, global, and random patterns can be combined for linear-time attention with full-sequence coverage.
  • Models like Longformer, BigBird, and Sparse Transformer pioneered these patterns for long-document tasks.
  • GPU efficiency requires block-sparse implementations; naive sparse attention is slower than dense on modern hardware.
  • Flash Attention has partially superseded sparse attention for moderate lengths, but sparse + Flash combinations push the frontier for 100K+ sequences.
One-line summary: Sparse attention skips most of the N² attention scores using structured patterns — trading exact attention for linear scaling.
