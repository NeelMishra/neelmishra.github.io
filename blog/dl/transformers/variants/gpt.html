    GPT: Generative Pre-trained Transformer

    ML Series · March 2026

    The Generative Paradigm

    In June 2018, OpenAI published the original GPT paper. The core idea was straightforward: take the decoder half of the transformer, pretrain it on a large corpus using next-token prediction, then fine-tune on downstream tasks. Unlike BERT, which processes all tokens at once with bidirectional attention, GPT generates text left to right, predicting each token based only on the tokens that came before it.

    This autoregressive approach turned out to be extraordinarily scalable. As models grew from GPT (117M parameters) to GPT-2 (1.5B) to GPT-3 (175B) to GPT-4, their capabilities expanded in ways that surprised even their creators. The decoder-only architecture became the foundation of modern large language models.

    Decoder-Only Architecture

    GPT uses only the decoder portion of the original transformer, but without the cross-attention layers (since there is no encoder to attend to). Each decoder block contains:

    • Masked self-attention: each token can attend only to itself and previous tokens
    • Feed-forward network: applied independently to each position
    • Layer normalization and residual connections

    The key difference from BERT is the attention mask. In BERT, every token attends to every other token. In GPT, attention is causal: token at position i can only attend to tokens at positions 0 through i. This is enforced by a triangular mask applied to the attention scores before softmax.

    Causal Masking

    The causal mask is a lower-triangular matrix that prevents each token from attending to future tokens. Before computing softmax over attention scores, positions above the diagonal are set to negative infinity, which drives their attention weights to zero.

    Each row represents a query token, each column a key token. The purple cells (1) indicate allowed attention; the pink cells (0) are masked. Token "sat" at position 2 can attend to "The" (0), "cat" (1), and "sat" (2), but not to "on" (3), "the" (4), or "mat" (5).

    Pretraining: Next Token Prediction

    The training objective is simple: given a sequence of tokens, predict the next token at every position. For a sequence [t1, t2, ..., tn], the model predicts:

    • P(t2 | t1)
    • P(t3 | t1, t2)
    • P(t4 | t1, t2, t3)
    • ... and so on for every position

    The loss is the average cross-entropy between the predicted probability distribution and the actual next token across all positions. This is computationally efficient because every position in the sequence provides a training signal (unlike BERT's MLM, where only 15% of positions contribute to the loss).

    Training efficiency: Next-token prediction provides a training signal at every position in the sequence. A single forward pass through 2048 tokens yields 2047 prediction losses. This is roughly 6.5x more sample-efficient than BERT's MLM, where only about 15% of tokens are masked.

    Scaling Laws: GPT-2 to GPT-4

    One of the most important discoveries in the GPT line of research is the existence of predictable scaling laws. As you increase model size, dataset size, and compute, performance improves smoothly and predictably on a log-log scale. This means you can forecast how well a larger model will perform before spending the compute to train it.

    Model Parameters Training Data Context Length Year
    GPT 117M BookCorpus (4.5 GB) 512 tokens 2018
    GPT-2 1.5B WebText (40 GB) 1024 tokens 2019
    GPT-3 175B Filtered CommonCrawl + books + Wikipedia (570 GB) 2048 tokens 2020
    GPT-4 Undisclosed (rumored ~1.8T MoE) Undisclosed (multi-trillion tokens) 8K / 32K / 128K tokens 2023

    The Kaplan et al. (2020) scaling laws paper showed that loss decreases as a power law with respect to model parameters, dataset size, and compute budget. The Chinchilla paper (Hoffmann et al., 2022) refined these laws and showed that many models were undertrained: you should scale data and model size proportionally for optimal compute efficiency.

    In-Context Learning

    GPT-3's most surprising capability was in-context learning: the ability to perform new tasks at inference time without any gradient updates, simply by conditioning on a few examples provided in the prompt. This was qualitatively different from anything seen before.

    Zero-Shot

    The model receives only a task description:

    Translate English to French:
sea otter =>

    The model must infer the task from the instruction alone.

    One-Shot

    The model receives one example:

    Translate English to French:
sea otter => loutre de mer
cheese =>

    Few-Shot

    The model receives several examples:

    Translate English to French:
sea otter => loutre de mer
peppermint => menthe poivree
plush giraffe => girafe en peluche
cheese =>

    GPT-3 showed that few-shot performance improves with model scale. Smaller models see little benefit from more examples, but the 175B parameter model can leverage them effectively. This established in-context learning as a primary mode of interaction with large language models and reduced the need for task-specific fine-tuning.

    Why does in-context learning work? The full mechanism is still an active area of research. One hypothesis is that during pretraining on diverse web text, the model encounters many implicit "task demonstrations" (e.g., a document that lists translations, a page with question-answer pairs). The model learns to recognize and continue patterns. At inference time, the few-shot examples activate this pattern-matching capability.

    Few-Shot Prompting vs Fine-Tuning

    The GPT approach offers two paths to task adaptation:

    • Few-shot prompting: No training required. Provide examples in the prompt. Works for many tasks but performance depends on prompt design and model scale.
    • Fine-tuning: Train the model on task-specific data with gradient updates. Higher performance ceiling but requires labeled data and compute.

    In practice, modern workflows often combine both: fine-tune a base model on instruction-following data (instruction tuning), then use few-shot prompting at inference time to steer the model toward specific tasks. RLHF (Reinforcement Learning from Human Feedback) further aligns the model with human preferences.

    Architectural Details

    Modern GPT-style models include several refinements over the original architecture:

    • Pre-norm: Layer normalization is applied before (not after) each sub-layer. This stabilizes training at large scales.
    • RoPE or ALiBi: Rotary positional embeddings or attention with linear biases replace learned absolute positional embeddings, enabling better length generalization.
    • Grouped query attention: Key and value heads are shared across multiple query heads, reducing memory usage during inference with minimal quality loss.
    • SwiGLU activation: Replaces ReLU in the feed-forward network for improved training dynamics.
    • Flash Attention: An IO-aware exact attention algorithm that reduces memory reads/writes, enabling longer context windows without approximation.

    Limitations

    • No bidirectional context: Each token can only see tokens before it. For tasks requiring full-context understanding (e.g., classification), this is a disadvantage compared to encoder-only models.
    • Hallucination: GPT models generate plausible-sounding text that may be factually incorrect. The model maximizes likelihood of the next token, not factual accuracy.
    • Compute cost: Training GPT-4 scale models requires millions of dollars in compute. Inference costs are also significant for billion-parameter models.
    • Context window: Despite improvements, the finite context window means the model cannot consider arbitrarily long histories.
