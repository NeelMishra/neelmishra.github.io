    Regularization and Early Stopping

    July 2026 · 15 min read

    GBM is powerful because it keeps lowering training loss round after round. That is also the danger. Without regularization, the sequence of trees can eventually fit noise, outliers, and quirks of the training split. A well-tuned GBM is therefore not just "many trees"; it is many trees restrained by shrinkage, limited depth, sampling, and validation-based stopping.

    Learning Rate and Number of Trees

    The learning rate $\nu$ scales every tree before it is added:

    $$F_m(x) = F_{m-1}(x) + \nu h_m(x).$$

    Smaller $\nu$ means each tree has less influence, so the model needs more boosting rounds. This tradeoff is central to GBM. A small learning rate with enough trees usually generalizes better than a large learning rate with few trees, because the optimization path is smoother and less likely to overshoot.

    Useful default. A learning rate around $0.05$ to $0.1$ is often a reasonable starting point for tabular problems. If validation loss is noisy or overfitting arrives quickly, lower the learning rate and allow more trees.

    Tree Capacity Knobs

    The base tree controls how complex each correction can be. GBM usually prefers shallow trees because the ensemble will add many of them.

    max depth

    Limits the number of nested splits. It controls interaction order and the size of each correction.

    min samples leaf

    Prevents leaves supported by only a few rows. Larger leaves make smoother corrections.

    min split gain

    Requires a split to improve the objective by enough before it is accepted.

    leaf value limits

    Caps individual leaf steps, useful when gradients are large or labels contain outliers.

    Stochastic Gradient Boosting

    Classic GBM trains each tree on all rows and all candidate features. Stochastic GBM injects randomness by using a row subsample, a column subsample, or both at each round. This does two things: it speeds up training and decorrelates nearby trees. The idea is related to random forests, but the trees are still sequential and still fit current gradients.

    • Row subsampling trains each boosting round on a random fraction of rows, often $0.5$ to $0.9$.
    • Column subsampling restricts the split search to a random subset of features.
    • Per-level or per-node sampling repeats feature sampling deeper inside a tree.

    Early Stopping

    The most important regularizer is a validation set. After every boosting round, measure loss on data not used to fit that tree. Training loss will usually keep falling. Validation loss falls at first, reaches a minimum, then rises when the ensemble begins fitting training-specific noise. Early stopping selects the number of trees at the validation minimum.

    A Practical Tuning Order

    A reliable tuning process starts with a small, conservative model, then increases capacity only when validation loss indicates underfitting:

    1. Choose the loss and metric that match the problem.
    2. Start with shallow trees, moderate leaf size, and $\nu$ around $0.05$ to $0.1$.
    3. Use a high maximum number of trees and let early stopping select the actual count.
    4. If both train and validation loss are high, increase depth or reduce leaf size.
    5. If train loss is low but validation loss rises early, lower depth, increase leaf size, add subsampling, or lower the learning rate.

    Reading Failure Modes

    When a GBM fails, the validation curve usually tells the story. If training and validation losses both plateau high, the model lacks capacity or the features are insufficient. If training loss keeps improving while validation loss rises, the model has too much capacity for the data. If validation loss is unstable from one run to another, the split may be small, labels may be noisy, or row subsampling may be too aggressive.

    The regularization knobs above describe classic GBM. Part 5 shows how modern libraries add more structure around the same ideas: second-order split gains, histogram bins, explicit penalties, default missing-value directions, and categorical handling.
