    Loss Functions and Pseudo-Residuals

    July 2026 · 17 min read

    Part 1 described GBM as "fit the next tree to the residuals." That sentence is exact for squared-error regression, but GBM is more general. The target for the next tree is always the negative gradient of the loss with respect to the current prediction. For losses other than squared error, that gradient target is called a pseudo-residual.

    The Stage-Wise Objective

    Suppose the current model is $F_{m-1}(x)$ and the next tree is $h_m(x)$. Boosting chooses a function that lowers the empirical risk

    $$\mathcal{L}(F) = \sum_{i=1}^{N} L\big(y_i, F(x_i)\big).$$

    Instead of searching over all possible functions at once, GBM takes one functional gradient step. It computes

    $$r_i^{(m)} = -\left[\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right]_{F=F_{m-1}},$$

    fits a tree to pairs $(x_i, r_i^{(m)})$, and adds the tree to the ensemble. The loss decides what "mistake" means; the tree decides how to generalize that mistake across regions of feature space.

    Common Losses

    Task Loss Prediction scale Negative gradient
    Regression $\tfrac12(y-F)^2$ Raw numeric prediction $y-F$, the ordinary residual
    Binary classification $-y\log p-(1-y)\log(1-p)$ Logit $F$, with $p=\sigma(F)$ $y-p$, the probability error
    Robust regression Huber loss Raw numeric prediction Residual clipped to a threshold
    Counts Poisson deviance Log rate $F$, with $\lambda=e^F$ $y-\lambda$, observed minus expected count

    The Initial Prediction

    The starting prediction $F_0$ is the best constant model for the chosen loss. For squared error it is the mean target. For absolute error it is the median. For binary logistic loss it is the log-odds of the positive class:

    $$F_0 = \log\frac{\bar{y}}{1-\bar{y}}.$$

    This matters because every later tree is a correction around that baseline. A poor baseline wastes early rounds correcting an avoidable offset.

    Binary Logistic GBM

    For binary labels $y_i \in \{0,1\}$, GBM usually maintains a raw score $F(x)$, not a probability directly. The probability is obtained by the sigmoid function:

    $$p_i = \sigma(F(x_i)) = \frac{1}{1+e^{-F(x_i)}}.$$

    The log loss for one example is

    $$L_i = -y_i\log p_i - (1-y_i)\log(1-p_i).$$

    Differentiating with respect to the logit gives $\partial L_i / \partial F = p_i-y_i$, so the pseudo-residual is $y_i-p_i$. This is intuitive: if the label is $1$ and the model predicts $p=0.2$, the residual is $+0.8$; if the label is $0$ and the model predicts $p=0.8$, the residual is $-0.8$.

    Scores versus probabilities. The trees add to the logit $F(x)$. Probabilities are derived at prediction time with a sigmoid. This keeps the additive model unconstrained while ensuring final probabilities remain between $0$ and $1$.

    Leaf Values and Line Search

    Fitting a tree to pseudo-residuals chooses the tree structure, but each leaf still needs a value. In squared-error GBM, the leaf value can be the average residual in that leaf. More generally, a leaf value is chosen by a one-dimensional minimization:

    $$\gamma_j = \arg\min_{\gamma}\sum_{x_i \in R_j} L\big(y_i, F_{m-1}(x_i)+\gamma\big),$$

    where $R_j$ is a terminal region of the new tree. For logistic loss, many implementations use a Newton step inside the leaf:

    $$\gamma_j \approx \frac{\sum_{x_i \in R_j}(y_i-p_i)}{\sum_{x_i \in R_j}p_i(1-p_i)}.$$

    The numerator is the sum of gradients; the denominator is the sum of curvatures. This is the bridge from first-order gradient boosting to the second-order systems discussed in Part 5.

    p[i] = current_probability_or_prediction(x[i])
    residual[i] = negative_gradient(loss, y[i], p[i])
    fit tree structure to residual[i]
    for each leaf R:
        leaf_value = best_step_for_loss(R)
    add learning_rate * leaf_value to rows in that leaf

    Robust Losses

    Squared error gives very large gradients to outliers because the residual grows without bound. Huber loss behaves like squared error near zero and absolute error far away. Its pseudo-residual is a clipped residual, so one extreme point cannot dominate an entire tree. Quantile loss, used for prediction intervals, pushes predictions toward a chosen percentile instead of the conditional mean.

    The main takeaway is that GBM is a loss-driven framework, not one fixed algorithm. Changing the loss changes the pseudo-residuals, the initial prediction, and the leaf update, while the same shallow-tree machinery remains in place. Part 3 focuses on that tree machinery.
