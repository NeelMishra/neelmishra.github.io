    Gradient Boosted Machines: Foundations

    July 2026 · 18 min read

    A gradient boosted machine, usually shortened to GBM, is an additive model made from many small decision trees. A single tree partitions the feature space once and predicts a constant inside each leaf. A GBM keeps adding new shallow trees, each one correcting the current ensemble's mistakes. The result is a flexible model that usually beats one large tree because it learns in many controlled steps instead of memorizing the training set in one uncontrolled jump.

    This series is the GBM companion to the Decision Trees deep dive. Decision trees supplied the base learner: axis-aligned splits, leaf values, and recursive growth. GBM supplies the training strategy around those trees: residuals, pseudo-residuals, shrinkage, subsampling, and modern second-order implementations. The matching implementation series lives in From Scratch · C++ · Gradient Boosted Machines.

    The Additive Tree Model

    The model is a sum of trees. For regression the prediction after $M$ rounds is

    $$F_M(x) = F_0(x) + \nu \sum_{m=1}^{M} h_m(x),$$

    where $F_0$ is a simple starting prediction, each $h_m$ is a shallow regression tree, and $0 < \nu \le 1$ is the learning rate. The important word is additive. Tree $m$ is not replacing the older trees; it is adding one correction to the current prediction.

    Single tree

    One model tries to explain the target at once. A deep tree can fit complex patterns, but it has high variance and can chase noise.

    Random forest

    Many deep trees are trained independently and averaged. The main effect is variance reduction through bagging and feature randomness.

    GBM

    Many shallow trees are trained sequentially. The main effect is bias reduction through small, directed corrections.

    Residual Fitting for Squared Loss

    For squared-error regression, GBM is easiest to see through residuals. Start with the mean target, $F_0 = \bar{y}$. At round $m$, compute the residual for every row:

    $$r_i^{(m)} = y_i - F_{m-1}(x_i).$$

    Then train a regression tree $h_m$ to predict those residuals and update the model:

    $$F_m(x) = F_{m-1}(x) + \nu h_m(x).$$

    The tree is therefore learning what the current ensemble still cannot explain. If a row is underpredicted, its residual is positive and the next tree pushes the prediction upward. If a row is overpredicted, its residual is negative and the next tree pushes downward.

    ▶ Residual Fitting, One Boosting Round at a Time

    The green curve is the current ensemble. Orange stems are residuals. Each step adds a small tree-shaped correction, so the ensemble walks toward the nonlinear target in controlled increments.

    Why This Is Gradient Descent

    The residual recipe is not a trick specific to squared error; it is the first example of gradient descent in function space. Ordinary gradient descent updates parameters by moving opposite the gradient. GBM updates predictions by adding a function that approximates the negative gradient of the loss with respect to the current prediction.

    For squared loss, $L_i = \tfrac12(y_i - F(x_i))^2$, the derivative with respect to $F(x_i)$ is

    $$\frac{\partial L_i}{\partial F(x_i)} = F(x_i) - y_i.$$

    The negative gradient is therefore $y_i - F(x_i)$, exactly the residual. For logistic loss the negative gradient becomes a probability error. For Huber loss it becomes a clipped residual. Part 2 builds those pseudo-residuals carefully.

    initialize F0 to the best constant prediction
for m = 1 to M:
    for each row i:
        residual[i] = negative_gradient(loss, y[i], F(x[i]))
    fit a shallow regression tree h_m to residual
    choose a leaf value for each terminal region
    F(x) = F(x) + learning_rate * h_m(x)

    Why Many Small Trees Work

    GBM usually uses shallow trees, often depth 2 to 6. A shallow tree is weak: it captures only a few interactions. But adding many weak trees gives the model two useful properties. First, each step is constrained, so the model cannot immediately contort around noise. Second, later trees focus on errors left by earlier trees, so hard regions receive more modeling capacity.

    Boosting is sequential. A random forest can train every tree independently. A GBM cannot, because tree $m$ depends on the predictions made by trees $1$ through $m-1$. That dependency is the cost of boosting, and it is also the source of its accuracy.
