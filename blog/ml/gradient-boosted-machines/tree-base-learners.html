    Trees as Base Learners

    July 2026 · 16 min read

    A GBM uses decision trees, but it uses them differently from a standalone tree or a random forest. The tree is not trained to predict $y$ directly. It is trained to predict a gradient target produced by the current ensemble. Because that target changes after every boosting round, each tree is a local correction rather than a complete model.

    A CART Regression Tree on Residuals

    The base learner is usually a CART regression tree. At a node it considers candidate splits of the form $x_f < t$. For each split it measures how well the residuals can be represented by two constants, one per child. With residual targets $r_i$, the split score can be written as a reduction in squared error:

    $$\Delta = \sum_{i \in R}(r_i-\bar{r}_R)^2 - \sum_{i \in R_L}(r_i-\bar{r}_{L})^2 - \sum_{i \in R_R}(r_i-\bar{r}_{R})^2.$$

    The best split is the one with the largest reduction. This is the same sorted-sweep idea from CART and pruning, but the labels are now pseudo-residuals.

    build_residual_tree(rows, depth):
    if depth limit or leaf is too small:
        return leaf(mean residual in rows)
    best_split = split with largest squared-error reduction
    left_rows, right_rows = partition rows by best_split
    return node(best_split,
                build_residual_tree(left_rows, depth + 1),
                build_residual_tree(right_rows, depth + 1))

    Depth Controls Interactions

    A depth-one tree, also called a stump, can use only one feature in a correction. A depth-two tree can model a two-way interaction, because the second split happens inside a region created by the first split. More depth means each tree can express higher-order feature interactions, but it also raises variance.

    Depth 1

    Main effects only. Useful for very smooth additive patterns and as a conservative baseline.

    Depth 2 to 4

    Common default range. Captures moderate interactions while keeping each tree weak.

    Depth 6+

    High capacity. Often needs stronger shrinkage, subsampling, or early stopping.

    Weak does not mean useless. In boosting, a weak tree is useful precisely because it cannot solve the whole task alone. It contributes a small correction, then the next tree works on what remains.

    Terminal Regions and Leaf Values

    Once the tree structure is chosen, it partitions the feature space into leaves $R_1,\dots,R_J$. For squared-error GBM, the leaf value is simply the average residual in that region:

    $$\gamma_j = \frac{1}{|R_j|}\sum_{x_i \in R_j} r_i.$$

    Every training row in that region receives the same update $\nu\gamma_j$. This piecewise-constant update is why GBM predictions are sums of small rectangular corrections. For logistic or Poisson losses, the leaf value can use a Newton-style ratio, but the same region structure remains.

    The Prediction Update

    After the leaf values are assigned, the tree becomes a function:

    $$h_m(x) = \sum_{j=1}^{J}\gamma_j \mathbf{1}[x \in R_j].$$

    The ensemble update is

    $$F_m(x) = F_{m-1}(x) + \nu h_m(x).$$

    The learning rate $\nu$ is not a small detail. It is the knob that turns a tree from a full jump into a controlled step. A lower learning rate usually requires more trees, but it often improves generalization.

    Why Trees Fit Boosting So Well

    GBM can use other base learners, but trees are the dominant choice for tabular data because they bring several practical advantages:

    • They handle nonlinear thresholds without feature scaling.
    • They naturally model interactions through nested splits.
    • They are robust to monotone transformations of individual features.
    • They produce sparse local updates, so later rounds can focus on hard regions.

    There is also an important limitation: ordinary CART trees are greedy. Every boosting round inherits the local decisions made by its tree. Modern GBM systems reduce the cost of searching for good splits with histograms, second-order split gains, and clever missing-value handling, covered in Part 5.

    Complete Training Loop

    Putting the pieces together, tree-based GBM is a short loop around a regression-tree learner:

    F = best constant prediction
for each round m:
    residuals = negative gradient at F
    tree = CART regression tree fit to residuals
    for each leaf:
        compute leaf value for the original loss
    F = F + learning_rate * tree
return sum of all trees

    The implementation companion builds that loop in plain C++ in Regression GBM from Scratch. Before implementation, Part 4 explains how the loop is regularized so it does not simply overfit one small correction at a time.
