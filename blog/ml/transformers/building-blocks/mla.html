    Multi-Head Latent Attention: Extreme Compression via Low-Rank Projection

    January 2025 · 12 min read

    GQA (part 11) reduces the KV cache by grouping heads. Multi-Head Latent Attention (MLA, DeepSeek-V2) goes further: instead of caching full Key and Value vectors, compress them into a low-rank latent space $\mathbf{c}_{KV} \in \mathbb{R}^{d_c}$ (where $d_c \ll d_{\mathrm{head}}$), cache the latent, and reconstruct $K, V$ on-the-fly during attention. This reduces cache by a factor of $(d_{\mathrm{head}} / d_c) \times H$ compared to standard MHA. MLA also decouples position embeddings (RoPE) into a separate rotated path for efficiency. DeepSeek-V2 achieves 1.7× longer context than comparable models on the same memory budget, or inference 2.3× faster with comparable quality.

    The Compression Idea

    In standard attention, each token position caches the full $d_{\mathrm{head}}$-dimensional Key and Value vectors for all $H$ heads (or $G$ heads in GQA). This is expensive. MLA observes that over multiple positions, much of the variation in $K$ and $V$ is redundant or low-rank. We can project $K$ and $V$ into a smaller latent space $d_c$ (e.g., $d_c = 512$ instead of $8 \times 128 = 1024$ for 8 heads), cache the latent codes, and reconstruct full $K, V$ during attention via an up-projection. The reconstruction is on-the-fly and differentiable, so training is unchanged; only caching and inference patterns shift.

    Forward Pass: Compression and Reconstruction

    Given input $X_t$ at position $t$:

    1. Project to latent: $\mathbf{c}_{K,t} = X_t W^{DK} \in \mathbb{R}^{d_c}$, $\mathbf{c}_{V,t} = X_t W^{DV} \in \mathbb{R}^{d_c}$, where $W^{DK}, W^{DV} \in \mathbb{R}^{d_{\mathrm{model}} \times d_c}$ are down-projection matrices (smaller).
    2. Cache latent: Store $\mathbf{c}_{K,t}$ and $\mathbf{c}_{V,t}$ (only $2 d_c$ scalars per position, tiny!).
    3. Query projection: Project current input to queries as usual: $Q_t = X_t W^Q \in \mathbb{R}^{H \times d_k}$.
    4. On-the-fly reconstruction (per cached position $\tau$): For each cached position $\tau$, reconstruct: $$K_\tau = \mathbf{c}_{K,\tau} W^{UK} \in \mathbb{R}^{H \times d_k}, \quad V_\tau = \mathbf{c}_{V,\tau} W^{UV} \in \mathbb{R}^{H \times d_k}$$ where $W^{UK}, W^{UV} \in \mathbb{R}^{d_c \times (H \times d_k)}$ are up-projection matrices.
    5. Attention: Compute standard scaled-dot-product attention with the reconstructed $K, V$ — no structural changes to the attention mechanism.

    The trade-off: we pay a small cost (matrix multiplies for up-projection) during attention, but cache memory shrinks dramatically.

    Decoupled Rotary Embeddings

    MLA also decouples RoPE (Rotary Position Embeddings) from the main attention. Instead of applying RoPE to $Q$ and $K$ directly, MLA maintains a separate small rotated query/key path:

    • Main attention: Uses the latent-reconstructed $K, V$ (unrotated or rotated as one block).
    • Rotated attention (small): Projects $Q$ and $K$ to a smaller rotated subspace, applies RoPE there, and concatenates outputs.

    This allows precise position tracking without per-head RoPE overhead, further reducing computation and cache costs.

    Complexity and Memory Gains

    Aspect MHA MLA (DeepSeek-V2)
    KV cache per layer per token $2H d_k$ (e.g., $16 \times 128 = 2048$ for 16 heads) $2 d_c$ (e.g., 512 for $d_c = 256$)
    Total cache (100 layers, $t$ tokens) $2H d_k \times 100 \times t$ $2 d_c \times 100 \times t$ (~4× smaller)
    Reconstruction cost per step N/A $O(d_c \times H d_k)$ FLOPs (small, batched)
    Quality (on benchmark) Baseline 100% ~99–100% (minimal loss)

    DeepSeek-V2 Deployment

    DeepSeek-V2 (2024) is a 236B parameter mixture-of-experts model that uses MLA throughout. Reported results:

    • Context length: Supports 128K context (vs. ~4–32K for typical models of similar size).
    • Inference speed: 2.3× faster than standard attention on long sequences.
    • Quality: Matches or exceeds comparable models (e.g., Llama 2 70B, GPT-3.5) on standard benchmarks.
    • Training efficiency: Reduced cache during training speeds up backprop and allows larger batch sizes.

    MLA is a key ingredient in DeepSeek-V2's ability to achieve high quality at long context on limited memory.

    Implementation Sketch

    def multi_head_latent_attention(X, d_c=512):
    """
    X: input tensor (batch, seq_len, d_model)
    d_c: latent compression dimension (e.g., 512)
    """
    # Down-projection to latent
    c_k = X @ W_DK  # (batch, seq_len, d_c)
    c_v = X @ W_DV  # (batch, seq_len, d_c)
    
    # Cache latent (very small)
    kv_cache_latent = (c_k, c_v)  # total: 2 * d_c per position
    
    # Query projection (standard)
    Q = X @ W_Q  # (batch, seq_len, H * d_k)
    Q = Q.view(batch, seq_len, H, d_k)
    
    # Retrieve cached latents (autoregressive)
    c_k_cache, c_v_cache = kv_cache_latent  # from previous positions
    
    # Up-project cached latents on-the-fly
    K_cached = c_k_cache @ W_UK  # (batch, past_len, H * d_k)
    V_cached = c_v_cache @ W_UV  # (batch, past_len, H * d_k)
    K_cached = K_cached.view(batch, past_len, H, d_k)
    V_cached = V_cached.view(batch, past_len, H, d_k)
    
    # Standard attention (scores against reconstructed K, V)
    scores = Q @ K_cached.transpose(-2, -1) / math.sqrt(d_k)
    attn_weights = softmax(scores, dim=-1)
    output = attn_weights @ V_cached
    
    # Output projection
    output = output.view(batch, seq_len, H * d_k) @ W_O
    return output

    Trade-offs and Considerations

    Computation vs. Memory: MLA shifts cost from memory to compute (reconstruction FLOPs). On memory-bound hardware (GPUs during inference), this is a win. On compute-bound scenarios, the gain is smaller.

    Latent dimension $d_c$: Smaller $d_c$ saves memory but may hurt quality. DeepSeek-V2 uses $d_c = 512$ with $H = 16$ heads ($d_k = 128$), achieving good balance.

    Training cost: MLA introduces no structural changes; training is standard. The latent projections are learned end-to-end. No special initialization tricks are needed.
