    Global Attention: Bridging Local and Full

    January 2026 · 11 min read

    Part 5 introduced sliding-window attention: fast but local. But models need to relate distant tokens too. Global attention is a hybrid: a small set of "special" tokens attend to and are attended by the entire sequence, while regular tokens use sliding-window attention. This is the key innovation in Longformer that preserves long-range modeling capacity without quadratic cost.

    The Problem with Pure Windowing

    Sliding-window attention with window $w$ has a receptive field that grows linearly with depth: after $k$ layers, a token can see at most $kw$ tokens away. For a $w=512$ window and 12 layers, that's $6144$ tokens—reasonable for most documents. But for truly long sequences or when you need immediate global context (e.g., classification), this is insufficient.

    Global attention is the solution: designate $g$ special tokens (e.g., $[CLS]$, task prompts) that act as "hub" nodes, attending to all $n$ positions and being attended by all.

    The Hybrid Pattern

    In a sequence of length $n$, designate positions $i_1, \ldots, i_g$ as global. The attention matrix becomes:

    • Regular tokens: Attend to sliding window + the $g$ global tokens.
    • Global tokens: Attend to all $n$ positions.

    So row $i$ of the attention matrix is:

    • Zero-masked outside window $[i-w, i+w]$, EXCEPT for global position columns which are unmasked.

    Similarly, every token attends to global positions (because every column corresponding to a global position is unmasked).

    Example: Classification with [CLS]

    In BERT-style models, the first token is $[CLS]$, used for classification. Make it global:

    • $[CLS]$ (position 0) attends to all $n$ tokens.
    • Token at position $i$ ($i > 0$) attends to positions in $[i-w, i+w]$ and also to position 0 ($[CLS]$).

    This allows $[CLS]$ to aggregate global information, and all other tokens to tap into it via the $[CLS]$ representation.

    Complexity

    SettingTimeSpaceNotes
    Sliding-window alone$O(nwd)$$O(nw)$Local only
    Sliding-window + $g$ global$O(n(w+g)d)$$O(n(w+g))$For $g \ll w$, negligible overhead
    Full attention$O(n^2 d)$$O(n^2)$No sparsity

    If $g=2$ and $w=512$ for a $10000$-token sequence, we save $\sim 95\%$ of the $O(n^2)$ cost while preserving global connectivity.

    Longformer's Implementation

    Longformer (Beltagy et al., 2020) formalizes this:

    • Uses a sliding window of size 512 for most layers.
    • Designates a subset of positions as "global" (configurable; often task-specific).
    • Achieves better than linear scaling with sequence length on long-document QA tasks.

    Multi-Head with Mixed Patterns

    Longformer uses different attention patterns in different heads:

    • Some heads: sliding-window.
    • Other heads: global attention to specific tokens (e.g., task tokens, sentence boundaries).

    This lets the model mix local and global reasoning within a single attention layer.

    Implementation Sketch

    def global_windowed_attention(Q, K, V, window_size, global_positions):
    """
    Q, K, V: (n, d)
    window_size: w (local window)
    global_positions: list of indices marked as global
    """
    scores = torch.matmul(Q, K.transpose(-1, -2)) / (Q.shape[-1] ** 0.5)
    
    # Create mask: allow window + global positions
    mask = torch.full_like(scores, float('-inf'))
    for i in range(len(Q)):
        # Window: positions [i-w, i+w]
        for j in range(max(0, i-window_size), min(len(Q), i+window_size+1)):
            mask[i, j] = 0
        # Global positions: attend all
        for g in global_positions:
            mask[i, g] = 0
    
    scores = scores + mask
    weights = F.softmax(scores, dim=-1)
    output = torch.matmul(weights, V)
    return output

    The Trade-Off

    Global attention introduces $O(ng)$ overhead compared to pure sliding-window. If $g$ is small and fixed, this is dominated by $O(nw)$. But you recover the ability to model long-range dependencies via the global tokens, which now act as highways for information flow.
