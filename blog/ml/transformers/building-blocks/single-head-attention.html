Single-Head Attention

April 2026 · ML Series · Transformers

Every Transformer layer runs the same core operation: take a query, compare it against a set of keys, and return a weighted mix of the corresponding values. That operation is scaled dot-product attention. This post takes it apart piece by piece: the matrices, the scaling factor, softmax, masking, and what the gradients look like.

1 · The Intuition: Why Attention?

Imagine you are in a vast library. You have a question in mind - your query. The library has thousands of books, each with a label on the spine - the key. And inside every book is the actual content - the value. To answer your question you don’t read every book cover to cover. You glance at the labels, find the most relevant ones, pull those books off the shelf, and combine the information you need.

That is exactly what attention does, but with vectors instead of books.

The bottleneck that attention solves

Before Transformers, sequence models like RNNs and LSTMs processed tokens one at a time, compressing the entire history into a single hidden-state vector. This created an information bottleneck: by the time the model reached the 50th token, the information from token 1 was squeezed through many nonlinear transformations and largely diluted.

The encoder–decoder attention mechanism introduced in Bahdanau et al. (2014) partially solved this by letting the decoder peek at every encoder state. But it was still bolted on top of a recurrent backbone.

The Transformer (Vaswani et al., 2017) went all in: attention is all you need. Every token can directly attend to every other token in a single matrix operation - no recurrence, no convolution. The result is a model that:

  • Has constant-length paths between any two positions (O(1) dependency distance).
  • Is trivially parallelizable because there is no sequential dependency during training.
  • Can learn position-dependent patterns (syntax, coreference, long-range agreement) that RNNs struggle with.
Key insight: Attention replaces the fixed-capacity hidden state with a dynamic, content-based lookup. Each position decides at runtime which other positions are relevant, and by how much.

2 · Query, Key, Value Framework

Every attention operation starts with three ingredients derived from the input. Given an input matrix X ∈ ℜn×d (where n is the sequence length and d is the model dimension), we compute:

Q = X · WQ (queries) shape: [n, dk] K = X · WK (keys) shape: [n, dk] V = X · WV (values) shape: [n, dv]

where WQ ∈ ℜd×dk, WK ∈ ℜd×dk, and WV ∈ ℜd×dv are learned parameter matrices.

What each projection means

  • Query (Q): “What am I looking for?” Each row of Q represents a question that a particular token is asking of the rest of the sequence.
  • Key (K): “What do I have to offer?” Each row of K represents the “label” that describes what information a token carries.
  • Value (V): “Here is my actual content.” Each row of V is the information that will be returned when this token is deemed relevant.

Why three separate projections?

A natural question: why not just use X directly, or use a single projection? The answer is that the roles are fundamentally asymmetric.

  1. Decoupled matching and retrieval. The query–key interaction decides where to attend. The value provides what to retrieve. If we used the same vector for both, the model would be forced to use the same representation for matching and for content - a severe constraint.
  2. Richer function space. With three independent weight matrices the model can learn to project the same input into three different subspaces optimized for their respective roles. Empirically, this dramatically increases expressiveness.
  3. Asymmetric attention in encoder–decoder models. In cross-attention the queries come from the decoder while keys and values come from the encoder. This is only possible because Q, K, V are separate projections.

3 · The Attention Formula Step by Step

The complete formula for scaled dot-product attention is:

Attention(Q, K, V) = softmax( Q · KT / √dk ) · V

This single line hides four distinct computational steps. Let’s peel them apart.

Step 1 - Compute Raw Scores: Q · KT

We compute the dot product between every query vector and every key vector. If we have n tokens, Q is [n, dk] and KT is [dk, n], so the result is an [n, n] matrix.

Entry (i, j) of this matrix is the dot product qi · kj - a scalar that measures how much token i should attend to token j. A higher dot product means greater similarity in the learned subspace.

scores = Q · KT shape: [n, n] scores[i][j] = ∑d Q[i][d] × K[j][d]

Think of it as: for every token, we compute a “compatibility score” with every other token. This is an all-pairs comparison - exactly the n² interactions that give attention its power (and its cost).

Step 2 - Scale by 1/√dk

We divide every entry in the score matrix by √dk, where dk is the dimension of the key vectors.

scaled_scores = scores / √dk

Why? Without scaling, the dot products grow proportionally to dk. When the scores are large, softmax pushes almost all the weight onto a single token (saturation), and the gradients become vanishingly small. Dividing by √dk keeps the variance of the scores at roughly 1 regardless of the head dimension. We’ll prove this rigorously in Section 4.

Step 3 - Softmax (Row-wise)

We apply the softmax function independently to each row of the scaled score matrix. Row i gives us the attention weights that token i assigns to every token in the sequence.

weights = softmax(scaled_scores, dim=-1) shape: [n, n] softmax(z)j = exp(zj) / ∑k exp(zk)

After softmax, each row sums to 1 and every entry is non-negative. The row is now a probability distribution over the sequence - it tells us the proportion of “attention” that this token devotes to each position.

Tokens whose keys were similar to our query get high weight; dissimilar tokens get weight near zero. In practice, the distribution is often fairly peaked: a token might put 60% of its weight on one neighbor, 25% on another, and scatter the rest.

Step 4 - Weighted Sum of Values

Finally, we multiply the weight matrix by the value matrix V to produce the output:

output = weights · V shape: [n, dv]

Row i of the output is a weighted combination of all the value vectors, where the weights were determined in Step 3. In pseudo-math:

output[i] = ∑j=0..n-1 weights[i][j] × V[j]

This is the “retrieval” step: the query found the most relevant keys, and now we blend their values to form the final representation for that position. Because this is a weighted sum (not a hard lookup), the output is differentiable everywhere - which is essential for training with gradient descent.

Putting it all together, single-head attention computes a soft, differentiable lookup: each token sends out a query, matches it against all keys, and retrieves a weighted blend of values. The entire operation is two matrix multiplications and a softmax - nothing more.

4 · Why Scale by √dk?

This is one of the most frequently asked questions about Transformers, and the answer is beautifully simple once you see the math.

The variance argument

Assume the components of the query and key vectors are independent random variables drawn from N(0, 1) - zero mean, unit variance. The dot product of two such vectors q and k, both in ℜdk, is:

q · k = ∑i=1dk qi · ki

Each product qi·ki has:

  • Mean: E[qi·ki] = E[qi]·E[ki] = 0
  • Variance: Var(qi·ki) = E[qi2]·E[ki2] = 1×1 = 1

Because the dk terms are independent, their variances add:

Var(q · k) = dk × 1 = dk

So the standard deviation of the raw dot product is √dk. As dk grows, the scores become larger in magnitude.

What happens to softmax with large inputs?

Softmax is extremely sensitive to the scale of its inputs. If you multiply all inputs by a constant α > 1, the output distribution becomes sharper. In the limit, softmax becomes an argmax - all weight on the largest element.

When the attention scores have standard deviation √dk instead of 1, it’s as if we multiplied the inputs by √dk. The resulting attention weights become near-one-hot: almost all weight on a single token, almost zero everywhere else. This has two catastrophic consequences:

  1. Gradient vanishing. The gradient of softmax with respect to a near-saturated input is approximately 0. Back-propagation stalls.
  2. Loss of expressiveness. A near-one-hot attention is essentially a hard lookup - no blending. The model loses the ability to combine information from multiple positions.

The fix

scaled_scores = (q · k) / √dk Var(scaled_scores) = dk / dk = 1 ✓

By dividing by √dk, we normalize the variance of the scores back to 1 regardless of the head dimension. Softmax receives inputs in a healthy range, produces a soft distribution, and gradients flow freely.

In practice: The original Transformer uses dk = 64, so the scaling factor is 1/√64 = 1/8 = 0.125. Without this factor, dot products would have standard deviation ~8, pushing softmax deep into saturation.

7 · Masking

In many settings we need to prevent certain positions from attending to others. The most common case is the causal mask (also called look-ahead mask) used in decoder self-attention: token i must not attend to any token j > i because those tokens haven’t been generated yet at inference time.

How masking works

After computing the scaled scores but before softmax, we add a mask matrix M to the scores:

Attention(Q,K,V) = softmax( Q·KT / √dk + M ) · V

The mask M has the same shape as the score matrix [n, n]. For positions that are allowed, M[i][j] = 0 (no effect). For positions that are forbidden, M[i][j] = −∞.

Why −∞ works

When we add −∞ to a score and then apply softmax:

softmax(−∞) = exp(−∞) / (∑ exp(...)) = 0 / (∑ exp(...)) = 0

The forbidden positions get exactly zero weight. The remaining positions are re-normalized to sum to 1. It’s as if those positions don’t exist.

The causal mask (lower-triangular)

For autoregressive generation, the mask is a lower-triangular matrix: position i can attend to positions 0, 1, …, i (itself and all predecessors) but not to i+1, i+2, … Here is the pattern for n=5:

Other types of masks

  • Padding mask: Ignores padding tokens in variable-length batches. If token j is a pad token, set M[·][j] = −∞ for every row.
  • Encoder self-attention: Typically no mask at all (or padding mask only). Every token can attend to every other token because the entire input is available.
  • Cross-attention: The decoder queries attend to all encoder positions (no causal restriction on the encoder side).
Implementation detail: In practice, we use a very large negative number like −109 or −104 instead of true −∞ to avoid NaN issues in floating-point arithmetic. PyTorch’s float('-inf') also works because softmax implementations handle it gracefully.

8 · With vs Without Scaling

To make the scaling effect concrete, let’s compare what happens when we use dk = 64 (a realistic head dimension) with and without the 1/√dk factor.

❌ Without Scaling

Assume q·k values drawn from N(0, dk) = N(0, 64). Typical raw scores:

scores = [12.3, -8.7, 15.1, -3.2] softmax → [0.0006, 0.0000, 0.9994, 0.0000]

The distribution is near-one-hot. Almost all weight on one token. The Jacobian of softmax here is nearly zero - gradients vanish during back-propagation. The model cannot learn to distribute attention.

Gradient magnitude: ~0.0006

✓ With Scaling (÷ √64 = ÷ 8)

Same q·k values, but divided by 8. Scores become N(0, 1):

scaled = [1.54, -1.09, 1.89, -0.40] softmax → [0.278, 0.020, 0.395, 0.040]

The distribution is soft and spread out. Multiple tokens receive meaningful weight. Gradients are healthy and the model can learn nuanced patterns of attention.

Gradient magnitude: ~0.22

The difference is dramatic: ~370× larger gradients when scaling is applied. This is why the seemingly trivial 1/√dk factor is critical. Without it, training Transformers with dk ≥ 64 is extremely unstable.

9 · Complexity Analysis

Time complexity

OperationShapeCost
Q = X · WQ[n,d]×[d,dk]O(n · d · dk)
K = X · WK[n,d]×[d,dk]O(n · d · dk)
V = X · WV[n,d]×[d,dv]O(n · d · dv)
Q · KT[n,dk]×[dk,n]O(n² · dk)
softmax[n,n]O(n²)
weights · V[n,n]×[n,dv]O(n² · dv)

The dominant cost is the Q · KT multiplication and the weights · V multiplication, both O(n² · d). This quadratic dependence on sequence length n is the single biggest bottleneck of standard Transformers.

Space complexity

The attention weight matrix [n, n] must be stored in memory (for backpropagation). This is O(n²) per head, per layer.

  • For n = 512: 262K entries per head - manageable.
  • For n = 2048: 4.2M entries per head - still OK for modern GPUs.
  • For n = 8192: 67M entries per head - becoming expensive.
  • For n = 32768: 1.07B entries per head - requires special techniques.

Why quadratic cost matters

Doubling the sequence length quadruples the attention computation and memory. This is why standard Transformers struggle with very long sequences (documents, code, genomics). The quadratic cost has driven extensive research into efficient alternatives:

  • FlashAttention: Same O(n²) arithmetic, but uses tiling to avoid materializing the full n×n matrix, dramatically reducing memory usage.
  • Linear attention (Katharopoulos et al., 2020): Approximates softmax attention using kernel feature maps, achieving O(n) time and space.
  • Sparse attention (Longformer, BigBird): Each token only attends to a subset of positions, reducing cost to O(n√n) or O(n).
  • State-space models (Mamba, S4): Replace attention entirely with recurrence-like operations that are O(n) in sequence length.
Perspective: Despite the quadratic cost, standard attention remains the default for most practical sequence lengths (≤ 8K tokens). The expressiveness of full pairwise attention is hard to match. FlashAttention makes it feasible even for 32K+ contexts by exploiting GPU memory hierarchies.

10 · Implementation

Here is a clean, self-contained implementation of single-head scaled dot-product attention in Python-like pseudo-code. Every line maps directly to the math we covered above.

import numpy as np def scaled_dot_product_attention(Q, K, V, mask=None): """ Single-head scaled dot-product attention. Args: Q: query matrix [n, d_k] K: key matrix [n, d_k] V: value matrix [n, d_v] mask: optional mask [n, n] (0 = attend, -inf = block) Returns: output: [n, d_v] context vectors weights: [n, n] attention weights (for visualization) """ d_k = Q.shape[-1] # Step 1: Compute raw attention scores scores = Q @ K.T # [n, n] # Step 2: Scale by 1/sqrt(d_k) scores = scores / np.sqrt(d_k) # [n, n] # Step 2.5: Apply mask (optional) if mask is not None: scores = scores + mask # masked positions get -inf # Step 3: Softmax over last dimension (keys) weights = softmax(scores, axis=-1) # [n, n], each row sums to 1 # Step 4: Weighted sum of values output = weights @ V # [n, d_v] return output, weights def softmax(x, axis=-1): """Numerically stable softmax.""" x_max = np.max(x, axis=axis, keepdims=True) e_x = np.exp(x - x_max) # subtract max for stability return e_x / np.sum(e_x, axis=axis, keepdims=True) def create_causal_mask(n): """Create a lower-triangular causal mask for n tokens.""" mask = np.full((n, n), -np.inf) mask = np.triu(mask, k=1) # upper triangle = -inf return mask # lower triangle + diagonal = 0 # ---- Example usage ---- n, d_k, d_v = 3, 4, 4 X = np.random.randn(n, 8) # input: 3 tokens, d_model=8 # Learned projection matrices (normally from nn.Linear) W_Q = np.random.randn(8, d_k) * 0.1 W_K = np.random.randn(8, d_k) * 0.1 W_V = np.random.randn(8, d_v) * 0.1 # Project input to Q, K, V Q = X @ W_Q # [3, 4] K = X @ W_K # [3, 4] V = X @ W_V # [3, 4] # Run attention (no mask = encoder-style) out, attn = scaled_dot_product_attention(Q, K, V) # Run attention with causal mask (decoder-style) mask = create_causal_mask(n) out_causal, attn_causal = scaled_dot_product_attention(Q, K, V, mask)

PyTorch equivalent (one-liner)

import torch import torch.nn.functional as F # PyTorch 2.0+ provides a fused, memory-efficient implementation: output = F.scaled_dot_product_attention( query, # [batch, n, d_k] key, # [batch, n, d_k] value, # [batch, n, d_v] attn_mask=mask, # optional [n, n] is_causal=False, # set True for decoder dropout_p=0.0, scale=None # defaults to 1/sqrt(d_k) ) # Under the hood, this can dispatch to FlashAttention or # Memory-Efficient Attention depending on hardware and shapes.

11 · Summary

Here are the key takeaways from this post:

  • Attention is a soft, differentiable lookup. Each token emits a query, matches it against keys, and retrieves a weighted combination of values. This replaces the fixed-capacity hidden state of RNNs with a dynamic, content-based mechanism.
  • Q, K, V are three independent learned projections of the same input. They decouple the “what am I looking for” (Q), “what do I advertise” (K), and “what do I contain” (V) roles.
  • The formula is four steps: (1) compute dot-product scores QKT, (2) scale by 1/√dk, (3) apply softmax row-wise, (4) weighted sum of V.
  • Scaling by √dk is essential. Without it, the variance of scores grows with dk, softmax saturates, and gradients vanish. The scaling normalizes variance to 1.
  • Masking is implemented by adding −∞ to forbidden positions before softmax, driving their weights to exactly 0. The causal mask (lower triangular) prevents attending to future tokens.
  • Complexity is O(n² · d) in time and O(n²) in space. This quadratic cost is the primary scalability limitation and has driven research into FlashAttention, linear attention, and sparse attention.
  • Single-head attention is the building block of multi-head attention (covered in Part 2), which runs multiple heads in parallel with different learned projections, then concatenates their outputs.
