    Interleaved Head Attention: Mixing Attention Patterns

    January 2025 · 11 min read

    In standard attention, all heads follow the same masking pattern (typically causal for autoregressive models). Interleaved Head Attention (IHA) — deployed in models like Gemma and MPT — assigns different attention patterns to different heads within the same layer. Some heads might be causal (attend only to past positions), others sliding-window (attend to a local window), and others global (attend to all positions). Each head specializes for a different structural role. This heterogeneity allows the model to trade local coherence (sliding-window heads), causal structure (causal heads), and global context (global heads) all within a single layer, improving efficiency and quality on long sequences without the parameter overhead of separate modules.

    Motivation: Why Homogeneous Masking Is Suboptimal

    Standard multi-head attention uses the same mask for all heads, so every head processes the same receptive field. But different semantic tasks need different patterns:

    • Local coherence: Tokens nearby should maintain local syntax and discourse continuity. A sliding-window or causal mask suffices.
    • Long-range dependencies: Distant tokens (pronouns and referents, discourse markers) need global attention.
    • Causal structure: In generation, respecting causality prevents information leakage.

    Assigning all heads the same mask under-utilizes capacity: causal-only heads "waste" their attention budget on local syntax when a sliding-window pattern would suffice. Interleaving patterns lets each head specialize.

    Common Attention Patterns

    Within a single layer, we can assign different masks to different heads:

    Pattern Receptive Field Use Case
    Causal Positions $\leq$ current Autoregressive structure; no information leakage
    Sliding-window ±W positions from current Local context (syntax, discourse)
    Global All positions Long-range semantics (pronouns, themes)
    Strided Every $k$-th position (e.g., $k=2$) Summarization of larger context

    Gemma and MPT: Interleaving in Practice

    Gemma 2 (Google, 2024): Uses a mix of causal and local-sliding-window heads within the same layer. Early transformer layers emphasize local patterns; middle layers blend causal and sliding-window; late layers add more global attention. This hierarchical specialization mirrors linguistic structure.

    MPT (Mosaic ML, 2023): The "Mixture of Patterns" design combines causal heads with block-diagonal or "attention-with-sparse-pattern" heads. MPT models achieve strong performance on very long sequences (up to 65K tokens) partly because interleaving allows efficient context integration without full quadratic attention cost.

    Benefits and Trade-offs

    Pros:

    • Better capacity utilization: each head specializes for its pattern.
    • Efficiency on long sequences: sliding-window and strided heads are $O(n \cdot W)$ instead of $O(n^2)$.
    • Improved quality: global context is preserved (via global heads) while local computation is efficient (via local heads).
    • No additional parameters: only masking changes; model size is identical.

    Cons:

    • Implementation complexity: different masks per head require careful kernel design or padding tricks.
    • Training overhead: masking may slow down some attention backends if not carefully optimized.
    • Architectural design choice: deciding which heads get which pattern requires experimentation.

    Computational Complexity by Pattern

    Pattern FLOPs Memory (QK matrix)
    Causal (standard) $O(n^2 d_k)$ $O(n^2)$
    Sliding-window (W=256) $O(n \cdot W \cdot d_k)$ $O(n \cdot W)$ (4–64× smaller)
    Global $O(n^2 d_k)$ $O(n^2)$ (same as causal)
    Strided (stride=4) $O(n^2 / 16 \cdot d_k)$ $O(n^2 / 16)$ (16× smaller)

    Design Choices: Which Heads Get Which Patterns?

    The pattern assignment is a hyperparameter. Some heuristics:

    • Layerwise progression: Early layers (syntax) use local/sliding-window. Middle layers blend causal and local. Late layers (semantics) add more global.
    • Uniform interleaving: Assign patterns in a round-robin across heads (e.g., heads 0, 4, 8, ... global; heads 1, 5, 9, ... causal; etc.).
    • Empirical tuning: Train multiple configurations, measure perplexity, pick the best.

    Gemma 2 and MPT both use layer-dependent assignments, which seems to work well empirically.

    Implementation Sketch

    def interleaved_head_attention(Q, K, V, num_heads, attention_patterns):
    """
    Q, K, V: (batch, seq_len, d_model) → reshaped to (batch, num_heads, seq_len, d_k)
    attention_patterns: list of str, len=num_heads, e.g., ['causal', 'sliding', 'global', ...]
    """
    batch, seq_len, d_model = Q.shape
    d_k = d_model // num_heads
    
    # Standard projections
    Q = linear_Q(Q).view(batch, seq_len, num_heads, d_k).transpose(1, 2)
    K = linear_K(K).view(batch, seq_len, num_heads, d_k).transpose(1, 2)
    V = linear_V(V).view(batch, seq_len, num_heads, d_k).transpose(1, 2)
    
    outputs = []
    for h in range(num_heads):
        pattern = attention_patterns[h]
        Q_h = Q[:, h, :, :]  # (batch, seq_len, d_k)
        K_h = K[:, h, :, :]
        V_h = V[:, h, :, :]
        
        # Compute scores
        scores = Q_h @ K_h.transpose(-2, -1) / math.sqrt(d_k)  # (batch, seq_len, seq_len)
        
        # Apply mask based on pattern
        if pattern == 'causal':
            mask = lower_triangular_mask(seq_len)  # attend to past only
        elif pattern == 'sliding':
            mask = sliding_window_mask(seq_len, window_size=256)
        elif pattern == 'global':
            mask = ones(seq_len, seq_len)  # attend to all
        
        scores = scores + (1 - mask) * (-1e9)  # apply mask
        attn_weights = softmax(scores, dim=-1)
        out_h = attn_weights @ V_h  # (batch, seq_len, d_k)
        outputs.append(out_h)
    
    # Concatenate heads
    output = torch.cat(outputs, dim=-1)  # (batch, seq_len, d_model)
    output = linear_O(output)
    return output

    Wrapping Up the Series

    From self-attention (part 1) through IHA, we've traced the evolution of attention mechanisms across the efficiency frontier. Modern transformers combine several of these ideas: GQA for cache efficiency (Llama 3), MLA for even tighter compression (DeepSeek-V2), and interleaved patterns for balanced local/global reasoning (Gemma 2). The field is moving toward heterogeneous architectures where different mechanisms coexist, rather than a single unified design.
