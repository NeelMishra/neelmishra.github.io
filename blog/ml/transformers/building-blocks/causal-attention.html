    Causal-Attention: The Autoregressive Mask

    January 2026 · 10 min read

    Decoder-only models (GPT, Llama) generate text token-by-token, left-to-right. They must never "look ahead" to tokens yet to be generated. Part 2 showed how softmax creates attention weights; this part adds a crucial constraint: the causal mask, which forces each token to attend only to its past (and itself).

    Why Causality?

    During *autoregressive* generation, at inference time, the model generates tokens one by one. If training allowed the model to attend to future tokens, it would see answers to problems it hasn't solved yet—a form of cheating. At test time, those future tokens don't exist yet, so the model would suddenly fail.

    The fix: apply a causal mask during training that mirrors the test-time constraint. The model learns that it cannot attend to tokens $j > i$ when computing output position $i$.

    Generative vs. Bidirectional: Encoder models (BERT) use full self-attention: each position attends to all others. Decoder/generative models (GPT) use causal self-attention: only to the past. This is not a quirk—it's a requirement for autoregressive generation to work correctly.

    The Causal Mask Mechanism

    Before applying softmax, we modify the attention scores:

    $$s'_{ij} = \begin{cases} s_{ij} & \text{if } j \leq i \\ -\infty & \text{if } j > i \end{cases}$$

    Then softmax proceeds normally:

    $$w_{ij} = \text{softmax}(s'_{ij})$$

    When $s'_{ij} = -\infty$, the exponential $e^{-\infty}$ is effectively 0. The softmax row renormalizes, and the masked position gets exactly 0 attention.

    In code, the mask is a triangular matrix added before softmax:

    import torch
import torch.nn.functional as F

def apply_causal_mask(scores, mask_value=-1e9):
    """
    scores: (batch, n_heads, seq_len, seq_len)
    Returns masked scores (future positions set to mask_value)
    """
    batch, heads, n, _ = scores.shape
    # Create lower triangular mask (1 where j <= i, 0 where j > i)
    causal_mask = torch.tril(torch.ones(n, n, device=scores.device))
    # Expand and apply
    mask = causal_mask.unsqueeze(0).unsqueeze(0)  # (1, 1, n, n)
    masked_scores = scores.masked_fill(mask == 0, mask_value)
    attn_weights = F.softmax(masked_scores, dim=-1)
    return attn_weights

    The Attention Matrix Shape

    In standard (non-causal) attention, the $n \times n$ matrix is unrestricted. In causal attention, it has a specific structure:

    • Main diagonal and lower triangle: computed normally, then softmaxed.
    • Upper triangle: all zeros (after masking and softmax).

    This is efficient in one sense: you only need to compute $n(n+1)/2$ attention scores instead of $n^2$. In practice, most implementations still compute the full matrix then mask, accepting the wasted computation as a trade-off for simplicity.

    Training vs. Inference Dynamics

    Training: Causal masking is applied. All $n$ positions are processed in one forward pass, each attending only to positions $\leq i$. This is efficient (parallel).

    Inference: Tokens are generated one at a time. Position 0 attends only to itself (weight 1.0). Position 1 attends to 0 and 1. Etc. In many implementations, the mask is implicit during inference because only past tokens exist in the KV cache.

    KV Cache and Causal Masking

    Modern generation uses a KV cache to avoid recomputing attention. At each step, we keep the full history of K and V, but only compute Q for the new token. The causal structure is automatic: new Q only attends to past K/V by construction.

    Generation Without KV Cache (Naive)

    A model without KV caching recomputes attention for all past positions at each step. For a 1000-token sequence, the last token sees a $1000 \times 1000$ matrix, then you generate one more token and re-compute a $1001 \times 1001$ matrix. This is $O(n^3)$ total work—very slow. This is why caching is critical.
