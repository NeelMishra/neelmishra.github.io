Feed-Forward Network

April 2026 · ML Series · Transformers

What Does the FFN Do in a Transformer?

Every transformer block has two core sub-layers: multi-head self-attention and a position-wise feed-forward network (FFN). Attention handles the communication between tokens. The FFN handles the computation at each token independently. Without the FFN, a transformer would be limited to linearly mixing token representations, never transforming any individual token through a nonlinear function.

Think of it this way. Attention lets token 5 borrow information from tokens 0 through 100. But once that information arrives at position 5, what does the model do with it? The FFN takes that enriched representation and pushes it through a nonlinear transformation, extracting higher-level features, combining semantic fragments, and mapping raw patterns into useful abstractions. This is where the real "thinking" happens for each individual token.

A useful analogy: attention is a meeting where everyone shares their notes. The FFN is each person going back to their desk and actually processing what they heard. The meeting (attention) is collaborative and cross-positional. The desk work (FFN) is individual and independent.

Key Insight: If you remove the FFN and stack only attention layers with residual connections, the entire network collapses to a linear function of the input embeddings (since attention is a weighted sum, which is linear). The FFN is what gives the transformer its universal approximation power.

Concretely, the FFN introduces two things the attention mechanism cannot provide on its own:

  • Nonlinearity: The activation function (ReLU, GELU, SwiGLU, etc.) breaks the linearity of matrix multiplications. This is essential for the network to learn complex, nonlinear relationships in the data.
  • Per-token feature extraction: While attention determines what information each token should see, the FFN determines how to transform that information. It acts as a two-layer MLP applied identically to every position.

In the original "Attention Is All You Need" paper, the authors describe it simply: the FFN is "applied to each position separately and identically." This brevity belies its importance. The FFN typically contains about two-thirds of all the parameters in a transformer block, making it the largest component by parameter count.

The Architecture: Expand Then Compress

The feed-forward network in a transformer is a simple two-layer MLP. It takes an input vector, projects it to a higher-dimensional space, applies a nonlinear activation function, and then projects it back down to the original dimension. The formula is:

FFN(x) = W₂ · activation(W₁ · x + b₁) + b₂ Where: x ∈ ℝ^d_model (input vector for a single token) W₁ ∈ ℝ^(d_ff × d_model) (first weight matrix - expands) b₁ ∈ ℝ^d_ff (first bias) W₂ ∈ ℝ^(d_model × d_ff) (second weight matrix - compresses) b₂ ∈ ℝ^d_model (second bias)

The key design decision is the shape of the intermediate layer. The input and output dimensions are both d_model, but the hidden dimension d_ff is much larger, typically 4 times d_model. This creates an expand-then-compress architecture, sometimes called a "bottleneck in reverse."

Why Two Layers, Not One?

A single linear layer would simply compute W · x + b, which is a linear transformation. No matter how many single linear layers you stack, the result is still a single linear transformation (because the composition of linear functions is linear). The activation function between the two layers is what makes the difference. It introduces a nonlinear boundary that allows the network to represent arbitrarily complex functions.

In fact, a two-layer MLP with a nonlinear activation function is a universal approximator. Given enough neurons in the hidden layer, it can approximate any continuous function to arbitrary precision. The expand-compress design leverages this theoretical power.

The Expand-Compress Flow

Step 1: Expansion (d_model → d_ff)

The input vector x of dimension d_model is multiplied by W₁ and passed through an activation function. This projects the representation into a much higher-dimensional space where the network has more room to perform nonlinear computations. Each neuron in this intermediate layer can detect a different feature or pattern.

Step 2: Compression (d_ff → d_model)

The expanded representation is multiplied by W₂, projecting it back to the original d_model dimension. This compression forces the network to distill the intermediate computation into a compact representation that fits the residual stream. The output is added back to the input via the residual connection.

This expand-compress pattern is not unique to transformers. It appears in ResNet bottleneck blocks, autoencoders (in reverse), and mixture-of-experts architectures. The transformer's specific contribution is applying it position-wise, meaning the exact same weights process every token independently.

Why the 4x Expansion?

The original "Attention Is All You Need" paper used d_model = 512 and d_ff = 2048, establishing the 4x ratio that became a near-universal default. But why 4x?

The honest answer is that it is an empirical design choice. The authors tried it, it worked well, and subsequent research confirmed it was a reasonable trade-off. But we can reason about why:

  • Too small (1x or 2x): The hidden layer would have the same dimension or only slightly more than the input. The FFN would have limited capacity to detect diverse features. The network's representational power suffers.
  • Just right (4x): Four times the dimension gives the FFN enough neurons to learn a rich set of feature detectors while keeping the parameter count and compute cost manageable.
  • Too large (8x, 16x): More capacity, more parameters, more computation. Diminishing returns set in quickly because the additional neurons learn increasingly redundant or noisy features. The extra FLOPs are rarely worth it.

Activation Functions

The activation function sits between the two linear layers and is the sole source of nonlinearity in the FFN. The choice of activation has a measurable impact on model quality, training stability, and inference speed. Let us examine the three most important options in detail.

ReLU: The Original Choice

ReLU(x) = max(0, x) Output: x > 0 → x (identity: passes the value through) x ≤ 0 → 0 (kills the value completely)

ReLU was the activation function used in the original Transformer paper. It is simple, fast, and computationally trivial (a single comparison and branch). It introduces a hard threshold at zero: positive values pass through unchanged, and negative values are zeroed out entirely.

This hard threshold has a crucial consequence: sparsity. In a trained FFN with ReLU, roughly 80-95% of the intermediate neurons output zero for any given input. Only a small subset of neurons "fire" for each token. This sparsity is computationally beneficial (sparse activations can be stored efficiently) but also problematic.

The problem is dead neurons. If a neuron's input is consistently negative across all training examples (due to unfortunate weight initialization or aggressive learning rates), the neuron's output is always zero. Since the gradient of ReLU is zero for negative inputs, the neuron receives no gradient signal and can never recover. It is permanently "dead," contributing nothing to the model's capacity. In large models, a significant fraction of neurons can die this way.

GELU: The Modern Default

GELU(x) = x · Φ(x) Where Φ(x) is the CDF of the standard normal distribution. Approximation: GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))) Output: x ≫ 0 → ≈ x (passes through like ReLU) x ≪ 0 → ≈ 0 (kills the value like ReLU) x ≈ 0 → smooth transition (NOT a hard cutoff)

GELU (Gaussian Error Linear Unit) was introduced by Hendrycks and Gimpel in 2016 and became the default in BERT, GPT-2, GPT-3, and most modern transformers. It behaves like ReLU for large positive and large negative values, but in the critical region near zero, it provides a smooth, differentiable transition instead of a hard cutoff.

Why does the smooth transition matter? Consider a neuron whose pre-activation value hovers around zero. With ReLU, this neuron flickers between fully active and fully inactive as the input changes slightly. The gradient is discontinuous: exactly 1 on the positive side, exactly 0 on the negative side. This causes noisy gradient estimates during training.

GELU eliminates this problem. Near zero, the activation and its gradient change smoothly. Slightly negative values still contribute a small, non-zero output, which means the neuron can still receive gradient signal and adjust. There are no dead neurons with GELU, because no region of the input space has a permanently zero gradient.

The intuitive interpretation is "stochastic gating." GELU can be seen as multiplying the input by a Bernoulli random variable where the probability of the gate being open equals Φ(x), the probability that a standard normal variable is less than x. Large positive inputs are almost certainly passed through. Large negative inputs are almost certainly blocked. Inputs near zero are passed through with approximately 50% probability. The function computes the expected value of this stochastic process.

SwiGLU: The State-of-the-Art

SwiGLU(x, W, V, W₂) = (Swish(xW) ⊙ xV) W₂ Where: Swish(z) = z · σ(z) (σ is the sigmoid function) ⊙ denotes element-wise multiplication Weight matrices: W ∈ ℝ^(d_model × d_ff) - gate projection V ∈ ℝ^(d_model × d_ff) - value projection W₂ ∈ ℝ^(d_ff × d_model) - output projection

SwiGLU, introduced by Shazeer (2020), combines the Swish activation with a Gated Linear Unit (GLU). It is the activation function used in PaLM, LLaMA, Mistral, and most state-of-the-art large language models as of 2024.

The key difference from standard FFN is the gating mechanism. Instead of applying a single activation to the expanded representation, SwiGLU computes two separate projections of the input:

  • Gate path: Swish(x · W) computes a gating signal. The Swish activation (z × sigmoid(z)) produces values that are smooth and unbounded above zero but bounded below (similar to GELU). This gate controls how much of each feature passes through.
  • Value path: x · V computes the raw feature values without any activation. These are the actual values that the gate selectively passes or blocks.

The element-wise product of gate and value means each intermediate neuron has its own learned gating mechanism. The model can learn to open or close each gate based on the input, providing fine-grained control over information flow.

Because SwiGLU uses three weight matrices (W, V, W₂) instead of two (W₁, W₂), it has 50% more parameters if d_ff is kept the same. To maintain parameter parity with a standard FFN, the intermediate dimension is typically set to (2/3) × 4 × d_model. For example, LLaMA-7B uses d_model=4096 and d_ff=11008 (which is approximately 2.68 × 4096).

Activation Function Impact on Training

In practice, the choice of activation function has a measurable but not dramatic impact. Switching from ReLU to GELU typically improves final validation loss by 1-3% on language modeling benchmarks. Switching from GELU to SwiGLU provides another 1-2% improvement. These gains compound across billions of training tokens.

The more significant impact is on training stability. GELU and SwiGLU produce smoother loss curves with fewer spikes, which is critical when training at scale with large batch sizes and aggressive learning rates. The smooth gradients make the optimization landscape easier to navigate.

Position-Wise Independence

The FFN applies the exact same transformation to every position in the sequence, independently. Position 0 and position 100 are processed by identical weights. There is no communication between positions inside the FFN. Each token is transformed in complete isolation.

This is a fundamental design choice that cleanly separates the two operations in each transformer block:

  • Attention = Communication: tokens exchange information with each other. The output at each position is a function of all positions.
  • FFN = Computation: each token processes its information privately. The output at each position is a function of only that position's input.

Position-Wise Processing: Same Weights, Different Inputs

Three tokens processed independently through the same FFN weights.

This design has several important consequences:

  • Parallelism: Because positions are independent, the FFN can process all positions simultaneously as a single batched matrix multiplication. There are no sequential dependencies. On GPUs, this is extremely efficient.
  • Translation equivariance: The FFN treats all positions equally. If you shift the entire sequence by one position, the FFN output shifts by one position too. The position-specific behavior comes entirely from positional encodings and the attention mechanism, not from the FFN.
  • Parameter efficiency: Sharing weights across all positions means the FFN does not need separate parameters for each position. A sequence of 1000 tokens uses the same FFN parameters as a sequence of 10 tokens.

Mathematically, if X is the input matrix of shape (n, d_model) where n is the sequence length, the FFN computes:

FFN(X) = activation(X · W₁ᵀ + b₁) · W₂ᵀ + b₂ This is equivalent to: For each position i ∈ {0, 1, ..., n-1}: FFN(x_i) = W₂ · activation(W₁ · x_i + b₁) + b₂ No mixing between positions. Token x_i never sees token x_j inside the FFN.

The Key-Value Memory Interpretation

One of the most fascinating discoveries about feed-forward networks came from Geva et al. (2021) in their paper "Transformer Feed-Forward Layers Are Key-Value Memories." They showed that the FFN is not just a generic nonlinear transformation; it acts as an associative memory that stores and retrieves factual knowledge.

The interpretation is beautifully simple. Consider the first weight matrix W₁ and the second weight matrix W₂:

Keys: Rows of W₁

Each row of W₁ (there are d_ff rows) acts as a key pattern. When the input vector x is multiplied by W₁, the dot product W₁[i] · x measures how well the input matches the i-th key. After the activation function, this becomes a scalar "match score" that is high when the input strongly matches key i, and near zero otherwise.

Values: Columns of W₂

Each column of W₂ (there are d_ff columns) acts as a value vector. When the activation output is multiplied by W₂, each column of W₂ is weighted by the corresponding activation. High activation for neuron i means column i of W₂ contributes strongly to the output. Low activation means that column is effectively suppressed.

Putting it together: the FFN computes a weighted sum of value vectors, where the weights are determined by how well the input matches each key. This is exactly how a key-value memory works.

FFN(x) = Σᵢ activation(kᵢ · x) · vᵢ Where kᵢ = i-th row of W₁ (key) vᵢ = i-th column of W₂ (value) This is a soft lookup: 1. Compare input x against all d_ff keys 2. Compute match scores via activation 3. Return weighted sum of corresponding values

What Do Individual Neurons Store?

Geva et al. found that individual FFN neurons encode specific, interpretable patterns. By analyzing which inputs maximally activate specific neurons, they discovered neurons that fire for:

  • Specific topics: A neuron that activates strongly for inputs related to "sports," "medicine," or "programming."
  • Syntactic patterns: A neuron that fires when the input is a verb in past tense, or when the current position follows a preposition.
  • Factual knowledge: A neuron that activates for "The capital of France is..." and whose corresponding value vector pushes the output distribution toward "Paris."
  • Language-specific patterns: Neurons that fire for text in specific languages.
Implications: This key-value memory interpretation has profound implications for model interpretability and editing. If a model incorrectly states that "The capital of Australia is Sydney," you can identify the responsible FFN neuron and modify its value vector to correct the fact. This is the basis of "knowledge editing" techniques like ROME (Rank-One Model Editing) and MEMIT.

Upper Layers vs Lower Layers

Further research revealed a gradient of abstraction across layers:

  • Lower layers (1-4): FFN neurons tend to detect shallow patterns like character n-grams, punctuation contexts, and positional features. The "keys" match surface-level input features.
  • Middle layers (5-8): Neurons detect syntactic and semantic patterns. Keys match part-of-speech contexts, semantic categories, and mid-level linguistic features.
  • Upper layers (9-12): Neurons encode high-level, task-specific patterns. In GPT-style models, these layers contain neurons that directly influence the next-token prediction, with value vectors that align with specific vocabulary items.

This layered specialization emerges naturally from training. No explicit signal tells the model to organize this way; it is a consequence of the architecture and the learning objective.

Notice that the input and output have the same shape: (batch, n, d_model). This is critical because the FFN output is added to the input via the residual connection. If the shapes did not match, the addition would fail.

A standard transformer block has two sub-layers: multi-head attention and FFN. Let us compare their parameter counts for d_model=512, d_ff=2048, h=8 heads:

The FFN accounts for approximately two-thirds of all parameters in each transformer block. This is a surprising and important fact. Despite attention getting most of the publicity, the FFN is where most of the model's capacity lives. This is consistent with the key-value memory interpretation: the FFN's parameters encode the model's knowledge, while attention's parameters encode how to route information between positions.

Dropout in FFN

Dropout is applied inside the FFN as a regularization technique to prevent overfitting. The original Transformer paper applies dropout in two locations within the FFN:

FFN(x) = Dropout₂(W₂ · Dropout₁(activation(W₁ · x + b₁)) + b₂) Dropout₁: Applied after the activation function, before the second linear layer. Randomly zeros out neurons in the d_ff-dimensional intermediate representation. Dropout₂: Applied after the entire FFN output, before the residual addition. Randomly zeros out dimensions of the d_model-dimensional output.

How Dropout Works

During training, each element is independently set to zero with probability p (typically 0.1), and the surviving elements are scaled by 1/(1-p) to maintain the expected value. During inference, dropout is disabled and all elements pass through unchanged.

In the FFN context, dropout after the activation is particularly effective because it forces the network to not rely on any single intermediate neuron. If neuron 42 is the only one that detects "capital city" contexts, dropout forces the network to distribute this knowledge across multiple neurons, improving generalization.

Dropout Rates in Practice

ModelFFN Dropout RateNotes

A notable trend: as models get larger and training data grows more abundant, dropout rates decrease toward zero. Very large models (GPT-3 175B, LLaMA, PaLM) often use no dropout at all, relying instead on the sheer volume and diversity of training data for regularization. The intuition is that dropout's regularization benefit diminishes when the model sees enough unique data to generalize without it.

Implementation

Here is the standard FFN implementation, followed by the SwiGLU variant.

Standard FFN (ReLU / GELU)

class FeedForward(Module): def __init__(self, d_model, d_ff, dropout=0.1): self.W1 = Linear(d_model, d_ff) # expand: d_model → d_ff self.W2 = Linear(d_ff, d_model) # compress: d_ff → d_model self.activation = GELU() self.dropout = Dropout(dropout) def forward(self, x): # x shape: (batch, seq_len, d_model) h = self.W1(x) # (batch, seq_len, d_ff) h = self.activation(h) # (batch, seq_len, d_ff) - nonlinearity h = self.dropout(h) # (batch, seq_len, d_ff) - regularize out = self.W2(h) # (batch, seq_len, d_model) return out # added to residual stream outside

SwiGLU Variant (LLaMA-style)

class SwiGLUFeedForward(Module): def __init__(self, d_model, d_ff): # d_ff is typically (2/3) × 4 × d_model, rounded to multiple of 256 self.W_gate = Linear(d_model, d_ff, bias=False) self.W_value = Linear(d_model, d_ff, bias=False) self.W_out = Linear(d_ff, d_model, bias=False) def forward(self, x): # x shape: (batch, seq_len, d_model) gate = self.W_gate(x) # (batch, seq_len, d_ff) gate = silu(gate) # Swish/SiLU activation on gate path value = self.W_value(x) # (batch, seq_len, d_ff) - no activation h = gate * value # (batch, seq_len, d_ff) - element-wise gating out = self.W_out(h) # (batch, seq_len, d_model) return out

FFN Inside a Transformer Block

class TransformerBlock(Module): def __init__(self, d_model, n_heads, d_ff, dropout=0.1): self.attention = MultiHeadAttention(d_model, n_heads) self.ffn = FeedForward(d_model, d_ff, dropout) self.norm1 = LayerNorm(d_model) self.norm2 = LayerNorm(d_model) self.dropout1 = Dropout(dropout) self.dropout2 = Dropout(dropout) def forward(self, x, mask=None): # Sub-layer 1: Multi-Head Attention + Residual attn_out = self.attention(x, x, x, mask) x = x + self.dropout1(attn_out) # residual connection x = self.norm1(x) # layer normalization # Sub-layer 2: Feed-Forward Network + Residual ffn_out = self.ffn(x) x = x + self.dropout2(ffn_out) # residual connection x = self.norm2(x) # layer normalization return x
Pre-Norm vs Post-Norm: The code above shows post-norm (original Transformer). Modern models like GPT-2+ and LLaMA use pre-norm, where LayerNorm is applied before each sub-layer: x = x + FFN(norm(x)). Pre-norm training is more stable, especially for deep models, because the residual stream maintains its scale.

Summary

  • Role: The FFN provides nonlinear, per-token computation. Attention handles cross-token communication; the FFN handles per-token transformation. Without the FFN, a transformer would be limited to linear operations.
  • Architecture: Two linear layers with an activation in between. The first expands from d_model to d_ff (typically 4× d_model), and the second compresses back. Formula: FFN(x) = W₂ · activation(W₁ · x + b₁) + b₂.
  • 4x Expansion: The 4× ratio balances representational capacity against compute cost. It is an empirical default that has proven robust across model scales from 100M to 175B parameters.
  • Activation Functions: ReLU (original, simple, dead neuron problem) → GELU (smooth, no dead neurons, modern default) → SwiGLU (gated, 3 matrices, state-of-the-art). Each generation brings modest but consistent quality gains.
  • Position-Wise: The same weights apply to every token independently. This cleanly separates communication (attention) from computation (FFN) and enables efficient parallelization.
  • Key-Value Memory: Rows of W₁ act as "keys" and columns of W₂ act as "values." The FFN functions as an associative memory storing factual knowledge, explaining why individual neurons encode interpretable concepts.
  • Parameters: The FFN accounts for ~67% of parameters in each transformer block. For a standard FFN: 2 × d_model × d_ff + d_ff + d_model.
  • Dropout: Applied after activation and/or after the FFN output. Rate of 0.1 is standard, but large modern models often use 0.0.
