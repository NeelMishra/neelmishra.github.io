    Self-Attention: The Foundation

    January 2026 · 12 min read

    Self-attention is the core mechanism of modern transformers. Given a sequence of tokens, each token learns to retrieve relevant context from all other tokens (and itself) by computing learned projections Q, K, V and combining them via a dot-product interaction. This post derives the formula, explains the design choices, and walks through a full computation.

    The Setup: Contextual Representations

    Imagine you have a sequence of embeddings $X \in \mathbb{R}^{n \times d}$ where $n$ is sequence length and $d$ is embedding dimension. You want to transform each token's representation by incorporating context from the entire sequence. A naive approach would apply a dense layer to the concatenation of all tokens—but that's $O(nd^2)$ parameters and breaks permutation properties.

    Self-attention solves this differently: each position learns *how much* to attend to each other position. The result is permutation-equivariant (rearranging inputs rearranges outputs accordingly) and uses a bounded, learnable interaction between pairs of tokens.

    Three Matrices: Q, K, V

    Given input $X \in \mathbb{R}^{n \times d}$, we compute three linear projections:

    $$Q = X W^Q \in \mathbb{R}^{n \times d_k}, \quad K = X W^K \in \mathbb{R}^{n \times d_k}, \quad V = X W^V \in \mathbb{R}^{n \times d_v}$$

    where $W^Q, W^K \in \mathbb{R}^{d \times d_k}$ and $W^V \in \mathbb{R}^{d \times d_v}$. Typically $d_k = d_v = d / h$ in multi-head attention (h heads), but here assume they're all $d$ for simplicity. The three projection matrices are *learned*—this is where the model's expressiveness comes from.

    • Query (Q): "What am I looking for?" Each query is a learned representation of what information a token needs.
    • Key (K): "What am I offering?" Keys represent the content each position advertises.
    • Value (V): "Here's the content." Values are what actually get combined when a query matches a key.

    The Scaled Dot-Product Formula

    Self-attention combines Q, K, V via:

    $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

    Let's break this down:

    • $QK^T \in \mathbb{R}^{n \times n}$: Each entry $(i,j)$ is the dot product of query $i$ and key $j$. High values mean "query $i$ is similar to key $j$."
    • Scale by $1/\sqrt{d_k}$: Stabilizes gradients. When $d_k$ is large, dot products grow large; scaling keeps them in a reasonable range for softmax to differentiate.
    • Apply softmax row-wise: Each row becomes a probability distribution over $n$ positions. Softmax ensures positive weights summing to 1.
    • Multiply by $V \in \mathbb{R}^{n \times d_v}$: Each output is a weighted combination of all values, where row $i$ is weighted by the attention weights from query $i$.

    The output is $\mathbb{R}^{n \times d_v}$—one representation per position, each attending to the full sequence.

    Why "Self"?

    The term "self-attention" means Q, K, V all come from the *same* input $X$. This contrasts with cross-attention (Part 4) where Q comes from one sequence and K, V from another. In a self-attention layer, every token simultaneously attends to every other token (and itself), enabling rich contextual flow.

    Permutation-Equivariance and Why It Matters

    Suppose we permute the input sequence by a permutation $\pi$: $(x_1, \ldots, x_n) \mapsto (x_{\pi(1)}, \ldots, x_{\pi(n)})$. What happens to the output?

    Because attention is computed position-wise—each position independently computes a weighted average—the output permutes identically: output position $\pi(i)$ becomes the old output position $i$. This is permutation-equivariance: permuting input permutes output the same way. In contrast, a global feedforward (like a large MLP on the full flattened sequence) would not have this property and would need explicit position encodings to break symmetry.

    Why equivariance is important: Transformers rely on equivariance to handle variable-length sequences, to compose attention layers, and to enable parallelization. If the output changed arbitrarily with input permutation, the model would have to "memorize" the positions, wasting capacity.

    Time and Space Complexity

    The main computational bottleneck is $QK^T$, which is $\mathbb{R}^{n \times n}$. Computing it takes:

    • Time: $O(n^2 d)$ (matrix multiply $QK^T$) plus $O(n^2)$ for softmax per row.
    • Space: $O(n^2)$ to store the attention matrix itself.

    For long sequences (e.g., full document, image patches), this becomes prohibitive. Parts 5–7 explore sparse and efficient approximations.

    PyTorch Implementation Sketch

    import torch
import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, d_model, d_k=None):
        super().__init__()
        self.d_k = d_k or d_model
        self.W_Q = nn.Linear(d_model, self.d_k)
        self.W_K = nn.Linear(d_model, self.d_k)
        self.W_V = nn.Linear(d_model, self.d_k)
    
    def forward(self, X):
        # X shape: (batch, seq_len, d_model)
        Q = self.W_Q(X)  # (batch, seq_len, d_k)
        K = self.W_K(X)  # (batch, seq_len, d_k)
        V = self.W_V(X)  # (batch, seq_len, d_k)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        attn_weights = F.softmax(scores, dim=-1)
        output = torch.matmul(attn_weights, V)  # (batch, seq_len, d_k)
        return output
