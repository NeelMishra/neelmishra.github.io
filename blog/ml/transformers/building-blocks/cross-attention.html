    Cross-Attention: Bridging Two Streams

    January 2026 · 11 min read

    Parts 1–3 covered self-attention (query, key, value from the same sequence). Cross-attention is a variant where the query comes from one sequence and key/value from another. This is fundamental to encoder-decoder models (original Transformer, MT), multimodal fusion (Flamingo), and attention-based retrieval. This part derives the mechanism and explores its applications.

    Cross-Attention Definition

    Given two sequences—a source (encoded context) $X_s \in \mathbb{R}^{n_s \times d}$ and a target (query sequence) $X_t \in \mathbb{R}^{n_t \times d}$—cross-attention computes:

    $$Q = X_t W^Q \quad (n_t \times d_k)$$ $$K = X_s W^K, \quad V = X_s W^V \quad (n_s \times d_k, n_s \times d_v)$$ $$\text{CrossAttn}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

    The key difference: $Q$ depends on $n_t$, while $K$ and $V$ depend on $n_s$. The attention matrix $QK^T$ is $(n_t \times n_s)$ rather than square. Each target position can attend to all source positions; no position is masked (unless a padding mask is applied separately).

    Intuition: Cross-attention is "query the source." Each target token asks: "What information from the source is relevant to me?" The target token's query searches the source's keys and retrieves the corresponding values.

    Encoder-Decoder Architecture

    The original Transformer (Vaswani et al., 2017) uses this pattern for machine translation:

    1. Encoder: Stack of self-attention layers on source tokens (e.g., English sentence). Output: contextual representations $C \in \mathbb{R}^{n_s \times d}$.
    2. Decoder: Stack of layers, each with two attention types:
      • Self-attention on target (with causal masking) to allow each target token to see past target tokens.
      • Cross-attention with $Q$ from target, $K, V$ from encoder output $C$. Allows each target token to query the full source.
    3. Decoder loop: Generate tokens one-by-one. For each step, re-run self-attention (KV-cached) and cross-attention (encoder output is fixed, so $K, V$ are also fixed).

    Multimodal Fusion: Flamingo and Perceiver

    Modern vision-language models like Flamingo apply cross-attention to fuse image and text:

    • Source (images): Image patches encoded by a vision model (e.g., ViT). $X_s \in \mathbb{R}^{n_\text{patches} \times d}$.
    • Target (text): Text tokens decoded autoregressively. $X_t \in \mathbb{R}^{n_\text{text} \times d}$.
    • Cross-attention: Each text token queries the image patches. The model learns to retrieve visual information relevant to what it's trying to generate.

    This allows vision and language to interact asymmetrically: language is causal (autoregressive), images are not.

    Dimension Mismatch and Projection

    If the encoder and decoder operate on different vocabularies or feature dimensions, the projections $W^Q, W^K, W^V$ naturally reconcile them:

    • Encoder output dimension: $d_\text{enc}$
    • Decoder hidden dimension: $d_\text{dec}$
    • $W^Q \in \mathbb{R}^{d_\text{dec} \times d_k}$ (applied to decoder hidden states)
    • $W^K, W^V \in \mathbb{R}^{d_\text{enc} \times d_k}$ (applied to encoder output)

    As long as $d_k$ matches, cross-attention works. This flexibility is powerful for multimodal models where image and text have different native dimensions.

    Why No Causal Mask in Cross-Attention?

    The source is typically a fixed, complete context (e.g., a full sentence to translate, or a complete image). There's no causality constraint: each target token can see the entire source. Masking is unnecessary unless you're implementing prefix-constrained generation (rarely done).

    Complexity

    Cross-attention score matrix is $(n_t \times n_s)$, so:

    • Time: $O(n_t n_s d)$ for the matrix multiplications.
    • Space: $O(n_t n_s)$ to store the attention matrix.

    If $n_t \approx n_s \approx n$, this is $O(n^2 d)$, same as self-attention. But if the source is large (e.g., 1M image patches) and target is small (e.g., 100 text tokens), cross-attention is $O(100 \cdot 10^6 \cdot d)$—much cheaper than $O((10^6)^2 d)$ self-attention on the images alone.

    PyTorch: Cross-Attention Layer

    class CrossAttention(nn.Module):
    def __init__(self, d_model, d_k=None):
        super().__init__()
        self.d_k = d_k or d_model
        self.W_Q = nn.Linear(d_model, self.d_k)     # applied to target
        self.W_K = nn.Linear(d_model, self.d_k)     # applied to source
        self.W_V = nn.Linear(d_model, self.d_k)     # applied to source
    
    def forward(self, Q_seq, KV_seq):
        """
        Q_seq: (batch, n_target, d_model)  — decoder hidden states
        KV_seq: (batch, n_source, d_model) — encoder output
        Returns: (batch, n_target, d_k)
        """
        Q = self.W_Q(Q_seq)
        K = self.W_K(KV_seq)
        V = self.W_V(KV_seq)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        weights = F.softmax(scores, dim=-1)
        output = torch.matmul(weights, V)
        return output
