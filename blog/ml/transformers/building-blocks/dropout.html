    Dropout Layer

    April 2026 · ML Series · Transformers

    Why Dropout Exists

    Dropout is a regularization layer: it deliberately injects noise during training so the network cannot become too dependent on one exact activation path. In a transformer, this matters because the same residual stream is reused across many layers. If a model can solve a training example by leaning on a small set of features, those features can become brittle shortcuts rather than robust patterns.

    The idea is simple: during each training forward pass, randomly set some values to zero. The next step sees a slightly damaged version of the representation and must still produce a useful answer. Across many batches, the model learns to distribute information across multiple neurons, attention links, and residual updates instead of relying on one lucky feature.

    Core intuition: Dropout trains a family of noisy subnetworks that share weights. At inference time, we use the full network, with no random masking.

    The Operation

    For every scalar activation x_i, dropout samples a binary mask value m_i. The mask is 0 with probability p and 1 with probability 1 - p. Then it multiplies the activation by the mask:

    m_i ~ Bernoulli(1 - p) basic dropout: y_i = x_i * m_i if m_i = 0 -> value is dropped if m_i = 1 -> value survives

    This is element-wise and shape-preserving. If the input tensor has shape (batch, seq_len, d_model), the output has the same shape. Dropout does not add parameters, does not change dimensions, and does not learn anything directly. Its effect comes entirely from stochastic masking during training.

    Example with p = 0.3

    Roughly 30% of elements are zeroed on each training pass. A different random mask is sampled on the next pass, even for the same input.

    Inverted Dropout

    Basic dropout has one problem: if we zero out a fraction p of the values, the expected magnitude shrinks by 1 - p. A vector with sum 100 becomes a vector with expected sum 90 when p = 0.1. That means later layers would see systematically smaller activations during training than during inference.

    Modern libraries use inverted dropout. The surviving values are scaled up by 1 / (1 - p) during training:

    inverted dropout: y_i = (x_i * m_i) / (1 - p) Expected value: E[y_i] = P(m_i = 1) * x_i / (1 - p) + P(m_i = 0) * 0 = (1 - p) * x_i / (1 - p) = x_i

    This keeps the expected activation unchanged. The training-time output is noisy, but it is unbiased. At inference time, dropout can simply return the input unchanged.

    Drop rate p = 0.1
    10% of elements are zeroed on average.
    Keep rate q = 1 - p = 0.9
    90% of elements survive on average.
    Scale 1 / q = 1.111...
    Surviving elements are scaled up.

    Training vs Inference

    Dropout is one of the few transformer layers whose behavior changes between training and inference. During training, it samples a random mask. During inference, it is disabled.

    Mode Dropout behavior Why
    Training Sample a mask, zero selected values, scale survivors by 1/(1-p) Regularizes the model by forcing robust representations
    Evaluation / inference Return the input unchanged Use the full learned network deterministically
    Common bug: Forgetting to switch a model to evaluation mode leaves dropout active during inference. The same prompt can produce unnecessary random variation because internal activations are still being zeroed.

    Where Dropout Appears in Transformers

    Dropout is not one single box in the transformer diagram. It is a small operation reused at several points where the model might otherwise overfit to exact feature paths.

    Embedding dropout

    Applied after token embeddings and positional information are combined, before the first transformer block.

    Attention dropout

    Applied to attention probabilities after softmax in many implementations, randomly hiding token-to-token links.

    Residual dropout

    Applied to attention and FFN outputs before they are added back into the residual stream.

    FFN activation dropout

    Often applied between the two FFN linear layers, after the activation function.

    # Decoder-style pre-norm transformer block x = token_embedding(ids) + positional_encoding x = dropout(x, p) # embedding dropout attn_in = layer_norm_1(x) scores = QK^T / sqrt(d_k) probs = softmax(mask(scores)) probs = dropout(probs, p) # attention dropout attn_out = probs @ V attn_out = output_projection(attn_out) x = x + dropout(attn_out, p) # residual dropout ffn_in = layer_norm_2(x) h = activation(ffn_in @ W1 + b1) h = dropout(h, p) # FFN activation dropout ffn_out = h @ W2 + b2 x = x + dropout(ffn_out, p) # residual dropout

    Not every model uses every location. Some minimal implementations only use residual dropout. Some attention kernels fuse or omit attention-probability dropout. Many large modern LLMs use p = 0 because data scale and other regularizers are enough. But the conceptual role is the same: make the training path noisy while preserving the inference path.

    Attention Dropout

    Attention dropout deserves special attention because it is applied to a probability matrix, not a normal hidden vector. After softmax, each query token has a distribution over key tokens. Dropout can randomly zero some of those links:

    attention_probs = softmax(scores) attention_probs = dropout(attention_probs, p) context = attention_probs @ V

    With inverted dropout, the row sum is not guaranteed to be exactly 1 for a single training sample. Instead, the expected row sum remains 1. This is intentional: the attention distribution becomes a noisy communication graph. A token cannot always rely on the same neighbor being available, so it must learn backup paths.

    Residual Dropout

    Residual dropout is applied to the update before it is merged back into the residual stream:

    x_next = x + dropout(sublayer(layer_norm(x)), p)

    This is different from dropping the residual stream itself. The identity path x remains intact, so gradients still have a clean highway through the stack. Only the learned update is made noisy. This is why residual dropout plays well with deep transformers: it regularizes each block without destroying the skip connection.

    Good: Drop the update

    x + dropout(F(x))

    The residual highway stays available. The learned transformation is regularized.

    Risky: Drop the stream

    dropout(x + F(x))

    The identity path itself becomes noisy, which can make optimization less stable.

    Choosing the Dropout Rate

    The dropout rate is a training hyperparameter. In transformers, common values are small because the model already has strong regularization from data scale, weight decay, label smoothing, augmentation, and early stopping.

    Setting Typical p Notes
    Original Transformer base 0.1 Classic default from "Attention Is All You Need"
    Small dataset fine-tuning 0.1 - 0.3 More dropout can help when overfitting is visible
    Large-scale pretraining 0.0 - 0.1 Very large LLMs often reduce or disable dropout
    Inference disabled No random mask should be sampled

    If training loss is much lower than validation loss, dropout may help. If both losses are high, dropout can make underfitting worse. It is not a substitute for enough data, clean labels, a sensible learning rate, or a correctly sized model.

    Implementation

    A minimal framework implementation has only two branches: return the input during inference, otherwise sample the mask and scale the survivors.

    def dropout(x, p, training): if not training or p == 0: return x keep_prob = 1 - p mask = bernoulli(shape=x.shape, probability=keep_prob) return x * mask / keep_prob

    The important engineering detail is that the training flag must propagate through the whole model. In PyTorch, model.train() enables dropout and model.eval() disables it. In TensorFlow/Keras, the training argument controls the same behavior.

    Summary

    • Dropout randomly zeros activations during training and does nothing during inference.
    • Inverted dropout scales survivors by 1/(1-p), so the expected activation stays unchanged.
    • Transformers use dropout around embeddings, attention probabilities, FFN activations, and residual updates.
    • Residual dropout regularizes the learned update while preserving the identity path.
    • Small models and small datasets often benefit from dropout; very large pretrained LLMs often use little or none.
