Positional Encoding

April 2026 · ML Series · Transformers

The Problem: Attention Has No Sense of Order

Self-attention, the core operation of every Transformer, computes a weighted sum over all positions in the input sequence. For each query token, it calculates a similarity score against every key token, normalizes these scores through softmax, and uses the resulting weights to blend the value vectors. This mechanism is extraordinarily powerful. It is also completely blind to order.

More precisely, self-attention is permutation equivariant. If you take a sequence of input vectors and shuffle them into any order, the output of self-attention is shuffled in exactly the same way. The attention weights between any pair of tokens depend only on the content of those tokens, never on where they sit in the sequence. The function treats its input as a set, not a sequence.

This is a profound limitation. Consider two sentences:

"The dog bites the man"

The dog is the aggressor. The man is the victim.

"The man bites the dog"

The man is the aggressor. The dog is the victim. An entirely different meaning.

Both sentences contain the same set of tokens: {the, dog, bites, the, man}. A pure self-attention layer, with no position information, would produce identical intermediate representations for both sentences. The two meanings are indistinguishable. This is clearly unacceptable for any language task.

The same problem arises in every domain where transformers are used. In code, x = y + z and z = y + x assign different variables. In music, the order of notes defines the melody. In proteins, the sequence of amino acids determines the folding structure. Order is not optional information—it is fundamental.

A Formal Demonstration

Let us make the permutation equivariance precise. Let X be a matrix of shape (n × d) representing n token vectors. Let Π be any n × n permutation matrix. Self-attention computes:

Attention(X) = softmax(X W_Q (X W_K)^T / sqrt(d_k)) · X W_V

If we feed in the permuted input ΠX instead of X:

Attention(ΠX) = softmax(ΠX W_Q (ΠX W_K)^T / sqrt(d_k)) · ΠX W_V = softmax(Π · X W_Q W_K^T X^T · Π^T / sqrt(d_k)) · Π · X W_V = Π · softmax(X W_Q W_K^T X^T / sqrt(d_k)) · X W_V = Π · Attention(X)

The permutation passes straight through. The attention output is simply the original output, reshuffled. No positional information is captured or generated by the attention mechanism itself. If we want the Transformer to understand order, we must explicitly inject it.

Absolute vs Relative Position

There are two fundamentally different philosophies for encoding position. Understanding both is essential, because modern architectures mix and match these ideas extensively.

Absolute Position Encoding

Idea: assign each position in the sequence a unique vector. Position 0 gets one vector, position 1 gets another, and so on. This vector is added to (or concatenated with) the token embedding before the first attention layer.

What it encodes: "I am at position 7." Each token knows its own index.

Examples: Sinusoidal encoding (original Transformer), learned positional embeddings (BERT, GPT-2).

Limitation: Two tokens at positions 3 and 7 both know their own positions, but the relationship "4 apart" must be implicitly learned by the attention layers. It is not directly encoded.

Relative Position Encoding

Idea: instead of telling each token where it is, tell each pair of tokens how far apart they are. The encoding modifies the attention scores directly, adding a bias that depends on the distance (i − j) between positions i and j.

What it encodes: "Token j is 4 positions to my left." Each pair knows their relative distance.

Examples: Relative Position Representations (Shaw et al.), T5's relative bias, RoPE, ALiBi.

Advantage: Directly encodes the relationships that matter most for attention. "Look at the token 2 positions back" is a natural instruction for many linguistic patterns.

The original Transformer paper (Vaswani et al., 2017) used absolute sinusoidal encoding. However, the authors noted that sinusoidal encoding has a special property: PE(pos + k) can be expressed as a linear transformation of PE(pos). This means that even though the encoding is absolute, the model can learn to recover relative distances. Modern methods like RoPE take this further by directly encoding relative position into the attention computation.

Neither approach is strictly superior. Absolute encoding is simpler and works well for fixed-length tasks. Relative encoding generalizes better to longer sequences and more naturally captures the local patterns that dominate natural language. In practice, the choice depends on the application and the desired extrapolation behavior.

Sinusoidal Positional Encoding

The original Transformer paper introduced a beautifully elegant solution: encode position using sine and cosine functions at different frequencies. This approach requires no learned parameters—the encoding is entirely deterministic.

The Formula

PE(pos, 2i) = sin(pos / 10000^(2i / d_model)) PE(pos, 2i + 1) = cos(pos / 10000^(2i / d_model))

This is a matrix of shape (max_seq_len, d_model). Each row is the positional encoding for one position. Each column corresponds to one dimension. Let us unpack every component.

Component: pos

The position index in the sequence: 0, 1, 2, ..., n−1. This is the "input" to the encoding function. Position 0 is the first token, position 1 is the second, and so on. Each position gets its own unique row in the PE matrix.

Component: i (dimension index)

The dimension index within the encoding vector. It ranges from 0 to d_model/2 − 1. Each value of i corresponds to a pair of dimensions: dimension 2i uses sin, dimension 2i+1 uses cos. For d_model = 512, there are 256 unique frequency levels, each producing a (sin, cos) pair.

Component: 10000^(2i / d_model) — the wavelength denominator

This is the heart of the encoding. It controls the frequency of each sinusoid. When i = 0 (the first dimension pair), the denominator is 10000^0 = 1, so the argument is simply pos. The sinusoid oscillates very quickly, completing a full cycle every 2π ≈ 6.28 positions.

When i = d_model/2 − 1 (the last dimension pair), the denominator is 10000^1 = 10000, so the argument is pos / 10000. The sinusoid oscillates extremely slowly, completing one full cycle every 2π × 10000 ≈ 62,832 positions.

The intermediate dimensions interpolate geometrically between these extremes. This creates a spectrum of frequencies spanning over four orders of magnitude.

Why Sin and Cos Together?

Using both sin and cos for each frequency is not arbitrary. A single sinusoid has an ambiguity problem: sin(θ) = sin(π − θ), so two different positions can produce the same value. By pairing sin and cos at the same frequency, we get a unique (sin, cos) pair for every position within each wavelength. This is the same idea as representing a point on a circle using (cos θ, sin θ) rather than just one coordinate.

Geometrically, for each dimension pair, the positional encoding traces a circle (or a helix, if we include the position axis). Different frequencies trace circles at different speeds. The complete positional encoding is a point in a high-dimensional space that combines all these circular trajectories, creating a unique fingerprint for every position.

Frequency Intuition: A Multi-Resolution Number System

The best way to build intuition for sinusoidal encoding is to think of it as a continuous analog of binary representation.

The Binary Analogy

In binary, we represent a number using multiple digits at different scales:

Position 0: 0000 Position 1: 0001 Position 2: 0010 Position 3: 0011 Position 4: 0100 Position 5: 0101 Position 6: 0110 Position 7: 0111

Notice the pattern. The rightmost bit (ones digit) flips every position. The next bit (twos digit) flips every 2 positions. The next (fours digit) flips every 4 positions. Each digit operates at a different frequency, and together they uniquely identify every position.

Sinusoidal encoding follows the same principle, but replaces hard binary flips with smooth sinusoidal waves:

Low-index dimensions (i near 0)

Like the ones digit in binary. These are the fast oscillators. They cycle rapidly, changing significantly between adjacent positions. They encode fine-grained, local position information. The model can use these dimensions to distinguish position 42 from position 43.

High-index dimensions (i near d_model/2)

Like the highest-order digit in binary. These are the slow oscillators. They change very gradually over hundreds or thousands of positions. They encode coarse, global position information. The model can use these dimensions to distinguish "near the beginning" from "near the middle" from "near the end."

Why Multiple Resolutions Matter

Natural language contains patterns at multiple scales. Some patterns are extremely local: word order within a phrase ("the big red dog" vs "the red big dog"), subject-verb agreement within a clause, or the position of punctuation. Other patterns span longer distances: the relationship between a pronoun and its antecedent paragraph-level earlier, or the connection between a topic sentence and a concluding summary.

By encoding position at many different frequencies simultaneously, the sinusoidal encoding gives the Transformer access to position information at every scale it might need. The high-frequency dimensions handle local patterns, and the low-frequency dimensions handle long-range patterns. The model, through its attention weights, learns to read whichever dimensions are relevant for each particular pattern.

Intuition Summary

Think of each dimension pair as a clock hand rotating at a different speed. The first pair is a second hand (fast), the next is a minute hand (moderate), and the last is an hour hand (very slow). At any given moment, the combination of all the clock hands gives a unique reading. Two moments close in time have similar hour-hand positions but different second-hand positions. Two moments far apart differ in everything. This is exactly how sinusoidal positional encoding works.

The Key Property: Relative Position is Linear

The most important mathematical property of sinusoidal encoding is that a fixed offset in position corresponds to a fixed linear transformation of the encoding vector. In other words, PE(pos + k) = M_k · PE(pos), where M_k is a matrix that depends only on k, not on pos.

This means the model can learn to attend to "the token k positions to the left" using a single set of learned weights, regardless of where in the sequence it currently is. This is the bridge between absolute encoding and relative position awareness.

The Rotation Matrix Proof

Consider a single dimension pair (2i, 2i+1). The sinusoidal encoding for position pos is:

PE(pos) = [ sin(pos · ω_i), cos(pos · ω_i) ] where ω_i = 1 / 10000^(2i / d_model)

Now consider position pos + k. Using the angle addition formulas for sin and cos:

sin((pos + k) · ω) = sin(pos · ω) cos(k · ω) + cos(pos · ω) sin(k · ω) cos((pos + k) · ω) = cos(pos · ω) cos(k · ω) - sin(pos · ω) sin(k · ω)

In matrix form:

[ sin((pos+k) · ω) ] [ cos(kω) sin(kω) ] [ sin(pos · ω) ] [ ] = [ ] [ ] [ cos((pos+k) · ω) ] [ -sin(kω) cos(kω) ] [ cos(pos · ω) ] PE(pos + k) = R_k · PE(pos)

The matrix R_k is a rotation matrix. It rotates the (sin, cos) pair by an angle kω_i. Crucially, R_k depends only on k and i, not on pos. The same rotation works for any starting position.

The full encoding vector across all d_model dimensions can be written as a block-diagonal rotation:

PE(pos + k) = BlockDiag(R_k^{(0)}, R_k^{(1)}, ..., R_k^{(d/2-1)}) · PE(pos)

Each 2×2 block rotates its dimension pair by a different angle (because each pair has a different frequency ω_i). The full transformation is a composition of independent rotations in d_model/2 orthogonal subspaces.

Why This Matters for Attention

In self-attention, the model computes q_i · k_j for each pair of positions. If Q and K are functions of the positional encoding, then the dot product q_i · k_j will contain information about the positions i and j. Because the encoding of pos + k is a linear function of the encoding of pos, the attention weight matrix W can learn to "decode" relative positions. Specifically, a linear layer applied to PE(pos + k) can extract the relative offset k from PE(pos) using a fixed set of weights.

This is a remarkable property. Even though the encoding is absolute (each position gets a fixed vector), the relationships between encodings are structured enough for the model to recover relative information. It is this property that inspired the development of RoPE, which makes the relative encoding explicit rather than implicit.

Addition, Not Concatenation

In the original Transformer, positional encoding is added to the token embedding, not concatenated:

input_to_attention = TokenEmbedding(token) + PositionalEncoding(pos)

This is a design choice that warrants explanation. Why not concatenate, which would keep the two sources of information cleanly separated?

Why Addition Works

Reason 1: Dimension Preservation

Addition keeps the dimensionality at d_model. Concatenation would double it to 2×d_model, which increases the parameter count of every subsequent weight matrix (Q, K, V projections, feed-forward layers). For large d_model, this is a substantial computational cost for what turns out to be a marginal benefit.

Reason 2: The Orthogonality Argument

In high-dimensional spaces, randomly chosen vectors are approximately orthogonal. Token embeddings and positional encodings live in the same d_model-dimensional space, but they occupy largely non-overlapping subspaces. The model can learn to project onto directions that isolate position from content, or attend to combinations of both. Addition is not as destructive as it might seem in low dimensions because high-dimensional geometry provides ample room.

Reason 3: Implicit Bias Interpretation

Adding the positional encoding is equivalent to adding a position-dependent bias to the token embedding. The first attention layer sees x + PE(pos), which is like saying "this token is here." The model learns through its weight matrices to use or ignore this positional bias as needed. Some attention heads learn to focus primarily on position (syntactic heads), while others focus on content (semantic heads).

Reason 4: Empirical Validation

Vaswani et al. tested concatenation and found no meaningful improvement over addition. Subsequent work has consistently confirmed this: the simple addition approach works well across tasks and domains. The overhead of concatenation is not justified by the results.

What Happens Inside the Model

After addition, the combined vector enters the Q, K, V projections. Consider the query projection:

q_i = W_Q · (embed_i + PE(i)) = W_Q · embed_i + W_Q · PE(i) = content_query + position_query

The query decomposes into a content-based component and a position-based component. The same decomposition applies to keys. When computing the attention score q_i · k_j, we get four terms:

q_i · k_j = (content_q_i + pos_q_i) · (content_k_j + pos_k_j) = content_q · content_k (content-to-content) + content_q · pos_k (content-to-position) + pos_q · content_k (position-to-content) + pos_q · pos_k (position-to-position)

The model learns weight matrices that balance these four components. Some heads may amplify the position-to-position term (attending by proximity), while others emphasize content-to-content (attending by meaning). This decomposition shows that addition does not lose information—it combines it in a way the model can learn to disentangle.

Learned Positional Embeddings

An alternative to the fixed sinusoidal encoding is to simply learn a positional embedding table during training. This is the approach used by BERT, GPT-2, GPT-3, and most modern encoder-only and decoder-only models.

How It Works

Instead of computing PE(pos) via a formula, the model maintains a trainable parameter matrix of shape (max_seq_len, d_model). Each row is initialized randomly and updated through backpropagation, just like any other parameter in the model.

# Learned positional embedding class LearnedPositionalEmbedding(nn.Module): def __init__(self, max_seq_len, d_model): super().__init__() self.embedding = nn.Embedding(max_seq_len, d_model) def forward(self, x): # x: (batch, seq_len, d_model) positions = torch.arange(x.size(1), device=x.device) return x + self.embedding(positions)

Advantages

  • Maximum flexibility. The embedding can learn any pattern, not just sinusoidal waves. If certain positions have special roles (like [CLS] at position 0 in BERT), the model can learn a unique embedding for them.
  • Task adaptation. For specific downstream tasks, the optimal positional representation may differ from what sine and cosine provide. Learning lets the model adapt.
  • Simplicity. No formula to implement or explain. It is just a lookup table.

Disadvantages

  • No extrapolation. If the model is trained with max_seq_len = 512, it has no embedding for position 513. It literally does not know what to do. Sinusoidal encoding, by contrast, can generate a valid encoding for any position by simply evaluating the formula.
  • Extra parameters. The positional embedding table adds max_seq_len × d_model parameters. For max_seq_len = 2048 and d_model = 768, this is 1.57 million parameters—not huge compared to the full model, but not free either.
  • No built-in relative awareness. There is no guarantee that the learned embeddings have the linear relative-position property of sinusoidal encodings. The model might learn something like it, but it is not structurally guaranteed.

Sinusoidal vs Learned: Does It Matter?

The original Transformer paper reported nearly identical performance between sinusoidal and learned positional embeddings. Subsequent work (GPT-2, BERT) defaulted to learned embeddings for their simplicity and flexibility. In practice, the choice between sinusoidal and learned rarely makes a significant difference for models trained on sequences within the training length.

The difference becomes important when extrapolation is required—when the model needs to handle sequences longer than those seen during training. This is where methods like RoPE and ALiBi have clear advantages.

RoPE: Rotary Position Embeddings

Rotary Position Embeddings (RoPE), introduced by Su et al. (2021), are used in LLaMA, LLaMA 2, GPT-NeoX, PaLM, and many of the most capable modern LLMs. RoPE takes the rotation insight from sinusoidal encoding and applies it in the most natural place: directly inside the attention computation.

Core Idea

Instead of adding position to the input embeddings, RoPE rotates the query and key vectors based on their position. The rotation angle depends on both the position and the dimension pair. The key insight is that when you compute the dot product of a rotated query and a rotated key, the rotation angles combine in a way that depends only on the relative position between the two tokens.

The Formula

For a pair of dimensions (2i, 2i+1) in the query vector at position m:

q_m^{(2i)} = q^{(2i)} cos(mθ_i) - q^{(2i+1)} sin(mθ_i) q_m^{(2i+1)} = q^{(2i)} sin(mθ_i) + q^{(2i+1)} cos(mθ_i) where θ_i = 1 / 10000^(2i / d_model)

The same rotation is applied to the key vector at position n. When computing the dot product q_m · k_n, the rotation matrices combine:

q_m · k_n = sum_i [ (q^{(2i)} cos(mθ) - q^{(2i+1)} sin(mθ)) · (k^{(2i)} cos(nθ) - k^{(2i+1)} sin(nθ)) + (q^{(2i)} sin(mθ) + q^{(2i+1)} cos(mθ)) · (k^{(2i)} sin(nθ) + k^{(2i+1)} cos(nθ)) ]

After simplification using trigonometric identities, this reduces to terms involving cos((m−n)θ_i) and sin((m−n)θ_i). The dot product depends only on (m − n), the relative distance, not on the absolute positions m and n.

Why RoPE is Powerful

  • Relative position is baked in. The dot product q_m · k_n automatically encodes relative position (m − n) without needing learned relative position biases or lookup tables.
  • No extra parameters. Like sinusoidal encoding, RoPE is deterministic and parameter-free. It only requires computing rotation angles from a fixed formula.
  • Extrapolation. Because the rotation angle is a continuous function of position, RoPE can handle positions beyond the training range (though performance does degrade for very long extrapolations without techniques like NTK-aware scaling or YaRN).
  • Compatibility with KV caching. The rotated keys can be cached during autoregressive generation. The position is encoded in the key at the time of computation, so cached keys remain valid.

RoPE Geometric Intuition

Imagine each query and key as a collection of 2D arrows (one per dimension pair). RoPE rotates each arrow by an angle proportional to the position. When the query at position m and the key at position n are compared via dot product, the result depends on the angle difference (m − n)θ_i. This is just the cosine similarity in a rotated coordinate system, which naturally captures relative distance.

ALiBi: Attention with Linear Biases

ALiBi (Press et al., 2022) takes a radically different approach: no positional encoding at all. Instead, it directly modifies the attention scores by subtracting a penalty proportional to the distance between positions.

The Formula

score(i, j) = q_i · k_j - m · |i - j|

Where m is a slope that differs per attention head. In a model with 8 heads, the slopes are typically set as a geometric sequence:

m_1 = 1/1, m_2 = 1/2, m_3 = 1/4, m_4 = 1/8, m_5 = 1/16, m_6 = 1/32, m_7 = 1/64, m_8 = 1/128

Or more generally, m_h = 2^(−8h/H) for head h out of H total heads.

How It Works

The penalty −m·|i−j| acts like a recency bias. Nearby tokens have small penalties and are easy to attend to. Distant tokens have large penalties and are harder to attend to. The strength of this bias varies by head: heads with large m have a strong local bias (focusing on nearby tokens), while heads with small m have a weaker bias (allowing longer-range attention).

Example: 4-token sequence, m = 0.5

The bias matrix −m·|i−j| for this head looks like:

pos 0 pos 1 pos 2 pos 3 pos 0 [ 0.0 -0.5 -1.0 -1.5] pos 1 [-0.5 0.0 -0.5 -1.0] pos 2 [-1.0 -0.5 0.0 -0.5] pos 3 [-1.5 -1.0 -0.5 0.0]

This bias is added to the raw Q·K^T scores before softmax. After softmax, the effect is that distant tokens are exponentially downweighted relative to nearby tokens.

Why ALiBi Excels at Extrapolation

  • No position encoding to break. There are no position embeddings that were trained only up to some maximum length. The linear bias is defined for any distance, so the model naturally handles longer sequences.
  • Smooth degradation. As tokens get further apart, the penalty grows linearly, causing attention to smoothly decrease. There are no discontinuities or out-of-distribution artifacts.
  • Simplicity. ALiBi requires no trainable position parameters. The slopes are fixed and the bias matrix can be precomputed. Implementation is trivially simple.

ALiBi was used in the BLOOM model (176B parameters) and has been shown to train on 1024-token sequences and extrapolate effectively to 2048 tokens without fine-tuning, whereas models with learned embeddings fail catastrophically at lengths beyond their training range.

Implementation: Generating Sinusoidal Encoding

Here is a clean pseudo-code implementation that generates the sinusoidal positional encoding matrix. This is the code that runs once during model initialization and produces the fixed encoding table.

def sinusoidal_positional_encoding(max_len, d_model): # Create an empty matrix: (max_len, d_model) PE = zeros(max_len, d_model) # Position indices: [0, 1, 2, ..., max_len - 1] pos = arange(0, max_len).reshape(max_len, 1) # column vector # Dimension indices for each pair: [0, 1, 2, ..., d_model/2 - 1] i = arange(0, d_model, 2) # step by 2 # Compute the denominator: 10000^(2i / d_model) # Use log-space for numerical stability: # 10000^(2i/d) = exp(2i * ln(10000) / d) div_term = exp(i * (-log(10000.0) / d_model)) # Compute sin for even indices, cos for odd indices PE[:, 0::2] = sin(pos * div_term) # even columns PE[:, 1::2] = cos(pos * div_term) # odd columns return PE

Key Implementation Detail: Log-Space Computation

The denominator 10000^(2i/d_model) spans many orders of magnitude (from 1 to 10000). Computing it directly as pow(10000, 2*i/d_model) can cause numerical precision issues for large values. The standard trick is to work in log space:

10000^(2i/d) = exp(2i · ln(10000) / d) = exp(-2i · ln(10000) / d)^(-1) In code: div_term = exp(i * (-log(10000) / d_model)) This computes the RECIPROCAL of the denominator (i.e., ω_i), avoiding overflow.

PyTorch Implementation

import torch import math class SinusoidalPositionalEncoding(torch.nn.Module): def __init__(self, d_model, max_len=5000): super().__init__() pe = torch.zeros(max_len, d_model) pos = torch.arange(0, max_len).unsqueeze(1).float() div = torch.exp( torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model) ) pe[:, 0::2] = torch.sin(pos * div) pe[:, 1::2] = torch.cos(pos * div) # Register as non-trainable buffer self.register_buffer('pe', pe.unsqueeze(0)) def forward(self, x): # x: (batch, seq_len, d_model) return x + self.pe[:, :x.size(1), :]

The key detail is register_buffer: the positional encoding is stored as a non-trainable buffer. It is saved with the model state but is not updated by the optimizer. This is the correct approach because the sinusoidal values are fixed by definition.

Summary

  • Self-attention is permutation equivariant. Without position information, the Transformer treats its input as an unordered set and cannot distinguish "dog bites man" from "man bites dog."
  • Positional encoding injects order. It gives each token a position-dependent signal that the model uses to understand sequence structure.
  • Sinusoidal encoding uses sin/cos waves at geometrically spaced frequencies, creating a unique fingerprint per position with no learned parameters. Low dimensions oscillate fast (local detail), high dimensions oscillate slowly (global structure).
  • The linear property of sinusoidal encoding allows PE(pos + k) to be expressed as a rotation of PE(pos), bridging absolute encoding with relative position awareness.
  • Addition to embeddings works because high-dimensional spaces allow content and position information to coexist without destructive interference. The model learns to separate them.
  • Learned embeddings offer maximum flexibility but cannot extrapolate beyond the training sequence length.
  • RoPE rotates Q and K vectors by position, explicitly encoding relative position in the attention dot product. It is the dominant method in modern LLMs.
  • ALiBi skips encoding entirely and adds a distance-proportional penalty to attention scores, achieving excellent length extrapolation with zero parameters.
  • The field has moved from absolute (sinusoidal, learned) toward relative (RoPE, ALiBi) methods, driven by the need for longer context windows and better generalization.
