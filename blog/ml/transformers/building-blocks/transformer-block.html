    The Transformer Block

    April 2026 · ML Series · Transformers

    The Big Picture

    Every modern language model, from BERT to GPT-4 to LLaMA, is built from a single repeating unit: the transformer block. Take this block, stack N copies of it, and you have a transformer. The magic is not in any one component but in how those components compose together.

    Each transformer block contains exactly two sub-layers:

    1. Multi-Head Self-Attention - lets every token attend to every other token, capturing relationships across the entire sequence.
    2. Position-wise Feed-Forward Network (FFN) - a two-layer MLP applied independently to each token position, giving the model nonlinear capacity to transform the attended representations.

    Each sub-layer is wrapped with two critical mechanisms:

    • Residual (skip) connection - adds the sub-layer’s input directly to its output.
    • Layer normalization - stabilises the distribution of activations.

    This pattern - normalize, transform, add back - is repeated millions of times during training and inference. Understanding it deeply is essential to understanding transformers at all.

    Key insight: The transformer block is a function from (seq_len, d_model) to (seq_len, d_model). It preserves the shape of its input. This is what makes stacking possible: the output of block i is directly usable as the input of block i+1.

    The original “Attention Is All You Need” paper (Vaswani et al., 2017) used N = 6 blocks in both encoder and decoder. Modern models use 32 (LLaMA-7B), 48 (GPT-3), or even 80+ (GPT-4 rumored). The number of blocks is the primary knob for controlling model depth and capacity.

    Let’s zoom into each piece, understand why it exists, and then animate data flowing through the entire block step by step.

    Residual Connections: Why They’re Essential

    The Vanishing Gradient Problem

    Consider a deep network with L layers. During backpropagation, the gradient of the loss with respect to the parameters of layer l is computed via the chain rule:

    ∂Loss/∂θl = ∂Loss/∂aL · ∂aL/∂aL-1 · ... · ∂al+1/∂al · ∂al/∂θl

    Each factor ∂ai+1/∂ai is the Jacobian of layer i+1 with respect to its input. If the spectral norm of these Jacobians is consistently less than 1, the product shrinks exponentially with depth. After 12 or more layers, gradients reaching the early layers become vanishingly small, and those layers effectively stop learning.

    This is exactly what happened with pre-ResNet CNNs and deep RNNs. A 20-layer plain network could perform worse than a 10-layer one, not because of overfitting, but because the deeper network could not be trained at all.

    The Gradient Highway

    Residual connections (He et al., 2015) elegantly solve this by creating a shortcut for gradient flow. Instead of computing y = F(x), we compute:

    y = F(x) + x

    The gradient of y with respect to x is now:

    ∂y/∂x = ∂F(x)/∂x + I

    That + I (the identity matrix) is the critical part. Even if ∂F(x)/∂x has small singular values, the gradient can always flow through the identity path. The gradient highway is always open.

    For a stack of N residual blocks, the gradient from block N to block 1 includes a path that is just I · I · ... · I = I. This means the loss has a direct, unattenuated gradient path to every single block in the network. Without this, transformers with 48 or 80 blocks would be impossible to train.

    Gradient Flow: With vs Without Residuals

    In practice, the sub-layer function F(x) learns to compute a delta - a small correction to the identity. Early in training, the weights are small and F(x) ≈ 0, so the block acts almost like an identity function. As training progresses, each block learns increasingly useful transformations while always preserving the ability for gradients to flow unimpeded.

    Analogy: Think of residual connections like a highway with exits. Information flows freely along the highway (the skip connection). Each block is an exit ramp where the data can optionally be transformed, and then the result is merged back onto the highway. Even if one exit ramp is “closed” (produces near-zero output), traffic still flows.

    Pre-Norm vs Post-Norm

    Where you place the layer normalization relative to the sub-layer and residual connection is the single most important architectural decision in transformer block design. It affects training stability, learning rate sensitivity, and whether you need warmup.

    Quick Recap: What LayerNorm Does

    Layer normalization (Ba et al., 2016) normalises each token’s feature vector independently:

    LayerNorm(x) = γ · (x − μ) / √(σ² + ε) + β where μ = mean(x), σ² = var(x) (computed over the d_model dimension) γ, β are learned scale and shift parameters of shape (d_model,)

    It ensures that the input to each sub-layer has zero mean and unit variance (before the learned affine transformation), preventing the activations from drifting to extreme values as they pass through many layers.

    Post-Norm (Original Transformer)

    The original “Attention Is All You Need” paper placed layer normalization after the residual addition:

    x = LayerNorm(x + Sublayer(x))

    The data flow is: Input → Sublayer → Add residual → LayerNorm → Output.

    Pre-Norm (Modern Standard)

    GPT-2, GPT-3, LLaMA, and nearly all modern transformers use pre-norm, where layer normalization is applied before the sub-layer:

    x = x + Sublayer(LayerNorm(x))

    The data flow is: Input → LayerNorm → Sublayer → Add residual → Output.

    Side-by-Side Comparison

    🔴 Post-Norm (Original, 2017)

    x′ = LayerNorm(x + MHA(x, x, x)) x″ = LayerNorm(x′ + FFN(x′))

    Pros:

    • Slightly better final performance in some settings
    • Output of each block is normalised

    Cons:

    • Training is unstable without careful learning rate warmup
    • Gradients through the residual path pass through LayerNorm, which can distort them
    • Sensitive to initialisation
    • Harder to scale beyond 12-24 layers without tricks

    🟢 Pre-Norm (Modern, GPT-2+)

    x′ = x + MHA(LayerNorm(x), ...) x″ = x′ + FFN(LayerNorm(x′))

    Pros:

    • Much more stable training
    • Residual path is never normalised - gradients flow freely
    • Can use larger learning rates, less warmup needed
    • Scales to 80+ layers with no special tricks

    Cons:

    • Needs a final LayerNorm after the last block
    • Marginally lower performance in some careful comparisons

    Why Pre-Norm Is More Stable

    The key insight is about the residual stream. In pre-norm, the main data path looks like:

    x → (+) → (+) → (+) → ... → output ↑ ↑ ↑ Sub1 Sub2 Sub3

    The main stream (the horizontal arrows) is a simple sum: output = x + Sub1(LN(x)) + Sub2(LN(x′)) + .... The residual path is never passed through a normalisation layer. This means gradients flowing backward through the residual path are never rescaled, clipped, or distorted by LayerNorm’s Jacobian.

    In post-norm, the residual stream passes through LayerNorm at every block. Each LayerNorm applies a nonlinear transformation (because of the division by standard deviation), and its Jacobian can amplify or suppress gradient components. After 12+ LayerNorm applications, the cumulative distortion makes the optimisation landscape rugged and hard to navigate without very careful warmup schedules.

    Rule of thumb: If you’re building a transformer from scratch, use pre-norm. It’s what GPT-2, GPT-3, GPT-4, LLaMA, Mistral, and nearly every modern LLM uses. Post-norm mainly survives in older BERT-style models and some specialised settings.

    The Complete Forward Pass (Pre-Norm)

    Let’s trace through every operation in a single pre-norm transformer block. The input is a tensor x of shape (batch, seq_len, d_model).

    Sub-Layer 1: Multi-Head Self-Attention

    Step 1 - Layer Normalization:

    h = LayerNorm(x) // shape: (B, T, d_model)

    Each token vector is independently normalised to have zero mean and unit variance across the d_model dimension. The learned scale γ and shift β then re-scale it. This stabilises the input to the attention mechanism, preventing the queries and keys from having wildly different magnitudes.

    Step 2 - Multi-Head Attention:

    h = MultiHeadAttention(Q=h, K=h, V=h) // shape: (B, T, d_model)

    For self-attention, Q, K, and V all come from the same source (h). Inside MHA: each head projects to d_k = d_model/h, computes scaled dot-product attention, and the heads are concatenated and projected back to d_model. Each token now contains information aggregated from the entire sequence based on learned attention patterns.

    Step 3 - Dropout:

    h = Dropout(h, p=0.1) // shape: (B, T, d_model)

    During training, each element is randomly zeroed with probability p and the remaining elements are scaled by 1/(1−p). This provides regularisation, preventing co-adaptation of features. During inference, dropout is a no-op.

    Step 4 - Residual Addition:

    x = x + h // shape: (B, T, d_model)

    The original input x is added element-wise to the attention output. This is the skip connection. Even if the attention sub-layer produced poor outputs early in training, the original signal is preserved.

    Sub-Layer 2: Position-wise Feed-Forward Network

    Step 5 - Layer Normalization:

    h = LayerNorm(x) // shape: (B, T, d_model)

    The (now attention-enriched) x is normalised again before entering the FFN. This is a separate LayerNorm with its own γ and β parameters.

    Step 6 - Feed-Forward Network:

    h = W₂ · GELU(W₁ · h + b₁) + b₂ // shape: (B, T, d_model) ↑ ↑ (d_ff → d_model) (d_model → d_ff)

    The FFN first expands from d_model to d_ff (typically 4 × d_model), applies a nonlinear activation (GELU in modern models, ReLU in the original), and then compresses back to d_model. This expand-activate-compress pattern gives each token a “thinking step” where the model can compute complex nonlinear functions of the attended representation.

    Step 7 - Dropout:

    h = Dropout(h, p=0.1) // shape: (B, T, d_model)

    Step 8 - Residual Addition:

    x = x + h // shape: (B, T, d_model)

    Again, the residual connection preserves the information from before the FFN. The output of the entire block is this final x, ready to be fed into the next block.

    Shape invariant: Notice that the input and output are both (B, T, d_model). Every operation inside the block either preserves this shape (LayerNorm, Dropout, residual add) or internally changes it and changes it back (MHA’s head split/concat, FFN’s expand/compress). This shape invariance is what lets us stack blocks arbitrarily.

    Encoder Block vs Decoder Block

    The transformer architecture comes in two flavours of blocks, and a decoder block has a critical extra sub-layer.

    Encoder Block (2 sub-layers)

    The encoder block is what we described above:

    1. Multi-Head Self-Attention - every token attends to every other token (bidirectional). No masking is applied, so token 3 can attend to tokens 1, 2, 4, 5, etc.
    2. Feed-Forward Network - position-wise nonlinear transformation.

    Models like BERT and RoBERTa use only encoder blocks. They see the entire sequence at once, which is ideal for classification, NER, and tasks where context from both directions is useful.

    Decoder Block (3 sub-layers)

    The decoder block has three sub-layers:

    1. Masked Multi-Head Self-Attention - token i can only attend to tokens ≤ i. A causal mask (upper triangular of −∞) prevents information from future tokens from leaking into the present. This is essential for autoregressive generation.
    2. Multi-Head Cross-Attention - Q comes from the decoder, but K and V come from the encoder output. This is how the decoder “reads” the encoded input. Without this, the decoder would be generating text without any conditioning on the source.
    3. Feed-Forward Network - same position-wise FFN as the encoder.

    Each sub-layer has its own residual connection and LayerNorm (pre-norm or post-norm, depending on the architecture).

    Architecture Comparison

    Decoder-only models (GPT-2, GPT-3, GPT-4, LLaMA, Mistral, Falcon) drop the cross-attention sub-layer entirely. Their blocks have just two sub-layers: masked self-attention + FFN. This is the dominant architecture for large language models today.

    Cross-Attention in Detail

    In the encoder-decoder transformer (used in machine translation, T5, BART), the cross-attention sub-layer works as follows:

    Q = decoder_hidden @ W_Q // from decoder state K = encoder_output @ W_K // from encoder output V = encoder_output @ W_V // from encoder output Attention(Q, K, V) = softmax(QKT / √d_k) V

    This means the decoder “queries” the encoder. Each decoder token decides how much to attend to each position in the encoded source sequence. For translation, when generating “le” in French, the cross-attention might heavily attend to “the” in the English source.

    Stacking Blocks: Depth and Emergent Specialisation

    Why Stack?

    A single transformer block can compute one round of self-attention and one nonlinear transformation. This is enough to capture direct pairwise relationships (e.g., “the cat sat”), but natural language has compositional structure that requires iterative refinement.

    Consider the sentence: “The keys to the cabinet were rusty.” To correctly resolve that “were” agrees with “keys” (not “cabinet”), the model needs to:

    1. Identify that “to the cabinet” is a prepositional phrase (syntactic structure)
    2. Determine the head noun is “keys” (dependency parsing)
    3. Apply agreement rules (morphological reasoning)

    Each block refines the representations, allowing later blocks to build on the patterns detected by earlier blocks.

    Emergent Depth Specialisation

    Research (e.g., Tenney et al., 2019; Clark et al., 2019) has shown that transformer layers naturally specialise:

    Layer DepthWhat It CapturesExamples
    Layers 1-3 (early)Surface / lexical patternsWord identity, POS tags, local context
    Layers 4-8 (middle)Syntactic structureParse trees, dependency arcs, NP/VP chunking
    Layers 9-12 (later)Semantic / task-specificCoreference, entailment, world knowledge
    Layers 20+ (deep)Abstract / reasoningMulti-hop inference, planning, complex logic

    This specialisation is not explicitly programmed; it emerges from the training objective. Early layers learn what’s “easy” (local patterns), and later layers build on those representations to solve harder problems.

    Parameter Count Per Block

    Let’s calculate exactly how many trainable parameters a single transformer block contains. We use d = d_model = 512 and d_ff = 2048.

    Multi-Head Attention Parameters

    W_Q: d × d = 512 × 512 = 262,144 b_Q: d = 512 W_K: d × d = 512 × 512 = 262,144 b_K: d = 512 W_V: d × d = 512 × 512 = 262,144 b_V: d = 512 W_O: d × d = 512 × 512 = 262,144 b_O: d = 512 Total MHA = 4 × (d² + d) = 4 × 262,656 = 1,050,624

    Note: the split into 8 heads doesn’t change the parameter count. Each head has its own d_k × d_k slice of the weight matrices, and together the slices form the full d × d matrices.

    Feed-Forward Network Parameters

    W₁: d × d_ff = 512 × 2048 = 1,048,576 b₁: d_ff = 2,048 W₂: d_ff × d = 2048 × 512 = 1,048,576 b₂: d = 512 Total FFN = d × d_ff + d_ff + d_ff × d + d = 2,099,712

    Layer Normalization Parameters

    LayerNorm₁: γ (d) + β (d) = 512 + 512 = 1,024 LayerNorm₂: γ (d) + β (d) = 512 + 512 = 1,024 Total LN = 2 × 2d = 2,048

    The FFN dominates: Two-thirds of the parameters in each block are in the feed-forward network, not in attention. This is because the FFN has two large matrices (d × d_ff and d_ff × d), while attention has four matrices that are only d × d. With the standard d_ff = 4d, the FFN has 8d² parameters vs attention’s 4d².

    Implementation: Pseudo-Code

    Here is a clean pseudo-code implementation of a single pre-norm transformer block. This corresponds directly to the forward pass we traced above.

    class TransformerBlock: def __init__(self, d_model, n_heads, d_ff, dropout_rate=0.1): // Sub-layer 1: Multi-Head Attention self.ln1 = LayerNorm(d_model) self.mha = MultiHeadAttention(d_model, n_heads) self.drop1 = Dropout(dropout_rate) // Sub-layer 2: Feed-Forward Network self.ln2 = LayerNorm(d_model) self.ffn = FeedForward(d_model, d_ff) self.drop2 = Dropout(dropout_rate) def forward(self, x, mask=None): // Sub-layer 1: Attention with pre-norm + residual h = self.ln1(x) // normalize h = self.mha(Q=h, K=h, V=h, mask=mask) // attend h = self.drop1(h) // regularise x = x + h // residual add // Sub-layer 2: FFN with pre-norm + residual h = self.ln2(x) // normalize h = self.ffn(h) // transform h = self.drop2(h) // regularise x = x + h // residual add return x // shape: (B, T, d_model)

    And the full transformer (stack of N blocks):

    class Transformer: def __init__(self, n_layers, d_model, n_heads, d_ff, vocab_size, max_seq_len): self.embedding = Embedding(vocab_size, d_model) self.pos_enc = PositionalEncoding(max_seq_len, d_model) self.blocks = [TransformerBlock(d_model, n_heads, d_ff) for _ in range(n_layers)] self.final_ln = LayerNorm(d_model) // final norm (pre-norm needs this) self.head = Linear(d_model, vocab_size) def forward(self, tokens, mask=None): x = self.embedding(tokens) + self.pos_enc(tokens) for block in self.blocks: x = block(x, mask) // stack N blocks x = self.final_ln(x) // pre-norm needs a final LN logits = self.head(x) // project to vocabulary return logits // shape: (B, T, vocab_size)
    The final LayerNorm: In pre-norm architectures, the output of the last block is not normalised (because the norm comes before each sub-layer, not after). So we need one extra LayerNorm after all blocks, before the output head. This is often forgotten but is critical for stable training.

    Feed-Forward Network Detail

    class FeedForward: def __init__(self, d_model, d_ff): self.W1 = Linear(d_model, d_ff) // expand: 512 → 2048 self.W2 = Linear(d_ff, d_model) // compress: 2048 → 512 def forward(self, x): x = self.W1(x) // (B, T, d_ff) x = GELU(x) // nonlinear activation x = self.W2(x) // (B, T, d_model) return x

    The FFN is sometimes called a “two-layer MLP” or “position-wise” because it is applied identically and independently to each token position. There is no interaction between positions in the FFN - that’s the job of attention.

    Common Variations

    The standard transformer block described above is a starting point. Modern architectures have introduced numerous variations, each with specific trade-offs.

    Parallel Attention + FFN (PaLM, GPT-J)

    Instead of running attention and FFN sequentially, compute them in parallel and add both residuals:

    x = x + Attention(LayerNorm(x)) + FFN(LayerNorm(x))

    This removes the sequential dependency between attention and FFN within a block, allowing better hardware utilisation (both can be computed simultaneously on different parts of the accelerator). PaLM (2022) showed this works at scale with minimal quality loss.

    Trade-off: Slightly lower quality in some benchmarks, but 15% faster training throughput.

    Grouped Query Attention (GQA)

    Standard multi-head attention has separate K and V projections for each head. GQA groups multiple query heads to share the same K/V heads:

    Standard MHA: h heads, h K/V heads → 4d² params Grouped (GQA): h heads, g K/V heads (g<h) → (2 + 2g/h)d² params Multi-Query: h heads, 1 K/V head → (2 + 2/h)d² params

    LLaMA-2 uses GQA with 8 K/V groups for 32 query heads. This dramatically reduces KV cache size during inference (important for serving) while preserving quality.

    SwiGLU Activation (LLaMA, Mistral)

    Replace the standard FFN with a gated variant:

    Standard: FFN(x) = W₂ · GELU(W₁ · x) SwiGLU: FFN(x) = W₂ · (SiLU(W₁ · x) ⊙ W₃ · x)

    SwiGLU adds a third weight matrix W₃ that acts as a gate (element-wise multiplication ⊙). This provides a learnable gating mechanism that can selectively suppress or amplify features. It adds ~50% more FFN parameters but improves quality enough that d_ff is typically reduced to compensate (e.g., d_ff = 2.67d instead of 4d).

    RMSNorm (LLaMA, Mistral)

    Replace LayerNorm with the simpler Root Mean Square normalisation:

    RMSNorm(x) = x / √(mean(x²) + ε) · γ

    RMSNorm drops the mean-centering step and the bias parameter β. It’s computationally cheaper (one fewer reduction operation) and empirically works just as well. Used in LLaMA, Mistral, and most recent open-source LLMs.

    Rotary Position Embeddings (RoPE)

    Instead of adding positional encodings to the input embeddings, RoPE rotates the Q and K vectors in the attention mechanism by position-dependent angles. This elegantly encodes relative position information directly where it matters (in the attention scores) and allows for length extrapolation.

    Mixture of Experts (MoE)

    Replace the single FFN with a set of FFN “experts,” and a router network that selects the top-k experts for each token. Mixtral-8x7B has 8 experts per block and routes each token to 2 of them. This allows massively scaling parameters without proportionally increasing compute (because only 2 of 8 experts run per token).

    Sliding Window Attention (Mistral)

    Restrict the attention window to the W most recent tokens instead of the full sequence. This reduces the attention complexity from O(T²) to O(T·W), making it feasible to process much longer sequences. Combined with stacking, a model with window size W and L layers has an effective receptive field of L·W tokens.

    Summary

    The transformer block is deceptively simple: normalise → attend → add → normalise → transform → add. Yet this pattern, repeated dozens or hundreds of times, is the computational engine behind every modern language model.

    Key takeaways:

    • Residual connections create a gradient highway that makes deep stacking possible.
    • Pre-norm is the modern standard: it puts LayerNorm before each sub-layer, leaving the residual path clean and gradient-friendly.
    • The block preserves the shape (B, T, d_model), enabling arbitrary stacking.
    • Two-thirds of the parameters are in the FFN, not in attention.
    • Encoder blocks use bidirectional attention; decoder blocks add causal masking (and optionally cross-attention).
    • Modern variations (GQA, SwiGLU, RMSNorm, RoPE, MoE) refine each component for better quality, efficiency, or scalability.
    • Deeper models naturally specialise: early layers learn syntax, later layers learn semantics and reasoning.

    The transformer block is the atom of modern AI. Understanding it deeply - not just as a diagram but as a precise sequence of linear algebra operations with specific shapes, parameters, and gradient properties - is the foundation for understanding everything built on top of it.
