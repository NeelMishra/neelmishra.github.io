    Multi-Head Attention

    March 2026 · ML Series · Transformers

    Why a Single Head is Not Enough

    In the previous article, we saw how self-attention computes a single set of attention weights: each token produces one query, one key, and one value. The attention weights capture one type of relationship between tokens.

    But language encodes many types of relationships simultaneously. Consider the sentence "The cat sat on the mat because it was tired." When processing the word "it," we need to resolve at least three different relationships:

    • Coreference: "it" refers to "cat" (semantic relationship)
    • Syntax: "it" is the subject of "was tired" (grammatical relationship)
    • Proximity: "it" is close to "mat" and "because" (positional relationship)

    A single attention head computes a single weighted average. It must compress all these different relationship types into one set of weights. In practice, this forces the head to compromise: it might partially attend to "cat" for coreference and partially attend to "was" for syntax, producing a muddled representation that does neither well.

    Multi-head attention solves this by running multiple attention computations in parallel, each with its own learned projection matrices. Each head can specialize in a different type of relationship. The results are concatenated and projected back to the original dimension.

    How Multi-Head Attention Works

    The procedure has four steps:

    1. Split: Project the input into h separate sets of Q, K, V using h different sets of weight matrices.
    2. Attend: Run scaled dot-product attention independently on each head.
    3. Concatenate: Stack all head outputs side by side.
    4. Project: Multiply the concatenated result by a final weight matrix W_O to produce the output.
    MultiHead(Q, K, V) = Concat(head_1, head_2, ..., head_h) * W_O where head_i = Attention(X * W_Q_i, X * W_K_i, X * W_V_i)

    Each head operates on a smaller dimension. If the model dimension is d_model and there are h heads, each head works with d_k = d_model / h dimensions. This keeps the total computation cost the same as a single-head attention with the full dimension.

    Concrete Example: 512-Dim, 8 Heads

    The original Transformer uses d_model = 512 with h = 8 heads. Here is exactly what happens at each step.

    OperationInput ShapeWeight ShapeOutput Shape
    Input embedding Xn x 512n x 512
    Project Q (per head)n x 512512 x 64n x 64
    Project K (per head)n x 512512 x 64n x 64
    Project V (per head)n x 512512 x 64n x 64
    Q * K^T (per head)(n x 64) * (64 x n)n x n
    Scale + softmaxn x nn x n
    Weights * V (per head)(n x n) * (n x 64)n x 64
    Concat all 8 heads8 x (n x 64)n x 512
    Output projection W_On x 512512 x 512n x 512

    Notice that the output has the same shape as the input: n x 512. This is essential because multi-head attention layers are stacked repeatedly in the Transformer, and each layer must accept and produce the same shape.

    Key insight: The dimension split is the reason multi-head attention has the same computational cost as single-head attention. A single head with d_k = 512 computes Q * K^T as (n x 512) times (512 x n): cost O(n^2 * 512). Eight heads each computing (n x 64) times (64 x n): cost 8 * O(n^2 * 64) = O(n^2 * 512). Identical total cost, but the model can learn 8 different attention patterns instead of one.

    What Different Heads Learn

    Empirical analysis of trained Transformer models reveals that different heads specialize in distinct linguistic functions. This is not explicitly programmed. It emerges from training.

    Syntactic heads

    Some heads learn to attend from a word to its syntactic governor. For example, a head might consistently attend from a verb to its subject, or from an adjective to the noun it modifies. These heads learn the dependency structure of the language without explicit syntactic supervision.

    Semantic heads

    Other heads learn coreference and semantic similarity. When processing a pronoun like "it," these heads attend strongly to the antecedent noun. When processing a word with multiple senses, these heads attend to the disambiguating context words.

    Positional heads

    Some heads develop simple positional patterns: attend to the previous token, attend to the next token, or attend to the first token in the sequence (often a special [CLS] or [BOS] token). These heads provide local context and anchoring information.

    Rare pattern heads

    In large models, some heads learn to detect rare but important patterns: negation, quotation boundaries, list structures, or domain-specific terminology. These heads activate infrequently but contribute critical information when they do.

    Implementation in Pseudo-Code

    The implementation performs all heads in parallel using a single batched matrix multiplication, then reshapes the result. This is more efficient than running h separate attention computations.

    // Multi-Head Attention forward pass
function multi_head_attention(X, W_Q, W_K, W_V, W_O, h):
    // X: input (batch x n x d_model)
    // W_Q, W_K, W_V: (d_model x d_model) each
    // W_O: output projection (d_model x d_model)
    // h: number of heads

    batch, n, d_model = X.shape
    d_k = d_model / h     // dimension per head

    // Project all heads at once
    Q = X @ W_Q           // (batch x n x d_model)
    K = X @ W_K           // (batch x n x d_model)
    V = X @ W_V           // (batch x n x d_model)

    // Reshape to separate heads
    // (batch x n x d_model) -> (batch x h x n x d_k)
    Q = reshape(Q, [batch, n, h, d_k]).transpose(1, 2)
    K = reshape(K, [batch, n, h, d_k]).transpose(1, 2)
    V = reshape(V, [batch, n, h, d_k]).transpose(1, 2)

    // Scaled dot-product attention (all heads in parallel)
    scores = Q @ K.transpose(-2, -1)   // (batch x h x n x n)
    scores = scores / sqrt(d_k)
    weights = softmax(scores, dim=-1)  // (batch x h x n x n)
    attn_out = weights @ V              // (batch x h x n x d_k)

    // Concatenate heads
    // (batch x h x n x d_k) -> (batch x n x d_model)
    attn_out = attn_out.transpose(1, 2).reshape(batch, n, d_model)

    // Final linear projection
    output = attn_out @ W_O  // (batch x n x d_model)

    return output

    The key implementation trick is the reshape and transpose. Instead of creating h separate weight matrices of size (d_model x d_k), we use a single (d_model x d_model) matrix and reshape the output into h heads. This allows the matrix multiplication to be executed as a single operation on the GPU, which is substantially faster than h separate smaller multiplications.

    Why the Output Projection W_O?

    After concatenation, the output is (n x d_model). Why multiply by another (d_model x d_model) matrix?

    The concatenated output is a simple stacking of h independent results. Each slot of 64 dimensions came from a different head and operates in a different subspace. The output projection W_O learns to combine information from all heads into a unified representation. Without it, the model cannot mix information across heads.

    Think of it this way: each head produces a partial answer from its own perspective. The output projection is the "committee vote" that integrates all perspectives into a single consensus representation.

    Complexity Analysis

    The total parameter count and computation cost of multi-head attention:

    Parameters

    W_Q: d_model x d_model = 512 x 512 = 262,144 W_K: d_model x d_model = 512 x 512 = 262,144 W_V: d_model x d_model = 512 x 512 = 262,144 W_O: d_model x d_model = 512 x 512 = 262,144 Total: 4 * d_model^2 = 4 * 262,144 = 1,048,576 parameters

    Computation

    Per head: O(n^2 * d_k) for attention, O(n * d_model * d_k) for projections.

    With h heads, each using d_k = d_model / h:

    Attention: h * O(n^2 * d_k) = h * O(n^2 * d_model/h) = O(n^2 * d_model) Projections: O(n * d_model^2) for Q, K, V combined Output proj: O(n * d_model^2) Total: O(n^2 * d_model + n * d_model^2)

    This is identical to the cost of single-head attention with full dimension. The dimension split means we do not pay extra computation for multiple heads. We get richer representations for free (in terms of FLOPs), at the cost of a modest increase in parameters from the output projection.

    Key insight: Multi-head attention is not more expensive than single-head attention. The dimension split ensures the total computation stays the same. What changes is the capacity: h independent attention patterns instead of one. This is why multi-head attention works so well in practice. It is strictly more expressive at the same cost.

    Choosing the Number of Heads

    The number of heads h must divide d_model evenly. Common choices:

    • Transformer-base: d_model = 512, h = 8, d_k = 64
    • Transformer-large: d_model = 1024, h = 16, d_k = 64
    • GPT-2 small: d_model = 768, h = 12, d_k = 64
    • GPT-3: d_model = 12288, h = 96, d_k = 128
    • LLaMA 70B: d_model = 8192, h = 64, d_k = 128

    The per-head dimension d_k typically stays between 64 and 128 across model sizes. As models grow larger, they add more heads rather than increasing the per-head dimension. This suggests that 64 to 128 dimensions per head is sufficient to capture individual attention patterns, and the benefit of scale comes from having more patterns, not wider patterns.

    Summary

    Multi-head attention runs h independent attention computations in parallel, each on a d_model/h dimensional subspace. The results are concatenated and linearly projected back to d_model dimensions. This gives the model the capacity to jointly attend to information from different representation subspaces at different positions. The total computational cost matches single-head attention, but the expressiveness is strictly greater.
