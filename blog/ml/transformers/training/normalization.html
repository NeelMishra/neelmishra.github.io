    Layer Normalization in Transformers

    March 2026 · ML Series · Architecture

    Why Normalization Is Needed

    Deep networks suffer from a problem called internal covariate shift: the distribution of inputs to each layer changes as the parameters of the preceding layers are updated during training. Each layer must continuously adapt to a moving target, which slows convergence and makes training sensitive to initialization and learning rate choices.

    Without normalization, the activations in a deep transformer tend to grow or shrink across layers. After 12 or more layers, the magnitude of the hidden states can differ by orders of magnitude between the first and last layers. This has several consequences:

    • Gradients become very large (exploding) or very small (vanishing) during backpropagation, making optimization unstable.
    • The effective learning rate differs between layers. Parameters in early layers receive much larger or smaller gradient updates than parameters in later layers.
    • The residual connections accumulate values with inconsistent scales, making the addition operation in the residual path less meaningful.

    Normalization layers re-center and re-scale activations at each layer, maintaining stable distributions throughout the network. This allows the use of larger learning rates, reduces sensitivity to initialization, and generally produces faster, more reliable convergence.

    Batch Normalization vs Layer Normalization

    Batch Normalization (Ioffe and Szegedy, 2015) normalizes across the batch dimension: for each feature, it computes the mean and variance over all examples in the mini-batch. Layer Normalization (Ba et al., 2016) normalizes across the feature dimension: for each example, it computes the mean and variance over all features. This difference is fundamental.

    Why Batch Norm fails for transformers

    • Variable sequence lengths. In a batch of sequences with different lengths, the statistics at position k are computed over a varying number of examples (only sequences long enough to have position k contribute). Padding masks complicate the computation further.
    • Small effective batch sizes. At rare token positions, the batch may contain very few examples, making the batch statistics unreliable.
    • Inference dependency. Batch Norm requires maintaining running mean and variance statistics from training, which must be applied during inference. For autoregressive models that generate one token at a time, the "batch" at inference is size 1, making batch statistics meaningless.

    Layer Normalization avoids all of these issues. It computes statistics independently for each example and each position. There is no dependency on other examples in the batch, no issue with variable lengths, and no running statistics to maintain.

    The LayerNorm Formula

    Given an input vector x of dimension d (the hidden state at one position in one sequence), LayerNorm computes:

    LayerNorm(x) = γ ⊙ (x − μ) / √(σ2 + ε) + β

    where:

    • μ = (1/d) ∑i=1d xi is the mean across the feature dimension
    • σ2 = (1/d) ∑i=1d (xi − μ)2 is the variance across the feature dimension
    • ε is a small constant (typically 10−5 or 10−6) for numerical stability
    • γ and β are learned parameter vectors of dimension d (scale and shift)
    • ⊙ denotes element-wise multiplication

    The normalization step (subtract mean, divide by standard deviation) forces the output to have zero mean and unit variance. The learned parameters γ and β then allow the network to undo this normalization if it is harmful for a particular layer. In practice, the network learns γ and β values that are close to 1 and 0 respectively for most dimensions, with task-specific deviations where needed.

    // LayerNorm implementation
def layer_norm(x, gamma, beta, eps=1e-5):
    // x: (batch, seq_len, d_model)
    mu    = x.mean(dim=-1, keepdim=True)
    var   = x.var(dim=-1, keepdim=True)
    x_hat = (x - mu) / sqrt(var + eps)
    return gamma * x_hat + beta
    Key insight: LayerNorm normalizes each token's hidden state independently. Token at position 5 and token at position 50 are normalized using different means and variances computed from their own feature vectors. There is no interaction between positions or between examples in the batch.

    Pre-Norm vs Post-Norm

    The original transformer ("Attention Is All You Need") places LayerNorm after the residual addition. This is called Post-Norm:

    // Post-Norm (original transformer)
h = x + MultiHeadAttention(x)
h = LayerNorm(h)               // normalize after residual
out = h + FeedForward(h)
out = LayerNorm(out)             // normalize after residual

    Most modern transformers (GPT-2, GPT-3, LLaMA, PaLM) use Pre-Norm, where LayerNorm is applied before the sublayer:

    // Pre-Norm (modern transformers)
h = x + MultiHeadAttention(LayerNorm(x))   // normalize before sublayer
out = h + FeedForward(LayerNorm(h))        // normalize before sublayer

    Why Pre-Norm is preferred

    The key difference is the gradient flow through the residual path. In Post-Norm, the residual connection feeds into LayerNorm, so the gradient must flow through the normalization operation. In Pre-Norm, the residual connection is a clean identity path from the output back to the input. The gradient flows directly through the addition without any nonlinear transformation.

    This has practical consequences:

    • Training stability. Pre-Norm models are easier to train. They tolerate larger learning rates and require less warmup. Post-Norm models frequently diverge without careful learning rate tuning.
    • Deep models. Pre-Norm enables training very deep transformers (100+ layers) without gradient pathology. Post-Norm struggles beyond 12 to 24 layers without additional techniques.
    • Convergence speed. Pre-Norm typically converges faster in the early phases of training.

    However, Post-Norm has been shown to achieve slightly better final performance in some cases when training is stable. The hypothesis is that normalizing after the residual addition provides a stronger regularization effect. Some architectures use a hybrid approach or add extra normalization to get the benefits of both.

    RMSNorm

    Root Mean Square Layer Normalization (Zhang and Sennrich, 2019) simplifies LayerNorm by removing the mean-centering step. Instead of subtracting the mean and dividing by the standard deviation, RMSNorm divides by the root mean square of the input:

    RMSNorm(x) = γ ⊙ x / RMS(x), where RMS(x) = √((1/d) ∑i=1d xi2)

    This removes the mean computation and the beta (shift) parameter. The motivation is that the re-centering in LayerNorm is not necessary for the regularization effect; the re-scaling alone is enough to stabilize training. The advantages of RMSNorm:

    • Computational savings. Removing the mean computation and the shift parameter reduces the FLOPs per normalization operation. For models with many layers and large hidden dimensions, this adds up.
    • Fewer parameters. Only γ (no β), so the normalization layer has half the trainable parameters.
    • Equivalent performance. Empirically, RMSNorm matches LayerNorm performance on language modeling tasks while being faster.

    RMSNorm is used in LLaMA, LLaMA 2, Mistral, and several other recent large language models. It has become the default choice for new architectures that prioritize training efficiency.

    // RMSNorm implementation
def rms_norm(x, gamma, eps=1e-6):
    // x: (batch, seq_len, d_model)
    rms = sqrt(x.pow(2).mean(dim=-1, keepdim=True) + eps)
    return gamma * (x / rms)
    Key insight: RMSNorm demonstrates a broader principle in deep learning: simpler operations that preserve the essential property (in this case, scale normalization) often work as well as more complex ones. The mean-centering in LayerNorm adds flexibility, but the scale normalization is what actually prevents activation explosion and enables stable training.

    Impact on Training Dynamics

    Normalization has a dramatic effect on training stability. Without normalization, training loss often exhibits erratic behavior: sudden spikes, plateaus, or divergence. With normalization, the loss curve is smooth and monotonically decreasing (with minor fluctuations from stochastic sampling).

    The difference is especially pronounced for deeper models. A 48-layer transformer without normalization is practically untrainable with standard learning rates. The same architecture with Pre-Norm LayerNorm trains as reliably as a 6-layer model.

    Normalization also affects the geometry of the loss landscape. Normalized networks have smoother loss surfaces with fewer sharp minima. The Hessian (matrix of second derivatives) has more uniform eigenvalues, which means gradient descent performs well in all parameter directions rather than being dominated by a few high-curvature directions. This allows the optimizer to use a single learning rate effectively across all parameters.

    Summary

    • Layer Normalization normalizes across features (not the batch), making it suitable for variable-length sequences and autoregressive generation.
    • Pre-Norm placement (normalize before the sublayer) provides cleaner gradient flow and is used by most modern transformers.
    • RMSNorm drops the mean-centering step while maintaining equivalent performance, offering computational savings.
    • Normalization is not optional for transformers. It enables stable training, larger learning rates, and deeper architectures. The choice between LayerNorm and RMSNorm is secondary to the choice of placement (Pre-Norm vs Post-Norm).
