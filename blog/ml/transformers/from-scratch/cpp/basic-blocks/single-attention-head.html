Single Attention Head in C++

ML Series Transformers > From Scratch > C++ > Basic Blocks

What We Are Building

A single attention head takes a sequence of token embeddings and produces a new sequence where each token is a weighted combination of all other tokens. The weights are computed by comparing queries against keys, then using the result to mix values.

The math for scaled dot-product attention is:

Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V Where: Q = X * W_Q (queries) K = X * W_K (keys) V = X * W_V (values) d_k = dimension of keys

We will implement this in pure C++ with no external libraries. By the end of this page, you will have a working AttentionHead class that takes an input matrix and returns the attention output.

Prerequisites: Basic C++ (vectors, classes) and familiarity with matrix multiplication. If you have read the attention theory pages in this series, you already know the math.

A Minimal Matrix Class

We need a simple 2D matrix that supports multiplication and element access. No need for a full linear algebra library. Here is a minimal implementation:

struct Matrix { int rows, cols; vector<vector<float>> data; Matrix(int r, int c) : rows(r), cols(c), data(r, vector<float>(c, 0.0f)) {} // Matrix multiplication: (m x n) * (n x p) = (m x p) Matrix operator*(const Matrix& other) const { Matrix result(rows, other.cols); for (int i = 0; i < rows; i++) for (int j = 0; j < other.cols; j++) for (int k = 0; k < cols; k++) result.data[i][j] += data[i][k] * other.data[k][j]; return result; } // Transpose: (m x n) -> (n x m) Matrix transpose() const { Matrix result(cols, rows); for (int i = 0; i < rows; i++) for (int j = 0; j < cols; j++) result.data[j][i] = data[i][j]; return result; } // Scale every element by a constant void scale(float factor) { for (auto& row : data) for (auto& val : row) val *= factor; } };
Performance note: This uses vector<vector<float>> for clarity. In production code, you would use a flat 1D array (row-major layout) for cache efficiency. We keep it simple here since the goal is understanding, not speed.

Implementing Softmax

Softmax converts raw scores into probabilities. Each row of the attention scores matrix gets softmaxed independently (each token attends to all tokens, and those attention weights must sum to 1).

softmax(x_i) = exp(x_i) / sum(exp(x_j) for all j)

The standard numerical trick: subtract the row maximum before exponentiating to avoid overflow.

void softmax_rows(Matrix& m) { for (int i = 0; i < m.rows; i++) { // Find max for numerical stability float max_val = *max_element(m.data[i].begin(), m.data[i].end()); // Exponentiate and sum float sum = 0.0f; for (int j = 0; j < m.cols; j++) { m.data[i][j] = exp(m.data[i][j] - max_val); sum += m.data[i][j]; } // Normalize so row sums to 1 for (int j = 0; j < m.cols; j++) m.data[i][j] /= sum; } }

Without the max subtraction, exp(large_number) overflows to infinity. If x = [100, 200, 300], then exp(300) is ~10130, which exceeds float range. Subtracting 300 first gives exp([0, -100, -200]) = [1, tiny, tinier], which is safe.

The Attention Head

Now we combine everything into an AttentionHead class. It holds three weight matrices (W_Q, W_K, W_V) and applies the full attention computation.

struct AttentionHead { Matrix W_Q, W_K, W_V; int d_k; AttentionHead(int d_model, int head_dim) : W_Q(d_model, head_dim), W_K(d_model, head_dim), W_V(d_model, head_dim), d_k(head_dim) { // Initialize with small random values random_init(W_Q); random_init(W_K); random_init(W_V); } Matrix forward(const Matrix& X) { // Step 1: Project input into Q, K, V Matrix Q = X * W_Q; // (n x d_model) * (d_model x d_k) = (n x d_k) Matrix K = X * W_K; // same shape Matrix V = X * W_V; // same shape // Step 2: Compute attention scores Matrix scores = Q * K.transpose(); // (n x d_k) * (d_k x n) = (n x n) // Step 3: Scale by sqrt(d_k) scores.scale(1.0f / sqrt((float)d_k)); // Step 4: Softmax each row softmax_rows(scores); // each row sums to 1 // Step 5: Weighted sum of values return scores * V; // (n x n) * (n x d_k) = (n x d_k) } };
Shape tracking is everything. If the dimensions do not align at each step, the code will crash or produce garbage. The comments after each line show the matrix dimensions. This is the single most important debugging skill when implementing transformers.

Walkthrough with Numbers

Let us trace a tiny example. Suppose we have 3 tokens with d_model = 4 and head_dim (d_k) = 2.

X (3 x 4): three tokens, each a 4-dimensional embedding.

Multiply X by W_Q, W_K, W_V (each 4 x 2) to get Q, K, V (each 3 x 2). Now each token has a 2D query, key, and value.

Q * K^T gives a 3 x 3 matrix. Entry (i, j) measures how much token i should attend to token j. Higher values mean more attention.

Divide every score by sqrt(2) = 1.414. This prevents the dot products from being too large before softmax. Without scaling, softmax would produce near-one-hot distributions, killing gradient flow.

Each row becomes a probability distribution. Row i tells us: "how much should token i attend to each of the 3 tokens?" The values in each row sum to 1.

Multiply the attention weights (3 x 3) by V (3 x 2). Each output row is a weighted average of all value vectors, weighted by how much attention each token pays to the others.

Full Working Code

Here is the complete, compilable C++ code. Copy it, compile with g++ -std=c++17 -O2 attention.cpp -o attention, and run it.

#include <iostream> #include <vector> #include <cmath> #include <algorithm> #include <random> using namespace std; struct Matrix { int rows, cols; vector<vector<float>> data; Matrix(int r, int c) : rows(r), cols(c), data(r, vector<float>(c, 0.0f)) {} Matrix operator*(const Matrix& o) const { Matrix res(rows, o.cols); for (int i = 0; i < rows; i++) for (int j = 0; j < o.cols; j++) for (int k = 0; k < cols; k++) res.data[i][j] += data[i][k] * o.data[k][j]; return res; } Matrix transpose() const { Matrix res(cols, rows); for (int i = 0; i < rows; i++) for (int j = 0; j < cols; j++) res.data[j][i] = data[i][j]; return res; } void scale(float f) { for (auto& row : data) for (auto& v : row) v *= f; } void print(const string& name) const { cout << name << " (" << rows << "x" << cols << "):\n"; for (auto& row : data) { for (float v : row) printf(" %6.3f", v); cout << "\n"; } } }; void random_init(Matrix& m) { mt19937 rng(42); normal_distribution<float> dist(0.0f, 0.1f); for (auto& row : m.data) for (auto& v : row) v = dist(rng); } void softmax_rows(Matrix& m) { for (int i = 0; i < m.rows; i++) { float mx = *max_element(m.data[i].begin(), m.data[i].end()); float sum = 0; for (int j = 0; j < m.cols; j++) { m.data[i][j] = exp(m.data[i][j] - mx); sum += m.data[i][j]; } for (int j = 0; j < m.cols; j++) m.data[i][j] /= sum; } } Matrix attention(const Matrix& X, Matrix& W_Q, Matrix& W_K, Matrix& W_V, int d_k) { Matrix Q = X * W_Q; Matrix K = X * W_K; Matrix V = X * W_V; Matrix scores = Q * K.transpose(); scores.scale(1.0f / sqrt((float)d_k)); softmax_rows(scores); scores.print("Attention Weights"); return scores * V; } int main() { int seq_len = 3, d_model = 4, d_k = 2; // Create input: 3 tokens, d_model=4 Matrix X(seq_len, d_model); X.data = {{1.0,0.5,0.2,0.8},{0.3,0.9,0.1,0.6},{0.7,0.4,0.8,0.2}}; X.print("Input X"); // Weight matrices Matrix W_Q(d_model, d_k), W_K(d_model, d_k), W_V(d_model, d_k); random_init(W_Q); random_init(W_K); random_init(W_V); Matrix output = attention(X, W_Q, W_K, W_V, d_k); output.print("Attention Output"); }
