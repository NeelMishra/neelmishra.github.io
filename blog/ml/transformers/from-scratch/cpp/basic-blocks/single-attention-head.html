Single Attention Head in C++ (LibTorch)

ML Series Transformers > From Scratch > C++ > Basic Blocks

What We Are Building

A single attention head takes a sequence of token embeddings and produces a new sequence where each token is a weighted combination of all other tokens. The weights come from comparing queries against keys, then using the result to mix values.

The math for scaled dot-product attention:

Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V Where: Q = X * W_Q (queries) K = X * W_K (keys) V = X * W_V (values) d_k = dimension of keys

We implement this using LibTorch (the C++ frontend for PyTorch). LibTorch handles tensor operations, autograd, and module registration so we can focus on the transformer logic rather than writing matrix multiplication from scratch.

The Class Structure

Our SingleAttentionHead inherits from torch::nn::Module. It holds three linear layers (Q, K, V projections) and knows n_emb (embedding dimension) and head_size (output dimension per head).

class SingleAttentionHead : public torch::nn::Module { int n_emb = 32; int head_size = 16; torch::nn::Linear Q{nullptr}, K{nullptr}, V{nullptr}; public: SingleAttentionHead(int n_emb, int head_size); torch::Tensor forward(torch::Tensor x); };

LibTorch linear layers must be initialized in the constructor using register_module. We declare them as nullptr first and assign them during construction. This is standard LibTorch practice: you cannot construct torch::nn::Linear without specifying its dimensions, so we defer it.

Constructor: Registering Layers

Each linear layer projects from n_emb to head_size with no bias (following the original "Attention is All You Need" paper).

SingleAttentionHead(int n_emb, int head_size) : n_emb(n_emb), head_size(head_size) { this->Q = register_module( "Q", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false) ) ); this->K = register_module( "K", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false) ) ); this->V = register_module( "V", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false) ) ); }
register_module does two things: (1) gives the layer a name for serialization and debugging, and (2) registers its parameters so the optimizer can find them during training. If you skip this, the weights will not be updated during backpropagation.

The Q, K, V projections in the original transformer paper do not use bias terms. The linear layers compute Q = X * W_Q and nothing more. Setting .bias(false) matches this. Some implementations do include bias, but the standard practice is to omit it.

The Forward Pass

The forward method takes input x of shape (batch, seq_len, n_emb) and returns the attention output of shape (batch, seq_len, head_size).

torch::Tensor forward(torch::Tensor x) { // Step 1: Project into Q, K, V torch::Tensor Q_vec = this->Q(x); // (B, T, head_size) torch::Tensor K_vec = this->K(x); // (B, T, head_size) torch::Tensor V_vec = this->V(x); // (B, T, head_size) // Step 2: Compute scaled attention scores torch::Tensor raw_attention = torch::matmul(Q_vec, K_vec.transpose(-2, -1)) // (B, T, T) / sqrt((float)(head_size)); // Step 3: Apply causal mask torch::Tensor mask = torch::tril( torch::ones({x.size(1), x.size(1)}) ); raw_attention.masked_fill_(mask == 0, -1e9f); // Step 4: Softmax over last dimension torch::Tensor weights = torch::softmax(raw_attention, -1); // Step 5: Weighted sum of values torch::Tensor out = torch::matmul(weights, V_vec); // (B, T, head_size) return out; }

Let us break down each step:

this->Q(x) calls the linear layer's forward method, computing x * W_Q. The input shape is (B, T, n_emb) and the output is (B, T, head_size). Same for K and V. Each token now has a query, key, and value vector.

torch::matmul(Q_vec, K_vec.transpose(-2, -1)) computes the dot product between every pair of query and key vectors. .transpose(-2, -1) swaps the last two dimensions (T, head_size) to (head_size, T), so the matmul gives (B, T, T). Division by sqrt(head_size) prevents values from growing too large before softmax.

The causal mask zeros out future positions. Softmax converts scores to probabilities. Finally, we multiply weights by V to get the output: each token becomes a weighted average of all (allowed) value vectors.

Causal Masking

For autoregressive models (like GPT), token i should only attend to tokens 0 through i, not future tokens. We enforce this with a lower-triangular mask.

// Create lower-triangular mask torch::Tensor mask = torch::tril(torch::ones({T, T})); // mask looks like: // [[1, 0, 0], // [1, 1, 0], // [1, 1, 1]] // Fill positions where mask==0 with -infinity raw_attention.masked_fill_(mask == 0, -1e9f);

After masking, those positions hold a very large negative number. When softmax exponentiates them, exp(-1e9) is essentially 0. So the resulting attention weights for future positions are zero: the model cannot "look ahead." We use -1e9 instead of negative infinity to avoid NaN issues.

Encoder vs Decoder: This causal mask is only for decoder-style (autoregressive) attention. For an encoder (like BERT), skip the masking entirely and allow every token to attend to every other token.

Full Working Code

Here is the complete class. Drop this into a LibTorch project and it is ready to use.

#include <torch/torch.h> class SingleAttentionHead : public torch::nn::Module { int n_emb = 32; int head_size = 16; torch::nn::Linear Q{nullptr}, K{nullptr}, V{nullptr}; public: SingleAttentionHead(int n_emb, int head_size) : n_emb(n_emb), head_size(head_size) { this->Q = register_module("Q", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false))); this->K = register_module("K", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false))); this->V = register_module("V", torch::nn::Linear( torch::nn::LinearOptions(n_emb, head_size).bias(false))); } torch::Tensor forward(torch::Tensor x) { torch::Tensor Q_vec = this->Q(x); torch::Tensor K_vec = this->K(x); torch::Tensor V_vec = this->V(x); torch::Tensor raw_attention = torch::matmul(Q_vec, K_vec.transpose(-2, -1)) / sqrt((float)head_size); torch::Tensor mask = torch::tril( torch::ones({x.size(1), x.size(1)})); raw_attention.masked_fill_(mask == 0, -1e9f); torch::Tensor weights = torch::softmax(raw_attention, -1); torch::Tensor out = torch::matmul(weights, V_vec); return out; } }; // Usage example int main() { auto head = std::make_shared<SingleAttentionHead>(32, 16); // Random input: batch=1, seq_len=8, n_emb=32 torch::Tensor x = torch::randn({1, 8, 32}); torch::Tensor out = head->forward(x); std::cout << "Input shape: " << x.sizes() << std::endl; std::cout << "Output shape: " << out.sizes() << std::endl; // Output: [1, 8, 16] }
