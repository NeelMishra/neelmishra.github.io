Layer Normalization

April 2025 · From Scratch in C++ · LibTorch

LayerNorm stabilizes the hidden representations inside a Transformer by normalizing each token's feature vector to zero mean and unit variance, then applying a learned scale and shift. For the full theory — why normalization matters, BatchNorm vs LayerNorm, RMSNorm, and Pre-Norm vs Post-Norm — see the dedicated Layer Normalization theory blog. This page covers the C++ LibTorch implementation.

1 · What LayerNorm Does

Given a feature vector x = [x0, x1, ..., xd-1] for a single token, LayerNorm computes:

x̂i = γi · (xi − μ) / √(σ² + ε) + βi where: μ = mean of x across all d features σ² = variance of x across all d features ε = small constant (default 1e-5) to avoid division by zero γ, β = learned per-feature scale and shift (both shape [d])

This is applied independently to every token in every batch element. The shape of the tensor does not change.

2 · The Code

Unlike dropout, LayerNorm has learnable parameters (γ and β), so we use the torch::nn::LayerNorm module:

// Declare (nullptr until constructed) torch::nn::LayerNorm ln{nullptr}; // Construct and register in the constructor ln = register_module( "ln", torch::nn::LayerNorm( torch::nn::LayerNormOptions({n_embd}) ) );

The argument to LayerNormOptions is the normalized shape: a vector of the dimensions that LayerNorm should normalize over. For Transformers, this is always {n_embd} — the last dimension.

Why {n_embd} with braces? LayerNormOptions takes a std::vector<int64_t>, not a single int. The braces create a one-element vector. You could also normalize over multiple trailing dimensions (e.g. {H, W, C} for images), but for Transformers it is always just the embedding dimension.

3 · Constructor Options

OptionDefaultMeaning
normalized_shape (required) Dimensions to normalize over. For us: {n_embd}
.eps() 1e-5 Added to variance before the square root to prevent division by zero
.elementwise_affine() true Whether to learn γ and β. Almost always left true.

Example with custom epsilon:

torch::nn::LayerNorm( torch::nn::LayerNormOptions({n_embd}).eps(1e-6) )

4 · The Forward Pass

Calling the module is a single line:

auto normed = ln(x); // x: [B, T, n_embd] → normed: [B, T, n_embd]

Internally, for each of the B × T tokens:

StepOperationResult
1 Compute mean across n_embd features μ (scalar per token)
2 Compute variance across n_embd features σ² (scalar per token)
3 Normalize: (xi − μ) / √(σ² + ε) Zero mean, unit variance vector
4 Scale and shift: γi · normi + βi Final output

The batch and sequence dimensions are untouched. LayerNorm only operates on the last dimension (the features). This is why it works identically regardless of sequence length or batch size.

5 · Learned Parameters

Each LayerNorm module has exactly 2 × n_embd learnable parameters:

ParameterShapeInitialized toRole
γ (weight) [n_embd] all ones Per-feature scale
β (bias) [n_embd] all zeros Per-feature shift

At initialization, γ = 1 and β = 0, so LayerNorm starts as pure normalization (zero mean, unit variance). During training, the model learns to scale up features that matter and shift the distribution where needed.

Parameter count is tiny. For n_embd = 512, one LayerNorm has 1,024 parameters. The FeedForward in the same block has ~525K. LayerNorm accounts for less than 0.1% of a block's parameters.

6 · Where It Lives in a Transformer

In a pre-norm Transformer block (the style used in GPT-2 and most modern models), LayerNorm is applied before each sub-layer:

// Inside TransformerBlock::forward() // ln1 normalizes before attention auto attn_out = mha.forward(ln1(x)); x = x + attn_out; // ln2 normalizes before feed-forward auto ffn_out = ffn.forward(ln2(x)); x = x + ffn_out;

ln1 and ln2 are separate modules with their own learned γ and β. They are not shared — the attention path and FFN path benefit from independent normalization.

Some architectures also apply a final LayerNorm after the last Transformer block, before the output head:

// After all N TransformerBlocks x = ln_final(x); logits = lm_head(x);
