    BERT: Bidirectional Encoder Representations

    ML Series · March 2026

    The Breakthrough

    In October 2018, Google published BERT (Bidirectional Encoder Representations from Transformers). The paper demonstrated that pretraining a deep bidirectional transformer on unlabeled text, then fine-tuning on task-specific data, could set new state-of-the-art results on eleven NLP benchmarks simultaneously. BERT changed how the field approached NLP: rather than designing task-specific architectures, you pretrain a general-purpose encoder and adapt it.

    The key insight was bidirectionality. Previous approaches like GPT used left-to-right language modeling, which meant each token could only see the tokens before it. ELMo used separate left-to-right and right-to-left models and concatenated their outputs. BERT trained a single model where every token attends to every other token in both directions. This produced richer contextual representations.

    Encoder-Only Architecture

    BERT uses only the encoder portion of the original transformer. There is no decoder, no causal masking, and no autoregressive generation. The model takes a sequence of tokens as input and produces a contextualized representation for each token as output.

    The architecture consists of:

    • Token embeddings: each input token is mapped to a dense vector
    • Segment embeddings: indicate whether a token belongs to sentence A or sentence B (for sentence-pair tasks)
    • Position embeddings: learned absolute positional encodings
    • Stacked encoder blocks: each block contains multi-head self-attention followed by a feed-forward network, with layer normalization and residual connections

    BERT-Base has 12 encoder blocks, 768 hidden dimensions, 12 attention heads, and 110 million parameters. BERT-Large scales to 24 blocks, 1024 hidden dimensions, 16 attention heads, and 340 million parameters.

    Masked Language Modeling (MLM)

    BERT's primary pretraining objective is Masked Language Modeling. During training, 15% of the input tokens are selected for masking. Of these selected tokens:

    • 80% are replaced with the special [MASK] token
    • 10% are replaced with a random token from the vocabulary
    • 10% are left unchanged

    The model must predict the original token for each masked position. Because attention is bidirectional, the model can use context from both sides of the masked token to make its prediction.

    Why the 80/10/10 split? If masked positions always used [MASK], the model would learn that it only needs to predict tokens at [MASK] positions during fine-tuning, when [MASK] never appears. The random replacement and unchanged cases force the model to maintain good representations for all tokens, not just masked ones.

    Next Sentence Prediction (NSP)

    BERT's second pretraining objective is Next Sentence Prediction. The model receives two sentences separated by a [SEP] token and must predict whether the second sentence actually follows the first in the original corpus. 50% of the time the pair is genuine, and 50% of the time the second sentence is randomly sampled from a different document.

    The NSP prediction uses the [CLS] token representation, which is passed through a binary classifier. The idea was to help BERT learn relationships between sentences, which is important for tasks like question answering and natural language inference.

    NSP controversy: Later work (RoBERTa, ALBERT) showed that NSP may not be necessary. RoBERTa removed it entirely and achieved better results. The consensus is that longer training sequences and more data matter more than the NSP objective. However, NSP remains historically significant as part of the original BERT design.

    Fine-Tuning for Downstream Tasks

    After pretraining, BERT is adapted to specific tasks by adding a thin task-specific layer and fine-tuning all parameters end-to-end. The pretrained weights provide a strong initialization, and fine-tuning adjusts them for the target task using labeled data.

    Sentence Classification

    For tasks like sentiment analysis or textual entailment, the [CLS] token representation is fed into a linear layer followed by softmax. The entire model is fine-tuned on the labeled dataset with cross-entropy loss.

    # Pseudocode for BERT classification fine-tuning
cls_output = bert(input_ids)[0][:, 0, :]   # [CLS] representation
logits = linear_layer(cls_output)            # project to num_classes
loss = cross_entropy(logits, labels)

    Named Entity Recognition (NER)

    For token-level tasks, each token's output representation is fed into a linear layer that classifies it (e.g., as Person, Organization, Location, or Other). The model learns to assign entity tags to each token in the sequence.

    Question Answering

    For extractive QA (SQuAD-style), the input is a question-passage pair. BERT outputs a start logit and an end logit for each token in the passage. The answer span is the substring from the highest-scoring start position to the highest-scoring end position. This is remarkably effective: BERT achieved human-level performance on SQuAD 1.1 when it was released.

    BERT vs GPT

    BERT and GPT represent two fundamentally different approaches to pretraining transformers. Their architectural choices lead to different strengths:

    Property BERT GPT
    Architecture Encoder-only Decoder-only
    Attention Bidirectional (full) Causal (left-to-right)
    Pretraining task Masked Language Modeling Next token prediction
    Context usage Both left and right context Only left context
    Adaptation Fine-tuning with task head Prompting or fine-tuning
    Best for Understanding (classification, NER) Generation (text, code, dialogue)
    Can generate text? Not naturally (no causal masking) Yes (autoregressive)

    BERT Variants

    The success of BERT spawned a family of improved encoder-only models:

    • RoBERTa (2019): Removed NSP, trained longer with more data, dynamic masking. Consistently outperforms BERT on benchmarks.
    • ALBERT (2019): Parameter sharing across layers, factorized embeddings. Achieved comparable performance with far fewer parameters.
    • DeBERTa (2020): Disentangled attention (separate content and position), enhanced mask decoder. Exceeded human performance on SuperGLUE.
    • DistilBERT (2019): Knowledge distillation from BERT-Base. 40% smaller, 60% faster, retains 97% of BERT's performance.
    • ELECTRA (2020): Replaced MLM with a replaced-token-detection task. More sample-efficient pretraining.

    Limitations

    BERT has several well-known limitations:

    • No generation: BERT cannot generate text naturally because it lacks causal masking. You can use it for fill-in-the-blank tasks, but it is not designed for open-ended generation.
    • Fixed context length: BERT-Base supports a maximum of 512 tokens. Longer documents must be truncated or processed in chunks.
    • Pretraining cost: BERT-Large was trained on 16 TPUs for 4 days. Reproducing it from scratch requires significant compute.
    • MLM inefficiency: Only 15% of tokens are masked per training step, so the model receives a training signal from a small fraction of the input. This makes MLM pretraining less sample-efficient than next-token prediction.

    Impact

    BERT fundamentally changed NLP. Before BERT, each task required a custom architecture. After BERT, the standard approach became "pretrain a large model on unlabeled text, then fine-tune on your task." This transfer learning paradigm reduced the amount of labeled data needed, made state-of-the-art NLP accessible to smaller teams, and established pretrained transformers as the default starting point for virtually all NLP work.

    Even though decoder-only models now dominate headlines, encoder-only models remain the practical choice for many production systems where the task is classification, search, or information extraction rather than generation.
