Speculative Decoding

ML Series · Transformers · Variants

1 · The Decoding Bottleneck

Autoregressive decoding generates one token per forward pass. Each pass loads the entire model's weights from HBM — billions of parameters — to produce a single token. This makes decoding memory-bandwidth-bound, not compute-bound:

LLaMA-2 70B decoding: Model weights: 140 GB (FP16) HBM bandwidth: 2 TB/s (A100) Time to load weights: 140 / 2000 = 70 ms per token Actual compute needed: ~0.14 TFLOP (for 1 token) A100 capacity: 312 TFLOP/s Compute time: 0.14 / 312 = 0.45 ms GPU utilization: 0.45 / 70 = 0.6% ← almost entirely idle!

We're loading 140 GB of weights per token but only doing 0.45 ms of useful compute. What if we could get more tokens per weight load?

2 · The Draft-then-Verify Idea

Speculative decoding uses two models:

A small, fast draft model (e.g., 1B params) generates K candidate tokens autoregressively. This is cheap — the draft model loads maybe 2 GB of weights per token instead of 140 GB.

The large target model (e.g., 70B) processes all K draft tokens in a single forward pass. It computes the target probability for each draft token simultaneously. This turns K sequential weight loads into one.

Each draft token is accepted or rejected based on a comparison between draft and target probabilities. Accepted tokens are kept; at the first rejection, a corrected token is sampled from the target. You always get at least 1 token, and up to K+1.

The key insight: Verifying K tokens with the target model costs about the same as generating 1 token (one forward pass either way), but you might accept all K — getting K tokens for the price of 1. The output distribution is mathematically identical to standard decoding.

3 · The Algorithm

def speculative_decode(target_model, draft_model, prompt, K): tokens = prompt while not done: # Phase 1: Draft K tokens with small model draft_tokens = [] draft_probs = [] for i in range(K): q = draft_model(tokens + draft_tokens) # fast! t = sample(q) draft_tokens.append(t) draft_probs.append(q[t]) # Phase 2: Verify ALL K tokens in ONE target forward pass target_logits = target_model(tokens + draft_tokens) # batch of K positions # Phase 3: Accept/reject each draft token n_accepted = 0 for i in range(K): p_target = target_logits[i] # target prob for position i p_draft = draft_probs[i] # draft prob for token i r = uniform(0, 1) if r < min(1, p_target[draft_tokens[i]] / p_draft): tokens.append(draft_tokens[i]) # ACCEPT n_accepted += 1 else: # REJECT: sample correction from adjusted distribution adjusted = max(0, p_target - p_draft) adjusted = adjusted / sum(adjusted) tokens.append(sample(adjusted)) break # stop at first rejection # If all K accepted, sample one bonus token from target if n_accepted == K: tokens.append(sample(target_logits[K])) # K+1 tokens! return tokens

4 · Rejection Sampling Guarantee

The acceptance criterion ensures the output distribution exactly matches the target model's distribution. This is not an approximation — it's mathematically proven:

For each draft token x_i: Accept with probability: min(1, p_target(x_i) / p_draft(x_i)) If p_target(x) ≥ p_draft(x): always accept (target wants this token ≥ as much) If p_target(x) < p_draft(x): accept with prob p_target/p_draft (proportional) On rejection, sample from: p_adjusted(x) = max(0, p_target(x) - p_draft(x)) / Z This ensures: P(output = x) = p_target(x) for all x ← EXACT
This is the magic: You get faster inference with zero quality degradation. The output is statistically indistinguishable from running the target model alone. No approximation, no quantization artifacts, no quality loss.

5 · The Math

Expected acceptance rate for position i: α = Σ_x min(p_target(x), p_draft(x)) = 1 - ½ · ||p_target - p_draft||₁ (related to total variation distance) Expected tokens per round: E[tokens] = (1 - α^(K+1)) / (1 - α) for geometric acceptance With α = 0.8, K = 5: E[tokens] = (1 - 0.8⁶) / (1 - 0.8) = (1 - 0.262) / 0.2 = 3.69 tokens per round
Speedup calculation: Standard: 1 token per target forward pass → T_target per token Speculative: E[tokens] tokens per (K × T_draft + T_target) Speedup = E[tokens] × T_target / (K × T_draft + T_target) With E[tokens] = 3.69, K = 5, T_draft = T_target / 20: Speedup = 3.69 × T / (5 × T/20 + T) = 3.69 / 1.25 = 2.95×

7 · Speedup Analysis

Acceptance Rate (α) K=3 K=5 K=8
0.6 1.8× (2.2 tokens) 1.9× (2.4 tokens) 1.9× (2.5 tokens)
0.7 2.1× (2.6 tokens) 2.3× (3.0 tokens) 2.4× (3.2 tokens)
0.8 2.4× (3.0 tokens) 2.8× (3.7 tokens) 3.1× (4.1 tokens)
0.9 2.7× (3.4 tokens) 3.4× (4.6 tokens) 4.0× (5.7 tokens)
Diminishing returns: Increasing K beyond 5–8 gives little benefit because the probability of accepting all K tokens drops exponentially: P(all accepted) = α^K. At α=0.8 and K=8, there's only a 17% chance all 8 are accepted. Most rounds end with 3–4 accepted tokens regardless of K.

8 · Choosing Draft Models

The draft model must be fast and well-aligned with the target:

  • Smaller version of same family: LLaMA-7B as draft for LLaMA-70B. Good alignment, standard approach.
  • Quantized target: 4-bit quantized version of the target as draft. Perfect alignment since it's the same model, just lower precision.
  • Distilled model: A model specifically trained to mimic the target's distribution. Best alignment but requires training.
  • N-gram model: Extremely fast but low acceptance rate. Works for repetitive text (code, structured output).
Draft Model Speed (vs target) Acceptance Rate Net Speedup
Same-family 10× smaller ~15–20× 70–85% 2.0–3.0×
4-bit quantized target ~2–3× 90–95% 1.5–2.0×
Distilled model ~10–15× 80–90% 2.5–3.5×
N-gram / lookup ~1000× 30–50% 1.3–1.8×

9 · Self-Speculative Decoding

Can we avoid needing a separate draft model entirely? Several approaches use the target model itself:

Use the target model's intermediate layers as the draft: run tokens through the first L/3 layers only, take the prediction as the draft token. No separate model needed, but requires architecture support for early exit heads.

Add small prediction heads to the target model that predict multiple future tokens in parallel. Each head is a 1-layer MLP that predicts token at position +1, +2, +3, etc. Draft generation is free (computed alongside the normal forward pass).

Use the target model's features (hidden states from the last layer) to autoregressively draft future tokens with a lightweight predictor. Achieves high acceptance rates because it operates in feature space, not token space.

10 · Implementation

Key implementation considerations: 1. KV cache management: - Draft model has its own KV cache - On rejection, roll back both draft and target KV caches - Target model's cache grows by n_accepted + 1 per round 2. Batch verification: - Target model processes K tokens as a "pseudo-batch" - Position IDs: [N, N+1, N+2, ..., N+K-1] for draft tokens - One forward pass → K+1 logit distributions - Causal mask ensures each position only sees previous tokens 3. Tree-based speculation: - Instead of linear chain, draft model proposes a tree of candidates - Target verifies all branches in one pass (with attention mask) - Accepts the longest valid path in the tree - SpecInfer / Sequoia: 20-40% additional speedup over linear speculation

11 · Ecosystem

  • vLLM — built-in speculative decoding with configurable draft models
  • HuggingFace TGI — supports speculative decoding for common model pairs
  • TensorRT-LLM (NVIDIA) — optimized speculative decoding kernels
  • Medusa — add-on prediction heads for self-speculative decoding, works with any model
  • EAGLE / EAGLE-2 — feature-level self-speculative decoding, state-of-the-art acceptance rates
  • SpecInfer — tree-based speculation with multiple draft models in parallel
  • Lookahead Decoding — uses Jacobi iteration to generate multiple tokens without a draft model

12 · Summary

  • Speculative decoding uses a fast draft model to propose K tokens, then verifies them in parallel with one target model forward pass.
  • Rejection sampling ensures the output distribution is mathematically identical to standard decoding — zero quality loss.
  • Typical speedup: 2–3× for well-aligned draft models, with K=5 and α=0.8.
  • Self-speculative methods (Medusa, EAGLE) eliminate the need for a separate draft model.
  • Tree-based speculation further improves throughput by exploring multiple continuation paths.
  • Now standard in production serving: vLLM, TGI, TensorRT-LLM all support it.
One-line summary: Speculative decoding drafts cheap guesses and batch-verifies them with the real model — same quality, 2–3× faster.
