    Vision Transformers (ViT)

    ML Series · March 2026

    Transformers Meet Computer Vision

    In October 2020, Dosovitskiy et al. published "An Image Is Worth 16x16 Words," introducing the Vision Transformer (ViT). The paper asked a simple question: can a standard transformer encoder, with minimal modifications, match or exceed convolutional neural networks on image classification? The answer was yes, provided sufficient training data.

    The key insight was that images can be treated as sequences of patches, just as sentences are sequences of tokens. By cutting an image into a grid of fixed-size patches, flattening each patch into a vector, and projecting it into the transformer's embedding space, the entire NLP transformer pipeline can be applied to vision tasks with almost no architectural changes.

    Splitting Images into Patches

    A standard ViT takes a 224x224 pixel image and splits it into non-overlapping 16x16 patches. This produces a grid of 14x14 = 196 patches. Each patch is a 16x16x3 = 768-dimensional vector (when flattened from the three RGB channels).

    These raw patch vectors are then linearly projected into the transformer's hidden dimension using a learnable linear layer. This is equivalent to a convolutional layer with kernel size 16x16 and stride 16. The result is a sequence of 196 patch embeddings, each representing a local region of the image.

    Why 16x16? The patch size controls the sequence length: smaller patches produce longer sequences and capture finer detail, but increase the computational cost quadratically (since self-attention is O(n^2)). The 16x16 default provides a good balance for 224x224 images. For higher-resolution images, smaller patches (e.g., 14x14) can be used at the cost of longer sequences.

    Patch Embeddings

    The linear projection that converts raw patches into embeddings can be implemented in two equivalent ways:

    • Flatten + Linear: Reshape each 16x16x3 patch into a 768-dimensional vector, then multiply by a learnable weight matrix W of shape (768, D) to get a D-dimensional embedding.
    • Conv2d: Apply a 2D convolution with kernel size 16, stride 16, and D output channels. This produces the same result as flatten + linear but is more efficient in practice because it processes the entire image in one operation.
    # PyTorch implementation of patch embedding
import torch.nn as nn

# Equivalent to flatten + linear:
patch_embed = nn.Conv2d(
    in_channels=3,
    out_channels=768,     # transformer hidden dim
    kernel_size=16,
    stride=16
)

# Input: (batch, 3, 224, 224)
# Output: (batch, 768, 14, 14) -> reshape to (batch, 196, 768)

    Position Embeddings

    Unlike CNNs, transformers have no inherent notion of spatial position. Without positional information, the model would treat the patches as an unordered set, losing all spatial structure. ViT uses learnable 1D position embeddings, one for each patch position plus the [CLS] token.

    Surprisingly, learned 1D position embeddings work nearly as well as hand-crafted 2D position encodings. The model learns spatial relationships during training: nearby patches develop similar position embeddings, and the 2D grid structure emerges naturally in the learned representations.

    Position embedding visualization: When researchers visualize the cosine similarity between position embeddings, they observe a clear 2D grid pattern. Position embeddings for patch (2, 3) are most similar to those for patches (1, 3), (3, 3), (2, 2), and (2, 4), which are its immediate 2D neighbors. The model discovers spatial locality without being told about it.

    The [CLS] Token

    Following BERT's convention, ViT prepends a learnable [CLS] (classification) token to the sequence of patch embeddings. This token participates in self-attention with all patch tokens through every layer. Its output representation, after the final encoder layer, is fed into a classification head (an MLP) to produce the final class prediction.

    The [CLS] token serves as a global aggregation mechanism: through self-attention, it collects information from all patches in the image. This is conceptually similar to global average pooling in CNNs, but learned rather than hard-coded.

    An alternative to the [CLS] token is global average pooling over all patch token outputs. Some ViT variants use this approach and achieve comparable performance. The choice is largely a design preference.

    ViT vs CNN Comparison

    Property Vision Transformer (ViT) CNN (e.g., ResNet)
    Inductive bias Minimal (only sequence order) Strong (locality, translation equivariance)
    Receptive field Global from layer 1 (full self-attention) Local, grows with depth
    Data efficiency Needs large datasets (JFT-300M, ImageNet-21K) Works well with smaller datasets
    Scalability Scales well with data and compute Diminishing returns at very large scale
    Attention cost O(n^2) in number of patches O(n) in number of pixels
    Input resolution flexibility Can handle variable resolutions (with interpolated position encodings) Fixed architecture per resolution (typically)
    Feature hierarchy Uniform dimensionality across layers Progressively lower resolution, higher channels
    Pretraining paradigm Supervised, self-supervised (MAE, DINO) Supervised (ImageNet)

    When ViT Works Better

    ViT outperforms CNNs when you have access to large-scale training data. The original paper showed that:

    • On ImageNet alone (1.2M images), ViT underperforms a well-tuned ResNet of comparable compute.
    • On ImageNet-21K (14M images), ViT matches ResNets.
    • On JFT-300M (300M images), ViT significantly outperforms the best CNNs.

    The reason is inductive bias. CNNs have built-in knowledge about images: local connectivity (nearby pixels are related) and translation equivariance (a cat in the top-left looks the same as a cat in the bottom-right). These biases are helpful when data is limited because they constrain the model toward reasonable solutions. ViT lacks these biases, so it must learn spatial relationships from data. With enough data, the learned relationships are more flexible than the hard-coded ones.

    When CNNs Win

    CNNs remain the better choice in several scenarios:

    • Small datasets: With fewer than a million training images, CNNs generalize better due to their inductive biases.
    • Edge deployment: CNNs are easier to quantize, prune, and deploy on mobile and embedded hardware. The convolution operation is well-optimized on all platforms.
    • Dense prediction: Tasks like semantic segmentation and object detection require per-pixel or per-region predictions. While ViT variants (SegFormer, DETR) exist for these tasks, CNN-based architectures remain competitive and simpler.
    • Real-time inference: For fixed-resolution inputs, CNNs are often faster because convolutions have lower memory bandwidth requirements than self-attention.

    ViT Variants and Extensions

    The success of ViT spawned a large family of vision transformer models:

    • DeiT (Data-efficient Image Transformers, 2021): Introduced a knowledge distillation strategy and strong data augmentation. DeiT showed that ViT can match CNNs on ImageNet alone, without JFT pretraining.
    • Swin Transformer (2021): Uses shifted windows to compute local attention, creating a hierarchical feature map similar to CNNs. This made transformers competitive on dense prediction tasks.
    • MAE (Masked Autoencoders, 2022): Self-supervised pretraining by masking 75% of patches and training the model to reconstruct them. Analogous to BERT's MLM but for images.
    • DINO / DINOv2 (2021/2023): Self-supervised training using self-distillation (no labels). Produces features that are useful across many vision tasks without any fine-tuning.
    • BEiT (2022): BERT-style pretraining for vision, predicting visual tokens for masked patches. Bridges NLP and vision pretraining paradigms.

    Practical Considerations

    When using ViT in practice:

    • Start with pretrained weights. Training ViT from scratch on a small dataset will underperform a pretrained CNN. Always initialize from a checkpoint pretrained on a large dataset (ImageNet-21K or similar).
    • Fine-tune at higher resolution. ViT can be pre-trained at 224x224 and fine-tuned at 384x384 by interpolating position embeddings. This often improves accuracy.
    • Use strong augmentation. Because ViT has less inductive bias, it benefits more from data augmentation techniques like RandAugment, Mixup, and CutMix.
    • Consider hybrid models. Some architectures use a CNN stem (first few layers) followed by transformer blocks. This combines the CNN's data efficiency with the transformer's flexibility.

    Impact on Computer Vision

    ViT demonstrated that the transformer architecture is not specific to language. The same attention mechanism that models relationships between words in a sentence can model relationships between patches in an image. This opened the door to unified architectures that process multiple modalities (text, images, audio, video) with the same model structure, leading to multimodal models like CLIP, DALL-E, and GPT-4V.

    The vision transformer also shifted the field's focus from architectural engineering to data and scale. Rather than designing specialized convolution patterns, the emphasis moved to collecting larger datasets, designing better pretraining objectives, and scaling compute. This mirrors the trajectory that NLP followed after the original transformer paper.
