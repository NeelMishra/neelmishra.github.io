    Positional Encoding

    ML Series · March 2026

    The Problem: Transformers Have No Sense of Order

    Recurrent networks process tokens one at a time, left to right. The hidden state at position 5 depends on positions 1 through 4, so order is baked into the computation. Convolutions have a fixed receptive field that gives each output a spatial relationship to its neighbors. Both architectures encode position implicitly through their structure.

    Transformers do neither. Self-attention computes a weighted sum over all positions simultaneously. The attention weight between tokens i and j depends only on the content of those tokens (their query and key vectors), not on whether i comes before or after j. The operation is permutation-equivariant: if you shuffle the input tokens, the output gets shuffled in exactly the same way, with exactly the same values.

    This means that, without any additional mechanism, the sentences:

    The cat sat on the mat
mat the on sat cat the

    produce identical attention patterns and identical outputs. Every token attends to every other token with the same weights, because the content at each position is the same set of words. The model has no way to distinguish "the" at position 0 from "the" at position 4. It cannot learn that subjects tend to come before verbs, or that adjectives precede nouns.

    For language, order is meaning. "Dog bites man" and "man bites dog" have the same words but opposite meanings. Any useful language model must have access to positional information. The question is how to provide it.

    The core requirement: We need a way to inject position information into the model so that the representation of each token reflects not just what the token is, but where it appears in the sequence.

    Sinusoidal Positional Encoding

    The original Transformer paper (Vaswani et al., 2017) introduced sinusoidal positional encoding. The idea is elegant: construct a vector of the same dimension as the word embedding, fill it with values computed from sine and cosine functions at different frequencies, and add it to the embedding before feeding it into the encoder or decoder.

    The Formula

    For a token at position pos in the sequence and dimension index i in the encoding vector (where the model dimension is d):

    PE(pos, 2i) = sin(pos / 10000^(2i/d)) PE(pos, 2i + 1) = cos(pos / 10000^(2i/d))

    Even-indexed dimensions get a sine, odd-indexed dimensions get a cosine. Both use the same frequency, which depends on the dimension index i. The denominator 10000^(2i/d) creates a geometric progression of wavelengths from 2π (at i = 0) to 10000 · 2π (at i = d/2 - 1).

    Frequency Intuition

    Think of each pair of dimensions (2i, 2i+1) as a clock running at a different speed:

    • Low dimensions (small i): high frequency. The sine and cosine oscillate rapidly, changing significantly from one position to the next. These dimensions capture fine-grained, local position differences.
    • High dimensions (large i): low frequency. The sine and cosine change very slowly across positions. These dimensions capture coarse, global position information.

    The analogy to binary representation is helpful. In a binary counter, the least significant bit flips every step, the next bit flips every two steps, the next every four, and so on. Sinusoidal encoding does the same thing but with smooth, continuous oscillations instead of discrete bit flips. Each dimension oscillates at a wavelength that is a geometric progression from short to long.

    Why It Works

    The combination of many different frequencies means that every position gets a unique encoding vector. No two positions share the same pattern of sine and cosine values across all dimensions. The model can learn to read these patterns to determine absolute position, and more importantly, to determine relative position between any two tokens.

    Implementation in Code

    import numpy as np

def sinusoidal_encoding(max_len, d_model):
    pe = np.zeros((max_len, d_model))
    position = np.arange(max_len).reshape(-1, 1)
    div_term = np.exp(
        np.arange(0, d_model, 2) * -(np.log(10000.0) / d_model)
    )
    pe[:, 0::2] = np.sin(position * div_term)
    pe[:, 1::2] = np.cos(position * div_term)
    return pe

    The div_term computation uses the log-space trick to avoid numerical overflow: instead of computing 10000^(2i/d) directly, it computes exp(-2i/d * log(10000)), which is mathematically equivalent but numerically stable.

    Why Sine and Cosine? The Rotation Matrix Interpretation

    The choice of sine and cosine is not arbitrary. It has a deep geometric reason: relative position can be expressed as a linear transformation of absolute position encodings.

    The Relative Position Property

    For any fixed offset k, there exists a linear transformation M (a matrix that depends only on k, not on pos) such that:

    PE(pos + k) = M_k · PE(pos)

    This means the model can learn to compute "what is 3 positions ahead" as a simple matrix multiplication, regardless of whether it is going from position 2 to 5 or from position 50 to 53. The transformation is the same.

    The Rotation Matrix

    Consider a single pair of dimensions (2i, 2i+1) with frequency ωi = 1/100002i/d. The encoding at position pos for this pair is:

    [sin(ω_i · pos), cos(ω_i · pos)]

    To shift from position pos to position pos + k, apply the standard 2D rotation matrix:

    | sin(ω_i · (pos+k)) | | cos(ω_i · k) sin(ω_i · k) | | sin(ω_i · pos) | | | = | | · | | | cos(ω_i · (pos+k)) | | -sin(ω_i · k) cos(ω_i · k) | | cos(ω_i · pos) |

    This is the angle addition formula from trigonometry, rewritten in matrix form. The matrix on the right depends only on the offset k and the frequency ωi, not on the absolute position. The full transformation matrix Mk is block-diagonal, with one such 2x2 rotation block for each dimension pair.

    Why Linearity Matters

    Attention operates through dot products and linear projections. If relative position can be expressed as a linear transformation, the attention mechanism can learn weight matrices that implicitly perform this transformation. A query at position pos can learn to "look for the token 3 positions back" by learning weights that correspond to the inverse rotation M-3.

    If the positional encoding used a non-linear function (say, a hash of the position), there would be no simple relationship between PE(pos) and PE(pos + k). The model would have to memorize the relationship for every pair of positions, rather than learning a general "relative offset" pattern.

    Key insight: Sinusoidal encodings encode position as a point on a high-dimensional circle (or more precisely, a torus). Moving from one position to the next is a rotation. Rotations are linear transformations, and linear transformations are exactly what neural network layers can learn efficiently.

    How Positional Encoding Shifts Embeddings

    Word embeddings map tokens to points in a high-dimensional vector space. Words with similar meanings cluster together. When we add a positional encoding vector to an embedding, we shift the point in that space. The same word at different positions ends up at different locations.

Adding the positional encoding vector shifts it to a new location. Different positions produce different shifts, so the same word at position 0 lands in a different part of the space than the same word at position 5.

    Learned vs Fixed Positional Encodings

    The sinusoidal encoding described above is fixed: the values are computed from a formula and never change during training. An alternative is to make the positional encoding a learned parameter: initialize a matrix of shape (max_length, d_model) randomly and let the optimizer update it during training, just like word embeddings.

    Comparison

    Property Fixed (Sinusoidal) Learned
    Extra parameters None max_len × d_model
    Generalization to longer sequences Natural extrapolation: the formula works for any position, even beyond training length Cannot extrapolate: positions beyond max_len have no learned embedding
    Training signal Not needed; structure is built in Requires sufficient training data to learn useful patterns
    Relative position Built-in via rotation matrix property Must be discovered by the optimizer; not guaranteed
    Performance (original paper) Nearly identical to learned Nearly identical to fixed
    Used in practice Original Transformer, some inference-focused models BERT, GPT-2, most modern models (combined with other techniques)

    Vaswani et al. reported that sinusoidal and learned encodings performed nearly identically on their translation task. They chose sinusoidal because it generalizes to sequence lengths not seen during training. In practice, most subsequent models (BERT, GPT-2, GPT-3) use learned encodings, often combined with other positional techniques.

    Relative Positional Encodings

    Sinusoidal and learned encodings are both absolute: each position gets a fixed vector regardless of context. Relative positional encodings instead encode the distance between two tokens, which is often more useful for attention computation. Several important variants have emerged.

    RoPE (Rotary Position Embedding)

    RoPE (Su et al., 2021) applies the rotation matrix idea directly inside the attention computation. Instead of adding a positional vector to the embedding, RoPE rotates the query and key vectors by an angle proportional to their position. The dot product between a rotated query at position m and a rotated key at position n depends only on the relative offset m - n.

    The rotation is applied in 2D blocks, exactly like the sinusoidal rotation matrix described above. The key difference is that the rotation happens inside the attention mechanism rather than at the input. This means position information is injected at every layer, not just at the input.

    RoPE is used in LLaMA, PaLM, and many other modern large language models. It has become the dominant positional encoding scheme for decoder-only transformers.

    ALiBi (Attention with Linear Biases)

    ALiBi (Press et al., 2022) takes a different approach entirely. Instead of modifying embeddings or rotations, it adds a linear bias to the attention scores. The bias is proportional to the distance between the query and key positions:

    attention_score(q_m, k_n) = q_m · k_n - m_head · |m - n|

    Each attention head uses a different slope mhead (set as a geometric sequence, not learned). This means nearby tokens get higher attention scores than distant ones, with the penalty scaling linearly with distance. Different heads have different slopes, so some heads focus locally (steep slope) and others attend more broadly (gentle slope).

    ALiBi adds no parameters, requires no changes to the embedding, and has strong length extrapolation: a model trained on sequences of length 1024 can often generalize to length 2048 or beyond without fine-tuning.

    Other Approaches

    • Relative position bias (T5): Learns a bias for each possible relative distance, bucketed logarithmically for long distances. Simple and effective.
    • Sandwich encoding: Injects positional encoding at multiple layers, not just the input.
    • NoPE: Recent work showing that decoder-only models with causal masking can sometimes perform well without explicit positional encoding, because the causal mask itself provides some positional signal.

    Why It Works: Multi-Scale Position Representation

    The fundamental reason sinusoidal positional encoding works well is that it provides a multi-scale representation of position. This is worth examining in detail.

    What a Single Frequency Captures

    A single sine wave at frequency ω has wavelength 2π/ω. Two positions that differ by exactly one wavelength produce the same sine value. So a single frequency can distinguish positions within one period, but positions separated by a full period become indistinguishable (they "alias").

    With a high frequency (short wavelength), nearby positions are easily distinguished, but positions far apart may collide. With a low frequency (long wavelength), distant positions are well-separated, but nearby positions look almost identical.

    Combining Multiple Frequencies

    By using many frequencies simultaneously, from very high to very low, the encoding avoids the weaknesses of any single frequency. The high-frequency components resolve nearby positions; the low-frequency components resolve distant ones. The combination produces a unique fingerprint for every position within a very large range.

    This is analogous to how the Fourier transform represents a signal: any signal can be decomposed into a sum of sinusoids at different frequencies. The positional encoding works in reverse: it constructs a unique signal (the position vector) by combining sinusoids at geometrically spaced frequencies.

    Dot Product and Distance

    The dot product between two positional encoding vectors PE(pos) and PE(pos + k) depends only on the offset k. For small k, the vectors are similar (high dot product). As k increases, the dot product oscillates but generally decreases. This gives the attention mechanism a natural "proximity bias" even before any training occurs: tokens that are close together in the sequence start with higher mutual attention, which aligns well with the local structure of natural language.

    Summary

    • Transformers process all positions in parallel and have no inherent notion of sequence order.
    • Positional encoding solves this by adding a position-dependent vector to each token embedding.
    • Sinusoidal encoding uses sine and cosine functions at geometrically increasing wavelengths, producing a unique multi-scale fingerprint for every position.
    • The sine/cosine choice enables relative position to be computed via a linear (rotation) transformation.
    • Learned encodings are a viable alternative but cannot extrapolate to unseen lengths.
    • Modern approaches (RoPE, ALiBi) inject position information directly into the attention computation for better performance and length generalization.
