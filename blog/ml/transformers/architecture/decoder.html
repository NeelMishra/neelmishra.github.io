    The Decoder Block

    ML Series · March 2026

    How the Decoder Differs from the Encoder

    The encoder and decoder share the same fundamental components: multi-head attention, feed-forward networks, residual connections, and layer normalization. But the decoder adds two critical differences:

    1. Masked self-attention. The decoder cannot look at future tokens. Each position can only attend to itself and earlier positions. This is enforced by applying a mask to the attention scores before softmax.
    2. Cross-attention. After masked self-attention, the decoder performs a second attention operation where queries come from the decoder and keys/values come from the encoder output. This is how the decoder reads the input sequence.

    Each decoder block therefore has three sub-layers (each with residual + layer norm), compared to the encoder's two.

    Decoder Block Structure

    Masked Self-Attention

    In the encoder, every token can attend to every other token. The decoder cannot do this. When generating the third output token, the model must not see tokens 4, 5, 6, etc. because those have not been generated yet. This is the autoregressive property: each token is predicted based only on previous tokens.

    The masking is implemented by adding a large negative number (typically negative infinity) to the attention scores at positions that correspond to future tokens. After softmax, these positions receive a weight of essentially zero.

    # Masked attention computation: scores = Q @ K.T / sqrt(d_k) # [batch, heads, tgt_len, tgt_len] scores = scores + mask # mask has -inf at illegal positions weights = softmax(scores, dim=-1) # -inf becomes 0.0 after softmax output = weights @ V # [batch, heads, tgt_len, d_v]

    The Triangular Mask Matrix

    The mask is an upper triangular matrix filled with negative infinity. Position i can attend to positions 0 through i (the diagonal and below), but not to positions i+1 and beyond (above the diagonal).

    The green cells (value 0, meaning no modification to the attention score) form a lower triangular matrix. Token t2, for instance, can attend to t0, t1, and t2, but not to t3, t4, or t5. This mask is computed once and reused across all heads and all decoder layers.

    Cross-Attention: Reading the Encoder Output

    The second attention sub-layer in the decoder is where the decoder reads the input sequence. The mechanism is identical to self-attention, but the sources of Q, K, V differ:

    • Queries (Q): come from the decoder (the output of the masked self-attention sub-layer)
    • Keys (K): come from the encoder output
    • Values (V): come from the encoder output
    # Cross-attention: Q = decoder_state @ W_Q # [batch, tgt_len, d_model] K = encoder_output @ W_K # [batch, src_len, d_model] V = encoder_output @ W_V # [batch, src_len, d_model] scores = Q @ K.T / sqrt(d_k) # [batch, heads, tgt_len, src_len] # No mask here! Decoder can attend to ALL source positions weights = softmax(scores, dim=-1) # [batch, heads, tgt_len, src_len] output = weights @ V # [batch, heads, tgt_len, d_v]

    Notice the attention matrix shape: [tgt_len, src_len]. Each decoder position produces a query that attends across all encoder positions. There is no mask here because the decoder is allowed to see the entire input sequence. The only restriction (from masked self-attention) is on seeing future output tokens.

    Why separate K/V projections? The encoder output is projected through its own W_K and W_V matrices (different from the encoder's internal attention weights). This gives the cross-attention layer its own learned mapping of what information to extract from the encoder. The same encoder output is used as K/V input to every decoder layer, but each layer has its own projection weights.

    Step-by-Step: Generating One Token at a Time

    During inference, the decoder generates tokens autoregressively. It starts with a special start-of-sequence token and generates one token per forward pass:

    Click "Next Token" to begin autoregressive generation.

    Each step feeds the entire generated sequence so far into the decoder. The decoder produces a probability distribution over the vocabulary at the last position. The chosen token is appended to the sequence, and the process repeats until an end-of-sequence token is produced (or a maximum length is reached).

    This is fundamentally O(n) forward passes for a sequence of length n. Each forward pass processes the growing sequence. With KV caching (storing intermediate key/value computations from prior steps), the per-step cost is reduced because only the new token's computations need to run through the attention layers. The cached K/V values from prior tokens are reused.

    Teacher Forcing vs Autoregressive Inference

    During training, the decoder does not generate tokens one at a time. Instead, the entire target sequence (shifted right by one position) is fed in at once. This is called teacher forcing: at every position, the model receives the correct previous token as input, regardless of what it would have predicted.

    PropertyTraining (Teacher Forcing)Inference (Autoregressive)
    Decoder input Entire target sequence (shifted right) Tokens generated so far
    Forward passes One pass for all positions (parallel) One pass per generated token (sequential)
    Loss computed at All positions simultaneously Not applicable (greedy or sampled output)
    Previous token source Ground truth (always correct) Model's own prediction (may be wrong)
    Mask required Yes (prevent seeing future targets) Not needed (future tokens do not exist yet)
    Speed Fast (fully parallelized) Slow (sequential token generation)

    The mask during training simulates the autoregressive constraint. Position 3 cannot see positions 4, 5, etc., even though all target tokens are present in the batch. This makes training parallel while maintaining the same information flow that inference will use.

    Exposure bias: During training, the decoder always sees correct previous tokens. During inference, it sees its own (potentially incorrect) predictions. This mismatch is called exposure bias. Techniques like scheduled sampling partially address it, but most Transformer models train with pure teacher forcing.

    The Final Linear + Softmax Layer

    After the last decoder block, the output at each position is a vector of dimension d_model (512). To convert this into a prediction over the vocabulary, two operations are applied:

    # Final output projection: logits = decoder_output @ W_vocab # [batch, tgt_len, vocab_size] # W_vocab: [d_model, vocab_size] = [512, 37000] probs = softmax(logits, dim=-1) # [batch, tgt_len, vocab_size] # Each position: probability distribution over 37000 tokens

    The linear projection W_vocab is the largest single weight matrix in the model: 512 × 37,000 = 18.9 million parameters. In many implementations, this matrix is tied (shared) with the input embedding matrix, which has shape [vocab_size, d_model]. Weight tying reduces parameter count and often improves performance.

    Decoding Strategies

    Given the probability distribution, several strategies exist for selecting the next token:

    • Greedy decoding: always pick the highest-probability token. Fast but can miss better overall sequences.
    • Beam search: maintain k candidate sequences and expand each at every step. Pick the best complete sequence. Standard for machine translation.
    • Top-k sampling: sample from the top k most probable tokens. Used in creative generation tasks.
    • Top-p (nucleus) sampling: sample from the smallest set of tokens whose cumulative probability exceeds p. Adapts the candidate set size dynamically.
    • Temperature scaling: divide logits by a temperature T before softmax. T < 1 sharpens the distribution (more deterministic), T > 1 flattens it (more random).

    Putting It All Together

    The decoder block is the encoder block with two additions: the causal mask (ensuring autoregressive generation) and the cross-attention layer (reading the encoder's output). During training, the entire target sequence is processed in parallel with masking. During inference, tokens are generated one at a time.

    The three sub-layers per decoder block are:

    1. Masked multi-head self-attention + residual + layer norm
    2. Multi-head cross-attention + residual + layer norm
    3. Feed-forward network + residual + layer norm

    Modern decoder-only models (GPT, LLaMA) drop the encoder and cross-attention entirely. They keep only masked self-attention and the feed-forward network. The entire model is a stack of decoder blocks, and the "input" is simply the prompt prepended to the generation.
