    Transformer Architecture Overview

    ML Series · March 2026

    The Paper That Changed Everything

    In June 2017, Vaswani et al. published "Attention Is All You Need". The paper introduced a sequence-to-sequence architecture that relied entirely on attention mechanisms, discarding the recurrent and convolutional layers that had dominated NLP for years. The model was called the Transformer.

    The original task was machine translation (English to German, English to French). The Transformer achieved state-of-the-art BLEU scores while training significantly faster than the best existing models. Training took 3.5 days on 8 GPUs, compared to weeks for comparable RNN-based systems.

    The key insight was simple but powerful: self-attention can replace recurrence. Instead of processing tokens one at a time (left to right), the Transformer processes all tokens in parallel and lets each token attend to every other token in the sequence. This removes the sequential bottleneck that made RNNs slow to train and difficult to scale.

    Why this matters: Every major language model since 2018 (BERT, GPT, T5, LLaMA, etc.) is built on the Transformer architecture. Understanding it is foundational to understanding modern AI.

    The Encoder-Decoder Architecture

    The original Transformer follows an encoder-decoder pattern. The encoder reads the entire input sequence and produces a rich representation. The decoder generates the output sequence one token at a time, attending to both its own previously generated tokens and the encoder's output.

    The diagram shows the two halves of the architecture. The left stack is the encoder: it processes the input sequence through layers of self-attention and feed-forward networks. The right stack is the decoder: it generates the output sequence, attending both to itself (masked, so it cannot see future tokens) and to the encoder output (via cross-attention).

    Data Flow: Input Tokens to Output Tokens

    Here is the complete path a sentence takes through the Transformer during translation:

    1. Tokenization. The input sentence is split into subword tokens. Each token is mapped to an integer ID from a fixed vocabulary.
    2. Embedding. Each token ID is looked up in an embedding matrix (size vocab_size x d_model), producing a dense vector of dimension d_model (512 in the original paper).
    3. Positional encoding. A position-dependent signal is added to each embedding so the model knows token order. (Covered in detail in the positional encoding article.)
    4. Encoder processing. The sequence of vectors passes through N=6 identical encoder blocks. Each block refines the representations through self-attention (letting each token gather context from all other tokens) and a feed-forward network (processing each position independently).
    5. Decoder input. The target sequence (shifted right by one position, with a start token prepended) is embedded and position-encoded in the same way.
    6. Decoder processing. Each decoder block first performs masked self-attention (each position can only attend to earlier positions), then cross-attention (attending to the encoder output), then a feed-forward network.
    7. Projection. The decoder output at each position is projected through a linear layer to vocabulary size, then passed through softmax to produce a probability distribution over the next token.
    8. Token selection. During training, the loss is computed against the true next token. During inference, the highest-probability token (or a sampled token) is selected and fed back as input for the next step.

    Why the Architecture Was Revolutionary

    Three properties made the Transformer a fundamental shift in sequence modeling:

    No Recurrence

    RNNs process tokens sequentially: the hidden state at position t depends on the hidden state at position t-1. This creates a chain of dependencies that is inherently serial. The Transformer replaces this with self-attention, where every token can attend to every other token in a single operation. The computation for position 50 does not wait for positions 1 through 49.

    Fully Parallelizable Training

    Because there is no sequential dependency during the forward pass (encoder side) and because the decoder uses teacher forcing during training (the true target is provided at every position), the entire training computation can be parallelized across sequence positions. This maps efficiently to GPU hardware, where thousands of cores can operate simultaneously.

    Direct Long-Range Connections

    In an RNN, information from position 1 must pass through every intermediate hidden state to reach position 100. Each step risks information loss. In a Transformer, position 100 can directly attend to position 1 in a single attention operation. The path length between any two positions is O(1), not O(n).

    The scaling insight: Removing recurrence did not just make training faster. It made the architecture scalable. You can increase model size, sequence length, and batch size, and the hardware utilization remains high. This is why Transformers enabled the scaling laws that drive modern LLMs.

    RNN vs LSTM vs Transformer

    The following table summarizes the key architectural differences:

    Property Vanilla RNN LSTM / GRU Transformer
    Sequential processing Yes (inherent) Yes (inherent) No (fully parallel)
    Max path length O(n) O(n) O(1)
    Complexity per layer O(n · d²) O(n · d²) O(n² · d)
    Long-range dependencies Poor (vanishing gradient) Better (gating helps) Excellent (direct attention)
    Training parallelism Low Low High
    Memory for context Fixed-size hidden state Fixed-size cell + hidden Grows with sequence length
    Position awareness Implicit (order of processing) Implicit (order of processing) Explicit (positional encoding)
    Hardware utilization Low (sequential ops) Low (sequential ops) High (matrix multiplications)

    The O(n² · d) attention cost is the Transformer's main limitation. For very long sequences, attention becomes expensive. This has motivated research into efficient attention variants (Linformer, Performer, Flash Attention), but the basic architecture remains dominant for sequences up to several thousand tokens.

    Key Building Blocks

    The Transformer is composed of a small number of repeated components. Each is covered in depth in subsequent articles. Here is a preview:

    Scaled Dot-Product Attention

    The core operation. Computes a weighted sum of values, where the weight for each value is determined by the compatibility (dot product) of a query with the corresponding key. Scaled by 1/sqrt(d_k) to prevent softmax saturation.

    Multi-Head Attention

    Runs attention in parallel across multiple "heads," each projecting queries, keys, and values to a lower dimension. The outputs are concatenated and projected back. This allows the model to attend to information from different representation subspaces simultaneously.

    Position-wise Feed-Forward Network

    A two-layer fully connected network applied identically to each position. The inner dimension expands (512 to 2048), applies a nonlinearity (ReLU in the original), then projects back to d_model. This is where most of the model's parameters reside.

    Residual Connections + Layer Norm

    Every sub-layer (attention or FFN) has a residual connection around it, followed by layer normalization. The residual connection provides a gradient highway during backpropagation. Layer normalization stabilizes training by normalizing activations.

    Original Paper Hyperparameters

    For reference, the "base" model from the paper used:

    • d_model = 512 (embedding dimension and residual stream width)
    • N = 6 (number of encoder and decoder blocks)
    • h = 8 (number of attention heads)
    • d_k = d_v = 64 (per-head key/value dimension, i.e. 512/8)
    • d_ff = 2048 (inner dimension of the feed-forward network)
    • Vocabulary = ~37,000 (shared source/target BPE vocabulary)
    • Dropout = 0.1 (applied to attention weights and sub-layer outputs)

    The "big" model doubled d_model to 1024 and increased heads to 16, achieving slightly better BLEU scores at higher computational cost.
