    Regression Mastery: Linear and Logistic Models

    July 2026 · 11 min read

    Regression, in the sense used across this series, is the problem of estimating a conditional summary of an outcome $Y$ given predictors $X$. The summary is a conditional mean when $Y$ is continuous and a conditional probability when $Y$ is binary. Both cases reduce to the same question: given a value of $X$, what number best represents the typical value of $Y$ observed at that $X$? Linear regression answers this question for continuous $Y$, and logistic regression answers a closely related question for binary $Y$. This page lays out the shared framework, then splits into two dedicated tracks.

    Conditional Mean Versus Class Probability

    A regression model estimates a population function rather than memorizing individual observations. A continuous outcome has the squared-loss target $\mu(x)=E[Y\mid X=x]$. A binary outcome has $E[Y\mid X=x]=P(Y=1\mid X=x)$ because $Y\in\{0,1\}$, but a logistic model additionally constrains that mean to $[0,1]$ through $p(x)=\sigma(x^\top\beta)$. The same conditional-expectation principle therefore underlies both tracks, while the support of the outcome determines the link and likelihood.

    $$E[Y\mid X=x]=0\cdot P(Y=0\mid x)+1\cdot P(Y=1\mid x)=P(Y=1\mid x).$$

    Model Selection and the Shared Score Contract

    The feature contract is identical in both tracks: a validated row becomes $\phi(x)=(1,x_1,\ldots,x_p,\text{derived terms})$, and the model computes one real score $s=\phi(x)^\top\beta$. Linear regression returns $s$ as an estimate of a continuous conditional mean. Logistic regression returns $\sigma(s)=1/(1+e^{-s})$ as a class probability and applies a separately chosen threshold only when a class decision is required. Fit, score, predict, and evaluate therefore remain separate operations in theory and in the matching C++ design.

    • Continuous, unbounded response: the identity link and least squares provide an interpretable conditional-mean model.
    • Binary response: the logit link and Bernoulli log-loss preserve valid probabilities.
    • Counts, positive skew, censoring, or repeated dependence: neither default is automatically adequate; a distribution and dependence structure matching the data is required.

    ▶ Assumption and Failure-Map Decision Engine

    Each scene is derived from an outcome type and observed model diagnostics. The engine selects the estimand, link, loss, uncertainty correction, and escalation path rather than displaying a sample-average convergence story.

    The first two scenes satisfy the basic support assumptions and select linear or logistic regression. The remaining scenes encode failures observed from the data: a fan-shaped residual plot changes the covariance estimator, curvature changes the feature map, label leakage invalidates evaluation, and a count response changes the response family. A counterexample is the linear probability model: it estimates a conditional mean for binary $Y$, but can return $-0.12$ or $1.08$ and has variance $p(x)(1-p(x))$, so ordinary homoscedastic standard errors are inappropriate.

    Two Link Functions, One Fitting Principle

    Both tracks in this series model the conditional summary as a linear combination of the predictors, $X\beta$, but they connect that linear combination to the target through a different link function.

    • Linear regression uses the identity link: $E[Y\mid X] = X\beta$ directly, so the fitted linear combination is itself the estimated conditional mean.
    • Logistic regression uses the logit link: $\log\dfrac{p(X)}{1-p(X)} = X\beta$, so the linear combination models log-odds, and the conditional probability is recovered through the sigmoid, $p(X) = \dfrac{1}{1+e^{-X\beta}}$.

    Both estimators are fit by minimizing an expected loss: squared error for linear regression, and negative log-likelihood (log-loss) for logistic regression. The two loss functions are not interchangeable arbitrarily. Squared error targets a conditional mean whenever the second moment is finite; symmetry and constant variance are separate assumptions used for particular efficiency and inference claims. Log-loss targets a probability and is implied by the Bernoulli likelihood.

    The target property follows directly for squared loss. Conditional on $X=x$, differentiating $E[(Y-a)^2\mid x]$ with respect to $a$ gives $2(a-E[Y\mid x])$, so the minimizer is the conditional mean. For Bernoulli log-loss, differentiating $-p\log q-(1-p)\log(1-q)$ gives $(q-p)/(q(1-q))$, whose unique interior zero is $q=p$.

    Failure Modes That Cross Both Tracks

    Several practical issues are not specific to either link function and appear in both tracks under slightly different names.

    • Multicollinearity. Highly correlated predictors inflate coefficient variance in linear regression and cause the same instability, plus slow or non-convergent optimization, in logistic regression.
    • Extrapolation. A linear regression line extended past the observed range of $X$ produces a number with no empirical support behind it, and a logistic regression score extended past the observed range produces a probability that can be badly miscalibrated for the same reason.
    • Omitted-variable and specification bias. Leaving out relevant structure can shift included coefficients in both models. The exact linear-regression product formula does not transfer unchanged to logistic regression because the logit link is nonlinear and odds ratios are non-collapsible.
    • Leakage. A feature that encodes information from the future relative to the prediction point inflates apparent accuracy identically whether the loss is squared error or log-loss.
    Interview trap. Least squares on $0/1$ labels produces the linear probability model. Its fitted values are not constrained to $[0,1]$, and its conditional error variance is $p(x)(1-p(x))$, so the homoskedastic OLS standard-error formula is invalid. Robust standard errors can address that covariance problem, but not the range violation or linear probability shape. Logistic regression preserves probability support and matches the Bernoulli likelihood.

    Assumptions and a Worked Selection Example

    Both tracks require representative sampling, a stable feature definition between fit and prediction, no target leakage, adequate overlap in the feature region used for prediction, and a conditional functional form rich enough for the estimand. Linear OLS coefficient inference additionally needs full rank and exogeneity; the textbook covariance formula needs homoscedastic, uncorrelated errors. Logistic maximum likelihood needs identifiable coefficients and no complete separation for a finite unregularized solution.

    For rows $(\text{hours},Y)=(1,0),(2,0),(3,1),(4,1)$, the response is binary, so the estimand is $P(Y=1\mid\text{hours})$. With $\beta=(-3,1)$, a row with three hours has score $s=-3+1(3)=0$ and probability $\sigma(0)=0.5$. The same score used as a linear prediction would happen to equal $0$ here, but scores at six hours would equal $3$, an impossible probability. Complete separation in this tiny sample is an edge case: unregularized logistic coefficients diverge, while the ranking may still look perfect.
