    Multiclass Logistic and Softmax Regression

    July 2026 · 13 min read

    Binary logistic regression generalizes to $K$ classes in two ways. One-vs-rest trains a separate binary model for each class and combines their scores; multinomial, or softmax, regression models all classes jointly with a single likelihood. The multinomial form is the direct generalization of the Bernoulli model and the one that produces a coherent probability distribution over classes, and it reduces exactly to binary logistic regression when $K=2$.

    One-vs-Rest

    The one-vs-rest strategy trains $K$ independent binary classifiers, the $k$-th distinguishing class $k$ from the union of all other classes. A new example is scored by all $K$ models and assigned the class whose model gives the highest probability. The appeal is simplicity and reuse: any binary classifier becomes a multiclass one with no new mathematics, and the $K$ fits are embarrassingly parallel.

    The cost is coherence. Each binary model is calibrated against a different partition of the data, so their probabilities are estimated on incompatible scales and do not sum to one; combining them requires an ad hoc renormalization that has no probabilistic justification. Class imbalance is also worse for one-vs-rest, since the rest group is typically much larger than the one group in every sub-problem. One-vs-rest remains a reasonable default when a joint fit is inconvenient, but it does not model the categorical outcome directly.

    Multinomial Softmax Regression

    Softmax regression assigns each class $k$ its own coefficient vector $\beta_k$ and models the class probabilities jointly. With per-class scores $z_k=\beta_k^\top\tilde{x}$, the probability of class $k$ is the softmax of the scores,

    $$\Pr(Y=k\mid x)=\operatorname{softmax}(z)_k=\frac{e^{z_k}}{\sum_{l=1}^{K}e^{z_l}}.$$

    The denominator normalizes across classes, so the probabilities are nonnegative and sum to one by construction, giving a genuine categorical distribution. This is the maximum-entropy distribution consistent with the linear score constraints, the multiclass counterpart of the Bernoulli exponential family, and it is the model that binary logistic regression is a special case of.

    The Softmax and Log-Sum-Exp

    The softmax has a shift invariance that is central to both its identifiability and its numerical stability: adding the same constant $c$ to every score leaves the probabilities unchanged, because the factor $e^{c}$ cancels between numerator and denominator,

    $$\operatorname{softmax}(z+c\mathbf{1})_k=\frac{e^{z_k+c}}{\sum_l e^{z_l+c}}=\frac{e^{c}e^{z_k}}{e^{c}\sum_l e^{z_l}}=\operatorname{softmax}(z)_k.$$

    Evaluated naively, $e^{z_k}$ overflows for large scores and underflows for very negative ones, poisoning the ratio. The shift invariance supplies the fix: subtracting the maximum score $m=\max_l z_l$ before exponentiating leaves the result unchanged while guaranteeing the largest exponent is zero, so no term overflows. The related log-sum-exp function, which appears in the loss, is computed the same way,

    $$\operatorname{logsumexp}(z)=\ln\sum_{l}e^{z_l}=m+\ln\sum_{l}e^{z_l-m}.$$

    The animation walks through this computation on the scores $z=(12,10,8)$, whose stable softmax is $(0.8668,0.1173,0.0159)$ and whose log-sum-exp is $12.1429$, and then contrasts the naive and stable routes on the shifted scores $(1000,1001,1002)$, where the naive exponential overflows to infinity while the stable computation returns $(0.0900,0.2447,0.6652)$.

    Categorical Likelihood, Gradient, and Hessian

    With the label encoded as a one-hot vector $y_i$ whose entry $y_{ik}$ is one for the true class and zero otherwise, the per-row negative log-likelihood is the categorical cross-entropy,

    $$\ell_i=-\sum_{k=1}^{K}y_{ik}\ln p_{ik}=-\ln p_{i,c_i}=\operatorname{logsumexp}(z_i)-z_{i,c_i},$$

    where $c_i$ is the true class. The last form mirrors the binary softplus expression and is the one to compute, since it never materializes a probability before taking a logarithm. The gradient with respect to the score of class $k$ is the same residual structure as the binary case,

    $$\frac{\partial \ell_i}{\partial z_{ik}}=p_{ik}-y_{ik},\qquad \nabla_{\beta_k}J=\frac{1}{n}\sum_{i=1}^{n}(p_{ik}-y_{ik})\,\tilde{x}_i,$$

    a predicted-minus-observed residual times the features, stacked one block per class. The Hessian has a block structure indexed by class pairs, built from the softmax derivative $\partial p_{ik}/\partial z_{il}=p_{ik}(\delta_{kl}-p_{il})$,

    $$\nabla^2_{\beta_k,\beta_l}J=\frac{1}{n}\sum_{i=1}^{n}p_{ik}(\delta_{kl}-p_{il})\,\tilde{x}_i\tilde{x}_i^\top.$$

    The matrix $\operatorname{diag}(p_i)-p_ip_i^\top$ at each row is positive semidefinite, so the full Hessian is positive semidefinite and the categorical cross-entropy is convex in the stacked coefficients, exactly as in the binary model. Newton and iteratively reweighted least squares extend directly, at a cost that grows with the number of classes because the coupled system spans all $K$ score blocks.

    Reference-Class Identifiability

    The shift invariance that stabilizes the softmax also makes the parameterization redundant. Adding any common vector $v$ to every class coefficient, $\beta_k\mapsto\beta_k+v$, shifts every score by the same $v^\top\tilde{x}$ and leaves all probabilities unchanged, so the likelihood cannot distinguish the two parameterizations and the model is overparameterized by one full coefficient vector. The estimate is identified only up to that redundancy, which appears as a singular Hessian if left unaddressed.

    Two conventions remove it. Fixing a reference class, typically $\beta_K=0$, expresses the remaining coefficients as log-odds contrasts against that class and recovers exactly binary logistic regression when $K=2$, since the single free vector $\beta_1$ becomes the binary coefficient. Alternatively, an L2 penalty on all class coefficients breaks the redundancy by selecting the symmetric minimum-norm representative, which is why regularized softmax implementations can keep all $K$ vectors. A numeric check confirms the invariance: on a three-class model, recentering every class vector by subtracting the third class's vector leaves the fitted probabilities identical to the full parameterization, so the reference choice is a labeling convention, not a modeling assumption.

    Decision Regions

    The predicted class is the one with the largest score, and because the softmax is monotone in the scores, $\arg\max_k p_k=\arg\max_k z_k$; the normalization never changes the decision. The boundary between two classes $k$ and $l$ is where their scores tie, $\beta_k^\top\tilde{x}=\beta_l^\top\tilde{x}$, a hyperplane, so the feature space is carved into convex polyhedral regions with piecewise-linear boundaries meeting at junctions.     Calibration and Cost-Sensitive Decisions

    The softmax returns a normalized probability vector, but normalized is not the same as calibrated: the predicted probabilities can still systematically over- or under-state the true class frequencies, and multiclass calibration is checked per class or with a multiclass reliability analysis rather than assumed. As in the binary case, probability estimation and decision-making are separate layers. The argmax rule is optimal only under symmetric, equal misclassification costs. When costs differ, the decision minimizes expected cost,

    $$\hat{c}(x)=\arg\min_{a}\sum_{k=1}^{K}C(a,k)\,\Pr(Y=k\mid x),$$

    choosing the action $a$ with the lowest expected cost against a cost matrix $C(a,k)$, which can differ from the most probable class whenever a rare class carries a heavy penalty for being missed. Keeping the probability layer calibrated is what makes this expected-cost calculation trustworthy; an uncalibrated model can rank classes correctly yet compute the wrong expected costs.

    Edge Cases and the Implementation

    • Overparameterization. Keeping all $K$ coefficient vectors without a reference class or a penalty leaves a singular Hessian; fix a reference or regularize.
    • Overflow. A softmax without the max-shift overflows on large scores; the log-sum-exp form in the loss must be used, not a probability-first cross-entropy.
    • Separation persists. A class perfectly separated from the rest sends its coefficients to infinity exactly as in the binary case, and the same penalty is the cure.
    • One-vs-rest probabilities. Combining independently trained binary models requires renormalization and yields less coherent probabilities than a joint softmax fit.
