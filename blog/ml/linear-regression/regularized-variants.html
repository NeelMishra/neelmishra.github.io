    Regularized Linear Regression Variants

    July 2026 · 17 min read

    Regularization adds a penalty to the least-squares objective. The penalty deliberately biases coefficients toward simpler solutions, often reducing variance and improving test performance.

    VariantObjectiveEffectBest when
    Ridge$\text{SSE}+\lambda\sum_j w_j^2$Shrinks coefficients smoothly toward zero.Features are correlated and all may carry signal.
    Lasso$\text{SSE}+\lambda\sum_j |w_j|$Can set coefficients exactly to zero.Feature selection matters and a sparse model is desired.
    Elastic Net$\text{SSE}+\lambda_1\sum_j |w_j|+\lambda_2\sum_j w_j^2$Combines sparsity with grouped shrinkage.There are many correlated features and sparse selection is still useful.

    Geometry

    Ridge uses a round penalty ball, so coefficients shrink continuously. Lasso uses a diamond-shaped penalty, whose corners make exact zeros more likely. Elastic Net blends both shapes.

    Always scale first. Ridge and Lasso penalize coefficient size. If features are on different scales, the penalty is not comparable across features.

    Choosing Lambda

    The regularization strength $\lambda$ is a hyperparameter. It is usually chosen with validation or cross-validation. Too little regularization behaves like ordinary least squares. Too much regularization underfits by forcing useful coefficients too close to zero.
