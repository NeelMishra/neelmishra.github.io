    Linear Regression: Foundations

    July 2026 · 20 min read

    Linear regression predicts a numeric target by adding weighted feature values. It is simple enough to solve exactly, but deep enough to introduce the core language of supervised learning: model, loss, residual, optimization, assumptions, regularization, and diagnostics. It is also the baseline that more complex models must beat, and every idea in this post reappears, in a sharper form, later in machine learning: penalized loss in ridge regression is the same shape as weight decay in a neural network, and the normal equations here are the same projection idea behind kernel ridge regression.

    The Model

    For one feature, the model is a line:

    $$\hat y = b + wx.$$

    For many features, it is a weighted sum:

    $$\hat y = b + w_1x_1 + w_2x_2 + \cdots + w_dx_d = b + w^\top x.$$

    The word "linear" refers to linearity in the parameters $b$ and $w$, not necessarily to the raw inputs. This one distinction is the difference between four models that people often treat as unrelated:

    Simple regression

    One feature, one weight: $\hat y = b + wx$. Easiest to visualize and to build intuition on.

    Multiple regression

    Several raw features $x_1,\dots,x_d$, one weight per feature: $\hat y = b + w^\top x$. Coefficients become partial effects, holding other features fixed.

    Polynomial regression

    One raw input $t$, features are its powers $t, t^2, t^3,\dots$. Curved in $t$, but still a weighted sum of features, so still linear regression.

    Basis expansion

    Features are any fixed functions of the input, $\phi_1(t),\dots,\phi_k(t)$: logs, indicators, splines, sinusoids. Same linear machinery, richer inputs.

    Concretely, suppose $t$ is a house's floor area. Simple regression fits $\hat y = b + w_1 t$. Multiple regression adds an unrelated feature such as the building's age: $\hat y = b + w_1 t + w_2\,\text{age}$. Polynomial regression instead lets the single feature $t$ curve, by adding $t^2$ as a second column: $\hat y = b + w_1 t + w_2 t^2$, which can bend to fit diminishing returns on very large houses. Basis expansion generalizes the same trick to non-polynomial shapes, for example $\hat y = b + w_1\log t + w_2\sin(2\pi t/365)$ for a feature with a yearly cycle. In every case the model is still $\hat y = b + w^\top x$ once $x$ is understood as "the feature vector actually fed to the model," which may be engineered from the raw inputs. This is why the matrix form in Least Squares Geometry can treat simple, multiple, polynomial, and basis-expanded regression as exactly the same optimization problem: only the columns of the design matrix change.

    Residuals and Least Squares

    The residual for row $i$ is $e_i = y_i-\hat y_i$, the signed gap between what happened and what the model predicted. Ordinary least squares (OLS) chooses parameters that minimize the sum of squared residuals:

    $$\min_{b,w}\sum_i (y_i-b-w^\top x_i)^2.$$

    Squaring serves three purposes at once: it makes every term non-negative so residuals cannot cancel out by sign, it makes large errors disproportionately expensive (a residual of 4 contributes 16 times more than a residual of 1), and it gives a smooth, differentiable objective with a closed-form solution when the features carry enough independent information. That last property is not automatic. It fails when features are redundant, and Part 2 makes precise what "enough independent information" means. Squaring large errors expensive also has a cost: a single wildly wrong observation can dominate the objective and drag the fitted line toward it. Part 5 covers losses built specifically to resist that.

    Why Linear Regression Still Matters

    • Interpretability: each coefficient is a marginal effect under the fitted linear model, in the original units of the feature.
    • Optimization: the OLS objective is a convex quadratic, so every stationary point is a global minimum and there are never bad local minima to get stuck in. The minimizer is unique only when the design matrix has full column rank; under rank deficiency (e.g. duplicated or perfectly collinear features) the minimum value is still unique, but the set of minimizing coefficient vectors becomes an entire affine subspace rather than a single point.
    • Diagnostics: residual plots expose nonlinearity, outliers, heteroscedasticity, and leakage, because the model is simple enough that failures are visible rather than hidden inside a black box.
    • Baseline value: if a complex model only barely beats linear regression, the extra complexity, latency, and opacity may not be worth it. Linear regression is the control group every other regression model is tested against.

    What the Rest of This Series Covers

    Ordinary least squares answers "what is the best line under squared error, with no constraints." Every other post in this series changes one of those three clauses: the geometry behind the solution, how it is computed, what counts as a penalty on the coefficients, or what counts as an error in the first place.

    • The dedicated OLS deep dive: seven focused posts move from the closed-form line through multiple regression and FWL, Gauss-Markov guarantees, inference, reliable solvers, leverage, and influence diagnostics.
    • The geometry (Part 2): projection, normal equations, rank, multicollinearity, and weighted least squares, which reweights each row's contribution to the loss instead of treating every observation equally.
    • The computation (Part 3): gradient descent and its batch, stochastic, and mini-batch forms, plus the online and streaming setting where data arrives one row at a time.
    • The penalty (Part 4): ridge, lasso, and elastic net, which trade a little bias for a lot less variance, and their Bayesian reading as priors on the coefficients.
    • The loss (Part 5): least absolute deviations, Huber regression, and quantile regression, for when the mean is not the quantity worth predicting or outliers should not dominate.
    • The checks (Part 6): the assumptions behind valid inference, and the diagnostic plots that reveal when they break.
