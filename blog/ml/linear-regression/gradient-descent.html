    Gradient Descent for Linear Regression

    July 2026 · 16 min read

    Closed-form least squares is elegant, but iterative optimization is often more practical for large datasets, streaming data, sparse features, and regularized variants. Gradient descent updates weights by moving opposite the slope of the loss.

    Gradient

    Using mean squared error,

    $$J(w)=\frac{1}{2n}\|Xw-y\|^2,$$

    the gradient is

    $$\nabla J(w)=\frac{1}{n}X^\top(Xw-y).$$

    The update is $w \leftarrow w-\eta\nabla J(w)$, where $\eta$ is the learning rate.

    ▶ Walking Down the Loss Surface

    Each step moves the parameters downhill. Too small is slow; too large can overshoot.

    Batch, SGD, and Mini-Batch

    • Batch gradient descent: uses all rows for each update. It is stable but can be slow.
    • Stochastic gradient descent: uses one row at a time. It is noisy but cheap per update.
    • Mini-batch gradient descent: uses a small batch. It balances noise and efficiency.

    Feature Scaling

    Gradient descent is sensitive to feature scale. If one feature is measured in dollars and another in years, the loss surface can be stretched into a narrow valley. Standardization makes the valley rounder and updates more stable.

    Rule of thumb. If an iterative linear model is slow or unstable, scale features before tuning the learning rate.
