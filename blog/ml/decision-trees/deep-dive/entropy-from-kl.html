    Where Entropy Comes From: KL Divergence

    June 2026 · 15 min read

    Part 2 stated entropy as a definition: for a class distribution $p = (p_1,\ldots,p_K)$, $$H(p) = -\sum_{i=1}^{K} p_i \log_2 p_i.$$ That formula is useful for scoring decision-tree nodes, but it can look arbitrary on first contact. Why should $-p_i\log p_i$ be the right contribution of class $i$? Why should a uniform node be maximally impure? Why should a pure node have zero uncertainty?

    This part shows that entropy is not a standalone trick invented for trees. It falls out of the Kullback-Leibler divergence, usually abbreviated KL divergence, which is the fundamental measure of how far one probability distribution is from another. Once KL is interpreted as extra coding cost, entropy appears as the best possible coding cost of a distribution, cross-entropy appears as the cost of using the wrong code, and information gain becomes a precise statement about reducing avoidable uncertainty inside a tree.

    KL Divergence

    Let $p$ and $q$ be two discrete probability distributions over the same $K$ categories. The KL divergence from $q$ to $p$ is

    $$D_{\mathrm{KL}}(p \,\|\, q) = \sum_{i=1}^{K} p_i \log_2 \frac{p_i}{q_i}.$$

    The notation $D_{\mathrm{KL}}(p \,\|\, q)$ should be read carefully. Samples are assumed to come from $p$, while the comparison distribution or coding model is $q$. The expectation is taken under $p$, so outcomes that are common under $p$ matter more than outcomes that are rare under $p$.

    The term for category $i$ is $p_i\log_2(p_i/q_i)$. If $p_i$ is larger than $q_i$, then $q$ underestimates that category and the logarithm is positive. If $p_i$ is smaller than $q_i$, then $q$ overestimates that category and the logarithm is negative. Individual terms can be negative, but the weighted sum cannot be negative.

    • Non-negativity. $D_{\mathrm{KL}}(p \,\|\, q) \ge 0$. This is Gibbs' inequality.
    • Zero only on equality. $D_{\mathrm{KL}}(p \,\|\, q) = 0$ if and only if $p=q$ for all categories with probability mass.
    • ASYMMETRY. In general, $D_{\mathrm{KL}}(p \,\|\, q) \ne D_{\mathrm{KL}}(q \,\|\, p)$. KL is a divergence, not a metric. It does not behave like distance, and it does not satisfy symmetry.

    The most important interpretation is coding cost. An optimal prefix code for distribution $q$ assigns roughly $-\log_2 q_i$ bits to category $i$. If the real data is generated by $p$, the expected length of that $q$-optimized code is $-\sum_i p_i\log_2 q_i$. The optimal code for the true distribution $p$ would cost $-\sum_i p_i\log_2 p_i$. Their difference is exactly KL divergence:

    $$\sum_i p_i\log_2\frac{p_i}{q_i}.$$

    So $D_{\mathrm{KL}}(p \,\|\, q)$ is the expected extra number of bits needed to encode samples from $p$ when using a code optimized for $q$ instead of a code optimized for $p$.

    ▶ KL Divergence, Term by Term

    Each step highlights one category, adds $p_i\log_2(p_i/q_i)$, and shows how the running divergence accumulates against a uniform reference distribution.

    Swapping the roles of $p$ and $q$ gives a different value because the expectation changes and the logarithmic penalty changes direction. With $q$ uniform over $K$ classes, the relationship becomes especially revealing: $$D_{\mathrm{KL}}(p\|\text{uniform}) = \log_2 K - H(p).$$ Therefore maximizing divergence from uniform is the same as minimizing entropy. A node becomes less uncertain precisely when its class distribution moves away from the uniform distribution and toward a concentrated distribution.

    Cross-Entropy = Entropy + KL

    Cross-entropy measures the expected code length when samples come from $p$ but the code is optimized for $q$:

    $$H(p,q) = -\sum_{i=1}^{K} p_i \log_2 q_i.$$

    It decomposes into the entropy of the true distribution plus the KL penalty for using $q$:

    $$\begin{aligned} H(p,q) &= -\sum_i p_i\log_2 q_i \\ &= -\sum_i p_i\log_2 p_i + \sum_i p_i\log_2 p_i - \sum_i p_i\log_2 q_i \\ &= -\sum_i p_i \log_2 p_i + \sum_i p_i\log_2\frac{p_i}{q_i} \\ &= H(p) + D_{\mathrm{KL}}(p\|q). \end{aligned}$$

    The algebra adds and subtracts the same quantity, $\sum_i p_i\log_2 p_i$, then groups the result into two interpretable pieces. The first piece, $H(p)$, depends only on the true data-generating distribution. The second piece, $D_{\mathrm{KL}}(p\|q)$, is the avoidable cost caused by using a mismatched distribution.

    Entropy is therefore the special case of cross-entropy where the model distribution is correct: $$H(p) = H(p,p).$$ It is the irreducible expected cost of encoding $p$ using its own optimal code. KL is the penalty for using the wrong code $q$. Cross-entropy can never be lower than entropy because KL is non-negative, so entropy is the floor of cross-entropy.

    Back to Trees

    At a classification node, the empirical class frequencies define a distribution $p$. A pure node has a one-hot distribution such as $(1,0,0,\ldots,0)$, so its entropy is $0$. There is no uncertainty left: once a sample reaches that node, the class is known from the training data at that node. A perfectly balanced $K$-class node has $p_i=1/K$ for every class, so it has entropy $\log_2 K$, the maximum possible value.

    In this sense, entropy measures how far the node is from certainty and how close it is to uniform confusion. A split is valuable when it replaces one mixed parent distribution with children whose weighted average entropy is smaller. That is exactly the information gain from Part 2:

    $$\mathrm{Gain} = H(p_{\text{parent}}) - \sum_c \frac{n_c}{n}H(p_c).$$

    Because $D_{\mathrm{KL}}(p\|\text{uniform}) = \log_2 K - H(p)$, lowering child entropy is equivalent to driving each child distribution toward larger KL divergence from uniform. In ordinary tree language, the split tries to make the children purer. In information-theoretic language, it tries to move each child away from uniform uncertainty and toward a one-hot distribution.

    Key insight. Information gain is not just a heuristic score. It is the reduction in irreducible coding cost inside the children, equivalently a push away from the uniform distribution and toward certainty.

    The same decomposition also connects trees to the broader loss functions used across machine learning. Training a probabilistic classifier with cross-entropy loss minimizes $H(\text{true},\text{pred})$. Since the entropy of the true label distribution is fixed with respect to the model predictions, minimizing cross-entropy is the same as minimizing $D_{\mathrm{KL}}(\text{true}\|\text{pred})$. Decision-tree impurity and neural-network cross-entropy therefore share the same mathematical backbone: both care about reducing the penalty for representing the true distribution with the wrong or too-diffuse distribution.

    Closing

    Entropy began as the impurity formula used to score decision-tree nodes, but KL divergence explains why that formula is natural. KL measures the extra bits paid for using a mismatched distribution, cross-entropy splits into entropy plus that KL penalty, and entropy itself is the best achievable cross-entropy when the code matches the data. For a tree, reducing entropy means making child nodes more certain, which is the same direction as increasing their divergence from uniform confusion.
