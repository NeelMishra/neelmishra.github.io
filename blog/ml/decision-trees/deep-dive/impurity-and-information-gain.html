    Impurity & Information Gain

    June 2026 · 16 min read

    A split is good when it makes the children purer than the parent. If the parent contains a mixture of classes and each child contains a more concentrated class distribution, the split has exposed structure that a leaf prediction can use. To compare candidate splits, however, the training algorithm needs a number for "impurity": a scalar summary of how mixed a node is. This formalizes the greedy score introduced in Part 1, where each feature-threshold pair was evaluated by the reduction in weighted impurity after the split.

    Three Impurity Measures

    Consider a classification node with $K$ possible classes. Let $p_k$ be the proportion of training examples in the node that belong to class $k$, so $p_k \ge 0$ and $\sum_k p_k = 1$. An impurity measure maps the vector $(p_1,\ldots,p_K)$ to a nonnegative number. The number should be small when one class dominates and large when the classes are evenly mixed.

    • Entropy: $$H = -\sum_k p_k \log_2 p_k.$$ Entropy measures the uncertainty of the class label in bits. A pure node has no uncertainty because the label is known once the node is reached. A balanced node has the most uncertainty because no class is preferred by the local distribution.
    • Gini impurity: $$G = \sum_k p_k(1-p_k) = 1 - \sum_k p_k^2.$$ Gini impurity can be read as the probability of making a mistake if a label is drawn according to the node distribution and then a second label is guessed independently from the same distribution. It avoids the logarithm, so it is cheaper to compute than entropy.
    • Misclassification error: $$E = 1 - \max_k p_k.$$ Misclassification error is the error rate of a leaf that predicts the majority class at the node. It is intuitive, but it reacts only when the identity or size of the majority class changes enough to alter that majority error.

    Why Gini Is a Mistake Probability

    The probability interpretation comes from a small two-draw experiment at a node. First draw the true label $Y$ from the node's empirical class distribution, so class $k$ appears with probability $p_k$. Then make an independent randomized guess $\hat{Y}$ from that same distribution, so the guess is also class $k$ with probability $p_k$. The guess is correct exactly when both draws choose the same class. For one class $k$, that joint probability is $p_k \cdot p_k = p_k^2$. Summing over all classes gives the probability of a correct randomized guess:

    $$P(\hat{Y}=Y)=\sum_k p_k^2.$$

    The mistake probability is therefore the complement:

    $$P(\hat{Y}\ne Y)=1-\sum_k p_k^2.$$

    That is exactly the Gini formula. The alternate form follows by expanding the sum:

    $$\sum_k p_k(1-p_k)=\sum_k p_k-\sum_k p_k^2=1-\sum_k p_k^2.$$

    For a concrete binary node with proportions $(0.7,0.3)$, the randomized guess is wrong in two ways: the true label is class 1 and the guess is class 2, or the true label is class 2 and the guess is class 1. The probability is $0.7\cdot0.3 + 0.3\cdot0.7 = 0.42$, which matches $2p(1-p)$. Notice that this is not the same as majority-vote error, which would be $0.3$. Gini measures how often two independent labels sampled from the node disagree; majority-vote error measures how often the single most common label would be wrong.

    The computational point is also literal. Entropy needs a logarithm for each nonzero class probability, while Gini needs only squaring, addition, and one subtraction. A tree builder evaluates thousands or millions of candidate splits, and the class counts on the left and right side of a threshold are updated repeatedly during sorted sweeps. Computing $1-\sum_k p_k^2$ from those counts is cheap and numerically straightforward, which is why CART-style trees commonly use Gini even though entropy often gives very similar split rankings.

    All three measures are $0$ for a pure node, where one $p_k=1$ and all other proportions are $0$. All three are maximal at the uniform distribution, where every class is equally likely. For two classes with proportions $p$ and $1-p$, entropy is

    $$H(p) = -p\log_2 p - (1-p)\log_2(1-p),$$

    Gini is

    $$G(p) = p(1-p) + (1-p)p = 2p(1-p),$$

    and misclassification error is

    $$E(p) = 1 - \max(p,1-p) = \min(p,1-p).$$

    At $p=0.5$, the binary node is maximally mixed. Entropy peaks at $1$ bit, Gini peaks at $0.5$, and misclassification error peaks at $0.5$. At $p=0$ and $p=1$, the node is pure and every measure returns $0$.

    ▶ Entropy, Gini & Misclassification vs p

    For a two-class node, the horizontal axis is $p$, the proportion of class 1. The curves show how each impurity measure rises as the node becomes mixed, then falls back to zero as the other class becomes pure.

    Entropy and Gini track each other closely: both are smooth, strictly concave for the binary case, and most sensitive around balanced mixtures. Misclassification error is piecewise-linear and flatter. That flatness makes it a poor choice for growing a tree, since it often sees zero gain for a split that clearly improves purity under entropy or Gini. It is still useful for pruning, where the question is closer to final prediction error than local split sensitivity.

    Information Gain

    Once impurity is defined, the split score is the impurity reduction. For a node with $N$ points and children indexed by $c$, with $N_c$ points reaching child $c$, the gain is

    $$\Delta\text{IG} = I(\text{parent}) - \frac{1}{N}\sum_{c} N_c\, I(\text{child}_c),$$

    where $I$ can be entropy, Gini, or another impurity function. With entropy, this is usually called information gain. With Gini, the same formula is often called Gini gain or impurity decrease. The tree-growing algorithm evaluates every candidate split at the current node and picks the split with the largest gain. The split is accepted because it produces the greatest immediate drop in weighted child impurity.

    A small binary example shows the arithmetic. Suppose the parent node has $4$ examples of class A and $4$ examples of class B. Its class proportions are $(1/2,1/2)$, so the parent entropy is

    $$H(\text{parent}) = -\frac{1}{2}\log_2\frac{1}{2} - \frac{1}{2}\log_2\frac{1}{2} = 1 \text{ bit}.$$

    Now consider a split into a left child with $\{3A,1B\}$ and a right child with $\{1A,3B\}$. Each child has proportions $(3/4,1/4)$ up to order, so each child entropy is

    $$H(3/4,1/4) = -\frac{3}{4}\log_2\frac{3}{4} - \frac{1}{4}\log_2\frac{1}{4} \approx 0.811.$$

    Both children contain $4$ of the $8$ parent points, so the weighted child entropy is

    $$\frac{4}{8}(0.811) + \frac{4}{8}(0.811) = 0.811.$$

    The information gain is therefore

    $$\Delta\text{IG} = 1 - 0.811 = 0.189 \text{ bits}.$$

    The split is better than doing nothing because the children are less mixed than the parent, but it is not a perfect split because both children still contain both classes.

    Which Measure, and Why

    Entropy and Gini usually produce almost identical trees. They have the same qualitative shape, they both reward purity, and they both prefer splits that isolate concentrated class distributions. Differences appear only when candidate splits are extremely close in score, and such differences are often smaller than the variance caused by small changes in the training sample.

    Measure Formula What it emphasizes Best when Main caution
    Entropy $-\sum_k p_k\log_2 p_k$ Expected surprise, measured in bits. It is very sensitive to changes in mixed class distributions. Use it when the information-theoretic interpretation matters, when explaining ID3/C4.5, or when using information gain and gain ratio. It requires logarithms and plain information gain can favor features with many possible split outcomes.
    Gini impurity $1-\sum_k p_k^2$ Probability that two independent labels sampled from the node distribution disagree. Use it as the practical default for CART-style classification trees, especially when many candidate splits must be scored quickly. It is not the majority-class error. It measures randomized disagreement, so its value can be larger than the leaf's final prediction error.
    Misclassification error $1-\max_k p_k$ Error rate of a leaf that predicts the majority class. Use it for pruning, reporting leaf error, or reasoning about the final class decision after the tree structure already exists. It is too flat for growing trees because many useful splits do not change the majority class enough to change the score.

    As a rule of thumb, Gini is the best default for growing CART-style classification trees because it is fast and usually chooses splits very close to entropy. Choose entropy when the coding and information-gain interpretation is part of the model story, or when following ID3/C4.5. Reserve misclassification error for pruning or evaluation, not for the greedy split search.

    Gini is the default criterion in CART-style classification trees because it is cheaper: no logarithm is required, only sums of squared class proportions. Entropy is more directly connected to coding, uncertainty, and information theory, so it is common in ID3 and C4.5 presentations. Misclassification error is generally too insensitive for growing trees because it ignores many distributional improvements that do not change the majority-class error. It is more natural after a tree has already been grown, especially in pruning criteria that care about leaf prediction error.

    When Entropy Is Still Used

    If the goal is simply to grow a strong CART-style classifier quickly, Gini usually wins the engineering argument: it is cheaper and its split rankings are often almost the same as entropy's. Entropy is still used when the objective being explained or optimized is explicitly information-theoretic:

    • ID3 and C4.5: these algorithms are defined around information gain, and C4.5's gain ratio is an entropy-based correction for many-valued split bias.
    • Communication and coding interpretation: entropy says how many bits are needed, on average, to encode the class label after reaching a node. This makes a split's gain easy to read as saved bits.
    • Log-loss and probability modeling: cross-entropy, KL divergence, and calibration all live in the same family. If the surrounding model story is about predicted probabilities, entropy keeps the tree criterion aligned with that language.
    • Teaching and derivation: entropy has a deeper derivation from expected surprise and KL divergence, while Gini is often presented as a fast impurity surrogate with similar behavior.

    A concrete disagreement shows the tradeoff. Suppose a binary parent node has $4$ examples of class A and $8$ examples of class B. Two candidate splits are:

    Candidate Children Entropy gain Gini gain Who prefers it?
    Split 1 Left $(0A,5B)$, right $(4A,3B)$ $0.344$ bits $0.159$ Entropy
    Split 2 Left $(3A,1B)$, right $(1A,7B)$ $0.286$ bits $0.174$ Gini

    Entropy chooses Split 1 because it creates one perfectly pure child and gives a larger reduction in expected code length. Gini chooses Split 2 because it lowers the randomized label-disagreement probability more. This is not a universal "Gini failure" for classification accuracy; it is a criterion mismatch. If saved bits, log-loss alignment, or information gain is the target, Gini can pick the wrong split for that target. If the target is ordinary CART tree growth under a cheap impurity surrogate, there is no broad practical defeat of Gini to point to. In that setting, entropy mostly loses on cost and usually does not buy enough different behavior to justify itself.

    Bias toward many-valued splits. Plain information gain tends to favor features with many possible split outcomes, since such features can fragment the data into small, artificially pure children. C4.5 addresses this with gain ratio, which normalizes information gain by the intrinsic information of the split. The normalization penalizes splits that create many branches without providing a proportionate reduction in class uncertainty.
