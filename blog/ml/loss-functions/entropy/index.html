    Entropy Foundations

    July 2026 · 18 min read

    Entropy measures uncertainty in a probability distribution. In machine learning it appears in two closely related roles: as an impurity measure for distributions over labels, and as the core ingredient of cross-entropy losses for probabilistic classifiers. The cleanest starting point is not trees or neural networks, but surprise.

    Surprise of One Outcome

    An outcome with probability $p$ carries

    $$I(p) = -\log_2 p$$

    bits of information. A certain event with $p=1$ has $0$ bits of surprise. A fair coin outcome has $1$ bit. An event with probability $1/8$ has $3$ bits, because three yes/no questions can identify one item among eight equally likely possibilities.

    Entropy as Expected Surprise

    A distribution has many possible outcomes, so entropy averages the surprise over all outcomes:

    $$H(P) = -\sum_{i=1}^{K} p_i \log_2 p_i.$$

    The weights are the probabilities themselves. Rare events are highly surprising when they occur, but they contribute less often. Common events are less surprising, but they contribute often. Entropy balances both effects.

    Why Uniform Has Maximum Entropy

    If all $K$ outcomes are equally likely, then $p_i=1/K$ and

    $$H(P) = -\sum_{i=1}^{K}\frac{1}{K}\log_2\frac{1}{K} = \log_2 K.$$

    No outcome is predictable, so uncertainty is as large as possible. Any skew in the distribution makes some outcomes easier to guess, which lowers the average surprise.

    Zero terms are harmless. The convention is $0\log 0=0$, defined by the limit $\lim_{p\to 0^+}p\log p=0$. An impossible event contributes nothing because it never occurs.

    Machine-Learning Connection

    In decision trees, entropy measures how mixed the labels are at a node. In classifiers, cross-entropy measures how many bits are wasted when predicted probabilities do not match the true distribution. Both uses come from the same formula: probability-weighted surprise.

    entropy(probabilities):
    total = 0
    for p in probabilities:
        if p > 0:
            total += -p * log2(p)
    return total
