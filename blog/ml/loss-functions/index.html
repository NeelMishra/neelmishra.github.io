    Popular Loss Functions

    July 2026 · Series overview

    A loss function is the objective that a learning algorithm actually optimizes. Accuracy, F1, AUC, RMSE, and calibration error may be the metrics reported to stakeholders, but gradient-based training needs a differentiable signal. The loss supplies that signal: it converts a prediction and a target into a number whose gradient says how to change the model.

    This series collects the most common loss criteria used in machine learning and explains why each one has the shape it does. The first sub-series covers entropy, because entropy sits underneath decision-tree impurity, cross-entropy classification loss, KL divergence, maximum-entropy modeling, calibration, and information gain.

    Loss versus Metric

    A metric answers the evaluation question: how good is the final model for the task? A loss answers the training question: how should the model be updated after one batch? The two are related but not identical. Accuracy is easy to understand, but it is flat almost everywhere, so it does not provide useful gradients. Cross-entropy is less directly interpretable to non-specialists, but it changes smoothly as probabilities improve or degrade.

    Training needs shape. A useful loss is not just low when predictions are good. It must also have a shape that gives meaningful learning signals before predictions are good.

    Beyond Entropy

    The same framework can later expand to squared error, absolute error, Huber loss, hinge loss, focal loss, ranking losses, contrastive losses, and likelihood-based objectives. Entropy comes first because it connects the broadest set of ideas: probability, coding, uncertainty, classification, trees, and model calibration.
