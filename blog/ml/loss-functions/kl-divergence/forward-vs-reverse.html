    Forward KL versus Reverse KL

    July 2026 · 15 min read

    The direction of KL changes the behavior. Forward KL, $D_{KL}(P\|Q)$, averages under the true distribution $P$. If $P$ puts mass somewhere and $Q$ assigns almost none, the penalty is large. This makes forward KL mode covering: the model is pushed to cover every region where the data appears.

    Reverse KL, $D_{KL}(Q\|P)$, averages under the model distribution $Q$. If $Q$ avoids a region, it pays no cost there. This makes reverse KL more mode seeking: it may focus on one plausible mode rather than spreading mass across all modes.

    Example

    If a true distribution has two separated clusters and the model family can represent only one Gaussian, forward KL tends to place a wide Gaussian covering both clusters. Reverse KL tends to pick one cluster and fit it sharply. Neither behavior is universally better; each matches a different approximation question.

    Where It Appears

    • Maximum likelihood and cross-entropy: often behaves like minimizing forward KL from data to model.
    • Variational inference: common ELBO derivations use reverse KL from approximate posterior to true posterior.
    • Policy optimization: KL penalties constrain how far a new policy moves from an old policy.
