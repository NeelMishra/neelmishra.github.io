    Tree SHAP for Ensembles and Global Summaries

    July 2026 · 16 min read

    Tree ensembles add predictions from many trees. Tree SHAP follows the same additivity: compute attributions for each tree, then sum attributions feature by feature. This is why Tree SHAP works naturally for random forests and gradient boosted trees.

    Additivity Across Trees

    If an ensemble prediction is the sum of tree predictions, then each feature's SHAP value is the sum of that feature's SHAP value from each tree. The baseline is also the sum of tree baselines. The row-level identity remains intact:

    ensemble prediction = ensemble baseline + summed feature contributions

    ▶ Add Local Contributions Across Trees

    Each tree contributes a small attribution vector for the row. The ensemble explanation is their feature-wise sum.

    From Local to Global

    A global SHAP importance score is usually computed as the mean absolute SHAP value for each feature over many rows. That means the global plot is not a separate explanation method. It is a summary of row-level explanations.

    • Local force plot: one row, signed contributions.
    • Global bar plot: average contribution magnitude per feature.
    • Beeswarm plot: many row-level contributions shown together.
    • Dependence plot: one feature's value versus its SHAP contribution.
    Always keep the row-level source in mind. If a global plot says a feature is important, that means the feature often has large local attributions. It does not mean the feature always pushes in the same direction.
