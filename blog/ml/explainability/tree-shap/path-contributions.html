    Tree SHAP Path Contributions

    July 2026 · 18 min read

    A tree path is a sequence of feature tests. For one row, the model knows which branch was actually taken at each test. Tree SHAP asks a more subtle question: how much did each feature on that path change the expected output when it became known?

    Conditional Expectations Along a Tree

    At the root, before any row-specific feature is known, the expected output is the training-weighted average over leaves. After a split feature is known, the row moves into one branch and the expected output changes. Tree SHAP distributes these expectation changes fairly across all possible orders in which path features could have become known.

    ▶ Expectations Narrow Along the Path

    The baseline averages all leaves. Each known split restricts the reachable leaves and changes the expected output.

    Path Fractions

    Exact Tree SHAP tracks how much probability mass flows down the hot branch taken by the row and the cold branch not taken by the row. In tree-path-dependent mode, those fractions come from training sample counts in the tree. In interventional mode, absent features are integrated over a background dataset.

    Not Just Features on One Path

    For a single tree, only features tested on reachable paths can affect the row's leaf. In an ensemble, different trees test different features. The final row-level explanation is the sum of contributions across all trees.

    Path awareness is the advantage. Generic explainers ask the model many artificial questions. Tree SHAP uses the branch structure to compute how knowing features changes the expected leaf value.
