    Why Tree SHAP Beats Generic Explainers for Trees

    July 2026 · 17 min read

    For tree models, Tree SHAP has a structural advantage. A decision tree already records how a row moves through a sequence of feature tests. Tree SHAP uses that structure directly. LIME and Kernel SHAP treat the model as a black box and must rediscover local behavior by perturbing or masking inputs.

    Comparison

    Explainer For one tree-model row Strength Weakness
    LIME Samples nearby rows and fits a local surrogate. Simple and model-agnostic. Approximate, sensitive to kernel width and perturbation design.
    Kernel SHAP Samples feature coalitions and fits weighted regression. SHAP semantics for arbitrary black-box models. Expensive and noisy unless many coalitions are sampled.
    Global SHAP summary Aggregates row-level SHAP values across many rows. Shows broad feature importance and distribution of effects. Does not itself explain one row unless the local values are shown.
    Tree SHAP Uses tree paths to compute row-level SHAP values. Exact and fast for supported tree models. Specialized to tree models and still depends on background semantics.

    ▶ Generic Sampling versus Tree Paths

    Generic explainers sample around the model. Tree SHAP follows the tree structure that the model actually uses for the row.

    Why Global SHAP Is Not the Rival

    A global SHAP bar chart usually plots the mean absolute SHAP value for each feature across a dataset. That chart is useful, but it is made from local explanations. Tree SHAP is the row-level engine. Global SHAP is an aggregation view built on top of row-level attributions.

    The practical recommendation. For random forests, gradient boosted trees, and other tree ensembles, use Tree SHAP for row-level explanations. Aggregate those local values when a global summary is needed. Use LIME or Kernel SHAP when the model is not a tree or the tree internals are unavailable.
