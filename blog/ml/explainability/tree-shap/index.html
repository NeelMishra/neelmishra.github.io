    Tree SHAP: Row-Level Explanations for Tree Models

    July 2026 · 18 min read

    Tree SHAP is the specialized SHAP algorithm for decision trees, random forests, and gradient boosted tree ensembles. It explains one row at a time by decomposing the row's prediction into a baseline plus feature contributions. For tree models, it is usually preferable to LIME and generic Kernel SHAP because it uses the actual tree paths instead of fitting an approximate local surrogate or sampling coalitions blindly.

    The important correction is that global SHAP is not a competing explainer. Global SHAP plots are aggregations of row-level SHAP values. Tree SHAP produces the row-level attributions; global bar plots, beeswarm plots, and dependence plots summarize those local attributions across a dataset.

    Row-Level Additive Explanation

    For one row, Tree SHAP writes the tree-model prediction as

    prediction = baseline + feature_1 contribution + feature_2 contribution + ...

    The baseline is the expected model output over a background distribution. Each contribution says how much knowing that row's feature value moved the prediction away from the baseline.

    ▶ From Tree Path to Row-Level Contributions

    A row walks down a tree. Each split narrows the expected prediction, and Tree SHAP distributes the final prediction difference back to the features on the path.

    Why Tree SHAP Is Better for Trees

    • Exact for supported tree semantics: it computes SHAP values from tree structure instead of approximating with a local line.
    • Fast for tree ensembles: it avoids brute-force coalition enumeration and exploits path structure.
    • Row-level by design: every explanation is for a specific row, then global summaries are built by aggregation.
    • Faithful to the trained model: the explanation follows the same splits the tree model actually uses.
    Tree SHAP beats LIME and Kernel SHAP only in the tree-model setting. LIME is still useful for arbitrary black boxes. Kernel SHAP is still the model-agnostic SHAP fallback. But when the model is a tree ensemble, Tree SHAP is the specialized tool that is faster, less noisy, and more faithful.
