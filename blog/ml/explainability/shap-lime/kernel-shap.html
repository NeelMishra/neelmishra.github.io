    Kernel SHAP: SHAP as Weighted Regression

    July 2026 · 18 min read

    Kernel SHAP estimates Shapley values by turning feature coalitions into a weighted linear regression problem. Each coalition is represented by a binary mask: feature present or feature absent. The model is queried on rows built from those masks, and the weighted regression coefficients estimate SHAP attributions.

    The SHAP Kernel

    For $M$ features and a coalition size $s$, the Kernel SHAP weight is

    $$w(s)=\frac{M-1}{\binom{M}{s}s(M-s)}.$$

    Coalitions with very small or very large size receive high weight because they anchor the baseline and the full prediction. Middle-sized coalitions receive lower weight.

    ▶ Coalition Masks Become a Weighted Regression

    Each mask turns features on or off. Kernel SHAP fits an additive model whose coefficients explain the prediction difference.

    Algorithm

    choose row x0 and background rows
sample binary coalition masks
for each mask:
    build a masked row from x0 and background values
    evaluate black-box model f
    assign SHAP kernel weight
fit weighted linear model: output = baseline + mask * phi
return coefficients phi

    The Missing Feature Problem

    Kernel SHAP needs a way to evaluate the model when a feature is absent. In tabular data this usually means replacing absent features with values from a background dataset. The choice of background is not cosmetic. It defines what "missing" means and changes the baseline.

    Kernel SHAP is model-agnostic but not assumption-free. The black-box model can be anything, but the masking strategy and background distribution define the explanation question.
