    Shapley Values for Model Explanations

    July 2026 · 18 min read

    SHAP is based on Shapley values, a way to divide a total payoff among cooperating players. In explainability, the players are features, the payoff is the prediction difference from a baseline, and a feature's attribution is its average marginal contribution across all possible feature coalitions.

    The Formula

    For feature $j$ among $M$ features, the Shapley value is

    $$\phi_j=\sum_{S\subseteq F\setminus\{j\}}\frac{|S|!(M-|S|-1)!}{M!}\left[v(S\cup\{j\})-v(S)\right].$$

    The value function $v(S)$ is the model output when only coalition $S$ is known and the other features are treated as missing or integrated out.

    ▶ Averaging Marginal Contributions

    Feature B is added after different coalitions. Its Shapley value averages the marginal lift it contributes in every context.

    Why Shapley Values Are Special

    The Shapley allocation is the unique attribution satisfying a set of fairness axioms: all credit sums to the prediction difference, identical features receive identical credit, unused features receive zero credit, and contributions behave consistently when the model changes.

    Why Exact SHAP Is Expensive

    The formula sums over all coalitions not containing feature $j$. With $M$ features, that is $2^{M-1}$ terms per feature. Exact computation is therefore impossible for wide tabular data unless the model structure gives shortcuts. Tree SHAP uses tree structure for those shortcuts; Kernel SHAP estimates values by sampling coalitions.

    Baseline matters. SHAP explains the difference between the prediction and an expected baseline. Different background datasets can change the baseline and therefore the attributions.
