    Machine Learning Explainability

    July 2026 · Series overview

    Explainability asks why a model made a prediction, which features mattered, and whether those reasons are stable enough to trust. It is not the same as accuracy. A model can be accurate and opaque, or explainable but weak. The goal of this series is to build a practical vocabulary for explaining trained models without pretending that every explanation is causal truth.

    The first populated sub-series is SHAP & LIME, two model-agnostic local explanation families. LIME explains one prediction by fitting a simple local surrogate around the point. SHAP explains one prediction by assigning feature contributions using Shapley values from cooperative game theory. The matching implementation path lives in From Scratch · C++ · Explainability.

    Local versus Global

    A local explanation answers why the model predicted one row the way it did. A global explanation summarizes model behavior across many rows. SHAP and LIME are most naturally local, but local explanations can be aggregated into global summaries such as mean absolute attribution.

    Explanations are model behavior, not causality. If a model learned a spurious shortcut, SHAP and LIME can faithfully reveal that shortcut. They do not prove that the shortcut causes the outcome in the real world.
