    Iterative (Model-Based) Imputation

    June 2026 · 7 min read

    Iterative imputation treats every incomplete column as a small supervised problem: predict it from the other columns. Cycling through the columns and refitting until the fills stop changing is the idea behind MICE (multivariate imputation by chained equations). It captures relationships that a flat mean or even KNN can miss.

    The Chained-Equations Loop

    Start by seeding every gap with a simple fill, say the column mean. Then repeat: pick a column with missing values, treat its observed rows as the training target and the other columns as predictors, fit a model (often linear regression), and overwrite that column's gaps with the model's predictions. Move to the next column and reuse the freshly updated values. One full sweep over all columns is an iteration; a handful of iterations usually suffices for the fills to converge.

    Formally, for column $c$ the update is $\hat{x}_{rc} = f_c(x_{r,\,-c})$, where $f_c$ is fit on rows that observe $c$ and $x_{r,-c}$ are the other (already imputed) columns of row $r$.

    When to Use It

    ✓ Reach for it when

    • Several columns are missing and meaningfully correlated.
    • Missingness is MAR, so the other columns explain the gaps.
    • A flat fill visibly distorts relationships you care about.
    • You can afford to fit and store a model per column.

    ✗ Avoid it when

    • You need a fast, dependency-free baseline.
    • Columns are nearly independent; the extra machinery buys little.
    • Data is MNAR; even a good model of observed columns stays biased.
    • You want uncertainty, then use multiple imputation with several draws.

    From Scratch in C++

    A minimal one-predictor version: impute column $c$ by least-squares regression on column $p$, fit on the rows where both are present. Real MICE loops this over all columns, with all other columns as predictors, for several passes.

    #include <bits/stdc++.h>
using namespace std;

// One regression pass: fill gaps in column c from column p (least squares).
void regressImpute(vector<vector<double>>& X, int c, int p) {
    double sx = 0, sy = 0, sxy = 0, sxx = 0; int n = 0;
    for (auto& row : X)
        if (!isnan(row[c]) && !isnan(row[p])) {      // fit on complete pairs
            sx += row[p]; sy += row[c];
            sxy += row[p]*row[c]; sxx += row[p]*row[p]; n++;
        }
    double b = (n*sxy - sx*sy) / (n*sxx - sx*sx);    // slope
    double a = (sy - b*sx) / n;                       // intercept
    for (auto& row : X)
        if (isnan(row[c]) && !isnan(row[p])) row[c] = a + b*row[p];
}

int main() {
    double NA = nan("");
    // column 1 is roughly 2*column 0; fill its gap from that relationship.
    vector<vector<double>> X = {{1,2},{2,4},{3,NA},{4,8}};
    regressImpute(X, 1, 0);
    printf("filled X[2][1] = %g\n", X[2][1]);
}
    filled X[2][1] = 6

    The fit recovers slope $2$ and intercept $0$, so the gap at row $2$ becomes $2 \times 3 = 6$, a value a flat column mean ($\tfrac{2+4+8}{3}\approx4.7$) would have missed.

    Fit on train, then freeze. Learn the per-column models on the training rows and reuse them to transform validation, test, and production rows. Inside cross-validation the imputer must be refit within each fold, otherwise validation rows leak into the fills.
