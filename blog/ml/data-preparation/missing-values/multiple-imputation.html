    Multiple Imputation

    June 2026 · 7 min read

    Every single-value fill pretends it guessed the gap correctly. That confidence is fake: the imputed value is uncertain, and a model fit on one completed dataset reports standard errors that are too small. Multiple imputation fixes this by creating several completed datasets, each with a slightly different random fill, analysing all of them, and pooling the results so the imputation uncertainty shows up in the final estimate.

    Three Steps and Rubin's Rules

    Impute. Generate $m$ completed datasets by drawing each fill with noise, for example sampling from the predictive distribution of a model rather than taking its mean. Analyse. Run the same analysis on each dataset, producing estimates $\hat{Q}_1,\dots,\hat{Q}_m$ with variances $U_1,\dots,U_m$. Pool. Combine them with Rubin's rules:

    $$\bar{Q} = \frac{1}{m}\sum_j \hat{Q}_j, \qquad \bar{U} = \frac{1}{m}\sum_j U_j, \qquad B = \frac{1}{m-1}\sum_j (\hat{Q}_j - \bar{Q})^2.$$

    The total variance $T = \bar{U} + \left(1 + \tfrac{1}{m}\right)B$ adds the within-imputation variance $\bar{U}$ to the between-imputation variance $B$. That second term is exactly the uncertainty single imputation throws away.

    When to Use It

    ✓ Reach for it when

    • You report estimates with confidence intervals or p-values (inference, not just prediction).
    • Missingness is MAR and a fair amount of data is missing.
    • Understating uncertainty would be costly: clinical, economic, scientific work.
    • You can afford to run the analysis $m$ times (typically $m = 5$–$50$).

    ✗ Avoid it when

    • You only need point predictions; a single good imputation is usually enough.
    • Compute is tight, since everything downstream is repeated $m$ times.
    • Data is MNAR; pooling honest variance still does not remove the bias.
    • The pipeline cannot carry $m$ parallel datasets through to the estimate.

    From Scratch in C++

    A compact illustration: estimate a column mean under missingness. Each imputation fills gaps by sampling observed values, the estimate is the resulting mean, and the pool follows Rubin's rules.

    #include <bits/stdc++.h>
using namespace std;

struct Pooled { double estimate, totalVar; };

// m imputations of a column mean, pooled by Rubin's rules.
Pooled multipleImpute(const vector<double>& col, int m, mt19937& rng) {
    vector<double> obs;
    for (double v : col) if (!isnan(v)) obs.push_back(v);
    uniform_int_distribution<int> pick(0, (int)obs.size() - 1);

    vector<double> est(m); double within = 0;
    for (int j = 0; j < m; ++j) {
        vector<double> full; double s = 0;
        for (double v : col) {                       // draw a fill for each gap
            double f = isnan(v) ? obs[pick(rng)] : v;
            full.push_back(f); s += f;
        }
        int n = full.size();
        double mean = s / n; est[j] = mean;
        double var = 0; for (double f : full) var += (f - mean) * (f - mean);
        within += var / (n * (n - 1.0));             // U_j: variance of the mean
    }
    double qbar = 0; for (double e : est) qbar += e; qbar /= m;
    double between = 0; for (double e : est) between += (e - qbar) * (e - qbar);
    between /= (m - 1);
    return { qbar, within / m + (1.0 + 1.0 / m) * between };   // T = Ubar + (1+1/m)B
}

int main() {
    double NA = nan("");
    vector<double> col = {2, 4, NA, 6, NA, 8};
    mt19937 rng(42);
    Pooled p = multipleImpute(col, 20, rng);
    printf("pooled mean = %.3f   total variance = %.3f\n", p.estimate, p.totalVar);
}
    pooled mean = 4.883 total variance = 1.328

    The between-imputation term widens the variance beyond what any single fill would report, which is the whole point: the final interval admits that the gaps were guessed.

    Draw with noise, fit on train. Multiple imputation only captures uncertainty if each fill is a random draw, not a repeated point estimate. As with every method here, the imputation model is learned on the training data and reused, so the test set never leaks into its own fills. This closes the Handling Missing Data series.
