    KNN Imputation

    June 2026 · 7 min read

    A flat mean ignores everything else known about a row. KNN imputation does better: to fill a gap, it finds the rows most similar to this one on the features that are present, and borrows their value for the missing feature. When columns are correlated, near neighbours carry real information about the hole.

    How It Works

    For a row $r$ with a gap in column $c$, measure distance to every other row that observes $c$, using only the columns both rows share. Take the $k$ closest and combine their $c$ values, usually a plain or distance-weighted mean. Because some columns are themselves missing, the distance is averaged over the shared dimensions:

    $$d(r,i) = \sqrt{\frac{1}{|S|}\sum_{j \in S}\left(x_{rj} - x_{ij}\right)^2}, \quad S = \{ j \neq c : x_{rj}, x_{ij} \text{ both present} \}.$$

    When to Use It

    ✓ Reach for it when

    • Features are correlated, so neighbours genuinely predict the gap.
    • Missingness is MAR and explainable from the other columns.
    • The dataset is small to medium, where pairwise distance is affordable.
    • You want a nonparametric fill that captures local structure.

    ✗ Avoid it when

    • Data is large; the naive cost is $O(n^2 d)$ per imputed column.
    • Dimensionality is high, where distances concentrate and "nearest" loses meaning.
    • Features are on wildly different scales and you have not scaled them.
    • A cheap mean or median is already accurate enough.

    From Scratch in C++

    Fill one cell from the mean of its $k$ nearest complete neighbours. Scaling the features first keeps the distance fair.

    #include <bits/stdc++.h>
using namespace std;

// Impute X[r][c] from the mean of c over the k nearest rows that observe c.
double knnImputeCell(const vector<vector<double>>& X, int r, int c, int k) {
    int n = X.size(), d = X[0].size();
    vector<pair<double,double>> cand;          // (distance, value at c)
    for (int i = 0; i < n; ++i) {
        if (i == r || isnan(X[i][c])) continue;
        double s = 0; int m = 0;
        for (int j = 0; j < d; ++j)
            if (j != c && !isnan(X[r][j]) && !isnan(X[i][j])) {
                double diff = X[r][j] - X[i][j]; s += diff * diff; m++;
            }
        if (m) cand.push_back({sqrt(s / m), X[i][c]});  // shared-dim distance
    }
    sort(cand.begin(), cand.end());
    int t = min((int)cand.size(), k);
    double sum = 0; for (int i = 0; i < t; ++i) sum += cand[i].second;
    return t ? sum / t : nan("");
}

int main() {
    double NA = nan("");
    vector<vector<double>> X = {{1,1,NA}, {1,1,10}, {1,2,12}, {9,9,99}};
    printf("filled X[0][2] = %g\n", knnImputeCell(X, 0, 2, 2));
}
    filled X[0][2] = 11

    Rows $1$ and $2$ are the closest to row $0$ on the first two columns, so the gap is filled with the mean of their third column, $\tfrac{10+12}{2}=11$, while the far-away row $3$ is ignored.

    Fit on train, scale first. Build the neighbour pool from the training rows only and reuse it to impute test rows, so the test set never informs its own fills. Standardize features before measuring distance, otherwise a large-range column dominates and the neighbours are chosen by the wrong axis.
