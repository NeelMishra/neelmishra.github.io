    Mean, Median, and Mode Imputation

    June 2026 · 6 min read

    The fastest imputation replaces every gap in a column with one summary of that column's observed values: the mean or median for numeric data, the mode for categorical data. It is a strong baseline, costs almost nothing, and is trivial to apply at serving time, as long as the fill value is learned only from the training data.

    Which Summary

    The mean, $\bar{x} = \frac{1}{n}\sum_i x_i$, is natural for roughly symmetric numeric columns, but it is pulled toward extreme values. The median is the middle order statistic and barely moves when a few points are extreme, which makes it the safer default for skewed columns. The mode, the most frequent value, is the only sensible single-value fill for categories.

    A column with an outlier shows the difference at once: for $\{1, 2, 4, 100\}$ the mean is $26.75$, larger than every non-outlier, while the median is $3$, right inside the cluster.

    When to Use It

    ✓ Reach for it when

    • You need a quick, explainable baseline before anything fancier.
    • Missingness is MCAR or mild MAR and the missing-rate is low.
    • The column is numeric (mean/median) or categorical (mode).
    • Serving must be cheap: store one number per column and reuse it.

    ✗ Avoid it when

    • Columns are strongly correlated; a flat fill ignores that structure (use KNN or model-based).
    • The missing-rate is high; one repeated value shrinks variance and distorts correlations.
    • Data is MNAR; the fill hides that absence was informative (add an indicator).
    • You need calibrated uncertainty; single imputation understates it (see multiple imputation).

    From Scratch in C++

    Compute the statistic over present values only, then apply it to the gaps. The median is shown; swapping in the mean is a one-line change.

    #include <bits/stdc++.h>
using namespace std;

double columnMedian(vector<double> col) {
    vector<double> v;
    for (double x : col) if (!isnan(x)) v.push_back(x);  // skip the gaps
    sort(v.begin(), v.end());
    int n = v.size();
    return n % 2 ? v[n/2] : (v[n/2 - 1] + v[n/2]) / 2.0;
}

void imputeWith(vector<double>& col, double fill) {
    for (double& x : col) if (isnan(x)) x = fill;
}

int main() {
    double NA = nan("");
    vector<double> col = {1, 2, NA, 4, 100, NA};
    double fill = columnMedian(col);          // learned on train
    imputeWith(col, fill);
    printf("median fill = %g  ->  ", fill);
    for (double x : col) printf("%g ", x);
}
    median fill = 3 -> 1 2 3 4 100 3
    Fit on train, apply to test. Compute the mean, median, or mode on the training column only and reuse that stored value to fill the test and production columns. Computing it on the full dataset before the split leaks information, the same rule the from-scratch C++ imputer follows.
