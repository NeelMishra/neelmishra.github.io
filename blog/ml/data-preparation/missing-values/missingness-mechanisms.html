    Missingness Mechanisms: MCAR, MAR, MNAR

    June 2026 · 7 min read

    Before choosing how to fill a gap, ask why it is there. The mechanism that produced the missingness decides whether deletion is safe, simple imputation is honest, or the absence itself carries signal. Let $M$ be the missingness indicator, with $M=1$ when a value is absent, and let $X_{obs}$ and $X_{mis}$ be the observed and missing parts of the data.

    MCAR — Missing Completely at Random

    Missingness is independent of everything: $P(M \mid X_{obs}, X_{mis}) = P(M)$. A random file glitch or a lab machine failing on a random subset. Under MCAR the observed rows are a representative sample, so deletion is unbiased; the only cost is throwing data away.

    MAR — Missing at Random

    Missingness depends on observed columns but not on the hidden value once those are known: $P(M \mid X_{obs}, X_{mis}) = P(M \mid X_{obs})$. A sensor fails more often in cold weather, and temperature is recorded. The missingness is explainable from columns you already have, which is exactly what lets KNN and regression imputation borrow strength from correlated features.

    MNAR — Missing Not at Random

    Missingness depends on the hidden value itself: high earners skip the income field, severe patients skip the survey. Here the missing values differ systematically from the observed ones, so naive imputation is biased and the fact of being missing often predicts the target.

    When This Matters

    The mechanism is a diagnosis, not a fill. It tells you which downstream method is defensible.

    ✓ Use the framing to

    • Justify deletion only when MCAR is plausible.
    • Pick model-based imputation when MAR holds on observed columns.
    • Add a missingness indicator when MNAR is possible.
    • Decide whether absence should be a feature or just patched.

    ✗ Do not

    • Assume MCAR by default; test it against observed columns first.
    • Trust a single imputation under suspected MNAR without sensitivity checks.
    • Forget that MNAR cannot be confirmed from the data alone, it needs domain knowledge.

    A Quick MAR Probe in C++

    You cannot prove MCAR, but you can falsify it: if a column's missing-rate differs across groups of an observed flag $g$, the missingness depends on $g$, so it is not MCAR (and is at least MAR in $g$).

    #include <bits/stdc++.h>
using namespace std;

// Missing-rate of column y within each group of an observed binary flag g.
pair<double,double> missRateByGroup(const vector<double>& y, const vector<int>& g) {
    long n0 = 0, m0 = 0, n1 = 0, m1 = 0;
    for (size_t i = 0; i < y.size(); ++i) {
        bool miss = isnan(y[i]);
        if (g[i] == 0) { n0++; m0 += miss; }
        else           { n1++; m1 += miss; }
    }
    return { n0 ? (double)m0 / n0 : 0.0,
             n1 ? (double)m1 / n1 : 0.0 };
}

int main() {
    double NA = nan("");
    vector<double> y = {1, NA, 3, NA, 5, NA, 7, 8};  // the column with gaps
    vector<int>    g = {0,  0, 0,  1, 1,  1, 1, 0};  // an observed flag (e.g. cold=1)
    auto r = missRateByGroup(y, g);
    printf("missing rate  g=0: %.2f   g=1: %.2f\n", r.first, r.second);
}
    missing rate g=0: 0.25 g=1: 0.50

    The rates differ across $g$, evidence against MCAR. In practice you would test every observed column and use a significance test, but the principle is this single comparison.

    Mechanism first, method second. MCAR makes deletion defensible, MAR makes imputation from observed columns useful, and MNAR warns that absence itself may be predictive. The rest of this series, from deletion to multiple imputation, is a menu of responses to this one question.
