    Backward Fill

    June 2026 · 6 min read

    Backward fill, also called next observation carried backward (NOCB), replaces each gap with the nearest value seen after it. It is the mirror image of forward fill: instead of reaching into the past, it reaches into the future. That single difference is what makes it useful in some places and dangerous in others.

    How It Works

    Walk through the series from the end to the start, remembering the last value seen (which, going backwards, is the next value going forwards). Fill each gap with it. Trailing gaps, after the final observation, have no future value and stay missing.

    index:  0   1   2   3   4   5
input:  NA  3   NA  NA  5   NA
bfill:  3   3   5   5   5   NA

    For a gap at time $t$, the fill is $x_t \leftarrow x_{\tau}$ where $\tau = \min\{ s > t : x_s \text{ observed} \}$. Notice the leading gap at index $0$ is now filled (from the $3$ that follows), while the trailing gap at index $5$ remains.

    When to Use It

    Backward fill is the right tool when the next known value is the best estimate of a preceding gap, and when looking into the future is acceptable.

    ✓ Reach for it when

    • A value becomes known slightly late and should be back-dated to the gap.
    • You are doing offline analysis or plotting, where causality does not matter.
    • You need to fill the leading gaps that forward fill leaves behind.
    • The quantity is "the next scheduled value" (next price tick, next census).

    ✗ Avoid it when

    • You build features for forecasting: it injects the future into the past (leakage).
    • Rows are unordered, where "next" is meaningless.
    • The series moves fast, so a future value is a poor stand-in for the gap.
    • You need the last trailing values filled, those have no successor.

    From Scratch in C++

    The same idea as forward fill, but the pass runs right to left.

    #include <bits/stdc++.h>
using namespace std;
static const double NA = nan("");

// Backward fill: carry the next observed value into earlier gaps (in place).
void backwardFill(vector<double>& x) {
    double next = NA;
    for (int i = (int)x.size() - 1; i >= 0; --i) {
        if (!isnan(x[i])) next = x[i];          // most recent value going backwards
        else if (!isnan(next)) x[i] = next;     // fill from the future
    }
}

int main() {
    vector<double> x = {NA, 3, NA, NA, 5, NA};
    backwardFill(x);
    for (double v : x) isnan(v) ? printf("NA ") : printf("%g ", v);
}
    3 3 5 5 5 NA
    Watch for leakage. Backward fill copies a future value into an earlier row. In any model that will predict the future from the past, this leaks the answer and inflates offline scores. Use it only for non-causal work, or pair it after a forward fill purely to patch leading gaps in static, already-observed data.
