    Forward and Backward Fill

    June 2026 · 7 min read

    For ordered data, the cheapest good guess for a gap is often a neighbour. Forward fill copies the last value seen before the gap; backward fill copies the next value seen after it. Both rely on the assumption that a value persists between observations, which holds for many time series and almost never holds for unordered tabular rows.

    Two Directions

    Take a short series with three gaps and fill it each way. Forward fill leaves the leading gap empty; backward fill leaves the trailing gap empty.

    index:  0   1   2   3   4   5
input:  NA  3   NA  NA  5   NA
ffill:  NA  3   3   3   5   5     (last value carried forward)
bfill:  3   3   5   5   5   NA    (next value carried backward)

    Forward fill is causal: a filled value at time $t$ uses only $x_s$ for $s < t$, so it is safe for forecasting features. Backward fill uses $x_s$ for $s > t$, which means it borrows from the future. The dedicated pages go deeper: Forward Fill and Backward Fill.

    The Forward-Then-Backward Combination

    Forward fill cannot patch leading gaps and backward fill cannot patch trailing gaps. Running forward fill first, then backward fill on whatever remains, leaves no gaps in a series that has at least one observation, while keeping past-derived values everywhere except the leading edge.

    When to Use It

    ✓ Reach for it when

    • Rows are ordered, usually a time series, and values persist between updates.
    • You model states or gauges: status flags, setpoints, slowly varying meters.
    • You want a fast fill with no fitted parameters and no cross-row arithmetic.
    • You need a no-gap series and accept a forward-then-backward pass to cover both ends.

    ✗ Avoid it when

    • Rows have no order, so "previous" and "next" are undefined.
    • The signal changes quickly; a copied neighbour misrepresents the gap.
    • You build forecasting features and use backward fill, which leaks the future.
    • Gaps are long; a single value gets stamped across a wide span and flattens variance.

    From Scratch in C++

    Two linear passes in opposite directions. Combine them to leave no interior or edge gaps.

    #include <bits/stdc++.h>
using namespace std;
static const double NA = nan("");

void forwardFill(vector<double>& x) {
    double last = NA;
    for (double& v : x)
        if (!isnan(v)) last = v;
        else if (!isnan(last)) v = last;     // past only (causal)
}

void backwardFill(vector<double>& x) {
    double next = NA;
    for (int i = (int)x.size() - 1; i >= 0; --i)
        if (!isnan(x[i])) next = x[i];
        else if (!isnan(next)) x[i] = next;  // future (use with care)
}

int main() {
    vector<double> x = {NA, 3, NA, NA, 5, NA};
    forwardFill(x);     // NA 3 3 3 5 5
    backwardFill(x);    // 3  3 3 3 5 5  (patches the leading gap)
    for (double v : x) printf("%g ", v);
}
    3 3 3 3 5 5
    Fill within a group, not across. Apply the passes inside each entity (per device, per ticker, per patient) so a value never bleeds from one series into another, and prefer forward fill alone when the features feed a forecast. See Forward Fill and Backward Fill for the causality details.
