    Missingness Indicators

    June 2026 · 5 min read

    Every imputation erases one fact: that the value was missing in the first place. When the absence is informative, that fact is worth keeping. A missingness indicator preserves it by adding a binary column $M$ that records whether the original entry was present, then letting an ordinary imputer fill the value so the numeric pipeline can still run.

    How It Works

    For each column that may be missing, create a companion flag $M_{rc} = 1$ if $x_{rc}$ was absent and $0$ otherwise, then impute $x_{rc}$ as usual. The model now sees two signals: the filled value, and whether to trust it. A tree can split on the flag; a linear model gets a separate coefficient for "was missing." The imputed number keeps arithmetic defined; the flag keeps the information.

    When to Use It

    ✓ Reach for it when

    • Missingness may be MNAR, so absence itself predicts the target.
    • A field is optional and skipping it is a deliberate choice (e.g. income).
    • You want the safety of imputation without losing the signal it hides.
    • The model can exploit an extra binary feature (trees, linear models, nets).

    ✗ Avoid it when

    • Missingness is plainly MCAR; the flag is pure noise then.
    • A column is missing for only a handful of rows, risking overfit on the flag.
    • You already have many columns and cannot afford to double the sparse ones.

    From Scratch in C++

    Append the flag first, then fill the value, so the record of the gap survives the imputation.

    #include <bits/stdc++.h>
using namespace std;

// Append a 0/1 "was missing" flag for column c, then fill c with `fill`.
void addIndicatorAndFill(vector<vector<double>>& X, int c, double fill) {
    for (auto& row : X) {
        double flag = isnan(row[c]) ? 1.0 : 0.0;   // record absence first
        if (isnan(row[c])) row[c] = fill;          // then patch the value
        row.push_back(flag);                       // new trailing indicator column
    }
}

int main() {
    double NA = nan(""), fill = 0.0;
    vector<vector<double>> X = {{5}, {NA}, {7}};
    addIndicatorAndFill(X, 0, fill);
    for (auto& r : X) printf("value=%g  was_missing=%g\n", r[0], r[1]);
}
    value=5 was_missing=0 value=0 was_missing=1 value=7 was_missing=0
    Compute the flag from the raw input. The indicator must reflect the original data, so build it before any fill and from the training definition of "missing." A practical baseline is median or mode imputation plus a was-missing flag, then add complexity only if validation rewards it.
