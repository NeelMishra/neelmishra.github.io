    Deletion: Listwise and Pairwise

    June 2026 · 6 min read

    The simplest response to a gap is to drop it. Deletion needs no fitted parameters and introduces no invented values, which makes it transparent. The catch is that the rows you remove are only ignorable when the data is missing completely at random; otherwise deletion quietly reshapes the distribution it leaves behind.

    Listwise Deletion

    Listwise (complete-case) deletion removes any row that has at least one missing value. With $d$ columns each independently missing with probability $p$, a row survives with probability $(1-p)^d$, so the loss compounds as the table grows wider. One stray gap in one column can discard an otherwise complete row.

    Pairwise Deletion

    Pairwise deletion keeps more data by computing each quantity from the rows that have the columns it needs. A correlation between columns $A$ and $B$ uses rows where both are present; a correlation between $A$ and $C$ uses a different subset. It wastes less, but each statistic is computed on a different sample, which can produce inconsistent results, for example a covariance matrix that is not positive semidefinite.

    When to Use It

    ✓ Reach for it when

    • Missingness looks MCAR, so the remaining rows stay representative.
    • Only a small fraction of rows are affected.
    • You have abundant data and can spend some on simplicity.
    • You want a clean, parameter-free baseline before trying imputation.

    ✗ Avoid it when

    • Missingness is MAR or MNAR, where deletion biases the sample.
    • Many columns or a high missing-rate would shrink the data sharply.
    • Rare classes or rare events could be wiped out.
    • At inference you must score every row, including incomplete ones.

    From Scratch in C++

    Listwise deletion is one scan that keeps only fully observed rows.

    #include <bits/stdc++.h>
using namespace std;

// Listwise deletion: keep only rows with no missing entry.
vector<vector<double>> dropMissingRows(const vector<vector<double>>& X) {
    vector<vector<double>> kept;
    for (const auto& row : X) {
        bool complete = true;
        for (double v : row) if (isnan(v)) { complete = false; break; }
        if (complete) kept.push_back(row);
    }
    return kept;
}

int main() {
    double NA = nan("");
    vector<vector<double>> X = {{1,2,3}, {4,NA,6}, {7,8,9}, {NA,1,1}};
    auto Y = dropMissingRows(X);
    printf("kept %zu of %zu rows\n", Y.size(), X.size());
}
    kept 2 of 4 rows
    Decide the rule on train, apply everywhere. Listwise deletion is fine for the training table, but you cannot delete rows you must score in production. If incomplete rows appear at inference, pair deletion during training with an imputer for serving, so the pipeline always has a value to work with.
