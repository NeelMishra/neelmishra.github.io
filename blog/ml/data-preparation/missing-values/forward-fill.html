    Forward Fill

    June 2026 · 6 min read

    Forward fill, also called last observation carried forward (LOCF), replaces each gap with the most recent value seen before it. It only makes sense when the rows have an order, usually time: the last known reading is often the best guess for a value that has not changed yet.

    How It Works

    Walk through the series in order, remembering the last value that was actually observed. Whenever you hit a gap, copy that remembered value into it. Leading gaps, the ones before the first real observation, have no past value to draw from and stay missing.

    index:  0   1   2   3   4   5
input:  NA  3   NA  NA  5   NA
ffill:  NA  3   3   3   5   5

    If the data is a state that persists until it changes, like a thermostat setpoint or an account status, this is exactly right: between updates the value genuinely is the last one. Formally, the fill for a gap at time $t$ is $x_t \leftarrow x_{\tau}$ where $\tau = \max\{ s < t : x_s \text{ observed} \}$.

    When to Use It

    The decisive question is whether the previous known value is a sensible stand-in for the gap, and whether using only past information is required.

    ✓ Reach for it when

    • Data is ordered or a time series (sensor logs, prices, status fields).
    • The signal is step-like or slowly changing between updates.
    • You need the last known state (last quote, last meter reading).
    • You build features for forecasting and must stay causal (past only).

    ✗ Avoid it when

    • Rows are unordered tabular data, where "previous" has no meaning.
    • The series changes quickly, so a stale value misrepresents the gap.
    • Gaps are long; one value gets stamped across a wide stretch and flattens variance.
    • The very first values are missing, leading gaps remain and need a default or a backward fill.

    From Scratch in C++

    A single left-to-right pass. A floating-point NaN marks a gap, and isnan tests for it.

    #include <bits/stdc++.h>
using namespace std;
static const double NA = nan("");

// Forward fill: carry the last observed value into later gaps (in place).
void forwardFill(vector<double>& x) {
    double last = NA;
    for (double& v : x) {
        if (!isnan(v)) last = v;            // remember the most recent real value
        else if (!isnan(last)) v = last;    // fill from the past only
    }
}

int main() {
    vector<double> x = {NA, 3, NA, NA, 5, NA};
    forwardFill(x);
    for (double v : x) isnan(v) ? printf("NA ") : printf("%g ", v);
}
    NA 3 3 3 5 5

    The leading gap is left as NA because nothing precedes it. The pass is $O(n)$ and touches only values at or before each index, which is what keeps it causal.

    Causal, but fit per series. Forward fill uses only the past, so it does not leak future information into forecasting features. Apply it within each entity (per device, per ticker) so a value never carries across unrelated series, and resolve any leading gaps with a default or a one-step backward fill.
