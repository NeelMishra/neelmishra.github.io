    Feature Scaling

    June 2026 · 14 min read

    When one feature ranges over thousands and another over hundredths, any model that compares features by magnitude is dominated by the large-scale feature, and the small one is effectively ignored. Scaling puts every feature on a comparable footing. This part covers the standard scalers, the geometry of why scaling matters, and the one model family that can safely skip it.

    Three Scalers

    Standardization (z-score) centers each feature to mean $0$ and scales it to standard deviation $1$:

    $$z = \frac{x - \mu}{\sigma}.$$

    Min-max normalization maps each feature linearly into $[0, 1]$:

    $$x' = \frac{x - \min}{\max - \min}.$$

    Robust scaling replaces the mean and standard deviation with the median and interquartile range, so a few extreme outliers do not set the scale. Z-score is the usual default; min-max is handy when a bounded range is needed (image pixels, some neural inputs); robust scaling is preferred when the feature has heavy outliers. In every case the parameters ($\mu, \sigma$ or $\min, \max$ or median, IQR) are learned on the training data.

    The Geometry: Distance Is Dominated by the Big Feature

    Most of the methods that need scaling rely on Euclidean distance or on a gradient, and both are sensitive to feature magnitude. Consider distance: with features $x_1$ (range $0$ to $100$) and $x_2$ (range $0$ to $1$), the squared distance $(\Delta x_1)^2 + (\Delta x_2)^2$ is almost entirely $(\Delta x_1)^2$, because $\Delta x_1$ can be a hundred times larger. The second feature might as well not exist. The animation makes this concrete with a nearest-neighbor classifier: in raw units the query's nearest neighbor is the wrong class, and standardizing the two features flips it to the right one.

    ▶ Scaling Flips the Nearest Neighbor

    A query point (blue) is classified by its nearest neighbor. Class A (green) sits at low $x_2$, class B (orange) at high $x_2$. In raw units the wide feature $x_1$ dominates the distance; after standardizing, $x_2$ counts equally and the nearest neighbor changes class.

    The raw panel plots the two features at the same pixel scale, so the data collapses into a nearly horizontal band: visually, $x_2$ barely varies, which is exactly why it is drowned out of the distance. The standardized panel rescales both axes to unit variance, the band opens up into two clear class clusters, and the query now sits with its true class.

    Which Models Care, and Which Do Not

    Scaling matters for any model whose answer depends on feature magnitude:

    • Distance-based methods (k-nearest-neighbors, k-means, anything with a Euclidean or cosine metric), for the reason the animation shows.
    • Gradient-based models (linear and logistic regression, neural networks). Unscaled features stretch the loss surface into a long narrow valley, so gradient descent zig-zags and converges slowly.
    • Regularized models (ridge, lasso, SVM). The penalty $\lambda \sum_j w_j^2$ treats all weights alike, which is only fair if the features share a scale.
    Trees do not need it. A decision tree, random forest, or boosted tree splits one feature at a time at a threshold, so any monotonic rescaling leaves every split, and therefore the whole model, unchanged. Scaling a tree-based pipeline is harmless but pointless. This is a useful tell: if the model only ever compares a feature to a constant, scale does not matter; if it combines features by distance or weighted sum, it does.

    Fit on Train, Apply to Test

    The scaler is fit on the training set, learning each feature's $\mu$ and $\sigma$ (or $\min$ and $\max$), and those same numbers transform the test set. The test data does not come out at exactly mean $0$, standard deviation $1$, and that is correct: re-estimating the statistics on the test set would leak it into the pipeline (the central rule from Part 1). The from-scratch standardizer, with this fit/transform split made explicit, is in From Scratch · C++ · Feature Scaling.
