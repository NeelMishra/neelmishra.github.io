Vertex AI Pipelines: GCP ML Pipelines

MLOps Series CI/CD for ML

Vertex AI Pipelines is Google Cloud's fully managed service for running ML workflows. Built on the Kubeflow Pipelines SDK, it eliminates the need to manage Kubernetes clusters while providing deep integration with GCP services like BigQuery, Cloud Storage, and Vertex AI Model Registry.

Vertex AI Overview

Vertex AI is Google Cloud's unified ML platform. Vertex AI Pipelines is the workflow orchestration layer that ties together data preparation, training, evaluation, and deployment into reproducible, automated pipelines.

Key advantages of Vertex AI Pipelines over self-managed Kubeflow:

  • Serverless execution — no Kubernetes cluster to manage; Google handles infrastructure
  • Native GCP integration — first-class connectors for BigQuery, Cloud Storage, Dataflow, and Vertex AI services
  • Built-in metadata tracking — Vertex ML Metadata stores lineage, parameters, and metrics automatically
  • Managed scheduling — cron-based pipeline schedules without external orchestrators
Pricing consideration: Vertex AI Pipelines charges per pipeline run plus the cost of underlying compute (custom jobs, training jobs, endpoints). For high-frequency retraining pipelines, costs can escalate quickly. Use caching aggressively and right-size your machine types to control spend.

Pipeline Components

Vertex AI Pipelines uses the same KFP v2 SDK as Kubeflow, so component definitions are portable. However, Vertex AI also provides pre-built Google Cloud Pipeline Components (GCPC) for common GCP operations.

# Vertex AI pipeline with Google Cloud components from kfp import dsl from google_cloud_pipeline_components.v1.bigquery import BigqueryQueryJobOp from google_cloud_pipeline_components.v1.vertex_ai import ModelUploadOp, EndpointDeployOp @dsl.component(base_image="python:3.10", packages_to_install=["scikit-learn", "pandas", "google-cloud-storage"]) def train_model(data_uri: str, model_out: dsl.Output[dsl.Model]): import pandas as pd from sklearn.ensemble import GradientBoostingClassifier df = pd.read_csv(data_uri) model = GradientBoostingClassifier() model.fit(df.drop("target", axis=1), df["target"]) # Save model artifact save_model(model, model_out.path) @dsl.pipeline(name="vertex-training-pipeline") def vertex_pipeline(project: str, bq_query: str): # Step 1 — Extract data from BigQuery bq_task = BigqueryQueryJobOp( project=project, query=bq_query, destination_uri="gs://my-bucket/data/extract.csv", ) # Step 2 — Train model train_task = train_model(data_uri=bq_task.outputs["destination_uri"]) # Step 3 — Upload model to Vertex AI Model Registry upload_task = ModelUploadOp( project=project, display_name="fraud-detector-v2", artifact_uri=train_task.outputs["model_out"], serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/sklearn-cpu:latest", ) # Step 4 — Deploy to endpoint EndpointDeployOp( project=project, model=upload_task.outputs["model"], deployed_model_display_name="fraud-detector-live", machine_type="n1-standard-4", traffic_split={"0": 100}, )

Metadata Management

Vertex ML Metadata automatically tracks artifacts, executions, and lineage for every pipeline run. You can also log custom metadata to enrich the audit trail.

# Logging custom metadata in Vertex AI from google.cloud import aiplatform aiplatform.init(project="my-project", location="us-central1") # Log experiment metrics with aiplatform.start_run("training-run-042") as run: run.log_params({ "learning_rate": 0.01, "n_estimators": 200, "max_depth": 6, }) run.log_metrics({ "accuracy": 0.943, "f1_score": 0.921, "auc_roc": 0.978, }) run.log_time_series_metrics({"loss": 0.087}, step=100) # Query lineage for a deployed model model = aiplatform.Model("projects/my-project/locations/us-central1/models/12345") lineage = model.get_model_evaluation() print(lineage.metrics)

Scheduling

Vertex AI Pipelines supports native scheduling — no need for Cloud Scheduler or Airflow to trigger periodic runs.

# Scheduling a Vertex AI pipeline from google.cloud import aiplatform aiplatform.init(project="my-project", location="us-central1") # Create a scheduled pipeline job schedule = aiplatform.PipelineJobSchedule( pipeline_job=aiplatform.PipelineJob( display_name="weekly-retraining", template_path="gs://my-bucket/pipelines/training_v2.json", parameter_values={"project": "my-project", "bq_query": query}, ), display_name="weekly-retrain-schedule", ) schedule.create( cron="0 3 * * 0", # every Sunday at 3 AM max_concurrent_run_count=1, max_run_count=52, # run for one year )
Tip: Use Vertex AI's built-in scheduling for straightforward cron patterns. For complex trigger logic (data-volume-based, event-driven), integrate with Cloud Functions or Pub/Sub to submit pipeline runs programmatically.

GCP Integration

Vertex AI Pipelines has deep integration with the broader GCP ecosystem:

  • BigQuery — extract training data, write evaluation results, run feature engineering queries
  • Cloud Storage — store datasets, model artifacts, and pipeline specs
  • Vertex AI Feature Store — serve pre-computed features for both training and online inference
  • Vertex AI Model Registry — version and manage models with deployment metadata
  • Vertex AI Endpoints — deploy models for online prediction with autoscaling
  • Cloud Monitoring — track pipeline metrics, set up alerts on failure rates or latency

Vertex AI Pipelines

  • Serverless — no cluster management
  • KFP v2 SDK compatible
  • Deep GCP service integration
  • Built-in scheduling and metadata
  • Best for GCP-native teams

Cloud Composer (Airflow)

  • General-purpose orchestration
  • Broader ecosystem of operators
  • Better for mixed workloads (ML + ETL)
  • Higher operational overhead
  • Best for complex multi-system workflows

Monitoring

Monitoring Vertex AI pipelines involves tracking both pipeline health (run success, step durations, resource usage) and model quality (prediction accuracy, drift, latency).

# Vertex AI Model Monitoring setup from google.cloud import aiplatform # Enable model monitoring on an endpoint endpoint = aiplatform.Endpoint("projects/my-project/locations/us-central1/endpoints/6789") monitoring_job = aiplatform.ModelDeploymentMonitoringJob.create( display_name="fraud-model-monitoring", endpoint=endpoint, logging_sampling_strategy={"random_sample_config": {"sample_rate": 0.1}}, schedule_config={"monitor_interval": {"seconds": 3600}}, alert_config={ "email_alert_config": {"user_emails": ["ml-team@example.com"]}, }, objective_configs=[{ "training_dataset": training_data_uri, "training_prediction_skew_detection_config": { "skew_thresholds": {"feature_1": {"value": 0.3}}, }, }], )

Key monitoring dimensions for Vertex AI Pipelines:

  • Pipeline run status — success/failure rates, average duration, step-level breakdowns
  • Model prediction quality — accuracy trends, feature drift, prediction drift
  • Infrastructure metrics — endpoint latency, CPU/GPU utilisation, autoscaler behaviour
  • Cost tracking — per-pipeline and per-endpoint costs via Cloud Billing dashboards
