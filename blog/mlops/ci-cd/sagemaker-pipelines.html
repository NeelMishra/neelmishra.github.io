SageMaker Pipelines: AWS ML Workflows

MLOps Series CI/CD for ML

Amazon SageMaker Pipelines is AWS's native service for building, automating, and managing end-to-end ML workflows. Tightly integrated with the SageMaker ecosystem — processing jobs, training jobs, model registry, and endpoints — it provides a Python SDK for defining repeatable, production-grade pipelines.

SageMaker Overview

Amazon SageMaker is a comprehensive ML platform covering the entire ML lifecycle — from data labelling and feature engineering to training, deployment, and monitoring. SageMaker Pipelines is the workflow automation layer that ties these capabilities into repeatable, versioned pipelines.

Core concepts in SageMaker Pipelines:

  • Pipeline — a DAG of steps defined in Python, compiled to a JSON definition
  • Steps — individual units of work (processing, training, evaluation, model registration, condition checks)
  • Parameters — runtime inputs that can be changed without modifying pipeline code
  • Properties — outputs from one step that can be referenced by downstream steps
  • Caching — step-level caching to skip unchanged steps on re-execution
Cost optimisation: SageMaker charges for compute time on processing and training instances. Use Spot Instances (managed spot training) for up to 90% cost savings on fault-tolerant training jobs. Set use_spot_instances=True and max_wait in your Estimator configuration.

Pipeline Steps

SageMaker Pipelines provides a rich set of built-in step types that map to SageMaker's managed services:

# SageMaker Pipeline — complete definition from sagemaker.workflow.pipeline import Pipeline from sagemaker.workflow.parameters import ParameterString, ParameterFloat from sagemaker.workflow.steps import ProcessingStep, TrainingStep from sagemaker.workflow.step_collections import RegisterModel # Define pipeline parameters input_data = ParameterString(name="InputData", default_value="s3://bucket/data") accuracy_threshold = ParameterFloat(name="AccuracyThreshold", default_value=0.9) instance_type = ParameterString(name="TrainInstanceType", default_value="ml.m5.xlarge") # Build the pipeline pipeline = Pipeline( name="fraud-detection-pipeline", parameters=[input_data, accuracy_threshold, instance_type], steps=[process_step, train_step, eval_step, condition_step], ) # Submit to SageMaker pipeline.upsert(role_arn=role) execution = pipeline.start( parameters={"InputData": "s3://bucket/data/2024-q1"} )

Processing Steps

Processing steps run data preparation, feature engineering, and evaluation scripts on managed compute instances. They wrap SageMaker Processing jobs.

# ProcessingStep for data preparation from sagemaker.sklearn.processing import SKLearnProcessor from sagemaker.processing import ProcessingInput, ProcessingOutput from sagemaker.workflow.steps import ProcessingStep sklearn_processor = SKLearnProcessor( framework_version="1.2-1", role=role, instance_type="ml.m5.xlarge", instance_count=1, ) process_step = ProcessingStep( name="PreprocessData", processor=sklearn_processor, inputs=[ ProcessingInput( source=input_data, destination="/opt/ml/processing/input", ) ], outputs=[ ProcessingOutput( output_name="train", source="/opt/ml/processing/train", ), ProcessingOutput( output_name="test", source="/opt/ml/processing/test", ), ], code="scripts/preprocess.py", )

Training Steps

Training steps launch managed training jobs on SageMaker. They support built-in algorithms, custom containers, and framework estimators (PyTorch, TensorFlow, XGBoost, etc.).

# TrainingStep with XGBoost estimator from sagemaker.estimator import Estimator from sagemaker.workflow.steps import TrainingStep from sagemaker.inputs import TrainingInput xgb_estimator = Estimator( image_uri=sagemaker.image_uris.retrieve("xgboost", region, version="1.7-1"), role=role, instance_count=1, instance_type=instance_type, output_path="s3://bucket/models/", use_spot_instances=True, # up to 90% savings max_wait=7200, max_run=3600, hyperparameters={ "max_depth": "6", "eta": "0.2", "objective": "binary:logistic", "num_round": "200", }, ) train_step = TrainingStep( name="TrainXGBoost", estimator=xgb_estimator, inputs={ "train": TrainingInput( s3_data=process_step.properties.ProcessingOutputConfig .Outputs["train"].S3Output.S3Uri, content_type="text/csv", ), }, )
Warning: Spot training instances can be interrupted. Always set max_wait higher than max_run to allow time for spot instance recovery. For time-critical pipelines, use on-demand instances instead.

Model Registration

After training and evaluation, the RegisterModel step adds the model to SageMaker Model Registry with version metadata, approval status, and deployment configuration.

# RegisterModel step from sagemaker.workflow.step_collections import RegisterModel register_step = RegisterModel( name="RegisterFraudModel", estimator=xgb_estimator, model_data=train_step.properties.ModelArtifacts.S3ModelArtifacts, content_types=["text/csv"], response_types=["text/csv"], inference_instances=["ml.m5.large", "ml.m5.xlarge"], transform_instances=["ml.m5.xlarge"], model_package_group_name="FraudDetectionModels", approval_status="PendingManualApproval", model_metrics={ "accuracy": eval_accuracy, "f1_score": eval_f1, }, )

The Model Registry supports approval workflows:

  • PendingManualApproval — a human must approve before the model can be deployed
  • Approved — automatically eligible for deployment
  • Rejected — blocked from deployment, with a reason logged

Conditional Execution

SageMaker Pipelines supports conditional logic to branch pipeline execution based on step outputs. This is essential for quality gates — only register and deploy models that meet performance thresholds.

# Conditional step — only register if accuracy exceeds threshold from sagemaker.workflow.conditions import ConditionGreaterThanOrEqualTo from sagemaker.workflow.condition_step import ConditionStep from sagemaker.workflow.functions import JsonGet from sagemaker.workflow.fail_step import FailStep # Extract accuracy from evaluation output eval_accuracy = JsonGet( step_name=eval_step.name, property_file="evaluation", json_path="metrics.accuracy.value", ) condition = ConditionGreaterThanOrEqualTo( left=eval_accuracy, right=accuracy_threshold, ) fail_step = FailStep( name="ModelQualityFailed", error_message="Model accuracy below threshold", ) condition_step = ConditionStep( name="CheckAccuracy", conditions=[condition], if_steps=[register_step], # register if condition passes else_steps=[fail_step], # fail pipeline if below threshold )

SageMaker Pipelines

  • Native SageMaker integration
  • Built-in step types for common ML tasks
  • Model Registry with approval workflows
  • Conditional branching and fail steps
  • Best for AWS-native ML workflows

Step Functions for ML

  • General-purpose AWS workflow orchestration
  • Visual workflow designer
  • Broader AWS service integration (Lambda, ECS, etc.)
  • Less ML-specific abstractions
  • Best for mixed ML + non-ML workflows
