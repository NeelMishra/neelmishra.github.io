Chain-of-Thought: Reasoning in LLMs

June 2025 · 18 min read

Large language models can solve complex reasoning problems — but only if you ask them the right way. Chain-of-Thought (CoT) prompting is the technique of eliciting intermediate reasoning steps from an LLM before it arrives at a final answer. Instead of jumping to a conclusion, the model "thinks out loud," dramatically improving accuracy on arithmetic, commonsense, and symbolic reasoning tasks.

Standard Prompting vs Chain-of-Thought

In standard prompting, a model receives a question and is expected to produce an answer directly. This works well for simple factual recall but collapses on multi-step reasoning. Chain-of-Thought prompting adds exemplar reasoning traces that teach the model to decompose complex problems before answering.

Standard Prompt

Q: Roger has 5 tennis balls. He buys 2 more cans of 3. How many does he have?

A: 11

The model guesses the final number directly — no reasoning visible. Works sometimes, fails on harder problems.

CoT Prompt

Q: Roger has 5 tennis balls. He buys 2 more cans of 3. How many does he have?

A: Roger started with 5 balls. 2 cans of 3 balls each is 2 × 3 = 6 balls. 5 + 6 = 11.

The model shows its work. Each step is verifiable, and the decomposition reduces errors.

Key insight: Chain-of-Thought doesn't change the model's weights — it changes what the model attends to during generation. The reasoning trace acts as a scratchpad that keeps intermediate results in the context window.

Wei et al. (2022) showed that CoT prompting improved accuracy on GSM8K math benchmarks from 17.9% to 58.1% with PaLM 540B — a 3× improvement from prompting alone. On symbolic reasoning tasks (last letter concatenation, coin flip), the gains were even larger.

Few-Shot Chain-of-Thought (Wei et al. 2022)

The original CoT paper demonstrated few-shot CoT: you provide a handful of question–reasoning–answer exemplars in the prompt, then pose the target question. The model mimics the demonstrated reasoning style.

# Few-Shot CoT prompt template def build_fewshot_cot_prompt(examples, question): prompt = "" for ex in examples: prompt += f"Q: {ex['question']}\n" prompt += f"A: Let's work through this step by step.\n" prompt += f"{ex['reasoning']}\n" prompt += f"The answer is {ex['answer']}.\n\n" prompt += f"Q: {question}\nA: Let's work through this step by step.\n" return prompt # Example usage examples = [ { "question": "A store has 4 shelves with 8 books each. " "3 books are removed. How many remain?", "reasoning": "Total books = 4 × 8 = 32. After removing 3: 32 - 3 = 29.", "answer": "29" }, { "question": "A cafeteria had 23 apples. They used 20 for lunch " "and bought 6 more. How many do they have?", "reasoning": "Started with 23. Used 20: 23 - 20 = 3. Bought 6: 3 + 6 = 9.", "answer": "9" } ] result = build_fewshot_cot_prompt(examples, "If you have 15 coins and lose half, how many remain?")

The quality of exemplars matters significantly. Effective CoT demonstrations share several traits:

  • Diverse problem types — cover different reasoning patterns the model may encounter
  • Clear step separation — each logical step should be explicit and atomic
  • Correct reasoning — errors in exemplars propagate (the model copies the reasoning style)
  • Appropriate complexity — exemplars should match or slightly exceed target difficulty
Watch out: Few-shot CoT is sensitive to exemplar ordering and phrasing. Small changes in wording can swing accuracy by 10-20% on some benchmarks. Always evaluate multiple prompt variants.

Zero-Shot CoT: "Let's Think Step by Step"

Kojima et al. (2022) made a surprising discovery: simply appending "Let's think step by step" to a prompt — with no exemplars at all — elicits chain-of-thought reasoning. This zero-shot approach works because instruction-tuned models have internalized reasoning patterns during training.

Without Trigger

Q: A juggler can juggle 16 balls. Half are golf balls, and half of the golf balls are blue. How many blue golf balls?

A: 8 ✗

With "Let's think step by step"

Q: A juggler can juggle 16 balls. Half are golf balls, and half of the golf balls are blue. How many blue golf balls?

A: 16 balls total. Half are golf balls: 16/2 = 8. Half of golf balls are blue: 8/2 = 4 ✓

Zero-shot CoT is remarkably effective across model families. The trigger phrase works because it shifts the model's generation distribution toward step-by-step tokens rather than jumping to a final answer. Variations of the trigger include:

  • "Let's think step by step" — the original and most studied
  • "Let's work this out in a step by step way to be sure we have the right answer" — Zhou et al. found this can outperform the original
  • "Think carefully, then answer" — shorter but still effective
  • "Before answering, reason through each part of the problem" — more directive
# Zero-Shot CoT — two-stage extraction def zero_shot_cot(llm, question): # Stage 1: elicit reasoning reasoning_prompt = f"{question}\nLet's think step by step." reasoning = llm.generate(reasoning_prompt) # Stage 2: extract the final answer extract_prompt = ( f"{question}\n{reasoning}\n" f"Therefore, the answer (number only) is:" ) answer = llm.generate(extract_prompt) return answer, reasoning
Two-stage extraction is important. Zero-shot CoT often buries the final answer inside a paragraph of reasoning. Stage 2 forces the model to commit to a concise answer, making it parseable for downstream evaluation.

Self-Consistency: Majority Vote over Reasoning Paths

Wang et al. (2022) introduced self-consistency, which improves CoT by sampling multiple independent reasoning paths and taking a majority vote on the final answer. The intuition: a correct answer is more likely to be reached via diverse reasoning paths, while incorrect answers tend to be idiosyncratic.

import collections def self_consistency(llm, question, n_samples=10, temperature=0.7): """Sample multiple CoT paths and return majority answer.""" answers = [] reasoning_paths = [] for _ in range(n_samples): # Sample with temperature > 0 for diversity reasoning = llm.generate( f"{question}\nLet's think step by step.", temperature=temperature ) # Extract final answer from the reasoning answer = extract_answer(reasoning) answers.append(answer) reasoning_paths.append(reasoning) # Majority vote vote_counts = collections.Counter(answers) best_answer = vote_counts.most_common(1)[0][0] confidence = vote_counts[best_answer] / n_samples return { "answer": best_answer, "confidence": confidence, "vote_distribution": dict(vote_counts), "paths": reasoning_paths }

Self-consistency consistently outperforms single-sample CoT. On GSM8K, it pushes PaLM 540B from 58.1% (single CoT) to 74.4%. The method works because:

  • Error decorrelation: Different reasoning paths make different mistakes — averaging cancels out random errors
  • Coverage: Some paths may discover valid solution strategies that others miss
  • Calibration: The vote distribution provides a natural confidence estimate
Cost consideration: Self-consistency requires N× the API calls of single-sample CoT. With n=10 samples, you pay 10× the token cost. For production use, consider using n=5 and only escalating to n=20 for low-confidence answers.

Tree-of-Thought: Branching Exploration

Yao et al. (2023) generalized CoT from a single linear chain to a tree of thoughts. The key idea: at each reasoning step, the model generates multiple candidate continuations (branches), evaluates them, and selectively expands the most promising ones using search algorithms like BFS or DFS.

from collections import deque class TreeOfThought: def __init__(self, llm, n_branches=3, max_depth=5): self.llm = llm self.n_branches = n_branches self.max_depth = max_depth def generate_thoughts(self, state, step): """Generate candidate next-steps from current state.""" prompt = ( f"Given the problem state:\n{state}\n\n" f"Generate {self.n_branches} distinct possible next steps " f"for step {step}. Return each on a separate line." ) response = self.llm.generate(prompt) return parse_thoughts(response) def evaluate_thought(self, state, thought): """Score a candidate thought: sure / maybe / impossible.""" prompt = ( f"State: {state}\nProposed step: {thought}\n\n" f"Is this step correct and promising? " f"Reply: sure / maybe / impossible" ) return self.llm.generate(prompt).strip().lower() def solve_bfs(self, problem): """Breadth-first search over thought tree.""" queue = deque([("", problem, 0)]) # (path, state, depth) solutions = [] while queue: path, state, depth = queue.popleft() if depth >= self.max_depth: solutions.append((path, state)) continue thoughts = self.generate_thoughts(state, depth + 1) for thought in thoughts: score = self.evaluate_thought(state, thought) if score != "impossible": new_state = f"{state}\nStep {depth+1}: {thought}" new_path = f"{path} → {thought}" queue.append((new_path, new_state, depth + 1)) return self._pick_best(solutions)

Tree-of-Thought excels on problems that require exploration and backtracking — tasks where a single greedy path often leads to dead ends. Key application areas:

BFS (Breadth-First)

  • Explores all branches at each depth before going deeper
  • Guarantees finding the shallowest solution
  • Higher memory usage — keeps all frontier nodes
  • Best for: creative writing, planning tasks with many valid short paths

DFS (Depth-First)

  • Explores one branch fully before backtracking
  • Lower memory — only stores current path
  • May find deep solutions faster
  • Best for: math proofs, puzzles, game of 24, crosswords
ToT vs CoT vs Self-Consistency: Think of CoT as a single reasoning chain, self-consistency as sampling multiple independent chains, and ToT as a structured search tree where branches are evaluated and pruned at each step. ToT subsumes both: a ToT with breadth=1 and no evaluation is CoT; a ToT with independent paths and final voting is self-consistency.

When CoT Helps and When It Doesn't

Chain-of-Thought is not a universal improvement. Understanding when it helps — and when it actively hurts — is critical for production prompt engineering.

✓ CoT Helps With

  • Multi-step arithmetic — GSM8K, SVAMP, AQuA
  • Commonsense reasoning — StrategyQA, CommonsenseQA
  • Symbolic manipulation — last letter concatenation, coin flip tracking
  • Logical deduction — date understanding, sports reasoning
  • Code generation — breaking down complex algorithms step by step
  • Planning tasks — multi-constraint scheduling, game solving

✗ CoT Hurts or Doesn't Help

  • Simple factual recall — "What is the capital of France?" CoT adds latency with no benefit
  • Single-step tasks — sentiment classification, named entity recognition
  • Small models (< 10B) — CoT degrades below a scale threshold; models produce incoherent reasoning
  • Creative writing — step-by-step reasoning can make text mechanical
  • Perceptual tasks — image captioning, speech recognition
The scale threshold: CoT only emerges reliably in models with >~60B parameters (or heavily instruction-tuned smaller models). With small models, the "reasoning" is often confabulated — the model generates plausible-looking but logically broken chains that lead to worse answers than direct prompting.

A practical decision framework for choosing the right technique:

# Decision framework for CoT technique selection def choose_cot_strategy(task): if task.requires_reasoning == False: return "standard_prompting" # Don't force CoT on simple tasks if task.has_labeled_exemplars: base = "few_shot_cot" # Best accuracy with good exemplars else: base = "zero_shot_cot" # "Let's think step by step" if task.requires_high_accuracy: if task.allows_multiple_calls: base = "self_consistency" # Majority vote over N paths if task.requires_exploration: if task.search_space_is_wide: return "tree_of_thought_bfs" # Broad exploration else: return "tree_of_thought_dfs" # Deep, focused search return base

Summary

  • Chain-of-Thought prompting elicits step-by-step reasoning that dramatically improves LLM accuracy on complex tasks — without changing model weights.
  • Few-Shot CoT (Wei et al. 2022) provides exemplar reasoning traces; the model mirrors the demonstrated decomposition style.
  • Zero-Shot CoT (Kojima et al. 2022) uses the trigger phrase "Let's think step by step" — no exemplars needed, surprisingly effective.
  • Self-Consistency (Wang et al. 2022) samples multiple reasoning paths and takes a majority vote, boosting accuracy by 10-20% over single CoT.
  • Tree-of-Thought (Yao et al. 2023) structures reasoning as a search tree with branching, evaluation, and pruning — powerful for problems requiring exploration and backtracking.
  • CoT helps most on multi-step reasoning tasks with models at sufficient scale; it offers no benefit (and can harm) on simple retrieval, classification, or small models.
