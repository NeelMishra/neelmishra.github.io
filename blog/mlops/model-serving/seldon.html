Seldon Core: K8s-Native Serving

MLOps Series Model Serving & Inference

Seldon Core Overview

Seldon Core is an open-source platform for deploying ML models on Kubernetes. It uses a Custom Resource Definition (CRD) to define model deployments declaratively, with built-in support for inference graphs, A/B testing, canary rollouts, and explainability.

Key value: Seldon lets ML engineers define complex serving topologies (preprocessing → model → explainer) as Kubernetes-native resources. The operator handles scaling, routing, and traffic splitting automatically.

SeldonDeployment CRD

The SeldonDeployment CRD is the fundamental building block. It defines predictors, each containing a graph of model components.

# Basic SeldonDeployment
apiVersion: machinelearning.seldon.io/v1
kind: SeldonDeployment
metadata:
  name: fraud-detector
  namespace: ml-serving
spec:
  predictors:
  - name: default
    replicas: 3
    graph:
      name: fraud-model
      implementation: SKLEARN_SERVER
      modelUri: gs://models/fraud/v1
      envSecretRefName: gcs-credentials
    componentSpecs:
    - spec:
        containers:
        - name: fraud-model
          resources:
            requests:
              memory: "2Gi"
              cpu: "1"
            limits:
              memory: "4Gi"
              nvidia.com/gpu: 1

Pre-built Servers

Framework Servers

  • SKLEARN_SERVER — scikit-learn
  • XGBOOST_SERVER — XGBoost
  • TENSORFLOW_SERVER — TensorFlow
  • MLFLOW_SERVER — MLflow models

Advanced Servers

  • TRITON_SERVER — NVIDIA Triton
  • HUGGINGFACE_SERVER — Transformers
  • TEMPO_SERVER — Custom pipelines
  • CUSTOM — Your own container

Inference Graphs

Seldon's inference graphs let you compose complex pipelines: transformers → predictors → combiners → output transformers — all defined declaratively.

# Inference graph: transformer → model → output transformer
apiVersion: machinelearning.seldon.io/v1
kind: SeldonDeployment
metadata:
  name: nlp-pipeline
spec:
  predictors:
  - name: default
    graph:
      name: output-transform
      type: OUTPUT_TRANSFORMER
      children:
      - name: classifier
        type: MODEL
        implementation: SKLEARN_SERVER
        modelUri: s3://models/sentiment/v2
        children:
        - name: tokenizer
          type: TRANSFORMER
          implementation: CUSTOM
          image: myrepo/tokenizer:v1

# Client → tokenizer → classifier → output-transform → Client

A/B Testing & Canary

Seldon supports traffic splitting between predictors natively — no service mesh configuration required.

# A/B test: 90% stable, 10% experimental
spec:
  predictors:
  - name: stable
    replicas: 3
    traffic: 90
    graph:
      name: model-v1
      implementation: SKLEARN_SERVER
      modelUri: s3://models/v1
  - name: experimental
    replicas: 1
    traffic: 10
    graph:
      name: model-v2
      implementation: SKLEARN_SERVER
      modelUri: s3://models/v2
Gradual rollout: Start with 1-5% traffic to the new model. Monitor error rates and prediction distributions before increasing. Seldon's built-in Prometheus metrics make this straightforward.

Custom Servers

# Custom Python model server
class FraudModel:
    def __init__(self):
        self.model = load_model()

    def predict(self, X, features_names=None):
        """Called for each request batch."""
        return self.model.predict_proba(X)

    def send_feedback(self, features, feature_names, reward, truth):
        """Called when feedback/label is received."""
        self.log_feedback(features, reward, truth)

    def tags(self):
        return {"model_version": "v2", "team": "fraud"}

    def metrics(self):
        return [{"type": "GAUGE", "key": "model_score", "value": 0.95}]

Monitoring & Explainability

Seldon integrates with Prometheus for metrics, Elasticsearch for request logging, and provides an explainer component for model interpretability.

# Add explainer to deployment
spec:
  predictors:
  - name: default
    graph:
      name: classifier
      implementation: SKLEARN_SERVER
      modelUri: s3://models/v1
    explainer:
      type: AnchorTabular
      modelUri: s3://explainers/v1

# Query explanation endpoint
curl -X POST http://seldon/seldon/ml-serving/fraud-detector/api/v1.0/explain \
  -H "Content-Type: application/json" \
  -d '{"data": {"ndarray": [[0.5, 1.2, 3.4, 0.8]]}}'
Seldon Deploy (Enterprise): Adds a UI dashboard, drift detection, outlier detection, and centralized model management on top of open-source Seldon Core.

Production Patterns

Installation

# Install Seldon Core operator
helm install seldon-core seldon-core-operator \
  --repo https://storage.googleapis.com/seldon-charts \
  --namespace seldon-system --create-namespace \
  --set usageMetrics.enabled=true \
  --set istio.enabled=true \
  --set ambassador.enabled=false

Resource Management

# HPA for Seldon deployments
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: fraud-detector-hpa
spec:
  scaleTargetRef:
    apiVersion: machinelearning.seldon.io/v1
    kind: SeldonDeployment
    name: fraud-detector
  minReplicas: 2
  maxReplicas: 20
  metrics:
  - type: Pods
    pods:
      metric:
        name: seldon_api_executor_server_requests_seconds_count
      target:
        type: AverageValue
        averageValue: "100"
