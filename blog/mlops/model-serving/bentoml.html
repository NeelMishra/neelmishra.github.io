BentoML: Build and Ship ML Services

MLOps Series Model Serving & Inference

Why BentoML?

BentoML bridges the gap between model training and production deployment. It provides a Pythonic, developer-friendly way to package models as production-ready services — with built-in adaptive batching, multi-model composition, and one-click deployment to any cloud.

✅ Strengths

  • Pythonic API — feels like writing FastAPI
  • Framework-agnostic (PyTorch, TF, sklearn, XGBoost)
  • Built-in adaptive batching
  • Containerize with one command
  • BentoCloud for managed deployment

⚠️ Trade-offs

  • Less GPU optimization than Triton
  • Smaller ecosystem than KServe
  • No native Kubernetes CRD
  • Newer — less battle-tested at massive scale

Service Decorator

The @bentoml.service decorator turns a Python class into a deployable ML service. Each method decorated with @bentoml.api becomes an HTTP endpoint.

# service.py — BentoML service definition
import bentoml
import numpy as np
from typing import List

@bentoml.service(
    resources={"gpu": 1, "memory": "4Gi"},
    traffic={"timeout": 60, "concurrency": 32},
)
class FraudDetector:
    def __init__(self):
        import torch
        self.model = bentoml.pytorch.load_model("fraud_model:latest")
        self.device = torch.device("cuda")
        self.model.to(self.device).eval()

    @bentoml.api
    async def predict(self, features: List[float]) -> dict:
        import torch
        tensor = torch.tensor([features], device=self.device)
        with torch.no_grad():
            score = self.model(tensor)
        return {"fraud_score": score.item()}

    @bentoml.api
    async def batch_predict(self, batch: List[List[float]]) -> List[dict]:
        import torch
        tensor = torch.tensor(batch, device=self.device)
        with torch.no_grad():
            scores = self.model(tensor)
        return [{"fraud_score": s.item()} for s in scores]

Runners & Adaptive Batching

BentoML's runner system provides automatic adaptive batching. Individual requests are transparently combined into batches based on load.

# Saving a model to BentoML model store
import bentoml
import torch

model = MyFraudModel()
model.load_state_dict(torch.load("weights.pt"))

saved_model = bentoml.pytorch.save_model(
    "fraud_model",
    model,
    signatures={"__call__": {"batchable": True, "batch_dim": 0}},
    labels={"team": "fraud", "stage": "production"},
    metadata={"accuracy": 0.97, "dataset": "v3"}
)
print(f"Saved: {saved_model.tag}")
Adaptive batching: Unlike static batching (fixed batch size + timeout), BentoML dynamically adjusts based on request arrival rate and model inference time. High load → larger batches; low load → immediate processing.

Bento Packaging

A Bento is BentoML's deployment artifact — it bundles the service code, model artifacts, Python dependencies, and Docker configuration.

# bentofile.yaml
service: "service:FraudDetector"
labels:
  team: fraud-detection
  env: production

include:
  - "*.py"

python:
  requirements_txt: "./requirements.txt"

docker:
  distro: debian
  cuda_version: "12.1"
  system_packages:
    - libgomp1
# Build and manage Bentos
bentoml build                     # Package into a Bento
bentoml list                      # List all Bentos
bentoml containerize fraud_detector:latest  # Build Docker image
bentoml push fraud_detector:latest          # Push to BentoCloud

Deployment Options

☁️ BentoCloud (Managed)

  • bentoml deploy .
  • Auto-scaling, zero to N
  • Built-in monitoring
  • A/B testing support

🐳 Docker / K8s (Self-managed)

  • bentoml containerize
  • Standard Kubernetes deployment
  • Helm charts available
  • Full infrastructure control
# Kubernetes deployment with generated container
apiVersion: apps/v1
kind: Deployment
metadata:
  name: fraud-detector
spec:
  replicas: 3
  template:
    spec:
      containers:
      - name: fraud-detector
        image: fraud_detector:latest
        ports:
        - containerPort: 3000
        resources:
          limits:
            nvidia.com/gpu: 1
            memory: "4Gi"
        readinessProbe:
          httpGet:
            path: /healthz
            port: 3000

Multi-Model Composition

# Composing multiple models in one service
@bentoml.service
class SearchPipeline:
    embedder = bentoml.depends(EmbeddingService)
    ranker = bentoml.depends(RankingService)

    @bentoml.api
    async def search(self, query: str) -> List[dict]:
        embedding = await self.embedder.encode(query)
        candidates = retrieve_candidates(embedding)
        ranked = await self.ranker.rank(query, candidates)
        return ranked[:10]
Best practice: Use bentoml.depends() for model composition instead of loading multiple models in one service. This gives independent scaling — your embedder can scale differently from your ranker.
