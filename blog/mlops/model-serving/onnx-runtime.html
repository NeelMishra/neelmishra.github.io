ONNX & ONNX Runtime

ONNX (Open Neural Network Exchange) is a standard format for representing ML models. ONNX Runtime (ORT) is Microsoft's high-performance inference engine that runs ONNX models on any hardware — CPU, GPU, NPU, or edge devices.

ONNX Format

  • Framework-agnostic model format
  • Export from PyTorch, TF, sklearn
  • Versioned operator sets
  • Graph-based representation

ONNX Runtime

  • Cross-platform inference engine
  • Automatic graph optimizations
  • Hardware-specific execution providers
  • 1.2-5× faster than native frameworks

Exporting to ONNX

# Export PyTorch model to ONNX
import torch

model = MyModel()
model.eval()
dummy_input = torch.randn(1, 3, 224, 224)

torch.onnx.export(
    model, dummy_input, "model.onnx",
    input_names=["image"],
    output_names=["logits"],
    dynamic_axes={
        "image": {0: "batch_size"},
        "logits": {0: "batch_size"}
    },
    opset_version=17
)
# Export sklearn model to ONNX
from skl2onnx import convert_sklearn
from skl2onnx.common.data_types import FloatTensorType

model = RandomForestClassifier()
model.fit(X_train, y_train)

onnx_model = convert_sklearn(
    model, "random_forest",
    [("features", FloatTensorType([None, 10]))]
)
with open("model.onnx", "wb") as f:
    f.write(onnx_model.SerializeToString())

Graph Optimizations

ORT automatically applies graph optimizations that fuse operations, eliminate redundancies, and reshape computation graphs for better performance.

# Run with optimizations
import onnxruntime as ort

# Level 0: No optimization
# Level 1: Basic (constant folding, redundant node elimination)
# Level 2: Extended (operator fusion, layout optimization)
# Level 99: All (includes hardware-specific optimizations)

opts = ort.SessionOptions()
opts.graph_optimization_level = ort.GraphOptimizationLevel.ORT_ENABLE_ALL
opts.optimized_model_filepath = "model_optimized.onnx"

session = ort.InferenceSession("model.onnx", opts)

# Key optimizations applied automatically:
# - Conv + BatchNorm fusion
# - MatMul + Add → Gemm fusion
# - Attention layer fusion (for transformers)
# - Constant folding
# - Node elimination (identity, dropout in eval)
Transformer optimization: ORT's transformer-specific optimizations (attention fusion, skip layer norm fusion) can deliver 2-3× speedup for BERT/GPT models compared to naive ONNX execution.

Execution Providers

Execution providers (EPs) are hardware-specific backends that ORT dispatches operations to. Use the right EP for your hardware.

# GPU inference with CUDA EP
session = ort.InferenceSession(
    "model.onnx",
    providers=["CUDAExecutionProvider", "CPUExecutionProvider"]
)

# TensorRT EP (fastest for NVIDIA GPUs)
session = ort.InferenceSession(
    "model.onnx",
    providers=[
        ("TensorrtExecutionProvider", {
            "trt_max_workspace_size": 2 * 1024 * 1024 * 1024,
            "trt_fp16_enable": True,
        }),
        "CUDAExecutionProvider",
        "CPUExecutionProvider"
    ]
)

# Run inference
result = session.run(
    ["logits"],
    {"image": input_array}
)

GPU Providers

  • CUDA — NVIDIA GPUs
  • TensorRT — Optimized NVIDIA
  • ROCm — AMD GPUs
  • DirectML — Windows GPU

CPU / Edge Providers

  • CPU — Default, everywhere
  • OpenVINO — Intel hardware
  • CoreML — Apple Silicon
  • NNAPI — Android NPU

ONNX Quantization

# Static quantization with calibration
from onnxruntime.quantization import quantize_static, CalibrationDataReader

class MyCalibReader(CalibrationDataReader):
    def __init__(self, data):
        self.data = iter(data)

    def get_next(self):
        try:
            return {"image": next(self.data)}
        except StopIteration:
            return None

quantize_static(
    model_input="model.onnx",
    model_output="model_int8.onnx",

Deployment Patterns

# Production inference with session pooling
import onnxruntime as ort

# Configure for production
opts = ort.SessionOptions()
opts.inter_op_num_threads = 4
opts.intra_op_num_threads = 4
opts.execution_mode = ort.ExecutionMode.ORT_PARALLEL
opts.graph_optimization_level = ort.GraphOptimizationLevel.ORT_ENABLE_ALL
opts.enable_mem_pattern = True
opts.enable_cpu_mem_arena = True

session = ort.InferenceSession("model_int8.onnx", opts,
    providers=["CUDAExecutionProvider"])

# Use io_binding for zero-copy GPU inference
binding = session.io_binding()
binding.bind_input("image", "cuda", 0, np.float32, [1,3,224,224], gpu_buffer.data_ptr())
binding.bind_output("logits", "cuda")
session.run_with_iobinding(binding)
ONNX as interchange: Train in PyTorch → export to ONNX → optimize with ORT → deploy anywhere. This decouples your training framework from your inference stack, letting you use the best tool for each.
