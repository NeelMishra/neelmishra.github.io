Anatomy of Inference Latency

End-to-end latency isn't just model inference. It's the sum of every stage: network transfer, deserialization, preprocessing, feature lookup, model execution, postprocessing, and serialization.

# Typical latency breakdown (100ms total budget)
Network (client → server)   : ████        ~15ms
Deserialization              : █           ~3ms
Feature store lookup         : ████        ~12ms
Preprocessing                : ███         ~10ms
Model inference (GPU)        : ████████    ~35ms
Postprocessing               : ██          ~5ms
Serialization                : █           ~3ms
Network (server → client)    : ████        ~15ms
Overhead (queuing, routing)  : █           ~2ms
                               Total:      ~100ms
Key insight: Many teams focus only on model inference (35ms above) and ignore everything else (65ms). The non-model overhead is often the majority of latency. Profile the full path before optimizing.

Profiling Techniques

# End-to-end profiling with instrumentation
import time
from contextlib import contextmanager

@contextmanager
def timer(stage_name, metrics):
    start = time.perf_counter_ns()
    yield
    elapsed_ms = (time.perf_counter_ns() - start) / 1e6
    metrics[stage_name] = elapsed_ms

async def profiled_predict(request):
    metrics = {}

    with timer("deserialize", metrics):
        features = parse_request(request)

    with timer("feature_lookup", metrics):
        enriched = await feature_store.get(features.user_id)

    with timer("preprocess", metrics):
        tensor = preprocess(features, enriched)

    with timer("inference", metrics):
        output = model(tensor)

    with timer("postprocess", metrics):
        result = postprocess(output)

    # Log detailed timing
    emit_metrics(metrics)
    return result

Protocol Optimization

  • gRPC over REST (2-5× faster)
  • HTTP/2 multiplexing
  • Connection pooling
  • Keep-alive connections

Payload Optimization

  • Protobuf binary serialization
  • Compress large payloads (gzip)
  • Send indices, not full vectors
  • Batch multiple requests

Postprocessing Optimization

# Common postprocessing bottlenecks

# ❌ Slow: Python loop over predictions
results = []
for logit in output:
    prob = softmax(logit)
    label = labels[argmax(prob)]
    results.append({"label": label, "score": max(prob)})

# ✅ Fast: Vectorized operations
probs = torch.softmax(output, dim=-1)
scores, indices = probs.max(dim=-1)
results = [{"label": labels[i], "score": s.item()}
           for i, s in zip(indices, scores)]

Optimization Checklist

  • ✅ Profile first — measure each stage before optimizing
  • ✅ Use gRPC instead of REST for internal services
  • ✅ Move preprocessing to GPU (DALI, torchvision transforms)
  • ✅ Apply TensorRT or torch.compile for model optimization
  • ✅ Enable dynamic batching in your serving framework
  • ✅ Quantize to INT8 for 2-4× speedup
  • ✅ Cache repeated computations (embeddings, features)
  • ✅ Use async feature store lookups (don't block on I/O)
  • ✅ Set p99 latency budgets per stage
  • ✅ Monitor tail latency, not just averages
Amdahl's Law applies: If inference is 35% of total latency, even a 10× inference speedup only reduces total latency by 31%. Always optimize the biggest bottleneck first — it's often not the model itself.
