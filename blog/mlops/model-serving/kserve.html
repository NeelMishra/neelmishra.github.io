KServe: Serverless ML on Kubernetes

MLOps Series Model Serving & Inference

KServe Overview

KServe (formerly KFServing) is the Kubernetes standard for serverless ML inference. Built on Knative, it provides scale-to-zero, GPU autoscaling, and a standardized inference protocol across all major frameworks.

✅ Key Features

  • Scale-to-zero with Knative
  • V2 Inference Protocol (standardized)
  • Multi-framework support
  • Canary rollouts built-in
  • ModelMesh for high-density serving

📦 Components

  • Predictor — runs the model
  • Transformer — pre/post processing
  • Explainer — model interpretability
  • InferenceGraph — DAG routing

InferenceService CRD

The InferenceService CRD is the primary abstraction. A single YAML definition creates the full serving infrastructure — pods, services, ingress, autoscaler, and model loading.

# Basic InferenceService
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: fraud-detector
  namespace: ml-serving
spec:
  predictor:
    model:
      modelFormat:
        name: sklearn
      storageUri: "s3://models/fraud/v1"
      resources:
        requests:
          cpu: "1"
          memory: "2Gi"
        limits:
          cpu: "2"
          memory: "4Gi"

Supported Model Formats

# Different model formats
# scikit-learn
modelFormat: { name: sklearn }
storageUri: "s3://models/sklearn/v1"

# PyTorch (TorchServe backend)
modelFormat: { name: pytorch }
storageUri: "gs://models/pytorch/v1"

# TensorFlow
modelFormat: { name: tensorflow }
storageUri: "s3://models/tf/v1"

# XGBoost
modelFormat: { name: xgboost }
storageUri: "s3://models/xgb/v1"

# NVIDIA Triton (any format Triton supports)
modelFormat: { name: triton }
storageUri: "s3://models/triton-repo/"

Predictor / Transformer / Explainer

KServe's three-component architecture cleanly separates concerns in the inference pipeline.

# Full InferenceService with all components
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: nlp-classifier
spec:
  transformer:
    containers:
    - name: tokenizer
      image: myrepo/tokenizer:v1
      resources:
        requests: { cpu: "500m", memory: "1Gi" }
  predictor:
    model:
      modelFormat: { name: pytorch }
      storageUri: "s3://models/bert-classifier/v2"
      resources:
        limits: { nvidia.com/gpu: 1 }
  explainer:
    containers:
    - name: shap-explainer
      image: myrepo/shap-explainer:v1

Autoscaling & Scale-to-Zero

KServe leverages Knative's autoscaling capabilities, including scale-to-zero for cost savings and GPU-aware scaling.

# Autoscaling annotations
metadata:
  annotations:
    # Scale to zero after 5 minutes of inactivity
    autoscaling.knative.dev/target: "10"
    autoscaling.knative.dev/metric: "concurrency"
    autoscaling.knative.dev/scale-to-zero-pod-retention-period: "5m"

    # Or use RPS-based scaling
    autoscaling.knative.dev/target: "100"
    autoscaling.knative.dev/metric: "rps"

    # Scaling bounds
    autoscaling.knative.dev/min-scale: "0"
    autoscaling.knative.dev/max-scale: "20"
Cold start warning: Scale-to-zero means the first request after idle period will experience a cold start (model loading). For GPU models, this can be 30-120 seconds. Set min-scale: 1 for latency-sensitive services.

Canary Rollouts

# Canary: 90% v1, 10% v2
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: fraud-detector
spec:
  predictor:
    canaryTrafficPercent: 10
    model:
      modelFormat: { name: sklearn }
      storageUri: "s3://models/fraud/v2"   # New version

# Promote canary to 100%
kubectl patch inferenceservice fraud-detector \
  --type=merge -p '{"spec":{"predictor":{"canaryTrafficPercent":100}}}'

# Rollback — set canary to 0%
kubectl patch inferenceservice fraud-detector \
  --type=merge -p '{"spec":{"predictor":{"canaryTrafficPercent":0}}}'

Advanced Features

InferenceGraph (DAG Routing)

# Route requests based on conditions
apiVersion: serving.kserve.io/v1alpha1
kind: InferenceGraph
metadata:
  name: smart-router
spec:
  nodes:
    root:
      routerType: Sequence
      steps:
      - serviceName: language-detector
        name: detect-lang
      - condition: "[@.lang == 'en']"
        serviceName: english-model
      - condition: "[@.lang == 'es']"
        serviceName: spanish-model

Inference Protocol V2

# V2 protocol — standardized across all servers
# Health check
GET /v2/health/ready

# Model metadata
GET /v2/models/fraud-detector

# Inference
POST /v2/models/fraud-detector/infer
{
  "inputs": [{
    "name": "features",
    "shape": [1, 10],
    "datatype": "FP32",
    "data": [0.1, 0.5, 3.2, ...]
  }]
}

Installation & Setup

# Install KServe with Knative (serverless mode)
kubectl apply -f https://github.com/kserve/kserve/releases/download/v0.12.0/kserve.yaml
kubectl apply -f https://github.com/kserve/kserve/releases/download/v0.12.0/kserve-cluster-resources.yaml

# Install in RawDeployment mode (no Knative dependency)
kubectl apply -f kserve-install.yaml
kubectl patch configmap/inferenceservice-config -n kserve \
  --type=merge -p '{"data":{"deploy":"{\"defaultDeploymentMode\": \"RawDeployment\"}"}}'
Serverless vs Raw: Knative mode gives you scale-to-zero and autoscaling out of the box but requires Knative + Istio. RawDeployment mode works with standard Kubernetes deployments — simpler to set up but you manage scaling yourself.
