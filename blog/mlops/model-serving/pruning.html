Model Pruning: Structured and Unstructured

MLOps Series Model Serving & Inference

Why Prune Models?

Most neural networks are over-parameterized — 80-90% of weights can be removed with minimal accuracy impact. Pruning removes these redundant parameters, yielding smaller, faster models for inference.

Key insight: Pruning is complementary to quantization. Pruning reduces the number of operations; quantization reduces the cost per operation. Combined, they can deliver 10-100× speedup over the original FP32 model.

Unstructured Pruning

Unstructured pruning sets individual weights to zero, creating sparse weight matrices. It achieves the highest compression ratios but requires sparse computation support for actual speedup.

# Unstructured pruning with PyTorch
import torch
import torch.nn.utils.prune as prune

model = MyModel()

# Prune 50% of weights in each linear layer
for name, module in model.named_modules():
    if isinstance(module, torch.nn.Linear):
        prune.l1_unstructured(module, name="weight", amount=0.5)

# Check sparsity
total_zeros = sum((p == 0).sum().item() for p in model.parameters())
total_params = sum(p.numel() for p in model.parameters())
print(f"Sparsity: {total_zeros/total_params:.1%}")

# Make pruning permanent
for name, module in model.named_modules():
    if isinstance(module, torch.nn.Linear):
        prune.remove(module, "weight")
Sparse computation gap: Most hardware doesn't accelerate sparse matrices well. NVIDIA A100 supports 2:4 structured sparsity natively (2× speedup), but general unstructured sparsity gets no speedup on most GPUs.

Structured Pruning

Structured pruning removes entire channels, filters, or attention heads — producing a genuinely smaller dense model that runs faster on any hardware without special sparse support.

Unstructured

  • Individual weight zeros
  • Higher compression ratio
  • Needs sparse hardware/libraries
  • Irregular memory access

Structured

  • Entire channels/filters removed
  • Works on all hardware
  • Real speedup guaranteed
  • Lower compression ratio
# Structured pruning — remove entire channels
import torch.nn.utils.prune as prune

# Remove 30% of output channels (structured along dim=0)
for name, module in model.named_modules():
    if isinstance(module, torch.nn.Conv2d):
        prune.ln_structured(
            module, name="weight",
            amount=0.3, n=2, dim=0  # L2 norm, output channels
        )

# After pruning, rebuild the model with smaller layers
def rebuild_model(pruned_model):
    """Create a compact model without zero channels."""
    new_model = CompactModel()
    for (name, pruned_layer), new_layer in zip(
        pruned_model.named_modules(), new_model.modules()
    ):
        mask = pruned_layer.weight.sum(dim=[1,2,3]) != 0
        new_layer.weight.data = pruned_layer.weight[mask]
    return new_model

Magnitude Pruning

The simplest and most effective criterion: remove weights with the smallest absolute values. Works surprisingly well because small weights contribute least to the output.

# Iterative magnitude pruning (IMP)
def iterative_magnitude_pruning(model, target_sparsity=0.9, steps=10):
    per_step = 1 - (1 - target_sparsity) ** (1 / steps)

    for step in range(steps):
        # Prune
        for module in model.modules():
            if hasattr(module, "weight"):
                prune.l1_unstructured(module, "weight", amount=per_step)

        # Fine-tune to recover accuracy
        fine_tune(model, train_loader, epochs=2)

        sparsity = compute_sparsity(model)
        accuracy = evaluate(model, test_loader)
        print(f"Step {step}: sparsity={sparsity:.1%}, acc={accuracy:.3f}")

    return model

Movement Pruning

Movement pruning considers not just magnitude but how weights change during fine-tuning. Weights that move toward zero are pruned, while small weights that grow are kept.

# Movement pruning concept
class MovementPruner:
    def compute_scores(self, model):
        scores = {}
        for name, param in model.named_parameters():
            if "weight" in name:
                # Score = weight × gradient (movement direction)
                # Negative score → weight moving toward zero → prune it
                scores[name] = (param.data * param.grad).abs()
        return scores

    def prune(self, model, sparsity):
        scores = self.compute_scores(model)
        threshold = self.compute_threshold(scores, sparsity)
        for name, param in model.named_parameters():
            if name in scores:
                mask = scores[name] > threshold
                param.data *= mask.float()
Movement > magnitude for fine-tuned models: When adapting a pretrained model (BERT, GPT), movement pruning significantly outperforms magnitude pruning because it identifies which pretrained weights are important for the downstream task.

Lottery Ticket Hypothesis

The Lottery Ticket Hypothesis (Frankle & Carlin, 2019) states that dense networks contain sparse subnetworks ("winning tickets") that, when trained in isolation from the same initialization, match the full network's accuracy.

# Lottery Ticket algorithm
def find_winning_ticket(model_class, data, target_sparsity):
    # 1. Initialize network
    model = model_class()
    initial_weights = copy_weights(model)

    # 2. Train to completion
    train(model, data, epochs=50)

    # 3. Prune smallest weights
    mask = magnitude_prune(model, target_sparsity)

    # 4. Reset to initial weights (rewinding)
    model = model_class()
    load_weights(model, initial_weights)
    apply_mask(model, mask)

    # 5. Retrain the sparse subnetwork
    train(model, data, epochs=50, mask=mask)

    return model, mask

Combining Pruning + Quantization

# Optimal compression pipeline
def compress_model(model, train_loader, test_loader):
    # Step 1: Structured pruning (30% channels)
    pruned = structured_prune(model, amount=0.3)
    fine_tune(pruned, train_loader, epochs=5)

    # Step 2: Unstructured pruning (50% of remaining)
    pruned = magnitude_prune(pruned, amount=0.5)
    fine_tune(pruned, train_loader, epochs=3)

    # Step 3: INT8 quantization
    quantized = quantize_int8(pruned, calibration_loader)

    # Result: ~10× smaller, ~5× faster
    print(f"Size reduction: {original_size / compressed_size:.1f}×")
    print(f"Accuracy: {evaluate(quantized, test_loader):.3f}")
    return quantized
Production recommendation: For guaranteed speedup, use structured pruning + quantization. Unstructured pruning is more aggressive but only pays off if your deployment target supports sparse computation (NVIDIA A100 2:4 sparsity, or CPU with sparse BLAS).
