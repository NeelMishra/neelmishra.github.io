Distillation Concept

Knowledge distillation trains a small "student" model to mimic a large "teacher" model. The student learns not just the correct answers (hard labels) but the teacher's full probability distribution (soft labels), which encode rich inter-class relationships.

Why soft labels? Hard label [1, 0, 0] says "this is a cat." Soft label [0.7, 0.2, 0.1] says "this is a cat, somewhat like a dog, nothing like a car." This inter-class similarity information is the "dark knowledge" the student learns.

Soft Labels & Temperature

Temperature scaling controls the softness of the probability distribution. Higher temperature produces softer distributions that expose more inter-class relationships.

# Temperature-scaled softmax
import torch
import torch.nn.functional as F

def soft_labels(logits, temperature):
    """Higher temperature → softer distribution."""
    return F.softmax(logits / temperature, dim=-1)

# Example: logits = [5.0, 3.0, 0.1]
# T=1: [0.84, 0.11, 0.05]  — peaked (hard)
# T=3: [0.49, 0.30, 0.21]  — softer
# T=5: [0.41, 0.33, 0.26]  — very soft

# Typical temperature range: 2-10
# Higher T for more inter-class info transfer

Training Process

The distillation loss combines the standard cross-entropy loss (hard labels) with a KL divergence loss matching student and teacher soft distributions.

# Knowledge distillation training loop
import torch
import torch.nn.functional as F

def distillation_loss(student_logits, teacher_logits, labels, T=4.0, alpha=0.7):
    # Soft loss: match teacher's distribution
    soft_loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=-1),
        F.softmax(teacher_logits / T, dim=-1),
        reduction="batchmean"
    ) * (T * T)  # Scale by T² to match gradient magnitude

    # Hard loss: standard classification
    hard_loss = F.cross_entropy(student_logits, labels)

    # Weighted combination
    return alpha * soft_loss + (1 - alpha) * hard_loss

# Training loop
teacher.eval()
for batch, labels in train_loader:
    with torch.no_grad():
        teacher_logits = teacher(batch)
    student_logits = student(batch)
    loss = distillation_loss(student_logits, teacher_logits, labels)
    loss.backward()
    optimizer.step()
Temperature and alpha tuning: T=4, alpha=0.7 is a good starting point. Higher alpha emphasizes soft labels (teacher knowledge). Higher T reveals more inter-class relationships but can be too noisy for very different classes.

Feature Distillation

Beyond output logits, the student can also learn to match the teacher's intermediate feature representations. This transfers deeper structural knowledge.

# Feature distillation — match intermediate layers
class FeatureDistillationLoss(torch.nn.Module):
    def __init__(self, teacher_dim, student_dim):
        super().__init__()
        # Projection to align dimensions
        self.projector = torch.nn.Linear(student_dim, teacher_dim)

    def forward(self, student_features, teacher_features):
        projected = self.projector(student_features)
        # Match normalized features (cosine similarity)
        s_norm = F.normalize(projected, dim=-1)
        t_norm = F.normalize(teacher_features, dim=-1)
        return (1 - (s_norm * t_norm).sum(dim=-1)).mean()

# Total loss = output_distillation + β × feature_distillation

Distillation Variants

Response-Based

  • Match output logits / probabilities
  • Simplest to implement
  • KL divergence loss
  • Example: Hinton et al. (2015)

Feature-Based

  • Match intermediate representations
  • FitNets, PKT, NST
  • Requires layer alignment
  • Better for architecture mismatch

Relation-Based

  • Match sample relationships
  • Pairwise / triplet similarities
  • CRD, RKD methods
  • Captures structural knowledge

Self-Distillation

  • Same architecture teacher & student
  • Born-Again Networks
  • Train → distill → retrain
  • Surprisingly effective

Multi-Teacher Distillation

# Ensemble of teachers
def multi_teacher_loss(student_logits, teacher_logits_list, labels, T=4):
    # Average soft labels from multiple teachers
    avg_teacher = torch.stack([
        F.softmax(t / T, dim=-1) for t in teacher_logits_list
    ]).mean(dim=0)

    soft_loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=-1),
        avg_teacher, reduction="batchmean"
    ) * (T * T)

    hard_loss = F.cross_entropy(student_logits, labels)
    return 0.7 * soft_loss + 0.3 * hard_loss
Distillation + Pruning + Quantization: The ultimate compression pipeline. Distill to a smaller architecture → prune redundant weights → quantize to INT8. This can yield 50-100× compression with 1-3% accuracy drop.
