Multi-Armed Bandits for Model Selection

MLOps Series Model Serving & Inference

The Explore-Exploit Problem

When deploying multiple model variants, you face a trade-off: explore (try different models to learn which is best) vs exploit (route traffic to the model you currently believe is best). Multi-armed bandits (MAB) solve this optimally — minimizing the cost of exploration while converging to the best model.

A/B testing waste: In a 50/50 A/B test, half your traffic goes to the inferior model for the entire test duration. Bandits adaptively shift traffic toward the winning model, reducing regret — the total cost of serving suboptimal predictions.

Epsilon-Greedy

The simplest bandit strategy: exploit the best-known model (1-ε) of the time, and explore a random model ε of the time.

# Epsilon-greedy model selection
import random

class EpsilonGreedy:
    def __init__(self, models, epsilon=0.1):
        self.models = models
        self.epsilon = epsilon
        self.rewards = {m: [] for m in models}

    def select_model(self):
        if random.random() < self.epsilon:
            return random.choice(self.models)  # Explore
        return max(self.models, key=lambda m:
            mean(self.rewards[m]) if self.rewards[m] else 0
        )  # Exploit

    def update(self, model, reward):
        self.rewards[model].append(reward)

Upper Confidence Bound (UCB)

UCB balances exploration and exploitation by choosing the model with the highest upper confidence bound — rewarding both high average reward and uncertainty (under-explored models).

# UCB1 algorithm
import math

class UCB1:
    def __init__(self, models):
        self.models = models
        self.counts = {m: 0 for m in models}
        self.rewards = {m: 0.0 for m in models}
        self.total = 0

    def select_model(self):
        # First: try each model once
        for m in self.models:
            if self.counts[m] == 0:
                return m

        # UCB1 score = mean_reward + sqrt(2 * ln(total) / count)
        def ucb_score(m):
            mean = self.rewards[m] / self.counts[m]
            exploration = math.sqrt(2 * math.log(self.total) / self.counts[m])
            return mean + exploration

        return max(self.models, key=ucb_score)

    def update(self, model, reward):
        self.counts[model] += 1
        self.rewards[model] += reward
        self.total += 1

Thompson Sampling

Thompson sampling is a Bayesian approach: maintain a probability distribution over each model's reward rate, sample from each, and select the model with the highest sample. It naturally balances exploration (uncertain models get higher variance samples) and exploitation.

# Thompson Sampling for model selection
import numpy as np

class ThompsonSampling:
    def __init__(self, models):
        self.models = models
        # Beta distribution parameters (successes, failures)
        self.alpha = {m: 1 for m in models}  # Prior: uniform
        self.beta = {m: 1 for m in models}

    def select_model(self):
        # Sample from each model's Beta distribution
        samples = {
            m: np.random.beta(self.alpha[m], self.beta[m])
            for m in self.models
        }
        return max(samples, key=samples.get)

    def update(self, model, reward):
        # reward is 0 or 1 (binary outcome)
        if reward:
            self.alpha[model] += 1
        else:
            self.beta[model] += 1
Thompson sampling is often the best choice for model selection. It converges faster than epsilon-greedy, is simpler to implement than UCB variants, and naturally handles non-stationary rewards (model performance can change over time).

Contextual Bandits

Standard bandits assume one model is globally best. Contextual bandits learn that different models may be best for different contexts (user segments, input types, time of day).

# Contextual bandit — model selection based on features
class ContextualBandit:
    def __init__(self, models, context_dim):
        self.models = models
        # Linear model per arm predicting expected reward
        self.weights = {m: np.zeros(context_dim) for m in models}
        self.covariance = {m: np.eye(context_dim) for m in models}

    def select_model(self, context):
        # LinUCB: predict reward + confidence bound
        scores = {}
        for m in self.models:
            mean = context @ self.weights[m]
            cov_inv = np.linalg.inv(self.covariance[m])
            uncertainty = np.sqrt(context @ cov_inv @ context)
            scores[m] = mean + 2.0 * uncertainty
        return max(scores, key=scores.get)

Bandits vs A/B Testing

A/B Testing

  • Fixed traffic split (50/50)
  • Runs for predetermined duration
  • Strong statistical guarantees
  • Higher regret (more traffic to loser)
  • Best for: measuring precise effect sizes

Multi-Armed Bandits

  • Adaptive traffic allocation
  • Continuously optimizes
  • Lower regret (less wasted traffic)
  • Weaker statistical inference
  • Best for: optimizing real-time outcomes
When NOT to use bandits: If you need precise effect size estimates, confidence intervals, or causal inference — use A/B testing. Bandits optimize for total reward, not for learning. They can't tell you "Model B is 3.2% better with 95% confidence."
