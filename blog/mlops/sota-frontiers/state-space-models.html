State Space Models: Mamba, S4

MLOps Series SOTA & Frontiers

Transformers revolutionised sequence modelling, but their O(n²) self-attention cost becomes a bottleneck on long sequences—genomics (16 k+ tokens), audio (160 k samples/s), and long-document QA. State Space Models (SSMs) offer a fundamentally different paradigm: they model sequences through a latent continuous-time dynamical system, yielding O(n) training and O(1) per-step inference. This post traces the arc from classical linear SSMs through the HiPPO framework, S4, and finally Mamba—the selective SSM that matches or exceeds Transformer quality at a fraction of the compute.

Continuous-Time SSM Fundamentals

A continuous-time state space model maps an input signal u(t) ∈ ℝ to an output y(t) ∈ ℝ through a latent state x(t) ∈ ℝN:

// Continuous-time SSM equations x'(t) = A x(t) + B u(t) // state equation (N×N dynamics) y(t) = C x(t) + D u(t) // observation equation

Here A ∈ ℝN×N governs how state evolves, B ∈ ℝN×1 projects input into state space, C ∈ ℝ1×N reads out the state, and D is typically set to zero. The state dimension N controls model capacity; typical values range from 16 to 64.

To process discrete sequences (tokens, samples), we discretize the continuous system using a step size Δ. The zero-order hold (ZOH) discretization gives:

// Zero-Order Hold discretization Ā = exp(Δ · A) // discrete state matrix B̄ = (Δ · A)⁻¹ · (Ā - I) · Δ · B // discrete input matrix // Recurrent form (inference) x[k] = Ā x[k-1] + B̄ u[k] y[k] = C x[k]
Key insight: The same SSM can be computed in two equivalent ways — recurrently (O(1) per step, ideal for autoregressive generation) or via a global convolution (parallelisable on GPU, ideal for training). This dual view is the foundation of efficient SSM architectures.

HiPPO Framework & Long Memory

The critical question for any SSM is: how should we initialise A? A random matrix leads to rapid forgetting. The HiPPO (High-order Polynomial Projection Operators) framework (Gu et al., 2020) provides a principled answer: choose A so the state x(t) maintains an optimal polynomial approximation of the input history up to time t.

The key HiPPO matrix (LegS variant) is:

// HiPPO-LegS matrix (N×N) A[n,k] = -(2n + 1)1/2 · (2k + 1)1/2 if n > k A[n,k] = -(n + 1) if n == k A[n,k] = 0 if n < k

This gives the state vector x(t) a remarkable property: the n-th component stores the coefficient of the n-th shifted Legendre polynomial approximation of u on [0, t]. As t grows, the approximation window naturally stretches — the model remembers the entire history at decreasing resolution, similar to how humans remember distant events less precisely.

Why HiPPO matters: Without HiPPO initialisation, SSMs fail at sequences longer than ~256 tokens. With it, they handle 16k+ tokens. HiPPO is to SSMs what positional encodings are to Transformers — the mechanism enabling long-range reasoning.

S4: Structured State Spaces

S4 (Gu, Goel, Ré, 2021) combined HiPPO initialisation with three key innovations to make SSMs practical at scale:

  1. Diagonal-plus-low-rank (DPLR) parameterisation — the HiPPO matrix is decomposed as A = Λ − PQ* where Λ is diagonal and P, Q are rank-1 vectors. This reduces the kernel computation from O(N³) to O(N).
  2. FFT-based convolution — the SSM kernel K of length L is computed in closed form via the Cauchy kernel trick at the roots of unity, then applied via FFT in O(L log L).
  3. Learnable Δ, C, D — while A is initialised from HiPPO and held (near-)fixed, the discretization step Δ and readout C are learned, allowing each layer to attend to different timescales.
# S4 kernel computation (simplified) def s4_kernel(A, B, C, delta, L): # Discretise A_bar = torch.matrix_exp(delta * A) # (N, N) B_bar = torch.linalg.solve(A, (A_bar - I)) @ B # (N, 1) # Build kernel via Cauchy dot product at roots of unity omega = torch.exp(2j * pi * torch.arange(L) / L) # K_hat[i] = C @ (omega[i]·I − A_bar)^{-1} @ B_bar ← Cauchy trick K_hat = cauchy_dot(C, A_bar, B_bar, omega) # O(N·L) K = torch.fft.ifft(K_hat).real[:L] # (L,) return K def s4_forward(u, K): # u: (batch, L) K: (L,) return torch.fft.ifft(torch.fft.fft(u) * torch.fft.fft(K)).real
S4's limitation: The SSM kernel is input-independent — the same convolution filter is applied regardless of input content. This means S4 cannot perform content-based reasoning (e.g., "copy this token when you see the trigger"). This is the motivation for Mamba.

Mamba: Selective State Spaces

Mamba (Gu & Dao, 2023) addresses S4's core limitation by making the SSM parameters input-dependent. The key idea: B, C, and Δ are functions of the current input, enabling the model to selectively propagate or forget information based on content.

Input-Dependent Gating

In standard S4, (B, C, Δ) are fixed learned parameters. In Mamba:

# Mamba selective scan — input-dependent parameters def mamba_block(x, D_param): # x: (batch, L, d_model) # Branch 1: SSM path x_ssm = linear_proj(x, d_inner) # expand dimension x_ssm = conv1d(x_ssm, kernel=4) # local context x_ssm = silu(x_ssm) # activation # Input-dependent SSM parameters (THE key innovation) B = linear_B(x_ssm) # (batch, L, N) C = linear_C(x_ssm) # (batch, L, N) delta = softplus(linear_delta(x_ssm)) # (batch, L, d_inner) # Selective scan (replaces global convolution) y = selective_scan(x_ssm, A, B, C, delta) # O(n) via parallel scan # Branch 2: Gate path z = silu(linear_z(x)) # gating signal # Combine out = y * z # element-wise gating return linear_out(out) + D_param * x # skip connection + D

Making Δ input-dependent is especially powerful. When Δ is large, the discretized Ā = exp(ΔA) ≈ 0, effectively resetting the hidden state — the model forgets past context. When Δ is small, Ā ≈ I, and the state is preserved. This gives Mamba a learned forget gate, analogous to the gate in an LSTM but operating in continuous state space.

Selectivity enables in-context learning: On the Induction Heads synthetic task (copying a pattern after a trigger), vanilla S4 fails completely while Mamba solves it perfectly — proving that input-dependent parameters enable content-based routing through state.

Hardware-Aware Parallel Scan

Input-dependent parameters break the convolution view — you can no longer pre-compute a single kernel K. Naively, this forces sequential recurrence: O(n) steps, each with O(N) work, with terrible GPU utilisation. Mamba solves this with a hardware-aware parallel scan algorithm.

Parallel Prefix Scan

The recurrence x[k] = Ā[k] x[k-1] + B̄[k] u[k] is a linear scan. Any linear scan can be parallelised via the Blelloch prefix-sum algorithm in O(n/p + log n) work across p processors:

# Parallel scan: associative operator ⊕ # (a₁, b₁) ⊕ (a₂, b₂) = (a₂ · a₁, a₂ · b₁ + b₂) def parallel_scan(A_bars, Bu_bars): # A_bars: (batch, L, N) — discretized per-step A # Bu_bars: (batch, L, N) — discretized per-step B·u pairs = zip(A_bars, Bu_bars) # L pairs # Up-sweep: reduce pairs with ⊕ in O(log L) steps # Down-sweep: propagate prefix products return prefix_sum(pairs, op=ssm_combine) # all x[0..L-1]

Memory-Efficient Kernel Fusion

Mamba's CUDA implementation fuses the entire selective scan into a single kernel that lives in GPU SRAM (on-chip memory), avoiding expensive HBM (high-bandwidth memory) round-trips:

  1. Discretization of (Δ, A, B) → (Ā, B̄) is computed on the fly inside the kernel.
  2. The scan operates on tiles that fit in SRAM (~192 KB on A100).
  3. No materialisation of the full (batch, L, d_inner, N) state tensor in HBM — a 10-30× memory saving.
Implementation detail: Mamba's speed advantage comes primarily from this kernel fusion, not from algorithmic complexity alone. A naive PyTorch parallel scan is often slower than FlashAttention-2 due to poor memory access patterns. The custom CUDA kernel is essential.

SSMs vs Transformers: O(n) vs O(n²)

🧬 State Space Models

  • Training: O(n) via parallel scan
  • Inference: O(1) per token (constant state)
  • Memory: O(1) KV-cache equivalent
  • Long context: native — no context window limit
  • Weakness: harder to do precise retrieval ("find the 3rd mention of X")

⚡ Transformers

  • Training: O(n²) attention (FlashAttention helps constant)
  • Inference: O(n) per token (KV cache grows)
  • Memory: O(n) KV cache
  • Long context: requires tricks (RoPE, ALiBi, sliding window)
  • Strength: excellent at precise retrieval and in-context learning

The practical implication: at sequence length 1M tokens, a Transformer's KV cache occupies ~60 GB for a 7B model, while Mamba's fixed-size state uses ~1 MB. For autoregressive generation, Mamba achieves 5× higher throughput at 128k context compared to a similarly-sized Transformer.

Jamba & Hybrid Architectures

In practice, pure SSMs and pure Transformers each have blind spots. Jamba (AI21 Labs, 2024) showed that interleaving Mamba layers with attention layers yields a model that inherits the best of both worlds:

  • Mamba layers — handle long-range dependencies efficiently (majority of layers, e.g., 6 out of every 8).
  • Attention layers — provide precise in-context retrieval where needed (e.g., 2 out of every 8).
  • MoE integration — Jamba uses Mixture-of-Experts in the MLP blocks for parameter efficiency, achieving 52B total parameters but only 12B active.
# Jamba block pattern (simplified) class JambaBlock: def __init__(self, layer_idx, config): # Alternate: 6 Mamba layers, then 2 Attention layers if layer_idx % 8 < 6: self.mixer = MambaLayer(config.d_model, config.ssm_state_size) else: self.mixer = AttentionLayer(config.d_model, config.n_heads) # MoE MLP: top-2 out of 16 experts active self.mlp = MoELayer(config.d_model, n_experts=16, top_k=2) def forward(self, x): x = x + self.mixer(norm(x)) # pre-norm residual x = x + self.mlp(norm(x)) return x

Jamba's 52B model fits in a single 80GB A100 (vs. needing 2+ GPUs for a 70B Transformer) and processes 256k token contexts natively. Other hybrid designs include Mamba-2 (Gu & Dao, 2024), which reframes the selective scan as structured masked attention, enabling direct use of optimised attention kernels.

Mamba-2 insight: The structured state space duality (SSD) framework shows that selective SSMs are equivalent to a specific form of linear attention with a semiseparable mask matrix. This unifies the SSM and Transformer paradigms under a single algebraic framework, enabling shared optimisation techniques.

Interview Cheat-Sheet

Rapid-fire answers for technical interviews at Anthropic, OpenAI, and similar labs:

❓ "Why not just use linear attention?"

Linear attention (Katharopoulos et al.) replaces softmax(QKT)V with φ(Q)·(φ(K)TV) — O(n) but empirically much weaker than softmax attention. Mamba approaches the problem differently: instead of approximating attention, it uses a principled dynamical systems formulation with learned selectivity. Mamba-2 proves the two are algebraically related but Mamba's parameterisation is more expressive.

❓ "When would you NOT use Mamba?"

When you need precise in-context retrieval — tasks like "return the value associated with key X from the context". Attention's direct token-to-token comparison excels here. Also, for codebases with extensive FlashAttention/vLLM infrastructure, the migration cost may not justify the switch. Hybrid architectures like Jamba hedge both bets.

❓ "Explain selectivity in one sentence."

Selectivity means the SSM's dynamics (B, C, Δ) are functions of the input at each timestep, enabling the model to gate what enters and leaves the hidden state based on content — like a continuous-time LSTM without separate gates.

❓ "What is the HiPPO matrix?"

A specific initialisation for the state matrix A derived from optimally projecting the input history onto a basis of orthogonal polynomials. It ensures the hidden state compresses all past inputs with mathematically optimal approximation quality, enabling SSMs to handle very long sequences.

The big picture: SSMs represent a paradigm shift from "every token attends to every other token" (Transformers) to "information flows through a compressed, evolving state" (SSMs). The field is converging toward hybrid architectures that combine both mechanisms, using attention for precision and SSMs for efficiency. Understanding both is essential for anyone working at the frontier of large-scale sequence modelling.
