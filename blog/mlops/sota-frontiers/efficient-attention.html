Efficient Attention: Linear, Sparse, Multi-Head

MLOps Series SOTA & Frontiers

Multi-Head Attention Review

Standard Multi-Head Attention (MHA) as introduced in Attention Is All You Need (Vaswani et al., 2017) projects the input into h independent query, key, and value heads, computes scaled dot-product attention in parallel, then concatenates and projects back. For a model with hidden dimension d and h heads, each head operates on vectors of dimension dk = d / h.

# Standard Multi-Head Attention
def multi_head_attention(X, W_q, W_k, W_v, W_o, n_heads):
    B, S, D = X.shape
    head_dim = D // n_heads

    Q = (X @ W_q).view(B, S, n_heads, head_dim)  # [B, S, h, d_k]
    K = (X @ W_k).view(B, S, n_heads, head_dim)  # [B, S, h, d_k]
    V = (X @ W_v).view(B, S, n_heads, head_dim)  # [B, S, h, d_k]

    # Scaled dot-product attention per head
    scores = (Q @ K.transpose(-1, -2)) / sqrt(head_dim)  # [B, h, S, S]
    attn = softmax(scores, dim=-1) @ V                   # [B, h, S, d_k]

    # Concat heads and project
    out = attn.reshape(B, S, D) @ W_o  # [B, S, D]
    return out
Cost breakdown: For sequence length S and model dimension D, MHA computes h separate S×S attention matrices. Total FLOPs: 4·S·D² + 2·S²·D. The S² term dominates for long sequences — at S=128K with D=4096, the quadratic term is 8192× larger than the linear term.

The critical bottleneck is twofold: (1) the O(S²) memory for the attention matrix, and (2) the KV cache size during inference which scales as 2 × L × h × d_k × S per sequence. Every optimization in this post attacks one or both of these bottlenecks.

Multi-Query Attention (MQA)

Multi-Query Attention (Shazeer, 2019) keeps h independent query heads but collapses all key and value projections into a single shared KV head. Every query head attends against the same K and V tensors. This reduces KV cache memory by a factor of h (typically 32–128×) and dramatically improves inference throughput by reducing memory bandwidth pressure.

# Multi-Query Attention: 1 KV head shared across all query heads
def multi_query_attention(X, W_q, W_k, W_v, W_o, n_heads):
    B, S, D = X.shape
    head_dim = D // n_heads

    Q = (X @ W_q).view(B, S, n_heads, head_dim)   # h query heads
    K = (X @ W_k).view(B, S, 1, head_dim)         # 1 KV head
    V = (X @ W_v).view(B, S, 1, head_dim)         # 1 KV head

    # Broadcast K, V across all query heads
    K = K.expand(B, S, n_heads, head_dim)          # no memory copy
    V = V.expand(B, S, n_heads, head_dim)

    scores = (Q @ K.transpose(-1, -2)) / sqrt(head_dim)
    attn = softmax(scores, dim=-1) @ V
    return attn.reshape(B, S, D) @ W_o
Quality tradeoff: MQA reduces KV cache by h× but loses per-head KV specialization. PaLM (540B) reported <0.5% quality loss with MQA. Falcon-40B ships with MQA, achieving 2.5× faster decoding than MHA at batch size 64. However, for smaller models (≤7B), the quality drop can be 1–2% on benchmarks like MMLU, making GQA the preferred compromise.

Grouped-Query Attention (GQA)

Grouped-Query Attention (Ainslie et al., 2023) is the middle ground between MHA and MQA. Instead of all heads sharing one KV set, query heads are partitioned into G groups, each with its own KV head. With h query heads and G KV heads, each KV head serves h/G query heads. GQA-1 = MQA, GQA-h = MHA.

# Grouped-Query Attention
def grouped_query_attention(X, W_q, W_k, W_v, W_o, n_heads, n_kv_heads):
    B, S, D = X.shape
    head_dim = D // n_heads
    group_size = n_heads // n_kv_heads  # queries per KV group

    Q = (X @ W_q).view(B, S, n_heads, head_dim)
    K = (X @ W_k).view(B, S, n_kv_heads, head_dim)
    V = (X @ W_v).view(B, S, n_kv_heads, head_dim)

    # Expand KV to match query heads via repeat_interleave
    K = K.repeat_interleave(group_size, dim=2)  # [B, S, n_heads, d_k]
    V = V.repeat_interleave(group_size, dim=2)

    scores = (Q @ K.transpose(-1, -2)) / sqrt(head_dim)
    attn = softmax(scores, dim=-1) @ V
    return attn.reshape(B, S, D) @ W_o
Industry adoption: GQA is the de facto standard for modern LLMs. LLaMA-2 70B uses GQA with 8 KV heads (8:1 ratio, 64 query heads). LLaMA-3 8B/70B/405B all use GQA-8. Mistral 7B uses GQA-8. Gemma uses GQA-1 (MQA) for the 2B model and GQA-8 for 7B+. The uptraining approach from the GQA paper showed that converting an MHA checkpoint to GQA via mean-pooling KV head weights, then fine-tuning for 5% of original training, recovers nearly all quality.

Flash Attention 1 / 2 / 3

Flash Attention (Dao et al., 2022) is an IO-aware exact attention algorithm that never materializes the full S×S attention matrix in HBM. Instead, it uses a tiling strategy to compute attention in blocks that fit in GPU SRAM (shared memory), reducing HBM reads/writes from O(S²) to O(S²/M) where M is SRAM size. The key insight is the online softmax trick: maintaining running max and sum statistics so softmax can be computed incrementally per tile.

# Flash Attention core loop (simplified)
# Q, K, V are in HBM; attention is computed tile-by-tile in SRAM
def flash_attention(Q, K, V, block_size=128):
    S = Q.shape[-2]
    O = zeros_like(Q)          # output accumulator in HBM
    l = zeros(S)               # running softmax denominator
    m = full(S, -inf)         # running row-wise max

    for j in range(0, S, block_size):     # outer loop: KV tiles
        K_j = K[j:j+block_size]                 # load KV tile to SRAM
        V_j = V[j:j+block_size]

        for i in range(0, S, block_size): # inner loop: Q tiles
            Q_i = Q[i:i+block_size]             # load Q tile to SRAM

            # Compute local attention scores
            S_ij = Q_i @ K_j.T / sqrt(d)       # [block, block] in SRAM

            # Online softmax update (the key trick)
            m_new = max(m[i:i+block_size], S_ij.max(dim=-1))
            P_ij = exp(S_ij - m_new)            # numerically stable
            l_new = exp(m[i:i+block_size] - m_new) * l[i:i+block_size] + P_ij.sum(-1)

            # Rescale previous output and accumulate
            O[i:i+block_size] = (exp(m[i:i+block_size] - m_new).unsqueeze(-1)
                                 * O[i:i+block_size] + P_ij @ V_j)

            m[i:i+block_size] = m_new
            l[i:i+block_size] = l_new

    # Final normalization
    O = O / l.unsqueeze(-1)
    return O
Flash Attention memory: Standard attention requires O(S²) HBM for the attention matrix. Flash Attention requires only O(S) — just the output, running statistics, and the input Q/K/V. For S=8192 with D=128 in FP16, standard attention allocates 128 MB for the attention matrix alone; Flash Attention uses ~0 extra HBM beyond inputs/outputs.

Flash Attention v1 (Dao 2022)

  • Tiled attention with online softmax
  • No S×S materialization in HBM
  • Recomputes attention in backward pass
  • 2–4× speedup over PyTorch standard attention
  • Memory: O(S) vs O(S²)
  • A100 throughput: ~124 TFLOPS (62% utilization)

Flash Attention v2 (Dao 2023)

  • Better work partitioning across warps
  • Reduced non-matmul FLOPs by 4×
  • Parallelizes over sequence length in backward
  • 2× faster than FA v1, up to 230 TFLOPS on A100
  • Reaches 73% of theoretical max FLOPS
  • Supports head dimensions up to 256

Flash Attention v3 (Shah et al. 2024)

  • Targets Hopper GPUs (H100, H200)
  • Exploits wgmma (warp group MMA) instructions
  • Overlaps softmax with GEMM via warp specialization
  • FP8 quantized attention with block-level scaling
  • 740 TFLOPS on H100 (75% utilization)
  • 1.5–2× faster than FA v2 on H100

Standard PyTorch Attention

  • Materializes full S×S attention matrix
  • Stores attention weights for backward
  • Memory: O(S²) — 128 MB at S=8K
  • OOMs at S ≥ 16K on 40GB A100
  • A100 throughput: ~30–50 TFLOPS
  • Simple to implement and debug

Sliding Window Attention

Sliding Window Attention (SWA) restricts each token to attend only to the W most recent tokens instead of the full context. This gives O(S·W) complexity instead of O(S²). Critically, information still propagates across the full context through stacking: after L layers of window size W, the effective receptive field is L × W tokens.

# Sliding Window Attention (as in Mistral 7B)
def sliding_window_attention(Q, K, V, window_size=4096):
    S = Q.shape[-2]
    O = zeros_like(Q)

    for i in range(S):
        start = max(0, i - window_size + 1)
        # Each token attends only to its local window
        K_win = K[start:i+1]
        V_win = V[start:i+1]

        scores = Q[i] @ K_win.T / sqrt(d)
        O[i] = softmax(scores) @ V_win

    return O

# Mistral 7B configuration:
#   window_size W = 4,096 tokens
#   num_layers L = 32
#   effective receptive field = L × W = 131,072 tokens
#   KV cache per sequence = W × 2 × L × n_kv × d_k × 2 bytes
#                         = 4096 × 2 × 32 × 8 × 128 × 2
#                         = 0.54 GB (fixed, regardless of context!)
Mistral's hybrid approach: Mistral 7B combines SWA with GQA-8 and a rolling KV cache. The KV cache is a fixed-size circular buffer — once it's full, the oldest entries are overwritten. This means the cache never grows beyond W entries per layer, enabling constant memory inference even at 128K+ context lengths. Combined with Flash Attention, Mistral achieves 5.4× the throughput of LLaMA-2 7B at 16K context.
Limitation: Pure sliding window loses precise long-range attention. Tokens beyond the window can only influence through indirect propagation across layers. For tasks requiring exact retrieval from early context (e.g., "what was the first item in the list?"), SWA can degrade. Mixtral 8×7B addresses this with a combination of SWA layers and full-attention layers.

Linear Attention & the Kernel Trick

Standard attention computes softmax(QKT/√d)V, which requires materializing the S×S matrix. Linear attention replaces the softmax kernel with a decomposable feature map φ, rewriting attention as φ(Q)(φ(K)TV). By computing φ(K)TV first (an O(D²·S) operation), we avoid the S×S bottleneck entirely.

# Standard attention: O(S² × D)
# attn = softmax(Q @ K.T / sqrt(d)) @ V
#        ~~~~~~~ S×S matrix ~~~~~~~

# Linear attention with kernel trick: O(S × D²)
def linear_attention(Q, K, V, feature_map=elu_plus_one):
    # Apply feature map φ to Q and K
    Q_prime = feature_map(Q)  # [B, S, D] → [B, S, D']
    K_prime = feature_map(K)

    # Key insight: change multiplication order
    # Instead of: (Q_prime @ K_prime.T) @ V    ← S×S intermediate
    # Compute:    Q_prime @ (K_prime.T @ V)    ← D'×D intermediate
    KV = K_prime.transpose(-1, -2) @ V  # [D', D] — small!
    out = Q_prime @ KV                   # [B, S, D]

    # Normalize (since we replaced softmax)
    Z = Q_prime @ K_prime.sum(dim=-2, keepdim=True).transpose(-1, -2)
    return out / (Z + 1e-6)

# Common feature maps:
def elu_plus_one(x): return elu(x) + 1       # Katharopoulos et al.
def random_features(x): return cos(x @ W_rand) # Performer (Choromanski)

✅ Linear Attention Pros

  • O(S·D²) time — linear in sequence length
  • O(D²) memory for KV state (constant!)
  • Causal version: recurrent form, O(1) per step
  • Infinite context window theoretically possible
  • No need for KV cache — state is a fixed-size matrix

❌ Linear Attention Cons

  • Significant quality degradation (2–5% on MMLU)
  • Feature map approximation loses precision
  • Cannot model sharp, peaked attention patterns
  • Struggles with retrieval and copying tasks
  • No Flash Attention equivalent (already memory-efficient)
Modern linear variants: RetNet (Microsoft, 2023) reformulates linear attention with multi-scale exponential decay, achieving GPT-quality at 7B parameters. Mamba (Gu & Dao, 2023) uses selective state spaces with hardware-aware algorithms, matching Transformer quality on language tasks with O(S) complexity. RWKV-v6 combines linear attention with time-mixing for competitive results on long-context benchmarks. These represent the strongest challenges to standard softmax attention.

Sparse Attention Patterns

Sparse attention reduces the S×S cost by restricting which token pairs can attend to each other. Different sparsity patterns trade off between computational savings and information flow quality.

Strided / Block Sparse

Introduced by Sparse Transformer (Child et al., 2019). Combines local windowed attention with strided patterns where every token attends to tokens at fixed intervals. Complexity: O(S·√S).

Used in GPT-3's initial experiments. The strided pattern ensures global information flow while the local window captures fine-grained context.

Longformer / BigBird

Longformer (Beltagy et al., 2020) combines local sliding window, dilated sliding window, and global attention on select tokens (e.g., [CLS]). BigBird adds random sparse attention. Both achieve O(S) complexity.

Longformer supports 16K tokens on a single GPU; BigBird extends to 36K. Both match RoBERTa quality on downstream tasks.

Mixture of Attention Heads

Some architectures use different attention patterns per head: some heads do local attention, some do global, some do strided. This lets the model learn which heads need long-range vs. local context.

Jamba (AI21, 2024) interleaves Transformer attention layers with Mamba SSM layers, getting the best of both worlds.

Dynamic / Learned Sparsity

Routing Transformer (Roy et al., 2021) uses k-means clustering on queries/keys to dynamically determine which tokens attend to each other. Reformer (Kitaev et al., 2020) uses LSH hashing for O(S·log S) attention.

Dynamic approaches add routing overhead but can match full attention quality better than fixed patterns.

The production stack in 2024: The winning combination for most deployments is GQA + Flash Attention v2/v3 + PagedAttention (vLLM). GQA shrinks the KV cache by 4–8×, Flash Attention eliminates the attention matrix memory bottleneck, and PagedAttention handles KV cache memory management with near-zero fragmentation. Add sliding window attention for workloads where 90%+ of queries don't need full context, and you get another 15–20% throughput boost.

When to Use What

  • GQA-8: Default for any new model. Negligible quality loss, 4–8× KV savings.
  • Flash Attention v2: Always. No quality loss, 2–7× speed, less memory.
  • SWA: Chat/dialogue, code completion — where recent context matters most.
  • Linear/SSM: Research or infinite-context edge cases. Not yet production-grade for general LLMs.

Key Metrics to Watch

  • Time-to-first-token (TTFT): Flash Attention helps prefill speed.
  • KV cache GB/seq: GQA/MQA are the levers here.
  • Max concurrent sequences: = (GPU mem − model) / KV per seq.
  • Cost per 1M tokens: The ultimate metric — combine all factors.

Efficient attention is one of the fastest-moving areas in ML systems research. The gap between a naively-served model and one using GQA + Flash Attention + PagedAttention can be 10–30× in throughput and 5–10× in cost efficiency. Understanding these mechanisms isn't optional for anyone deploying LLMs at scale — it's the difference between a viable product and one that bankrupts you on compute.
