Open Source LLMs: Llama, Mistral, DeepSeek, Qwen

MLOps Series SOTA & Frontiers

The open-source LLM landscape has evolved from a curiosity into a genuine alternative to proprietary APIs. In 2023, the best open models lagged GPT-4 by 20–30 points on major benchmarks. By late 2024, that gap shrank to single digits — and in specialized domains like math and code, open models now lead. This post maps the four dominant families, their licenses, capabilities, and practical deployment considerations.

Open Model Release Timeline

The pace of open model releases has been staggering. Each major release forced competitors to respond within weeks, creating a virtuous cycle of rapid improvement. Below is an SVG timeline of the key milestones:

Key observation: The cadence has collapsed from yearly to quarterly releases. Meta shipped Llama 3, 3.1, 3.2, and 3.3 within eight months. DeepSeek went from V2 to R1 in the same period. This forces MLOps teams to treat model upgrades as a routine operational concern, not an annual migration event.

The Llama Family

Meta's Llama series is the most widely deployed open model family, with an ecosystem spanning thousands of fine-tuned variants, quantized builds, and commercial deployments. Each generation brought architectural and training improvements:

Llama 1 (Feb 2023)

Sizes: 7B, 13B, 33B, 65B

Architecture: Standard decoder-only transformer with RMSNorm, SwiGLU, rotary positional embeddings (RoPE).

Training: 1.4T tokens from publicly available data. The 13B model matched GPT-3 (175B) — proving smaller, well-trained models could compete with large ones.

License: Research-only (leaked within a week, catalyzing the open-source movement).

Llama 2 (Jul 2023)

Sizes: 7B, 13B, 70B (+ Chat variants)

Key advances: 2T training tokens (40% more), grouped-query attention (GQA) for the 70B, 4K context length, RLHF for chat models with >1M human annotations.

License: Llama 2 Community License — free for commercial use under 700M monthly active users.

Impact: First truly commercially usable open LLM at frontier quality.

Llama 3 / 3.1 (Apr–Jul 2024)

Sizes: 8B, 70B, 405B

Key advances: 15T+ training tokens, 128K vocabulary (4× Llama 2), 128K context for 3.1, GQA across all sizes. The 405B became the first open model to approach GPT-4 performance.

Training infra: 16K H100 GPUs, custom training stack. Published a 92-page paper detailing every training decision.

License: Llama 3.1 Community License (similar terms, minor refinements).

Llama 3.2 / 3.3 (Sep–Dec 2024)

Sizes: 1B, 3B (lightweight), 11B-Vision, 90B-Vision (3.2); 70B (3.3)

Key advances: 3.2 introduced multimodal vision + lightweight edge models. 3.3 70B matches 3.1 405B on many benchmarks via improved training recipes.

Ops note: 1B/3B target mobile/edge. Vision models need image preprocessor pipeline integration. 3.3 70B is the sweet spot for most production deployments.

# Llama model selection helper
def select_llama_model(use_case, infra):
    if use_case == "edge" or infra.gpu_memory_gb < 8:
        return "Llama-3.2-3B"           # 4-bit: ~2GB VRAM

    if use_case == "vision":
        if infra.gpu_memory_gb >= 48:
            return "Llama-3.2-90B-Vision" # needs multi-GPU
        return "Llama-3.2-11B-Vision"    # fits single A100-40GB

    if use_case == "general":
        if infra.gpu_count >= 4:
            return "Llama-3.3-70B"        # best quality/cost ratio
        return "Llama-3.1-8B"            # strong single-GPU option

    if use_case == "max_quality":
        return "Llama-3.1-405B"          # 8× A100-80GB minimum

The Mistral Family

Mistral AI, founded by ex-Meta and ex-DeepMind researchers, took a different path: smaller models with architectural innovations, particularly Mixture-of-Experts (MoE) and sliding window attention. Their models consistently punch above their weight class.

Mistral 7B (Sep 2023)

Architecture: Sliding window attention (4096 tokens), GQA, 32K context via rolling buffer. First model released via magnet link — no sign-up, no restrictions.

Performance: Beat Llama 2 13B on every benchmark while being half the size. Beat Llama 1 34B on many tasks.

License: Apache 2.0 — fully permissive, no usage restrictions.

Impact: Proved that 7B-class models could be genuinely useful. Spawned thousands of community fine-tunes.

Mixtral 8×7B (Dec 2023)

Architecture: Sparse MoE — 8 expert FFNs per layer, top-2 routing. Total params: 46.7B, active per token: ~12.9B. This gives near-dense-model quality at much lower inference cost.

Performance: Matched or beat Llama 2 70B while using ~3× less compute per token. Excellent multilingual performance across 5 languages.

License: Apache 2.0.

Ops note: Needs ~90GB for full weights (all experts loaded), but only ~25GB active per forward pass. Memory-bandwidth bound, not compute-bound.

Mistral Large 2 (Jul 2024)

Size: 123B dense parameters, 128K context.

Performance: Competitive with Llama 3.1 405B and GPT-4o on code, math, and multilingual tasks — at 1/3 the parameter count.

License: Mistral Research License — not Apache 2.0. Free for research, commercial use requires agreement.

Ops note: Dense architecture means higher per-token compute vs Mixtral, but simpler serving pipeline (no expert routing overhead).

Mistral Small / Codestral (2024)

Mistral Small 22B: Optimized for low-latency, single-GPU deployment. Strong reasoning for its size class.

Codestral 22B: Code-specialized model with >80 language support. Fill-in-the-middle (FIM) trained. Strong HumanEval scores (81.1%).

License: Both under Mistral Non-Production License (MNPL) initially; Codestral Mamba under Apache 2.0.

Mistral licensing fragmentation: Unlike Meta (consistent license) or Alibaba (Apache 2.0 everywhere), Mistral uses different licenses per model. Always verify the specific model's license before production deployment. The Apache 2.0 models (7B, Mixtral) are safe; newer models may carry restrictions.

The DeepSeek Family

DeepSeek, a Chinese AI lab backed by the quantitative hedge fund High-Flyer, has produced some of the most architecturally innovative open models. Their key contributions are Multi-head Latent Attention (MLA) and DeepSeekMoE, both of which dramatically reduce inference cost.

DeepSeek-V2 (May 2024)

Architecture: 236B total params, 21B active per token. Introduces MLA — compresses KV-cache by projecting keys and values into a low-rank latent space, reducing KV-cache memory by ~93% vs standard MHA.

DeepSeekMoE: Fine-grained experts (64 routed + 2 shared) with auxiliary load-balancing loss. More experts = smoother routing and better specialization.

Performance: Matched Mixtral 8×22B and Llama 3 70B at 1/5th the inference cost.

License: MIT (weights) + Apache 2.0 (code) — maximally permissive.

DeepSeek-V3 (Dec 2024)

Architecture: 671B total params, 37B active. Same MLA + MoE foundation as V2 but scaled significantly. Added Multi-Token Prediction (MTP) as an auxiliary training objective.

Training cost: $5.6M on 2048 H800 GPUs — remarkably efficient. Used FP8 mixed-precision training with minimal accuracy loss.

Performance: Matched GPT-4o and Claude 3.5 Sonnet on most benchmarks. Best open model at time of release.

License: MIT.

DeepSeek-R1 (Jan 2025)

Architecture: Built on V3 base with RL-based reasoning training. Produces extended chain-of-thought reasoning traces, similar to OpenAI's o1.

Key innovation: Showed that pure RL (GRPO) without supervised fine-tuning can produce emergent reasoning behaviors — the model learns to self-verify, backtrack, and decompose problems.

Distilled variants: R1-Distill-Qwen-1.5B through R1-Distill-Llama-70B — reasoning capabilities transferred to smaller models.

License: MIT.

DeepSeek-Coder V2 (Jun 2024)

Architecture: 236B total / 21B active, same MoE base as V2. Specialized for code with 338 programming languages and 128K context.

Performance: Beat GPT-4 Turbo on code benchmarks (HumanEval: 90.2%) at the time of release.

Ops note: Code models benefit significantly from speculative decoding — code has predictable token patterns. Use draft-model speculation for 2–3× throughput improvement.

License: MIT.

# DeepSeek MLA — KV-cache compression
class MultiHeadLatentAttention:
    def __init__(self, d_model=5120, n_heads=128, d_latent=512):
        # Standard MHA: KV-cache = 2 × n_heads × d_head × seq_len
        # MLA: KV-cache = 2 × d_latent × seq_len (93% smaller)
        self.down_proj = Linear(d_model, d_latent)     # compress
        self.up_proj_k = Linear(d_latent, d_model)     # expand to keys
        self.up_proj_v = Linear(d_latent, d_model)     # expand to values

    def forward(self, x, cache=None):
        # Compress hidden state into low-rank latent
        latent = self.down_proj(x)           # [B, L, d_latent]
        # Only store the latent in KV-cache (not full K, V)
        if cache is not None:
            cache.append(latent)
        # Expand back to full K, V for attention computation
        k = self.up_proj_k(latent)            # [B, L, d_model]
        v = self.up_proj_v(latent)            # [B, L, d_model]
        return k, v
Why MLA matters for ops: A standard 70B model at 128K context can consume 40+ GB of KV-cache alone. MLA compresses this to ~3 GB, enabling longer contexts and higher batch sizes on the same hardware. This directly translates to 3–5× higher throughput in production vLLM/SGLang deployments.

The Qwen Family

Alibaba's Qwen (通义千问) series is the most comprehensive open model family, covering text, vision, audio, code, and math in a single model line. Their rapid iteration (three major releases in 12 months) and consistent Apache 2.0 licensing make them a strong choice for production systems.

Qwen 1.5 (Nov 2023 → Feb 2024)

Sizes: 0.5B, 1.8B, 4B, 7B, 14B, 32B, 72B, 110B

Key features: Broadest size range of any family. 32K context, strong multilingual performance (especially CJK languages). GQA across all sizes.

License: Apache 2.0 for all sizes — rare for 72B+ class models.

Impact: The small variants (0.5B–4B) became go-to choices for on-device deployment.

Qwen 2 (Jun 2024)

Sizes: 0.5B, 1.5B, 7B, 57B-A14B (MoE), 72B

Key advances: 128K context, improved tokenizer (151K vocab), 27+ language support. The 57B-A14B MoE variant offered excellent efficiency — only 14B active params for near-72B performance.

Variants: Qwen2-Math, Qwen2-Code — specialized models trained on domain-specific data.

License: Apache 2.0 (except Qwen2-72B which uses Qwen License).

Qwen 2.5 (Sep 2024)

Sizes: 0.5B, 1.5B, 3B, 7B, 14B, 32B, 72B

Training: 18T tokens — more than Llama 3.1's 15T. Massive synthetic data usage for math and code. The 72B model matched Llama 3.1 405B on several key benchmarks.

Variants: Qwen2.5-Coder (1.5B–32B), Qwen2.5-Math (1.5B–72B). The Coder 32B model achieved 92.7% on HumanEval.

License: Apache 2.0 for all sizes including 72B.

QwQ / Qwen-VL / Qwen-Audio

QwQ-32B (Nov 2024): Reasoning model (o1-style) with 32K thinking tokens. Strong math/code reasoning at just 32B parameters.

Qwen-VL: Vision-language models from 2B to 72B. Supports arbitrary resolution images, video understanding, and OCR. Competitive with GPT-4V.

Qwen-Audio: Speech understanding + generation, supports 8+ languages.

Ops note: All modalities share the same base architecture, simplifying multi-modal serving.

Qwen's breadth advantage: No other open family covers text, code, math, vision, audio, and reasoning in a single ecosystem with consistent licensing. For teams that need multimodal capabilities without managing multiple model families, Qwen is the most practical choice.

License Comparison

Licensing is the most common blocker in production deployments. Open weights ≠ open source. Understanding the restrictions is critical before building production systems:

🟢 MIT License

Used by: DeepSeek (all models)

Permits: Commercial use, modification, distribution, sublicensing, private use. No attribution required in binary distributions.

Restrictions: None beyond preserving copyright notice.

Verdict: Maximum freedom. No legal review needed.

🟢 Apache 2.0

Used by: Mistral (7B, Mixtral), Qwen (most models)

Permits: Same as MIT + explicit patent grant. Requires preserving NOTICE file and stating changes.

Restrictions: Must include license text. Cannot use trademarks.

Verdict: Excellent for production. Patent grant adds protection.

🟡 Llama Community License

Used by: Llama 2, 3, 3.1, 3.2, 3.3

Permits: Commercial use, fine-tuning, hosting.

Restrictions: >700M monthly active users need special Meta license. Cannot use to train other LLMs. Must include "Built with Llama" attribution.

Verdict: Fine for most companies. Enterprises with huge user bases need Meta approval.

🔴 Research / Non-Production

Used by: Mistral Large 2, some Codestral variants

Permits: Research, evaluation, non-commercial use.

Restrictions: No commercial deployment without separate agreement. No redistribution in some cases.

Verdict: Evaluate only. Do not deploy to production without explicit commercial license.

# License compliance checker
def check_license_compliance(model_name, deployment):
    license_db = {
        "llama":    {"type": "community", "mau_limit": 700_000_000},
        "mistral-7b": {"type": "apache-2.0"},
        "mixtral":  {"type": "apache-2.0"},
        "mistral-large": {"type": "research"},
        "deepseek": {"type": "mit"},
        "qwen":     {"type": "apache-2.0"},
    }
    lic = license_db.get(model_name)

    if lic["type"] == "research":
        raise LicenseError("Cannot deploy research-licensed model")

    if lic["type"] == "community":
        if deployment.monthly_active_users > lic["mau_limit"]:
            raise "Contact Meta for enterprise license"
        if deployment.use_case == "train_other_llm":
            raise "Llama output cannot train competing LLMs"

    return "✅ Compliant"

Benchmark Comparison

Benchmarks provide directional guidance but should not be the sole basis for model selection. Real-world performance often diverges from published numbers due to evaluation contamination, prompt sensitivity, and task distribution differences. That said, here are the key comparisons across standardized benchmarks:

📊 MMLU (Massive Multitask Language Understanding)

Measures: Broad knowledge across 57 academic subjects.

Llama 3.1 405B: 88.6%

Llama 3.3 70B: 86.0%

DeepSeek-V3: 88.5%

Qwen 2.5 72B: 86.1%

Mixtral 8×7B: 70.6%

GPT-4o (reference): 88.7%

💻 HumanEval (Code Generation)

Measures: Python function completion from docstrings (164 problems).

Llama 3.1 405B: 89.0%

DeepSeek-Coder-V2: 90.2%

Qwen 2.5-Coder 32B: 92.7%

DeepSeek-V3: 82.6% (general model)

Mistral Large 2: 92.0%

GPT-4o (reference): 90.2%

🔢 MATH (Competition Mathematics)

Measures: Challenging math problems at competition level (5 difficulty levels).

Llama 3.1 405B: 73.8%

DeepSeek-V3: 90.2%

DeepSeek-R1: 97.3% (with reasoning)

Qwen 2.5-Math 72B: 85.9%

QwQ 32B: 90.6% (with reasoning)

GPT-4o (reference): 76.6%

🌍 Multilingual & Other

Best multilingual: Qwen 2.5 72B (27+ languages, esp. CJK) and Mistral Large 2 (12+ languages).

Best long-context: Qwen 2.5 (128K native) and DeepSeek-V3 (128K). Llama 3.1+ also supports 128K.

Best instruction following: DeepSeek-V3 leads on IFEval (86.1%). Llama 3.1 405B at 87.5%.

Best reasoning: DeepSeek-R1 and QwQ 32B dominate all reasoning benchmarks.

Benchmark caveats: (1) Published numbers are self-reported — independent evaluations often differ by 2–5 points. (2) MMLU and HumanEval are increasingly saturated; prefer MMLU-Pro, LiveCodeBench, and SWE-bench for differentiation. (3) Benchmark contamination is a real concern — some models may have seen test data during training. Always validate on your own evaluation set.

Serving Considerations per Family

Each model family has distinct operational characteristics that affect your serving infrastructure decisions. These go beyond raw model size — architecture, quantization friendliness, and ecosystem tooling all matter.

# Serving configuration by model family
def configure_serving(model_family, target_throughput, latency_sla_ms):
    configs = {
        "llama": {
            "engine": "vLLM",           # best ecosystem support
            "quantization": "AWQ-4bit", # excellent quality retention
            "features": ["PagedAttention", "continuous_batching"],
            "speculative": "Llama-3.2-1B",  # draft model for 70B
        },
        "mistral_moe": {
            "engine": "vLLM",           # good MoE support
            "quantization": "GPTQ-4bit", # MoE + quant needs care
            "features": ["expert_parallelism", "tensor_parallel"],
            "note": "memory-bandwidth bound, not compute",
        },
        "deepseek": {
            "engine": "SGLang",         # best MLA + MoE support
            "quantization": "FP8",      # V3 trained in FP8
            "features": ["MLA_cache", "expert_parallel", "MTP_decode"],
            "note": "MLA reduces KV-cache 93%, huge batch sizes",
        },
        "qwen": {
            "engine": "vLLM",           # or TGI, both well-supported
            "quantization": "GPTQ-4bit", # widely available on HF
            "features": ["PagedAttention", "continuous_batching"],
            "note": "large vocab (151K) — embedding layer is big",
        },
    }
    return configs[model_family]

Key operational differences by family:

  • Llama — Best ecosystem support. Every serving framework, quantization tool, and fine-tuning library supports Llama day-one. GGUF quantizations are available within hours of release. The safest choice for teams that prioritize operational simplicity.
  • Mistral MoE — Memory-bandwidth bound (all experts must be loaded). Expert parallelism across GPUs is essential for Mixtral-class models. Quantization is trickier for MoE — some experts may be more sensitive to precision loss. Test each quantization scheme on your specific tasks.
  • DeepSeek — MLA enables massive batch sizes but requires engine support (SGLang > vLLM for DeepSeek). The 671B V3 model can serve at reasonable cost thanks to MoE + MLA, but requires 8+ GPUs with fast interconnect. FP8 quantization is nearly lossless since V3 was trained in FP8.
  • Qwen — Large vocabulary (151K tokens) means the embedding and lm_head layers consume ~1.2 GB for the 72B model. This is usually negligible but matters for extremely memory-constrained deployments. Pre-quantized GPTQ models are available on Hugging Face for all sizes.
Universal serving tip: Regardless of model family, always benchmark with your actual traffic distribution. Synthetic benchmarks (fixed input/output lengths) systematically overestimate throughput by 30–50% compared to real-world variable-length workloads. Use tools like llmperf or genai-perf with production-representative prompts.

Model Selection Decision Framework

Choosing the right open model for production requires balancing multiple dimensions: performance, cost, license compatibility, ecosystem support, and operational complexity. Here is a systematic decision framework:

# Production model selection decision tree
def select_open_model(requirements):
    # Step 1: Filter by license
    if requirements.license == "must_be_permissive":
        candidates = ["deepseek", "qwen", "mistral-7b", "mixtral"]
    elif requirements.mau > 700_000_000:
        candidates = ["deepseek", "qwen", "mistral-7b", "mixtral"]
    else:
        candidates = ["all_families"]   # Llama included

    # Step 2: Filter by capability
    if requirements.needs_vision:
        return "Qwen2-VL-72B" or "Llama-3.2-90B-Vision"
    if requirements.needs_reasoning:
        return "DeepSeek-R1" or "QwQ-32B"
    if requirements.primary_task == "code":
        return "Qwen2.5-Coder-32B" or "DeepSeek-Coder-V2"

    # Step 3: Filter by infrastructure
    if requirements.max_gpus == 1:
        if requirements.gpu_memory_gb <= 16:
            return "Qwen2.5-7B-AWQ"    # or Llama-3.1-8B-AWQ
        if requirements.gpu_memory_gb <= 48:
            return "Qwen2.5-32B-AWQ"   # or Mistral-Small-22B

    # Step 4: Optimize for cost vs quality
    if requirements.priority == "quality":
        return "DeepSeek-V3"          # best quality/cost at scale
    if requirements.priority == "ecosystem":
        return "Llama-3.3-70B"       # most tooling, community
    if requirements.priority == "throughput":
        return "DeepSeek-V3"          # MLA = massive batches
    if requirements.priority == "simplicity":
        return "Llama-3.1-8B"        # runs everywhere

Decision dimensions to evaluate:

  • License compatibility: Check legal requirements first — this eliminates options immediately. MIT/Apache 2.0 are safest. Llama Community License works for most companies but adds restrictions.
  • Task-specific performance: Run your own evals. A model that scores 5 points lower on MMLU might score 10 points higher on your specific domain. DeepSeek excels at math/code; Qwen excels at multilingual; Llama excels at general instruction-following.
  • Infrastructure fit: Match model size to available hardware. A well-quantized 70B model on 2× A100 often outperforms a poorly-served 405B model on 8× A100.
  • Ecosystem maturity: Llama has the deepest ecosystem — any problem you encounter, someone has solved it. DeepSeek models may require newer framework versions for MLA support.
  • Update cadence: Plan for model upgrades every 3–6 months. Design serving infrastructure for easy model swaps — containerize, version model artifacts, maintain evaluation pipelines.

🏆 Best for Startups

Pick: Qwen 2.5 7B/14B (Apache 2.0)

Maximum license flexibility, strong performance, broad size range from 0.5B to 72B allows you to scale without switching families. Pre-quantized models available everywhere.

🏢 Best for Enterprise

Pick: Llama 3.3 70B

Best ecosystem support, Meta backing provides long-term stability, community license works for almost all enterprise use cases. Largest pool of fine-tuned variants and operational knowledge.

🔬 Best for AI/ML Teams

Pick: DeepSeek-V3 / R1

Highest raw performance, MIT license enables any use, architectural innovations (MLA, MoE) push serving efficiency boundaries. Best choice when you have GPU infra and ML engineering talent.

📱 Best for Edge / Mobile

Pick: Qwen 2.5 0.5B–3B or Llama 3.2 1B–3B

Both families offer sub-4B models optimized for mobile. Qwen has more size options; Llama has better GGUF/llama.cpp ecosystem for on-device inference.

The meta-strategy: Don't bet on a single family. Design your serving infrastructure to be model-agnostic using OpenAI-compatible APIs (vLLM, SGLang, Ollama all support this). This lets you hot-swap models as the landscape evolves — because in six months, the best model will be one that doesn't exist yet.
