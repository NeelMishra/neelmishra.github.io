Retrieval: Dense, Sparse, Hybrid

MLOps Series RAG Systems

Retrieval is the backbone of every RAG system. The quality of your generated answers is bounded by the quality of retrieved context. This post covers the three dominant retrieval paradigms—dense, sparse, and hybrid—explaining the mathematics, trade-offs, and implementation details you need to choose and combine them effectively.

Dense Retrieval

Dense retrieval encodes both queries and documents into fixed-dimensional vectors using a neural encoder (typically a bi-encoder Transformer). Relevance is measured by the cosine similarity or inner product between the query vector q and document vector d:

sim(q, d) = (q · d) / (‖q‖ · ‖d‖)

The key advantage of dense retrieval is semantic matching. The query "how to fix a flat tire" will match a document containing "tire puncture repair steps" even though they share few exact terms. This is because the encoder has learned to project semantically similar texts into nearby regions of the embedding space.

# Dense retrieval with sentence-transformers from sentence_transformers import SentenceTransformer import numpy as np model = SentenceTransformer('all-MiniLM-L6-v2') # Encode documents once at index time documents = [ "Retrieval-Augmented Generation combines search with LLMs.", "BM25 is a probabilistic ranking function for text retrieval.", "HNSW builds a navigable small-world graph for fast ANN search.", "Product Quantization compresses vectors for memory efficiency.", ] doc_embeddings = model.encode(documents, normalize_embeddings=True) # Encode query at search time query = "How does RAG work?" query_embedding = model.encode([query], normalize_embeddings=True) # Cosine similarity (dot product after normalization) scores = np.dot(doc_embeddings, query_embedding.T).flatten() ranked = np.argsort(scores)[::-1] for idx in ranked: print(f"[{scores[idx]:.3f}] {documents[idx]}")
Model choice matters: For production RAG, consider models fine-tuned for retrieval like bge-large-en-v1.5, e5-large-v2, or gte-large. They outperform general-purpose models by 5-10 points on MTEB benchmarks.

Limitations of dense retrieval include: (1) poor handling of exact keyword matches and rare terms, (2) domain shift when the encoder was trained on different data, and (3) the need to re-encode all documents when switching models.

Sparse Retrieval

Sparse retrieval represents documents and queries as high-dimensional sparse vectors where each dimension corresponds to a vocabulary term. Only the terms present in the text have non-zero weights. The two most important sparse methods are TF-IDF and BM25.

Sparse methods excel at exact keyword matching and rare term sensitivity. If a user searches for an error code like "CUDA_ERROR_OUT_OF_MEMORY", a sparse method will reliably find documents containing that exact string, whereas a dense encoder might not capture its significance.

TF-IDF Explained

Term Frequency–Inverse Document Frequency assigns each term a weight that increases with its frequency in a document but decreases with its frequency across the corpus. Formally:

TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)

Where TF(t, d) is the number of times term t appears in document d (often normalized by document length), and IDF(t, D) = log(N / df(t)) where N is the total number of documents and df(t) is the number of documents containing term t.

The IDF component down-weights common terms like "the" and "is" (which appear in nearly every document) and up-weights rare, discriminative terms like "quantization" or "HNSW".

# TF-IDF from scratch import math from collections import Counter def compute_tf(doc_tokens): counts = Counter(doc_tokens) total = len(doc_tokens) return {t: c / total for t, c in counts.items()} def compute_idf(corpus_tokens): N = len(corpus_tokens) df = Counter() for doc in corpus_tokens: for term in set(doc): df[term] += 1 return {t: math.log(N / v) for t, v in df.items()} def tfidf_score(query_tokens, doc_tokens, idf): tf = compute_tf(doc_tokens) score = 0.0 for term in query_tokens: if term in tf: score += tf[term] * idf.get(term, 0.0) return score

BM25 Algorithm

BM25 (Best Matching 25) is a probabilistic ranking function that improves on TF-IDF with two key refinements: term frequency saturation (via parameter k1) and document length normalization (via parameter b). The scoring formula is:

BM25(q, d) = Σ IDF(t) × [ f(t,d) × (k1 + 1) ] / [ f(t,d) + k1 × (1 − b + b × |d| / avgdl) ]

Where f(t,d) is the raw term frequency of t in d, |d| is the document length, avgdl is the average document length in the corpus, k1 (typically 1.2–2.0) controls how quickly TF saturates, and b (typically 0.75) controls length normalization strength.

When k1 = 0, BM25 reduces to IDF-only scoring. As k1 → ∞, it approaches raw TF×IDF. The saturation curve means that the 10th occurrence of a term contributes much less than the 2nd occurrence — an intuitive property that TF-IDF lacks.

# BM25 implementation import math from collections import Counter class BM25: def __init__(self, corpus, k1=1.5, b=0.75): self.k1 = k1 self.b = b self.corpus = [doc.lower().split() for doc in corpus] self.N = len(self.corpus) self.avgdl = sum(len(d) for d in self.corpus) / self.N self.df = Counter() for doc in self.corpus: for term in set(doc): self.df[term] += 1 def idf(self, term): n = self.df.get(term, 0) return math.log((self.N - n + 0.5) / (n + 0.5) + 1.0) def score(self, query, doc_idx): doc = self.corpus[doc_idx] dl = len(doc) tf = Counter(doc) s = 0.0 for term in query.lower().split(): f = tf.get(term, 0) numerator = f * (self.k1 + 1) denominator = f + self.k1 * (1 - self.b + self.b * dl / self.avgdl) s += self.idf(term) * numerator / denominator return s def rank(self, query, top_k=10): scores = [(i, self.score(query, i)) for i in range(self.N)] scores.sort(key=lambda x: x[1], reverse=True) return scores[:top_k]
Why BM25 is still relevant: Despite the rise of dense retrievers, BM25 remains competitive for keyword-heavy queries, domain-specific jargon, and zero-shot scenarios where no labeled data exists for fine-tuning dense models.

Hybrid Approaches

Hybrid retrieval combines dense and sparse signals. The intuition is that each method has blind spots the other covers: dense handles paraphrases and semantic similarity, while sparse handles exact keywords and rare terms. Two common fusion strategies are:

Score-Level Fusion

Normalize and linearly combine scores: α·dense + (1−α)·sparse. Simple but sensitive to score distribution differences.

Rank-Level Fusion (RRF)

Combine rankings: Σ 1/(k + rank_i). Robust to score scale differences. K=60 is standard.

End-to-End Implementation

Here is a complete hybrid retrieval pipeline combining sentence-transformers for dense and rank-bm25 for sparse retrieval:

# Hybrid retrieval: dense + BM25 with RRF from sentence_transformers import SentenceTransformer from rank_bm25 import BM25Okapi import numpy as np # Initialize models encoder = SentenceTransformer('all-MiniLM-L6-v2') corpus = ["doc1 text...", "doc2 text...", ...] tokenized = [doc.lower().split() for doc in corpus] bm25 = BM25Okapi(tokenized) # Dense embeddings doc_embs = encoder.encode(corpus, normalize_embeddings=True) def hybrid_search(query, top_k=10, rrf_k=60): # Dense scores q_emb = encoder.encode([query], normalize_embeddings=True) dense_scores = np.dot(doc_embs, q_emb.T).flatten() dense_rank = np.argsort(dense_scores)[::-1] # Sparse BM25 scores sparse_scores = bm25.get_scores(query.lower().split()) sparse_rank = np.argsort(sparse_scores)[::-1] # Reciprocal Rank Fusion rrf_scores = {} for rank, idx in enumerate(dense_rank): rrf_scores[idx] = rrf_scores.get(idx, 0) + 1 / (rrf_k + rank + 1) for rank, idx in enumerate(sparse_rank): rrf_scores[idx] = rrf_scores.get(idx, 0) + 1 / (rrf_k + rank + 1) sorted_ids = sorted(rrf_scores, key=rrf_scores.get, reverse=True) return sorted_ids[:top_k]

This hybrid approach consistently outperforms either retriever alone. On the BEIR benchmark, hybrid retrieval improves NDCG@10 by 3–8 points over the best single retriever across diverse domains.
