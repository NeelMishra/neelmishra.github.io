Sparse Dispatch and Combine from Scratch

The core MoE systems problem is moving tokens to experts efficiently. Build dispatch masks, grouped expert batches, combine weights, and vectorized alternatives.

April 2026 · 36 min read MLOps · Mixture of Experts · Part 3

The dispatch problem

After routing, each token has one or more expert ids. GPUs, however, do not want thousands of tiny calls like “run expert 7 on token 31.” They want contiguous batches. Sparse dispatch converts token-level routing decisions into per-expert mini-batches.

Route

Token 0 chooses experts 2 and 5; token 1 chooses 5 and 1.

Dispatch

Pack all tokens for expert 5 into one buffer and run expert 5 once.

Combine

Scatter outputs back to token positions using router weights.

Dispatch masks

The clearest representation is a boolean dispatch mask with shape (tokens, experts, capacity). A true entry means “copy this token into this expert slot.”

def make_dispatch_and_combine(top_idx, top_weight, num_experts, capacity):
    tokens, k = top_idx.shape
    dispatch = torch.zeros(tokens, num_experts, capacity, device=top_idx.device, dtype=torch.bool)
    combine = torch.zeros(tokens, num_experts, capacity, device=top_weight.device, dtype=top_weight.dtype)
    counters = torch.zeros(num_experts, device=top_idx.device, dtype=torch.long)
    dropped = 0

    for t in range(tokens):
        for j in range(k):
            e = int(top_idx[t, j])
            slot = int(counters[e])
            if slot < capacity:
                dispatch[t, e, slot] = True
                combine[t, e, slot] = top_weight[t, j]
                counters[e] += 1
            else:
                dropped += 1
    return dispatch, combine, counters, dropped

This representation is easy to reason about but memory-heavy. It is still excellent for tests because it makes correctness obvious.

Packed expert batches

Given a dispatch mask, one line can build expert inputs:

# tokens: (T, D), dispatch: (T, E, C)
expert_inputs = torch.einsum("td,tec->ecd", tokens, dispatch.float())

The resulting tensor has shape (experts, capacity, d_model). Each expert gets a fixed-capacity batch. Unused slots are zeros and should not contribute to the final output because their combine weights remain zero.

Combine step

After running experts, combine maps (experts, capacity, d_model) back to (tokens, d_model):

# expert_outputs: (E, C, D), combine: (T, E, C)
combined = torch.einsum("ecd,tec->td", expert_outputs, combine)

For top-2 routing, a token can have two non-zero combine entries. The weighted sum becomes the MoE layer output for that token.

Vectorized reference MoE

This version uses fixed-capacity buffers and einsum for readability. It assumes all experts share the same architecture but not the same weights.

class MaskedDispatchMoE(nn.Module):
    def forward(self, x):
        b, s, d = x.shape
        tokens = x.reshape(b * s, d)
        logits = self.router(tokens.float())
        probs = torch.softmax(logits, dim=-1)
        weight, idx = torch.topk(probs, self.top_k, dim=-1)
        weight = weight / weight.sum(dim=-1, keepdim=True).clamp_min(1e-9)

        capacity = math.ceil(self.capacity_factor * tokens.shape[0] * self.top_k / self.num_experts)
        dispatch, combine, counts, dropped = make_dispatch_and_combine(idx, weight, self.num_experts, capacity)
        expert_inputs = torch.einsum("td,tec->ecd", tokens, dispatch.to(tokens.dtype))

        expert_outputs = []
        for e, expert in enumerate(self.experts):
            expert_outputs.append(expert(expert_inputs[e]))
        expert_outputs = torch.stack(expert_outputs, dim=0)

        out = torch.einsum("ecd,tec->td", expert_outputs, combine)
        return out.reshape(b, s, d), {"counts": counts, "dropped": dropped}

Performance notes

  • Dense masks are simple but can be too large for long contexts and many experts.
  • Production kernels pack tokens with prefix sums and sort/group by expert id.
  • Fixed capacity improves static shapes but can waste FLOPs on padding.
  • Dynamic packing reduces padding but complicates compilation, distributed communication, and CUDA graph capture.
  • Always benchmark end-to-end tokens/sec; a beautiful dispatch algorithm can still lose if it creates synchronization points.
