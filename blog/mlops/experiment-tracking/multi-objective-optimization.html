Multi-Objective Optimization in HPO

MLOps Series Experiment Tracking

Why Multi-Objective Optimization?

Real-world ML systems rarely optimize a single metric. A production model must balance accuracy against inference latency, model size, fairness constraints, and resource cost. Single-objective optimization forces you to collapse these trade-offs into one number, losing information about the trade-off surface.

Multi-objective optimization (MOO) finds the set of solutions where no objective can be improved without degrading another. This set — the Pareto frontier — gives decision-makers a menu of options rather than a single "best" model.

Example trade-offs: Accuracy vs. latency (edge deployment), F1-score vs. model size (mobile), precision vs. recall (fraud detection), accuracy vs. fairness (lending models).

The Pareto Frontier

A solution is Pareto-optimal if no other solution is better in all objectives simultaneously. The collection of all Pareto-optimal solutions forms the Pareto frontier — a curve (or surface, in 3+ dimensions) representing the best achievable trade-offs.

Points on the Pareto frontier represent non-dominated solutions. The gray points are dominated — there exists at least one Pareto-optimal solution that is better in every objective. The frontier reveals the cost of improving one metric: moving right (higher accuracy) requires moving up (higher latency).

NSGA-II Algorithm

NSGA-II (Non-dominated Sorting Genetic Algorithm II) is the most widely used multi-objective evolutionary algorithm. It maintains a population of candidate solutions and evolves them through selection, crossover, and mutation while preserving diversity along the Pareto frontier.

The algorithm works in three stages per generation: (1) non-dominated sorting ranks solutions into fronts, (2) crowding distance measures how spread out solutions are within each front, and (3) selection prefers solutions from better fronts and, within the same front, solutions with higher crowding distance. 

# NSGA-II core concepts (simplified)
import numpy as np

def non_dominated_sort(objectives):
    # objectives: array of shape (n_solutions, n_objectives)
    n = len(objectives)
    fronts = []
    domination_count = np.zeros(n, dtype=int)
    dominated_set = [[] for _ in range(n)]

    for i in range(n):
        for j in range(n):
            if dominates(objectives[i], objectives[j]):
                dominated_set[i].append(j)
            elif dominates(objectives[j], objectives[i]):
                domination_count[i] += 1

    # First front: solutions dominated by no one
    front = [i for i in range(n) if domination_count[i] == 0]
    fronts.append(front)
    return fronts

def dominates(a, b):
    # a dominates b if a is no worse in all objectives and better in at least one
    return all(ai <= bi for ai, bi in zip(a, b)) and \
           any(ai < bi for ai, bi in zip(a, b))
Why crowding distance matters: Without it, the algorithm would cluster solutions in one region of the Pareto front. Crowding distance ensures diversity by favoring solutions in less-crowded regions, giving you a well-spread set of trade-off options.

Scalarization Methods

Scalarization converts a multi-objective problem into a single-objective one by combining objectives with weights. While simpler than full MOO, it requires choosing weights upfront — which means you're encoding preferences before seeing the trade-off surface. 

# Scalarization approaches
import numpy as np

def linear_scalarization(accuracy, latency, w_acc=0.7, w_lat=0.3):
    # Weighted sum — simple but misses concave Pareto regions
    return w_acc * accuracy - w_lat * latency

def chebyshev_scalarization(objectives, weights, ideal_point):
    # Minimax approach — handles non-convex Pareto fronts
    diffs = weights * np.abs(objectives - ideal_point)
    return np.max(diffs)

def epsilon_constraint(accuracy, latency, latency_budget=50):
    # Optimize accuracy subject to latency constraint
    if latency > latency_budget:
        return None  # Infeasible — violates constraint
    return accuracy

Linear Scalarization

Fast, single-objective. Cannot find solutions on concave regions of the Pareto front. Weight selection is subjective.

Chebyshev Scalarization

Handles non-convex fronts. Computationally efficient. Still requires weight specification but is more robust.

Implementation with Optuna

Optuna natively supports multi-objective optimization via its NSGAIISampler. You define an objective function that returns multiple values, and Optuna maintains the Pareto frontier across trials. 

# Multi-objective HPO with Optuna
import optuna
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import cross_val_score
import time

def objective(trial):
    # Suggest hyperparameters
    n_estimators = trial.suggest_int("n_estimators", 50, 500)
    max_depth = trial.suggest_int("max_depth", 2, 10)
    learning_rate = trial.suggest_float("learning_rate", 0.01, 0.3, log=True)

    model = GradientBoostingClassifier(
        n_estimators=n_estimators,
        max_depth=max_depth,
        learning_rate=learning_rate
    )

    # Objective 1: accuracy (maximize)
    accuracy = cross_val_score(model, X, y, cv=5).mean()

    # Objective 2: inference latency (minimize)
    model.fit(X, y)
    start = time.perf_counter()
    model.predict(X[:100])
    latency = time.perf_counter() - start

    return accuracy, latency

# Create multi-objective study
study = optuna.create_study(
    directions=["maximize", "minimize"],
    sampler=optuna.samplers.NSGAIISampler(seed=42)
)

study.optimize(objective, n_trials=200)

# Get Pareto-optimal trials
pareto_trials = study.best_trials
for t in pareto_trials:
    print(f"Accuracy={t.values[0]:.4f}, Latency={t.values[1]:.4f}ms")
    print(f"  Params: {t.params}")
Watch out: More trials does not always mean a better Pareto front. If objectives are noisy (e.g., latency varies between runs), increase trial repetitions or use robust statistical estimators for each objective.

Visualization and Decision Making

Optuna provides built-in visualization for multi-objective studies. The Pareto front plot, parallel coordinate plot, and contour plots help stakeholders understand trade-offs and select the best model for their constraints. 

# Visualizing multi-objective results
from optuna.visualization import (
    plot_pareto_front,
    plot_parallel_coordinate,
    plot_param_importances
)

# Pareto front — shows trade-off surface
fig_pareto = plot_pareto_front(
    study,
    target_names=["Accuracy", "Latency (ms)"]
)
fig_pareto.show()

# Parallel coordinates — shows parameter interactions
fig_parallel = plot_parallel_coordinate(
    study,
    params=["n_estimators", "max_depth", "learning_rate"]
)
fig_parallel.show()

# Decision: pick model meeting latency budget
latency_budget = 0.01  # 10ms
feasible = [t for t in study.best_trials if t.values[1] < latency_budget]
best = max(feasible, key=lambda t: t.values[0])
print(f"Selected model: accuracy={best.values[0]:.4f}, params={best.params}")
Decision framework: (1) Plot the Pareto front. (2) Apply hard constraints (latency < X, size < Y). (3) Among feasible solutions, select the one maximizing the primary metric. Document the reasoning for auditing.
