Experiment Analysis: Statistical Significance

MLOps Series Experiment Tracking

Why Statistical Testing Matters

Comparing two models by their average accuracy on a single test set is misleading. A model scoring 91.3% versus 90.8% may not actually be better — the difference could be noise from the data split, random initialization, or hardware variability. Statistical testing quantifies whether an observed difference is likely real.

Without statistical rigor, teams chase phantom improvements, waste compute on insignificant gains, and deploy models that regress in production. Every experiment comparison should answer: "Is this difference larger than what we'd expect from chance alone?"

Rule of thumb: If you can't state the p-value or confidence interval for a model comparison, you don't know if the improvement is real. Report effect size alongside statistical significance.

T-Tests for Model Comparison

The paired t-test is the workhorse of model comparison. When two models are evaluated on the same k-fold splits, you get paired performance scores. The test checks whether the mean difference across folds is significantly different from zero. 

# Paired t-test for model comparison across k-fold
import numpy as np
from scipy import stats
from sklearn.model_selection import cross_val_score

def compare_models(model_a, model_b, X, y, cv=10, alpha=0.05):
    # Use same folds for both models
    scores_a = cross_val_score(model_a, X, y, cv=cv, scoring="accuracy")
    scores_b = cross_val_score(model_b, X, y, cv=cv, scoring="accuracy")

    # Paired t-test on fold-level differences
    differences = scores_b - scores_a
    t_stat, p_value = stats.ttest_rel(scores_b, scores_a)

    # Effect size (Cohen's d for paired samples)
    cohens_d = np.mean(differences) / np.std(differences, ddof=1)

    print(f"Model A: {scores_a.mean():.4f} ± {scores_a.std():.4f}")
    print(f"Model B: {scores_b.mean():.4f} ± {scores_b.std():.4f}")
    print(f"Mean diff: {differences.mean():.4f}")
    print(f"t-stat: {t_stat:.3f}, p-value: {p_value:.4f}")
    print(f"Cohen's d: {cohens_d:.3f}")
    print(f"Significant: {p_value < alpha}")

    return {"t_stat": t_stat, "p_value": p_value, "cohens_d": cohens_d}
Caution: The standard paired t-test assumes independence between folds — which is violated in k-fold CV because training sets overlap. Use the corrected resampled t-test (Nadeau & Bengio, 2003) for more reliable p-values. 
# Corrected resampled t-test (Nadeau & Bengio)
def corrected_ttest(scores_a, scores_b, n_train, n_test):
    differences = scores_b - scores_a
    n = len(differences)
    mean_diff = np.mean(differences)
    var_diff = np.var(differences, ddof=1)

    # Correction factor for overlapping training sets
    correction = (1 / n) + (n_test / n_train)
    corrected_var = correction * var_diff

    t_stat = mean_diff / np.sqrt(corrected_var)
    p_value = 2 * stats.t.sf(np.abs(t_stat), df=n - 1)

    return t_stat, p_value

Bootstrapping Methods

Bootstrapping makes no distributional assumptions. You resample the test set predictions with replacement many times, compute the metric for each resample, and build a distribution of the metric difference. This empirical distribution tells you how uncertain the comparison is. 

# Bootstrap comparison of two models
import numpy as np

def bootstrap_compare(y_true, preds_a, preds_b, n_bootstrap=10000, alpha=0.05):
    n = len(y_true)
    rng = np.random.default_rng(seed=42)
    diff_scores = []

    for _ in range(n_bootstrap):
        # Resample with replacement
        idx = rng.choice(n, size=n, replace=True)

        acc_a = np.mean(preds_a[idx] == y_true[idx])
        acc_b = np.mean(preds_b[idx] == y_true[idx])
        diff_scores.append(acc_b - acc_a)

    diff_scores = np.array(diff_scores)

    # Percentile confidence interval
    lower = np.percentile(diff_scores, 100 * alpha / 2)
    upper = np.percentile(diff_scores, 100 * (1 - alpha / 2))

    # p-value: proportion of bootstraps where diff <= 0
    p_value = np.mean(diff_scores <= 0)

    print(f"Bootstrap mean diff: {diff_scores.mean():.4f}")
    print(f"95% CI: [{lower:.4f}, {upper:.4f}]")
    print(f"p-value: {p_value:.4f}")

    return diff_scores, (lower, upper), p_value
Advantage of bootstrapping: Works for any metric — AUC, F1, BLEU, custom business metrics. No assumptions about the distribution shape. Use 10,000+ resamples for stable confidence intervals.

Confidence Intervals

A confidence interval (CI) is more informative than a p-value alone. It tells you the range of plausible effect sizes, not just whether the effect exists. A 95% CI of [0.001, 0.015] means the true improvement is likely between 0.1% and 1.5% — possibly too small to matter in practice. 

# Multiple CI methods for model performance
from scipy import stats
import numpy as np

def wilson_ci(n_correct, n_total, alpha=0.05):
    # Wilson score interval — better than normal approx for proportions
    z = stats.norm.ppf(1 - alpha / 2)
    p_hat = n_correct / n_total
    denom = 1 + z**2 / n_total
    center = (p_hat + z**2 / (2 * n_total)) / denom
    margin = z * np.sqrt(p_hat * (1 - p_hat) / n_total + z**2 / (4 * n_total**2)) / denom
    return center - margin, center + margin

def bootstrap_ci(scores, n_bootstrap=10000, alpha=0.05):
    # BCa bootstrap interval
    rng = np.random.default_rng(42)
    boot_means = []
    for _ in range(n_bootstrap):
        sample = rng.choice(scores, size=len(scores), replace=True)
        boot_means.append(np.mean(sample))
    boot_means = np.array(boot_means)
    return np.percentile(boot_means, [100*alpha/2, 100*(1-alpha/2)])

Visualization Techniques

Good visualizations communicate uncertainty at a glance. Instead of bar charts with single numbers, use box plots across cross-validation folds, bootstrap distribution histograms, and error bar plots with confidence intervals. 

# Visualization for experiment comparison
import matplotlib.pyplot as plt
import numpy as np

def plot_model_comparison(results_dict, metric_name="Accuracy"):
    fig, axes = plt.subplots(1, 3, figsize=(15, 5))

    models = list(results_dict.keys())
    scores = [results_dict[m] for m in models]

    # 1. Box plot of fold scores
    axes[0].boxplot(scores, labels=models)
    axes[0].set_ylabel(metric_name)
    axes[0].set_title("Cross-Validation Distribution")

    # 2. Bootstrap difference distribution
    diff_dist, ci, pval = bootstrap_compare(
        y_true, preds_a, preds_b
    )
    axes[1].hist(diff_dist, bins=50, alpha=0.7, color="#0a8f6a")
    axes[1].axvline(x=0, color="red", linestyle="--")
    axes[1].set_title("Bootstrap Δ Distribution")

    # 3. Error bar plot with CIs
    means = [np.mean(s) for s in scores]
    cis = [1.96 * np.std(s) / np.sqrt(len(s)) for s in scores]
    axes[2].barh(models, means, xerr=cis, color="#0a8f6a", alpha=0.7)
    axes[2].set_title("Mean ± 95% CI")

    plt.tight_layout()
    plt.savefig("model_comparison.png", dpi=150)
    plt.show()

Common Pitfalls

Statistical testing in ML has several traps that lead to false conclusions. Understanding these pitfalls is as important as knowing the tests themselves.

⚠ Multiple Comparisons

Comparing 20 models pairwise gives 190 tests. At α=0.05, you'll get ~10 false positives. Apply Bonferroni correction: use α/n as your significance threshold.

⚠ p-Hacking

Running tests until you find significance, adding folds, or changing metrics post-hoc inflates false positive rates. Pre-register your analysis plan before running experiments.

⚠ Ignoring Effect Size

A large dataset can make a 0.01% improvement statistically significant. Always report effect size (Cohen's d) alongside p-values. Decide the minimum meaningful improvement beforehand.

⚠ Test Set Reuse

Evaluating many models on the same test set causes adaptive overfitting. Use fresh holdout data for final comparisons, or use Dwork et al.'s reusable holdout mechanism.

# Bonferroni correction for multiple comparisons
from scipy import stats
import itertools

def pairwise_compare_corrected(model_scores, alpha=0.05):
    models = list(model_scores.keys())
    pairs = list(itertools.combinations(models, 2))
    n_tests = len(pairs)
    corrected_alpha = alpha / n_tests  # Bonferroni

    print(f"Running {n_tests} pairwise tests")
    print(f"Corrected α = {corrected_alpha:.5f}")

    for m1, m2 in pairs:
        t_stat, p_val = stats.ttest_rel(
            model_scores[m1], model_scores[m2]
        )
        sig = "*" if p_val < corrected_alpha else ""
        print(f"  {m1} vs {m2}: p={p_val:.5f} {sig}")
Best practice: Report results as "Model B improves accuracy by 0.9% (95% CI: [0.3%, 1.5%], p=0.003, Cohen's d=0.45)". This gives readers the magnitude, uncertainty, significance, and practical effect size in one sentence.
