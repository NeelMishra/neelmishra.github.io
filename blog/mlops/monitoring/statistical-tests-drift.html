    Statistical Tests: KS, PSI, Chi-Square, MMD

    MLOps SeriesMonitoring & Observability

    Introduction

    Drift detection is the backbone of ML monitoring. A model trained on historical data will inevitably encounter distribution shifts in production—seasonal patterns change, user behaviour evolves, and upstream data pipelines break in subtle ways. Statistical tests give us a principled, quantitative framework for answering the question: "Has the data distribution changed enough to worry about?"

    This post covers four workhorses of drift detection: the Kolmogorov–Smirnov test for continuous features, the Population Stability Index for binned distributions, the Chi-Square test for categorical features, and Maximum Mean Discrepancy for high-dimensional or multivariate data. For each test we cover the mathematical foundation, practical implementation, and guidance on when to use it.

    Prerequisites: Basic familiarity with probability distributions, hypothesis testing (p-values, null hypotheses), and Python’s NumPy/SciPy stack.

    Kolmogorov–Smirnov Test

    The KS test is a non-parametric test that compares two empirical cumulative distribution functions (ECDFs). The test statistic D is the maximum absolute vertical distance between the two ECDFs:

    D = sup |Fref(x) − Fprod(x)|

    Because it operates on CDFs rather than binned histograms, the KS test is distribution-free—it makes no assumptions about the underlying shape of the data. This makes it an excellent default choice for continuous numerical features.

    Strengths: distribution-free, well-understood critical values, works on any continuous data. Weaknesses: sensitive to sample size (large samples almost always reject H₀), only captures univariate drift.

    import numpy as np from scipy import stats def ks_drift_test(reference, production, alpha=0.05): # Two-sample Kolmogorov-Smirnov test for drift detection # Returns statistic D, p-value, and boolean drift flag reference = np.asarray(reference) production = np.asarray(production) # Scipy's two-sample KS test statistic, p_value = stats.ks_2samp(reference, production) return { "test": "kolmogorov_smirnov", "statistic": round(statistic, 6), "p_value": round(p_value, 6), "drift_detected": p_value < alpha, "alpha": alpha, "ref_size": len(reference), "prod_size": len(production) } def ks_per_feature(ref_df, prod_df, columns, alpha=0.05): # Run KS test on every numerical column and collect results results = {} for col in columns: results[col] = ks_drift_test( ref_df[col].dropna().values, prod_df[col].dropna().values, alpha=alpha ) return results

    Population Stability Index (PSI)

    PSI was originally developed in credit-risk modelling to measure how much a score distribution has shifted between two time periods. Unlike the KS test, PSI produces a single scalar that is easy to threshold and trend over time.

    The formula bins both distributions into k buckets and sums a symmetric divergence term for each bucket:

    PSI = Σi (Pi − Qi) · ln(Pi / Qi)

    PSI < 0.10

    No significant shift. The production distribution closely matches the reference. No action required.

    PSI 0.10 – 0.25

    Moderate shift. Investigate root cause. May be seasonal variation or a data pipeline change that warrants monitoring.

    PSI > 0.25

    Significant shift. The distribution has materially changed. Trigger model retraining or manual review.

    Bin Strategy

    Use 10–20 equal-frequency (quantile) bins derived from the reference set. Avoid equal-width bins which can create empty buckets on skewed features.

    import numpy as np def compute_psi(reference, production, n_bins=10, epsilon=1e-4): # Population Stability Index with quantile binning reference = np.asarray(reference) production = np.asarray(production) # Create quantile-based bin edges from the reference set quantiles = np.linspace(0, 100, n_bins + 1) bin_edges = np.percentile(reference, quantiles) bin_edges[0] = -np.inf bin_edges[-1] = np.inf # Compute proportions for each bin ref_counts = np.histogram(reference, bins=bin_edges)[0] prod_counts = np.histogram(production, bins=bin_edges)[0] ref_proportions = ref_counts / len(reference) + epsilon prod_proportions = prod_counts / len(production) + epsilon # PSI formula: sum of (P - Q) * ln(P / Q) psi_values = (prod_proportions - ref_proportions) * np.log( prod_proportions / ref_proportions ) total_psi = float(np.sum(psi_values)) return { "test": "psi", "psi": round(total_psi, 6), "per_bin_psi": [round(v, 6) for v in psi_values], "severity": "none" if total_psi < 0.1 else "moderate" if total_psi < 0.25 else "significant", "n_bins": n_bins }
    Pitfall: PSI is sensitive to the number of bins. Too few bins mask localised shifts; too many create noisy per-bin estimates. Stick to 10–20 quantile-based bins and validate on held-out data before deploying to production.

    Chi-Square Test

    When your feature is categorical—country codes, device types, error categories—the KS test and PSI are inappropriate because there is no natural ordering. The Chi-Square goodness-of-fit test compares observed category frequencies to expected frequencies from the reference distribution.

    χ² = Σi (Oi − Ei)² / Ei

    where Oi is the observed count in category i and Ei is the expected count derived from the reference proportions scaled to the production sample size.

    import numpy as np from scipy import stats from collections import Counter def chi_square_drift(ref_categories, prod_categories, alpha=0.05): # Chi-square test for categorical feature drift ref_counts = Counter(ref_categories) prod_counts = Counter(prod_categories) # Align categories across both distributions all_cats = sorted(set(ref_counts.keys()) | set(prod_counts.keys())) ref_freq = np.array([ref_counts.get(c, 0) for c in all_cats], dtype=float) prod_freq = np.array([prod_counts.get(c, 0) for c in all_cats], dtype=float) # Scale reference frequencies to production sample size ref_proportions = ref_freq / ref_freq.sum() expected = ref_proportions * prod_freq.sum() # Remove categories with zero expected count mask = expected > 0 observed = prod_freq[mask] expected = expected[mask] statistic, p_value = stats.chisquare(observed, f_exp=expected) return { "test": "chi_square", "statistic": round(float(statistic), 6), "p_value": round(float(p_value), 6), "drift_detected": p_value < alpha, "degrees_of_freedom": int(mask.sum()) - 1, "n_categories": len(all_cats) } def detect_new_categories(ref_categories, prod_categories): # Identify categories that appear in production but not reference ref_set = set(ref_categories) prod_set = set(prod_categories) novel = prod_set - ref_set missing = ref_set - prod_set return { "novel_categories": sorted(novel), "missing_categories": sorted(missing), "novel_count": len(novel), "missing_count": len(missing) }
    Note: Always check for novel categories that appear in production but were absent during training. A pure Chi-Square test will miss these if you only test the known category set. The detect_new_categories helper above catches this edge case.

    Maximum Mean Discrepancy (MMD)

    When you need to compare multivariate distributions—multiple features simultaneously, or high-dimensional embeddings—univariate tests fall short because they ignore feature correlations. MMD is a kernel-based two-sample test that operates in a reproducing kernel Hilbert space (RKHS).

    The key idea: map both samples into a high-dimensional feature space via a kernel function, then measure the distance between their mean embeddings. If the kernel is characteristic (e.g., Gaussian RBF), MMD equals zero if and only if the two distributions are identical.

    MMD² = E[k(x,x′)] + E[k(y,y′)] − 2E[k(x,y)]

    import numpy as np def gaussian_kernel(x, y, sigma=1.0): # RBF kernel: k(x, y) = exp(-||x - y||^2 / (2 * sigma^2)) diff = x[:, None, :] - y[None, :, :] sq_dist = np.sum(diff ** 2, axis=-1) return np.exp(-sq_dist / (2 * sigma ** 2)) def compute_mmd(reference, production, sigma=1.0): # Biased estimator of MMD^2 with Gaussian RBF kernel reference = np.asarray(reference) production = np.asarray(production) k_xx = gaussian_kernel(reference, reference, sigma) k_yy = gaussian_kernel(production, production, sigma) k_xy = gaussian_kernel(reference, production, sigma) mmd_sq = k_xx.mean() + k_yy.mean() - 2 * k_xy.mean() return float(mmd_sq) def mmd_permutation_test(reference, production, sigma=1.0, n_perms=500, alpha=0.05): # Permutation test for statistical significance of MMD reference = np.asarray(reference) production = np.asarray(production) observed_mmd = compute_mmd(reference, production, sigma) # Pool both samples and shuffle to build null distribution combined = np.vstack([reference, production]) n_ref = len(reference) null_mmds = [] for _ in range(n_perms): perm = np.random.permutation(len(combined)) perm_ref = combined[perm[:n_ref]] perm_prod = combined[perm[n_ref:]] null_mmds.append(compute_mmd(perm_ref, perm_prod, sigma)) null_mmds = np.array(null_mmds) p_value = float(np.mean(null_mmds >= observed_mmd)) return { "test": "mmd", "mmd_squared": round(observed_mmd, 8), "p_value": round(p_value, 4), "drift_detected": p_value < alpha, "n_permutations": n_perms, "sigma": sigma }
    Computational cost: Naïve MMD is O(n²) in sample size due to the kernel matrix. For large datasets, use random Fourier features or the linear-time MMD estimator to keep costs manageable.

    Choosing the Right Test

    The choice of statistical test depends on three factors: the data type of the feature, the dimensionality of the comparison, and the computational budget available in your monitoring pipeline.

    Continuous & Univariate

    Use KS test as the default. It is fast, distribution-free, and well-understood. Add PSI as a complementary scalar summary for trending.

    Categorical

    Use Chi-Square. It handles any number of categories and has clear degrees-of-freedom semantics. Supplement with a novel-category detector.

    Multivariate / Embeddings

    Use MMD with a Gaussian kernel. It captures joint distribution shifts and correlation changes that univariate tests miss entirely.

    Score / Binned Distributions

    Use PSI. Particularly useful when you need a single number that maps cleanly to severity thresholds (green / amber / red).

    class DriftTestRouter: # Automatically select the right drift test per feature def __init__(self, schema): # schema: dict mapping feature_name -> "continuous" | "categorical" self.schema = schema def test_feature(self, name, ref_data, prod_data, alpha=0.05): dtype = self.schema.get(name, "continuous") if dtype == "categorical": return chi_square_drift(ref_data, prod_data, alpha) else: return ks_drift_test(ref_data, prod_data, alpha) def test_all(self, ref_df, prod_df, alpha=0.05): # Run appropriate test for every feature in the schema results = {} for name in self.schema: results[name] = self.test_feature( name, ref_df[name].values, prod_df[name].values, alpha ) return results

    Putting It All Together

    In production you rarely run a single test in isolation. Instead, you build a drift monitoring pipeline that iterates over every feature, selects the appropriate test, computes the statistic, and emits structured results for downstream alerting and dashboarding.

    import json from datetime import datetime class DriftMonitor: # End-to-end drift monitoring for a feature set def __init__(self, schema, alert_callback=None): self.router = DriftTestRouter(schema) self.alert_callback = alert_callback self.history = [] def run(self, ref_df, prod_df, alpha=0.05): # Execute all drift tests and persist results results = self.router.test_all(ref_df, prod_df, alpha) report = { "timestamp": datetime.utcnow().isoformat(), "features": results, "drifted_features": [ k for k, v in results.items() if v.get("drift_detected", False) ] } self.history.append(report) # Fire alert if any feature drifted if report["drifted_features"] and self.alert_callback: self.alert_callback(report) return report def summary(self): # Return latest drift summary as JSON if not self.history: return None return json.dumps(self.history[-1], indent=2)
    Best practice: Log every drift test result—not just the alerts—to a time-series store. This gives you a historical baseline to distinguish genuine drift from statistical noise and helps you tune thresholds over time.
