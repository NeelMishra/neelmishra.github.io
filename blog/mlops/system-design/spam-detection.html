Design: Email / Message Spam Detection

MLOps Series ML System Design

Requirements & Scope

Spam detection protects billions of inboxes from unwanted messages. The challenge: spammers evolve faster than static rules can keep up.

Functional

  • Classify messages as spam / ham in real time
  • Support text, image, and link-based spam
  • Provide "Report spam" & "Not spam" feedback
  • Per-user personalisation (one user's spam is another's newsletter)

Non-Functional

  • Latency < 50 ms per message
  • False positive rate < 0.01 %
  • Process 10 B+ messages / day
  • Model updates every 30 minutes

Feature Extraction

Spam detection features come from the message content, sender metadata, and network signals.

Content Features

TF-IDF n-grams, character-level trigrams (catches obfuscated text like "V1@gra"), URL count, attachment types, and language detection.

Sender Features

Account age, sender reputation score, authentication results (SPF, DKIM, DMARC), and sending volume in the last hour. 

# Feature extraction pipeline
def extract_features(message):
    return {
        # Content
        "url_count":       count_urls(message.body),
        "has_attachment":  bool(message.attachments),
        "char_trigrams":   hash_ngrams(message.body, n=3),
        "subject_caps_ratio": caps_ratio(message.subject),
        # Sender
        "sender_age_days": (now() - message.sender.created_at).days,
        "sender_reputation": get_reputation(message.sender.id),
        "spf_pass":        message.auth.spf == "pass",
        "dkim_pass":       message.auth.dkim == "pass",
        # Network
        "sender_volume_1h": get_send_volume(message.sender.id, hours=1),
        "recipient_count":  len(message.recipients),
    }

Behavioural Features

How recipients interact with messages from this sender: open rate, reply rate, and "mark as spam" rate. These are computed from historical logs and updated hourly.

Feature hashing: Use feature hashing (hashing trick) for text features to keep the feature vector a fixed size even as vocabulary evolves.

Real-Time Classification

The classifier must score messages in under 50 ms, ruling out expensive transformer models for the primary path.

Two-Stage Approach

Stage 1: A lightweight logistic regression model with hashed features handles 95 % of traffic. Stage 2: Borderline messages (score 0.3–0.7) are re-scored by a deeper GBDT model. 

# Two-stage classification
def classify_message(features):
    # Stage 1: fast linear model
    score = linear_model.predict_proba(features)[0][1]

    if score > 0.9:
        return "spam", score
    if score < 0.1:
        return "ham", score

    # Stage 2: deeper model for ambiguous cases
    score = gbdt_model.predict_proba(features)[0][1]
    label = "spam" if score > 0.5 else "ham"
    return label, score

Feedback Loops

User feedback ("Report spam", "Not spam") is the most valuable training signal. It captures evolving spam patterns that the model hasn't seen yet. 

# Feedback ingestion pipeline
def process_feedback(message_id, action):
    # action: "report_spam" or "not_spam"
    label = 1 if action == "report_spam" else 0
    features = get_cached_features(message_id)
    write_to_training_store(features, label)

    # Update sender reputation in real time
    if action == "report_spam":
        decrease_reputation(message.sender_id, delta=0.1)
    else:
        increase_reputation(message.sender_id, delta=0.05)

Continuous Retraining

The model is retrained every 30 minutes on the latest feedback data, giving it a window of hours — not weeks — to adapt to new spam campaigns.

Feedback poisoning: Adversaries may report legitimate messages as spam. Use weighted feedback: trusted users' reports count more than new accounts.

Adversarial Robustness

Spammers actively try to evade detection. Common evasion techniques include character substitution ("V1@gra"), image-based text, URL shorteners, and account farming.

Defences

Character normalisation (homoglyph mapping), OCR on images, URL expansion and reputation scoring, and sender behaviour clustering all help counter evasion. 

# Homoglyph normalisation
HOMOGLYPHS = {"0": "o", "1": "l", "@": "a", "$": "s", "3": "e"}

def normalise_text(text):
    normalised = ""
    for ch in text.lower():
        normalised += HOMOGLYPHS.get(ch, ch)
    return normalised
Arms race: Treat spam detection as a security problem. Red-team your own system regularly and monitor for sudden drops in detection rate.

System Architecture

The architecture separates the real-time scoring path (top) from the feedback and retraining path (bottom), allowing the model to evolve continuously without affecting serving latency.
