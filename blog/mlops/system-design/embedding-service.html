Design: Embedding Service

MLOps Series ML System Design

Requirements & Scope

An embedding service provides dense vector representations of entities (users, items, queries, documents) for downstream ML systems — recommendations, search, and similarity matching.

Functional

  • Lookup pre-computed embeddings by entity ID
  • Compute embeddings on-the-fly for new entities
  • Nearest-neighbour search over embedding space
  • Version embeddings with model updates

Non-Functional

  • Lookup latency < 5 ms p99
  • Store 1 B+ embeddings (256-dim float32)
  • ANN search < 20 ms for top-100
  • Zero-downtime embedding updates

Pre-Computation

For known entities (existing users, items in catalogue), embeddings are pre-computed in batch and stored for fast lookup.

Batch Embedding Pipeline

# Batch embedding computation with Ray
import ray, torch, numpy as np

@ray.remote(num_gpus=1)
def compute_embeddings(entity_batch, model_path):
    model = torch.load(model_path)
    model.eval()
    embeddings = {}
    with torch.no_grad():
        for entity in entity_batch:
            emb = model.encode(entity.features)
            embeddings[entity.id] = emb.cpu().numpy()
    return embeddings

# Distribute across GPU workers
batches = partition(all_entities, batch_size=10000)
futures = [compute_embeddings.remote(b, MODEL_PATH) for b in batches]
results = ray.get(futures)

Incremental Updates

New entities or entities with changed features are re-embedded incrementally (hourly) rather than recomputing the entire catalogue.

Embedding versioning: When the embedding model changes, all embeddings must be recomputed. Store embeddings with a version tag and support dual-read during migration.

Caching & Serving

Embedding lookups are on the critical path of many ML systems. A tiered caching strategy minimises latency.

Storage Tiers

Hot Tier (In-Memory)

  • Redis / Memcached
  • Sub-millisecond lookup
  • Store top 10 M popular entities

Warm Tier (SSD)

  • RocksDB / DynamoDB
  • 1–5 ms lookup
  • Store all 1 B embeddings
# Tiered embedding lookup
def get_embedding(entity_id):
    # L1: In-memory cache
    emb = redis_cache.get(f"emb:{entity_id}")
    if emb:
        return deserialize(emb)

    # L2: SSD-backed store
    emb = rocksdb_store.get(entity_id)
    if emb:
        redis_cache.setex(f"emb:{entity_id}", 3600, serialize(emb))
        return emb

    # L3: Compute on-the-fly
    emb = model.encode(get_features(entity_id))
    store_embedding(entity_id, emb)
    return emb

Vector Search

Nearest-neighbour search over the embedding space enables similarity-based retrieval — "find items similar to X".

ANN Algorithms

HNSW

  • High recall (> 95 %)
  • Memory-intensive (graph structure)
  • Best for < 100 M vectors

IVF + PQ

  • Compressed vectors (8× smaller)
  • Scalable to billions
  • Slightly lower recall
# FAISS vector search with IVF+PQ
import faiss

def build_search_index(embeddings, dim=256, nlist=4096, m=32):
    quantizer = faiss.IndexFlatIP(dim)
    index = faiss.IndexIVFPQ(quantizer, dim, nlist, m, 8)
    index.train(embeddings)
    index.add(embeddings)
    index.nprobe = 64  # search 64 clusters
    return index

def search(index, query_embedding, k=100):
    distances, ids = index.search(query_embedding.reshape(1, -1), k)
    return list(zip(ids[0], distances[0]))
Index rebuild: When embeddings change (new model), the ANN index must be rebuilt. Use blue-green index deployment to avoid search downtime during rebuilds.

Scaling Strategies

Storing and searching billions of embeddings requires careful engineering.

Sharding

Partition the embedding space across multiple shards by entity ID hash. Each shard holds a subset of embeddings and its own ANN index. Search queries fan out to all shards and merge results.

Quantisation

Product Quantisation (PQ) compresses 256-dim float32 vectors from 1 KB to ~64 bytes — a 16× memory reduction. This makes billion-scale in-memory indexing feasible.

System Architecture

The architecture supports both pre-computed lookups (fast path) and on-the-fly embedding computation (slow path with GPU). The ANN index is rebuilt periodically and swapped atomically.
