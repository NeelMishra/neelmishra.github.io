Design: News Feed Ranking

MLOps Series ML System Design

Requirements & Scope

The news feed is the primary surface of social platforms. Ranking determines what billions of users see every time they open the app.

Functional

  • Rank posts from friends, pages, and groups
  • Mix organic, sponsored, and suggested content
  • Support multiple content types (text, image, video)
  • Respect user preferences & mute lists

Non-Functional

  • Latency < 200 ms p99
  • Serve 2 B+ daily active users
  • 99.99 % availability
  • Model refresh hourly

Candidate Selection

A user may have thousands of candidate posts from friends and followed pages. Candidate selection prunes this to ~500 using lightweight signals.

Source-Based Retrieval

Fan-out-on-read pulls recent posts from each connection. For high-follower accounts (celebrities), posts are pre-aggregated (fan-out-on-write) into a global pool. 

# Candidate generation: fan-out-on-read
def get_candidates(user_id, max_per_source=50):
    friends = get_friends(user_id)
    candidates = []
    for friend in friends:
        posts = get_recent_posts(friend, limit=max_per_source)
        candidates.extend(posts)

    # Add trending and suggested content
    candidates += get_trending(user_id, limit=100)
    candidates += get_suggested(user_id, limit=100)
    return candidates[:2000]  # cap for ranking
Fan-out trade-off: Fan-out-on-write is expensive for celebrity accounts (millions of followers). Hybrid approaches use write for regular users and read for high-follower accounts.

Scoring Model

The scoring model predicts multiple engagement signals and combines them into a single relevance score.

Multi-Task Learning

A shared-bottom architecture predicts P(like), P(comment), P(share), P(click), and P(hide) simultaneously. Each task head shares a common feature backbone. 

# Multi-task feed ranker
import torch, torch.nn as nn

class FeedRanker(nn.Module):
    def __init__(self, input_dim):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(input_dim, 256), nn.ReLU(),
            nn.Linear(256, 128), nn.ReLU()
        )
        self.head_like    = nn.Linear(128, 1)
        self.head_comment = nn.Linear(128, 1)
        self.head_share   = nn.Linear(128, 1)
        self.head_hide    = nn.Linear(128, 1)

    def forward(self, x):
        h = self.shared(x)
        return {
            "like":    torch.sigmoid(self.head_like(h)),
            "comment": torch.sigmoid(self.head_comment(h)),
            "share":   torch.sigmoid(self.head_share(h)),
            "hide":    torch.sigmoid(self.head_hide(h)),
        }

Value Model

Individual predictions are combined into a final value score with tunable weights: 

score = w_like * P(like) + w_comment * P(comment)
      + w_share * P(share) - w_hide * P(hide)

Diversity & Freshness

A pure relevance ranking produces repetitive feeds. Diversity and freshness constraints improve long-term retention.

Diversity Rules

No more than 2 consecutive posts from the same author. Ensure a mix of content types (text, photos, videos, links) in every viewport. Use MMR (Maximal Marginal Relevance) to penalise redundancy.

Freshness Decay

A time-decay function reduces scores of older posts: 

import math

def freshness_boost(post_age_hours, half_life=6.0):
    return math.exp(-0.693 * post_age_hours / half_life)
Filter bubbles: Over-optimising for engagement can create echo chambers. Inject "out-of-network" content periodically to broaden user exposure.

Engagement Optimisation

Long-term engagement metrics (daily active users, session length) matter more than short-term clicks. The value model weights are tuned through A/B experiments targeting long-term retention. 

# Long-term value estimation
def compute_ltv_score(predictions, weights):
    # Negative actions penalised heavily
    score = (weights["like"] * predictions["like"]
           + weights["comment"] * predictions["comment"]
           + weights["share"] * predictions["share"]
           - weights["hide"] * predictions["hide"]
           - weights["report"] * predictions["report"])
    return score
Integrity constraints: Downrank misinformation, clickbait, and borderline content even if they drive engagement. This protects the platform's long-term health.

System Architecture

The architecture is designed for horizontal scaling: each service (graph fetch, ranking, re-ranking) runs independently and can be scaled to handle peak traffic (Sunday evenings, breaking news events).
