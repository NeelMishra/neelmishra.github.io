Design: Anomaly Detection Platform

MLOps Series ML System Design

Requirements & Scope

An anomaly detection platform monitors metrics, logs, and events across an entire infrastructure, surfacing unusual patterns before they become outages.

Functional

  • Detect point, contextual, and collective anomalies
  • Support multiple data types (metrics, logs, traces)
  • Self-tuning thresholds per metric
  • Root-cause analysis suggestions

Non-Functional

  • Detection latency < 5 minutes
  • Monitor 1 M+ time series
  • False positive rate < 5 %
  • Horizontal scalability

Statistical Methods

Simple statistical methods are surprisingly effective as a baseline and scale to millions of time series.

Z-Score / Modified Z-Score

Flag points beyond ±3 standard deviations. Use the modified Z-score (based on MAD) for robustness to existing outliers. 

# Modified Z-score anomaly detection
import numpy as np

def modified_zscore(data, threshold=3.5):
    median = np.median(data)
    mad = np.median(np.abs(data - median))
    scores = 0.6745 * (data - median) / (mad + 1e-8)
    return np.abs(scores) > threshold

Exponential Smoothing

EWM (exponentially weighted moving average) adapts to level shifts and trends. An anomaly is detected when the residual exceeds a dynamic threshold.

Seasonality matters: A "spike" at 9 AM on Monday might be normal weekly traffic. Always decompose the time series into trend, seasonal, and residual components before flagging.

ML Approaches

When statistical methods fall short, ML models capture complex, multi-dimensional anomalies.

Isolation Forest

Randomly partitions the feature space; anomalies require fewer splits to isolate. Efficient for high-dimensional tabular data. 

# Isolation Forest for multi-dimensional anomalies
from sklearn.ensemble import IsolationForest

def train_detector(training_data, contamination=0.01):
    model = IsolationForest(
        n_estimators=200,
        contamination=contamination,
        random_state=42
    )
    model.fit(training_data)
    return model

def detect_anomalies(model, new_data):
    preds = model.predict(new_data)  # -1 = anomaly, 1 = normal
    scores = model.decision_function(new_data)
    return preds == -1, scores

Autoencoder-Based Detection

Train an autoencoder on normal data; anomalies produce high reconstruction error. Works well for complex, non-linear patterns.

Statistical

  • Fast, interpretable
  • Scales to millions of metrics
  • Limited to univariate patterns

ML-Based

  • Captures complex patterns
  • Multi-dimensional
  • Needs labelled data for tuning

Time-Series Anomalies

Time-series anomalies come in three flavours: point (single outlier), contextual (normal value at wrong time), and collective (sequence of values that are jointly abnormal).

Forecasting-Based Detection

Use a forecasting model (Prophet, DeepAR, N-BEATS) to predict expected values. Flag deviations beyond the prediction interval. 

# Prophet-based anomaly detection
from prophet import Prophet

def detect_ts_anomalies(df, sensitivity=0.99):
    m = Prophet(interval_width=sensitivity)
    m.fit(df)  # df has columns: ds, y
    forecast = m.predict(df)
    df["anomaly"] = (
        (df["y"] < forecast["yhat_lower"]) |
        (df["y"] > forecast["yhat_upper"])
    )
    return df
Alert fatigue: Too many false alerts desensitise operators. Use adaptive thresholds, alert deduplication, and severity-based routing.

Alerting & Triage

Raw anomaly detections must be transformed into actionable alerts with context, severity, and suggested root causes.

Alert Aggregation

Group related anomalies by service, time window, and correlation. A single root cause (e.g., database slowdown) may trigger hundreds of downstream anomalies — group them into one incident. 

# Alert aggregation logic
def aggregate_alerts(alerts, time_window_sec=300):
    groups = []
    for alert in sorted(alerts, key=lambda a: a.timestamp):

Each metric gets its own lightweight model instance in the model store. The detection engine evaluates all models in parallel, processing the entire metric space within the 5-minute SLA.
