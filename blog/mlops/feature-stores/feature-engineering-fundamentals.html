Why Feature Engineering Matters

Raw data is rarely suitable for machine learning models. A database might store a timestamp like 2024-03-15 14:32:07, but a model predicting food delivery times needs features like hour of day, is_weekend, or minutes since last order. The process of transforming raw data into informative signals is feature engineering, and it is often the single largest lever for improving model performance.

Consider a churn-prediction model. Feeding it raw columns—user ID, timestamp, event type—yields mediocre accuracy. But engineering features like days since last login, ratio of sessions this week vs. last month, and number of support tickets dramatically increases predictive power. Feature engineering is where domain knowledge meets data science.

Key Insight: In many Kaggle competitions, winning solutions attribute 70–80% of their edge to feature engineering rather than model architecture.

Features generally fall into four broad families: numerical, categorical, text, and temporal. Mastering the techniques for each type is the foundation of any ML pipeline.

Numerical Features

Numerical data—prices, counts, measurements—seems ready to use, but models benefit from proper scaling and transformation. A feature ranging from 0 to 1,000,000 can dominate gradient updates, while a feature from 0 to 1 is nearly ignored.

Standard Scaling (Z-Score)

Transforms each value to have zero mean and unit variance. Ideal when features follow a roughly Gaussian distribution. 

from sklearn.preprocessing import StandardScaler
import numpy as np

# Raw price data with wide range
prices = np.array([[10], [200], [3500], [45000], [780]])

scaler = StandardScaler()
scaled = scaler.fit_transform(prices)

print(scaled)
# Each value now centered around 0 with std ≈ 1

Min-Max Normalization

Squeezes values into a fixed [0, 1] range. Useful for neural networks with sigmoid or tanh activations, but sensitive to outliers. 

from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler(feature_range=(0, 1))
normalized = scaler.fit_transform(prices)

print(normalized)
# All values now between 0 and 1

Log Transforms

Heavily right-skewed distributions (e.g., income, page views) benefit from a log transform to reduce the influence of extreme values. 

import numpy as np

# Skewed page-view data
page_views = np.array([1, 5, 20, 150, 10000, 80000])

log_views = np.log1p(page_views)   # log(1 + x) handles zeros
print(log_views)
# [0.69, 1.79, 3.04, 5.02, 9.21, 11.29]

Binning (Discretization)

Converts a continuous feature into discrete buckets. Useful when the relationship between a feature and target is non-linear or step-wise (e.g., age groups for insurance pricing). 

import pandas as pd

ages = pd.Series([22, 35, 47, 58, 73, 19, 41])

bins = [0, 25, 40, 60, 100]
labels = ["young", "mid", "senior", "elder"]

age_groups = pd.cut(ages, bins=bins, labels=labels)
print(age_groups)
# [young, mid, senior, senior, elder, young, senior]
Watch out: Binning destroys information. Use it only when domain knowledge suggests that discrete groupings are more meaningful than raw values.

Categorical Features

Categorical features like country, browser type, or product category must be encoded as numbers before most models can use them. The choice of encoding has a significant impact on model quality and training speed.

One-Hot Encoding

Creates a binary column for each category. Safe and interpretable, but can explode the feature space when cardinality is high. 

import pandas as pd

df = pd.DataFrame({"color": ["red", "blue", "green", "red", "blue"]})

encoded = pd.get_dummies(df, columns=["color"], prefix="color")
print(encoded)
# color_blue  color_green  color_red
#     0           0            1
#     1           0            0
#     0           1            0

Label Encoding

Assigns an integer to each category. Compact, but introduces an artificial ordering that tree-based models handle well while linear models may misinterpret. 

from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
df["color_encoded"] = le.fit_transform(df["color"])
print(df)
# red → 2, blue → 0, green → 1

Target Encoding

Replaces each category with the mean of the target variable for that category. Powerful for high-cardinality features but prone to overfitting without regularization. 

import pandas as pd

df = pd.DataFrame({
    "city": ["NYC", "LA", "NYC", "LA", "Chicago", "Chicago"],
    "price": [350, 200, 400, 180, 150, 170]
})

# Mean price per city
city_means = df.groupby("city")["price"].mean()
df["city_target_enc"] = df["city"].map(city_means)
print(df)
# NYC → 375.0, LA → 190.0, Chicago → 160.0
Tip: Always compute target encoding on the training set only, then apply the mapping to validation and test sets. Computing it on the full dataset leaks information.

Encoding Method Comparison

One-Hot Encoding

  • Pros: No ordinal assumption; works with any model
  • Cons: High dimensionality for many categories
  • Best for: Low-cardinality features (<20 values)

Label Encoding

  • Pros: Compact; single column
  • Cons: Implies false ordering
  • Best for: Tree-based models (XGBoost, LightGBM)

Target Encoding

  • Pros: Captures target relationship; compact
  • Cons: Overfitting risk without regularization
  • Best for: High-cardinality features (zip codes, user IDs)

Frequency Encoding

  • Pros: Simple; captures popularity
  • Cons: Categories with same frequency collide
  • Best for: Quick baseline when cardinality is moderate

Text Features

Unstructured text—product reviews, support tickets, search queries—must be converted to numerical representations. Classic approaches include bag-of-words and TF-IDF; modern pipelines often add embeddings on top.

Bag of Words

Counts the occurrence of each word in a document. Simple but ignores word order and semantics. 

from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "the food was great",
    "great service and great food",
    "terrible food and bad service"
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

print(vectorizer.get_feature_names_out())
print(X.toarray())
# Matrix of word counts per document

TF-IDF

Weights words by how important they are to a document relative to the entire corpus. Common words like “the” get down-weighted while discriminative words get boosted. 

from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer(max_features=1000, stop_words="english")
X_tfidf = vectorizer.fit_transform(corpus)

print(X_tfidf.shape)
# (3, N) — each row is a TF-IDF vector

Tokenization and Preprocessing

Before vectorization, text usually needs cleaning: lowercasing, removing punctuation, stemming or lemmatization, and stop-word removal. 

import re

def clean_text(text):
    text = text.lower()
    text = re.sub(r"[^a-z0-9\s]", "", text)   # remove punctuation
    tokens = text.split()
    stop_words = {"the", "a", "an", "is", "was", "and", "or"}
    tokens = [t for t in tokens if t not in stop_words]
    return " ".join(tokens)

print(clean_text("The Food was GREAT!!!"))
# "food great"
Modern Alternative: For production systems, consider using pre-trained sentence embeddings (e.g., Sentence-BERT) instead of TF-IDF. They capture semantic meaning and generalize better to unseen vocabulary.

Temporal Features

Timestamps are one of the richest sources of features. A single datetime column can yield dozens of derived features: hour, day of week, month, quarter, is_holiday, time since an event, and cyclical encodings.

Basic Extractions

import pandas as pd

df = pd.DataFrame({
    "timestamp": pd.to_datetime([
        "2024-03-15 14:32:07",
        "2024-07-04 09:15:00",
        "2024-12-25 23:59:59"
    ])
})

df["hour"]       = df["timestamp"].dt.hour
df["day_of_week"] = df["timestamp"].dt.dayofweek   # 0=Mon, 6=Sun
df["month"]      = df["timestamp"].dt.month
df["is_weekend"] = df["day_of_week"].isin([5, 6]).astype(int)
print(df)

Cyclical Encoding

Hours 23 and 0 are close in time but far apart numerically. Cyclical encoding using sine and cosine preserves this circular relationship. 

import numpy as np

# Encode hour-of-day as cyclical features
df["hour_sin"] = np.sin(2 * np.pi * df["hour"] / 24)
df["hour_cos"] = np.cos(2 * np.pi * df["hour"] / 24)

# Encode day-of-week as cyclical features
df["dow_sin"] = np.sin(2 * np.pi * df["day_of_week"] / 7)
df["dow_cos"] = np.cos(2 * np.pi * df["day_of_week"] / 7)

print(df[["hour", "hour_sin", "hour_cos"]])

Lag Features

For time-series problems, the value at previous time steps is often the strongest predictor. Lag features shift values by n periods. 

import pandas as pd

df = pd.DataFrame({
    "date": pd.date_range("2024-01-01", periods=7, freq="D"),
    "sales": [100, 120, 90, 150, 130, 170, 160]
})

# Create lag features
df["sales_lag_1"] = df["sales"].shift(1)
df["sales_lag_7"] = df["sales"].shift(7)

# Rolling statistics
df["sales_rolling_3"] = df["sales"].rolling(window=3).mean()

print(df)
Data Leakage Alert: When creating lag features, ensure you only use past data relative to the prediction point. Including future values accidentally is one of the most common causes of data leakage in time-series models.

Best Practices

Feature engineering is as much about discipline as creativity. Follow these practices to build robust, maintainable feature pipelines.

Feature Selection

Not every feature helps. Too many features lead to overfitting, slower training, and harder debugging. Use techniques like mutual information, recursive feature elimination, or simply tracking feature importances from tree models. 

from sklearn.feature_selection import mutual_info_classif
import numpy as np

# Compute mutual information between each feature and target
mi_scores = mutual_info_classif(X_train, y_train)
feature_ranking = np.argsort(mi_scores)[::-1]

# Keep only top-k features
top_k = 20
selected = feature_ranking[:top_k]
X_selected = X_train[:, selected]

Avoiding Data Leakage

Data leakage happens when information from outside the training window bleeds into features. Common sources include:

  • Fitting scalers or encoders on the full dataset instead of only the training fold
  • Using future data in lag or rolling features
  • Target encoding computed on the same data used for training
Golden Rule: Every transformation must be fit on training data and applied (transform only) to validation and test data. This includes scalers, encoders, imputers, and any statistics-based feature.

Document Your Features

Maintain a feature catalog that records for each feature: its name, source table, transformation logic, owner, and creation date. This becomes critical when working in teams and is exactly the problem that feature stores solve at scale—which we cover in the next post.
