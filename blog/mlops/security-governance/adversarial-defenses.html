Adversarial Defenses: Robustness and Detection

MLOps Series Security & Governance

Introduction

The previous post catalogued the major adversarial attack families — evasion, poisoning, and backdoors. This post focuses on the other side of the equation: how to defend against them. We cover four pillars of adversarial defense: adversarial training, certified defenses, input preprocessing, and detection methods.

Adversarial defense is not a single technique but a spectrum of trade-offs. Stronger robustness typically costs clean accuracy, adds training time, or increases inference latency. The right defense strategy depends on your threat model, deployment context, and tolerance for performance degradation.

Defense philosophy: Think of adversarial defenses as layers of an onion — each layer increases the cost of attack. The goal is not perfect invulnerability (which is impossible) but making attacks expensive enough to deter real adversaries.
Beware of false security: Many proposed defenses have been broken by stronger adaptive attacks. Always evaluate your defenses against adaptive adversaries — not just the attacks they were designed for. The AutoAttack benchmark is a good starting point.

Adversarial Training

Adversarial training is the most established and empirically effective defense against evasion attacks. The core idea is simple: during training, generate adversarial examples on the fly and include them in the loss computation. This forces the model to learn a smoother, more robust decision boundary.

Standard Adversarial Training (Madry et al.)

The PGD-based adversarial training procedure solves a min-max optimisation problem: the inner loop maximises the loss by crafting adversarial examples, while the outer loop minimises the loss on those adversarial examples. 

# Complete adversarial training loop
import torch

def adversarial_train(model, train_loader, epochs=100, epsilon=8/255,
                       alpha=2/255, pgd_steps=10, lr=0.1):
    """Train model with PGD adversarial training."""
    optimiser = torch.optim.SGD(
        model.parameters(), lr=lr, momentum=0.9, weight_decay=5e-4
    )
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
        optimiser, T_max=epochs
    )

    for epoch in range(epochs):
        model.train()
        total_loss = 0

        for x, y in train_loader:
            # Inner maximisation: generate PGD adversarial examples
            x_adv = pgd_attack(model, x, y, epsilon, alpha, pgd_steps)

            # Outer minimisation: update model on adversarial data
            output = model(x_adv)
            loss = torch.nn.functional.cross_entropy(output, y)

            optimiser.zero_grad()
            loss.backward()
            optimiser.step()
            total_loss += loss.item()

        scheduler.step()
        avg_loss = total_loss / len(train_loader)
        print(f"Epoch {epoch+1}/{epochs} — Loss: {avg_loss:.4f}")

    return model

TRADES — Trading Off Robustness and Accuracy

TRADES (Zhang et al., 2019) decomposes the adversarial loss into a clean accuracy term and a robustness regulariser. By adjusting the balance parameter β, practitioners can explicitly control the accuracy-robustness trade-off. 

# TRADES loss function
def trades_loss(model, x, y, epsilon=8/255, beta=6.0, pgd_steps=10):
    """TRADES: decomposed clean + robustness loss."""
    # Clean loss term
    logits_clean = model(x)
    loss_clean = torch.nn.functional.cross_entropy(logits_clean, y)

    # Generate adversarial examples maximising KL divergence
    x_adv = x.clone().detach() + 0.001 * torch.randn_like(x)
    for _ in range(pgd_steps):
        x_adv.requires_grad_(True)
        with torch.enable_grad():
            loss_kl = torch.nn.functional.kl_div(
                torch.nn.functional.log_softmax(model(x_adv), dim=1),
                torch.nn.functional.softmax(logits_clean.detach(), dim=1),
                reduction="batchmean"
            )
        grad = torch.autograd.grad(loss_kl, x_adv)[0]
        x_adv = x_adv.detach() + (2/255) * grad.sign()
        delta = torch.clamp(x_adv - x, -epsilon, epsilon)
        x_adv = torch.clamp(x + delta, 0.0, 1.0).detach()

    # Robustness loss: KL divergence between clean and adv predictions
    logits_adv = model(x_adv)
    loss_robust = torch.nn.functional.kl_div(
        torch.nn.functional.log_softmax(logits_adv, dim=1),
        torch.nn.functional.softmax(logits_clean.detach(), dim=1),
        reduction="batchmean"
    )

    return loss_clean + beta * loss_robust
Practical note: Adversarial training with PGD is 3–10× slower than standard training (due to the inner PGD loop). Techniques like Free Adversarial Training (Shafahi et al., 2019) and FGSM-RS reduce this cost by reusing gradient computations.

Standard AT (Madry)

  • Strongest empirical robustness
  • Significant clean accuracy drop (3–8 %)
  • 3–10× training cost increase
  • No explicit accuracy-robustness control

TRADES

  • Comparable robustness to Madry AT
  • Smaller clean accuracy drop (1–4 %)
  • Similar training cost to Madry AT
  • Tunable β controls the trade-off explicitly

Certified Defenses

While adversarial training provides empirical robustness (tested against known attacks), certified defenses provide mathematical guarantees: within a specified perturbation radius, the model's prediction is provably unchanged. This is crucial for safety-critical applications where "probably robust" is not enough.

Randomised Smoothing

Randomised smoothing (Cohen et al., 2019) converts any base classifier into a certifiably robust one. The idea is to classify an input by taking a majority vote over predictions on Gaussian-noised copies of the input. The radius of certified robustness depends on the noise level σ and the gap between the top-two predicted classes. 

# Randomised smoothing: certified prediction
import torch
from scipy.stats import norm

class SmoothedClassifier:
    def __init__(self, base_model, sigma=0.25, n_samples=1000, alpha=0.001):
        self.model    = base_model
        self.sigma    = sigma
        self.n        = n_samples
        self.alpha    = alpha

    def certify(self, x):
        """Return predicted class and certified L2 radius."""
        counts = torch.zeros(self.model.num_classes)
        for _ in range(self.n):
            noisy_x = x + self.sigma * torch.randn_like(x)
            pred = self.model(noisy_x).argmax(dim=1)
            counts[pred] += 1

        top_class = counts.argmax().item()
        p_lower = proportion_confint(counts[top_class], self.n, self.alpha)

        if p_lower > 0.5:
            radius = self.sigma * norm.ppf(p_lower)
            return top_class, radius
        return -1, 0.0  # abstain — cannot certify

Interval Bound Propagation (IBP)

IBP propagates interval bounds through each layer of the network, computing the worst-case output range for any input within the perturbation set. If the worst-case output for the correct class remains above all other classes, the prediction is certified. IBP is exact for linear layers and provides sound over-approximations for ReLU activations.

Scalability challenge: Certified defenses currently offer weaker robustness than empirical methods (adversarial training) on large-scale benchmarks (ImageNet). They excel in domains where provable guarantees matter more than absolute accuracy — such as medical imaging or autonomous driving.

Input Preprocessing Defenses

Input preprocessing defenses apply transformations to the input before feeding it to the model, aiming to destroy adversarial perturbations while preserving the semantic content. They are attractive because they require no model retraining and can be deployed as a drop-in module in the serving pipeline.

Common Preprocessing Techniques

  • JPEG compression — Compressing and decompressing the input removes high-frequency perturbations that many adversarial attacks exploit.
  • Spatial smoothing — Gaussian or median blurring reduces pixel-level noise at the cost of some detail.
  • Bit-depth reduction — Quantising pixel values (e.g., from 256 to 16 levels) eliminates small perturbations.
  • Feature squeezing — Combines multiple preprocessing methods and compares predictions; divergence indicates an adversarial input.
# Input preprocessing defense pipeline
import torch
from torchvision import transforms

class PreprocessingDefense:
    def __init__(self, jpeg_quality=75, blur_kernel=3, bit_depth=4):
        self.jpeg_quality = jpeg_quality
        self.blur = transforms.GaussianBlur(blur_kernel, sigma=1.0)
        self.bit_depth = bit_depth

    def jpeg_compress(self, x):
        """Simulate JPEG compression/decompression."""
        # Encode to JPEG bytes and decode back
        from io import BytesIO
        from PIL import Image
        img = transforms.functional.to_pil_image(x)
        buffer = BytesIO()
        img.save(buffer, format="JPEG", quality=self.jpeg_quality)
        buffer.seek(0)
        img_decoded = Image.open(buffer)
        return transforms.functional.to_tensor(img_decoded)

    def reduce_bits(self, x):
        """Reduce colour bit depth."""
        levels = 2 ** self.bit_depth
        return torch.round(x * levels) / levels

    def defend(self, x):
        """Apply all preprocessing steps."""
        x = self.jpeg_compress(x)
        x = self.blur(x)
        x = self.reduce_bits(x)
        return x
Limitation: Preprocessing defenses are effective against non-adaptive attackers. An adaptive attacker can incorporate the preprocessing into their attack loop (differentiable approximation) and craft perturbations that survive it. Always combine preprocessing with other defense layers.

Feature Squeezing Detection

Feature squeezing runs the input through multiple squeezers (JPEG, blur, bit-reduction) and compares predictions across the original and squeezed versions. If predictions diverge significantly, the input is flagged as potentially adversarial. This turns preprocessing into a detection mechanism rather than a pure defense.

Detection Methods

Rather than trying to correctly classify adversarial inputs, detection methods aim to identify and reject them. This is often more practical for production systems, where rejecting a suspicious input (and falling back to a human reviewer) is acceptable.

Statistical Detection

Adversarial inputs often exhibit statistical properties that differ from natural data — unusual activation patterns in intermediate layers, atypical confidence distributions, or anomalous feature statistics. Detectors trained on these signals can flag adversarial inputs with high accuracy. 

# Activation-based adversarial detector
class ActivationDetector:
    def __init__(self, model, layer_name, threshold=2.5):
        self.model      = model
        self.layer_name = layer_name
        self.threshold  = threshold
        self.stats      = None   # fitted from clean data

    def fit(self, clean_loader):
        """Compute activation statistics from clean data."""
        activations = []
        for x, _ in clean_loader:
            act = self.model.get_activations(x, self.layer_name)
            activations.append(act.flatten(1))
        all_acts = torch.cat(activations)
        self.stats = {
            "mean": all_acts.mean(dim=0),
            "std":  all_acts.std(dim=0) + 1e-6
        }

    def detect(self, x):
        """Return True if input appears adversarial."""
        act = self.model.get_activations(x, self.layer_name).flatten(1)
        z_scores = (act - self.stats["mean"]) / self.stats["std"]
        max_z = z_scores.abs().max(dim=1).values
        return max_z > self.threshold

Confidence Calibration

Well-calibrated models assign lower confidence to adversarial inputs that are near the decision boundary. Temperature scaling and Bayesian approaches (MC-Dropout) produce better-calibrated uncertainties that serve as natural adversarial detectors.

Backdoor Detection

Detecting backdoors in trained models requires specialised techniques:

  • Neural Cleanse — Reverse-engineers the smallest trigger pattern that causes misclassification to a given class. A suspiciously small trigger indicates a backdoor.
  • Activation Clustering — Clusters intermediate-layer activations and identifies outlier clusters that correspond to backdoor-triggered inputs.
  • Spectral Signatures — Analyses the singular values of activation matrices; backdoor samples create detectable spectral anomalies.
  • STRIP — Blends test inputs with clean images; backdoor inputs produce abnormally consistent predictions regardless of the blend.
Detection is not prevention. Detection methods reduce the attack success rate but cannot eliminate risk entirely. Combine detection with adversarial training and input preprocessing for a robust defense-in-depth strategy.

Practical Deployment Guide

Choosing the right defense stack depends on your threat model, performance budget, and deployment constraints. Below is a decision framework for production ML systems.

High-Security (Safety-Critical)

  • Adversarial training (PGD or TRADES)
  • Certified defenses (randomised smoothing)
  • Multi-layer detection pipeline
  • Model signing and provenance tracking
  • Accept 3–5 % clean accuracy cost

Standard (Business Applications)

  • Input preprocessing (JPEG + spatial smoothing)
  • Feature squeezing for detection
  • Confidence thresholding for rejection
  • Rate limiting on prediction APIs
  • Minimal clean accuracy impact

Defense Integration Architecture

# Production defense pipeline
class RobustInferencePipeline:
    def __init__(self, model, config):
        self.model        = model    # adversarially-trained model
        self.preprocessor = PreprocessingDefense(config)
        self.detector     = ActivationDetector(model, config.detect_layer)
        self.rate_limiter = RateLimiter(config.max_qps)
        self.monitor      = PredictionMonitor(config.alert_rules)

    def predict(self, request):
        # Layer 1: Rate limiting
        if not self.rate_limiter.allow(request.client_id):
            raise RateLimitError()

        # Layer 2: Input preprocessing
        x = self.preprocessor.defend(request.features)

        # Layer 3: Adversarial detection
        if self.detector.detect(x):
            self.monitor.flag(request, "adversarial_suspected")
            return FallbackResponse(reason="input_rejected")

        # Layer 4: Robust model prediction
        prediction = self.model.predict(x)

        # Layer 5: Confidence check
        if prediction.confidence < config.min_confidence:
            self.monitor.flag(request, "low_confidence")
            return FallbackResponse(reason="uncertain")

        # Layer 6: Monitoring
        self.monitor.log(request, prediction)
        return prediction

Evaluation Checklist

Before deploying any defense, validate it against this checklist:

  • Clean accuracy — Measure the accuracy drop on standard test data. Is it within your tolerance?
  • Robust accuracy — Test against PGD-20, PGD-100, and AutoAttack. Are the numbers consistent?
  • Adaptive attack — Can an attacker who knows your defense bypass it? Run BPDA (Backward Pass Differentiable Approximation) for non-differentiable defenses.
  • Latency impact — Measure inference latency with defenses enabled. Does it meet your SLA?
  • False positive rate — For detection methods, what fraction of clean inputs are incorrectly rejected?
Start pragmatically: For most production systems, begin with input preprocessing + confidence thresholding (zero retraining cost), then add adversarial training if your threat model warrants it. Only invest in certified defenses if you need provable guarantees for compliance or safety reasons.
