Fairness in ML: Metrics, Bias Detection

MLOps Series Security & Governance

Introduction

Machine learning models are increasingly used for high-stakes decisions — hiring, lending, criminal justice, healthcare. Yet these models can inherit and amplify biases present in historical training data, leading to systematically unfair outcomes for protected demographic groups.

Fairness in ML is not merely a philosophical concern; it is a legal, ethical, and business imperative. Regulations like the EU AI Act, US Equal Credit Opportunity Act, and Title VII of the Civil Rights Act impose concrete requirements on automated decision-making systems. Understanding how to measure, detect, and mitigate bias is an essential skill for every ML practitioner.

Key insight: There is no single definition of "fairness." Different metrics encode different notions of equity, and it has been mathematically proven that several fairness criteria cannot be simultaneously satisfied (the impossibility theorem of Chouldechova, 2017). Choosing the right metric requires understanding the social context of your application.

Fairness Metrics

Fairness metrics quantify whether a model treats different demographic groups equitably. Each metric captures a different aspect of fairness, and the choice depends heavily on the application domain.

Demographic Parity

Definition: P(Ŷ=1 | A=0) = P(Ŷ=1 | A=1)

The positive prediction rate should be equal across groups. Also known as statistical parity. Does not account for differences in base rates.

Use when: The goal is equal representation in outcomes regardless of qualification differences.

Equalized Odds

Definition: P(Ŷ=1 | Y=y, A=0) = P(Ŷ=1 | Y=y, A=1) for y ∈ {0,1}

True positive rates and false positive rates should be equal across groups. Conditions on the true label.

Use when: Both false positives and false negatives have significant costs.

Disparate Impact

Definition: P(Ŷ=1 | A=0) / P(Ŷ=1 | A=1) ≥ 0.8

The 80% rule from US employment law. If the selection rate for a protected group is less than 80% of the rate for the majority group, there is evidence of disparate impact.

Use when: Regulatory compliance (EEOC guidelines).

Calibration

Definition: P(Y=1 | Ŷ=p, A=a) = p for all groups a

A predicted probability of p should mean the true probability is p, regardless of group membership. Important for risk scoring systems.

Use when: Predicted probabilities are used for decision-making (e.g., credit risk scores).

Computing Fairness Metrics in Python

# Computing core fairness metrics from predictions import numpy as np def demographic_parity(y_pred, sensitive_attr): """Difference in positive prediction rates between groups.""" groups = np.unique(sensitive_attr) rates = {} for g in groups: mask = sensitive_attr == g rates[g] = np.mean(y_pred[mask]) return max(rates.values()) - min(rates.values()) def equalized_odds(y_true, y_pred, sensitive_attr): """Difference in TPR and FPR across groups.""" groups = np.unique(sensitive_attr) tpr, fpr = {}, {} for g in groups: mask = sensitive_attr == g pos = y_true[mask] == 1 neg = y_true[mask] == 0 tpr[g] = np.mean(y_pred[mask][pos]) if pos.sum() > 0 else 0 fpr[g] = np.mean(y_pred[mask][neg]) if neg.sum() > 0 else 0 tpr_diff = max(tpr.values()) - min(tpr.values()) fpr_diff = max(fpr.values()) - min(fpr.values()) return {"tpr_diff": tpr_diff, "fpr_diff": fpr_diff} def disparate_impact_ratio(y_pred, sensitive_attr, privileged=1): """80% rule: ratio of positive rates (unprivileged / privileged).""" priv_mask = sensitive_attr == privileged unpriv_mask = ~priv_mask rate_priv = np.mean(y_pred[priv_mask]) rate_unpriv = np.mean(y_pred[unpriv_mask]) return rate_unpriv / rate_priv if rate_priv > 0 else 0
Impossibility theorem: It is mathematically impossible to simultaneously satisfy demographic parity, equalized odds, and calibration when base rates differ across groups (Chouldechova, 2017; Kleinberg et al., 2016). You must choose which fairness criterion matters most for your specific use case.

Bias Detection

Bias can enter the ML pipeline at multiple stages. A systematic approach to detection examines each stage independently.

Sources of Bias

Slice-Based Analysis

Aggregate model metrics hide disparities. Always evaluate performance per demographic slice in addition to overall metrics. A model with 95% accuracy overall might have 70% accuracy for an underrepresented group.

# Slice-based fairness audit def fairness_audit(y_true, y_pred, y_prob, sensitive_attrs): """Run a comprehensive fairness audit across slices.""" report = {} for attr_name, attr_values in sensitive_attrs.items(): report[attr_name] = {} for group in np.unique(attr_values): mask = attr_values == group report[attr_name][group] = { "count": mask.sum(), "positive_rate": np.mean(y_pred[mask]), "accuracy": np.mean(y_pred[mask] == y_true[mask]), "tpr": recall(y_true[mask], y_pred[mask]), "fpr": false_positive_rate(y_true[mask], y_pred[mask]), "avg_score": np.mean(y_prob[mask]), } # Compute pairwise disparities groups = list(report[attr_name].keys()) report[attr_name]["disparate_impact"] = ( report[attr_name][groups[0]]["positive_rate"] / max(report[attr_name][groups[1]]["positive_rate"], 1e-10) ) return report
Proxy variables: Even when sensitive attributes (race, gender) are excluded from features, models can learn proxies — ZIP code correlates with race, name correlates with ethnicity. Detecting proxy bias requires correlation analysis between features and protected attributes.

Mitigation Strategies

Bias mitigation techniques fall into three categories based on when they are applied in the ML lifecycle.

Pre-processing Techniques

Modify the training data before model training to reduce bias at its source.

  • Reweighting: Assign higher sample weights to underrepresented group–label combinations to balance the effective training distribution.
  • Resampling: Over-sample minority groups or under-sample majority groups to achieve demographic balance.
  • Disparate Impact Remover: Transform features to remove correlation with the sensitive attribute while preserving rank ordering within groups.

In-processing Techniques

Modify the learning algorithm itself to enforce fairness constraints during training.

  • Adversarial Debiasing: Train a predictor and an adversary simultaneously. The adversary tries to predict the sensitive attribute from the predictor's output; the predictor is penalised when the adversary succeeds.
  • Constrained Optimization: Add fairness constraints (e.g., equalized odds) directly to the optimisation objective using Lagrange multipliers.
  • Fair Regularisation: Add a penalty term to the loss function that penalises fairness metric violations.

Post-processing Techniques

Adjust model outputs after training to satisfy fairness criteria.

  • Threshold Adjustment: Use different classification thresholds for different groups to equalise error rates. The simplest and most common approach.
  • Calibrated Equalized Odds: Optimise group-specific thresholds to jointly minimise a loss function subject to equalized odds constraints.
  • Reject Option Classification: Give favourable outcomes to disadvantaged group members and unfavourable outcomes to advantaged group members near the decision boundary.

Pre-processing

Pros: Model-agnostic, simple to implement, preserves model choice flexibility.

Cons: May lose information, doesn't guarantee model-level fairness, can reduce overall accuracy.

In-processing

Pros: Directly optimises for fairness during training, can achieve better accuracy-fairness tradeoffs.

Cons: Model-specific, more complex to implement, may require custom training loops.

Fairness Tools & Libraries

Several mature open-source libraries provide production-ready fairness tooling.

AI Fairness 360 (AIF360)

IBM's comprehensive fairness toolkit with 70+ fairness metrics, 10+ bias mitigation algorithms, and support for pre-/in-/post-processing interventions.

Practical Fairness Workflow

Integrating fairness into your MLOps pipeline requires a structured, repeatable process that runs alongside standard model development.

Step-by-Step Process

  1. Define fairness requirements — work with stakeholders to select appropriate fairness metrics, identify protected groups, and set acceptable disparity thresholds.
  2. Measure baseline bias — before any mitigation, compute fairness metrics on the current model to quantify existing disparities.
  3. Apply mitigation — select and implement pre-/in-/post-processing techniques based on your constraints (model type, latency, interpretability needs).
  4. Validate and iterate — re-compute fairness metrics after mitigation. Check that overall performance has not degraded unacceptably. Iterate if needed.
  5. Monitor in production — continuously track fairness metrics on live data. Distribution shifts can introduce new biases over time.

CI/CD Integration

Add fairness gates to your model CI/CD pipeline. A model should not be promoted to production if it fails fairness thresholds — just as it would not be promoted if it failed accuracy thresholds.

# Fairness gate in CI/CD pipeline def fairness_gate(model, X_test, y_test, sensitive_features, thresholds): """Block deployment if fairness thresholds are violated.""" dp_diff = demographic_parity(model.predict(X_test), sensitive_features) eo = equalized_odds(y_test, model.predict(X_test), sensitive_features) di = disparate_impact_ratio(model.predict(X_test), sensitive_features) violations = [] if dp_diff > thresholds["max_dp_diff"]: violations.append(f"Demographic parity gap: {dp_diff:.3f}") if di < thresholds["min_disparate_impact"]: violations.append(f"Disparate impact: {di:.3f}") if eo["tpr_diff"] > thresholds["max_tpr_diff"]: violations.append(f"TPR gap: {eo['tpr_diff']:.3f}") if violations: raise FairnessViolation("Deployment blocked:\n" + "\n".join(violations)) return True
Ongoing commitment: Fairness is not a one-time check. Models drift, populations shift, and societal norms evolve. Embed fairness monitoring into your production observability stack alongside latency, throughput, and accuracy metrics.
