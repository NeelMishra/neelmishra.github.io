Google Vertex AI: Managed ML on GCP

MLOps Series Platform Engineering

Vertex AI Platform Overview

Vertex AI is Google Cloud's unified ML platform, consolidating previously separate services (AI Platform Training, AI Platform Prediction, AutoML) into a single API surface. It provides custom training, managed prediction, pipelines, feature store, model monitoring, and experiment tracking under one roof.

Unified SDK: The google-cloud-aiplatform Python SDK provides a consistent interface to all Vertex AI services. Initialize it once with your project and location, then access training, prediction, pipelines, and more.
# Initialize the Vertex AI SDK from google.cloud import aiplatform aiplatform.init( project="my-ml-project", location="us-central1", staging_bucket="gs://my-ml-staging", experiment="image-classification-v2", ) print(f"Project: {aiplatform.initializer.global_config.project}") print(f"Location: {aiplatform.initializer.global_config.location}")

Key advantages of Vertex AI include native BigQuery integration for feature engineering, tight coupling with Google Cloud Storage for large datasets, and access to TPU v4/v5 hardware for training large models.

Custom Training Jobs

Vertex AI custom training supports three modes: pre-built containers with your training script, custom Docker containers, and AutoML. For most teams, pre-built containers are the fastest path—you write a standard Python training script and Vertex handles the environment.

# Custom training with a pre-built PyTorch container from google.cloud import aiplatform job = aiplatform.CustomTrainingJob( display_name="pytorch-image-classifier", script_path="trainer/task.py", container_uri="us-docker.pkg.dev/vertex-ai/training/pytorch-gpu.2-1:latest", requirements=["torchvision==0.16.0", "pillow>=10.0"], model_serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/pytorch-gpu.2-1:latest", ) model = job.run( replica_count=1, machine_type="n1-standard-8", accelerator_type="NVIDIA_TESLA_V100", accelerator_count=1, args=[ "--epochs=50", "--batch-size=64", "--learning-rate=0.001", "--data-dir=gs://my-ml-data/train/", ], model_display_name="image-classifier-v2", )
GPU quotas: GCP enforces GPU quotas per region. Before launching training, check your available quota with gcloud compute regions describe us-central1. Request quota increases well in advance of production training runs.

For distributed training, Vertex AI supports multi-worker configurations with automatic cluster setup:

# Distributed custom training with multiple workers from google.cloud import aiplatform worker_pool_specs = [ { # Chief worker "machine_spec": { "machine_type": "n1-standard-16", "accelerator_type": "NVIDIA_TESLA_V100", "accelerator_count": 2, }, "replica_count": 1, "container_spec": { "image_uri": "us-docker.pkg.dev/vertex-ai/training/pytorch-gpu.2-1:latest", "args": ["--distributed", "--epochs=100"], }, }, { # Worker pool "machine_spec": { "machine_type": "n1-standard-16", "accelerator_type": "NVIDIA_TESLA_V100", "accelerator_count": 2, }, "replica_count": 3, # 3 additional workers "container_spec": { "image_uri": "us-docker.pkg.dev/vertex-ai/training/pytorch-gpu.2-1:latest", "args": ["--distributed", "--epochs=100"], }, }, ] custom_job = aiplatform.CustomJob( display_name="distributed-training", worker_pool_specs=worker_pool_specs, ) custom_job.run()

Prediction & Serving

Vertex AI Prediction deploys models to managed endpoints with autoscaling, traffic splitting for A/B tests, and built-in monitoring. Models are deployed to endpoints, and each endpoint can host multiple model versions with configurable traffic allocation.

# Deploy a model to an endpoint from google.cloud import aiplatform # Upload model to Vertex AI Model Registry model = aiplatform.Model.upload( display_name="image-classifier-v2", artifact_uri="gs://my-ml-models/classifier/v2/", serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/pytorch-gpu.2-1:latest", ) # Create endpoint and deploy endpoint = aiplatform.Endpoint.create( display_name="classifier-endpoint", ) endpoint.deploy( model=model, deployed_model_display_name="v2-production", machine_type="n1-standard-4", accelerator_type="NVIDIA_TESLA_T4", accelerator_count=1, min_replica_count=1, max_replica_count=5, traffic_percentage=100, ) # Make a prediction prediction = endpoint.predict( instances=[{"image_bytes": {"b64": encoded_image}}] ) print(f"Prediction: {prediction.predictions}")

Online Prediction

Latency: Low (dedicated instances)

Scaling: Autoscaling based on traffic

Best for real-time applications with consistent traffic patterns.

Batch Prediction

Latency: Minutes to hours

Scaling: Automatic job-level scaling

Best for processing large datasets offline (daily scoring, bulk inference).

For batch prediction on large datasets:

# Batch prediction job batch_job = model.batch_predict( job_display_name="daily-scoring-job", gcs_source="gs://my-ml-data/batch-input/input.jsonl", gcs_destination_prefix="gs://my-ml-data/batch-output/", machine_type="n1-standard-4", accelerator_type="NVIDIA_TESLA_T4", accelerator_count=1, starting_replica_count=2, max_replica_count=10, ) batch_job.wait()

Vertex Pipelines

Vertex Pipelines is built on Kubeflow Pipelines v2 and provides serverless orchestration of ML workflows. Pipelines are defined as Python functions decorated with KFP components, compiled to YAML, and executed on managed infrastructure.

# Define pipeline components with KFP v2 from kfp import dsl from kfp.dsl import Input, Output, Dataset, Model, Metrics @dsl.component( base_image="python:3.10", packages_to_install=["pandas", "scikit-learn"], ) def preprocess_data( raw_data_uri: str, train_data: Output[Dataset], test_data: Output[Dataset], split_ratio: float = 0.8, ): import pandas as pd from sklearn.model_selection import train_test_split df = pd.read_csv(raw_data_uri) train_df, test_df = train_test_split(df, train_size=split_ratio) train_df.to_csv(train_data.path, index=False) test_df.to_csv(test_data.path, index=False) @dsl.component( base_image="us-docker.pkg.dev/vertex-ai/training/pytorch-gpu.2-1:latest", ) def train_model( train_data: Input[Dataset], model_artifact: Output[Model], metrics: Output[Metrics], epochs: int = 50, ): # Training logic here metrics.log_metric("accuracy", 0.94) metrics.log_metric("loss", 0.12)

Compose components into a pipeline and submit it:

# Compose and run the pipeline @dsl.pipeline(name="training-pipeline") def ml_pipeline(raw_data_uri: str, epochs: int = 50): preprocess_task = preprocess_data(raw_data_uri=raw_data_uri) train_task = train_model( train_data=preprocess_task.outputs["train_data"], epochs=epochs, ) # Compile and submit from kfp import compiler compiler.Compiler().compile(ml_pipeline, "pipeline.yaml") job = aiplatform.PipelineJob( display_name="training-run-001", template_path="pipeline.yaml", parameter_values={ "raw_data_uri": "gs://my-data/raw/data.csv", "epochs": 100, }, ) job.run(service_account="ml-pipeline@my-project.iam.gserviceaccount.com")
Pipeline caching: Vertex Pipelines caches component outputs by default. If a component's inputs haven't changed, it reuses the previous output. This dramatically speeds up iterative development—change your training code without re-running preprocessing.

Vertex Feature Store

Vertex Feature Store provides a centralized, managed repository for ML features. It integrates tightly with BigQuery, allowing you to define feature transformations in SQL and serve them with low latency for online predictions.

# Create a Feature Store and Entity Type from google.cloud import aiplatform feature_store = aiplatform.Featurestore.create( featurestore_id="ml_features", online_store_fixed_node_count=1, ) # Create an entity type customer_entity = feature_store.create_entity_type( entity_type_id="customer", description="Customer features for churn prediction", ) # Add features customer_entity.batch_create_features( feature_configs={ "avg_purchase_value": {"value_type": "DOUBLE"}, "total_orders": {"value_type": "INT64"}, "days_since_last_order": {"value_type": "INT64"}, "is_premium": {"value_type": "BOOL"}, }, ) # Ingest features from BigQuery customer_entity.ingest_from_bq( feature_ids=["avg_purchase_value", "total_orders", "days_since_last_order", "is_premium"], feature_time="update_time", bq_source_uri="bq://my-project.features.customer_features", entity_id_field="customer_id", )
Online serving nodes: Each online serving node adds cost even when idle. Start with online_store_fixed_node_count=1 and scale up based on actual read QPS measured in production.

Reading features for online prediction:

# Read features for online serving entity_views = feature_store.read( entity_type_id="customer", entity_ids=["c001", "c002"], feature_ids=["avg_purchase_value", "total_orders"], ) print(entity_views.to_dict())

Model Registry & Versioning

Vertex AI Model Registry provides a central catalog for all trained models. Each model can have multiple versions, and the registry tracks lineage—linking each version to its training job, dataset, and pipeline run.

# Upload a new model version from google.cloud import aiplatform model = aiplatform.Model.upload( display_name="churn-predictor", artifact_uri="gs://my-ml-models/churn/v3/", serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/sklearn-cpu.1-2:latest", description="v3 - added feature interactions", labels={ "team": "ml-platform", "dataset_version": "2024-01", "framework": "sklearn", }, is_default_version=False, # promote explicitly ) # List all versions of a model models = aiplatform.Model.list( filter='display_name="churn-predictor"', order_by="create_time desc", ) for m in models: print(f"{m.display_name} | {m.version_id} | {m.create_time}")

Vertex AI Model Registry

Scope: GCP-native model catalog

Lineage: Automatic with Vertex Pipelines

Best for: Teams fully on GCP

MLflow Model Registry

Scope: Cloud-agnostic

Lineage: Manual tracking required

Best for: Multi-cloud or hybrid setups

Promotion workflow: Use labels and aliases to manage model lifecycle stages. Train with is_default_version=False, validate in staging, then promote to default only after passing evaluation gates in your pipeline.
