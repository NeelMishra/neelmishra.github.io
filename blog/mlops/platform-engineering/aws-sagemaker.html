MLOps Series Platform Engineering

SageMaker Platform Overview

AWS SageMaker is a fully managed service that covers the entire ML lifecycle: data labeling, feature engineering, model training, hyperparameter tuning, deployment, and monitoring. It abstracts away infrastructure management so teams can focus on model quality.

Why SageMaker? It provides managed infrastructure for every stage of ML. You define what to run (training script, container image, instance type) and SageMaker handles provisioning, scaling, and teardown automatically.

The SageMaker Python SDK provides a high-level interface that wraps boto3 calls, making it the preferred way to interact with the service programmatically. Every SageMaker resource is tied to an execution role that controls IAM permissions.

# Initialize SageMaker session import sagemaker from sagemaker import Session, get_execution_role session = sagemaker.Session() role = get_execution_role() bucket = session.default_bucket() print(f"Role: {role}") print(f"Bucket: {bucket}") print(f"Region: {session.boto_region_name}")

Training Jobs

SageMaker training jobs are the core compute primitive. You provide a training script, specify the instance type and count, point to your data in S3, and SageMaker handles the rest—spinning up instances, pulling your container, running the training, and uploading the model artifact.

# PyTorch Estimator for custom training from sagemaker.pytorch import PyTorch estimator = PyTorch( entry_point="train.py", source_dir="src/", role=role, instance_count=1, instance_type="ml.p3.2xlarge", framework_version="2.1", py_version="py310", hyperparameters={ "epochs": 50, "learning-rate": 0.001, "batch-size": 64, "model-type": "resnet50", }, metric_definitions=[ {"Name": "train:loss", "Regex": "train_loss: ([0-9\\.]+)"}, {"Name": "val:accuracy", "Regex": "val_acc: ([0-9\\.]+)"}, ], max_run=3600 * 4, # 4 hour timeout checkpoint_s3_uri=f"s3://{bucket}/checkpoints/", ) # Launch training with S3 data channels estimator.fit({ "train": f"s3://{bucket}/data/train/", "validation": f"s3://{bucket}/data/val/", })
Metric definitions matter: SageMaker scrapes CloudWatch metrics from stdout using regex patterns. If your metric_definitions regex doesn't match your training script's print format, you'll get no metrics in the console.

For distributed training across multiple GPU instances, SageMaker supports data parallelism and model parallelism out of the box:

# Distributed training with SageMaker Data Parallelism from sagemaker.pytorch import PyTorch distributed_estimator = PyTorch( entry_point="train_distributed.py", source_dir="src/", role=role, instance_count=4, # 4 nodes instance_type="ml.p4d.24xlarge", # 8 GPUs each framework_version="2.1", py_version="py310", distribution={ "smdistributed": { "dataparallel": {"enabled": True} } }, ) distributed_estimator.fit({"train": train_s3_uri})

Real-Time Endpoints

Once a model is trained, SageMaker can deploy it as a real-time inference endpoint. The service provisions compute, loads the model, and exposes an HTTPS endpoint with built-in autoscaling, A/B testing, and monitoring.

# Deploy trained model to a real-time endpoint predictor = estimator.deploy( initial_instance_count=2, instance_type="ml.g4dn.xlarge", endpoint_name="image-classifier-prod", ) # Invoke the endpoint import json response = predictor.predict( json.dumps({"image_url": "s3://bucket/test.jpg"}) ) print(f"Prediction: {response}")

For production deployments, configure autoscaling based on invocation metrics:

# Auto-scaling configuration via boto3 import boto3 client = boto3.client("application-autoscaling") # Register scalable target client.register_scalable_target( ServiceNamespace="sagemaker", ResourceId="endpoint/image-classifier-prod/variant/AllTraffic", ScalableDimension="sagemaker:variant:DesiredInstanceCount", MinCapacity=2, MaxCapacity=10, ) # Target-tracking scaling policy client.put_scaling_policy( PolicyName="invocations-target-tracking", ServiceNamespace="sagemaker", ResourceId="endpoint/image-classifier-prod/variant/AllTraffic", ScalableDimension="sagemaker:variant:DesiredInstanceCount", PolicyType="TargetTrackingScaling", TargetTrackingScalingPolicyConfiguration={ "TargetValue": 750.0, # invocations per instance per minute "PredefinedMetricSpecification": { "PredefinedMetricType": "SageMakerVariantInvocationsPerInstance" }, "ScaleInCooldown": 300, "ScaleOutCooldown": 60, }, )

Processing Jobs

SageMaker Processing provides managed compute for data preprocessing, postprocessing, and model evaluation. Unlike training jobs, processing jobs are optimized for CPU-heavy ETL workloads like feature engineering and data validation.

# SKLearn Processing Job for feature engineering from sagemaker.processing import ProcessingInput, ProcessingOutput from sagemaker.sklearn.processing import SKLearnProcessor processor = SKLearnProcessor( role=role, instance_type="ml.m5.4xlarge", instance_count=1, framework_version="1.2-1", sagemaker_session=session, ) processor.run( code="preprocessing.py", inputs=[ ProcessingInput( source=f"s3://{bucket}/raw-data/", destination="/opt/ml/processing/input", ) ], outputs=[ ProcessingOutput( source="/opt/ml/processing/output/train", destination=f"s3://{bucket}/processed/train/", ), ProcessingOutput( source="/opt/ml/processing/output/test", destination=f"s3://{bucket}/processed/test/", ), ], arguments=["--split-ratio", "0.8"], )
Processing vs. Glue: Use SageMaker Processing for ML-specific data transforms (normalization, tokenization, feature creation). Use AWS Glue for general-purpose ETL on large data lake workloads.

Feature Store

SageMaker Feature Store provides a centralized repository for storing, retrieving, and sharing ML features. It supports both an online store (low-latency reads for inference) and an offline store (batch reads for training) backed by S3.

# Create a Feature Group from sagemaker.feature_store.feature_group import FeatureGroup import pandas as pd # Define feature definitions from a DataFrame df = pd.DataFrame({ "customer_id": ["c001", "c002"], "avg_purchase": [45.2, 102.8], "purchase_count": [12, 58], "churn_risk": [0.15, 0.72], "event_time": ["2024-01-15T10:00:00Z", "2024-01-15T10:00:00Z"], }) feature_group = FeatureGroup( name="customer-features", sagemaker_session=session, ) feature_group.load_feature_definitions(data_frame=df) feature_group.create( s3_uri=f"s3://{bucket}/feature-store/", record_identifier_name="customer_id", event_time_feature_name="event_time", role_arn=role, enable_online_store=True, ) # Ingest features feature_group.ingest(data_frame=df, max_workers=3, wait=True)

To retrieve features at inference time from the online store:

# Read features at inference time record = session.boto_session.client( "sagemaker-featurestore-runtime" ).get_record( FeatureGroupName="customer-features", RecordIdentifierValueAsString="c001", ) features = { f["FeatureName"]: f["ValueAsString"] for f in record["Record"] } print(f"Customer features: {features}")
Online store latency: The online store targets single-digit millisecond reads but adds cost. Only enable it for feature groups that are actually used in real-time inference paths.

SageMaker Pipelines

SageMaker Pipelines is a purpose-built CI/CD service for ML. It orchestrates preprocessing, training, evaluation, and conditional deployment into a single, repeatable workflow defined in Python code.

# Define a SageMaker Pipeline from sagemaker.workflow.pipeline import Pipeline from sagemaker.workflow.steps import ProcessingStep, TrainingStep from sagemaker.workflow.conditions import ConditionGreaterThanOrEqualTo from sagemaker.workflow.condition_step import ConditionStep from sagemaker.workflow.parameters import ParameterString, ParameterFloat # Pipeline parameters instance_type = ParameterString(name="TrainingInstanceType", default_value="ml.p3.2xlarge") accuracy_threshold = ParameterFloat(name="AccuracyThreshold", default_value=0.90) # Step 1: Preprocessing preprocess_step = ProcessingStep( name="PreprocessData", processor=processor, inputs=[...], outputs=[...], code="preprocess.py", ) # Step 2: Training train_step = TrainingStep( name="TrainModel", estimator=estimator, inputs={"train": preprocess_step.properties.ProcessingOutputConfig .Outputs["train"].S3Output.S3Uri}, ) # Step 3: Conditional deployment accuracy_condition = ConditionGreaterThanOrEqualTo( left=train_step.properties.FinalMetricDataList[0].Value, right=accuracy_threshold, ) condition_step = ConditionStep( name="CheckAccuracy", conditions=[accuracy_condition], if_steps=[deploy_step], # deploy if accuracy >= threshold else_steps=[notify_step], # notify team otherwise ) # Assemble pipeline pipeline = Pipeline( name="image-classification-pipeline", parameters=[instance_type, accuracy_threshold], steps=[preprocess_step, train_step, condition_step], sagemaker_session=session, ) pipeline.upsert(role_arn=role) execution = pipeline.start()
Pipeline versioning: Every pipeline execution is recorded with its parameters, input data, and output artifacts. This provides full lineage tracking—you can always trace a deployed model back to the exact training run and dataset.
