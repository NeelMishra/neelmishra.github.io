    Number System and Base Conversion

    May 2026 · 34 min read

    Bit manipulation begins before the first &, |, or ^. You need to be comfortable reading the same value in different number systems: decimal for humans, binary for bits, octal for 3-bit groups, and hexadecimal for compact 4-bit groups.

    A number system is defined by its base, also called its radix. Base 10 has ten digits, base 2 has two digits, base 8 has eight digits, and base 16 has sixteen digits. Conversion is not magic; it is just preserving the same value while changing how that value is grouped.

    Positional Notation

    In base b, each digit must be between 0 and b - 1. The rightmost digit has weight b0, the next digit has weight b1, then b2, and so on.

    The logic is bundling. In base 10, ten ones become one ten, ten tens become one hundred, and so on. In base 2, two ones become one two, two twos become one four, and so on. Moving one position left multiplies the place value by the base; moving one position right divides it by the base.

    Why digits stop at b - 1: If a digit ever reaches b, it is no longer a single digit in that base. It carries into the next position. For example, in base 2 the digit 2 is impossible because two ones become 102.

    dkdk-1...d1d0 in base b means:

    d[k] * b^k + d[k-1] * b^(k-1) + ... + d[1] * b + d[0]
    347 base 10 = 3 * 10^2 + 4 * 10^1 + 7 * 10^0
1011 base 2 = 1 * 2^3  + 0 * 2^2  + 1 * 2^1 + 1 * 2^0 = 11
2F base 16  = 2 * 16^1 + 15 * 16^0 = 47

    Common Bases in Programming

    BaseNameDigitsC++ literal styleWhy it matters
    2Binary0, 10b101010Actual bit patterns used by bitwise operators.
    8Octal0..7052Each octal digit equals exactly 3 binary bits.

    Important: 10 does not always mean ten. It means "one group of the base and zero ones." So 102 = 2, 108 = 8, 1010 = 10, and 1016 = 16.

    Converting Any Base to Decimal

    To convert from base b to decimal, expand every digit by its positional weight and sum the terms.

    The logic: a written number is already a sum of bundles. Base-to-decimal conversion simply counts how many decimal ones those bundles contain. A digit d at position k means d copies of bk.

    Convert 101101 base 2 to decimal:

1 * 2^5 + 0 * 2^4 + 1 * 2^3 + 1 * 2^2 + 0 * 2^1 + 1 * 2^0
= 32 + 0 + 8 + 4 + 0 + 1
= 45
    Convert 735 base 8 to decimal:

7 * 8^2 + 3 * 8^1 + 5 * 8^0
= 448 + 24 + 5
= 477
    Convert 2AF base 16 to decimal:

2 * 16^2 + A * 16^1 + F * 16^0
= 2 * 256 + 10 * 16 + 15
= 687

    Faster Mental Method: Left to Right

    Instead of writing powers explicitly, process digits from left to right. Repeatedly multiply the current answer by the base and add the next digit.

    2AF base 16:

ans = 0
ans = ans * 16 + 2  = 2
ans = ans * 16 + 10 = 42
ans = ans * 16 + 15 = 687

    This is Horner's method. The logic is that appending one digit to the right shifts the old value one base-position left. A left shift in base b multiplies by b, and then the new digit is added in the ones place.

    For digits 2, A, F in base 16:

After "2":   value = 2
After "2A":  value = 2 * 16 + 10
After "2AF": value = (2 * 16 + 10) * 16 + 15

    Converting Decimal to Any Base

    To convert decimal N to base b, repeatedly divide by b. Record each remainder. The first remainder is the rightmost digit, so read the remainders from bottom to top.

    The logic comes from the division algorithm:

    N = b * quotient + remainder, where 0 <= remainder < b

    In base b, the rightmost digit is exactly N mod b, because every higher position is a multiple of b. Dividing by b removes that rightmost digit and exposes the next one. That is why repeated division discovers digits from right to left.

    Convert 45 decimal to binary:

45 / 2 = 22 remainder 1
22 / 2 = 11 remainder 0
11 / 2 = 5  remainder 1
 5 / 2 = 2  remainder 1
 2 / 2 = 1  remainder 0
 1 / 2 = 0  remainder 1

Read bottom-to-top: 101101
45 = 101101 base 2
    Convert 687 decimal to hexadecimal:

687 / 16 = 42 remainder 15  -> F
 42 / 16 = 2  remainder 10  -> A
  2 / 16 = 0  remainder 2   -> 2

Read bottom-to-top: 2AF
687 = 2AF base 16
    Why remainders work: In base b, the last digit stores the value modulo b. Integer division by b removes that last digit and shifts every remaining digit one position right.

    Converting Negative Numbers

    For ordinary bases such as binary, octal, decimal, hexadecimal, or base 7, the base itself is positive. A negative value is handled by converting the absolute value and then attaching a leading minus sign. The sign is not one of the base digits.

    The logic is that the base digits describe only the magnitude. The leading minus sign is an instruction to multiply the whole value by -1. So -1011012 means -(1011012), not a different set of binary place values.

    Negative Decimal to Desired Base

    To convert a negative decimal number -N to base b:

    1. Ignore the sign and convert N to base b using repeated division.
    2. Put - in front of the converted digits.
    Convert -45 decimal to binary:

Convert +45 first:
45 / 2 = 22 remainder 1
22 / 2 = 11 remainder 0
11 / 2 = 5  remainder 1
 5 / 2 = 2  remainder 1
 2 / 2 = 1  remainder 0
 1 / 2 = 0  remainder 1

+45 = 101101 base 2
-45 = -101101 base 2
    Convert -687 decimal to hexadecimal:

Convert +687 first:
687 / 16 = 42 remainder 15 -> F
 42 / 16 = 2  remainder 10 -> A
  2 / 16 = 0  remainder 2  -> 2

+687 = 2AF base 16
-687 = -2AF base 16

    Negative Desired Base to Decimal

    To convert a negative number written in base b back to decimal, remove the leading minus sign, convert the remaining digits normally, and then negate the decimal answer.

    Convert -101101 base 2 to decimal:

Ignore the sign first:
101101 base 2 = 1 * 2^5 + 0 * 2^4 + 1 * 2^3 + 1 * 2^2 + 0 * 2 + 1
              = 32 + 8 + 4 + 1
              = 45

Apply the sign:
-101101 base 2 = -45 decimal
    Convert -2AF base 16 to decimal:

Ignore the sign first:
2AF base 16 = 2 * 16^2 + A * 16 + F
            = 512 + 160 + 15
            = 687

Apply the sign:
-2AF base 16 = -687 decimal
    Representation note: -1011012 is mathematical signed notation in a positive base, not an exotic negative-radix system and not a fixed-width two's-complement bit pattern. For example, -45 in an 8-bit signed integer is stored as 11010011, not as the characters -101101. The next section shows the conversion, and the next post goes deeper into storage details.

    Signed Binary and Two's Complement

    If someone says "signed binary" in programming, they usually mean a fixed-width two's-complement representation. You must know the width first, such as 8-bit, 16-bit, 32-bit, or 64-bit. The same decimal value has different signed binary strings at different widths.

    The logic behind two's complement is modular arithmetic. With w bits, the hardware stores only the remainder modulo 2w. So a negative number n is stored as the non-negative residue 2w + n. Addition still works because both signed and unsigned interpretations use the same wrapped bit pattern.

    Range of a signed w-bit integer:

    -2^(w - 1) to 2^(w - 1) - 1

    Negative Decimal to Signed Binary

    To convert a negative decimal number n to w-bit signed binary:

    1. Check that n is inside the signed w-bit range.
    2. Compute 2w + n. Since n is negative, this is the same as 2w - |n|.
    3. Convert that non-negative result to binary using exactly w bits.
    Convert -45 decimal to 8-bit signed binary:

8-bit signed range = -128 to 127, so -45 fits.

2^8 + (-45) = 256 - 45 = 211
211 decimal = 11010011 binary

Therefore:
-45 as 8-bit signed binary = 11010011

    Why this gives the right signed value: 211 and -45 differ by exactly 256. In 8-bit arithmetic, values that differ by 28 have the same stored bit pattern, because the 9th bit is discarded.

    The same number in 16 bits keeps the same lower 8 bits but extends the sign with leading ones:

    Convert -45 decimal to 16-bit signed binary:

2^16 + (-45) = 65536 - 45 = 65491
65491 decimal = 1111111111010011 binary

Grouped:
-45 as 16-bit signed binary = 11111111 11010011

    Signed Binary to Decimal

    To convert a w-bit signed binary string to decimal:

    1. If the leftmost bit is 0, convert it as an ordinary positive binary number.
    2. If the leftmost bit is 1, convert it as an unsigned value U, then compute U - 2w.
    Convert 11010011 as 8-bit signed binary:

Leftmost bit is 1, so the value is negative.

Unsigned value:
11010011 base 2 = 211 decimal

Signed value:
211 - 2^8 = 211 - 256 = -45

Therefore:
11010011 as 8-bit signed binary = -45

    The logic behind subtracting 2w: when the sign bit is 1, the unsigned value lies in the upper half of the residue range. Two's complement chooses the negative representative of that residue by moving it back one full cycle of size 2w.

    You can also use the invert-and-add-one shortcut to find the magnitude of a negative two's-complement number:

    11010011
invert bits: 00101100
add 1:       00101101 = 45

So the original signed value is -45.

    The shortcut works because, within w bits, ~x equals 2w - 1 - x. Therefore ~x + 1 = 2w - x, which is exactly -x modulo 2w.

    Width matters: 11010011 as an 8-bit signed number is -45, but as a 16-bit signed number written as 00000000 11010011, it is 211. A signed interpretation always needs the bit width.

    Fast Conversion Between Binary, Octal, and Hex

    Binary, octal, and hexadecimal are special because their bases are powers of two: 8 = 23 and 16 = 24. That means you can convert by grouping bits instead of doing full arithmetic.

    The logic: one hex digit stores exactly four binary decisions because 16 has 24 possible values, from 0 to 15. One octal digit stores exactly three binary decisions because 8 = 23. Grouping is not a trick; it is positional notation with a bigger power-of-two base.

    Binary to Hexadecimal

    Group binary bits into chunks of four from right to left. Pad the leftmost group with zeros if needed. Convert each 4-bit group to one hex digit.

    Binary groupHex digitBinary groupHex digit

    0001110019
    001021010A
    001131011B
    010041100C

    011061110E
    011171111F
     
    Convert 110101101 base 2 to hexadecimal:

Pad left: 0001 1010 1101
Groups:   0001 1010 1101
Hex:         1    A    D

110101101 base 2 = 1AD base 16

    Hexadecimal to Binary

    Replace each hex digit by its 4-bit binary group.

    Convert 0x2F to binary:

2 -> 0010
F -> 1111

0x2F = 0010 1111 base 2

    Binary to Octal

    Octal uses 3-bit groups because 8 = 23.

    Convert 101101 base 2 to octal:

Group by 3 bits: 101 101
Octal digits:      5   5

101101 base 2 = 55 base 8

    Converting From One Base to Another

    The most reliable general strategy is a two-step bridge through decimal:

    The logic is that the value is independent of its representation. Decimal is just a convenient checkpoint because we are used to arithmetic in base 10. Once you know the actual value, you can regroup it into any target base.

    1. Convert the source number from base a to decimal.
    2. Convert that decimal value to the target base b.
    Convert 735 base 8 to hexadecimal:

Step 1: octal -> decimal
735 base 8 = 7 * 8^2 + 3 * 8 + 5 = 477

Step 2: decimal -> hexadecimal
477 / 16 = 29 remainder 13 -> D
 29 / 16 = 1  remainder 13 -> D
  1 / 16 = 0  remainder 1  -> 1

735 base 8 = 1DD base 16

    When both bases are powers of two, skip decimal and go through binary groups. This works because both representations are just different group sizes over the same underlying bits. For example, octal to hex can be done as octal → binary → hex:

    735 base 8:

7 -> 111
3 -> 011
5 -> 101

Binary: 111011101
Group by 4: 0001 1101 1101
Hex:          1    D    D

735 base 8 = 1DD base 16

    Fractional Parts

    Fractional conversion has two common directions: decimal fraction to another base, and source-base fraction back to decimal.

    Decimal Fraction to Base

    To convert a decimal fraction to base b, multiply the fraction by b. The integer part of the result becomes the next digit. Keep the remaining fraction and repeat.

    The logic is the fractional mirror of repeated division. Multiplying by b shifts the base-b point one place to the right, so the next hidden fractional digit moves into the integer position where you can read it.

    Convert 0.625 decimal to binary:

0.625 * 2 = 1.25   -> digit 1, remaining 0.25
0.25  * 2 = 0.5    -> digit 0, remaining 0.5
0.5   * 2 = 1.0    -> digit 1, remaining 0

0.625 decimal = 0.101 base 2

    Base Fraction to Decimal

    To convert a fractional number from base b to decimal, expand every digit using positional weights. Digits left of the point use non-negative powers like b0, b1, b2. Digits right of the point use negative powers: b-1, b-2, b-3, and so on.

    The logic is the same bundling idea, but now the bundles are smaller than one. The first digit after the point counts pieces of size 1 / b, the second counts pieces of size 1 / b2, and so on.

    Convert 101.101 base 2 to decimal:

Integer side:
1 * 2^2 + 0 * 2^1 + 1 * 2^0 = 4 + 0 + 1 = 5

Fraction side:
1 * 2^-1 + 0 * 2^-2 + 1 * 2^-3
= 1/2 + 0/4 + 1/8
= 0.5 + 0 + 0.125
= 0.625

Final answer:
101.101 base 2 = 5.625 decimal
    Convert 2A.F base 16 to decimal:

Integer side:
2 * 16^1 + A * 16^0 = 2 * 16 + 10 = 42

Fraction side:
F * 16^-1 = 15 / 16 = 0.9375

Final answer:
2A.F base 16 = 42.9375 decimal

    A code-friendly version is to scan the fractional digits left to right with a decreasing place value:

    fraction = 0
place = 1 / base

for each digit d after the point:
    fraction += d * place
    place /= base

    When converting decimal fractions to another base, some values never terminate. For example, 0.1 repeats forever in base 2, which is why floating-point arithmetic can produce results like 0.30000000000000004.

    C++ Conversion Toolkit

    For contest code, use the standard library when the value fits into an integer type. Use a manual converter when you need to control validation or output base.

    #include <algorithm>
#include <bitset>
#include <stdexcept>
#include <string>
using namespace std;

int digitValue(char c) {
    if ('0' <= c && c <= '9') return c - '0';
    if ('A' <= c && c <= 'Z') return c - 'A' + 10;
    if ('a' <= c && c <= 'z') return c - 'a' + 10;
    return -1;
}

char digitChar(int x) {
    return x < 10 ? char('0' + x) : char('A' + x - 10);
}

long long toDecimal(const string& s, int base) {
    if (base < 2 || base > 36) throw invalid_argument("unsupported base");
    if (s.empty() || s == "-") throw invalid_argument("empty number");
    bool negative = s[0] == '-';
    long long ans = 0;
    for (int i = negative ? 1 : 0; i < (int)s.size(); i++) {
        char c = s[i];
        int d = digitValue(c);
        if (d < 0 || d >= base) throw invalid_argument("digit outside base");
        ans = ans * base + d;
    }
    return negative ? -ans : ans;
}

string fromDecimal(long long n, int base) {
    if (base < 2 || base > 36) throw invalid_argument("unsupported base");
    if (n == 0) return "0";
    bool negative = n < 0;
    unsigned long long x = negative ? 0ULL - (unsigned long long)n : (unsigned long long)n;
    unsigned long long b = (unsigned long long)base;
    string out;
    while (x > 0) {
        out.push_back(digitChar((int)(x % b)));
        x /= b;
    }
    if (negative) out.push_back('-');
    reverse(out.begin(), out.end());
    return out;
}

string convertBase(const string& s, int fromBase, int toBase) {
    return fromDecimal(toDecimal(s, fromBase), toBase);
}

string toSignedBinary(long long n, int width) {
    if (width <= 0 || width > 63) throw invalid_argument("unsupported width");
    long long low = -(1LL << (width - 1));
    long long high = (1LL << (width - 1)) - 1;
    if (n < low || n > high) throw invalid_argument("value outside signed width");

    unsigned long long mod = 1ULL << width;
    unsigned long long x = n < 0 ? mod - (unsigned long long)(-n) : (unsigned long long)n;
    string out(width, '0');
    for (int i = width - 1; i >= 0; i--) {
        out[i] = char('0' + (x & 1ULL));
        x >>= 1;
    }
    return out;
}

long long signedBinaryToDecimal(const string& bits) {
    if (bits.empty() || bits.size() > 63) throw invalid_argument("unsupported width");
    unsigned long long x = 0;
    for (char c : bits) {
        if (c != '0' && c != '1') throw invalid_argument("not a binary digit");
        x = x * 2 + (c - '0');
    }
    int width = (int)bits.size();
    if (bits[0] == '0') return (long long)x;
    unsigned long long mod = 1ULL << width;
    return -(long long)(mod - x);
}
    toDecimal("101101", 2)        // 45
toDecimal("-101101", 2)       // -45
fromDecimal(687, 16)          // "2AF"
fromDecimal(-687, 16)         // "-2AF"
convertBase("735", 8, 16)     // "1DD"
convertBase("-101101", 2, 16) // "-2D"
toSignedBinary(-45, 8)         // "11010011"
toSignedBinary(-45, 16)        // "1111111111010011"
signedBinaryToDecimal("11010011") // -45
bitset<8>(42).to_string()    // "00101010"
    Overflow note: The code above assumes the converted value fits in long long. For very large numbers, convert using string division/remainders instead of passing through a fixed-width integer.

    Common Mistakes

    • Reading digits as decimal: 10102 is ten, but 101010 is one thousand ten.
    • Forgetting to reverse remainders: repeated division produces digits from least significant to most significant.
    • Treating the minus sign as a digit: for normal positive bases, convert the absolute value first and keep the sign separately.
    • Interpreting signed binary without width: 11010011 needs a width before it can be read as a signed two's-complement value.
    • Using positive powers after the point: fractional digits use b-1, b-2, ..., not b1, b2, ....
    • Using invalid digits: 8 is invalid in octal, and G is invalid in hexadecimal.
    • Dropping leading zeros too early: leading zeros do not change the value, but they matter when showing fixed-width bit patterns such as 00001010.
    • Confusing value with representation: 42, 0b101010, 052, and 0x2A are different representations of the same value.

    Summary

    • A base tells you how many digit symbols exist and what each position is worth.
    • Base to decimal unpacks each digit into decimal ones: digit d at position k contributes d * bk.
    • Decimal to base reverses that unpacking: N mod b reveals the rightmost digit, and dividing by b removes it.
    • Negative values in ordinary positive bases keep the sign outside the digits because the digits encode magnitude and the sign multiplies the whole value by -1.
    • Signed binary in programming usually means fixed-width two's complement: values are stored modulo 2w, so negative n is stored as 2w + n.
    • Binary, octal, and hex conversions are fast because octal groups 3 bits and hex groups 4 bits; those group sizes come from 8 = 23 and 16 = 24.
    • Any base-to-base conversion can be done as source base → value → target base; decimal is only a convenient checkpoint.
    • For decimal fractions to another base, multiplication by the base reveals the next digit; for source-base fractions to decimal, negative powers count pieces smaller than one.
