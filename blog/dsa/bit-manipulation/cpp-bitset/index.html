    std::bitset Essentials

    June 2026 · 22 min read

    std::bitset is the standard library's fixed-size array of bits, packed tightly into machine words. It turns the manual shift-and-mask work from the rest of this series into named member functions, while keeping the speed of raw bit operations. This first post covers the fundamentals: how to declare a bitset, how its bits are numbered, and how to read, modify, and convert it. The next post covers the operators and idioms, and the third covers performance and real use cases.

    What std::bitset Is

    A std::bitset<N> stores exactly N bits, where N is a compile-time constant. The bits live in as few unsigned long words as needed, so a bitset<1000> occupies roughly 128 bytes rather than 1000 bytes. It lives in the <bitset> header.

    #include <bitset>
#include <iostream>

std::bitset<8>  flags;     // 8 bits, all zero by default
std::bitset<64> mask;      // 64 bits, fits in one machine word
std::bitset<1000> sieve;   // 1000 bits, ~16 words
    The size is part of the type. bitset<8> and bitset<16> are different, incompatible types. Because N must be known at compile time, a bitset whose length is only known at runtime is not possible; that case needs std::vector<bool> or boost::dynamic_bitset, covered in Part 3.

    Declaring and Initializing

    A default-constructed bitset is all zeros. It can also be built from an unsigned integer or from a string of '0' and '1' characters.

    std::bitset<8> a;                 // 00000000
std::bitset<8> b(42);             // 00101010  (low bits of the integer)
std::bitset<8> c("00101010");     // 00101010  (from a string)
std::bitset<8> d(0xFF);           // 11111111

// Only the low N bits of the integer are kept:
std::bitset<4> e(42);             // 1010  (42 = 101010, top bits dropped)

    The string constructor reads the leftmost character as the most significant bit, matching how the value is printed. A substring and custom zero/one characters are also accepted:

    std::string s = "XXOOXX";
std::bitset<6> f(s, 0, 6, 'O', 'X');   // treat 'X' as 1, 'O' as 0  -> 110011

// Start at offset 2, take 4 characters:
std::bitset<4> g("11001010", 2, 4);    // reads "0010" -> 0010

    The Indexing Convention

    This is the single most important rule, and the easiest to get wrong. Bit position 0 is the least significant bit, the one with weight $2^0$. Positions grow toward the most significant bit. Printing, however, shows the most significant bit first, so the printed order is the reverse of the index order.

    std::bitset<8> x("00101010");
//  printed:   0 0 1 0 1 0 1 0
//  index:     7 6 5 4 3 2 1 0
//                 ^         ^
//             x[5]==1     x[1]==1

std::cout << x[0];   // 0  (rightmost, weight 1)
std::cout << x[1];   // 1  (weight 2)
std::cout << x[5];   // 1  (weight 32)
    Index 0 is on the right; print order is on the left. If a loop or an animation ever looks mirrored, this convention is almost always the reason.

    Reading Bits

    There are two ways to read a single bit, plus several whole-bitset queries.

    std::bitset<8> x(42);   // 00101010

x[3];          // proxy reference; no bounds check (UB if out of range)
x.test(3);     // bounds-checked; throws std::out_of_range if pos >= N

x.count();     // number of set bits (popcount)  -> 3
x.size();      // total number of bits            -> 8
x.any();       // true if at least one bit is set -> true
x.none();      // true if no bit is set           -> false
x.all();       // true if every bit is set        -> false

    Use operator[] in hot loops where the index is already known to be valid, and test() when the index might be out of range and a thrown exception is preferable to undefined behavior.

    Modifying Bits

    Each mutating member has a whole-bitset form (no argument) and a single-bit form (a position, and optionally a value).

    std::bitset<8> x;     // 00000000

x.set();          // set every bit        -> 11111111
x.set(2);         // set bit 2            -> ...00000100 within the above
x.set(2, false);  // clear bit 2 (value form)
x.reset();        // clear every bit      -> 00000000
x.reset(5);       // clear bit 5
x.flip();         // toggle every bit
x.flip(0);        // toggle bit 0

// operator[] also assigns through a proxy:
x[7] = 1;         // set the most significant bit

    The single-bit forms are bounds-checked and throw std::out_of_range for an invalid position, exactly like test().

    Converting Out

    A bitset converts back to text or to an unsigned integer.

    std::bitset<8> x(42);   // 00101010

x.to_string();              // "00101010"
x.to_string('.', '#');      // "..#.#.#."  (custom zero, one glyphs)
x.to_ulong();               // 42 as unsigned long
x.to_ullong();              // 42 as unsigned long long
    Conversion can throw. to_ulong() and to_ullong() raise std::overflow_error when the value does not fit in the target type. A bitset<100> with a high bit set cannot become a 64-bit integer, so guard the call or keep N within the integer width when a numeric conversion is required.

    Stream I/O

    Bitsets stream directly to and from iostreams. Output prints most significant bit first; input reads a run of '0' and '1' characters.

    std::bitset<8> x(42);
std::cout << x << "\n";        // 00101010

std::bitset<8> y;
std::cin  >> y;                // reads up to 8 of '0'/'1' from input

    This makes printing a binary representation a one-liner, which is why earlier posts in this series used std::bitset<32>(value) simply to display bit patterns.

    Summary

    • std::bitset<N> is a compile-time fixed array of N bits, packed into words and zero-initialized by default.
    • Build one from an integer (low N bits kept) or from a '0'/'1' string (leftmost character is the most significant bit).
    • Bit 0 is the least significant bit; printing reverses that order.
    • operator[] is unchecked and fast; test(), set(pos), reset(pos), and flip(pos) are bounds-checked.
    • count, size, any, none, and all answer whole-bitset questions cheaply.
    • to_ulong and to_ullong throw overflow_error when the value will not fit.
