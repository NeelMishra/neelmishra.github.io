    bitset Operators & Idioms

    June 2026 · 24 min read

    Part 1 covered a single bitset in isolation. The real power shows up when whole bitsets are combined with the bitwise operators, all of which run across every bit at once. This post walks through those operators, the way shifts behave inside a fixed width, and the everyday idioms that turn a bitset into a flag set or a mask. It assumes the Essentials are familiar, especially the rule that bit 0 is the least significant bit.

    Full-Width Bitwise Operators

    The operators &, |, ^, and ~ apply position by position across two bitsets of the same size N. They behave exactly like the integer operators from Part 4, just over the entire fixed width.

    std::bitset<8> a("11001010");
std::bitset<8> b("01101100");

a & b;    // 01001000   bits set in both
a | b;    // 11101110   bits set in either
a ^ b;    // 10100110   bits set in exactly one
~a;       // 00110101   every bit flipped

// Compound forms mutate in place:
a &= b;
a |= b;
a ^= b;
    The widths must match. bitset<8> and bitset<16> cannot be combined directly; the compiler rejects it. Convert one explicitly (for example through to_ullong() and a new bitset) when mixing widths is genuinely intended.

    Shifting Within a Fixed Width

    The shift operators << and >> move every bit by a fixed amount. Because the width is fixed at N, bits that move past either end are simply dropped, and zeros fill in behind them. There is no sign extension and no arithmetic shift; a bitset is unsigned by nature.

    std::bitset<8> x("00010110");

x << 2;   // 01011000   top two bits fell off, zeros entered at the bottom
x >> 2;   // 00000101   bottom two bits fell off, zeros entered at the top

x <<= 1;  // in-place left shift
x >>= 1;  // in-place right shift

    A left shift by k is the bitset analogue of multiplying by $2^k$ and keeping only the low N bits. This is the operation behind subset-sum and reachability DP, where a single dp <<= w shifts an entire set of reachable sums by a weight in one instruction. Part 3 returns to that use case.

    Comparison and Equality

    Two bitsets of the same width compare with == and !=. There is no < ordering operator, because there is no single agreed meaning for one bit pattern being less than another.

    std::bitset<8> a(42), b(42), c(43);
a == b;   // true
a != c;   // true
// a < c;  // does not compile

    When an ordering is needed (for example to use bitsets as map keys), convert to an integer with to_ullong() for small widths, or hash the bitset, shown below.

    Bitsets as Flag Sets and Masks

    A common use is a compact set of named boolean flags. Give each flag a bit position, then set, clear, and test by name.

    enum Perm { READ = 0, WRITE = 1, EXEC = 2, ADMIN = 3 };

std::bitset<8> perms;
perms.set(READ);
perms.set(WRITE);

perms.test(WRITE);          // true
perms.test(EXEC);           // false

// Combine two permission sets:
std::bitset<8> required;
required.set(READ);
required.set(EXEC);

// Does perms contain every required bit?
bool ok = (perms & required) == required;   // subset test

    The subset test (a & b) == b is worth memorizing: it is true exactly when every bit of b is also set in a. It is the bitset form of asking whether one set contains another.

    The Print-versus-Index Trap

    The most frequent bug when building masks is mixing up the two directions. Construction from a string and printing both put the most significant bit on the left, while operator[] and the position arguments count from the least significant bit on the right.

    std::bitset<8> m("00000001");
// printed leftmost-first: 0 0 0 0 0 0 0 1
// the single 1 is the LEAST significant bit, so:
m[0];   // true
m[7];   // false

std::bitset<8> n;
n.set(7);                 // sets the MOST significant bit
std::cout << n;           // 10000000
    Reading a string sets high bits on the left; set(pos) counts from the right. When a mask looks reversed, line up the printed string against the indices before changing any logic.

    Iterating Over Set Bits

    A portable scan tests each position in turn. It is simple and correct, and runs in $O(N)$.

    std::bitset<32> s(0b10110);
for (std::size_t i = 0; i < s.size(); ++i) {
    if (s[i]) {
        // i is the position of a set bit
    }
}

    On GCC and Clang, two non-standard members jump directly between set bits, which is much faster when the bitset is sparse. They are a libstdc++ extension, so guard any portable code that relies on them.

    // GCC / libstdc++ extension, not standard C++:
for (std::size_t i = s._Find_first(); i < s.size(); i = s._Find_next(i)) {
    // visits only the set bits, skipping runs of zeros
}

    Hashing and Using a bitset as a Key

    The standard library specializes std::hash for std::bitset, so a bitset can be the key of an unordered_set or unordered_map with no extra work.

    #include <unordered_set>

std::unordered_set<std::bitset<16>> seen;
std::bitset<16> state(0b1011);
if (!seen.count(state)) {
    seen.insert(state);
}

    This makes a bitset a natural representation for a visited-set in state-space search, where each state is a compact subset and equality is a fast word comparison.

    Test membership / subset(a & b) == b
    Toggle a flags.flip(pos)
    Union / intersection of setsa | b  /  a & b

    Shift a set of values up by ks <<= k
    Is the set empty / full?s.none()  /  s.all()

    Summary

    • &, |, ^, and ~ act across the whole fixed width and require matching sizes.
    • Shifts drop bits that fall off either end and fill with zeros; there is no sign extension.
    • Only == and != are defined; convert or hash when an ordering is needed.
    • A bitset is a compact flag set; (a & b) == b is the subset test.
    • Construction and printing are most-significant-first, while indexing is least-significant-first.
    • std::hash is specialized for bitset, so it works as an unordered-container key out of the box.
