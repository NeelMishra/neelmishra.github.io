Floyd-Warshall Algorithm

All-pairs shortest paths in O(V³) using dynamic programming.

When to Use Floyd-Warshall

When you need shortest paths between every pair of vertices in a graph, Floyd-Warshall is the go-to algorithm. It handles negative weights (but not negative cycles) and works on both directed and undirected graphs.

  • Small dense graphs — V ≤ 500, any edge structure
  • Transitive closure — can vertex i reach vertex j?
  • All-pairs problems — diameter, center, minimax paths
  • Negative cycle detection — check if dist[i][i] < 0

The DP Recurrence

Let dist[i][j] be the shortest path from i to j. We introduce intermediate vertices one by one:

Floyd-Warshall
// Initialize dist[i][j] = weight(i,j) or ∞
// dist[i][i] = 0

for k = 0 to V-1:          // intermediate vertex
    for i = 0 to V-1:      // source
        for j = 0 to V-1:  // destination
            dist[i][j] = min(dist[i][j],
                             dist[i][k] + dist[k][j])

Why k must be the outer loop: When we process intermediate vertex k, we need dist[i][k] and dist[k][j] to already reflect paths using intermediates {0, ..., k-1}. If i or j were outer, this invariant breaks.

Path Reconstruction

Maintain a next[i][j] matrix. Initialize next[i][j] = j if edge (i,j) exists. When updating dist[i][j] through k, set next[i][j] = next[i][k].

vector<int> getPath(int i, int j, vector<vector<int>>& nxt) {
    if (nxt[i][j] == -1) return {}; // no path
    vector<int> path = {i};
    while (i != j) {
        i = nxt[i][j];
        path.push_back(i);
    }
    return path;
}

Negative Cycle Detection

After running Floyd-Warshall, check the diagonal: if dist[i][i] < 0 for any i, then vertex i is on a negative cycle. Any path dist[a][b] where dist[a][i] + dist[i][b] involves a negative-cycle vertex has dist = -∞.

Transitive Closure (Warshall’s Algorithm)

Replace arithmetic with boolean logic: reach[i][j] = reach[i][j] || (reach[i][k] && reach[k][j]). This gives O(V³) reachability for all pairs.

void transitiveClosure(vector<vector<bool>>& reach, int n) {
    for (int k = 0; k < n; k++)
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                reach[i][j] = reach[i][j] || (reach[i][k] && reach[k][j]);
}

C++ Implementation

#include <bits/stdc++.h>
using namespace std;
const long long INF = 1e18;

void floydWarshall(vector<vector<long long>>& dist, vector<vector<int>>& nxt) {
    int n = dist.size();
    for (int k = 0; k < n; k++)
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++)
                if (dist[i][k] < INF && dist[k][j] < INF
                    && dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                    nxt[i][j] = nxt[i][k];
                }
}

int main() {
    int n = 4;
    vector<vector<long long>> dist(n, vector<long long>(n, INF));
    vector<vector<int>> nxt(n, vector<int>(n, -1));

    for (int i = 0; i < n; i++) dist[i][i] = 0;
    auto addEdge = [&](int u, int v, long long w) {
        dist[u][v] = w; nxt[u][v] = v;
    };
    addEdge(0,1,3); addEdge(0,2,8); addEdge(0,3,7);
    addEdge(1,2,1); addEdge(2,3,2); addEdge(3,1,1);

    floydWarshall(dist, nxt);

    // Print shortest distances
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            cout << (dist[i][j] >= INF ? "INF" : to_string(dist[i][j])) << "\t";
        cout << "\n";
    }
}

Complexity Analysis

Floyd-Warshall

Time: O(V³)
Space: O(V²)
All-pairs. Handles negative weights. Simple to implement.

vs V × Dijkstra

V × Dijkstra: O(V(V+E) log V).
Better for sparse graphs (E « V²).
Floyd-Warshall wins for dense graphs or V ≤ 500.

CP Patterns

1. Shortest Path in Small Graphs

When V ≤ 400–500, Floyd-Warshall is the easiest all-pairs solution. No need for adjacency lists.

2. Minimax / Maximin Paths

Replace min(dist[i][j], dist[i][k]+dist[k][j]) with min(dist[i][j], max(dist[i][k], dist[k][j])) for the minimax path (minimize the maximum edge weight).

3. Widest Path

Find the path maximizing the minimum edge weight (bottleneck path): max(dist[i][j], min(dist[i][k], dist[k][j])).

4. Counting Shortest Paths

Maintain a count matrix alongside dist. When dist improves, reset count. When equal, add counts.

Summary

  • Floyd-Warshall computes all-pairs shortest paths in O(V³).
  • The DP recurrence adds intermediate vertices one by one — k must be the outer loop.
  • Detects negative cycles via dist[i][i] < 0.
  • Path reconstruction uses a next[][] matrix.
  • Variants: transitive closure, minimax paths, widest paths, counting paths.
  • Best for dense graphs with V ≤ 500.
