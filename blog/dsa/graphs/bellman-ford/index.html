Bellman-Ford Algorithm

Single-source shortest paths with negative weights and negative cycle detection.

When to Use Bellman-Ford

Dijkstra fails on graphs with negative edge weights. The Bellman-Ford algorithm handles negative weights correctly and can also detect negative-weight cycles — something no other SSSP algorithm does as naturally.

  • Currency arbitrage — detect profitable exchange rate cycles
  • Differential constraints — constraint satisfaction via shortest paths
  • Routing protocols — RIP uses a distributed Bellman-Ford variant
  • At-most-K-edges paths — run only K iterations instead of V-1

The Algorithm

Bellman-Ford relaxes every edge, V-1 times. After V-1 iterations, all shortest paths are found (a shortest path has at most V-1 edges).

Bellman-Ford
dist[source] = 0, dist[*] = ∞

for i = 1 to V-1:
    for each edge (u, v, w):
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w

// Negative cycle check (Vth iteration)
for each edge (u, v, w):
    if dist[u] + w < dist[v]:
        → negative cycle exists!

Why V-1 iterations? A shortest path in a graph with V vertices uses at most V-1 edges. In iteration i, all shortest paths using ≤ i edges are correct. So after V-1 iterations, all shortest paths are finalized.

Negative Cycle Detection

If the V-th iteration still reduces some distance, a negative cycle is reachable from the source. Vertices on or reachable from the cycle have dist = -∞.

Arbitrage Example

Model currencies as nodes, exchange rates as edges with weight = -log(rate). A negative cycle means the product of exchange rates around the cycle > 1 — free money!

// Arbitrage detection: edges are -log(exchange_rate)
// A negative cycle in this graph = profitable arbitrage
bool hasArbitrage(int n, vector<tuple<int,int,double>>& edges) {
    vector<double> dist(n, 0); // start all at 0 to detect any cycle
    for (int i = 0; i < n; i++)
        for (auto& [u, v, w] : edges)
            if (dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
    // Vth iteration check
    for (auto& [u, v, w] : edges)
        if (dist[u] + w < dist[v]) return true;
    return false;
}

SPFA Optimization

The Shortest Path Faster Algorithm uses a queue: only enqueue vertices whose distance actually changed. Average case is much faster than O(VE), but worst case remains the same.

vector<long long> spfa(int src, int n, vector<vector<pair<int,int>>>& adj) {
    vector<long long> dist(n, LLONG_MAX);
    vector<bool> inQueue(n, false);
    queue<int> q;
    dist[src] = 0; q.push(src); inQueue[src] = true;

    while (!q.empty()) {
        int u = q.front(); q.pop(); inQueue[u] = false;
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (!inQueue[v]) { q.push(v); inQueue[v] = true; }
            }
        }
    }
    return dist;
}

Warning: SPFA is banned/unreliable in many CP contests because adversarial test cases can force O(VE). Prefer standard Bellman-Ford or Dijkstra when possible.

C++ Implementation

Standard Bellman-Ford

#include <bits/stdc++.h>
using namespace std;

struct Edge { int u, v, w; };

// Returns dist array; dist[v] == LLONG_MIN if v is on/reachable from negative cycle
vector<long long> bellmanFord(int src, int n, vector<Edge>& edges) {
    vector<long long> dist(n, LLONG_MAX);
    dist[src] = 0;

    // V-1 relaxation passes
    for (int i = 0; i < n - 1; i++)
        for (auto& e : edges)
            if (dist[e.u] != LLONG_MAX && dist[e.u] + e.w < dist[e.v])
                dist[e.v] = dist[e.u] + e.w;

    // Detect negative cycles (Vth pass)
    for (auto& e : edges)
        if (dist[e.u] != LLONG_MAX && dist[e.u] + e.w < dist[e.v])
            dist[e.v] = LLONG_MIN; // reachable from negative cycle

    return dist;
}

int main() {
    int n = 5, src = 0;
    vector<Edge> edges = {{0,1,4},{0,2,3},{1,2,-2},{1,3,2},{2,3,5},{3,4,1}};
    auto dist = bellmanFord(src, n, edges);
    for (int i = 0; i < n; i++)
        cout << "dist[" << i << "] = " << dist[i] << "\n";
}

Complexity Analysis

Bellman-Ford

Time: O(V · E)
Space: O(V)
Handles negative weights. Detects negative cycles.

vs Dijkstra

Dijkstra: O((V+E) log V) but no negative weights.
Bellman-Ford: Slower but more general.
Use Dijkstra when all weights ≥ 0.

CP Patterns

1. Shortest Path with at Most K Edges

Run only K iterations instead of V-1. After iteration K, dist[v] is the shortest path using ≤ K edges. Used in problems like “Cheapest Flights Within K Stops.”

2. Differential Constraints

A system of inequalities xj - xi ≤ wij maps directly to shortest-path edges. Run Bellman-Ford on the constraint graph.

3. Finding Negative Cycles

To find the actual cycle: track parent[], then from any vertex that improves on the V-th iteration, follow parents V times to reach a vertex guaranteed to be on the cycle, then trace the cycle.

Summary

  • Bellman-Ford finds SSSP in O(VE), handling negative edge weights.
  • Relax all edges V-1 times; a V-th pass detects negative cycles.
  • SPFA is a queue-based optimization but has worst-case O(VE) and is unreliable in contests.
  • Use for: negative weights, at-most-K-edges paths, differential constraints, arbitrage detection.
