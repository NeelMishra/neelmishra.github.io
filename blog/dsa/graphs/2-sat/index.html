2-SAT

Solving Boolean satisfiability with implication graphs & strongly connected components

June 2025

Intuition & Motivation

2-SAT is a special case of the Boolean satisfiability problem where every clause has exactly two literals. While general SAT is NP-complete, 2-SAT is solvable in linear time using graph theory — specifically, by reducing it to finding strongly connected components.

In competitive programming, 2-SAT appears whenever you have binary choices with pairwise constraints: “if you pick A, you must also pick B”, “you can’t pick both C and D”, etc.

Problem Statement

Given n Boolean variables x1, x2, …, xn and m clauses, each of the form (a ∨ b) where a, b are literals (xi or ¬xi), determine if there exists an assignment making all clauses true. If yes, construct one.

Example

Variables: x1, x2, x3. Clauses:

  • (x1 ∨ ¬x2)
  • (¬x1 ∨ x3)
  • (x2 ∨ x3)
  • (¬x3 ∨ ¬x1)

Solution: x1=false, x2=false, x3=true satisfies all clauses.

The Implication Graph

The key insight is converting each clause into two implications:

(a ∨ b) ≡ (¬a ⇒ b) ∧ (¬b ⇒ a)

Each variable xi gets two nodes: one for xi (true) and one for ¬xi (false). Each implication becomes a directed edge. This creates the implication graph.

SCC-Based Solution

The 2-SAT instance is satisfiable if and only if no variable xi and its negation ¬xi are in the same SCC of the implication graph.

Build the implication graph

For each clause (a ∨ b), add edges ¬a → b and ¬b → a. Graph has 2n nodes and 2m edges.

Find all SCCs

Use Kosaraju’s or Tarjan’s algorithm to find SCCs and compute the topological order of the condensation (DAG of SCCs).

Check satisfiability

For each variable xi: if xi and ¬xi are in the same SCC, output UNSATISFIABLE.

If xi ⇔ ¬xi (reachable in both directions), the formula is contradictory.

Construct assignment

For each variable xi: set xi = true if the SCC containing xi has a higher topological order (appears later) than the SCC containing ¬xi. Otherwise set xi = false.

The later SCC “wins” because implications flow toward it.

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

struct TwoSAT {
    int n;
    vector<vector<int>> adj, radj;
    vector<int> order, comp;
    vector<bool> vis;

    TwoSAT(int n) : n(n), adj(2*n), radj(2*n), comp(2*n), vis(2*n) {}

    // x = valX OR y = valY
    void addClause(int x, bool valX, int y, bool valY) {
        // (!valX ? x : x+n) -> (valY ? y : y+n)
        int u = x * 2 + (valX ? 0 : 1);
        int nu = x * 2 + (valX ? 1 : 0);
        int v = y * 2 + (valY ? 0 : 1);
        int nv = y * 2 + (valY ? 1 : 0);
        adj[nu].push_back(v);   // !a -> b
        radj[v].push_back(nu);
        adj[nv].push_back(u);   // !b -> a
        radj[u].push_back(nv);
    }

    void dfs1(int u) {
        vis[u] = true;
        for (int v : adj[u]) if (!vis[v]) dfs1(v);
        order.push_back(u);
    }

    void dfs2(int u, int c) {
        comp[u] = c;
        for (int v : radj[u]) if (comp[v] == -1) dfs2(v, c);
    }

    // Returns true if satisfiable, fills 'result'
    bool solve(vector<bool>& result) {
        fill(vis.begin(), vis.end(), false);
        for (int i = 0; i < 2*n; i++)
            if (!vis[i]) dfs1(i);
        fill(comp.begin(), comp.end(), -1);
        int c = 0;
        for (int i = 2*n - 1; i >= 0; i--)
            if (comp[order[i]] == -1) dfs2(order[i], c++);

        result.resize(n);
        for (int i = 0; i < n; i++) {
            if (comp[2*i] == comp[2*i+1]) return false;
            result[i] = comp[2*i] > comp[2*i+1];
        }
        return true;
    }
};

int main() {
    int n, m;
    cin >> n >> m;
    TwoSAT sat(n);

    for (int i = 0; i < m; i++) {
        int a, b; cin >> a >> b;
        bool va = a > 0, vb = b > 0;
        a = abs(a) - 1; b = abs(b) - 1;
        sat.addClause(a, va, b, vb);
    }

    vector<bool> result;
    if (sat.solve(result)) {
        cout << "SATISFIABLE\n";
        for (int i = 0; i < n; i++)
            cout << "x" << i+1 << " = " << (result[i] ? "true" : "false") << "\n";
    } else {
        cout << "UNSATISFIABLE\n";
    }
}

Complexity

Time: O(n + m) — building the implication graph is O(m), Kosaraju’s SCC is O(n + m).

Space: O(n + m) for the graph.

Variations & Extensions

2-SAT with Forced Variables

To force xi = true, add clause (xi ∨ xi). This creates edge ¬xi → xi.

Lexicographically Smallest Solution

Process variables in order. For each, try setting to false first; check if consistent with remaining implications using BFS/DFS on the implication graph. O(n(n+m)) total.

Relationship to Other Problems

  • 3-SAT: NP-complete — no polynomial algorithm known
  • Horn-SAT: Clauses with at most one positive literal; solvable in linear time by unit propagation
  • MAX-2-SAT: Maximise satisfied clauses; NP-hard, but good approximation algorithms exist

Common CP Patterns

  • “Assign binary labels with pairwise constraints” → Direct 2-SAT
  • “Two teams, incompatibility constraints” → Model as 2-SAT with XOR clauses
  • “Placement problems (left/right, top/bottom)” → Each object gets a Boolean variable for its position
  • “Network configuration with at-most-one constraints” → “Not both” clauses
  • “Interval scheduling with conflicts” → 2-SAT on interval endpoints
