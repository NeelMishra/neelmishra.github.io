Minimum Spanning Tree

Kruskal’s, Prim’s & Borůvka’s — connecting everything at minimum cost

June 2025

Intuition & Motivation

Imagine you are an engineer tasked with connecting n cities with fibre-optic cables. Each potential cable has a cost. You need every city reachable from every other, using as few cables as possible and minimising total cost. That is exactly the Minimum Spanning Tree (MST) problem.

A spanning tree of a connected graph G = (V, E) is a subgraph that is a tree containing all |V| vertices. It has exactly |V| − 1 edges. Among all spanning trees, an MST has the smallest total edge weight.

The Cut Property (Why MST Works)

Every MST algorithm relies on the Cut Property:

For any cut (S, V\S) of a connected weighted graph, the minimum-weight edge crossing the cut belongs to every MST (assuming unique edge weights).

A cut partitions vertices into two non-empty sets. The lightest edge connecting the two sides must be in the MST. This greedy insight powers all three algorithms below.

Cycle Property: Conversely, the heaviest edge in any cycle does not belong to any MST. This is the “dual” of the cut property and equally important.

Kruskal’s Algorithm

Idea: Sort all edges by weight. Process them in ascending order. Add an edge to the MST if it connects two different components (i.e., doesn’t form a cycle). Use Disjoint Set Union (DSU) for near-constant-time component checks.

Step-by-Step Walkthrough

Consider a graph with 5 vertices (0–4) and edges: 0-1(2), 0-3(6), 1-2(3), 1-4(5), 2-3(1), 2-4(4), 3-4(7).

Sort edges & pick 2–3 (w=1)

Lightest edge. Vertices 2 and 3 are in different components → add.

MST edges: {2-3} • Cost so far: 1

Pick 0–1 (w=2)

0 and 1 in different components → add.

MST edges: {2-3, 0-1} • Cost: 3

Pick 1–2 (w=3)

1 in {0,1}, 2 in {2,3} — different components → add. Merges into {0,1,2,3}.

MST edges: {2-3, 0-1, 1-2} • Cost: 6

Pick 2–4 (w=4)

4 is still alone; 2 is in the big component → add. All 5 vertices connected!

MST edges: {2-3, 0-1, 1-2, 2-4} • Total MST cost: 10 ✓

Remaining edges: skip

Edges 1-4(5), 0-3(6), 3-4(7) connect vertices in the same component → skip. Done with |V|−1 = 4 edges.

C++ Implementation (Kruskal’s with DSU)

#include <bits/stdc++.h>
using namespace std;

struct DSU {
    vector<int> par, rnk;
    DSU(int n) : par(n), rnk(n, 0) { iota(par.begin(), par.end(), 0); }
    int find(int x) { return par[x] == x ? x : par[x] = find(par[x]); }
    bool unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;
        if (rnk[a] < rnk[b]) swap(a, b);
        par[b] = a;
        if (rnk[a] == rnk[b]) rnk[a]++;
        return true;
    }
};

int main() {
    int n, m;
    cin >> n >> m;
    vector<tuple<int,int,int>> edges(m);
    for (auto& [w, u, v] : edges) cin >> u >> v >> w;
    sort(edges.begin(), edges.end());

    DSU dsu(n);
    long long cost = 0;
    int cnt = 0;
    for (auto [w, u, v] : edges) {
        if (dsu.unite(u, v)) {
            cost += w;
            if (++cnt == n - 1) break;
        }
    }
    if (cnt < n - 1) cout << "No MST (disconnected)\n";
    else cout << "MST cost: " << cost << "\n";
}

Complexity: O(E log E) for sorting + O(E α(V)) for DSU ≈ O(E log E).

Prim’s Algorithm

Idea: Grow the MST one vertex at a time. Start from any vertex. At each step, pick the cheapest edge connecting an MST vertex to a non-MST vertex. Use a priority queue (min-heap).

Step-by-Step Walkthrough

Same graph. Start from vertex 0.

Start at vertex 0

MST = {0}. Push edges: (0-1, w=2), (0-3, w=6) into heap.

MST vertices: {0}

Pop (0-1, w=2)

1 not in MST → add. Push from 1: (1-2, w=3), (1-4, w=5).

MST: {0,1} • Cost: 2

Pop (1-2, w=3)

2 not in MST → add. Push from 2: (2-3, w=1), (2-4, w=4).

MST: {0,1,2} • Cost: 5

Pop (2-3, w=1)

3 not in MST → add.

MST: {0,1,2,3} • Cost: 6

Pop (2-4, w=4)

4 not in MST → add. All vertices connected. Done!

MST: {0,1,2,3,4} • Total: 10 ✓

C++ Implementation (Prim’s with Priority Queue)

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<pair<int,int>>> adj(n);
    for (int i = 0; i < m; i++) {
        int u, v, w; cin >> u >> v >> w;
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    vector<bool> inMST(n, false);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    pq.push({0, 0});
    long long cost = 0;
    int cnt = 0;

    while (!pq.empty() && cnt < n) {
        auto [w, u] = pq.top(); pq.pop();
        if (inMST[u]) continue;
        inMST[u] = true;
        cost += w; cnt++;
        for (auto [v, wt] : adj[u])
            if (!inMST[v]) pq.push({wt, v});
    }
    cout << "MST cost: " << cost << "\n";
}

Complexity: O((V + E) log V) with binary heap. With Fibonacci heap: O(E + V log V).

Borůvka’s Algorithm

Idea: In each round, every component simultaneously picks its cheapest outgoing edge. All such edges are added. This halves components each round, giving O(log V) rounds.

Borůvka’s is inherently parallel — each component acts independently. It’s the basis for randomised MST algorithms achieving expected O(E) time.

Pseudocode

while number_of_components > 1:
    for each component C:
        cheapest[C] = lightest edge leaving C
    for each component C:
        add cheapest[C] to MST
        merge the two endpoints' components

Complexity: O(E log V) — each round scans all edges O(E), at most O(log V) rounds.

Comparing the Three Algorithms

Kruskal’s

  • Time: O(E log E)
  • Best for: Sparse graphs
  • Uses: DSU
  • Approach: Global edge sort

Prim’s

  • Time: O((V+E) log V)
  • Best for: Dense graphs
  • Uses: Priority queue
  • Approach: Grow from source

Borůvka’s

  • Time: O(E log V)
  • Best for: Parallel settings
  • Uses: DSU + per-component min
  • Approach: Simultaneous merging

Variations & Extensions

Second-Best MST

Find the MST, then for each non-MST edge (u,v,w), the best alternative tree comes from adding this edge and removing the heaviest edge on the MST path u→v. Use LCA with path-max queries (Binary Lifting / Sparse Table) for O(1) per query after O(V log V) preprocessing.

MST Uniqueness

An MST is unique iff for every non-MST edge (u,v,w), the maximum weight on the MST path u→v is strictly less than w. If any non-MST edge ties with the path-max, multiple MSTs exist.

Directed MST (Minimum Arborescence)

For directed graphs, use Edmonds’ algorithm (Chu–Liu/Edmonds) in O(EV), or O(E log V) with advanced data structures.

Common CP Patterns

  • “Connect all nodes at minimum cost” → Direct MST
  • “Maximum spanning tree” → Negate weights, run MST, negate back
  • “MST + path queries” → Build MST, use LCA for path-max/min
  • “Kruskal’s reconstruction tree” → Internal node = merging edge; enables offline min-bottleneck path queries
  • “Minimum bottleneck spanning tree” → MST itself minimises the heaviest edge (minimax path property)
  • “Steiner tree” → NP-hard generally; small k terminals: DP on subsets O(3^k · n + 2^k · n²)
