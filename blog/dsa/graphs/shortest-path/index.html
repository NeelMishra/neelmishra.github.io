    Shortest Path: BFS & 0-1 BFS

    July 2026 · 20 min read

    Finding the shortest path between two vertices is one of the most fundamental graph problems. In this post we focus on two elegant O(V+E) algorithms: BFS for unweighted graphs and 0-1 BFS for graphs where every edge weight is either 0 or 1. Each technique gets a detailed visual walkthrough, an interactive animation, clean C++ code, and a complexity analysis.

    Why Shortest Path Matters

    Shortest path problems appear everywhere in computing:

    • Navigation: GPS systems find the shortest route between locations.
    • Networking: Routers find the fewest hops to deliver a packet.
    • Social graphs: “Degrees of separation” between users.
    • Game AI: Characters pathfind through grids and tile maps.
    • Competitive programming: Dozens of problems reduce to BFS or 0-1 BFS.
    Algorithm landscape: For unweighted graphs, BFS gives shortest paths in O(V+E). For edges with weights 0 or 1, 0-1 BFS achieves the same complexity using a deque. For arbitrary non-negative weights, Dijkstra runs in O((V+E) log V). This post covers the first two.

    BFS for Unweighted Shortest Path

    In an unweighted graph every edge has the same cost (1). BFS explores nodes layer by layer — first all nodes at distance 0 (the source), then distance 1, then distance 2, and so on. This guarantees that the first time BFS reaches a node, it has found the shortest path.

    function bfsShortestPath(source, adj, n):
    dist[] = {∞, ∞, ..., ∞}
    dist[source] = 0
    queue = [source]
    while queue is not empty:
    u = queue.dequeue()
    for v in adj[u]:
      if dist[v] == ∞:
        dist[v] = dist[u] + 1
        queue.enqueue(v)
    return dist

    Multi-Source BFS

    Sometimes we need the shortest distance from any of several sources. Instead of running BFS from each source, we enqueue all sources at once with dist = 0. The BFS wavefront expands from all sources simultaneously.

    Classic example: Given a grid with multiple fire sources, find the distance from each cell to the nearest fire. Enqueue all fire cells with dist = 0, then run standard BFS.

    0-1 BFS: Shortest Path with Weights 0 and 1

    When edge weights are restricted to 0 and 1, we can still find shortest paths in O(V+E) — no need for Dijkstra’s O((V+E) log V). The trick: use a deque (double-ended queue) instead of a regular queue or priority queue.

    Key idea: When relaxing an edge with weight 0, push the neighbor to the front of the deque (same distance as current node). For weight 1, push to the back. This maintains the deque in non-decreasing distance order.
    function zeroOneBFS(source, adj, n):
    dist[] = {∞, ∞, ..., ∞}
    dist[source] = 0
    deque = [source]
    while deque is not empty:
    u = deque.popFront()
    for (v, w) in adj[u]:
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
        if w == 0: deque.pushFront(v)
        else: deque.pushBack(v)
    return dist

    BFS on Grids

    Many competitive programming problems model the world as a 2D grid. Each cell is a node and edges connect adjacent cells (4- or 8-directional). BFS on grids is identical to BFS on graphs:

    1. Enqueue the starting cell with distance 0.
    2. For each dequeued cell, try all 4 (or 8) neighbors.
    3. If a neighbor is in-bounds, not a wall, and unvisited, set its distance and enqueue it.
    0-1 BFS on grids: Some grid problems have two types of movement — e.g., moving to an empty cell costs 0, but breaking a wall costs 1. This maps directly to 0-1 BFS: weight-0 for free moves, weight-1 for wall-breaking.

    Common grid patterns:

    • Shortest path in maze — standard BFS
    • Nearest distance to special cell — multi-source BFS
    • Minimum wall removals — 0-1 BFS
    • Minimum cost to reach exit (with free/paid tiles) — 0-1 BFS

    C++ Implementations

    BFS Shortest Path (Unweighted)

    vector<int> bfs(int src, const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<int> dist(n, INT_MAX);
    queue<int> q;
    dist[src] = 0;
    q.push(src);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == INT_MAX) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return dist;
}

    Multi-Source BFS

    vector<int> multiSourceBFS(const vector<int>& sources,
                            const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<int> dist(n, INT_MAX);
    queue<int> q;
    for (int s : sources) {
        dist[s] = 0;
        q.push(s);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == INT_MAX) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return dist;
}

    0-1 BFS

    vector<int> zeroOneBFS(int src,
    const vector<vector<pair<int,int>>>& adj) {
    int n = adj.size();
    vector<int> dist(n, INT_MAX);
    deque<int> dq;
    dist[src] = 0;
    dq.push_back(src);
    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (w == 0) dq.push_front(v);
                else        dq.push_back(v);
            }
        }
    }
    return dist;
}

    Grid BFS (4-directional)

    vector<vector<int>> gridBFS(int sr, int sc,
    const vector<string>& grid) {
    int R = grid.size(), C = grid[0].size();
    vector<vector<int>> dist(R, vector<int>(C, INT_MAX));
    int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
    queue<pair<int,int>> q;
    dist[sr][sc] = 0;
    q.push({sr, sc});
    while (!q.empty()) {
        auto [r, c] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nr = r+dx[d], nc = c+dy[d];
            if (nr>=0 && nr<R && nc>=0 && nc<C
                && grid[nr][nc]!='#'
                && dist[nr][nc]==INT_MAX) {
                dist[nr][nc] = dist[r][c] + 1;
                q.push({nr, nc});
            }
        }
    }
    return dist;
}

    Complexity Analysis

    BFS (Unweighted)

    Time: O(V + E) — each vertex enqueued once, each edge examined once.

    Space: O(V) for the distance array and queue.

    Multi-Source BFS

    Time: O(V + E) — identical to single-source; we just start with multiple sources.

    Space: O(V) for dist and queue.

    0-1 BFS

    Time: O(V + E) — each vertex pushed at most twice (via 0-edge and 1-edge), processed once.

    Space: O(V) for dist and deque.

    vs. Dijkstra

    Dijkstra: O((V+E) log V) with a heap. 0-1 BFS is faster when all weights are 0 or 1 — the deque replaces the heap.

    Why O(V+E)? In BFS, each node enters the queue exactly once. In 0-1 BFS, a node may be re-pushed if a shorter path is found, but since weights are 0 or 1 the total pushes are bounded by O(V+E).

    Summary

    • BFS computes shortest paths in unweighted graphs in O(V+E) by exploring layer-by-layer.
    • Multi-source BFS finds the nearest source for every node by starting from all sources simultaneously.
    • 0-1 BFS handles weight-0 and weight-1 edges in O(V+E) using a deque: push front for weight 0, push back for weight 1.
    • On grids, each cell is a node and adjacency is defined by 4 or 8 directions. All three variants apply directly.
    • For arbitrary non-negative weights, graduate to Dijkstra’s algorithm (covered in the next post).
