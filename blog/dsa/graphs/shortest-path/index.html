    Shortest Path: BFS & 0-1 BFS

    July 2026 · 25 min read

    Finding the shortest path between two nodes is one of the most fundamental problems in graph theory and competitive programming. While Dijkstra’s algorithm handles general weighted graphs, many problems involve unweighted or 0/1-weighted edges where simpler, faster algorithms apply. In this post we cover BFS for unweighted shortest paths, multi-source BFS, and the elegant 0-1 BFS technique — each with visual walkthroughs, interactive animations, clean C++ code, and competitive programming patterns.

    Introduction

    The “shortest path” problem comes in several flavors, each suited to a different algorithm:

    • Unweighted graphs: Every edge has cost 1. Plain BFS gives shortest paths in O(V + E).
    • 0/1-weighted graphs: Edges cost either 0 or 1. A modified BFS using a deque (0-1 BFS) runs in O(V + E).
    • Non-negative weights: Dijkstra’s algorithm runs in O((V + E) log V) with a priority queue.
    • Negative weights (no negative cycles): Bellman-Ford in O(VE), or SPFA as a practical optimization.
    • All-pairs: Floyd-Warshall in O(V³).
    Key insight: In competitive programming, most shortest-path problems on unweighted or 0/1-weighted graphs should be solved with BFS or 0-1 BFS — not Dijkstra. Using Dijkstra where BFS suffices adds an unnecessary log factor and more complex code.

    This post focuses on the first two categories: standard BFS and 0-1 BFS. We will also cover multi-source BFS, grid BFS, and BFS with augmented state spaces — patterns that appear in hundreds of competitive programming problems.

    BFS for Unweighted Graphs

    In an unweighted graph (or equivalently, all edges have weight 1), BFS from a source s naturally discovers nodes in order of their distance from s. The first time BFS reaches a node, it has found the shortest path to that node.

    Why BFS Gives Shortest Paths

    BFS explores nodes layer by layer. All nodes at distance d are processed before any node at distance d + 1. Since the queue is FIFO, and every edge adds exactly 1 to the distance, BFS guarantees:

    Theorem: When BFS first visits a vertex v, the path used has length equal to the shortest-path distance from the source. Formally, dist[v] = δ(s, v) for all reachable vertices v.

    Proof sketch: By induction on distance d. Base case: dist[s] = 0 = δ(s, s). Inductive step: assume all vertices at distance ≤ d have correct dist values. A vertex v at distance d + 1 must have a neighbor u at distance d. Since BFS processes u first (by induction), it will enqueue v with dist[v] = dist[u] + 1 = d + 1 = δ(s, v). Any earlier discovery of v would require a shorter path, contradicting δ(s, v) = d + 1.

    Step-by-Step BFS Example

    Let’s trace BFS from node 0 on this 8-node undirected graph, tracking distances as we go:

    Initialize: enqueue source node 0

    Set dist[0] = 0. All other distances are ∞. Queue = [0].

    Process node 0: discover neighbors 1, 2

    Set dist[1] = 1, dist[2] = 1. Queue = [1, 2]. Layer 1 discovered.

    Process layer 1: nodes 1 and 2 discover 3, 4, 5

    Node 1 discovers 3 (d=2) and 4 (d=2). Node 2 discovers 3 (already queued). Node 3 discovers 5 (d=2). Queue = [3, 4, 5]. Layer 2 complete.

    Process layer 2: discover 6 and 7

    Node 4 discovers 6 (d=3) and 7 (d=3). All nodes reached!

    Final distances: 0→0, 1→1, 2→1, 3→2, 4→2, 5→2, 6→3, 7→3

    Multi-Source BFS

    Sometimes we need shortest distances from multiple starting nodes simultaneously. Instead of running BFS from each source separately (which would be O(k · (V + E))), we enqueue all sources at the start with distance 0. The BFS then expands outward from all sources in parallel, like a wavefront.

    Classic analogy: Think of fire spreading from multiple ignition points on a grid. Each fire source starts at time 0, and the fire expands one cell per time step. Multi-source BFS computes when the fire reaches each cell.

    Fire Spreading Example

    Two fire sources (nodes A and D) spread simultaneously through a 6-node graph. Each node’s label shows the time the fire arrives:

    Initialize: both sources at t = 0

    Enqueue A and D with dist = 0. Queue = [A, D].

    t = 1: fire expands from both sources

    From A: discover B (t=1), C (t=1). From D: discover E (t=1), F (t=1).

    All nodes reached! The wavefronts “meet in the middle.”

    Multi-source BFS is equivalent to adding a virtual super-source node connected to all real sources with zero-weight edges, then running standard BFS from that super-source.

    0-1 BFS

    When edges have weights of either 0 or 1, we can modify BFS to handle these weights without a priority queue. The key idea: use a deque (double-ended queue) instead of a regular queue.

    • When traversing a 0-weight edge: push the neighbor to the front of the deque (it’s at the same distance).
    • When traversing a 1-weight edge: push the neighbor to the back of the deque (it’s one step farther).
    Why it works: The deque maintains the invariant that elements are sorted by distance. Zero-cost neighbors are at the same distance as the current node, so they go to the front. One-cost neighbors are one step farther, so they go to the back. This mimics Dijkstra’s behavior without the O(log n) overhead of a priority queue.

    0-1 BFS Walkthrough

    Consider this graph where dashed edges have weight 0 and solid edges have weight 1. We find shortest distances from node 0:

    Initialize: deque = [0], dist[0] = 0

    Edges: 0⟶1 (w=1), 0⟶2 (w=0), 1⟶3 (w=1), 2⟶3 (w=1), 2⟶4 (w=0), 3⟶5 (w=1), 4⟶5 (w=1).

    Deque: [0]

    Pop 0: push 2 to front (w=0), push 1 to back (w=1)

    dist[2] = 0 + 0 = 0. dist[1] = 0 + 1 = 1. Node 2 goes to front because it costs 0!

    Deque: [2, 1] ← 2 at front (w=0), 1 at back (w=1)

    Pop 2 (from front): push 4 to front (w=0), push 3 to back (w=1)

    dist[4] = 0 + 0 = 0. dist[3] = 0 + 1 = 1. Node 4 also at distance 0!

    Deque: [4, 1, 3] ← 4 at front, 3 at back

    Pop 4: push 5 to back (w=1). Then pop 1, 3, discover 5 already reached.

    dist[5] = 0 + 1 = 1. Done! Nodes {0, 2, 4} form a “zero-cost cluster” at distance 0.

    Final: dist = [0, 1, 0, 1, 0, 1]. Zero-weight edges create “free” clusters.

    Comparison: BFS vs 0-1 BFS vs Dijkstra

    Algorithm Edge Weights Data Structure Complexity When to Use
    BFS All = 1 (unweighted) Queue O(V + E) Grids, mazes, social networks
    0-1 BFS 0 or 1 only Deque O(V + E) Road/rail toggling, flip problems
    Dijkstra ≥ 0 (any non-negative) Priority Queue O((V+E) log V) Weighted roads, network routing
    Bellman-Ford Any (inc. negative) Array O(VE) Negative weights, arbitrage
    CP tip: If you see a problem where edges are unweighted or have weights 0/1, immediately think BFS or 0-1 BFS. Using Dijkstra is overkill and might TLE on tight constraints (e.g., V = 106).

    BFS on Grids

    Many competitive programming problems model the world as a 2D grid. Each cell is a node, and adjacent cells (up/down/left/right) are connected by edges. BFS on grids is the bread and butter of contest graph problems.

    4-Directional vs 8-Directional

    4-Directional

    Moves: up, down, left, right. Each cell has at most 4 neighbors.

    int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

    Used in: maze problems, flood fill, shortest path on grid.

    8-Directional

    Also includes diagonals. Each cell has at most 8 neighbors.

    int dx[] = {-1,-1,-1,0,0,1,1,1};
int dy[] = {-1,0,1,-1,1,-1,0,1};

    Used in: chess king movement, some pathfinding problems.

    Handling Obstacles

    Most grid BFS problems have blocked cells. The pattern is simple: before enqueuing a neighbor, check that it’s (1) within bounds and (2) not an obstacle:

    for (int d = 0; d < 4; d++) {
    int nx = x + dx[d], ny = y + dy[d];
    if (nx >= 0 && nx < R && ny >= 0 && ny < C
        && grid[nx][ny] != '#' && dist[nx][ny] == -1) {
        dist[nx][ny] = dist[x][y] + 1;
        q.push({nx, ny});
    }
}
    Grid BFS tip: Always use dist[nx][ny] == -1 (or ∞) as the visited check rather than a separate boolean array. This saves memory and is simpler. Initialize dist to -1, set dist[start] = 0, and you’re good.

    BFS with State Space

    Some problems require augmenting the BFS state beyond just position. The key idea: the “node” in your BFS isn’t just a vertex — it’s the full state you need to track.

    Bitmask BFS

    When you need to collect items (keys, coins, etc.) and the number of items is small (≤ 20), represent the collected set as a bitmask. The BFS state becomes (position, bitmask).

    // State: (node, collected_keys_bitmask)
// Total states: V * 2^K where K = number of keys
int dist[MAXN][1 << MAXK];
memset(dist, -1, sizeof(dist));
dist[start][0] = 0;
queue<pair<int,int>> q;
q.push({start, 0});

while (!q.empty()) {
    auto [u, mask] = q.front(); q.pop();
    if (mask == (1 << K) - 1) {
        cout << dist[u][mask]; // found all keys
        return;
    }
    for (auto [v, key_at_v] : adj[u]) {
        int nmask = mask | key_at_v;
        if (dist[v][nmask] == -1) {
            dist[v][nmask] = dist[u][mask] + 1;
            q.push({v, nmask});
        }
    }
}

    BFS with Keys and Doors

    A classic pattern: a grid has keys (a-f) and locked doors (A-F). Key ‘a’ opens door ‘A’. You can only pass through a door if you have the corresponding key. BFS state = (row, col, keys_bitmask).

    State space explosion: With K keys, the state space is R × C × 2K. For a 30×30 grid with 6 keys, that’s 30 × 30 × 64 = 57,600 states — very manageable. But with 20 keys on a 100×100 grid, you get 1010 states, which won’t fit in memory.

    C++ Implementations

    Clean, competitive-programming-ready code for each algorithm variant.

    1. BFS Shortest Path

    #include <bits/stdc++.h>
using namespace std;

vector<int> bfs(int src, const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<int> dist(n, -1);
    queue<int> q;
    dist[src] = 0;
    q.push(src);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return dist;
}

int main() {
    int n = 8, m = 9;
    vector<vector<int>> adj(n);
    vector<pair<int,int>> edges = {
        {0,1},{0,2},{1,3},{1,4},{2,3},{3,5},{4,6},{5,6},{4,7}
    };
    for (auto [u, v] : edges) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
    auto dist = bfs(0, adj);
    for (int i = 0; i < n; i++)
        cout << "dist[" << i << "] = " << dist[i] << "\n";
}

    2. Multi-Source BFS

    #include <bits/stdc++.h>
using namespace std;

vector<int> multiBFS(const vector<int>& sources,
                      const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<int> dist(n, -1);
    queue<int> q;
    for (int s : sources) {
        dist[s] = 0;
        q.push(s);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return dist;
}

int main() {
    int R = 5, C = 5;
    vector<string> grid(R);
    vector<int> sources;
    vector<vector<int>> adj(R * C);
    int dx[] = {-1,1,0,0}, dy[] = {0,0,-1,1};
    for (int i = 0; i < R; i++) {
        cin >> grid[i];
        for (int j = 0; j < C; j++) {
            if (grid[i][j] == 'F')
                sources.push_back(i * C + j);
            for (int d = 0; d < 4; d++) {
                int ni = i + dx[d], nj = j + dy[d];
                if (ni >= 0 && ni < R && nj >= 0 && nj < C
                    && grid[ni][nj] != '#')
                    adj[i*C+j].push_back(ni*C+nj);
            }
        }
    }
    auto dist = multiBFS(sources, adj);
    for (int i = 0; i < R; i++) {
        for (int j = 0; j < C; j++)
            cout << dist[i*C+j] << " ";
        cout << "\n";
    }
}

    3. 0-1 BFS

    #include <bits/stdc++.h>
using namespace std;

vector<int> bfs01(int src,
    const vector<vector<pair<int,int>>>& adj) {
    int n = adj.size();
    vector<int> dist(n, INT_MAX);
    deque<int> dq;
    dist[src] = 0;
    dq.push_back(src);
    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (w == 0)
                    dq.push_front(v);
                else
                    dq.push_back(v);
            }
        }
    }
    return dist;
}

int main() {
    int n = 7;
    vector<vector<pair<int,int>>> adj(n);
    adj[0].push_back({1, 1}); adj[0].push_back({2, 0});
    adj[1].push_back({3, 1});
    adj[2].push_back({3, 0}); adj[2].push_back({4, 1});
    adj[3].push_back({5, 1});
    adj[4].push_back({5, 0}); adj[4].push_back({6, 1});
    adj[5].push_back({6, 1});

    auto dist = bfs01(0, adj);
    for (int i = 0; i < n; i++)
        cout << "dist[" << i << "] = " << dist[i] << "\n";
    // Output: 0 1 0 0 1 1 2
}

    Complexity Analysis

    All three BFS variants achieve O(V + E) time complexity:

    • Standard BFS: Each vertex is enqueued and dequeued exactly once. Each edge is examined once (twice for undirected). Total: O(V + E).
    • Multi-source BFS: Same as standard BFS. The multiple sources just change the initial queue contents, not the asymptotic behavior. Total: O(V + E).
    • 0-1 BFS: Each vertex can be added to the deque at most twice (once to front, once to back), but is processed at most once (due to the dist check). Total: O(V + E).
    Space complexity: All variants use O(V) for the distance array and O(V) for the queue/deque. The adjacency list takes O(V + E). Total space: O(V + E).

    Comparison with Dijkstra

    Dijkstra’s with a binary heap runs in O((V + E) log V). For dense graphs where E = O(V²), this becomes O(V² log V), while BFS remains O(V²). For sparse graphs with V = 106, the log factor can mean the difference between passing and TLE in competitive programming.

    Competitive Programming Patterns

    Pattern 1: Grid + BFS = Shortest Path

    Given a grid with obstacles, find shortest path from start to end. This is the most common BFS pattern in CP. Model cells as nodes, adjacent passable cells as edges.

    Pattern 2: Complement Graph BFS

    Problem: given a dense graph (described by which edges are missing), find shortest paths. Maintain a set of unvisited nodes. For each BFS level, iterate over unvisited nodes and check if they are not neighbors of the current node (meaning they are connected in the complement). This runs in O(V + E) where E is the number of missing edges.

    // Complement graph BFS in O(V + M) where M = missing edges
set<int> unvisited;
for (int i = 0; i < n; i++) if (i != src) unvisited.insert(i);
queue<int> q;
q.push(src); dist[src] = 0;
while (!q.empty()) {
    int u = q.front(); q.pop();
    set<int> next_unvisited;
    for (int v : unvisited) {
        if (!adj_missing[u].count(v)) {
            dist[v] = dist[u] + 1;
            q.push(v);
        } else {
            next_unvisited.insert(v);
        }
    }
    unvisited = next_unvisited;
}

    Pattern 3: Bidirectional BFS

    When searching for shortest path between two specific nodes in a large graph, start BFS from both ends simultaneously. When the two frontiers meet, you’ve found the shortest path. This reduces the search space from O(bd) to O(bd/2) where b is the branching factor and d is the distance.

    Pattern 4: 0-1 BFS on Grids

    Common in problems where some movements are “free” (weight 0) and others cost 1. Examples: walking on roads (cost 0) vs building a new road through forest (cost 1). Model as a 0-1 BFS on the grid.

    Pattern 5: Level-by-Level BFS

    When you need to process BFS one layer at a time (e.g., “what is the set of nodes at distance exactly k?”), track the queue size at each level:

    while (!q.empty()) {
    int sz = q.size();  // nodes at current distance
    for (int i = 0; i < sz; i++) {
        int u = q.front(); q.pop();
        // process u at distance 'level'
        for (int v : adj[u]) { /* enqueue */ }
    }
    level++;
}

    Summary

    Let’s recap the key takeaways:

    1. BFS gives shortest paths on unweighted graphs in O(V + E). It explores layer-by-layer, guaranteeing optimality.
    2. Multi-source BFS handles multiple starting points by enqueuing all sources initially. Same complexity, same guarantee.
    3. 0-1 BFS extends BFS to 0/1-weighted graphs using a deque: push to front for weight 0, push to back for weight 1. Still O(V + E).
    4. State-space BFS (bitmask BFS, keys/doors) augments the node with extra state. The BFS itself doesn’t change — only the definition of “node” does.
    5. Grid BFS is the most common application in competitive programming. Master the 4-directional template and the obstacle check.
    6. Use BFS/0-1 BFS instead of Dijkstra whenever edge weights allow. The log factor matters on large inputs.
    Practice tip: Solve at least 15–20 BFS problems before a contest. Focus on recognizing which variant applies: standard BFS, multi-source, 0-1 BFS, or state-space BFS. The algorithm is simple — the hard part is modeling the problem correctly.
