Bipartite Graphs & Matching

Two-colorable graphs, maximum matchings, and the elegant theorems that connect them.

What Is a Bipartite Graph?

A graph is bipartite if its vertices can be divided into two disjoint sets A and B such that every edge connects a vertex in A to a vertex in B. Equivalently, the graph is 2-colorable — we can assign one of two colors to each vertex so that no two adjacent vertices share a color.

Key theorem: A graph is bipartite if and only if it contains no odd-length cycle. This gives us both a structural characterization and a practical test: if a BFS/DFS coloring succeeds, the graph is bipartite; if we find a conflict, we’ve found an odd cycle.

Bipartite graphs appear everywhere:

  • Job assignment — workers on one side, tasks on the other
  • Stable matching — applicants vs. positions
  • Network flows — many max-flow problems reduce to bipartite matching
  • Competitive programming — coloring, matching, and cover problems

Checking Bipartiteness (BFS 2-Coloring)

The standard approach is BFS 2-coloring: pick an uncolored vertex, color it 0, and propagate. For every neighbor, if it’s uncolored assign the opposite color; if it’s already the same color, the graph is not bipartite.

BFS Bipartite Check
color[start] = 0
queue.push(start)

while queue is not empty:
    u = queue.front(); queue.pop()
    for each neighbor v of u:
        if color[v] == -1:
            color[v] = 1 - color[u]
            queue.push(v)
        else if color[v] == color[u]:
            return NOT BIPARTITE

return BIPARTITE

Maximum Bipartite Matching

A matching in a bipartite graph is a set of edges with no shared endpoints. A maximum matching is one with the largest possible number of edges. The standard approach for unweighted bipartite matching is based on augmenting paths.

Augmenting path: A path that starts at an unmatched vertex in A, alternates between unmatched and matched edges, and ends at an unmatched vertex in B. Flipping the matched/unmatched status of every edge on this path increases the matching size by 1.

Current matching: {b–x, c–z} (size 2). Augmenting path: a → x → b → y. After flipping: matching becomes {a–x, b–y, c–z} (size 3 — perfect matching!).

Kuhn’s Algorithm (Hungarian Method for Unweighted)

For each unmatched vertex in A, run a DFS to find an augmenting path. If found, flip the path. This is the simplest maximum bipartite matching algorithm, running in O(V · E).

Hopcroft-Karp Algorithm

Kuhn’s algorithm finds one augmenting path per iteration. Hopcroft-Karp improves this by finding a maximal set of shortest augmenting paths in each phase using BFS, then augmenting all of them with DFS.

Hopcroft-Karp
while BFS finds shortest augmenting paths:
    for each free vertex u in A:
        DFS(u) to augment along shortest path

// BFS builds layers by distance from free vertices in A
// DFS augments along layer edges only

Each BFS phase takes O(E) and increases the shortest augmenting path length. Since the shortest augmenting path length can increase at most O(√V) times, the total complexity is O(E · √V). This is the go-to algorithm for large bipartite matching problems in competitive programming.

Kuhn’s Algorithm

O(V · E). Simple DFS-based. Good for small graphs (≤ 1000 vertices). Easy to implement.

Hopcroft-Karp

O(E · √V). BFS + DFS phases. Required for large graphs (≤ 105 vertices). Slightly more complex.

Hungarian Algorithm (Weighted Matching)

When edges have weights and we want a maximum (or minimum) weight perfect matching, we use the Hungarian algorithm. It works on a cost matrix and uses potential functions to find the optimal assignment.

Hungarian Algorithm Outline
1. Initialize potentials u[i] = 0, v[j] = 0
2. For each row i:
   a. Find augmenting path using reduced costs
      reduced_cost[i][j] = cost[i][j] - u[i] - v[j]
   b. Update potentials to maintain feasibility
   c. Augment along the path found
3. Total cost = sum of u[i] + sum of v[j]

The Hungarian algorithm runs in O(V³) for a V×V cost matrix. It solves the assignment problem: given n workers and n jobs with a cost matrix, find the minimum-cost one-to-one assignment.

CP tip: For maximum weight matching, negate all weights and run the Hungarian algorithm for minimum weight.

König’s Theorem

One of the most beautiful results in graph theory:

König’s Theorem: In a bipartite graph, the size of the maximum matching equals the size of the minimum vertex cover.

A vertex cover is a set of vertices such that every edge has at least one endpoint in the set. The minimum vertex cover is the smallest such set.

This theorem gives us powerful equivalences in bipartite graphs:

  • Maximum matching = Minimum vertex cover (König)
  • Minimum vertex cover + Maximum independent set = V
  • So: Maximum independent set = V − Maximum matching
  • By LP duality: Minimum edge cover = V − Maximum matching (when no isolated vertices)

How to find the minimum vertex cover from a maximum matching:

  1. Find a maximum matching M.
  2. Let U be the set of unmatched vertices in A.
  3. Run alternating BFS/DFS from U (alternate matched/unmatched edges). Let Z be all reachable vertices.
  4. Minimum vertex cover = (A ∖ Z) ∪ (B ∩ Z).

C++ Implementation

Bipartite Check (BFS)

#include <bits/stdc++.h>
using namespace std;

bool isBipartite(int n, const vector<vector<int>>& adj) {
    vector<int> color(n, -1);
    for (int start = 0; start < n; start++) {
        if (color[start] != -1) continue;
        queue<int> q;
        color[start] = 0;
        q.push(start);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : adj[u]) {
                if (color[v] == -1) {
                    color[v] = 1 - color[u];
                    q.push(v);
                } else if (color[v] == color[u]) {
                    return false;
                }
            }
        }
    }
    return true;
}

Kuhn’s Algorithm (Maximum Bipartite Matching)

// Left vertices: 0..n-1, Right vertices: 0..m-1
// adj[u] = list of right vertices connected to left vertex u

vector<vector<int>> adj;
vector<int> matchL, matchR;
vector<bool> used;

bool dfs(int u) {
    for (int v : adj[u]) {
        if (!used[v]) {
            used[v] = true;
            if (matchR[v] == -1 || dfs(matchR[v])) {
                matchL[u] = v;
                matchR[v] = u;
                return true;
            }
        }
    }
    return false;
}

int maxMatching(int n, int m) {
    matchL.assign(n, -1);
    matchR.assign(m, -1);
    int result = 0;
    for (int u = 0; u < n; u++) {
        used.assign(m, false);
        if (dfs(u)) result++;
    }
    return result;
}

Hopcroft-Karp Algorithm

#include <bits/stdc++.h>
using namespace std;

struct HopcroftKarp {
    int n, m;
    vector<vector<int>> adj;
    vector<int> matchL, matchR, dist;

    HopcroftKarp(int n, int m) : n(n), m(m), adj(n) {}

    void addEdge(int u, int v) { adj[u].push_back(v); }

    bool bfs() {
        queue<int> q;
        for (int u = 0; u < n; u++) {
            if (matchL[u] == -1) {
                dist[u] = 0;
                q.push(u);
            } else {
                dist[u] = INT_MAX;
            }
        }
        bool found = false;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : adj[u]) {
                int w = matchR[v];
                if (w == -1) {
                    found = true;
                } else if (dist[w] == INT_MAX) {
                    dist[w] = dist[u] + 1;
                    q.push(w);
                }
            }
        }
        return found;
    }

    bool dfs(int u) {
        for (int v : adj[u]) {
            int w = matchR[v];
            if (w == -1 || (dist[w] == dist[u] + 1 && dfs(w))) {
                matchL[u] = v;
                matchR[v] = u;
                return true;
            }
        }
        dist[u] = INT_MAX;
        return false;
    }

    int solve() {
        matchL.assign(n, -1);
        matchR.assign(m, -1);
        dist.resize(n);
        int ans = 0;
        while (bfs()) {
            for (int u = 0; u < n; u++) {
                if (matchL[u] == -1 && dfs(u)) ans++;
            }
        }
        return ans;
    }
};

Complexity Analysis

AlgorithmTimeSpaceBest For
BFS 2-ColoringO(V + E)O(V)Bipartite check
Kuhn’s (DFS augmenting)O(V · E)O(V + E)Small matching (≤ 1000)
Hopcroft-KarpO(E · √V)O(V + E)Large matching (≤ 105)
HungarianO(V³)O(V²)Weighted assignment

Bipartite check is linear — identical to BFS. Kuhn’s is simple but can be slow for large inputs. Hopcroft-Karp is the standard for competitive programming. The Hungarian algorithm handles weighted cases in cubic time.

Summary

  • A graph is bipartite iff it has no odd cycle. Check via BFS 2-coloring in O(V + E).
  • Maximum matching in bipartite graphs: use Kuhn’s O(VE) for small inputs, Hopcroft-Karp O(E√V) for large.
  • Weighted matching (assignment problem): Hungarian algorithm in O(V³).
  • König’s theorem: max matching = min vertex cover. Gives maximum independent set = V − max matching.
  • Bipartite matching is a foundation for network flow, assignment, and scheduling problems in CP.
