    Network Flow: From Ford-Fulkerson to Dinic’s

    April 2026 · 30 min read

    Network flow models how much “stuff” (water, data, traffic) can move from a source to a sink through a network with limited-capacity edges. The maximum flow problem is one of the most beautiful in combinatorial optimization — elegant theory meets practical algorithms. This post covers the core concepts, three progressively better algorithms, and their implementations.

    What is Network Flow?

    A flow network is a directed graph where:

    • There is a source s (produces flow) and a sink t (absorbs flow)
    • Each edge (u, v) has a capacity c(u, v) ≥ 0
    • Flow on an edge cannot exceed its capacity

    A valid flow must satisfy three constraints:

    1. Capacity constraint: 0 ≤ f(u,v) ≤ c(u,v) for every edge
    2. Flow conservation: for every vertex except s and t, flow in = flow out
    3. Skew symmetry: f(u,v) = −f(v,u)

    The maximum flow problem: find a flow that maximizes the total flow from s to t.

    Real-world analogy: Think of water pipes. Each pipe has a maximum throughput (capacity). The source is a reservoir, the sink is a city. How much water can you deliver?

    The Residual Graph

    The residual graph is the key concept behind all flow algorithms. After sending some flow, the residual graph shows what adjustments are still possible:

    • Forward edge: remaining capacity = c(u,v) − f(u,v). You can push more flow.
    • Backward edge: capacity = f(u,v). You can “cancel” existing flow (reroute it).

    Original: edge with cap 10, flow 6

    Residual: forward 4, backward 6

    Why backward edges matter: They let algorithms “undo” bad choices. Without them, a greedy approach can get stuck at a non-optimal flow. Backward edges enable rerouting.

    Max-Flow Min-Cut Theorem

    One of the most elegant results in graph theory:

    Theorem: In any flow network, the maximum flow from s to t equals the minimum cut capacity separating s from t.

    A cut (S, T) partitions vertices into two sets: S containing s, T containing t. The cut capacity is the sum of capacities of edges crossing from S to T.

    Practical use: After finding max flow, BFS from s on the residual graph. Reachable vertices = S, unreachable = T. Saturated edges crossing S→T form the min cut.

    Ford-Fulkerson Method

    The foundation of all max-flow algorithms. The idea is beautifully simple:

    function FordFulkerson(graph, s, t):
    maxFlow = 0 while there exists an augmenting path P from s to t in the residual graph: bottleneck = min residual capacity along P augment flow along P by bottleneck maxFlow += bottleneck return maxFlow

    An augmenting path is any path from s to t in the residual graph where every edge has positive residual capacity.

    Warning: Ford-Fulkerson is a method, not an algorithm — it doesn’t specify how to find augmenting paths. With arbitrary paths (DFS), it may not terminate if capacities are irrational. With integer capacities, it runs in O(E × max_flow), which can be very slow.

    Edmonds-Karp Algorithm

    Edmonds-Karp = Ford-Fulkerson with BFS to find the shortest augmenting path (fewest edges). This single change guarantees O(VE²) time, regardless of capacity values.

    Why BFS is better: BFS finds paths with the fewest edges. It can be proven that the shortest augmenting path length never decreases, and each edge can be critical (bottleneck) at most O(V) times. This gives O(VE) augmentations, each costing O(E) for BFS = O(VE²).

    Step-by-Step: Edmonds-Karp Walkthrough

    Let’s trace Edmonds-Karp on our example network. BFS always finds the shortest path.

    Total flow: 4

    BFS finds path: s → 2 → 4 → t

    Dinic’s Algorithm

    Dinic’s improves on Edmonds-Karp by finding multiple augmenting paths per BFS phase.

    Two key concepts

    Level Graph

    BFS from s assigns levels. The level graph keeps only edges going from level i to level i+1. This ensures all paths are shortest.

    Blocking Flow

    Find a blocking flow in the level graph: a flow such that every s→t path has at least one saturated edge. Use DFS with a pointer optimization to avoid re-scanning dead-end edges.

    function Dinic(graph, s, t):
    maxFlow = 0 while BFS builds a level graph from s that reaches t: while blockingFlow = DFS from s to t in level graph: augment by blockingFlow maxFlow += blockingFlow return maxFlow

    Key insight: After each blocking flow, the distance from s to t in the residual graph strictly increases. Since the distance is at most V-1, there are at most V-1 BFS phases. Each blocking flow takes O(VE). Total: O(V²E).

    Special case: In unit-capacity graphs (all capacities = 1), Dinic’s runs in O(E√V). This makes it ideal for bipartite matching (equivalent to max flow on a unit-capacity network).

    C++ Implementation: Edmonds-Karp

    #include <bits/stdc++.h>
using namespace std;

struct Edge {
    int to, rev;    // destination and index of reverse edge
    int cap, flow;  // capacity and current flow
};

class MaxFlow {
    int n;
    vector<vector<Edge>> graph;

    // BFS to find augmenting path, returns parent edge info
    bool bfs(int s, int t, vector<int>& parent, vector<int>& parentEdge) {
        fill(parent.begin(), parent.end(), -1);
        parent[s] = s;
        queue<pair<int,int>> q;
        q.push({s, INT_MAX});

        while (!q.empty()) {
            auto [u, pushed] = q.front(); q.pop();
            for (int i = 0; i < graph[u].size(); i++) {
                Edge& e = graph[u][i];
                int residual = e.cap - e.flow;
                if (parent[e.to] == -1 && residual > 0) {
                    parent[e.to] = u;
                    parentEdge[e.to] = i;
                    if (e.to == t) return true;
                    q.push({e.to, min(pushed, residual)});
                }
            }
        }
        return false;
    }

public:
    MaxFlow(int n) : n(n), graph(n) {}

    void addEdge(int from, int to, int cap) {
        graph[from].push_back({to, (int)graph[to].size(), cap, 0});
        graph[to].push_back({from, (int)graph[from].size()-1, 0, 0});
    }

    int edmondsKarp(int s, int t) {
        int totalFlow = 0;
        vector<int> parent(n), parentEdge(n);

        while (bfs(s, t, parent, parentEdge)) {
            // Find bottleneck
            int bottleneck = INT_MAX;
            int v = t;
            while (v != s) {
                int u = parent[v];
                bottleneck = min(bottleneck,
                    graph[u][parentEdge[v]].cap - graph[u][parentEdge[v]].flow);
                v = u;
            }
            // Augment flow along path
            v = t;
            while (v != s) {
                int u = parent[v];
                graph[u][parentEdge[v]].flow += bottleneck;
                graph[v][graph[u][parentEdge[v]].rev].flow -= bottleneck;
                v = u;
            }
            totalFlow += bottleneck;
        }
        return totalFlow;
    }
};

int main() {
    MaxFlow mf(6); // s=0, 1, 2, 3, 4, t=5
    mf.addEdge(0,1,10); mf.addEdge(0,2,10);
    mf.addEdge(1,2,2);  mf.addEdge(1,3,4);
    mf.addEdge(1,4,8);  mf.addEdge(2,4,9);
    mf.addEdge(3,5,10); mf.addEdge(4,3,6);
    mf.addEdge(4,5,10);
    cout << "Max Flow: " << mf.edmondsKarp(0, 5) << endl;
    // Output: Max Flow: 19
}

    C++ Implementation: Dinic’s

    class Dinic {
    int n;
    vector<vector<Edge>> graph;
    vector<int> level, iter;

    bool bfs(int s, int t) {
        fill(level.begin(), level.end(), -1);
        level[s] = 0;
        queue<int> q;
        q.push(s);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (auto& e : graph[u]) {
                if (e.cap - e.flow > 0 && level[e.to] == -1) {
                    level[e.to] = level[u] + 1;
                    q.push(e.to);
                }
            }
        }
        return level[t] != -1;
    }

    int dfs(int u, int t, int pushed) {
        if (u == t || pushed == 0) return pushed;
        for (int& i = iter[u]; i < graph[u].size(); i++) {
            Edge& e = graph[u][i];
            if (level[e.to] != level[u] + 1) continue;
            int residual = e.cap - e.flow;
            int d = dfs(e.to, t, min(pushed, residual));
            if (d > 0) {
                e.flow += d;
                graph[e.to][e.rev].flow -= d;
                return d;
            }
        }
        return 0;
    }

public:
    Dinic(int n) : n(n), graph(n), level(n), iter(n) {}

    void addEdge(int from, int to, int cap) {
        graph[from].push_back({to, (int)graph[to].size(), cap, 0});
        graph[to].push_back({from, (int)graph[from].size()-1, 0, 0});
    }

    int maxflow(int s, int t) {
        int flow = 0;
        while (bfs(s, t)) {
            fill(iter.begin(), iter.end(), 0);
            int d;
            while ((d = dfs(s, t, INT_MAX)) > 0)
                flow += d;
        }
        return flow;
    }
};

    Finding the Minimum Cut

    After computing max flow, the min cut is free:

    1. Run BFS from s on the residual graph (edges with remaining capacity > 0)
    2. Reachable vertices = set S. Unreachable = set T.
    3. Edges from S to T in the original graph that are fully saturated = the min cut edges.
    vector<pair<int,int>> findMinCut(int s) {
    // After running max flow, BFS on residual
    vector<bool> reachable(n, false);
    queue<int> q;
    q.push(s); reachable[s] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto& e : graph[u]) {
            if (!reachable[e.to] && e.cap - e.flow > 0) {
                reachable[e.to] = true;
                q.push(e.to);
            }
        }
    }
    // Edges crossing the cut
    vector<pair<int,int>> cut;
    for (int u = 0; u < n; u++)
        if (reachable[u])
            for (auto& e : graph[u])
                if (!reachable[e.to] && e.cap > 0)
                    cut.push_back({u, e.to});
    return cut;
}

    Applications

    🔗 Bipartite Matching

    Maximum matching in a bipartite graph = max flow in a network with source connected to left nodes, right nodes connected to sink, all capacities 1. Use Dinic’s for O(E√V).

    ✂️ Image Segmentation

    Separate foreground from background: pixels are nodes, neighboring pixels have edges. Min cut = optimal boundary. Used in computer vision.

    ⚾ Baseball Elimination

    Can team X still win the season? Model remaining games as a flow network. If max flow saturates all game edges, X is eliminated.

    📋 Project Selection

    Choose projects with dependencies to maximize profit. Model as min-cut: source = take project, sink = skip project. Classic “closure” problem.

    Practice Problems

    Network flow rarely appears as a direct "find max flow" question. The real skill is recognizing that a problem reduces to flow and modeling it correctly. Here are curated problems from easiest to hardest, grouped by pattern.

    Pure Max Flow

    Medium

    CSES — Download Speed

    Classic max flow: given a network of computers with connection speeds, find the maximum download speed from server to client. Direct Edmonds-Karp / Dinic’s application — the best first problem to implement.

    Hard

    SPOJ — FASTFLOW

    Straightforward max flow on a large graph (up to 5000 nodes, 30000 edges). Tests your implementation speed — Dinic’s is essentially required. Good for stress-testing your template.

    Min Cut / Max-Flow Min-Cut Duality

    Medium

    CSES — Police Chase

    Find the minimum number of streets to block so a burglar cannot escape from junction 1 to junction n. Direct min-cut application: run max flow, then BFS on residual to find the cut edges.

    Hard

    Codeforces 653D — Delivery Bears

    Bears deliver honey through a network with weight limits. Binary search on the weight per bear, then max flow on edges that can support that weight. Combines binary search + max flow modeling.

    Bipartite Matching via Flow

    Medium

    CSES — School Dance

    Find the maximum number of boy-girl pairs for a school dance. Classic maximum bipartite matching — model as flow with source → boys → girls → sink, all capacity 1. Dinic’s runs in O(E√V).

    Hard

    CSES — Coin Grid

    Given a grid with coins, find the minimum number of rows and columns to clear so no coins remain. Reduces to minimum vertex cover in a bipartite graph, which by König’s theorem equals maximum matching (= max flow).

    Advanced Flow Modeling

    Hard

    CSES — Distinct Routes

    Find the maximum number of edge-disjoint paths from node 1 to node n, and output them. Max flow gives the count; reconstruct paths by tracing flow. Tests flow decomposition.

    Hard

    Project Selection (Closure Problem)

    Select a subset of projects with dependencies to maximize total profit. Model as min-cut: source-side = take, sink-side = skip. Assign capacities based on profits/costs. See this Codeforces blog on min-cut duality for worked examples.

    Hard

    Minimum Path Cover in DAG

    Find the minimum number of vertex-disjoint paths that cover all vertices. Equals n − max matching on the bipartite split of the DAG. Good exercise in reducing path-cover to flow. Practice on Library Checker — Bipartite Matching.

    More practice: The USACO Guide — Max Flow and USACO Guide — Min Cut pages maintain up-to-date problem lists with difficulty ratings. For competitive programming, CP-Algorithms: Max Flow has implementations and further problem links.

    Summary

    • Flow network: directed graph with source, sink, and edge capacities
    • Residual graph: shows remaining capacity + backward edges for rerouting
    • Max-flow = min-cut: the fundamental duality theorem
    • Ford-Fulkerson: the method — find augmenting path, push flow, repeat
    • Edmonds-Karp: BFS for shortest path → O(VE²), simple and reliable
    • Dinic’s: level graph + blocking flow → O(V²E), fast in practice
    • After max flow: BFS on residual gives you the min cut for free
    Decision tree: Simple problem? → Edmonds-Karp. Competitive programming? → Dinic’s. Bipartite matching? → Dinic’s (O(E√V)). Dense graph? → Push-Relabel.
