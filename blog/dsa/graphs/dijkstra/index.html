Dijkstra’s Algorithm

The gold standard for single-source shortest paths in non-negative weighted graphs.

Why Dijkstra’s Algorithm?

When edges carry non-negative weights, BFS no longer gives shortest paths. We need an algorithm that accounts for edge costs. Dijkstra’s algorithm solves the single-source shortest path problem by greedily extracting the nearest unvisited vertex and relaxing its neighbors.

Published by Edsger Dijkstra in 1959, it remains one of the most important graph algorithms. Applications include:

  • GPS navigation — finding the fastest route between two locations
  • Network routing — OSPF protocol uses Dijkstra to compute routing tables
  • Game AI — pathfinding on weighted terrain maps
  • Competitive programming — countless shortest-path problems

The Algorithm

Dijkstra maintains a distance array dist[] initialized to ∞ for all vertices except the source (which is 0). A priority queue (min-heap) always extracts the vertex with the smallest tentative distance.

Dijkstra’s Algorithm
dist[source] = 0
pq.push({0, source})

while pq is not empty:
    (d, u) = pq.extractMin()
    if d > dist[u]: continue   // stale entry
    for each edge (u, v, w):
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            pq.push({dist[v], v})

Key insight: When we extract vertex u from the priority queue, dist[u] is finalized. This works because all edge weights are ≥ 0, so no future path through unvisited vertices can improve it.

Why Non-Negative Weights?

If an edge has negative weight, a vertex we already “finalized” might later find a shorter path through that negative edge. Dijkstra’s greedy assumption breaks.

Step-by-Step Walkthrough

Let’s trace Dijkstra on a 7-node weighted graph starting from node 0.

Initialize

dist = [0, ∞, ∞, ∞, ∞, ∞, ∞]. Push (0, node 0) into the priority queue.

Extract node 0 (dist=0)

Relax neighbors: dist[1] = 0+4 = 4, dist[2] = 0+1 = 1.

dist = [0, 4, 1, ∞, ∞, ∞, ∞]

Finalized: node 0

Extract node 2 (dist=1)

Relax: dist[1] = min(4, 1+2) = 3 (improved!), dist[4] = 1+5 = 6.

dist = [0, 3, 1, ∞, 6, ∞, ∞]

Finalized: node 2

Extract node 1 (dist=3)

Relax: dist[3] = 3+1 = 4.

dist = [0, 3, 1, 4, 6, ∞, ∞]

Finalized: node 1

Extract node 3 (dist=4)

Relax: dist[4] = min(6, 4+3) = 6 (no change), dist[5] = 4+1 = 5.

dist = [0, 3, 1, 4, 6, 5, ∞]

Finalized: node 3

Extract node 5 (dist=5)

Relax: dist[6] = 5+4 = 9.

dist = [0, 3, 1, 4, 6, 5, 9]

Finalized: node 5

Extract node 4 (dist=6)

Relax: dist[6] = min(9, 6+2) = 8 (improved!).

dist = [0, 3, 1, 4, 6, 5, 8]

Finalized: node 4

Extract node 6 (dist=8)

No outgoing edges to relax. Done!

Final distances: [0, 3, 1, 4, 6, 5, 8]

Finalized: node 6 — All done!

Why Negative Weights Break Dijkstra

Consider this simple counterexample:

Dijkstra finalizes A with dist=1, then C with dist=3 (via S→A→C). But the true shortest path to C is S→B→C = 5 + (-4) = 1. Since A was already finalized, Dijkstra never reconsiders it. Use Bellman-Ford for graphs with negative edges.

Priority Queue Variants

Binary Heap (std::priority_queue)

Most common in CP. O((V+E) log V). Uses lazy deletion — if we pop a vertex with a stale distance, we skip it.

Fibonacci Heap

O(V log V + E) amortized. Theoretically faster for dense graphs, but complex to implement and rarely faster in practice due to constant factors.

Dial’s Algorithm

When edge weights are small integers ≤ C, use an array of C+1 buckets. O(V·C + E). Useful in specific CP problems.

Indexed Priority Queue

Supports decrease-key in O(log V). Avoids lazy deletion overhead. O((V+E) log V) but with better constants.

C++ Implementation

Standard Dijkstra with Priority Queue

#include <bits/stdc++.h>
using namespace std;

vector<long long> dijkstra(int src, const vector<vector<pair<int,int>>>& adj) {
    int n = adj.size();
    vector<long long> dist(n, LLONG_MAX);
    priority_queue<pair<long long,int>, vector<pair<long long,int>>,
                   greater<pair<long long,int>>> pq;

    dist[src] = 0;
    pq.push({0, src});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue; // stale entry

        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}

With Path Reconstruction

pair<vector<long long>, vector<int>> dijkstra_path(int src,
    const vector<vector<pair<int,int>>>& adj) {
    int n = adj.size();
    vector<long long> dist(n, LLONG_MAX);
    vector<int> parent(n, -1);
    priority_queue<pair<long long,int>, vector<pair<long long,int>>,
                   greater<>> pq;

    dist[src] = 0;
    pq.push({0, src});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                parent[v] = u;
                pq.push({dist[v], v});
            }
        }
    }
    return {dist, parent};
}

// Reconstruct path from src to dst
vector<int> getPath(int dst, const vector<int>& parent) {
    vector<int> path;
    for (int v = dst; v != -1; v = parent[v])
        path.push_back(v);
    reverse(path.begin(), path.end());
    return path;
}

For most competitive programming problems: V, E ≤ 105–106, so the binary heap version is fast enough.

Competitive Programming Patterns

1. Shortest Path on Grid

Treat each cell as a node. 4 neighbors are edges. If cells have costs, use Dijkstra instead of BFS. If costs are 0 or 1, prefer 0-1 BFS.

2. Multi-Source Dijkstra

Push all source nodes into the priority queue at distance 0. Finds the nearest source for every vertex. Used in problems like “distance to nearest facility.”

3. Modified Edge Weights

Some problems require transforming edge weights. Example: “maximize the minimum edge on a path” — use negative log weights, or modify the relaxation condition.

4. Dijkstra on States

Add extra dimensions: dist[node][state]. Example: “shortest path using at most K toll roads” — state = number of toll roads used.

5. Bidirectional Dijkstra

Run Dijkstra from both source and target simultaneously. Stop when the two searches meet. Approximately 2× faster in practice for point-to-point queries.

Summary

  • Dijkstra’s algorithm finds single-source shortest paths in O((V+E) log V) for non-negative weighted graphs.
  • Uses a priority queue to greedily extract the nearest unfinalized vertex.
  • Does not work with negative edge weights — use Bellman-Ford instead.
  • The “lazy deletion” approach with if (d > dist[u]) continue; is the standard CP idiom.
  • State-augmented Dijkstra dist[node][state] handles many advanced CP problems.
