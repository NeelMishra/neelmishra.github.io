Block-Cut Tree

Decomposing graphs into biconnected components for fast vertex-connectivity queries.

Introduction

An articulation point (or cut vertex) is a vertex whose removal increases the number of connected components. A biconnected component is a maximal subgraph that has no cut vertex — equivalently, every pair of edges in it lies on a common simple cycle.

The block-cut tree compresses an undirected graph into a tree by representing each biconnected component (called a block) and each cut vertex as nodes, with edges connecting a cut vertex to every block it belongs to. This transforms messy vertex-connectivity questions into simple tree queries.

Key definitions:

  • Cut vertex — removing it disconnects the graph (or increases component count)
  • Block — a maximal 2-connected subgraph, a bridge with its endpoints, or an isolated vertex
  • Bridge — an edge whose removal disconnects the graph; it forms a block by itself

Applications in competitive programming include:

  • Vertex connectivity queries — how many cut vertices lie on every path between two nodes
  • Component counting — how many components result from removing a vertex
  • Path queries — reduce graph path problems to tree path problems via LCA

Finding Biconnected Components

We use a modified DFS maintaining two arrays and an edge stack:

  • disc[u] — discovery time of vertex u
  • low[u] — smallest discovery time reachable from the subtree of u via back edges
  • An edge stack that accumulates edges as DFS progresses
Biconnected Components via DFS
dfs(u, parent):
    disc[u] = low[u] = timer++
    for each neighbor v of u:
        if v is unvisited:
            push edge (u,v) onto stack
            dfs(v, u)
            low[u] = min(low[u], low[v])
            if low[v] ≥ disc[u]:
                pop edges from stack until (u,v)
                → these edges form one biconnected component
        else if v ≠ parent and disc[v] < disc[u]:
            push edge (u,v) onto stack
            low[u] = min(low[u], disc[v])

Key insight: When low[v] ≥ disc[u], the subtree rooted at v cannot reach any ancestor of u without passing through u. So u is a cut vertex (unless it is the DFS root with only one child), and the edges popped form one block.

Cut Vertex Detection

A vertex u is a cut vertex if and only if:

  • u is the DFS root and has ≥ 2 DFS children, or
  • u is not the root and has some child v with low[v] ≥ disc[u]

Building the Block-Cut Tree

Once we have the biconnected components and cut vertices, the block-cut tree is straightforward to build:

  1. Create one block node for each biconnected component.
  2. Create one cut-vertex node for each cut vertex.
  3. Add an edge between a cut vertex and a block if that vertex belongs to the block.

Properties of the block-cut tree:

  • It is always a tree (or forest for disconnected graphs).
  • Every cut vertex has degree ≥ 2 in the tree (it connects at least two blocks).
  • Non-cut vertices appear inside exactly one block and are not separate tree nodes.
  • The path between any two blocks in the tree passes through exactly those cut vertices that separate them in the original graph.

Bridge Tree Variant

A closely related structure is the bridge tree (also called the block-cut tree for edges). Instead of vertex-biconnected components, it uses 2-edge-connected components:

Block-Cut Tree

Decomposes into vertex-biconnected components (blocks). Cut vertices connect blocks. Used for vertex connectivity queries.

Bridge Tree

Decomposes into 2-edge-connected components. Bridges connect components. Used for edge connectivity queries.

A 2-edge-connected component is a maximal subgraph where every pair of vertices has at least two edge-disjoint paths. To build the bridge tree: find all bridges, then contract each 2-edge-connected component into a single node. The bridges become the tree edges.

C++ Implementation

Finding Biconnected Components & Cut Vertices

#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int disc[MAXN], low[MAXN], timer_cnt;
bool is_cut[MAXN];
stack<pair<int,int>> stk;
vector<vector<pair<int,int>>> bcc; // biconnected components as edge lists

void dfs(int u, int par) {
    disc[u] = low[u] = timer_cnt++;
    int children = 0;
    for (int v : adj[u]) {
        if (disc[v] == -1) {
            children++;
            stk.push({u, v});
            dfs(v, u);
            low[u] = min(low[u], low[v]);
            // cut vertex check
            if ((par == -1 && children > 1) ||
                (par != -1 && low[v] >= disc[u]))
                is_cut[u] = true;
            // pop biconnected component
            if (low[v] >= disc[u]) {
                bcc.emplace_back();
                while (stk.top() != make_pair(u, v)) {
                    bcc.back().push_back(stk.top());
                    stk.pop();
                }
                bcc.back().push_back(stk.top());
                stk.pop();
            }
        } else if (v != par && disc[v] < disc[u]) {
            stk.push({u, v});
            low[u] = min(low[u], disc[v]);
        }
    }
}

int n, m;
void find_bcc() {
    memset(disc, -1, sizeof disc);
    memset(is_cut, false, sizeof is_cut);
    timer_cnt = 0;
    bcc.clear();
    for (int i = 0; i < n; i++)
        if (disc[i] == -1) dfs(i, -1);
}

Building the Block-Cut Tree

// After calling find_bcc():
// Block nodes are indexed n, n+1, ..., n+bcc.size()-1.
// Cut-vertex nodes keep their original index 0..n-1.
vector<int> tree[MAXN * 2];
int block_id[MAXN]; // for non-cut vertices: which block they belong to

void build_block_cut_tree() {
    // clear tree
    for (int i = 0; i < n + (int)bcc.size(); i++) tree[i].clear();
    memset(block_id, -1, sizeof block_id);

    for (int b = 0; b < (int)bcc.size(); b++) {
        int bnode = n + b;
        set<int> verts;
        for (auto& [u, v] : bcc[b]) {
            verts.insert(u);
            verts.insert(v);
        }
        for (int v : verts) {
            if (is_cut[v]) {
                tree[v].push_back(bnode);
                tree[bnode].push_back(v);
            } else {
                block_id[v] = bnode; // non-cut vertex belongs to this block
            }
        }
    }
}

// Usage: paths in block-cut tree via LCA.
// For a non-cut vertex v, its tree node is block_id[v].
// For a cut vertex v, its tree node is v itself.

Note on multi-edges: If the graph may have parallel edges between the same pair of vertices, track the edge index instead of the parent vertex to avoid skipping valid back edges.

Applications

1. Components After Vertex Removal

Removing a non-cut vertex never disconnects the graph. Removing a cut vertex v splits the graph into exactly deg(v) components in the block-cut tree (where deg is v’s degree in the tree).

2. Vertex-Disjoint Paths

Two vertices u and v have at least 2 vertex-disjoint paths between them if and only if they belong to the same biconnected component. Otherwise the number of cut vertices on the tree path between them limits connectivity.

3. Number of Cut Vertices on a Path

The number of cut vertices that every simple path from u to v must pass through equals the number of cut-vertex nodes on the unique path between u and v in the block-cut tree. Solve with LCA + depth counting.

4. 2-Edge-Connected Check

A graph is 2-edge-connected (no bridges) if and only if it has a single 2-edge-connected component. Check by verifying that no edge is a bridge — equivalently, every edge lies on a cycle.

Summary

  • Biconnected components (blocks) are maximal subgraphs with no cut vertex. Found in O(V+E) via DFS with an edge stack.
  • The block-cut tree has one node per block and one per cut vertex; edges connect cut vertices to their blocks. It is always a tree.
  • The bridge tree is the edge-connectivity analog: contract 2-edge-connected components, keep bridges as edges.
  • Both structures reduce graph connectivity queries to tree queries (LCA, path sums, etc.).
  • Cut vertex detection: root with ≥ 2 DFS children, or non-root u with a child v where low[v] ≥ disc[u].
