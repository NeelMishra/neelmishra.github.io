Minimum Cost Maximum Flow

Send the maximum flow through a network while minimizing total cost — a powerful framework behind assignment, matching, and transportation problems.

What is Min-Cost Max-Flow?

In a standard max-flow problem every edge has a capacity and we want to push as much flow as possible from source to sink. In the Minimum Cost Maximum Flow (MCMF) variant each edge also carries a cost per unit of flow. The goal is twofold:

  1. Achieve the maximum flow from source s to sink t.
  2. Among all maximum-flow solutions, find the one with minimum total cost.

Formally, for a directed graph G = (V, E) where each edge e has capacity cap(e) and cost cost(e), we want:

MCMF Objective
Maximize   |f|  =  total flow from s to t
Minimize   Σ  f(e) · cost(e)   over all edges e
subject to capacity and flow-conservation constraints

Applications are everywhere in competitive programming and beyond:

  • Assignment problem — assign n workers to n jobs at minimum total cost
  • Transportation problem — ship goods from factories to warehouses at minimum shipping cost
  • Min-cost bipartite matching — find a maximum-weight/minimum-weight matching
  • Project scheduling — allocate resources with penalties for delays

Successive Shortest Paths Algorithm

The most intuitive MCMF algorithm uses successive shortest paths: repeatedly find the cheapest augmenting path from s to t in the residual graph and push flow along it.

Successive Shortest Paths (SPFA)
total_flow = 0, total_cost = 0
while true:
    Run SPFA / Bellman-Ford from s in residual graph
    if t is unreachable: break

    // Find bottleneck capacity along the shortest path s→t
    bottleneck = min residual capacity on path
    total_flow += bottleneck
    total_cost += bottleneck × dist[t]

    // Augment: push bottleneck flow along the path
    Update residual capacities (forward -=, reverse +=)

return (total_flow, total_cost)

Why SPFA / Bellman-Ford?

The residual graph contains reverse edges with negative cost. If a forward edge has cost c, its reverse edge has cost −c. Dijkstra cannot handle negative edges directly, so we use SPFA (Shortest Path Faster Algorithm) or Bellman-Ford to find shortest paths in the residual graph.

Why Cheapest Path First?

By always augmenting along the shortest-cost path, each unit of flow is sent as cheaply as possible. A classical result in network flow theory guarantees that this greedy strategy produces a globally optimal (minimum cost) maximum flow.

Step-by-Step Walkthrough

Let’s trace MCMF on a 5-node network. Each edge label shows capacity/cost. Source = S(0), Sink = T(4).

Shortest path: S → A → B → C → T  (cost = 1+2+1+3 = 7)

SPFA finds the cheapest s–t path in the residual graph. Bottleneck capacity along this path = min(3, 2, 3, 3) = 2.

Push 2 units of flow. Update residual capacities.

Flow = 2  |  Cost = 2 × 7 = 14

Shortest path: S → A → T  (cost = 1+6 = 7)

The A→B edge is now saturated (residual = 0). The next cheapest path uses the direct A→T edge. Bottleneck = min(1, 2) = 1.

Push 1 unit of flow.

Flow = 3  |  Cost = 14 + 1 × 7 = 21

Shortest path: S → B → C → T  (cost = 4+1+3 = 8)

S→A is now saturated. Use the S→B edge. Bottleneck = min(2, 1, 1) = 1.

Push 1 unit of flow.

Flow = 4  |  Cost = 21 + 1 × 8 = 29

Shortest path: S → B → C → T  (cost = 4+1+3 = 8)

Same path still exists. Bottleneck = min(1, 0…)—wait, B→C has residual 0 now? Let’s check: after iteration 3 B→C residual is 0. SPFA cannot find another s–t path.

No augmenting path found. Algorithm terminates.

Max Flow = 4  |  Min Cost = 29

Result: Maximum flow of 4 units at minimum total cost of 29.

Johnson’s Potential Technique

SPFA/Bellman-Ford runs in O(VE) per iteration, which is slow for large graphs. We can speed this up using Johnson’s potentials to transform costs so that all residual edge costs become non-negative, enabling Dijkstra for subsequent iterations.

How It Works

  1. First iteration: Run SPFA/Bellman-Ford from s to compute shortest distances h[] (the potential function).
  2. Re-weight edges: For each edge (u, v) with cost c, define the reduced cost: c′(u,v) = c(u,v) + h[u] − h[v]. This is guaranteed to be ≥ 0.
  3. Subsequent iterations: Use Dijkstra with the reduced costs. After each iteration, update potentials: h[v] += dist[v].
Potential Update Rule
// After Dijkstra finds shortest distances d[]:
for each vertex v:
    h[v] = h[v] + d[v]

// Reduced cost for edge (u, v):
c'(u, v) = cost(u, v) + h[u] - h[v]   ≥ 0

Why Does This Work?

The potential h[] comes from valid shortest-path distances. For any edge on a shortest path, the reduced cost is 0. For other edges it is positive. After augmenting flow, the reverse edge has reduced cost 0 (since it lies on a shortest path in reverse). This means all edges in the new residual graph have non-negative reduced costs.

This technique is exactly how Johnson’s algorithm for all-pairs shortest paths works — one Bellman-Ford pass to establish potentials, then Dijkstra from each source.

Applications

Assignment Problem

Given n workers and n jobs with a cost matrix, assign each worker to exactly one job to minimize total cost. Model as a bipartite graph with S→workers (cap 1, cost 0), workers→jobs (cap 1, cost cij), jobs→T (cap 1, cost 0). MCMF gives the optimal assignment.

Min-Cost Bipartite Matching

Generalization of assignment: find a maximum matching of minimum cost in a bipartite graph. Directly solvable with MCMF. Also handles the case where we want maximum weight matching (negate costs).

Transportation Problem

Ship goods from m factories (with supply) to n warehouses (with demand) at minimum shipping cost. Each factory connects to S with supply capacity; each warehouse connects to T with demand capacity.

Minimum Path Cover with Costs

In a DAG, cover all vertices with the minimum number of paths, where each edge has an associated cost. Reduce to min-cost bipartite matching on the split graph.

Modeling Tip

Many MCMF problems require creative graph construction. The key insight: any problem that can be phrased as “select a set of items/paths maximizing quantity while minimizing cost, subject to capacity constraints” is likely an MCMF problem. Look for:

  • A natural source (supply) and sink (demand)
  • Capacity constraints on how much “flow” can pass through an edge or node
  • Per-unit costs on edges

C++ Implementation

SPFA-based MCMF

#include <bits/stdc++.h>
using namespace std;

struct Edge {
    int to, cap, cost, flow;
};

struct MCMF {
    int n;
    vector<Edge> edges;
    vector<vector<int>> g; // adjacency list of edge indices
    vector<int> dist, parent, parentEdge;
    vector<bool> inQueue;

    MCMF(int n) : n(n), g(n), dist(n), parent(n),
                   parentEdge(n), inQueue(n) {}

    void addEdge(int from, int to, int cap, int cost) {
        g[from].push_back(edges.size());
        edges.push_back({to, cap, cost, 0});
        g[to].push_back(edges.size());
        edges.push_back({from, 0, -cost, 0}); // reverse edge
    }

    bool spfa(int s, int t) {
        fill(dist.begin(), dist.end(), INT_MAX);
        fill(inQueue.begin(), inQueue.end(), false);
        dist[s] = 0;
        inQueue[s] = true;
        queue<int> q;
        q.push(s);

        while (!q.empty()) {
            int u = q.front(); q.pop();
            inQueue[u] = false;
            for (int id : g[u]) {
                Edge& e = edges[id];
                if (e.cap > e.flow && dist[u] + e.cost < dist[e.to]) {
                    dist[e.to] = dist[u] + e.cost;
                    parent[e.to] = u;
                    parentEdge[e.to] = id;
                    if (!inQueue[e.to]) {
                        inQueue[e.to] = true;
                        q.push(e.to);
                    }
                }
            }
        }
        return dist[t] != INT_MAX;
    }

    pair<int,int> solve(int s, int t) {
        int totalFlow = 0, totalCost = 0;
        while (spfa(s, t)) {
            // Find bottleneck
            int pushFlow = INT_MAX;
            for (int v = t; v != s; v = parent[v])
                pushFlow = min(pushFlow,
                    edges[parentEdge[v]].cap - edges[parentEdge[v]].flow);

            // Augment along the path
            for (int v = t; v != s; v = parent[v]) {
                edges[parentEdge[v]].flow += pushFlow;
                edges[parentEdge[v] ^ 1].flow -= pushFlow; // reverse edge
            }
            totalFlow += pushFlow;
            totalCost += pushFlow * dist[t];
        }
        return {totalFlow, totalCost};
    }
};

Usage Example

int main() {
    int n, m;
    cin >> n >> m;

}

Dijkstra + Potentials Variant

struct MCMF_Dijkstra {
    int n;
    vector<Edge> edges;
    vector<vector<int>> g;
    vector<int> h, dist, parent, parentEdge;

    MCMF_Dijkstra(int n) : n(n), g(n), h(n, 0), dist(n),
                            parent(n), parentEdge(n) {}

    void addEdge(int from, int to, int cap, int cost) {
        g[from].push_back(edges.size());
        edges.push_back({to, cap, cost, 0});
        g[to].push_back(edges.size());
        edges.push_back({from, 0, -cost, 0});
    }

    bool dijkstra(int s, int t) {
        fill(dist.begin(), dist.end(), INT_MAX);
        dist[s] = 0;
        priority_queue<pair<int,int>, vector<pair<int,int>>,
                       greater<pair<int,int>>> pq;
        pq.push({0, s});

        while (!pq.empty()) {
            auto [d, u] = pq.top(); pq.pop();
            if (d > dist[u]) continue;
            for (int id : g[u]) {
                Edge& e = edges[id];
                if (e.cap > e.flow) {
                    // Reduced cost using potentials
                    int newDist = dist[u] + e.cost + h[u] - h[e.to];
                    if (newDist < dist[e.to]) {
                        dist[e.to] = newDist;
                        parent[e.to] = u;
                        parentEdge[e.to] = id;
                        pq.push({newDist, e.to});
                    }
                }
            }
        }
        // Update potentials
        for (int i = 0; i < n; i++)
            if (dist[i] < INT_MAX) h[i] += dist[i];
        return dist[t] != INT_MAX;
    }

    pair<int,int> solve(int s, int t) {
        // Initialize potentials with SPFA (handles negative costs)
        fill(h.begin(), h.end(), INT_MAX);
        h[s] = 0;
        queue<int> q;
        vector<bool> inQ(n, false);
        q.push(s); inQ[s] = true;
        while (!q.empty()) {
            int u = q.front(); q.pop(); inQ[u] = false;
            for (int id : g[u]) {
                Edge& e = edges[id];
                if (e.cap > e.flow && h[u] + e.cost < h[e.to]) {
                    h[e.to] = h[u] + e.cost;
                    if (!inQ[e.to]) { q.push(e.to); inQ[e.to] = true; }
                }
            }
        }

        int totalFlow = 0, totalCost = 0;
        while (dijkstra(s, t)) {
            int pushFlow = INT_MAX;
            for (int v = t; v != s; v = parent[v])
                pushFlow = min(pushFlow,
                    edges[parentEdge[v]].cap - edges[parentEdge[v]].flow);
            for (int v = t; v != s; v = parent[v]) {
                edges[parentEdge[v]].flow += pushFlow;
                edges[parentEdge[v] ^ 1].flow -= pushFlow;
            }
            totalFlow += pushFlow;
            totalCost += pushFlow * (h[t] - h[s]); // true cost
        }
        return {totalFlow, totalCost};
    }
};

Complexity Analysis

VariantTime per IterationTotal TimeNotes
SPFA-basedO(V · E)O(V · E · F)F = max flow value; simple to implement
Dijkstra + PotentialsO((V + E) log V)O((V + E) log V · F)Faster per iteration; one-time SPFA init
Cycle-CancelingO(V · E) per cycleO(V · E2 · C)C = max cost; rarely used in CP
Network Simplex—O(V · E · log V · log(VC))Strongly polynomial; fastest in practice

In competitive programming, SPFA-based MCMF is the most common due to simplicity. The Dijkstra+potentials variant is preferred when the flow value is large and you need the extra speed from O((V+E) log V) per augmentation.

Space: O(V + E) for the adjacency list and edge storage. Each edge requires a forward and reverse entry, so total edge storage is 2E.

Summary

  • MCMF finds the maximum flow with minimum total cost in a network with capacities and per-unit costs.
  • The Successive Shortest Paths algorithm repeatedly finds the cheapest augmenting path via SPFA and pushes flow along it.
  • Johnson’s potentials allow using Dijkstra after the first SPFA pass, improving per-iteration time from O(VE) to O((V+E) log V).
  • MCMF models a wide range of problems: assignment, transportation, min-cost matching, and more.
  • The edge ^ 1 trick gives O(1) access to the reverse edge when edges are stored in pairs.
  • Total complexity is O(V · E · F) with SPFA or O((V+E) log V · F) with Dijkstra + potentials.
