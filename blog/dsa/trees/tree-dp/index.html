    Tree DP — Dynamic Programming on Trees

    July 2025 · 25 min read

    Introduction

    Tree DP (Dynamic Programming on Trees) is the single most important technique for competitive programming problems on trees. The idea is simple: exploit the fact that removing any edge from a tree splits it into two independent subtrees. This optimal substructure makes trees a natural fit for DP.

    The core pattern: root the tree, then compute dp values bottom-up from leaves to root. Each node’s dp value is computed from the dp values of its children. Since a rooted tree is a DAG (parent → children), every node is processed exactly once, giving O(n) time for most tree DP problems.

    Why Tree DP matters: It appears in ~40% of tree problems on Codeforces and nearly every CSES tree problem. Master this, and you can solve most competitive programming tree problems.

    We’ll cover the fundamental patterns — subtree DP, max independent set, tree diameter, the powerful rerooting technique, and tree coloring — each with interactive animations and contest-ready C++ code.

    Subtree DP Pattern

    The simplest tree DP: compute a value for every subtree. For each node v, define dp[v] as some function of the dp values of its children. Process nodes bottom-up (post-order DFS).

    Example: Subtree Sizes

    The most basic tree DP: sz[v] = 1 + ∑ sz[c] for each child c of v. Every leaf has sz = 1.

    function dfs(v, parent):
    sz[v] = 1 for child in adj[v]: if child == parent: continue dfs(child, v) sz[v] += sz[child]

    Interactive: Subtree Size DFS

    ▶ Subtree Sizes — Bottom-Up DFS

    Watch DFS fill the sz[] array from leaves to root. Each node’s size = 1 + sum of children’s sizes.

    The subtree-size DP is a building block for many advanced techniques: centroid decomposition (find centroids via subtree sizes), heavy-light decomposition (pick heavy child), and rerooting (adjust dp values when the root changes).

    Maximum Independent Set

    A classic tree DP problem: select the maximum number of nodes such that no two selected nodes are adjacent (connected by an edge).

    The Recurrence

    For each node v, track two states:

    • dp[v][0] = max independent set size in subtree of v, when v is not selected
    • dp[v][1] = max independent set size in subtree of v, when v is selected
    Transitions:
    dp[v][0] = ∑c ∈ children(v) max(dp[c][0], dp[c][1])
    dp[v][1] = 1 + ∑c ∈ children(v) dp[c][0]

    If v is not taken, each child can be taken or not — pick whichever is better. If v is taken, no child can be taken. The answer is max(dp[root][0], dp[root][1]).

    Walkthrough

    Nodedp[v][0] (not taken)dp[v][1] (taken)Best
    3 (leaf)011
    4 (leaf)011
    5 (leaf)011
    1max(0,1)+max(0,1) = 21+0+0 = 12
    2max(0,1) = 11+0 = 11
    0 (root)max(2,1)+max(1,1) = 31+2+1 = 44

    Answer = max(dp[0][0], dp[0][1]) = max(3, 4) = 4. The optimal set is {0, 3, 4, 5}: taking root 0 forces children 1 and 2 to be excluded, but grandchildren 3, 4, and 5 can all be taken.

    Key insight: When v is taken, its children cannot be taken, but grandchildren can. This is why dp[0][1] = 1 + dp[1][0] + dp[2][0] = 1 + 2 + 1 = 4 is larger than dp[0][0] = 3 here.

    Tree Diameter via DP

    The diameter of a tree is the number of edges on the longest path between any two nodes. We can compute it in a single DFS using tree DP.

    The Idea

    For each node v, compute depth[v] = length of the longest path going downward from v. The longest path through v uses the two deepest children.

    depth[v] = max(depth[c] + 1) over all children c
    diameter candidate at v = sum of two largest (depth[c] + 1) values 
    int diameter = 0;
vector<int> depth(n, 0);

function<void(int,int)> dfs = [&](int v, int par) {
    int best1 = 0, best2 = 0; // two longest downward paths
    for (int c : adj[v]) {
        if (c == par) continue;
        dfs(c, v);
        int d = depth[c] + 1;
        if (d >= best1) { best2 = best1; best1 = d; }
        else if (d > best2) { best2 = d; }
    }
    depth[v] = best1;
    diameter = max(diameter, best1 + best2);
};
dfs(0, -1);

    Time: O(n). Space: O(n) for the depth array plus O(h) recursion stack.

    Key insight: The diameter path passes through some node v where it uses the two longest downward paths in different subtrees. By checking best1 + best2 at every node, we find the global optimum.

    Rerooting Technique

    Many problems ask: “compute the answer for every possible root”. Naively running tree DP n times gives O(n²). The rerooting technique solves this in O(n) with two passes.

    The Idea

    When we move the root from node v to its child c:

    • c gains everything that was “above” it (the rest of the tree through v)
    • v loses the subtree of c

    We can express this as a recurrence that “pushes” information from parent to child.

    Phase 1 Root at node 0. Compute dp[v] for all nodes bottom-up.
    Phase 2 Top-down: for each edge v → c, compute ans[c] by adjusting ans[v] to account for the root moving from v to c.

    General Template

    The rerooting template works for any DP where the combine operation is “nice” (associative, has an inverse or can be decomposed). The key operations:

    1. base(v): the dp value of a single node with no children
    2. combine(acc, child_dp): merge a child’s contribution into the accumulator
    3. finalize(acc, v): produce dp[v] from the accumulated children values
    // Phase 1: bottom-up DFS (root at 0)
    function dfs1(v, parent):
    dp[v] = base(v) for child in adj[v]: if child == parent: continue dfs1(child, v) dp[v] = combine(dp[v], dp[child]) dp[v] = finalize(dp[v], v)

    // Phase 2: top-down reroot
    function dfs2(v, parent):
    // ans[v] is already set (from parent or root) // Build prefix/suffix of children contributions children = [c for c in adj[v] if c != parent] prefix[0] = base(v) for i = 0 to len(children)-1: prefix[i+1] = combine(prefix[i], dp[children[i]]) suffix[len] = identity for i = len(children)-1 down to 0: suffix[i] = combine(suffix[i+1], dp[children[i]]) for i, c in enumerate(children): // contribution from everything except subtree(c) without_c = combine(prefix[i], suffix[i+1]) without_c = combine(without_c, parent_contrib[v]) ans[c] = finalize(combine(base(c), all_children_of_c, without_c), c) dfs2(c, v)

    The prefix/suffix trick lets us “remove” any single child’s contribution in O(1), which is essential when the combine operation doesn’t have a simple inverse (e.g., min, gcd).

    ▶ Rerooting — Sum of Subtree Sizes for Each Root

    Phase 1 computes dp[] bottom-up from root 0. Phase 2 reroots top-down to compute ans[] for every node.

    Rerooting Example: Sum of Distances

    The canonical rerooting problem. Given a tree with n nodes, for each node v, compute the sum of distances from v to all other nodes. (CSES: Tree Distances I, LeetCode 834)

    Approach

    Phase 1 Root at node 0. Compute sz[v] (subtree size) and down[v] (sum of distances from v to all nodes in its subtree).

    down[v] = ∑c ∈ children(v) (down[c] + sz[c])
    Each of the sz[c] nodes in child c’s subtree is 1 edge farther from v than from c.

    Phase 2 Top-down: when we move the root from v to child c:

    • All sz[c] nodes in c’s subtree get 1 closer (distance decreases by sz[c])
    • All n - sz[c] other nodes get 1 farther (distance increases by n - sz[c])
    ans[c] = ans[v] - sz[c] + (n - sz[c]) = ans[v] + n - 2 · sz[c]

    Worked Example

    Consider a tree with 6 nodes:

    Phase 1 computation:

    Nodesz[v]down[v]
    3, 4, 5 (leaves)10
    130+1 + 0+1 = 2
    220+1 = 1
    06(2+3) + (1+2) = 8

    Phase 2 computation: ans[0] = down[0] = 8. Then propagate using ans[c] = ans[v] + n − 2·sz[c]:

    Nodeans[v]Computation
    08ans[0] = down[0] = 8
    18ans[0] + 6 − 2·3 = 8 + 0 = 8
    210ans[0] + 6 − 2·2 = 8 + 2 = 10
    312ans[1] + 6 − 2·1 = 8 + 4 = 12
    412ans[1] + 6 − 2·1 = 8 + 4 = 12
    514ans[2] + 6 − 2·1 = 10 + 4 = 14
    Verification: From node 0, distances are 0+1+1+2+2+2 = 8 ✔. From node 1: 1+0+2+1+1+3 = 8 ✔. From node 5: 3+4+2+4+4+0 — wait, let’s check: dist(5,0)=2, dist(5,1)=3, dist(5,2)=1, dist(5,3)=4, dist(5,4)=4, dist(5,5)=0 → 2+3+1+4+4 = 14 ✔

    C++ Implementation

    vector<vector<int>> adj(n);
// ... build adjacency list ...

vector<long long> sz(n, 1), down(n, 0), ans(n, 0);

// Phase 1: bottom-up
function<void(int,int)> dfs1 = [&](int v, int p) {
    for (int c : adj[v]) {
        if (c == p) continue;
        dfs1(c, v);
        sz[v] += sz[c];
        down[v] += down[c] + sz[c];
    }
};

// Phase 2: top-down reroot
function<void(int,int)> dfs2 = [&](int v, int p) {
    for (int c : adj[v]) {
        if (c == p) continue;
        ans[c] = ans[v] + n - 2 * sz[c];
        dfs2(c, v);
    }
};

dfs1(0, -1);
ans[0] = down[0];
dfs2(0, -1);
// ans[v] = sum of distances from v to all other nodes

    Tree Coloring DP

    Count the number of ways to color a tree with k colors such that no two adjacent nodes share the same color. This is the chromatic polynomial of a tree.

    The Recurrence

    For a tree, the chromatic polynomial has a beautiful closed form, but the DP perspective is instructive:

    • The root can be any of k colors.
    • Each child must differ from its parent: k - 1 choices.
    • Since subtrees are independent (only connected through the parent), the counts multiply.
    dp[v] = ∏c ∈ children(v) ((k - 1) · dp[c])
    With dp[leaf] = 1. Total = k · dp[root].

    For a tree with n nodes: answer = k · (k-1)n-1. The DP confirms this: the root has k choices, and each of the n-1 other nodes has k-1 choices (must differ from parent only, since parent is the only adjacent node on the path from root).

    Harder Variant: Limited Colors per Node

    When each node has a set of allowed colors, the product formula breaks down and we need genuine tree DP:

    // dp[v][c] = number of valid colorings of subtree(v)
//            where v is colored c
vector<vector<long long>> dp(n, vector<long long>(k, 0));

function<void(int,int)> dfs = [&](int v, int p) {
    for (int c = 0; c < k; c++)
        if (allowed[v][c]) dp[v][c] = 1;

    for (int u : adj[v]) {
        if (u == p) continue;
        dfs(u, v);
        for (int c = 0; c < k; c++) {
            long long sum = 0;
            for (int d = 0; d < k; d++)
                if (d != c) sum += dp[u][d];
            dp[v][c] *= sum;
        }
    }
};
dfs(0, -1);
long long ans = 0;
for (int c = 0; c < k; c++) ans += dp[0][c];

    Time: O(n · k²). Can be optimized to O(n · k) using the “total minus same” trick: for each child, compute total = ∑ dp[u][d], then contribution for color c is total - dp[u][c].

    C++ Implementations

    Maximum Independent Set (Contest-Ready)

    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n; cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v; cin >> u >> v; u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // dp[v][0] = not taken, dp[v][1] = taken
    vector<array<int, 2>> dp(n, {0, 0});

    function<void(int, int)> dfs = [&](int v, int p) {
        dp[v][1] = 1; // taking v contributes 1
        for (int c : adj[v]) {
            if (c == p) continue;
            dfs(c, v);
            dp[v][0] += max(dp[c][0], dp[c][1]);
            dp[v][1] += dp[c][0];
        }
    };
    dfs(0, -1);
    cout << max(dp[0][0], dp[0][1]) << "\n";
}

    Rerooting Template (Sum of Distances)

    #include <bits/stdc++.h>
using namespace std;

int main() {
    int n; cin >> n;
    vector<vector<int>> adj(n);
    for (int i = 0; i < n - 1; i++) {
        int u, v; cin >> u >> v; u--; v--;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    vector<long long> sz(n, 1), down(n, 0), ans(n);

    // Phase 1: bottom-up from root 0
    function<void(int, int)> dfs1 = [&](int v, int p) {
        for (int c : adj[v]) {
            if (c == p) continue;
            dfs1(c, v);
            sz[v] += sz[c];
            down[v] += down[c] + sz[c];
        }
    };

    // Phase 2: top-down reroot
    function<void(int, int)> dfs2 = [&](int v, int p) {
        for (int c : adj[v]) {
            if (c == p) continue;
            ans[c] = ans[v] + n - 2 * sz[c];
            dfs2(c, v);
        }
    };

    dfs1(0, -1);
    ans[0] = down[0];
    dfs2(0, -1);

    for (int i = 0; i < n; i++)
        cout << ans[i] << " \n"[i == n - 1];
}

    Generic Rerooting Template

    For problems where the combine operation has no inverse (e.g., max, gcd), use the prefix-suffix approach:

    // Generic rerooting with prefix-suffix decomposition
// Customize: Val, identity, base, combine, finalize

using Val = long long;
const Val identity = 0;

Val base(int v) { return 0; }
Val combine(Val a, Val b) { return a + b; }
Val finalize(Val acc, int v) { return acc + 1; }

vector<Val> dp, ans;
vector<vector<int>> adj;
int n;

void dfs1(int v, int p) {
    dp[v] = base(v);
    for (int c : adj[v]) {
        if (c == p) continue;
        dfs1(c, v);
        dp[v] = combine(dp[v], dp[c]);
    }
    dp[v] = finalize(dp[v], v);
}

void dfs2(int v, int p, Val fromParent) {
    // Collect children
    vector<int> ch;
    for (int c : adj[v]) if (c != p) ch.push_back(c);

    int m = ch.size();
    vector<Val> pref(m + 1, identity), suf(m + 1, identity);
    for (int i = 0; i < m; i++)
        pref[i + 1] = combine(pref[i], dp[ch[i]]);
    for (int i = m - 1; i >= 0; i--)
        suf[i] = combine(suf[i + 1], dp[ch[i]]);

    // ans[v] considers all children + parent contribution
    ans[v] = finalize(combine(pref[m], fromParent), v);

    for (int i = 0; i < m; i++) {
        Val without_ci = combine(combine(pref[i], suf[i + 1]), fromParent);
        Val parentContrib = finalize(combine(base(v), without_ci), v);
        dfs2(ch[i], v, parentContrib);
    }
}

// Usage:
// dfs1(0, -1);
// dfs2(0, -1, identity);
// ans[v] is the answer when v is the root

    Common Patterns & Tips

    When to Use Tree DP

    • “For each node” or “for each subtree” — almost always tree DP
    • “For every root” — rerooting technique
    • Optimization on a tree (max weight, min cost) — tree DP with appropriate states
    • Counting (paths, colorings, matchings) — tree DP with multiplication

    Common State Designs

    Problem TypeDP StateExample
    Binary choice per nodedp[v][0/1]Max Independent Set, Min Vertex Cover
    Color per nodedp[v][color]Tree Coloring, Guard placement
    Path trackingdp[v] = best downward pathDiameter, Max Path Sum
    Subtree aggregatedp[v] = f(subtree)Subtree sum, sizes
    All-roots answerdp[v] + rerootingSum of distances, Max depth from any root

    Vertex DP vs Edge DP

    Most tree DP problems use vertex-based states. But some problems are more natural as edge DP:

    • Edge coloring / weighting: dp on edges rather than nodes
    • Bridge-related problems: Consider each edge’s contribution

    Conversion trick: you can convert edge DP to vertex DP by “assigning” each edge to its child node (in a rooted tree, every non-root node has exactly one parent edge).

    Common Mistakes

    • Forgetting the parent check: Always skip the parent in DFS to avoid infinite loops.
    • Integer overflow: Tree DP with multiplication (colorings, matchings) often needs long long or modular arithmetic.
    • Rerooting without inverse: If your combine is max, you can’t simply “subtract” a child. Use prefix-suffix decomposition.
    • 1-indexed vs 0-indexed: Many CP problems use 1-indexed nodes. Be consistent.

    Practice Problems

    Roughly ordered by difficulty. Start with the first few, then work through the rerooting problems.

    • CSES — Subordinates Subtree DP
      Compute subtree sizes. Direct application of the basic pattern.
    • CSES — Tree Diameter Path DP
      Find the longest path in the tree. Use the two-best-depths technique.
    • CSES — Tree Distances I Rerooting
      For each node, find the maximum distance to any other node. Classic rerooting.
    • CSES — Tree Distances II Rerooting
      Sum of distances from each node to all others. The exact problem we solved above.
    • CF 1187E — Tree Painting Rerooting
      Maximize sum of subtree sizes over all removal orders. Reduces to rerooting.
    • CF 1092F — Tree with Maximum Cost Rerooting
      Maximize ∑ a[v] · dist(v, root). Beautiful rerooting application.
    • AtCoder DP Contest — P (Independent Set) Tree DP
      Count the number of independent sets. dp[v][0/1] with multiplication.
    • AtCoder ABC220F — Distance Sums 2 Rerooting
      Sum of distances. Identical to CSES Tree Distances II.
    • CF 219D — Choosing Capital for Treeland Rerooting
      Directed tree: minimize reversed edges. Rerooting with edge direction tracking.
    • CSES — Tree Matching Tree DP
      Maximum matching in a tree. dp[v][0/1] based on whether v is matched to a child.
