    Operations

    March 2026 · 25 min read

    Now comes the fun part. We have the recursive structure: a vEB(u) node with min, max, a summary, and √u clusters. Let's implement every operation and see exactly why each one touches only one recursive path, giving us O(log log u).

    Throughout this post, we'll use this running example: u = 16, set S = {2, 3, 7, 14}.

    Min and Max: O(1)

    The simplest operations. Since we cache min and max directly in each node:

    int vEB_min(vEB* V) { return V->min; }
int vEB_max(vEB* V) { return V->max; }

    That's it. O(1). No recursion needed.

    Member: O(log log u)

    Check whether a value x exists in the structure.

    bool vEB_member(vEB* V, int x) {
    if (x == V->min || x == V->max) return true;
    if (V->u == 2) return false;           // base case: not min or max, so absent
    return vEB_member(V->cluster[high(x)], low(x));
}

    First check if x is the min or max (O(1)). If not, recurse into the appropriate cluster. At each level, the universe size square-roots down, so we recurse O(log log u) times.

    Successor: O(log log u)

    This is the most important operation. Given x, find the smallest element in the set that is greater than x.

    int vEB_successor(vEB* V, int x) {
    // Base case
    if (V->u == 2) {
        if (x == 0 && V->max == 1) return 1;
        return -1;  // no successor
    }

    // Case 1: x < min → min is the successor
    if (V->min != -1 && x < V->min)
        return V->min;

    // Case 2: successor is within the same cluster
    int hi = high(x), lo = low(x);
    int maxInCluster = vEB_max(V->cluster[hi]);  // O(1)
    if (maxInCluster != -1 && lo < maxInCluster) {
        int offset = vEB_successor(V->cluster[hi], lo);
        return index(hi, offset);
    }

    // Case 3: successor is in a later cluster
    int nextCluster = vEB_successor(V->summary, hi);
    if (nextCluster == -1) return -1;  // no successor
    int offset = vEB_min(V->cluster[nextCluster]);  // O(1)
    return index(nextCluster, offset);
}

    The key insight: we only make one recursive call. Either we recurse into the current cluster (Case 2) or into the summary (Case 3), but never both. The max check in Case 2 is O(1) because max is cached.

    Insert: O(log log u)

    Insert a value x into the vEB tree.

    void vEB_insert(vEB* V, int x) {
    // Empty tree: just set min and max
    if (V->min == -1) {
        V->min = V->max = x;
        return;                         // O(1), no recursion!
    }

    // If x < min, swap: the old min gets pushed into a cluster
    if (x < V->min) swap(x, V->min);

    if (V->u > 2) {
        int hi = high(x), lo = low(x);
        if (vEB_min(V->cluster[hi]) == -1) {
            // Cluster was empty: update summary AND insert into cluster
            vEB_insert(V->summary, hi);           // recursive call
            V->cluster[hi]->min = lo;              // O(1): empty cluster insert
            V->cluster[hi]->max = lo;
        } else {
            vEB_insert(V->cluster[hi], lo);        // recursive call
        }
    }

    if (x > V->max) V->max = x;
}

    There are two branches, but only one recursive call executes:

    • Cluster was empty: We recurse into the summary to register this cluster. The cluster itself gets a direct min/max set (O(1)).
    • Cluster was non-empty: We recurse into the cluster. The summary already knows about it, so no summary update needed.

    Predecessor: O(log log u)

    The mirror of successor. Given x, find the largest element in the set smaller than x.

    int vEB_predecessor(vEB* V, int x) {
    // Base case
    if (V->u == 2) {
        if (x == 1 && V->min == 0) return 0;
        return -1;
    }

    // Case 1: x > max → max is the predecessor
    if (V->min != -1 && x > V->max)
        return V->max;

    // Case 2: predecessor is within the same cluster
    int hi = high(x), lo = low(x);
    int minInCluster = vEB_min(V->cluster[hi]);  // O(1)
    if (minInCluster != -1 && lo > minInCluster) {
        int offset = vEB_predecessor(V->cluster[hi], lo);
        return index(hi, offset);
    }

    // Case 3: predecessor is in an earlier cluster
    int prevCluster = vEB_predecessor(V->summary, hi);
    if (prevCluster == -1) {
        // Check if min itself is the predecessor (min is NOT in any cluster)
        if (V->min != -1 && x > V->min) return V->min;
        return -1;
    }
    int offset = vEB_max(V->cluster[prevCluster]);  // O(1)
    return index(prevCluster, offset);
}

    Notice the extra check in Case 3: since min is not stored in any cluster, we check it explicitly when falling through. Same pattern: one recursive call.

    Delete: O(log log u)

    Delete is the most intricate operation, but follows the same pattern: at most one recursive call per level.

    void vEB_delete(vEB* V, int x) {
    // Only one element? Clear it.
    if (V->min == V->max) {
        V->min = V->max = -1;
        return;
    }

    // Base case: u == 2
    if (V->u == 2) {
        V->min = (x == 0) ? 1 : 0;
        V->max = V->min;
        return;
    }

    // If deleting the min, replace it with the next smallest
    if (x == V->min) {
        int firstCluster = vEB_min(V->summary);       // O(1)
        x = index(firstCluster, vEB_min(V->cluster[firstCluster])); // O(1)
        V->min = x;  // new min; now delete x from its cluster
    }

    // Delete x from its cluster
    int hi = high(x), lo = low(x);
    vEB_delete(V->cluster[hi], lo);                    // ONE recursive call

    // If cluster became empty, update summary
    if (vEB_min(V->cluster[hi]) == -1) {
        vEB_delete(V->summary, hi);                    // ONE recursive call
        // If we deleted the max, find the new max
        if (x == V->max) {
            int summaryMax = vEB_max(V->summary);      // O(1)
            if (summaryMax == -1) V->max = V->min;
            else V->max = index(summaryMax, vEB_max(V->cluster[summaryMax]));
        }
    } else if (x == V->max) {
        V->max = index(hi, vEB_max(V->cluster[hi]));  // O(1)
    }
}

    Wait, there are two recursive calls in the code (delete from cluster AND delete from summary). But the crucial insight: the summary delete only runs when the cluster became empty, and an empty cluster's delete was a base case O(1) (single element removal). So at most one of the two is a non-trivial recursive call.

    The pattern for every operation: At each level, we do O(1) work and make at most one recursive call into a vEB(√u) structure. The recurrence T(u) = T(√u) + O(1) solves to T(u) = O(log log u).

    Operation Summary

    OperationTimeKey Technique
    min / maxO(1)Cached in node
    memberO(log log u)Recurse into cluster[high(x)]
    successorO(log log u)Check cluster max first, then summary
    predecessorO(log log u)Check cluster min first, then summary; check global min
    insertO(log log u)Empty cluster insert is O(1); only one path recurses
    deleteO(log log u)Summary cleanup only when cluster empties (trivial)
