    Threaded Binary Trees — Introduction

    March 2026 · 8 min read

    Why Threaded Trees?

    In a standard binary tree with n nodes, there are n + 1 null pointers. Each leaf contributes two nulls, and internal nodes with one child contribute one. That's a lot of wasted memory doing nothing.

    A threaded binary tree replaces these null pointers with threads — pointers to the in-order predecessor or successor. This enables O(1) space in-order traversal without a stack or recursion.

    Key insight: In an n-node binary tree, exactly n+1 of the 2n pointer fields are null. Threaded trees put those wasted pointers to work.

    Types of Threading

    Single Threaded (Right-Threaded)

    The most common variant. Null right pointers are replaced with threads to the in-order successor.

    Double Threaded

    Both null left and right pointers are threaded:

    • Null left → points to the in-order predecessor
    • Null right → points to the in-order successor

    Node Structure

    Each node needs a boolean flag to distinguish a real child pointer from a thread:

    struct ThreadedNode {
    int val;
    ThreadedNode* left;
    ThreadedNode* right;
    bool leftThread;   // true = left points to predecessor
    bool rightThread;  // true = right points to successor

    ThreadedNode(int v)
        : val(v), left(nullptr), right(nullptr),
          leftThread(true), rightThread(true) {}
};
    Memory trade-off: We replace n+1 null pointers with useful threads, at the cost of 2 boolean flags per node (2 bits each). Net memory usage is typically better.

    Comparison with Standard Trees

    Feature Standard Binary Tree Threaded Binary Tree
    In-order traversalO(n) time, O(h) space (stack)O(n) time, O(1) space
    Find successorO(h) worst caseO(1) amortized
    Find predecessorO(h) worst caseO(1) (double threaded)
    Insert complexityStandard BST insertSlightly more complex (maintain threads)
    Null pointers wastedn + 10
    Extra storage per nodeNone2 boolean flags

    Summary

    • Threaded trees eliminate null pointer waste by storing in-order predecessor/successor links.
    • Right-threaded (single) is the most common; double-threaded adds predecessor links too.
    • Key benefit: stack-free, recursion-free in-order traversal in O(1) space.
    • Each node carries two boolean flags to distinguish children from threads.
