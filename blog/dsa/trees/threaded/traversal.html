    Threaded Tree Traversal

    March 2026 · 9 min read

    In-order Traversal (O(1) Space)

    The whole point of threading: traverse in-order without a stack, without recursion, using only the thread pointers.

    function threadedInorder(root):
    cur = leftmost(root) while cur != null: visit(cur) if cur.rightThread: cur = cur.right // follow thread to successor else: cur = leftmost(cur.right) // go to leftmost in right subtree

    ▶ Threaded In-order Traversal Animation

    Watch the cursor follow children and threads to traverse without any stack.

    C++ Implementation

    ThreadedNode* leftmost(ThreadedNode* node) {
    while (node && !node->leftThread)
        node = node->left;
    return node;
}

void threadedInorder(ThreadedNode* root) {
    ThreadedNode* cur = leftmost(root);
    while (cur) {
        visit(cur);
        if (cur->rightThread) {
            cur = cur->right;  // follow thread
        } else {
            cur = leftmost(cur->right);  // real right child
        }
    }
}

    Connection to Morris Traversal

    Morris traversal (covered in Tree Algorithms) is essentially building temporary threads on the fly, using them to traverse, and then removing them. A threaded tree has these links permanently.

    Morris vs Threaded:
    • Morris: Temporary threads. O(1) space, O(n) time. Modifies and restores the tree on the fly.
    • Threaded tree: Permanent threads. O(1) space, O(n) time. No modification during traversal.
    • Morris is preferred when you can't modify the tree structure permanently.

    Pre-order Traversal on Threaded Trees

    Pre-order is also possible without a stack on a threaded tree:

    void threadedPreorder(ThreadedNode* root) {
    ThreadedNode* cur = root;
    while (cur) {
        visit(cur);
        if (!cur->leftThread) {
            cur = cur->left;
        } else if (!cur->rightThread) {
            cur = cur->right;
        } else {
            // Follow threads up, then take right child
            while (cur && cur->rightThread)
                cur = cur->right;
            if (cur) cur = cur->right;
        }
    }
}

    Real-World Applications

    • Database indexes: B-tree leaf pages are often threaded for efficient range scans.
    • Compiler symbol tables: In-order iteration over identifiers without recursion overhead.
    • Real-time systems: Deterministic traversal with bounded stack usage (zero).
    • Memory-constrained devices: Embedded systems where stack space is extremely limited.

    Summary

    • Threaded traversal follows the leftmost path, then uses threads to jump to successors.
    • No stack, no recursion — just pointer chasing at O(1) extra space.
    • Morris traversal is the "on-the-fly threading" version of this concept.
    • Pre-order traversal is also possible stack-free on threaded trees.
