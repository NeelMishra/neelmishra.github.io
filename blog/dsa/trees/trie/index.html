    Tries: Prefix Trees

    March 2026 · 18 min read

    Imagine you're building an autocomplete system. A user types "app" and you need to instantly show all words starting with "app": apple, application, approve, appetite. How do you do this efficiently?

    A hash map can check if "apple" exists in O(1), but it can't find all words starting with "app" without scanning every key. A sorted array can binary-search for the prefix range, but insertions cost O(n). A BST comparison-sorts whole strings, paying O(m log n) where m is the string length.

    A trie (pronounced "try," from retrieval) solves this naturally: it stores strings character by character along tree edges, so every word sharing a prefix shares the same path from the root. Finding all words with prefix "app" means walking three edges and collecting everything below.

    The Structure

    A trie is a tree where:

    • The root represents the empty string.
    • Each edge is labeled with a character.
    • Each node represents the prefix formed by the characters on the path from root to that node.
    • Some nodes are marked as end-of-word (shown as filled circles below).

    Why Not Just Use a Hash Map?

    OperationHash MapSorted ArrayBSTTrie
    Exact lookupO(m)O(m log n)O(m log n)O(m)
    InsertO(m)O(n)O(m log n)O(m)
    Prefix searchO(n · m) ✗O(m log n + k)O(m log n + k)O(p + k) ✓
    Autocomplete✗PossiblePossibleNatural ✓
    Sorted iteration✗✓✓✓ (DFS)

    (m = key length, n = number of keys, p = prefix length, k = number of matches)

    The trie wins on prefix search: walk p edges for the prefix, then collect all descendants. No scanning, no sorting, no hash collisions. The time depends on the prefix length and the number of results, not the total size of the dataset.

    The Trie Node

    struct TrieNode {
    TrieNode* children[26];  // for lowercase a-z
    bool isEnd;              // marks end of a word

    TrieNode() : isEnd(false) {
        memset(children, 0, sizeof(children));
    }
};

    Each node has an array of 26 child pointers (for lowercase English). children[0] is the 'a' edge, children[1] is the 'b' edge, etc. A null pointer means "no word continues with that character from here."

    Space note: The 26-pointer array is simple but wastes memory when most children are null. We'll cover more space-efficient alternatives (hash maps per node, compressed tries) in the Variants post.

    Where Tries Are Used

    • Autocomplete: Phone keyboards, search engines, IDEs. Type a prefix, traverse the trie, collect suggestions.
    • Spell checkers: Check if a word exists in the dictionary. Suggest corrections by exploring nearby trie paths (edit distance).
    • IP routing: Longest prefix match on IP addresses. Each bit of the IP is an edge in a bitwise trie.
    • T9 predictive text: Map number sequences to all possible words (each number maps to 3-4 letters).
    • Genome sequence matching: Store DNA strings (alphabet = {A, C, G, T}).
    • File systems: Directory path lookup is essentially a trie over path components.
