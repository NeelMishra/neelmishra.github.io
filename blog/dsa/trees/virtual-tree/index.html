    Virtual Tree (Auxiliary Tree)

    April 2026 · 18 min read

    Introduction

    You’re given a tree with n = 105 nodes. There are q queries, and each query specifies a small set of k “important” nodes. You need to answer something about the subtree induced by those k nodes — minimum cost to connect them, sum of path weights, etc. The catch: k << n, and the sum of all ki across queries is bounded (say, ≤ 105).

    A naïve approach processes the entire n-node tree per query, giving O(qn) total — way too slow. What if we could compress the tree so that each query operates on a tree of only O(k) nodes, preserving all structural information we need?

    That is exactly what a Virtual Tree (also called Auxiliary Tree or """ in Chinese competitive programming) does.

    Key Idea: Given k important nodes on a tree of n nodes, we can build a compressed tree containing only O(k) nodes that preserves all ancestor–descendant relationships between the important nodes. This compressed tree is called the virtual tree.

    What is a Virtual Tree?

    Given a rooted tree T and a set S of k query nodes, the virtual tree V(S) is the minimal subtree of T that:

    1. Contains every node in S.
    2. Contains the pairwise LCA of every pair of nodes in S.
    3. Contains the root of T (or the LCA of the entire set S as the virtual root).
    4. Preserves the ancestor–descendant ordering of T.

    An edge (u, v) in the virtual tree represents the path from u to v in the original tree (with all intermediate non-important nodes compressed away). The weight of this virtual edge equals the sum of edge weights (or the number of edges) on the original path.

    Size bound: If |S| = k, then V(S) has at most 2k − 1 nodes. This is because k query nodes produce at most k − 1 distinct pairwise LCAs (when sorted by Euler tour order, consecutive pairs yield all necessary LCAs).

    Construction Algorithm

    The construction uses a beautiful stack-based algorithm that processes query nodes in Euler tour order (also called DFS-in-time order). Here’s the high-level approach:

    Algorithm Overview

    function buildVirtualTree(queryNodes):
    // Step 1: Sort by Euler tour entry time sort(queryNodes, by = tin[node]) // Step 2: Add LCAs of consecutive pairs merged = queryNodes.copy() for i = 0 to len(queryNodes) - 2: merged.add(LCA(queryNodes[i], queryNodes[i+1])) // Also add root if needed // Step 3: Deduplicate & sort merged set by tin merged = unique(sort(merged, by = tin)) // Step 4: Build tree with stack stack = [merged[0]] // virtual root for i = 1 to len(merged) - 1: u = merged[i] l = LCA(u, stack.top()) if l ≠ stack.top():   while stack.size() > 1 and tin[l] ≤ tin[stack.second()]:     addEdge(stack.second(), stack.top())     stack.pop()   if tin[l] < tin[stack.top()]:     addEdge(l, stack.top())     stack.pop()     stack.push(l) stack.push(u) // Flush remaining stack while stack.size() > 1: addEdge(stack.second(), stack.top()) stack.pop()

    Step-by-Step Explanation

    1. Sort by DFS entry time (tin): This ensures we process nodes in the same order a DFS would visit them, so ancestors always appear before descendants.
    2. Compute LCAs of consecutive pairs: A key insight — when sorted by tin, the LCA of any two query nodes equals the LCA of some pair of consecutive query nodes. So we only need k − 1 LCA computations.
    3. Stack-based tree building: We maintain a stack representing the current “rightmost path” from the virtual root. For each new node, we pop finished branches and link them, then push the new node.

    Why It Works

    Pairwise LCAs Suffice

    Claim: To preserve the tree structure among k query nodes, we only need to add their pairwise LCAs.

    Proof sketch: Consider the path between any two query nodes u and v. This path goes up from u to LCA(u, v) and down to v. Any branching point where different query nodes “split off” must be the LCA of some pair. If we include all such branching points (LCAs), we capture the complete topology.

    More formally: the Steiner tree of S in T (the minimal subtree connecting all nodes in S) consists of the paths between pairs of “adjacent” query nodes in Euler tour order, and every internal branching node on these paths is an LCA of some consecutive pair in the sorted order.

    At Most 2k − 1 Nodes

    Claim: The virtual tree has at most 2k − 1 nodes.

    Proof: We start with k query nodes and add at most k − 1 LCA nodes (one for each consecutive pair). Some LCAs may coincide with query nodes or with each other, so after deduplication we get at most k + (k − 1) = 2k − 1 nodes.

    In fact, the virtual tree is a tree on these ≤ 2k − 1 nodes, so it has ≤ 2k − 2 edges. Every operation (DP, DFS, etc.) on the virtual tree runs in O(k) time.

    Tight example: Consider a complete binary tree. If we pick all leaves on alternating positions, every internal node on the path between consecutive leaves is a distinct LCA. The virtual tree can indeed have exactly 2k − 1 nodes.

    Implementation Details

    Prerequisites

    • Euler Tour: Precompute tin[v] (entry time) for every node via DFS. This is used to sort query nodes and to determine ancestor relationships (isAncestor(u, v) checks if tin[u] ≤ tin[v] ≤ tout[u]).
    • LCA with Binary Lifting: Precompute up[v][j] = 2j-th ancestor of v. LCA queries run in O(log n). Alternatively, use Sparse Table + Euler tour for O(1) LCA after O(n log n) preprocessing.
    • Depth / Distance: Precompute depth[v] or dist[v] (distance from root) to compute edge weights in the virtual tree as dist(u,v) = depth[u] + depth[v] − 2 · depth[LCA(u,v)].

    The Sorting Trick

    The crucial invariant: when query nodes are sorted by tin[], the LCA of any two query nodes is equal to the LCA of some pair of consecutive nodes in this sorted order. This is because the DFS entry times form an ordering where “close” nodes in the tour are also “close” in the tree.

    Formal proof: Suppose nodes a, b, c appear in tin-order with a < b < c. Then LCA(a, c) is an ancestor of LCA(a, b) (or they’re equal). So the LCA of a and c is either LCA(a, b) or LCA(b, c) or an ancestor of both — and any such ancestor is already the LCA of some consecutive pair.

    Stack-Based Building

    The stack maintains a path from the virtual root downward. When we process the next node u:

    1. Compute l = LCA(u, stack.top()).
    2. If l is the same as stack.top(), then u is a descendant — just push u.
    3. Otherwise, u is in a different subtree. Pop nodes from the stack (linking them as children) until we find where l fits. If l isn’t already in the stack, push it.
    4. Push u.

    After processing all nodes, flush the remaining stack to link the last branch.

    Example: Steiner Tree Problem

    Problem Statement

    Given a weighted tree with n nodes, answer q queries. Each query gives k nodes. Find the minimum total edge weight of a connected subtree that contains all k given nodes.

    Constraints: n ≤ 105, ∑ki ≤ 105

    Approach

    1. Build the virtual tree for the k query nodes.
    2. Each edge in the virtual tree corresponds to a path in the original tree. The edge weight = distance in original tree.
    3. The answer is simply the sum of all edge weights in the virtual tree, because the virtual tree is the Steiner tree of the query nodes!
    Why? The Steiner tree in a tree is unique — it’s the minimal subtree connecting all query nodes. The virtual tree captures exactly this: the query nodes, their pairwise LCAs (the branching points), and the paths between them. Every edge in the virtual tree represents a necessary path segment.

    Step-by-Step Walkthrough

    Consider a tree where each edge has weight 1. Query nodes S = {8, 10, 11} on our 11-node tree:

    1. Euler tour order: tin values give us the sorted order: [8, 10, 11].
    2. LCAs of consecutive pairs: LCA(8, 10) = 2, LCA(10, 11) = 1.
    3. Merged & sorted: [1, 2, 8, 10, 11] (5 nodes ≤ 2·3 − 1).
    4. Virtual tree edges: 1→2 (weight 1), 1→6 (weight 2), 2→8 (weight 2), 2→10 (weight 2), 6→11 (weight 1).
    5. Answer: 1 + 2 + 2 + 2 + 1 = 8 (total edges on the Steiner tree paths).

    C++ Implementation

    Here’s a complete, contest-ready implementation. The code precomputes LCA with binary lifting and builds the virtual tree using the stack-based approach.

    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
const int LOG  = 18;

int n, q;
vector<pair<int,long long>> adj[MAXN]; // adj[u] = {v, weight}
int tin[MAXN], tout[MAXN], timer_cnt;
int dep[MAXN], up[MAXN][LOG];
long long dist[MAXN]; // distance from root

// ── Euler Tour + Binary Lifting ──
void dfs(int u, int p, int d, long long dd) {
    tin[u] = ++timer_cnt;
    dep[u] = d;
    dist[u] = dd;
    up[u][0] = p;
    for (int j = 1; j < LOG; j++)
        up[u][j] = up[up[u][j-1]][j-1];
    for (auto [v, w] : adj[u]) {
        if (v == p) continue;
        dfs(v, u, d + 1, dd + w);
    }
    tout[u] = timer_cnt;
}

bool isAnc(int u, int v) {
    return tin[u] <= tin[v] && tout[v] <= tout[u];
}

int lca(int u, int v) {
    if (isAnc(u, v)) return u;
    if (isAnc(v, u)) return v;
    for (int j = LOG - 1; j >= 0; j--)
        if (!isAnc(up[u][j], v))
            u = up[u][j];
    return up[u][0];
}

// ── Virtual Tree Construction ──
vector<pair<int,long long>> vAdj[MAXN]; // virtual tree adjacency
vector<int> vNodes;                      // nodes in the virtual tree

void buildVirtualTree(vector<int>& nodes) {
    // Sort by Euler tour entry time
    sort(nodes.begin(), nodes.end(),
         [](int a, int b){ return tin[a] < tin[b]; });

    // Add LCAs of consecutive pairs
    int sz = nodes.size();
    for (int i = 0; i + 1 < sz; i++)
        nodes.push_back(lca(nodes[i], nodes[i+1]));

    // Deduplicate and sort
    sort(nodes.begin(), nodes.end(),
         [](int a, int b){ return tin[a] < tin[b]; });
    nodes.erase(unique(nodes.begin(), nodes.end()), nodes.end());

    // Stack-based tree construction
    vNodes = nodes;
    for (int u : vNodes) vAdj[u].clear();

    stack<int> stk;
    stk.push(nodes[0]);

    for (int i = 1; i < (int)nodes.size(); i++) {
        int u = nodes[i];
        int l = lca(u, stk.top());

        if (l != stk.top()) {
            // Pop and link until we find where l fits
            while (stk.size() > 1) {
                int top = stk.top();
                stk.pop();
                int sec = stk.top();
                if (tin[l] <= tin[sec]) {
                    long long w = dist[top] - dist[sec];
                    vAdj[sec].push_back({top, w});
                    vAdj[top].push_back({sec, w});
                } else {
                    long long w = dist[top] - dist[l];
                    vAdj[l].push_back({top, w});
                    vAdj[top].push_back({l, w});
                    stk.push(l);
                    break;
                }
            }
        }
        stk.push(u);
    }

    // Flush remaining stack
    while (stk.size() > 1) {
        int top = stk.top(); stk.pop();
        int sec = stk.top();
        long long w = dist[top] - dist[sec];
        vAdj[sec].push_back({top, w});
        vAdj[top].push_back({sec, w});
    }
}

// ── Example: Steiner Tree (sum of virtual edge weights) ──
long long steinerCost() {
    long long total = 0;
    int root = vNodes[0];
    // Simple DFS on virtual tree
    function<void(int, int)> dfs2 = [&](int u, int par) {
        for (auto [v, w] : vAdj[u]) {
            if (v == par) continue;
            total += w;
            dfs2(v, u);
        }
    };
    dfs2(root, -1);
    return total;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n;
    for (int i = 0; i < n - 1; i++) {
        int u, v; long long w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }
    dfs(1, 0, 0, 0);

    cin >> q;
    while (q--) {
        int k; cin >> k;
        vector<int> nodes(k);
        for (int& x : nodes) cin >> x;

        buildVirtualTree(nodes);
        cout << steinerCost() << "\n";
    }
    return 0;
}
    Contest tip: Don’t forget to clear the virtual adjacency lists! Use vAdj[u].clear() for each node in vNodes at the start of buildVirtualTree, rather than clearing the entire array each query.

    Complexity Analysis

    PhasePer QueryDetails
    PreprocessingO(n log n)One-time: Euler tour O(n) + Binary lifting O(n log n)
    Sort query nodesO(k log k)Sort k nodes by tin[] value
    LCA computationsO(k log n)k − 1 consecutive LCA queries, each O(log n)
    Merge & dedupO(k log k)Sort and unique the merged set of ≤ 2k nodes
    Stack buildO(k)Each node pushed/popped at most once
    DP / DFS on VTO(k)Virtual tree has ≤ 2k − 1 nodes
    Total per queryO(k log n)Dominated by LCA queries (or O(k log k) if using O(1) LCA)
    Total over all queriesO(∑ki · log n)With ∑ki ≤ 105, this is very fast
    Optimization: Using Sparse Table for O(1) LCA queries, total per query drops to O(k log k) (dominated by sorting). With radix sort on tin[] values, this can be further reduced to O(k) per query.

    When to Use Virtual Trees

    Virtual trees are the right tool when:

    • Multiple queries, each on a small subset: q queries, each involving ki nodes, with ∑ki bounded. Without a virtual tree you’d process O(n) per query; with it, O(ki) per query.
    • Tree DP on a subset: You need to run a DP (e.g., minimum-cost Steiner tree, painting subtrees, counting paths) but only care about specific nodes.
    • Offline + online hybrid: Precompute the heavy structure (LCA, Euler tour) once, then answer each query online by building a small virtual tree.

    Common problem patterns:

    • Steiner tree on a tree (min-cost subtree connecting k nodes)
    • DP on a subset of nodes (e.g., assign colors to k nodes, minimize cost)
    • Counting paths/edges between specified nodes
    • Subtree queries where only a few nodes are “active”
    • Problems that say “∑ki ≤ C” — almost always a virtual tree hint
    Red flag in problem statements: “It is guaranteed that the sum of ki over all queries does not exceed 2 × 105.” This almost certainly calls for a virtual tree approach.

    Summary

    • A virtual tree compresses a tree of n nodes to O(k) nodes by keeping only the query nodes and their pairwise LCAs.
    • Construction: sort by Euler tour order, compute consecutive LCAs, build with a stack — all in O(k log n).
    • The virtual tree has ≤ 2k − 1 nodes, making any DP/DFS on it O(k).
    • Classic application: Steiner tree = just sum virtual edge weights.
    • Look for the hint “∑ki ≤ C” in problem constraints.
