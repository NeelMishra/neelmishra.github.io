    Small to Large Merging (DSU on Tree)

    July 2025 · 22 min read

    Introduction

    Consider this deceptively simple problem: given a rooted tree where every vertex has a color, for each vertex count the number of distinct colors in its subtree.

    The brute-force approach is clear—run a DFS from every vertex and collect colors in its subtree. But if you do that naively, the total work is O(n²) in the worst case (imagine a long chain). For n = 105, that’s far too slow.

    Key question: Can we answer subtree aggregate queries in better than O(n²) without heavy data structures like segment trees on Euler tour?

    The answer is Small to Large Merging, also called DSU on Tree or the heavy-light merge technique. It solves a broad class of subtree aggregate problems in O(n log n) total time with elegant simplicity.

    The idea applies whenever you need to compute, for each vertex, some aggregate over all vertices in its subtree—distinct counts, frequency maximums, sum of values by color, and more. The technique has two equivalent formulations:

    1. Set merging with swap — maintain a set/map per vertex, always merge the smaller into the larger.
    2. Euler Tour + heavy child — exploit heavy-light decomposition to avoid redundant work during DFS.

    Both achieve the same O(n log n) complexity. Let’s understand why.

    The Key Insight

    When we merge two sets, the cost is proportional to the size of the smaller set (we iterate over the smaller one and insert each element into the larger one). The critical observation is:

    Small-to-large rule: Always merge the smaller set into the larger set. Swap the pointers if needed so the “current” set is always the larger one.

    This simple rule gives us an amortized total cost of O(n log n) across all merges in the entire tree.

    Why O(n log n)?

    🔎 Proof sketch

    Track a single element x. Every time x’s set is merged into a larger one, the set containing x at least doubles in size (because we merged the smaller into the larger, so the result is ≥ 2 × |smaller|).

    Since the largest possible set has n elements, x can be “moved” at most log2 n times before the set reaches size n.

    There are n elements total, each moving at most O(log n) times. Therefore, the total number of insertions across all merges is O(n log n).

    This is the same principle behind union-by-size in Disjoint Set Union (DSU)—hence the name “DSU on Tree.”

    Algorithm Walkthrough

    Let’s trace the algorithm on a concrete tree. Each node has a color, and we want to find the number of distinct colors in every subtree.

    Red Blue Yellow Green Purple

    We process the tree bottom-up via DFS. Each leaf starts with a set containing just its own color. When we process an internal node, we merge all children’s sets into one, always pouring the smaller set into the larger. After merging, we add the current node’s own color and record answer[v] = set.size().

    Implementation Strategy

    The key implementation trick is to swap pointers rather than copy data. Each vertex owns a pointer/reference to a set (or map). After processing all children of a vertex v:

    1. Find the child c with the largest set.
    2. Swap set[v] with set[c] so v now owns the big set.
    3. For each other child c′, iterate over set[c′] and insert each element into set[v].
    4. Insert v’s own color into set[v].
    5. Record the answer: ans[v] = set[v].size().
    function dfs(v, parent):
    for child in adj[v]:
    if child ≠ parent: dfs(child, v)
    // find child with largest set
    heavy ← child with max |set[child]|
    swap(set[v], set[heavy]) // O(1) pointer swap
    for other child c ≠ heavy:
    for elem in set[c]: // iterate smaller set
      set[v].insert(elem)
    set[v].insert(color[v])
    ans[v] ← |set[v]|

    The swap is O(1) because std::set (and std::map) swap their internal pointers in constant time. This is what makes the technique efficient—we never copy the large set.

    Euler Tour + Heavy Child Variant

    The set-merging approach is intuitive but uses O(n) memory for the sets. An alternative formulation uses a single global data structure and avoids explicit merging by exploiting the DFS order.

    Heavy child decomposition

    For each internal node, the heavy child is the child with the largest subtree size. All other children are light children.

    The algorithm

    Instead of maintaining separate sets, we use a single global frequency array (or map). For each node v:

    1. Process all light children first: DFS into each light child, compute its answer, then undo (remove) all its contributions from the global structure.
    2. Process the heavy child last: DFS into the heavy child, compute its answer, but keep its contributions in the global structure.
    3. Re-add light children: Iterate over all vertices in each light child’s subtree and add them back to the global structure.
    4. Add v itself to the global structure, then record ans[v].
    Why keep the heavy child? The heavy child’s subtree is the largest, so re-traversing it would be the most expensive. By keeping its data and only re-adding the smaller (light) subtrees, we get the same O(n log n) guarantee.

    C++ Implementation

    Approach 1: Set merging with swap

    This is the most straightforward implementation. We maintain a std::set<int> per vertex and swap + merge bottom-up.

    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int color[MAXN], ans[MAXN];
set<int> *st[MAXN];

void dfs(int v, int p) {
    // create a new set for this vertex
    st[v] = new set<int>();
    st[v]->insert(color[v]);

    for (int u : adj[v]) {
        if (u == p) continue;
        dfs(u, v);

        // always merge smaller into larger
        if (st[u]->size() > st[v]->size())
            swap(st[v], st[u]);

        // pour smaller set into larger
        for (int c : *st[u])
            st[v]->insert(c);

        delete st[u];
        st[u] = nullptr;
    }
    ans[v] = st[v]->size();
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> color[i];
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    dfs(1, 0);

    for (int i = 1; i <= n; i++)
        cout << ans[i] << " \n"[i == n];
}
    Why pointers? Using set<int>* allows O(1) swaps. If you used set<int> directly, swap() would still be O(1) for std::set (it swaps internal pointers), but the pointer approach makes ownership semantics clearer and allows manual memory management with delete.

    Approach 2: Euler tour + heavy child (global counter)

    This approach uses a single global frequency array. We precompute the Euler tour to quickly iterate over any subtree, and use sz[] to identify heavy children.

    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int color[MAXN], ans[MAXN];
int sz[MAXN], in_time[MAXN], euler[MAXN], timer_val;
int cnt[MAXN];  // global color frequency
int distinct;   // global distinct counter

void calc_sz(int v, int p) {
    sz[v] = 1;
    for (int u : adj[v])
        if (u != p) { calc_sz(u, v); sz[v] += sz[u]; }
}

void build_euler(int v, int p) {
    euler[timer_val] = v;
    in_time[v] = timer_val++;
    for (int u : adj[v])
        if (u != p) build_euler(u, v);
}

void add(int v) {
    cnt[color[v]]++;
    if (cnt[color[v]] == 1) distinct++;
}

void remove_node(int v) {
    cnt[color[v]]--;
    if (cnt[color[v]] == 0) distinct--;
}

void dfs(int v, int p, bool keep) {
    // find heavy child
    int heavy = -1, mx = 0;
    for (int u : adj[v]) {
        if (u != p && sz[u] > mx) {
            mx = sz[u]; heavy = u;
        }
    }

    // 1. process light children: solve then remove
    for (int u : adj[v])
        if (u != p && u != heavy)
            dfs(u, v, false);

    // 2. process heavy child: solve and keep
    if (heavy != -1)
        dfs(heavy, v, true);

    // 3. re-add light children's subtrees
    for (int u : adj[v]) {
        if (u != p && u != heavy) {
            for (int t = in_time[u]; t < in_time[u] + sz[u]; t++)
                add(euler[t]);
        }
    }
    add(v);  // add current node

    ans[v] = distinct;

    // 4. if v is a light child, undo everything
    if (!keep) {
        for (int t = in_time[v]; t < in_time[v] + sz[v]; t++)
            remove_node(euler[t]);
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> color[i];
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    calc_sz(1, 0);
    timer_val = 0;
    build_euler(1, 0);
    distinct = 0;
    dfs(1, 0, true);

    for (int i = 1; i <= n; i++)
        cout << ans[i] << " \n"[i == n];
}

    Example: Distinct Colors per Subtree

    This is the classic problem: CSES – Distinct Colors. Given a rooted tree with colored nodes, output the number of distinct colors in each subtree.

    Full solution (Approach 1)

    #include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<int> c(n + 1);
    for (int i = 1; i <= n; i++) cin >> c[i];

    vector<vector<int>> adj(n + 1);
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    vector<int> ans(n + 1);
    vector<set<int>*> st(n + 1, nullptr);

    // iterative DFS with post-order processing
    vector<int> order, parent(n + 1, 0);
    vector<bool> visited(n + 1, false);
    stack<int> stk;
    stk.push(1); visited[1] = true;
    while (!stk.empty()) {
        int v = stk.top(); stk.pop();
        order.push_back(v);
        for (int u : adj[v]) {
            if (!visited[u]) {
                visited[u] = true;
                parent[u] = v;
                stk.push(u);
            }
        }
    }

    // process in reverse order (leaves first)
    reverse(order.begin(), order.end());
    for (int v : order) {
        st[v] = new set<int>();
        st[v]->insert(c[v]);

        for (int u : adj[v]) {
            if (u == parent[v] || !st[u]) continue;
            if (st[u]->size() > st[v]->size())
                swap(st[v], st[u]);
            for (int x : *st[u])
                st[v]->insert(x);
            delete st[u];
            st[u] = nullptr;
        }
        ans[v] = st[v]->size();
    }

    for (int i = 1; i <= n; i++)
        cout << ans[i] << " \n"[i == n];
}

    Example: Most Frequent Color

    Another classic application: for each vertex, find the sum of all colors that appear the maximum number of times in its subtree (CF 600E – Lomsat gelral).

    Here we need not just distinct counting but frequency tracking. The Euler tour approach is ideal since we maintain a global frequency array and can easily track the maximum frequency and its color sum.

    Key modifications

    • Maintain cnt[c] = frequency of color c in the current subtree.
    • Track maxFreq = current maximum frequency.
    • Track sumOfMax = sum of all colors with frequency equal to maxFreq.
    • On add(v): increment cnt[color[v]]. If it exceeds maxFreq, update maxFreq and set sumOfMax = color[v]. If it equals maxFreq, add color[v] to sumOfMax.
    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
vector<int> adj[MAXN];
int color[MAXN], sz[MAXN];
int in_time[MAXN], euler_arr[MAXN], timer_val;
long long ans[MAXN];
int cnt[MAXN];
int maxFreq;
long long sumOfMax;

void calc_sz(int v, int p) {
    sz[v] = 1;
    for (int u : adj[v])
        if (u != p) { calc_sz(u, v); sz[v] += sz[u]; }
}

void build_euler(int v, int p) {
    euler_arr[timer_val] = v;
    in_time[v] = timer_val++;
    for (int u : adj[v])
        if (u != p) build_euler(u, v);
}

void add(int v) {
    cnt[color[v]]++;
    if (cnt[color[v]] > maxFreq) {
        maxFreq = cnt[color[v]];
        sumOfMax = color[v];
    } else if (cnt[color[v]] == maxFreq) {
        sumOfMax += color[v];
    }
}

void remove_node(int v) {
    cnt[color[v]]--;
}

void dfs(int v, int p, bool keep) {
    int heavy = -1, mx = 0;
    for (int u : adj[v])
        if (u != p && sz[u] > mx) { mx = sz[u]; heavy = u; }

    // process light children
    for (int u : adj[v])
        if (u != p && u != heavy)
            dfs(u, v, false);

    // process heavy child (keep its data)
    if (heavy != -1)
        dfs(heavy, v, true);

    // re-add light subtrees
    for (int u : adj[v])
        if (u != p && u != heavy)
            for (int t = in_time[u]; t < in_time[u] + sz[u]; t++)
                add(euler_arr[t]);

    add(v);
    ans[v] = sumOfMax;

    if (!keep) {
        // undo: remove this entire subtree
        for (int t = in_time[v]; t < in_time[v] + sz[v]; t++)
            remove_node(euler_arr[t]);
        maxFreq = 0;
        sumOfMax = 0;
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> color[i];
    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    calc_sz(1, 0);
    timer_val = 0;
    build_euler(1, 0);
    maxFreq = 0; sumOfMax = 0;
    dfs(1, 0, true);

    for (int i = 1; i <= n; i++)
        cout << ans[i] << " \n"[i == n];
}
    Subtlety: When we undo (remove) a light subtree, we can’t efficiently reverse the maxFreq/sumOfMax computation. Instead, we reset maxFreq = 0 and sumOfMax = 0 after removing all nodes. This works because we recompute these values when we add nodes again for the parent. The heavy child’s contribution stays, and light subtrees get re-added.

    Complexity Analysis

    Set merging approach

    As proven earlier, each element is moved at most O(log n) times. Each move involves one set::insert which is O(log n) for a balanced BST. Total: O(n log² n).

    If you replace std::set with an std::unordered_set, insertions become amortized O(1), giving O(n log n) total—but with worse cache behavior and higher constant factor.

    Euler tour approach

    Each vertex is added and removed at most O(log n) times total. Why? A vertex v is added once for each ancestor where v belongs to a light subtree. The number of light edges on any root-to-leaf path is at most O(log n) (because each light edge leads to a subtree of size ≤ half the parent’s subtree). Each add/remove is O(1) with array access. Total: O(n log n).

    Space complexity

    Both approaches use O(n) space. The set approach stores each element in exactly one set at any time. The Euler tour approach uses a single global array of size equal to the number of distinct colors.
