    Red-Black Deletion

    March 2026 · 26 min read

    Deletion is the most involved operation on a red-black tree. But the same principle applies: do the BST delete, then fix property violations. The key concept unique to deletion is double-black.

    The Double-Black Concept

    When we remove a black node, every path through that node loses one black node. This violates property 5 (equal black-height). To track this, we imagine the replacement node carries an "extra black." A node that's already black becomes double-black. A red node that absorbs the extra black just becomes black (problem solved instantly).

    The goal of the fix-up is to eliminate the double-black by either absorbing the extra black into a red neighbor (making it black) or propagating it up to the root (where it disappears, reducing the tree's overall black-height by 1).

    The Four Fix-Up Cases

    Let X be the double-black node, S be its sibling, and P be its parent.

    Case 1: Sibling S is Red

    Rotate P toward X and swap colors of P and S. This converts to cases 2, 3, or 4 (sibling becomes black).

    Case 2: Sibling S is Black, Both S's Children are Black

    Remove one black from X and S (make S red), push the extra black up to P. If P was red, it becomes black (done). If P was black, P becomes double-black (repeat cases from P).

    Case 3: Sibling S is Black, S's Near Child is Red, Far Child is Black

    Rotate S toward its red near child and swap colors. This converts to Case 4.

    Case 4: Sibling S is Black, S's Far Child is Red

    This is the terminal case. Rotate P toward X, color S with P's color, color P black, color S's far child black. The double-black is eliminated. Done.

    Deletion Case Summary

    CaseSiblingSibling's childrenActionResult
    1RedAnyRotate P, swap P/S colorsConvert to 2/3/4
    2BlackBoth blackRecolor S red, push extra black upMay cascade
    3BlackNear red, far blackRotate S, swap colorsConvert to Case 4
    4BlackFar redRotate P, recolorDone!
    At most 3 rotations: Case 1 does 1 rotation and leads to 2/3/4. Case 3 does 1 rotation and leads to Case 4. Case 4 does 1 rotation and terminates. Total: at most 3 rotations. Case 2 only recolors (no rotation) and may cascade, but cascading only does recolorings, not rotations. This is why red-black delete is at most 3 rotations.

    Complete Delete Code

    void rbDelete(RBTree& T, RBNode* z) {
    RBNode* y = z;
    Color yOrigColor = y->color;
    RBNode* x;

    if (z->left == T.nil) {
        x = z->right;
        transplant(T, z, z->right);
    } else if (z->right == T.nil) {
        x = z->left;
        transplant(T, z, z->left);
    } else {
        y = minimum(z->right);  // in-order successor
        yOrigColor = y->color;
        x = y->right;
        if (y->parent == z) {
            x->parent = y;
        } else {
            transplant(T, y, y->right);
            y->right = z->right;
            y->right->parent = y;
        }
        transplant(T, z, y);
        y->left = z->left;
        y->left->parent = y;
        y->color = z->color;
    }

    if (yOrigColor == BLACK)
        deleteFixup(T, x);  // x might be double-black
}

void deleteFixup(RBTree& T, RBNode* x) {
    while (x != T.root && x->color == BLACK) {
        if (x == x->parent->left) {
            RBNode* s = x->parent->right;  // sibling
            if (s->color == RED) {           // Case 1
                s->color = BLACK;
                x->parent->color = RED;
                leftRotate(T, x->parent);
                s = x->parent->right;
            }
            if (s->left->color == BLACK && s->right->color == BLACK) {
                s->color = RED;              // Case 2
                x = x->parent;
            } else {
                if (s->right->color == BLACK) {
                    s->left->color = BLACK;  // Case 3
                    s->color = RED;
                    rightRotate(T, s);
                    s = x->parent->right;
                }
                s->color = x->parent->color; // Case 4
                x->parent->color = BLACK;
                s->right->color = BLACK;
                leftRotate(T, x->parent);
                x = T.root;  // done!
            }
        } else {
            // Mirror (x is right child)
            // ... same logic with left/right swapped
        }
    }
    x->color = BLACK;
}
