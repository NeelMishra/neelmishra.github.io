    Red-Black Insertion

    March 2026 · 24 min read

    We know the five properties. Now let's insert into a red-black tree. The process: BST insert, color the new node red, then fix any violations.

    Why New Nodes Are Red

    We always color new nodes red. Why? Because inserting a red node never violates property 5 (black-height). It doesn't add a black node to any path, so all path counts stay the same.

    The only property that might break is property 4 (no red-red). If the parent of our new red node is also red, we have a violation. But that's a local problem we can fix with recoloring and at most 2 rotations.

    If we inserted a black node instead, we'd violate property 5 on every path through that node. That's a global problem, much harder to fix.

    The Three Fix-Up Cases

    After inserting node Z (red), if Z's parent P is also red, we have a red-red violation. We look at Z's uncle U (P's sibling) to decide what to do:

    Case 1: Uncle is Red → Recolor

    Both P and U are red. This is a "4-node" in the 2-3-4 view. We split it by recoloring: make P and U black, make grandparent G red. Then move up to G and repeat (G might now violate with its parent).

    Case 2: Uncle is Black, Z is Inner Child → Rotate to Case 3

    Z is on the "inside" (e.g., Z is right child of P, and P is left child of G). This is a zig-zag. We rotate Z and P to straighten the zig-zag into a line, turning it into Case 3.

    Case 3: Uncle is Black, Z is Outer Child → Rotate + Recolor

    Z is on the "outside" (e.g., Z is left child of P, P is left child of G). We rotate G in the opposite direction and swap colors of P and G. Done. No more violations.

    Case Summary

    CaseUncle colorZ positionActionContinues?
    1RedAnyRecolor P, U, GYes (check G)
    2BlackInner childRotate Z-P (straighten)Becomes Case 3
    3BlackOuter childRotate G + recolor P/GNo (done!)
    The pattern: Case 1 pushes the problem up (recoloring). Cases 2+3 fix it with at most 2 rotations and 1 recoloring. The worst case is O(log n) recolorings (Case 1 cascading up) plus at most 2 rotations total.

    Complete Insert Code

    void rbInsert(RBTree& T, RBNode* z) {
    // 1. Standard BST insert
    RBNode* y = T.nil;
    RBNode* x = T.root;
    while (x != T.nil) {
        y = x;
        x = (z->key < x->key) ? x->left : x->right;
    }
    z->parent = y;
    if (y == T.nil) T.root = z;
    else if (z->key < y->key) y->left = z;
    else y->right = z;

    z->left = z->right = T.nil;
    z->color = RED;

    // 2. Fix violations
    insertFixup(T, z);
}

void insertFixup(RBTree& T, RBNode* z) {
    while (z->parent->color == RED) {
        if (z->parent == z->parent->parent->left) {
            RBNode* uncle = z->parent->parent->right;
            if (uncle->color == RED) {
                // Case 1: uncle is red
                z->parent->color = BLACK;
                uncle->color = BLACK;
                z->parent->parent->color = RED;
                z = z->parent->parent;
            } else {
                if (z == z->parent->right) {
                    // Case 2: inner child
                    z = z->parent;
                    leftRotate(T, z);
                }
                // Case 3: outer child
                z->parent->color = BLACK;
                z->parent->parent->color = RED;
                rightRotate(T, z->parent->parent);
            }
        } else {
            // Mirror: parent is right child of grandparent
            // (same logic with left/right swapped)
            RBNode* uncle = z->parent->parent->left;
            if (uncle->color == RED) {
                z->parent->color = BLACK;
                uncle->color = BLACK;
                z->parent->parent->color = RED;
                z = z->parent->parent;
            } else {
                if (z == z->parent->left) {
                    z = z->parent;
                    rightRotate(T, z);
                }
                z->parent->color = BLACK;
                z->parent->parent->color = RED;
                leftRotate(T, z->parent->parent);
            }
        }
    }
    T.root->color = BLACK;  // ensure root is always black
}
