    Lowest Common Ancestor (LCA)

    July 2025 · 20 min read

    Introduction & Problem Statement

    Given a rooted tree with n nodes and two nodes u and v, the Lowest Common Ancestor (LCA) is the deepest node that is an ancestor of both u and v. A node is considered its own ancestor.

    More formally: LCA(u, v) is the node w such that w is an ancestor of both u and v, and no descendant of w has this property.

    LCA is a fundamental primitive in competitive programming. Nearly every “path query on a tree” reduces to it: distance between nodes, path aggregation (min, max, sum, xor), tree isomorphism, and many more. Two efficient approaches dominate contest use:

    • Binary Lifting — O(n log n) preprocess, O(log n) query. Simple, versatile, easy to extend with path aggregates.
    • Euler Tour + Sparse Table — O(n log n) preprocess, O(1) query. Fastest possible queries; great when query count is very large.

    Naive Approach — O(n) per Query

    The simplest approach: compute the path from root to u and root to v, then find the deepest common node. Alternatively, walk both nodes up to the same depth, then walk both up together until they meet.

    function naiveLCA(u, v):
    // Bring both to the same depth while depth[u] > depth[v]: u = parent[u] while depth[v] > depth[u]: v = parent[v] // Walk up together while u ≠ v: u = parent[u] v = parent[v] return u

    Time: O(n) per query in the worst case (a skewed tree). Fine for a single query, but with Q queries on a tree of size n, the total O(nQ) is too slow when both n and Q reach 105–106.

    Naive LCA(7, 5) walk-through

    Node 7 is at depth 3, node 5 is at depth 2. First bring node 7 up to depth 2: parent[7] = 3. Now both are at depth 2. They differ (3 ≠ 5), so walk up: parent[3] = 1, parent[5] = 2. Still differ (1 ≠ 2). Walk up again: parent[1] = 0, parent[2] = 0. Match! LCA = 0.

    LCA(7, 5) = 0

    Binary Lifting

    Binary lifting is the workhorse LCA technique in competitive programming. The key insight: instead of climbing one edge at a time, precompute “express lanes” that let us jump 1, 2, 4, 8, … 2k steps at once. Any distance can be decomposed into powers of two, so we can reach any ancestor in O(log n) jumps.

    Preprocessing: The Lifting Table

    We build a table up[v][k] = the 2k-th ancestor of node v.

    • up[v][0] = parent[v] (base case — 20 = 1 step up)
    • up[v][k] = up[ up[v][k-1] ][k-1] (jump 2k-1 twice = 2k)

    For a tree with n nodes, we need LOG = ⌊log2(n)⌋ + 1 levels. The table has n × LOG entries, built in O(n log n) time.

    LCA Query in O(log n)

    The query has two phases:

    1. Equalise depths: lift the deeper node up until both are at the same depth. Decompose the depth difference into powers of two and jump accordingly.
    2. Binary search for LCA: for k from LOG-1 down to 0, if up[u][k] ≠ up[v][k], jump both up by 2k. At the end, u and v are children of the LCA, so return up[u][0].
    function lca(u, v):
    if depth[u] < depth[v]: swap(u, v) // Phase 1: bring u up to depth of v diff = depth[u] - depth[v] for k = LOG-1 down to 0: if diff & (1 << k): u = up[u][k] if u == v: return u // Phase 2: climb together for k = LOG-1 down to 0: if up[u][k] ≠ up[v][k]:   u = up[u][k]; v = up[v][k] return up[u][0]

    K-th Ancestor Queries

    A bonus of binary lifting: we can answer “what is the k-th ancestor of node v?” in O(log n). Decompose k into its binary representation and jump through the set bits:

    function kthAncestor(v, k):
    for i = 0 to LOG-1: if k & (1 << i): v = up[v][i] return v

    For example, the 3rd ancestor of node 7: 3 = 21 + 20. Jump: up[7][0] = 3, then up[3][1] = 0. So the 3rd ancestor of 7 is node 0 (the root). ✓

    C++ Implementation — Binary Lifting

    const int MAXN = 2e5 + 5;
const int LOG = 18; // ceil(log2(2e5))

int up[MAXN][LOG], depth[MAXN];
vector<int> adj[MAXN];

void dfs(int v, int p, int d) {
    up[v][0] = p;
    depth[v] = d;
    for (int k = 1; k < LOG; k++)
        up[v][k] = up[up[v][k-1]][k-1];
    for (int u : adj[v])
        if (u != p) dfs(u, v, d + 1);
}

int lca(int u, int v) {
    if (depth[u] < depth[v]) swap(u, v);
    int diff = depth[u] - depth[v];
    for (int k = 0; k < LOG; k++)
        if ((diff >> k) & 1) u = up[u][k];
    if (u == v) return u;
    for (int k = LOG - 1; k >= 0; k--)
        if (up[u][k] != up[v][k]) {
            u = up[u][k];
            v = up[v][k];
        }
    return up[u][0];
}

// Usage: dfs(root, root, 0); then lca(u, v);

    Tip: set up[root][0] = root (the root is its own parent) so that jumping beyond the root stays at the root, avoiding out-of-bounds issues.

    Euler Tour + Sparse Table (RMQ)

    This approach reduces LCA to a Range Minimum Query (RMQ) problem, which can be solved in O(1) per query using a Sparse Table. The idea:

    1. Euler Tour: traverse the tree with DFS, recording each node when you enter and when you return from each child. This produces an array of length 2n − 1.
    2. Reduction to RMQ: record the depth at each position. LCA(u, v) is the node with the minimum depth between the first occurrences of u and v in the Euler tour.
    3. Sparse Table: preprocess the depth array for O(1) range minimum queries.

    Reduction to RMQ

    To find LCA(7, 5): look at first[7]=3, first[5]=10. The range is [3, 10]. The minimum depth in this range is depth[8]=0 at index 8, where euler[8]=0. So LCA(7,5) = 0.

    To find LCA(7, 4): first[7]=3, first[4]=6. Range [3, 6]. Min depth = depth[5]=1 at index 5, euler[5]=1. So LCA(7,4) = 1. ✓

    Sparse Table for O(1) RMQ

    A Sparse Table answers range minimum queries in O(1) after O(n log n) preprocessing. For each index i and power k, store the index of the minimum element in the range [i, i + 2k − 1].

    // Build
    for i = 0 to m-1: sparse[i][0] = i
    for k = 1 to LOG:
    for i = 0 to m - (1 << k): l = sparse[i][k-1], r = sparse[i + (1 << (k-1))][k-1] sparse[i][k] = (depth[l] ≤ depth[r]) ? l : r
    // Query [L, R]
    function query(L, R):
    k = ⌊log2(R - L + 1)⌋ l = sparse[L][k], r = sparse[R - (1 << k) + 1][k] return (depth[l] ≤ depth[r]) ? l : r

    C++ Implementation — Euler Tour + Sparse Table

    const int MAXN = 2e5 + 5;
const int LOG = 19;

vector<int> adj[MAXN];
int euler[2 * MAXN], dep[2 * MAXN], first[MAXN];
int sp[2 * MAXN][LOG]; // sparse table (stores indices)
int timer_et = 0;

void dfs(int v, int p, int d) {
    first[v] = timer_et;
    euler[timer_et] = v;
    dep[timer_et] = d;
    timer_et++;
    for (int u : adj[v]) {
        if (u == p) continue;
        dfs(u, v, d + 1);
        euler[timer_et] = v;
        dep[timer_et] = d;
        timer_et++;
    }
}

void buildSparse() {
    int n = timer_et;
    for (int i = 0; i < n; i++) sp[i][0] = i;
    for (int k = 1; (1 << k) <= n; k++)
        for (int i = 0; i + (1 << k) - 1 < n; i++) {
            int l = sp[i][k-1];
            int r = sp[i + (1 << (k-1))][k-1];
            sp[i][k] = (dep[l] <= dep[r]) ? l : r;
        }
}

int query(int L, int R) {
    if (L > R) swap(L, R);
    int k = __lg(R - L + 1);
    int l = sp[L][k], r = sp[R - (1 << k) + 1][k];
    return (dep[l] <= dep[r]) ? l : r;
}

int lca(int u, int v) {
    return euler[query(first[u], first[v])];
}

// dfs(root, -1, 0);

    Comparison: Binary Lifting vs Euler Tour + RMQ

    Aspect Binary Lifting Euler Tour + Sparse Table
    Preprocessing O(n log n) O(n log n)
    Query O(log n) O(1)
    Space O(n log n) O(n log n)
    K-th ancestor O(log n) — built-in Not directly supported
    Path aggregation Easy to extend — store min/max/sum along jumps Needs separate structure
    Online / Offline Online Online
    Implementation Simpler, shorter code Slightly more code (Euler tour + Sparse Table)
    Best for General purpose; path queries; when k-th ancestor is needed Very high query count (Q > 106); when O(1) matters

    Verdict: Binary Lifting is the default choice in most contests. It’s simpler, handles k-th ancestor, and is easy to extend with path aggregation. Use Euler Tour + RMQ when you need absolute O(1) per query or the problem has an extremely high query count.

    Applications in Competitive Programming

    Distance Between Nodes

    The distance between nodes u and v in a tree is:

    dist(u, v) = depth[u] + depth[v] - 2 * depth[lca(u, v)]

    This is one of the most common uses of LCA. Any problem asking for the shortest path between two nodes in a tree reduces to this formula.

    dist(7, 5)

    depth[7] = 3, depth[5] = 2, LCA(7,5) = 0, depth[0] = 0.
    dist = 3 + 2 − 2 × 0 = 5.

    dist(7, 5) = 3 + 2 − 0 = 5 edges

    Path Queries & Path Aggregation

    Many problems ask for the minimum, maximum, sum, or XOR of edge weights (or node values) along the path from u to v. With binary lifting, extend the up table to also store aggregates:

    // Store max edge weight along 2^k jumps
int mx[MAXN][LOG];

void dfs(int v, int p, int d, int w) {
    up[v][0] = p;
    mx[v][0] = w; // weight of edge to parent
    depth[v] = d;
    for (int k = 1; k < LOG; k++) {
        up[v][k] = up[up[v][k-1]][k-1];
        mx[v][k] = max(mx[v][k-1], mx[up[v][k-1]][k-1]);
    }
    for (auto [u, wt] : adj[v])
        if (u != p) dfs(u, v, d + 1, wt);
}

int pathMax(int u, int v) {
    int ans = 0;
    if (depth[u] < depth[v]) swap(u, v);
    int diff = depth[u] - depth[v];
    for (int k = 0; k < LOG; k++)
        if ((diff >> k) & 1) {
            ans = max(ans, mx[u][k]);
            u = up[u][k];
        }
    if (u == v) return ans;
    for (int k = LOG - 1; k >= 0; k--)
        if (up[u][k] != up[v][k]) {
            ans = max(ans, max(mx[u][k], mx[v][k]));
            u = up[u][k]; v = up[v][k];
        }
    ans = max(ans, max(mx[u][0], mx[v][0]));
    return ans;
}

    Subtree & Path on Flat Arrays

    Combining LCA with Euler tour (in/out times) unlocks powerful patterns:

    • Subtree queries: use in-time/out-time to flatten subtrees into contiguous ranges; apply segment tree or BIT.
    • Path updates: use the “path = u→LCA + v→LCA” decomposition with difference arrays on the Euler tour.
    • Virtual tree / auxiliary tree: given a subset of k important nodes, build a compressed tree containing only those nodes and their pairwise LCAs. Size is O(k). Heavily used in problems like “answer queries on a subset of nodes.”

    Summary

    • LCA is the deepest common ancestor of two nodes; a fundamental tree primitive.
    • Binary Lifting: O(n log n) / O(log n). The default in contests. Supports k-th ancestor and path aggregation out of the box.
    • Euler Tour + Sparse Table: O(n log n) / O(1). Use when you need the fastest queries or have Q > 106.
    • Distance formula: dist(u,v) = depth[u] + depth[v] − 2·depth[LCA(u,v)].
    • LCA is a building block for HLD, virtual trees, centroid decomposition, and countless CP problems.
