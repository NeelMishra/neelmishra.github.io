    Tree Hashing & Isomorphism

    July 2026 · 18 min read

    Introduction

    Two trees are isomorphic if one can be transformed into the other by reordering children at every node. The shapes are the same; only the labeling of children differs. This sounds simple, but it is surprisingly hard to check by brute force: with n nodes, there are up to n! possible relabelings.

    Tree hashing solves this efficiently. The idea: assign each subtree a hash value such that two subtrees receive the same hash if and only if they are isomorphic. Once every subtree is hashed, comparing two trees reduces to comparing two integers.

    This technique appears across competitive programming:

    • Checking if two trees are isomorphic — the direct application.
    • Counting distinct subtrees in a single tree.
    • Finding symmetry axes or automorphisms of a tree.
    • Canonical forms for memoization in tree DP.
    • Detecting duplicate sub-structures (e.g., Codeforces subtree problems).

    We’ll build up from rooted hashing to unrooted isomorphism, cover the deterministic AHU algorithm, and finish with full C++ implementations.

    Rooted Tree Hashing

    Core Idea

    For a rooted tree, we define the hash of a node v recursively:

    h(v) = f( sorted( h(c1), h(c2), …, h(ck) ) )

    where c1…ck are children of v, and f is a hash function over multisets.

    The sorting step is the key insight. Two nodes with the same multiset of children hashes — regardless of child order — get the same hash. Leaves all get the same base hash (no children), so all leaves are isomorphic, which is correct.

    Hash Functions

    There are several ways to combine children hashes into a parent hash. Each trades off collision resistance vs. speed.

    MethodFormulaCollision Risk
    Polynomial rolling h(v) = 1 + Σ hash(ci) · pi mod M (sorted) Low with large prime M
    XOR with random h(v) = XOR of R[h(ci)] where R is a random map Moderate; fast
    Sorted tuple hash Sort children hashes, treat as a tuple, hash the tuple Very low (tuple is unique)
    Map-based canonical Use std::map to assign unique IDs to new multisets Zero (deterministic)

    In competitive programming, the map-based approach is most popular: maintain a global map from sorted-children-hash-vectors to unique IDs. No collisions, no prime tuning, and O(n log n) total time.

    // Map-based deterministic hashing (zero collisions)
    map<vector<int>, int> canon;
    int nextId = 1; // 1 = leaf hash

    function hashTree(v, parent):
    childHashes = [] for u in adj[v]: if u ≠ parent: childHashes.push(hashTree(u, v)) sort(childHashes) if childHashes not in canon: canon[childHashes] = ++nextId return canon[childHashes]

    A leaf has an empty childHashes vector, which maps to ID 1. A node whose two children are both leaves has childHashes = [1, 1], mapped to ID 2, and so on. Two subtrees get the same ID if and only if they are isomorphic.

    Unrooted Tree Isomorphism

    Rooted hashing works when both trees share the same root. But what if the trees are unrooted? We can’t just pick an arbitrary root — different root choices produce different hash values even for isomorphic trees.

    The solution: root both trees at their center(s), then compare rooted hashes.

    What is the Tree Center?

    The center of a tree is the set of nodes that minimize the maximum distance to any other node (the eccentricity). Every tree has either one center or two adjacent centers.

    Key property: The center of a tree is invariant under isomorphism. If tree A is isomorphic to tree B, then the center(s) of A map to the center(s) of B under any valid isomorphism.

    Finding the Center: Leaf Peeling

    Iteratively remove all current leaves. The last remaining node(s) are the center(s). This is equivalent to repeatedly removing the outermost layer of the tree:

    function findCenter(tree):
    leaves = {v : degree[v] == 1} remaining = n while remaining > 2: newLeaves = {} for leaf in leaves: remaining -= 1 for neighbor of leaf: degree[neighbor] -= 1 if degree[neighbor] == 1: newLeaves.add(neighbor) leaves = newLeaves return leaves // 1 or 2 nodes

    Complete Unrooted Isomorphism Check

    With the center(s) found, the algorithm is:

    1. Find centers of both trees.
    2. If one tree has 1 center and the other has 2 (or vice versa), they cannot be isomorphic.
    3. Root both trees at their center(s) and compute hashes.
    4. If both have 1 center: compare root hashes directly.
    5. If both have 2 centers: try both orientations of one tree’s edge and compare.

    AHU Algorithm

    The AHU (Aho, Hopcroft, Ullman) algorithm from 1974 provides a deterministic, collision-free canonical form for rooted trees in O(n log n) time. It encodes tree structure as a canonical string using parentheses.

    Level-by-Level Canonical Encoding

    The encoding works bottom-up:

    1. Leaves get the canonical string "()".
    2. For an internal node, compute the canonical strings of all children, sort them lexicographically, concatenate, and wrap in parentheses.
    3. The root’s canonical string uniquely represents the tree’s isomorphism class.
    function ahuCanonical(v, parent):
    childStrings = [] for u in adj[v]: if u ≠ parent: childStrings.push(ahuCanonical(u, v)) sort(childStrings) return "(" + join(childStrings) + ")"

    Example

    Consider a tree rooted at node 1 with structure: 1→{2,3}, 2→{4,5}, 3→{6}.

    Two trees are isomorphic if and only if their canonical strings are identical. This is deterministic — no hash collisions possible.

    Complexity

    The naive string-based approach is O(n²) due to string concatenation. To achieve O(n log n), replace strings with integer IDs: use a map from sorted child-ID vectors to unique IDs, exactly as in our hash-based approach. The map lookups cost O(log n) per node, giving O(n log n) total.

    AHU vs. Hashing: In practice, the map-based hash approach from Section 2 is the efficient AHU algorithm. The parenthesis encoding is a conceptual tool; the map-based integer IDs are the implementation. They produce the same result: a unique integer ID per isomorphism class.

    Detecting Isomorphic Subtrees

    Given a single rooted tree, find all duplicate subtree structures. This is directly solved by tree hashing: compute the hash for every node, then group nodes by hash. Any group with more than one node contains duplicate subtrees.

    Counting distinct subtrees: The number of distinct subtree shapes equals the number of distinct hashes. In the tree above, there are 3 distinct shapes: a single leaf (h=1), a node with two leaves (h=2), and the full tree (h=3).

    This technique directly solves LeetCode 652: Find Duplicate Subtrees.

    C++ Implementation

    Rooted Tree Hashing

    #include <bits/stdc++.h>
using namespace std;

map<vector<int>, int> canon;
int nextId = 0;

int hashSubtree(int v, int par, const vector<vector<int>>& adj) {
    vector<int> childHashes;
    for (int u : adj[v]) {
        if (u != par)
            childHashes.push_back(hashSubtree(u, v, adj));
    }
    sort(childHashes.begin(), childHashes.end());
    if (!canon.count(childHashes))
        canon[childHashes] = nextId++;
    return canon[childHashes];
}

    Finding Tree Center

    vector<int> findCenter(int n, const vector<vector<int>>& adj) {
    vector<int> degree(n, 0);
    queue<int> leaves;
    for (int i = 0; i < n; i++) {
        degree[i] = adj[i].size();
        if (degree[i] <= 1) leaves.push(i);
    }
    int remaining = n;
    while (remaining > 2) {
        int sz = leaves.size();
        remaining -= sz;
        queue<int> nextLeaves;
        while (sz--) {
            int v = leaves.front(); leaves.pop();
            for (int u : adj[v]) {
                if (--degree[u] == 1)
                    nextLeaves.push(u);
            }
        }
        leaves = nextLeaves;
    }
    vector<int> centers;
    while (!leaves.empty()) {
        centers.push_back(leaves.front());
        leaves.pop();
    }
    return centers;
}

    Full Unrooted Isomorphism Check

    bool areIsomorphic(int n1, const vector<vector<int>>& adj1,
                   int n2, const vector<vector<int>>& adj2) {
    if (n1 != n2) return false;

    auto c1 = findCenter(n1, adj1);
    auto c2 = findCenter(n2, adj2);
    if (c1.size() != c2.size()) return false;

    // Reset canonical map for fresh comparison
    canon.clear(); nextId = 0;

    int h1 = hashSubtree(c1[0], -1, adj1);

    // Try rooting tree2 at each center
    for (int root2 : c2) {
        canon.clear(); nextId = 0;
        // Recompute tree1 hash (map was cleared)
        h1 = hashSubtree(c1[0], -1, adj1);
        int h2 = hashSubtree(root2, -1, adj2);
        if (h1 == h2) return true;
    }
    return false;
}

    Optimization: Instead of clearing the map and recomputing, you can use a single canonical map for both trees simultaneously. Since the map assigns IDs to structural shapes, both trees can share it. Then just compare the root IDs directly.

    Optimized: Shared Map Version

    bool areIsomorphicOpt(int n1, const vector<vector<int>>& adj1,
                      int n2, const vector<vector<int>>& adj2) {
    if (n1 != n2) return false;
    auto c1 = findCenter(n1, adj1);
    auto c2 = findCenter(n2, adj2);
    if (c1.size() != c2.size()) return false;

    canon.clear(); nextId = 0;

    int h1 = hashSubtree(c1[0], -1, adj1);

    // Same canonical map — same structure → same ID
    for (int root2 : c2) {
        int h2 = hashSubtree(root2, -1, adj2);
        if (h1 == h2) return true;
    }
    return false;
}

    Applications

    Symmetry Detection

    A rooted tree is symmetric (has a non-trivial automorphism) if there exist two children of some node with the same hash. By scanning all nodes, you can find every axis of symmetry in O(n log n).

    Counting Distinct Subtrees

    After hashing all nodes, the number of distinct hashes equals the number of structurally distinct subtrees. This is useful in problems that ask “how many unique shapes appear as subtrees?”

    Tree Automorphisms

    An automorphism is a bijection from the tree to itself that preserves adjacency. The number of automorphisms can be computed by counting, at each node, how many children have identical hashes (they can be permuted freely). The total is the product over all nodes of k! where k is the count of identical-hash children groups.

    Canonical Forms for Tree DP

    When a DP on trees needs to avoid recomputing identical subtree structures, the hash serves as a memoization key. If two subtrees are isomorphic, their DP values are identical, so you compute once and reuse.

    Connection to Graph Isomorphism

    Tree isomorphism is solvable in polynomial time (in fact, O(n log n)), while general graph isomorphism has no known polynomial algorithm (it’s not known to be NP-complete either). Trees are the “easy” special case, and tree hashing is the tool that makes it easy.

    Summary

    • Rooted tree hashing: Assign each subtree a unique hash based on sorted children hashes. Map-based approach gives zero collisions in O(n log n).
    • Unrooted isomorphism: Find the tree center(s) by leaf peeling, root at center, then use rooted hashing.
    • AHU algorithm: The canonical parenthesis encoding is conceptually equivalent to the map-based hash. Both yield deterministic O(n log n) isomorphism testing.
    • Duplicate subtrees: Hash all nodes, group by hash to find structural duplicates.
    • Automorphisms: Count children with identical hashes to compute the automorphism group size.

    Tree hashing is one of those techniques that, once you know it, becomes a building block for dozens of tree problems. The core idea — sort children hashes and map to a canonical ID — is simple, fast, and powerful.
