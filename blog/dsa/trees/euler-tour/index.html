    Euler Tour Technique (Tree Flattening)

    July 2025 · 22 min read

    Introduction

    Trees are everywhere in competitive programming, but answering range-style queries on them (sum of all values in a subtree, update every node in a subtree, sum along a path) is hard because trees don't have a natural linear order. Arrays do. If only we could turn a tree into an array, we'd unlock the entire arsenal of range query data structures: segment trees, BITs (Fenwick trees), sparse tables, and more.

    That's exactly what the Euler Tour Technique does. It performs a DFS on the tree and records the entry time and exit time of each node. The entry time becomes the node's index in a flattened array. The magic property: every subtree maps to a contiguous range in this array.

    This single observation transforms tree problems into array problems:

    • Subtree sum query → range sum query on the flat array
    • Subtree update → range update on the flat array
    • Path query (with LCA) → two range queries combined
    Key idea: A DFS traversal assigns each node a timestamp. The subtree of any node v occupies the contiguous range [tin[v], tout[v]] in the flattened array. This converts all subtree operations into range operations on a 1D array.

    Euler Tour Construction

    The construction is a standard DFS. We maintain a global timer. When we first enter a node, we record tin[v] = timer++. After visiting all children, we record tout[v] = timer - 1 (or tout[v] = timer++ in the edge-based variant). The entry time becomes the node's position in the flat array.

    global timer = 0
    function euler_tour(v, parent):
    tin[v] = timer++
    flat[tin[v]] = value[v]
    for child in adj[v]:
    if child ≠ parent:
      euler_tour(child, v)
    tout[v] = timer - 1

    Subtree Queries

    This is the payoff. After the Euler tour, the subtree of any node v is exactly the contiguous range [tin[v], tout[v]] in the flat array. Why? Because DFS enters node v, then recursively visits every descendant before exiting v. All descendants get timestamps between tin[v] and tout[v].

    Subtree ↔ Range Mapping: subtree(v) = flat[tin[v] .. tout[v]]. This means any subtree query becomes a range query on the flat array.

    Let's see this concretely. Suppose each node has a value equal to its label. The subtree of node 2 contains nodes {2, 5, 6}. In the flat array, these occupy positions [1, 3] (i.e., indices 1, 2, 3):

    Subtree Sum with a BIT (Fenwick Tree)

    Once we have the flat array, a BIT gives us O(log n) point updates and range sum queries:

    • Subtree sum of v = bit.query(tout[v]) - bit.query(tin[v] - 1)
    • Update value of node v = bit.update(tin[v], delta)

    This is the standard BIT range sum. The Euler tour's only job is to convert the tree structure into the correct array indices.

    Edge-Based vs Vertex-Based Euler Tour

    There are two common variants of the Euler tour, differing in what occupies the flat array and how the timer increments on exit:

    Vertex-Based (most common)

    Each node appears once in the flat array at position tin[v]. The timer increments only on entry. tout[v] is set to timer - 1 after visiting all children.

    Array size = n (one slot per node). Used for subtree queries on node values.

    // Vertex-based: flat array has n entries
    function euler_tour_vertex(v, par):
    tin[v] = timer++
    for child in adj[v]:
    if child ≠ par: euler_tour_vertex(child, v)
    tout[v] = timer - 1

    Edge-Based (entry + exit)

    Each node appears twice: once at entry (tin[v]) and once at exit (tout[v]). The timer increments on both entry and exit.

    Array size = 2n. At entry, store +value[v]; at exit, store -value[v]. This variant is useful for path queries: the prefix sum up to position tin[v] gives the sum from root to v.

    // Edge-based: flat array has 2n entries
    function euler_tour_edge(v, par):
    tin[v] = timer++
    flat[tin[v]] = +value[v]
    for child in adj[v]:
    if child ≠ par: euler_tour_edge(child, v)
    tout[v] = timer++
    flat[tout[v]] = -value[v]

    Subtree Updates

    The Euler tour makes subtree updates just as easy as subtree queries:

    Point Update on a Single Node

    To update the value of node v by delta, simply do a point update at index tin[v] in the flat array:

    // Update value of node v
bit.update(tin[v], delta);

// Query subtree sum of node u
long long subtree_sum = bit.query(tout[u]) - bit.query(tin[u] - 1);

    Range Update on an Entire Subtree

    To add delta to every node in the subtree of v, use a range update on [tin[v], tout[v]]. With a BIT supporting range updates and point queries (using the difference trick), this is O(log n):

    // Add delta to every node in subtree of v
bit.range_update(tin[v], tout[v], delta);

// Point query: what is the current value of node u?
long long val = bit.point_query(tin[u]);
    Both at once: If you need both range updates (on subtrees) and range queries (subtree sums), use a BIT with the range-update-range-query technique (two BITs), or a lazy segment tree.

    Example: Subtree Add + Subtree Sum

    Initial state

    Values: node 1=3, node 2=5, node 3=1, node 4=2, node 5=4, node 6=7, node 7=6, node 8=9. Flat array: [3, 5, 4, 7, 1, 6, 2, 9].

    Update: add +10 to subtree of node 2

    Subtree of node 2 = {2, 5, 6} occupies flat[1..3]. Range update: add 10 to positions 1, 2, 3. Flat array becomes [3, 15, 14, 17, 1, 6, 2, 9].

    Query: subtree sum of node 1

    Subtree of node 1 = entire tree = flat[0..7]. Sum = 3+15+14+17+1+6+2+9 = 67. (Was 37 before the update; added 10 × 3 nodes = 30.)

    Path Queries via Euler Tour + LCA

    Subtree queries are the "easy win" of the Euler tour. Path queries (e.g., sum of values on the path from u to v) are harder but still solvable.

    Method 1: Edge-Based Euler Tour (Root-to-Node)

    With the edge-based variant, prefix_sum(tin[v]) gives the root-to-v sum. To get the sum on the path from u to v:

    path_sum(u, v) = root_to(u) + root_to(v) - 2 * root_to(lca(u,v)) + value[lca(u,v)]

    where root_to(x) = prefix_sum(tin[x]) in the edge-based array. This is the inclusion-exclusion principle: add both root-to-node paths, subtract the twice-counted root-to-LCA portion, then add back the LCA's own value (since it was subtracted one too many times).

    Method 2: Vertex-Based + BIT + LCA

    With the vertex-based Euler tour, define dist(v) = sum from root to v (precomputed). Then:

    path_sum(u, v) = dist[u] + dist[v] - dist[lca(u,v)] - dist[parent[lca(u,v)]]

    If node values change, you need to maintain dist[] dynamically. One approach: when node v's value changes by delta, update the BIT at tin[v] by +delta and at tout[v]+1 by -delta. Then dist[x] = bit.prefix_sum(tin[x]) gives the current root-to-x sum.

    Complexity: Each path query requires one LCA computation (O(log n) with binary lifting, O(1) with sparse table + Euler tour RMQ) plus O(1) or O(log n) for the BIT queries. Overall: O(log n) per query.

    C++ Implementation

    Here's a clean, contest-ready implementation of the Euler tour with a BIT for subtree sum queries and point updates.

    #include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int tin[MAXN], tout[MAXN], timer_val;
long long bit[MAXN];  // 1-indexed BIT
int n, q;

// ---- BIT (Fenwick Tree) ----
void bit_update(int i, long long delta) {
    for (++i; i <= n; i += i & (-i))
        bit[i] += delta;
}

long long bit_query(int i) {
    long long s = 0;
    for (++i; i > 0; i -= i & (-i))
        s += bit[i];
    return s;
}

long long bit_range(int l, int r) {
    return bit_query(r) - (l ? bit_query(l - 1) : 0);
}

// ---- Euler Tour ----
void euler_tour(int v, int par) {
    tin[v] = timer_val++;
    for (int u : adj[v]) {
        if (u != par)
            euler_tour(u, v);
    }
    tout[v] = timer_val - 1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> q;

    vector<long long> val(n + 1);
    for (int i = 1; i <= n; i++)
        cin >> val[i];

    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // Build Euler tour (1-indexed nodes)
    timer_val = 0;
    euler_tour(1, 0);

    // Initialize BIT with node values
    for (int i = 1; i <= n; i++)
        bit_update(tin[i], val[i]);

    // Process queries
    while (q--) {
        int type;
        cin >> type;
        if (type == 1) {
            // Point update: change value of node v
            int v;
            long long x;
            cin >> v >> x;
            bit_update(tin[v], x - val[v]);
            val[v] = x;
        } else {
            // Subtree sum query on node v
            int v;
            cin >> v;
            cout << bit_range(tin[v], tout[v]) << "\n";
        }
    }
    return 0;
}

    This solves the classic CSES Subtree Queries problem directly.

    Euler Tour + Lazy Segment Tree (Subtree Update + Subtree Query)

    For problems requiring both subtree range updates and subtree range queries, replace the BIT with a lazy propagation segment tree:

    struct LazySegTree {
    int n;
    vector<long long> tree, lazy;

    LazySegTree(int n) : n(n), tree(4 * n, 0), lazy(4 * n, 0) {}

    void push_down(int node, int lo, int hi) {
        if (lazy[node]) {
            int mid = (lo + hi) / 2;
            apply(2 * node, lo, mid, lazy[node]);
            apply(2 * node + 1, mid + 1, hi, lazy[node]);
            lazy[node] = 0;
        }
    }

    void apply(int node, int lo, int hi, long long val) {
        tree[node] += val * (hi - lo + 1);
        lazy[node] += val;
    }

    void update(int node, int lo, int hi, int l, int r, long long val) {
        if (r < lo || hi < l) return;
        if (l <= lo && hi <= r) { apply(node, lo, hi, val); return; }
        push_down(node, lo, hi);
        int mid = (lo + hi) / 2;
        update(2 * node, lo, mid, l, r, val);
        update(2 * node + 1, mid + 1, hi, l, r, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    long long query(int node, int lo, int hi, int l, int r) {
        if (r < lo || hi < l) return 0;
        if (l <= lo && hi <= r) return tree[node];
        push_down(node, lo, hi);
        int mid = (lo + hi) / 2;
        return query(2 * node, lo, mid, l, r)
             + query(2 * node + 1, mid + 1, hi, l, r);
    }

    // Public API
    void subtree_update(int v, long long delta) {
        update(1, 0, n - 1, tin[v], tout[v], delta);
    }

    long long subtree_query(int v) {
        return query(1, 0, n - 1, tin[v], tout[v]);
    }
};

    Applications in Competitive Programming

    The Euler tour technique is a fundamental building block. Here are the most common applications:

    1. Subtree Sum / Min / Max Queries

    The most direct application. Flatten the tree, build a segment tree or BIT on the flat array, and answer subtree aggregate queries in O(log n). Works for any associative operation.

    2. Subtree Update + Query

    Add a value to every node in a subtree (range update on [tin[v], tout[v]]) and query individual node values (point query at tin[u]) or subtree sums. Use a BIT with range update support or a lazy segment tree.

    3. Coloring / Marking Subtrees

    Mark all nodes in a subtree with a color. Since the subtree is a contiguous range, you can use a segment tree with lazy propagation to "paint" ranges. This appears in problems like "assign a color to the entire subtree of v, then query the color of node u."

    4. Counting Distinct Values in Subtrees

    Flatten the tree with the Euler tour. Now "distinct values in subtree of v" becomes "distinct values in range [tin[v], tout[v]]". This can be solved offline with Mo's algorithm on the flat array, or online with a persistent segment tree.

    5. Combined with Heavy-Light Decomposition (HLD)

    HLD also assigns DFS-based indices to nodes, and in fact HLD's index assignment is an Euler tour with a specific child ordering (heavy child first). The Euler tour property (subtree = contiguous range) is preserved. HLD adds the bonus that any root-to-node path is decomposed into O(log n) contiguous segments, enabling O(log² n) path queries with a segment tree.

    6. LCA via Euler Tour + RMQ

    The Euler tour can be used to reduce LCA to Range Minimum Query. Build an array of (depth, node) pairs in the order nodes are visited (including re-entries). The LCA of u and v is the node with minimum depth between their first occurrences. With a sparse table, this gives O(n log n) preprocessing and O(1) per LCA query.

    7. Re-Rooting Tricks

    Some problems ask you to re-root the tree and answer queries. The Euler tour structure remains valid for subtree queries under re-rooting: you just need to handle three cases for "subtree of v when rooted at r" based on where r falls relative to v's original subtree.

    Summary

    The Euler Tour Technique is one of the most versatile tools in the competitive programmer's toolkit. The core idea is simple: a single DFS assigns timestamps that flatten the tree into an array where every subtree is a contiguous range. This unlocks the entire world of range query data structures for tree problems.

    • Construction: O(n) DFS. Record tin[v] and tout[v].
    • Subtree of v: flat[tin[v]..tout[v]].
    • Vertex-based: n-element array. Best for subtree queries.
    • Edge-based: 2n-element array with +/- values. Enables root-to-node prefix sums.
    • Path queries: Combine with LCA using inclusion-exclusion.
    • Pair with: BIT, segment tree, lazy segment tree, sparse table, Mo's algorithm, HLD.
