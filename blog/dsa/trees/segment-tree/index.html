    Segment Trees

    March 2026 · 12 min read

    Motivation

    Given an array, a segment tree answers range queries (sum, min, max, GCD) and supports point/range updates — both in O(log n).

    Without a segment tree, range-sum queries take O(n) per query. Prefix sums give O(1) queries but O(n) updates. Segment trees balance both at O(log n).

    ApproachQueryUpdate
    Brute forceO(n)O(1)
    Prefix sumsO(1)O(n)
    Segment treeO(log n)O(log n)

    Structure

    A segment tree for an array of size n is a full binary tree with 2n-1 nodes (approximately). Each node stores the aggregate for a range:

    • Leaf nodes correspond to individual elements.
    • Internal nodes store the aggregate (e.g., sum) of their children’s ranges.
    • The root covers the entire array [0, n-1].

    Build

    Build bottom-up: leaves hold individual values, each internal node = left child + right child. Time: O(n).

    function build(arr, tree, node, start, end):
    if start == end: tree[node] = arr[start]; return mid = (start + end) / 2 build(arr, tree, 2*node, start, mid) build(arr, tree, 2*node+1, mid+1, end) tree[node] = tree[2*node] + tree[2*node+1]

    Range Query

    To query sum of [l, r]: starting at root, if the node’s range is fully inside [l, r], return its value. If fully outside, return 0. Otherwise split into left and right children.

    function query(tree, node, start, end, l, r):
    if r < start or end < l: return 0 // outside if l ≤ start and end ≤ r: return tree[node] // fully inside mid = (start + end) / 2 return query(..., 2*node, start, mid, l, r) + query(..., 2*node+1, mid+1, end, l, r)

    Point Update

    Update index i to a new value. Walk from the leaf up to the root, updating sums along the path. Time: O(log n).

    function update(tree, node, start, end, idx, val):
    if start == end: tree[node] = val; return mid = (start + end) / 2 if idx ≤ mid: update(..., 2*node, start, mid, idx, val) else: update(..., 2*node+1, mid+1, end, idx, val) tree[node] = tree[2*node] + tree[2*node+1]

    C++ Implementation

    #include <vector>

class SegmentTree {
    int n;
    std::vector<int> tree;

    void build(const std::vector<int>& arr, int node, int s, int e) {
        if (s == e) { tree[node] = arr[s]; return; }
        int mid = (s + e) / 2;
        build(arr, 2*node, s, mid);
        build(arr, 2*node+1, mid+1, e);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    int query(int node, int s, int e, int l, int r) {
        if (r < s || e < l) return 0;
        if (l <= s && e <= r) return tree[node];
        int mid = (s + e) / 2;
        return query(2*node, s, mid, l, r)
             + query(2*node+1, mid+1, e, l, r);
    }

    void update(int node, int s, int e, int idx, int val) {
        if (s == e) { tree[node] = val; return; }
        int mid = (s + e) / 2;
        if (idx <= mid) update(2*node, s, mid, idx, val);
        else update(2*node+1, mid+1, e, idx, val);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

public:
    SegmentTree(const std::vector<int>& arr) : n(arr.size()), tree(4 * n) {
        build(arr, 1, 0, n - 1);
    }
    int query(int l, int r) { return query(1, 0, n-1, l, r); }
    void update(int idx, int val) { update(1, 0, n-1, idx, val); }
};

    Applications

    • Range sum / min / max queries with updates.
    • Count of elements in a range satisfying a condition.
    • Lazy propagation for range updates (add to all in [l,r]).
    • 2D segment trees for matrix queries.
    • Persistent segment trees for versioned queries.

    Summary

    • Segment trees support O(log n) range queries and point updates.
    • Built bottom-up in O(n), stored in an array of size 4n.
    • Lazy propagation extends to O(log n) range updates.
    • Fundamental for competitive programming and database indexing.
