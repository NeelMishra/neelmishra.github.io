    Treaps & Implicit Treaps

    July 2025 · 35 min read

    Introduction

    A treap is a binary tree where each node stores two values: a key (maintaining BST order) and a random priority (maintaining max-heap order). The name is a portmanteau: Tree + Heap = Treap.

    The key insight: if priorities are chosen uniformly at random, the expected height of the treap is O(log n), giving us a balanced BST without the complexity of AVL or Red-Black rebalancing. Instead, all operations reduce to just two primitives: split and merge.

    Why Treaps? In competitive programming, treaps are the go-to balanced BST because they're easy to code (under 50 lines of C++), support split/merge for interval operations, and extend naturally into implicit treaps — a structure that can replace segment trees with lazy propagation while supporting insert, delete, reverse at arbitrary positions.

    Treap Properties

    PropertyMaintained ByConstraint
    KeyBST invariantleft.key < node.key < right.key
    PriorityMax-heap invariantnode.priority ≥ children.priority
    HeightRandom prioritiesO(log n) expected

    The treap is unique for a given set of (key, priority) pairs — there is exactly one treap that satisfies both BST order on keys and heap order on priorities. This uniqueness, combined with random priorities, guarantees O(log n) expected height.

    Split Operation

    The split(t, key) operation divides a treap t into two treaps: L (all keys ≤ key) and R (all keys > key). This runs in O(log n) expected time by walking down the tree.

    function split(t, key) → (L, R):
    if t == null: return (null, null) if t.key ≤ key: // root goes to L, recurse on right subtree (t.right, R) = split(t.right, key) return (t, R) else: // root goes to R, recurse on left subtree (L, t.left) = split(t.left, key) return (L, t)

    Merge Operation

    The merge(L, R) operation combines two treaps where all keys in L < all keys in R. It picks the root with the higher priority and recursively merges the other subtree. Expected time: O(log n).

    function merge(L, R) → T:
    if L == null: return R if R == null: return L if L.priority > R.priority: // L's root becomes the merged root L.right = merge(L.right, R) return L else: // R's root becomes the merged root R.left = merge(L, R.left) return R

    Merge: Step by Step

    ▶ Merge Animation

    Merge two treaps L={1,3,4} and R={6,7,9}. The node with higher priority becomes the root at each step.

    Precondition: merge(L, R) requires max(L.keys) < min(R.keys). This is always satisfied when L and R come from a split, or when inserting a single node.

    Insert & Delete

    Once you have split and merge, insertion and deletion are trivially simple — no rotations, no case analysis, no rebalancing code.

    Insert

    To insert key k with random priority:

    1. split(t, k) → get L (keys ≤ k) and R (keys > k)
    2. Create new node n with key=k, priority=random()
    3. t = merge(merge(L, n), R)
    function insert(t, key):
    (L, R) = split(t, key) node = new Node(key, rand()) return merge(merge(L, node), R)

    Delete

    To delete key k:

    1. split(t, k-1) → L (keys < k) and temp (keys ≥ k)
    2. split(temp, k) → mid (key = k) and R (keys > k)
    3. t = merge(L, R) — mid is discarded
    function erase(t, key):
    (L, temp) = split(t, key - 1) (mid, R) = split(temp, key) // mid contains only node(s) with this key; discard it return merge(L, R)

    Explicit Treap: C++ Implementation

    struct Node {
    int key, pri, sz;
    Node *l, *r;
    Node(int k) : key(k), pri(rand()), sz(1), l(0), r(0) {}
};

int sz(Node* t) { return t ? t->sz : 0; }
void upd(Node* t) { if (t) t->sz = 1 + sz(t->l) + sz(t->r); }

void split(Node* t, int key, Node*& l, Node*& r) {
    if (!t) { l = r = 0; return; }
    if (t->key <= key) {
        split(t->r, key, t->r, r);
        l = t;
    } else {
        split(t->l, key, l, t->l);
        r = t;
    }
    upd(t);
}

void merge(Node*& t, Node* l, Node* r) {
    if (!l || !r) { t = l ? l : r; return; }
    if (l->pri > r->pri) {
        merge(l->r, l->r, r);
        t = l;
    } else {
        merge(r->l, l, r->l);
        t = r;
    }
    upd(t);
}

void insert(Node*& t, int key) {
    Node *l, *r;
    split(t, key, l, r);
    merge(t, l, new Node(key));
    merge(t, t, r);
}

void erase(Node*& t, int key) {
    Node *l, *mid, *r;
    split(t, key - 1, l, mid);
    split(mid, key, mid, r);
    delete mid;          // free one node
    merge(t, l, r);
}

    Implicit Treaps

    This is where treaps become a competitive programming powerhouse. Instead of storing explicit keys, we use the subtree size to implicitly determine each node's position — the "implicit key" of a node is sz(t->left) + 1 within its subtree.

    This transforms the treap into a sequence data structure that supports:

    • Insert at position i — O(log n)
    • Delete at position i — O(log n)
    • Range reverse [l, r] — O(log n)
    • Range query / update — O(log n) with lazy propagation
    • Split / concatenate arrays — O(log n)
    Key Idea: In an implicit treap, there are no keys at all. Each node stores a value (the array element) and a random priority. The node's position in the array is determined by how many nodes are in its left subtree. When we split by position k, we split by "the first k elements go left, the rest go right."

    Implicit Indexing Visualized

    Implicit Split by Position

    To split by position k (first k elements go to L, rest go to R), we modify split to count subtree sizes instead of comparing keys:

    function split(t, k) → (L, R): // first k elements → L
    if t == null: return (null, null) push(t) // propagate lazy tags leftSize = sz(t.left) if leftSize ≥ k: // root has index ≥ k → root goes to R (L, t.left) = split(t.left, k) return (L, t) else: // root has index < k → root goes to L (t.right, R) = split(t.right, k - leftSize - 1) return (t, R)

    Array Operations via Implicit Treap

    OperationImplementationTime
    Insert at position isplit(i) → merge(L, node, R)O(log n)
    Delete at position isplit(i), split(1) → merge(L, R)O(log n)
    Reverse [l, r]split to isolate [l,r], flip lazy flag, merge backO(log n)
    Range sum [l, r]split to isolate, read .sum, merge backO(log n)
    Cyclic shiftsplit + merge in different orderO(log n)

    Lazy Propagation on Implicit Treap

    Just like segment trees, implicit treaps support lazy propagation for range operations. The most common use case in CP is range reverse: swap the left and right children and toggle a rev flag.

    Push-Down for Reverse

    function push(t):
    if t != null and t.rev: swap(t.left, t.right) if t.left: t.left.rev ^= 1 if t.right: t.right.rev ^= 1 t.rev = 0

    To reverse the subarray [l, r]:

    1. split(t, l) → A (first l elements), BC (rest)
    2. split(BC, r-l+1) → B (elements l..r), C (rest)
    3. Set B.rev ^= 1
    4. merge(A, B, C)

    Extending to Other Operations

    The same lazy pattern works for:

    • Range assignment: Set all values in [l, r] to some value v
    • Range add: Add delta to all values in [l, r]
    • Range min/max/sum: Maintain aggregate values, push lazy updates down

    The push() function must be called at the start of every split and merge to ensure lazy tags are propagated before any structural changes.

    Why not just use a segment tree? Segment trees can't efficiently insert/delete at arbitrary positions or reverse ranges. Implicit treaps handle all of these in O(log n). They're the Swiss Army knife of sequence data structures.

    Applications

    Rope Data Structure

    An implicit treap is essentially a rope — a balanced BST for sequences. Text editors use ropes for efficient insert, delete, and undo operations on strings. With an implicit treap, all these are O(log n).

    Reversible Arrays

    The classic CP application: given an array, support reverse(l, r) queries and then output the final array. This is impossible with segment trees alone but trivial with an implicit treap + lazy reverse flag.

    Order-Statistic Operations

    Since each node tracks subtree size, we get kth element and count less than x for free:

    function kth(t, k): // 1-indexed
    leftSize = sz(t.left) if k == leftSize + 1: return t if k ≤ leftSize: return kth(t.left, k) return kth(t.right, k - leftSize - 1)

    Persistent Treaps

    Because split and merge are functional-style operations (they create new nodes rather than modifying in place), treaps naturally extend to persistent versions. Each operation produces a new root while keeping old versions intact — useful for offline queries and time-travel data structures.

    C++ Implementations

    Contest-Ready: Explicit Treap (with kth & count)

    mt19937 rng(chrono::steady_clock::now().time_since_epoch().count());

struct Node {
    int key, pri, sz;
    Node *l = 0, *r = 0;
    Node(int k) : key(k), pri(rng()), sz(1) {}
};

int sz(Node* t) { return t ? t->sz : 0; }
void pull(Node* t) { if (t) t->sz = 1 + sz(t->l) + sz(t->r); }

// Split: keys <= key go to l, keys > key go to r
void split(Node* t, int key, Node*& l, Node*& r) {
    if (!t) { l = r = 0; return; }
    if (t->key <= key)
        split(t->r, key, t->r, r), l = t;
    else
        split(t->l, key, l, t->l), r = t;
    pull(t);
}

void merge(Node*& t, Node* l, Node* r) {
    if (!l || !r) { t = l ? l : r; return; }
    if (l->pri > r->pri)
        merge(l->r, l->r, r), t = l;
    else
        merge(r->l, l, r->l), t = r;
    pull(t);
}

void insert(Node*& t, int key) {
    Node *l, *r;
    split(t, key, l, r);
    merge(l, l, new Node(key));
    merge(t, l, r);
}

void erase(Node*& t, int key) {
    Node *l, *mid, *r;
    split(t, key - 1, l, mid);
    split(mid, key, mid, r);
    delete mid;
    merge(t, l, r);
}

// kth smallest (1-indexed)
int kth(Node* t, int k) {
    int left = sz(t->l);
    if (k == left + 1) return t->key;
    if (k <= left) return kth(t->l, k);
    return kth(t->r, k - left - 1);
}

// Count keys strictly less than key
int countLess(Node* t, int key) {
    if (!t) return 0;
    if (t->key < key) return 1 + sz(t->l) + countLess(t->r, key);
    return countLess(t->l, key);
}

    Contest-Ready: Implicit Treap with Lazy Reverse

    mt19937 rng(chrono::steady_clock::now().time_since_epoch().count());

struct Node {
    int val, pri, sz;
    bool rev;
    Node *l = 0, *r = 0;
    Node(int v) : val(v), pri(rng()), sz(1), rev(0) {}
};

int sz(Node* t) { return t ? t->sz : 0; }

void pull(Node* t) {
    if (t) t->sz = 1 + sz(t->l) + sz(t->r);
}

void push(Node* t) {
    if (t && t->rev) {
        swap(t->l, t->r);
        if (t->l) t->l->rev ^= 1;
        if (t->r) t->r->rev ^= 1;
        t->rev = 0;
    }
}

// Split first k elements into l, rest into r
void split(Node* t, int k, Node*& l, Node*& r) {
    if (!t) { l = r = 0; return; }
    push(t);
    int left = sz(t->l);
    if (left >= k)
        split(t->l, k, l, t->l), r = t;
    else
        split(t->r, k - left - 1, t->r, r), l = t;
    pull(t);
}

void merge(Node*& t, Node* l, Node* r) {
    push(l); push(r);
    if (!l || !r) { t = l ? l : r; return; }
    if (l->pri > r->pri)
        merge(l->r, l->r, r), t = l;
    else
        merge(r->l, l, r->l), t = r;
    pull(t);
}

// Reverse range [l, r] (0-indexed)
void reverse(Node*& t, int l, int r) {
    Node *a, *b, *c;
    split(t, l, a, b);
    split(b, r - l + 1, b, c);
    b->rev ^= 1;
    merge(b, b, c);
    merge(t, a, b);
}

// Insert value v at position pos (0-indexed)
void insertAt(Node*& t, int pos, int v) {
    Node *l, *r;
    split(t, pos, l, r);
    merge(l, l, new Node(v));
    merge(t, l, r);
}

// Delete element at position pos (0-indexed)
void eraseAt(Node*& t, int pos) {
    Node *l, *mid, *r;
    split(t, pos, l, mid);
    split(mid, 1, mid, r);
    delete mid;
    merge(t, l, r);
}

// Print in-order (the array)
void print(Node* t) {
    if (!t) return;
    push(t);
    print(t->l);
    cout << t->val << ' ';
    print(t->r);
}
    Contest tip: Use mt19937 for random priorities instead of rand(). The Mersenne Twister has much better distribution, reducing the chance of pathological tree heights. Some competitive programmers also use (int)rng() to get signed ints for priorities.

    Summary

    • A treap is a BST by key + max-heap by random priority → O(log n) expected height.
    • Split and merge are the only two primitives; all operations build on them.
    • Implicit treaps use subtree sizes as implicit keys, turning the treap into a powerful sequence data structure.
    • With lazy propagation, implicit treaps support range reverse, range assignment, and range queries — all in O(log n).
    • Treaps are simpler to code than splay trees and more versatile than segment trees for dynamic sequences.
    • In contests, the implicit treap is your best friend when you need insert/delete at arbitrary positions or range reversal.
