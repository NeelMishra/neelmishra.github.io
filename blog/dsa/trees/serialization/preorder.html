    Preorder Serialization with Null Markers

    March 2026 · 10 min read

    This is the single most important serialization technique. Master it, and the rest are variations on the same idea. The approach is dead simple: do a preorder traversal (root, left, right), write each value, and write a # for every null pointer you encounter.

    The Idea

    In a preorder traversal, the root is always the first element. So when we deserialize, we know the first token is the root. Then we recursively build the left subtree from the next tokens, then the right subtree from whatever remains. The # markers tell us exactly when a subtree ends.

    Count the tokens: 5 values + 6 null markers = 11 tokens. For a binary tree with n nodes, there are always n + 1 null pointers (each internal node has 2 child slots, leaves contribute nulls). So the serialized string always has 2n + 1 tokens.

    Why This Encoding Is Unambiguous

    Here's the key insight: in the token stream, every node value is followed by exactly two subtrees (each of which is itself a valid serialization). A # token represents an empty subtree. This recursive structure means there's exactly one way to parse the token stream.

    Formally, define the balance of a serialization as: start at 0, add +1 for each value token, subtract 1 for each # token. A valid serialization of a single tree has these properties:

    • The balance starts at 0 and ends at -1 (we read n values and n + 1 nulls).
    • The balance never reaches -1 before the last token (otherwise the tree would have "ended" prematurely).

    This is directly analogous to balanced parentheses: each value "opens" two child slots, each null "closes" one. The serialization is valid iff the parenthesis structure is valid.

    Serialization: Tree to String

    The recursive code is minimal:

    std::string serialize(TreeNode* root) {
    if (!root) return "#,";
    return std::to_string(root->val) + ","
         + serialize(root->left)
         + serialize(root->right);
}

    This builds the string by concatenation. In practice, you'd use a std::ostringstream or pass a reference to avoid O(n²) string copies:

    void serialize(TreeNode* root, std::ostringstream& out) {
    if (!root) { out << "# "; return; }
    out << root->val << " ";
    serialize(root->left, out);
    serialize(root->right, out);
}

    Tracing the Calls

    For the tree [1, 2, 3, 4, null, null, 5], the call sequence is:

    serialize(1) → write "1"
    serialize(2) → write "2"
    serialize(4) → write "4"
    serialize(null) → write "#"
    serialize(null) → write "#"
    serialize(null) → write "#"
    serialize(3) → write "3"
    serialize(null) → write "#"
    serialize(5) → write "5"
    serialize(null) → write "#"
    serialize(null) → write "#"

    Result: "1 2 4 # # # 3 # 5 # #"

    Deserialization: String to Tree

    Deserialization is the mirror image. Read tokens from left to right. Each non-null token becomes a node; then recursively build its left and right children from the subsequent tokens.

    TreeNode* deserialize(std::istringstream& in) {
    std::string token;
    in >> token;
    if (token == "#") return nullptr;

    TreeNode* node = new TreeNode(std::stoi(token));
    node->left  = deserialize(in);   // consumes left subtree tokens
    node->right = deserialize(in);   // consumes right subtree tokens
    return node;
}

    The stream acts as a shared cursor. The beauty is that each recursive call consumes exactly the right number of tokens, you don't need to calculate offsets or boundaries. The # markers naturally terminate the recursion.

    Iterative Deserialization

    You can also deserialize iteratively using an explicit stack. This avoids recursion depth issues for deep trees:

    TreeNode* deserializeIterative(const std::string& data) {
    std::istringstream in(data);
    std::string token;
    in >> token;
    if (token == "#") return nullptr;

    TreeNode* root = new TreeNode(std::stoi(token));
    // Stack of (node, which_child_next): 0 = left, 1 = right
    std::stack<std::pair<TreeNode*, int>> stk;
    stk.push({root, 0});

    while (in >> token && !stk.empty()) {
        TreeNode* child = (token == "#") ? nullptr
                        : new TreeNode(std::stoi(token));
        auto& [parent, side] = stk.top();
        if (side == 0) {
            parent->left = child;
            side = 1;       // next time, assign right
        } else {
            parent->right = child;
            stk.pop();      // both children assigned
        }
        if (child) stk.push({child, 0});
    }
    return root;
}

    The stack tracks which node still needs its right child assigned. When we read a token:

    1. If the top-of-stack node needs a left child (side=0), assign it and switch to side=1.
    2. If it needs a right child (side=1), assign it and pop (this node is complete).
    3. If the new node is non-null, push it onto the stack so we process its children next.

    Postorder Variant

    You can also serialize using postorder (left, right, root) with null markers. The twist: you deserialize in reverse, reading from the end of the string. The last token is the root. Before that come the right subtree tokens, then the left subtree tokens.

    // Serialize: postorder
void serializePost(TreeNode* root, std::ostringstream& out) {
    if (!root) { out << "# "; return; }
    serializePost(root->left, out);
    serializePost(root->right, out);
    out << root->val << " ";   // root last
}

// Deserialize: read tokens in reverse, build right-then-left
TreeNode* deserializePost(std::vector<std::string>& tokens, int& i) {
    if (i < 0 || tokens[i] == "#") { i--; return nullptr; }
    TreeNode* node = new TreeNode(std::stoi(tokens[i--]));
    node->right = deserializePost(tokens, i);  // right first!
    node->left  = deserializePost(tokens, i);
    return node;
}

    Postorder is less common in interviews but shows up in serialization of expression trees (reverse Polish notation is postorder serialization of an expression tree).

    Edge Cases to Watch

    • Empty tree: serialize(null) should produce "#". deserialize("#") should return null.
    • Single node: "1 # #". Both children are null.
    • Left-skewed/Right-skewed: The serialization works, but recursion depth = O(n). Use the iterative version for production.
    • Negative values: "-1 -2 # # #" works fine, as long as your delimiter isn't -.
    • Large values: Use space or comma as delimiter, not fixed-width encoding.
    • Duplicate values: Not a problem. Structure is encoded by null markers, not by value uniqueness.

    Complexity Analysis

    Time: O(n) for both serialize and deserialize. Each node visited exactly once.

    Space (output): O(n). The string has 2n + 1 tokens.

    Space (auxiliary): O(h) for the recursion stack (or explicit stack), where h is the height of the tree.

    When to Use Preorder Serialization

    Use it when:

    • You need the simplest, most bug-free implementation under interview pressure.
    • You're serializing a general binary tree (not necessarily a BST).
    • You want streaming deserialization: you can start building the tree as soon as you read the first token.
    • The tree may have duplicate values (two-traversal approaches struggle with duplicates).

    Don't use it when:

    • You need the output to be human-readable (level-order is much easier to visualize).
    • You're working with a BST and want a more compact encoding (see the advanced page).
