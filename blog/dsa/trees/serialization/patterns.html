    Serialization Interview Patterns

    March 2026 · 11 min read

    Serialization questions appear frequently in interviews because they test several skills at once: tree traversal, recursion, string parsing, and the ability to design a correct encode/decode pair. Let's walk through the major problem patterns.

    Pattern 1: Serialize and Deserialize Binary Tree

    LeetCode 297 Hard

    Design an algorithm to serialize a binary tree to a string and deserialize the string back to the original tree. There are no restrictions on how your serialization format works, so long as the round-trip produces an identical tree.

    This is the canonical problem. The "hard" rating is generous if you've practiced: the preorder + null markers approach is 10 lines of code.

    Recommended Approach: Preorder + Null Markers

    class Codec {
public:
    std::string serialize(TreeNode* root) {
        if (!root) return "# ";
        return std::to_string(root->val) + " "
             + serialize(root->left)
             + serialize(root->right);
    }

    TreeNode* deserialize(std::string data) {
        std::istringstream in(data);
        return build(in);
    }

private:
    TreeNode* build(std::istringstream& in) {
        std::string tok;
        in >> tok;
        if (tok == "#") return nullptr;
        auto* node = new TreeNode(std::stoi(tok));
        node->left  = build(in);
        node->right = build(in);
        return node;
    }
};
    Interview tip: State your approach before coding. Say: "I'll use preorder traversal with null markers. Serialize writes root, then left subtree, then right subtree, with '#' for null. Deserialize reads tokens in the same order. The stream cursor naturally advances through each subtree."

    Pattern 2: Construct Tree from Traversals

    LeetCode 105: Construct from Preorder + Inorder Medium

    Given preorder and inorder traversal arrays, reconstruct the binary tree. Values are unique.

    LeetCode 106: Construct from Inorder + Postorder Medium

    Same idea, but with inorder and postorder. Postorder's root is the last element.

    The key insight for both: one traversal gives the root, the other tells you how to split into left and right subtrees. Build a hash map of inorder values to indices for O(1) lookup.

    // LeetCode 105 pattern
class Solution {
    std::unordered_map<int,int> inMap;
    int preIdx = 0;
public:
    TreeNode* buildTree(std::vector<int>& pre, std::vector<int>& ino) {
        for (int i = 0; i < ino.size(); i++)
            inMap[ino[i]] = i;
        return build(pre, 0, ino.size() - 1);
    }

    TreeNode* build(std::vector<int>& pre, int inL, int inR) {
        if (inL > inR) return nullptr;
        int rootVal = pre[preIdx++];
        TreeNode* root = new TreeNode(rootVal);
        root->left  = build(pre, inL, inMap[rootVal] - 1);
        root->right = build(pre, inMap[rootVal] + 1, inR);
        return root;
    }
};
    Common mistake: Forgetting that for postorder + inorder (LC 106), you must build the right subtree before the left (process postorder from the end, right-to-left).

    Pattern 3: Verify Preorder Serialization

    LeetCode 331: Verify Preorder Serialization Medium

    Given a comma-separated string of values and '#' markers, determine if it's a valid preorder serialization of some binary tree. You may not reconstruct the tree.

    Use the slot counting technique. Think of each node as consuming one slot and producing two. Each '#' consumes one slot and produces zero.

    bool isValidSerialization(std::string preorder) {
    int slots = 1;  // start with 1 slot (for the root)
    std::istringstream in(preorder);
    std::string token;

    while (std::getline(in, token, ',')) {
        slots--;                    // consume a slot
        if (slots < 0) return false;  // no slot available
        if (token != "#")
            slots += 2;            // non-null produces 2 child slots
    }
    return slots == 0;  // all slots consumed
}

    This is elegant: O(n) time, O(1) space. No tree reconstruction needed. The invariant is that slots must never go negative (would mean we're placing a node where no slot exists) and must end at exactly 0 (all slots filled).

    Pattern 4: Serialize/Deserialize BST

    LeetCode 449: Serialize and Deserialize BST Medium

    Design a serialization for a BST that is as compact as possible. The BST property must be used to achieve better compression than the general binary tree approach.

    Use preorder without null markers. The BST property tells you where the left subtree ends and the right begins.

    class Codec {
public:
    std::string serialize(TreeNode* root) {
        std::string s;
        serializeHelper(root, s);
        return s;
    }

    TreeNode* deserialize(std::string data) {
        std::istringstream in(data);
        std::vector<int> vals;
        int v;
        while (in >> v) vals.push_back(v);
        int i = 0;
        return build(vals, i, INT_MIN, INT_MAX);
    }

private:
    void serializeHelper(TreeNode* r, std::string& s) {
        if (!r) return;
        s += std::to_string(r->val) + " ";
        serializeHelper(r->left, s);
        serializeHelper(r->right, s);
    }

    TreeNode* build(std::vector<int>& v, int& i, int lo, int hi) {
        if (i >= v.size() || v[i] < lo || v[i] > hi)
            return nullptr;
        TreeNode* node = new TreeNode(v[i++]);
        node->left  = build(v, i, lo, node->val);
        node->right = build(v, i, node->val, hi);
        return node;
    }
};

    Space: n values instead of 2n+1 tokens. That's almost half the size.

    Pattern 5: N-ary Tree Serialization

    LeetCode 428: Serialize and Deserialize N-ary Tree Hard

    Same as LC 297 but for N-ary trees where each node can have any number of children.

    Use DFS with a child-count prefix or an end-of-children sentinel:

    class Codec {
public:
    std::string serialize(Node* root) {
        if (!root) return "#";
        std::string s = std::to_string(root->val) + " "
                      + std::to_string(root->children.size()) + " ";
        for (auto* child : root->children)
            s += serialize(child) + " ";
        return s;
    }

    Node* deserialize(std::string data) {
        std::istringstream in(data);
        return build(in);
    }

private:
    Node* build(std::istringstream& in) {
        std::string tok;
        in >> tok;
        if (tok == "#") return nullptr;
        int val = std::stoi(tok);
        int childCount;
        in >> childCount;
        Node* node = new Node(val);
        for (int i = 0; i < childCount; i++)
            node->children.push_back(build(in));
        return node;
    }
};

    Pattern 6: Tree Isomorphism via Serialization

    Two trees are isomorphic if one can be transformed into the other by swapping children. You can check this by computing a canonical serialization and comparing strings.

    // Canonical form: for each node, serialize both orderings
// of children and pick the lexicographically smaller one
std::string canonical(TreeNode* root) {
    if (!root) return "#";
    std::string left  = canonical(root->left);
    std::string right = canonical(root->right);
    // Canonical: smaller child serialization goes first
    if (left > right) std::swap(left, right);
    return "(" + std::to_string(root->val)
         + left + right + ")";
}

bool isIsomorphic(TreeNode* a, TreeNode* b) {
    return canonical(a) == canonical(b);
}

    Pattern 7: Subtree Matching via Serialization

    LeetCode 572: Subtree of Another Tree Easy

    Given two trees s and t, check if t is a subtree of s.

    Serialize both trees with preorder + null markers, then check if t's serialization is a substring of s's serialization. But be careful: add a delimiter before each value to avoid false matches (e.g., "12" matching "2").

    bool isSubtree(TreeNode* s, TreeNode* t) {
    std::string ss = serialize(s);
    std::string st = serialize(t);
    return ss.find(st) != std::string::npos;
}

std::string serialize(TreeNode* root) {
    if (!root) return ",#";   // comma delimiter prevents "12" matching "2"
    return "," + std::to_string(root->val)
         + serialize(root->left)
         + serialize(root->right);
}
    Pitfall: Without the delimiter, serialize(node with val=12) starts with "12" which contains "2". A tree with root 12 would falsely "contain" a subtree with root 2.

    Common Mistakes

    MistakeWhat Goes WrongFix
    Using traversal without null markersMultiple trees map to the same stringAlways include structure info (nulls or second traversal)
    String concatenation in recursionO(n²) time from copyingUse stream or pass reference
    Two-traversal with duplicatesAmbiguous inorder indexUse null-marker approach instead
    Missing delimiter"12" matches "2" in substring searchPrefix each token with a delimiter
    Not handling empty treedeserialize("") crashesCheck for empty/null input first
    Postorder: building left before rightWrong tree structureRead tokens from end; build right subtree first
    BST: not passing boundsLose the BST property during reconstructionThread min/max bounds through recursion

    Interview Decision Tree

    When you hit a serialization problem, follow this flowchart:

    Is it a BST?
    YES → Preorder only, deserialize with bounds. (LC 449)
    NO → Is it a general binary tree?
    YES → Preorder + null markers. (LC 297)
    NO → Is it N-ary?
    YES → DFS with child-count prefix. (LC 428)

    Are you given two traversals?
    Pre + In → Root from pre[0], split by inorder index. (LC 105)
    Post + In → Root from post[last], split by inorder index. (LC 106)

    Asked to verify (not reconstruct)?
    → Slot counting: +2 for node, -1 for each token. (LC 331)
