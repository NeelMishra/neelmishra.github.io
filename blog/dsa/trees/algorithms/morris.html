    Morris In-order Traversal

    March 2026 · 8 min read

    The Problem

    Standard in-order traversal uses O(h) space — either via an explicit stack or the implicit call stack of recursion. For a balanced tree with a million nodes, that's ~20 stack frames. For a skewed tree, it's O(n).

    Can we do in-order traversal with O(1) extra space? Yes — Morris traversal achieves exactly this by temporarily modifying the tree itself.

    The Core Idea

    The recursion stack exists because after visiting the left subtree, we need to find our way back to the current node. Without a stack, we have no breadcrumb trail. Morris traversal solves this by creating temporary threads — pointers from the rightmost node of each left subtree back to the current node.

    These threads are exactly the same concept as in threaded binary trees, except they're created on the fly and removed after use, leaving the tree unmodified when we're done.

    Algorithm Step by Step

    Starting at the root, for each node cur:

    1. If cur has no left child: Visit cur, move right (cur = cur->right). If a thread exists, this takes us back up to the ancestor that created the thread.
    2. If cur has a left child: Find the in-order predecessor of cur (the rightmost node in the left subtree).
      • If the predecessor's right is null: Create a thread (pred->right = cur). This is our breadcrumb. Move left (cur = cur->left).
      • If the predecessor's right points to cur: We've returned via the thread. The left subtree is fully traversed. Remove the thread (pred->right = nullptr), visit cur, move right.

    Why It Works

    Each node is visited at most twice:

    • First encounter: We haven't explored its left subtree yet. We create a thread and go left.
    • Second encounter: We arrive back via the thread. The left subtree is done. We visit the node and go right.

    The two encounters are distinguished by the predecessor's right pointer: null (first visit) vs pointing to cur (second visit).

    The Code

    void morrisInorder(TreeNode* root) {
    TreeNode* cur = root;
    while (cur) {
        if (!cur->left) {
            // No left subtree: visit and go right
            visit(cur);
            cur = cur->right;
        } else {
            // Find in-order predecessor
            TreeNode* pred = cur->left;
            while (pred->right && pred->right != cur)
                pred = pred->right;

            if (!pred->right) {
                // First visit: create thread, go left
                pred->right = cur;
                cur = cur->left;
            } else {
                // Second visit: remove thread, visit, go right
                pred->right = nullptr;
                visit(cur);
                cur = cur->right;
            }
        }
    }
}

    Time: O(n) — despite the inner while loop for finding predecessors, each edge is traversed at most twice (once to create a thread, once to remove it). Amortized over all nodes, the total work is still linear.
    Space: O(1) extra — no stack, no recursion, no visited set. Only a few pointer variables.

    The Trade-off

    Morris traversal temporarily mutates the tree. While the traversal is in progress, the tree structure is invalid (it has cycles). This means:

    • Not safe in concurrent/multi-threaded contexts without synchronization.
    • If the traversal is interrupted (e.g., by an exception), the tree is left in a broken state.
    • If the tree is read-only (e.g., const TreeNode*), Morris traversal cannot be used.

    The tree is always restored to its original state by the end — but only if the traversal completes normally.

    When to use Morris: When O(1) space is a hard requirement and you can tolerate temporary tree mutation. In interview settings, mention it as an optimization after presenting the stack-based approach. In production, the stack-based version is usually preferred for safety.

    Morris Preorder Variant

    The only difference from in-order: visit the node on the first encounter (when creating the thread) instead of the second:

    void morrisPreorder(TreeNode* root) {
    TreeNode* cur = root;
    while (cur) {
        if (!cur->left) {
            visit(cur);
            cur = cur->right;
        } else {
            TreeNode* pred = cur->left;
            while (pred->right && pred->right != cur)
                pred = pred->right;
            if (!pred->right) {
                visit(cur);         // visit HERE (first encounter)
                pred->right = cur;
                cur = cur->left;
            } else {
                pred->right = nullptr;
                // DON'T visit (second encounter)
                cur = cur->right;
            }
        }
    }
}
