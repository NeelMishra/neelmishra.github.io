    Find the Degree of Each Vertex

    Weekly Contest 497 · Q1 LeetCode #3898 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given a 2D integer array matrix of size n x n representing the adjacency matrix of an undirected graph with n vertices labeled from 0 to n - 1.

    • matrix[i][j] = 1 indicates that there is an edge between vertices i and j.
    • matrix[i][j] = 0 indicates that there is no edge between vertices i and j.

    The degree of a vertex is the number of edges connected to it.

    Return an integer array ans of size n where ans[i] represents the degree of vertex i.

    Example 1:

    Input: matrix = [[0,1,1],[1,0,1],[1,1,0]]
    Output: [2,2,2]

    Example 2:

    Input: matrix = [[0,1,0],[1,0,0],[0,0,0]]
    Output: [1,1,0]

    Example 3:

    Input: matrix = [[0]]
    Output: [0]

    Constraints:

    • 1 <= n == matrix.length == matrix[i].length <= 100
    • matrix[i][i] == 0
    • matrix[i][j] is either 0 or 1
    • matrix[i][j] == matrix[j][i]

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The graph is given to us as an adjacency matrix, and the degree of vertex i is exactly the number of vertices adjacent to it. In an adjacency matrix that means: count the number of 1s in row i.

    Because the matrix is symmetric (undirected graph) and the diagonal is zero (no self-loops), simply summing row i gives the degree without any double-counting concerns. No graph traversal or auxiliary structure is needed — this is a one-pass row sum per vertex.

    Brute Force

    For each vertex i, scan every other vertex j and check whether matrix[i][j] == 1. Increment a counter on each hit.

    class Solution {
public:
    vector<int> findDegrees(vector<vector<int>>& matrix) {
        int n = matrix.size();
        vector<int> ans(n, 0);
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < n; ++j) {
                if (matrix[i][j] == 1) ans[i]++;
            }
        }
        return ans;
    }
};

    Complexity: Time O(n²), Space O(1) auxiliary (output excluded).

    Optimal Approach

    Since each entry is 0 or 1, the row sum is the degree. Replace the inner conditional with a plain accumulation over the row.

    class Solution {
public:
    vector<int> findDegrees(vector<vector<int>>& matrix) {
        int n = matrix.size();
        vector<int> ans(n);
        for (int i = 0; i < n; ++i)
            for (int x : matrix[i]) ans[i] += x;
        return ans;
    }
};

    Why it works: Each row of an adjacency matrix lists incidence indicators for one vertex. With n ≤ 100, n² work is at most 10⁴ operations — there is no algorithmically faster path.

    Complexity: Time O(n²), Space O(n) for the answer.

    Dry Run

    Take Example 2: matrix = [[0,1,0],[1,0,0],[0,0,0]].

    Row iRow contentsSumans[i]
    0[0, 1, 0]0+1+0 = 11
    1[1, 0, 0]1+0+0 = 11
    2[0, 0, 0]0+0+0 = 00

    Final answer: [1, 1, 0].

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute (per-cell check)O(n²)O(n)Conditional increment per cell.
    Optimal (row sum)O(n²)O(n)Same asymptotics, simpler inner loop and cache-friendly.

    Edge Cases & Pitfalls

    • Single vertex (n = 1): The lone row is all zeros and the answer is [0]; do not special-case it.
    • Isolated vertices: An entirely-zero row legitimately gives degree 0 — make sure the result vector is initialized to 0.
    • Self-loop assumption: The problem guarantees matrix[i][i] == 0; do not skip the diagonal manually if you also rely on summing the entire row.
    • Symmetry: Don’t halve the count thinking each edge is double-counted. We sum each row independently, not the whole matrix.
    • Output type: Return vector<int>; integer overflow is impossible because each degree is at most n - 1 ≤ 99.
