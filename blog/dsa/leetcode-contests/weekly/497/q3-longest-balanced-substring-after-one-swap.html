    Longest Balanced Substring After One Swap

    Weekly Contest 497 · Q3 LeetCode #3900 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given a binary string s consisting only of characters '0' and '1'.

    A string is balanced if it contains an equal number of '0's and '1's.

    You can perform at most one swap between any two characters in s. Then, you select a balanced substring from s.

    Return an integer representing the maximum length of the balanced substring you can select.

    Example 1:

    Input: s = "100001"
    Output: 4

    Swap the underlined characters in "100001"; the string becomes "101000", and the prefix "1010" is balanced (length 4).

    Example 2:

    Input: s = "111"
    Output: 0 (the empty substring is balanced).

    Constraints:

    • 1 <= s.length <= 105
    • s consists only of the characters '0' and '1'.

    Source: LeetCode · Statement © LeetCode.

    Intuition

    Let pre[i] be the running difference (#1s − #0s) in s[0..i]. A substring s[l+1..r] is balanced exactly when pre[r] = pre[l]. So without any swap, the longest balanced substring is the maximum gap between two indices having the same prefix value — a classic prefix-sum + first-occurrence trick.

    Now “at most one swap” lets us rebalance a substring whose count is off by exactly two — swapping one inner character with one outside character changes the inside difference by ±2. So besides equal prefix values, we also care about pairs (l, r) with pre[r] − pre[l] = ±2. Such a window has two more 1s than 0s (or vice versa); we can balance it iff there is at least one 0 outside the window (resp. one 1) to swap in.

    Concretely we store, for each prefix value, all positions where it appears. The earliest occurrence gives the longest candidate window; if that candidate fails the “character available outside” check we fall back to the second-earliest occurrence.

    Brute Force

    Try every pair of positions to swap (or no swap at all), then for each resulting string scan all substrings and count 0s and 1s. Take the maximum balanced length over all attempts.

    class Solution {
public:
    int longestBalanced(string s) {
        int n = s.size(), best = 0;
        auto scan = [&](const string& t) {
            for (int i = 0; i < n; ++i) {
                int z = 0, o = 0;
                for (int j = i; j < n; ++j) {
                    (t[j] == '0') ? ++z : ++o;
                    if (z == o) best = max(best, j - i + 1);
                }
            }
        };
        scan(s);
        for (int i = 0; i < n; ++i)
            for (int j = i + 1; j < n; ++j) {
                if (s[i] == s[j]) continue;
                swap(s[i], s[j]); scan(s); swap(s[i], s[j]);
            }
        return best;
    }
};

    Complexity: Time O(n4) (n² swap pairs × n² substring scan), Space O(1). Far too slow for n = 105.

    Optimal Approach

    Walk the string once maintaining the prefix sum pre = (#1) − (#0). For each prefix value we remember, in order, the indices where it occurs. At index i with current prefix pre, three candidate windows update the answer:

    • No swap: the earliest position with the same prefix value pre — difference is zero, already balanced.
    • Swap a 1 out for a 0 in: needs a window with pre − 2 at the left end. Window length is L = i − ℓ; it contains (L − 2)/2 zeros, so we need at least one 0 outside, i.e. (L − 2)/2 < cnt0. If the earliest ℓ fails this test, try the second-earliest.
    • Swap a 0 out for a 1 in: symmetric, looking for prefix pre + 2 and requiring (L − 2)/2 < cnt1.
    class Solution {
public:
    int longestBalanced(string s) {
        int n = s.size();
        int cnt0 = count(s.begin(), s.end(), '0');
        int cnt1 = n - cnt0;
        unordered_map<int, vector<int>> pos;
        pos[0] = {-1};
        int ans = 0, pre = 0;
        for (int i = 0; i < n; ++i) {
            pre += (s[i] == '1') ? 1 : -1;
            pos[pre].push_back(i);

            ans = max(ans, i - pos[pre][0]);

            auto it = pos.find(pre - 2);
            if (it != pos.end()) {
                auto& p = it->second;
                if ((i - p[0] - 2) / 2 < cnt0) ans = max(ans, i - p[0]);
                else if (p.size() > 1)        ans = max(ans, i - p[1]);
            }
            it = pos.find(pre + 2);
            if (it != pos.end()) {
                auto& p = it->second;
                if ((i - p[0] - 2) / 2 < cnt1) ans = max(ans, i - p[0]);
                else if (p.size() > 1)        ans = max(ans, i - p[1]);
            }
        }
        return ans;
    }
};

    Why it works: One swap changes the inside count of 1s by exactly ±1, so it can only fix windows whose (#1) − (#0) imbalance is in {0, ±2}. The earliest occurrence of a given prefix maximises window length; if that earliest window already contains all 0s of s (resp. all 1s), no outside character is available to swap in, and we fall back to the second-earliest occurrence — which is enough, because moving the left boundary one occurrence to the right strictly removes characters from the window.

    Complexity: Time O(n) amortised (hash-map lookups and appends), Space O(n).

    Dry Run

    Take Example 1: s = "100001", so cnt0 = 4, cnt1 = 2. We track pre (with the convention pos[0] = {-1}):

    is[i]prepos snapshotans update
    01+1{0:[-1], 1:[0]}same-prefix gap 0
    100{0:[-1,1], 1:[0]}1 − (−1) = 2
    20−1+ -1:[2]—
    30−2+ -2:[3]pre+2=0 earliest at −1, len 3−(−1)=4; window has (4−2)/2 = 1 < cnt1 = 2 ✓ ⇒ ans = 4
    40−3+ -3:[4]pre+2=−1 earliest at 2, len 2 — no improvement
    51−2-2:[3,5]same-prefix gap 5−3 = 2

    Maximum is 4, matching the expected output. The swap that realises it: turn the leftmost 1 into a 0 outside the window (positions 0 and 4 swap) so the prefix "0100" becomes balanced — equivalent to the doocs example, just a different witness.

    Edge Cases & Pitfalls

    • All same character ("111" or "000"): No swap helps; the answer is 0, which the algorithm naturally returns because no two prefix values differ by 0 with positive gap and no ±2 window has a partner outside.
    • Sentinel pos[0] = {−1}: Forgetting it causes you to miss balanced prefixes that begin at index 0.
    • “Outside character available” check: Use a strict inequality. If the window already contains all zeros of s, there is no 0 to swap in; you must fall back to the next-earliest left endpoint.
    • Same-character swap doesn’t help: Swapping two equal characters changes nothing, so the “at most one swap” option includes the no-op case — the prefix-equal case handles this implicitly.
    • Empty substring: Length 0 is balanced and is the floor of ans; never return a negative number.
    • Hash map ordering: Push the current index before reading pos[pre][0] so that the lookup never sees a missing key, and so that the first occurrence remains at index 0.
