    Angles of a Triangle

    Problem Statement

    You are given a positive integer array sides of length 3.

    Determine if there exists a triangle with positive area whose three side lengths are given by the elements of sides.

    If such a triangle exists, return an array of three floating-point numbers representing its internal angles (in degrees), sorted in non-decreasing order. Otherwise, return an empty array.

    Answers within 10-5 of the actual answer will be accepted.

    Example 1:

    Input: sides = [3,4,5]
    Output: [36.86990,53.13010,90.00000]

    Example 2:

    Input: sides = [2,4,2]
    Output: [] (degenerate, zero area)

    Constraints:

    • sides.length == 3
    • 1 <= sides[i] <= 1000

    Source: LeetCode · Statement © LeetCode.

    Intuition

    Two pieces of geometry collapse this problem to constant work. First, the triangle inequality: three positive lengths form a triangle of positive area iff the sum of the two smallest is strictly greater than the largest. Sort the sides and test once.

    Second, given a valid triangle, the law of cosines tells us each angle from the three side lengths. With sides a ≤ b ≤ c we get cos A = (b² + c² − a²) / (2bc), and similarly for B; the third angle is 180° − A − B. Because a ≤ b ≤ c, the angles come out automatically in non-decreasing order, so no extra sorting is needed.

    Brute Force

    One unsophisticated approach: compute every angle independently with acos, sort the three results, and validate the triangle inequality with all three pairwise checks instead of the two-smallest shortcut.

    class Solution {
public:
    vector<double> internalAngles(vector<int>& sides) {
        int a = sides[0], b = sides[1], c = sides[2];
        if (a + b <= c || a + c <= b || b + c <= a) return {};
        double PI = acos(-1.0);
        double A = acos((1.0*b*b + 1.0*c*c - 1.0*a*a) / (2.0*b*c)) * 180.0 / PI;
        double B = acos((1.0*a*a + 1.0*c*c - 1.0*b*b) / (2.0*a*c)) * 180.0 / PI;
        double C = acos((1.0*a*a + 1.0*b*b - 1.0*c*c) / (2.0*a*b)) * 180.0 / PI;
        vector<double> ans = {A, B, C};
        sort(ans.begin(), ans.end());
        return ans;
    }
};

    Complexity: Time O(1), Space O(1). Three acos calls plus a 3-element sort.

    Optimal Approach

    Sort the sides once so that a ≤ b ≤ c. Reject if a + b ≤ c (degenerate / impossible). Otherwise compute two angles via the law of cosines and recover the third by complement to 180°. Since the largest side faces the largest angle, the output triple (A, B, C) is already non-decreasing.

    class Solution {
public:
    vector<double> internalAngles(vector<int>& sides) {
        sort(sides.begin(), sides.end());
        int a = sides[0], b = sides[1], c = sides[2];
        if (a + b <= c) return {};
        double PI = acos(-1.0);
        double A = acos((1.0*b*b + 1.0*c*c - 1.0*a*a) / (2.0*b*c)) * 180.0 / PI;
        double B = acos((1.0*a*a + 1.0*c*c - 1.0*b*b) / (2.0*a*c)) * 180.0 / PI;
        double C = 180.0 - A - B;
        return {A, B, C};
    }
};

    Why it works: The law of cosines is exact for any non-degenerate triangle. Casting to double with 1.0 * avoids overflow during b*b and produces the cosine in (−1, 1), so acos is well-defined.

    Complexity: Time O(1), Space O(1).

    Dry Run

    Take Example 1: sides = [3, 4, 5]. After sort, a=3, b=4, c=5.

    1. Triangle inequality: 3 + 4 = 7 > 5 ✓.
    2. cos A = (4² + 5² − 3²) / (2 · 4 · 5) = (16+25−9)/40 = 32/40 = 0.8. A = acos(0.8) · 180/π ≈ 36.8699°.
    3. cos B = (3² + 5² − 4²) / (2 · 3 · 5) = (9+25−16)/30 = 18/30 = 0.6. B = acos(0.6) · 180/π ≈ 53.1301°.
    4. C = 180 − 36.8699 − 53.1301 = 90.0000°.

    Output: [36.86990, 53.13010, 90.00000], already non-decreasing.

    Example 2: sides = [2, 4, 2] sorts to [2, 2, 4]; 2 + 2 = 4, not strictly greater than 4, so we return [].

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute (3 acos + sort)O(1)O(1)Three transcendental calls and an array sort.
    Optimal (sort + 2 acos)O(1)O(1)Third angle from 180° − A − B; output already ordered.

    Edge Cases & Pitfalls

    • Degenerate triangle (a + b == c): Zero area; the problem requires positive area, so use a strict ≤ rejection.
    • Integer overflow: b*b with b = 1000 fits in int, but multiplying three ints in 2*b*c could mislead — promote to double using 1.0 * to be safe and to keep the division floating-point.
    • Equilateral triangle: All sides equal yields three exact 60° angles; floating-point round-off may give 59.99999… — that is within the 10−5 tolerance.
    • Sorting input mutation: The optimal solution sorts sides in place. If the caller relies on the original order, copy first.
    • Numerical clamping: If you compute the third angle independently with acos, round-off can push the cosine just outside [-1, 1]. Recovering it as 180 − A − B avoids that pitfall entirely.

    vector<double> internalAngles(vector<int>& sides) {

        double B = acos((1.0*a*a + 1.0*c*c - 1.0*b*b) / (2.0*a*c)) * 180.0 / PI;
        double C = 180.0 - A - B;
        return {A, B, C};
    }
};
