    Multi Source Flood Fill

    Weekly Contest 498 · Q3 LeetCode #3905 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given two integers n and m representing the number of rows and columns of a grid, respectively.

    You are also given a 2D integer array sources, where sources[i] = [ri, ci, colori] indicates that the cell (ri, ci) is initially colored with colori. All other cells are initially uncolored and represented as 0.

    At each time step, every currently colored cell spreads its color to all adjacent uncolored cells in the four directions: up, down, left, and right. All spreads happen simultaneously.

    If multiple colors reach the same uncolored cell at the same time step, the cell takes the color with the maximum value.

    The process continues until no more cells can be colored.

    Return a 2D integer array representing the final state of the grid, where each cell contains its final color.

    Example 1:

    Input:  n = 3, m = 3, sources = [[0,0,1],[2,2,2]]
Output: [[1,1,2],[1,2,2],[2,2,2]]
Explanation: at time step 2, cells (0,2), (1,1), (2,0) are reached by both
colors simultaneously and take the larger value (2).

    Example 2:

    Input:  n = 3, m = 3, sources = [[0,1,3],[1,1,5]]
Output: [[3,3,3],[5,5,5],[5,5,5]]

    Example 3:

    Input:  n = 2, m = 2, sources = [[1,1,5]]
Output: [[5,5],[5,5]]

    Constraints:

    • 1 <= n, m <= 105
    • 1 <= n * m <= 105
    • 1 <= sources.length <= n * m
    • sources[i] = [ri, ci, colori]
    • 0 <= ri <= n - 1, 0 <= ci <= m - 1
    • 1 <= colori <= 106
    • All (ri, ci) in sources are distinct.

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The natural model is a multi-source BFS where every source starts at distance 0. Each uncolored cell ultimately gets the color of the source(s) closest to it (Manhattan distance through the grid), and on ties the largest source color wins.

    The catch: ordinary multi-source BFS, started from one big queue, breaks the tie rule. If you simply enqueue all sources, the first source you happen to push will "claim" border cells one step before the others are processed — that's an artifact of FIFO order, not the simultaneous-spread semantics. So we process the BFS one full layer at a time: gather all candidate (uncolored, neighbor-of-frontier) cells at the current step, take the max color among colors that reached each candidate, paint them, then promote them to the next frontier.

    Equivalently — and what we actually code — at each layer iterate all frontier cells, for every uncolored neighbor record the maximum of all colors arriving at this step in a hash map keyed by cell, then commit them in one batch.

    Brute Force

    Simulate the rules verbatim: at each time step, sweep the entire grid and for every uncolored cell collect the colors of its already-colored neighbors; if any exists, paint the cell with the maximum. Repeat until a full sweep makes no changes.

    class Solution {
public:
    vector<vector<int>> floodFill(int n, int m, vector<vector<int>>& sources) {
        vector<vector<int>> g(n, vector<int>(m, 0));
        for (auto& s : sources) g[s[0]][s[1]] = s[2];
        int dr[4] = {-1,1,0,0}, dc[4] = {0,0,-1,1};
        while (true) {
            vector<vector<int>> nxt = g;
            bool changed = false;
            for (int r = 0; r < n; ++r)
                for (int c = 0; c < m; ++c) if (g[r][c] == 0) {
                    int best = 0;
                    for (int d = 0; d < 4; ++d) {
                        int nr = r + dr[d], nc = c + dc[d];
                        if (nr >= 0 && nr < n && nc >= 0 && nc < m)
                            best = max(best, g[nr][nc]);
                    }
                    if (best > 0) { nxt[r][c] = best; changed = true; }
                }
            g = nxt;
            if (!changed) break;
        }
        return g;
    }
};

    Time O((n·m)²) in the worst case (one cell colored per sweep), space O(n·m). Correct, but quadratic.

    Optimal Approach

    Layered multi-source BFS. Initialise the frontier with all source cells. At every time step, scan each cell in the current frontier; for every uncolored neighbor, record the max color that reaches it during this step (use a small hash map keyed by the linearized cell index). After processing the frontier, commit all collected colors to the grid and promote those cells to be the next frontier. Stop when the frontier is empty.

    class Solution {
public:
    vector<vector<int>> floodFill(int n, int m, vector<vector<int>>& sources) {
        vector<vector<int>> g(n, vector<int>(m, 0));
        vector<pair<int,int>> frontier;
        for (auto& s : sources) {
            g[s[0]][s[1]] = s[2];
            frontier.push_back({s[0], s[1]});
        }
        int dr[4] = {-1,1,0,0}, dc[4] = {0,0,-1,1};
        while (!frontier.empty()) {
            unordered_map<int,int> best;  // cell id -> max color this step
            for (auto& [r, c] : frontier) {
                int col = g[r][c];
                for (int d = 0; d < 4; ++d) {
                    int nr = r + dr[d], nc = c + dc[d];
                    if (nr < 0 || nr >= n || nc < 0 || nc >= m) continue;
                    if (g[nr][nc] != 0) continue;
                    int id = nr * m + nc;
                    auto it = best.find(id);
                    if (it == best.end()) best[id] = col;
                    else it->second = max(it->second, col);
                }
            }
            frontier.clear();
            for (auto& [id, col] : best) {
                int r = id / m, c = id % m;
                g[r][c] = col;
                frontier.push_back({r, c});
            }
        }
        return g;
    }
};

    Why it works. Painting the whole layer in one batch enforces the simultaneous-spread rule: a cell sees every neighbor that was colored strictly before this step, picks the max among arrivals at this step, and only then becomes a source for the next layer. Each cell is added to best at most a constant number of times (once per direction) and each cell is colored at most once.

    Time O(n·m), space O(n·m).

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute simulationO((n·m)²)O(n·m)Recopies grid each step; works but slow.

    Edge Cases & Pitfalls

    • Naive single-queue multi-source BFS is wrong. If you use one shared FIFO without level batching, the source you happen to push first claims contested cells; layers must be processed atomically.
    • Tie-break is "max color", not "first color reached". Within a layer you must compute the max over arriving colors per cell before painting.
    • Don't read partly-painted cells in the same layer. Either commit the layer in a batch (as above) or expand from a snapshot of the previous frontier; otherwise a cell painted earlier in the loop becomes a source within the same time step.
    • Skinny grids. The constraint n, m ≤ 105 with n·m ≤ 105 means one of the dimensions can be huge (e.g. 1×100000). Use linearized cell ids r·m + c rather than vector<vector> visited markers if you want tighter memory.
    • Sources cover the grid. If sources.length == n·m, the BFS terminates after step 0; the loop must handle an empty subsequent frontier cleanly.
    • Color value 0 is special. 0 means "uncolored"; all real colors are ≥ 1, so the in-place sentinel works.

    C++ Code

    class Solution {
public:
    vector<vector<int>> floodFill(int n, int m, vector<vector<int>>& sources) {
        vector<vector<int>> g(n, vector<int>(m, 0));
        vector<pair<int,int>> frontier;
        frontier.reserve(sources.size());
        for (auto& s : sources) {
            g[s[0]][s[1]] = s[2];
            frontier.push_back({s[0], s[1]});
        }
        const int dr[4] = {-1, 1, 0, 0};
        const int dc[4] = { 0, 0,-1, 1};
        while (!frontier.empty()) {
            unordered_map<int, int> best;
            best.reserve(frontier.size() * 2);
            for (auto& [r, c] : frontier) {
                int col = g[r][c];
                for (int d = 0; d < 4; ++d) {
                    int nr = r + dr[d], nc = c + dc[d];
                    if (nr < 0 || nr >= n || nc < 0 || nc >= m) continue;
                    if (g[nr][nc] != 0) continue;
                    int id = nr * m + nc;
                    auto it = best.find(id);
                    if (it == best.end()) best.emplace(id, col);
                    else if (col > it->second) it->second = col;
                }
            }
            frontier.clear();
            frontier.reserve(best.size());
            for (auto& [id, col] : best) {
                int r = id / m, c = id % m;
                g[r][c] = col;
                frontier.push_back({r, c});
            }
        }
        return g;
    }
};
