    Smallest Stable Index I

    Weekly Contest 498 · Q1 LeetCode #3903 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given an integer array nums of length n and an integer k.

    For each index i, define its instability score as max(nums[0..i]) - min(nums[i..n - 1]).

    In other words:

    • max(nums[0..i]) is the largest value among the elements from index 0 to index i.
    • min(nums[i..n - 1]) is the smallest value among the elements from index i to index n - 1.

    An index i is called stable if its instability score is less than or equal to k.

    Return the smallest stable index. If no such index exists, return -1.

    Example 1:

    Input:  nums = [5,0,1,4], k = 3
Output: 3
Explanation: index 0 → 5−0=5, index 1 → 5−0=5, index 2 → 5−1=4,
             index 3 → 5−4=1 ≤ 3. Smallest stable index is 3.

    Example 2:

    Input:  nums = [3,2,1], k = 1
Output: -1
Explanation: every index has instability score 2, none ≤ 1.

    Example 3:

    Input:  nums = [0], k = 0
Output: 0

    Constraints:

    • 1 <= nums.length <= 100
    • 0 <= nums[i] <= 109
    • 0 <= k <= 109

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The instability score at index i is a difference between two monotonic quantities. As i increases, the prefix maximum max(nums[0..i]) can only grow or stay the same, and the suffix minimum min(nums[i..n-1]) can only grow or stay the same. The two terms move in opposite ways relative to the score, but each is cheap to maintain.

    So we can compute both sides once: a single right-to-left sweep gives us suffMin[i] = min(nums[i..n-1]), and then a single left-to-right sweep maintains the running prefix max. At every i we have both pieces in O(1) and check whether prefMax − suffMin[i] ≤ k. With n ≤ 100 the constraints are tiny, but the same idea scales — and is in fact what Q2 forces you to use.

    Brute Force

    For every index i recompute the prefix max and the suffix min from scratch. With n ≤ 100 this comfortably fits.

    class Solution {
public:
    int firstStableIndex(vector<int>& nums, int k) {
        int n = nums.size();
        for (int i = 0; i < n; ++i) {
            int mx = nums[0];
            for (int j = 1; j <= i; ++j) mx = max(mx, nums[j]);
            int mn = nums[i];
            for (int j = i + 1; j < n; ++j) mn = min(mn, nums[j]);
            if (mx - mn <= k) return i;
        }
        return -1;
    }
};

    Time O(n²), space O(1).

    Optimal Approach

    Precompute suffMin in one backward pass, then sweep forward maintaining a running prefMax. At each step compare prefMax − suffMin[i] against k.

    class Solution {
public:
    int firstStableIndex(vector<int>& nums, int k) {
        int n = nums.size();
        vector<int> suffMin(n);
        suffMin[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; --i)
            suffMin[i] = min(suffMin[i + 1], nums[i]);
        int prefMax = 0;
        for (int i = 0; i < n; ++i) {
            prefMax = max(prefMax, nums[i]);
            if (prefMax - suffMin[i] <= k) return i;
        }
        return -1;
    }
};

    Why it works. Both quantities are monotone in i, so any work done by the inner loops in the brute force is redundant — extending a prefix only adds a candidate to the max, and shrinking a suffix only removes one from the min. We replace recomputation by an incremental update.

    Time O(n), space O(n) (or O(1) if you scan once and store suffMin in-place).

    Edge Cases & Pitfalls

    • Single-element array. n = 1 always satisfies nums[0] − nums[0] = 0 ≤ k, so the answer is 0.
    • Inclusive endpoints. Both nums[i] participates in the prefix max and the suffix min — the windows overlap exactly at i. Off-by-one slicing breaks the score.
    • Strict vs. non-strict. The condition is ≤ k, not < k; missing this misclassifies boundary cases (e.g. k = 0 with constant arrays).
    • Initial prefMax. If you initialize prefMax with 0, it works only because nums[i] ≥ 0. For general inputs use INT_MIN or seed with nums[0].
    • Sorted-descending input. The score is constant nums[0] − nums[n-1] at every index; the answer is 0 if it fits, else -1.
