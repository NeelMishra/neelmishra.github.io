    Count Good Integers on a Grid Path

    Weekly Contest 498 · Q4 LeetCode #3906 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given two integers l and r, and a string directions consisting of exactly three 'D' characters and three 'R' characters.

    For each integer x in the range [l, r] (inclusive), perform the following steps:

    1. If x has fewer than 16 digits, pad it on the left with leading zeros to obtain a 16-digit string.
    2. Place the 16 digits into a 4 × 4 grid in row-major order (the first 4 digits form the first row from left to right, the next 4 digits form the second row, and so on).
    3. Starting at the top-left cell (row = 0, column = 0), apply the 6 characters of directions in order: 'D' increments the row by 1, 'R' increments the column by 1.
    4. Record the sequence of digits visited along the path (including the starting cell), producing a sequence of length 7.

    The integer x is considered good if the recorded sequence is non-decreasing.

    Return an integer representing the number of good integers in the range [l, r].

    Example 1:

    Input:  l = 8, r = 10, directions = "DDDRRR"
Output: 2
Explanation: x=8 → path digits [0,0,0,0,0,0,8] (good);
             x=9 → [0,0,0,0,0,0,9] (good);
             x=10 → [0,0,0,0,0,1,0] (not good).

    Example 2:

    Input:  l = 123456789, r = 123456790, directions = "DDRRDR"
Output: 1
Explanation: 123456789 visits [0,0,2,3,4,8,9] (good);
             123456790 visits [0,0,2,3,4,9,0] (not good).

    Example 3:

    Input:  l = 1288561398769758, r = 1288561398769758, directions = "RRRDDD"
Output: 0

    Constraints:

    • 1 <= l <= r <= 9 × 1015
    • directions.length == 6
    • directions consists of exactly three 'D' and three 'R' characters.

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The path is determined entirely by directions, so it visits a fixed set of 7 cells out of the 16. Mark those 7 positions in the row-major 16-digit string as "key positions". A number x (padded to 16 digits) is good iff the digits at those 7 key positions form a non-decreasing sequence — the other 9 digits can be anything from 0–9.

    Counting integers in [l, r] with such a digit-pattern condition is the classic digit DP setup: define f(x) = #good integers in [0, x] and return f(r) − f(l − 1). The DP scans the 16 digit positions, tracks the last key-cell digit (so the next key cell can constrain itself to be at least that), and a tight flag indicating whether we are still on the upper bound's prefix.

    Memoization works only when tight = false; states are (pos, last_key_digit), giving 16 × 10 = 160 unique non-tight states.

    Brute Force

    Enumerate every x in [l, r], pad to 16 digits, walk the path, and check non-decreasing.

    class Solution {
public:
    long long countGoodIntegersOnPath(long long l, long long r, string directions) {
        vector<int> idx;          // 7 key positions in 0..15
        int row = 0, col = 0;
        idx.push_back(0);
        for (char ch : directions) {
            if (ch == 'D') ++row; else ++col;
            idx.push_back(row * 4 + col);
        }
        long long ans = 0;
        for (long long x = l; x <= r; ++x) {
            char s[17]; snprintf(s, 17, "%016lld", x);
            bool ok = true;
            for (int i = 1; i < (int)idx.size() && ok; ++i)
                if (s[idx[i]] < s[idx[i - 1]]) ok = false;
            if (ok) ++ans;
        }
        return ans;
    }
};

    Time O((r − l + 1) · 7). With r up to 9·1015 this is hopeless; we need digit DP.

    Optimal Approach

    Build a boolean array key[0..15] from directions with true at the 7 visited positions. Define calc(x) = number of integers in [0, x] whose 16-digit padded form has non-decreasing digits at the key positions; return calc(r) − calc(l − 1).

    Inside calc, run a digit DP dfs(pos, last, tight):

    • pos: current digit index (0..16).
    • last: last digit placed at a key position (or 0 if none yet).
    • tight: whether the current prefix still matches the upper bound's prefix.

    At each position, the lower bound for the digit is last if this is a key position (to keep non-decreasing), else 0; the upper bound is s[pos] if tight, else 9.

    class Solution {
    bool key[16]{};
    long long memo[16][10];
    string s;
    long long dfs(int pos, int last, bool tight) {
        if (pos == 16) return 1;
        if (!tight && memo[pos][last] != -1) return memo[pos][last];
        int lo = key[pos] ? last : 0;
        int hi = tight ? (s[pos] - '0') : 9;
        long long res = 0;
        for (int d = lo; d <= hi; ++d) {
            res += dfs(pos + 1,
                       key[pos] ? d : last,
                       tight && (d == hi));
        }
        if (!tight) memo[pos][last] = res;
        return res;
    }
    long long calc(long long x) {
        if (x < 0) return 0;
        char buf[17]; snprintf(buf, 17, "%016lld", x);
        s = buf;
        for (auto& row : memo) for (auto& v : row) v = -1;
        return dfs(0, 0, true);
    }
public:
    long long countGoodIntegersOnPath(long long l, long long r, string directions) {
        for (auto& b : key) b = false;
        int row = 0, col = 0;
        key[0] = true;
        for (char c : directions) {
            if (c == 'D') ++row; else ++col;
            key[row * 4 + col] = true;
        }
        return calc(r) - calc(l - 1);
    }
};

    Why it works. Non-decreasing on the key cells depends only on the most recent key digit, so a single state last captures every constraint affecting future positions. Non-key cells are free choices in [0, 9] and don't affect last. The tight flag is the standard upper-bound trick of digit DP.

    Time O(D² · L) = O(10² · 16) = O(1600) per calc; space O(L · D) = 160 entries.

    Dry Run

    Take Example 1: l = 8, r = 10, directions = "DDDRRR".

    Path cells: start (0,0), then D D D R R R → (1,0), (2,0), (3,0), (3,1), (3,2), (3,3). Linearised key positions: {0, 4, 8, 12, 13, 14, 15}.

    For x = 10 padded to 0000000000000010:

    1. Digits at key positions = [s[0], s[4], s[8], s[12], s[13], s[14], s[15]] = [0, 0, 0, 0, 0, 1, 0].
    2. That sequence drops from 1 → 0 at the last step, so 10 is not good.

    Computing calc(10) − calc(7):

    • calc(10) walks digit-by-digit; the only good integers ≤ 10 with non-decreasing key digits are 0..9 (their key sequence is [0,0,0,0,0,0,d] which is non-decreasing for any single digit d). So calc(10) = 10.
    • calc(7) = 8 by the same argument (integers 0..7).
    • Answer = 10 − 8 = 2, matching the example.

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute enumerationO(r − l)O(1)Up to 9·1015 values; TLE.
    Digit DPO(L · D²) per calc (L = 16, D = 10)O(L · D)Two calc calls; effectively O(1).

    Edge Cases & Pitfalls

    • Always pad to 16 digits. r ≤ 9·1015 needs exactly 16 digit slots. Padding to fewer (e.g., the natural width of r) corrupts the row-major layout of the 4×4 grid.
    • Key cell at pos = 0. The starting cell (0,0) is always a key cell. Initialise last = 0 so the first key digit can be any 0–9.
    • Memoize only when tight is false. Tight states depend on the upper bound's prefix and aren't reusable across calls.
    • Reset memo between calc(r) and calc(l-1). The string s changes, so previously-memoized values for tight = true aren't valid; the simplest correct path is to reset all of memo.
    • calc(l - 1) when l = 1. Handle x = 0: padded form is sixteen zeros, key digits are [0,0,0,0,0,0,0] which is non-decreasing, so calc(0) = 1. Make sure your function returns 0 for x < 0.
    • last only updates on key positions. Non-key positions must pass last through unchanged; updating it everywhere overcounts/undercounts.
    • 64-bit return. The answer can exceed 231; use long long throughout.

    C++ Code

    class Solution {
    bool key[16]{};
    long long memo[16][10];
    string s;

    long long dfs(int pos, int last, bool tight) {
        if (pos == 16) return 1;
        if (!tight && memo[pos][last] != -1) return memo[pos][last];
        int lo = key[pos] ? last : 0;
        int hi = tight ? (s[pos] - '0') : 9;
        long long res = 0;
        for (int d = lo; d <= hi; ++d) {
            res += dfs(pos + 1,
                       key[pos] ? d : last,
                       tight && (d == hi));
        }
        if (!tight) memo[pos][last] = res;
        return res;
    }

    long long calc(long long x) {
        if (x < 0) return 0;
        char buf[17]; snprintf(buf, 17, "%016lld", x);
        s = buf;
        for (int i = 0; i < 16; ++i)
            for (int j = 0; j < 10; ++j) memo[i][j] = -1;
        return dfs(0, 0, true);
    }

public:
    long long countGoodIntegersOnPath(long long l, long long r, string directions) {
        for (int i = 0; i < 16; ++i) key[i] = false;
        int row = 0, col = 0;
        key[0] = true;
        for (char c : directions) {
            if (c == 'D') ++row; else ++col;
            key[row * 4 + col] = true;
        }
        return calc(r) - calc(l - 1);
    }
};
