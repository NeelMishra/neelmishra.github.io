    Check Adjacent Digit Differences

    Weekly Contest 502 · Q1 LeetCode #? Easy · 3 pts View on LeetCode ↗
    Topics strings simulation
    Tags string adjacent-pair digit-difference ad-hoc linear-scan

    Problem Statement

    You are given a string s consisting of digits.

    Return true if the absolute difference between every pair of adjacent digits is at most 2, otherwise return false.

    The absolute difference between a and b is defined as abs(a - b).

    Example 1:

    Input:  s = "132"
Output: true
Explanation: |1-3|=2, |3-2|=1. Both differences are at most 2.

    Example 2:

    Input:  s = "129"
Output: false
Explanation: |2-9|=7, which is greater than 2.

    Constraints:

    • 2 <= s.length <= 100
    • s consists only of digits.

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The predicate is purely local: each adjacent pair is checked independently, and the overall verdict is the logical AND of all per-pair verdicts. A single linear pass with an early return false on the first violator is asymptotically optimal — you can’t certify the whole string without inspecting every pair at least once.

    One small trick: the digit characters '0' through '9' are contiguous in ASCII, so subtracting two raw chars is identical to subtracting their numeric values — the '0' offset cancels. No stoi, no character-to-int conversion, no parsing.

    Brute Force

    There isn’t a meaningfully slower correct approach — even the "verbose" version is linear. Shown for clarity:

    class Solution {
public:
    bool hasSameDigits(string s) {
        for (int i = 1; i < (int)s.size(); ++i) {
            int a = s[i]     - '0';
            int b = s[i - 1] - '0';
            if (abs(a - b) > 2) return false;
        }
        return true;
    }
};

    Time O(n), space O(1).

    Optimal Approach

    Same asymptotic complexity, but tighter constants and fewer lines. Subtract chars directly because the '0' offset cancels:

    class Solution {
public:
    bool hasSameDigits(string s) {
        for (int i = 1; i < (int)s.size(); ++i)
            if (abs(s[i] - s[i - 1]) > 2) return false;
        return true;
    }
};

    Time O(n), space O(1).

    Why it works: if any adjacent pair fails, the whole string fails — short-circuit. If we reach the end of the loop, every pair satisfied the bound, so the verdict is true.

    Dry Run

    Take s = "132":

    is[i-1]s[i]|diff|OK?
    1'1''3'2yes (2 ≤ 2)
    2'3''2'1yes

    Loop completes without a violation → return true.

    For s = "129", at i = 2 we have |'2' - '9'| = 7 > 2 → return false immediately.

    Edge Cases & Pitfalls

    • Minimum length is 2: the loop body always runs at least once. For a hypothetical relaxed variant with n <= 1, the natural answer is true (no pairs to violate).
    • Bound is non-strict: "at most 2" is <= 2, not < 2. Off-by-one here flips Example 1 (|1-3|=2 must pass).
    • Char arithmetic: works because digits are contiguous in ASCII. If the alphabet ever expands beyond digits, switch back to explicit s[i] - '0' conversion to be defensive.
    • Signed/unsigned compare: s.size() is size_t; the cast (int)s.size() suppresses the benign warning.
