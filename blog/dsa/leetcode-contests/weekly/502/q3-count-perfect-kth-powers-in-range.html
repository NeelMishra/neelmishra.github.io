    Count Perfect Kth Powers in Range

    Weekly Contest 502 · Q3 LeetCode #? Medium · 5 pts View on LeetCode ↗
    Topics math
    Tags enumeration integer-overflow kth-power perfect-power special-case-k=1 bounded-iteration

    Problem Statement

    You are given three integers l, r, and k.

    An integer y is said to be a perfect kth power if there exists an integer x such that y = x^k.

    Return the number of integers y in the range [l, r] (inclusive) that are perfect kth powers.

    Example 1:

    Input:  l = 1, r = 9, k = 3
Output: 2
Explanation: Perfect cubes in [1, 9] are 1 = 1^3 and 8 = 2^3.

    Example 2:

    Input:  l = 8, r = 30, k = 2
Output: 3
Explanation: Perfect squares in [8, 30] are 9, 16, 25.

    Constraints:

    • 0 <= l <= r <= 109
    • 1 <= k <= 30

    Source: LeetCode · Statement © LeetCode.

    Intuition

    Instead of testing every y in [l, r], enumerate the bases. The candidate set is exactly $\{0^k, 1^k, 2^k, \ldots\}$, and we stop as soon as $x^k > r$. With $r \le 10^9$ and $k \ge 2$ the base $x$ is at most $\lfloor r^{1/k} \rfloor$:

    kmax x to try
    2$\approx 31{,}623$
    3$\approx 1{,}000$
    10$\approx 8$
    30$\approx 2$

    So for $k \ge 2$ this is trivially fast. The trap is k = 1: every integer is a perfect 1st power, so naive enumeration runs $\Theta(r) = 10^9$ iterations → TLE. Handle it as a closed-form r - l + 1.

    Negative bases add no new values in $[0, 10^9]$: for even $k$, $(-x)^k = x^k$ is already counted; for odd $k$, $(-x)^k < 0$ falls outside the range since $l \ge 0$. So we only iterate $x \ge 0$.

    Brute Force

    For each y in [l, r], check whether y is a perfect kth power by iterating x until $x^k \ge y$.

    class Solution {
public:
    int perfectPowers(int l, int r, int k) {
        int count = 0;
        for (int y = l; y <= r; ++y) {
            for (long long x = 0; ; ++x) {
                long long p = 1;
                bool over = false;
                for (int i = 0; i < k; ++i) { p *= x; if (p > (long long)2e9) { over = true; break; } }
                if (over || p > y) break;
                if (p == y) { ++count; break; }
            }
        }
        return count;
    }
};

    Time $O((r - l + 1) \cdot r^{1/k})$ — up to $\sim 10^9 \cdot 31{,}623 \approx 3 \times 10^{13}$ for $k = 2$. Hopeless.

    Optimal Approach

    Flip the iteration: enumerate bases $x = 0, 1, 2, \ldots$ and count those with $l \le x^k \le r$.

    1. If $k = 1$: every integer in [l, r] is a perfect 1st power. Return r - l + 1.
    2. Otherwise loop $x$ from $0$ upward. Compute $p = x^k$ inside an inner loop, breaking during multiplication if $p > r$ (this both speeds termination and guards against overflow).
    3. Stop the outer loop when the final $p > r$.
    class Solution {
public:
    int perfectPowers(int l, int r, int k) {
        if (k == 1) return r - l + 1;     // fast path; avoid 10^9-iter loop
        int count = 0;
        for (long long x = 0; ; ++x) {
            long long p = 1;
            bool stop = false;
            for (int i = 0; i < k; ++i) {
                p *= x;
                if (x > 1 && p > r) { stop = true; break; }
            }
            if (stop || p > r) break;
            if (p >= l) ++count;
        }
        return count;
    }
};

    Time $O(r^{1/k})$ (with the $k = 1$ fast path constant). Space $O(1)$.

    Why the overflow guard works: we break out of the inner loop the first time $p$ exceeds $r$. Before that break, $p$ was at most $r$, so the next multiplication produces at most $p \cdot x \le 10^9 \cdot 31{,}623 \approx 3 \times 10^{13}$ — comfortably inside long long. The x > 1 guard prevents tripping the break for $x \in \{0, 1\}$ where $p$ stays small forever.

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute force (test every y)O((r - l + 1) · r1/k)O(1)$\sim 3 \times 10^{13}$ for $k = 2$ — hopeless.
    Enumerate bases (this solution)O(r1/k)O(1)$k = 1$ closed-form. Otherwise ≤ ~31,623 iterations.

    Edge Cases & Pitfalls

    • $k = 1$ TLE trap: without the closed-form fast path, the loop runs up to $10^9$ iterations. Special-case it as r - l + 1.
    • $x = 0$: $0^k = 0$ for $k \ge 1$. The inner loop multiplies p *= 0 $k$ times, leaving p = 0. If l == 0 this contributes one count; if l > 0 it’s skipped naturally.
    • $x = 1$: $1^k = 1$. We never break inside the inner loop because x > 1 guards the early-exit. After the loop, p = 1; if $l \le 1 \le r$ it counts.
    • Overflow: $x^k$ can exceed int easily ($31{,}623^2 \approx 10^9$ already, and one more multiplication crosses $10^{13}$). Use long long for p and x; the inner-loop early break keeps every intermediate value under $\sim 3 \times 10^{13}$.
    • Negative bases: not needed. Even $k$ duplicates positive bases; odd $k$ produces negatives outside $[l, r]$ since $l \ge 0$.
    • $l = 0$: 0 is a perfect kth power (0 = 0^k) for any $k \ge 1$ and counts in [0, r]. The natural iteration handles it.
    • Empty inner range: if $r < 0$ (impossible here, but defensive) every $x$ produces $p \ge 0 > r$, and the loop breaks immediately.
