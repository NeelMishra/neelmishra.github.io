    Valid Elements in an Array

    Problem Statement

    You are given an integer array nums.

    An element nums[i] is considered valid if it satisfies at least one of the following conditions:

    • It is strictly greater than every element to its left.
    • It is strictly greater than every element to its right.

    The first and last elements are always valid.

    Return an array of all valid elements in the same order as they appear in nums.

    Example 1:

    Input:  nums = [1,2,4,2,3,2]
Output: [1,2,4,3,2]
Explanation: nums[0] and nums[5] are always valid; nums[1] and nums[2] are
strictly greater than every element to their left; nums[4] is strictly greater
than every element to its right.

    Example 2:

    Input:  nums = [5,5,5,5]
Output: [5,5]
Explanation: Only the first and last elements are valid; no other element is
strictly greater than all elements on either side.

    Example 3:

    Input:  nums = [1]
Output: [1]

    Constraints:

    • 1 <= nums.length <= 100
    • 1 <= nums[i] <= 100

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The validity predicate decomposes into two independent facts about each index i: is nums[i] larger than the maximum of nums[0..i-1], or is it larger than the maximum of nums[i+1..n-1]? The first and last indices are valid trivially because the corresponding side is empty.

    A direct double loop recomputes those prefix and suffix maxima for every i. Since both maxima are monotonic running aggregates, a single backward pass to record the suffix maxima plus a single forward pass with a rolling prefix max is enough — every index is decided in O(1) after the O(n) preprocessing.

    

    Brute Force

    For each index, scan everything to its left and to its right and check the strict-greater predicate.

    class Solution {
public:
    vector<int> findValidElements(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans;
        for (int i = 0; i < n; ++i) {
            bool leftOk = true, rightOk = true;
            for (int j = 0;     j < i; ++j) if (nums[j] >= nums[i]) leftOk  = false;
            for (int j = i + 1; j < n; ++j) if (nums[j] >= nums[i]) rightOk = false;
            if (leftOk || rightOk) ans.push_back(nums[i]);
        }
        return ans;
    }
};

    Time O(n2), space O(1) beyond the output.

    

    Optimal Approach

    Pre-compute right[i] = max(nums[i..n-1]) in a backwards sweep. Then walk left to right with a running left max. Index i is valid iff nums[i] > left (strictly greater than everything seen so far) or nums[i] > right[i+1] (strictly greater than everything still to come), with the boundary i == n - 1 handled explicitly.

    class Solution {
public:
    vector<int> findValidElements(vector<int>& nums) {
        int n = nums.size();
        vector<int> right(n);
        right[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; --i)
            right[i] = max(right[i + 1], nums[i]);
        int left = 0;
        vector<int> ans;
        for (int i = 0; i < n; ++i) {
            if (nums[i] > left || i == n - 1 || nums[i] > right[i + 1])
                ans.push_back(nums[i]);
            left = max(left, nums[i]);
        }
        return ans;
    }
};

    Time O(n), space O(n) for the suffix-max array.

    Why it works: the suffix-max array answers "is there anything bigger to the right?" in O(1), and the rolling left answers the symmetric left-side question. Together they encode the exact two clauses of the validity predicate without any redundant scanning.

    Edge Cases & Pitfalls

    • Single element: n == 1, the lone element is always valid.
    • All-equal array: nothing in the middle is strictly greater than its neighbours; only the first and last positions survive (e.g. [5,5,5,5] → [5,5]).
    • Strictly monotone arrays: a strictly increasing array yields all elements via the left rule; a strictly decreasing array yields all elements via the right rule.
    • Strict inequality matters: using >= instead of > would wrongly accept ties.
    • Initial left sentinel: seeding it with 0 works only because nums[i] >= 1 by the constraints; for unconstrained inputs use INT_MIN.
    • Right-index off-by-one: querying right[i + 1] requires the explicit i == n - 1 guard or you read past the end.
