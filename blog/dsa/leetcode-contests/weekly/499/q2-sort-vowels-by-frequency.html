    Sort Vowels by Frequency

    Weekly Contest 499 · Q2 LeetCode #3913 View on LeetCode ↗ Reference (doocs) ↗

    Problem Statement

    You are given a string s consisting of lowercase English characters.

    Rearrange only the vowels in the string so that they appear in non-increasing order of their frequency.

    If multiple vowels have the same frequency, order them by the position of their first occurrence in s.

    Return the modified string. Vowels are 'a', 'e', 'i', 'o', and 'u'. The frequency of a letter is the number of times it occurs in the string.

    Example 1:

    Input:  s = "leetcode"
Output: "leetcedo"
Explanation: Vowels are ['e','e','o','e'] with frequencies e=3, o=1.
Sorting non-increasing by frequency and refilling vowel slots gives "leetcedo".

    Example 2:

    Input:  s = "aeiaaioooa"
Output: "aaaaoooiie"
Explanation: a=4, o=3, i=2, e=1. Refilling vowel positions in this order
yields "aaaaoooiie".

    Example 3:

    Input:  s = "baeiou"
Output: "baeiou"
Explanation: Each vowel appears once, so first-occurrence order wins and
the string is unchanged.

    Constraints:

    • 1 <= s.length <= 105
    • s consists of lowercase English letters.

    Source: LeetCode · Statement © LeetCode.

    Intuition

    The vowel positions in s are immutable; only the multiset of letters that occupy those positions is being shuffled. So the question reduces to: in what order do we want to lay the vowels back down?

    The required order is "frequency descending, then first-occurrence ascending". The vowel alphabet has only five letters, so building that order is constant work: count each vowel, remember each one's first index, sort the (at most five) distinct vowels by the composite key, and stream them out into the saved vowel slots.

    The naive approach — collect all vowel chars into a list and sort that list — also works and is short to write, but doing the sort over the 5-element distinct-vowel list keeps the comparator cost independent of n.

    Brute Force

    Pull every vowel character into a vector, sort that vector with a comparator that ranks letters by frequency (desc) and first-occurrence index (asc), then write them back into the vowel positions of s.

    class Solution {
public:
    string sortVowels(string s) {
        auto isV = [](char c) {
            return c == 'a' || c == 'e' || c == 'i' || c == 'o' || c == 'u';
        };
        unordered_map<char, int> cnt, first;
        for (int i = 0; i < (int)s.size(); ++i) {
            if (isV(s[i])) {
                ++cnt[s[i]];
                if (!first.count(s[i])) first[s[i]] = i;
            }
        }
        vector<char> v;
        for (char c : s) if (isV(c)) v.push_back(c);
        sort(v.begin(), v.end(), [&](char a, char b) {
            if (cnt[a] != cnt[b]) return cnt[a] > cnt[b];
            return first[a] < first[b];
        });
        int k = 0;
        for (char& c : s) if (isV(c)) c = v[k++];
        return s;
    }
};

    Time O(n log n) in the worst case (almost-all-vowels string), space O(n).

    Optimal Approach

    Sort only the (up to five) distinct vowels by the composite key, build a single output buffer in that order, then walk the string and replace each vowel with the next character from the buffer.

    class Solution {
public:
    string sortVowels(string s) {
        const string vs = "aeiou";
        auto idx = [&](char c) -> int {
            size_t p = vs.find(c);
            return p == string::npos ? -1 : (int)p;
        };
        array<int, 5> cnt{}, firstIdx{};
        firstIdx.fill(INT_MAX);
        for (int i = 0; i < (int)s.size(); ++i) {
            int j = idx(s[i]);
            if (j >= 0) {
                if (cnt[j] == 0) firstIdx[j] = i;
                ++cnt[j];
            }
        }
        vector<int> order;
        for (int j = 0; j < 5; ++j) if (cnt[j] > 0) order.push_back(j);
        sort(order.begin(), order.end(), [&](int a, int b) {
            if (cnt[a] != cnt[b]) return cnt[a] > cnt[b];
            return firstIdx[a] < firstIdx[b];
        });
        string filled;
        for (int j : order) filled.append(cnt[j], vs[j]);
        int p = 0;
        for (char& c : s) if (idx(c) != -1) c = filled[p++];
        return s;
    }
};

    Time O(n), space O(n) for the filled buffer (the comparator runs over a 5-element list).

    Why it works: "non-increasing frequency, ties broken by first occurrence" is a total order on at most five distinct vowels, so we can pre-compute a permutation of the vowels and refill the slots in a single sweep. Consonants are never touched.

    Dry Run

    Take s = "aeiaaioooa".

    1. Count + first-occurrence pass:
      VowelCountFirst index
      a40
      e11
      i22
      o36
    2. Sort the distinct-vowel list by (count desc, first-idx asc): [a, o, i, e].
    3. Build filled buffer: "aaaa" + "ooo" + "ii" + "e" = "aaaaoooiie".
    4. Replay over s: every position in "aeiaaioooa" is a vowel, so the answer is just the filled buffer: "aaaaoooiie".

    Complexity Recap

    ApproachTimeSpaceNotes
    Brute forceO(n log n)O(n)Sort all vowel chars with a hash-map comparator.
    OptimalO(n)O(n)Sort only the ≤ 5 distinct vowels; refill in one pass.

    Edge Cases & Pitfalls

    • No vowels at all: consonant-only string is returned unchanged (the refill loop never fires).
    • Single character: trivially returned as-is.
    • All five vowels with equal frequency: first-occurrence order alone determines the output (Example 3).
    • Tie-breaker direction: ties go to earlier first occurrence; reversing the inequality silently fails on cases like "baeiou".
    • Consonants must stay put: only vowel positions are overwritten; do not touch the rest of the string or you change the answer.
    • Uppercase out of scope: constraints guarantee lowercase only; do not include 'A','E',... in the vowel set or the position sweep desyncs.
