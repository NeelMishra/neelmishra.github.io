    Burst Balloons

    June 2026 · 32 min read

    Burst Balloons is the problem that separates people who memorised interval DP from people who understand it. The naive framing leads to subproblems that refuse to separate. The fix is a single change of perspective, the same "decide the last operation" move from Matrix Chain Multiplication, but here it is genuinely non-obvious, which is exactly why the problem is worth your time.

    The Problem

    You have $n$ balloons in a row, balloon $i$ painted with number $a_i$. When you burst balloon $i$, you earn $a_{i-1} \cdot a_i \cdot a_{i+1}$ coins, where $a_{i-1}$ and $a_{i+1}$ are the balloons currently adjacent to $i$. If a neighbour has already burst, you use the next surviving balloon. Out-of-range neighbours are treated as a balloon painted $1$. Burst all the balloons; maximise total coins.

    Example: $a = [3, 1, 5, 8]$. One good order is burst $1$, then $5$, then $3$, then $8$, earning $3\cdot1\cdot5 + 3\cdot5\cdot8 + 1\cdot3\cdot8 + 1\cdot8\cdot1 = 15 + 120 + 24 + 8 = 167$.

    Why "Which Balloon First" Fails

    The instinct is to fix the balloon you burst first. Say you burst balloon $k$ first, earning $a_{-1} \cdot a_k \cdot a_{n}$. Now the row splits into the balloons left of $k$ and the balloons right of $k$. But here is the catch: those two groups are not independent. When you later burst the rightmost balloon of the left group, its right neighbour is whatever balloon survives in the right group. The two halves remain coupled through the hole that $k$ left behind. The subproblem you wanted to recurse on still depends on the other half. The recursion does not close.

    This is the lesson. Fixing the first operation couples the halves; fixing the last operation decouples them. Whenever an interval problem resists a clean split, ask what happens if you fix the last thing that occurs in the interval instead of the first.

    The Reframing: Which Balloon Bursts Last

    Pad the array with a virtual balloon $1$ on each end: work with $b = [1, a_0, a_1, \dots, a_{n-1}, 1]$, indices $0$ to $n+1$. Now consider an open interval $(i, j)$, meaning the balloons strictly between index $i$ and index $j$ (the boundaries $i$ and $j$ are not burst within this subproblem; they are the walls that survive on either side).

    Ask: in the optimal plan for interval $(i, j)$, which balloon $k$ (with $i < k < j$) is burst last? If $k$ is last, then at the moment it pops, every other balloon in $(i, j)$ is already gone, so its neighbours are exactly the walls $i$ and $j$. Bursting $k$ last earns $b_i \cdot b_k \cdot b_j$, a value that does not depend on the order of anything else. And before that final pop, the balloons in $(i, k)$ and $(k, j)$ were burst entirely within their own sub-intervals, walled off by $i, k$ and $k, j$ respectively. Now the halves are truly independent.

    Let $dp[i][j]$ be the most coins from bursting all balloons strictly inside the open interval $(i, j)$:

    $$dp[i][j] = \max_{i < k < j} \Big(\, dp[i][k] + dp[k][j] + b_i \cdot b_k \cdot b_j \,\Big),$$

    with $dp[i][j] = 0$ whenever $j \le i + 1$ (no balloons strictly between). The final answer is $dp[0][n+1]$, the whole padded row.

    Read the split carefully. The sub-intervals are $(i, k)$ and $(k, j)$, sharing the index $k$ as a wall, not $(i, k-1)$ and $(k+1, j)$. Because $k$ is burst last, it stands as a boundary for both halves the entire time they are being cleared. This shared-endpoint split is the signature of "burst last" interval DP.

    The Code

    int maxCoins(vector<int>& a) {
    int n = a.size();
    vector<int> b(n + 2, 1);
    for (int i = 0; i < n; ++i) b[i + 1] = a[i];   // b[0]=b[n+1]=1 walls

    vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));
    // fill by increasing gap = j - i; need at least one balloon strictly inside
    for (int gap = 2; gap <= n + 1; ++gap) {
        for (int i = 0; i + gap <= n + 1; ++i) {
            int j = i + gap;
            for (int k = i + 1; k < j; ++k) {       // k is burst LAST in (i, j)
                int coins = dp[i][k] + dp[k][j] + b[i] * b[k] * b[j];
                dp[i][j] = max(dp[i][j], coins);
            }
        }
    }
    return dp[0][n + 1];
}

    Three nested loops over $O(n)$ ranges give $O(n^3)$ time and $O(n^2)$ space, the same budget as MCM. The only structural differences from MCM are the boundary padding, the open-interval semantics, and the shared-endpoint split $(i,k) + (k,j)$.

    ▶ Burst-Last Interval DP — a = [3, 1, 5, 8]

    Padded row b = [1, 3, 1, 5, 8, 1]. We fill dp[i][j] over open intervals by increasing gap. For each interval, every interior balloon k is tried as the last to burst, earning b[i]*b[k]*b[j] plus the two already-solved halves. Step to watch the optimum build up to dp[0][5].

    Tracing the Winning Order

    To recover the burst order, store the argmax $\text{last}[i][j] = k$. The balloon $\text{last}[0][n+1]$ is the globally last balloon you pop; recurse into $(0, k)$ and $(k, n+1)$ for the order within each wall. Flattening that recursion (post-order, with the root balloon emitted last) reproduces a coin-optimal sequence. For $a = [3,1,5,8]$ the table yields the answer $167$ found by hand above.

    MCM vs Burst Balloons, Side by Side

    Split$[i,k] + [k{+}1,j]$$(i,k) + (k,j)$ (shared $k$)

    They are the same algorithm wearing different clothes. The hard part of Burst Balloons is not the code, it is realising that "last" is the thing to fix and that the boundary balloons survive as walls.

    Pitfalls

    • Forgetting the padding. Without the two virtual $1$ balloons you have to special-case the edges of the row, which is error-prone. Padding makes every interior balloon look the same.
    • Using $(i, k{-}1)$ and $(k{+}1, j)$. That is the closed-interval split and it double-counts or skips $k$. Because $k$ is burst last and acts as a wall, the correct halves share $k$: $(i, k)$ and $(k, j)$.
    • Maximising instead of minimising, or vice versa. Burst Balloons maximises coins; MCM minimises cost. Initialise the diagonal accordingly ($0$ works for both since empty intervals contribute nothing).

    What's Next

    One more interval problem to lock in the pattern from the opposite direction. In Minimum Cost to Cut a Stick the split represents the first cut you make inside a segment, and the merge term is simply the segment length. It shows that "first" can be correct too, as long as the cost of the operation does not depend on the order of the rest.
