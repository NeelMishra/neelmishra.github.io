    Matrix Chain Multiplication

    June 2026 · 30 min read

    Matrix Chain Multiplication (MCM) is the canonical interval partition DP. It teaches the move that every interval problem reuses: instead of asking "where do I cut first", you ask "which operation happens last", and that single reframing makes the two halves independent. If you internalise MCM, Burst Balloons and Cut a Stick become variations on a theme.

    Why Multiplication Order Matters

    Matrix multiplication is associative: $(AB)C = A(BC)$. The result never changes. But the cost does. Multiplying a $p \times q$ matrix by a $q \times r$ matrix costs $p \cdot q \cdot r$ scalar multiplications and yields a $p \times r$ matrix.

    Take three matrices with dimensions $A: 10 \times 100$, $B: 100 \times 5$, $C: 5 \times 50$.

    • $(AB)C$: first $AB$ costs $10 \cdot 100 \cdot 5 = 5000$ and gives $10 \times 5$; then times $C$ costs $10 \cdot 5 \cdot 50 = 2500$. Total $7500$.
    • $A(BC)$: first $BC$ costs $100 \cdot 5 \cdot 50 = 25000$ and gives $100 \times 50$; then $A$ times that costs $10 \cdot 100 \cdot 50 = 50000$. Total $75000$.

    Same answer matrix, but one order is ten times cheaper. With $n$ matrices the number of parenthesizations is the Catalan number $C_{n-1}$, which explodes, so we cannot try them all. We need DP.

    Encoding the Chain

    A chain of $n$ matrices $M_1, \dots, M_n$ is described by a single dimension array $p_0, p_1, \dots, p_n$, where matrix $M_i$ has shape $p_{i-1} \times p_i$. Consecutive matrices share a dimension, which is exactly why the chain is multipliable. For the example above, $p = [10, 100, 5, 50]$.

    The Key Idea: Fix the Last Multiplication

    Consider the product $M_i M_{i+1} \cdots M_j$. However you parenthesize it, there is exactly one multiplication performed last, and it splits the chain into a left product $M_i \cdots M_k$ and a right product $M_{k+1} \cdots M_j$ for some $k$ with $i \le k < j$. The left collapses to a $p_{i-1} \times p_k$ matrix, the right to a $p_k \times p_j$ matrix, and the final multiplication costs $p_{i-1} \cdot p_k \cdot p_j$.

    Crucially, once you fix $k$, the cost of the left side and the cost of the right side are independent subproblems. That gives the interval recurrence. Let $dp[i][j]$ be the minimum cost to multiply $M_i \cdots M_j$:

    $$dp[i][j] = \min_{i \le k < j} \Big(\, dp[i][k] + dp[k+1][j] + p_{i-1}\,p_k\,p_j \,\Big),$$ $$dp[i][i] = 0 \quad (\text{a single matrix needs no multiplication}).$$
    Last, not first. If you tried to fix the multiplication done first, the resulting two groups would still have to be combined in a way that depends on the order of everything else, and the subproblems would not separate cleanly. Fixing the last multiplication is what makes the left and right halves fully independent. This "decide the last operation" instinct is the heart of interval DP.

    Fill Order: By Increasing Length

    $dp[i][j]$ depends only on strictly shorter intervals ($dp[i][k]$ and $dp[k+1][j]$ both span fewer matrices). So we must compute all length-2 intervals before length-3, and so on. The classic bottom-up loop iterates over chain length $\ell$ from $2$ to $n$:

    // p has size n+1; matrices are 1..n, M_i is p[i-1] x p[i]
int n = p.size() - 1;
vector<vector<long long>> dp(n + 1, vector<long long>(n + 1, 0));
vector<vector<int>> cut(n + 1, vector<int>(n + 1, 0));   // for reconstruction

for (int len = 2; len <= n; ++len) {
    for (int i = 1; i + len - 1 <= n; ++i) {
        int j = i + len - 1;
        dp[i][j] = LLONG_MAX;
        for (int k = i; k < j; ++k) {
            long long cost = dp[i][k] + dp[k + 1][j]
                           + (long long)p[i - 1] * p[k] * p[j];
            if (cost < dp[i][j]) { dp[i][j] = cost; cut[i][j] = k; }
        }
    }
}
return dp[1][n];

    Three nested loops, each up to $n$, give $O(n^3)$ time and $O(n^2)$ space. For the usual $n \le 100$ or so this is instant.

    Top-down is often easier to get right. A memoized recursion solve(i, j) that returns $0$ when $i = j$ and otherwise loops $k$ from $i$ to $j-1$ produces the identical answer without you having to reason about fill order: the recursion naturally evaluates shorter intervals first. Reach for bottom-up only when you need the tightest constant factor.

    Recovering the Parenthesization

    The cut[i][j] table records the winning split. Recurse on it to print the optimal grouping:

    string build(int i, int j, vector<vector<int>>& cut) {
    if (i == j) return "M" + to_string(i);
    int k = cut[i][j];
    return "(" + build(i, k, cut) + " x " + build(k + 1, j, cut) + ")";
}

    For the worked $p = [10, 100, 5, 50]$ example this prints ((M1 x M2) x M3), the $7500$-cost order we found by hand.

    The Interval DP Template

    Strip MCM of its specifics and you get the shape that the rest of this series reuses:

    for length = 2 .. n:
    for each interval [i, j] of that length:
        dp[i][j] = best over split k in [i, j) of
                   dp[i][k] + dp[k+1][j] + merge(i, k, j)

    Only merge(i, k, j) changes between problems. In MCM it is $p_{i-1} p_k p_j$. In Burst Balloons the split represents the last balloon popped, and the merge term is the product of the two surviving neighbours. In Cut a Stick the split is the first cut inside a segment and the merge term is the segment's length.

    Common Pitfalls

    • Dimension off-by-one. $M_i$ is $p_{i-1} \times p_i$. The merge term $p_{i-1} p_k p_j$ uses the outer left dimension $p_{i-1}$, the split dimension $p_k$, and the outer right dimension $p_j$. Get these wrong and you will compute a plausible but incorrect cost.
    • Filling in the wrong order. Iterating $i$ and $j$ in plain increasing order reads sub-intervals before they are ready. Always loop by length (or memoize).
    • Initialising the diagonal to infinity. $dp[i][i] = 0$, not $\infty$. A single matrix is free.

    What's Next

    MCM gave you the "fix the last operation" reflex on a problem where the split point is obvious. Burst Balloons hides the trick: the natural "which balloon first" framing fails, and only "which balloon last" makes the halves independent. It is the single best exercise for cementing interval DP.
