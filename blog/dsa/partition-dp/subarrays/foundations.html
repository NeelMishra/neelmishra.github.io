    Partitioning Into K Subarrays

    June 2026 · 27 min read

    A huge slice of interview and contest DP is one template in disguise: split an array into exactly $k$ contiguous subarrays so that some aggregate over the pieces is optimal. Split Array Largest Sum, Largest Sum of Averages, the Painter's Partition, Allocate Books, and "minimum total cost to cut into $k$ groups" are all the same recurrence with a different cost function. This post derives that template once so the rest of the Partition DP subarray series is just plugging in costs.

    This is the piece-count extension of front partition: there we had a one-index state $dp[i]$; here we add a dimension for "how many subarrays so far".

    The Problem Family

    You are given an array $a_0, \dots, a_{n-1}$ and an integer $k$. A valid solution cuts the array into exactly $k$ non-empty contiguous blocks. Each block $[l, r]$ has a cost $\text{cost}(l, r)$, and the score of a partition is some combination of the block costs, usually one of:

    • minimise the maximum block cost (Split Array Largest Sum, Painter's Partition, Book Allocation),
    • maximise / minimise the sum of block costs (Largest Sum of Averages, minimum cost partitions),
    • count the number of valid partitions.

    The defining trait, exactly as in the front family, is that a block's cost depends only on its own elements. A range sum, a range average, a range max, a "cost to paint this stretch": all self-contained. That independence is what makes a per-prefix state correct.

    The State and Recurrence

    Use a length-based prefix again. Define

    $$dp[i][p] = \text{best score for the first } i \text{ elements split into exactly } p \text{ subarrays}.$$

    Fix the last subarray. It ends at index $i-1$ and starts at some index $j$ (so it is the closed range $[j, i-1]$). Everything before it, the first $j$ elements, is split into $p-1$ subarrays, an already-solved subproblem $dp[j][p-1]$. Then:

    $$dp[i][p] = \operatorname*{opt}_{p-1 \le j \le i-1} \Big(\, \text{combine}\big(dp[j][p-1],\; \text{cost}(j, i-1)\big) \,\Big).$$

    The base case is $dp[0][0] = $ identity (0 for sums, $-\infty$ or $+\infty$ depending on direction for the rest), with all unreachable cells set to the "impossible" sentinel. The lower bound $j \ge p-1$ just encodes that you need at least one element per earlier block, so the first $j$ elements must number at least $p-1$.

    Objectivecombine(a, c)opt
    Min largest block$\max(a, c)$$\min$
    Max sum of block scores$a + c$$\max$
    Min sum of block costs$a + c$$\min$
    One template, three problems. Notice that only the combine operator and the opt direction change between rows. The loop structure, the prefix, and the fill order are identical. Internalise the skeleton and each new problem is a five-minute adaptation.

    The Cost Oracle: Prefix Sums

    Almost every block cost reduces to a range sum, so precompute prefix sums once and answer $\text{sum}(l, r)$ in $O(1)$:

    vector<long long> pre(n + 1, 0);
for (int i = 0; i < n; ++i) pre[i + 1] = pre[i] + a[i];
auto sum = [&](int l, int r) { return pre[r + 1] - pre[l]; };   // closed [l, r]

    For averages, divide the range sum by the length. For "max in range" costs (rare in this family) you would precompute a sparse table instead. The point is to make cost(l, r) an $O(1)$ lookup so the DP itself is the only quadratic factor. See the prefix-sum deep dive if this is unfamiliar.

    The Template in C++

    Here is the min-largest-block instance, the one the animation traces. The other objectives swap two lines:

    const long long INF = LLONG_MAX / 4;
// dp[i][p] = min possible largest-block-sum for first i elements in p blocks
vector<vector<long long>> dp(n + 1, vector<long long>(k + 1, INF));
dp[0][0] = 0;
for (int p = 1; p <= k; ++p)
    for (int i = p; i <= n; ++i)                 // need >= p elements for p blocks
        for (int j = p - 1; j < i; ++j) {        // last block is [j, i-1]
            if (dp[j][p - 1] == INF) continue;
            long long blk = sum(j, i - 1);
            long long cand = max(dp[j][p - 1], blk);   // combine = max
            dp[i][p] = min(dp[i][p], cand);            // opt = min
        }
return dp[n][k];

    Three nested loops bounded by $n, n, k$ give $O(n^2 k)$ time and $O(nk)$ space. For the typical $n \le 1000,\ k \le n$ this is comfortably fast.

    Exactly K vs At Most K

    Some problems (Largest Sum of Averages) allow at most $k$ groups rather than exactly $k$. Two clean ways to handle it:

    • Answer aggregation. Compute $dp[n][p]$ for every $p$ and take the best over $p = 1 \dots k$. For "at most $k$" with a monotone benefit from more cuts, the optimum is at $p = k$, but aggregating is always safe.
    • Reframe the base case. Let $dp[i][p]$ mean "first $i$ elements in at most $p$ groups" and allow $dp[i][p] = dp[i][p-1]$ (use fewer groups) as one transition. This folds the aggregation into the recurrence.

    Reconstructing the Cuts

    Store the argopt: from[i][p] = j, the start of the last block that achieved $dp[i][p]$. Walk back from $(n, k)$: emit block $[\,\text{from}[i][p],\, i-1\,]$, then move to $(\text{from}[i][p],\, p-1)$, repeat until $p = 0$, and reverse. This recovers the actual $k$ subarrays, not just the score.

    When To Prefer Binary Search Instead

    For the min largest block objective specifically, there is a famous alternative: binary search on the answer. Guess a cap $X$ and greedily check whether the array can be cut into $\le k$ blocks each with sum $\le X$; the feasibility is monotone in $X$, so binary search finds the smallest feasible $X$ in $O(n \log(\sum a))$, beating the $O(n^2 k)$ DP. We build that solution in full in Part 2. The DP still matters: it generalises to objectives where no monotone feasibility test exists (sum of averages, for instance).

    Pitfalls

    • Forgetting the reachability sentinel. $dp[j][p-1]$ may be unreachable (too few elements for $p-1$ blocks). Skip those, or you will combine with a garbage value.
    • Wrong loop bounds. You need $i \ge p$ (at least one element per block) and $j \ge p-1$. Off-by-one here silently allows empty blocks.
    • Mixing inclusive and exclusive indices. With the length-based prefix, the last block is the closed range $[j, i-1]$ and the carried state is $dp[j][p-1]$. Write it down before coding.
