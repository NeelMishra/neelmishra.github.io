    Interval Patterns

    March 2026 · 35 min read

    Introduction

    The merge intervals algorithm from the previous post is the foundation. This post covers six patterns that build on the same core ideas: sort intervals, scan with comparisons, and use overlap conditions to make decisions. Each pattern has a different goal, but the thinking process is similar.

    1. Insert Interval: Insert a new interval into a sorted non-overlapping list.
    2. Non-overlapping Intervals: Minimum removals to eliminate all overlaps.
    3. Meeting Rooms I: Can one person attend all meetings?
    4. Meeting Rooms II: Minimum conference rooms needed.
    5. Interval Intersection: Find all common intervals between two lists.
    6. Minimum Number of Arrows: Fewest points to hit all intervals.

    Pattern 1: Insert Interval

    LeetCode 57

    Given a sorted list of non-overlapping intervals and a new interval, insert the new interval and merge if necessary. The output should remain sorted and non-overlapping.

    Example: intervals = [[1,3],[6,9]], newInterval = [2,5]. Output: [[1,5],[6,9]].

    Approach

    Scan through the existing intervals from left to right. There are three phases:

    Before the new interval

    All intervals that end before the new interval starts have no overlap. Add them to the result as-is.

    Overlapping with the new interval

    Any interval whose start is less than or equal to the new interval's end (and was not already added in Phase 1) overlaps. Merge them all by taking the minimum start and maximum end.

    After the new interval

    All remaining intervals start after the merged interval ends. Add them to the result as-is.

    Code

    // C++
vector<vector<int>> insert(vector<vector<int>>& intervals,
                           vector<int>& newInterval) {
    vector<vector<int>> result;
    int i = 0, n = intervals.size();

    // Phase 1: intervals ending before newInterval starts
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.push_back(intervals[i++]);
    }

    // Phase 2: overlapping intervals
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = min(newInterval[0], intervals[i][0]);
        newInterval[1] = max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.push_back(newInterval);

    // Phase 3: intervals starting after newInterval ends
    while (i < n) {
        result.push_back(intervals[i++]);
    }

    return result;
}
    # Python
def insert(intervals, newInterval):
    result = []
    i, n = 0, len(intervals)

    # Phase 1: before the new interval
    while i < n and intervals[i][1] < newInterval[0]:
        result.append(intervals[i])
        i += 1

    # Phase 2: overlapping with the new interval
    while i < n and intervals[i][0] <= newInterval[1]:
        newInterval[0] = min(newInterval[0], intervals[i][0])
        newInterval[1] = max(newInterval[1], intervals[i][1])
        i += 1
    result.append(newInterval)

    # Phase 3: after the new interval
    while i < n:
        result.append(intervals[i])
        i += 1

    return result

    Time: $O(n)$ since the list is already sorted. Space: $O(n)$ for the output.

    Pattern 2: Non-overlapping Intervals

    LeetCode 435

    Given a list of intervals, find the minimum number of intervals to remove so that the remaining intervals do not overlap.

    Example: [[1,2],[2,3],[3,4],[1,3]]. Remove [1,3] to get [[1,2],[2,3],[3,4]]. Answer: 1.

    Greedy Insight

    This is an activity selection problem. Sort by end time. When two intervals overlap, always keep the one that ends earlier (it leaves more room for future intervals). Count the removals.

    Sort by end

    Sort all intervals by their end value. When ends are equal, the start does not matter.

    Greedy scan

    Keep track of the end of the last kept interval. For each interval: if its start is less than the last end, it overlaps, so remove it (increment counter). Otherwise, keep it and update the last end.

    Visual

    Intervals sorted by end time, shown on a timeline. Each step decides whether to keep or discard the current interval.

    Code

    // C++
int eraseOverlapIntervals(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end(),
         [](const auto& a, const auto& b) {
             return a[1] < b[1];
         });

    int removals = 0;
    int lastEnd = intervals[0][1];

    for (int i = 1; i < intervals.size(); i++) {
        if (intervals[i][0] < lastEnd) {
            removals++;  // overlap: remove this one
        } else {
            lastEnd = intervals[i][1];  // no overlap: keep it
        }
    }
    return removals;
}
    # Python
def eraseOverlapIntervals(intervals):
    intervals.sort(key=lambda x: x[1])
    removals = 0
    last_end = intervals[0][1]

    for i in range(1, len(intervals)):
        if intervals[i][0] < last_end:
            removals += 1  # overlap: remove this one
        else:
            last_end = intervals[i][1]  # no overlap: keep it

    return removals

    Time: $O(n \log n)$. Space: $O(1)$ extra (or $O(\log n)$ for sorting).

    Why sort by end, not start? Sorting by end is the classic activity selection strategy. Keeping the interval that ends earliest maximizes the remaining time for future intervals. Sorting by start does not guarantee this.

    Pattern 3: Meeting Rooms I

    LeetCode 252

    Given a list of meeting time intervals, determine if a person can attend all meetings (no two meetings overlap).

    Example: [[0,30],[5,10],[15,20]]. The meeting [5,10] overlaps with [0,30]. Answer: false.

    Approach

    Sort by start time. If any meeting starts before the previous one ends, there is a conflict.

    Code

    // C++
bool canAttendMeetings(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end());
    for (int i = 1; i < intervals.size(); i++) {
        if (intervals[i][0] < intervals[i - 1][1]) {
            return false;
        }
    }
    return true;
}
    # Python
def canAttendMeetings(intervals):
    intervals.sort()
    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False
    return True

    Time: $O(n \log n)$. Space: $O(1)$ extra.

    Pattern 4: Meeting Rooms II

    LeetCode 253

    Given a list of meeting time intervals, find the minimum number of conference rooms required so that no two overlapping meetings share a room.

    Example: [[0,30],[5,10],[15,20]]. We need 2 rooms: one for [0,30] and one that handles [5,10] then [15,20].

    Approach: Event Line Sweep

    Create a list of events: each interval contributes a +1 at its start and a -1 at its end. Sort all events by time (break ties by processing -1 before +1, so that a room freed at time $t$ can be used by a meeting starting at time $t$). Sweep through and track the running count. The maximum count at any point is the answer.

    Create events

    For each interval [s,e], create two events: (s, +1) for a meeting starting and (e, -1) for a meeting ending.

    Sort events

    Sort by time. If two events have the same time, process the end event first.

    Sweep

    Walk through events, maintaining a running count of active meetings. Track the maximum.

    Visual: Room Allocation Timeline

    Watch how meetings are assigned to rooms. The line sweep counts how many meetings are active at each point in time.

    Code

    // C++
int minMeetingRooms(vector<vector<int>>& intervals) {
    vector<pair<int,int>> events;
    for (auto& iv : intervals) {
        events.push_back({iv[0], 1});   // meeting starts
        events.push_back({iv[1], -1});  // meeting ends
    }
    // Sort by time; end events (-1) before start events (+1) at same time
    sort(events.begin(), events.end());

    int rooms = 0, maxRooms = 0;
    for (auto& [time, delta] : events) {
        rooms += delta;
        maxRooms = max(maxRooms, rooms);
    }
    return maxRooms;
}
    # Python
def minMeetingRooms(intervals):
    events = []
    for start, end in intervals:
        events.append((start, 1))    # meeting starts
        events.append((end, -1))     # meeting ends
    events.sort()  # -1 < 1, so ends sort before starts at same time

    rooms = 0
    max_rooms = 0
    for time, delta in events:
        rooms += delta
        max_rooms = max(max_rooms, rooms)
    return max_rooms

    Time: $O(n \log n)$ for sorting the events. Space: $O(n)$ for the events list.

    Alternative approach: Use a min-heap (priority queue) keyed on end times. For each meeting (sorted by start), if the earliest ending meeting has ended, reuse that room (pop the heap). Otherwise, allocate a new room. The heap size at the end is the answer. Same time complexity but sometimes easier to reason about.

    Min-Heap Approach

    // C++ with priority queue
int minMeetingRooms(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end());
    priority_queue<int, vector<int>, greater<int>> pq; // min-heap of end times

    for (auto& iv : intervals) {
        if (!pq.empty() && pq.top() <= iv[0]) {
            pq.pop();  // reuse the room
        }
        pq.push(iv[1]);
    }
    return pq.size();
}
    # Python with heapq
import heapq

def minMeetingRooms(intervals):
    intervals.sort()
    heap = []  # min-heap of end times

    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heappop(heap)  # reuse the room
        heapq.heappush(heap, end)

    return len(heap)

    Pattern 5: Interval List Intersections

    LeetCode 986

    Given two sorted lists of non-overlapping intervals, find the intersection of these two lists. Each intersection is itself an interval (a range common to both).

    Example:

    A = [[0,2],[5,10],[13,23],[24,25]]

    B = [[1,5],[8,12],[15,24],[25,26]]

    Output: [[1,2],[5,5],[8,10],[15,23],[24,24],[25,25]]

    Approach: Two Pointer Scan

    Use one pointer for each list. At each step, the two current intervals either overlap or they do not. If they overlap, the intersection is [max(starts), min(ends)]. Then advance the pointer for the interval that ends first (it cannot intersect with anything beyond the current interval in the other list).

    Compare current intervals

    Let A[i] and B[j] be the current intervals. Compute lo = max(A[i].start, B[j].start) and hi = min(A[i].end, B[j].end). If lo is less than or equal to hi, the intersection is [lo, hi].

    Advance the pointer

    If A[i].end is less than B[j].end, advance i (A[i] can no longer intersect with B[j+1] or later). Otherwise, advance j.

    Code

    // C++
vector<vector<int>> intervalIntersection(
    vector<vector<int>>& A, vector<vector<int>>& B) {
    vector<vector<int>> result;
    int i = 0, j = 0;

    while (i < A.size() && j < B.size()) {
        int lo = max(A[i][0], B[j][0]);
        int hi = min(A[i][1], B[j][1]);
        if (lo <= hi) {
            result.push_back({lo, hi});
        }
        // Advance the pointer with the smaller end
        if (A[i][1] < B[j][1]) i++;
        else j++;
    }
    return result;
}
    # Python
def intervalIntersection(A, B):
    result = []
    i, j = 0, 0

    while i < len(A) and j < len(B):
        lo = max(A[i][0], B[j][0])
        hi = min(A[i][1], B[j][1])
        if lo <= hi:
            result.append([lo, hi])
        # Advance the pointer with the smaller end
        if A[i][1] < B[j][1]:
            i += 1
        else:
            j += 1

    return result

    Time: $O(n + m)$ where $n$ and $m$ are the lengths of the two lists. Space: $O(k)$ where $k$ is the number of intersection intervals in the output.

    Pattern 6: Minimum Number of Arrows to Burst Balloons

    LeetCode 452

    Balloons are represented as intervals on a horizontal line. An arrow shot vertically at position $x$ bursts all balloons where $start \le x \le end$. Find the minimum number of arrows needed to burst all balloons.

    Example: [[10,16],[2,8],[1,6],[7,12]]. Two arrows at positions 6 and 11 burst all four balloons. Answer: 2.

    Approach

    This is equivalent to the non-overlapping intervals problem, viewed differently. Sort by end. Shoot an arrow at the end of the first balloon. Any balloon that starts after this arrow's position needs a new arrow.

    Sort by end

    Sort balloons by their right edge (end coordinate).

    Greedy arrow placement

    Place the first arrow at the end of the first balloon. For each subsequent balloon: if its start is after the arrow position, it is not hit. Place a new arrow at this balloon's end.

    Visual

    Balloons on a number line. Arrows are shot at specific positions. Watch how each arrow bursts all overlapping balloons.

    Code

    // C++
int findMinArrowShots(vector<vector<int>>& points) {
    sort(points.begin(), points.end(),
         [](const auto& a, const auto& b) {
             return a[1] < b[1];
         });

    int arrows = 1;
    int arrowPos = points[0][1];

    for (int i = 1; i < points.size(); i++) {
        if (points[i][0] > arrowPos) {
            arrows++;
            arrowPos = points[i][1];
        }
    }
    return arrows;
}
    # Python
def findMinArrowShots(points):
    points.sort(key=lambda x: x[1])
    arrows = 1
    arrow_pos = points[0][1]

    for i in range(1, len(points)):
        if points[i][0] > arrow_pos:
            arrows += 1
            arrow_pos = points[i][1]

    return arrows

    Time: $O(n \log n)$. Space: $O(1)$ extra.

    Connection to Non-overlapping Intervals: The minimum number of arrows equals the maximum set of non-overlapping intervals (after the greedy grouping). Equivalently, arrows = total intervals minus max removals to make intervals non-overlapping is not quite right. More precisely: each group of mutually overlapping balloons needs exactly one arrow, and the greedy sort-by-end approach finds these groups optimally.

    Pattern Comparison

    PatternSort ByCore IdeaTime
    Insert Interval (already sorted) Three-phase scan: before, overlap, after $O(n)$
    Non-overlapping Intervals End Greedy: keep the one ending earliest $O(n \log n)$
    Meeting Rooms I Start Check consecutive overlap $O(n \log n)$
    Meeting Rooms II Event time Line sweep with +1/-1 events $O(n \log n)$
    Interval Intersection (already sorted) Two pointers, advance the smaller end $O(n + m)$
    Min Arrows End Greedy arrow at rightmost point of first group $O(n \log n)$

    Key Takeaways

    • Most interval problems reduce to: sort, then scan with one or two pointers.
    • Sort by start when order matters (merge, insert, meeting rooms I). Sort by end for greedy selection (non-overlapping, arrows).
    • The overlap condition is the same everywhere: one interval starts before the other ends.
    • Meeting Rooms II introduces the line sweep technique, which generalizes to many scheduling problems.
    • Interval intersection uses the two-pointer technique because both lists are already sorted.
    • The greedy arrow placement is equivalent to activity selection: group overlapping intervals and count the groups.
