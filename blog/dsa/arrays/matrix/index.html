    Matrix Fundamentals

    March 2026 · 20 min read

    What Is a Matrix?

    A matrix is a two-dimensional array arranged in rows and columns. In programming, you represent it as an array of arrays. A matrix with m rows and n columns is called an m x n matrix. Each element is accessed by its row index and column index.

    Matrices appear everywhere in programming: image processing (pixels in a grid), game boards, graph adjacency matrices, dynamic programming tables, and spreadsheet-like data. Mastering matrix traversal patterns is essential because many interview problems are fundamentally about visiting cells in a specific order.

    Key Idea: Every matrix problem reduces to deciding which cells to visit and in what order. The traversal pattern determines everything.

    Row-Major Storage

    Computer memory is linear, a flat sequence of addresses. When you create a 2D array, the system must map two dimensions into one. Most languages (C, C++, Java, Python) use row-major order: all elements of row 0 come first, then all of row 1, and so on.

    For a 3x4 matrix, element at position (i, j) maps to the linear index i * numCols + j. This means accessing elements in row order is cache-friendly because they are stored contiguously in memory. Column-wise access jumps across memory, which is slower.

    Green = Row 0, Orange = Row 1, Purple = Row 2. All stored back to back in memory.

    Converting 2D to 1D

    For a matrix with C columns: linearIndex = row * C + col

    For the reverse: row = linearIndex / C, col = linearIndex % C

    Row-by-Row Traversal

    The simplest traversal. Visit every element from left to right in each row, starting from row 0. This is the most cache-friendly pattern because it follows the exact order elements are stored in memory.

    Press Step to begin. 
    // Row-by-row traversal
void rowTraversal(vector<vector<int>>& mat) {
    int m = mat.size(), n = mat[0].size();
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            // process mat[i][j]
        }
    }
}
// Time: O(m * n), Space: O(1)

    Column-by-Column Traversal

    Visit every element from top to bottom in each column, starting from column 0. This is less cache-friendly than row traversal because you jump across rows in memory. Still useful when problems require processing data column-wise, for example, finding the first non-zero element in each column.

    Press Step to begin. 
    // Column-by-column traversal
void columnTraversal(vector<vector<int>>& mat) {
    int m = mat.size(), n = mat[0].size();
    for (int j = 0; j < n; j++) {
        for (int i = 0; i < m; i++) {
            // process mat[i][j]
        }
    }
}
// Time: O(m * n), Space: O(1)

    Diagonal Traversal

    There are two flavors. The primary diagonals run from top-left to bottom-right. All cells on the same primary diagonal share the same value of i - j. The anti-diagonals run from top-right to bottom-left, and all cells share the same value of i + j.

    A common pattern is to iterate over each diagonal by looping over the starting cells along the top row and left column.

    Press Step to begin. 
    // Primary diagonal traversal (top-left to bottom-right)
void diagonalTraversal(vector<vector<int>>& mat) {
    int m = mat.size(), n = mat[0].size();
    // Start from each cell in the first row
    for (int col = 0; col < n; col++) {
        int i = 0, j = col;
        while (i < m && j < n) {
            // process mat[i][j]
            i++; j++;
        }
    }
    // Start from each cell in the first column (skip 0,0)
    for (int row = 1; row < m; row++) {
        int i = row, j = 0;
        while (i < m && j < n) {
            // process mat[i][j]
            i++; j++;
        }
    }
}
// Time: O(m * n), Space: O(1)

    Diagonal Indexing Trick

    For an m x n matrix, there are m + n - 1 primary diagonals. Diagonal number d starts at row = max(0, d - n + 1), col = max(0, n - 1 - d) ... or simply enumerate using the two-loop approach above. The key insight: cells (i, j) and (i+1, j+1) are always on the same primary diagonal.

    Zigzag (Anti-Diagonal) Traversal

    This pattern visits cells along anti-diagonals, alternating direction: the first anti-diagonal goes downward, the next goes upward, and so on. This is exactly what LeetCode 498 "Diagonal Traverse" asks for.

    All cells on anti-diagonal d satisfy i + j = d. There are m + n - 1 anti-diagonals total (numbered 0 through m + n - 2). For even-numbered diagonals, walk upward (decreasing row). For odd-numbered diagonals, walk downward (increasing row).

    Press Step to begin. 
    // Zigzag (anti-diagonal) traversal
vector<int> zigzagTraversal(vector<vector<int>>& mat) {
    int m = mat.size(), n = mat[0].size();
    vector<int> result;
    for (int d = 0; d < m + n - 1; d++) {
        if (d % 2 == 0) {
            // Walk upward: row decreases, col increases
            int r = min(d, m - 1);
            int c = d - r;
            while (r >= 0 && c < n) {
                result.push_back(mat[r][c]);
                r--; c++;
            }
        } else {
            // Walk downward: row increases, col decreases
            int c = min(d, n - 1);
            int r = d - c;
            while (r < m && c >= 0) {
                result.push_back(mat[r][c]);
                r++; c--;
            }
        }
    }
    return result;
}
// Time: O(m * n), Space: O(1) extra (excluding output)

    Spiral Traversal

    Spiral traversal visits a matrix layer by layer, starting from the outermost ring and moving inward. Each layer consists of four legs: right across the top, down the right side, left across the bottom, and up the left side. After completing one ring, the boundaries shrink inward and the process repeats.

    Maintain four boundaries: top, bottom, left, right. After traversing each edge, shrink the corresponding boundary. Stop when the boundaries cross.

    Press Step to begin. 
    // Spiral order traversal
vector<int> spiralOrder(vector<vector<int>>& mat) {
    vector<int> result;
    if (mat.empty()) return result;
    int top = 0, bottom = mat.size() - 1;
    int left = 0, right = mat[0].size() - 1;

    while (top <= bottom && left <= right) {
        // Traverse right across top row
        for (int j = left; j <= right; j++)
            result.push_back(mat[top][j]);
        top++;

        // Traverse down the right column
        for (int i = top; i <= bottom; i++)
            result.push_back(mat[i][right]);
        right--;

        // Traverse left across bottom row
        if (top <= bottom) {
            for (int j = right; j >= left; j--)
                result.push_back(mat[bottom][j]);
            bottom--;
        }

        // Traverse up the left column
        if (left <= right) {
            for (int i = bottom; i >= top; i--)
                result.push_back(mat[i][left]);
            left++;
        }
    }
    return result;
}
// Time: O(m * n), Space: O(1) extra (excluding output)

    Extra Boundary Checks

    The checks if (top <= bottom) before the leftward pass and if (left <= right) before the upward pass are essential. Without them, you will double-count elements when the remaining area is a single row or single column.

    Boundary and Index Tricks

    Several indexing patterns come up repeatedly in matrix problems:

    Four Directional Neighbors

    For BFS/DFS on grids, you visit the four neighbors of cell (i, j): up, down, left, right. Define a direction array:

    int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

for (int d = 0; d < 4; d++) {
    int ni = i + dx[d], nj = j + dy[d];
    if (ni >= 0 && ni < m && nj >= 0 && nj < n) {
        // (ni, nj) is a valid neighbor
    }
}

    Eight Directional Neighbors

    For problems like number of islands (diagonals count), add the four diagonal directions:

    int dx[] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[] = {-1, 0, 1, -1, 1, -1, 0, 1};

    Transposing Indices

    To transpose a matrix in place (square matrix), swap mat[i][j] with mat[j][i] for all i < j:

    for (int i = 0; i < n; i++)
    for (int j = i + 1; j < n; j++)
        swap(mat[i][j], mat[j][i]);

    Summary

    • A matrix is a 2D array stored in row-major order in memory. Access row-by-row for best cache performance.
    • The five core traversal patterns are: row-by-row, column-by-column, diagonal, zigzag, and spiral.
    • Spiral traversal uses four boundary variables that shrink inward after each pass.
    • Diagonal cells share i - j (primary) or i + j (anti-diagonal).
    • Direction arrays simplify neighbor enumeration in grid BFS/DFS problems.
