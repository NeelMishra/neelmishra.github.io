    Subarray Patterns

    March 2026 · 22 min read

    This post covers five classic subarray patterns that appear repeatedly in interviews. Each uses a different technique from the overview. For every pattern, we walk through the intuition, a visual step-through, and complete C++ code.

    Pattern 1: Subarray Sum Equals K

    Problem (LeetCode 560): Given an integer array nums and an integer k, return the total number of subarrays whose sum equals k. The array can contain negative numbers.

    Idea

    Maintain a running prefix sum. At each index j, the prefix sum is curSum. A subarray ending at j with sum exactly k means that some earlier prefix had value curSum - k. Instead of scanning backwards to find such prefixes, store all prefix sum frequencies in a hashmap. Each lookup is O(1).

    Initialize the map with {0: 1} to account for subarrays that start from index 0.

    Walkthrough

    Consider nums = [3, 4, -7, 1, 3, 3, 1, -4] and k = 7. The prefix sums are [0, 3, 7, 0, 1, 4, 7, 8, 4]. At index 1 (prefix 7), we look for 7 - 7 = 0 in the map. It is there (the empty prefix), so we found [3, 4]. At index 5 (prefix 7), we look for 0 again. It is there twice (prefix at start and prefix at index 2), giving us [1, 3, 3] and the full subarray [3, 4, -7, 1, 3, 3].

    ▶ Subarray Sum = K Step-Through

    nums = [3, 4, -7, 1, 3, 3, 1, -4], k = 7. Track prefix sums and hashmap lookups.

    Array
    Running Prefix Sum
    HashMap

    C++ Code

    #include <vector>
#include <unordered_map>

int subarraySum(const std::vector<int>& nums, int k) {
    std::unordered_map<long long, int> prefixCount;
    prefixCount[0] = 1;
    long long curSum = 0;
    int count = 0;

    for (int num : nums) {
        curSum += num;
        auto it = prefixCount.find(curSum - k);
        if (it != prefixCount.end())
            count += it->second;
        prefixCount[curSum]++;
    }
    return count;
}

// subarraySum({3,4,-7,1,3,3,1,-4}, 7) = 4
// Subarrays: [3,4], [3,4,-7,1,3,3], [1,3,3], [7] wait...
// Let me verify: prefix = 0,3,7,0,1,4,7,8,4
//   i=0: cur=3, need=-4 no
//   i=1: cur=7, need=0  yes(1) count=1  [3,4]
//   i=2: cur=0, need=-7 no
//   i=3: cur=1, need=-6 no
//   i=4: cur=4, need=-3 no
//   i=5: cur=7, need=0  yes(2) count=3  [3,4,-7,1,3,3] and [1,3,3]
//   i=6: cur=8, need=1  yes(1) count=4  [3,3,1]
//   i=7: cur=4, need=-3 no
// Total = 4

    Time: O(n). Space: O(n).

    Key insight: The hashmap stores how many times each prefix sum has occurred. When a prefix sum repeats, it means the sum of the subarray between those two occurrences is zero. This generalizes to any target sum k.

    Pattern 2: Maximum Length Subarray with Sum at Most K

    Problem: Given an array of positive integers and an integer k, find the length of the longest subarray whose sum is at most k.

    Idea

    Since all values are positive, the sum of any subarray increases as we expand and decreases as we shrink. This monotonicity property makes sliding window the right tool.

    Expand the right end to include elements. When the window sum exceeds k, shrink from the left until the sum is within the limit again. Track the maximum window size seen so far.

    Walkthrough

    Consider nums = [1, 2, 1, 0, 1, 1, 0] and k = 4. The sliding window starts empty. We expand right until sum exceeds 4, then shrink left. The longest valid window has length 5 (elements [1, 0, 1, 1, 0] with sum 3).

    ▶ Maximum Length Subarray with Sum at Most K

    nums = [1, 2, 1, 0, 1, 1, 0], k = 4. Watch the sliding window find the longest valid subarray.

    C++ Code

    #include <vector>
#include <algorithm>

int maxLenSubarrayAtMostK(const std::vector<int>& nums, int k) {
    int n = nums.size();
    int left = 0, maxLen = 0;
    long long curSum = 0;

    for (int right = 0; right < n; ++right) {
        curSum += nums[right];
        while (curSum > k && left <= right) {
            curSum -= nums[left];
            ++left;
        }
        maxLen = std::max(maxLen, right - left + 1);
    }
    return maxLen;
}

// maxLenSubarrayAtMostK({1,2,1,0,1,1,0}, 4) = 5
// Window [1,0,1,1,0] has sum 3 <= 4, length 5.

    Time: O(n). Space: O(1).

    Why this fails with negatives: If the array had negative values, shrinking from the left might increase the sum instead of decreasing it. The monotonicity breaks and the sliding window gives wrong results during shrinking.

    Pattern 3: Subarrays with Exactly K Distinct Elements

    Problem (LeetCode 992): Given an integer array nums and an integer k, return the number of subarrays that contain exactly k distinct integers.

    Idea

    Directly counting subarrays with exactly K distinct elements using a sliding window is hard because when you shrink the window, the number of distinct elements might not change if there are duplicates. The trick is to decompose the problem:

    exactly(K) = atMost(K) - atMost(K - 1)

    The function atMost(K) counts subarrays with at most K distinct elements. This is straightforward with a sliding window and a frequency map: expand right, and if distinct count exceeds K, shrink left until it is at most K again. At each position, the number of valid subarrays ending at right is right - left + 1.

    Walkthrough

    Consider nums = [1, 2, 1, 2, 3] and k = 2. We compute atMost(2) = 12 and atMost(1) = 5. So exactly(2) = 12 - 5 = 7.

    ▶ atMost(K) Sliding Window

    nums = [1, 2, 1, 2, 3], computing atMost(2). At each step, count subarrays ending at right.

    Freq Map

    C++ Code

    #include <vector>
#include <unordered_map>

int atMost(const std::vector<int>& nums, int k) {
    int n = nums.size();
    std::unordered_map<int, int> freq;
    int left = 0, result = 0;

    for (int right = 0; right < n; ++right) {
        freq[nums[right]]++;
        while ((int)freq.size() > k) {
            int lv = nums[left];
            if (--freq[lv] == 0)
                freq.erase(lv);
            ++left;
        }
        result += right - left + 1;
    }
    return result;
}

int subarraysWithKDistinct(const std::vector<int>& nums, int k) {
    return atMost(nums, k) - atMost(nums, k - 1);
}

// subarraysWithKDistinct({1,2,1,2,3}, 2) = 7
// atMost(2) = 12, atMost(1) = 5, exact = 7

    Time: O(n). Space: O(K) for the frequency map.

    Why right - left + 1?

    When the window spans indices [left, right], every subarray that ends at right and starts at any index from left to right is valid (it has at most K distinct elements, since it is a subset of the current window). That gives exactly right - left + 1 new valid subarrays at each step.

    Pattern 4: Product of Array Except Self

    Problem (LeetCode 238): Given an integer array nums, return an array answer where answer[i] is equal to the product of all elements of nums except nums[i]. You must solve it without using division.

    Idea

    This is a prefix/suffix product problem. For each index i, the answer is the product of everything to its left multiplied by the product of everything to its right.

    Build two arrays:

    • leftProduct[i] = product of nums[0] * nums[1] * ... * nums[i-1]
    • rightProduct[i] = product of nums[i+1] * nums[i+2] * ... * nums[n-1]

    Then answer[i] = leftProduct[i] * rightProduct[i].

    To save space, compute the left pass into the result array, then multiply in the right pass directly using a running product variable.

    Walkthrough

    Consider nums = [1, 2, 3, 4]. The left products are [1, 1, 2, 6] and the right products are [24, 12, 4, 1]. Multiplying element-wise gives [24, 12, 8, 6].

    ▶ Product of Array Except Self

    nums = [1, 2, 3, 4]. First compute left products, then right products, then combine.

    C++ Code

    #include <vector>

std::vector<int> productExceptSelf(const std::vector<int>& nums) {
    int n = nums.size();
    std::vector<int> answer(n, 1);

    // Left pass: answer[i] = product of nums[0..i-1]
    int leftProd = 1;
    for (int i = 0; i < n; ++i) {
        answer[i] = leftProd;
        leftProd *= nums[i];
    }

    // Right pass: multiply in product of nums[i+1..n-1]
    int rightProd = 1;
    for (int i = n - 1; i >= 0; --i) {
        answer[i] *= rightProd;
        rightProd *= nums[i];
    }

    return answer;
}

// productExceptSelf({1,2,3,4}) = {24,12,8,6}

    Time: O(n). Space: O(1) extra (the output array is not counted).

    No division allowed. A tempting approach is to compute the total product and divide by each element. But this fails when any element is zero (division by zero) and the problem explicitly forbids division.

    Pattern 5: Subarrays with Given XOR

    Problem: Given an integer array nums and an integer target, count the number of subarrays whose XOR equals target.

    Idea

    XOR has the same prefix property as addition. Define a prefix XOR array where pre[i] = nums[0] ^ nums[1] ^ ... ^ nums[i-1]. Then the XOR of subarray nums[l..r] is pre[r+1] ^ pre[l].

    We want pre[r+1] ^ pre[l] = target, which means pre[l] = pre[r+1] ^ target (because XOR is its own inverse). So the algorithm is identical to the sum = K pattern, just replacing addition with XOR and subtraction with XOR.

    Maintain a running prefix XOR and a hashmap of prefix XOR frequencies. At each step, look up curXor ^ target in the map.

    Walkthrough

    Consider nums = [4, 2, 2, 6, 4] and target = 6. The prefix XORs are [0, 4, 6, 4, 2, 6]. At index 1 (prefix XOR = 6), we look for 6 ^ 6 = 0. It is in the map, so we found [4, 2]. At index 3 (prefix XOR = 2), we look for 2 ^ 6 = 4. It is in the map twice (prefix at index 0 and index 2), giving two subarrays. The animation below shows each step.

    ▶ Subarray XOR = Target Step-Through

    nums = [4, 2, 2, 6, 4], target = 6. Track prefix XOR and hashmap.

    Array
    Running Prefix XOR
    HashMap {prefXor: count}

    C++ Code

    #include <vector>
#include <unordered_map>

int subarrayXor(const std::vector<int>& nums, int target) {
    std::unordered_map<int, int> prefixCount;
    prefixCount[0] = 1;
    int curXor = 0, count = 0;

    for (int num : nums) {
        curXor ^= num;
        int need = curXor ^ target;
        auto it = prefixCount.find(need);
        if (it != prefixCount.end())
            count += it->second;
        prefixCount[curXor]++;
    }
    return count;
}

// subarrayXor({4,2,2,6,4}, 6) = 4
// Prefix XORs: 0,4,6,4,2,6
//   i=0: cur=4, need=4^6=2 no
//   i=1: cur=6, need=6^6=0 yes(1) count=1  [4,2]
//   i=2: cur=4, need=4^6=2 no
//   i=3: cur=2, need=2^6=4 yes(2) count=3  [2,6] and [4,2,2,6]
//   i=4: cur=6, need=6^6=0 yes(1) count=4  [4,2,2,6,4]? No.
//     Actually pre[0]=0, so subarray is nums[0..4], XOR=6. Yes.
// Total = 4

    Time: O(n). Space: O(n).

    Why XOR Works Like Sum

    Both XOR and addition satisfy the prefix cancellation property. For sums, sum(l..r) = pre[r+1] - pre[l]. For XOR, xor(l..r) = pre[r+1] ^ pre[l]. The key property is that XOR is its own inverse: a ^ a = 0. So if pre[r+1] ^ pre[l] = target, then pre[l] = pre[r+1] ^ target. This is exactly analogous to pre[l] = pre[r+1] - target in the sum case.

    Key Takeaways

    • Prefix sum + hashmap is the universal pattern for "count subarrays with property X" where X can be checked via a prefix function.
    • If the array has only positive values and the constraint is a sum bound, prefer sliding window for its O(1) space and simplicity.
    • The "exactly K = atMost K minus atMost (K-1)" decomposition is essential for distinct-element counting problems. It converts a hard constraint into two easy ones.
    • Product except self is a prefix/suffix pattern, not a subarray problem per se, but it uses the exact same left-right decomposition idea.
    • XOR behaves identically to addition for prefix purposes. Any technique that works with prefix sums can be adapted to prefix XOR.
