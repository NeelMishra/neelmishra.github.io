    Subarray Problems: Overview

    March 2026 · 18 min read

    Subarray vs Subsequence vs Subset

    Before working through subarray problems you need to be clear about three terms that sound similar but have very different meanings. Getting them confused leads to choosing wrong algorithms.

    Subarray (also called a contiguous subarray): a slice of consecutive elements from the array. If the array is [a, b, c, d, e], then [b, c, d] is a subarray but [a, c, e] is not. A subarray is fully determined by its start and end index.

    Subsequence: elements chosen in order but not necessarily adjacent. From [a, b, c, d, e], both [a, c, e] and [b, d] are valid subsequences. You must preserve relative order, but you can skip elements.

    Subset: any selection of elements with no ordering requirement at all. {d, a, c} is a valid subset of {a, b, c, d, e}.

    The contiguity constraint is what makes subarray problems unique. Because elements must be adjacent, techniques like sliding window, prefix sums, and two pointers become available. Subsequence problems typically need dynamic programming, and subset problems are usually exponential.

    Counting Subarrays

    An array of length n has exactly n(n+1)/2 non-empty subarrays. This comes from choosing a start index (n choices) and an end index (at least start, at most n-1). The total is 1 + 2 + ... + n = n(n+1)/2.

    For n = 5, that gives 15 subarrays. For n = 1000, it is 500,500. This quadratic count explains why brute force enumeration of all subarrays is O(n2) and is sometimes acceptable for small inputs, but you need O(n) or O(n log n) algorithms when n reaches 105 or beyond.

    A useful identity: the number of subarrays that include element at index i is (i + 1) * (n - i). This counts all (start, end) pairs where start is between 0 and i, and end is between i and n-1. This identity appears surprisingly often when computing contribution of each element to some aggregate.

    ▶ Enumerate All Subarrays

    Step through all 10 subarrays of [a, b, c, d], grouped by starting index.

    // Enumerate all subarrays of arr (for understanding, not for production)
void printAllSubarrays(const std::vector<int>& arr) {
    int n = arr.size();
    for (int start = 0; start < n; ++start) {
        for (int end = start; end < n; ++end) {
            // arr[start..end] is one subarray
            for (int k = start; k <= end; ++k)
                std::cout << arr[k] << " ";
            std::cout << "\n";
        }
    }
    // Total printed: n*(n+1)/2 subarrays
}

    Key Techniques for Subarray Problems

    Most subarray problems can be solved using one of four core techniques. Each has specific conditions under which it applies. Understanding when to use which technique is the real skill.

    Prefix Sums

    The prefix sum array lets you compute the sum of any subarray in O(1) after O(n) preprocessing. Given array arr, build pre where pre[0] = 0 and pre[i] = pre[i-1] + arr[i-1]. Then sum(l, r) = pre[r+1] - pre[l].

    Prefix sums are the foundation of nearly every subarray sum problem. They transform the question "what is the sum of arr[l..r]?" into a subtraction of two known values.

    #include <vector>

std::vector<long long> buildPrefix(const std::vector<int>& arr) {
    int n = arr.size();
    std::vector<long long> pre(n + 1, 0);
    for (int i = 0; i < n; ++i)
        pre[i + 1] = pre[i] + arr[i];
    return pre;
}

// Sum of arr[l..r] inclusive
long long rangeSum(const std::vector<long long>& pre, int l, int r) {
    return pre[r + 1] - pre[l];
}

    arr = [2, -1, 3, 5, -2, 4]. Watch the prefix array fill in one element at a time.

    When to use: Any problem asking about subarray sums. Often combined with a hashmap to find pairs of prefix values with a specific difference.

    Sliding Window

    Maintain two pointers (left and right) that define a window. Expand right to include more elements; shrink left when some condition is violated. Each element enters and leaves the window at most once, giving O(n) total work.

    The sliding window requires a monotonicity property: if a window is valid, then a smaller window (obtained by moving right left or left right) should also satisfy some useful condition. For sum constraints with positive values, expanding always increases the sum and shrinking always decreases it, which is exactly the monotonicity you need.

    // Shortest subarray with sum >= target (positive values only)
int minSubArrayLen(int target, const std::vector<int>& nums) {
    int n = nums.size(), left = 0, minLen = INT_MAX;
    long long curSum = 0;
    for (int right = 0; right < n; ++right) {
        curSum += nums[right];
        while (curSum >= target) {
            minLen = std::min(minLen, right - left + 1);
            curSum -= nums[left++];
        }
    }
    return minLen == INT_MAX ? 0 : minLen;
}

    When to use: Longest/shortest subarray with a sum or count constraint, when values are positive (or the constraint has the right monotonicity). Also applies to string problems with character frequency constraints. See the Sliding Window series for full coverage.

    Kadane's Algorithm

    Kadane's algorithm finds the contiguous subarray with the maximum (or minimum) sum in O(n) time and O(1) space. At each position, decide: extend the previous subarray or start a new one from the current element. You start fresh whenever the accumulated sum is negative.

    int maxSubArray(const std::vector<int>& nums) {
    int curMax = nums[0], maxSoFar = nums[0];
    for (int i = 1; i < (int)nums.size(); ++i) {
        curMax = std::max(nums[i], curMax + nums[i]);
        maxSoFar = std::max(maxSoFar, curMax);
    }
    return maxSoFar;
}

    ▶ Kadane's Algorithm

    nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]. Track curMax and maxSoFar at each step.

    When to use: Maximum subarray sum, minimum subarray sum, maximum product subarray (with modification). The key signal is "find the best contiguous segment" without a fixed target. See the Kadane's Algorithm post.

    Prefix Sum + Hashmap

    This combines prefix sums with a hashmap to solve "count subarrays with sum = K" in O(n). At each index, the running prefix sum is curSum. A subarray ending here with sum K requires a previous prefix equal to curSum - K. Store prefix sum frequencies in a map; each lookup is O(1).

    #include <unordered_map>

int subarraySum(const std::vector<int>& nums, int k) {
    std::unordered_map<long long, int> prefixCount;
    prefixCount[0] = 1;           // empty prefix
    long long curSum = 0;
    int count = 0;
    for (int num : nums) {
        curSum += num;
        auto it = prefixCount.find(curSum - k);
        if (it != prefixCount.end())
            count += it->second;
        prefixCount[curSum]++;
    }
    return count;
}

    ▶ Prefix Sum + Hashmap (Sum = K)

    nums = [1, 2, 3, -2, 5], k = 5. Watch the prefix sums and hashmap build step by step.

    HashMap {prefixSum: count}
    Critical detail: Always initialize the hashmap with {0: 1}. Without it, subarrays starting at index 0 that sum to K will be missed.

    When to use: Count or find subarrays with a specific sum, XOR, or remainder property. The hashmap stores prefix values; the lookup finds complements. Works even when array contains negative numbers (where sliding window fails).

    Decision Framework

    When you encounter a subarray problem, run through these questions in order:

    Detailed Breakdown

    Step 1: Is it about the best (max/min) subarray with no target value? If you want the maximum sum contiguous subarray, or the minimum sum, or the maximum product, you are in Kadane's territory. The key is that there is no external target; you optimize over all possible subarrays.

    Step 2: Are all values non-negative? If yes and the problem involves a sum constraint (at least K, at most K, equal to K), sliding window is usually the cleanest approach. Expanding the window always increases the sum; shrinking always decreases it. This monotonicity is what makes the two-pointer technique valid.

    Step 3: Can values be negative? Then sliding window breaks because adding an element might decrease the sum. Use prefix sums instead. If you need to count subarrays with a specific sum, pair prefix sums with a hashmap. If you need the shortest subarray with sum at least K, use a monotone deque on prefix sums.

    Step 4: Does the problem involve distinct elements? Problems like "number of subarrays with exactly K distinct elements" combine sliding window with a frequency map. You often need the "exactly K = atMost K minus atMost (K-1)" trick.

    Step 5: Is the operation XOR instead of sum? XOR has the same prefix property as addition: xor(l, r) = prefix[r+1] ^ prefix[l]. So the prefix + hashmap approach works identically. Just replace addition with XOR.

    Common Pitfalls

    • Forgetting the empty prefix. When using prefix sum + hashmap, always initialize with {0: 1}. Without it, subarrays that start at index 0 with the target sum get missed.
    • Applying sliding window with negative values. If the array can have negatives, shrinking the window does not guarantee the sum decreases. The window approach silently gives wrong answers.
    • Off-by-one in prefix arrays. Using a prefix array of size n+1 (with pre[0] = 0) eliminates the special case at index 0. Prefer this convention.
    • Integer overflow. Prefix sums can grow large. Use long long in C++ when values are up to 104 and n is up to 105.
    • Confusing "at most K" with "exactly K". For problems asking exactly K distinct elements, use the identity: exactly(K) = atMost(K) - atMost(K-1). Never try to maintain an exact count directly in a sliding window.

    Key Takeaways

    • A subarray is a contiguous slice. This constraint enables efficient algorithms that do not work for subsequences or subsets.
    • An array of length n has n(n+1)/2 subarrays. This quadratic count means brute force is O(n2) at best.
    • Prefix sums are the workhorse. They reduce range sum queries to O(1) and, combined with a hashmap, solve "count subarrays with sum K" in O(n).
    • Sliding window works when monotonicity holds (typically positive values). It gives O(n) time with O(1) space.
    • Kadane's algorithm is the go-to for "best contiguous segment" with no external target.
    • Use the decision framework: identify whether you need a target, whether values can be negative, and whether the constraint is on sums or distinct elements.
