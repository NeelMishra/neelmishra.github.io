    Prefix Sum — Patterns

    March 2026 · 12 min read

    Pattern 1: Subarray Sum Equals K

    Problem: Given an array and an integer k, find the total number of contiguous subarrays whose sum equals k. (LeetCode 560)

    Brute force would check all O(n²) pairs of (start, end) — far too slow for large arrays.

    Key Insight: If the prefix sum at index j is pre[j], and the prefix sum at some earlier index i was pre[i], then the subarray arr[i+1..j] has sum pre[j] - pre[i]. We want this to equal k, meaning we need pre[i] = pre[j] - k.

    So at each index, we ask: "how many times have I seen a prefix sum equal to curSum - k?" A hashmap tracks how many times each prefix sum has occurred. We initialize it with {0: 1} to handle the case where a subarray starting from index 0 itself sums to k.

    Why the hashmap works: Instead of checking all previous indices, the hashmap gives us the count in O(1). Each element is visited once, so the total time is O(n).

    function subarraySum(arr, k):
    count = 0, curSum = 0 map = {0: 1} // empty prefix for num in arr: curSum += num if (curSum - k) in map:   count += map[curSum - k] map[curSum] = map.get(curSum, 0) + 1 return count

    ▶ Subarray Sum = K Animation

    arr = [1, 2, 3, -2, 5], k = 5. Watch prefix sums and the hashmap track matches.

    int subarraySum(std::vector<int>& nums, int k) {
    std::unordered_map<int, int> prefixCount;
    prefixCount[0] = 1;
    int count = 0, curSum = 0;
    for (int num : nums) {
        curSum += num;
        if (prefixCount.count(curSum - k))
            count += prefixCount[curSum - k];
        prefixCount[curSum]++;
    }
    return count;
}

    Complexity: O(n) time, O(n) space. This is one of the most frequently asked interview patterns — it shows up in problems about subarray sums, equal partitions, and binary array splits.

    Pattern 2: Equilibrium Index

    Problem: Find an index where the sum of elements to its left equals the sum to its right. (Also called Pivot Index — LeetCode 724)

    A naive approach computes left sum and right sum separately for each index, taking O(n²). But we can do much better.

    Key Insight: If we know the totalSum of the array starting, then at any index i:

    • rightSum = totalSum - leftSum - arr[i]
    • We want leftSum == rightSum, i.e., leftSum == totalSum - leftSum - arr[i]

    We sweep left to right, maintaining a running leftSum. At each index we check the condition, then add arr[i] to leftSum before moving on. This single pass gives us O(n) time with O(1) extra space.

    ▶ Equilibrium Index Animation

    arr = [1, 3, 5, 2, 2]. Find where left sum = right sum.

    int equilibriumIndex(std::vector<int>& arr) {
    int total = 0;
    for (int x : arr) total += x;
    int leftSum = 0;
    for (int i = 0; i < arr.size(); ++i) {
        int rightSum = total - leftSum - arr[i];
        if (leftSum == rightSum) return i;
        leftSum += arr[i];
    }
    return -1;
}

    Complexity: O(n) time, O(1) space.

    Pattern 3: Product of Array Except Self

    Problem: Return an array where each element is the product of all elements except itself — without using division. (LeetCode 238)

    With division, it's trivial: compute total product, divide by each element. But the constraint says no division (and there could be zeros, which make division undefined).

    Key Insight: For index i, the answer is (product of everything to the left of i) × (product of everything to the right of i). This is a prefix/suffix product pattern:

    • Left pass: Build array where left[i] = product of arr[0..i-1] (everything before i).
    • Right pass: Sweep from right, multiplying each left[i] by the running right product (everything after i).

    Two passes, no division, O(1) extra space (the output array doesn't count). This technique generalizes: anytime you need "combine everything except the current element", think left-pass + right-pass.

    ▶ Product Except Self Animation

    arr = [1, 2, 3, 4]. Build left products, then right products, then combine.

    std::vector<int> productExceptSelf(std::vector<int>& nums) {
    int n = nums.size();
    std::vector<int> result(n, 1);
    // Left pass: result[i] = product of all elements to the left
    int leftProd = 1;
    for (int i = 0; i < n; ++i) {
        result[i] = leftProd;
        leftProd *= nums[i];
    }
    // Right pass: multiply by product of all elements to the right
    int rightProd = 1;
    for (int i = n - 1; i >= 0; --i) {
        result[i] *= rightProd;
        rightProd *= nums[i];
    }
    return result;
}

    Complexity: O(n) time, O(1) extra space (output array doesn't count).

    Pattern 4: Running Sum & Difference Arrays

    The running sum is simply the prefix sum by another name. But the truly powerful idea here is the difference array — the inverse of the prefix sum.

    Motivating problem: You have an array of zeros and receive many queries of the form "add +val to every element from index l to r". A naive approach takes O(n) per query (looping through the range). With q queries, that's O(n·q). Can we do better?

    Difference array trick: Instead of updating every element in the range, mark just the boundaries:

    • diff[l] += val — the increase starts here
    • diff[r+1] -= val — the increase stops here

    Each update is now O(1). After all updates, reconstruct the actual array by computing the prefix sum of diff[]. The prefix sum "spreads" each boundary mark across the correct range. Total cost: O(1) per update + O(n) reconstruction = O(q + n).

    This is used everywhere: booking calendars ("add 1 to days 3-7"), image processing, and competitive programming range-update problems.

    // Range update [l, r] by +val using difference array:
    diff[l] += val
    diff[r+1] -= val // if r+1 < n

    // Reconstruct actual array:
    arr[0] = diff[0]
    arr[i] = arr[i-1] + diff[i]

    ▶ Difference Array Animation

    Apply range update [1,4] += 3, then [2,5] += 2, then reconstruct.

    // Difference array range update
void rangeAdd(std::vector<int>& diff, int l, int r, int val) {
    diff[l] += val;
    if (r + 1 < diff.size()) diff[r + 1] -= val;
}

// Reconstruct original via prefix sum
std::vector<int> reconstruct(std::vector<int>& diff) {
    std::vector<int> arr(diff.size());
    arr[0] = diff[0];
    for (int i = 1; i < diff.size(); ++i)
        arr[i] = arr[i-1] + diff[i];
    return arr;
}

    Complexity: O(1) per range update, O(n) to reconstruct.
