    Prefix Sum — Fundamentals

    March 2026 · 10 min read

    What Is a Prefix Sum?

    A prefix sum (also called a cumulative sum) is a derived array where each element at index i stores the sum of all elements from index 0 to i of the original array.

    Imagine you're a cashier tallying receipts. Instead of re-adding all receipts from scratch every time someone asks "what's the total from receipt #3 to #7?", you maintain a running total. That running total at each point is the prefix sum.

    Given an array arr of length n, the prefix sum array pre is defined as:

    pre[0] = arr[0]
    pre[i] = pre[i-1] + arr[i] // for i ≥ 1

    Each entry pre[i] stores arr[0] + arr[1] + ... + arr[i]. This simple preprocessing step unlocks O(1) range sum queries, turning what would be O(n) per query into constant time after an O(n) build.

    Let's trace through an example. Given arr = [2, 4, 1, 3, 5, 2, 7]:

    • pre[0] = 2 (just the first element)
    • pre[1] = 2 + 4 = 6
    • pre[2] = 6 + 1 = 7
    • pre[3] = 7 + 3 = 10
    • pre[4] = 10 + 5 = 15
    • pre[5] = 15 + 2 = 17
    • pre[6] = 17 + 7 = 24 (the total of the entire array)

    Building the Prefix Array

    The build is a single left-to-right pass. At each index, you only need the previous prefix value and the current element — no need to look back at the entire array. This is what makes it O(n).

    Range Sum Query in O(1)

    This is where the magic happens. Once the prefix array is built, the sum of any subarray arr[l..r] is:

    rangeSum(l, r) = pre[r] - (l > 0 ? pre[l-1] : 0)

    Why does this work? pre[r] contains the sum of arr[0..r]. But we only want arr[l..r], not the part before l. So we subtract pre[l-1], which is the sum of arr[0..l-1] — the part we don't want. What remains is exactly arr[l] + arr[l+1] + ... + arr[r].

    Think of it like a measuring tape: to find the length between two marks, you subtract the smaller mark from the larger one. No need to measure each segment individually.

    Edge case: when l = 0, there's nothing to subtract — the answer is simply pre[r].

    C++ Implementation

    The implementation is straightforward — two functions, each just a few lines. Notice that buildPrefix is a single pass (O(n)), and rangeSum does one subtraction (O(1)):

    #include <vector>

// Build prefix sum array
std::vector<int> buildPrefix(const std::vector<int>& arr) {
    int n = arr.size();
    std::vector<int> pre(n);
    pre[0] = arr[0];
    for (int i = 1; i < n; ++i)
        pre[i] = pre[i-1] + arr[i];
    return pre;
}

// Range sum query in O(1)
int rangeSum(const std::vector<int>& pre, int l, int r) {
    return pre[r] - (l > 0 ? pre[l-1] : 0);
}

// Example
int main() {
    std::vector<int> arr = {2, 4, 1, 3, 5, 2, 7};
    auto pre = buildPrefix(arr);
    // rangeSum(pre, 1, 4) = pre[4] - pre[0] = 15 - 2 = 13
    // rangeSum(pre, 0, 6) = pre[6] = 24
}

    Complexity

    The fundamental tradeoff: spend O(n) time once to preprocess, then enjoy O(1) per query forever. This is a classic time-space tradeoff — we use O(n) extra memory to make queries instantaneous.

    OperationTimeSpace
    Build prefix arrayO(n)O(n)
    Range sum queryO(1)—
    Brute force range sumO(n) per queryO(1)

    If you have q queries, prefix sum reduces total time from O(n·q) to O(n + q).

    When to use prefix sums: Whenever you need repeated range sum (or range XOR, range product) queries on a static array. If the array changes, consider a Segment Tree instead.

    Key Takeaways

    • Prefix sum is one of the most fundamental array preprocessing techniques.
    • Build once in O(n), query any range in O(1).
    • The formula: sum(l, r) = pre[r] - pre[l-1] (handle l=0 edge case).
    • Forms the building block for many advanced patterns (next post).
