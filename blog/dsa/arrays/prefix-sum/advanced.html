    2D Prefix Sum & Advanced

    March 2026 · 14 min read

    2D Prefix Sum

    The 1D prefix sum lets us query any subarray sum in O(1). The 2D prefix sum extends this to matrices: given an m × n grid, precompute a prefix matrix so any sub-rectangle sum can be answered in O(1).

    The idea: pre[i][j] stores the sum of all elements in the rectangle from (0,0) to (i,j). To compute it, we use the same "add what's above + what's to the left, subtract the double-counted corner" logic:

    // Build 2D prefix sum:
    pre[i][j] = grid[i][j]
    + pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1]

    // Query sum of sub-rectangle (r1,c1) to (r2,c2):
    sum = pre[r2][c2]
    - pre[r1-1][c2] - pre[r2][c1-1] + pre[r1-1][c1-1]

    The formula uses inclusion-exclusion, a principle from set theory:

    • Build: To get the sum of the rectangle ending at (i,j), add the current cell plus what's above (pre[i-1][j]) plus what's to the left (pre[i][j-1]), but that double-counts the top-left corner (pre[i-1][j-1]), so subtract it once.
    • Query: Start with the big rectangle pre[r2][c2]. Subtract the strip above (pre[r1-1][c2]) and the strip to the left (pre[r2][c1-1]). But now you've subtracted the top-left corner twice, so add it back (pre[r1-1][c1-1]).

    Both operations are O(1), just four lookups and some arithmetic. The build is O(m×n), same as reading the entire matrix once.

    // Build 2D prefix sum
std::vector<std::vector<int>> build2D(
    const std::vector<std::vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    std::vector<std::vector<int>> pre(m, std::vector<int>(n, 0));
    for (int i = 0; i < m; ++i)
        for (int j = 0; j < n; ++j) {
            pre[i][j] = grid[i][j];
            if (i > 0) pre[i][j] += pre[i-1][j];
            if (j > 0) pre[i][j] += pre[i][j-1];
            if (i > 0 && j > 0) pre[i][j] -= pre[i-1][j-1];
        }
    return pre;
}

// Query sub-rectangle sum in O(1)
int query2D(const std::vector<std::vector<int>>& pre,
            int r1, int c1, int r2, int c2) {
    int sum = pre[r2][c2];
    if (r1 > 0) sum -= pre[r1-1][c2];
    if (c1 > 0) sum -= pre[r2][c1-1];
    if (r1 > 0 && c1 > 0) sum += pre[r1-1][c1-1];
    return sum;
}

    XOR Prefix Sum

    The prefix sum trick works for any associative operation that has an inverse. Addition has subtraction. And XOR? XOR is its own inverse: a ^ a = 0, and a ^ 0 = a. This means we can build a "prefix XOR" and query any range XOR in O(1).

    Why this matters: XOR prefix sums are used in problems involving parity, finding missing numbers, toggling bits in ranges, and cryptographic applications. The principle is identical to the additive prefix sum, just replace + with ^ and - with ^ (since XOR undoes itself).

    xorPre[i] = arr[0] ^ arr[1] ^ ... ^ arr[i]

    rangeXOR(l, r) = xorPre[r] ^ (l > 0 ? xorPre[l-1] : 0)

    ▶ XOR Prefix Animation

    arr = [3, 5, 2, 6, 1, 4]. Build XOR prefix then query ranges.

    // XOR prefix sum
std::vector<int> buildXORPrefix(const std::vector<int>& arr) {
    int n = arr.size();
    std::vector<int> xp(n);
    xp[0] = arr[0];
    for (int i = 1; i < n; ++i)
        xp[i] = xp[i-1] ^ arr[i];
    return xp;
}

int rangeXOR(const std::vector<int>& xp, int l, int r) {
    return xp[r] ^ (l > 0 ? xp[l-1] : 0);
}

    Subarrays Divisible by K

    Problem: Count subarrays whose sum is divisible by k. (LeetCode 974)

    This is a beautiful application of modular arithmetic combined with prefix sums.

    Key Insight: A subarray arr[i+1..j] has sum pre[j] - pre[i]. For this to be divisible by k, we need (pre[j] - pre[i]) % k == 0, which means pre[j] % k == pre[i] % k. In other words: if two prefix sums have the same remainder mod k, the subarray between them is divisible by k.

    So we maintain a frequency array of size k that counts how many prefix sums have each remainder. At each step:

    • Compute the current prefix sum's remainder: mod = ((prefixSum % k) + k) % k (the extra +k handles negative numbers in C++/Java).
    • Add the count of previous prefix sums with the same remainder, that's how many valid subarrays end here.
    • Increment the frequency for this remainder.

    We initialize modCount[0] = 1 to account for subarrays starting from index 0 whose total sum is directly divisible by k.

    ▶ Subarrays Divisible by K Animation

    arr = [4, 5, 0, -2, -3, 1], k = 5. Track prefix sum remainders.

    int subarraysDivByK(std::vector<int>& nums, int k) {
    std::vector<int> modCount(k, 0);
    modCount[0] = 1;
    int count = 0, prefixSum = 0;
    for (int num : nums) {
        prefixSum += num;
        int mod = ((prefixSum % k) + k) % k; // handle negatives
        count += modCount[mod];
        modCount[mod]++;
    }
    return count;
}

    Complexity: O(n) time, O(k) space.

    Key Takeaways

    • 2D prefix sum extends the 1D idea to matrices using inclusion-exclusion.
    • XOR prefix sums work because XOR is self-inverse.
    • Remainder-based prefix sums solve divisibility problems elegantly.
    • Prefix sums are the simplest range query structure. For dynamic data, upgrade to segment trees or BITs.
