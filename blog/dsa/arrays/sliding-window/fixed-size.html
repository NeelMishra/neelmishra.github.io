    Fixed-Size Sliding Window

    March 2026 · 22 min read

    Introduction

    A fixed-size sliding window maintains a window of exactly $k$ elements that moves one position at a time across the array. At each step, one element enters the window on the right and one element leaves on the left. The window size never changes.

    This is the simpler of the two sliding window variants. The pattern is mechanical: build the initial window, then slide it forward, updating your running aggregate with each move.

    Fixed-size window invariant: At every step after initialization, the window contains exactly $k$ consecutive elements: arr[i-k+1..i].

    Problem: Maximum Sum Subarray of Size K

    Given an integer array and a number $k$, find the maximum sum among all contiguous subarrays of size $k$.

    Example: arr = [4, 2, 1, 7, 8, 1, 2, 8, 1, 0], k = 3. The answer is 16 (the subarray [7, 8, 1] at indices 3 through 5).

    Step-by-Step Walkthrough

    Build the initial window

    Sum the first $k = 3$ elements: 4 + 2 + 1 = 7. Set maxSum = 7.

    Slide to position 1

    Remove arr[0] = 4, add arr[3] = 7. New sum: 7 - 4 + 7 = 10. Max: 10.

    Slide to position 2

    Remove arr[1] = 2, add arr[4] = 8. New sum: 10 - 2 + 8 = 16. Max: 16.

    Continue sliding

    Each further window produces a sum less than 16, so the answer stays 16.

    Code

    int maxSumSubarray(vector<int>& arr, int k) {
    int n = arr.size();
    int windowSum = 0;

    // Build initial window
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];
    }
    int maxSum = windowSum;

    // Slide the window
    for (int i = k; i < n; i++) {
        windowSum += arr[i];       // new element enters
        windowSum -= arr[i - k];   // old element leaves
        maxSum = max(maxSum, windowSum);
    }
    return maxSum;
}
// Time: O(n), Space: O(1)

    Problem: Averages of Subarrays of Size K

    Given an array of $n$ numbers and an integer $k$, compute the average of every contiguous subarray of size $k$.

    Example: arr = [1, 3, 2, 6, -1, 4, 1, 8, 2], k = 5.

    Expected output: [2.2, 2.8, 2.4, 3.6, 2.8].

    Code

    vector<double> subarrayAverages(vector<int>& arr, int k) {
    int n = arr.size();
    vector<double> result;

    double windowSum = 0;
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];
    }
    result.push_back(windowSum / k);

    for (int i = k; i < n; i++) {
        windowSum += arr[i] - arr[i - k];
        result.push_back(windowSum / k);
    }
    return result;
}
// Time: O(n), Space: O(n-k+1) for output

    Problem: Maximum of Each Window (Naive)

    Given an array and window size $k$, find the maximum element in each window of size $k$.

    The naive approach scans each window for its maximum, giving $O(n \times k)$ time. We present the naive version here; the optimized deque-based $O(n)$ solution is covered in the Competition Patterns post.

    vector<int> maxInWindow_naive(vector<int>& arr, int k) {
    int n = arr.size();
    vector<int> result;
    for (int i = 0; i <= n - k; i++) {
        int windowMax = arr[i];
        for (int j = i + 1; j < i + k; j++) {
            windowMax = max(windowMax, arr[j]);
        }
        result.push_back(windowMax);
    }
    return result;
}
// Time: O(n*k), Space: O(n-k+1)
    Why naive here? This version illustrates the limitation. With a plain fixed-size window, we cannot maintain the maximum element in $O(1)$ per step because removing the maximum requires knowing the next largest. The deque solution (see Part 4) solves this elegantly.

    Problem: Count Anagram Occurrences

    Given a string s and a pattern p, count how many substrings of s of length |p| are anagrams of p.

    Example: s = "cbaebabacd", p = "abc". Answer: 2 (substrings "cba" at index 0 and "bac" at index 6).

    Approach

    Use a fixed-size window of size |p|. Maintain a frequency count of characters in the window and compare it to the frequency count of p. Instead of comparing all 26 entries each time, track how many characters have matching frequencies (a counter called matches).

    int countAnagrams(string s, string p) {
    int n = s.size(), m = p.size();
    if (m > n) return 0;

    vector<int> pFreq(26, 0), wFreq(26, 0);
    for (char c : p) pFreq[c - 'a']++;

    int matches = 0, count = 0;

    // Initialize: count matching frequencies
    for (int i = 0; i < 26; i++) {
        if (pFreq[i] == 0) matches++;  // both zero = match
    }

    for (int i = 0; i < n; i++) {
        // Add character entering window
        int idx = s[i] - 'a';
        wFreq[idx]++;
        if (wFreq[idx] == pFreq[idx]) matches++;
        else if (wFreq[idx] == pFreq[idx] + 1) matches--;

        // Remove character leaving window
        if (i >= m) {
            int ridx = s[i - m] - 'a';
            wFreq[ridx]--;
            if (wFreq[ridx] == pFreq[ridx]) matches++;
            else if (wFreq[ridx] == pFreq[ridx] - 1) matches--;
        }

        if (matches == 26) count++;
    }
    return count;
}
// Time: O(n), Space: O(1) (fixed 26-char alphabet)

    Edge Cases

    • k > n: No valid window exists. Return empty or an appropriate sentinel value.
    • k = 1: Each element is its own window. The answer is simply the max/min/average of the array itself.
    • k = n: Only one window (the entire array). Return the aggregate of all elements.
    • All elements equal: Every window produces the same result. The algorithm still works correctly.
    • Negative numbers: The sliding window technique handles negatives correctly for sum-based problems. The max sum might be negative if all elements are negative.

    Common Mistakes

    • Off-by-one in loop bounds: The sliding loop runs from i = k to i < n, not i <= n.
    • Forgetting the initial window: The initial window must be built separately before the sliding loop starts.
    • Using the wrong removal index: When the right edge is at index i, the element leaving is at index i - k, not i - k + 1.
    • Integer overflow: For large arrays with large values, use long long instead of int for the sum.

    Key Takeaways

    • Fixed-size windows maintain exactly $k$ elements. The window slides one position per step.
    • Build the initial window first, then slide by removing one element and adding another.
    • The technique works for sums, averages, frequency counts, and any aggregate that can be updated incrementally.
    • For aggregates that are not incrementally updatable (like maximum), you need additional data structures (covered in the patterns post).
    • Time complexity is always $O(n)$ for a single pass with $O(1)$ updates per step.
