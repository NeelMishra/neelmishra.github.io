    Sliding Window: Competition Patterns

    March 2026 · 28 min read

    Introduction

    This post covers four advanced sliding window patterns that appear frequently in coding competitions and interviews. Each pattern builds on the basics from the previous posts but adds a twist that requires careful thinking.

    1. At Most K Distinct: A variable-size window with a frequency constraint.
    2. Counting Subarrays: Using the "at most" trick to count exact matches.
    3. Sliding Window Maximum: Combining a fixed-size window with a monotonic deque.
    4. Minimum Window Sort: Identifying the smallest subarray to sort to make the whole array sorted.

    Pattern 1: Longest Substring with At Most K Distinct Characters

    Given a string s and an integer k, find the length of the longest substring that contains at most k distinct characters.

    Example: s = "eceba", k = 2. The answer is 3 ("ece").

    Approach

    Use a hash map to track character frequencies in the window. Expand the window with the right pointer. When the number of distinct characters exceeds k, shrink from the left until the count drops back to k.

    Expand

    Add s[right] to the frequency map. If this is a new character, increment the distinct count.

    Shrink if needed

    While distinct count exceeds k, decrement the frequency of s[left]. If a frequency drops to zero, remove it from the map (distinct count decreases). Advance left.

    Update answer

    The current window [left..right] has at most k distinct characters. Update the maximum length.

    String: "eceba", k = 2. Step through to see the window expand and shrink as the distinct character count changes.

    Code

    int lengthOfLongestSubstringKDistinct(string s, int k) {
    int n = s.size();
    unordered_map<char, int> freq;
    int left = 0, maxLen = 0;

    for (int right = 0; right < n; right++) {
        freq[s[right]]++;

        while ((int)freq.size() > k) {
            freq[s[left]]--;
            if (freq[s[left]] == 0) freq.erase(s[left]);
            left++;
        }
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Time: O(n), Space: O(k)

    Pattern 2: Count Subarrays with Exactly K Distinct

    Given an array and an integer k, count the number of subarrays with exactly k distinct elements.

    Example: arr = [1, 2, 1, 2, 3], k = 2. The answer is 7.

    The "At Most" Trick

    Directly counting subarrays with exactly k distinct elements is tricky with a single sliding window. The key insight is:

    Identity: exactly(k) = atMost(k) - atMost(k - 1)
    The number of subarrays with exactly $k$ distinct elements equals the number with at most $k$ distinct minus the number with at most $k - 1$ distinct.

    Counting subarrays with "at most K" distinct

    For each position of the right pointer, the number of valid subarrays ending at right is right - left + 1 (because every starting point from left to right gives a valid subarray).

    int atMostKDistinct(vector<int>& arr, int k) {
    int n = arr.size();
    unordered_map<int, int> freq;
    int left = 0, count = 0;

    for (int right = 0; right < n; right++) {
        freq[arr[right]]++;

        while ((int)freq.size() > k) {
            freq[arr[left]]--;
            if (freq[arr[left]] == 0) freq.erase(arr[left]);
            left++;
        }
        count += right - left + 1;  // all subarrays ending at right
    }
    return count;
}

int exactlyKDistinct(vector<int>& arr, int k) {
    return atMostKDistinct(arr, k) - atMostKDistinct(arr, k - 1);
}
// Time: O(n), Space: O(k)

    Why right - left + 1?

    Consider the window [left..right]. Every subarray that ends at right and starts at any index from left to right has at most k distinct elements. That is right - left + 1 subarrays:

    • [left..right]
    • [left+1..right]
    • ...
    • [right..right]

    Pattern 3: Sliding Window Maximum

    Given an array and window size k, find the maximum element in each window of size k as the window slides from left to right.

    Example: arr = [1, 3, -1, -3, 5, 3, 6, 7], k = 3. Output: [3, 3, 5, 5, 6, 7].

    Approach: Monotonic Deque

    A deque (double-ended queue) maintains indices of array elements in decreasing order of their values. The front of the deque always holds the index of the current window's maximum.

    Remove expired elements

    If the front of the deque is outside the window (its index is less than i - k + 1), pop it from the front.

    Maintain monotonicity

    Pop elements from the back of the deque whose values are less than or equal to arr[i]. They can never be the maximum for any future window.

    Push current index

    Add the current index i to the back of the deque.

    Record the maximum

    Once the window is fully formed (i >= k - 1), the front of the deque is the index of the maximum element. Record arr[deque.front()].

    Interactive Animation: Sliding Window Maximum

    Array: [1, 3, -1, -3, 5, 3, 6, 7], k = 3. The deque stores indices; the front (leftmost) always points to the window maximum.

    Press Step to begin.

    Code

    vector<int> maxSlidingWindow(vector<int>& arr, int k) {
    int n = arr.size();
    deque<int> dq;  // stores indices
    vector<int> result;

    for (int i = 0; i < n; i++) {
        // Remove indices outside the window
        if (!dq.empty() && dq.front() < i - k + 1) {
            dq.pop_front();
        }

        // Remove smaller elements from back
        while (!dq.empty() && arr[dq.back()] <= arr[i]) {
            dq.pop_back();
        }

        dq.push_back(i);

        // Record maximum once window is formed
        if (i >= k - 1) {
            result.push_back(arr[dq.front()]);
        }
    }
    return result;
}
// Time: O(n), Space: O(k)
    Why $O(n)$? Each index is pushed onto the deque exactly once and popped at most once. Across all iterations, the total number of push and pop operations is at most $2n$.

    Why the Monotonic Deque Works

    The deque contains indices sorted in decreasing order of their array values. Here is why each operation is correct:

    • Pop from front (expired): If the front index is outside the current window, it cannot be the maximum. Remove it.
    • Pop from back (smaller): If arr[dq.back()] ≤ arr[i], then dq.back() can never be the maximum in any window that includes index i. The new element is larger and has a later index (stays in the window longer). Remove the smaller element.
    • Front is always the max: After the two removal steps, the front of the deque holds the index of the largest element in the current window.

    Pattern 4: Minimum Window Sort

    Given an array, find the length of the smallest subarray that, if sorted, would make the entire array sorted.

    Example: arr = [1, 2, 5, 3, 7, 10, 9, 12]. The answer is 5 (sort the subarray [5, 3, 7, 10, 9] from index 2 to 6).

    Approach

    This problem is not a traditional sliding window, but uses a related two-pointer scan from both ends. The idea is to find the boundaries of the unsorted region.

    Find the right boundary

    Scan from left to right, tracking the running maximum. The last position where arr[i] < max_so_far is the right boundary of the unsorted region.

    Find the left boundary

    Scan from right to left, tracking the running minimum. The last position where arr[i] > min_so_far is the left boundary.

    Compute the answer

    The minimum window to sort is [left_boundary..right_boundary], with length right_boundary - left_boundary + 1.

    Interactive Animation: Minimum Window Sort

    Array: [1, 2, 5, 3, 7, 10, 9, 12]. Step through two passes: left-to-right to find the right boundary, then right-to-left to find the left boundary.

    Press Step to begin.

    Code

    int findUnsortedSubarray(vector<int>& arr) {
    int n = arr.size();
    int maxSoFar = INT_MIN, minSoFar = INT_MAX;
    int rightBound = -1, leftBound = -1;

    // Left to right: find right boundary
    for (int i = 0; i < n; i++) {
        maxSoFar = max(maxSoFar, arr[i]);
        if (arr[i] < maxSoFar) rightBound = i;
    }

    // Right to left: find left boundary
    for (int i = n - 1; i >= 0; i--) {
        minSoFar = min(minSoFar, arr[i]);
        if (arr[i] > minSoFar) leftBound = i;
    }

    if (rightBound == -1) return 0;  // already sorted
    return rightBound - leftBound + 1;
}
// Time: O(n), Space: O(1)
    Intuition: If an element is less than the maximum seen so far (scanning left to right), it is "out of place" and must be included in the sorted subarray. Similarly from the right side. The two boundaries together define the minimal unsorted window.

    Choosing the Right Pattern

    Problem Signal Pattern
    "Longest with at most k distinct"At Most K Distinct
    "Count subarrays with exactly k"atMost(k) - atMost(k-1)
    "Maximum in each window of size k"Monotonic Deque
    "Minimum subarray to sort"Two-pass boundary scan
    "Longest with at most k replacements"windowSize - maxFreq > k
    "Number of subarrays with sum = k"Prefix sums (sliding window may not apply with negatives)

    Key Takeaways

    • The "at most K distinct" pattern uses a hash map and shrinks the window when distinct count exceeds K.
    • To count subarrays with exactly K distinct, compute atMost(K) - atMost(K-1). The identity right - left + 1 counts all valid subarrays ending at each position.
    • The monotonic deque maintains the current window maximum in $O(1)$ per query. Each element is pushed and popped at most once, giving $O(n)$ total.
    • Minimum window sort uses two passes to find the boundaries of the unsorted region in $O(n)$ time.
    • Always check whether the problem has the monotonic property required for sliding window. If not (e.g., negative numbers in sum problems), consider prefix sums or other approaches.
