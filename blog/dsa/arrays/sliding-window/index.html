    Sliding Window Overview

    March 2026 · 18 min read

    What is the Sliding Window Technique?

    The sliding window technique is a method for processing arrays (or strings) by maintaining a window that moves across the data, one step at a time. Instead of recalculating everything from scratch for each position, you update the result incrementally by adding the new element entering the window and removing the element leaving it.

    Think of it like looking through a rectangular frame placed over a long filmstrip. You slide the frame one position to the right, and you only need to account for what appeared on the right edge and what disappeared from the left edge. Everything in the middle stays the same.

    Core Idea: Reuse computation from the previous window position. Only process the elements that changed (one entering, one leaving) rather than reprocessing the entire subarray.

    Why Use Sliding Window?

    Consider the problem: find the maximum sum of any contiguous subarray of size $k$ in an array of $n$ elements.

    Brute Force: $O(n \times k)$

    For every starting index, sum up $k$ elements. If $n = 10{,}000$ and $k = 1{,}000$, that is 10 million operations.

    int maxSum = INT_MIN;
for (int i = 0; i <= n - k; i++) {
    int sum = 0;
    for (int j = i; j < i + k; j++) {
        sum += arr[j];
    }
    maxSum = max(maxSum, sum);
}

    Sliding Window: $O(n)$

    Compute the sum of the first window. Then, for each subsequent position, subtract the element leaving the window and add the element entering it. Only two operations per step, regardless of $k$.

    int sum = 0;
for (int i = 0; i < k; i++) sum += arr[i];
int maxSum = sum;

for (int i = k; i < n; i++) {
    sum += arr[i] - arr[i - k];   // add new, remove old
    maxSum = max(maxSum, sum);
}
    Key Insight: The sliding window reduces work by recognizing that consecutive windows share $k - 1$ elements. Only the two boundary elements change.

    Fixed-Size vs Variable-Size Windows

    There are two main variants of the sliding window technique. They differ in how the window boundaries move.

    Aspect Fixed-Size Window Variable-Size Window
    Window size Constant $k$, given in the problem Changes dynamically based on a condition
    Left pointer movement Advances exactly one step per iteration Advances when the window violates a constraint
    Right pointer movement Advances exactly one step per iteration Advances to expand the window
    Typical problems Max sum of size k, averages of subarrays Longest substring, minimum window substring
    Complexity $O(n)$ $O(n)$ amortized (each element enters and exits the window at most once)

    Fixed-Size Window: How It Moves

    In a fixed-size window, both the left and right edges of the window advance by one position at each step. The window always contains exactly $k$ elements.

    Initialize

    Compute the aggregate (sum, max, count, etc.) for the first $k$ elements. This is your initial window.

    Slide Right

    Add the element at position $i$ (the new right edge). Remove the element at position $i - k$ (the old left edge).

    Update Answer

    Compare the current aggregate with your running answer (e.g., take the maximum).

    Repeat

    Continue until the right edge reaches the end of the array. Total work: $O(n)$.

    Variable-Size Window: How It Moves

    In a variable-size window, the right pointer expands the window to include more elements, and the left pointer contracts it when a constraint is violated. The window size changes dynamically.

    Expand

    Move the right pointer forward. Add the new element to the window state (hash map, counter, sum, etc.).

    Check Constraint

    Is the window still valid? If yes, update your answer. If no, proceed to shrink.

    Shrink

    Move the left pointer forward, removing elements from the window state, until the constraint is satisfied again.

    Repeat

    Continue expanding and shrinking. Each element enters and exits the window at most once, so the total work is $O(n)$.

    Code Templates

    Fixed-Size Window Template

    int slidingWindowFixed(vector<int>& arr, int k) {
    int n = arr.size();
    // 1. Build initial window
    int windowSum = 0;
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];
    }
    int answer = windowSum;

    // 2. Slide window
    for (int i = k; i < n; i++) {
        windowSum += arr[i];       // add entering element
        windowSum -= arr[i - k];   // remove leaving element
        answer = max(answer, windowSum);
    }
    return answer;
}

    Variable-Size Window Template

    int slidingWindowVariable(vector<int>& arr, int target) {
    int n = arr.size();
    int left = 0;
    int windowSum = 0;
    int answer = INT_MAX;

    for (int right = 0; right < n; right++) {
        windowSum += arr[right];       // expand window

        while (windowSum >= target) {  // shrink while valid
            answer = min(answer, right - left + 1);
            windowSum -= arr[left];
            left++;
        }
    }
    return (answer == INT_MAX) ? 0 : answer;
}

    When to Use Sliding Window

    Look for these signals in a problem statement:

    • Contiguous subarray or substring: The problem asks about a contiguous segment of elements.
    • "Maximum/minimum length" of a subarray satisfying some condition.
    • "Sum of subarray of size k": Direct fixed-size window application.
    • Character frequency constraints: "At most k distinct characters", "no repeating characters".
    • $O(n)$ expected: If brute force is $O(n^2)$ and involves nested loops over subarrays, sliding window often applies.
    Rule of Thumb: If the problem involves finding an optimal contiguous subarray and the brute force requires examining all subarrays, consider whether a sliding window can maintain the answer incrementally.

    When Sliding Window Does NOT Apply

    • Non-contiguous subarrays: If elements do not need to be adjacent (e.g., subsequence problems), sliding window does not help.
    • No monotonic property: If adding an element can both help and hurt (e.g., arrays with negative numbers in certain sum problems), the shrink step may not work correctly.
    • Global optimization without structure: Problems requiring DP or divide-and-conquer where local window state is insufficient.

    Complexity Analysis

    Variant Time Space Why
    Fixed-size $O(n)$ $O(1)$ Single pass, constant work per step
    Variable-size $O(n)$ amortized $O(k)$ or $O(|\Sigma|)$ Each element enters and exits window at most once; space depends on auxiliary data structure
    Brute force $O(n \times k)$ or $O(n^2)$ $O(1)$ Recalculates full subarray each time

    Key Takeaways

    • The sliding window technique converts $O(n \times k)$ or $O(n^2)$ brute force into $O(n)$ by reusing overlapping computation.
    • Fixed-size windows have a constant width and slide one step at a time.
    • Variable-size windows expand with the right pointer and shrink with the left pointer based on a constraint.
    • Each element enters and exits the window at most once, guaranteeing linear time.
    • The technique applies to contiguous subarrays and substrings where a monotonic property holds.
