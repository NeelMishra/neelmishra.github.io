    Variable-Size Sliding Window

    March 2026 · 25 min read

    Introduction

    Unlike the fixed-size variant, a variable-size sliding window lets the window grow and shrink dynamically. The right pointer expands the window to explore new elements, and the left pointer contracts it when a constraint is violated.

    This pattern is extremely common in string problems and subarray problems where you are asked to find the longest, shortest, or most optimal contiguous segment satisfying some condition.

    Key Invariant: Each element enters the window at most once (when the right pointer moves past it) and leaves the window at most once (when the left pointer moves past it). Total work across all steps: $O(n)$.

    General Template

    Most variable-size window problems follow this structure:

    int left = 0;
// window state: hash map, counter, sum, etc.

for (int right = 0; right < n; right++) {
    // 1. Add arr[right] to window state

    // 2. While window is invalid, shrink from left
    while (/* window violates constraint */) {
        // Remove arr[left] from window state
        left++;
    }

    // 3. Window is now valid; update answer
    // answer = max(answer, right - left + 1);
}

    The key decisions are:

    • What data structure maintains the window state? (hash map, counter, sum)
    • What is the validity condition? (no repeats, sum ≤ target, at most k distinct)
    • Are we maximizing or minimizing the window size?

    Problem: Longest Substring Without Repeating Characters

    Given a string s, find the length of the longest substring that contains no repeating characters.

    Example: s = "abcabcbb". The answer is 3 (the substring "abc").

    Approach

    Maintain a hash map that stores the last seen index of each character. When we encounter a character that is already in the window, move the left pointer past its previous occurrence.

    Expand

    Move right forward. Add s[right] to the hash map with its index.

    Check for duplicate

    If s[right] was already in the map and its stored index is at or after left, we have a repeat inside the window.

    Shrink

    Move left to one past the previous occurrence of s[right]. This skips over the duplicate.

    Update answer

    The current valid window length is right - left + 1. Take the maximum.

    Code

    int lengthOfLongestSubstring(string s) {
    int n = s.size();
    unordered_map<char, int> lastSeen;
    int left = 0, maxLen = 0;

    for (int right = 0; right < n; right++) {
        char c = s[right];
        if (lastSeen.count(c) && lastSeen[c] >= left) {
            left = lastSeen[c] + 1;  // shrink past duplicate
        }
        lastSeen[c] = right;
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Time: O(n), Space: O(min(n, |alphabet|))

    Problem: Minimum Window Substring

    Given strings s and t, find the shortest substring of s that contains all characters of t (including duplicates).

    Example: s = "ADOBECODEBANC", t = "ABC". The answer is "BANC".

    Approach

    This is a "shrink to optimize" pattern. We expand the window until it contains all characters of t, then shrink from the left to find the smallest valid window.

    Build target frequency map

    Count the frequency of each character in t. Track how many unique characters need to be matched (required).

    Expand the window

    Move right forward. Update the window frequency map. If a character's count matches its target count, increment formed.

    Shrink while valid

    When formed == required (all characters matched), try shrinking from the left. Record the window size if it is smaller than the current best. Remove the leftmost character and advance left.

    Track minimum

    Keep track of the start index and length of the smallest valid window found so far.

    Code

    string minWindow(string s, string t) {
    int n = s.size(), m = t.size();
    if (m > n) return "";

    unordered_map<char, int> tFreq, wFreq;
    for (char c : t) tFreq[c]++;

    int required = tFreq.size();  // unique chars to match
    int formed = 0;               // unique chars currently matched
    int left = 0;
    int bestLen = INT_MAX, bestStart = 0;

    for (int right = 0; right < n; right++) {
        char c = s[right];
        wFreq[c]++;
        if (tFreq.count(c) && wFreq[c] == tFreq[c]) {
            formed++;
        }

        // Shrink while window is valid
        while (formed == required) {
            int len = right - left + 1;
            if (len < bestLen) {
                bestLen = len;
                bestStart = left;
            }
            char lc = s[left];
            wFreq[lc]--;
            if (tFreq.count(lc) && wFreq[lc] < tFreq[lc]) {
                formed--;
            }
            left++;
        }
    }
    return (bestLen == INT_MAX) ? "" : s.substr(bestStart, bestLen);
}
// Time: O(n + m), Space: O(|alphabet|)

    Problem: Minimum Size Subarray Sum

    Given an array of positive integers and a target, find the length of the shortest contiguous subarray whose sum is greater than or equal to the target. Return 0 if no such subarray exists.

    Example: arr = [2, 3, 1, 2, 4, 3], target = 7. The answer is 2 (the subarray [4, 3]).

    Approach

    Since all elements are positive, adding more elements can only increase the sum. This monotonic property makes the sliding window valid: expand to increase the sum, shrink to decrease it.

    int minSubArrayLen(int target, vector<int>& arr) {
    int n = arr.size();
    int left = 0, windowSum = 0;
    int minLen = INT_MAX;

    for (int right = 0; right < n; right++) {
        windowSum += arr[right];

        while (windowSum >= target) {
            minLen = min(minLen, right - left + 1);
            windowSum -= arr[left];
            left++;
        }
    }
    return (minLen == INT_MAX) ? 0 : minLen;
}
// Time: O(n), Space: O(1)
    Why does this work? Each element is added once (when right passes it) and removed at most once (when left passes it). The total number of add + remove operations across the entire loop is at most $2n$, giving $O(n)$ time.

    Problem: Longest Repeating Character Replacement

    Given a string s and an integer k, find the length of the longest substring that contains only one character, after performing at most k replacements.

    Example: s = "AABABBA", k = 1. The answer is 4 (replace the B at index 3 to get "AAAA" or similar).

    Approach

    Maintain a frequency map of characters in the window. The number of characters that need to be replaced is windowSize - maxFreq, where maxFreq is the count of the most frequent character in the window. If this exceeds k, shrink the window.

    int characterReplacement(string s, int k) {
    int n = s.size();
    vector<int> freq(26, 0);
    int left = 0, maxFreq = 0, maxLen = 0;

    for (int right = 0; right < n; right++) {
        freq[s[right] - 'A']++;
        maxFreq = max(maxFreq, freq[s[right] - 'A']);

        // Window size - max freq = chars to replace
        while ((right - left + 1) - maxFreq > k) {
            freq[s[left] - 'A']--;
            left++;
        }
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Time: O(n), Space: O(1)
    Subtlety: We do not need to update maxFreq when shrinking. The answer can only improve when maxFreq increases, so keeping a stale (too-high) maxFreq is safe. The window just might not shrink as aggressively, but it never incorrectly enlarges the answer.

    Variable-Size Window Patterns

    Variable-size windows show up in two flavors:

    1. Find the Longest (maximize window)

    Expand the right pointer. When the window becomes invalid, shrink the left pointer just enough to restore validity. Update the answer after shrinking.

    // Longest subarray/substring satisfying condition
for (right = 0; right < n; right++) {
    // add arr[right]
    while (/* invalid */) {
        // remove arr[left]; left++
    }
    answer = max(answer, right - left + 1);
}

    2. Find the Shortest (minimize window)

    Expand the right pointer. When the window becomes valid, update the answer and keep shrinking to find the shortest valid window.

    // Shortest subarray/substring satisfying condition
for (right = 0; right < n; right++) {
    // add arr[right]
    while (/* valid */) {
        answer = min(answer, right - left + 1);
        // remove arr[left]; left++
    }
}

    The Role of the Hash Map

    In most variable-size window problems, a hash map (or frequency array) is the core data structure maintaining window state. The map tracks:

    • Character frequencies: How many times each character appears in the current window.
    • Last seen index: The most recent index where a character was seen (useful for the "no repeats" problem).
    • Match count: How many characters have met their required frequency.

    For small alphabets (e.g., 26 lowercase letters), a fixed-size array is more efficient than a hash map. For larger character sets (e.g., Unicode), use an unordered_map.

    Edge Cases

    • Empty string/array: Return 0 or empty string depending on the problem.
    • All identical characters: The window may never need to shrink (longest without repeats = 1; longest with k replacements = entire string if k is large enough).
    • Target impossible: For minimum window substring, if t contains characters not in s, return empty.
    • Single character: The window is always of size 1. Make sure your code handles this without off-by-one errors.
    • Negative numbers: Variable-size windows on sums do not work with negative numbers because adding an element might decrease the sum, breaking the monotonic property. Use prefix sums or other techniques instead.

    Key Takeaways

    • Variable-size windows expand with the right pointer and shrink with the left pointer.
    • Each element enters and exits the window at most once, giving $O(n)$ amortized time.
    • A hash map or frequency array typically maintains the window state.
    • "Find longest" problems shrink when invalid; "find shortest" problems shrink while valid.
    • The technique requires a monotonic property: expanding the window can only make the condition "more violated" (or "more satisfied"), never both.
