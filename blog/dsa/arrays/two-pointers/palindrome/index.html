    Palindrome Patterns

    March 2026 · 22 min read

    Introduction

    A palindrome reads the same forward and backward. While this sounds like a simple definition, it gives rise to a surprisingly rich family of algorithm problems. The two-pointer technique is at the heart of almost every efficient palindrome solution.

    There are two fundamental strategies for palindrome problems with two pointers:

    • Shrink from ends: Place one pointer at the start and one at the end, then move them toward each other. Used for checking if something is a palindrome.
    • Expand from center: Place both pointers at a center position, then move them outward. Used for finding palindromic substrings.

    This page covers seven problems that span the full range of palindrome techniques. Each one builds on ideas from the previous, so reading them in order is recommended.

    Prerequisites: You should be comfortable with the Two Pointers Overview and the Opposite-End Pointers pattern before diving in.

    1. Valid Palindrome (LeetCode 125)

    Problem: Given a string s, determine if it is a palindrome, considering only alphanumeric characters and ignoring cases.

    Example: "A man, a plan, a canal: Panama" is a palindrome. After filtering: "amanaplanacanalpanama".

    Approach: Two Pointers from the Ends

    Place a left pointer at index 0 and a right pointer at the last index. At each step:

    1. Skip the left pointer forward past any non-alphanumeric character.
    2. Skip the right pointer backward past any non-alphanumeric character.
    3. Compare the two characters (case-insensitive). If they do not match, return false.
    4. Move both pointers inward and repeat until they meet or cross.

    Why Two Pointers are Ideal

    A palindrome is defined by the relationship between symmetric positions. Two pointers naturally represent this symmetry: the left pointer and the right pointer always point at positions that should hold the same character. No extra data structure is needed.

    C++ Implementation

    bool isPalindrome(string s) {
    int left = 0, right = s.size() - 1;
    while (left < right) {
        // Skip non-alphanumeric from the left
        while (left < right && !isalnum(s[left]))
            left++;
        // Skip non-alphanumeric from the right
        while (left < right && !isalnum(s[right]))
            right--;
        // Compare characters (case-insensitive)
        if (tolower(s[left]) != tolower(s[right]))
            return false;
        left++;
        right--;
    }
    return true;
}

    Time and Space

    Time: O(n) where n is the length of the string. Each character is visited at most once by each pointer.

    Space: O(1). We do not create a filtered copy of the string.

    2. Valid Palindrome II (LeetCode 680)

    Problem: Given a string s, return true if it can become a palindrome after deleting at most one character.

    Example: "abca" returns true. Removing 'b' gives "aca" or removing 'c' gives "aba". Both are palindromes.

    Approach: Shrink with a Second Chance

    Start with the standard two-pointer palindrome check. When you encounter the first mismatch at positions left and right, you have two options:

    1. Delete the character at left (skip it) and check if s[left+1..right] is a palindrome.
    2. Delete the character at right (skip it) and check if s[left..right-1] is a palindrome.

    If either branch succeeds, the answer is true. If neither works, the answer is false.

    Branching Decision

    When s[L] != s[R], one of these characters must be the one we delete. We do not know which, so we try both. Each branch is itself a standard palindrome check, so the total work remains O(n).

    C++ Implementation

    bool checkPalin(const string& s, int lo, int hi) {
    while (lo < hi) {
        if (s[lo] != s[hi]) return false;
        lo++; hi--;
    }
    return true;
}

bool validPalindrome(string s) {
    int left = 0, right = s.size() - 1;
    while (left < right) {
        if (s[left] != s[right]) {
            // Try skipping left or skipping right
            return checkPalin(s, left + 1, right)
                || checkPalin(s, left, right - 1);
        }
        left++;
        right--;
    }
    return true;
}

    Time and Space

    Time: O(n). The outer loop runs at most n/2 steps, and the helper runs at most n/2 steps. We call the helper at most twice.

    Space: O(1). No extra data structures.

    Step-Through Animation: The Fork

    Input: "abcbda". Can we make it a palindrome by removing at most one character?

    Common Mistake: Only trying one branch (e.g., always skipping the left character). You must try both branches because either deletion could produce a valid palindrome.

    3. Palindrome Linked List (LeetCode 234)

    Problem: Given the head of a singly linked list, return true if it is a palindrome.

    Example: 1 -> 2 -> 2 -> 1 is a palindrome.

    Approach: Fast/Slow + Reverse + Compare

    With an array, we would use two pointers from the ends. A singly linked list only goes forward. The trick is to combine three techniques:

    1. Find the middle using fast and slow pointers. When fast reaches the end, slow is at the middle.
    2. Reverse the second half of the list starting from slow.
    3. Compare the first half and the reversed second half node by node.

    Finding the Middle

    The slow pointer moves one node at a time. The fast pointer moves two nodes at a time. When fast reaches the end (or null), slow is at the midpoint. For even-length lists, slow ends at the start of the second half.

    Reversing the Second Half

    Starting from slow, reverse the pointers so the second half points backward. This lets us traverse it from the "end" toward the middle.

    Comparing Both Halves

    With the first half going forward from the head and the reversed second half going backward from the tail, we can compare values node by node. If all values match, it is a palindrome.

    Visual: Linked List Palindrome

    C++ Implementation

    bool isPalindrome(ListNode* head) {
    if (!head || !head->next) return true;

    // Step 1: Find the middle with fast/slow
    ListNode* slow = head;
    ListNode* fast = head;
    while (fast->next && fast->next->next) {
        slow = slow->next;
        fast = fast->next->next;
    }

    // Step 2: Reverse the second half
    ListNode* prev = nullptr;
    ListNode* curr = slow->next;
    while (curr) {
        ListNode* nxt = curr->next;
        curr->next = prev;
        prev = curr;
        curr = nxt;
    }

    // Step 3: Compare both halves
    ListNode* p1 = head;
    ListNode* p2 = prev; // head of reversed half
    while (p2) {
        if (p1->val != p2->val) return false;
        p1 = p1->next;
        p2 = p2->next;
    }
    return true;
}

    Time and Space

    Time: O(n). Three passes through the list (find middle, reverse, compare).

    Space: O(1). We reverse in place and use only pointer variables.

    Restore the list: In production code, you should reverse the second half back after comparing to leave the list in its original state. The solution above modifies the input.

    4. Palindromic Substrings (LeetCode 647)

    Problem: Given a string s, return the number of palindromic substrings in it. A single character is considered a palindrome.

    Example: "aaa" has 6 palindromic substrings: "a", "a", "a", "aa", "aa", "aaa".

    Approach: Expand Around Center

    This is the second fundamental palindrome pattern. Instead of shrinking from the ends, we start at each possible center and expand outward.

    For a string of length n, there are 2n - 1 possible centers:

    • n centers on characters (odd-length palindromes)
    • n - 1 centers between adjacent characters (even-length palindromes)

    For each center, expand with two pointers moving in opposite directions. As long as the characters match, we have a palindrome. Count each successful expansion.

    C++ Implementation

    int countSubstrings(string s) {
    int n = s.size(), count = 0;
    for (int center = 0; center < 2 * n - 1; center++) {
        int left = center / 2;
        int right = left + (center % 2);
        // Expand outward while characters match
        while (left >= 0 && right < n
               && s[left] == s[right]) {
            count++;
            left--;
            right++;
        }
    }
    return count;
}

    Odd vs Even Length Palindromes

    An odd-length palindrome like "aba" has a single character center ('b'). An even-length palindrome like "abba" has a center between the two 'b's. To catch both types, we iterate over all 2n-1 possible centers. The formula left = center/2, right = left + center%2 elegantly handles both cases.

    Step-Through Animation: Expand Around Center

    Input: "abba". Count all palindromic substrings by expanding from each center.

    Time and Space

    Time: O(n2). For each of the 2n-1 centers, expansion can take up to O(n) steps in the worst case.

    Space: O(1). Only pointer variables.

    5. Longest Palindromic Substring (LeetCode 5)

    Problem: Given a string s, return the longest palindromic substring in s.

    Example: "babad" returns "bab" or "aba".

    Approach: Expand Around Center (Track Best)

    This problem uses the same expand-from-center technique as Palindromic Substrings. The only difference: instead of counting palindromes, we track the start and length of the longest one found so far.

    C++ Implementation

    string longestPalindrome(string s) {
    int n = s.size(), bestStart = 0, bestLen = 1;
    for (int center = 0; center < 2 * n - 1; center++) {
        int left = center / 2;
        int right = left + (center % 2);
        while (left >= 0 && right < n
               && s[left] == s[right]) {
            int len = right - left + 1;
            if (len > bestLen) {
                bestStart = left;
                bestLen = len;
            }
            left--;
            right++;
        }
    }
    return s.substr(bestStart, bestLen);
}

    Expand-from-Center vs Dynamic Programming

    Many textbooks teach a DP approach where dp[i][j] is true if s[i..j] is a palindrome. Let us compare the two approaches:

    PropertyExpand from CenterDynamic Programming
    TimeO(n2)O(n2)
    SpaceO(1)O(n2)
    Cache friendlinessExcellent (sequential access)Poor (2D table)
    Code complexitySimpleModerate
    Best forFinding the longest palindromeWhen you need all dp[i][j] answers
    Interview tip: Expand-from-center is almost always the preferred approach. It is simpler to code, uses O(1) space, and runs faster in practice. The DP approach is mainly useful when you need to look up whether arbitrary substrings are palindromes (e.g., palindrome partitioning).

    The DP Approach (for reference)

    string longestPalindromeDP(string s) {
    int n = s.size();
    if (n < 2) return s;
    vector<vector<bool>> dp(n, vector<bool>(n, false));
    int start = 0, maxLen = 1;

    // Every single character is a palindrome
    for (int i = 0; i < n; i++) dp[i][i] = true;

    // Check substrings of length 2
    for (int i = 0; i < n - 1; i++) {
        if (s[i] == s[i + 1]) {
            dp[i][i + 1] = true;
            start = i; maxLen = 2;
        }
    }

    // Check substrings of length 3 and above
    for (int len = 3; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            if (s[i] == s[j] && dp[i + 1][j - 1]) {
                dp[i][j] = true;
                if (len > maxLen) {
                    start = i; maxLen = len;
                }
            }
        }
    }
    return s.substr(start, maxLen);
}

    6. Minimum Insertions to Make a Palindrome

    Problem: Given a string s, find the minimum number of character insertions needed to make it a palindrome.

    Example: "abc" requires 2 insertions to become "cbabc" or "abcba".

    The Connection to LCS

    This problem has an elegant reduction. The minimum number of insertions equals:

    min_insertions = n - LPS(s)

    where LPS(s) is the length of the Longest Palindromic Subsequence of s. And the LPS of s equals the Longest Common Subsequence of s and its reverse.

    Intuition

    The longest palindromic subsequence is the largest "palindrome skeleton" already present in the string. Every character that is not part of this skeleton needs a matching insertion to make the full string a palindrome. So we insert exactly n - LPS characters.

    C++ Implementation

    int minInsertions(string s) {
    int n = s.size();
    string rev(s.rbegin(), s.rend());

    // LCS of s and reverse(s) = LPS of s
    vector<vector<int>> dp(n + 1, vector<int>(n + 1, 0));
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            if (s[i - 1] == rev[j - 1])
                dp[i][j] = dp[i - 1][j - 1] + 1;
            else
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
    int lps = dp[n][n];
    return n - lps;
}

    Visual: LPS Reduction

    Time and Space

    Time: O(n2) for the LCS computation.

    Space: O(n2) for the DP table. Can be optimized to O(n) with rolling rows.

    7. Pattern Recognition: Choosing Your Approach

    The biggest challenge with palindrome problems is recognizing which strategy to use. Here is a decision framework.

    Shrink from Ends vs Expand from Center

    CriterionShrink from EndsExpand from Center
    GoalVerify if a string/range is a palindromeFind/count palindromic substrings
    DirectionInward (L goes right, R goes left)Outward (L goes left, R goes right)
    Typical problemsValid Palindrome, Palindrome IIPalindromic Substrings, Longest Palindrome
    Initial stateL=0, R=n-1L=R=center (or L=center, R=center+1)
    TimeO(n) per checkO(n2) total over all centers

    Use Shrink from Ends When...

    You are given a complete string or range and need to determine if it is a palindrome. The question is a yes/no verification.

    Signals: "Is this a palindrome?", "Can it be a palindrome after one deletion?", "Check if the linked list is a palindrome."

    Use Expand from Center When...

    You need to find or count palindromic substrings. The question asks to search for palindromes within a larger string.

    Signals: "Count palindromic substrings", "Find the longest palindromic substring", "Partition into palindromes."

    Common Traps and Edge Cases

    Empty and single-character strings: Both are palindromes by definition. Make sure your code handles "" and "a" without errors.
    Even vs odd length: When expanding from center, forgetting the even-length case (center between two characters) is one of the most common bugs. Always iterate over 2n-1 centers.
    Case sensitivity and special characters: In LC 125, many solutions fail because they forget to normalize. Always clarify in an interview whether to ignore case and non-alphanumeric characters.

    Summary

    • Palindrome problems revolve around two core two-pointer strategies: shrink from ends (verification) and expand from center (discovery).
    • Shrink from ends is O(n) and ideal for checking if a string is a palindrome, with variants for skipping characters or branching on deletion.
    • Expand from center is O(n2) and ideal for counting or finding palindromic substrings. It uses O(1) space and outperforms DP in practice.
    • Linked-list palindrome problems combine fast/slow pointers with list reversal and then apply the shrink approach.
    • Minimum-insertion and partitioning problems reduce to LCS/LPS via dynamic programming.
    • Always handle edge cases: empty strings, single characters, even vs odd lengths, and character normalization.
