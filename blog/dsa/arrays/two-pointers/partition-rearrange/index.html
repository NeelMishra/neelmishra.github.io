    Partition & Rearrange

    March 2026 · 25 min read

    Introduction

    Partitioning is one of the most fundamental operations in computer science. At its core, partitioning means rearranging an array so that elements satisfying a condition appear on one side, and the rest appear on the other. This single idea underpins quicksort, quickselect, the Dutch National Flag problem, and a family of array rearrangement problems.

    In this post we study partition schemes in depth, then apply them to classic interview problems. Every technique here runs in $O(n)$ time and $O(1)$ extra space unless stated otherwise.

    What you will learn: Lomuto and Hoare partition schemes, QuickSelect for order statistics, greedy partition labels, wiggle sort (both variants), and alternating sign rearrangement. Each section includes C++ code and interactive step-through animations.

    1. Lomuto Partition Scheme

    The Lomuto partition scheme is the simpler of the two classic partition algorithms. It uses a single scanning pointer that walks left to right, maintaining the invariant that everything to the left of a boundary index is less than or equal to the pivot.

    How It Works

    Choose the pivot

    Pick the last element of the subarray as the pivot. Let i = lo - 1. This marks the boundary of the "less than or equal" region.

    Scan with j

    Walk j from lo to hi - 1. If arr[j] <= pivot, increment i and swap arr[i] with arr[j]. This grows the "small" region by one element.

    Place the pivot

    After the scan, swap arr[i + 1] with arr[hi]. The pivot is now at index i + 1, with all smaller elements to its left and all larger elements to its right.

    C++ Implementation

    int lomutoPartition(vector<int>& arr, int lo, int hi) {
    int pivot = arr[hi];
    int i = lo - 1;

    for (int j = lo; j < hi; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[hi]);
    return i + 1;
}

    Complexity

    • Time: $O(n)$ per partition call. In quicksort, average $O(n \log n)$, worst $O(n^2)$.
    • Space: $O(1)$ extra.
    • Swaps: Up to $n - 1$ swaps in the worst case (every element is smaller than the pivot).

    2. Hoare Partition Scheme

    Hoare partition uses two pointers that start at opposite ends of the subarray and walk towards each other. When the left pointer finds an element greater than or equal to the pivot and the right pointer finds one less than or equal to the pivot, the two elements are swapped. The pointers continue until they cross.

    Why Hoare Is Faster in Practice

    Fewer swaps: Lomuto swaps every element that is less than or equal to the pivot past the boundary, even if the element is already on the correct side. Hoare only swaps when both pointers have found misplaced elements. On average, Hoare performs about three times fewer swaps than Lomuto.

    How It Works

    Choose the pivot

    Pick arr[lo] as the pivot. Set left = lo - 1 and right = hi + 1.

    Advance left

    Increment left until arr[left] >= pivot.

    Advance right

    Decrement right until arr[right] <= pivot.

    Swap or stop

    If left < right, swap arr[left] and arr[right] and repeat from Step 2. Otherwise, return right as the partition index.

    C++ Implementation

    int hoarePartition(vector<int>& arr, int lo, int hi) {
    int pivot = arr[lo];
    int left = lo - 1;
    int right = hi + 1;

    while (true) {
        do { left++;  } while (arr[left]  < pivot);
        do { right--; } while (arr[right] > pivot);

        if (left >= right) return right;
        swap(arr[left], arr[right]);
    }
}
    Caution: Hoare partition does not place the pivot at the returned index. The return value right guarantees that elements in [lo..right] are all <= elements in [right+1..hi]. When using Hoare with quicksort, recurse on [lo..right] and [right+1..hi].

    Complexity

    • Time: $O(n)$ per partition call.

    • Space: $O(1)$ extra.

    3. QuickSelect: Kth Largest Element (LeetCode 215)

    QuickSelect applies the partition idea to find the $k$-th largest (or smallest) element without fully sorting the array. After one partition, the pivot is at its final sorted position. If that position matches what we need, we are done. Otherwise, we recurse into only one side.

    Algorithm

    Partition

    Use Lomuto (or Hoare) to partition the array. Let p be the index where the pivot lands.

    Compare

    If p == n - k, then arr[p] is the $k$-th largest. Return it.

    Recurse

    If the target index is less than p, recurse on the left half. Otherwise, recurse on the right half.

    C++ Implementation

    // LeetCode 215: Kth Largest Element in an Array
int findKthLargest(vector<int>& nums, int k) {
    int target = nums.size() - k;  // convert to 0-indexed
    int lo = 0, hi = nums.size() - 1;

    while (lo <= hi) {
        int p = lomutoPartition(nums, lo, hi);
        if (p == target) return nums[p];
        else if (p < target) lo = p + 1;
        else hi = p - 1;
    }
    return -1; // unreachable for valid input
}

    4. Partition Labels (LeetCode 763)

    Given a string, partition it into as many parts as possible so that each letter appears in at most one part. Return the sizes of these parts.

    This is not a quicksort-style partition but uses the same mental model: define regions and decide where boundaries fall. It is a greedy problem with a two-pointer flavor.

    Approach

    Record last occurrences

    For each character, store the index of its last occurrence in the string.

    Scan with greedy extension

    Maintain a start and end for the current partition. As you scan character by character, update end = max(end, lastOccurrence[ch]).

    Close the partition

    When the current index equals end, the current partition is complete. Record its size and start a new partition.

    C++ Implementation

    // LeetCode 763: Partition Labels
vector<int> partitionLabels(string s) {
    int last[26] = {};
    for (int i = 0; i < (int)s.size(); i++) {
        last[s[i] - 'a'] = i;
    }

    vector<int> result;
    int start = 0, end = 0;

    for (int i = 0; i < (int)s.size(); i++) {
        end = max(end, last[s[i] - 'a']);
        if (i == end) {
            result.push_back(end - start + 1);
            start = end + 1;
        }
    }
    return result;
}

    5. Wiggle Sort (LeetCode 280)

    Rearrange the array so that a[0] <= a[1] >= a[2] <= a[3] >= ... This is a non-strict wiggle pattern.

    Key Insight

    You do not need to sort. A single pass suffices. At each index i:

    • If i is odd, we need a[i] >= a[i-1]. If violated, swap a[i] and a[i-1].
    • If i is even, we need a[i] <= a[i-1]. If violated, swap.

    This greedy approach works because fixing a violation at position i never breaks the condition at position i-1.

    C++ Implementation

    // LeetCode 280: Wiggle Sort
void wiggleSort(vector<int>& nums) {
    for (int i = 1; i < (int)nums.size(); i++) {
        if ((i % 2 == 1 && nums[i] < nums[i - 1]) ||
            (i % 2 == 0 && nums[i] > nums[i - 1])) {
            swap(nums[i], nums[i - 1]);
        }
    }
}

    6. Wiggle Sort II (LeetCode 324)

    Rearrange the array so that nums[0] < nums[1] > nums[2] < nums[3] > ... The key difference from Wiggle Sort I is the strict inequality. Simple adjacent swaps no longer work because equal elements can violate the condition.

    Approach: Median + Virtual Indexing

    Find the median

    Use QuickSelect (see Section 3) to find the median in $O(n)$ average time.

    Three-way partition

    Partition the array into three groups: elements less than the median, equal to the median, and greater than the median. This is the Dutch National Flag partition.

    Virtual index mapping

    Map index i to a "virtual" index (2*i + 1) % (n | 1). This interleaves positions so that small elements go to even indices and large elements go to odd indices, ensuring strict inequality.

    C++ Implementation

    // LeetCode 324: Wiggle Sort II
void wiggleSortII(vector<int>& nums) {
    int n = nums.size();

    // Find median using nth_element (QuickSelect internally)
    auto midIt = nums.begin() + n / 2;
    nth_element(nums.begin(), midIt, nums.end());
    int median = *midIt;

    // Virtual index mapping
    auto vi = [&](int i) { return (2 * i + 1) % (n | 1); };

    // Three-way partition with virtual indexing
    int lo = 0, hi = n - 1, cur = 0;

    while (cur <= hi) {
        if (nums[vi(cur)] > median) {
            swap(nums[vi(cur)], nums[vi(lo)]);
            lo++;
            cur++;
        } else if (nums[vi(cur)] < median) {
            swap(nums[vi(cur)], nums[vi(hi)]);
            hi--;
        } else {
            cur++;
        }
    }
}
    Why virtual indexing? Without it, after three-way partitioning, elements equal to the median might cluster next to each other in adjacent odd/even positions, violating strict inequality. Virtual indexing spreads equal elements apart so they never sit next to another equal element.

    Complexity

    • Time: $O(n)$ average (dominated by QuickSelect).
    • Space: $O(1)$ extra. The virtual indexing is computed on the fly.

    7. Rearrange Array Elements by Sign (LeetCode 2149)

    Given an array with equal numbers of positive and negative integers, rearrange it so that positive and negative elements alternate, starting with a positive element. The relative order within positives and within negatives must be preserved.

    Approach: Two Write Pointers

    Create result array

    Allocate a result array of the same size. Maintain two write pointers: posIdx = 0 (for even positions) and negIdx = 1 (for odd positions).

    Scan once

    Iterate through the original array. Place each positive element at posIdx and advance by 2. Place each negative element at negIdx and advance by 2.

    C++ Implementation

    // LeetCode 2149: Rearrange Array Elements by Sign
vector<int> rearrangeArray(vector<int>& nums) {
    int n = nums.size();
    vector<int> result(n);
    int posIdx = 0, negIdx = 1;

    for (int x : nums) {
        if (x > 0) {
            result[posIdx] = x;
            posIdx += 2;
        } else {
            result[negIdx] = x;
            negIdx += 2;
        }
    }
    return result;
}

    8. Comparison: Lomuto vs Hoare vs Dutch National Flag

    All three are partition schemes, but they differ in how many regions they create and how they manage pointers.

    Aspect Lomuto Hoare Dutch National Flag
    Regions created 2 (less/equal, greater) 2 (less/equal, greater/equal) 3 (less, equal, greater)
    Pointers 1 boundary + 1 scanner 2 converging pointers 3 pointers (lo, mid, hi)
    Pivot placement Pivot ends at exact position Pivot is not placed exactly All equal-to-pivot elements are grouped
    Average swaps ~n/2 ~n/6 ~n/3
    Handles duplicates Poorly (degrades to $O(n^2)$ quicksort) Better (still degrades with all-equal) Optimally (groups duplicates in the middle)
    Ease of implementation Simplest Moderate (off-by-one pitfalls) Moderate
    Best use case Teaching, quickselect Quicksort (fewer swaps) Arrays with many duplicates, Sort Colors

    Diagram: Partition Region Layout

    Interview tip: If the problem involves partitioning around a single value and there may be many duplicates, prefer the Dutch National Flag (three-way) partition. It avoids worst-case behavior in quicksort by collapsing all equal elements into the middle. You should also know Lomuto for QuickSelect, because its exact pivot placement makes it easy to reason about the pivot index.

    Pattern Reference

    Pattern: Single-Pointer Partition

    One boundary pointer, one scanner. The boundary separates "processed" from "unprocessed" elements. Scanner walks left to right, swapping into the boundary region when the condition is met.

    Used in: Lomuto partition, Move Zeroes, Sort by Parity, Rearrange by Sign.

    Pattern: Two-Pointer Convergence

    Two pointers start at opposite ends and walk toward each other. Each pointer skips elements already on the correct side, and swaps happen when both pointers find misplaced elements.

    Used in: Hoare partition, Container With Most Water, Trapping Rain Water.

    Pattern: Three-Way / Dutch National Flag

    Three pointers (lo, mid, hi) manage three regions. The middle pointer scans, routing elements to the low or high region as needed.

    Used in: Sort Colors, Wiggle Sort II, any problem with three categories of elements.

    Pattern: Greedy Partition Boundaries

    Track the farthest reach of each element (e.g., last occurrence). Greedily extend the current partition boundary. Close a partition when the scan index reaches the boundary.

    Used in: Partition Labels, Jump Game, Video Stitching.

    Summary

    Partition and rearrangement problems are a natural extension of the two-pointer technique. They appear constantly in interviews because they test your ability to manage array invariants in a single pass.

    • Lomuto Partition: Simplest scheme. Single scanner, pivot ends in exact position. Good for QuickSelect.
    • Hoare Partition: Two converging pointers, fewer swaps on average. Better for quicksort in practice.
    • QuickSelect: Apply partition once, recurse into one side. Average $O(n)$ for order statistics.
    • Partition Labels: Greedy extension of partition boundaries using last-occurrence tracking.
    • Wiggle Sort: Single-pass greedy swaps. Works for non-strict inequality.
    • Wiggle Sort II: Median-finding + three-way partition + virtual indexing for strict inequality.
    • Rearrange by Sign: Two independent write pointers filling even and odd positions.

    The unifying theme: define regions, maintain clear invariants about what each region contains, and advance pointers to grow or shrink those regions until every element is in the right place.
