    Reverse Techniques

    March 2026 · 20 min read

    Reversing is one of the most useful primitives in array and string manipulation. On its own it solves several classic problems, and when combined with itself it unlocks powerful tricks like rotating an array in O(n) time with O(1) extra space. This post walks through every important reversal problem, from the basic two-pointer swap all the way to the three-reverse rotation trick and its mathematical proof.

    1. Reverse String (LeetCode 344)

    Problem. Given a character array s, reverse it in place. You must do this by modifying the input array with O(1) extra memory.

    Idea

    Place one pointer at the start and another at the end. Swap the characters they point to, then move both pointers inward. Repeat until they meet or cross.

    Initialize pointers

    Set left = 0 and right = s.size() - 1. These mark the two ends of the region that still needs to be reversed.

    Swap and shrink

    Swap s[left] with s[right]. Then increment left and decrement right. The reversed portion now includes one more character on each side.

    Terminate

    When left >= right, every character has been swapped into its mirror position. The array is fully reversed.

    C++ Code

    class Solution {
public:
    void reverseString(vector<char>& s) {
        int left = 0, right = s.size() - 1;
        while (left < right) {
            swap(s[left], s[right]);
            left++;
            right--;
        }
    }
};

    Time: O(n) where n is the length of the string. Each element is visited at most once.
    Space: O(1). Only two integer variables are used.

    Why swap from both ends? If you reversed by shifting every element one position (like insertion), each shift is O(n) and you would need n/2 shifts, giving O(n2). The two-pointer swap visits each element exactly once.

    2. Reverse Vowels of a String (LeetCode 345)

    Problem. Given a string s, reverse only the vowels and return the modified string. Vowels are a, e, i, o, u (both lowercase and uppercase).

    Idea

    This is the same two-pointer approach as reversing a full string, but with a twist: each pointer skips over consonants. It only stops when it lands on a vowel. Once both pointers point to vowels, swap them and continue inward.

    Skip-and-Swap

    Advance left past consonants until it finds a vowel. Advance right backward past consonants until it finds a vowel. If left < right, swap and move both inward. Otherwise, stop.

    C++ Code

    class Solution {
public:
    string reverseVowels(string s) {
        auto isVowel = [](char c) {
            c = tolower(c);
            return c == 'a' || c == 'e' || c == 'i'
                || c == 'o' || c == 'u';
        };
        int left = 0, right = s.size() - 1;
        while (left < right) {
            while (left < right && !isVowel(s[left]))  left++;
            while (left < right && !isVowel(s[right])) right--;
            if (left < right) {
                swap(s[left], s[right]);
                left++;
                right--;
            }
        }
        return s;
    }
};

    Time: O(n). Each pointer moves inward across the string at most once.
    Space: O(1) extra (ignoring the output string).

    Step-Through Animation

    Input: "leetcode". Vowels are highlighted in orange. Watch the pointers skip consonants and only swap vowels. Finalized vowels turn green.

    Edge cases. A string with no vowels returns unchanged. A string that is all vowels gets fully reversed. Single-character strings are trivially handled by the left < right guard.

    3. Reverse Words in a String (LeetCode 151)

    Problem. Given a string s, reverse the order of words. A word is a maximal substring of non-space characters. The returned string should have a single space separating words with no leading or trailing spaces.

    The Two-Pass Technique

    The elegant in-place approach has two passes:

    Reverse the entire string

    After this, the words are in the correct order but each individual word is spelled backward.

    "the sky is blue" becomes "eulb si yks eht"

    Reverse each word individually

    Walk through the reversed string, find each word boundary, and reverse the characters within that word.

    "eulb si yks eht" becomes "blue is sky the"

    Between the two passes, you also clean up extra spaces. But the core idea is: two reversals undo each other at the character level but preserve the word-level reordering.

    Why Does This Work?

    Think of the string as a sequence of blocks [W1][W2][W3]. Reversing the whole string gives [W3_rev][W2_rev][W1_rev]. Now reversing each block individually gives [W3][W2][W1]. The words are now in reversed order, spelled correctly.

    C++ Code

    class Solution {
public:
    string reverseWords(string s) {
        // Step 1: reverse entire string
        reverse(s.begin(), s.end());

        int n = s.size();
        int write = 0;  // write pointer for cleaned result

        for (int i = 0; i < n; i++) {
            if (s[i] == ' ') continue;

            // add space before word (except for first word)
            if (write != 0) s[write++] = ' ';

            // copy word characters
            int j = i;
            while (j < n && s[j] != ' ')
                s[write++] = s[j++];

            // reverse the word we just wrote
            reverse(s.begin() + write - (j - i),
                    s.begin() + write);
            i = j;
        }
        s.resize(write);
        return s;
    }
};

    Time: O(n). Two passes over the string.
    Space: O(1) if done in place on the character array. The above modifies the string itself and resizes at the end.

    The read/write pointer. Notice how this combines the reverse technique with a same-direction two-pointer pattern. The read pointer j scans ahead to find word boundaries while the write pointer write compacts the result, skipping extra spaces.

    Worked Example

    Input: " hello world "

    After full reverse" dlrow olleh "
    Copy + reverse word 1"world" written at position 0
    Copy + reverse word 2"world hello" at positions 0..10

    4. Reverse String II (LeetCode 541)

    Problem. Given a string s and an integer k, for every 2k characters counting from the start, reverse the first k characters. If fewer than k characters remain, reverse all of them. If between k and 2k characters remain, reverse the first k and leave the rest.

    Idea

    Process the string in chunks of 2k. In each chunk, reverse only the first half (the first k characters). The second half is left untouched.

    Chunk-Based Reversal

    For i = 0, 2k, 4k, 6k, ... reverse the range [i, min(i+k, n)). The min handles the case where fewer than k characters remain in the final chunk.

    C++ Code

    class Solution {
public:
    string reverseStr(string s, int k) {
        int n = s.size();
        for (int i = 0; i < n; i += 2 * k) {
            int left = i;
            int right = min(i + k - 1, n - 1);
            while (left < right) {
                swap(s[left], s[right]);
                left++;
                right--;
            }
        }
        return s;
    }
};

    Time: O(n). Every character is part of at most one reversal.
    Space: O(1).

    Worked Example

    Input: s = "abcdefg", k = 2.

    ChunkRangeActionResult
    i = 0[0, 3]Reverse [0, 1]"bacdefg"
    i = 4[4, 6]Reverse [4, 5]"bacdfeg"

    Output: "bacdfeg"

    Common mistake: Forgetting the boundary check when fewer than k characters remain. Always use min(i + k - 1, n - 1) for the right boundary to avoid out-of-bounds access.

    5. Rotate Array (LeetCode 189)

    Problem. Given an integer array nums, rotate the array to the right by k steps. For example, [1,2,3,4,5,6,7] rotated by 3 gives [5,6,7,1,2,3,4].

    The Naive Approach

    Rotate one position at a time, k times. Each single rotation is O(n), so the total is O(n*k). For large arrays and large k this is far too slow.

    The Three-Reverse Trick

    There is an elegant O(n) time, O(1) space solution that uses exactly three reversals:

    Reverse the entire array

    [1, 2, 3, 4, 5, 6, 7] becomes [7, 6, 5, 4, 3, 2, 1]

    Reverse the first k elements

    [7, 6, 5, 4, 3, 2, 1] becomes [5, 6, 7, 4, 3, 2, 1] (reversed indices 0 to k-1)

    Reverse the remaining n-k elements

    [5, 6, 7, 4, 3, 2, 1] becomes [5, 6, 7, 1, 2, 3, 4] (reversed indices k to n-1)

    Proof: Why Three Reversals Work

    Let the array be A = [a0, a1, ..., an-k-1, an-k, ..., an-1]. Split it into two parts:

    • L = first (n-k) elements: [a0, ..., an-k-1]
    • R = last k elements: [an-k, ..., an-1]

    So A = L + R (concatenation). We want R + L.

    Algebraic argument

    Reversing the whole array: rev(A) = rev(L + R) = rev(R) + rev(L).

    Now reverse the first k elements (which is rev(R)): we get R + rev(L).

    Finally reverse the last n-k elements (which is rev(L)): we get R + L.

    This is exactly the right rotation by k positions.

    The key identity is: reversing a concatenation reverses the order of the parts and reverses each part internally. The two subsequent sub-reversals undo the internal reversal of each part, leaving only the reordering.

    C++ Code

    class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;  // handle k >= n
        if (k == 0) return;

        reverse(nums.begin(), nums.end());           // reverse all
        reverse(nums.begin(), nums.begin() + k);     // reverse first k
        reverse(nums.begin() + k, nums.end());       // reverse rest
    }
};

    Time: O(n). Three passes, each at most O(n).
    Space: O(1). Only pointer variables.

    Do not forget k %= n. If k equals n, the array is unchanged. If k is larger than n, rotating by k is the same as rotating by k mod n. Omitting this causes out-of-bounds errors.

    6. Left Rotate by d Positions

    Problem. Rotate the array to the left by d positions. For example, [1,2,3,4,5] rotated left by 2 gives [3,4,5,1,2].

    Connection to Right Rotation

    A left rotation by d is the same as a right rotation by n - d. So you can use the exact same three-reverse trick with k = n - d.

    Alternatively, there is a direct three-reverse approach for left rotation that reverses in a slightly different order:

    Reverse the first d elements

    [1, 2, 3, 4, 5] becomes [2, 1, 3, 4, 5]

    Reverse the remaining n-d elements

    [2, 1, 3, 4, 5] becomes [2, 1, 5, 4, 3]

    Reverse the entire array

    [2, 1, 5, 4, 3] becomes [3, 4, 5, 1, 2]

    C++ Code

    void leftRotate(vector<int>& nums, int d) {
    int n = nums.size();
    d %= n;
    if (d == 0) return;

    reverse(nums.begin(), nums.begin() + d);    // reverse first d
    reverse(nums.begin() + d, nums.end());      // reverse rest
    reverse(nums.begin(), nums.end());           // reverse all
}

    Time: O(n). Space: O(1).

    Equivalence Table

    OperationThree-Reverse Order
    Right rotate by krev(all), rev(first k), rev(last n-k)
    Left rotate by drev(first d), rev(last n-d), rev(all)
    Left rotate by d (via right)Right rotate by (n - d)
    Both orderings work because the three reversals are an involution group. You can also prove the left-rotate version by the same algebraic argument: if A = L + R, then rev(rev(L) + rev(R)) = rev(rev(R)) + rev(rev(L)) = R + L.

    7. Connection Between Reversal and Cyclic Shifts

    A cyclic shift (rotation) moves every element by a fixed offset, wrapping around at the boundary. Formally, a right cyclic shift by k maps element at index i to index (i + k) mod n.

    The Key Insight

    Any cyclic shift can be decomposed into exactly three reversals. This is not a coincidence. It follows from a deeper fact in group theory: the reversal operation (an involution) and the cyclic shift operation together generate the dihedral group Dn, which describes all symmetries of a regular n-gon.

    Reversal Decomposition of Cyclic Shifts

    For any array of length n and any shift amount k (0 ≤ k < n), the cyclic right shift by k equals:

    shift(A, k) = rev(rev(A[0..n-k-1]) + rev(A[n-k..n-1]))

    Which simplifies to the three-reverse trick: reverse all, reverse first k, reverse last n-k.

    Why Exactly Three?

    One reversal can only produce the mirror image of a sequence. A single reversal cannot produce an arbitrary cyclic shift (except the trivial shift by 0 or the full reversal). Two reversals can produce some shifts but not all. Three reversals are both necessary and sufficient to express any cyclic shift as a composition of reversals.

    The GCD Connection (Juggling Algorithm)

    There is an alternative O(n) rotation approach called the juggling algorithm that moves elements in cycles of length n / gcd(n, k). Each cycle visits gcd(n, k) starting positions. This is mathematically elegant but harder to implement correctly and has worse cache behavior than the three-reverse trick.

    Comparison of Rotation Methods

    MethodTimeSpacePassesCache Friendly
    Brute force (shift one at a time)O(n * k)O(1)k passesYes
    Extra arrayO(n)O(n)2 passesYes
    Three-reverse trickO(n)O(1)3 passesYes
    Juggling (GCD cycles)O(n)O(1)1 passNo
    In practice, use the three-reverse trick. It is the easiest to code, the easiest to verify, and has excellent cache locality because each pass is a linear scan. The juggling algorithm hops around the array and causes cache misses on large inputs.

    Beyond Arrays: Strings and Linked Lists

    The three-reverse technique works on any sequence that supports efficient reversal:

    • Strings: Use exactly the same algorithm. In C++ you can use std::reverse on iterators into a std::string.
    • Linked lists: Reversing a singly linked list is O(n). So rotating a linked list by k also takes O(n) time with the same three-reverse strategy, though in practice it is simpler to just relink the tail to the head.
    • Deques and circular buffers: Rotation is a native O(1) operation since you simply adjust the start/end pointers.
