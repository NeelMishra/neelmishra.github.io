    Hashing Patterns

    March 2026 · 30 min read

    Once you understand how hash tables work, the next step is recognizing where to apply them. Hashing shows up in a huge number of array problems, usually as the tool that drops a brute-force $O(n^2)$ solution down to $O(n)$. This post covers six essential patterns, each with an explanation, a step-through visual, and complete C++ code.

    Pattern 1: Frequency Counting

    Core idea: Count how many times each element appears using a hash map. This is the simplest and most common use of hashing.

    Problem: Given an array, find the element that appears more than $\lfloor n/2 \rfloor$ times (the majority element). (LeetCode 169)

    The brute force approach checks every element against every other element: $O(n^2)$. With a hash map, you count frequencies in a single pass and then scan the map for the majority.

    int majorityElement(std::vector<int>& nums) {
    std::unordered_map<int, int> freq;
    int n = nums.size();
    for (int x : nums) {
        freq[x]++;
        if (freq[x] > n / 2)
            return x;
    }
    return -1;  // guaranteed to exist per problem statement
}

    Complexity: $O(n)$ time, $O(n)$ space.

    Pattern template: Whenever a problem asks "how many times does X appear?" or "find the most/least frequent element," your first instinct should be a frequency map. This covers problems like first unique character, top K frequent elements, sort by frequency, and finding duplicates.

    Pattern 2: Anagram Detection

    Core idea: Two strings are anagrams if and only if they have the same character frequencies.

    Problem: Given two strings s and t, determine if t is an anagram of s. (LeetCode 242)

    Build a frequency map for s, then decrement for each character in t. If every count returns to zero, they are anagrams.

    bool isAnagram(std::string s, std::string t) {
    if (s.size() != t.size()) return false;
    std::unordered_map<char, int> freq;
    for (char c : s) freq[c]++;
    for (char c : t) {
        freq[c]--;
        if (freq[c] < 0) return false;
    }
    return true;
}

    Complexity: $O(n)$ time, $O(1)$ space (at most 26 lowercase letters).

    Optimization: For lowercase English letters only, use an int freq[26] array instead of a hash map. Array access is faster than hash map access, and 26 slots use negligible memory.

    Pattern 3: Subarray Sum Equals K

    Core idea: Combine prefix sums with a hash map. For each index, ask "how many previous prefix sums equal currentSum - k?"

    Problem: Given an array and an integer k, find the total number of contiguous subarrays whose sum equals k. (LeetCode 560)

    If the prefix sum at index $j$ is pre[j] and the prefix sum at some earlier index $i$ was pre[i], then the subarray arr[i+1..j] has sum pre[j] - pre[i]. We want this to equal k, so we need pre[i] = pre[j] - k.

    A hash map tracks how many times each prefix sum has occurred. At each step, look up currentSum - k in the map. Initialize the map with {0: 1} to handle subarrays starting from index 0.

    int subarraySum(std::vector<int>& nums, int k) {
    std::unordered_map<int, int> prefixCount;
    prefixCount[0] = 1;
    int count = 0, curSum = 0;
    for (int num : nums) {
        curSum += num;
        if (prefixCount.count(curSum - k))
            count += prefixCount[curSum - k];
        prefixCount[curSum]++;
    }
    return count;
}

    Complexity: $O(n)$ time, $O(n)$ space.

    Pattern 4: Longest Consecutive Sequence

    Core idea: Use a hash set for $O(1)$ membership checks. Only start counting from the beginning of a sequence (where num - 1 is not in the set).

    Problem: Given an unsorted array, find the length of the longest consecutive elements sequence. Must run in $O(n)$. (LeetCode 128)

    Sorting gives $O(n \log n)$. To achieve $O(n)$, insert all elements into a hash set. Then for each element, check if it is the start of a sequence (i.e., num - 1 is not in the set). If it is, count how many consecutive numbers follow.

    int longestConsecutive(std::vector<int>& nums) {
    std::unordered_set<int> numSet(nums.begin(), nums.end());
    int best = 0;
    for (int num : numSet) {
        if (!numSet.count(num - 1)) {   // start of a sequence
            int cur = num;
            int length = 1;
            while (numSet.count(cur + 1)) {
                cur++;
                length++;
            }
            best = std::max(best, length);
        }
    }
    return best;
}

    Complexity: $O(n)$ time, $O(n)$ space. Each element is visited at most twice (once in the outer loop, once in the inner while loop).

    Why is this O(n)? The "check if sequence start" condition ensures the inner while loop only runs for actual sequence starts. Each element is part of exactly one sequence, so the total work across all iterations of the inner loop is $O(n)$.

    Pattern 5: Group Anagrams

    Core idea: Use a canonical form (sorted string or frequency signature) as the hash map key to group words that are anagrams of each other.

    Problem: Given an array of strings, group the anagrams together. (LeetCode 49)

    Two words are anagrams if they have the same characters in the same frequencies. If we sort each word, anagrams produce the same sorted string. Use that sorted string as the key in a hash map, and the value is the list of original words.

    std::vector<std::vector<std::string>> groupAnagrams(
        std::vector<std::string>& strs) {
    std::unordered_map<std::string, std::vector<std::string>> groups;
    for (auto& s : strs) {
        std::string key = s;
        std::sort(key.begin(), key.end());
        groups[key].push_back(s);
    }
    std::vector<std::vector<std::string>> result;
    for (auto& [key, group] : groups)
        result.push_back(group);
    return result;
}

    Complexity: $O(n \cdot k \log k)$ where $n$ is the number of strings and $k$ is the maximum string length (due to sorting each string). Space is $O(n \cdot k)$.

    Frequency-key alternative

    Instead of sorting, you can build a frequency string as the key. For each word, count characters and encode as "a1b0c0...z0". This avoids sorting and runs in $O(n \cdot k)$.

    std::string freqKey(const std::string& s) {
    int count[26] = {};
    for (char c : s) count[c - 'a']++;
    std::string key;
    for (int i = 0; i < 26; i++) {
        key += std::string(1, 'a' + i);
        key += std::to_string(count[i]);
    }
    return key;
}

    Pattern 6: Two Sum Variants

    The Two Sum problem from the fundamentals post is the gateway to a family of hash-based pair-finding problems. Here are the key variants.

    Variant A: Two Sum with indices

    The classic version. Return the indices of two numbers that add up to the target. Already covered in the fundamentals post.

    Variant B: Two Sum with sorted input

    If the array is sorted, you do not need a hash map. Use two pointers (one at each end) and move them inward. This uses $O(1)$ space instead of $O(n)$.

    std::vector<int> twoSumSorted(std::vector<int>& nums, int target) {
    int lo = 0, hi = nums.size() - 1;
    while (lo < hi) {
        int sum = nums[lo] + nums[hi];
        if (sum == target) return {lo, hi};
        else if (sum < target) lo++;
        else hi--;
    }
    return {};
}

    Variant C: Two Sum with duplicates (count pairs)

    Problem: Count the number of pairs (i, j) with i < j such that nums[i] + nums[j] == target.

    Build a frequency map as you scan. For each element x, the number of valid pairs it forms is freq[target - x] (the count of complements seen so far).

    int countPairs(std::vector<int>& nums, int target) {
    std::unordered_map<int, int> freq;
    int count = 0;
    for (int x : nums) {
        int complement = target - x;
        if (freq.count(complement))
            count += freq[complement];
        freq[x]++;
    }
    return count;
}

    Variant D: Three Sum

    Problem: Find all unique triplets that sum to zero. (LeetCode 15)

    Fix one element, then reduce to Two Sum on the remaining array. Sort first to handle duplicates by skipping repeated values.

    std::vector<std::vector<int>> threeSum(std::vector<int>& nums) {
    std::sort(nums.begin(), nums.end());
    std::vector<std::vector<int>> result;
    int n = nums.size();
    for (int i = 0; i < n - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue;
        int lo = i + 1, hi = n - 1;
        while (lo < hi) {
            int sum = nums[i] + nums[lo] + nums[hi];
            if (sum == 0) {
                result.push_back({nums[i], nums[lo], nums[hi]});
                while (lo < hi && nums[lo] == nums[lo + 1]) lo++;
                while (lo < hi && nums[hi] == nums[hi - 1]) hi--;
                lo++; hi--;
            } else if (sum < 0) lo++;
            else hi--;
        }
    }
    return result;
}

    Complexity: $O(n^2)$ time, $O(1)$ extra space (excluding output).

    When to Reach for a Hash Map

    Use this checklist to identify hashing problems:

    • Need O(1) lookups? If you are checking "have I seen this before?" or "does this complement exist?", a hash map or hash set is the answer.
    • Counting occurrences? Frequency map.
    • Grouping by some property? Use the property as the key (sorted string, frequency signature, remainder, etc.).
    • Prefix sum + lookup? If you need to find subarrays with a specific sum, combine prefix sums with a hash map.
    • Reducing from O(n^2) to O(n)? If the brute force checks all pairs, hashing can often eliminate one of the two nested loops.

    Key Takeaways

    • Frequency counting is the simplest hash pattern: one pass, count everything.
    • Anagram detection reduces to comparing character frequencies via a hash map.
    • Subarray sum equals K combines prefix sums with a hash map for $O(n)$ counting.
    • Longest consecutive sequence uses a hash set and only starts counting from sequence boundaries.
    • Group anagrams uses a canonical key (sorted string or frequency string) to partition words.
    • Two Sum and its variants (count pairs, three sum) are the foundation for all pair-finding hash problems.
    • The universal signal: if brute force is $O(n^2)$ because of a nested lookup, a hash map can often make it $O(n)$.
