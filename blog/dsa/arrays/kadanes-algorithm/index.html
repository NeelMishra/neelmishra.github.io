    Kadane's Algorithm

    March 2026 · 18 min read

    Given an array of integers, find the contiguous subarray with the largest sum. This is the Maximum Subarray Problem, one of the most fundamental questions in algorithm design. It appears in interviews, competitive programming, and real-world applications like financial analysis (finding the best period of stock gains) and image processing (maximum brightness rectangle).

    We will build from brute force all the way to Kadane's elegant O(n) solution, then look at important variants.

    The Problem

    Maximum Subarray Sum: Given an integer array arr[0..n-1], find indices $i$ and $j$ ($0 \le i \le j < n$) such that $\sum_{k=i}^{j} arr[k]$ is maximized. Return this maximum sum.

    A few important notes:

    • The subarray must be contiguous (consecutive elements).
    • The array can contain negative numbers. If all elements are negative, the answer is the single least-negative element.
    • The subarray must contain at least one element.

    Example

    Consider the array [-2, 1, -3, 4, -1, 2, 1, -5, 4].

    The maximum subarray is [4, -1, 2, 1] with sum 6.

    Brute Force: O(n^3)

    The most straightforward approach: enumerate every possible subarray, compute its sum, and track the maximum.

    Three Nested Loops

    For each starting index $i$, for each ending index $j \ge i$, compute $\sum_{k=i}^{j} arr[k]$ with a third loop. Compare against the running maximum.

    int maxSubarrayBruteN3(vector<int>& arr) {
    int n = arr.size();
    int maxSum = INT_MIN;
    for (int i = 0; i < n; i++) {
        for (int j = i; j < n; j++) {
            int sum = 0;
            for (int k = i; k <= j; k++) {
                sum += arr[k];
            }
            maxSum = max(maxSum, sum);
        }
    }
    return maxSum;
}

    Analysis

    The number of subarrays is $\binom{n}{2} + n = \frac{n(n+1)}{2}$. For each, we spend up to $O(n)$ computing the sum. Total work:

    $$T(n) = \sum_{i=0}^{n-1}\sum_{j=i}^{n-1}(j - i + 1) = \frac{n(n+1)(n+2)}{6} = O(n^3)$$

    This is far too slow for competitive programming constraints (typically $n \le 10^5$).

    Optimized Brute Force: O(n^2)

    The inner sum loop is wasteful. When we extend the subarray from $[i..j]$ to $[i..j{+}1]$, we only add one element. We can maintain a running sum instead of recomputing from scratch.

    Eliminate the Third Loop

    For each starting index $i$, maintain sum and extend it by adding arr[j] as $j$ increases. This removes the innermost loop entirely.

    int maxSubarrayBruteN2(vector<int>& arr) {
    int n = arr.size();
    int maxSum = INT_MIN;
    for (int i = 0; i < n; i++) {
        int sum = 0;
        for (int j = i; j < n; j++) {
            sum += arr[j];
            maxSum = max(maxSum, sum);
        }
    }
    return maxSum;
}

    Analysis

    $$T(n) = \sum_{i=0}^{n-1}(n - i) = \frac{n(n+1)}{2} = O(n^2)$$

    Better, but still too slow for $n = 10^5$. We need a fundamentally different approach.

    Prefix Sum Perspective

    Before diving into Kadane's, let us see the problem through prefix sums. Define $P[k] = \sum_{i=0}^{k-1} arr[i]$ with $P[0] = 0$. Then the sum of subarray $[i..j]$ is:

    $$\text{sum}(i, j) = P[j+1] - P[i]$$

    The maximum subarray sum equals:

    $$\max_{0 \le i \le j < n} (P[j+1] - P[i]) = \max_{j} \left( P[j+1] - \min_{i \le j} P[i] \right)$$

    If we track the minimum prefix sum seen so far as we scan left to right, we get an O(n) algorithm. This is essentially what Kadane's does, expressed differently.

    Kadane's Algorithm: O(n)

    Kadane's algorithm is a dynamic programming approach. The key insight: at each position $j$, the maximum subarray ending at $j$ is either:

    • The element $arr[j]$ alone (start a new subarray), or
    • The element $arr[j]$ appended to the best subarray ending at $j{-}1$.
    Recurrence: Let $\text{currentSum}(j)$ be the maximum subarray sum ending at index $j$: $$\text{currentSum}(j) = \max\bigl(arr[j],\;\text{currentSum}(j{-}1) + arr[j]\bigr)$$ The answer is $\max_{j} \text{currentSum}(j)$.

    Equivalently: if currentSum drops below zero, we reset it (starting fresh is better than dragging a negative prefix). Otherwise, we keep extending.

    Why Does This Work?

    A subarray ending at position $j$ either includes position $j{-}1$ or it does not. If the best sum ending at $j{-}1$ is negative, including it would only reduce the total, so we discard it. If it is non-negative, extending is always at least as good as starting fresh. This greedy choice is optimal because it considers every possible ending position.

    C++ Implementation

    int kadane(vector<int>& arr) {
    int currentSum = arr[0];
    int maxSum = arr[0];
    for (int i = 1; i < (int)arr.size(); i++) {
        currentSum = max(arr[i], currentSum + arr[i]);
        maxSum = max(maxSum, currentSum);
    }
    return maxSum;
}

    Returning the Subarray Indices

    tuple<int,int,int> kadaneWithIndices(vector<int>& arr) {
    int currentSum = arr[0], maxSum = arr[0];
    int start = 0, end = 0, tempStart = 0;
    for (int i = 1; i < (int)arr.size(); i++) {
        if (arr[i] > currentSum + arr[i]) {
            currentSum = arr[i];
            tempStart = i;
        } else {
            currentSum += arr[i];
        }
        if (currentSum > maxSum) {
            maxSum = currentSum;
            start = tempStart;
            end = i;
        }
    }
    return {maxSum, start, end};
}

    Detailed Walkthrough

    Let us trace through the algorithm manually on [-2, 1, -3, 4, -1, 2, 1, -5, 4]:

    Initialize

    currentSum = arr[0] = -2, maxSum = -2.

    The best subarray ending at index 0 is just [-2].

    arr[1] = 1

    currentSum = max(1, -2 + 1) = max(1, -1) = 1. Starting fresh is better.

    maxSum = max(-2, 1) = 1. Best subarray: [1].

    arr[2] = -3

    currentSum = max(-3, 1 + (-3)) = max(-3, -2) = -2. Extending is less bad.

    maxSum = max(1, -2) = 1. Best subarray still [1].

    arr[3] = 4

    currentSum = max(4, -2 + 4) = max(4, 2) = 4. Starting fresh wins.

    maxSum = max(1, 4) = 4. New best: [4].

    arr[4] = -1

    currentSum = max(-1, 4 + (-1)) = max(-1, 3) = 3. Extend.

    maxSum = max(4, 3) = 4. Best subarray still [4].

    arr[5] = 2

    currentSum = max(2, 3 + 2) = max(2, 5) = 5. Extend.

    maxSum = max(4, 5) = 5. New best: [4, -1, 2].

    arr[6] = 1

    currentSum = max(1, 5 + 1) = max(1, 6) = 6. Extend.

    maxSum = max(5, 6) = 6. New best: [4, -1, 2, 1].

    arr[7] = -5

    currentSum = max(-5, 6 + (-5)) = max(-5, 1) = 1. Extend (1 > -5).

    maxSum = max(6, 1) = 6. Best unchanged.

    arr[8] = 4

    currentSum = max(4, 1 + 4) = max(4, 5) = 5. Extend.

    maxSum = max(6, 5) = 6. Final answer: 6. Subarray: [4, -1, 2, 1].

    Why Kadane's Is Correct

    Define $\text{OPT}(j)$ as the maximum sum of any subarray ending at index $j$. Any subarray ending at $j$ either:

    1. Contains only element $j$: sum = $arr[j]$.
    2. Extends a subarray ending at $j{-}1$: sum = $\text{OPT}(j{-}1) + arr[j]$.

    Therefore $\text{OPT}(j) = \max(arr[j], \text{OPT}(j{-}1) + arr[j])$, which is exactly the recurrence Kadane's computes. The global maximum is $\max_j \text{OPT}(j)$, which equals maxSum.

    This is a valid dynamic programming solution because every subarray must end at some index, and we consider all ending positions.

    Edge Cases

    • All negative: [-3, -5, -1, -8]. Kadane's correctly returns -1 (the least negative element).
    • All positive: [2, 3, 1, 5]. The entire array is the answer: sum = 11.
    • Single element: [7]. Returns 7.
    • Contains zero: [0, -1, 3]. Returns 3.
    • Large negative followed by large positive: [-100, 50]. Returns 50 (starts fresh).

    Divide and Conquer: O(n log n)

    For completeness, here is the divide and conquer approach. Split the array at the midpoint. The maximum subarray either lies entirely in the left half, entirely in the right half, or crosses the midpoint.

    int maxCrossing(vector<int>& arr, int lo, int mid, int hi) {
    int leftSum = INT_MIN, sum = 0;
    for (int i = mid; i >= lo; i--) {
        sum += arr[i];
        leftSum = max(leftSum, sum);
    }
    int rightSum = INT_MIN;
    sum = 0;
    for (int i = mid + 1; i <= hi; i++) {
        sum += arr[i];
        rightSum = max(rightSum, sum);
    }
    return leftSum + rightSum;
}

int maxSubarrayDC(vector<int>& arr, int lo, int hi) {
    if (lo == hi) return arr[lo];
    int mid = lo + (hi - lo) / 2;
    int leftMax = maxSubarrayDC(arr, lo, mid);
    int rightMax = maxSubarrayDC(arr, mid + 1, hi);
    int crossMax = maxCrossing(arr, lo, mid, hi);
    return max({leftMax, rightMax, crossMax});
}

    Recurrence: $T(n) = 2T(n/2) + O(n) = O(n \log n)$ by the Master Theorem. This is instructive but strictly worse than Kadane's for this problem.

    Variants Overview

    Kadane's idea generalizes to several important variants. Each is covered in detail on the Variants page.

    Maximum Circular Subarray Sum

    The array is circular: the subarray can wrap around from the end to the beginning. The trick: the maximum circular subarray is either a normal (non-wrapping) subarray, or equals totalSum - minSubarraySum. Apply Kadane's twice: once for max, once for min.

    Maximum Product Subarray

    Instead of sum, find the contiguous subarray with the largest product. Negative numbers complicate things: a negative times a negative becomes positive. Track both the running maximum and running minimum product at each position.

    Minimum Subarray Sum

    Flip the signs: negate every element, run Kadane's, negate the result. Or modify the recurrence directly:

    $$\text{currentMin}(j) = \min(arr[j], \text{currentMin}(j{-}1) + arr[j])$$

    Maximum Subarray with at Most K Deletions

    Allow removing up to $k$ elements from the chosen subarray to maximize the sum. Requires a 2D DP state: $dp[i][j]$ = best subarray ending at index $i$ with exactly $j$ deletions used.

    Summary

    • The maximum subarray problem asks for the largest contiguous sum in an array.
    • Brute force takes $O(n^3)$ or $O(n^2)$. Divide and conquer achieves $O(n \log n)$.
    • Kadane's algorithm solves it in $O(n)$ with $O(1)$ space by maintaining currentSum and resetting when it drops below the current element.
    • The core recurrence: $\text{currentSum}(j) = \max(arr[j], \text{currentSum}(j{-}1) + arr[j])$.
    • Kadane's idea extends to circular arrays, product subarrays, minimum subarrays, and deletion variants.
