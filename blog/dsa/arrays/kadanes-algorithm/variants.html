    Kadane's Variants

    March 2026 · 22 min read

    Kadane's algorithm is not limited to finding the maximum contiguous subarray sum. The same "extend or restart" idea adapts to several important variants. This page covers three of them in depth: minimum subarray sum, maximum circular subarray sum, and maximum product subarray. Each includes a visual walkthrough, code, and analysis.

    Variant 1: Minimum Subarray Sum

    Problem: Given an integer array, find the contiguous subarray with the smallest sum. Return that sum.

    Approach

    Mirror Kadane's algorithm. Instead of tracking the maximum subarray ending at each position, track the minimum. At each index $j$:

    $$\text{currentMin}(j) = \min\bigl(arr[j],\;\text{currentMin}(j{-}1) + arr[j]\bigr)$$

    If currentMin becomes positive and the current element is smaller, we restart. The global answer is $\min_j \text{currentMin}(j)$.

    Alternatively, negate every element, run standard Kadane's, and negate the result. Both approaches are $O(n)$ time, $O(1)$ space.

    C++ Implementation

    int minSubarraySum(vector<int>& arr) {
    int currentMin = arr[0];
    int globalMin = arr[0];
    for (int i = 1; i < (int)arr.size(); i++) {
        currentMin = min(arr[i], currentMin + arr[i]);
        globalMin = min(globalMin, currentMin);
    }
    return globalMin;
}

    Returning Indices

    tuple<int,int,int> minSubarrayWithIndices(vector<int>& arr) {
    int currentMin = arr[0], globalMin = arr[0];
    int start = 0, end = 0, tempStart = 0;
    for (int i = 1; i < (int)arr.size(); i++) {
        if (arr[i] < currentMin + arr[i]) {
            currentMin = arr[i];
            tempStart = i;
        } else {
            currentMin += arr[i];
        }
        if (currentMin < globalMin) {
            globalMin = currentMin;
            start = tempStart;
            end = i;
        }
    }
    return {globalMin, start, end};
}

    Complexity: $O(n)$ time, $O(1)$ space. Identical to standard Kadane's.

    Variant 2: Maximum Circular Subarray Sum

    Problem (LeetCode 918): Given a circular integer array, find the maximum sum of a non-empty subarray. The array wraps around: after the last element, the next element is the first.

    Key Insight

    A subarray of a circular array is either:

    1. Non-wrapping: a normal contiguous subarray. Kadane's finds this.
    2. Wrapping: takes elements from both the end and the beginning of the array.

    The wrapping case is the complement of a non-wrapping subarray. If we remove a middle portion (a contiguous subarray) from the total, what remains wraps around. To maximize the wrap, we minimize the middle portion.

    Circular Maximum

    $$\text{circularMax} = \max\bigl(\text{kadaneMax},\;\text{totalSum} - \text{kadaneMin}\bigr)$$

    Where kadaneMax is the standard maximum subarray sum, kadaneMin is the minimum subarray sum, and totalSum is $\sum arr[i]$.

    Edge case: if all elements are negative, totalSum - kadaneMin = 0 (the minimum subarray is the entire array). In this case, we must use kadaneMax (the least-negative single element), since the subarray must be non-empty.

    Visual Walkthrough

    Step through the circular subarray algorithm on [5, -3, 5]. The non-wrapping max is 7 (just [5, -3, 5]), but the wrapping subarray [5, 5] (wrapping around, skipping -3) gives sum 10.

    Click "Step" to walk through the circular max subarray on [5, -3, 5]. Total = 7.

    C++ Implementation

    int maxSubarrayCircular(vector<int>& arr) {
    int n = arr.size();
    int totalSum = 0;
    int kadaneMax = arr[0], currentMax = arr[0];
    int kadaneMin = arr[0], currentMin = arr[0];

    totalSum += arr[0];
    for (int i = 1; i < n; i++) {
        totalSum += arr[i];
        currentMax = max(arr[i], currentMax + arr[i]);
        kadaneMax = max(kadaneMax, currentMax);
        currentMin = min(arr[i], currentMin + arr[i]);
        kadaneMin = min(kadaneMin, currentMin);
    }

    // If all elements are negative, kadaneMin == totalSum
    // and the wrapping case gives 0 (empty subarray not allowed)
    if (kadaneMax < 0) return kadaneMax;

    return max(kadaneMax, totalSum - kadaneMin);
}

    Why the Edge Case Matters

    When all elements are negative, kadaneMin equals the entire array sum (the minimum subarray is everything). Then totalSum - kadaneMin = 0, which represents an empty subarray. Since the problem requires at least one element, we return kadaneMax instead.

    Complexity: $O(n)$ time (single pass computing four values), $O(1)$ space.

    Detailed Example

    Array: [8, -1, 3, 4], totalSum = 14.

    • kadaneMax = 14 (entire array).
    • kadaneMin = -1 (element at index 1).
    • Wrapping sum = 14 - (-1) = 15. This corresponds to [3, 4, 8] (wrapping).
    • Answer = max(14, 15) = 15.

    Array: [-3, -2, -5], totalSum = -10.

    • kadaneMax = -2.
    • kadaneMin = -10 (entire array).
    • Wrapping sum = -10 - (-10) = 0. But subarray must be non-empty.
    • Since kadaneMax < 0, answer = -2.

    Variant 3: Maximum Product Subarray

    Problem (LeetCode 152): Given an integer array, find the contiguous subarray with the largest product. The array can contain positive, negative, and zero values.

    Why Standard Kadane's Fails

    With sums, a negative prefix is always bad. With products, a negative prefix can become very positive if we multiply by another negative number. Consider [-2, 3, -4]:

    • The product of the entire array is $(-2) \times 3 \times (-4) = 24$.
    • If we discarded the negative prefix -2, we would miss this.

    The fix: track both the maximum product and the minimum product ending at each position. The minimum (most negative) product can flip to become the maximum after multiplying by a negative element.

    Dual Tracking

    At each index $j$:

    $$\text{maxProd}(j) = \max\bigl(arr[j],\;\text{maxProd}(j{-}1) \times arr[j],\;\text{minProd}(j{-}1) \times arr[j]\bigr)$$

    $$\text{minProd}(j) = \min\bigl(arr[j],\;\text{maxProd}(j{-}1) \times arr[j],\;\text{minProd}(j{-}1) \times arr[j]\bigr)$$

    The global answer is $\max_j \text{maxProd}(j)$.

    Detailed Trace

    Array: [2, 3, -2, 4, -1].

    Initialize

    maxProd = 2, minProd = 2, globalMax = 2.

    arr[1] = 3

    Candidates: 3, 2*3=6, 2*3=6. maxProd = 6, minProd = 3.

    globalMax = 6. Current best product: [2, 3] = 6.

    arr[2] = -2

    Candidates: -2, 6*(-2)=-12, 3*(-2)=-6. maxProd = -2, minProd = -12.

    globalMax = 6. The negative element is destructive. We restart at -2. But we keep minProd = -12 (it might flip later).

    arr[3] = 4

    Candidates: 4, (-2)*4=-8, (-12)*4=-48. maxProd = 4, minProd = -48.

    globalMax = 6. Still restart because extending from -2 gives -8.

    arr[4] = -1

    Candidates: -1, 4*(-1)=-4, (-48)*(-1)=48. maxProd = 48, minProd = -4.

    globalMax = 48. The minimum product flipped! (-48) * (-1) = 48 corresponds to subarray [2, 3, -2, 4, -1].

    Final answer: 48, achieved by the entire array.

    C++ Implementation

    int maxProduct(vector<int>& arr) {
    int n = arr.size();
    int maxProd = arr[0], minProd = arr[0], globalMax = arr[0];

    for (int i = 1; i < n; i++) {
        // If arr[i] is negative, swapping max and min before
        // multiplying simplifies the logic
        if (arr[i] < 0) swap(maxProd, minProd);

        maxProd = max(arr[i], maxProd * arr[i]);
        minProd = min(arr[i], minProd * arr[i]);
        globalMax = max(globalMax, maxProd);
    }
    return globalMax;
}

    Why the Swap Trick Works

    When arr[i] is negative, multiplying a large positive number makes it very negative, and multiplying a very negative number makes it very positive. By swapping maxProd and minProd before the multiplication, we ensure that:

    • maxProd * arr[i] is the old minimum times the negative (which could be large positive).
    • minProd * arr[i] is the old maximum times the negative (which becomes very negative).

    This elegantly handles the sign flip without complex conditional logic.

    Handling Zeros

    Zeros reset both maxProd and minProd to 0, effectively restarting the subarray. This is correct: any subarray containing zero has product zero, and we start fresh at the next element.

    Complexity: $O(n)$ time, $O(1)$ space.

    Comparison of Variants

    VariantKey ModificationTimeSpace
    Maximum Subarray Sum Standard Kadane's $O(n)$ $O(1)$
    Minimum Subarray Sum Replace max with min $O(n)$ $O(1)$
    Maximum Circular max(kadaneMax, total - kadaneMin) $O(n)$ $O(1)$
    Maximum Product Track both max and min product $O(n)$ $O(1)$

    Summary

    • Minimum subarray: flip max to min in the recurrence (or negate, run Kadane's, negate).
    • Circular subarray: the wrapping maximum equals total sum minus the minimum subarray. Handle the all-negative edge case.
    • Product subarray: track both max and min products because a negative times a negative gives a positive.
    • All three variants run in $O(n)$ time and $O(1)$ space, preserving Kadane's efficiency.
