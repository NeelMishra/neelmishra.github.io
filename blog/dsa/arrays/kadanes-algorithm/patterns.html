    Kadane's: Competition Patterns

    March 2026 · 24 min read

    Kadane's algorithm is more than a standalone technique. In competitive programming and interviews, it appears as a building block inside larger problems. This page covers how to recognize when Kadane's applies, how it combines with prefix sums, and its powerful 2D extension for finding the maximum-sum rectangle in a matrix.

    Recognizing Kadane's Problems

    Not every "maximum" or "subarray" problem needs Kadane's. Here is a decision framework for quickly identifying when the technique applies.

    Kadane's Applies When

    The problem asks for a contiguous subarray that optimizes a sum or product.

    The array may contain negative values (otherwise the answer is trivially the entire array).

    You need an optimal subarray ending at each position, and the global optimum is the best among these.

    Kadane's Does NOT Apply When

    The subarray has a fixed size (use sliding window instead).

    You need non-contiguous elements (use DP or greedy on subsequences).

    The constraint is on the subarray length, not its aggregate value.

    The problem involves multiple subarrays or partitions.

    Quick Test: Can the problem be phrased as "at index $j$, should I extend the current subarray or start fresh"? If yes, Kadane's (or a variant) likely works.

    Common Disguises

    Kadane's often hides behind different problem statements. Here are patterns that reduce to maximum subarray:

    Best Time to Buy and Sell Stock (LeetCode 121)

    Compute the array of daily price changes (differences). The maximum subarray sum of these differences is the maximum profit from one transaction. Why? A buy at day $i$ and sell at day $j$ yields profit equal to the sum of changes from day $i{+}1$ to day $j$.

    Maximum Absolute Sum (LeetCode 1749)

    Find the subarray whose absolute sum is largest. Run Kadane's for max and for min. The answer is $\max(|\text{kadaneMax}|, |\text{kadaneMin}|)$.

    Maximum Sum with One Deletion (LeetCode 1186)

    Run forward Kadane's to get maxEndingAt[i] and backward Kadane's to get maxStartingAt[i]. Then for each index $i$, the best subarray that skips element $i$ is maxEndingAt[i-1] + maxStartingAt[i+1].

    Longest Turbulent Subarray (LeetCode 978)

    Run a modified Kadane's where "extend" only continues if the sign of the difference alternates. The "currentLength" replaces "currentSum", restarting when the alternation breaks.

    Combining Kadane's with Prefix Sums

    Recall from the overview that the maximum subarray sum can be expressed using prefix sums:

    $$\max_{0 \le i \le j < n}\bigl(P[j{+}1] - P[i]\bigr) = \max_j\bigl(P[j{+}1] - \min_{0 \le k \le j} P[k]\bigr)$$

    This perspective is powerful because it generalizes to constrained problems where pure Kadane's cannot be used directly.

    Maximum Subarray Sum with Length Constraint

    Problem: Find the maximum subarray sum where the subarray length is between $L$ and $R$ (inclusive).

    Standard Kadane's considers all ending positions but cannot enforce a length constraint. The prefix sum approach handles this naturally:

    int maxSumWithLengthConstraint(vector<int>& arr, int L, int R) {
    int n = arr.size();
    // Build prefix sums: P[0] = 0, P[i] = arr[0] + ... + arr[i-1]
    vector<int> P(n + 1, 0);
    for (int i = 0; i < n; i++) P[i + 1] = P[i] + arr[i];

    int maxSum = INT_MIN;
    // For each ending position j, find the best starting position
    // such that L <= j - i + 1 <= R, i.e. j - R + 1 <= i <= j - L + 1
    // Equivalently, P[j+1] - P[i] where j-R+1 <= i <= j-L+1
    // We need the minimum P[i] in the valid range

    // Use a sliding window minimum (deque) over prefix sums
    deque<int> dq; // stores indices into P
    for (int j = L - 1; j < n; j++) {
        // Add P[j - L + 1] to the deque (new valid start)
        int newIdx = j - L + 1;
        while (!dq.empty() && P[dq.back()] >= P[newIdx])
            dq.pop_back();
        dq.push_back(newIdx);

        // Remove indices that are too far left
        while (!dq.empty() && dq.front() < j - R + 1)
            dq.pop_front();

        if (!dq.empty())
            maxSum = max(maxSum, P[j + 1] - P[dq.front()]);
    }
    return maxSum;
}

    Complexity: $O(n)$ time. Each index enters and leaves the deque at most once.

    When to Prefer Prefix Sums Over Kadane's

    • Length constraints (subarray length between $L$ and $R$): prefix sums + sliding window minimum.
    • Multiple queries on different ranges: prefix sums precompute once, answer each query in $O(1)$.
    • 2D problems: prefix sums reduce a 2D problem to 1D Kadane's (see below).
    • Modular arithmetic: "maximum subarray sum mod $m$" needs prefix sums and a balanced BST for $O(n \log n)$.

    2D Kadane's: Maximum Rectangle Sum

    Problem: Given an $m \times n$ matrix of integers, find the rectangle (contiguous submatrix) with the largest sum.

    This is a classic extension of the maximum subarray problem from 1D to 2D. The brute force approach enumerates all $O(m^2 n^2)$ rectangles and sums each in $O(mn)$ time, giving $O(m^3 n^3)$. We can do much better.

    The Key Idea

    Reduce 2D to 1D

    Fix two row boundaries: top row $r_1$ and bottom row $r_2$. Collapse all columns between these rows into a 1D array where each entry is the column sum from row $r_1$ to row $r_2$. Then run 1D Kadane's on this collapsed array to find the best left and right column boundaries.

    By iterating over all $O(m^2)$ pairs of row boundaries, we cover every possible rectangle. For each pair, the collapsed array construction and Kadane's scan both take $O(n)$ time.

    Algorithm Steps

    Fix Top Row

    For each top row $r_1$ from 0 to $m{-}1$, initialize a temporary column-sum array temp[0..n-1] to all zeros.

    Expand Bottom Row

    For each bottom row $r_2$ from $r_1$ to $m{-}1$, add row $r_2$ to the temp array: temp[c] += matrix[r2][c] for each column $c$.

    Run 1D Kadane's

    Run Kadane's algorithm on temp[0..n-1]. If the result exceeds the global maximum, update the answer and record the boundaries.

    Complexity

    $O(m^2)$ row pairs, each taking $O(n)$ for Kadane's. Total: $O(m^2 \cdot n)$. If $m > n$, transpose the matrix so $m \le n$ and get $O(\min(m,n)^2 \cdot \max(m,n))$.

    C++ Implementation

    int maxRectangleSum(vector<vector<int>>& matrix) {
    int m = matrix.size(), n = matrix[0].size();
    int globalMax = INT_MIN;

    for (int r1 = 0; r1 < m; r1++) {
        vector<int> temp(n, 0);
        for (int r2 = r1; r2 < m; r2++) {
            // Add row r2 to the column sums
            for (int c = 0; c < n; c++)
                temp[c] += matrix[r2][c];

            // Run 1D Kadane's on temp[]
            int currentSum = temp[0], maxSum = temp[0];
            for (int c = 1; c < n; c++) {
                currentSum = max(temp[c], currentSum + temp[c]);
                maxSum = max(maxSum, currentSum);
            }
            globalMax = max(globalMax, maxSum);
        }
    }
    return globalMax;
}

    Returning the Rectangle Boundaries

    struct Rectangle {
    int r1, c1, r2, c2, sum;
};

Rectangle maxRectangleWithBounds(vector<vector<int>>& matrix) {
    int m = matrix.size(), n = matrix[0].size();
    Rectangle best = {0, 0, 0, 0, INT_MIN};

    for (int r1 = 0; r1 < m; r1++) {
        vector<int> temp(n, 0);
        for (int r2 = r1; r2 < m; r2++) {
            for (int c = 0; c < n; c++)
                temp[c] += matrix[r2][c];

            // Kadane's with index tracking
            int cs = temp[0], ms = temp[0];
            int tStart = 0, bestL = 0, bestR = 0;
            for (int c = 1; c < n; c++) {
                if (temp[c] > cs + temp[c]) {
                    cs = temp[c]; tStart = c;
                } else {
                    cs += temp[c];
                }
                if (cs > ms) {
                    ms = cs; bestL = tStart; bestR = c;
                }
            }
            if (ms > best.sum)
                best = {r1, bestL, r2, bestR, ms};
        }
    }
    return best;
}

    Complexity: $O(m^2 \cdot n)$ time, $O(n)$ space for the temp array.

    Advanced Competition Patterns

    Forward-Backward Kadane's

    Several problems require running Kadane's from both directions. Compute two arrays:

    • maxEndingAt[i]: maximum subarray sum ending at index $i$ (forward Kadane's).
    • maxStartingAt[i]: maximum subarray sum starting at index $i$ (backward Kadane's).

    These enable problems like:

    Maximum Sum with One Deletion

    For each index $i$ that we consider deleting, the best subarray that skips $i$ is maxEndingAt[i-1] + maxStartingAt[i+1]. Also compare against the standard Kadane's (no deletion). Total: $O(n)$.

    int maxSumWithOneDeletion(vector<int>& arr) {
    int n = arr.size();
    vector<int> maxEnd(n), maxStart(n);

    // Forward pass
    maxEnd[0] = arr[0];
    for (int i = 1; i < n; i++)
        maxEnd[i] = max(arr[i], maxEnd[i - 1] + arr[i]);

    // Backward pass
    maxStart[n - 1] = arr[n - 1];
    for (int i = n - 2; i >= 0; i--)
        maxStart[i] = max(arr[i], maxStart[i + 1] + arr[i]);

    // Best without deletion
    int best = *max_element(maxEnd.begin(), maxEnd.end());

    // Best with one deletion
    for (int i = 1; i < n - 1; i++)
        best = max(best, maxEnd[i - 1] + maxStart[i + 1]);

    return best;
}

    Kadane's on Transformed Arrays

    Many problems reduce to Kadane's after a clever transformation of the input:

    ProblemTransformationThen Apply
    Best stock trade Compute daily changes: diff[i] = price[i] - price[i-1] Kadane's on diff[]
    Maximum alternating sum Multiply odd-index elements by -1 Kadane's on transformed array
    Maximum subarray with equal 0s and 1s Replace 0 with -1 Longest subarray with sum 0 (prefix sum + hash map)
    Maximum weighted subarray Apply weights: w[i] * arr[i] Kadane's on weighted array

    Kadane's with DP Extensions

    When the "extend or restart" decision has more than two states, Kadane's generalizes into multi-dimensional DP.

    K-Concatenation Maximum Sum

    Given array arr repeated $k$ times, find maximum subarray sum. If $k = 1$, standard Kadane's. If $k = 2$, run Kadane's on the doubled array. If $k > 2$ and the total array sum is positive, the answer is Kadane's on the doubled array plus $(k{-}2) \times \text{totalSum}$.

    long long kConcatenationMaxSum(vector<int>& arr, int k) {
    long long n = arr.size();
    long long totalSum = 0;
    for (int x : arr) totalSum += x;

    // Kadane's on single array
    auto kadane = [](vector<int>& a) {
        long long cs = 0, ms = 0;
        for (int x : a) {
            cs = max((long long)x, cs + x);
            ms = max(ms, cs);
        }
        return ms;
    };

    if (k == 1) return kadane(arr);

    // Kadane's on doubled array
    vector<int> doubled(arr.begin(), arr.end());
    doubled.insert(doubled.end(), arr.begin(), arr.end());
    long long doubledMax = kadane(doubled);

    if (k == 2 || totalSum <= 0) return doubledMax;

    return doubledMax + (long long)(k - 2) * totalSum;
}

    Decision Tree: Which Technique to Use

    When you encounter a subarray problem, use this decision process:

    Is the subarray size fixed?

    Yes: use sliding window. No: continue.

    Does it ask for contiguous elements only?

    No: this is a subsequence problem, use DP. Yes: continue.

    Is the optimization a sum (or reducible to one)?

    Yes: Kadane's or a variant. No: consider specialized approaches (product variant, two pointers, etc.).

    Are there additional constraints (length, deletions, circular)?

    Length constraint: prefix sums + deque. One deletion: forward-backward Kadane's. Circular: max vs (total - min). No constraints: standard Kadane's.

    Summary

    • Kadane's applies whenever you need the optimal contiguous subarray for a sum-like aggregate. Learn to spot the "extend or restart" structure.
    • Stock trading, alternating sums, and weighted subarrays all reduce to Kadane's after transforming the input.
    • Prefix sums give an equivalent view that handles length constraints and modular arithmetic.
    • 2D Kadane's finds the maximum rectangle sum in $O(m^2 n)$ by fixing row pairs and running 1D Kadane's on column sums.
    • Forward-backward Kadane's handles deletion problems, and multi-copy extensions handle K-concatenation.
